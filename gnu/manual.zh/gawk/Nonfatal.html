5.10启用非致命输出

本节介绍了gawk特定的功能。

在标准awk ，输出为print要么printf到不存在的文件或其他一些I / O错误（例如，填满磁盘）是致命错误。 

$ gawk 'BEGIN { print "hi" > "/no/such/file" }'
error→ gawk: cmd. line:1: fatal: can't redirect to `/no/such/file' (No
error→ such file or directory)

gawk使您有可能检测到已发生错误，从而使您有可能从错误中恢复，或者至少在退出前打印您选择的错误消息。您可以通过以下两种方式之一执行此操作：

  • 对于所有输出文件，通过将任何值分配给PROCINFO["NONFATAL"] 。
  • 在每个文件的基础上，通过将任何值分配给PROCINFO[filename, "NONFATAL"] 。这里， filename是您希望输出不致命的文件的名称。

启用非致命输出后，必须检查ERRNO在每个相关之后print要么printf声明看是否出了问题。初始化也是个好主意ERRNO尝试输出前为零。例如：

$ gawk '
> BEGIN {
>     PROCINFO["NONFATAL"] = 1
>     ERRNO = 0
>     print "hi" > "/no/such/file"
>     if (ERRNO) {
>         print("Output failed:", ERRNO) > "/dev/stderr"
>         exit 1
>     }
> }'
error→ Output failed: No such file or directory

这里， gawk没有产生致命错误；相反，它让awk程序代码可以检测到问题并进行处理。

此机制也适用于标准输出和标准错误。对于标准输出，您可以使用PROCINFO["-", "NONFATAL"]要么PROCINFO["/dev/stdout", "NONFATAL"] 。对于标准错误，请使用PROCINFO["/dev/stderr", "NONFATAL"] 。

尝试打开TCP / IP套接字时（请参阅使用gawk用于网络编程 ）， gawk尝试多次。的GAWK_SOCK_RETRIES环境变量（请参阅其他环境变量部分）允许您覆盖gawk的内置默认尝试次数。但是，一旦为给定的套接字启用了非致命性的I / O， gawk仅重试一次，依靠awk级别的代码以注意出现问题。
