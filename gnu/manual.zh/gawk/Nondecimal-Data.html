12.1允许非十进制输入数据

如果你跑gawk与--non-decimal-data选项，您可以在输入数据中包含非十进制值：

$ echo 0123 123 0x123 |
> gawk --non-decimal-data '{ printf "%d, %d, %d\n", $1, $2, $3 }'
-| 83, 123, 291

要使用此功能，请编写程序，以便gawk将您的数据视为数字：

$ echo 0123 123 0x123 | gawk '{ print $1, $2, $3 }'
-| 0123 123 0x123

的print语句将其表达式视为字符串。尽管这些字段在必要时可以用作数字，但它们仍然是字符串，因此print不会尝试对其进行数字处理。您需要在字段中添加零以强制将其视为数字。例如：

$ echo 0123 123 0x123 | gawk --non-decimal-data '
> { print $1, $2, $3
>   print $1 + 0, $2 + 0, $3 + 0 }'
-| 0123 123 0x123
-| 83 123 291

因为十进制数据的前导零是很常见的，并且因为使用此功能可能导致令人惊讶的结果，所以默认设置为禁用它。如果需要，必须显式请求它。

注意： 不建议使用此选项。它会严重破坏旧程序。而是使用strtonum()函数来转换您的数据（请参阅“ 字符串操作函数”一节）。这使您的程序更易于编写和阅读，并导致令人惊讶的结果。

此选项可能在将来的版本中消失gawk 。
