17.4.13.1 API版本常量和变量

该API提供了“主要”和“次要”版本号。API版本可在编译时使用，因为C预处理程序定义了该版本以支持条件编译，还提供了枚举常量以方便调试：

API版本C预处理器定义枚举常数
重大的gawk_api_major_versionGAWK_API_MAJOR_VERSION
次要gawk_api_minor_versionGAWK_API_MINOR_VERSION

表17.3： gawk API版本常数

将新功能添加到API时，次要版本会增加。此类新功能总是添加到API的末尾struct 。

如果任何数据类型更改大小或成员顺序，或者任何现有功能更改签名，则主要版本增加（次要版本重置为零）。

可能会针对API的一个版本编译扩展，但由某个版本的API加载扩展gawk使用其他版本。因此，正在运行的主要和次要API版本gawk包含在API中struct作为只读常量整数：

api->major_version

运行的主要版本gawk 。

api->minor_version

次要版本的运行gawk 。

由扩展决定是否存在API不兼容。通常，这样的检查就足够了：

if (   api->major_version != GAWK_API_MAJOR_VERSION
    || api->minor_version < GAWK_API_MINOR_VERSION) {
        fprintf(stderr, "foo_extension: version mismatch with gawk!\n");
        fprintf(stderr, "\tmy version (%d, %d), gawk version (%d, %d)\n",
                GAWK_API_MAJOR_VERSION, GAWK_API_MINOR_VERSION,
                api->major_version, api->minor_version);
        exit(1);
}
