17.4.14样板代码

如前所述（请参阅“ 如何在更高层次上工作”一节），所提供的函数定义实际上是宏。要使用这些宏，扩展名必须使用此处所述的预定义名称，向源文件顶部提供少量样板代码（变量和函数）。所需的样板也在注释中提供。 gawkapi.h头文件：

/* Boilerplate code: */
int plugin_is_GPL_compatible;

static gawk_api_t *const api;

static awk_ext_id_t ext_id;
static const char *ext_version = NULL; /* or … = "some string" */

static awk_ext_func_t func_table[] = {
    { "name", do_name, 1, 0, awk_false, NULL },
    /* … */
};

/* EITHER: */

static awk_bool_t (*init_func)(void) = NULL;

/* OR: */

static awk_bool_t
init_my_extension(void)
{
    …
}

static awk_bool_t (*init_func)(void) = init_my_extension;

dl_load_func(func_table, some_name, "name_space_in_quotes")

这些变量和函数如下：

int plugin_is_GPL_compatible;

这断言该扩展与GNU GPL兼容（请参阅GNU通用公共许可证 ）。如果您的扩展程序没有此功能， gawk不会加载它（请参阅“ 扩展许可”部分）。

static gawk_api_t *const api;

这个全球static变量应设置为指向gawk_api_t指针gawk传递给您dl_load()功能。所有宏都使用此变量。

static awk_ext_id_t ext_id;

此全局静态变量应设置为awk_ext_id_t重视gawk传递给您dl_load()功能。所有宏都使用此变量。

static const char *ext_version = NULL; /* or … = "some string" */

这个全球static变量应设置为NULL ，或指向提供扩展名和版本的字符串。

static awk_ext_func_t func_table[] = { … };

这是一个或多个的数组awk_ext_func_t结构，如前所述（请参阅“ 注册扩展功能”一节）。然后可以将其循环以多次调用add_ext_func() 。

static awk_bool_t (*init_func)(void) = NULL;
                   OR
static awk_bool_t init_my_extension(void) { … }
static awk_bool_t (*init_func)(void) = init_my_extension;

如果需要进行一些初始化工作，则应定义一个执行该工作的函数（创建变量，打开文件等）。然后定义init_func指向您的函数的指针。该函数应返回awk_false失败时，或awk_true如果一切顺利。

如果不需要进行任何初始化，请定义指针并将其初始化为NULL 。

dl_load_func(func_table, some_name, "name_space_in_quotes")

此宏扩展为dl_load()执行所有必要初始化的函数。

所有变量和数组的要点是让dl_load()功能（来自dl_load_func()宏）执行所有标准工作。它执行以下操作：

  1. 检查API版本。如果扩展主版本不匹配gawk ，或者次要扩展版本大于gawk ，它会显示一条致命错误消息并退出。
  2. 检查MPFR和GMP版本。如果不匹配，它将显示致命错误消息并退出。
  3. 加载中定义的功能func_table 。如果其中任何一个加载失败，它会显示警告消息，然后继续。
  4. 如果init_func指针不是NULL ，调用它指向的函数。如果返回awk_false ，打印警告消息。
  5. 如果ext_version不是NULL ，向中注册版本字符串gawk 。
