7.4.10 exit声明

的exit陈述原因awk立即停止执行当前规则并停止处理输入；任何剩余的输入都将被忽略。的exit语句编写如下：

exit [return code]

当一个exit语句从一个执行BEGIN规则，程序立即停止处理所有内容。没有输入记录被读取。但是，如果END存在规则，作为执行exit声明， END规则执行（见的BEGIN和END特殊模式 ）。如果exit用于人体END规则，它将导致程序立即停止。

一个exit不属于BEGIN要么END规则停止为当前记录执行任何其他自动规则，跳过读取任何剩余的输入记录，并执行END规则是否有一个。 gawk也跳过任何ENDFILE规则他们不执行。

在这种情况下，如果您不想END规则以完成其工作，请在变量之前将变量设置为非零值exit语句并检查END规则。有关执行此操作的示例，请参见断言部分。

如果提供了一个参数exit ，其值用作退出状态代码awk处理。如果未提供任何参数， exit原因awk返回“成功”状态。在将参数提供给第一个参数的情况下exit陈述，然后exit被称为第二次END没有参数的规则awk使用先前提供的退出值。 （直流）见章节gawk的退出状态以获取更多信息。

例如，假设发生了难以处理或无法处理的错误情况。按照惯例，程序通过以非零状态退出来报告此情况。一个awk程序可以使用exit带有非零参数的语句，如以下示例所示：

BEGIN {
    if (("date" | getline date_now) <= 0) {
        print "Can't get system date" > "/dev/stderr"
        exit 1
    }

    print "current date is", date_now
    close("date")
}

注意：为了完全可移植，退出值应介于0和126之间（包括0和126）。负值以及127或更大的值可能无法在不同的操作系统上产生一致的结果。
