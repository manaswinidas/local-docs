17.4.15对API版本1的更改

当前的API与该API的版本1 不二进制兼容。您必须重新编译扩展程序才能将其与当前版本的gawk 。

幸运的是，该API仍保持源代码兼容性，并且与以前的API兼容，但可能会牺牲一些编译时警告。主要区别在于awk_ext_func_t结构，并在C实现函数中添加第三个参数（请参见注册扩展功能部分 ）。

以下是从API版本1更改为版本2的各个功能的列表：

  • 现在，数值可以具有MPFR / MPZ变体（请参见“ 通用数据类型”一节 ）。
  • 有新的字符串类型： AWK_REGEX和AWK_STRNUM （请参阅“ 通用数据类型”部分 ）。
  • 的ezalloc()宏是新的（请参阅“ 内存分配功能和便利宏”部分 ）。
  • 的awk_ext_func_t结构改变了。代替num_expected_args ，现在有max_expected和min_required （请参阅部分注册扩展功能 ）。
  • 对于get_record() ，输入解析器现在可以指定字段宽度（请参阅“ 自定义输入解析器”部分）。
  • 扩展现在可以产生非致命的错误消息（请参阅“ 打印消息”部分）。
  • 展平数组时，现在可以指定索引和值类型（请参见“ 数组函数”一节）。
  • 的get_file() API是新的（请参阅“ 访问和操纵重定向”部分 ）。
