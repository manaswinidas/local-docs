B.3.1.3使用gawk在PC操作系统上

本节中的信息适用于以下网络的MinGW和DJGPP端口gawk 。请参阅使用部分gawk在Cygwin环境中以获取有关Cygwin端口的信息。

在MS-Windows下，MinGW环境同时支持“ |&运营商和TCP / IP网络（请参阅使用gawk用于网络编程 ）。DJGPP环境不支持“ |& '。

MS-Windows版本gawk在描述程序文件搜索的AWKPATH环境变量 。但是，分号（而不是冒号）在AWKPATH变量。如果AWKPATH未设置或为空，则默认搜索路径为“ .;c:/lib/awk;c:/gnu/lib/awk '。

在MS-Windows下， gawk （以及许多其他文本程序）静默翻译行尾“ \r\n ' 至 ' \n '输入和' \n ' 至 ' \r\n '输出。一个特别的BINMODE变量（ce）允许控制这些翻译，其解释如下：

  • 如果BINMODE是"r"或其中之一，则在读取时设置二进制模式（即读取时不翻译）。
  • 如果BINMODE是"w"或两个，则在写入时设置二进制模式（即，写入时不转换）。
  • 如果BINMODE是"rw"要么"wr"或三种，同时为读写设置了二进制模式。
  • BINMODE=non-null-string是相同的 ' BINMODE=3 '（即，读或写均不翻译）。然而， gawk如果字符串不是以下之一，则发出警告消息"rw"要么"wr" 。

标准输入和标准输出的模式仅设置一次（在读取命令行之后，但在处理任何awk程序）。设置BINMODE标准输入或标准输出可通过使用适当的“ -v BINMODE=N命令行中的'选项。 BINMODE在打开文件或管道时设置，并且不能在中途更改。

在兼容POSIX的系统上，此变量的值无效。因此，如果您认为程序可以在多个不同的系统上运行，则可能需要使用BINMODE ，您只需简单地（在程序中或在命令行上）无条件设置它，而不必担心程序在其上运行的操作系统。

名字BINMODE选择匹配mawk （请参阅其他免费提供的部分awk实现 ）。 mawk和gawk处理BINMODE类似地然而， mawk添加一个“ -W BINMODE=N '选项和可以设置的环境变量BINMODE ， RS和ORS 。文件binmode[1-3].awk （下gnu/lib/awk在一些准备好的二进制分布中）已被选中以匹配mawk的-W BINMODE=N ' 选项。这些可以更改或丢弃；特别是RS给出最少的“惊喜”是有争议的。 mawk使用“ RS = "\r\n" '如果在读取时设置了二进制模式，则该模式适用于具有MS-DOS样式的行尾的文件。

为了说明，以下示例在写入时为标准输出和其他文件设置了二进制模式，并设置了ORS作为“常规” MS-DOS样式的行尾：

gawk -v BINMODE=2 -v ORS="\r\n" …

要么：

gawk -v BINMODE=w -f binmode2.awk …

这些结果与“ -W BINMODE=2 '选项mawk 。以下将记录分隔符更改为"\r\n"并在读取时设置二进制模式，但不影响标准输入的模式：

gawk -v RS="\r\n" -e "BEGIN { BINMODE = 1 }" …

要么：

gawk -f binmode1.awk …

在第一个示例中，使用正确的引号将RS可以移入BEGIN规则。
