17.4.10.2 Cookie的变量访问和更新

标量cookie是不透明的句柄，可用于访问全局变量或数组。这是一种优化措施，可避免在其中查找变量gawk每次需要访问时的的符号表。前面在“ 通用数据类型”中对此进行了讨论。

以下功能使您可以使用标量cookie：

awk_bool_t sym_lookup_scalar(awk_scalar_t cookie,
                             awk_valtype_t wanted,
                             awk_value_t *result);

检索标量cookie的当前值。使用以下方法获取标量Cookie后sym_lookup() ，您可以使用此功能更有效地获取其价值。如果无法检索该值，则返回false。

awk_bool_t sym_update_scalar(awk_scalar_t cookie, awk_value_t *value);

更新与标量cookie关联的值。如果新值不是类型，则返回false AWK_STRING ， AWK_STRNUM ， AWK_REGEX ， 要么AWK_NUMBER 。在此，预定义变量也可能不会更新。

这是不是在第一眼明显如何与标饼干或他们存在的理由真的是工作。从理论上讲sym_lookup()和sym_update()例程是您真正需要使用变量的全部。例如，您可能有代码来查找变量的值，评估条件，然后可能根据该评估的结果来更改变量的值，如下所示：

/*  do_magic --- do something really great */

static awk_value_t *
do_magic(int nargs, awk_value_t *result)
{
    awk_value_t value;

    if (   sym_lookup("MAGIC_VAR", AWK_NUMBER, & value)
        && some_condition(value.num_value)) {
            value.num_value += 42;
            sym_update("MAGIC_VAR", & value);
    }

    return make_number(0.0, result);
}

这段代码看起来（很简单）。所以有什么问题？

好吧，请考虑如果awk与您的扩展程序相关联的代码调用magic()功能（在C中由do_magic() ），每条记录一次，同时处理数十万或数百万条记录。的MAGIC_VAR每个函数调用在符号表中查找变量一次或两次！

查找符号表实际上只是开销。获取代表变量的cookie并使用它获取变量的值并根据需要更新它的效率要高得多。 107

因此，使用cookie的方式如下。首先，将扩展程序的变量安装在gawk的符号表使用sym_update() ， 照常。然后使用获取变量的标量cookie sym_lookup() ：

static awk_scalar_t magic_var_cookie;    /* cookie for MAGIC_VAR */

static void
my_extension_init()
{
    awk_value_t value;


    /* install initial value */
    sym_update("MAGIC_VAR", make_number(42.0, & value));

    /* get the cookie */
    sym_lookup("MAGIC_VAR", AWK_SCALAR, & value);

    /* save the cookie */
    magic_var_cookie = value.scalar_cookie;
    …
}

接下来，使用本节中的例程通过cookie检索和更新值。从而， do_magic()现在变成这样：

/*  do_magic --- do something really great */

static awk_value_t *
do_magic(int nargs, awk_value_t *result)
{
    awk_value_t value;

    if (   sym_lookup_scalar(magic_var_cookie, AWK_NUMBER, & value)
        && some_condition(value.num_value)) {
            value.num_value += 42;
            sym_update_scalar(magic_var_cookie, & value);
    }
    …

    return make_number(0.0, result);
}

注意：前面的代码为演示目的省略了错误检查。您的扩展代码应更健壮，并仔细检查API函数的返回值。
