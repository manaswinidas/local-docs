B.2.1编译gawk用于Unix之类的系统

正常的安装步骤应该在所有现代的，基于Unix的商业系统，GNU / Linux，基于BSD的系统以及适用于MS-Windows的Cygwin环境中都可以进行。

提取完后gawk分配， cd至gawk-5.0.0 。与大多数GNU软件一样，您可以配置gawk通过运行configure程序。该程序是使用GNU Autoconf自动生成的Bourne Shell脚本。（在Autoconf-生成自动配置脚本中对Autoconf软件进行了详细说明，可以在Free Software Foundation的网站上在线找到该软件 。）

配置gawk ，只需运行configure ：

sh ./configure

这产生了一个Makefile和config.h为您的系统量身定制。的config.h文件描述了有关系统的各种事实。您可能要编辑Makefile改变CFLAGS变量，用于控制传递给C编译器的命令行选项（例如优化级别或用于调试的编译）。

另外，您可以为大多数添加自己的值make命令行上的变量，例如CC和CFLAGS ，在跑步时configure ：

CC=cc CFLAGS=-g sh ./configure

查看档案INSTALL在里面gawk分发所有详细信息。

跑步后configure并可能编辑了Makefile ，输入：

make

此后不久，您应该具有的可执行版本gawk 。这里的所有都是它的！验证gawk运行正常，运行“ make check '。所有测试均应成功。如果这些步骤不起作用，或者任何测试失败，请检查README_d目录以查看是否发现了已知问题。如果此处未描述故障，请发送错误报告（请参阅报告问题和错误部分 ）。

当然，一旦建立gawk ，您可能希望安装它。为此，您需要运行命令“ make install '，作为具有适当权限的用户。具体操作方法因系统而异，但是在许多系统上，您可以使用sudo命令这样做。该命令将变为“ sudo make install '。可能会要求您输入密码，并且您之前必须已经被设置为允许运行密码的用户。 sudo命令。
