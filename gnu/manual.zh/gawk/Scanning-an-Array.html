8.1.5扫描阵列的所有元素

在使用数组的程序中，通常有必要使用对数组的每个元素执行一次的循环。在其他语言中，数组是连续的并且索引被限制为非负整数，这很容易：所有有效索引都可以通过从最低索引到最高索引进行计数来找到。这项技术无法完成awk ，因为任何数字或字符串都可以是数组索引。所以awk有一种特殊的for扫描数组的语句：

for (var in array)
    body

该循环执行body每个索引一次array该程序以前使用过的变量var设置为该索引。

以下程序使用这种形式的for声明。第一条规则通过将一个单词存储到数组中来扫描输入记录并记下哪些单词在输入中出现（至少一次） used以单词作为索引。第二条规则扫描used查找输入中出现的所有不同单词。它会打印每个长度超过10个字符的单词，并打印此类单词的数量。有关内置函数的更多信息，请参见字符串操作函数一节。 length() 。 

# Record a 1 for each word that is used at least once
{
    for (i = 1; i <= NF; i++)
        used[$i] = 1
}



# Find number of distinct words more than 10 characters long
END {
    for (x in used) {
        if (length(x) > 10) {
            ++num_long_words
            print x
        }
    }
    print num_long_words, "words longer than 10 characters"
}

有关此类型的更多示例，请参见生成字用法计数部分。

该语句访问数组元素的顺序由内部的数组元素的内部排列确定awk和标准awk无法控制或更改。如果将新元素添加到这会导致问题array通过循环体中的语句；无法预测for循环将到达他们。同样，改变var循环内可能会产生奇怪的结果。最好避免这种事情。

作为信息点， gawk设置在循环开始之前要迭代的元素列表，并且不对其进行更改。但不是所有的awk版本会这样做。考虑这个程序，命名为loopcheck.awk ：

BEGIN {
    a["here"] = "here"
    a["is"] = "is"
    a["a"] = "a"
    a["loop"] = "loop"
    for (i in a) {
        j++
        a[j] = j
        print i
    }
}

这是运行时发生的情况gawk （和mawk ）：

$ gawk -f loopcheck.awk
-| here
-| loop
-| a
-| is

与BWK对比awk ：

$ nawk -f loopcheck.awk
-| loop
-| here
-| is
-| a
-| 1
