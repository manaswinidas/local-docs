3.3.2关于间隔表达式的一些注意事项

间隔表达式传统上不可用awk 。它们被添加为POSIX标准的一部分，以使awk和egrep彼此一致。

最初，因为旧程序可能会使用“ { '和' }在正则表达式常量中， gawk与正则表达式中的间隔表达式不匹配。

但是，从4.0版开始， gawk默认情况下会匹配间隔表达式。这是因为与POSIX的兼容性对大多数人来说变得越来越重要gawk用户要比旧程序兼容。

对于使用“ { '和' } '在regexp常量中，最好始终使用反斜杠将其转义。然后，regexp常量有效，并且可以使用任何版本的regexp常量以它们希望的方式工作awk 。 16

最后，当“ { '和' } '以无法解释为间隔表达式的方式出现在regexp常量中（例如/q{a}/ ），那么他们就代表自己。

如前所述，区间表达式传统上无法在awk 。在2019年3月，Brian Kernighan的awk （最后）获得了它们。

但是，由于数十年来一直无法使用它们， gawk在兼容模式下，继续不提供它们（请参阅“ 命令行选项”一节）。

（16）

如果您使用带有正则表达式运算符或函数的字符串常量，请使用两个反斜杠。
