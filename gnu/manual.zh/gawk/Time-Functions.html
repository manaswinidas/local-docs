9.1.5时间函数

awk程序通常用于处理包含时间戳信息的日志文件，该日志文件指示何时写入特定日志记录。许多程序都以时间戳记返回的形式记录其时间戳。 time()系统调用，是自特定时期以来的秒数。在兼容POSIX的系统上，它是自1970-01-01 00:00:00 UTC以来的秒数，不计算leap秒。 52所有已知的符合POSIX标准的系统通过2 31支持时间戳从0 - 1，这是足够的，通过2038年1月19日3点14分07秒UTC代表次。许多系统支持更大范围的时间戳，包括表示时代之前的时间的负时间戳。

为了更轻松地处理此类日志文件并生成有用的报告， gawk提供以下用于处理时间戳的功能。他们是gawk扩展；在POSIX标准中未指定它们。 53但是，最新版本的mawk （请参阅其他免费提供的部分awk实现 ）也支持这些功能。可选参数放在方括号（[]）中：

mktime(datespec [ , utc-flag ])

转datespec转换为时间戳，格式与systime() 。它类似于ISO C中同名的功能。论据datespec ，是以下形式的字符串"YYYY MM DD HH MM SS [DST]" 。该字符串由六个或七个数字组成，分别代表包括世纪在内的全年，从1到12的月份，从1到31的月份，一天中从0到23的小时，从0到0的分钟。 59，第二个从0到60、54和可选的夏令时标志。

这些数字的值不必在指定的范围内；例如，小时-1表示午夜前1个小时。假设使用零起点公历，年份1之前的年份为0，年份0之前的年份为-1。如果utc-flag存在并且为非零或非空，该时间假定为UTC时区；否则，该时间假定为本地时区。如果DST夏令时标志为正，该时间假定为夏令时；如果为零，则假定该时间为标准时间；如果为负（默认值）， mktime()尝试确定夏令时在指定时间内是否有效。

如果datespec没有足够的元素，或者结果时间超出范围， mktime()返回-1。

strftime( [ format [ ,timestamp [ ,utc-flag ]]])

格式化指定的时间timestamp根据内容format字符串并返回结果。它类似于ISO C中具有相同名称的功能。 utc-flag存在并且为非零或非空，该值的格式为UTC（协调世界时，以前为GMT或格林威治标准时间）。否则，该值将格式化为本地时区。的timestamp格式与systime()功能。如果不timestamp提供了参数， gawk使用一天中的当前时间作为时间戳。没有format论据strftime()使用的值PROCINFO["strftime"]作为格式字符串（请参阅预定义变量部分）。默认字符串值为"%a %b %e %H:%M:%S %Z %Y" 。此格式字符串产生的输出与date效用。您可以将新值分配给PROCINFO["strftime"]更改默认格式；有关各种格式指令，请参见以下列表。

systime()

返回当前时间，以自系统纪元以来的秒数为单位。在POSIX系统上，这是自1970-01-01 00:00:00 UTC以来的秒数，不计算leap秒。在其他系统上，它可能是不同的数字。

的systime()函数可让您将日志文件中的时间戳与当前时间进行比较。特别是，很容易确定记录特定记录的时间。它还允许您使用“自纪元以来的秒数”格式生成日志记录。

的mktime()函数允许您将日期和时间的文本表示形式转换为时间戳。这使得比较日期和时间之前/之后变得容易，尤其是在处理来自外部源（例如日志文件）的日期和时间数据时。

的strftime()功能可让您轻松地将时间戳转换为人类可读的信息。它在本质上与sprintf()函数（请参阅“ 字符串操作函数”部分 ），因为它会将所有非格式规范字符逐字复制到返回的字符串中，同时用日期和时间值替换格式规范中的format串。

strftime() 1999 ISO C标准55保证支持以下日期格式规范：

%a

语言环境的缩写工作日名称。

%A

语言环境的完整工作日名称。

%b

语言环境的缩写月份名称。

%B

语言环境的完整月份名称。

%c

语言环境的“适当”日期和时间表示。（这是 ' %A %B %d %T %Y ' 在里面"C"语言环境。）

%C

本年的世纪部分。这是年份除以100并被截断为下一个较小的整数。

%d

以十进制数字表示的月份中的日期（01-31）。

%D

等同于指定“ %m/%d/%y '。

%e

每月的某天，如果只有一位数字，则用空格填充。

%F

等同于指定“ %Y-%m-%d '。这是ISO 8601日期格式。

%g

ISO 8601周编号的年份模100，十进制数字（00-99）。例如，2012年1月1日位于2011年的第53周。因此，即使其年份为2012，其ISO 8601周编号的年份也为2011年。同样，2012年12月31日位于2013年的第1周。因此，即使其年份为2012，其ISO周编号的年份为2013。

%G

ISO周编号的全年，以十进制数字表示。

%h

相当于 ' %b '。

%H

小时（24小时制），为十进制数字（00-23）。

%I

小时（12小时制）为十进制数字（01–12）。

%j

一年中的天，以十进制数字（001–366）为单位。

%m

以十进制数字表示的月份（01-12）。

%M

分钟，十进制数字（00–59）。

%n

换行符（ASCII LF）。

%p

与12小时制相关的AM / PM指定的语言环境。

%r

语言环境的12小时制时间。（这是 ' %I:%M:%S %p ' 在里面"C"语言环境。）

%R

等同于指定“ %H:%M '。

%S

第二个十进制数字（00-60）。

%t

TAB字符。

%T

等同于指定“ %H:%M:%S '。

%u

工作日为十进制数字（1–7）。星期一是第一天。

%U

一年中的星期数（第一个星期日为一周的第一天），以十进制数表示（00-53）。

%V

一年中的星期数（以第一个星期一作为一周的第一天），以十进制数字（01-53）表示。确定星期数的方法由ISO 8601规定。（例如：如果包含1月1日的一周在新年中有四天或更多天，则为第一周；否则为上一年的最后一周[52或53]，而下一周为第一周。）

%w

工作日为十进制数字（0–6）。星期日是零天。

%W

一年中的星期数（以第一个星期一作为一周的第一天），以十进制数表示（00-53）。

%x

语言环境的“适当”日期表示形式。（这是 ' %A %B %d %Y ' 在里面"C"语言环境。）

%X

语言环境的“适当”时间表示。（这是 ' %T ' 在里面"C"语言环境。）

%y

以100为模的年，以十进制数表示（00–99）。

%Y

整年作为十进制数字（例如2015）。

%z

'中的时区偏移+HHMM格式（例如，产生RFC 822 / RFC 1036日期标头所需的格式）。

%Z

时区名称或缩写；如果无法确定时区，则没有字符。

%OI %Om %OM %OS %Ou %OU %OV %Ow %OW %Oy

以下示例是awk POSIX的实施date效用。通常， date实用程序以众所周知的格式打印当前日期和时间。但是，如果您提供以“ + '， date将非格式说明符字符复制到标准输出，并根据字符串中的格式说明符解释当前时间。例如：

$ date '+Today is %A, %B %d, %Y.'
-| Today is Monday, September 22, 2014.

这里是gawk版本date效用。它有一个外壳“包装器”来处理-u选项，这要求date就像时区设置为UTC一样运行：

#! /bin/sh
#
# date --- approximate the POSIX 'date' command

case $1 in
-u)  TZ=UTC0     # use UTC
     export TZ
     shift ;;
esac

gawk 'BEGIN  {
    format = PROCINFO["strftime"]
    exitval = 0

    if (ARGC > 2)
        exitval = 1
    else if (ARGC == 2) {
        format = ARGV[1]
        if (format ~ /^\+/)
            format = substr(format, 2)   # remove leading +
    }
    print strftime(format)
    exit exitval
}' "$@"
