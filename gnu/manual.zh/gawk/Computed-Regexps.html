3.6使用动态正则表达式

'的右侧~ ' 要么 ' !~ '运算符不必是正则表达式常量（即，斜杠之间的字符串）。它可以是任何表达式。该表达式将被求值，并在必要时转换为字符串。然后将字符串的内容用作正则表达式。以这种方式计算的正则表达式称为动态正则表达式或计算的正则表达式 ：

BEGIN { digits_regexp = "[[:digit:]]+" }
$0 ~ digits_regexp    { print }

这套digits_regexp到描述一个或多个数字的正则表达式，并测试输入记录是否与此正则表达式匹配。

注意：使用“ ~ '和' !~ '运算符，请注意，用斜杠括起来的regexp常量与用双引号括起来的字符串常量之间是有区别的。如果要使用字符串常量，则必须了解该字符串本质上被扫描了两次 ：第一次awk读取您的程序，并第二次将其与操作符左侧的字符串与右侧的模式匹配。对于任何字符串值表达式（例如digits_regexp ，如上例所示），而不仅仅是字符串常量。

如果字符串被扫描两次，会有什么区别？答案与转义序列有关，尤其是与反斜杠有关。要将反斜杠插入字符串内的正则表达式中，必须键入两个反斜杠。

例如， /\*/是文字'的正则表达式常量* '。只需要一个反斜杠。要对字符串执行相同的操作，必须输入"\\*" 。第一个反斜杠转义第二个反斜杠，以便字符串实际上包含两个字符' \ '和' * '。

鉴于您可以同时使用regexp和字符串常量来描述正则表达式，应该使用哪个？答案是“ regexp常量”，原因如下：

  • 字符串常量编写起来更复杂，阅读起来也更困难。使用regexp常量可使您的程序不易出错。不了解两种常量之间的差异是导致错误的常见原因。
  • 使用正则表达式常量更有效。 awk可以注意到，您已经提供了一个正则表达式，并以一种使模式匹配更有效的形式将其存储在内部。使用字符串常量时， awk必须首先将字符串转换为这种内部形式，然后执行模式匹配。
  • 使用regexp常量是更好的形式。它清楚地表明您打算进行正则表达式匹配。
使用\n在正则表达式的括号表达式中

一些较旧的版本awk不允许在动态正则表达式的方括号表达式内使用换行符：

$ awk '$0 ~ "[ \t\n]"'
error→ awk: newline in character class [
error→ ]...
error→  source line number 1
error→  context is
error→        $0 ~ "[ >>>  \t\n]" <<<

但是正则表达式常量中的换行符可以正常工作：

$ awk '$0 ~ /[ \t\n]/'
here is a sample line
-| here is a sample line
Ctrl-d

gawk没有这个问题，并且在实践中不太可能经常发生，但是值得一提，以备将来参考。
