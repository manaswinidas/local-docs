6.3.3布尔表达式

布尔表达式是比较表达式或匹配表达式的组合，使用布尔运算符“或”（' || '），“和”（' && '）， 并不是” （' ! '），以及括号以控制嵌套。通过组合组件表达式的真值来计算布尔表达式的真值。布尔表达式也称为逻辑表达式 。这些术语是等效的。

可以在可以使用比较和匹配表达式的任何地方使用布尔表达式。它们可以用于if ， while ， do和for语句（请参阅“操作中的控制语句”部分）。它们具有数值（如果为true，则为1，如果为false，则为零），如果布尔表达式的结果存储在变量中或用于算术运算，则它们会起作用。

另外，每个布尔表达式也是一个有效的模式，因此可以将其中一个用作模式来控制规则的执行。布尔运算符为：

boolean1 && boolean2

如果两者都为真boolean1和boolean2是真的。例如，以下语句如果同时包含两个“ edu '和' li '：

if ($0 ~ /edu/ && $0 ~ /li/) print

子表达式boolean2仅在以下情况下评估boolean1是真的。这可能会有所作为boolean2包含具有副作用的表达式。如果是 ' $0 ~ /foo/ && ($2 == bar++) '，变量bar如果没有子串'，则不会增加foo在记录中。

boolean1 || boolean2

如果至少以下一项为真boolean1要么boolean2是真的。例如，下面的语句打印输入的是包含所有记录“ edu ' 要么 ' li '：

if ($0 ~ /edu/ || $0 ~ /li/) print

子表达式boolean2仅在以下情况下评估boolean1是错误的。这可能会有所作为boolean2包含具有副作用的表达式。（因此，该测试从不真正区分包含“ edu '和' li '-立刻 ' edu匹配，则完整测试成功。）

! boolean

如果为真boolean是错误的。例如，以下程序会打印“ no home! '在不寻常的情况下HOME未定义环境变量：

BEGIN { if (! ("HOME" in ENVIRON))
            print "no home!" }

（ in运算符在参考数组元素中进行了介绍 。）

' && '和' ||操作员由于其工作方式而称为短路操作员。如果可以在评估过程的中途确定结果，则对完整表达式的评估是“短路的”。

以'结尾的陈述&& ' 要么 ' ||只需在其后加上换行符即可继续。但是，如果不使用反斜杠继续，就不能在这两个运算符的前面放置换行符（请参见awk语句与行 ）。

使用'的表达式的实际值! '运算符可以是一或为零，这取决于它所应用于的表达式的真值。' ! '运算符通常可用于将标志变量的含义从false更改为true，然后再次更改。例如，下面的程序是在特殊的包围线之间打印线的一种方法：

$1 == "START"   { interested = ! interested; next }
interested      { print }
$1 == "END"     { interested = ! interested; next }

变量interested ，与所有awk变量，开始时初始化为零，这也是错误的。当看到一行的第一个字段为“ START '， 的价值interested使用'切换为true ! '。下一条规则将打印行interested是真的。当看到一行的第一个字段为“ END '， interested被切换回false。 36

最常见的是， !在以下情况下使用'运算符if和while语句，通常在否定句中用逻辑来表达：

if (! some condition || some other condition) {
    … do whatever processing …
}

注意： next声明中所讨论的next声明 。 next告诉awk跳过其余规则，获取下一条记录，并在顶部重新开始处理规则。这样做的原因是避免打印括号“ START '和' END '行。
