13.5一个简单的国际化示例

现在，让我们看一下如何将简单的国际化和本地化的分步示例awk程序，使用guide.awk作为我们的原始资料：

BEGIN {
    TEXTDOMAIN = "guide"
    bindtextdomain(".")  # for testing
    print _"Don't Panic"
    print _"The Answer Is", 42
    print "Pardon me, Zaphod who?"
}

跑 ' gawk --gen-pot '来创建.pot文件：

$ gawk --gen-pot -f guide.awk > guide.pot

这将产生：

#: guide.awk:4
msgid "Don't Panic"
msgstr ""

#: guide.awk:5
msgid "The Answer Is"
msgstr ""

将保存此原始便携式对象模板文件，并将其重新用于应用程序翻译成的每种语言。的msgid是原始字符串， msgstr是翻译。

注意：未标记前导下划线的字符串不会出现在guide.pot文件。

接下来，必须翻译消息。这是假设的英语方言的翻译，被称为“ Mellow”： 94 

$ cp guide.pot guide-mellow.po
Add translations to guide-mellow.po …

以下是翻译：

#: guide.awk:4
msgid "Don't Panic"
msgstr "Hey man, relax!"

#: guide.awk:5
msgid "The Answer Is"
msgstr "Like, the scoop is"

注意：以下说明适用于带有GNU C库的GNU / Linux。请注意，实际步骤可能会随时间变化，以下描述可能并不适用于所有GNU / Linux发行版，并且在其他操作系统上的工作方式可能完全不同。

下一步是使目录包含二进制消息对象文件，然后创建guide.mo文件。该目录具有以下形式locale/LC_MESSAGES ，在哪里locale是C已知的语言环境名称gettext例行程序。

我们如何知道要使用哪个语言环境？事实证明，C使用了四种不同的环境变量gettext例行程序。按顺序，它们是$LANGUAGE ， $LC_ALL ， $LANG和$LC_MESSAGES 。 95因此，我们检查$LANGUAGE ：

$ echo $LANGUAGE
-| en_US.UTF-8

接下来，我们创建目录：

$ mkdir en_US.UTF-8 en_US.UTF-8/LC_MESSAGES

的msgfmt实用程序转换人类可读.po文件放入机器可读.mo文件。默认， msgfmt创建一个名为messages 。此文件必须重命名并放置在正确的目录中（使用-o选项），以便gawk可以找到它：

$ msgfmt guide-mellow.po -o en_US.UTF-8/LC_MESSAGES/guide.mo

最后，我们运行程序进行测试：

$ gawk -f guide.awk
-| Hey man, relax!
-| Like, the scoop is 42
-| Pardon me, Zaphod who?

如果三个替换功能为dcgettext() ， dcngettext()和bindtextdomain() （请参见awk可移植性问题 ）位于名为的文件中libintl.awk ，那么我们可以运行guide.awk保持不变，如下所示：

$ gawk --posix -f guide.awk -f libintl.awk
-| Don't Panic
-| The Answer Is 42
-| Pardon me, Zaphod who?
