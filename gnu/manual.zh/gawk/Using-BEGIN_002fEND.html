7.1.4.1启动和清理操作

一种BEGIN在读取第一个输入记录之前，规则仅执行一次。同样， END读取所有输入后，规则仅执行一次。例如：

$ awk '
> BEGIN { print "Analysis of \"li\"" }
> /li/  { ++n }
> END   { print "\"li\" appears in", n, "records." }' mail-list
-| Analysis of "li"
-| "li" appears in 4 records.

该程序在输入文件中查找记录数mail-list包含字符串“ li '。的BEGIN规则为报告打印标题。无需使用BEGIN初始化计数器的规则n为零awk自动执行此操作（请参见变量部分）。第二条规则增加变量n每次包含模式的记录li '被读取。的END规则输出的值n在运行结束时。

特殊图案BEGIN和END不能在范围内或布尔运算符中使用（实际上，它们不能与任何运算符一起使用）。一个awk程序可能有多个BEGIN和/或END规则。它们按照它们出现的顺序执行：所有BEGIN启动时的规则以及所有END终止规则。 BEGIN和END规则可以与其他规则混在一起。1987年的版本中添加了此功能awk并包含在POSIX标准中。的原始（1978）版本awk需要BEGIN放在程序开始处的规则END规则应放置在末尾，并且每个只能允许一个。不再需要这样做，但是从程序组织和可读性方面考虑，遵循此模板是一个好主意。

多BEGIN和END规则对于编写库函数很有用，因为每个库文件可以有自己的文件BEGIN和/或END规则进行自己的初始化和/或清除。在命令行上命名库函数的顺序控制着它们的顺序。 BEGIN和END规则被执行。因此，在库文件中编写此类规则时必须小心，以免执行顺序无关紧要。有关使用库函数的更多信息，请参见命令行选项部分。见库awk函数 ，提供了许多有用的库函数。

如果awk程序只有BEGIN规则和其他规则，则程序会在BEGIN规则运行。 37但是，如果END规则存在，即使程序中没有其他规则，也会读取输入。这在万一END规则检查FNR和NR变量。
