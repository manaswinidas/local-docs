2.7将其他文件包含到程序中

本节介绍了特定于gawk 。

的@include关键字可用于读取外部awk源文件。这使您能够拆分大块awk源文件分成更小，更易于管理的部分，还使您可以重复使用常见的文件awk来自各种代码awk脚本。换句话说，您可以组合在一起awk用于将特定任务执行到外部文件中的功能。这些文件可以像函数库一样使用，使用@include关键字与AWKPATH环境变量。请注意，使用-i选项。

让我们来看一个例子。我们将从两个（平凡的）开始awk脚本，即test1和test2 。这里是test1脚本：

BEGIN {
    print "This is script test1."
}

这是test2 ：

@include "test1"
BEGIN {
    print "This is script test2."
}

跑步gawk与test2产生以下结果：

$ gawk -f test2
-| This is script test1.
-| This is script test2.

gawk运行test2脚本，其中包括test1使用@include关键词。因此，包括外部awk源文件，您只需使用@include后跟要包含的文件名，并用双引号引起来。

注意：请记住，这是一种语言构造，文件名不能是字符串变量，而只能是双引号中的文字字符串常量。

要包含的文件可以嵌套；例如，给定第三个脚本，即test3 ：

@include "test2"
BEGIN {
    print "This is script test3."
}

跑步gawk与test3脚本产生以下结果：

$ gawk -f test3
-| This is script test1.
-| This is script test2.
-| This is script test3.

该文件名当然可以是路径名。例如：

@include "../io_funcs"

和：

@include "/usr/awklib/network"

都有效。的AWKPATH使用时环境变量可能具有很大的价值@include 。使用相同的规则AWKPATH在命令行文件搜索变量（见节的AWKPATH环境变量 ）适用于@include也。

这对构建非常有帮助gawk功能库。如果您的大型脚本具有有用的通用目的awk函数，您可以将其分解为库文件，然后将这些文件放在一个特殊目录中。然后，您可以使用文件的完整路径名或通过设置这些“库”来包含这些“库”。 AWKPATH相应地使用环境变量，然后使用@include仅包含完整路径名的文件部分。当然，您可以将库文件保存在多个目录中。工作环境越复杂，组织要包含的文件所需的目录就越多。

具备指定多个-f选项， @include机制并非严格必要。然而@include关键字可以帮助您构建独立的gawk程序，从而减少了编写复杂繁琐的命令行的需要。特别是， @include对于编写要从网页运行的CGI脚本非常有用。

查找中描述的源文件中的规则的AWKPATH环境变量也适用于加载了@include 。

最后，随附的文件@include被当作他们有“ @namespace "awk"在他们的开始。有关更多信息，请参见“ 更改命名空间”一节。
