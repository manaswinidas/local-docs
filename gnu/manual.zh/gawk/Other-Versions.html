B.5其他免费提供awk实作

在awk代码中添加如下注释是很有趣的：
      // Do C++ comments work? answer: yes! of course

— 迈克尔·布伦南 （ Michael Brennan）

还有许多其他免费提供的awk实现。本节简要介绍了在何处获取它们：

Unix系统awk

Brian Kernighan，Unix的原始设计师之一awk ，已实现awk免费提供。您可以从GitHub检索它：

git clone git://github.com/onetrueawk/awk bwkawk

此命令在名为的目录中创建Git存储库的副本bwkawk 。如果您省略了最后一个参数git命令行，在名为的目录中创建存储库副本awk 。

这个版本需要ISO C（1990标准）编译器； GCC的C编译器（GNU编译器集合）工作得很好。

要构建它，请查看makefile ，然后运行make 。请注意，编译结果命名为a.out ;您将不得不将其重命名为合理的名称。

请参阅“ 公共扩展摘要 ”部分以获取此扩展的列表。 awk不在POSIX中awk 。

顺带一提，Dan Bornstein创建了一个Git存储库，用于跟踪BWK的所有版本awk他可以找到的。可在git：//github.com/danfuzz/one-true-awk上获得 。

mawk

迈克尔·布伦南（Michael Brennan）编写了一个独立的实现awk ，称为mawk 。可以在GPL下使用（请参见GNU通用公共许可证 ），就像gawk是。

的原始发行网站mawk源代码不再拥有它。可从http://www.skeeve.com/gawk/mawk1.3.3.tar.gz获得副本。

2009年，托马斯·迪基（Thomas Dickey） mawk保养。基本信息可在项目的网页上找到 。下载URL是http://invisible-island.net/datafiles/release/mawk.tar.gz 。

一旦有了它， gunzip可用于解压缩该文件。安装类似于gawk的（请参阅“ 编译和安装”部分gawk在Unix-like Systems上 ）。

有关通用扩展的列表，请参见“ 通用扩展摘要 ”部分。 mawk不在POSIX中awk 。

mawk 2.0

2016年，迈克尔·布伦南（Michael Brennan）恢复mawk发展。他的开发快照可通过Git从项目的GitHub页面获得 。

awka

由安德鲁·萨姆纳（Andrew Sumner）撰写， awka翻译awk程序编译为C，对其进行编译，并将其与提供核心功能的函数库链接awk功能。它还具有许多扩展。

的awk翻译程序在GPL下发布，而库在LGPL下。

要得到awka ，请转到https://sourceforge.net/projects/awka 。

该项目似乎被冻结；自2001年左右以来，没有进行任何新的代码更改。

pawk

犹他大学的Nelson HF Beebe修改了BWK awk提供时序和配置信息。有别于gawk与--profile选项（请参阅剖析您的awk程序 ），因为它使用基于CPU的分析，而不是行数分析。您可以在ftp://ftp.math.utah.edu/pub/pawk/pawk-20030606.tar.gz或http://www.math.utah.edu/pub/pawk/pawk-20030606中找到它。 tar.gz。

忙箱awk

BusyBox是GPL许可的程序，在单个可执行文件中提供许多应用程序的小版本。它针对嵌入式系统。它包括POSIX的完整实现awk 。制作时，请注意不要make install '，因为它会覆盖您的其他应用程序的副本/usr/local/bin 。有关更多信息，请参见项目的主页 。

OpenSolaris POSIXawk

的版本awk在/usr/xpg4/bin和/usr/xpg6/bin在Solaris上或多或少与POSIX兼容。它们基于awk来自Mortice Kern Systems的PC产品。我们能够将这些代码编译并在GNU / Linux下工作1-2个小时。使它更一般地可移植（使用GNU Autoconf和/或Automake）将花费更多的工作，而至少就我们所知，这还没有完成。

源代码曾经可以从OpenSolaris网站获得。但是，该项目结束了，该网站也关闭了。幸运的是， Illumos项目使此实现可用。您可以从https://github.com/joyent/illumos-joyent/blob/master/usr/src/cmd/awk_xpg4一次查看一个文件。

goawk

这是一awk用Go编程语言编写的解释器 。它实现POSIX awk ，并进行了一些小扩展。源代码可从https://github.com/benhoyt/goawk获得 。作者写了一篇不错的文章来描述实现。

jawk

这是一个口译员awk用Java编写。它声称是完整的解释器，尽管因为它使用Java工具进行I / O和正则表达式匹配，所以它支持的语言不同于POSIX。 awk 。有关更多信息，请参见项目的主页 。

利伯莫克

这是一个可嵌入的awk解释器源自mawk 。有关更多信息，请参见http://repo.hu/projects/libmawk/ 。

Mircea Neacsu的可嵌入awk

Mircea Neacsu创建了一个可嵌入的awk解释器，基于BWK awk。可从https://github.com/neacsum/awk获得 。

pawk

这是一个声称带来的Python模块awk类似的功能。有关更多信息，请参见https://github.com/alecthomas/pawk 。（这与Nelson Beebe的BWK的修改版无关awk ，如前所述。）

QSE awk

这是一个可嵌入的awk口译员。有关更多信息，请参见https://code.google.com/p/qse/ 。

QTawk

这是一个独立的实现awk根据GPL分发。它在标准方面有大量扩展awk并且可能与它在语法上不是100％兼容的。有关更多信息，包括手册，请参见http://www.quiktrim.org/QTawk.html 。那里的下载链接已过期；有关最新的下载链接，请参见http://www.quiktrim.org/#AdditionalResources 。

该项目也可能被冻结；自2014年左右以来，未进行任何新的代码更改。

其他版本

另请参阅Wikipedia文章的“版本和实现”部分， 网址为awk有关其他版本的信息。
