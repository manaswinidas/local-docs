8.4的delete声明

要删除数组的单个元素，请使用delete声明：

delete array[index-expression]

一旦删除数组元素，该元素曾经拥有的任何值将不再可用。好像该元素从未被引用过或没有被赋予值。以下是删除数组中元素的示例：

for (i in frequencies)
    delete frequencies[i]

本示例从数组中删除所有元素frequencies 。元素删除后， for扫描数组的语句不报告该元素，并且使用in用于检查该元素是否存在的运算符将返回零（即false）：

delete foo[4]
if (4 in foo)
    print "This will never be printed"

需要注意的是删除一个元素是不一样为它指定一个空值（空字符串是很重要的， "" ）。例如：

foo[4] = ""
if (4 in foo)
  print "This is printed, even though foo[4] is empty"

删除不存在的元素不是错误。但是，如果--lint在命令行上提供（请参阅“ 命令行选项”一节）， gawk删除数组中不存在的元素时，发出警告消息。

数组的所有元素都可以通过删除语句中的下标来用一条语句删除。 delete声明如下：

delete array

使用此版本的delete与每次删除一个元素的等效循环相比，该语句的效率大约高三倍。

这种形式的delete BWK也支持该语句awk和mawk ，以及许多其他实现。

注意：多年来，使用delete没有下标是常见的扩展。在2012年9月，它被接受并包含在POSIX标准中。请参阅Austin Group网站 。

以下语句提供了一种可移植但不明显的方法来清除数组： 42 

split("", array)

的split()函数（请参阅“ 字符串操作函数”一节）首先清除目标数组。该调用要求它拆分空字符串。由于没有数据可拆分，因此该函数仅清除数组然后返回。

注意：从数组中删除所有元素不会更改其类型；您不能清除数组，然后使用该数组的名称作为标量（即常规变量）。例如对于以下不起作用：

a[1] = 3
delete a
a = 3
