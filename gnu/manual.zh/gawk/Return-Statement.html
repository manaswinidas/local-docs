9.2.4 return声明

如前面几个示例所示，用户定义函数的主体可以包含一个return声明。该语句将控制权返回给awk程序。它还可以用于返回一个值，以供其余awk程序。看起来像这样：

return [expression]

的expression部分是可选的。由于最有可能受到疏忽，如果您省略POSIX，则POSIX不会定义返回值是什么expression 。从技术上讲，这使返回值不确定，因此不可预测。但实际上，所有版本的awk只需返回空字符串，如果在数字上下文中使用该字符串，其作用类似于零。

一种return声明不带expression假定在每个函数定义的末尾。因此，如果控制到达函数体的末尾，那么从技术上讲，该函数将返回不可预测的值。实际上，它返回空字符串。awk如果您使用此类函数的返回值，则不会发出警告。

有时，您想针对其功能而不是针对其返回值编写函数。这样的功能对应于void使用C，C ++或Java或procedure在阿达因此，不返回任何值可能是适当的；只需记住，您不应该使用此类函数的返回值。

以下是一个用户定义函数的示例，该函数返回数组元素中最大数量的值：

function maxelt(vec,   i, ret)
{
     for (i in vec) {
          if (ret == "" || vec[i] > ret)
               ret = vec[i]
     }
     return ret
}

你打电话maxelt()带有一个参数，即数组名称。局部变量i和ret不打算成为争论；没有什么可以阻止您传递不止一个论点maxelt()但是结果会很奇怪。之前的多余空间i在功能参数列表中指示i和ret是局部变量。定义函数时，应遵循此约定。

以下程序使用maxelt()功能。它加载一个数组，调用maxelt() ，然后报告该数组中的最大数目：

function maxelt(vec,   i, ret)
{
     for (i in vec) {
          if (ret == "" || vec[i] > ret)
               ret = vec[i]
     }
     return ret
}


# Load all fields of each record into nums.
{
     for(i = 1; i <= NF; i++)
          nums[NR, i] = $i
}


END {
     print maxelt(nums)
}

给出以下输入：

 1 5 23 8 16
44 3 5 2 8 26
256 291 1396 2962 100
-6 467 998 1101
99385 11 0 225

该程序报告（可预测）阵列中的最大值为99,385。
