10.3.1注意数据文件边界

的BEGIN和END规则在您的开头和结尾分别执行一次awk程序，分别为（见节的BEGIN和END特殊模式 ）。我们（ gawk作者）曾经有一个用户错误地认为BEGIN规则在每个数据文件的开头执行，并且END规则在每个数据文件的末尾执行。

当得知情况并非如此时，用户要求我们向其中添加新的特殊模式gawk ，命名BEGIN_FILE和END_FILE ，这将具有所需的行为。他甚至向我们提供了这样做的代码。

将这些特殊模式添加到gawk没必要这项工作可以干净地完成awk本身，如以下库程序所示。它安排调用两个用户提供的函数， beginfile()和endfile() ，位于每个数据文件的开头和结尾。除了仅用九个即可解决问题（！）代码行，它可移植 ；这适用于任何实现awk ：

# transfile.awk
#
# Give the user a hook for filename transitions
#
# The user must supply functions beginfile() and endfile()
# that each take the name of the file being started or
# finished, respectively.

FILENAME != _oldfilename {
    if (_oldfilename != "")
        endfile(_oldfilename)
    _oldfilename = FILENAME
    beginfile(FILENAME)
}

END { endfile(FILENAME) }

必须在用户的“主”程序之前加载此文件，以便首先执行其提供的规则。

该规则依赖awk的FILENAME变量，它会针对每个新数据文件自动更改。当前文件名保存在私有变量中， _oldfilename 。如果FILENAME不等于_oldfilename ，则正在处理一个新的数据文件，有必要调用endfile()用于旧文件。因为endfile()仅当文件已被处理时才调用，程序首先检查以确保_oldfilename不是空字符串。然后，程序将当前文件名分配给_oldfilename和电话beginfile()用于文件。因为像所有awk变量_oldfilename初始化为空字符串，即使对于第一个数据文件，此规则也可以正确执行。

该程序还提供了END规则对最后一个文件进行最终处理。因为这END规则先于END “主”程序中提供的规则， endfile()首先被称为。再次，价值倍数BEGIN和END规则应该清楚。

如果同一数据文件在命令行上连续出现两次，则endfile()和beginfile()在第一遍的结尾和第二遍的开头不执行。以下版本解决了该问题：

# ftrans.awk --- handle datafile transitions
#
# user supplies beginfile() and endfile() functions

FNR == 1 {
    if (_filename_ != "")
        endfile(_filename_)
    _filename_ = FILENAME
    beginfile(FILENAME)
}

END { endfile(_filename_) }

事物计数显示了如何使用此库函数，以及如何简化编写主程序的过程。

那为什么gawk有BEGINFILE和ENDFILE ？

您可能想知道，如果beginfile()和endfile()功能可以胜任，为什么gawk有BEGINFILE和ENDFILE模式？

好问题。通常，如果awk无法打开文件，这将立即导致致命错误。在这种情况下，用户定义的函数无法处理此问题，因为调用它的机制依赖于打开的文件和第一条记录。因此，主要原因BEGINFILE是给您一个“钩子”来捕获无法处理的文件。 ENDFILE之所以存在对称性，是因为它提供了一种执行每个文件清理处理的简便方法。欲了解更多信息，请参阅该BEGINFILE和ENDFILE特殊图案 。
