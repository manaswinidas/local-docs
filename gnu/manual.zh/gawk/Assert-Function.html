10.2.2断言

在编写大型程序时，了解一个条件或一组条件通常是有用的。在进行特定计算之前，请先声明一下您认为的情况。这样的声明称为断言 。C语言提供了头文件和对应的assert()程序员可以用来声明的宏。如果断言失败，则assert()宏安排打印一条诊断消息，描述应该为真但不是的情况，然后杀死程序。在C中，使用assert()看起来像这样：

#include <assert.h>

int myfunc(int a, double b)
{
     assert(a <= 5 && b >= 17.1);
     …
}

如果断言失败，程序将输出类似于以下内容的消息：

prog.c:5: assertion failed: a <= 5 && b >= 17.1

使用C语言，可以将条件转换为字符串以用于打印诊断消息。这是不可能的awk ， 所以这assert()函数还需要所测试条件的字符串版本。以下是功能：

# assert --- assert that a condition is true. Otherwise, exit.

function assert(condition, string)
{
    if (! condition) {
        printf("%s:%d: assertion failed: %s\n",
            FILENAME, FNR, string) > "/dev/stderr"
        _assert_exit = 1
        exit 1
    }
}


END {
    if (_assert_exit)
        exit 1
}

的assert()功能测试condition参数。如果为假，则使用string描述失败条件的参数。然后设置变量_assert_exit到一个并执行exit声明。的exit陈述跳到END规则。如果END规则发现_assert_exit没错，它会立即退出。

测试目的END规则是保留其他任何东西END运行规则。断言失败时，程序应立即退出。如果没有断言失败，则_assert_exit仍然是假的END规则正常运行，而程序的其余部分END规则执行。为了使所有这些正常工作， assert.awk必须是读取的第一个源文件awk 。可以通过以下方式在程序中使用该功能：

function myfunc(a, b)
{
     assert(a <= 5 && b >= 17.1, "a <= 5 && b >= 17.1")
     …
}

如果断言失败，您会看到类似以下内容的消息：

mydata:1357: assertion failed: a <= 5 && b >= 17.1

这个版本的assert() 。一个END规则自动添加到程序调用assert() 。通常，如果程序仅包含一个BEGIN规则，不读取输入文件和/或标准输入。但是，由于该程序具有END规则， awk尝试读取输入数据文件或标准输入（请参阅“ 启动和清除操作”部分 ），很可能导致程序在等待输入时挂起。

有一个简单的解决方法：请确保BEGIN规则总是以exit声明。
