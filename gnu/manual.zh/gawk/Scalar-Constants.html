6.1.1.1数字和字符串常量

数字常数代表数字。该数字可以是整数，小数部分或科学（指数）表示形式的数字。 29这是一些数值常量都具有相同值的示例：

105
1.05e+2
1050e-1

字符串常量由用双引号引起来的一系列字符组成。例如：

"parrot"

表示内容为“ parrot '。弦乐gawk可以是任何长度，并且可以包含任何可能的八位ASCII字符，包括ASCII NUL （零字符代码）。其他awk某些字符代码可能难以实现。

某些语言允许您在行的末尾加反斜杠来继续长字符串。例如在C中：

#include <stdio.h>

int main()
{
    printf "hello, \
world\n");
    return 0;
}

在这种情况下，C编译器会同时删除反斜杠和换行符，从而产生一个字符串，就好像键入了' "hello, world\n" '。当单个字符串需要包含大量文本时，这很有用。

POSIX标准明确指出，字符串常量中不允许换行。实际上，所有awk如果尝试这样做，实现会报告错误。例如：

$ gawk 'BEGIN { print "hello, 
> world" }'
-| gawk: cmd. line:1: BEGIN { print "hello,
-| gawk: cmd. line:1:               ^ unterminated string
-| gawk: cmd. line:1: BEGIN { print "hello,
-| gawk: cmd. line:1:               ^ syntax error

尽管POSIX并未定义如果使用转义的换行符会发生什么，如前面的C示例中所示，所有已知版本的awk允许你这样做。不幸的是，每个人用这样的弦子做什么。 （直流） gawk ， mawk和OpenSolaris POSIX awk （请参阅其他免费提供的部分awk实现 ）消除反斜杠和换行符，如C中所示：

$ gawk 'BEGIN { print "hello, \
> world" }'
-| hello, world

在POSIX模式下（请参阅“ 命令行选项”部分 ）， gawk不允许转义的换行符。否则，它的行为就像刚刚描述的那样。

布莱恩·克尼根（Brian Kernighan） awk和BusyBox awk除去反斜杠，但保留完整的换行符，作为字符串的一部分：

$ nawk 'BEGIN { print "hello, \
> world" }'
-| hello, 
-| world
