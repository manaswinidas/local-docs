17.4.1简介

使用内部设施gawk通过调用传递到扩展中的函数指针来实现。

提供API函数指针用于以下几种操作：

  • 分配，重新分配和释放内存。
  • 注册功能。您可以注册：
    • -扩展功能
    • -退出回调
    • -版本字符串
    • -输入解析器
    • -输出包装
    • -两路处理器

    所有这些都将在本章后面详细讨论。

  • 打印致命，警告和“掉毛”警告消息。
  • 更新中ERRNO或取消设置。
  • 访问参数，包括将未定义的参数转换为数组。
  • 符号表访问：检索全局变量，创建一个或更改一个。
  • 创建和释放缓存的值；这为使用多个变量的值提供了一种有效的方法，并且可以赢得巨大的性能。
  • 操纵数组：
    • -检索，添加，删除和修改元素
    • -获取数组中元素的数量
    • -创建一个新数组
    • -清除数组
    • -展平数组以方便在其所有索引和元素上进行C风格的循环
  • 访问和操纵重定向。

  • 如果您的扩展程序使用MPFR功能，并且您希望从gawk和/或将此类值传递给它，则必须包括包含之前的标题 。

  • 尽管API仅使用ISO C 90功能，但有例外。 “构造函数”功能使用inline关键词。如果您的编译器不支持此关键字，则应将“ -Dinline=''在您的命令行上，或使用GNU Autotools并包含一个config.h文件中的扩展名。
  • 所有指针由填充gawk指向由管理的内存gawk且扩展名应将其视为只读。传递给所有字符串的内存gawk来自分机号必须来自呼叫gawk_malloc() ， gawk_calloc() ， 要么gawk_realloc() ，并由gawk从那时起。
  • API定义了几个简单的struct映射从中看到的值的awk 。值可以是double ，字符串或数组（例如在多维数组中或在创建新数组时）。

    字符串值同时保留指针和长度，因为允许嵌入NUL字符。

    注意：出于意图， gawk使用当前的多字节编码来维护字符串（由LC_xxx环境变量），并且不使用宽字符。这匹配如何gawk在内部存储字符串以及文件中可能输入和输出字符的方式。

    注意：通过以下方式将字符串值传递给扩展gawk总是以NUL终止。因此，将此类字符串值传递到标准库和系统例程是安全的。但是，因为gawk允许在字符串数据中嵌入NUL字符，在将该数据用作常规C字符串之前，应检查传递给扩展名的该字符串的长度是否与返回值相匹配。 strlen()为了它。

  • 在获取值（例如参数或全局变量或数组元素的参数）时，扩展名会请求特定类型（数字，字符串，标量，值cookie，数组或“未定义”）。当请求为“ undefined”时，返回值将具有实际的基础类型。

    但是，如果请求和实际类型不匹配，则访问函数将返回“ false”并填写实际值的类型，以便扩展名可以例如打印错误消息（例如“ scalar”通过了预期的数组”）。

您可以通过直接使用函数指针来调用API函数，但是接口并不那么漂亮。为了使扩展代码看起来更像常规代码， gawkapi.h头文件定义了几个应在代码中使用的宏。本节将宏视为函数。
