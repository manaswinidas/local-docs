7.1.3使用模式指定记录范围

范围模式由两个模式组成，两个模式之间用逗号隔开，形式为“ begpat, endpat '。它用于匹配连续输入记录的范围。第一种模式begpat ，控制范围的开始位置，而endpat控制图案结束的位置。例如，以下内容：

awk '$1 == "on", $1 == "off"' myfile

打印中的每条记录myfile介于on '/' off对（含）。

范围模式通过匹配开始begpat针对每个输入记录。记录匹配时begpat ，范围模式已打开 ，并且范围模式也与此记录匹配。只要范围模式保持打开状态，它就会自动匹配每个读取的输入记录。范围模式也匹配endpat针对每个输入记录；成功执行此操作后，范围模式将再次关闭以用于以下记录。然后范围模式回到检查begpat针对每个记录。

打开范围模式的记录和关闭范围模式的记录都与范围模式匹配。如果您不想对这些记录进行操作，则可以编写if规则操作中的语句，以将其与您感兴趣的记录区分开。

可以通过同一记录打开和关闭模式。如果记录满足两个条件，则仅对该记录执行操作。例如，假设两个相同的标记之间有文字（例如，“ % '符号），每行都应忽略。第一种尝试是将描述定界文本的范围模式与next声明（尚未讨论，请参见该next声明 ）。这导致awk跳过对当前记录的任何进一步处理，并从下一个输入记录重新开始。这样的程序如下所示：

/^%$/,/^%$/    { next }
               { print }

该程序失败，因为范围模式已由第一行（仅带有' % ' 在上面。要完成此任务，请使用标志以以下方式编写程序：

/^%$/     { skip = ! skip; next }
skip == 1 { next } # skip lines with `skip' set

在范围模式中，逗号（' , '）在所有运算符中具有最低的优先级（即，最后被求值）。因此，以下程序尝试将范围模式与另一个更简单的测试结合起来：

echo Yes | awk '/1/,/2/ || /Yes/'

该程序的目的是“ (/1/,/2/) || /Yes/ '。然而， awk将此解释为“ /1/, (/2/ || /Yes/) '。这不能更改或解决；范围模式不能与其他模式结合使用：

$ echo Yes | gawk '(/1/,/2/) || /Yes/'
error→ gawk: cmd. line:1: (/1/,/2/) || /Yes/
error→ gawk: cmd. line:1:           ^ syntax error

作为一个次要的兴趣点，尽管它的风格很差，但POSIX允许您在范围模式中的逗号后加换行符。 （直流）
