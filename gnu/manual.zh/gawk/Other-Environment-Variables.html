2.5.3其他环境变量

其他许多环境变量也会影响gawk的行为，但它们更加专业化。以下列表中的内容仅供普通用户使用：

GAWK_MSEC_SLEEP

指定连接重试之间的时间间隔（以毫秒为单位）。在不支持usleep()系统调用时，该值将舍入为整数秒数。

GAWK_READ_TIMEOUT

指定时间（以毫秒为单位） gawk等待输入，然后返回错误。请参阅读取超时输入部分。

GAWK_SOCK_RETRIES

控制次数gawk尝试在放弃之前重试双向TCP / IP（套接字）连接。请参阅使用部分gawk用于网络编程 。请注意， 启用非致命I / O时（请参阅“ 启用非致命输出”部分）， gawk仅尝试一次打开TCP / IP套接字。

POSIXLY_CORRECT

原因gawk切换到POSIX兼容模式，从而禁用所有传统扩展和GNU扩展。请参阅“ 命令行选项”部分 。

下表中的环境变量旨在供gawk开发人员进行测试和调整。它们可能会发生变化。变量是：

AWKBUFSIZE

此变量仅影响gawk在符合POSIX的系统上。值为“ exact '， gawk使用每个输入文件的大小作为要分配给I / O的内存缓冲区的大小。否则，该值应为数字，并且gawk使用该数字作为要分配的缓冲区的大小。（如果未设置此变量， gawk使用较小的文件大小和“默认”块大小，通常是文件系统的I / O块大小。）

AWK_HASH

如果此变量的值是' gst '， gawk切换为使用GNU Smalltalk的哈希函数来管理数组。此功能可能比标准功能快一些。

AWKREADFUNC

如果该变量存在， gawk切换为一次一行读取源文件，而不是逐块读取。存在此问题是为了调试非POSIX操作系统上的文件系统上的问题，在非POSIX操作系统上，I / O是在记录中而不是在块中执行的。

GAWK_MSG_SRC

如果该变量存在， gawk包括文件名和行号gawk从中生成警告和/或致命消息的源代码。其目的是帮助隔离消息的来源，因为有多个地方会产生相同的警告或错误消息。

GAWK_LOCALE_DIR

指定以下对象的已编译消息对象文件的位置gawk本身。这被传递给bindtextdomain()功能何时gawk启动。

GAWK_NO_DFA

如果该变量存在， gawk不会将DFA正则表达式匹配器用于“是否匹配”这类测试。这可能导致gawk慢一点。其目的是帮助隔离两个正则表达式匹配器之间的差异， gawk内部使用。（不应该存在差异，但有时理论和实践不会相互协调。）

GAWK_STACKSIZE

这指定了金额gawk必要时应扩展其内部评估堆栈。

INT_CHAIN_MAX

这指定了预期的最大项目数gawk将维持在散列链上，用于管理由整数索引的数组。

STR_CHAIN_MAX

这指定了预期的最大项目数gawk将保留在哈希链上，用于管理由字符串索引的数组。

TIDYMEM

如果该变量存在， gawk使用mtrace() GNU C库中的库调用可帮助跟踪可能的内存泄漏。
