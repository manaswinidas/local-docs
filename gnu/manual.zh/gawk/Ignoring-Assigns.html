10.3.5将分配视为文件名

有时候，您可能不想要awk处理命令行变量分配（请参阅“在命令行上分配变量”部分）。特别是，如果您的文件名包含“ =字符， awk将文件名视为分配，不对其进行处理。

一些用户建议使用其他命令行选项gawk禁用命令行分配。但是，使用库文件进行一些简单的编程就可以达到目的：

# noassign.awk --- library file to avoid the need for a
# special option that disables command-line assignments

function disable_assigns(argc, argv,    i)
{
    for (i = 1; i < argc; i++)
        if (argv[i] ~ /^[a-zA-Z_][a-zA-Z0-9_]*=.*/)
            argv[i] = ("./" argv[i])
}

BEGIN {
    if (No_command_assign)
        disable_assigns(ARGC, ARGV)
}

然后，您可以通过以下方式运行程序：

awk -v No_command_assign=1 -f noassign.awk -f yourprog.awk *

该函数通过遍历参数来工作。它以' ./ '匹配任何与变量赋值形式匹配的参数，然后将该参数转换为文件名。

指某东西的用途No_command_assign通过为变量提供一个真值，您可以在调用时禁用命令行分配。如果未设置，则最初为零（即false），因此命令行参数将保留。
