7.5.3使用ARGC和ARGV

传达信息的内置变量提供了以下程序，该程序描述了包含在其中的信息ARGC和ARGV ：

$ awk 'BEGIN {
>        for (i = 0; i < ARGC; i++)
>            print ARGV[i]
>      }' inventory-shipped mail-list
-| awk
-| inventory-shipped
-| mail-list

在这个例子中ARGV[0]包含“ awk '， ARGV[1]包含“ inventory-shipped '和ARGV[2]包含“ mail-list '。请注意awk程序未输入ARGV 。也不会输入其他命令行选项及其参数。这包括使用-v选项（请参阅“ 命令行选项”部分 ）。命令行上的普通变量分配被视为参数，并且确实显示在ARGV数组。在名为的文件中给出以下程序showargs.awk ：

BEGIN {
    printf "A=%d, B=%d\n", A, B
    for (i = 0; i < ARGC; i++)
        printf "\tARGV[%d] = %s\n", i, ARGV[i]
}
END   { printf "A=%d, B=%d\n", A, B }

运行它会产生以下结果：

$ awk -v A=1 -f showargs.awk B=2 /dev/null
-| A=1, B=0
-|        ARGV[0] = awk
-|        ARGV[1] = B=2
-|        ARGV[2] = /dev/null
-| A=1, B=2

程序可以改变ARGC和的元素ARGV 。每一次awk到达输入文件的末尾，它使用的下一个元素ARGV作为下一个输入文件的名称。通过在其中存储其他字符串，程序可以更改读取的文件。使用"-"代表标准输入。存储其他元素并递增ARGC导致读取其他文件。

如果值ARGC减少，从而从列表末尾删除输入文件。通过记录旧值ARGC在其他地方，程序可以将消除的参数视为文件名以外的其他内容。

要从列表中间删除文件，请存储空字符串（ "" ）成ARGV代替文件名。作为一项特殊功能， awk忽略被空字符串替换的文件名。另一种选择是使用delete从中删除元素的语句ARGV （见的delete声明 ）。

所有这些动作通常是在BEGIN规则，在开始实际处理输入之前。有关从中删除元素的每种方法的示例，请参见将大文件拆分为小节，并参见将输出复制为多个文件小节。 ARGV 。

实际将期权纳入awk程序，结束awk具有的选项--然后提供awk程序的选项，方式如下：

awk -f myprog.awk -- -v -q file1 file2 …

以下片段过程ARGV为了检查然后删除前面提到的命令行选项：

BEGIN {
    for (i = 1; i < ARGC; i++) {
        if (ARGV[i] == "-v")
            verbose = 1
        else if (ARGV[i] == "-q")
            debug = 1
        else if (ARGV[i] ~ /^-./) {
            e = sprintf("%s: unrecognized option -- %c",
                    ARGV[0], substr(ARGV[i], 2, 1))
            print e > "/dev/stderr"
        } else
            break
        delete ARGV[i]
    }
}

结束awk具有的选项--不需要gawk 。除非--posix已被指定， gawk默默地将任何无法识别的选项放入ARGV为了awk程序要处理。一旦看到未知选项， gawk停止寻找其他可能会识别的选项。上一个命令行gawk将会：

gawk -f myprog.awk -q -v file1 file2 …

因为-q不是有效的gawk选项，它和以下-v传递给awk程序。（请参阅处理命令行选项部分awk解析命令行选项的库函数。）

在设计程序时，应选择与以下选项不冲突的选项gawk是的，因为它将在将其余命令行传递到程序之前处理它接受的所有选项。使用“ #!与-E选项可能会有所帮助（请参阅“ 可执行文件”部分awk程序 ，请参阅“ 命令行选项”部分 ）。
