以下功能完成了大部分工作， awk_array_t具有从有效值获得的值的结果数组struct stat 。这项工作是通过单独的功能完成的，以支持stat()的功能gawk并支持fts()扩展名，包含在同一文件中，但此处未显示其代码（请参阅文件相关功能部分 ）。

第三个论点stat()之前没有讨论过。此参数是可选的。如果存在，它将导致do_stat()使用stat()系统调用而不是lstat()系统调用。这是通过使用函数指针完成的： statfunc 。 statfunc初始化为指向lstat() （代替stat() ）以获取文件信息（如果文件是符号链接）。但是，如果包含第三个参数， statfunc设置为指向stat() ，而不是。

这里是do_stat()函数，从变量声明和参数检查开始：

/* do_stat --- provide a stat() function for gawk */

static awk_value_t *
do_stat(int nargs, awk_value_t *result, struct awk_ext_func *unused)
{
    awk_value_t file_param, array_param;
    char *name;
    awk_array_t array;
    int ret;
    struct stat sbuf;
    /* default is lstat() */
    int (*statfunc)(const char *path, struct stat *sbuf) = lstat;

    assert(result != NULL);

然后是实际的工作。首先，该函数获取参数。接下来，它获取文件的信息。如果调用的函数（ lstat()要么stat() ）返回错误，代码集ERRNO并返回：

    /* file is first arg, array to hold results is second */
    if (   ! get_argument(0, AWK_STRING, & file_param)
        || ! get_argument(1, AWK_ARRAY, & array_param)) {
        warning(ext_id, _("stat: bad parameters"));
        return make_number(-1, result);
    }

    if (nargs == 3) {
        statfunc = stat;
    }

    name = file_param.str_value.str;
    array = array_param.array_cookie;

    /* always empty out the array */
    clear_array(array);

    /* stat the file; if error, set ERRNO and return */
    ret = statfunc(name, & sbuf);

    if (ret < 0) {
        update_ERRNO_int(errno);
        return make_number(ret, result);
    }

繁琐的工作是由fill_stat_array() ，如前所示。完成后，函数将返回以下结果： fill_stat_array() ：

    ret = fill_stat_array(name, array, & sbuf);

    return make_number(ret, result);
}
