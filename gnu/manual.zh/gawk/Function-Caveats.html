9.2.3.4关于调用函数的其他要点

一些awk实现使您可以调用尚未定义的函数。当程序实际尝试调用该函数时，它们仅在运行时报告问题。例如：

BEGIN {
    if (0)
        foo()
    else
        bar()
}
function bar() { … }
# note that `foo' is not defined

因为“ if '陈述永远不会是真的，这不是一个真正的问题foo()尚未定义。但是，通常情况下，如果程序调用未定义的函数，则会出现问题。

如果--lint已指定（请参阅“ 命令行选项”部分 ）， gawk报告对未定义函数的调用。

一些awk如果您使用以下任一实现，则会生成运行时错误next陈述或nextfile声明（见部分的next声明 ，并请参见本nextfile用户定义函数中的statement ）。 gawk没有此限制。

您可以调用一个函数并向其传递比其声明时更多的参数，例如：

function foo(p1, p2)
{
    …
}

BEGIN {
    foo(1, 2, 3, 4)
}

但是，这样做是不好的做法。被调用的函数无法通过传递给它的附加值来做任何事情，因此awk计算表达式，然后将其丢弃。

更重要的是，这样的调用会使接下来阅读程序的人感到困惑。 62函数参数通常是影响函数执行的计算的输入项。调用一个具有比其可接受的参数更多的参数的函数会给人一种错误的印象，即这些值对函数很重要，而实际上并不重要。

因为这是一个不好的做法， gawk每当执行此类函数调用时，都会无条件发出警告。（如果您不喜欢警告，请修正您的代码！毕竟这是不对的。）
