4.10.9要记住的要点getline

以下是一些杂点getline您应该牢记：

  • 什么时候getline改变价值$0和NF ， awk 不会自动跳转到该程序的启动和开始测试新的记录对每个模式。但是，将根据任何后续规则测试新记录。
  • 一些很老awk实施限制了awk程序可能只对其中一个开放。在gawk ，没有这样的限制。您可以在基础操作系统允许的情况下打开尽可能多的管道（和协同进程）。
  • 如果您使用一个有趣的副作用getline内部没有重定向BEGIN规则。因为未重定向getline从命令行数据文件中读取第一个getline命令原因awk设置值FILENAME 。一般， FILENAME里面没有值BEGIN规则，因为您尚未开始处理命令行数据文件。（直流）（见的BEGIN和END特殊模式 ;另请参阅传达信息的内置变量部分。）
  • 使用FILENAME与getline （' getline < FILENAME '）可能会引起混乱。 awk从当前输入文件打开一个单独的输入流。但是，通过不使用变量， $0和NF仍在更新。如果这样做，可能是偶然的，您应该重新考虑您要实现的目标。
  • 的总结getline Variants ，提供了一个表格，总结了getline变体以及它们会影响哪些变量。值得注意的是，不使用重定向的那些变体可能会导致FILENAME如果它们导致更新awk开始读取新的输入文件。
  • 如果分配的变量是带有副作用的表达式，则不同版本的awk遇到文件结束时的行为有所不同。某些版本不评估表达式。许多版本（包括gawk ）做。这是一个示例，由Duncan Moore提供：
    BEGIN {
    system("echo 1 > f")
    while ((getline a[++c] < "f") > 0) { }
    print c
}

    在这里，副作用是“ ++c '。是c如果在中的元素之前遇到文件结尾，则增加a被安排了？

    gawk对待getline类似于函数调用，并计算表达式' a[++c] '，然后尝试从中读取f 。但是，某些版本的awk仅当他们知道要分配一个字符串值时，才对表达式求值。
