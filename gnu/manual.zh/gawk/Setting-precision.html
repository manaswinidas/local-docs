16.4.4设置精度

gawk使用全局工作精度；它不会跟踪单个数字的精度或准确性。执行算术运算或调用内置函数会将结果四舍五入到当前的工作精度。默认工作精度为53位，您可以使用预定义变量进行修改PREC 。您还可以将值设置为表16.4中所示的预定义的不区分大小写的字符串之一，以模拟IEEE 754二进制格式。

PRECIEEE 754二进制格式
"half"16位半精度
"single"基本的32位单精度
"double"基本的64位双精度
"quad"基本的128位四倍精度
"oct"256位八重精度

表16.4：的预定义精度字符串PREC

以下示例说明了更改精度对算术运算的影响：

$ gawk -M -v PREC=100 'BEGIN { x = 1.0e-400; print x + 0
>   PREC = "double"; print x + 0 }'
-| 1e-400
-| 0

注意：注意浮点常量！从程序源代码读取浮点常量时， gawk使用默认精度（C的精度） double ），除非被特殊变量赋值所覆盖PREC在命令行上，将其内部存储为MPFR编号。使用更改精度PREC在程序文本中， 不会更改常量的精度。

如果您需要以比默认精度更高的精度来表示浮点常量，并且不能使用命令行分配来PREC ，则应尽可能将常量指定为字符串或有理数。下面的示例说明了打印浮点常量的各种方式之间的差异：

$ gawk -M 'BEGIN { PREC = 113; printf("%0.25f\n", 0.1) }'
-| 0.1000000000000000055511151
$ gawk -M -v PREC=113 'BEGIN { printf("%0.25f\n", 0.1) }'
-| 0.1000000000000000000000000
$ gawk -M 'BEGIN { PREC = 113; printf("%0.25f\n", "0.1") }'
-| 0.1000000000000000000000000
$ gawk -M 'BEGIN { PREC = 113; printf("%0.25f\n", 1/10) }'
-| 0.1000000000000000000000000
