退出调试器。请参阅“ quit '，在此列表的后面。

help
h

打印所有列表gawk调试器命令及其用法的简短摘要。 ' help command '显示有关命令的信息command 。

list [ - | + | n | filename:n | n – m | function ]
l [ - | + | n | filename:n | n – m | function ]

从当前源文件或名为的文件中打印指定的行（默认为15） filename 。可能的论点list如下面所述：

- （减去）

在最后打印的行之前打印行。

+

在最后打印行之后打印行。 list没有任何参数就做同样的事情。

n

以行号为中心打印行n 。

n –m

从打印行n至m 。

filename:n

以行号为中心打印行n在源文件中filename 。此命令可能会更改当前的源文件。

function

以功能开头为中心的打印行function 。此命令可能会更改当前的源文件。

quit
q

退出调试器。调试非常有趣，但是有时我们所有人都不得不承担生活中的其他义务，有时我们会发现错误并可以继续进行下一个！如前所述，如果您正在运行程序，则调试器会在您键入“ q ' 要么 ' quit '，以确保您确实要退出。

trace [ on | off ]

开启或关闭将要执行的指令的连续打印，以及awk他们实施的路线。默认是off 。

希望这些说明中的大多数“操作码”都是不言自明的，并且使用stepi和nexti而trace开启将使他们成为熟悉的朋友。
