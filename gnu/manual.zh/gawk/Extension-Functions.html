17.4.5.1注册扩展功能

扩展功能由以下记录描述：

typedef struct awk_ext_func {
    const char *name;
    awk_value_t *(*const function)(int num_actual_args,
                                   awk_value_t *result,
                                   struct awk_ext_func *finfo);
    const size_t max_expected_args;
    const size_t min_required_args;
    awk_bool_t suppress_lint;
    void *data;        /* opaque pointer to any extra state */
} awk_ext_func_t;

字段是：

const char *name;

新功能的名称。 awk级代码使用此名称调用该函数。这是一个常规的C字符串。

函数名称必须遵守以下规则awk身份标识。也就是说，它们必须以英文字母或下划线开头，后跟任意数量的字母，数字和下划线。函数名称中的字母大小写很重要。

awk_value_t *(*const function)(int num_actual_args,
                              awk_value_t *result,
                              struct awk_ext_func *finfo);

这是指向提供扩展功能的C函数的指针。该功能必须填写*result使用数字，字符串或正则表达式。 gawk获取任何字符串内存的所有权。如前所述，字符串内存必须来自以下一种gawk_malloc() ， gawk_calloc() ， 要么gawk_realloc() 。

的num_actual_args参数告诉C函数从调用中传递了多少实际参数awk码。

的finfo参数是指向的指针awk_ext_func_t为此功能。被调用函数可以根据需要访问其中的数据，也可以不访问。

该函数必须返回的值result 。这是为了方便内部调用代码gawk 。

const size_t max_expected_args;

这是函数期望接收的最大参数数量。如果调用的参数超过此数量，并且已启用了lint检查，则gawk打印警告消息。有关更多信息，请参见suppress_lint ，在此列表的后面。

const size_t min_required_args;

这是函数期望接收的最少参数。如果以较少的参数调用， gawk显示致命错误消息并退出。

awk_bool_t suppress_lint;

这个标志告诉gawk如果已启用棉绒检查并且呼叫中提供的参数比预期的多，则不打印棉绒消息。扩展功能可以判断gawk通过检查“ num_actual_args > finfo->max_expected_args '。如果是这样，并且该功能不希望打印更多的皮棉消息，则应设置finfo->suppress_lint至awk_true 。

void *data;

这是指向扩展函数在调用时可能希望提供的任何数据的不透明指针。通过awk_ext_func_t扩展功能的结构，并具有此指针可用，因此可以编写实现多个功能的单个C或C ++函数awk级扩展功能。

获得代表扩展功能的记录后，即可向gawk使用此API函数：

awk_bool_t add_ext_func(const char *name_space, awk_ext_func_t *func);

成功时此函数返回true，否则返回false。的name_space参数是命名空间，在其中放置功能（参见命名空间中gawk ）。使用空字符串（ "" ） 要么"awk"将功能放置在默认位置awk命名空间。的func指针是一个的地址struct代表您的功能，如上所述。

gawk不修改什么func指向，但扩展功能本身接收此指针并可以修改其指向的指针，因此有意不声明为const 。

的结合min_required_args ， max_expected_args和suppress_lint可能会造成混乱。这是您应该设置的方法。

任何数量的参数都是有效的

组min_required_args和max_expected_args归零并设置suppress_lint至awk_true 。

需要最少数量的参数，没有最大数量的限制

组min_required_args达到最低要求。组max_expected_args归零并设置suppress_lint至awk_true 。

需要最少数量的参数，预计最大数量

组min_required_args达到最低要求。组max_expected_args达到最大预期。组suppress_lint至awk_false 。

要求的参数个数最少，最大不能超过

组min_required_args达到最低要求。组max_expected_args达到最大预期。组suppress_lint至awk_false 。在扩展功能中，检查num_actual_args不超过f->max_expected_args 。如果是这样，则发出致命错误消息。
