8.3使用未初始化的变量作为下标

假设有必要编写一个程序以相反的顺序打印输入数据。一个合理的尝试（带有一些测试数据）可能看起来像这样：

$ echo 'line 1
> line 2
> line 3' | awk '{ l[lines] = $0; ++lines }
> END {
>     for (i = lines - 1; i >= 0; i--)
>        print l[i]
> }'
-| line 3
-| line 2

不幸的是，输入数据的第一行没有出现在输出中！

乍一看，我们认为该程序应该有效。变量lines未初始化，并且未初始化的变量的数值为零。所以， awk应该已经印出了l[0] 。

这里的问题是awk数组始终是字符串。未初始化的变量在用作字符串时具有值"" ，不为零。因此， line 1最终存储在l[""] 。以下版本的程序可以正常运行：

{ l[lines++] = $0 }
END {
    for (i = lines - 1; i >= 0; i--)
       print l[i]
}

在这里， ++ ' 势力lines为数字，从而使“旧值”的数字为零。然后将其转换为"0"作为数组下标。

尽管有些不寻常，但空字符串（ "" ）是有效的数组下标。（直流） gawk警告有关在以下情况下使用空字符串作为下标--lint在命令行上提供（请参阅“ 命令行选项”一节）。
