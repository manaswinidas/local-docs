13.4.2重新排列printf争论

格式化字符串printf和sprintf() （请参见使用部分printf Fancier Printing的声明（ ）提出了翻译的特殊问题。考虑以下情况： 92 

printf(_"String `%s' has %d characters\n",
          string, length(string)))

为此可能的德语翻译可能是：

"%d Zeichen lang ist die Zeichenkette `%s'\n"

问题应该很明显：格式规格的顺序与原始顺序不同！即使gettext()可以在运行时返回翻译后的字符串，它不能更改对printf 。

为了解决这个问题， printf格式说明符可能还有一个附加的可选元素，我们称其为位置说明符 。例如：

"%2$d Zeichen lang ist die Zeichenkette `%1$s'\n"

在此，位置说明符由一个整数（表示要使用的参数）和一个' $ '。计数是基于一的，并且不包含格式字符串本身。因此，在以下示例中，“ string '是第一个参数，' length(string) '是第二个：

$ gawk 'BEGIN {
>     string = "Don\47t Panic"
>     printf "%2$d characters live in \"%1$s\"\n",
>                         string, length(string)
> }'
-| 11 characters live in "Don't Panic"

位置说明符（如果存在）在格式说明中排在标志，字段宽度和/或精度之前。

位置说明符可以与动态字段宽度和精度功能一起使用：

$ gawk 'BEGIN {
>    printf("%*.*s\n", 10, 20, "hello")
>    printf("%3$*2$.*1$s\n", 20, 10, "hello")
> }'
-|      hello
-|      hello

注意：使用' * '和位置说明符' * '首先出现，然后是整数位置，然后是' $ '。这有点违反直觉。

gawk不允许您在同一字符串中混用常规格式说明符和带有位置说明符的说明符：

$ gawk 'BEGIN { printf "%d %3$s\n", 1, 2, "hi" }'
error→ gawk: cmd. line:1: fatal: must use `count$' on all formats or none

注意：有些病理情况gawk可能无法诊断。在这种情况下，输出可能不会达到您的期望。即使将它们混合，仍然是一个坏主意gawk没有检测到它。

尽管位置说明符可以直接在awk程序，其主要目的是帮助将格式字符串正确翻译成与首次编写该程序不同的语言。
