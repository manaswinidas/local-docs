17.7.9转储和还原阵列

的rwarray扩展添加了两个函数，名为writea()和reada() ， 如下：

@load "rwarray"

这是您加载扩展程序的方式。

ret = writea(file, array)

此函数采用字符串参数，该参数是要将数组转储到的文件的名称，而数组本身作为第二个参数。 writea()了解数组的数组。成功返回1，失败返回0。

ret = reada(file, array)

reada()是...的逆writea() ;它读取名为第一个参数的文件，并填充名为第二个参数的数组。它首先清除数组。同样，这里的返回值在成功时为1，在失败时为零。

由创建的数组reada()与写的相同writea()在某种意义上说内容是相同的。但是，由于实现问题，重新创建的数组的数组遍历顺序可能与原始数组的遍历顺序不同。作为数组遍历顺序awk默认情况下是undefined，这（从技术上）不是问题。如果需要保证特定的遍历顺序，请使用gawk为此（请参阅控制数组遍历和数组排序一节 ）。

该文件包含二进制数据。所有整数值均以网络字节顺序写入。但是，双精度浮点值被写为本地二进制数据。因此，理论上只包含字符串数据的数组可以在具有一个字节顺序的系统上转储，并在具有不同字节顺序的系统上恢复，但是尚未尝试过。

这是一个例子：

@load "rwarray"
…
ret = writea("arraydump.bin", array)
…
ret = reada("arraydump.bin", array)
