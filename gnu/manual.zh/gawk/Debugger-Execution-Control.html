14.3.2执行控制

现在已经准备好断点，可以开始运行程序并观察其行为了。与前面的示例相比，用于控制程序执行的命令更多：

commands [ n ]
silent
…
end

设置在断点或监视点停止时要执行的命令列表。 n是断点或监视点编号。如果没有数字，则使用最后一组。实际的命令从下一行开始，并以end命令。如果命令silent在列表中，不会显示有关在断点处停止和源代码行的常规消息。列表中恢复执行的任何命令（例如， continue ）终止列表（隐式end ），随后的命令将被忽略。例如：

gawk> commands
> silent
> printf "A silent breakpoint; i = %d\n", i
> info locals
> set i = 10
> continue
> end
gawk>
continue [ count ]
c [ count ]

恢复程序执行。如果从断点继续并且count指定后，在下一个位置忽略该断点count停止之前的时间。

finish

执行直到选择的堆栈帧返回。打印返回的值。

next [ count ]
n [ count ]

继续执行到下一个源代码行，逐步执行函数调用。争论count控制重复动作的次数，例如step 。

nexti [ count ]
ni [ count ]

执行一个（或count ）指令，逐步执行函数调用。

return [ value ]

取消执行函数调用。如果value （字符串或数字）被指定，它用作函数的返回值。如果在除最里面的那帧（当前执行的功能；即帧号0）以外的其他帧中使用，则除了选择所选择的那一个以外，还丢弃所有内部帧，并且该帧的调用方成为最里面的帧。

run
r

开始/重新开始执行程序。重新启动时，调试器将保留当前的断点，监视点，命令历史记录，自动显示变量和调试器选项。

step [ count ]
s [ count ]

继续执行，直到控制到达当前堆栈帧中的其他源代码行，进入该行中调用的任何函数。如果论点count除非遇到断点或观察点，否则将提供停止之前多次执行的步骤。

stepi [ count ]
si [ count ]

执行一个（或count ）指令，逐步进入函数调用。（为说明“指令”的含义gawk ，请参阅下方显示的输出dump在其他命令中 。）

until [[ filename: ] n | function ]
u [[ filename: ] n | function ]

不带任何参数的情况下，继续执行直到到达当前堆栈帧中当前行之后的行。使用自变量继续执行，直到到达指定位置，否则返回当前堆栈帧。
