10.3.2重新读取当前文件

对新的内置函数的另一个要求是，该函数可以重新读取当前文件。发出请求的用户不想使用getline （请参阅使用getline ）。

但是，只要您不在END规则，安排立即关闭当前输入文件然后从顶部重新开始很容易。由于缺少更好的名称，我们将调用该函数rewind() ：

# rewind.awk --- rewind the current file and start over

function rewind(    i)
{
    # shift remaining arguments up
    for (i = ARGC; i > ARGIND; i--)
        ARGV[i] = ARGV[i-1]

    # make sure gawk knows to keep going
    ARGC++

    # make current file next to get done
    ARGV[ARGIND+1] = FILENAME

    # do it
    nextfile
}

的rewind()功能取决于ARGIND变量（请参阅传达信息的内置变量部分），该变量特定于gawk 。它还依赖于nextfile关键字（见节的nextfile声明 ）。因此，您不应从ENDFILE规则。（无论如何，这是没有必要的，因为gawk尽快转到下一个文件ENDFILE规则完成！）

你要小心点rewind() 。如果不注意，最终可能导致无限递归。这是一个示例用法：

$ cat data
-| a
-| b
-| c
-| d
-| e

$ cat test.awk
-| FNR == 3 && ! rewound {
-|    rewound = 1
-|    rewind()
-| }
-| 
-| { print FILENAME, FNR, $0 }

$ gawk -f rewind.awk -f test.awk data 
-| data 1 a
-| data 2 b
-| data 1 a
-| data 2 b
-| data 3 c

-| data 4 d
-| data 5 e
