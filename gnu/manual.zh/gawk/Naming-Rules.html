15.5命名空间和组件命名规则

下列规则适用于名称空间和组件名称。

  • 将合格名称用作函数参数名称是语法错误。
  • 使用任何标准都是语法错误awk保留字（例如if要么for ），或任何标准内置函数的名称（例如sin()要么gsub() ）作为合格名称的一部分。因此，以下内容会产生语法错误：
    @namespace "example"

function gsub(str, pat, result) { … }
  • 外awk名称空间，其他名称gawk内置功能（例如gensub()要么strftime() ） 可以用作组件名称。相同的名称集可以用作名称空间名称，尽管这可能会造成混淆。
  • 额外的gawk内置函数仍可以从外部调用awk通过限定它们的名称空间。例如， awk::systime() 。这是一个有点愚蠢的示例，它展示了此规则和上一个规则：
    BEGIN {
    print "in awk namespace, systime() =", systime()
}

@namespace "testing"

function systime()
{
    print "in testing namespace, systime() =", awk::systime()
}

BEGIN {
    systime()
}

    运行时，它将产生如下输出：

    $ gawk -f systime.awk
-| in awk namespace, systime() = 1500488503
-| in testing namespace, systime() = 1500488503
  • gawk可以使用预定义的变量名： NF::NR是有效的，如果不是那么有用的话。
