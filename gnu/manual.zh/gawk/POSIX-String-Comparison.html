6.3.2.3基于语言环境整理顺序的字符串比较

POSIX标准通常说所有字符串比较都是根据语言环境的整理顺序执行的 。这是语言环境定义的字符排序顺序（有关更多讨论，请参见“您在哪里有所作为”部分）。此顺序通常与直接进行逐字节比较时获得的结果非常不同。 34

由于此行为与现有做法大不相同， gawk仅在POSIX模式下实现它（请参阅“ 命令行选项”一节）。这是一个说明差异的示例， en_US.UTF-8地区：

$ gawk 'BEGIN { printf("ABC < abc = %s\n",
>                     ("ABC" < "abc" ? "TRUE" : "FALSE")) }'
-| ABC < abc = TRUE
$ gawk --posix 'BEGIN { printf("ABC < abc = %s\n",
>                             ("ABC" < "abc" ? "TRUE" : "FALSE")) }'
-| ABC < abc = FALSE

幸运的是，自2016年8月起，不再需要根据区域设置整理顺序进行比较==和!=操作员。 35但是，仍然需要基于语言环境进行比较< ， <= ， >和>= 。因此，POSIX建议如下：

自从==操作员检查字符串是否相同，而不是是否相等排序，需要检查字符串是否相等的应用程序可以使用：

a <= b && a >= b

从4.2版开始， gawk继续使用语言环境整理顺序< ， <= ， >和>=仅在POSIX模式下。
