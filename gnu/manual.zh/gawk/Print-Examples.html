5.2 print声明示例

每print语句至少输出一行。但是，它不仅限于一行。如果项目值是包含换行符的字符串，则换行符将与字符串的其余部分一起输出。一个print语句可以用这种方式制作任意数量的行。

以下是打印包含嵌入式换行符（“ \n '是转义序列，用于表示换行符；请参阅转义序列部分）：

$ awk 'BEGIN { print "line one\nline two\nline three" }'
-| line one
-| line two
-| line three

下一个示例在inventory-shipped文件，打印每个输入记录的前两个字段，并在它们之间留一个空格：

$ awk '{ print $1, $2 }' inventory-shipped
-| Jan 13
-| Feb 15
-| Mar 15
…

使用print声明是省略两个项目之间的逗号。这通常具有使项目在输出中一起运行而没有空间的效果。这样做的原因是两个字符串表达式并列在awk意味着将它们串联起来。这是相同的程序，没有逗号：

$ awk '{ print $1 $2 }' inventory-shipped
-| Jan13
-| Feb15
-| Mar15
…

对于不熟悉的人inventory-shipped文件，两个示例的输出都没有多大意义。开头的标题行会使其更清晰。让我们在月份表中添加一些标题（ $1 ）和运来的绿色包装箱（ $2 ）。我们使用BEGIN规则（见节的BEGIN和END特殊图案 ），以便标题仅打印一次：

awk 'BEGIN {  print "Month Crates"
              print "----- ------" }
           {  print $1, $2 }' inventory-shipped

运行时，程序将打印以下内容：

Month Crates
----- ------
Jan 13
Feb 15
Mar 15
…

但是，唯一的问题是标题和表格数据不对齐！我们可以通过在两个字段之间打印一些空格来解决此问题：

awk 'BEGIN { print "Month Crates"
             print "----- ------" }
           { print $1, "     ", $2 }' inventory-shipped

如果要修复许多列，以这种方式排列列可能会变得非常复杂。计算两列或三列的空间很简单，但仅此一点便会占用大量时间。这就是为什么printf语句已创建（请参见使用部分printf发烧友声明 ）;它的特色之一是排列数据列。

注意：您可以继续print要么printf只需在逗号后面加上换行符即可声明（请参见awk语句与行 ）。
