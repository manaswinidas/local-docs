1.1.6 Shell报价问题

• DOS报价 ：  引用Windows批处理文件。

适用于中短awk程序，最方便的是在awk命令行。最好用单引号将整个程序括起来。无论您是在shell提示符下以交互方式输入程序，还是将其作为较大的shell脚本的一部分编写，都是如此：

awk 'program text' input-file1 input-file2 …

使用完外壳程序后，了解外壳程序引用规则会很有帮助。以下规则仅适用于POSIX兼容的Bourne样式的shell（例如Bash，GNU Bourne-Again Shell）。如果您使用C Shell，那您就一个人了。

在深入研究规则之前，我们将介绍一个出现在整个Web页面中的概念，即null或空字符串的概念。

空字符串是没有值的字符数据。换句话说，它是空的。它写在awk像这样的程序： "" 。在外壳中，可以使用单引号或双引号将其写成： ""要么'' 。尽管空字符串中没有字符，但确实存在。例如，考虑以下命令：

$ echo ""

在这里echo实用程序将接收单个参数，即使该参数中没有字符也是如此。在本网页的其余部分，我们将术语空字符串和空字符串互换使用。现在，参考报价规则：

  • 引用项可以与未引用项以及其他引用项并置。Shell将所有内容变成该命令的一个参数。
  • 在任何单个字符前加反斜杠（' \ '）引用该字符。外壳程序删除反斜杠并将加引号的字符传递给命令。
  • 单引号可以保护起首和结束引号之间的所有内容。外壳程序不解释引用的文本，将其逐字传递给命令。不可能在单引号中嵌入单引号。请参考中的评论awk尝试以下操作的程序示例。
  • 双引号可以保护大多数内容，避免介于开引号和结束引号之间。Shell至少对引用的文本执行变量和命令替换。不同的外壳可能会对双引号文本进行其他类型的处理。

    由于双引号中的某些字符由外壳程序处理，因此必须在文本内对其进行转义 。值得注意的是字符“ $ '，' ` '，' \ '和' " '，如果要将它们直接传递给程序，则所有这些都必须在双引号文本内加反斜杠。（前面的反斜杠首先被去除。）因此，前面在运行中看到的示例awk没有输入文件 ：

    awk 'BEGIN { print "Don\47t Panic!" }'

    可以这样写：

    $ awk "BEGIN { print \"Don't Panic!\" }"
-| Don't Panic!

    请注意，单引号在双引号内并不特殊。

  • 当空字符串作为非空命令行参数的一部分出现时，会将空字符串删除，同时保留明确的空对象。例如，指定字段分隔符FS应该设置为空字符串，使用：
    awk -F "" 'program' files # correct

    不要使用此：

    awk -F"" 'program' files  # wrong!

    在第二种情况下awk尝试使用程序的文本作为FS ，第一个文件名就是程序的文本！这最多会导致语法错误，最糟糕的是会导致行为混乱。

混合单引号和双引号很困难。您必须诉诸shell引用技巧，如下所示：

$ awk 'BEGIN { print "Here is a single quote <'"'"'>" }'
-| Here is a single quote <'>

该程序由三个串联的带引号的字符串组成。第一个和第三个被单引号，第二个被双引号。

可以“简化”为：

$ awk 'BEGIN { print "Here is a single quote <'\''>" }'
-| Here is a single quote <'>

自己判断这两个中哪个更易读。

另一种选择是使用双引号，将嵌入式转义， awk级双引号：

$ awk "BEGIN { print \"Here is a single quote <'>\" }"
-| Here is a single quote <'>

此选项也很痛苦，因为在高级版本中双引号，反斜杠和美元符号很常见。 awk程式。

第三种选择是对单引号和双引号字符使用八进制转义序列等效项（请参见转义序列部分），如下所示：

$ awk 'BEGIN { print "Here is a single quote <\47>" }'
-| Here is a single quote <'>
$ awk 'BEGIN { print "Here is a double quote <\42>" }'
-| Here is a double quote <">

这很好用，但是您应该清楚地说明转义符的含义。

第四个选项是使用命令行变量分配，如下所示：

$ awk -v sq="'" 'BEGIN { print "Here is a single quote <" sq ">" }'
-| Here is a single quote <'>

（在这里，两个字符串常量和sq被连接成单个字符串，由print ）

如果您确实需要单引号和双引号awk程序，最好将其移动到一个单独的文件中，在该文件中shell不会成为图片的一部分，您可以说出您的意思。
