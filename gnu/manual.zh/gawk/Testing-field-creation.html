4.8检查方式gawk正在拆分记录

如我们所见， gawk提供三种独立的方法将输入记录拆分为字段。使用的机制基于以下三个变量中的哪一个： FS ， FIELDWIDTHS ， 要么FPAT —最后分配给。此外，API输入解析器可能会选择覆盖记录解析机制。请参阅自定义输入解析器以获取有关此功能的更多信息。

使用后恢复正常的场分割FIELDWIDTHS和/或FPAT ，只需将一个值分配给FS 。您可以使用 ' FS = FS '而不必知道当前值FS 。

为了确定哪种字段拆分有效，请使用PROCINFO["FS"] （请参阅传达信息的内置变量部分）。值是"FS"如果使用常规字段拆分， "FIELDWIDTHS"如果正在使用固定宽度的字段拆分，或者"FPAT"如果使用基于内容的字段拆分：

if (PROCINFO["FS"] == "FS")
    regular field splitting …
else if (PROCINFO["FS"] == "FIELDWIDTHS")
    fixed-width field splitting …
else if (PROCINFO["FS"] == "FPAT")
    content-based field splitting …
else
    API input parser field splitting … (advanced feature)

在编写需要临时更改的功能时，此信息很有用FS要么FIELDWIDTHS ，读取一些记录，然后恢复原始设置（有关此类功能的示例，请参阅“ 读取用户数据库 ”一节）。
