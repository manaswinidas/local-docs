17.4.3内存分配功能和便利宏

API提供了许多内存分配功能，用于分配可传递给的内存。 gawk ，以及许多便利宏。本小节以扩展代码使用它们的方式将它们全部展示为函数原型：

void *gawk_malloc(size_t size);

呼叫正确版本的malloc()分配可能传递给的存储gawk 。

void *gawk_calloc(size_t nmemb, size_t size);

呼叫正确版本的calloc()分配可能传递给的存储gawk 。

void *gawk_realloc(void *ptr, size_t size);

呼叫正确版本的realloc()分配可能传递给的存储gawk 。

void gawk_free(void *ptr);

呼叫正确版本的free()释放分配给gawk_malloc() ， gawk_calloc() ， 要么gawk_realloc() 。

API必须提供这些功能，因为可以将扩展编译并链接到与C库不同的C库版本。 gawk可执行文件。 104如果gawk将使用其版本free()当记忆来自一个不相关的版本时malloc() ，可能会导致意外行为。

可以使用三个便捷宏从gawk_malloc() ， gawk_calloc和gawk_realloc() 。如果分配失败，则会导致gawk退出并显示致命错误消息。应将它们当作没有返回值的过程调用来使用：

#define emalloc(pointer, type, size, message) …

该宏的参数如下：

pointer

指向已分配存储的指针变量。

type

指针变量的类型。这用于为以下对象的调用创建演员表gawk_malloc() 。

size

要分配的字节总数。

message

致命错误消息的前缀消息。通常，这是使用宏的函数的名称。

例如，您可以这样分配一个字符串值：

awk_value_t result;
char *message;
const char greet[] = "Don't Panic!";

emalloc(message, char *, sizeof(greet), "myfunc");
strcpy(message, greet);
make_malloced_string(message, strlen(message), & result);


#define ezalloc(pointer, type, size, message) …

就像emalloc() ，但它调用gawk_calloc()代替gawk_malloc() 。参数与emalloc()宏，但是此宏保证返回的内存被初始化为零。

#define erealloc(pointer, type, size, message) …

就像emalloc() ，但它调用gawk_realloc()代替gawk_malloc() 。参数与emalloc()宏。

另外两个函数分配MPFR和GMP对象，以供需要创建然后返回此类值的扩展函数使用：

void *get_mpfr_ptr();

分配并初始化MPFR对象，并返回指向它的指针。如果分配失败， gawk退出并出现致命的“内存不足”错误。如果gawk在没有MPFR支持的情况下被编译，调用此函数将导致致命错误。

void *get_mpz_ptr();

分配和初始化GMP对象，并返回指向它的指针。如果分配失败， gawk退出并出现致命的“内存不足”错误。如果gawk在没有MPFR支持的情况下被编译，调用此函数将导致致命错误。

这两个函数都返回' void * '，因为gawkapi.h头文件不应依赖于 （和 ，来自 ）。实际的返回值是类型mpfr_ptr和mpz_ptr分别，并且在将结果分配给正确类型的变量之前，应适当地转换返回值。
