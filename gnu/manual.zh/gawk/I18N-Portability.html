13.4.3 awk可移植性问题

gawk特意选择的国际化功能，以尽可能减少对awk将它们用于其他版本的程序awk 。考虑以下程序：

BEGIN {
    TEXTDOMAIN = "guide"
    if (Test_Guide)   # set with -v
        bindtextdomain("/test/guide/messages")
    print _"don't panic!"
}

按照书面规定，它不适用于其他版本的awk 。但是，它实际上几乎是可移植的，几乎不需要更改：

  • 分配给TEXTDOMAIN不会有任何影响，因为TEXTDOMAIN其他不特别awk实现。
  • 非GNU版本的awk将标记的字符串视为名为的变量的串联_后面跟着一个字符串。 93通常，变量_具有空字符串（ "" ）作为其值，并保留原始字符串常量作为结果。
  • 通过定义“虚拟”功能来替换dcgettext() ， dcngettext()和bindtextdomain() ， awk可以使程序运行，但是所有消息均以原始语言输出。例如： 
    function bindtextdomain(dir, domain)
{
    return dir
}

function dcgettext(string, domain, category)
{
    return string
}

function dcngettext(string1, string2, number, domain, category)
{
    return (number == 1 ? string1 : string2)
}
  • 在中使用位置规范printf要么sprintf() 不便携。支持gettext()在C级别，许多系统的C版本sprintf()支持位置说明符。但是，只有在函数调用中提供了足够的参数时，它才有效。许多版本awk通过printf格式和参数与基础C库版本相同sprintf() ，但一次只能使用一种格式和参数。如果使用位置规范，将会发生任何人的猜测。但是，因为位置规范主要用于转换后的格式字符串，并且因为非GNU awk永远不要检索转换后的字符串，这在实践中应该不是问题。
