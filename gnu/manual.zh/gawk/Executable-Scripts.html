1.1.4可执行文件awk程式

一旦学到awk ，您可能想写自成体系的awk脚本，使用“ #!脚本机制。您可以在许多系统上执行此操作。 8例如，您可以更新文件advice看起来像这样：

#! /bin/awk -f

BEGIN { print "Don't Panic!" }

使该文件可执行后（使用chmod实用程序），只需键入“ advice在外壳上，系统安排运行awk就像您键入“ awk -f advice '：

$ chmod +x advice
$ advice
-| Don't Panic!

（我们假设您在外壳程序的搜索路径变量中有当前目录，[通常是$PATH ]。如果不是，您可能需要输入“ ./advice ”）。

自给自足awk当您要编写一个用户可以调用而无需知道该程序是用什么编写的程序时，脚本很有用awk 。

了解“ #! '

awk是一种解释性语言。这意味着awk实用程序读取您的程序，然后根据程序中的说明处理数据。（这与诸如C之类的编译语言不同，在C语言中，程序首先被编译为机器代码，然后由系统处理器直接执行。）的awk实用程序因此被称为口译员 。许多现代语言得到了解释。

以“ #! '列出了要运行的解释程序的完整文件名，以及传递给该解释程序的单个可选的初始命令行参数。然后，操作系统使用给定的参数和已执行程序的完整参数列表来运行解释器。列表中的第一个参数是文件的完整文件名。 awk程序。参数列表的其余部分包含以下任一选项awk ，或数据文件或两者。（请注意，在许多系统上awk可能在/usr/bin而不是/bin ）

某些系统将解释器名称的长度限制为32个字符。通常，这可以通过使用符号链接来处理。

您不应在“ #!的路径后行awk 。这是行不通的。操作系统将该行的其余部分视为一个自变量，并将其传递给awk 。这样做会导致混乱的行为-最有可能是某种形式的使用情况诊断awk 。

最后，价值ARGV[0] （请参阅“ 预定义变量”部分）因您的操作系统而异。有些系统将“ awk '在那里，有些人输入了awk （如/bin/awk ），有些则输入了脚本的名称（' advice '）。 （直流）不要依赖于ARGV[0]提供您的脚本名称。
