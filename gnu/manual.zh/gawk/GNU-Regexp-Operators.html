3.7 gawk -特定的正则表达式运算符

处理正则表达式的GNU软件提供了许多其他的正则表达式运算符。这些运算符在本节中介绍，并且特定于gawk ;它们在其他地方不可用awk实现。大多数其他运算符都处理单词匹配。就我们的目的而言， 单词是一个或多个字母，数字或下划线（' _ '）：

\s

匹配任何空白字符。将其视为“ [[:space:]] ' 。

\S

匹配任何非空格字符。将其视为“ [^[:space:]] ' 。

\w

匹配任何单词组成的字符，即，它匹配任何字母，数字或下划线。将其视为“ [[:alnum:]_] ' 。

\W

匹配任何非单词构成的字符。将其视为“ [^[:alnum:]_] ' 。

\<

匹配单词开头的空字符串。例如， /\火柴 ' away ' 但不是 ' stowaway '。

\>

匹配单词末尾的空字符串。例如， /stow\>/火柴 ' stow ' 但不是 ' stowaway '。

\y

与单词（即单词boundy y ）的开头或结尾处的空字符串匹配。例如， ' \yballs?\y '匹配' ball ' 要么 ' balls '，作为一个单独的词。

\B

匹配出现在两个单词组成字符之间的空字符串。例如， /\Brat\B/火柴 ' crate '，但不匹配' dirty rat '。' \B '本质上与'相反\y '。

还有两个其他操作缓冲区的运算符。在Emacs中， 缓冲区自然是Emacs缓冲区。其他GNU程序，包括gawk ，将要匹配的整个字符串视为缓冲区。操作员是：

\`

匹配缓冲区开头的空字符串（字符串）

\'

匹配缓冲区末尾的空字符串（字符串）

因为^ '和' $ '始终根据字符串的开头和结尾工作，这些运算符不会为awk 。提供它们是为了与其他GNU软件兼容。

在其他GNU软件中，字边界运算符为' \b '。但是，这与awk语言对“ \b '作为退格键，因此gawk使用不同的字母。一种替代方法是在GNU运算符中要求两个反斜杠，但这被认为太混乱了。当前使用“ \y '为GNU' \b '似乎是两种邪恶中的较小者。

各种命令行选项（请参阅“ 命令行选项”一节）控制着gawk解释正则表达式中的字符：

没有选择

在默认情况下， gawk提供POSIX正则表达式的所有功能以及前面介绍的GNU正则表达式运算符。正则表达式运算符中描述的GNU regexp运算符 。

--posix

仅匹配POSIX正则表达式； GNU运算符并不特殊（例如，“ \w '匹配文字' w '）。允许使用间隔表达式。

--traditional

匹配传统的Unix awk正则表达式。GNU运算符不是特殊的，并且间隔表达式也不可用。因为BWK awk支持它们，POSIX字符类（' [[:alnum:]] '等）。由八进制和十六进制转义序列描述的字符将按字面意义处理，即使它们代表正则表达式元字符。

--re-interval

如果允许，则在正则表达式中允许使用间隔表达式--traditional已提供。否则，默认情况下间隔表达式可用。
