17.7.3接口fork() ， wait()和waitpid()

的fork扩展添加了三个功能，如下所示：

@load "fork"

这是您加载扩展程序的方式。

pid = fork()

此功能创建一个新的过程。返回值在子代中为零，在父代中子代的进程ID号，或者在错误时返回-1。在后一种情况下， ERRNO表示问题。在孩子里PROCINFO["pid"]和PROCINFO["ppid"]更新以反映正确的值。

ret = waitpid(pid)

此函数采用数字参数，这是要等待的进程ID。返回值是waitpid()系统调用。

ret = wait()

此功能等待第一个孩子死亡。返回值是wait()系统调用。

没有对应的exec()功能。

这是一个例子：

@load "fork"
…
if ((pid = fork()) == 0)
    print "hello from the child"
else
    print "hello from the parent"
