17.4.5.6定制的双向处理器

两路处理器将输入解析器和用于两路I / O的输出包装器与“ |& '运算符（请参阅重定向输出print和printf ）。它使用相同的awk_input_parser_t和awk_output_buf_t如前所述的结构。

双向处理器由以下结构表示：

typedef struct awk_two_way_processor {
    const char *name;   /* name of the two-way processor */
    awk_bool_t (*can_take_two_way)(const char *name);
    awk_bool_t (*take_control_of)(const char *name,
                                  awk_input_buf_t *inbuf,
                                  awk_output_buf_t *outbuf);
    awk_const struct awk_two_way_processor *awk_const next;  /* for gawk */
} awk_two_way_processor_t;

字段如下：

const char *name;

双向处理器的名称。

awk_bool_t (*can_take_two_way)(const char *name);

如果此字段指向的函数要接管此文件名的双向I / O，则应返回true。它不应在其内更改任何状态（变量值等） gawk 。

awk_bool_t (*take_control_of)(const char *name,
                              awk_input_buf_t *inbuf,
                              awk_output_buf_t *outbuf);

该字段指向的函数应填写awk_input_buf_t和awk_output_buf_t指向的结构inbuf和outbuf ， 分别。这些结构已在前面描述。

awk_const struct two_way_processor *awk_const next;

供...使用gawk ;因此它被标记awk_const因此扩展名无法对其进行修改。

与输入解析器和输出处理器一样，您提供“是的，我可以接受”和“为此而接受”功能， XXX_can_take_two_way()和XXX_take_control_of() 。

您使用以下功能注册双向处理器：

void register_two_way_processor(awk_two_way_processor_t *two_way_processor);

注册由指向的双向处理器two_way_processor与gawk 。
