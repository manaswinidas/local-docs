11.2.4将大文件拆分

的split程序将大型文本文件拆分为较小的部分。用法如下： 76 

split [-count] [file] [prefix]

默认情况下，输出文件的名称为xaa ， xab ， 等等。每个文件中都有1,000行，最后一个文件可能例外。要更改每个文件中的行数，请在命令行中提供一个带减号的数字（例如，“ -500 '用于其中包含500行而不是1,000行的文件）。要将输出文件的名称更改为类似myfileaa ， myfileab ，依此类推，提供一个额外的参数来指定文件名前缀。

这是一个版本split在awk 。它使用ord()和chr() 在字符和数字之间转换中提供的函数。

该程序首先设置其默认值，然后进行测试以确保没有太多参数。然后依次查看每个参数。第一个参数可以是减号，后跟数字。如果是这样，它恰好看起来像一个负数，因此将其设为正数，即行数。数据文件名被跳过，最后一个参数用作输出文件名的前缀：

# split.awk --- do split in awk
#
# Requires ord() and chr() library functions
# usage: split [-count] [file] [outname]

BEGIN {
    outfile = "x"    # default
    count = 1000
    if (ARGC > 4)
        usage()

    i = 1
    if (i in ARGV && ARGV[i] ~ /^-[[:digit:]]+$/) {
        count = -ARGV[i]
        ARGV[i] = ""
        i++
    }
    # test argv in case reading from stdin instead of file
    if (i in ARGV)
        i++    # skip datafile name

    if (i in ARGV) {
        outfile = ARGV[i]
        ARGV[i] = ""
    }

    s1 = s2 = "a"
    out = (outfile s1 s2)
}

下一条规则将完成大部分工作。 tcount （临时计数）跟踪到目前为止已在输出文件中打印了多少行。如果大于count ，该关闭当前文件并开始一个新文件了。 s1和s2跟踪文件名的当前后缀。如果它们都是' z '，文件太大。除此以外， s1移动到字母表中的下一个字母，然后s2从“ a '：

{
    if (++tcount > count) {
        close(out)
        if (s2 == "z") {
            if (s1 == "z") {
                printf("split: %s is too large to split\n",
                       FILENAME) > "/dev/stderr"
                exit 1
            }
            s1 = chr(ord(s1) + 1)
            s2 = "a"
        }

        else
            s2 = chr(ord(s2) + 1)

        out = (outfile s1 s2)
        tcount = 1
    }
    print > out
}

的usage()函数仅显示错误消息并退出：

function usage()
{
    print("usage: split [-num] [file] [outname]") > "/dev/stderr"
    exit 1
}

这个程序有点草率；它依靠awk自动关闭最后一个文件，而不是在END规则。它还假设字母在字符集中是连续的，这对于EBCDIC系统而言并非如此。
