6.1.1.2八进制和十六进制数

在awk ，所有数字均为小数（即以10为底）。许多其他编程语言允许您以其他基数指定数字，通常是八进制（基数8）和十六进制（基数16）。以八进制表示，数字为0、1、2、3、4、5、6、7、10、11、12，依此类推。就像 ' 11 '十进制为1乘以10加1，因此' 11 '八进制为1乘8加1。等于9（十进制）。十六进制为16位数字。因为每天的十进制数字系统只有十位数（' 0 '–' 9 '）， 这些信 ' a '通过' f '代表其余的。 （字母中的大小写通常是无关紧要的；十六进制' a '和' A '具有相同的值。）因此， 11十六进制的'是16的1乘以1，等于十进制的17。

只看普通的' 11 '，您无法分辨它的基础。因此，在C，C ++和其他衍生自C的语言中，有一种特殊的符号表示基础。八进制数字以“ 0 '，十六进制数字以前导' 0x ' 要么 ' 0X '：

11

十进制值11

011

八进制11，十进制值9

0x11

十六进制11，十进制值17

此示例显示了不同之处：

$ gawk 'BEGIN { printf "%d, %d, %d\n", 011, 11, 0x11 }'
-| 9, 11, 17

当处理不能方便地用字符或正数表示的数据（例如各种二进制数据）时，能够在程序中使用八进制和十六进制常量非常有用。

gawk允许在程序文本中使用八进制和十六进制常量。但是，输入数据中的这些数字没有区别。默认情况下这样做会破坏旧程序。（如果您确实需要这样做，请使用--non-decimal-data命令行选项；请参阅允许非十进制输入数据部分 。）如果您使用八进制或十六进制数据，则可以使用strtonum()函数（请参阅字符串操作函数一节）将数据转换为数字。大多数时候，使用内置的位操作函数时，您将希望使用八进制或十六进制常量。有关更多信息，请参见位操作功能 。

与某些早期的C实现不同，“ 8 '和' 9 '在八进制常量中无效。例如， gawk对待' 018 '作为十进制18：

$ gawk 'BEGIN { print "021 is", 021 ; print 018 }'
-| 021 is 17
-| 18

八进制和十六进制源代码常量是gawk延期。如果gawk处于兼容模式（请参阅“ 命令行选项”一节），它们不可用。

常数的基数不会影响其值

将数字常量内部转换为数字后， gawk不再记得常数的原始形式是什么；内部值始终被使用。这对于将数字转换为字符串有特殊的影响：

$ gawk 'BEGIN { printf "0x11 is <%s>\n", 0x11 }'
-| 0x11 is <17>
