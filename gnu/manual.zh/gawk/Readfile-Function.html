10.2.8一次读取整个文件

通常，将文件的全部内容作为单个字符串存储在内存中很方便。一个简单但天真的方法可能如下：

function readfile1(file,    tmp, contents)
{
    if ((getline tmp < file) < 0)
        return

    contents = tmp RT
    while ((getline tmp < file) > 0)
        contents = contents tmp RT

    close(file)
    return contents
}

该功能读取file一次一个记录，在局部变量中建立文件的全部内容contents 。它有效，但不一定有效。

根据Denis Shirokov的建议，以下功能可一次性读取命名文件的全部内容：

# readfile.awk --- read an entire file at once

function readfile(file,     tmp, save_rs)
{
    save_rs = RS
    RS = "^$"
    getline tmp < file
    close(file)
    RS = save_rs

    return tmp
}

它通过设置起作用RS至 ' ^$ '，如果文件包含内容，则该正则表达式将永远不匹配。 gawk从文件读取数据到tmp ，尝试匹配RS 。每次读取后匹配失败，但是很快失败，因此gawk填充tmp文件的全部内容。（有关以下信息，请参见如何将输入分为记录部分： RT和RS ）

在这种情况下file为空，返回值为空字符串。因此，调用代码可能使用类似以下内容的代码：

contents = readfile("/some/path")
if (length(contents) == 0)
    # file was empty …

这将测试结果以查看其是否为空。等效测试将是“ contents == "" '。

请参阅读取整个文件一节以获取扩展功能，该功能还将整个文件读入内存。
