B.1.3内容gawk分配

的gawk发行版包含许多C源文件，文档文件，子目录以及与配置过程相关的文件（请参阅编译和安装部分gawk在Unix-like Systems上 ），以及与不同的非Unix操作系统有关的几个子目录：

各种“ .c '，' .y '和' .h '个文件

这些文件包含实际的gawk源代码。

support/*

例程的C标头和源文件gawk用途，但这不是其核心功能的一部分。例如，参数解析，正则表达式匹配和随机数生成例程都保存在这里。

ABOUT-NLS

包含有关GNU信息的文件gettext和翻译。

AUTHORS

包含有关作者身份的一些信息的文件gawk 。它的存在仅是为了满足自由软件基金会的学徒们的要求。

README
README_d/README.*

描述性文件： README对于gawk在Unix下，其余部分用于各种硬件和软件组合。

INSTALL

一个文件，概述配置和安装过程。

ChangeLog

修正了错误或进行了改进的源代码更改的详细列表。

ChangeLog.0

源代码更改的旧列表。

NEWS

更改列表gawk自上一个版本或补丁以来。

NEWS.0

较旧的更改列表gawk 。

COPYING

GNU通用公共许可证。

POSIX.STD

POSIX标准中有关行为的描述awk没有定义的地方gawk可能不完全符合POSIX标准应描述但不完全符合的清单。

doc/awkforai.txt

指向描述原因的简短文章的原始草案的指针gawk是人工智能（AI）编程的一种好语言。

doc/bc_notes

的简要说明gawk的“字节码”内部。

doc/gawk.1

的troff描述手册页面的来源gawk 。为方便Unix用户而分发。

doc/gawktexi.in
doc/sidebar.awk

此网页的Texinfo源文件。应该由doc/sidebar.awk在处理之前texi2dvi要么texi2pdf产生印刷文件，并makeinfo产生一个Info或HTML文件。的Makefile负责此处理并通过以下方式产生可打印的输出texi2dvi要么texi2pdf 。

doc/gawk.texi

doc/gawk.info

为该网页生成的信息文件。

doc/gawkinet.texi

Texinfo源文件用于TCP / IP Internetworking与gawk 。应该使用TeX处理（通过texi2dvi要么texi2pdf ），以产生印刷文件并makeinfo产生一个Info或HTML文件。

doc/gawkinet.info

生成的TCP / IP Internetworking的信息文件gawk 。

doc/igawk.1

的troff手册页的来源，其中描述了igawk程序以“使用库函数的简便方法”展示。（以来gawk可以自己做@include都没有处理igawk也不igawk.1已安装。）

doc/it/*

该网页的意大利语翻译文件，由Antonio Colombo和Marco Curreli制作和提供。

doc/Makefile.in

的po库包含消息翻译。

awklib/extract.awk

awklib/eg/*

的awklib目录包含的副本extract.awk （请参阅从Texinfo源文件中提取程序一节），可用于从此Web页面的Texinfo源文件中提取示例程序。它还包含一个Makefile.in文件，其中configure用于生成一个Makefile 。 Makefile.am由GNU Automake用于创建Makefile.in 。从库函数库awk功能 ，作为现成的文件包含在gawk分配。它们是安装过程中的一部分。该网页中的其余程序可在以下位置的相应子目录中找到： awklib/eg 。

extension/*

随附的示例扩展的源代码，手册页和基础结构文件gawk 。请参阅写作扩展部分gawk ， 想要查询更多的信息。

extras/*

其他非必需文件。当前，此目录包含一些要在其中安装的Shell启动文件。 /etc/profile.d协助操纵AWKPATH和AWKLIBPATH环境变量。有关更多信息，请参见外壳启动文件部分。

posix/*

构建所需的文件gawk在符合POSIX的系统上。

pc/*

构建所需的文件gawk在MS-Windows下（有关详细信息，请参阅在MS-Windows上安装 ）。

vms/*

构建所需的文件gawk在Vax / VMS和OpenVMS下（请参阅编译和安装部分gawk在Vax / VMS和OpenVMS上了解详细信息）。

test/*

一个测试套件gawk 。您可以使用 ' make check '来自顶层gawk目录运行您的版本gawk针对测试套件。如果gawk成功通过make check '，那么您可以对成功的移植充满信心。
