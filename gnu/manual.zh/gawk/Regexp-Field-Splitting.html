4.5.2使用正则表达式分隔字段

上一小节讨论了使用单个字符或简单字符串作为的值FS 。更普遍地讲， FS可以是包含任何正则表达式的字符串。在这种情况下，正则表达式的记录中的每个匹配项都将字段分开。例如，分配：

FS = ", \t"

将由逗号，空格和TAB组成的输入行的每个区域转换为字段分隔符。

对于正则表达式的普通示例，请尝试使用单个空格来分隔使用单个逗号的方式的字段。 FS可以设置为"[ ]" （左括号，空格，右括号）。这正则表达式匹配一个空格，没有别的（见部分正则表达式 ）。

这两种情况之间存在重要区别FS = " " '（单个空格）和' FS = "[ \t\n]+" '（与一个或多个空格，TAB或换行符匹配的正则表达式）。对于两个值FS ，字段由空格，TAB和/或换行符的运行 （多个相邻出现）分隔。但是，当FS是" " ， awk首先从记录中去除开头和结尾的空白，然后确定字段在哪里。例如，以下管道会打印“ b '：

$ echo ' a b c d ' | awk '{ print $2 }'
-| b

但是，此管道会打印“ a '（注意每个字母周围的多余空格）：

$ echo ' a  b  c  d ' | awk 'BEGIN { FS = "[ \t\n]+" }
>                                  { print $2 }'
-| a

在这种情况下，第一个字段为null或为空。

无论何时，剥离前导空格和尾随空格也起作用$0重新计算。例如，研究以下管道：

$ echo '   a b c d' | awk '{ print; $2 = $2; print }'
-|    a b c d
-| a b c d

首先print语句在读取时打印记录，并且前导空格保持不变。分配给$2重建$0通过串联$1通过$NF在一起，由OFS （默认情况下为空格）。因为查找时前导空格被忽略$1 ，它不是新的一部分$0 。最后，最后print语句打印新的$0 。

使用正则表达式进行字段拆分时，还需要注意其他一些细微之处。在POSIX标准或其他任何地方都没有很好地指定， ^ '表示在拆分字段时。“ ^ '仅在整个记录的开头匹配？还是每个字段分隔符都是一个新字符串？原来是不同的awk版本对这个问题的回答不同，您不应依赖程序中的任何特定行为。（直流）

作为信息点，BWK awk允许“ ^ '仅在记录的开头匹配。gawk也可以这样例如：

$ echo 'xxAA  xxBxx  C' |
> gawk -F '(^x+)|( +)' '{ for (i = 1; i <= NF; i++)
>                             printf "-->%s<--\n", $i }'
-| --><--
-| -->AA<--
-| -->xxBxx<--
-| -->C<--
