12.4使用gawk用于网络编程

除了能够打开通往同一系统上的协同处理程序的双向管道（请参阅与另一个进程的双向通信部分）之外，还可以通过IP与另一个系统上的另一个进程建立双向连接。网络连接。

您可以将其视为通往协同处理的非常长的双向管道。方式gawk通过识别以以下其中一个开头的特殊文件名来确定您要使用TCP / IP网络/inet/ '，' /inet4/ '， 要么 ' /inet6/ '。

特殊文件名的完整语法为/net-type/protocol/local-port/remote-host/remote-port 。这些组件是：

net-type

指定要建立的Internet连接类型。使用 ' /inet4/ '强制使用IPv4，以及' /inet6/以强制使用IPv6。普通的/inet/ '（以前是唯一的选项）使用系统默认值，很可能是IPv4。

protocol

通过IP使用的协议。这必须是“ tcp '， 要么 ' udp '，分别用于TCP或UDP IP连接。TCP应该用于大多数应用程序。

local-port

要使用的本地TCP或UDP端口号。使用端口号“ 0 '当您希望系统选择端口时。这是编写TCP或UDP客户端时应该执行的操作。您也可以使用众所周知的服务名称，例如“ smtp ' 要么 ' http '， 在这种情况下gawk尝试使用C确定预定义的端口号getaddrinfo()功能。

remote-host

您要连接的Internet主机的IP地址或标准域名。

remote-port

给定要使用的TCP或UDP端口号remote-host 。同样，使用“ 0 '（如果您不在乎），否则将使用知名的服务名称。

注意：无法打开双向插座会导致非致命错误返回到调用代码。的价值ERRNO指示错误（请参阅传达信息的内置变量部分）。

考虑以下非常简单的示例：

BEGIN {
    Service = "/inet/tcp/0/localhost/daytime"
    Service |& getline
    print $0
    close(Service)
}

该程序从本地系统的TCP读取当前日期和时间daytime服务器。然后打印结果并关闭连接。

因为这个主题很广泛，所以使用gawk用于TCP / IP编程的内容另行记录。请参阅TCP / IP与gawk ，这是gawk发行，更全面的介绍和讨论以及大量示例。

注意： gawk只能打开直接插座。当前无法访问通过安全套接字层（SSL）提供的服务。这包括其URL开头为“ https:// '。
