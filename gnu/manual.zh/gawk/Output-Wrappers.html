17.4.5.5定制输出包装器

输出包装器是输入解析器的镜像。它允许扩展将输出转移到使用' > ' 要么 ' >> I / O重定向运算符（请参见重定向输出print和printf ）。

输出包装器与输入解析器的结构非常相似：

typedef struct awk_output_wrapper {
    const char *name;   /* name of the wrapper */
    awk_bool_t (*can_take_file)(const awk_output_buf_t *outbuf);
    awk_bool_t (*take_control_of)(awk_output_buf_t *outbuf);
    awk_const struct awk_output_wrapper *awk_const next;  /* for gawk */
} awk_output_wrapper_t;

成员如下：

const char *name;

这是输出包装器的名称。

awk_bool_t (*can_take_file)(const awk_output_buf_t *outbuf);

这指向一个功能，可以检查awk_output_buf_t指向的结构outbuf 。如果输出包装器要接管文件，则应返回true，否则返回false。它不应在其内更改任何状态（变量值等） gawk 。

awk_bool_t (*take_control_of)(awk_output_buf_t *outbuf);

此字段指向的函数在以下情况下被调用gawk决定让输出包装器控制文件。它应该填写适当的成员awk_output_buf_t结构，如下所述，如果成功则返回true，否则返回false。

awk_const struct output_wrapper *awk_const next;

供...使用gawk ;因此它被标记awk_const因此扩展名无法对其进行修改。

的awk_output_buf_t结构看起来像这样：

typedef struct awk_output_buf {
    const char *name;   /* name of output file */
    const char *mode;   /* mode argument to fopen */
    FILE *fp;           /* stdio file pointer */
    awk_bool_t redirected;  /* true if a wrapper is active */
    void *opaque;       /* for use by output wrapper */
    size_t (*gawk_fwrite)(const void *buf, size_t size, size_t count,
                FILE *fp, void *opaque);
    int (*gawk_fflush)(FILE *fp, void *opaque);
    int (*gawk_ferror)(FILE *fp, void *opaque);
    int (*gawk_fclose)(FILE *fp, void *opaque);
} awk_output_buf_t;

在这里，您的扩展程序也会定义XXX_can_take_file()和XXX_take_control_of()检查和更新数据成员的功能awk_output_buf_t 。数据成员如下：

const char *name;

输出文件的名称。

const char *mode;

模式字符串（如在第二个参数中使用的fopen() ）打开文件。

FILE *fp;

的FILE来自的指针 。 gawk在尝试找到输出包装器之前先打开文件。

awk_bool_t redirected;

该字段必须由XXX_take_control_of()功能。

void *opaque;

该指针对gawk 。该扩展名应使用它来存储指向与该文件关联的任何私有数据的指针。

size_t (*gawk_fwrite)(const void *buf, size_t size, size_t count,
                      FILE *fp, void *opaque);
int (*gawk_fflush)(FILE *fp, void *opaque);
int (*gawk_ferror)(FILE *fp, void *opaque);
int (*gawk_fclose)(FILE *fp, void *opaque);

这些指针应设置为指向执行与如果需要，函数可以执行。 gawk将这些函数指针用于所有输出。 gawk初始化指针以指向仅调用常规函数的内部“传递”函数功能，因此扩展仅需要重新定义适合其功能的那些功能。

的XXX_can_take_file()功能应基于name和mode字段以及任何其他状态（例如awk变量值）。

什么时候gawk来电XXX_take_control_of() ，该函数应适当填写其他字段，但fp ，应该正常使用。

您可以使用以下功能注册输出包装器：

void register_output_wrapper(awk_output_wrapper_t *output_wrapper);

注册由指向的输出包装器output_wrapper与gawk 。
