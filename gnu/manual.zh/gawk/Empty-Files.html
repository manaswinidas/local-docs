10.3.4检查零长度文件

都知道awk实现静默跳过零长度文件。这是awk的针对规则的隐式读取记录和匹配循环：何时awk尝试从空文件中读取记录，它立即收到文件结束指示，关闭文件，然后继续执行下一个命令行数据文件， 而不执行任何用户级操作awk程序代码。

使用gawk的ARGIND变量（请参见预定义变量部分），可以检测何时跳过了空数据文件。与“ 注意数据文件边界”中提供的库文件类似，以下库文件调用名为zerofile()用户必须提供的。传递的参数是文件名和位置ARGV找到的地方：

# zerofile.awk --- library file to process empty input files

BEGIN { Argind = 0 }

ARGIND > Argind + 1 {
    for (Argind++; Argind < ARGIND; Argind++)
        zerofile(ARGV[Argind], Argind)
}

ARGIND != Argind { Argind = ARGIND }

END {
    if (ARGIND > Argind)
        for (Argind++; Argind <= ARGIND; Argind++)
            zerofile(ARGV[Argind], Argind)
}

用户级变量Argind允许awk跟踪其进度的程序ARGV 。只要程序检测到ARGIND大于' Argind + 1 '，表示跳过了一个或多个空文件。该动作然后调用zerofile()对于每个这样的文件，递增Argind一路上。

' Argind != ARGIND规则只是保持Argind在正常情况下是最新的。

最后， END rule捕获命令行参数末尾任何空文件的情况。注意，测试条件为for循环使用' <= '运算符，不是' < '。
