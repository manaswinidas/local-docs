4.10.5使用getline从管道

Omniscience有很多值得推荐的地方。否则，关注细节将很有用。

— 布赖恩·克尼根 （ Brian Kernighan）

命令的输出也可以通过管道传递到getline ，使用“ command | getline '。在这种情况下，字符串command作为shell命令运行，其输出通过管道传递到awk用作输入。这种形式的getline从管道一次读取一个记录。例如，以下程序将其输入复制到其输出，但以'开头的行除外@execute '，将其替换为通过将命令行的其余部分作为shell命令运行而产生的输出：

{
     if ($1 == "@execute") {
          tmp = substr($0, 10)        # Remove "@execute"
          while ((tmp | getline) > 0)
               print
          close(tmp)
     } else
          print
}

的close()调用函数以确保两个相同的' @execute '行出现在输入中，每执行一次该命令。请参阅“ 关闭输入和输出重定向”部分 。给定输入：

foo
bar
baz
@execute who
bletch

该程序可能会产生：

foo
bar
baz
arnold     ttyv0   Jul 13 14:22
miriam     ttyp0   Jul 13 14:23     (murphy:0)
bill       ttyp1   Jul 13 14:23     (murphy:0)
bletch

请注意，该程序运行了命令who并打印结果。（如果您亲自尝试该程序，则当然会得到不同的结果，具体取决于登录系统的人。）

这种变化getline将记录拆分为字段，设置值NF ，并重新计算的值$0 。的值NR和FNR没有改变。 RT被设置。

根据POSIX，“ expression | getline如果是if昧的expression包含除“ $ '-例如， ' "echo " "date" | getline '是不明确的，因为串联运算符未加括号。您应该将其写为“ ("echo " "date") | getline '如果您想让程序可移植到所有人awk实现。

注意：不幸的是， gawk在处理类似“ "echo " "date" | getline '。大多数版本（包括当前版本）都将其视为“ ("echo " "date") | getline '。（这也是BWK awk表现。）某些版本将其视为' "echo " ("date" | getline) '。（这是mawk表现。）简而言之，请始终使用显式括号，这样您就不必担心。
