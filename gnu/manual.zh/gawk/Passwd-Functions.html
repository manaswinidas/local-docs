10.5读取用户数据库

的PROCINFO数组（请参阅“ 预定义变量”部分）提供对当前用户的真实有效用户和组ID号以及（如果可用）用户辅助组集的访问权限。但是，由于这些是数字，因此它们无法为普通用户提供非常有用的信息。需要找到某种方式来查找与用户和组ID号关联的用户信息。本节介绍了用于从用户数据库检索信息的一组功能。有关从组数据库检索信息的类似套件，请参阅“ 读取组数据库 ”一节。

POSIX标准未定义保存用户信息的文件。相反，它提供了头文件和一些用于获取用户信息的C语言子例程。主要功能是getpwent() ，表示“获取密码”。“密码”来自原始用户数据库文件， /etc/passwd ，它存储用户信息以及加密的密码（因此而得名）。

虽然一个awk程序可以简单地阅读/etc/passwd直接，此文件可能不包含有关系统用户集的完整信息。 73为确保您能够生成可读且完整的用户数据库版本，有必要编写一个小型的C程序来调用getpwent() 。getpwent()定义为返回指向struct passwd 。每次调用它时，它都会返回数据库中的下一个条目。没有更多条目时，它将返回NULL空指针。发生这种情况时，C程序应调用endpwent()关闭数据库。以下是pwcat ，一个“捕获”密码数据库的C程序：

/*
 * pwcat.c
 *
 * Generate a printable version of the password database.
 */
#include <stdio.h>
#include <pwd.h>

int
main(int argc, char **argv)
{
    struct passwd *p;

    while ((p = getpwent()) != NULL)
        printf("%s:%s:%ld:%ld:%s:%s:%s\n",
            p->pw_name, p->pw_passwd, (long) p->pw_uid,
            (long) p->pw_gid, p->pw_gecos, p->pw_dir, p->pw_shell);

    endpwent();
    return 0;
}

如果您不了解C，请不必担心。来自的输出pwcat是传统的用户数据库/etc/passwd以冒号分隔的字段的格式。字段是：

登录名

用户的登录名。

加密密码

用户的加密密码。在某些系统上可能不可用。

用户身份

用户的数字用户ID号。（在某些系统上，它是C long ，而不是int 。因此，我们将其转换为long对于所有情况。）

组ID

用户的数字组ID号。（关于long与int在这里申请。）

全名

用户的全名，以及与该用户关联的其他信息。

主目录

用户的登录（或“主”）目录（shell程序员熟悉为$HOME ）。

登录外壳

用户登录时运行的程序。通常是一个外壳，例如Bash。

通过该介绍，以下是获取用户信息的一组功能。这里有几个函数，对应于相同名称的C函数：

# passwd.awk --- access password file information

BEGIN {
    # tailor this to suit your system
    _pw_awklib = "/usr/local/libexec/awk/"
}

function _pw_init(    oldfs, oldrs, olddol0, pwcat, using_fw, using_fpat)
{
    if (_pw_inited)
        return

    oldfs = FS
    oldrs = RS
    olddol0 = $0
    using_fw = (PROCINFO["FS"] == "FIELDWIDTHS")
    using_fpat = (PROCINFO["FS"] == "FPAT")
    FS = ":"
    RS = "\n"

    pwcat = _pw_awklib "pwcat"
    while ((pwcat | getline) > 0) {
        _pw_byname[$1] = $0
        _pw_byuid[$3] = $0
        _pw_bycount[++_pw_total] = $0
    }
    close(pwcat)
    _pw_count = 0
    _pw_inited = 1
    FS = oldfs
    if (using_fw)
        FIELDWIDTHS = FIELDWIDTHS
    else if (using_fpat)
        FPAT = FPAT
    RS = oldrs
    $0 = olddol0
}

的BEGIN规则将私有变量设置到目录，其中pwcat被储存了。因为它是用来帮助awk库例程，我们选择将其放入/usr/local/libexec/awk ;但是，您可能希望它位于系统上的其他目录中。

功能_pw_init()将用户信息的三个副本填充到三个关联数组中。数组按用户名索引（ _pw_byname ），按用户ID号（ _pw_byuid ），并按出现的顺序（ _pw_bycount ）。变量_pw_inited用于提高效率，因为_pw_init()只需要调用一次。

因为此功能使用getline从中读取信息pwcat ，它首先保存的值FS ， RS和$0 。它在变量中指出using_fw是否用FIELDWIDTHS是否有效。这样做是必要的，因为可以从用户程序中的任何位置调用这些功能，并且用户可以使用自己的方式来拆分记录和字段。这样可以在以后恢复正确的场分割机制。该测试仅适用于gawk 。如果使用则为假FS要么FPAT或其他awk实施。

检查使用的代码FPAT ，使用using_fpat和PROCINFO["FS"] ， 类似。

该函数的主要部分使用循环读取数据库行，将行拆分为字段，然后根据需要将行存储到每个数组中。循环完成后， _pw_init() 通过关闭管道进行清理，设置_pw_inited 到一个，并还原FS （和FIELDWIDTHS要么FPAT如有必要）， RS和$0 。指某东西的用途_pw_count 简短说明。

的getpwnam()函数将用户名作为字符串参数。如果该用户在数据库中，则返回相应的行。否则，它将依赖对不存在的元素的数组引用来创建以空字符串作为其值的元素：

function getpwnam(name)
{
    _pw_init()
    return _pw_byname[name]
}

同样， getpwuid()函数采用用户ID号参数。如果该用户号在数据库中，它将返回相应的行。否则，它返回空字符串：

function getpwuid(uid)
{
    _pw_init()
    return _pw_byuid[uid]
}

的getpwent()函数只是简单地遍历数据库，一次只有一个条目。它用_pw_count跟踪其当前位置_pw_bycount数组：

function getpwent()
{
    _pw_init()
    if (_pw_count < _pw_total)
        return _pw_bycount[++_pw_count]
    return ""
}

的endpwent() 功能重置_pw_count 为零，以便后续调用getpwent()从头再来：

function endpwent()
{
    _pw_count = 0
}

该套件中一个有意识的设计决策是每个子例程调用_pw_init() 初始化数据库数组。仅当用户的主程序实际调用这些功能之一时，才会产生运行单独的进程来生成用户数据库以及进行扫描的I / O的开销。如果此库文件与用户程序一起加载，但没有任何例程被调用过，则不会产生额外的运行时开销。（替代方法是移动_pw_init() 变成一个BEGIN规则，始终运行pwcat 。这简化了代码，但运行了可能永远不需要的额外过程。）

反过来，打电话_pw_init()不太贵，因为_pw_inited变量可防止程序多次读取数据。如果您担心要挤出最后一个周期awk程序，检查_pw_inited可以移出_pw_init()并在所有其他功能中重复使用。实际上，这不是必需的，因为大多数awk程序是受I / O约束的，这样的更改会使代码混乱。

的id 打印出用户信息中的程序使用这些功能。
