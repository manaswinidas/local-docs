16.5具有的任意精度整数运算gawk

当给出-M选项， gawk使用GMP任意精度整数执行所有整数算术运算。在源文件或数据文件中，任何看起来像整数的数字都将存储为任意精度的整数。整数的大小仅受可用内存的限制。例如，以下计算5 4 3 2 ，其结果超出了普通硬件双精度浮点值的限制：

$ gawk -M 'BEGIN {
>   x = 5^4^3^2
>   print "number of digits =", length(x)
>   print substr(x, 1, 20), "...", substr(x, length(x) - 19, 20)
> }'
-| number of digits = 183231
-| 62060698786608744707 ... 92256259918212890625

相反，如果您要使用任意精度的浮点值来计算相同的值，那么正确输出所需的精度（使用公式' prec = 3.322 * dps '）为3.322 x 183231或608693。

具有整数和浮点值的算术运算的结果是精度等于工作精度的浮点值。以下程序使用递归计算Sylvester序列101中的第八项：

$ gawk -M 'BEGIN {
>   s = 2.0
>   for (i = 1; i <= 7; i++)
>       s = s * (s - 1) + 1
>   print s
> }'
-| 113423713055421845118910464

输出与实际数字113,423,713,055,421,844,361,000,443有所不同，因为默认的53位精度不足以准确表示浮点结果。您可以提高精度（在这种情况下100位就足够了），也可以替换浮点常量' 2.0 '与一个整数，以使用整数算术执行所有计算以获取正确的输出。

有时gawk必须将任意精度整数隐式转换为任意精度浮点值。这主要是因为MPFR库并不总是提供相关的接口来按操作或函数的需要处理任意精度的整数或混合模式数字。在这种情况下，将精度设置为精确转换所需的最小值，并且为此不使用工作精度。如果这不是您所需要的，则可以使用一个替代方法，然后将整数首先转换为浮点，如下所示：

gawk -M 'BEGIN { n = 13; print (n + 0.0) % 2.0 }'

您可以通过将数字指定为浮点值开头来完全避免此问题：

gawk -M 'BEGIN { n = 13.0; print n % 2.0 }'

请注意，对于此特定示例，最好仅使用以下内容：

gawk -M 'BEGIN { n = 13; print n % 2 }'

将两个任意精度整数除以' / ' 要么 ' % '，结果通常是任意精度的浮点值（除非分母均匀地分为分子）。
