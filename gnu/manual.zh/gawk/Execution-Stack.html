14.3.4使用堆栈

每当您运行包含任何函数调用的程序时， gawk维护所有导致当前程序所在的函数调用的堆栈。您可以看到自己到达的位置，还可以在堆栈中四处移动，以查看调用您所在的函数的状态。用于执行此操作的命令是：

backtrace [ count ]
bt [ count ]
where [ count ]

打印所有函数调用（堆栈帧）或最里面的回溯count框架如果count > 0。打印最外面count框架如果count <0。回溯显示每个函数的名称和参数，源文件名和行号。别名where对于backtrace为可能习惯该命令的GDB长期用户提供。

down [ count ]

移动count （默认1）使堆栈向下朝最里面的帧方向移动。然后选择并打印框架。

frame [ n ]
f [ n ]

选择并打印堆叠框架n 。第0帧是当前正在执行或最内部的 （函数调用）；框架1是称为最里面的框架。编号最高的帧是主程序的帧。打印的信息包括帧号，函数和参数名称，源文件和源行。

up [ count ]

移动count （默认值为1）将堆栈朝最外面的帧向上对齐。然后选择并打印框架。
