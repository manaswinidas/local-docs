4.10.1使用getline没有参数

的getline命令可以不带参数使用，以从当前输入文件读取输入。在这种情况下，它所做的全部工作就是读取下一个输入记录，并将其拆分为多个字段。如果你处理完目前的战绩，但想要做下记录一些特殊的处理，现在这是有用的。例如：

# Remove text between /* and */, inclusive
{
    if ((i = index($0, "/*")) != 0) {
        out = substr($0, 1, i - 1)  # leading part of the string
        rest = substr($0, i + 2)    # ... */ ...
        j = index(rest, "*/")       # is */ in trailing part?
        if (j > 0) {
            rest = substr(rest, j + 2)  # remove comment
        } else {
            while (j == 0) {
                # get more text
                if (getline <= 0) {
                    print("unexpected EOF or error:", ERRNO) > "/dev/stderr"
                    exit
                }
                # build up the line using string concatenation
                rest = rest $0
                j = index(rest, "*/")   # is */ in trailing part?
                if (j != 0) {
                    rest = substr(rest, j + 2)
                    break
                }
            }
        }
        # build up the output line using string concatenation
        $0 = out rest
    }
    print $0
}

这个awk程序删除C样式的注释（' /* … */ '）。它使用了许多我们尚未介绍的功能，包括字符串连接（请参阅String Concatenation部分）和index()和substr()内置函数（请参阅“ 字符串操作函数”一节）。通过替换“ print $0与其他语句一起使用，您可以对已分解的输入执行更复杂的处理，例如搜索正则表达式的匹配项。（此程序有一个细微的问题-如果一个注释结束而另一个注释在同一行开始，则该程序将不起作用。）

这种形式的getline命令集NF ， NR ， FNR ， RT ，以及$0 。

注意：的新值$0用于测试任何后续规则的模式。原始值$0触发了执行的规则getline迷路了。相比之下， next语句读取新记录，但立即从程序中的第一条规则开始正常进行处理。见节的next声明 。
