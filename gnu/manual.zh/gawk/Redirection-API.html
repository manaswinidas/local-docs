17.4.12访问和操纵重定向

以下功能允许扩展访问和操纵重定向。

awk_bool_t get_file(const char *name,
                    size_t name_len,
                    const char *filetype,
                    int fd,
                    const awk_input_buf_t **ibufp,
                    const awk_output_buf_t **obufp);

查找文件name在gawk的内部重定向表。如果name是NULL要么name_len为零，返回当前打开的输入文件对应的数据FILENAME 。（这无法访问filetype参数，因此可能不确定）。如果文件尚未打开，请尝试将其打开。的filetype参数必须以零结尾，并且应该是以下之一：

">"

打开一个文件进行输出。

">>"

打开一个文件进行追加。

"<"

打开一个文件以供输入。

"|>"

打开输出管道。

"|<"

打开用于输入的管道。

"|&"

双向协同处理。

错误时返回awk_false 。否则，返回awk_true ，并在中返回有关重定向的其他信息ibufp和obufp指针。

对于输入重定向， *ibufp值应为非NULL和*obufp应该NULL 。对于输出重定向， *obufp值应为非NULL和*ibufp应该NULL 。对于双向协过程，两个值均应为非NULL 。

在通常情况下，扩展名感兴趣(*ibufp)->fd和/或fileno((*obufp)->fp) 。如果文件尚未打开，并且fd论点是非否定的gawk将使用该文件描述符，而不是以通常的方式打开文件。如果fd是非负数，但文件已经存在， gawk忽略fd并返回现有文件。呼叫者有责任注意fd在返回awk_input_buf_t也不是fd在返回awk_output_buf_t与要求的值相符。

请注意，管道当前不支持提供文件描述符。但是，提供文件描述符应该适用于输入，输出，附加和双向（协同处理）套接字。如果filetype是双向的gawk假设它是一个插座！请注意，在双向情况下，输入和输出文件描述符可能会有所不同。要检查是否成功，必须检查两者是否匹配。

预计该API函数将用于实现I / O复用和套接字库。
