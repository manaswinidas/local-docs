9.2.3.3通过值或引用传递函数参数

在awk ，当您声明函数时，无法明确声明参数是通过value还是由reference传递。

取而代之的是，传递约定是在运行时在调用函数时根据以下规则确定的：如果参数是数组变量，则它通过引用传递。否则，该参数将按值传递。

通过值传递参数意味着在调用函数时，会为其提供此参数值的副本 。调用者可以使用变量作为参数的表达式，但是被调用函数不知道这一点-它仅知道参数具有什么值。例如，如果您编写以下代码：

foo = "bar"
z = myfunc(foo)

那么你不应该想到myfunc()作为“变量foo 。”而是将参数视为字符串值"bar" 。如果功能myfunc()更改其局部变量的值，这对任何其他变量都没有影响。因此，如果myfunc()做这个：

function myfunc(str)
{
   print str
   str = "zzz"
   print str
}

更改其第一个参数变量str ，它不会改变的价值foo在呼叫者中。的作用foo在打电话myfunc()当其值（ "bar" ）进行了计算。如果str也存在于myfunc() ，函数体无法更改此外部值，因为它在执行过程中被阴影myfunc()并且无法从那里看到或更改。

但是，如果数组是函数的参数，则不会复制它们。取而代之的是，该数组本身可用于该函数的直接操作。这通常称为按引用调用 。给功能的主体内的阵列参数作出的改变可见之外的功能。

注意：如果不注意正在执行的操作，则在函数内部更改数组参数可能非常危险。例如：

function changeit(array, ind, nvalue)
{
     array[ind] = nvalue
}

BEGIN {
    a[1] = 1; a[2] = 2; a[3] = 3
    changeit(a, 2, "two")
    printf "a[1] = %s, a[2] = %s, a[3] = %s\n",
            a[1], a[2], a[3]
}

打印' a[1] = 1, a[2] = two, a[3] = 3 '，因为changeit()商店"two"在第二个要素中a 。
