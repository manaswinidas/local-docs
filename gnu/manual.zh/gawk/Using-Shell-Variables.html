7.2在程序中使用Shell变量

awk程序通常用作用Shell编写的较大程序的组件。例如，使用shell变量来保存awk程序搜索。有两种方法可以将shell变量的值输入到主体中awk程序。

一种常见的方法是使用外壳引号将变量的值替换为脚本内的程序。例如，考虑以下程序：

printf "Enter search pattern: "
read pattern
awk "/$pattern/ "'{ nmatches++ }
     END { print nmatches, "found" }' /path/to/data

的awk程序由两段引号组成，这两段引号串联在一起构成程序。第一部分用双引号引起来，它允许替换pattern引号内的shell变量。第二部分是单引号。

通过引用进行变量替换有效，但可能会造成混乱。它需要对Shell的引用规则有充分的了解（请参阅Shell Quoting Issues一节），并且在阅读程序时通常很难正确地匹配引用。

更好的方法是使用awk的变量分配功能（请参阅命令行上的分配变量部分），以将shell变量的值分配给awk变量。然后使用动态正则表达式来匹配模式（请参阅使用动态正则部分）。下面显示了如何使用此技术重做上一个示例：

printf "Enter search pattern: "
read pattern
awk -v pat="$pattern" '$0 ~ pat { nmatches++ }
       END { print nmatches, "found" }' /path/to/data

现在awk程序只是一个单引号字符串。分配 ' -v pat="$pattern" '仍需要双引号，以防万一$pattern 。的awk变量pat可以命名pattern也可以，但这会更加令人困惑。使用变量还提供了更大的灵活性，因为该变量可以在程序内部的任何位置使用-用于打印，作为数组下标或用于任何其他用途-无需在程序的每个点都使用引号。
