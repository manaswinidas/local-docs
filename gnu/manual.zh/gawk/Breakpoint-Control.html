14.3.1断点控制

正如我们前面所看到的，您可能希望在调试会话中要做的第一件事就是设置断点，因为否则程序将像不在调试器下一样运行。用于控制断点的命令是：

break [[ filename: ] n | function ] [ "expression" ]
b [[ filename: ] n | function ] [ "expression" ]

不带任何参数的情况下，请在要选择的堆栈帧中执行的下一条指令处设置断点。参数可以是以下之一：

n

在行号上设置断点n在当前源文件中。

filename:n

在行号上设置断点n在源文件中filename 。

function

在函数（的第一条指令）的入口处设置断点function 。

每个断点都分配有一个编号，该编号可使用以下命令从断点列表中删除delete命令。

使用断点，您还可以提供条件。这是一awk达到断点时调试器将评估的表达式（用双引号引起来）。如果条件为真，则调试器将停止执行并提示输入命令。否则，它将继续执行程序。

clear [[ filename: ] n | function ]

不带任何参数的情况下，删除要在所选堆栈帧中执行的下一条指令处的任何断点。如果程序在断点处停止，则会删除该断点，以使程序不会再次在该位置处停止。参数可以是以下之一：

n

删除行号设置的断点n在当前源文件中。

filename:n

删除行号设置的断点n在源文件中filename 。

function

删除在功能入口处设置的断点function 。

condition n "expression"

向现有断点或监视点添加条件n 。条件是awk 用双引号括起来的表达式，调试器在到达断点或监视点时将对其求值。如果条件为真，则调试器将停止执行并提示输入命令。否则，调试器将继续执行程序。如果未指定条件表达式，则将删除任何现有条件（即，使断点或监视点成为无条件的）。

delete [ n1 n2 …] [ n – m ]
d [ n1 n2 …] [ n – m ]

删除指定的断点或一系列断点。如果未提供任何参数，则删除所有定义的断点。

disable [ n1 n2 …| n – m ]

禁用指定的断点或一定范围的断点。不带任何参数的情况下，禁用所有断点。

enable [ del | once ] [ n1 n2 …] [ n – m ]
e [ del | once ] [ n1 n2 …] [ n – m ]

启用指定的断点或一系列断点。不带任何参数的情况下，启用所有断点。（可选）您可以指定如何启用断点：

del

临时启用断点，然后在程序停止时将其删除。

once

临时启用断点，然后在程序停止时禁用每个断点。

ignore n count

忽略断点号n下一个count次被击中。

tbreak [[ filename: ] n | function ]
t [[ filename: ] n | function ]

设置一个临时断点（仅启用一站式）。参数与for相同break 。
