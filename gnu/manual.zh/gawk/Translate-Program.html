11.3.3音译字符

系统tr实用程序将字符音译。例如，它通常用于将大写字母映射为小写字母以进行进一步处理：

generate data | tr 'A-Z' 'a-z' | process data …

tr需要两个字符列表。 78在处理输入时，将第一列表中的第一个字符替换为第二列表中的第一个字符，将第一列表中的第二个字符替换为第二列表中的第二个字符，依此类推。如果“从”列表中的字符多于“至”列表中的字符，则“至”列表的最后一个字符用于“从”列表中的其余字符。

曾几何时，用户建议将音译功能添加到gawk 。编写以下程序来证明可以使用用户级功能来进行字符音译。该程序不如系统完整tr实用程序，但可以完成大部分工作。

的translate程序是很久以前写的gawk获得了将字符串中的每个字符拆分为单独的数组元素的功能。因此，它重复使用了substr() ， index()和gsub()内置函数（请参阅“ 字符串操作函数”一节）。有两个功能。首先， stranslate() ，采用三个参数：

from

要翻译的字符列表

to

待翻译字符列表

target

进行翻译的字符串

关联数组使翻译部分相当容易。 t_ar包含“ to”字符，由“ from”字符索引。然后经过一个简单的循环from ，一次只输入一个字符。对于中的每个字符from ，如果字符出现在target ，它被替换为相应的to字符。

的translate()函数调用stranslate() ，使用$0作为目标。主程序设置两个全局变量， FROM和TO ，从命令行，然后进行更改ARGV以便awk从标准输入读取。

最后，处理规则只需调用translate()对于每条记录：

# translate.awk --- do tr-like stuff
# Bugs: does not handle things like tr A-Z a-z; it has
# to be spelled out. However, if `to' is shorter than `from',
# the last character in `to' is used for the rest of `from'.

function stranslate(from, to, target,     lf, lt, ltarget, t_ar, i, c,
                                                               result)
{
    lf = length(from)
    lt = length(to)
    ltarget = length(target)
    for (i = 1; i <= lt; i++)
        t_ar[substr(from, i, 1)] = substr(to, i, 1)
    if (lt < lf)
        for (; i <= lf; i++)
            t_ar[substr(from, i, 1)] = substr(to, lt, 1)
    for (i = 1; i <= ltarget; i++) {
        c = substr(target, i, 1)
        if (c in t_ar)
            c = t_ar[c]
        result = result c
    }
    return result
}

function translate(from, to)
{
    return $0 = stranslate(from, to, $0)
}

# main program
BEGIN {

    if (ARGC < 3) {
        print "usage: translate from to" > "/dev/stderr"
        exit
    }

    FROM = ARGV[1]
    TO = ARGV[2]
    ARGC = 2
    ARGV[1] = "-"
}

{
    translate(FROM, TO)
    print
}

可以在用户级功能中进行字符音译，但不一定高效，因此我们（ gawk开发人员）开始考虑添加内置功能。但是，在编写此程序后不久，我们得知Brian Kernighan添加了toupper()和tolower()对他的功能awk （请参阅“ 字符串操作函数”一节）。这些功能处理了大多数需要字符音译的情况，因此我们选择将这些功能添加到gawk以及然后保持足够的孤独。

此程序的一个明显改进是建立了t_ar在一个数组中只排列一次BEGIN规则。但是，这假设“发件人”和“发件人”列表在程序的整个生命周期内都不会改变。

另一个明显的改进是启用了范围，例如“ a-z '，如tr效用。看一下代码cut.awk （请参阅“ 切出字段和列”一节 ）以获取灵感。
