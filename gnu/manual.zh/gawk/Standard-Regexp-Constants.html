6.1.2.1标准正则表达式常量

在“ ~ ' 要么 ' !~ '运算符，一个regexp常量仅代表要匹配的regexp。但是，正则表达式常量（例如/foo/ ）可以像简单表达式一样使用。当一个正则表达式常量本身出现时，它的含义与它以某种模式出现的含义相同（即，“ ($0 ~ /foo/) '）。（直流）请参阅将表达式作为模式一节。这意味着以下两个代码段：

if ($0 ~ /barfly/ || $0 ~ /camelot/)
    print "found"

和：

if (/barfly/ || /camelot/)
    print "found"

完全相同。该规则的一个相当奇怪的结果是，以下布尔表达式是有效的，但没有达到其作者可能想要的目的：

# Note that /foo/ is on the left of the ~
if (/foo/ ~ $1) print "found foo"

此代码“显然”正在测试$1与正则表达式匹配/foo/ 。但实际上，“ /foo/ ~ $1 ' 真正意思 ' ($0 ~ /foo/) ~ $1 '。换句话说，首先将输入记录与正则表达式匹配/foo/ 。结果是零或一，取决于匹配的成功与否。然后将该结果与记录中的第一个字段进行匹配。因为您不太可能真正想要进行这种测试， gawk在程序中看到此构造时会发出警告。该规则的另一个结果是赋值语句：

matches = /foo/

为变量分配零或一matches ，具体取决于当前输入记录的内容。

常量正则表达式也用作gensub() ， sub()和gsub()功能，作为match()函数，并作为split()和patsplit()函数（请参阅“ 字符串操作函数”一节）。的现代实现awk ， 包含gawk ，允许第三个参数split()是一个正则表达式常量，但某些较旧的实现则不是。（直流）因为某些内置函数接受regexp常量作为参数，所以在尝试将regexp常量用作用户定义函数的参数时可能会引起混淆（请参阅“ 用户定义的函数”一节）。例如：

function mysub(pat, repl, str, global)
{
    if (global)
        gsub(pat, repl, str)
    else
        sub(pat, repl, str)
    return str
}



{
    …
    text = "hi! hi yourself!"
    mysub(/hi/, "howdy", text, 1)
    …
}

在此示例中，程序员希望将regexp常量传递给用户定义的函数mysub() ，进而将其传递给任一sub()要么gsub() 。但是，真正发生的是pat根据参数是否为参数分配一个值为1或0的值$0火柴/hi/ 。 gawk当看到正则表达式常量用作用户定义函数的参数时，发出警告，因为以这种方式传递真值可能不是预期的。
