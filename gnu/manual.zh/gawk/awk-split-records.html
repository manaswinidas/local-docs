4.1.1使用标准记录拆分awk

记录由称为记录分隔符的字符分隔 。默认情况下，记录分隔符是换行符。这就是为什么记录默认为单行的原因。要为记录分隔符使用其他字符，只需将该字符分配给预定义变量RS 。

像其他任何变量一样， RS可以在awk与赋值运算符“ = '（请参阅分配表达式部分）。新的记录分隔符应用引号引起来，该引号表示字符串常量。通常，正确的时间是在执行开始之前，即处理任何输入之前，以便使用适当的分隔符读取第一条记录。为此，请使用特殊BEGIN图案（见部分的BEGIN和END特殊模式 ）。例如：

awk 'BEGIN { RS = "u" }
     { print $0 }' mail-list

改变价值RS至 ' u '，然后再读取任何输入。新值是一个字符串，其第一个字符为字母“ u”；结果，记录由字母“ u”分隔。然后读取输入文件，并在awk程序（无模式的动作）将打印每条记录。因为每个print语句在其输出末尾添加一个换行符，这awk程序将输入与每个' u更改为换行符。这是在上运行程序的结果mail-list ：

请注意，名称“ Bill '未拆分。在原始数据文件中（请参见示例的数据文件部分），该行如下所示：

Bill         555-1675     bill.drowning@hotmail.com       A

它不包含“ u '，因此没有理由拆分记录，与其他记录不同，每个记录都有一个或多个' u '。实际上，该记录被视为先前记录的一部分；在输出中将它们分隔开的换行符是数据文件中的原始换行符，而不是由awk当它打印记录时！

更改记录分隔符的另一种方法是在命令行上使用变量分配功能（请参阅“ 其他命令行参数”一节 ）：

awk '{ print $0 }' RS="u" mail-list

这套RS至 ' u '处理之前mail-list 。

使用字母字符，例如“ u对于记录分隔符，极有可能产生奇怪的结果。使用不寻常的字符，例如“ /在大多数情况下，更有可能产生正确的行为，但并不能保证。道德是：知道您的数据。

gawk允许RS成为完整的正则表达式（稍后讨论；请参阅“使用gawk ）。即使这样，也可以使用正则表达式元字符，例如' . '作为值中的单个字符RS没有特殊效果：按字面意义对待。为了与两个Unix向后兼容，这是必需的awk以及POSIX。

当使用常规字符作为记录分隔符时，会发生一种异常情况gawk与POSIX完全兼容（请参阅“ 命令行选项”部分 ）。然后，以下（极端）管道将打印出令人惊讶的“ 1 '：

$ echo | gawk --posix 'BEGIN { RS = "a" } ; { print NF }'
-| 1

有一个字段，由换行符组成。内置变量的值NF是当前记录中的字段数。（在正常情况下， gawk将换行符视为空白，打印' 0 ' 作为结果。其他大多数版本awk也可以这样。）

即使输入文件中的最后一个字符不是其中的字符，到达输入文件的末尾也会终止当前输入记录。 RS 。（直流）

空字符串"" （不含任何字符的字符串）的特殊含义是RS 。这意味着记录由一个或多个空行分隔，仅此而已。有关更多详细信息，请参见“ 多行记录”部分。

如果您更改值RS在中间awk运行时，新值用于定界后续记录，但是当前正在处理的记录以及已经处理的记录不受影响。

确定记录结束后， gawk设置变量RT匹配输入中的文本RS 。
