16.4.1.3错误累积

在单次使用浮点数进行计算时，准确性损失通常不足以令人担忧。但是，如果计算的值是一系列浮点运算的结果，则该错误会累积并极大地影响计算本身。这是尝试使用pi的许多序列表示之一来计算pi的值：

BEGIN {
    x = 1.0 / sqrt(3.0)
    n = 6
    for (i = 1; i < 30; i++) {
        n = n * 2.0
        x = (sqrt(x * x + 1) - 1) / x
        printf("%.15f\n", n * x)
    }
}

运行时，早期错误会在以后的计算中传播，导致循环在尝试除零后过早终止：

$ gawk -f pi.awk
-| 3.215390309173475
-| 3.159659942097510
-| 3.146086215131467
-| 3.142714599645573
…
-| 3.224515243534819
-| 2.791117213058638
-| 0.000000000000000
error→ gawk: pi.awk:6: fatal: division by zero attempted

这是另一个示例，其中内部表示形式的不正确会产生意外的结果：

$ gawk 'BEGIN {
>   for (d = 1.1; d <= 1.5; d += 0.1)    # loop five times (?)
>       i++
>   print i
> }'
-| 4
