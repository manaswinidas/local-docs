5.7标准预打开数据流的特殊文件

通常，运行程序具有三个输入和输出流，它们已经可供读取和写入。这些被称为标准输入 ， 标准输出和标准错误输出 。这些打开的流（以及任何其他打开的文件或管道）通常由技术术语文件描述符来引用。

默认情况下，这些流已连接到您的键盘和屏幕，但通常会通过“ < '，' << '，' > '，' >> '，' >& '和' | '运算符。标准错误通常用于编写错误消息。之所以有两个独立的流，即标准输出和标准错误，是为了使它们可以分别重定向。

在传统的实现中awk ，是将错误消息写入标准错误的唯一方法awk程序如下：

print "Serious error detected!" | "cat 1>&2"

这是通过打开指向shell命令的管道来进行的，该命令可以访问它从命令行继承的标准错误流。 awk处理。这远非优雅，而且还需要一个单独的过程。所以人们写作awk程序通常不这样做。而是将错误消息发送到屏幕，如下所示：

print "Serious error detected!" > "/dev/tty"

（ /dev/tty是操作系统提供的特殊文件，已连接到键盘和屏幕。它代表“终端” 26 ，在现代系统中，它是键盘和屏幕，而不是串行控制台。）这通常具有相同的效果，但并非总是如此：尽管标准错误流通常是屏幕，但可以将其重定向；发生这种情况时，写入屏幕不正确。实际上，如果awk是从后台作业运行的，它可能根本没有终端。然后开/dev/tty失败。

gawk ，BWK awk和mawk提供用于访问三个标准流的特殊文件名。如果文件名与以下特殊名称之一匹配，则： gawk （或其中之一）重定向输入或输出，然后直接使用文件名所代表的描述符。这些特殊文件名适用于所有gawk已被移植到不仅仅是POSIX兼容的那些：

/dev/stdin

标准输入（文件描述符0）。

/dev/stdout

标准输出（文件描述符1）。

/dev/stderr

标准错误输出（文件描述符2）。

使用这些工具，编写错误消息的正确方法将变为：

print "Serious error detected!" > "/dev/stderr"

请注意在文件名周围使用引号。与其他任何重定向一样，该值必须是字符串。省略引号是一个常见错误，这会导致结果混乱。

gawk在POSIX兼容模式下，不会将这些文件名视为特殊文件名。但是，因为BWK awk支持他们， gawk确实支持它们，即使使用--traditional选项（请参阅“ 命令行选项”部分 ）。
