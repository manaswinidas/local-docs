3.8区分大小写

在匹配普通字符（即不是元字符）和括号内的表达式时，大小写通常在正则表达式中很重要。因此， w “在正则表达式中仅匹配小写字母” w “而不是大写” W '。

进行不区分大小写的匹配的最简单方法是使用方括号表达式-例如，“ [Ww] '。但是，如果您需要经常使用它，可能会很麻烦，并且这会使正则表达式更难以阅读。您可能有两种选择。

在程序的特定位置执行不区分大小写的匹配的一种方法是使用以下命令将数据转换为单个大小写tolower()要么toupper()内置的字符串函数（我们尚未讨论；请参阅“ 字符串操作函数”一节）。例如：

tolower($1) ~ /foo/  { … }

在与第一个字段匹配之前将其转换为小写。这适用于任何POSIX兼容awk 。

另一种方法，特定于gawk ，是设置变量IGNORECASE设置为非零值（请参阅预定义变量部分）。什么时候IGNORECASE不为零， 所有正则表达式和字符串操作均忽略大小写。

改变的价值IGNORECASE在程序运行时动态控制程序是否区分大小写。默认情况下，大小写是重要的，因为IGNORECASE （像大多数变量一样）初始化为零：

x = "aB"
if (x ~ /ab/) …   # this test will fail

IGNORECASE = 1
if (x ~ /ab/) …   # now it will succeed

通常，您不能使用IGNORECASE使某些规则不区分大小写，而其他规则不区分大小写，因为没有直接的方法来设置IGNORECASE仅用于特定规则的模式。 17为此，请使用方括号表达式或tolower() 。但是，您可以做的一件事IGNORECASE唯一的优点是一次动态地打开或关闭所有规则的区分大小写。

IGNORECASE可以在命令行或BEGIN规则（请参阅“ 其他命令行参数”部分；另请参见“ 启动和清除操作”部分 ）。设置IGNORECASE通过命令行可以使程序不区分大小写，而无需对其进行编辑。

在多字节语言环境中，根据语言环境字符集的宽字符值测试大写和小写字符之间的等效性。在版本5.0之前，基于ISO-8859-1（ISO Latin-1）字符集测试了单字节字符。但是，从5.0版开始，还将根据区域设置的字符集的值来测试单字节字符。 18岁
