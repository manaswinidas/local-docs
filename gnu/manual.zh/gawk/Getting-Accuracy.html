16.4.2获得所需的精度

可以使用任意精度算术得出准确的结果吗？没有简单的答案。使用浮点算术时，标准的代数规则通常不适用。除其他外，分布和关联定律并不完全成立，操作顺序对于您的计算可能很重要。舍入误差，累积精度损失和下溢通常很麻烦。

什么时候gawk测试表达式' 0.1 + 12.2 '和' 12.3 '对于使用机器双精度算术的相等性，它确定它们不相等！（请参阅小心比较值部分 。）您可以通过提高精度来获得所需的结果。在这种情况下有56位可以完成这项工作：

$ gawk -M -v PREC=56 'BEGIN { print (0.1 + 12.2 == 12.3) }'
-| 1

如果增加更多的位是好的，也许增加更多的精度更好？如果我们使用更大的值，会发生什么PREC ：

$ gawk -M -v PREC=201 'BEGIN { print (0.1 + 12.2 == 12.3) }'
-| 0

这不是错误gawk或在MPFR库中。容易忘记，用于存储值的有限位数通常只是经过适当舍入后的近似值。当且仅当两个操作数中的所有位完全相同时，相等性测试才会成功。因为在具有特定精度和有效舍入模式的浮点计算之后，这不一定是正确的，所以对相等性的直接测试可能不起作用。而是比较两个数字以查看它们是否在期望的增量内。

在小数点后十五位或更少的应用中，硬件双精度算术就足够了，通常更快。但是您需要记住，每个浮点运算都会遭受新的舍入误差，并带来灾难性的后果，正如我们之前计算pi值的尝试所说明的那样。在这种情况下，额外的精度可以极大地提高计算的稳定性和准确性。

此外，您应该理解重复加法不一定等于浮点算术中的乘法。在错误累积示例中：

$ gawk 'BEGIN {
>   for (d = 1.1; d <= 1.5; d += 0.1)    # loop five times (?)
>       i++
>   print i
> }'
-| 4

您可能会或可能不会通过选择任意大的值来成功获得正确的结果PREC 。在这种情况下，重新制定当前的问题通常是正确的方法。
