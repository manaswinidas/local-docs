6.2.2字符串连接

这似乎是一个好主意。

— 布赖恩·克尼根 （ Brian Kernighan）

只有一个字符串操作：串联。它没有特定的运算符来表示它。而是通过不使用运算符将表达式彼此相邻地写入来执行连接。例如：

$ awk '{ print "Field number one: " $1 }' mail-list
-| Field number one: Amelia
-| Field number one: Anthony
…

'之后的字符串常量中没有空格: '，这条线连在一起。例如：

$ awk '{ print "Field number one:" $1 }' mail-list
-| Field number one:Amelia
-| Field number one:Anthony
…

因为字符串连接没有显式运算符，所以通常需要使用括号将要连接的项目括起来，以确保它在正确的时间发生。例如，您可能期望以下代码片段串联在一起file和name ：

file = "file"
name = "name"
print "something meaningful" > file name

这在某些版本的Unix中产生语法错误awk 。 31必须使用以下内容：

print "something meaningful" > (file name)

除最常见的情况外，所有其他情况下都应在连接周围使用括号，例如“ = '。请注意字符串连接中使用的表达式种类。特别是，用于连接的表达式的求值顺序在awk语言。考虑以下示例：

BEGIN {
    a = "don't"
    print (a " " (a = "panic"))
}

未定义第二次分配是否a发生在值之前或之后a检索以产生级联值。结果可能是“ don't panic '， 要么 ' panic panic '。

当与其他运算符混合使用时，串联的优先级通常是违反直觉的。考虑以下示例：

$ awk 'BEGIN { print -12 " " -24 }'
-| -12-24

这“显然”是将-12，一个空格和-24连接在一起。但是空间消失在哪里了？答案在于运算符优先级和awk的自动转换规则。要获得所需的结果，请以这种方式编写程序：

$ awk 'BEGIN { print -12 " " (-24) }'
-| -12 -24

这种力量awk对待' -在“ -24一元。否则，其解析如下：

    -12 (" " - 24)
⇒ -12 (0 - 24)
⇒ -12 (-24)
⇒ -12-24

如前所述，将串联与其他运算符混合时，请加上括号 。否则，您永远不确定会得到什么。
