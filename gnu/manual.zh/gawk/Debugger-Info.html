14.3.5获取有关程序和调试器状态的信息

除了查看变量的值之外，通常还需要获取其他有关程序状态和调试环境本身的信息。的gawk调试器有一个提供此信息的命令，适当地称为info 。info与许多参数之一一起使用，这些参数可以准确告诉您您想知道的内容：

info what
i what

的价值what应该是以下之一：

args

列出所选框架的参数。

break

列出所有当前设置的断点。

display

列出自动显示列表中的所有项目。

frame

提供所选堆栈框架的描述。

functions

列出所有功能定义，包括源文件名和行号。

locals

列出所选框架的局部变量。

source

打印当前源文件的名称。每次程序停止时，当前源文件就是包含当前指令的文件。调试器首次启动时，当前源文件是通过-f选项。' list filename:lineno可以随时使用'命令更改当前源。

sources

列出所有程序源。

variables

列出所有全局变量。

watch

列出监视列表中的所有项目。

其他命令使您可以控制调试器，保存调试器状态以及从文件运行调试器命令的能力。这些命令是：

option [ name [ =value ]]
o [ name [ =value ]]

不带参数的情况下，显示可用的调试器选项及其当前值。 ' option name '显示命名选项的当前值。 ' option name=value '为命名选项分配新值。可用的选项有：

history_size

设置要保留在历史记录文件中的最大行数./.gawk_history 。默认值为100。

listsize

指定行数list打印。默认值为15。

outfile

发送gawk输出到文件；调试器输出仍将转为标准输出。空字符串（ "" ）将输出重置为标准输出。

prompt

更改调试器提示。默认值为“ gawk>  '。

save_history [ on | off ]

将命令历史记录保存到文件./.gawk_history 。默认是on 。

save_options [ on | off ]

将当前选项保存到文件./.gawkrc退出时。默认是on 。启动后，选项将读回到下一个会话。

trace [ on | off ]

打开或关闭指令跟踪。默认是off 。

save filename

将命令从当前会话保存到给定的文件名，以便可以使用source命令。

source filename

从文件运行命令；任何命令中的错误不会终止后续命令的执行。评论（以“ # '）可以在命令文件中使用。空行将被忽略；他们不重复最后一条命令。您不能通过拥有多个来重启程序run文件中的命令。另外，命令列表可能包括其他内容source命令;然而gawk为了避免无限递归，调试器不会多次重复获取同一文件。

除了或代替source命令，您可以使用-D file要么--debug=file命令行选项，以非交互方式从文件执行命令（请参阅“ 命令行选项”一节）。
