11.2.5将输出复制到多个文件

的tee该程序被称为“管道配件”。 tee将其标准输入复制到其标准输出，并将其复制到命令行上命名的文件。其用法如下：

tee [-a] file …

的-a选项告诉tee附加到命名文件，而不是截断它们并重新开始。

的BEGIN规则首先将所有命令行参数的副本复制到名为copy 。 ARGV[0]不需要，因此不会被复制。 tee不能用ARGV直接，因为awk尝试处理中的每个文件名ARGV作为输入数据。

如果第一个参数是-a ，然后是flag变量append设置为true，并且两者ARGV[1]和copy[1]被删除。如果ARGC小于两个，则没有提供文件名，并且tee打印用法消息并退出。最后， awk通过设置强制读取标准输入ARGV[1]至"-"和ARGC到两个：

# tee.awk --- tee in awk
#
# Copy standard input to all named output files.
# Append content if -a option is supplied.
#
BEGIN {
    for (i = 1; i < ARGC; i++)
        copy[i] = ARGV[i]

    if (ARGV[1] == "-a") {
        append = 1
        delete ARGV[1]
        delete copy[1]
        ARGC--
    }
    if (ARGC < 2) {
        print "usage: tee [-a] file ..." > "/dev/stderr"
        exit 1
    }
    ARGV[1] = "-"
    ARGC = 2
}

以下单个规则可以完成所有工作。因为没有模式，所以对输入的每一行都执行该模式。规则的正文只是将行打印到命令行上的每个文件中，然后打印到标准输出中：

{
    # moving the if outside the loop makes it run faster
    if (append)
        for (i in copy)
            print >> copy[i]
    else
        for (i in copy)
            print > copy[i]
    print
}

也可以这样编写循环：

for (i in copy)
    if (append)
        print >> copy[i]

    else
        print > copy[i]

这更加简洁，但是效率也较低。' if对每个记录和每个输出文件进行测试。通过复制循环主体，“ if每个输入记录仅测试一次。如果有N输入记录和M输出文件，第一个方法仅执行N ' if '语句，而第二条语句执行N*M ' if '语句。

最后， END规则通过关闭所有输出文件来清理：

END {
    for (i in copy)
        close(copy[i])
}
