5.9关闭输入和输出重定向

如果使用相同的文件名或相同的shell命令getline在执行过程中不止一次awk程序（请参见“ 显式输入与getline ），则仅在第一次打开文件（或执行命令）。那时，从该文件或命令中读取输入的第一条记录。下次使用相同的文件或命令getline ，从中读取另一个记录，依此类推。

同样，当打开文件或管道进行输出时， awk记住与之关联的文件名或命令，并且对同一文件或命令的后续写入将附加到先前的写入中。文件或管道保持打开状态，直到awk退出。

这意味着必须采取特殊步骤才能从头开始再次读取同一文件或重新运行Shell命令（而不是从同一命令读取更多输出）。的close()函数使这些事情成为可能：

close(filename)

要么：

close(command)

争论filename要么command可以是任何表达式。其值必须与用于打开文件或启动命令的字符串完全匹配（包括空格和其他“不相关”字符）。例如，如果您使用以下方法打开管道：

"sort -r names" | getline foo

那么您必须使用以下命令将其关闭：

close("sort -r names")

一旦执行此函数调用，下一个getline从该文件或命令，或下一个print要么printf到该文件或命令，然后重新打开该文件或重新运行该命令。因为用于关闭文件或管道的表达式必须与用于打开文件或运行命令的表达式完全匹配，所以最好使用变量来存储文件名或命令。前面的示例如下：

sortcom = "sort -r names"
sortcom | getline foo

…
close(sortcom)

这有助于避免您难以发现的印刷错误awk程式。这是关闭输出文件的一些原因：

  • 写入文件并稍后再读回awk程序。写入后关闭文件，然后开始阅读getline 。
  • 连续写入多个文件awk程序。如果文件没有关闭，最终awk一个进程中打开的文件数可能超过系统限制。最好在程序完成编写后将其关闭。
  • 完成命令。通过管道重定向输出时，只要管道是打开的，读取管道的命令通常会继续尝试读取输入。通常，这意味着该命令只有在关闭管道后才能真正完成其工作。例如，如果输出重定向到mail程序，直到关闭管道才真正发送该消息。
  • 使用相同的参数再次运行同一程序。这与为第一次运行提供更多输入不一样！

    例如，假设有一个程序将输出传递给mail程序。如果它输出几行重定向到该管道而不关闭它，则它们会发出一条包含几行的消息。相反，如果程序在输出的每一行之后关闭管道，则每一行都会发出一条单独的消息。

如果您使用的文件数量超出系统允许的数量， gawk尝试在数据文件之间多路复用可用的打开文件。 gawk能否执行此操作取决于您操作系统的功能，因此它可能并不总是有效。因此，始终使用它既是良好实践又是良好的可移植性建议close()处理完文件后，将它们放在文件上。实际上，如果您使用大量管道，则必须在完成后关闭命令。例如，考虑这样的事情：

{
    …
    command = ("grep " $1 " /some/file | my_prog -q " $3)
    while ((command | getline) > 0) {
        process output of command
    }
    # need close(command) here
}

本示例根据每个记录中的数据创建一个新管道。无需致电close()在评论中指出， awk创建子进程来运行命令，直到最终耗尽文件描述符以获取更多管道。

即使每个命令都已完成（如文件末尾从getline ），子进程不会终止； 27更重要的是，直到关闭管道的文件描述符才将其关闭并释放close()被称为或awk退出。

close()如果给出的参数不代表通过重定向打开的文件，管道或协同进程，则无提示地不执行任何操作。在这种情况下，它将返回负值，表示错误。此外， gawk套ERRNO表示错误的字符串。

另请注意， close(FILENAME) '对通过命令行命名的文件进行读取的隐式循环没有“魔术”效果。很可能关闭了从未通过重定向打开过的文件，因此awk默默地执行任何操作，除了返回负值。

使用“ |&为了与协过程进行通信，有时能够关闭双向管道的一端而不关闭另一端很有用。这是通过提供第二个参数来完成的close() 。与其他任何呼叫一样close() ，第一个参数是用于启动协同进程的命令或特殊文件的名称。第二个参数应该是一个字符串，具有两个值之一"to"要么"from" 。大小写无所谓。由于这是一项高级功能，因此讨论被推迟到与另一进程进行双向通信为止，后者将对此进行更详细的描述并给出一个示例。

command = "…"
command | getline info
retval = close(command)  # syntax error in many Unix awks
