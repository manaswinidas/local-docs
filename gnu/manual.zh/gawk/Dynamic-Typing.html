9.2.5函数及其对变量键入的影响

awk是一种非常流畅的语言。它可能是awk直到运行时才能知道标识符是代表标量变量还是数组。这是一个带注释的示例程序：

function foo(a)
{
    a[1] = 1   # parameter is an array
}

BEGIN {
    b = 1
    foo(b)  # invalid: fatal type mismatch

    foo(x)  # x uninitialized, becomes an array dynamically
    x = 1   # now not allowed, runtime error
}

在此示例中，第一个调用foo()产生致命错误，因此awk将不会报告第二个错误。但是，如果您注释掉该呼叫，则awk确实报告第二个错误。

通常，此类问题并不是大问题，但值得警惕。
