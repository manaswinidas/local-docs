1.1.6.1在MS-Windows批处理文件中报价

尽管此网页通常只担心POSIX系统和POSIX Shell，但对于许多用户而言，经常会出现以下问题，值得解决。

Microsoft Windows系统上的“外壳”使用双引号字符进行引用，并使得很难或不可能在命令行脚本中包含转义的双引号字符。下面的示例由Jeroen Brink提供，展示了如何从这个衬里脚本中转义双引号，该脚本将文件中的所有行打印在双引号包围的文件中：

{ print "\"" $0 "\"" }

在MS-Windows命令行中，可以按以下方式传递上面的单脚本：

gawk "{ print \"\042\" $0 \"\042\" }" file

在此示例中， \042 '是双引号的八进制代码； gawk将其转换为实双引号，以供print声明。

在MS-Windows中，转义双引号有点棘手，因为您使用反斜杠来转义双引号，但是反斜杠本身不会以通常的方式转义。实际上，它们是否重复，取决于是否有后续的双引号。用于双引号的MS-Windows规则如下：

  1. 对于原始字符串中的每个双引号，让N是前面的反斜杠数， N可能为零。更换这些N反斜杠加2 * N +1反斜杠
  2. 让N是在原始字符串后面加反斜杠的数量， N可能为零。更换这些N反斜杠加2 * N反斜杠
  3. 用双引号将结果字符串引起来。

所以要单引号脚本加双引号{ print "\"" $0 "\"" } '在上一个示例中，您可以这样做：

gawk "{ print \"\\\"\" $0 \"\\\"\" }" file

但是，使用“ \042 ' 代替 ' \\\"也是可能的，而且更容易阅读，因为没有反引号的反斜杠不需要重复。
