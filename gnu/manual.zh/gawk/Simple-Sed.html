11.3.8一个简单的流编辑器

的sed Utility是一个流编辑器 ，它是一个读取数据流，对其进行更改并将其传递的程序。它通常用于对大型文件或命令管道生成的数据流进行全局更改。虽然sed它本身就是一个复杂的程序，它最普遍的用途是在管道中间执行全局替换：

command1 < orig.data | sed 's/old/new/g' | command2 > result

这里， ' s/old/new/g告诉sed寻找正则表达式' old在每个输入行上，并将其全局替换为文字' new '（即一行中所有出现的事件）。这类似于awk的gsub()函数（请参阅“ 字符串操作函数”一节）。

以下程序， awksed.awk ，至少接受两个命令行参数：要查找的模式和用其替换的文本。任何其他参数都被视为要处理的数据文件名。如果未提供，则使用标准输入：

# awksed.awk --- do s/foo/bar/g using just print
#    Thanks to Michael Brennan for the idea

function usage()
{
    print "usage: awksed pat repl [files...]" > "/dev/stderr"
    exit 1
}


BEGIN {
    # validate arguments
    if (ARGC < 3)
        usage()


    RS = ARGV[1]
    ORS = ARGV[2]

    # don't use arguments as files
    ARGV[1] = ARGV[2] = ""
}


# look ma, no hands!
{
    if (RT == "")
        printf "%s", $0
    else
        print
}

该程序依赖gawk的能力RS是一个正则表达式，以及RT到终止记录的实际文本（请参见如何将输入拆分为记录部分 ）。

这个想法是有RS成为要寻找的模式。gawk自动设置$0模式匹配之间的文本。这是我们要保留的未经修改的文本。然后，通过设置ORS替换文字，简单print语句输出我们要保留的文本，然后输出替换文本。

这种方案有一个缺点，那就是如果最后一个记录没有以匹配的文本结尾，该怎么办RS 。用一个print语句无条件打印替换文本，这是不正确的。但是，如果文件未以匹配的文本结尾RS ， RT设置为空字符串。在这种情况下，我们可以打印$0使用printf （请参见使用部分printf特殊印刷声明 ）。

的BEGIN规则处理设置，检查正确数量的参数并调用usage()如果有问题。然后设置RS和ORS从命令行参数和设置ARGV[1]和ARGV[2]设置为空字符串，这样就不会将它们视为文件名（请参阅使用ARGC和ARGV ）。

的usage()函数打印错误消息并退出。最后，单个规则使用以下命令处理前面概述的打印方案print要么printf视情况而定，取决于RT 。
