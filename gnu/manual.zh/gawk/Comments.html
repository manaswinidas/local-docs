1.1.5中的评论awk程式

注释是为了方便读者而包含在程序中的一些文本；它实际上不是程序的可执行部分。注释可以解释该程序的功能及其工作方式。几乎所有的编程语言都有注释的规定，因为没有它们通常很难理解程序。

在里面awk语言，注释以数字符号字符（' # '）并继续到行尾。' # '不必是该行的第一个字符。的awk语言会忽略数字符号后的其余行。例如，我们可以将以下内容放入advice ：

# This program prints a nice, friendly message.  It helps
# keep novice users from being afraid of the computer.
BEGIN    { print "Don't Panic!" }

您可以将注释行放入键盘组成的一次性内容中awk程序，但这通常不是很有用；发表评论的目的是帮助您或其他人在以后阅读程序时理解该程序。

小心： 如一键扔掉中所述awk程序 ，您可以将中小型程序括在单引号中，以使您的shell脚本自成一体。这样做时， 请勿在注释（或程序中的其他任何地方）中加上撇号（即单引号）。Shell将引号解释为整个程序的结束引号。因此，通常外壳程序会显示有关不匹配引号的消息，如果awk实际运行时，它可能会显示有关语法错误的奇怪消息。例如，查看以下内容：

$ awk 'BEGIN { print "hello" } # let's be cute'
>

Shell看到前两个引号匹配，并且新的带引号的对象在命令行的末尾开始。因此，它以辅助提示提示，等待更多输入。使用Unix awk ，关闭带引号的字符串会产生以下结果：

$ awk '{ print "hello" } # let's be cute'
> '
error→ awk: can't open file be
error→  source line number 1

在'中的单引号前加上反斜杠let's则无济于事，因为反斜杠在单引号内并不特殊。下一节描述了Shell的引用规则。
