30.16.3测试网络组成员身份

通常不必扫描整个网络组，因为通常唯一有趣的问题是给定条目是否为所选网络组的一部分。

函数： int innetgr （const char * netgroup ，const char * host ，const char * user ，const char * domain ）

初步： MT不安全种族：netgrent语言环境| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

此函数测试参数是否指定了三元组host ， user和domain是网络组的一部分netgroup 。使用此功能的优点是

  1. 由于使用了内部锁定，因此没有其他网络组功能可以使用全局网络组状态。
  2. 该功能的实现比连续调用另一个函数更有效set / get / endnetgrent功能。

任何指针host ， user ， 要么domain可NULL这表示此位置接受任何值。这个名字也是如此-否则不应与其他任何字符串匹配。

返回值为1如果在网络组中找到与给定三元组匹配的条目。返回值为0如果找不到网络组本身，则该网络组不包含三元组或发生内部错误。
