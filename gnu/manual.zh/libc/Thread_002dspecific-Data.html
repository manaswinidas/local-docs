35.2.1特定于线程的数据

GNU C库实现了允许用户创建和管理特定于线程的数据的功能。如果提供了析构函数，则此类数据可能在线程退出时被破坏。定义了以下功能：

函数： int pthread_key_create （pthread_key_t * key ，无效（* destructor ）（无效*））

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

为调用线程创建特定于线程的数据键，引用key 。

用C ++ 11声明的对象thread_local关键字在线程特定的数据之前被销毁，因此不应在线程特定的数据析构函数中使用它们，甚至不应该将它们用作线程特定的数据的成员，因为后者是作为参数传递给析构函数的。

函数： int pthread_key_delete （pthread_key_t key ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

销毁线程专用数据key在调用线程中。线程专用数据的析构函数在销毁过程中不会被调用，在线程退出过程中也不会被调用。

函数： void * pthread_getspecific （pthread_key_t key ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

返回与之关联的特定于线程的数据key在调用线程中。

函数： int pthread_setspecific （pthread_key_t key ，const void * value ）

初步： MT安全| AS不安全的损坏堆| AC不安全的损坏的内存|请参阅POSIX安全概念 。

关联特定于线程value与key在调用线程中。
