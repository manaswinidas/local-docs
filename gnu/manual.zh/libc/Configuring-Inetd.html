16.11.2配置inetd

文件/etc/inetd.conf告诉inetd侦听哪些端口以及为其运行哪些服务器程序。通常，文件中的每个条目都是一行，但是您可以将其拆分为多行，前提是条目的第一行以外的所有行都以空格开头。以'开头的行# '是评论。

这是两个标准条目/etc/inetd.conf ：

ftp	stream	tcp	nowait	root	/libexec/ftpd	ftpd
talk	dgram	udp	wait	root	/libexec/talkd	talkd

条目具有以下格式：

service style protocol wait username program arguments

的service字段说明此程序提供的服务。它应该是在中定义的服务的名称/etc/services 。 inetd用途service决定侦听此条目的端口。

田野style和protocol指定用于侦听套接字的通信样式和协议。样式应为交流样式的名称，并转换为小写并以' SOCK_ '已删除-例如，' stream ' 要么 ' dgram '。 protocol应该是列在其中的协议之一/etc/protocols 。典型的协议名称是“ tcp '用于字节流连接，' udp '表示不可靠的数据报。

的wait字段应为“ wait ' 要么 ' nowait '。使用 ' wait '如果style是无连接样式，服务器一旦启动，便会处理多个请求。使用 ' nowait '如果inetd应该为收到的每条消息或请求启动一个新的过程。如果style使用连接，然后wait 必须是nowait '。

user是服务器应以其运行的用户名。 inetd以root用户身份运行，因此它可以任意设置其子级的用户ID。最好避免使用“ root为user如果你可以的话;但是某些服务器（例如Telnet和FTP）会读取用户名并自行设置密码。这些服务器最初必须是root用户，以便它们可以根据网络上的数据命令登录。

program和...一起arguments指定要运行以启动服务器的命令。 program应该是一个绝对文件名，指定要运行的可执行文件。 arguments由任意数量的空格分隔的单词组成，这些单词成为program 。中的第一个字arguments是参数零，按照惯例，它应该是程序名称本身（没有目录）。

如果您编辑/etc/inetd.conf ，你可以说inetd通过发送重新读取文件并遵循其新内容inetd处理SIGHUP信号。您必须使用ps确定进程的ID inetd处理过程，因为它不是固定的。
