3.5.2锁定的内存详细信息

内存锁与虚拟页面而不是真实框架关联。分页规则是：如果一帧至少支持一个锁定的页面，请不要将其分页出去。

内存锁不会堆叠。即，您不能两次锁定特定页面，因此必须先将其解锁两次才能真正解锁。它要么被锁定，要么未被锁定。

内存锁将一直持续到拥有内存的进程将其明确解锁为止。（但是进程终止和执行会导致虚拟内存停止存在，您可能会说这意味着它不再被锁定）。

子进程不继承内存锁。（但是请注意，在现代Unix系统上，在分叉之后，父级和子级的虚拟地址空间立即由相同的真实页面框架作为后盾，因此子级享有父级的锁）。请参阅创建流程 。

由于它可以影响其他进程，因此只有超级用户才能锁定页面。任何进程都可以解锁自己的页面。

系统对进程可以锁定的内存量及其专用的实际内存量设置了限制。请参阅资源限制 。

在Linux中，锁定页面的锁定程度不如您想象的那样。但是，不是共享内存的两个虚拟页面可以由同一真实框架支持。当内核知道两个虚拟页面包含相同的数据并且即使其中一个或两个虚拟页面被锁定时，内核也会以效率为目的进行此操作。

但是，当进程修改其中一个页面时，内核必须为它提供一个单独的框架，并用该页面的数据填充它。这称为写时复制页面错误 。这会花费少量时间，在病理情况下，获取该帧可能需要I / O。

为了确保您的程序不会发生这种情况，请不要仅锁定页面。也写信给他们，除非您知道永远不会写信给他们。并且要确保为堆栈预先分配了帧，请输入一个范围，该范围声明一个C自动变量，该变量大于所需的最大堆栈大小，将其设置为某种值，然后从其范围返回。
