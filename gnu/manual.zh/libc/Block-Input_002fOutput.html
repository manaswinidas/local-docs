12.11块输入/输出

本节介绍如何对数据块进行输入和输出操作。您可以使用这些函数读取和写入二进制数据，以及以固定大小的块（而不是按字符或行）读取和写入文本。

二进制文件通常用于读取和写入数据块，其格式与用于在运行程序中表示数据的格式相同。换句话说，可以将任意内存块（而不仅仅是字符或字符串对象）写入二进制文件，并由同一程序有意义地再次读取。

与使用格式化的I / O功能相比，以二进制形式存储数据通常效率更高。同样，对于浮点数，二进制形式避免了转换过程中精度的可能损失。另一方面，二进制文件无法使用许多标准文件实用程序（例如文本编辑器）轻松检查或修改，并且无法在该语言的不同实现或不同类型的计算机之间移植。

这些函数在中声明stdio.h 。

函数： size_t fread （无效* data ，size_t size ，size_t count ，文件* stream ）

初步： MT安全|不安全AS损坏| AC不安全锁损坏|请参阅POSIX安全概念 。

该功能最多读取count物体大小size进入阵列data ，来自信息流stream 。它返回实际读取的对象数，可能少于count如果发生读取错误或到达文件末尾。如果有一个函数返回零值（不读取任何内容） size要么count是零。

如果fread在对象中间遇到文件结尾时，它返回读取的完整对象的数量，并丢弃部分对象。因此，流保留在文件的实际末尾。

函数： size_t fread_unlocked （无效* data ，size_t size ，size_t count ，文件* stream ）

初步： MT-Safe竞赛：流|不安全AS损坏| AC不安全的腐败|请参阅POSIX安全概念 。

的fread_unlocked功能相当于fread函数，但它不会隐式锁定流。

此函数是GNU扩展。

功能： size_t fwrite （const void * data ，size_t size ，size_t count ，文件* stream ）

初步： MT安全|不安全AS损坏| AC不安全锁损坏|请参阅POSIX安全概念 。

该功能写到count物体大小size从数组data ，到信息流stream 。返回值通常是count ，如果呼叫成功。其他任何值都表示某种错误，例如空间不足。

功能： size_t fwrite_unlocked （const void * data ，size_t size ，size_t count ，文件* stream ）

初步： MT-Safe竞赛：流|不安全AS损坏| AC不安全的腐败|请参阅POSIX安全概念 。

的fwrite_unlocked功能相当于fwrite函数，但它不会隐式锁定流。

此函数是GNU扩展。
