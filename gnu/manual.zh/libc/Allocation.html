17.9.1分配伪终端

本小节描述了分配伪终端并使该伪终端可实际使用的功能。这些函数在头文件中声明stdlib.h 。

函数： int getpt （无效）

初步： MT安全| AS安全交流安全fd |请参阅POSIX安全概念 。

的getpt函数为下一个可用的主伪终端返回一个新的文件描述符。的正常返回值getpt是一个非负整数文件描述符。如果发生错误，则返回值-1 。下列errno为此功能定义了条件：

ENOENT

没有可用的免费主伪终端。

此函数是GNU扩展。

函数： int grantpt （int filedes ）

初步： MT安全区域| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

的grantpt函数更改与文件描述符关联的主伪终端设备相对应的从伪终端设备的所有权和访问权限filedes 。根据调用过程的真实用户ID设置所有者（请参阅Process Persona ），并将该组设置为特殊组（通常为tty ）或根据调用过程的真实组ID进行设置。设置访问许可，以便文件所有者既可以读写文件，也只能由组写文件。

在某些系统上，此功能是通过调用特殊功能来实现的setuid根程序（请参阅如何更改角色 ）。结果，为SIGCHLD信号（请参阅“ 作业控制信号” ）可能会干扰对grantpt 。

的正常返回值grantpt是0 ;如果失败，则返回-1 。下列errno为此功能定义了错误条件：

EBADF

的filedes参数不是有效的文件描述符。

EINVAL

的filedes参数不与主伪终端设备关联。

EACCES

对应于主机的从机伪终端设备与filedes无法访问。

函数： int unlockpt （int filedes ）

初步： MT安全| AS不安全堆/ bsd | AC不安全的内存fd |请参阅POSIX安全概念 。

的unlockpt函数解锁与文件描述符关联的主伪终端设备相对应的从伪终端设备filedes 。在许多系统上，从设备只能在解锁后才能打开，因此便携式应用程序应始终调用unlockpt在尝试打开奴隶之前。

的正常返回值unlockpt是0 ;如果失败，则返回-1 。下列errno为此功能定义了错误条件：

EBADF

的filedes参数不是有效的文件描述符。

EINVAL

的filedes参数不与主伪终端设备关联。

函数： char * ptsname （int filedes ）

初步： MT不安全种族：ptsname | AS不安全堆/ bsd | AC不安全的内存fd |请参阅POSIX安全概念 。

如果文件描述符filedes与主伪终端设备相关联， ptsname函数返回一个指针，该指针指向一个静态分配的，以空值结尾的字符串，该字符串包含关联的从属伪终端文件的文件名。后续的调用可能会覆盖此字符串ptsname 。

函数： int ptsname_r （int filedes ，字符* buf ，size_t len ）

初步： MT安全| AS不安全堆/ bsd | AC不安全的内存fd |请参阅POSIX安全概念 。

的ptsname_r功能类似于ptsname函数，除了将结果放入用户指定的缓冲区（从处开始）之外buf与长度len 。

此函数是GNU扩展。

以下示例说明了这些功能的典型用法：

int
open_pty_pair (int *amaster, int *aslave)
{
  int master, slave;
  char *name;

  master = getpt ();
  if (master < 0)
    return 0;

  if (grantpt (master) < 0 || unlockpt (master) < 0)
    goto close_master;
  name = ptsname (master);
  if (name == NULL)
    goto close_master;

  slave = open (name, O_RDWR);
  if (slave == -1)
    goto close_master;

  *amaster = master;
  *aslave = slave;
  return 1;

close_slave:
  close (slave);

close_master:
  close (master);
  return 0;
}
