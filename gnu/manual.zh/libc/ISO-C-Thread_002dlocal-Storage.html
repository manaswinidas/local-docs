35.1.6线程本地存储

GNU C库实现了提供线程本地存储的功能 ，可以通过该机制定义变量以具有唯一的每个线程存储，与线程寿命匹配的生存期以及用于清理唯一的每个线程存储的析构函数。

存在几种用于线程本地存储的数据类型和宏：

数据类型： tss_t

的tss_t数据类型标识线程特定的存储对象。即使共享，每个线程也会有其自己的变量实例，并具有不同的值。

数据类型： tss_dtor_t

的tss_dtor_t是类型的函数指针void (*) (void *) ，用作特定于线程的存储析构函数。当前线程调用时将调用该函数thrd_exit （但从不拨打电话tss_delete要么exit ）。

巨集： thread_local

thread_local用于标记具有线程存储持续时间的变量，这意味着该变量在线程启动时创建，并在线程结束时清除。

注意：对于C ++，需要C ++ 11或更高版本才能使用thread_local关键词。

巨集： TSS_DTOR_ITERATIONS

TSS_DTOR_ITERATIONS是一个整数常量表达式，表示线程终止时所有线程局部析构函数的最大迭代次数。该值为破坏线程本地存储提供了一个有限的限制。例如，考虑一个创建更多线程本地存储的析构函数。

以下功能用于管理线程本地存储：

函数： int tss_create （tss_t * tss_key ，tss_dtor_t destructor ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

tss_create创建一个新的特定于线程的存储密钥，并将其存储在由指向的对象中tss_key 。尽管不同线程可能使用相同的键值，但绑定到键的值由tss_set在每个线程的基础上进行维护，并在调用线程的生命周期内保持不变。

如果destructor不为NULL时，将设置一个析构函数，并在线程完成调用后调用该函数thrd_exit 。

该函数返回thrd_success如果tss_key已成功将线程设置为唯一值；除此以外， thrd_error返回并且值tss_key未定义。

函数： int tss_set （tss_t tss_key ，无效* val ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

tss_set设置由标识的特定于线程的存储的值tss_key对于当前线程val 。不同的线程可以为同一键设置不同的值。

该函数返回thrd_success要么thrd_error 。

函数： void * tss_get （tss_t tss_key ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

tss_get返回由标识的值tss_key保存在当前线程的线程专用存储中。不同的线程可能会获得由同一密钥标识的不同值。失败了tss_get返回零。

函数： void tss_delete （tss_t tss_key ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

tss_delete破坏由标识的特定于线程的存储tss_key 。
