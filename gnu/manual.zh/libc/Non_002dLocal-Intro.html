23.1非本地出口简介

作为使用非本地出口可能有用的情况的示例，假设您有一个交互式程序，该程序具有“主循环”来提示并执行命令。假设“读取”命令从文件读取输入，在处理输入时进行一些词法分析和解析。如果检测到低级输入错误，那么能够立即返回“主循环”而不用使每个词法分析，解析和处理阶段都必须显式处理错误情况将很有用。最初是由嵌套调用检测到的。

（另一方面，如果这些阶段中的每个阶段都必须在退出时进行大量清理（例如关闭文件，释放缓冲区或其他数据结构等），那么进行正常的返回会更合适并让每个阶段都进行自己的清理，因为非本地退出会绕过中间阶段及其相关的清理代码。或者，您可以使用非本地出口，但可以在返回“主循环”之前或之后显式地进行清理。）

在某些方面，非本地出口类似于使用“ return '从函数返回的语句。但是当return如果只放弃一个函数调用，将控制权转移回被调用的位置，那么非本地出口可能会放弃许多级别的嵌套函数调用。

您可以通过调用函数来识别非本地出口的返回点setjmp 。此函数保存有关执行环境的信息，在该环境中调用setjmp出现在一个类型的对象中jmp_buf 。调用后，程序通常会继续执行。 setjmp ，但是如果稍后通过调用退出此返回点longjmp与相应的jmp_buf 对象，控制权转移回该点setjmp被称为。的返回值setjmp用于区分普通收益和通过调用产生的收益longjmp ，因此致电setjmp通常出现在“ if声明。

这是上述示例程序的设置方式：


#include <setjmp.h>
#include <stdlib.h>
#include <stdio.h>

jmp_buf main_loop;

void
abort_to_main_loop (int status)
{
  longjmp (main_loop, status);
}

int
main (void)
{
  while (1)
    if (setjmp (main_loop))
      puts ("Back at main loop....");
    else
      do_command ();
}


void
do_command (void)
{
  char buffer[128];
  if (fgets (buffer, 128, stdin) == NULL)
    abort_to_main_loop (-1);
  else
    exit (EXIT_SUCCESS);
}

功能abort_to_main_loop无论从何处调用，都会立即将控制权转移回程序的主循环。

内部的控制流main函数起初可能看起来有些神秘，但实际上是setjmp 。正常拨打setjmp返回零，因此将执行条件的“ else”子句。如果abort_to_main_loop被称为执行的某处do_command ，那么它实际上看起来好像一样调用setjmp在main再次返回值是-1 。

因此，使用的一般模式setjmp看起来像：

if (setjmp (buffer))
  /* Code to clean up after premature return. */
  …
else
  /* Code to be executed normally after setting up the return point. */
  …
