1.2.1 ISO C

GNU C库与美国国家标准协会（ANSI）： 美国国家标准X3.159-1989（“ ANSI C”）以及后来国际标准化组织（ISO）： ISO / IEC 9899通过的C标准兼容： 1990年，“编程语言-C” 。我们在这里将该标准称为ISO C 因为这是批准方面的更通用的标准。组成GNU C库的头文件和库工具是ISO C 标准指定的头文件和库工具的超集。

如果您担心严格遵守ISO C 标准，则应使用“ -ansi使用GNU C编译器编译程序时使用'选项。这告诉编译器仅从库头文件中定义ISO标准功能，除非您明确要求其他功能。有关如何执行此操作的信息，请参见功能测试宏 。

能够将库限制为仅包含ISO C 功能非常重要，因为ISO C 限制了库实现可以定义的名称，而GNU扩展不符合这些限制。有关这些限制的更多信息，请参见保留名称 。

本手册并不试图为您提供有关ISO C 与较旧方言之间差异的完整详细信息。它提供了有关如何编写程序以在多个C语言下可移植工作的建议，但并不旨在完整性。
