16.6.1 Internet套接字地址格式

在Internet名称空间中，对于两个IPv4（ AF_INET ）和IPv6（ AF_INET6 ），套接字地址由主机地址和该主机上的端口组成。另外，您选择的协议可以有效地用作地址的一部分，因为本地端口号仅在特定协议内才有意义。

在头文件中定义了用于表示Internet名称空间中套接字地址的数据类型。 netinet/in.h 。

数据类型： struct sockaddr_in

这是用于表示Internet名称空间中套接字地址的数据类型。它具有以下成员：

sa_family_t sin_family

这标识了地址系列或套接字地址的格式。您应该存储值AF_INET在这个成员中。地址族按主机字节顺序存储。请参阅套接字地址 。

struct in_addr sin_addr

这是IPv4地址。有关如何获取值存储在此处的信息，请参见主机地址和主机名 。IPv4地址以网络字节顺序存储。

unsigned short int sin_port

这是端口号。请参阅端口 。端口号以网络字节顺序存储。

你打电话的时候bind要么getsockname ，您应指定sizeof (struct sockaddr_in)作为length参数（如果使用的是IPv4 Internet名称空间套接字地址）。

数据类型： struct sockaddr_in6

这是用于表示IPv6名称空间中套接字地址的数据类型。它具有以下成员：

sa_family_t sin6_family

这标识了地址系列或套接字地址的格式。您应该存储的值AF_INET6在这个成员中。请参阅套接字地址 。地址族按主机字节顺序存储。

struct in6_addr sin6_addr

这是主机的IPv6地址。有关如何获取值存储在此处的信息，请参见主机地址和主机名 。地址以网络字节顺序存储。

uint32_t sin6_flowinfo

如IPv6标头中所示，此方法将IPv6流量类和流标签值组合在一起。该字段以网络字节顺序存储。仅使用28个低位（按网络字节顺序编号）。 restig位必须为零。低20位是流标签，而20到27位是流量类别。通常，此字段为零。

uint32_t sin6_scope_id

对于本地链接地址，这标识该地址在其上有效的接口。范围ID以主机字节顺序存储。通常，此字段为零。

uint16_t sin6_port

这是端口号。请参阅端口 。端口号以网络字节顺序存储。
