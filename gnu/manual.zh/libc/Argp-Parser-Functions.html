25.3.5 Argp解析器函数

指向的功能parser在一个领域struct argp （请参阅Argp解析器 ）定义了响应于每个已解析的选项或参数而采取的操作。它还用作挂钩，允许解析器在解析期间在某些其他点执行任务。

Argp解析器函数具有以下类型签名：

error_t parser (int key, char *arg, struct argp_state *state)

其中的参数如下：

key

对于每个解析的选项， parser的值为key从该选项的key选项向量中的字段。请参阅Argp选项向量 。 parser在其他时间也使用特殊的保留键来调用，例如ARGP_KEY_ARG用于非选项参数。请参阅Argp特殊键 。

arg

如果key是一个选择arg是它的给定值。如果未指定任何值，则默认为零。仅具有非零的选项arg字段永远可以具有价值。这些必须始终具有一个值，除非OPTION_ARG_OPTIONAL标志被指定。如果要解析的输入为不允许的选项指定了一个值，则在parser曾经被称为。

如果key是ARGP_KEY_ARG ， arg是一个非选项参数。其他特殊键始终为零arg 。

state

state指向一个struct argp_state ，包含有关当前解析状态的有用信息，供以下人员使用parser 。请参阅Argp解析状态 。

什么时候parser被调用，它应该执行适合于key ，然后返回0为了成功， ARGP_ERR_UNKNOWN如果值key不由该解析器函数处理，如果发生实际错误，则不处理Unix错误代码。请参阅错误代码 。

巨集： int ARGP_ERR_UNKNOWN

Argp解析器函数应返回ARGP_ERR_UNKNOWN对于任何key它们无法识别的值，或用于非选项参数（ key == ARGP_KEY_ARG ），他们没有能力处理。

典型的解析器函数在以下位置使用switch语句key ：

error_t
parse_opt (int key, char *arg, struct argp_state *state)
{
  switch (key)
    {
    case option_key:
      action
      break;
    …
    default:
      return ARGP_ERR_UNKNOWN;
    }
  return 0;
}
