32.1一般容量限制

POSIX.1和POSIX.2标准指定了许多参数，这些参数描述了系统的容量限制。这些限制可以是给定操作系统的固定常数，也可以因计算机而异。例如，某些限制值可以由系统管理员在运行时或通过重建内核来配置，并且这不需要重新编译应用程序。

以下每个限制参数都有一个定义为limits.h仅当系统对相关参数有固定的统一限制时。如果系统允许不同的文件系统或文件具有不同的限制，则该宏未定义；使用sysconf找出在特定时间在特定计算机上适用的限制。参见Sysconf 。

每个参数还具有另一个宏，其名称以“ _POSIX '，它给出了在任何 POSIX系统上允许的限制的最小值。请参阅最低要求 。

巨集： int ARG_MAX

如果定义，则组合的不变最大最大长度argv和environ可以传递给exec功能。

巨集： int CHILD_MAX

如果定义，则在任何时候使用相同的真实用户ID可以存在的最大进程数。在BSD和GNU中，这由RLIMIT_NPROC资源限制；请参阅资源限制 。

巨集： int OPEN_MAX

如果定义，则单个进程可以同时打开的不变的最大文件数。在BSD和GNU中，这由RLIMIT_NOFILE资源限制；请参阅资源限制 。

巨集： int STREAM_MAX

如果定义，则单个进程可以同时打开的最大不变流数。请参阅打开流 。

巨集： int TZNAME_MAX

如果已定义，则为时区名称的不变最大长度。请参阅时区功能 。

这些限制宏始终在limits.h 。

巨集： int NGROUPS_MAX

一个进程可以拥有的最大补充组ID数。

该宏的值实际上是最大值的下限。也就是说，您可以指望能够拥有那么多补充组ID，但是一台特定的机器可能会让您拥有更多。您可以使用sysconf看看特定的机器是否会让您拥有更多功能（请参阅Sysconf ）。

巨集： ssize_t SSIZE_MAX

可以容纳在类型对象中的最大值ssize_t 。实际上，这是对单个操作中可以读取或写入的字节数的限制。

在所有POSIX系统中都定义了此宏，因为此限制永远不可配置。

巨集： int RE_DUP_MAX

构造中允许您保证的最大重复次数' \{min,max\}以正则表达式表示。

该宏的值实际上是最大值的下限。也就是说，您可以指望能够进行那么多重复，但是一台特定的机器可能会让您拥有更多重复。您可以使用sysconf看看特定的机器是否会让您拥有更多功能（请参阅Sysconf ）。甚至价值sysconf告诉您只是一个下限-较大的值可能会起作用。

在所有POSIX.2系统中都定义了此宏，因为POSIX.2表示即使没有特定的施加限制，也应始终对其进行定义。
