16.8.1创建一个套接字

创建套接字的原语是socket函数，在sys/socket.h 。

函数： int socket （int namespace ，int style ，int protocol ）

初步： MT安全| AS安全交流安全fd |请参阅POSIX安全概念 。

该函数创建一个套接字并指定通信方式style ，它应该是“ 通信样式”中列出的套接字样式之一。的namespace参数指定名称空间；肯定是PF_LOCAL （请参阅本地命名空间 ）或PF_INET （请参阅Internet命名空间 ）。protocol指定特定的协议（请参阅Socket概念 ）；零通常适合protocol 。

的返回值socket是新套接字的文件描述符，或者-1如果发生错误。下列errno为此功能定义了错误条件：

EPROTONOSUPPORT

的protocol要么style不受支持namespace指定。

EMFILE

该进程已经打开了太多的文件描述符。

ENFILE

系统已经打开了太多的文件描述符。

EACCES

该进程无权创建指定的套接字style要么protocol 。

ENOBUFS

系统用完了内部缓冲区空间。

由...返回的文件描述符socket函数支持读取和写入操作。但是，套接字与管道一样，不支持文件定位操作。

有关如何调用的示例socket函数，请参见“ 本地套接字示例 ”或“ Inet示例” 。
