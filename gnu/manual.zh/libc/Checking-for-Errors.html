2.1检查错误

大多数库函数都返回一个特殊值，以指示它们已失败。特殊值通常是-1 ，空指针或常量，例如EOF为此目的而定义的。但是此返回值仅告诉您发生了错误。要找出错误类型，您需要查看存储在变量中的错误代码errno 。该变量在头文件中声明errno.h 。

变量： volatile int errno

变量errno包含系统错误号。您可以更改的值errno 。

以来errno被宣布volatile ，它可能会被信号处理程序异步更改；请参阅定义处理程序 。但是，正确编写的信号处理程序将保存并恢复以下值： errno ，因此通常无需担心这种可能性，除非编写信号处理程序。

的初始值errno在程序启动时为零。在许多情况下，当库函数遇到错误时，它将设置errno设置为非零值以指示发生了什么特定的错误情况。每个功能的文档列出了该功能可能的错误条件。并非所有的库函数都使用这种机制。有些人则直接返回错误代码。

警告：可能设置了许多库功能errno即使它们没有遇到任何错误，也即使它们直接返回错误代码，也可以设置为无意义的非零值。因此，通过检查的值来检查是否发生错误通常是不正确的。 errno 。每个功能都记录了检查错误的正确方法。

可移植性注释： ISO C 指定errno作为“可修改的左值”而不是变量，从而允许将其实现为宏。例如，其扩展可能涉及函数调用，例如*__errno_location () 。实际上，这就是GNU / Linux和GNU / Hurd系统上的情况。每个系统上的GNU C库都执行适合特定系统的任何操作。

有一些库函数，例如sqrt和atan ，如果出现错误，则返回一个完全合法的值，但也可以设置errno 。对于这些功能，如果要检查是否发生错误，建议的方法是设置errno为零，然后再调用该函数。

所有错误代码都有符号名称；它们是在errno.h 。名称以“ E '和大写字母或数字；您应将此表格的名称视为保留名称。请参阅保留名称 。

错误代码值都是正整数，并且都是唯一的，只有一个例外： EWOULDBLOCK和EAGAIN是相同的。由于这些值是不同的，因此您可以将它们用作标签中的switch声明;只是不要同时使用EWOULDBLOCK和EAGAIN 。您的程序不应对这些符号常量的特定值做任何其他假设。

除了在GNU / Hurd系统上，几乎所有系统调用都可以返回EFAULT如果给定一个无效的指针作为参数。由于这种情况仅可能是由于您的程序中的错误而发生，并且由于它不会在GNU / Hurd系统上发生，因此我们通过不提及而节省了空间EFAULT在各个功能的说明中。

在某些Unix系统中，许多系统调用也可以返回EFAULT如果给定一个指向堆栈的指针作为参数，并且内核由于某种晦涩的原因而无法扩展堆栈。如果发生这种情况，您可能应该尝试使用静态或动态分配的内存，而不是该系统上的堆栈内存。
