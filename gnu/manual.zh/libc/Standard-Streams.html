12.2标准流

当。。。的时候main调用程序的函数，它已经打开了三个预定义的流，可供使用。这些代表已为该过程建立的“标准”输入和输出通道。

这些流在头文件中声明stdio.h 。

变量： FILE * stdin

标准输入流，它是程序的常规输入源。

变量： FILE * stdout

标准输出流，用于程序的常规输出。

变量： FILE * stderr

标准错误流，用于程序发出的错误消息和诊断。

在GNU系统上，您可以使用Shell提供的管道和重定向功能来指定与这些流相对应的文件或进程。（在文件系统接口中描述了Shell用于实现这些功能的原语。）大多数其他操作系统也提供类似的机制，但是如何使用它们的细节可能有所不同。

在GNU C库中， stdin ， stdout和stderr是普通变量，您可以像设置其他变量一样进行设置。例如，要将标准输出重定向到文件，您可以执行以下操作：

fclose (stdout);
stdout = fopen ("standard-output-file", "w");

但是请注意，在其他系统中stdin ， stdout和stderr是您无法以常规方式分配的宏。但是你可以使用freopen获得关闭并重新打开的效果。请参阅打开流 。

三流stdin ， stdout和stderr在程序启动时并不是没有方向性的（请参见Streams和I18N ）。
