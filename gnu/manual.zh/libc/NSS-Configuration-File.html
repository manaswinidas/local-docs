29.2 NSS配置文件

必须以某种方式告知NSS代码用户的意愿。因此，有文件/etc/nsswitch.conf 。对于每个数据库，此文件都包含有关查找过程如何工作的规范。该文件可能如下所示：

# /etc/nsswitch.conf
#
# Name Service Switch configuration file.
#

passwd:     db files nis
shadow:     files
group:      db files nis

hosts:      files nisplus nis dns
networks:   nisplus [NOTFOUND=return] files

ethers:     nisplus [NOTFOUND=return] db files
protocols:  nisplus [NOTFOUND=return] db files
rpc:        nisplus [NOTFOUND=return] db files
services:   nisplus [NOTFOUND=return] db files

您可以从上表中猜测出第一列是数据库。该行的其余部分指定查找过程的工作方式。请注意，您可以分别指定每个数据库的工作方式。用整体实现的旧方法无法做到这一点。

每个数据库的配置规范可以包含两个不同的项目：

  • 服务规范如files ， db ， 要么nis 。
  • 查找结果的反应像[NOTFOUND=return] 。
• NSS配置中的服务  NSS配置中的服务名称。
• NSS配置中的操作  对查找结果做出适当的反应。
• 关于NSS配置文件的说明  配置NSS时要注意的事项。
