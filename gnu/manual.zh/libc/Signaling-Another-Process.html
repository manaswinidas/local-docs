24.6.2发信号通知另一个进程

的kill功能可用于向另一个进程发送信号。尽管它具有名称，但是它可以用于使进程终止之外的许多事情。您可能希望在进程之间发送信号的一些情况示例如下：

  • 父进程启动子进程执行任务（也许让子进程运行无限循环），然后在不再需要任务时终止子进程。
  • 进程作为组的一部分执行，并且需要在发生错误或其他事件时终止或通知组中的其他进程。
  • 一起工作时，两个进程需要同步。

本部分假定您对流程的工作原理有所了解。有关此主题的更多信息，请参见流程 。

的kill函数在signal.h 。

函数： int kill （pid_t pid ，int signum ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的kill功能发送信号signum到指定的流程或流程组pid 。除了标准信号中列出的信号 ， signum也可以使用零值来检查pid 。

的pid指定接收信号的过程或过程组：

pid > 0

标识符为的进程pid 。（在Linux上，即使pid是不同于进程ID的线程ID。）

pid == 0

与发件人在同一进程组中的所有进程。

pid < -1

标识符为-的进程组pid 。

pid == -1

如果该进程具有特权，则将信号发送到除某些特殊系统进程之外的所有进程。否则，将信号发送给具有相同有效用户ID的所有进程。

进程可以通过类似这样的调用向自身发送信号kill (getpid(), signum) 。如果kill由进程用来向自身发送信号，并且信号未被阻塞，则kill传送至少一个信号（可能是其他一些未阻塞的未阻塞信号，而不是信号signum ）返回到该过程。

的返回值kill如果可以成功发送信号，则为零。否则，不发送信号，并且值为-1返回。如果pid指定向多个进程发送信号， kill如果可以将信号发送到至少其中之一，则表示成功。您无法分辨哪个进程收到了信号，或者是否全部收到了信号。

下列errno为此功能定义了错误条件：

EINVAL

的signum参数是无效或不受支持的数字。

EPERM

您无权向该进程或由命名的进程组中的任何进程发送信号pid 。

ESRCH

的pid参数不引用现有的进程或组。

函数： int tgkill （pid_t pid ，pid_t tid ，int signum ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的tgkill功能发送信号signum ID的线程或进程tid ， 如kill函数，但仅在线程的进程ID时tid等于pid 。如果目标线程属于另一个进程，则该函数将失败并显示ESRCH 。

的tgkill函数可以用来避免在错误的进程中向线程发送信号，如果调用者确保pid值不会被内核重用（例如，如果它是当前进程的进程ID，则返回getpid ）。

函数： int killpg （int pgid ，int signum ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

这类似于kill ，但发送信号signum进入过程组pgid 。提供此功能是为了与BSD兼容。使用kill做到这一点更方便。

作为一个简单的例子kill ，电话kill (getpid (), sig) 的作用与raise (sig) 。
