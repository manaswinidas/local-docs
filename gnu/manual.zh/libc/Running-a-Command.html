26.1运行命令

运行另一个程序的简单方法是使用system功能。该函数完成了运行子程序的所有工作，但并没有给您太多控制细节：您必须等到子程序终止后才能执行其他任何操作。

函数： int 系统 （const char * command ）

初步： MT安全| AS-Unsafe插件堆锁| AC不安全锁记忆|请参阅POSIX安全概念 。

该函数执行command作为shell命令。在GNU C库中，它始终使用默认外壳sh运行命令。特别是，它搜索以下目录PATH查找要执行的程序。返回值为-1如果无法创建shell进程，则为shell进程的状态。有关如何解释此状态代码的详细信息，请参见流程完成 。

如果command参数是一个空指针，返回值为零表示没有命令处理器可用。

此功能是多线程程序中的取消点。如果线程当时分配了一些资源（例如内存，文件描述符，信号量或其他内容），这将是一个问题system叫做。如果线程被取消，这些资源将保持分配状态，直到程序结束。为了避免这种情况system应该使用取消处理程序进行保护。
