14.11临时文件

如果您需要在程序中使用临时文件，则可以使用tmpfile打开它的功能。或者您可以使用tmpnam （更好： tmpnam_r ）函数为临时文件提供名称，然后您可以按通常的方式使用fopen 。

的tempnam功能就像tmpnam但可以让您选择要进入的目录临时文件，以及有关其文件名的外观。对于多线程程序，重要的是tempnam是可重入的，而tmpnam不是，因为它返回指向静态缓冲区的指针。

这些功能在头文件中声明stdio.h 。

函数： FILE * tmpfile （无效）

初步： MT安全| AS不安全堆锁| AC不安全的内存fd锁|请参阅POSIX安全概念 。

此函数为更新模式创建一个临时二进制文件，就像通过调用fopen有模式"wb+" 。该文件在关闭或程序终止时会自动删除。（在其他一些ISO C 系统上，如果程序异常终止，则可能无法删除该文件）。

此函数是可重入的。

当源编译时_FILE_OFFSET_BITS == 64在32位系统上，该功能实际上是tmpfile64 ，即LFS接口透明地替换了旧接口。

函数： FILE * tmpfile64 （无效）

初步： MT安全| AS不安全堆锁| AC不安全的内存fd锁|请参阅POSIX安全概念 。

该功能类似于tmpfile ，但是它返回的流指向使用tmpfile64 。因此，此流可用于32位计算机上大于2 ^ 31字节的文件。

请注意，返回类型仍为FILE * 。没有特别的FILE LFS接口的类型。

如果源使用_FILE_OFFSET_BITS == 64在32位计算机上，此功能在名称下可用tmpfile因此透明地替换了旧界面。

函数： char * tmpnam （字符* result ）

初步： MT不安全种族：tmpnam /！结果AS不安全|交流安全请参阅POSIX安全概念 。

此函数构造并返回一个不引用任何现有文件的有效文件名。如果result参数是一个空指针，返回值是一个指向内部静态字符串的指针，该字符串可能被后续调用修改，因此使该函数不可重入。否则， result参数应为至少包含以下内容的数组的指针L_tmpnam字符，然后将结果写入该数组。

有可能tmpnam如果您调用它太多次而没有删除以前创建的文件，则会失败。这是因为临时文件名的有限长度仅为有限数量的不同名称提供了空间。如果tmpnam失败，它将返回空指针。

警告：在构建路径名和创建文件之间，另一个进程可能使用以下命令创建了一个具有相同名称的文件： tmpnam ，可能会导致安全漏洞。该实现会生成很难预测的名称，但是在打开文件时，应使用O_EXCL旗。使用tmpfile要么mkstemp是避免此问题的安全方法。

功能： char * tmpnam_r （char * result ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

此功能几乎与tmpnam功能，除了result是一个空指针，它返回一个空指针。

这保证了可重入，因为tmpnam这里不可能发生。

警告 ：此功能与以下功能存在相同的安全问题tmpnam 。

巨集： int L_tmpnam

该宏的值是一个整数常量表达式，表示一个字符串的最小大小，该字符串的大小足以容纳由tmpnam功能。

巨集： int TMP_MAX

宏TMP_MAX是您可以使用多少个临时名称创建的下限tmpnam 。您可以依靠能够打电话tmpnam至少说了很多遍，否则它可能无法说明您做了太多的临时文件名。

使用GNU C库，您可以创建大量的临时文件名。如果实际上创建了文件，则在用尽名称之前，可能会用完磁盘空间。其他一些系统对临时文件的数量有固定的小限制。限制不得低于25 。

函数： char * tempnam （const char * dir ，const char * prefix ）

初步： MT安全环境| AS不安全堆| AC不安全的内存|请参阅POSIX安全概念 。

此函数生成唯一的临时文件名。如果prefix不是一个空指针，此字符串的最多五个字符用作文件名的前缀。返回值是一个新分配的字符串malloc ，因此您应该使用free当不再需要它时。

由于字符串是动态分配的，因此该函数是可重入的。

临时文件名的目录前缀是通过依次测试以下各项来确定的。该目录必须存在并且可写。

  • 环境变量TMPDIR ，如果已定义。出于安全原因，仅在程序未启用SUID或SGID的情况下才会发生这种情况。
  • 的dir参数，如果它不是空指针。
  • 的价值P_tmpdir宏。
  • 目录/tmp 。

定义此功能是为了实现SVID兼容性。

警告：在构建路径名和创建文件之间，另一个进程可能使用以下命令创建了一个具有相同名称的文件： tempnam ，可能会导致安全漏洞。该实现会生成很难预测的名称，但是在打开文件时，应使用O_EXCL旗。使用tmpfile要么mkstemp是避免此问题的安全方法。

SVID宏： char * P_tmpdir

此宏是临时文件的默认目录的名称。

较旧的Unix系统没有上述功能。相反，他们使用mktemp和mkstemp 。这两个功能都可以通过修改您传递的文件名模板字符串来工作。该字符串的最后六个字符必须为' XXXXXX '。这六个X的替换为六个字符，使整个字符串成为唯一的文件名。通常，模板字符串类似于“ /tmp/prefixXXXXXX '，并且每个程序使用唯一的prefix 。

注意：因为mktemp和mkstemp修改模板字符串， 不得将字符串常量传递给它们。字符串常量通常位于只读存储器中，因此当mktemp要么mkstemp试图修改字符串。这些函数在头文件中声明stdlib.h 。

函数： char * mktemp （char * template ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的mktemp函数通过修改生成唯一的文件名template如上所述。如果成功，则返回template作为修改。如果mktemp找不到唯一的文件名，这使得template一个空字符串并返回该字符串。如果template不以“ XXXXXX '， mktemp返回一个空指针。

警告：在构建路径名和创建文件之间，另一个进程可能使用以下命令创建了一个具有相同名称的文件： mktemp ，可能会导致安全漏洞。该实现会生成很难预测的名称，但是在打开文件时，应使用O_EXCL旗。使用mkstemp是避免此问题的安全方法。

函数： int mkstemp （char * template ）

初步： MT安全| AS安全交流安全fd |请参阅POSIX安全概念 。

的mkstemp函数生成唯一的文件名，就像mktemp可以，但是它还会为您打开文件open （请参阅打开和关闭文件 ）。如果成功，它将修改template并返回该文件的文件描述符，以供读取和写入。如果mkstemp无法创建唯一命名的文件，它将返回-1 。如果template不以“ XXXXXX '， mkstemp退货-1并且不修改template 。

使用模式打开文件0600 。如果该文件供其他用户使用，则必须显式更改此模式。

不像mktemp ， mkstemp实际上可以保证创建一个唯一的文件，该文件不可能与尝试创建临时文件的任何其他程序冲突。这是因为它可以通过调用open与O_EXCL标记，表示您要创建一个新文件，如果文件已经存在，则会出现错误。

功能： char * mkdtemp （char * template ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的mkdtemp函数使用唯一名称创建目录。如果成功，它将覆盖template目录名称，然后返回template 。与mktemp和mkstemp ， template应该是以'结尾的字符串XXXXXX '。

如果mkdtemp无法创建唯一命名的目录，它将返回NULL和集errno适当地。如果template不以“ XXXXXX '， mkdtemp退货NULL并且不修改template 。 errno将设置为EINVAL在这种情况下。

该目录是使用模式创建的0700 。

目录创建者mkdtemp不能与其他用户创建的临时文件或目录冲突。这是因为目录创建始终像open与O_EXCL 。请参阅创建目录 。

的mkdtemp功能来自OpenBSD。
