22.2限制资源使用

您可以指定进程的资源使用限制。当进程尝试超过限制时，它可能会收到信号，或者尝试这样做的系统调用可能会失败，具体取决于资源。每个过程最初都从其父级继承其极限值，但随后可以更改它们。

与资源相关联的每个进程有两个限制：

电流极限

当前限制是系统不允许使用量超出的值。也称为“软限制”，因为受限制的过程通常可以随意提高当前限制。

最大极限

最大限制是允许进程设置其当前限制的最大值。它也被称为“硬限制”，因为流程无法绕开它。进程可以降低自己的最大限制，但是只有超级用户可以增加最大限制。

用于的符号getrlimit ， setrlimit ， getrlimit64和setrlimit64在中定义sys/resource.h 。

函数： int getrlimit （int resource ，struct rlimit * rlp ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

阅读资源的当前和最大限制resource并将它们存储在*rlp 。

返回值为0成功与-1失败。唯一可能的errno错误条件是EFAULT 。

当源编译时_FILE_OFFSET_BITS == 64在32位系统上，该功能实际上是getrlimit64 。因此，LFS接口透明地替换了旧接口。

函数： int getrlimit64 （int resource ，struct rlimit64 * rlp ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

该功能类似于getrlimit但它的第二个参数是指向类型变量的指针struct rlimit64 ，这允许它读取不适合a成员的值struct rlimit 。

如果源使用_FILE_OFFSET_BITS == 64在32位计算机上，此功能可以在名称下使用getrlimit因此透明地替换了旧界面。

函数： int setrlimit （int resource ，const struct rlimit * rlp ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

存储资源的当前和最大限制resource在*rlp 。

返回值为0成功与-1失败。下列errno错误条件是可能的：

EPERM
  • 该过程试图将电流限制提高到最大限制之外。
  • 该过程试图提高最大限制，但不是超级用户。

当源编译时_FILE_OFFSET_BITS == 64在32位系统上，该功能实际上是setrlimit64 。因此，LFS接口透明地替换了旧接口。

函数： int setrlimit64 （int resource ，const struct rlimit64 * rlp ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

该功能类似于setrlimit但它的第二个参数是指向类型变量的指针struct rlimit64允许它设置不适合成员的值struct rlimit 。

如果源使用_FILE_OFFSET_BITS == 64在32位计算机上，该功能在名称下可用setrlimit因此透明地替换了旧界面。

数据类型： struct rlimit

此结构用于getrlimit接收极限值，并与setrlimit指定特定过程和资源的极限值。它具有两个字段：

rlim_t rlim_cur

当前限制

rlim_t rlim_max

最大限制。

对于getrlimit ，结构是输出；它接收当前值。对于setrlimit ，它指定新值。

对于LFS功能，在sys/resource.h 。

数据类型： struct rlimit64

这种结构类似于rlimit上面的结构，但其组成部分范围更广。它具有两个字段：

rlim64_t rlim_cur

这类似于rlimit.rlim_cur ，但类型不同。

rlim64_t rlim_max

这类似于rlimit.rlim_max ，但类型不同。

这是您可以为其指定限制的资源列表。内存和文件大小以字节为单位。

RLIMIT_CPU

进程可以使用的最大CPU时间。如果运行时间超过此时间，则会收到信号： SIGXCPU 。该值以秒为单位。请参阅操作错误信号 。

RLIMIT_FSIZE

进程可以创建的最大文件大小。尝试写入更大的文件会导致信号： SIGXFSZ 。请参阅操作错误信号 。

RLIMIT_DATA

该进程的最大数据存储空间。如果该进程尝试分配超出此数量的数据内存，则分配功能将失败。

RLIMIT_STACK

进程的最大堆栈大小。如果该进程尝试将其堆栈扩展到该大小以上，则会得到一个SIGSEGV信号。请参阅程序错误信号 。

RLIMIT_CORE

此过程可以创建的最大大小的核心文件。如果进程终止并且将转储一个大于此值的核心文件，则不会创建任何核心文件。因此，将此限制设置为零可防止创建核心文件。

RLIMIT_RSS

该过程应获得的最大物理内存量。该参数是系统调度程序和内存分配器的指南。当有剩余时，系统可以为进程提供更多的内存。

RLIMIT_MEMLOCK

可以锁定到物理内存中的最大内存量（因此它将永远不会被调出）。

RLIMIT_NPROC

使用相同用户ID可以创建的最大进程数。如果您已达到用户ID的限制， fork将失败EAGAIN 。请参阅创建流程 。

RLIMIT_NOFILE
RLIMIT_OFILE

进程可以打开的最大文件数。如果尝试打开的文件超过此数目，则打开尝试失败，并显示errnoEMFILE 。请参阅错误代码 。并非所有系统都支持此限制； GNU和4.4 BSD都可以。

RLIMIT_AS

该过程应获得的总内存的最大大小。例如，如果进程尝试分配超出此数量的更多内存，例如， brk ， malloc ， mmap要么sbrk ，分配功能失败。

RLIM_NLIMITS

不同资源限制的数量。任何有效resource操作数必须小于RLIM_NLIMITS 。

常数： rlim_t RLIM_INFINITY

当作为极限值提供时，此常数表示“无穷大”值setrlimit 。

ulimit并且命令符号在ulimit.h 。

如果有限制，则命令参数是唯一的参数。如果要设置限制，则有第二个参数： long intlimit这是您要设置限制的值。

GETFSIZE

获取文件大小的当前限制，以512字节为单位。

SETFSIZE

还有其他一些cmd在某些系统上可能会起作用的值，但不支持它们。

只有超级用户可以增加最大限制。

成功获得限制后，返回值ulimit是那个极限，永远不会是负数。成功设置限制后，返回值为零。函数失败时，返回值为-1和errno根据原因设置：

EPERM
