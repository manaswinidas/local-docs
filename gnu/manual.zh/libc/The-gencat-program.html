8.1.3生成消息目录文件

的gencat程序在X / Open标准中指定，并且GNU实现遵循该规范，因此将处理所有正确形成的输入文件。此外，还实施了一些扩展功能，这些扩展功能可帮助您更合理地使用catgets功能。

的gencat可以通过两种方式调用程序：

`gencat [Option …] [Output-File [Input-File …]]`

这是X / Open标准中定义的接口。如果不Input-File给定参数，将从标准输入中读取输入。将读取多个输入文件，就像它们是串联的一样。如果Output-File也缺少，输出将被写入标准输出。为了提供该接口，提供了另一个程序的接口。 

`gencat [Option …] -o Output-File [Input-File …]`

选项 ' -o '用于指定输出文件，所有文件参数均用作输入文件。

除了这个可以使用-要么/dev/stdin对于Input-File表示标准输入。对应一个可以使用-和/dev/stdout对于Output-File表示标准输出。使用-因为在使用设备名称的设备中，文件名是XNU / GNU扩展名，所以X / Open中允许使用文件名。

的gencat程序通过连接所有输入文件，然后将消息集的结果集合与可能存在的输出文件合并来工作 。这是通过从输出文件中删除所有设置/消息号元组与任何已生成消息匹配的消息，然后添加所有新消息来完成的。因此，要在忽略旧内容的情况下重新生成目录文件，需要删除输出文件（如果存在）。如果将输出写入标准输出，则不会发生合并。

下表显示了gencat程序。X / Open标准没有为程序指定任何选项，因此所有这些都是GNU扩展。

' -V '
' --version '

打印版本信息并退出。

' -h '
' --help '

打印列出所有可用选项的使用情况消息，然后成功退出。

' --new '

不要将输入文件中的新消息与输出文件中的旧内容合并。输出文件的旧内容将被丢弃。

' -H '
' --header=name '

此选项用于发出给输入文件中的集和消息指定的符号名称，以供程序使用。下一节将提供有关如何使用它的详细信息。的name此选项的参数指定输出文件的名称。它将包含多个C预处理器#define s将名称与数字相关联。

请注意，生成的文件仅包含输入文件中的符号。如果将输出与输出文件的先前内容合并，则生成旧输出文件的文件中可能存在的符号不在生成的头文件中。
