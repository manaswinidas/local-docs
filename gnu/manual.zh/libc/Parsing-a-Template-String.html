12.12.10解析模板字符串

您可以使用该功能parse_printf_format获取有关给定模板字符串期望的参数数量和类型的信息。此功能允许提供接口的解释器printf以避免传递来自用户程序的无效参数，这可能导致崩溃。

本节中描述的所有符号都在头文件中声明printf.h 。

功能： size_t parse_printf_format （const char * template ，size_t n ，int * argtypes ）

初步： MT安全区域| AS安全交流安全请参阅POSIX安全概念 。

此函数返回有关预期的参数数量和类型的信息。 printf模板字符串template 。信息存储在数组中argtypes ;此数组的每个元素都描述一个参数。该信息使用各种“ PA_ '宏，在下面列出。

争论n指定数组中元素的数量argtypes 。这是最大的元素数量parse_printf_format会尝试写。

parse_printf_format返回所需的参数总数template 。如果此数字大于n ，那么返回的信息仅描述第一个n论点。如果需要有关其他参数的信息，请分配更大的数组并调用parse_printf_format再次。

参数类型被编码为基本类型和修饰符标志位的组合。

巨集： int PA_FLAG_MASK

该宏是类型修饰符标志位的位掩码。你可以写表达式(argtypes[i] & PA_FLAG_MASK)仅提取参数的标志位，或(argtypes[i] & ~PA_FLAG_MASK)仅提取基本类型代码。

以下是代表基本类型的符号常量；它们代表整数值。

PA_INT

这指定基本类型为int 。

PA_CHAR

这指定基本类型为int ，投放至char 。

PA_STRING

这指定基本类型为char * ，以空值结尾的字符串。

PA_POINTER

这指定基本类型为void * ，一个任意指针。

PA_FLOAT

这指定基本类型为float 。

PA_DOUBLE

这指定基本类型为double 。

PA_LAST

您可以为自己的程序定义其他基本类型，作为与PA_LAST 。例如，如果您有数据类型“ foo '和' bar '自己专门printf转换，您可以将这些类型的编码定义为：

#define PA_FOO  PA_LAST
#define PA_BAR  (PA_LAST + 1)

这是修改基本类型的标志位。它们使用inclusive-or结合基本类型的代码。

PA_FLAG_PTR

如果设置此位，则表示编码类型是指向基本类型的指针，而不是立即值。例如， ' PA_INT|PA_FLAG_PTR '代表类型' int * '。

PA_FLAG_SHORT

如果设置了此位，则表明使用short 。（这对应于“ h '类型修饰符。）

PA_FLAG_LONG

如果设置了该位，则表明使用long 。（这对应于“ l '类型修饰符。）

PA_FLAG_LONG_LONG

如果设置了此位，则表明使用long long 。（这对应于“ L '类型修饰符。）

PA_FLAG_LONG_DOUBLE

这是的同义词PA_FLAG_LONG_LONG ，按常规使用，基本类型为PA_DOUBLE表示一种long double 。
