12.15文件结束和错误

本章描述的许多函数都返回宏的值EOF表示操作未成功完成。以来EOF用于报告文件结尾和随机错误，通常最好使用feof用于显式检查文件结尾的功能，以及ferror检查错误。这些功能检查作为流对象内部状态一部分的指示符，这些指示符设置是否通过对该流进行的先前I / O操作检测到适当的条件。

宏： int EOF

此宏是一个整数值，它由许多窄流函数返回以指示文件结束条件或某些其他错误情况。有了GNU C库， EOF是-1 。在其他库中，其值可能是其他负数。

该符号在stdio.h 。

宏： int WEOF

此宏是一个整数值，由许多宽流函数返回以指示文件结束条件或某些其他错误情况。有了GNU C库， WEOF是-1 。在其他库中，其值可能是其他负数。

该符号在wchar.h 。

函数： int feof （FILE * stream ）

初步： MT安全| AS安全AC不安全锁|请参阅POSIX安全概念 。

的feof当且仅当流的文件结束指示符时，函数返回非零stream被设置。

该符号在stdio.h 。

功能： int feof_unlocked （FILE * stream ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的feof_unlocked功能相当于feof函数，但它不会隐式锁定流。

此函数是GNU扩展。

该符号在stdio.h 。

函数： int ferror （文件* stream ）

初步： MT安全| AS安全AC不安全锁|请参阅POSIX安全概念 。

的ferror当且仅当流的错误指示符时，函数返回非零stream设置为1，表示在对该流的先前操作上已发生错误。

该符号在stdio.h 。

函数： int ferror_unlocked （文件* stream ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的ferror_unlocked功能相当于ferror函数，但它不会隐式锁定流。

此函数是GNU扩展。

该符号在stdio.h 。

除了设置与流相关的错误指示符之外，还设置了对流进行操作的功能errno与对文件描述符进行操作的相应低级函数相同。例如，执行输出到流的所有功能，例如fputc ， printf和fflush -在以下方面实施write ，以及所有errno为以下条件定义的错误条件write对这些功能有意义。有关描述符级I / O功能的更多信息，请参见低级I / O。
