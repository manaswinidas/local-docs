20.8.6浮点比较功能

当一个或其他操作数为NaN时，标准C比较运算符会引发异常。例如，

int v = a < 1.0;

如果出现以下情况将引发异常a是NaN。（这不会发生与==和!= ;在检查NaN时，它们分别分别返回false和true。）通常，此异常是不希望的。因此，ISO C99 定义了比较函数，这些函数在检查NaN时不会引发异常。所有功能都实现为宏，从而允许其参数为任何浮点类型。保证宏仅对它们的参数求值一次。TS 18661-1：2014为相等性比较添加了这样一个宏，该宏的确会引起NaN参数的异常；它还添加了功能，可对包括NaN在内的所有浮点值进行总排序，即使发出NaN信号也不会引发任何异常。

宏： int isgreater （ 实数浮动 x ， 实际浮动 y ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

此宏确定参数是否x大于y 。相当于(x) > (y) ，但如果x要么y是NaN。

宏： int isgreaterequal （ 实数浮点数 x ， 实际浮动 y ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

此宏确定参数是否x大于或等于y 。相当于(x) >= (y) ，但如果x要么y是NaN。

宏： int isless （ 实数浮动 x ， 实际浮动 y ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

此宏确定参数是否x小于y 。相当于(x) < (y) ，但如果x要么y是NaN。

宏： int islessequal （ 实数浮点数 x ， 实际浮动 y ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

此宏确定参数是否x小于或等于y 。相当于(x) <= (y) ，但如果x要么y是NaN。

Macro： int islessgreater （ 实数浮动 x ， 实际浮动 y ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

此宏确定参数是否x小于或大于y 。相当于(x) < (y) || (x) > (y) （尽管它只评估x和y一次），但如果x要么y是NaN。

这个宏不等于x != y ，因为如果x要么y是NaN。

宏： int isunordered （ 实浮点） x ， 实际浮动 y ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

该宏确定其参数是否无序。换句话说，如果x要么y是NaN，否则为false。

巨集： int iseqsig （ 实数浮动 x ， 实际浮动 y ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

该宏确定其参数是否相等。相当于(x) == (y) ，但会引发无效的异常并进行设置errno至EDOM如果任一自变量是NaN。

函数： int totalorder （双 精度 x ，双y ）
函数： int totalorderf （浮点数x ，浮动y ）
函数： int totalorderl （长 整数 x长双y ）
函数： int totalorderfN （_Float Nx ，_Float Ny ）
函数： int totalorderfNx （_Float N X x ，_Float N X y ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

这些函数确定IEEE 754-2008中定义的总订单关系是否适用于x和y ，如果为true，则返回非零；如果为false，则返回零。即使发出信号NaN也没有异常。如果它们是相同的浮点值（包括零和NaN的符号，以及NaN的有效载荷），则关系为真，或者x来之前y按以下顺序：负静默NaN，以减少有效载荷的顺序；负信令NaN，以有效载荷递减的顺序；负无穷大有限数，按升序排列，负零先于正零；正无穷大；积极的信号NaN，以增加有效载荷的顺序；积极安静的NaN，以增加有效载荷的顺序。

函数： int totalordermag （double x ，双y ）
函数： int totalordermagf （float x ，浮动y ）
函数： int totalordermagl （long double x长双y ）
函数： int totalordermagfN （_Float Nx ，_Float Ny ）
函数： int totalordermagfNx （_Float N X x ，_Float N X y ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

这些函数确定IEEE 754-2008中定义的总阶关系对于的绝对值是否成立。 x和y ，如果为true，则返回非零；如果为false，则返回零。即使发出信号NaN也没有异常。

并非所有机器都为这些操作提供硬件支持。在没有的机器上，宏可能会很慢。因此，当您不关心NaN时，不应使用这些功能。

注意：没有宏isequal要么isunequal 。它们是不必要的，因为==和!=如果一个或两个操作数均为NaN，则运算符不会引发异常。
