11.2.4文件名的可移植性

在讨论文件名的语法规则文件名 ，通常使用的GNU系统和其他POSIX系统的规则。但是，其他操作系统可能使用其他约定。

注意文件名可移植性问题对于您而言很重要的原因有两个：

  • 如果您的程序对文件名语法进行了假设，或包含嵌入式文字文件名字符串，则使其很难在使用不同语法约定的其他操作系统下运行。
  • 即使您不担心在运行其他操作系统的计算机上运行程序，也仍然可以访问使用不同命名约定的文件。例如，您可能能够通过网络访问另一台运行不同操作系统的计算机上的文件系统，或以其他操作系统使用的格式读写磁盘。

ISO C 标准几乎没有说明文件名语法，只是文件名是字符串。除了对文件名长度和可以在文件名中有效显示哪些字符的限制有所不同之外，不同的操作系统还对结构化目录，文件类型或扩展名等概念使用不同的约定和语法。有些概念（例如文件版本）可能在某些操作系统中受支持，而在其他操作系统中则不受支持。

POSIX.1标准允许实现对文件名语法进行附加限制，涉及文件名中允许使用哪些字符以及文件名和文件名组件字符串的长度。但是，在GNU系统上，文件名字符串中允许使用除空字符之外的任何字符，并且在GNU / Hurd系统上，文件名字符串的长度没有限制。
