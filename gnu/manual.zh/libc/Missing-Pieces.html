28.5.7丢失的碎片

本章中包含的示例Shell的代码摘录只是整个Shell程序的一部分。特别是，关于如何job和program数据结构被分配和初始化。

大多数实际的Shell提供了一个复杂的用户界面，该界面支持命令语言。变量文件名的缩写，替换和模式匹配；等等。所有这些都太复杂了，在这里无法解释！相反，我们专注于展示如何实现可从此类Shell调用的核心流程创建和作业控制功能。

下表总结了我们介绍的主要切入点：

void init_shell (void)

初始化外壳的内部状态。请参阅初始化Shell 。

void launch_job (job *j, int foreground)

启动工作j作为前台或后台作业。请参阅启动作业 。

void do_job_notification (void)

检查并报告已终止或停止的所有作业。可以同步调用，也可以在处理程序中调用SIGCHLD信号。请参阅停止和终止的作业 。

void continue_job (job *j, int foreground)

继续工作j 。请参阅继续停止的作业 。

当然，真正的外壳还希望提供其他功能来管理作业。例如，使用命令列出所有活动作业或发送信号（例如SIGKILL ）去工作。
