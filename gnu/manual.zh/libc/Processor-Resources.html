22.5了解可用的处理器

将线程或进程与共享内存一起使用允许应用程序利用系统可以提供的所有处理能力。如果任务可以并行化，则编写应用程序的最佳方法是随时运行与有处理器一样多的进程。要确定可用于系统的处理器数量

  sysconf (_SC_NPROCESSORS_CONF)

它返回操作系统配置的处理器数量。但是操作系统可能会禁用单个处理器，因此调用

  sysconf (_SC_NPROCESSORS_ONLN)

返回当前在线（即可用）的处理器数量。

对于这两条信息，GNU C库还提供了直接获取信息的功能。这些函数在中声明sys/sysinfo.h 。

函数： int get_nprocs_conf （void）

初步： MT安全| AS不安全堆锁| AC不安全锁fd mem |请参阅POSIX安全概念 。

的get_nprocs_conf函数返回操作系统配置的处理器数量。

此函数是GNU扩展。

函数： int get_nprocs （无效）

初步： MT安全| AS安全交流安全fd |请参阅POSIX安全概念 。

的get_nprocs函数返回可用处理器的数量。

此函数是GNU扩展。

在启动更多线程之前，应检查处理器是否尚未过度使用。Unix系统会计算平均负载 。该数字表示正在运行多少个进程。此数字是不同时间段（通常为1、5和15分钟）的平均值。

函数： int getloadavg （double loadavg []，int nelem ）

初步： MT安全| AS安全交流安全fd |请参阅POSIX安全概念 。

此功能获取系统的1、5和15分钟的平均负载。值放在loadavg 。 getloadavg将最多放置nelem元素放入数组中，但不得超过三个元素。返回值是写入的元素数loadavg ，或-1表示错误。

该函数在stdlib.h 。
