24.3.6初始信号动作

创建新流程（请参阅创建流程 ）后，它将继承其父流程的信号处理。但是，当您使用exec函数（请参阅执行文件 ），您定义了自己的处理程序的所有信号都将还原为它们SIG_DFL处理。（如果稍微考虑一下，这是有道理的；旧程序中的处理程序函数特定于该程序，甚至不存在于新程序映像的地址空间中。）当然，新程序可以建立自己的处理程序。

当程序由外壳运行时，外壳通常会将子进程的初始操作设置为SIG_DFL要么SIG_IGN ， 作为适当的。最好检查一下以确保外壳未设置以下命令的初始操作： SIG_IGN在建立自己的信号处理程序之前。

这是一个如何为SIGHUP ，但如果SIGHUP目前被忽略：

…
struct sigaction temp;

sigaction (SIGHUP, NULL, &temp);

if (temp.sa_handler != SIG_IGN)
  {
    temp.sa_handler = handle_sighup;
    sigemptyset (&temp.sa_mask);
    sigaction (SIGHUP, &temp, NULL);
  }
