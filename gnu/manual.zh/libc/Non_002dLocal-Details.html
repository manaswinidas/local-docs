23.2非本地出口的详细信息

以下是用于执行非本地出口的功能和数据结构的详细信息。这些设施在setjmp.h 。

数据类型： jmp_buf

类型的对象jmp_buf保留要通过非本地出口恢复的状态信息。一个的内容jmp_buf确定要返回的特定地点。

宏： int setjmp （jmp_buf state ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

通常情况下， setjmp将有关程序执行状态的信息存储在state并返回零。如果longjmp以后用于对此执行非本地退出state ， setjmp返回一个非零值。

函数： void longjmp （jmp_buf state ，int value ）

初步： MT安全| AS-Unsafe插件损坏了锁/锁|不安全的AC损坏锁/锁|请参阅POSIX安全概念 。

该功能将当前执行恢复到保存的状态state ，并从继续执行setjmp确定了返回点。从返回setjmp通过longjmp返回value传递给的论点longjmp ， 而不是0 。（但如果value给出为0 ， setjmp退货1 ）。

在使用上有很多晦涩但重要的限制setjmp和longjmp 。存在这些限制中的大多数限制是因为非本地出口在C编译器方面需要大量的魔术，并且可以以奇怪的方式与语言的其他部分进行交互。

的setjmp函数实际上是没有实际函数定义的宏，因此您不应该尝试“ #undef '或取其地址。此外， setjmp仅在以下情况下是安全的：

  • 作为选择或迭代语句的测试表达式（例如' if '，' switch '， 要么 ' while '）。
  • 作为相等或比较运算符的一个操作数，显示为选择或迭代语句的测试表达式。另一个操作数必须是整数常量表达式。
  • 作为一元'的操作数! '运算符，它显示为选择或迭代语句的测试表达式。
  • 本身作为表达式语句。

返回点仅在调用函数的动态范围内有效setjmp建立他们。如果你longjmp到在已经返回的函数中建立的返回点，可能会发生不可预测的灾难性事件。

您应该使用非零value争论longjmp 。而longjmp拒绝传回零参数作为来自的返回值setjmp ，其目的是作为防止意外滥用的安全网，并不是真正好的编程风格。
