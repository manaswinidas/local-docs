14.1工作目录

每个进程都有一个与其关联的目录，称为其当前工作目录或简称为working directory ，用于解析相对文件名（请参阅File Name Resolution ）。

登录并开始新会话时，您的工作目录最初设置为与系统用户数据库中的登录帐户关联的主目录。您可以使用以下命令找到任何用户的主目录getpwuid要么getpwnam功能;请参阅用户数据库 。

用户可以使用诸如shell之类的命令来更改工作目录cd 。本节中描述的功能是这些命令和其他程序用于检查和更改工作目录的原语。

这些函数的原型在头文件中声明unistd.h 。

函数： char * getcwd （char * buffer ，size_t size ）

初步： MT安全| AS不安全堆| AC不安全的内存fd |请参阅POSIX安全概念 。

的getcwd函数返回表示当前工作目录的绝对文件名，并将其存储在字符数组中buffer您提供的。的size参数是如何告诉系统的分配大小buffer 。

该函数的GNU C库版本还允许您为buffer论点。然后getcwd自动分配缓冲区，就像malloc （请参阅无限制分配 ）。如果size大于零，则缓冲区那么大；否则，缓冲区必须足够大以保存结果。

返回值为buffer成功时返回null，失败时返回null。下列errno为此功能定义了错误条件：

EINVAL

的size参数为零， buffer不是空指针。

ERANGE

的size参数小于工作目录名称的长度。您需要分配更大的数组，然后重试。

EACCES

读取或搜索文件名组成部分的权限被拒绝。

您可以实现GNU的行为getcwd (NULL, 0) 仅使用以下行为getcwd ：

char *
gnu_getcwd ()
{
  size_t size = 100;

  while (1)
    {
      char *buffer = (char *) xmalloc (size);
      if (getcwd (buffer, size) == buffer)
        return buffer;
      free (buffer);
      if (errno != ERANGE)
        return 0;
      size *= 2;
    }
}

有关更多信息，请参见Malloc示例 。 xmalloc ，它不是库函数，而是大多数GNU软件中使用的惯用名称。

不推荐使用的功能： char * getwd （char * buffer ）

初步： MT安全| AS不安全堆i18n | AC不安全的内存fd |请参阅POSIX安全概念 。

这类似于getcwd ，但无法指定缓冲区的大小。GNU C库提供getwd仅与BSD向后兼容。

的buffer参数至少应该是一个指向数组的指针PATH_MAX字节长（请参阅文件限制 ）。在GNU / Hurd系统上，文件名的大小没有限制，因此该空间不一定足以容纳目录名。这就是为什么不推荐使用此功能的原因。

函数： char * get_current_dir_name （无效）

初步： MT安全环境| AS不安全堆| AC不安全的内存fd |请参阅POSIX安全概念 。

的get_current_dir_name功能基本上相当于getcwd (NULL, 0) ，但PWD首先检查环境变量，如果它实际上与当前目录相对应，则返回该值。如果由中的值描述的路径是可见的，这是一个细微的差异。 PWD正在使用一个或多个符号链接，在这种情况下， getcwd将解析符号链接，因此产生不同的结果。

此函数是GNU扩展。

函数： int chdir （const char * filename ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

此功能用于将进程的工作目录设置为filename 。

的正常成功返回值chdir是0 。值-1返回表示错误。的errno为此功能定义的错误条件是通常的文件名语法错误（请参见文件名错误 ），以及ENOTDIR如果文件filename不是目录。

函数： int fchdir （int filedes ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

此函数用于将进程的工作目录设置为与文件描述符关联的目录filedes 。

的正常成功返回值fchdir是0 。值-1返回表示错误。下列errno为此功能定义了错误条件：

EACCES

拒绝由命名的目录的读取权限dirname 。

EBADF

的filedes参数不是有效的文件描述符。

ENOTDIR

文件描述符filedes与目录没有关联。

EINTR

该函数调用被信号中断。

EIO

发生I / O错误。
