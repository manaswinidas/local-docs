21.3.2处理器时间查询

的times函数返回有关进程消耗的处理器时间的信息struct tms 对象，以及进程的CPU时间。请参阅时间基础 。您应该包含头文件sys/times.h使用此功能。

数据类型： struct tms

的tms结构用于返回有关处理时间的信息。它至少包含以下成员：

clock_t tms_utime

这是调用进程在执行其程序指令时使用的总处理器时间。

clock_t tms_stime

这是系统代表调用进程使用的处理器时间。

clock_t tms_cutime

这是tms_utime价值观和tms_cutime调用进程的所有终止子进程的值，其状态已通过以下方式报告给父进程wait要么waitpid ;请参阅过程完成 。换句话说，它表示执行调用过程中所有终止的子进程的指令所用的总处理器时间，不包括尚未由报告的子进程。 wait要么waitpid 。

clock_t tms_cstime

这类似于tms_cutime ，但代表系统代表调用进程的所有终止子进程使用的总处理器时间。

所有时间均以时钟滴答数为单位。与CPU时间不同，这些是实际时间。与任何事件无关。请参阅创建流程 。

巨集： int CLK_TCK

这是每秒时钟滴答数的过时名称。使用sysconf (_SC_CLK_TCK)代替。

函数： clock_t 次 （结构tms * buffer ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的times函数将调用过程的处理器时间信息存储在buffer 。

返回值是自从过去任意点以来（例如自系统启动以来）的时钟滴答数。 times退货(clock_t)(-1)表示失败。

便携性注： clock ISO C 标准指定了“ CPU时间”中描述的功能。的times函数是POSIX.1的功能。在GNU系统上，CPU时间被定义为等于tms_utime和tms_stime传回的栏位times 。
