C.3推荐的编译工具

我们建议在尝试构建GNU C库之前安装以下GNU工具：

  • GNU make 4.0或更高版本

    截至发布时间，GNU make 4.2.1是经过验证的最新版本，可用于构建GNU C库。

  • GCC 6.2或更高版本

    需要GCC 6.2或更高版本。通常，建议使用已知可用于构建GNU C库的最新版本的编译器，因为较新的编译器通常会产生更好的代码。从发布之日起，GCC 9.1.1是经过验证可用于构建GNU C库的最新编译器。

    对于多体系结构支持，建议使用已支持GNU间接功能构建的GCC。这样可以确保为IFUNC解析器选择的功能生成正确的调试信息。可以通过将GCC配置为“ --enable-gnu-indirect-function ”，或者默认情况下通过设置“ default_gnu_indirect_function 'GCC源文件中特定体系结构的变量gcc/config.gcc 。

    您可以使用任何喜欢的编译器来编译使用GNU C库的程序。

    检查常见问题解答以了解特定平台上的特殊编译器问题。

  • GNU binutils 2.25或更高版本

    您必须使用GNU binutils （as和ld）来构建GNU C库。目前没有其他汇编器或链接器具有必需的功能。截至发布时间，GNU binutils 2.31.1是经过验证的最新版本，可用于构建GNU C库。

  • GNU texinfo 4.7或更高版本

    要正确翻译和安装Texinfo文档，您需要此版本的texinfo包。早期版本无法理解文档中使用的所有标签，并且信息文件的安装机制不存在或工作方式不同。截至发布时间， texinfo 6.6是经过验证的最新版本，可用于构建GNU C库。

  • GNU awk 3.1.2或更高版本

    awk在多个地方用于生成文件。一些gawk使用扩展名，包括asorti功能，是在3.1.2版中引入的gawk 。截至发布时间， gawk 4.2.1版是经过验证的最新版本，可用于构建GNU C库。

  • GNU bison 2.7或更高版本

    bison用于生成yacc中的解析器代码intl子目录。截至发布时间， bison 3.0.5版是经过验证的最新版本，可用于构建GNU C库。

  • Perl 5

    Perl不是必需的，但如果有的话，它会在某些测试中使用，并且mtrace程序，以构建GNU C库手册。截止发行时间perl 5.28.2版是经过验证的最新版本，可用于构建GNU C库。

  • GNU sed 3.02或更高版本

    Sed在多个地方用于生成文件。大多数脚本可与任何版本的sed 。截至发布时间， sed 4.5版是经过验证的最新版本，可用于构建GNU C库。

  • Python 3.4或更高版本

    需要Python来构建GNU C库。截至发布时，Python 3.7.4已被验证可用于构建和测试GNU C库。

  • PExpect 4.0

    漂亮的打印机测试通过测试程序来驱动GDB，并将其输出与打印机的输出进行比较。PExpect用于捕获GDB的输出，并且应与系统中的Python版本兼容。截至发布时间为止，PExpect 4.3是最新经过验证可用于测试漂亮打印机的产品。

  • GDB 7.8或更高版本，支持Python 2.7 / 3.4或更高版本

    为了使用漂亮的打印机，需要为GDB本身配置Python支持。请注意，具有Python的系统并不意味着GDB支持它，也不意味着系统的Python和GDB具有相同的版本。截至发布时GNU debugger 8.3是经过验证的最新版本，可以测试漂亮的打印机。

    除非存在支持Python的Python，PExpect和GDB，否则打印机测试会将其报告为UNSUPPORTED 。请注意，某些打印机测试要求使用调试符号来编译GNU C库。

如果您更改任何configure.ac您还将需要的文件

  • GNU autoconf 2.69（完全）

如果您更改任何消息翻译文件，则需要

  • GNU gettext 0.10.36或更高版本

    截至发布时间，GNU gettext版本0.19.8.1是经验证可用于构建GNU C库的最新版本。

如果您使用补丁程序升级源代码树，则可能还需要这些软件包，尽管我们会尽量避免这样做。
