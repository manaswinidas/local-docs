17.4.9.4其他特殊字符

巨集： int VLNEXT

这是特殊控制字符数组中LNEXT字符的下标。 termios.c_cc[VLNEXT]拥有角色本身。

只有在以下情况下才能识别LNEXT字符IEXTEN已设置，但同时处于规范和非规范模式。它禁用用户键入的下一个字符的任何特殊含义。即使该角色通常可以执行某些编辑功能或生成信号，也将其视为普通字符。这是类似的C-q Emacs中的命令。“ LNEXT”代表“下一个文字”。

LNEXT字符通常是C-v 。

该字符在BSD系统和GNU / Linux和GNU / Hurd系统上可用。

巨集： int VDISCARD

这是特殊控制字符数组中DISCARD字符的下标。 termios.c_cc[VDISCARD]拥有角色本身。

只有在以下情况下才能识别DISCARD字符IEXTEN已设置，但同时处于规范和非规范模式。它的作用是切换丢弃输出标志。设置此标志后，将丢弃所有程序输出。设置该标志还会丢弃当前在输出缓冲区中的所有输出。键入任何其他字符将重置标志。

该字符在BSD系统和GNU / Linux和GNU / Hurd系统上可用。

巨集： int VSTATUS

这是特殊控制字符数组中STATUS字符的下标。 termios.c_cc[VSTATUS]拥有角色本身。

STATUS字符的作用是打印出有关当前进程如何运行的状态消息。

仅在规范模式下且仅在以下情况下才能识别状态字符NOKERNINFO未设置。

该字符仅在BSD系统和GNU / Hurd系统上可用。
