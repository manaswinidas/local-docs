3.2.4.2示例程序摘录

即使跟踪功能不会影响程序的运行时行为，调用它也不是一个好主意mtrace在所有程序中。试想一下，您使用以下命令调试程序mtrace调试会话中使用的所有其他程序也会跟踪它们malloc电话。所有程序的输出文件都是相同的，因此无法使用。因此，应该打电话给mtrace仅当编译用于调试时。因此，程序可以像这样启动：

#include <mcheck.h>

int
main (int argc, char *argv[])
{
#ifdef DEBUGGING
  mtrace ();
#endif
  …
}

如果您想在程序的整个运行期间跟踪调用，那么这就是全部。或者，您可以随时致电来停止跟踪muntrace 。甚至可以通过重新调用再次重新启动跟踪mtrace 。但这可能导致不可靠的结果，因为可能会调用未调用的函数。请注意，不仅应用程序使用了跟踪的函数，库（包括C库本身）也使用了这些函数。

最后一点也是为什么打电话给它不是一个好主意muntrace在程序终止之前。仅当程序从以下位置返回后，才通知库有关程序终止的信息main或致电exit因此无法释放在此之前使用的内存。

所以最好的办法就是打电话mtrace作为程序中的第一个函数，永远不会调用muntrace 。因此，该程序几乎可以跟踪malloc函数（由程序的构造函数或使用的库执行的那些调用除外）。
