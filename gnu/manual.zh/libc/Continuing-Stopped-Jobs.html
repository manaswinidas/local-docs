28.5.6继续停止的作业

Shell可以通过发送一个SIGCONT向其处理组发出信号。如果作业在前台继续进行，则外壳程序应首先调用tcsetpgrp使作业可以访问终端，并恢复保存的终端设置。在前台继续执行作业后，外壳程序应等待作业停止或完成，就好像该作业刚刚在前台启动一样。

该示例Shell程序使用同一对函数处理新创建的作业和后续的作业， put_job_in_foreground 和put_job_in_background 。这些功能的定义在“ 前景”和“背景”中给出。继续停止的作业时，将传递非零值作为cont参数以确保SIGCONT发送信号，并根据需要重置终端模式。

这仅留下一个功能，用于更新Shell的内部簿记以了解正在进行的工作：

/* Mark a stopped job J as being running again.  */

void
mark_job_as_running (job *j)
{
  Process *p;

  for (p = j->first_process; p; p = p->next)
    p->stopped = 0;
  j->notified = 0;
}



/* Continue the job J.  */

void
continue_job (job *j, int foreground)
{
  mark_job_as_running (j);
  if (foreground)
    put_job_in_foreground (j, 1);
  else
    put_job_in_background (j, 1);
}
