17.1识别终端

本章中描述的功能仅适用于与终端设备相对应的文件。您可以使用以下命令查找文件描述符是否与终端相关联isatty功能。

本节中函数的原型在头文件中声明unistd.h 。

函数： int isatty （int filedes ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

该函数返回1如果filedes是与开放终端设备关联的文件描述符，否则为0 。

如果文件描述符与终端相关联，则可以使用ttyname功能。另请参阅ctermid功能，如“ 识别终端”中所述 。

函数： char * ttyname （int filedes ）

初步： MT不安全种族：ttyname | AS不安全堆锁| AC不安全锁fd mem |请参阅POSIX安全概念 。

如果文件描述符filedes与终端设备相关联， ttyname函数返回一个指针，该指针指向包含终端文件的文件名的静态分配的，以空终止的字符串。如果文件描述符未与终端关联，或者无法确定文件名，则该值为空指针。

函数： int ttyname_r （int filedes ，字符* buf ，size_t len ）

初步： MT安全| AS不安全堆| AC不安全的内存fd |请参阅POSIX安全概念 。

的ttyname_r功能类似于ttyname函数，除了将结果放入用户指定的缓冲区（从处开始）之外buf与长度len 。

的正常返回值ttyname_r是0 。否则，将返回错误编号以指示错误。下列errno为此功能定义了错误条件：

EBADF

的filedes参数不是有效的文件描述符。

ENOTTY

的filedes与终端无关。

ERANGE

缓冲区长度len太小，无法存储要返回的字符串。

ENODEV

的filedes终端设备与作为从属伪终端的终端设备相关联，但是无法确定与该设备相关联的文件名。这是一个GNU扩展。
