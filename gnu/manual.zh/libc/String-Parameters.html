32.12字符串值参数

POSIX.2定义了一种使用函数从操作系统获取字符串值参数的方法confstr ：

函数： size_t confstr （int parameter ，字符* buf ，size_t len ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

此函数读取字符串值系统参数的值，并将字符串存储到len起始的存储空间字节buf 。的parameter参数应为“ _CS_ '符号在下面列出。

的正常返回值confstr是您要求的字符串值的长度。如果您为提供空指针buf ， 然后confstr不尝试存储字符串；它只是返回其长度。值0表示错误。

如果您要求的字符串对于缓冲区而言太长（即，比len - 1 ）， 然后confstr存储那么多（为终止的空字符留出空间）。您可以说这是因为confstr返回大于或等于的值len 。

下列errno为此功能定义了错误条件：

EINVAL

的价值parameter是无效的。

当前只有一个参数可以读取confstr ：

_CS_PATH

该参数的值是搜索可执行文件的建议默认路径。这是用户登录后默认拥有的路径。

_CS_LFS_CFLAGS

返回的字符串指定如果使用C＃编译源，则必须向C编译器提供哪些附加标志。 _LARGEFILE_SOURCE功能选择宏；请参阅功能测试宏 。

_CS_LFS_LDFLAGS

返回的字符串指定如果使用编译源，则必须给链接器附加哪些标志。 _LARGEFILE_SOURCE功能选择宏；请参阅功能测试宏 。

_CS_LFS_LIBS

返回的字符串指定如果使用编译源，则必须将哪些其他库链接到应用程序。 _LARGEFILE_SOURCE功能选择宏；请参阅功能测试宏 。

_CS_LFS_LINTFLAGS

返回的字符串指定如果使用以下命令编译源，则必须给lint工具提供哪些附加标志： _LARGEFILE_SOURCE功能选择宏；请参阅功能测试宏 。

_CS_LFS64_CFLAGS

返回的字符串指定如果使用C＃编译源，则必须向C编译器提供哪些附加标志。 _LARGEFILE64_SOURCE功能选择宏；请参阅功能测试宏 。

_CS_LFS64_LDFLAGS

返回的字符串指定如果使用编译源，则必须给链接器附加哪些标志。 _LARGEFILE64_SOURCE功能选择宏；请参阅功能测试宏 。

_CS_LFS64_LIBS

返回的字符串指定如果使用编译源，则必须将哪些其他库链接到应用程序。 _LARGEFILE64_SOURCE功能选择宏；请参阅功能测试宏 。

_CS_LFS64_LINTFLAGS

返回的字符串指定如果使用以下命令编译源，则必须给lint工具提供哪些附加标志： _LARGEFILE64_SOURCE功能选择宏；请参阅功能测试宏 。

使用方式confstr对字符串大小没有任何限制的情况是调用它两次：首先调用它以获取长度，相应地分配缓冲区，然后调用confstr再次填充缓冲区，如下所示：

char *
get_default_path (void)
{
  size_t len = confstr (_CS_PATH, NULL, 0);
  char *buffer = (char *) xmalloc (len);

  if (confstr (_CS_PATH, buf, len + 1) == 0)
    {
      free (buffer);
      return NULL;
    }

  return buffer;
}
