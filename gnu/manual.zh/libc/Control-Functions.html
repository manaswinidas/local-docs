20.7浮点控制功能

IEEE 754 浮点实现允许程序员通过在控制字中设置位来决定是否为每个异常发生陷阱。在C语言中，陷阱导致程序接收到SIGFPE信号;请参阅信号处理 。

注意： IEEE 754 表示，陷阱处理程序会获得特殊情况的详细信息，并可设置结果值。C信号不提供任何机制来回传递此信息。因此，在C中捕获异常不是很有用。

在执行某些计算时，有时必须保存浮点单元的状态。该库提供的功能可以保存和还原异常标志，生成陷阱的异常集以及舍入模式。此信息称为浮点环境 。

保存和还原浮点环境的函数都使用类型为变量的变量fenv_t存储信息。此类型在fenv.h 。它的大小和内容是实现定义的。您不应尝试直接操作此类型的变量。

要保存FPU的状态，请使用以下功能之一：

函数： int fegetenv （fenv_t * envp ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

将浮点环境存储在指向的变量中envp 。

如果操作成功，该函数将返回零，否则返回非零值。

函数： int feholdexcept （fenv_t * envp ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

将当前浮点环境存储在指向的对象中envp 。然后清除所有异常标志，并将FPU设置为不捕获任何异常。并非所有的FPU都支持不捕获异常。如果feholdexcept无法设置此模式，它将返回非零值。如果成功，则返回零。

还原浮点环境的函数可以采用以下类型的参数：

  • 指向fenv_t对象，这些对象先前是通过调用来初始化的fegetenv要么feholdexcept 。
  • 特殊宏FE_DFL_ENV它代表程序启动时可用的浮点环境。
  • 以名称开头的实现定义的宏FE_并具有类型fenv_t * 。

    如果可能，GNU C库定义一个宏FE_NOMASK_ENV它代表了一个环境，其中引发的每个异常都会导致陷阱的发生。您可以使用以下命令测试此宏#ifdef 。仅在以下情况下定义_GNU_SOURCE被定义为。

    一些平台可能定义其他预定义环境。

要设置浮点环境，可以使用以下功能之一：

函数： int fesetenv （const fenv_t * envp ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

将浮点环境设置为envp 。

如果操作成功，该函数将返回零，否则返回非零值。

函数： int feupdateenv （const fenv_t * envp ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

喜欢fesetenv ，此功能会将浮点环境设置为envp 。但是，如果之前在状态字中标记了任何异常feupdateenv被呼叫后，呼叫后仍会被标记。换句话说，之后feupdateenv被调用时，状态字是前一个状态字与保存在其中的状态字的按位或envp 。

如果操作成功，该函数将返回零，否则返回非零值。

TS 18661-1：2014定义了其他功能，用于保存和恢复浮点控制模式（例如，舍入模式以及是否启用陷阱），而其他状态（例如凸起的标志）保持不变。

特殊宏FE_DFL_MODE可能会传递给fesetmode 。它代表程序启动时的浮点控制模式。

函数： int fegetmode （femode_t * modep ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

将浮点控制模式存储在由指向的变量中modep 。

如果操作成功，该函数将返回零，否则返回非零值。

函数： int fesetmode （const femode_t * modep ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

将浮点控制模式设置为modep 。

如果操作成功，该函数将返回零，否则返回非零值。

要控制个别异常（如果引发异常会导致陷阱），可以使用以下两个功能。

可移植性注意：这些功能都是GNU扩展。

函数： int Feenableexcept （int excepts ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

此功能为参数所指示的每个异常启用陷阱excepts 。状态位操作中描述了各个异常。仅启用指定的例外，其他例外的状态不变。

如果操作成功，该函数将返回先前启用的异常， -1除此以外。

函数： int fedisableexcept （int excepts ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

此功能禁用参数所指示的每个异常的陷阱excepts 。状态位操作中描述了各个异常。仅禁用指定的例外，其他例外的状态不变。

如果操作成功，该函数将返回先前启用的异常， -1除此以外。

函数： int fegetexcept （无效）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

该函数返回所有当前启用的异常的位掩码。它返回-1以防万一。
