30.14.2查找一组

您可以使用以下方法在组数据库中搜索有关特定组的信息： getgrgid要么getgrnam 。这些函数在中声明grp.h 。

功能： 结构组* getgrgid （gid_t gid ）

初步： MT不安全种族：grgid语言环境| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

该函数返回一个指向静态分配结构的指针，该结构包含有关其组ID为gid 。后续的调用可能会覆盖此结构getgrgid 。

空指针表示没有ID的组gid 。

函数： int getgrgid_r （gid_t gid ，结构组* result_buf ，字符* buffer ，size_t buflen ，结构组** result ）

初步： MT安全区域| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

该功能类似于getgrgid它返回有关其组ID为的组的信息gid 。但是，它填充了用户指向的结构result_buf信息而不是使用静态缓冲区。首先buflen指向的附加缓冲区的字节buffer用于包含其他信息，通常是结果结构的元素所指向的字符串。

如果有ID的组gid找到后，指针返回result指向包含所需数据的记录（即， result包含值result_buf ）。如果未找到组或发生错误，则指针返回result是一个空指针。该函数返回零或错误代码。如果缓冲buffer太小，无法包含所有必需的信息，错误代码ERANGE返回并errno被设定为ERANGE 。

功能： 结构体组* getgrnam （const char * name ）

初步： MT不安全种族：grnam语言环境| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

该函数返回一个指向静态分配结构的指针，该结构包含有关组名称为name 。后续的调用可能会覆盖此结构getgrnam 。

空指针表示没有命名的组name 。

函数： int getgrnam_r （const char * name ，结构组* result_buf ，字符* buffer ，size_t buflen ，结构组** result ）

初步： MT安全区域| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

该功能类似于getgrnam因为它返回有关其组名为的组的信息name 。喜欢getgrgid_r ，它使用用户提供的缓冲区result_buf和buffer ，而不是静态缓冲区。

返回值与相同getgrgid_r 。
