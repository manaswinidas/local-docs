12.14.3输入转换表

下表总结了各种转换规范：

' %d '

匹配以十进制表示的可选带符号整数。请参见数字输入转换 。

' %i '

匹配C语言定义的用于指定整数常量的任何格式的可选带符号整数。请参见数字输入转换 。

' %o '

匹配以八进制基数写入的无符号整数。请参见数字输入转换 。

' %u '

匹配以十进制基数编写的无符号整数。请参见数字输入转换 。

' %x '，' %X '

匹配以十六进制基数写入的无符号整数。请参见数字输入转换 。

' %e '，' %f '，' %g '，' %E '，' %G '

与可选签名的浮点数匹配。请参见数字输入转换 。

' %s '

匹配仅包含非空格字符的字符串。请参阅字符串输入转换 。'的存在l '修饰符确定输出是存储为宽字符串还是多字节字符串。如果“ %s '用于宽字符函数中，字符串被转换为多次调用wcrtomb转换为多字节字符串。这意味着缓冲区必须为MB_CUR_MAX每个宽字符读取的字节数。如果 ' %ls '在多字节函数中使用，结果被多次转换为宽字符mbrtowc然后再存储在用户提供的缓冲区中。

' %S '

这是“ %ls支持与Unix标准兼容。

' %[ '

匹配属于指定集合的字符串。请参阅字符串输入转换 。'的存在l '修饰符确定输出是存储为宽字符串还是多字节字符串。如果“ %[ '用于宽字符函数中，字符串被转换为多次调用wcrtomb转换为多字节字符串。这意味着缓冲区必须为MB_CUR_MAX每个宽字符读取的字节数。如果 ' %l[ '在多字节函数中使用，结果被多次转换为宽字符mbrtowc然后再存储在用户提供的缓冲区中。

' %c '

匹配一个或多个字符的字符串；读取的字符数由转换给出的最大字段宽度控制。请参阅字符串输入转换 。

如果“ %c在宽流函数中使用'，将读取的值在存储之前从宽字符转换为相应的多字节字符。请注意，此转换可能会产生一个以上的字节输出，因此提供的缓冲区必须足够大，最多可以MB_CUR_MAX每个字符的字节数。如果“ %lc在多字节函数中使用'，将输入视为多字节序列（而不是字节），并且将结果转换为对的调用mbrtowc 。

' %C '

这是“ %lc支持与Unix标准兼容。

' %p '

匹配指针值，该指针值与' %p的输出转换为printf 。请参阅其他输入转换 。

' %n '

此转换不读取任何字符；它记录了到目前为止通过此调用读取的字符数。请参阅其他输入转换 。

' %% '

这与文字' %输入流中的'字符。没有使用相应的参数。请参阅其他输入转换 。

如果转换规范的语法无效，则行为未定义。如果没有足够的函数参数为执行分配的模板字符串中的所有转换规范提供地址，或者参数的类型不正确，则该行为也未定义。另一方面，多余的参数将被忽略。
