32.9使用pathconf

当您的计算机允许不同文件的文件系统参数具有不同的值时，可以使用本节中的功能来找出适用于任何特定文件的值。

这些函数以及与之相关的常量parameter参数在头文件中声明unistd.h 。

函数： long int pathconf （const char * filename ，int parameter ）

初步： MT安全| AS不安全锁堆| AC不安全锁fd mem |请参阅POSIX安全概念 。

此函数用于查询适用于名为的文件的限制filename 。

的parameter参数应为“ _PC_ '常量在下面列出。

的正常返回值pathconf是您要求的值。值-1如果实现没有施加限制，以及发生错误，则返回。在前一种情况下， errno没有设置，而在后一种情况下， errno设置为指示问题的原因。因此，要稳定使用此功能的唯一方法是存储0进入errno就在打电话之前。

除了常见的文件名错误（请参阅“ 文件名错误” ），还为此函数定义了以下错误条件：

EINVAL

的价值parameter无效，或者实现不支持parameter用于特定文件。

函数： long int fpathconf （int filedes ，int parameter ）

初步： MT安全| AS不安全锁堆| AC不安全锁fd mem |请参阅POSIX安全概念 。

就像pathconf除了使用打开的文件描述符指定要为其请求信息的文件而不是文件名之外。

下列errno为此功能定义了错误条件：

EBADF

的filedes参数不是有效的文件描述符。

EINVAL

的价值parameter无效，或者实现不支持parameter用于特定文件。
