16.6.2.3主机地址功能

这些用于处理Internet地址的附加功能在头文件中声明arpa/inet.h 。它们以网络字节顺序表示Internet地址，以主机字节顺序表示网络号和网络内本地地址。见字节顺序 ，为网络和主机字节顺序的解释。

函数： int inet_aton （const char * name ，struct in_addr * addr ）

初步： MT安全区域| AS安全交流安全请参阅POSIX安全概念 。

此功能转换IPv4 Internet主机地址name从标准的点号符号转换为二进制数据并将其存储在struct in_addr那addr指着。 inet_aton如果地址有效，则返回非零；否则返回零。

功能： uint32_t inet_addr （const char * name ）

初步： MT安全区域| AS安全交流安全请参阅POSIX安全概念 。

此功能转换IPv4 Internet主机地址name从标准的数字符号转换为二进制数据。如果输入无效， inet_addr退货INADDR_NONE 。这是过时的界面inet_aton ，如上所述。这是过时的，因为INADDR_NONE是有效地址（255.255.255.255），并且inet_aton提供了一种更干净的方法来指示错误返回。

功能： uint32_t inet_network （const char * name ）

初步： MT安全区域| AS安全交流安全请参阅POSIX安全概念 。

此功能从地址中提取网络号name ，以标准的数字符号表示。返回的地址按主机顺序。如果输入无效， inet_network退货-1 。

该功能仅适用于传统的IPv4 A，B和C类网络类型。它不适用于无类地址，因此不应再使用。

功能： char * inet_ntoa （struct in_addr addr ）

初步： MT安全区域| AS不安全竞赛|交流安全请参阅POSIX安全概念 。

此功能转换IPv4 Internet主机地址addr转换为标准数字和点符号的字符串。返回值是指向静态分配缓冲区的指针。后续调用将覆盖相同的缓冲区，因此如果需要保存字符串，则应复制该字符串。

在多线程程序中，每个线程都有其自己的静态分配缓冲区。但随后的呼吁inet_ntoa在同一线程中将覆盖上一次调用的结果。

代替inet_ntoa较新的功能inet_ntop应该使用下面描述的方法，因为它既处理IPv4地址又处理IPv6地址。

函数： struct in_addr inet_makeaddr （uint32_t net ，uint32_t local ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

此功能通过组合网络号来生成IPv4 Internet主机地址net网络内本地地址local 。

函数： uint32_t inet_lnaof （结构in_addr addr ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

此函数返回Internet主机地址的本地网络内地址部分addr 。

该功能仅适用于传统的IPv4 A，B和C类网络类型。它不适用于无类地址，因此不应再使用。

函数： uint32_t inet_netof （结构in_addr addr ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

此函数返回Internet主机地址的网络号部分addr 。

该功能仅适用于传统的IPv4 A，B和C类网络类型。它不适用于无类地址，因此不应再使用。

函数： int inet_pton （int af ，const char * cp ，无效* buf ）

初步： MT安全区域| AS安全交流安全请参阅POSIX安全概念 。

此功能将Internet地址（IPv4或IPv6）从表示形式（文本）转换为网络（二进制）格式。 af应该是AF_INET要么AF_INET6 ，适用于要转换的地址类型。 cp是指向输入字符串的指针，并且buf是指向结果缓冲区的指针。调用者有责任确保缓冲区足够大。

函数： const char * inet_ntop （int af ，const void * cp ，字符* buf ，socklen_t len ）

初步： MT安全区域| AS安全交流安全请参阅POSIX安全概念 。

此功能将Internet地址（IPv4或IPv6）从网络（二进制）转换为表示形式（文本形式）。 af应该是AF_INET要么AF_INET6 ， 作为适当的。 cp是指向要转换的地址的指针。 buf应该是指向保存结果的缓冲区的指针，并且len是此缓冲区的长度。该函数的返回值将是该缓冲区地址。
