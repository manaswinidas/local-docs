1.3.2函数的宏定义

如果我们在本手册中将某些功能描述为功能，则它也可能具有宏定义。通常，这对程序的运行方式没有影响-宏定义与函数的作用相同。特别是，库函数的等效宏等效于对函数的一次评估，仅对参数进行一次评估。这些宏定义的主要原因是，有时它们可以产生比实际函数调用快得多的内联扩展。

即使将库函数的地址也定义为宏，也可以使用该地址。这是因为在这种情况下，函数的名称后没有左括号，这在语法上是识别宏调用所必需的。

您可能偶尔会希望避免使用函数的宏定义，这可能会使您的程序更易于调试。有两种方法可以执行此操作：

  • 您可以通过将函数名称放在括号中来避免在特定用途中使用宏定义。之所以可行，是因为该函数的名称未出现在可识别为宏调用的语法上下文中。
  • 您可以使用'取消显示整个源文件的任何宏定义。 #undef预处理程序指令，除非在该功能的描述中另有明确说明。

例如，假设头文件stdlib.h声明一个名为abs与

extern int abs (int);

并为abs 。然后，在：

#include <stdlib.h>
int f (int *i) { return abs (++*i); }

参考abs可能引用宏或函数。另一方面，在以下每个示例中，引用均是函数而不是宏。 

#include <stdlib.h>
int g (int *i) { return (abs) (++*i); }

#undef abs
int h (int *i) { return abs (++*i); }

由于对函数加倍的宏定义的行为与实际函数版本完全相同，因此通常不需要任何这些方法。实际上，删除宏定义通常只会使程序变慢。
