3.2.6.6增长对象

由于先后使用堆栈块中的内存，因此可以逐步建立对象，一次向对象末尾添加一个或多个字节。使用此技术，您无需知道要放入对象中的数据量，直到结束为止。我们称其为生长物体的技术。本节介绍将数据添加到生长对象的特殊功能。

开始种植物体时，您不需要做任何特别的事情。使用功能之一将数据添加到对象会自动启动它。但是，必须明确说出对象何时完成。这是通过功能完成的obstack_finish 。

这样完成的对象的实际地址在对象完成之前是未知的。在此之前，您总是有可能添加如此多的数据，以致必须将对象复制到新的块中。

当obstack用于增长的对象时，您不能将其用于其他对象的普通分配。如果尝试这样做，已经添加到增长对象中的空间将成为另一个对象的一部分。

功能： void obstack_blank （struct obstack * obstack-ptr ，int size ）

初步： MT安全竞赛：obstack-ptr | AS安全AC不安全的损坏的内存|请参阅POSIX安全概念 。

添加到生长物体的最基本功能是obstack_blank ，这会增加空间而不进行初始化。

功能： void obstack_grow （struct obstack * obstack-ptr ，无效* data ，int size ）

初步： MT安全竞赛：obstack-ptr | AS安全AC不安全的损坏的内存|请参阅POSIX安全概念 。

要添加一块初始化空间，请使用obstack_grow ，它是的增长对象类似物obstack_copy 。它增加了size字节数据到增长的对象，从中复制内容data 。

功能： void obstack_grow0 （struct obstack * obstack-ptr ，无效* data ，int size ）

初步： MT安全竞赛：obstack-ptr | AS安全AC不安全的损坏的内存|请参阅POSIX安全概念 。

这是...的增长对象类似物obstack_copy0 。它增加了size从复制的字节data ，后跟一个附加的空字符。

功能： void obstack_1grow （struct obstack * obstack-ptr ，字符c ）

初步： MT安全竞赛：obstack-ptr | AS安全AC不安全的损坏的内存|请参阅POSIX安全概念 。

要一次添加一个字符，请使用函数obstack_1grow 。它添加了一个包含c对增长的对象。

功能： void obstack_ptr_grow （struct obstack * obstack-ptr ，无效* data ）

初步： MT安全竞赛：obstack-ptr | AS安全AC不安全的损坏的内存|请参阅POSIX安全概念 。

一个指针的值加可以使用该功能obstack_ptr_grow 。它增加了sizeof (void *)包含值的字节data 。

功能： void obstack_int_grow （struct obstack * obstack-ptr ，int data ）

初步： MT安全竞赛：obstack-ptr | AS安全AC不安全的损坏的内存|请参阅POSIX安全概念 。

类型的单个值int可以通过使用obstack_int_grow功能。它增加了sizeof (int)个字节到增长的对象，并使用的值对其进行初始化data 。

功能： void * obstack_finish （struct obstack * obstack-ptr ）

初步： MT安全竞赛：obstack-ptr | AS安全AC不安全的腐败|请参阅POSIX安全概念 。

完成对象的生长后，请使用函数obstack_finish将其关闭并返回其最终地址。

完成对象后，即可将对象堆栈用于常规分配或增长另一个对象。

该函数可以在与以下条件相同的条件下返回空指针obstack_alloc （请参阅在Obstack中分配 ）。

当通过扩大对象来构建对象时，您可能需要随后知道它变成了多长时间。当您扩展对象时，无需跟踪它，因为您可以在使用函数完成对象之前从对象堆栈中找出长度obstack_object_size ，声明如下：

功能： int obstack_object_size （struct obstack * obstack-ptr ）

初步： MT安全竞赛：obstack-ptr | AS安全交流安全请参阅POSIX安全概念 。

此函数返回增长对象的当前大小（以字节为单位）。切记在完成对象之前调用此函数。完成后， obstack_object_size将返回零。

如果您开始增长对象并希望取消它，则应先完成然后释放它，如下所示：

obstack_free (obstack_ptr, obstack_finish (obstack_ptr));

如果没有物体在生长，这将无效。

您可以使用obstack_blank使用负的size参数使当前对象变小。只是不要尝试将其缩小到零长度以上，否则您将无法得知会发生什么。
