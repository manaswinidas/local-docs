12.21.2.2自定义流挂钩函数

以下是有关如何定义自定义流所需的四个挂钩函数的更多详细信息。

您应该将函数定义为从cookie读取数据，如下所示：

ssize_t reader (void *cookie, char *buffer, size_t size)

这非常类似于read功能;请参见I / O基本体 。您的功能应转移到size字节放入buffer ，并返回读取的字节数，或返回零以指示文件结束。您可以返回的值-1表示错误。

您应该定义将数据写入cookie的函数，如下所示：

ssize_t writer (void *cookie, const char *buffer, size_t size)

这非常类似于write功能;请参见I / O基本体 。您的功能应转移到size缓冲区中的字节数，并返回写入的字节数。您可以返回的值0表示错误。您不得返回任何负值。

您应将函数定义为对Cookie执行查找操作，如下所示：

int seeker (void *cookie, off64_t *position, int whence)

对于此功能， position和whence参数解释为fgetpos ;请参阅便携式定位 。

执行搜索操作后，您的函数应将相对于文件开头的结果文件位置存储在position 。您的函数应返回一个值0成功与-1表示错误。

您应该将函数定义为对适合关闭流的cookie进行清理操作，如下所示：

int cleaner (void *cookie)

您的函数应该返回-1表示错误，以及0除此以外。

数据类型： cookie_read_function_t

这是自定义流的读取功能应具有的数据类型。如果如上所述声明函数，则它将是该类型。

数据类型： cookie_write_function_t

自定义流的写函数的数据类型。

数据类型： cookie_seek_function_t

自定义流的搜索功能的数据类型。

数据类型： cookie_close_function_t

自定义流的关闭函数的数据类型。
