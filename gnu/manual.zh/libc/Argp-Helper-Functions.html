25.3.5.3在Argp解析器中使用的函数

Argp为argp用户提供了许多可用的功能（请参阅Argp分析器功能 ），主要用于产生错误消息。这些以他们的第一个论点state解析器函数的参数。请参阅Argp解析状态 。

功能： void argp_usage （const struct argp_state * state ）

初步： MT不安全种族：argpbuf env语言环境| AS-Unsafe堆i18n损坏| AC不安全的内存损坏锁|请参阅POSIX安全概念 。

输出所引用的argp解析器的标准用法消息state至state->err_stream并终止该程序exit (argp_err_exit_status) 。请参阅Argp全局变量 。

功能： void argp_error （const struct argp_state * state ，const char * fmt ，…）

初步： MT不安全种族：argpbuf env语言环境| AS-Unsafe堆i18n损坏| AC不安全的内存损坏锁|请参阅POSIX安全概念 。

打印printf格式字符串fmt以及后面的args，后跟程序名称和' : '，然后是' Try … --help ' 消息，并以退出状态终止该程序argp_err_exit_status 。请参阅Argp全局变量 。

功能： void argp_failure （const struct argp_state * state ，int status ，int errnum ，const char * fmt ，…）

初步： MT安全| AS不安全的损坏堆| AC不安全锁损坏了内存|请参阅POSIX安全概念 。

类似于标准的GNU错误报告功能error ，这会打印程序名称和“ : '，printf格式字符串fmt ，以及以下适当的参数。如果非零，则为errnum打印。如果status为非零，它将以该值作为退出状态终止程序。

和...之间的不同argp_failure和argp_error就是它argp_error用于解析错误 ，而argp_failure用于解析期间发生的其他问题，但不能反映输入的语法问题，例如选项的非法值，月亮的相位差等。

功能： void argp_state_help （const struct argp_state * state ，文件* stream ，未签名flags ）

初步： MT不安全种族：argpbuf env语言环境| AS-Unsafe堆i18n损坏| AC不安全的内存损坏锁|请参阅POSIX安全概念 。

输出由所引用的argp解析器的帮助消息state ， 至stream 。的flags参数确定生成哪种类型的帮助消息。请参阅Argp帮助标志 。

错误输出发送到state->err_stream ，并且打印的程序名称是state->name 。

如果以下功能可以抑制这些功能的输出或程序终止行为： ARGP_NO_EXIT要么ARGP_NO_ERRS标志传递给argp_parse 。参见Argp标志 。

如果导出argp解析器以供其他程序（例如，库）使用，则此行为很有用，并且可以在不需要响应解析错误而终止程序的上下文中使用。在用于此类常规用途的argp解析器中，并且在程序没有终止的情况下，对这些函数中的任何一个的调用都应跟随返回适当错误代码的代码：

if (bad argument syntax)
  {
     argp_usage (state);
     return EINVAL;
  }

如果仅在以下情况下使用解析器功能ARGP_NO_EXIT如果未设置，则可以省略返回值。
