3.2.6.9堆栈中的数据对齐

每个堆栈都有一个对齐边界 ; Obstack中分配的每个对象都会自动从指定边界倍数的地址开始。默认情况下，此边界是对齐的，以便对象可以保存任何类型的数据。

要访问对象堆栈的对齐边界，请使用宏obstack_alignment_mask ，其功能原型如下所示：

巨集： int obstack_alignment_mask （struct obstack * obstack-ptr ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

该值是位掩码。位为1表示对象地址中的对应位应为0。掩码值应小于2的幂。结果是所有对象地址都是2的幂的倍数。掩码的默认值是允许对齐的对象保存任何类型的数据的值：例如，如果其值为3，则可以将任何类型的数据存储在地址为4的倍数的位置。掩码值0表示对象可以以1的任意倍数开始（也就是说，不需要对齐）。

宏的扩展obstack_alignment_mask是左值，因此您可以通过赋值来更改蒙版。例如，此语句：

obstack_alignment_mask (obstack_ptr) = 0;

具有关闭指定对象堆栈中的对齐处理的效果。

请注意，对齐遮罩的更改要等到下一次在对象堆栈中分配或完成对象之后 ，才会生效。如果您不增加对象，则可以通过调用使新的对齐蒙版立即生效。 obstack_finish 。这将完成一个零长度的对象，然后对下一个对象进行适当的对齐。
