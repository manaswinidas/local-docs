13.12文件控制操作

本节介绍如何对文件描述符执行各种其他操作，例如查询或设置描述文件描述符状态的标志，操纵记录锁等。所有这些操作均由功能执行fcntl 。

第二个论点fcntl函数是指定执行哪个操作的命令。在头文件中声明了函数和宏，这些函数和宏命名与其一起使用的各种标志fcntl.h 。这些标志中的许多标志也被open功能;请参阅打开和关闭文件 。

函数： int fcntl （int filedes ，int command ，…）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的fcntl函数执行由指定的操作command在文件描述符上filedes 。某些命令需要提供其他参数。这些附加参数以及返回值和错误条件在各个命令的详细说明中给出。

简要地说，这是各种命令的清单。

F_DUPFD

复制文件描述符（返回另一个指向同一打开文件的文件描述符）。请参阅复制描述符 。

F_GETFD

获取与文件描述符关联的标志。请参阅描述符标志 。

F_SETFD

设置与文件描述符关联的标志。请参阅描述符标志 。

F_GETFL

获取与打开的文件关联的标志。请参阅文件状态标志 。

F_SETFL

设置与打开的文件关联的标志。请参阅文件状态标志 。

F_GETLK

测试文件锁。请参阅文件锁 。

F_SETLK

设置或清除文件锁定。请参阅文件锁 。

F_SETLKW

喜欢F_SETLK ，但要等待完成。请参阅文件锁 。

F_OFD_GETLK

测试一个打开的文件描述锁。请参阅打开文件描述锁 。特定于Linux。

F_OFD_SETLK

设置或清除打开的文件描述锁。请参阅打开文件描述锁 。特定于Linux。

F_OFD_SETLKW

喜欢F_OFD_SETLK ，但直到获得锁为止才阻塞。请参阅打开文件描述锁 。特定于Linux。

F_GETOWN

获取要接收的进程或进程组ID SIGIO信号。请参见中断输入 。

F_SETOWN

设置要接收的进程或进程组ID SIGIO信号。请参见中断输入 。

此功能是多线程程序中命令的取消点F_SETLKW （和LFS类似F_SETLKW64 ）和F_OFD_SETLKW 。如果线程当时分配了一些资源（例如内存，文件描述符，信号量或其他内容），这将是一个问题fcntl叫做。如果线程被取消，这些资源将保持分配状态，直到程序结束。为了避免这种情况fcntl应该使用取消处理程序进行保护。
