16.5.2本地命名空间的详细信息

要在本地名称空间中创建套接字，请使用常量PF_LOCAL作为namespace争论socket要么socketpair 。该常量在sys/socket.h 。

巨集： int PF_LOCAL

这指定了本地名称空间（其中套接字地址是本地名称）及其关联的协议系列。 PF_LOCAL是POSIX.1g使用的宏。

巨集： int PF_UNIX

这是的同义词PF_LOCAL ，出于兼容性考虑。

巨集： int PF_FILE

这是的同义词PF_LOCAL ，出于兼容性考虑。

在头文件中定义了用于在本地名称空间中指定套接字名称的结构sys/un.h ：

数据类型： struct sockaddr_un

此结构用于指定本地名称空间套接字地址。它具有以下成员：

short int sun_family

这标识了地址系列或套接字地址的格式。您应该存储值AF_LOCAL指定本地名称空间。请参阅套接字地址 。

char sun_path[108]

这是要使用的文件名。

不完整：为什么108是一个魔数？RMS建议将其设为零长度数组，并调整以下示例以供使用alloca根据文件名的长度分配适当的存储量。

您应该计算length本地名称空间中套接字地址的参数，作为sun_family组件和字符串长度（ 不是分配大小！）文件名字符串。这可以使用宏完成SUN_LEN ：

巨集： int SUN_LEN （ struct sockaddr_un * ptr ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

此宏计算本地名称空间中套接字地址的长度。
