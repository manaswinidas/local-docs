25.7.5内部端接

的_exit函数是用于终止进程的原语exit 。在头文件中声明unistd.h 。

函数： void _exit （int status ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的_exit函数是导致进程以状态终止的原语status 。调用此功能不会执行向其注册的清除功能atexit要么on_exit 。

函数： void _Exit （int status ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的_Exit函数是ISO C 等效于_exit 。ISO C 委员会成员不确定是否定义_exit和_Exit是兼容的，所以他们没有使用POSIX名称。

此函数在ISO C99 引入，并在stdlib.h 。

当进程由于任何原因（由于程序终止或由于信号终止）终止时，将发生以下情况：

  • 进程中所有打开的文件描述符都被关闭。请参阅低级I / O。请注意，当进程终止时，流不会自动刷新。参见Streams上的I / O。
  • 保存流程退出状态，以通过以下方式报告给父流程wait要么waitpid ;请参阅过程完成 。如果程序退出，则该状态包括程序退出状态作为其低8位。
  • 终止的任何子进程将被分配一个新的父进程。（在大多数系统上，包括GNU，这是init进程，进程ID为1。
  • 一种SIGCHLD信号发送到父进程。
  • 如果进程是具有控制终端的会话负责人，则SIGHUP信号发送到前台作业中的每个进程，并且控制终端与该会话解除关联。请参阅作业控制 。
  • 如果某个进程的终止导致一个进程组变为孤立状态，并且该进程组的任何成员都已停止，则SIGHUP信号和SIGCONT信号被发送到组中的每个进程。请参阅作业控制 。
