3.2.6.3在堆栈中分配

在堆中分配对象的最直接方法是obstack_alloc ，几乎就像malloc 。

功能： void * obstack_alloc （struct obstack * obstack-ptr ，int size ）

初步： MT安全竞赛：obstack-ptr | AS安全AC不安全的损坏的内存|请参阅POSIX安全概念 。

这分配了一个未初始化的块size堆中的字节数并返回其地址。这里obstack-ptr指定在哪个堆中分配块；它是地址struct obstack代表对象的对象。每个obstack函数或宏都需要您指定一个obstack-ptr作为第一个论点。

该函数调用obstack的obstack_chunk_alloc如果需要分配新的内存块，则执行该函数；它呼吁obstack_alloc_failed_handler如果通过分配内存obstack_chunk_alloc失败了

例如，这是一个分配字符串副本的函数str在特定的堆栈中，该变量在变量中string_obstack ：

struct obstack string_obstack;

char *
copystring (char *string)
{
  size_t len = strlen (string) + 1;
  char *s = (char *) obstack_alloc (&string_obstack, len);
  memcpy (s, string, len);
  return s;
}

要分配具有指定内容的块，请使用函数obstack_copy ，这样声明：

功能： void * obstack_copy （struct obstack * obstack-ptr ，无效* address ，int size ）

初步： MT安全竞赛：obstack-ptr | AS安全AC不安全的损坏的内存|请参阅POSIX安全概念 。

这将分配一个块并通过复制将其初始化size数据字节起始于address 。它呼吁obstack_alloc_failed_handler如果通过分配内存obstack_chunk_alloc失败了

功能： void * obstack_copy0 （struct obstack * obstack-ptr ，无效* address ，int size ）

初步： MT安全竞赛：obstack-ptr | AS安全AC不安全的损坏的内存|请参阅POSIX安全概念 。

喜欢obstack_copy ，但附加一个包含空字符的额外字节。该多余字节不计入参数中size 。

的obstack_copy0函数很方便地将字符序列作为以零结尾的字符串复制到堆中。这是一个用法示例：

char *
obstack_savestring (char *addr, int size)
{
  return obstack_copy0 (&myobstack, addr, size);
}

将此与之前的示例进行对比savestring使用malloc （请参阅基本分配 ）。
