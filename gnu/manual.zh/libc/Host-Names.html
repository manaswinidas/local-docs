16.6.2.4主机名

除了用于Internet地址的标准数字符号外，您还可以通过符号名称引用主机。象征性名称的优点是通常更容易记住。例如，Internet地址为“ 158.121.106.19也称为“ alpha.gnu.org ';和“ gnu.org '域可以简称为' alpha '。

在内部，系统使用数据库来跟踪主机名和主机号之间的映射。该数据库通常是文件/etc/hosts或名称服务器提供的等效名称。用于访问此数据库的函数和其他符号在中声明netdb.h 。它们是BSD功能，如果包含以下内容，则会无条件定义netdb.h 。

数据类型： struct hostent

此数据类型用于表示主机数据库中的条目。它具有以下成员：

char *h_name

这是主机的“正式”名称。

char **h_aliases

这些是主机的备用名称，表示为以空值终止的字符串向量。

int h_addrtype

这是主机地址类型；实际上，它的价值永远是AF_INET要么AF_INET6 ，后者用于IPv6主机。原则上，其他种类的地址也可以在数据库中表示，也可以在Internet地址中表示。如果这样做，您可能会在此字段中找到一个值，而不是AF_INET要么AF_INET6 。请参阅套接字地址 。

int h_length

这是每个地址的长度（以字节为单位）。

char **h_addr_list

这是主机地址的向量。（回想一下，主机可能连接到多个网络，并且每个网络上都有不同的地址。）向量由空指针终止。

char *h_addr

这是的同义词h_addr_list[0] ;换句话说，它是第一个主机地址。

就主机数据库而言，每个地址仅是一块内存h_length个字节长。但是在其他情况下，存在一个隐含的假设，即您可以将IPv4地址转换为struct in_addr或uint32_t 。主机地址struct hostent结构总是以网络字节顺序给出；参见字节顺序 。

您可以使用gethostbyname ， gethostbyname2要么gethostbyaddr在主机数据库中搜索有关特定主机的信息。信息以静态分配的结构返回；如果需要跨调用保存信息，则必须复制该信息。您也可以使用getaddrinfo和getnameinfo获取此信息。

功能： struct hostent * gethostbyname （const char * name ）

的gethostbyname函数返回有关主机的信息name 。如果查找失败，则返回空指针。

函数： struct hostent * gethostbyname2 （const char * name ，int af ）

初步： MT-不安全种族：hostbyname2 env语言环境| AS-Unsafe dlopen插件损坏了堆锁AC不安全锁损坏了内存fd |请参阅POSIX安全概念 。

的gethostbyname2功能就像gethostbyname ，但允许呼叫者指定所需的地址族（例如AF_INET要么AF_INET6 ）的结果。

函数： struct hostent * gethostbyaddr （const void * addr ，socklen_t length ，int format ）

初步： MT-不安全种族：hostbyaddr env语言环境| AS-Unsafe dlopen插件损坏了堆锁AC不安全锁损坏了内存fd |请参阅POSIX安全概念 。

的gethostbyaddr函数返回有关具有Internet地址的主机的信息addr 。参数addr实际上不是指向char的指针-它可以是指向IPv4或IPv6地址的指针。的length参数是地址的大小（以字节为单位） addr 。 format指定地址格式；对于IPv4 Internet地址，请指定值为AF_INET ;对于IPv6互联网地址，请使用AF_INET6 。

如果查找失败， gethostbyaddr返回一个空指针。

如果通过名称查找gethostbyname要么gethostbyaddr失败，您可以通过查看变量的值来找出原因h_errno 。（将这些功能设置为更简洁的设计errno ，但使用h_errno与其他系统兼容。）

这是您可能会发现的错误代码h_errno ：

HOST_NOT_FOUND

数据库中没有此类主机。

TRY_AGAIN

当无法联系名称服务器时，会发生这种情况。如果稍后再试，则可能会成功。

NO_RECOVERY

发生了不可恢复的错误。

NO_ADDRESS

主机数据库包含名称的条目，但没有关联的Internet地址。

首先，查找功能有一个共同点：它们不是可重入的，因此无法在多线程应用程序中使用。因此，为GNU C库提供了可以在这种情况下使用的一组新功能。

函数： int gethostbyname_r （const char * restrict name ，struct hostent *限制result_buf ，char *限制buf ，size_t buflen ，struct hostent **限制result ，int *限制h_errnop ）

初步： MT安全环境语言环境| AS-Unsafe dlopen插件损坏了堆锁AC不安全锁损坏了内存fd |请参阅POSIX安全概念 。

的gethostbyname_r函数返回有关主机的信息name 。调用者必须将指针传递给类型的对象struct hostent在里面result_buf参数。另外，该函数可能需要额外的缓冲区空间，并且调用方必须在其中传递一个指针和缓冲区的大小。 buf和buflen参数。

指向存储结果的缓冲区的指针在*result函数调用成功返回后。缓冲区作为buf只有在调用者完成了结果托管结构之后，或者已经复制了它（包括它指向的所有其他内存）后，才能释放该参数。如果发生错误或找不到条目，则指针*result是一个空指针。返回值为零表示成功。如果函数失败，则返回值为错误号。除了定义的错误gethostbyname也可以是ERANGE 。在这种情况下，应使用更大的缓冲区重复调用。其他错误信息未存储在全局变量中h_errno但是在由指向的对象中h_errnop 。

这是一个小例子：

struct hostent *
gethostname (char *host)
{
  struct hostent *hostbuf, *hp;
  size_t hstbuflen;
  char *tmphstbuf;
  int res;
  int herr;

  hostbuf = malloc (sizeof (struct hostent));
  hstbuflen = 1024;
  tmphstbuf = malloc (hstbuflen);

  while ((res = gethostbyname_r (host, hostbuf, tmphstbuf, hstbuflen,
                                 &hp, &herr)) == ERANGE)
    {
      /* Enlarge the buffer.  */
      hstbuflen *= 2;
      tmphstbuf = realloc (tmphstbuf, hstbuflen);
    }

  free (tmphstbuf);
  /*  Check for errors.  */
  if (res || hp == NULL)
    return NULL;
  return hp;
}

的gethostbyaddr_r函数返回有关具有Internet地址的主机的信息addr 。参数addr实际上不是指向char的指针-它可以是指向IPv4或IPv6地址的指针。的length参数是地址的大小（以字节为单位） addr 。 format指定地址格式；对于IPv4 Internet地址，请指定值为AF_INET ;对于IPv6互联网地址，请使用AF_INET6 。

类似于gethostbyname_r函数，调用者必须为结果和内部使用的内存提供缓冲区。如果成功，函数将返回零。否则，该值为错误号，其中ERANGE特殊含义是调用者提供的缓冲区太小。

您也可以一次扫描整个主机数据库一个条目sethostent ， gethostent和endhostent 。使用这些功能时请小心，因为它们不可重入。

函数： void sethostent （int stayopen ）

初步： MT不安全种族：托管环境AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

此功能将打开主机数据库以开始对其进行扫描。然后你可以打电话gethostent阅读条目。

如果stayopen参数为非零，这会设置一个标志，以便后续对gethostbyname要么gethostbyaddr不会关闭数据库（通常会关闭）。如果您多次调用这些函数，可以避免每次调用重新打开数据库，从而提高效率。

函数： struct hostent * gethostent （无效）

初步： MT不安全种族：hostent种族：hostentbuf env语言环境| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

该函数返回主机数据库中的下一个条目。如果没有更多条目，则返回空指针。

函数： void endhostent （无效）

初步： MT不安全种族：托管环境AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。
