14.6删除文件

您可以使用删除文件unlink要么remove 。

删除实际上删除了文件名。如果这是文件的唯一名称，则该文件也会被删除。如果文件还有其他名称（请参阅“ 硬链接” ），则在这些名称下仍可访问。

函数： int unlink （const char * filename ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的unlink函数删除文件名filename 。如果这是文件的唯一名称，则文件本身也会被删除。（实际上，如果发生这种情况时，如果有任何进程打开了文件，则删除将推迟到所有进程关闭文件为止。）

功能unlink在头文件中声明unistd.h 。

该函数返回0成功完成后，以及-1错误。除了常见的文件名错误（请参阅文件名错误 ）之外，以下内容errno为此功能定义了错误条件：

EACCES

要从中删除文件的目录的写许可权被拒绝，或者该目录设置了粘性位并且您不拥有该文件。

EBUSY

此错误表示文件正在被系统以无法取消链接的方式使用。例如，如果文件名指定了文件系统的根目录或安装点，则可能会看到此错误。

ENOENT

要删除的文件名不存在。

EPERM

在某些系统上unlink不能用于删除目录名称，或者至少只能由特权用户以这种方式使用。为避免此类问题，请使用rmdir删除目录。（在GNU / Linux和GNU / Hurd系统上unlink永远无法删除目录的名称。）

EROFS

包含要删除文件名的目录位于只读文件系统上，无法修改。

函数： int rmdir （const char * filename ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的rmdir函数删除目录。该目录必须为空才能删除。换句话说，它只能包含.和.. 。

在大多数其他方面， rmdir表现像unlink 。还有两个errno为以下条件定义的错误条件rmdir ：

ENOTEMPTY
EEXIST

要删除的目录不为空。

这两个错误代码是同义词。有些系统使用一个，而另一些使用。GNU / Linux和GNU / Hurd系统始终使用ENOTEMPTY 。

该函数的原型在头文件中声明unistd.h 。

函数： int remove （const char * filename ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

这是ISO C 函数，用于删除文件。它像unlink用于文件之类的rmdir用于目录。 remove在中声明stdio.h 。
