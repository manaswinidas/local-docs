17.4.9.2引起信号的字符

这些特殊字符可能在规范或非规范输入模式下处于活动状态，但仅当ISIG标志已设置（请参阅本地模式 ）。

巨集： int VINTR

这是特殊控制字符数组中INTR字符的下标。 termios.c_cc[VINTR]拥有角色本身。

INTR（中断）字符引发一个SIGINT与终端关联的前台作业中所有进程的信号。然后，将丢弃INTR字符本身。有关信号的更多信息，请参见信号处理 。

通常，INTR字符为C-c 。

巨集： int VQUIT

这是特殊控制字符数组中QUIT字符的下标。 termios.c_cc[VQUIT]拥有角色本身。

QUIT字符引发一个SIGQUIT与终端关联的前台作业中所有进程的信号。然后将QUIT字符本身丢弃。有关信号的更多信息，请参见信号处理 。

通常，QUIT字符为C-\ 。

巨集： int VSUSP

这是特殊控制字符数组中SUSP字符的下标。 termios.c_cc[VSUSP]拥有角色本身。

仅当实现支持作业控制时，才识别SUSP（挂起）字符（请参见作业控制 ）。它导致SIGTSTP信号发送到与终端关联的前台作业中的所有进程。然后，SUSP字符本身将被丢弃。有关信号的更多信息，请参见信号处理 。

通常，SUSP字符是C-z 。

很少有应用程序禁用SUSP字符的常规解释。如果您的程序这样做，它应该为用户提供其他机制来停止工作。当用户调用此机制时，程序应发送一个SIGTSTP向流程的流程组发出信号，而不仅仅是向流程本身发出信号。请参阅发信号通知另一个进程 。

巨集： int VDSUSP

这是特殊控制字符数组中DSUSP字符的下标。 termios.c_cc[VDSUSP]拥有角色本身。

仅当实现支持作业控制时，才识别DSUSP（挂起）字符（请参阅Job Control ）。它发送一个SIGTSTP仅当程序尝试将其读取为输入时，才发出信号，例如SUSP字符，但不是立即发出。并非所有具有作业控制功能的系统都支持DSUSP。仅兼容BSD的系统（包括GNU / Hurd系统）。

有关信号的更多信息，请参见信号处理 。

通常，DSUSP字符是C-y 。
