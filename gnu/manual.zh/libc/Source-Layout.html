D.1增加新功能

库的构建过程由makefile驱动，makefile大量使用了GNU的特殊功能make 。生成文件非常复杂，您可能不想尝试理解它们。但是它们的作用非常简单，只需要您在正确的位置定义一些变量即可。

库源分为子目录，按主题分组。

的string子目录具有所有的字符串操作功能， math具有所有的数学功能，等等。

每个子目录包含一个简单的makefile，称为Makefile ，它定义了一些make变量，然后包含全局makefile Rules像这样的一行：

include ../Rules

子目录makefile定义的基本变量是：

subdir

子目录的名称，例如stdio 。必须定义此变量。

headers

库的此部分中的头文件名，例如stdio.h 。

routines
aux

库此部分中的模块（源文件）的名称。这些应该是简单的名称，例如“ strlen '（而不是完整的文件名，例如strlen.c ）。使用routines用于定义库中函数的模块，以及aux用于包含诸如数据定义之类的辅助模块。但是的价值routines和aux只是串联在一起的，所以实际上没有实际区别。

tests

库的此部分的测试程序的名称。这些应该是简单的名称，例如“ tester '（而不是完整的文件名，例如tester.c ）。' make tests ' 将构建并运行所有测试程序。如果测试程序需要输入，请将测试数据放入一个名为test-program.input ;它会以其标准输入提供给测试程序。如果测试程序要使用参数运行，请将参数（全部放在一行中）放在一个名为test-program.args 。当测试通过时，测试程序应以零状态退出，而当测试表明库中有错误或构建错误时，测试程序应以非零状态退出。

others

与库的此部分关联的“其他”程序的名称。这些程序本身不是测试程序，而是库中附带的其他小程序。它们是由“ make others ' 。

install-lib
install-data
install

要由'安装的文件make install ' 。在“ install-lib '安装在'指定的目录中libdir在configparms要么Makeconfig （请参阅安装 ）。列出的文件install-data安装在由'指定的目录中datadir在configparms要么Makeconfig 。列出的文件install安装在由'指定的目录中bindir在configparms要么Makeconfig 。

distribute

该子目录中的其他文件应放入分发tar文件中。您无需在此处列出makefile本身或其他标准变量中列出的源文件和头文件。只定义distribute是否有不正常使用的文件应进入分发。

generated

生成的文件Makefile在此子目录中。这些文件将通过“ make clean ' ，它们将永远不会进入分发。

extra-objs

由构建的额外目标文件Makefile在此子目录中。这应该是文件名列表，例如foo.o ;实际上，可以在任何内置目录对象文件中找到这些文件。这些文件将通过“ make clean ' 。此变量用于构建所需的辅助对象文件others要么tests 。
