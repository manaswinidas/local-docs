13.13复制描述符

您可以复制一个文件描述符，或者分配另一个文件描述符，该文件描述符引用与原始文件相同的打开文件。重复的描述符共享一个文件位置和一组文件状态标志（请参阅文件状态标志 ），但是每个描述符都有其自己的一组文件描述符标志（请参见描述符标志 ）。

复制文件描述符的主要用途是实现输入或输出的重定向 ：即更改特定文件描述符所对应的文件或管道。

您可以使用fcntl与F_DUPFD命令，但也有方便的功能dup和dup2用于复制描述符。

的fcntl函数和标志在fcntl.h ，而dup和dup2在头文件中unistd.h 。

函数： int dup （int old ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

该函数复制描述符old到第一个可用的描述符编号（第一个当前未打开的编号）。相当于fcntl (old, F_DUPFD, 0) 。

函数： int dup2 （int old ，int new ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

此函数复制描述符old到描述符编号new 。

如果old是无效的描述符，则dup2什么也没做;它不会关闭new 。否则，新的重复项old替换描述符的任何先前含义new ，好像new首先关闭。

如果old和new是不同的数字，并且old是有效的描述符号，则dup2等效于：

close (new);
fcntl (old, F_DUPFD, new)

然而， dup2原子地做到这一点；通话中间没有瞬间dup2在哪new已关闭，尚未重复old 。

巨集： int F_DUPFD

此宏用作command争论fcntl ，以复制作为第一个参数给出的文件描述符。

在这种情况下，呼叫的形式为：

fcntl (old, F_DUPFD, next-filedes)

的next-filedes参数是类型int并指定返回的文件描述符应该是下一个大于或等于此值的可用描述符。

的返回值fcntl使用此命令通常是新文件描述符的值。返回值-1表示错误。下列errno为该命令定义了错误条件：

EBADF

的old参数无效。

EINVAL

的next-filedes参数无效。

EMFILE

没有更多可用的文件描述符-您的程序已经在使用最大值。在BSD和GNU中，最大值由可以更改的资源限制来控制。 有关资源的更多信息，请参见资源限制。 RLIMIT_NOFILE限制。

ENFILE不是可能的错误代码dup2因为dup2不创建文件的新开头；重复的描述符不计入以下限制ENFILE表示。 EMFILE之所以可能，是因为它指的是在一个进程中使用的唯一描述符数量的限制。

这是显示如何使用的示例dup2进行重定向。通常，重定向标准流（例如stdin ）是由shell或类似shell的程序在调用其中一个exec函数（请参阅执行文件 ）以在子进程中执行新程序。执行新程序时，它会先创建并初始化标准流，以指向相应的文件描述符。 main函数被调用。

因此，要将标准输入重定向到文件，shell可以执行以下操作：

pid = fork ();
if (pid == 0)
  {
    char *filename;
    char *program;
    int file;
    …
    file = TEMP_FAILURE_RETRY (open (filename, O_RDONLY));
    dup2 (file, STDIN_FILENO);
    TEMP_FAILURE_RETRY (close (file));
    execv (program, NULL);
  }
