9.2数组搜索功能

通常，在数组中搜索特定元素意味着可能必须检查所有元素。GNU C库包含执行线性搜索的函数。可以在以下位置找到以下两个功能的原型： search.h 。

功能： void * lfind （const void * key ，const void * base ，size_t * nmemb ，size_t size ，comparison_fn_t compar ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的lfind函数在数组中搜索*nmemb要点size指向的字节base对于与所指向的元素相匹配的元素key 。所指向的功能compar用于确定两个元素是否匹配。

返回值是一个指向数组中匹配元素的指针，从base如果找到。如果没有匹配的元素可用NULL返回。

该函数的平均运行时间为*nmemb / 2。仅当元素经常添加到数组中或从数组中删除时，才应使用此函数，在这种情况下，在搜索之前对数组进行排序可能没有用。

函数： void * lsearch （常量void * key ，无效* base ，size_t * nmemb ，size_t size ，comparison_fn_t compar ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的lsearch功能类似于lfind功能。它在给定数组中搜索元素，如果找到则返回它。区别在于，如果找不到匹配的元素，则lsearch函数添加由指向的对象key （尺寸为size字节）在数组的末尾，它会增加*nmemb反映出这种增加。

对于调用者来说，这意味着如果不确定数组是否包含元素one，则会从以下位置开始搜索为数组分配的内存： base必须至少有空间size更多字节。如果确定该元素在数组中，则最好使用lfind因此在调用时总是需要在数组中有更多空间lsearch 。

要在排序数组中搜索与键匹配的元素，请使用bsearch功能。该函数的原型在头文件中stdlib.h 。

函数： void * bsearch （常量void * key ，const void * array ，size_t count ，size_t size ，comparison_fn_t compare ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的bsearch函数搜索排序的数组array对于等同于key 。数组包含count元素，每个元素的大小size个字节。

的compare函数用于执行比较。该函数使用两个指针参数调用，并且应返回小于，等于或大于零的整数，该整数对应于是否将其第一个参数视为小于，等于或大于其第二个参数。的要素array必须已经根据此比较功能按升序排序。

返回值是指向匹配数组元素的指针，如果找不到匹配项，则返回null指针。如果数组包含多个匹配的元素，则未指定返回的元素。

该函数的名称源于使用二进制搜索算法实现的事实。
