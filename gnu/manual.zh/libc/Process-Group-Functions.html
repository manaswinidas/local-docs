28.6.2流程组功能

以下是用于处理过程组的功能的说明。您的程序应包含头文件sys/types.h和unistd.h使用这些功能。

函数： pid_t setsid （无效）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的setsid函数创建一个新的会话。调用进程成为会话领导者，并被放入一个新的进程组中，该进程组的进程组ID与该进程的进程ID相同。最初，新进程组中没有其他进程，新会话中也没有其他进程组。

此功能还使呼叫过程没有控制终端。

的setsid如果成功，函数将返回调用进程的新进程组ID。返回值为-1表示错误。下列errno为此功能定义了错误条件：

EPERM

调用过程已经是一个过程组组长，或者周围已经有另一个具有相同过程组ID的过程组。

函数： pid_t getsid （pid_t pid ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的getsid函数返回指定进程的会话负责人的进程组ID。如果一个pid是0 ，则返回当前进程的会话负责人的进程组ID。

发生错误时-1返回并errno被设置。下列errno为此功能定义了错误条件：

ESRCH

没有具有给定进程ID的进程pid 。

EPERM

调用过程和指定的过程pid在不同的会话中，并且该实现不允许使用ID访问该进程的会话负责人的进程组ID pid从调用过程中。

函数： pid_t getpgrp （无效）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的getpgrp函数返回调用进程的进程组ID。

函数： int getpgid （pid_t pid ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的getpgid函数返回流程的流程组ID pid 。您可以提供以下值0为了pid参数以获取有关调用过程的信息。

发生错误时-1返回并errno被设置。下列errno为此功能定义了错误条件：

ESRCH

没有具有给定进程ID的进程pid 。调用过程和指定的过程pid在不同的会话中，并且实现不允许使用ID来访问流程的流程组ID pid从调用过程中。

函数： int setpgid （pid_t pid ，pid_t pgid ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的setpgid功能把过程pid进入过程组pgid 。作为特殊情况， pid要么pgid可以为零，以指示调用进程的进程ID。

如果操作成功， setpgid返回零。否则返回-1 。下列errno为此功能定义了错误条件：

EACCES

子进程由命名pid已经执行了exec功能，因为它是分叉的。

EINVAL

的价值pgid无效。

ENOSYS

系统不支持作业控制。

EPERM

指示的过程pid参数是会话领导者，或者与调用进程不在同一个会话中，或者pgid参数与调用进程在同一会话中与进程组ID不匹配。

ESRCH

指示的过程pid参数不是调用过程也不是调用过程的子级。

函数： int setpgrp （pid_t pid ，pid_t pgid ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

这是BSD Unix的名称setpgid 。这两个功能完全相同。
