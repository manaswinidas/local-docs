24.9使用独立的信号栈

信号堆栈是存储器的特殊区域，在信号处理程序期间将用作执行堆栈。它应该相当大，以避免任何可能依次溢出的危险；宏SIGSTKSZ定义为信号堆栈的规范大小。您可以使用malloc为堆栈分配空间。然后打电话sigaltstack要么sigstack告诉系统将该空间用于信号堆栈。

您无需使用其他方式编写信号处理程序即可使用信号堆栈。自动从一个堆栈切换到另一个堆栈。（如果在使用信号堆栈的处理程序运行时检查堆栈跟踪，则某些机器上的某些非GNU调试器可能会感到困惑。）

有两个接口可以告诉系统使用单独的信号堆栈。 sigstack是较旧的界面，来自4.2 BSD。 sigaltstack是较新的界面，来自4.4 BSD。的sigaltstack接口的优点是它不需要您的程序知道堆栈的增长方向，这取决于特定的计算机和操作系统。

数据类型： stack_t

此结构描述了信号堆栈。它包含以下成员：

void *ss_sp

这指向信号栈的底部。

size_t ss_size

这是信号堆栈的大小（以字节为单位），其中“ ss_sp ' 指着。您应该将此设置为为堆栈分配的空间。

在中定义了两个宏signal.h在计算此大小时应使用的：

SIGSTKSZ

这是信号堆栈的规范大小。判断为正常使用就足够了。

MINSIGSTKSZ

这是操作系统仅用于实现信号传递所需的信号堆栈空间量。信号堆栈的大小必须大于此大小。

在大多数情况下，只需使用SIGSTKSZ对于ss_size足够了。但是，如果您知道程序的信号处理程序将需要多少堆栈空间，则可能需要使用其他大小。在这种情况下，您应该分配MINSIGSTKSZ信号堆栈的其他字节并增加ss_size相应地。

int ss_flags

该字段包含这些标志的按位或 ：

SS_DISABLE

这告诉系统不应使用信号栈。

SS_ONSTACK

这是由系统设置的，表示当前正在使用信号堆栈。如果未设置此位，则信号将在普通用户堆栈上传递。

功能： int sigaltstack （const stack_t * restrict stack ，stack_t *限制oldstack ）

初步： MT安全| AS-不安全锁/门| AC不安全锁/门|请参阅POSIX安全概念 。

的sigaltstack函数指定在信号处理期间使用的备用堆栈。当进程接收到信号并且其动作指示已使用信号堆栈时，系统会在执行该信号的处理程序时安排到当前安装的信号堆栈的开关。

如果oldstack不是一个空指针，有关当前安装的信号堆栈的信息将在其指向的位置返回。如果stack不是空指针，则将其安装为新堆栈，以供信号处理程序使用。

返回值为0成功与-1失败。如果sigaltstack失败，它设置errno这些值之一：

EINVAL

您试图禁用实际上正在使用的堆栈。

ENOMEM

备用堆栈的大小太小。它必须大于MINSIGSTKSZ 。

这是老sigstack接口。你应该用sigaltstack而是在具有此功能的系统上。

数据类型： struct sigstack

此结构描述了信号堆栈。它包含以下成员：

void *ss_sp

这是堆栈指针。如果堆栈在您的计算机上向下增长，则应指向您分配的区域的顶部。如果堆栈向上生长，则应指向底部。

int ss_onstack

如果进程当前正在使用此堆栈，则此字段为true。
