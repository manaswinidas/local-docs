25.6系统调用

系统调用是程序对内核进行的服务请求。该服务通常只有内核才有权执行，例如执行I / O。程序员通常不需要关心系统调用，因为GNU C库中有一些函数可以执行系统调用实际上所做的所有事情。这些功能通过使系统调用自身来起作用。例如，有一个系统调用可以更改文件的权限，但是您无需了解它，因为您可以使用GNU C库的chmod功能。

系统调用有时称为内核调用。

但是，有时候您想显式地进行系统调用，为此，GNU C库提供了syscall功能。 syscall比起诸如chmod ，但比在汇编程序指令中对系统调用进行编码要容易和可移植。

syscall当您使用系统专用的或比您使用的GNU C库更新的系统调用时，此功能最有用。 syscall以完全通用的方式实现；该函数不知道有关特定系统调用的内容或有效的信息。

的描述syscall本节中假定运行GNU C库的各种平台上的系统调用使用某种协议。该协议不是由任何权威机构定义的，但是我们也不会在这里描述它，因为有人正在编码syscall无论如何，可能不接受内核和C库源代码作为它们之间接口的规范。

syscall在中声明unistd.h 。

功能： long int syscall （long int sysno ，…）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

syscall执行通用系统调用。

sysno是系统电话号码。每种系统调用都由一个数字标识。所有可能的系统调用号的宏都在sys/syscall.h

其余参数依次是系统调用的参数，它们的含义取决于系统调用的类型。每种系统调用都有一定数量的参数，从零到五个。如果您编写的参数超出系统调用的数量，则将忽略右侧多余的参数。

返回值是系统调用的返回值，除非系统调用失败。在这种情况下， syscall退货-1和集errno到系统调用返回的错误代码。请注意，系统调用不会返回-1当他们成功时。

如果指定无效sysno ， syscall退货-1与errno = ENOSYS 。

例：


#include <unistd.h>
#include <sys/syscall.h>
#include <errno.h>

…

int rc;

rc = syscall(SYS_chmod, "/etc/passwd", 0444);

if (rc == -1)
   fprintf(stderr, "chmod failed, errno = %d\n", errno);

如果所有兼容星号对齐，则等效于以下首选代码：


#include <sys/types.h>
#include <sys/stat.h>
#include <errno.h>

…

int rc;

rc = chmod("/etc/passwd", 0444);
if (rc == -1)
   fprintf(stderr, "chmod failed, errno = %d\n", errno);
