3.2.6.7额外快速增长的对象

增长对象的常用功能会产生开销，以检查当前块中是否有空间进行新的增长。如果您经常以很小的增长步长来构造对象，那么此开销可能会很大。

您可以通过使用特殊的“快速增长”功能（无需检查即可增长对象）来减少开销。为了拥有强大的程序，您必须自己进行检查。如果您每次要向对象添加数据时都以最简单的方式进行此检查，则不会保存任何内容，因为这是普通的增长函数所做的。但是，如果您可以减少检查频率或提高检查效率，则可以使程序运行更快。

功能obstack_room返回当前块中的可用空间量。声明如下：

功能： int obstack_room （struct obstack * obstack-ptr ）

初步： MT安全竞赛：obstack-ptr | AS安全交流安全请参阅POSIX安全概念 。

这将返回可以安全地添加到当前正在增长的对象（或即将启动的对象）的字节数obstack-ptr使用快速增长的功能。

当您知道有空间时，可以使用以下快速增长功能向正在增长的对象添加数据：

功能： void obstack_1grow_fast （struct obstack * obstack-ptr ，字符c ）

初步： MT安全竞赛：obstack-ptr | AS安全AC不安全的损坏的内存|请参阅POSIX安全概念 。

功能obstack_1grow_fast添加一个包含字符的字节c不断增长的对象obstack-ptr 。

功能： void obstack_ptr_grow_fast （struct obstack * obstack-ptr ，无效* data ）

初步： MT安全竞赛：obstack-ptr | AS安全交流安全请参阅POSIX安全概念 。

功能obstack_ptr_grow_fast添加sizeof (void *)包含值的字节data不断增长的对象obstack-ptr 。

功能： void obstack_int_grow_fast （struct obstack * obstack-ptr ，int data ）

初步： MT安全竞赛：obstack-ptr | AS安全交流安全请参阅POSIX安全概念 。

功能obstack_int_grow_fast添加sizeof (int)包含值的字节data不断增长的对象obstack-ptr 。

功能： void obstack_blank_fast （struct obstack * obstack-ptr ，int size ）

初步： MT安全竞赛：obstack-ptr | AS安全交流安全请参阅POSIX安全概念 。

功能obstack_blank_fast添加size堆中成长对象的字节数obstack-ptr无需初始化它们。

当您使用检查空间时obstack_room而且您没有足够的空间来添加内容，因此快速增长的功能并不安全。在这种情况下，只需使用相应的普通增长函数即可。很快，这会将对象复制到新的块中。那么将有大量可用空间。

因此，每次使用普通的增长函数时，请使用obstack_room 。将对象复制到新的块后，将再次有足够的空间，因此程序将再次开始使用快速增长功能。

这是一个例子：

void
add_string (struct obstack *obstack, const char *ptr, int len)
{
  while (len > 0)
    {
      int room = obstack_room (obstack);
      if (room == 0)
        {
          /* Not enough room.  Add one character slowly,
             which may copy to a new chunk and make room.  */
          obstack_1grow (obstack, *ptr++);
          len--;
        }
      else
        {
          if (room > len)
            room = len;
          /* Add fast as much as we have room for. */
          len -= room;
          while (room-- > 0)
            obstack_1grow_fast (obstack, *ptr++);
        }
    }
}
