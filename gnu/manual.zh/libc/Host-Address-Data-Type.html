16.6.2.2主机地址数据类型

IPv4 Internet主机地址在某些上下文中表示为整数（类型uint32_t ）。在其他情况下，整数打包在类型为的结构内struct in_addr 。如果使用法一致，会更好，但是从结构中提取整数或将整数放入结构中并不难。

您会发现使用unsigned long int用于IPv4 Internet主机地址，而不是uint32_t要么struct in_addr 。历史上unsigned long int是32位数字，但在64位计算机上，这一情况已更改。使用unsigned long int如果此类型的计算机没有32位，则可能会破坏代码。 uint32_t由Unix98指定，并保证具有32位。

IPv6 Internet主机地址具有128位，并打包在一个结构类型中struct in6_addr 。

头文件中声明了以下Internet地址的基本定义netinet/in.h ：

数据类型： struct in_addr

在某些情况下，此数据类型用于包含IPv4 Internet主机地址。它只有一个字段，名为s_addr ，它将主机地址号记录为uint32_t 。

巨集： uint32_t INADDR_LOOPBACK

您可以使用此常量代表“本机的地址”，而不用查找其实际地址。这是IPv4互联网地址' 127.0.0.1 '，通常称为' localhost '。这个特殊常数免除了您查找自己机器地址的麻烦。另外，系统通常会实现INADDR_LOOPBACK特别是避免一台机器自言自语的情况下避免任何网络流量。

巨集： uint32_t INADDR_ANY

当绑定到一个地址时，您可以使用此常量来表示“任何传入地址”。请参阅设置地址 。这是通常要填写的地址sin_addr成员struct sockaddr_in当您想接受Internet连接时为 。

巨集： uint32_t INADDR_BROADCAST

此常数是您用于发送广播消息的地址。

巨集： uint32_t INADDR_NONE

该常数由某些函数返回以指示错误。

数据类型： struct in6_addr

此数据类型用于存储IPv6地址。它存储128位数据，可以通过多种方式访问（通过联合）。

常量： struct in6_addr in6addr_loopback

此常数是IPv6地址' ::1 '，即回送地址。请参阅上面的内容说明。宏IN6ADDR_LOOPBACK_INIT提供允许您将自己的变量初始化为该值。

常量： struct in6_addr in6addr_any

此常数是IPv6地址' :: '，即未指定的地址。请参阅上面的内容说明。宏IN6ADDR_ANY_INIT提供允许您将自己的变量初始化为该值。
