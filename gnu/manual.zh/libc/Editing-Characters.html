17.4.9.1用于输入编辑的字符

这些特殊字符仅在规范输入模式下有效。请参阅“ 规范”或“不规范” 。

宏： int VEOF

这是特殊控制字符数组中EOF字符的下标。 termios.c_cc[VEOF]拥有角色本身。

EOF字符仅在规范输入模式下才能识别。它用作换行符的方式与换行符相同，但是如果在行首键入EOF字符，则会导致read返回字节计数为零的字节，指示文件结束。EOF字符本身将被丢弃。

通常，EOF字符是C-d 。

巨集： int VEOL

这是特殊控制字符数组中EOL字符的下标。 termios.c_cc[VEOL]拥有角色本身。

仅在规范输入模式下才能识别EOL字符。它像换行符一样充当行终止符。EOL字符不会被丢弃；它被读取为输入行中的最后一个字符。

您无需使用EOL字符即可使RET结束一行。只需设置ICRNL标志。实际上，这是默认的事务状态。

巨集： int VEOL2

这是特殊控制字符数组中EOL2字符的下标。 termios.c_cc[VEOL2]拥有角色本身。

EOL2字符的工作方式类似于EOL字符（请参见上文），但可以是不同的字符。因此，通过将EOL设置为其中一个，将EOL2设置为另一个，可以指定两个字符来终止输入行。

EOL2字符是BSD扩展；它仅在BSD系统以及GNU / Linux和GNU / Hurd系统上存在。

巨集： int VERASE

这是特殊控制字符数组中ERASE字符的下标。 termios.c_cc[VERASE]拥有角色本身。

ERASE字符仅在规范输入模式下才能识别。当用户键入擦除字符时，先前键入的字符将被丢弃。（如果终端生成多字节字符序列，则可能导致超过一个字节的输入被丢弃。）这不能用于擦除当前文本行的开头。ERASE字符本身将被丢弃。

通常，ERASE字符是DEL 。

巨集： int VWERASE

这是特殊控制字符数组中WERASE字符的下标。 termios.c_cc[VWERASE]拥有角色本身。

仅在规范模式下才能识别WERASE字符。它将擦除先前输入的整个单词以及其后的所有空格；单词前的空白字符不会被删除。

“单词”的定义取决于ALTWERASE模式;请参阅本地模式 。

如果ALTWERASE如果未设置mode，则将单词定义为除空格或制表符之外的任何字符的序列。

如果ALTWERASE设置为mode时，单词定义为仅包含字母，数字和下划线的字符序列，可以选择后面跟一个不是字母，数字或下划线的字符。

WERASE字符通常是C-w 。

这是BSD扩展。

巨集： int VKILL

这是特殊控制字符数组中KILL字符的下标。 termios.c_cc[VKILL]拥有角色本身。

仅在标准输入模式下才能识别KILL字符。当用户键入杀死字符时，将丢弃当前输入行的全部内容。杀死角色本身也被丢弃。

杀死字符通常是C-u 。

巨集： int VREPRINT

这是特殊控制字符数组中REPRINT字符的下标。 termios.c_cc[VREPRINT]拥有角色本身。

REPRINT字符仅在规范模式下可识别。重新打印当前输入行。如果在键入时出现了一些异步输出，则可以让您再次清楚地看到正在键入的行。

REPRINT字符通常是C-r 。

这是BSD扩展。
