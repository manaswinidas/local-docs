3.2.4.3或多或少的聪明点子

你知道情况。该程序已准备好进行调试，并且在所有调试会话中均可正常运行。但是一旦启动它而没有调试，就会显示错误。一个典型的例子是只有当我们关闭调试时，内存泄漏才可见。如果您预见到这种情况，您仍然可以获胜。只需使用与以下小程序等效的东西：

#include <mcheck.h>
#include <signal.h>

static void
enable (int sig)
{
  mtrace ();
  signal (SIGUSR1, enable);
}

static void
disable (int sig)
{
  muntrace ();
  signal (SIGUSR2, disable);
}

int
main (int argc, char *argv[])
{
  …

  signal (SIGUSR1, enable);
  signal (SIGUSR2, disable);

  …
}

即，用户可以在需要的任何时间启动内存调试器，如果程序以MALLOC_TRACE在环境中放置。输出当然不会显示在第一个信号之前发生的分配，但是如果存在内存泄漏，则仍然会显示。
