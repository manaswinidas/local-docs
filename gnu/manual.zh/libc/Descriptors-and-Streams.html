13.4描述符和流

给定一个打开的文件描述符，您可以使用fdopen功能。您可以使用以下命令获取现有流的基础文件描述符fileno功能。这些函数在头文件中声明stdio.h 。

函数： FILE * fdopen （int filedes ，const char * opentype ）

初步： MT安全| AS不安全堆锁| AC不安全的内存锁|请参阅POSIX安全概念 。

的fdopen函数返回文件描述符的新流filedes 。

的opentype参数的解释方式与fopen功能（请参见打开流 ），但“ b '选项是不允许的；这是因为GNU系统在文本文件和二进制文件之间没有区别。也， "w"和"w+"不会导致文件被截断；这些仅在打开文件时才有效，在这种情况下，文件已被打开。您必须确保opentype参数匹配打开文件描述符的实际模式。

返回值是新的流。如果无法创建流（例如，如果文件描述符所指示的文件的模式不允许通过opentype参数），则返回空指针。

在其他一些系统中， fdopen可能无法检测到文件描述符的模式不允许通过以下方式指定访问opentype 。GNU C库始终对此进行检查。

举例说明使用fdopen功能，请参见创建管道 。

函数： int fileno （FILE * stream ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

此函数返回与流关联的文件描述符stream 。如果检测到错误（例如，如果stream无效），或者stream不对文件进行I / O， fileno返回-1 。

函数： int fileno_unlocked （FILE * stream ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的fileno_unlocked功能相当于fileno函数，除了状态为时不会隐式锁定流FSETLOCKING_INTERNAL 。

此函数是GNU扩展。

在中也定义了符号常量unistd.h用于属于标准流的文件描述符stdin ， stdout和stderr ;请参阅标准流 。

STDIN_FILENO

这个宏有价值0 ，这是标准输入的文件描述符。

STDOUT_FILENO

这个宏有价值1 ，这是标准输出的文件描述符。

STDERR_FILENO

这个宏有价值2 ，这是标准错误输出的文件描述符。
