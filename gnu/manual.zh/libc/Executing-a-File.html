26.5执行文件

本节介绍exec系列功能，用于将文件作为过程映像执行。您可以使用这些功能使子进程在分叉后执行新程序。

看看效果exec从被调用程序的角度来看，请参阅“ 程序基础知识” 。

该族中的函数在指定参数的方式上有所不同，但否则它们都做相同的事情。它们在头文件中声明unistd.h 。

功能： int execv （const char * filename ，char * const argv [] ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的execv函数执行名称为的文件filename作为新的过程映像。

的argv参数是一个以null结尾的字符串数组，用于为argv争论main要执行的程序的功能。该数组的最后一个元素必须为空指针。按照约定，此数组的第一个元素是程序的文件名，没有目录名。有关程序如何访问这些参数的完整详细信息，请参见程序参数 。

新过程映像的环境取自environ当前过程映像的变量；有关环境的信息，请参见环境变量 。

功能： int execl （const char * filename ，const char * arg0 ，…）

初步： MT安全| AS不安全堆| AC不安全的内存|请参阅POSIX安全概念 。

这类似于execv ，但argv字符串是单独指定的，而不是数组。必须将空指针作为最后一个这样的参数传递。

函数： int execve （const char * filename ，char * const argv [] ，char * const env [] ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

这类似于execv ，但允许您将新程序的环境明确指定为env论点。这应该是一个字符串数组，格式与environ变量;请参阅环境访问 。

函数： int execle （const char * filename ，const char * arg0 ，…，char * const env [] ）

初步： MT安全| AS不安全堆| AC不安全的内存|请参阅POSIX安全概念 。

这类似于execl ，但允许您为新程序明确指定环境。环境参数在标记最后一个的空指针之后传递argv参数，并且应该是一个字符串数组，其格式与environ变量。

功能： int execvp （const char * filename ，char * const argv [] ）

初步： MT安全环境| AS不安全堆| AC不安全的内存|请参阅POSIX安全概念 。

的execvp功能类似于execv ，除了它会搜索PATH环境变量（请参阅“ 标准环境” ）以从中查找文件的完整文件名filename如果filename不包含斜线。

此功能对于执行系统实用程序很有用，因为它会在用户选择的位置查找它们。外壳程序使用它来运行用户键入的命令。

函数： int execlp （const char * filename ，const char * arg0 ，…）

初步： MT安全环境| AS不安全堆| AC不安全的内存|请参阅POSIX安全概念 。

这个功能就像execl ，除了它执行的文件名搜索与execvp功能。

参数列表和环境列表的总和不能大于ARG_MAX个字节。请参阅一般限制 。在GNU / Hurd系统上，大小（与ARG_MAX ）包括每个字符串的字符数，加上一个char * ，再加上一个，四舍五入为a的大小的倍数char * 。其他系统可能有一些不同的计数规则。

这些功能通常不会返回，因为执行新程序会导致当前正在执行的程序完全消失。值-1失败时返回。除了常见的文件名错误（请参阅文件名错误 ）之外，以下内容errno为这些功能定义了错误条件：

E2BIG

新程序的参数列表和环境列表的组合大小大于ARG_MAX个字节。GNU / Hurd系统对参数列表的大小没有具体限制，因此不会出现此错误代码，但是您可能会得到ENOMEM相反，如果参数对于可用内存而言太大。

ENOEXEC

指定的文件格式不正确，因此无法执行。

ENOMEM

执行指定的文件需要更多的存储空间。

如果新文件的执行成功，它将更新文件的访问时间字段，就像已读取文件一样。有关文件访问时间的更多详细信息，请参见文件时间 。

没有指定再次关闭文件的时间点，而是在退出过程之前或执行另一个过程映像之前的某个时间点。

执行新的过程映像会完全更改内存的内容，仅将参数和环境字符串复制到新位置。但是该过程的许多其他属性没有改变：

  • 进程ID和父进程ID。请参阅过程创建概念 。
  • 会话和进程组成员身份。请参阅作业控制的概念 。
  • 真实用户ID和组ID，以及补充组ID。请参阅流程角色 。
  • 等待警报。请参阅设置警报 。
  • 当前工作目录和根目录。请参阅工作目录 。在GNU / Hurd系统上，执行setuid程序时不会复制根目录。而是将系统默认的根目录用于新程序。
  • 文件模式创建掩码。请参阅设置权限 。
  • 过程信号屏蔽；参见过程信号屏蔽 。
  • 待处理信号；请参阅阻塞信号 。
  • 与该过程相关联的已用处理器时间；请参阅处理器时间 。

如果设置了过程映像文件的set-user-ID和set-group-ID模式位，则分别影响过程的有效用户ID和有效组ID。这些概念在Process Persona中进行了详细讨论。

在现有过程映像中设置为忽略的信号在新过程映像中也设置为忽略。在新过程映像中，所有其他信号均设置为默认操作。有关信号的更多信息，请参见信号处理 。

在现有过程映像中打开的文件描述符在新过程映像中保持打开状态，除非它们具有FD_CLOEXEC （执行时关闭）标志设置。保持打开状态的文件从现有过程映像继承打开文件描述符的所有属性，包括文件锁。文件描述符在底层I / O中讨论。

相比之下，流无法生存exec函数，因为它们位于进程本身的内存中。除了重新创建的流之外，新的过程映像没有流。前置码中的每个流exec过程映像内部有一个描述符，这些描述符通过exec （前提是他们没有FD_CLOEXEC组）。新的过程映像可以使用以下命令将它们重新连接到新的流fdopen （请参阅描述符和流 ）。
