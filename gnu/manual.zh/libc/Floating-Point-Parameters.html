A.5.3.2浮点参数

可以通过包含头文件来访问这些宏定义。 float.h在您的程序中。

以'开头的宏名称FLT_ '指的是float类型，而名称以“ DBL_ '指的是double类型和名称以“ LDBL_ '指的是long double类型。（如果GCC不支持long double作为目标计算机上的不同数据类型，则“ LDBL_ '常数等于对应的常数double类型。）

在这些宏中，只有FLT_RADIX保证是一个常量表达式。此处列出的其他宏不能在需要常量表达式的地方可靠地使用，例如' #if预处理指令或静态数组的维数。

尽管ISO C 标准为大多数这些参数指定了最小值和最大值，但是GNU C实现使用描述目标机器浮点表示的任何值。因此，原则上，只有在目标机器合适的情况下，GNU C实际上才满足ISO C 要求。实际上，当前支持的所有机器都是合适的。

FLT_ROUNDS

该值表示浮点加法的舍入模式。以下值指示标准的舍入模式：

-1

模式是不确定的。

0

舍入为零。

1

四舍五入到最接近的数字。

2

取整朝着正无穷大。

3

取整朝着负无穷大。

任何其他值都表示机器相关的非标准舍入模式。

在大多数计算机上，该值为1 ，符合针对浮点的IEEE标准。

下表显示了某些值如何针对的每个可能值取整FLT_ROUNDS ，如果表示的其他方面与IEEE单精度标准匹配。 

                0      1             2             3
 1.00000003    1.0    1.0           1.00000012    1.0
 1.00000007    1.0    1.00000012    1.00000012    1.0
-1.00000003   -1.0   -1.0          -1.0          -1.00000012
-1.00000007   -1.0   -1.00000012   -1.0          -1.00000012
FLT_RADIX

这是指数表示形式的基数或基数的值。与本节中介绍的其他宏不同，它保证是一个常量表达式。在我们知道的所有机器上，除IBM 360和衍生产品外，该值为2。

FLT_MANT_DIG

这是基数- FLT_RADIX浮点数尾数中的数字float数据类型。以下表达式产生1.0 （即使从数学上讲也不应该如此），因为尾数位数有限：

float radix = FLT_RADIX;

1.0f + 1.0f / radix / radix / … / radix

哪里radix出现FLT_MANT_DIG次。

DBL_MANT_DIG
LDBL_MANT_DIG

这是基数- FLT_RADIX数据类型的浮点尾数中的数字double和long double ， 分别。

FLT_DIG

这是精度的小数位数float数据类型。从技术上讲，如果p和b是表示的精度和基数（分别），然后是小数精度q是最大十进制数字，因此任何浮点数都带有q基数10可以四舍五入为浮点数p基础b数字然后再返回，而无需更改q十进制数字。

该宏的值至少应为6 ，以满足ISO C 。

DBL_DIG
LDBL_DIG

这些类似于FLT_DIG ，但对于数据类型double和long double ， 分别。这些宏的值至少应为10 。

FLT_MIN_EXP

这是类型的最小可能的指数值float 。更确切地说，它是最小负整数，使得值FLT_RADIX升至此幂减去1可以表示为类型的归一化浮点数float 。

DBL_MIN_EXP
LDBL_MIN_EXP

这些类似于FLT_MIN_EXP ，但对于数据类型double和long double ， 分别。

FLT_MIN_10_EXP

这是最小负整数，使得10升至此幂减去1可以表示为类型的归一化浮点数float 。应该是-37甚至更少。

DBL_MIN_10_EXP
LDBL_MIN_10_EXP

这些类似于FLT_MIN_10_EXP ，但对于数据类型double和long double ， 分别。

FLT_MAX_EXP

这是类型的最大可能指数值float 。更确切地说，这是最大正整数，使得该值FLT_RADIX升至此幂减去1可以表示为类型的浮点数float 。

DBL_MAX_EXP
LDBL_MAX_EXP

这些类似于FLT_MAX_EXP ，但对于数据类型double和long double ， 分别。

FLT_MAX_10_EXP

这是最大正整数，使得10升至此幂减去1可以表示为类型的归一化浮点数float 。至少应该是37 。

DBL_MAX_10_EXP
LDBL_MAX_10_EXP

这些类似于FLT_MAX_10_EXP ，但对于数据类型double和long double ， 分别。

FLT_MAX

此宏的值是类型可以表示的最大数量float 。应该至少1E+37 。值具有类型float 。

最小的可表示数字是- FLT_MAX 。

DBL_MAX
LDBL_MAX

这些类似于FLT_MAX ，但对于数据类型double和long double ， 分别。宏的值的类型与它描述的类型相同。

FLT_MIN

此宏的值是最小可归一的正浮点数，可以在类型中表示float 。它应该不超过1E-37 。

DBL_MIN
LDBL_MIN

这些类似于FLT_MIN ，但对于数据类型double和long double ， 分别。宏的值的类型与它描述的类型相同。

FLT_EPSILON

这是1与最小浮点数类型之间的差float大于1。应该不大于1E-5 。

DBL_EPSILON
LDBL_EPSILON

这些类似于FLT_EPSILON ，但对于数据类型double和long double ， 分别。宏的值的类型与它描述的类型相同。该值不应大于1E-9 。
