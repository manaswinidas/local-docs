E.1 PowerPC专用的设施

在不声明特定于特定操作系统的特定于PowerPC的功能中声明了sys/platform/ppc.h 。

函数： uint64_t __ppc_get_timebase （无效）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

读取时基寄存器的当前值。

时基寄存器是一个64位寄存器，用于存储单调递增的值，该值以与系统相关的频率更新，该频率可能与处理器频率不同。有关更多信息，请参见Power ISA 2.06b-Book II-第5.2节 。

__ppc_get_timebase无需操作系统的帮助即可直接使用处理器的时基工具，因此非常高效。

函数： uint64_t __ppc_get_timebase_freq （无效）

初步： MT不安全初始化| AS-Unsafe损坏： AC-Unsafe损坏：请参阅POSIX安全概念 。

读取时基寄存器更新的当前频率。

该频率与处理器时钟或总线时钟无关。该频率也可能不是恒定的。有关更多信息，请参见Power ISA 2.06b-Book II-第5.2节 。

以下功能提供有关与其他处理器共享的资源的使用情况的提示。例如，如果等待锁的程序打算转移共享资源以供其他处理器使用，则可以使用它们。有关更多信息，请参见Power ISA 2.06b-Book II-第3.2节 。

函数： void __ppc_yield （void）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

提供一个暗示，如果释放专用于执行处理器的共享资源供其他处理器使用，则性能可能会得到改善。

函数： void __ppc_mdoio （void）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

提示如果释放专用于执行处理器的共享资源，直到对缓存禁止的存储的所有未完成的存储访问都已完成，则性能可能会得到改善。

函数： void __ppc_mdoom （void）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

提供一个暗示，如果释放专用于执行处理器的共享资源，直到完成对未包含在缓存中的数据的可缓存存储的所有未完成的存储访问之前，性能可能会得到改善。

函数： void __ppc_set_ppr_med （无效）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

将程序优先级寄存器设置为中等值（默认）。

程序优先级寄存器 （PPR）是一个64位寄存器，用于控制程序的优先级。通过调整PPR值，程序员可以通过使系统资源更有效地使用来提高系统吞吐量，特别是在竞争情况下。功能涵盖了三个可用的非特权状态__ppc_set_ppr_med （中-默认）， __ppc_set_ppc_low （低）和__ppc_set_ppc_med_low （中低）。有关更多信息，请参见Power ISA 2.06b-Book II-第3.1节 。

函数： void __ppc_set_ppr_low （无效）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

将程序优先级寄存器设置为较低的值。

函数： void __ppc_set_ppr_med_low （无效）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

将程序优先级寄存器设置为中低值。

Power ISA 2.07扩展了可以设置到程序优先级寄存器（PPR）的优先级。以下功能实现了新的优先级：非常低和中等高。

函数： void __ppc_set_ppr_very_low （无效）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

将程序优先级寄存器设置为非常低的值。

函数： void __ppc_set_ppr_med_high （无效）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

将程序优先级寄存器设置为中高值。中高优先级具有特权，并且只能在特定时间间隔内由问题状态程序设置。如果在时间间隔到期时程序优先级为中等高，或者在不允许的情况下尝试将优先级设置为中高，则将优先级设置为中。
