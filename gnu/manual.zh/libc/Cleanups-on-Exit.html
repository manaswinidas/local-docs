25.7.3退出时的清理

如果正常终止，您的程序可以安排运行自己的清理功能。如果要编写用于各种应用程序的库，那么坚持退出所有应用程序之前，要所有应用程序显式调用该库的清理函数是不可靠的。通过使用以下方法在库本身中设置清除功能，使清除对应用程序不可见的功能更加强大。 atexit要么on_exit 。

函数： int atexit （void（* function ）（无效））

初步： MT安全| AS不安全堆锁| AC不安全锁记忆|请参阅POSIX安全概念 。

的atexit功能注册功能function在正常程序终止时调用。的function不带参数调用。

的返回值atexit成功时为0，如果无法注册该函数，则为非零。

函数： int on_exit （void（* function ）（int status ，无效* arg ），无效* arg ）

初步： MT安全| AS不安全堆锁| AC不安全锁记忆|请参阅POSIX安全概念 。

此功能是以下功能的更强大的变体atexit 。它接受两个参数，一个函数function和一个任意指针arg 。在正常程序终止时， function用两个参数调用： status价值传递给exit和arg 。

GNU C库中包含此功能仅是为了与SunOS兼容，而其他实现可能不支持此功能。

这是一个简单的程序，说明了exit和atexit ：


#include <stdio.h>
#include <stdlib.h>

void
bye (void)
{
  puts ("Goodbye, cruel world....");
}

int
main (void)
{
  atexit (bye);
  exit (EXIT_SUCCESS);
}

执行此程序时，它仅打印消息并退出。
