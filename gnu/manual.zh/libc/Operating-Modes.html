13.15.3 I / O操作模式

操作模式会影响使用文件描述符的输入和输出操作的工作方式。这些标志由open并可以通过以下方式获取和更改fcntl 。

巨集： int O_APPEND

启用文件附加模式的位。如果设置，则全部write无论当前文件处于什么位置，操作都会在文件末尾写入数据，并对其进行扩展。这是附加到文件的唯一可靠方法。在追加模式下，可以确保写入的数据将始终转到文件的当前末尾，而不管写入该文件的其他进程如何。相反，如果仅将文件位置设置为文件末尾并写入，则在设置文件位置之后但在写入之前，另一个过程可以扩展文件，从而导致数据出现在文件实际末尾之前的某个位置。

巨集： int O_NONBLOCK

启用文件非阻塞模式的位。如果该位置1， read如果没有立即可用的输入，则对该文件的请求可以立即返回失败状态，而不是阻塞。同样write如果无法立即写入输出，则请求也可以立即返回失败状态。

请注意O_NONBLOCK该标志作为I / O操作模式和文件名转换标志都被重载；请参阅开放时间标志 。

巨集： int O_NDELAY

这是一个过时的名称O_NONBLOCK ，用于与BSD兼容。POSIX.1标准未定义它。

其余的操作模式是BSD和GNU扩展。它们仅存在于某些系统上。在其他系统上，未定义这些宏。

巨集： int O_ASYNC

启用异步输入模式的位。如果设置，则SIGIO输入可用时将生成信号。请参见中断输入 。

异步输入模式是BSD的功能。

巨集： int O_FSYNC

启用文件同步写入的位。如果设置，则每个write调用将确保返回之前将数据可靠地存储在磁盘上。

同步写入是BSD的功能。

巨集： int O_SYNC

这是另一个名字O_FSYNC 。它们具有相同的价值。

巨集： int O_NOATIME

如果该位置1， read不会更新文件的访问时间。请参阅文件时间 。进行备份的程序会使用它，因此备份文件不算作读取文件。只有文件所有者或超级用户可以使用此位。

这是一个GNU扩展。
