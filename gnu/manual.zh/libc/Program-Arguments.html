25.1程序参数

系统通过调用函数来启动C程序main 。您可以自行编写一个名为main —否则，您甚至都将无法正确链接程序。

在ISO C 您可以定义main要么不接受任何参数，要么接受两个代表程序命令行参数的参数，如下所示：

int main (int argc, char *argv[])

命令行参数是用于调用程序的shell命令中给定的空格分隔的标记。因此， cat foo bar '，则参数为' foo '和' bar '。程序可以查看其命令行参数的唯一方法是通过main 。如果main不带参数，那么您将无法进入命令行。

的价值argc参数是命令行参数的数量。的argv参数是C字符串的向量；它的元素是单独的命令行参数字符串。矢量中还包含正在运行的程序的文件名作为第一个元素；的价值argc计算此元素。空指针始终位于最后一个元素之后： argv[argc]是此空指针。

对于命令“ cat foo bar '， argc是3并且argv有三个要素， "cat" ， "foo"和"bar" 。

在Unix系统中，您可以定义main第三种方式，使用三个参数：

int main (int argc, char *argv[], char *envp[])

前两个参数是相同的。第三论点envp提供程序的环境；它与的值相同environ 。请参阅环境变量 。POSIX.1不允许使用这种三参数形式，因此，为了便于移植，最好编写main接受两个参数，并使用的值environ 。
