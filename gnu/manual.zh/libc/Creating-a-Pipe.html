15.1创建管道

创建管道的原语是pipe功能。这将创建管道的读取端和写入端。对于单个进程来说，使用管道与其自身通信不是很有用。在典型的用法中，一个流程在派生一个或多个子流程之前创建一个管道（请参阅创建流程 ）。然后将管道用于父进程或子进程之间或两个同级进程之间的通信。

的pipe函数在头文件中声明unistd.h 。

函数： int 管道 （int filedes [2] ）

初步： MT安全| AS安全交流安全fd |请参阅POSIX安全概念 。

的pipe函数创建一个管道，并将分别用于读取和写入管道末端的文件描述符放入filedes[0]和filedes[1] 。

记住输入端在前的一种简单方法是文件描述符0是标准输入，文件描述符1是标准输出。

如果成功pipe返回值0 。失败了-1返回。下列errno为此功能定义了错误条件：

EMFILE

该进程打开了太多文件。

ENFILE

整个系统中打开的文件过多。有关更多信息，请参见错误代码 。 ENFILE 。在GNU / Hurd系统上永远不会发生此错误。

这是一个创建管道的简单程序的示例。该程序使用fork函数（请参阅创建流程 ）来创建子流程。父进程将数据写入管道，子进程将读取该数据。 


#include <sys/types.h>
#include <unistd.h>
#include <stdio.h>
#include <stdlib.h>

/* Read characters from the pipe and echo them to stdout. */

void
read_from_pipe (int file)
{
  FILE *stream;
  int c;
  stream = fdopen (file, "r");
  while ((c = fgetc (stream)) != EOF)
    putchar (c);
  fclose (stream);
}

/* Write some random text to the pipe. */

void
write_to_pipe (int file)
{
  FILE *stream;
  stream = fdopen (file, "w");
  fprintf (stream, "hello, world!\n");
  fprintf (stream, "goodbye, world!\n");
  fclose (stream);
}

int
main (void)
{
  pid_t pid;
  int mypipe[2];


  /* Create the pipe. */
  if (pipe (mypipe))
    {
      fprintf (stderr, "Pipe failed.\n");
      return EXIT_FAILURE;
    }


  /* Create the child process. */
  pid = fork ();
  if (pid == (pid_t) 0)
    {
      /* This is the child process.
         Close other end first. */
      close (mypipe[1]);
      read_from_pipe (mypipe[0]);
      return EXIT_SUCCESS;
    }
  else if (pid < (pid_t) 0)
    {
      /* The fork failed. */
      fprintf (stderr, "Fork failed.\n");
      return EXIT_FAILURE;
    }
  else
    {
      /* This is the parent process.
         Close other end first. */
      close (mypipe[0]);
      write_to_pipe (mypipe[1]);
      return EXIT_SUCCESS;
    }
}
