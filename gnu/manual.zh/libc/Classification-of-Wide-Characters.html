4.3确定宽字符的字符类别

ISO C90的修订1 定义了对宽字符进行分类的函数。尽管原始ISO C90 标准已经定义了类型wchar_t ，未定义对其进行操作的功能。

宽字符分类功能的一般设计更为通用。它允许扩展到可用分类集，而不是始终可用的分类集。POSIX标准指定了扩展方式，并且已经在GNU C库实现中实现了该扩展。 localedef程序。

字符类功能通常由位集实现，每个字符都有位集。对于给定的字符，从表中读取适当的位集，并执行有关是否设置了某个位的测试。测试哪个位由类决定。

对于宽字符分类功能，这是可见的。有一个定义的类型分类类型，一个函数用于检索给定类的该值，以及一个函数使用分类值来测试给定字符是否在该类中。最重要的是，常规字符分类功能用于char可以定义对象。

数据类型： wctype_t

的wctype_t可以保存代表字符类的值。生成该值的唯一定义方法是使用wctype功能。

此类型在wctype.h 。

函数： wctype_t wctype （const char * property ）

初步： MT安全区域| AS安全交流安全请参阅POSIX安全概念 。

wctype返回一个值，该值代表一类由字符串标识的宽字符property 。除了一些标准属性外，每个语言环境还可以定义自己的语言环境。如果没有已知的给定名称的属性用于当前选择的语言环境LC_CTYPE类别，函数返回零。

在每个语言环境中已知的属性是：

"alnum""alpha""cntrl""digit"
"graph""lower""print""punct"
"space""upper""xdigit"

该函数在wctype.h 。

为了测试字符是否属于非标准类之一，ISO C 标准定义了一个全新的功能。

函数： int iswctype （wint_t wc ，wctype_t desc ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

如果此函数返回一个非零值wc在由指定的字符类中desc 。 desc必须事先通过成功调用返回wctype 。

该函数在wctype.h 。

为了更易于使用常用的分类函数，它们在C库中定义。无需使用wctype如果属性字符串是已知字符类之一。在某些情况下，需要构造属性字符串，然后重要的是wctype也可以处理标准类。

函数： int iswalnum （wint_t wc ）

初步： MT安全区域| AS安全交流安全请参阅POSIX安全概念 。

如果此函数返回一个非零值wc是字母数字字符（字母或数字）；换句话说，如果有iswalpha要么iswdigit对一个角色是真实的，那么iswalnum也是如此。

该功能可以使用

iswctype (wc, wctype ("alnum"))

声明在wctype.h 。

函数： int iswalpha （wint_t wc ）

初步： MT安全区域| AS安全交流安全请参阅POSIX安全概念 。

如果返回true wc是字母字符（字母）。如果iswlower要么iswupper对一个角色是真实的，那么iswalpha也是如此。

在某些语言环境中，可能还会有其他字符iswalpha是正确的-字母既不大写也不小写。但是在标准"C"语言环境，没有这样的附加字符。

该功能可以使用

初步： MT安全区域| AS安全交流安全请参阅POSIX安全概念 。

如果返回true wc是控制字符（即不是打印字符的字符）。

该功能可以使用

iswctype (wc, wctype ("cntrl"))

声明在wctype.h 。

函数： int iswdigit （wint_t wc ）

初步： MT安全区域| AS安全交流安全请参阅POSIX安全概念 。

如果返回true wc是一个数字（例如，“ 0 '通过' 9 '）。请注意，此函数不仅会返回十进制数字的非零值， 还会返回所有数字的非零值。结果是类似下面的代码不会为宽字符无条件地工作：

n = 0;
while (iswdigit (*wc))
  {
    n *= 10;
    n += *wc++ - L'0';
  }

该功能可以使用
