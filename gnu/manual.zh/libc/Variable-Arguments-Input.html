12.14.9变量参数输入函数

功能vscanf和朋友被提供，以便您可以定义自己的可变参数scanf类函数，它们使用与内置格式化输出函数相同的内部函数。这些功能类似于vprintf系列输出功能。有关如何使用变量的重要信息，请参见变量参数输出 。

可移植性注意：本节中列出的功能已在ISO C99 中引入，并且已作为GNU扩展提供。

功能： int vscanf （const char * template ，va_list ap ）

初步： MT安全区域| AS不安全的损坏堆| AC不安全的内存锁损坏|请参阅POSIX安全概念 。

该功能类似于scanf ，而不是直接采用可变数量的参数，而是采用参数列表指针ap类型的va_list （请参见可变函数 ）。

功能： int vwscanf （const wchar_t * template ，va_list ap ）

初步： MT安全区域| AS不安全的损坏堆| AC不安全的内存锁损坏|请参阅POSIX安全概念 。

该功能类似于wscanf ，而不是直接采用可变数量的参数，而是采用参数列表指针ap类型的va_list （请参见可变函数 ）。

功能： int vfscanf （FILE * stream ，const char * template ，va_list ap ）

初步： MT安全区域| AS不安全的损坏堆| AC不安全的内存锁损坏|请参阅POSIX安全概念 。

这相当于fscanf变量参数列表直接指定为vscanf 。

功能： int vfwscanf （FILE * stream ，const wchar_t * template ，va_list ap ）

初步： MT安全区域| AS不安全的损坏堆| AC不安全的内存锁损坏|请参阅POSIX安全概念 。

这相当于fwscanf变量参数列表直接指定为vwscanf 。

功能： int vsscanf （const char * s ，const char * template ，va_list ap ）

初步： MT安全区域| AS不安全堆| AC不安全的内存|请参阅POSIX安全概念 。

这相当于sscanf变量参数列表直接指定为vscanf 。

功能： int vswscanf （const wchar_t * s ，const wchar_t * template ，va_list ap ）

初步： MT安全区域| AS不安全堆| AC不安全的内存|请参阅POSIX安全概念 。

这相当于swscanf变量参数列表直接指定为vwscanf 。

在GNU C中，有一个特殊的结构可以用来让编译器知道函数使用了scanf样式的格式字符串。然后，它可以检查每次调用函数时参数的数量和类型，并在参数与格式字符串不匹配时发出警告。有关详细信息，请参见《 使用GNU CC》中的“ 声明函数的属性” 。
