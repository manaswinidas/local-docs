5.15.1 Argz函数

每个argz向量由指向第一个元素的指针表示，其类型为char *和大小的类型size_t ，两者都可以初始化为0代表一个空的argz向量。所有argz函数都将接受指针和大小参数，或者接受指向它们的指针（如果将对其进行修改）。

argz函数使用malloc / realloc分配/增长argz向量，因此使用这些函数创建的任何argz向量都可以通过使用释放free ;相反，任何可能增长字符串的argz函数都希望该字符串已使用malloc （那些仅检查或修改参数的argz函数可在任何类型的内存上使用）。请参阅无限制分配 。

所有执行内存分配的argz函数的返回类型为error_t ，然后返回0为了成功，以及ENOMEM如果发生分配错误。

这些功能在标准包含文件中声明argz.h 。

功能： error_t argz_create （char * const argv []，char ** argz ，size_t * argz_len ）

初步： MT安全| AS不安全堆| AC不安全的内存|请参阅POSIX安全概念 。

的argz_create函数转换Unix风格的参数向量argv （指向普通C字符串的指针的向量，以终止(char *)0 ;参见Program Arguments ）转换为具有相同元素的argz向量，并在argz和argz_len 。

功能： error_t argz_create_sep （const char * string ，int sep ，字符** argz ，size_t * argz_len ）

初步： MT安全| AS不安全堆| AC不安全的内存|请参阅POSIX安全概念 。

的argz_create_sep函数转换字符串string变成一个argz向量（返回argz和argz_len ），在每次出现字节时将其拆分为元素sep 。

功能： size_t argz_count （const char * argz ，size_t argz_len ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

返回argz向量中的元素数argz和argz_len 。

函数： void argz_extract （const char * argz ，size_t argz_len ，字符** argv ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的argz_extract函数转换argz向量argz和argz_len存入Unix风格的参数向量中argv ，通过将指针指向其中的每个元素argz进入argv ，然后是的终止符0 。 Argv必须预先分配足够的空间以容纳所有元素argz加上终止(char *)0 （ (argz_count (argz, argz_len) + 1) * sizeof (char *)个字节就足够了）。注意，将字符串指针存储到argv指向argz （它们不是副本），所以argz如果要更改它，则必须复制argv仍然活跃。此功能对于将元素传递到argz到exec函数（请参阅执行文件 ）。

函数： void argz_stringify （char * argz ，size_t len ，int sep ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的argz_stringify转换argz转换为普通字符串，其元素由字节分隔sep ，通过替换每个'\0'内argz （最后一个除外，它终止字符串）与sep 。这对于打印很方便argz以可读的方式。

函数： error_t argz_add （字符** argz ，size_t * argz_len ，const char * str ）

初步： MT安全| AS不安全堆| AC不安全的内存|请参阅POSIX安全概念 。

的argz_add函数添加字符串str到argz向量的结尾*argz和更新*argz和*argz_len相应地。

功能： error_t argz_add_sep （char ** argz ，size_t * argz_len ，const char * str ，int delim ）

初步： MT安全| AS不安全堆| AC不安全的内存|请参阅POSIX安全概念 。

的argz_add_sep功能类似于argz_add ，但是str在出现字节时将结果分成单独的元素delim 。例如，这对于将Unix搜索路径的成分添加到argz向量中非常有用，方法是使用值':'对于delim 。

函数： error_t argz_append （字符** argz ，size_t * argz_len ，const char * buf ，size_t buf_len ）

初步： MT安全| AS不安全堆| AC不安全的内存|请参阅POSIX安全概念 。

的argz_append功能追加buf_len字节起始于buf到argz向量*argz ，重新分配*argz容纳它，并添加buf_len至*argz_len 。

函数： void argz_delete （char ** argz ，size_t * argz_len ，字符* entry ）

初步： MT安全| AS不安全堆| AC不安全的内存|请参阅POSIX安全概念 。

如果entry指向argz向量中元素之一的开头*argz ， argz_delete函数将删除该条目并重新分配*argz ，修改*argz和*argz_len相应地。请注意，由于破坏性的argz函数通常会重新分配其argz参数，因此指向argz向量的指针例如entry然后将变为无效。

函数： error_t argz_insert （字符** argz ，size_t * argz_len ，字符* before ，const char * entry ）

初步： MT安全| AS不安全堆| AC不安全的内存|请参阅POSIX安全概念 。

的argz_insert函数插入字符串entry进入argz向量*argz在由指向的现有元素之前before ，重新分配*argz和更新*argz和*argz_len 。如果before是0 ， entry而是添加到末尾（就像通过argz_add ）。由于第一个元素实际上与*argz ，传入*argz作为...的价值before将导致entry在开始时被插入。

功能： char * argz_next （const char * argz ，size_t argz_len ，const char * entry ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的argz_next函数提供了一种迭代argz向量中元素的便捷方法argz 。它返回指向下一个元素的指针argz在元素之后entry ， 要么0如果后面没有元素entry 。如果entry是0 ，第一个元素argz返回。

此行为表明了两种迭代方式：

    char *entry = 0;
    while ((entry = argz_next (argz, argz_len, entry)))
      action;

（双括号是使某些C编译器关闭它们认为有问题的内容所必需的while -test）和：

    char *entry;
    for (entry = argz;
         entry;
         entry = argz_next (argz, argz_len, entry))
      action;

请注意，后者取决于argz的值为0如果为空（而不是指向空内存块的指针）；对于通过此处的函数创建的argz向量，将保持此不变性。

函数： error_t argz_replace （字符** argz ， size_t * argz_len ，常量字符* str ，const char * with ，未签名* replace_count ）

初步： MT安全| AS不安全堆| AC不安全的内存|请参阅POSIX安全概念 。

替换任何出现的字符串str在argz与with ，重新分配argz有必要的。如果replace_count不为零， *replace_count将增加执行的替换次数。
