24.2.8信号消息

上面我们提到，shell打印一条消息，描述终止子进程的信号。打印描述信号的消息的简单方法是使用功能strsignal和psignal 。这些功能使用信号编号来指定要描述的信号类型。信号号可能来自子进程的终止状态（请参见Process Completion ），也可能来自同一进程中的信号处理程序。

函数： char * strsignal （int signum ）

初步： MT不安全种族：strsignal语言环境| AS不安全的初始化i18n损坏的堆| AC不安全的初始化损坏的内存|请参阅POSIX安全概念 。

该函数返回指向静态分配的字符串的指针，该字符串包含描述信号的消息signum 。您不应该修改此字符串的内容。并且，由于可以在以后的调用中重写它，因此，如果以后需要引用它，则应该保存它的副本。

此函数是GNU扩展，在头文件中声明string.h 。

函数： void psignal （int signum ，const char * message ）

初步： MT安全区域| AS-Unsafe损坏了i18n堆| AC不安全锁损坏了内存|请参阅POSIX安全概念 。

此功能打印一条描述信号的消息signum到标准错误输出流stderr ;请参阅标准流 。

如果你打电话psignal与一个message它是一个空指针或一个空字符串， psignal只是打印对应的消息signum ，添加尾随换行符。

如果您提供非null message论据psignal使用此字符串在其输出之前添加前缀。它添加了一个冒号和一个空格字符来分隔message从对应的字符串signum 。

此函数是BSD功能，在头文件中声明signal.h 。

还有一个数组sys_siglist其中包含各种信号代码的消息。与BSD系统不同，此阵列存在strsignal 。
