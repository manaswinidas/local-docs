35.1.5条件变量

互斥不是唯一可用的同步机制。对于一些更复杂的任务，GNU C库还实现了条件变量 ，这些条件变量使程序员在解决复杂的同步问题时可以进行更高层次的思考。它们用于同步等待特定条件发生的线程。

条件变量的基本数据类型是cnd_t ：

数据类型： cnd_t

的cnd_t唯一标识条件变量对象。

以下函数用于处理条件变量：

函数： int cnd_init （cnd_t * cond ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

cnd_init初始化一个新的条件变量，由cond 。

该函数可能会返回thrd_success ， thrd_nomem ， 要么thrd_error 。

功能： int cnd_signal （cnd_t * cond ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

cnd_signal解除阻止当前正在等待由指向的条件变量的一个线程cond 。如果成功解除线程阻塞，则此函数返回thrd_success 。如果没有线程被阻塞，则此函数不执行任何操作并返回thrd_success 。否则，此函数返回thrd_error 。

功能： int cnd_broadcast （cnd_t * cond ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

cnd_broadcast解除阻止当前正在等待由指向的条件变量的所有线程cond 。该函数返回thrd_success成功。如果没有线程被阻塞，则此函数不执行任何操作并返回thrd_success 。否则，此函数返回thrd_error 。

函数： int cnd_wait （cnd_t * cond ，mtx_t * mutex ）

初步： MT安全| AS不安全锁| AC不安全锁|请参阅POSIX安全概念 。

cnd_wait原子解锁由指向的互斥锁mutex并阻止由指向的条件变量cond直到线程被发出信号cnd_signal要么cnd_broadcast 。该互斥锁在函数返回之前再次被锁定。

该函数返回thrd_success要么thrd_error 。

函数： int cnd_timedwait （cnd_t * restrict cond ，mtx_t *限制mutex ，const struct timespec * restrict time_point ）

初步： MT安全| AS不安全锁| AC不安全锁|请参阅POSIX安全概念 。

cnd_timedwait原子解锁由指向的互斥锁mutex并阻止由指向的条件变量cond直到线程被发出信号cnd_signal要么cnd_broadcast ，或直到由指出的日历时间time_point已经达到。该互斥锁在函数返回之前再次被锁定。

至于mtx_timedlock ，由于此功能需要绝对时间，因此如果需要持续时间，则必须手动计算日历时间。请参阅时间基础和日历时间 。

该函数可能会返回thrd_success ， thrd_nomem ， 要么thrd_error 。

函数： void cnd_destroy （cnd_t * cond ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

cnd_destroy销毁由指向的条件变量cond 。如果有线程在等待cond ，行为是不确定的。
