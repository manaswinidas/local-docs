24.3.4 sigaction功能实例

在“ 基本信号处理”中 ，我们举了一个使用终止信号建立简单处理程序的示例。 signal 。这是使用的等效示例sigaction ：

#include <signal.h>

void
termination_handler (int signum)
{
  struct temp_file *p;

  for (p = temp_file_list; p; p = p->next)
    unlink (p->name);
}

int
main (void)
{
  …
  struct sigaction new_action, old_action;

  /* Set up the structure to specify the new action. */
  new_action.sa_handler = termination_handler;
  sigemptyset (&new_action.sa_mask);
  new_action.sa_flags = 0;

  sigaction (SIGINT, NULL, &old_action);
  if (old_action.sa_handler != SIG_IGN)
    sigaction (SIGINT, &new_action, NULL);
  sigaction (SIGHUP, NULL, &old_action);
  if (old_action.sa_handler != SIG_IGN)
    sigaction (SIGHUP, &new_action, NULL);
  sigaction (SIGTERM, NULL, &old_action);
  if (old_action.sa_handler != SIG_IGN)
    sigaction (SIGTERM, &new_action, NULL);
  …
}

该程序只是加载new_action具有所需参数的结构，并将其传递给sigaction呼叫。的用法sigemptyset稍后描述；请参阅阻塞信号 。

在示例中使用signal ，我们避免处理先前设置为忽略的信号。在这里，我们可以通过使用的功能避免瞬间更改信号处理程序sigaction这样我们就可以检查当前操作，而无需指定新操作。

这是另一个例子。它检索有关当前操作的信息SIGINT而无需更改该操作。 

struct sigaction query_action;

if (sigaction (SIGINT, NULL, &query_action) < 0)
  /* sigaction returns -1 in case of error. */
else if (query_action.sa_handler == SIG_DFL)
  /* SIGINT is handled in the default, fatal manner. */
else if (query_action.sa_handler == SIG_IGN)
  /* SIGINT is ignored. */
else
  /* A programmer-defined signal handler is in effect. */
