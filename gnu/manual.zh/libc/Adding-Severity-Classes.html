12.22.2添加严重等级

除了使用环境变量外，还有一种引入严重性等级的可能性SEV_LEVEL 。这简化了在正在运行的程序中引入新类的任务。一个可以使用setenv要么putenv函数来设置环境变量，但这很麻烦。

函数： int addevereverity （int severity ，const char * string ）

初步： MT安全| AS不安全堆锁| AC不安全锁记忆|请参阅POSIX安全概念 。

此功能允许引入新的严重性等级，可以通过severity的参数fmtmsg功能。的severity的参数addseverity必须与具有相同名称的参数值匹配fmtmsg和string是实际消息中打印的字符串，而不是数字值。

如果string是NULL严重性等级，其数值根据severity已移除。

无法覆盖或删除默认的严重级别之一。所有呼叫addseverity与severity设置为默认类的值之一将失败。

返回值为MM_OK如果任务已成功执行。如果返回值为MM_NOTOK出了些问题。这可能意味着不再有可用的内存或必须删除的类不可用。

尽管《 X / Open可移植性指南》中没有指定此功能， fmtsmg功能是。它在System V系统上可用。
