1.3.1头文件

C程序使用的库实际上包括两部分：定义类型和宏以及声明变量和函数的头文件 ；以及包含变量和函数定义的实际库或档案库。

（回想一下，在C中， 声明仅提供信息，说明函数或变量存在并给出其类型。对于函数声明，可能还会提供有关其参数类型的信息。声明的目的是允许编译器正确处理对声明的变量和函数的引用。另一方面， 定义实际上是为变量分配存储空间或说明函数的功能。）

为了使用GNU C库中的功能，您应确保程序源文件包含适当的头文件。这样一来，编译器就可以使用这些工具的声明，并且可以正确处理对其的引用。编译完程序后，链接程序会将这些引用解析为存档文件中提供的实际定义。

标头文件由' #include ' 预处理程序指令。C语言支持此指令的两种形式：首先，

#include "header"

通常用于包含头文件header你写自己；其中将包含描述特定应用程序不同部分之间的接口的定义和声明。相比之下，

#include <file.h>

通常用于包含头文件file.h包含标准库的定义和声明。该文件通常由系统管理员安装在标准位置。您应该将第二种形式用于C库头文件。

通常，“ #include指令位于其他任何代码之前的C源文件的顶部。如果您在源文件中添加一些注释，以解释文件中的代码的作用（一个好主意），请在“ #include之后，紧接着功能测试宏定义（请参阅功能测试宏 ）。

有关使用头文件和“ #include指令，请参见《 GNU C预处理程序手册 》中的 头文件 。

GNU C库提供了几个头文件，每个头文件都包含一组相关设施的类型和宏定义以及变量和函数声明。这意味着您的程序可能需要包含多个头文件，具体取决于您所使用的设施。

某些库头文件会自动包含其他库头文件。但是，就编程风格而言，您不应依赖于此。最好明确地包含所使用的库工具所需的所有头文件。GNU C库头文件的编写方式与头文件是否被意外包含多次无关紧要。第二次包含头文件无效。同样，如果您的程序需要包含多个头文件，则包含它们的顺序也无关紧要。

兼容性说明：在任何ISO C 实现中，以任意顺序和任意次数包含标准头文件都是 。但是，传统上在许多较旧的C实现中情况并非如此。

严格来说，你不必包含头文件使用它声明的函数;您可以根据本手册中的说明自行声明该函数。但是通常最好包含头文件，因为它可以定义其他方式无法使用的类型和宏，并且可以为某些功能定义更有效的宏替换。这也是获得正确声明的肯定方法。
