25.3.5.2 Argp解析状态

argp解析器函数的第三个参数（请参阅Argp解析器函数 ）是指向a的指针。 struct argp_state ，其中包含有关选项解析状态的信息。

数据类型： struct argp_state

该结构具有以下字段，可以根据需要进行修改：

const struct argp *const root_argp

顶级argp解析器正在解析。请注意，这通常是不一样的struct argp传入argp_parse通过调用程序。参见Argp 。这是一个内部argp解析器，其中包含通过以下方式实现的选项argp_parse本身，例如“ --help '。

int argc
char **argv

正在解析的参数向量。这可以修改。

int next

中的索引argv下一个要解析的参数。这可以修改。

消耗输入中所有剩余参数的一种方法是设置state->next = state->argc ，也许在记录了next字段以查找消耗的参数。通过减少此字段，然后修改，可以立即重新分析当前选项state->argv[state->next]反映应该重新检查的选项。

unsigned flags

提供给的标志argp_parse 。尽管有些标志可能仅在argp_parse首先被调用。参见Argp标志 。

unsigned arg_num

在使用调用解析函数时key论点ARGP_KEY_ARG ，代表从0开始的当前arg的编号。每次递增ARGP_KEY_ARG通话返回。在其他所有时间，这是ARGP_KEY_ARG已处理的参数。

int quoted

如果非零，则索引在argv特殊'之后的第一个参数的--的论点。这样可以防止将其后的任何内容解释为选项。仅在参数解析进行到此点之后设置。

void *input

从的调用者传入的任意指针argp_parse ， 在里面input论点。

void **child_inputs

这些是将传递给子解析器的值。此向量的长度将与当前解析器中的子代数目相同。每个子解析器将被赋予以下值： state->child_inputs[i]作为其 state->input场，在哪里i是此解析器中子级的索引children领域。参见Argp儿童 。

void *hook

供解析器功能使用。初始化为0，否则被argp忽略。

char *name

打印消息时使用的名称。初始化为argv[0] ， 要么program_invocation_name如果argv[0]不可用。

FILE *err_stream
FILE *out_stream

argp打印时使用的stdio流。错误消息被打印到err_stream ，所有其他输出，例如“ --help '输出）到out_stream 。这些被初始化为stderr和stdout分别。请参阅标准流 。

void *pstate

专用，供argp实现使用。
