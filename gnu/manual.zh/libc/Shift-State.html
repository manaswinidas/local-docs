6.4.3非重入功能中的状态

在某些多字节字符代码中，任何特定字节序列的含义都不固定；它取决于同一字符串中较早出现的其他序列。通常，只有几个序列可以改变其他序列的含义。这几个称为移位序列 ，我们说它们为随后的其他序列设置移位状态 。

为了说明换挡状态和换挡顺序，假设我们决定该顺序0200 （仅一个字节）进入日语模式，其中，从0240至0377是单个字符，而0201进入Latin-1模式，其中从0240至0377是字符，并根据ISO Latin-1字符集进行解释。这是一个多字节代码，具有两个备用的移位状态（“日语模式”和“拉丁1模式”），以及两个指定特定移位状态的移位序列。

当使用的多字节字符代码具有移位状态时，则mblen ， mbtowc和wctomb在扫描字符串时必须保持并更新当前的换档状态。要使其正常工作，必须遵循以下规则：

  • 开始扫描字符串之前，请使用空指针为多字节字符地址调用函数，例如， mblen (NULL, 0) 。这会将换档状态初始化为其标准初始值。
  • 依次扫描字符串一个字符。不要“备份”和重新扫描已经扫描过的字符，也不要穿插处理不同的字符串。

这是使用的示例mblen遵循以下规则：

void
scan_string (char *s)
{
  int length = strlen (s);

  /* Initialize shift state.  */
  mblen (NULL, 0);

  while (1)
    {
      int thischar = mblen (s, length);
      /* Deal with end of string and invalid characters.  */
      if (thischar == 0)
        break;
      if (thischar == -1)
        {
          error ("invalid multibyte character");
          break;
        }
      /* Advance past this character.  */
      s += thischar;
      length -= thischar;
    }
}

功能mblen ， mbtowc和wctomb当使用使用移位状态的多字节代码时，它们是不可重入的。但是，没有其他库函数调用这些函数，因此您不必担心移位状态会神秘地改变。
