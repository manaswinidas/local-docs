14.9.2读取文件的属性

要检查文件的属性，请使用功能stat ， fstat和lstat 。他们将属性信息返回到struct stat宾语。这三个函数都在头文件中声明sys/stat.h 。

函数： int stat （const char * filename ，struct stat * buf ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的stat函数返回有关由命名的文件的属性的信息filename 中所指向的结构buf 。

如果filename是符号链接的名称，您获得的属性描述了链接指向的文件。如果链接指向不存在的文件名，则stat无法报告不存在的文件。

返回值为0如果操作成功，或者-1失败。除了常见的文件名错误（请参阅文件名错误 ，以下内容errno为此功能定义了错误条件：

ENOENT

由命名的文件filename不存在。

当源编译时_FILE_OFFSET_BITS == 64这个功能其实stat64因为LFS接口透明地替换了常规实现。

功能： int stat64 （const char * filename ，struct stat64 * buf ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

该功能类似于stat但它也可以在32位系统上处理大于2 ^ 31字节的文件。为了能够做到这一点，结果存储在类型为的变量中struct stat64到哪buf必须指出。

当源编译时_FILE_OFFSET_BITS == 64该功能在名称下可用stat因此透明地替换了32位计算机上小文件的接口。

函数： int fstat （int filedes ，struct stat * buf ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的fstat功能就像stat ，不同之处在于它使用打开的文件描述符作为参数而不是文件名。请参阅低级I / O。

喜欢stat ， fstat退货0成功与-1失败。下列errno错误条件是为fstat ：

EBADF

的filedes参数不是有效的文件描述符。

当源编译时_FILE_OFFSET_BITS == 64这个功能其实fstat64因为LFS接口透明地替换了常规实现。

函数： int fstat64 （int filedes ，struct stat64 * buf ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

该功能类似于fstat但可以在32位平台上处理大型文件。对于大文件，文件描述符filedes应该通过open64要么creat64 。的buf指针指向类型的变量struct stat64能够代表更大的值。

当源编译时_FILE_OFFSET_BITS == 64该功能在名称下可用fstat因此透明地替换了32位计算机上小文件的接口。

函数： int lstat （const char * filename ，struct stat * buf ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的lstat功能就像stat ，只是它不遵循符号链接。如果filename是符号链接的名称， lstat返回有关链接本身的信息；除此以外lstat像stat 。请参阅符号链接 。

当源编译时_FILE_OFFSET_BITS == 64这个功能其实lstat64因为LFS接口透明地替换了常规实现。
