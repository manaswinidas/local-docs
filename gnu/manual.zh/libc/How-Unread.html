12.10.2使用ungetc做未读

未读字符的功能称为ungetc ，因为它会逆转getc 。

函数： int ungetc （int c ，文件* stream ）

初步： MT安全|不安全AS损坏| AC不安全锁损坏|请参阅POSIX安全概念 。

的ungetc功能推回字符c进入输入流stream 。所以下一个输入stream会读c首先。

如果c是EOF ， ungetc什么都不做，只是返回EOF 。这可以让您打电话ungetc返回值为getc无需检查错误getc 。

推回的字符不必与实际从流中读取的最后一个字符相同。实际上，在使用取消读取字符之前，实际上不必从流中读取任何字符。 ungetc ！但这是编写程序的一种奇怪方法。通常ungetc仅用于未读从同一流中读取的字符。即使在以二进制模式打开的文件上，GNU C库也支持此功能，但其他系统可能不支持。

GNU C库仅支持一个推回字符，也就是说，它无法调用ungetc两次之间不做输入。其他系统可能会让您后退多个字符；然后从流中读取以相反的顺序检索字符。

推回字符不会更改文件。仅影响流的内部缓冲。如果有文件定位功能（如fseek ， fseeko要么rewind ;参见File Positioning ），则所有待处理的推回字符都将被丢弃。

在文件末尾的流上未读取字符会清除该流的文件结束指示符，因为它使输入字符可用。读取该字符后，尝试再次读取将遇到文件结尾。

函数： wint_t ungetwc （wint_t wc ，文件* stream ）

初步： MT安全|不安全AS损坏| AC不安全锁损坏|请参阅POSIX安全概念 。

的ungetwc函数的行为就像ungetc只是它推开了宽广的角色。

这是显示使用的示例getc和ungetc跳过空白字符。当此函数到达非空白字符时，它将取消读取该字符，以便在流的下一次读取操作中再次看到该字符。 

#include <stdio.h>
#include <ctype.h>

void
skip_whitespace (FILE *stream)
{
  int c;
  do
    /* No need to check for EOF because it is not
       isspace, and ungetc ignores EOF.  */
    c = getc (stream);
  while (isspace (c));
  ungetc (c, stream);
}
