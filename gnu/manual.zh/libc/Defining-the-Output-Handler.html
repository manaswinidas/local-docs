12.13.3定义输出处理程序

现在让我们看看如何定义作为参数传递给的处理程序和arginfo函数。 register_printf_function 。

兼容性说明：接口在GNU C库2.0版中进行了更改。以前第三个参数是类型va_list * 。

您应该使用以下原型定义处理程序函数：

int function (FILE *stream, const struct printf_info *info,
		    const void *const *args)

的stream传递给处理程序函数的参数是应该向其写入输出的流。

的info参数是指向结构的指针，该结构包含有关模板字符串中转换所包含的各种选项的信息。您不应在处理程序函数中修改此结构。有关此数据结构的说明，请参见“ 转换说明符选项” 。

的args是指向参数数据的指针的向量。参数数量是通过调用用户提供的参数信息函数确定的。

您的处理函数应返回一个值，就像printf可以：应返回已写入的字符数，或返回负值以指示错误。

数据类型： printf_function

这是处理程序函数应具有的数据类型。

如果您要使用parse_printf_format在您的应用程序中 ，您还必须定义一个函数以作为arginfo-function您安装的每个新转换的参数register_printf_function 。

您必须使用以下原型来定义这些功能：

int function (const struct printf_info *info,
		    size_t n, int *argtypes)

该函数的返回值应该是转换期望的参数数量。该函数还应填写不超过n的要素argtypes包含有关每个自变量类型的信息的数组。该信息使用各种“ PA_ '宏。（您会注意到这是相同的调用约定parse_printf_format本身使用。）

数据类型： printf_arginfo_function

此类型用于描述返回有关转换说明符使用的参数的数量和类型的信息的函数。
