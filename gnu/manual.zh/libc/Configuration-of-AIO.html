13.11.5如何优化AIO实施

POSIX标准未指定如何实现AIO功能。它们可能是系统调用，但也可以在用户级别上模拟它们。

在撰写本文时，可用的实现是用户级别的实现，它使用线程来处理排队的请求。尽管此实现需要对限制做出一些决定，但在GNU C库中最好避免硬限制。因此，GNU C库提供了一种根据个人用途调整AIO实现的方法。

数据类型： struct aioinit

此数据类型用于将配置或可调参数传递给实现。程序必须初始化此结构的成员，然后使用aio_init功能。

int aio_threads

该成员指定可以在任何一次使用的最大线程数。

int aio_num

此数字提供对同时入队请求的最大数量的估计。

int aio_locks

没用过。

int aio_usedba

没用过。

int aio_debug

没用过。

int aio_numusers

没用过。

int aio_reserved[2]

没用过。

功能： void aio_init （const struct aioinit * init ）

初步： MT安全| AS不安全锁| AC不安全锁|请参阅POSIX安全概念 。

必须在任何其他AIO函数之前调用此函数。称它为完全自愿，因为这仅是为了帮助AIO实施更好地执行。

致电之前aio_init ，类型变量的成员struct aioinit必须初始化。然后对此变量的引用作为参数传递给aio_init它本身可能不会注意这些提示。

该函数没有返回值，也没有定义错误情况。它是对Irix 6 SGI实现的建议的扩展。它不受POSIX.1b或Unix98的保护。
