15.3 FIFO特殊文件

FIFO特殊文件与管道相似，区别在于它以不同的方式创建。FIFO特殊文件不是匿名通信通道，而是通过调用以下命令进入文件系统mkfifo 。

以这种方式创建FIFO特殊文件后，任何进程都可以以与普通文件相同的方式打开该文件以进行读取或写入。但是，必须在两端同时打开它，然后才能继续对其进行任何输入或输出操作。打开FIFO以正常读取块，直到其他进程打开相同的FIFO进行写入为止，反之亦然。

的mkfifo函数在头文件中声明sys/stat.h 。

函数： int mkfifo （const char * filename ，mode_t mode ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的mkfifo函数使用名称创建FIFO特殊文件filename 。的mode参数用于设置文件的权限；请参阅设置权限 。

的正常成功返回值mkfifo是0 。如果有错误， -1返回。除了常见的文件名错误（请参阅文件名错误 ）之外，以下内容errno为此功能定义了错误条件：

EEXIST

命名文件已存在。

ENOSPC

目录或文件系统无法扩展。

EROFS

包含文件的目录位于只读文件系统上。
