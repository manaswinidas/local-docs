31.3.1.1 fstab文件

文件条目的内部表示是struct fstab ，定义于fstab.h 。

数据类型： struct fstab

此结构与getfsent ， getfsspec和getfsfile功能。

char *fs_spec

此元素描述从中安装文件系统的设备。通常，这是特殊设备的名称，例如硬盘分区，但也可能是或多或少的通用字符串。对于NFS ，它将是主机名和目录名的组合。

即使没有声明元素const它不应该被修改。失踪const具有历史原因，因为此函数早于ISO C 。对于此结构的其他字符串元素也是如此。

char *fs_file

这描述了本地系统上的安装点。即，访问此文件系统中的任何文件都将隐式或显式地将此字符串作为前缀。

char *fs_vfstype

这是文件系统的类型。根据底层内核的理解，它可以是任何字符串。

char *fs_mntops

这是一个字符串，其中包含通过mount呼叫。同样，这几乎可以是任何东西。可以有多个选项，其他选项之间用逗号隔开。每个选项都包含一个名称和一个可选值部分，由=字符。

如果必须处理此元素的值，则理想情况下应使用getsubopt功能;参见Suboptions 。

const char *fs_type

该名称选择不当。该元素指向一个字符串（可能在fs_mntops字符串），它描述了文件系统的安装方式。 fstab定义五个宏来描述可能的值：

FSTAB_RW

文件系统在读取和写入启用的情况下挂载。

FSTAB_RQ

文件系统在读取和写入启用的情况下挂载。写访问受配额限制。

FSTAB_RO

文件系统以只读方式挂载。

FSTAB_SW

这不是真正的文件系统，而是交换设备。

FSTAB_XX

此条目来自fstab文件被完全忽略。

测试这些值的相等性必须使用strcmp因为这些都是字符串。比较指针可能总是会失败。

int fs_freq

此元素以天为单位描述转储频率。

int fs_passno

此元素描述并行转储上的通过号。它与dump Unix系统上使用的实用程序。

读取的全部内容fstab GNU C库文件包含一组按常规方式设计的三个函数。

函数： int setfsent （无效）

初步： MT不安全竞赛：现在| AS不安全堆损坏锁| AC不安全损坏的锁内存fd |请参阅POSIX安全概念 。

此函数确保内部读取指针为fstab文件位于文件的开头。这可以通过打开文件或重置读取指针来完成。

由于文件句柄在libc内部，因此此函数不是线程安全的。

如果操作成功并且该函数返回了一个非零值getfs*函数可用于读取文件的条目。

功能： void endfsent （无效）

初步： MT不安全竞赛：现在| AS不安全堆损坏锁| AC不安全损坏的锁内存fd |请参阅POSIX安全概念 。

此功能可确保先前调用所获取的所有资源setfsent （通过显式或隐式调用getfsent ）被释放。

函数： struct fstab * getfsent （void）

初步： MT不安全种族：目前没有语言环境| AS不安全的损坏堆锁| AC不安全损坏的锁内存|请参阅POSIX安全概念 。

此函数返回的下一个条目fstab文件。如果这是对任何函数处理的第一次调用fstab自程序启动或上一次调用以来endfsent ，文件将被打开。

该函数返回一个指向类型变量的指针struct fstab 。所有线程共享此变量，因此此函数不是线程安全的。如果发生错误getfsent返回一个NULL指针。

功能： struct fstab * getfsspec （const char * name ）

初步： MT不安全种族：目前没有语言环境| AS不安全的损坏堆锁| AC不安全损坏的锁内存|请参阅POSIX安全概念 。

此函数返回的下一个条目fstab字符串等于的文件name指向fs_spec元件。由于通常每个特殊设备只有一个条目，因此对于同一个参数多次调用此函数是没有意义的。如果这是对任何函数处理的第一次调用fstab自程序启动或上一次调用以来endfsent ，文件将被打开。

该函数返回一个指向类型变量的指针struct fstab 。所有线程共享此变量，因此此函数不是线程安全的。如果发生错误getfsent返回一个NULL指针。

功能： struct fstab * getfsfile （const char * name ）

初步： MT不安全种族：目前没有语言环境| AS不安全的损坏堆锁| AC不安全损坏的锁内存|请参阅POSIX安全概念 。

此函数返回的下一个条目fstab字符串等于的文件name指向fs_file元件。由于通常每个安装点只有一个条目，因此对于同一个参数多次调用此函数是没有意义的。如果这是对任何函数处理的第一次调用fstab自程序启动或上一次调用以来endfsent ，文件将被打开。

该函数返回一个指向类型变量的指针struct fstab 。所有线程共享此变量，因此此函数不是线程安全的。如果发生错误getfsent返回一个NULL指针。
