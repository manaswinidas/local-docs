24.2.6操作错误信号

这些信号用于报告程序执行的操作所产生的各种错误。它们不一定表示程序中的编程错误，而是表示阻止操作系统调用完成的错误。所有这些控件的默认操作是导致进程终止。

巨集： int SIGPIPE

管道破损。如果使用管道或FIFO，则必须设计应用程序，以便一个进程打开管道进行读取，然后另一个进程开始写入。如果读取过程从未开始或意外终止，则写入管道或FIFO会引发SIGPIPE信号。如果SIGPIPE被阻止，处理或忽略，令人反感的呼叫失败并显示EPIPE代替。

管道和FIFO特殊文件在管道和FIFO中有更详细的讨论。

的另一个原因SIGPIPE是当您尝试输出到未连接的套接字时。请参阅发送数据 。

巨集： int SIGLOST

资源丢失。当您在NFS文件上拥有咨询锁定时，会生成此信号，并且NFS服务器将重新启动并忘记您的锁定。

在GNU / Hurd系统上， SIGLOST当任何服务器程序意外终止时生成。通常可以忽略信号。对死亡的服务器进行的任何调用都将返回错误。

巨集： int SIGXCPU

超过CPU时间限制。当进程超出其CPU时间的软资源限制时，将生成此信号。请参阅资源限制 。

巨集： int SIGXFSZ

超出文件大小限制。当进程尝试扩展文件时，将生成此信号，以使其超出进程的文件大小的软资源限制。请参阅资源限制 。
