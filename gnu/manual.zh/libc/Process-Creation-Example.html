26.9流程创建示例

这是一个示例程序，显示了如何编写类似于内置函数的函数system 。它执行它的command参数使用等价于' sh -c command '。 

#include <stddef.h>
#include <stdlib.h>
#include <unistd.h>
#include <sys/types.h>
#include <sys/wait.h>

/* Execute the command using this shell program.  */
#define SHELL "/bin/sh"


int
my_system (const char *command)
{
  int status;
  pid_t pid;


  pid = fork ();
  if (pid == 0)
    {
      /* This is the child process.  Execute the shell command. */
      execl (SHELL, SHELL, "-c", command, NULL);
      _exit (EXIT_FAILURE);
    }
  else if (pid < 0)
    /* The fork failed.  Report failure.  */
    status = -1;
  else
    /* This is the parent process.  Wait for the child to complete.  */
    if (waitpid (pid, &status, 0) != pid)
      status = -1;
  return status;
}

在此示例中，需要注意几件事。

记住第一argv提供给程序的参数表示正在执行的程序的名称。这就是为什么在execl ， SHELL提供一次以命名要执行的程序，第二次提供用于argv[0] 。

的execl如果成功，子进程中的调用不会返回。如果失败，则必须采取措施使子进程终止。只是返回一个错误的状态码return将使两个进程运行原始程序。相反，正确的行为是让子进程向其父进程报告失败。

呼叫_exit为此。使用的原因_exit代替exit是为了避免刷新完全缓冲的流，例如stdout 。这些流的缓冲区可能包含由父进程从父进程复制的数据。 fork ，最终将由父进程输出的数据。呼唤exit在孩子中将输出两次数据。请参阅内部端接 。
