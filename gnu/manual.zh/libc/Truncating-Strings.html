5.6复制时截断字符串

本节中描述的函数将字符串或数组的可能被截断的内容复制或连接到另一个，或者对于宽字符串也类似。它们遵循其标头约定中的字符串复制功能。请参见复制字符串和数组 。' str '函数在头文件中声明string.h和“ wc '函数在文件中声明wchar.h 。

功能： char * strncpy （char * restrict to ，const char *限制from ，size_t size ）

除此以外from必须是长度小于size 。在这种情况下strncpy复制所有from ，然后有足够的空字节加起来size全部字节。

功能： char * strncat （char * restrict to ，const char *限制from ，size_t size ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

这个功能就像strcat除了不超过size来自的字节from附加到to和from无需以null终止。一个空字节也总是附加到to ，因此总分配大小为to必须至少size + 1比其初始长度更长的字节。

的strncat功能可以这样实现：

char *
strncat (char *to, const char *from, size_t size)
{
  size_t len = strlen (to);
  memcpy (to + len, from, strnlen (from, size));
  to[len + strnlen (from, size)] = '\0';
  return to;
}

的行为strncat如果字符串重叠，则未定义。

作为伴侣strncpy ， strncat是为很少使用的数组设计的，该数组由非空字节后跟零个或多个空字节组成。如下所述，此功能通常是处理文本的不佳选择。此外，此功能还存在严重的性能问题。请参见串联字符串 。

函数： wchar_t * wcsncat （wchar_t * restrict wto ，const wchar_t *限制wfrom ，size_t size ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

这个功能就像wcscat除了不超过size来自的宽字符from附加到to和from无需以null终止。单个null宽字符也总是附加到to ，因此总分配大小为to必须至少wcsnlen (wfrom, size) + 1宽字符长于其初始长度。

的wcsncat功能可以这样实现：

wchar_t *
wcsncat (wchar_t *restrict wto, const wchar_t *restrict wfrom,
         size_t size)
{
  size_t len = wcslen (wto);
  memcpy (wto + len, wfrom, wcsnlen (wfrom, size) * sizeof (wchar_t));
  wto[len + wcsnlen (wfrom, size)] = L'\0';
  return wto;
}

的行为wcsncat如果字符串重叠，则未定义。

如下所述，此功能通常是处理文本的不佳选择。此外，此功能还存在严重的性能问题。请参见串联字符串 。

因为这些函数可能会突然截断字符串或宽字符串，所以它们通常是处理文本的较差选择。处理或连接多字节字符串时，它们可以在多字节字符内截断，因此结果不是有效的多字节字符串。当组合或串联多字节或宽字符串时，它们可能会在组合字符后截断输出，从而导致字素损坏。即使在处理单字节字符串时，它们也可能导致错误：例如，在计算仅ASCII的用户名时，截断的名称可以标识错误的用户。

尽管可以通过手动将对复制函数的调用替换为对截断函数的调用来防止某些缓冲区溢出，但是通常有更简单，更安全的自动技术导致缓冲区溢出来可靠地终止程序，例如GCC -fcheck-pointer-bounds和-fsanitize=address选项。请参阅使用GCC中的 用于调试程序或GCC的选项 。因为截断功能可以掩盖否则会被自动技术捕获的应用程序错误，因此仅当应用程序的基础逻辑需要截断时才应使用这些功能。

注意： GNU程序不应截断字符串或宽字符串以适应任意大小限制。请参见在GNU编码标准中 编写健壮的程序 。代替字符串截断函数，通常最好使用动态内存分配（请参见Unconstrained Allocation ）和函数，例如strdup要么asprintf构造字符串。
