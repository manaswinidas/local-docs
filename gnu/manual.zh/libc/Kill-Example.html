24.6.4使用kill沟通

这是一个更长的示例，显示了如何将信号用于进程间通信。这就是SIGUSR1和SIGUSR2提供信号。由于默认情况下这些信号是致命的，因此应该接收这些信号的过程必须通过signal要么sigaction 。

在此示例中，父进程派生了一个子进程，然后等待该子进程完成其初始化。子进程通过发送一个父进程来通知父进程何时准备就绪。 SIGUSR1信号，使用kill功能。 



#include <signal.h>
#include <stdio.h>
#include <sys/types.h>
#include <unistd.h>


/* When a SIGUSR1 signal arrives, set this variable. */
volatile sig_atomic_t usr_interrupt = 0;

void
synch_signal (int sig)
{
  usr_interrupt = 1;
}

/* The child process executes this function. */
void
child_function (void)
{
  /* Perform initialization. */
  printf ("I'm here!!!  My pid is %d.\n", (int) getpid ());

  /* Let parent know you’re done. */
  kill (getppid (), SIGUSR1);

  /* Continue with execution. */
  puts ("Bye, now....");
  exit (0);
}

int
main (void)
{
  struct sigaction usr_action;
  sigset_t block_mask;
  pid_t child_id;

  /* Establish the signal handler. */
  sigfillset (&block_mask);
  usr_action.sa_handler = synch_signal;
  usr_action.sa_mask = block_mask;
  usr_action.sa_flags = 0;
  sigaction (SIGUSR1, &usr_action, NULL);

  /* Create the child process. */
  child_id = fork ();
  if (child_id == 0)
    child_function ();          /* Does not return. */


  /* Busy wait for the child to send a signal. */
  while (!usr_interrupt)
    ;


  /* Now continue execution. */
  puts ("That's all, folks!");

  return 0;
}

本示例使用繁忙的等待，这很不好，因为它浪费了其他程序可能会使用的CPU周期。最好让系统等待直到信号到达。请参阅“ 等待信号”中的示例。
