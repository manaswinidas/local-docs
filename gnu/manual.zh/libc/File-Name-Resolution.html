11.2.2文件名解析

文件名由用斜杠（' / '）个字符。在GNU C库支持的系统上，多个连续的' / '字符等同于单个' /字符。

确定文件名引用哪个文件的过程称为文件名解析 。这是通过按从左到右的顺序检查组成文件名的组件，并将每个连续的组件放置在前一个组件命名的目录中来执行的。当然，被引用为目录的每个文件必须实际上存在，它们是目录而不是常规文件，并且具有适当的权限以供进程访问；否则，文件名解析将失败。

如果文件名以“ / '，文件名中的第一个组件位于进程的根目录中（通常系统上的所有进程都具有相同的根目录）。这样的文件名称为绝对文件名 。

否则，文件名中的第一个组件位于当前工作目录中（请参阅Working Directory ）。这种文件名称为相对文件名 。

文件名组件. （“点”）和.. （“圆点”）具有特殊含义。每个目录都有这些文件名组件的条目。文件名组件.指目录本身，而文件名部分..指的是其父目录 （包含相关目录链接的目录）。作为一种特殊情况..根目录中的根目录是指根目录本身，因为它没有父目录；从而/..是相同的/ 。

以下是文件名的一些示例：

/a

文件名为a ，位于根目录中。

/a/b

文件名为b ，位于名为a在根目录中。

a

文件名为a ，在当前工作目录中。

/a/./b

这和/a/b 。

./a

文件名为a ，在当前工作目录中。

../a

文件名为a ，位于当前工作目录的父目录中。

命名目录的文件名可以以' / '。您可以指定一个文件名/引用根目录，但是空字符串不是有意义的文件名。如果要引用当前工作目录，请使用文件名.要么./ 。

与某些其他操作系统不同，作为其文件名语法的一部分，GNU系统没有对文件类型（或扩展名）或文件版本的任何内置支持。许多程序和实用程序都使用约定的文件名-例如，包含C源代码的文件通常具有后缀' .c '-但是文件系统本身没有任何东西可以执行这种约定。
