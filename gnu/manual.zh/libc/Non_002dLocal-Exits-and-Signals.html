23.3非本地出口和信号

在BSD Unix系统中， setjmp和longjmp还保存和恢复被阻塞的信号集；请参阅阻塞信号 。但是，POSIX.1标准要求setjmp和longjmp不更改被阻止信号的集合，并提供了另外一对功能（ sigsetjmp和siglongjmp ）以获取BSD行为。

的行为setjmp和longjmp在GNU C库中由功能测试宏控制；请参阅功能测试宏 。GNU C库中的默认行为是POSIX.1行为，而不是BSD行为。

本节中的设施在头文件中声明setjmp.h 。

数据类型： sigjmp_buf

这类似于jmp_buf ，除了它还可以存储有关阻塞信号集的状态信息。

函数： int sigsetjmp （sigjmp_buf state ，int savesigs ）

初步： MT安全| AS-不安全锁/门| AC不安全锁/门|请参阅POSIX安全概念 。

这类似于setjmp 。如果savesigs不为零，则将阻塞信号集保存在state并将在以下情况下恢复siglongjmp稍后执行此state 。

功能： 无效 siglongjmp （sigjmp_buf state ，int value ）

初步： MT安全| AS-Unsafe插件损坏了锁/锁|不安全的AC损坏锁/锁|请参阅POSIX安全概念 。

这类似于longjmp除了它的类型state论点。如果sigsetjmp称这个state使用非零savesigs旗， siglongjmp还可以恢复被阻止的信号集。
