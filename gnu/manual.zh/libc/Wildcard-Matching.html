10.1通配符匹配

本节介绍如何将通配符模式与特定字符串进行匹配。结果是是或否的答案：字符串是否适合该模式。此处描述的符号都在中声明fnmatch.h 。

函数： int fnmatch （const char * pattern ，const char * string ，int flags ）

初步： MT安全环境语言环境| AS不安全堆| AC不安全的内存|请参阅POSIX安全概念 。

此函数测试字符串是否string匹配模式pattern 。它返回0他们是否匹配；否则，返回非零值FNM_NOMATCH 。论点pattern和string都是字符串。

争论flags是标志位的组合，可更改匹配的详细信息。请参阅以下有关已定义标志的列表。

在GNU C库中， fnmatch有时可能会通过返回不等于的非零值来报告“错误” FNM_NOMATCH 。

这些是flags参数：

FNM_FILE_NAME

对待“ / '字符，用于匹配文件名。如果设置了此标志，则通配符构造在pattern无法匹配“ /在string 。因此，匹配“ / '与显式' /在pattern 。

FNM_PATHNAME

这是别名FNM_FILE_NAME ;它来自POSIX.2。我们不建议使用此名称，因为我们不使用术语“路径名”作为文件名。

FNM_PERIOD

对待“ . '字符，特别是如果它出现在string 。如果设置了此标志，则通配符构造在pattern无法匹配“ . '作为的第一个字符string 。

如果同时设置FNM_PERIOD和FNM_FILE_NAME ，那么特殊待遇适用于“ . “关注” /以及. ' 在。。。之初string 。（外壳使用FNM_PERIOD和FNM_FILE_NAME标志一起匹配文件名。）

FNM_NOESCAPE

不要对待“ \ '字符特别是在模式中。一般， ' \ '引用了以下字符，并关闭了其特殊含义（如果有的话），以使其仅与自身匹配。启用报价后，模式' \? '仅匹配字符串' ? ”，因为该模式中的问号的作用类似于普通字符。

如果您使用FNM_NOESCAPE ， 然后 ' \ '是普通字符。

FNM_LEADING_DIR

忽略以'开头的尾随字符/在string ;也就是说，测试是否string以目录名称开头pattern火柴。

如果设置了此标志，则“ foo* ' 要么 ' foobar '作为模式将匹配字符串' foobar/frobozz '。

FNM_CASEFOLD

在比较中忽略大小写string至pattern 。

FNM_EXTMATCH

除了正常模式，还可以识别引入的扩展模式ksh 。这些模式以下表中说明的形式编写，其中pattern-list是一个|模式列表。

?(pattern-list)

如果该模式中的任何一个模式出现零次或一次，则该模式匹配pattern-list允许匹配输入字符串。

*(pattern-list)

如果该模式中的任何一个模式出现零次或多次，则该模式匹配pattern-list允许匹配输入字符串。

+(pattern-list)

如果该模式中的任何一个或多个模式出现一次，则该模式匹配pattern-list允许匹配输入字符串。

@(pattern-list)

如果该模式中的任何一种模式恰好出现一次，则该模式匹配pattern-list允许匹配输入字符串。

!(pattern-list)

如果输入字符串不能与字符串中的任何模式匹配，则模式匹配pattern-list 。
