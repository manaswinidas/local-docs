16.13网络数据库

许多系统附带一个数据库，该数据库记录了系统开发人员已知的网络列表。通常将其保存在文件中/etc/networks或名称服务器中的等效名称。该数据库对于路由诸如route ，但对于仅通过网络进行通信的程序没有用。我们提供了访问该数据库的函数，这些函数在netdb.h 。

数据类型： struct netent

此数据类型用于表示有关网络数据库中条目的信息。它具有以下成员：

char *n_name

这是网络的“正式”名称。

char **n_aliases

这些是网络的备用名称，表示为字符串向量。空指针终止数组。

int n_addrtype

这是网络号的类型；这总是等于AF_INET用于Internet网络。

unsigned long int n_net

这是网络号。网络号按主机字节顺序返回；参见字节顺序 。

使用getnetbyname要么getnetbyaddr函数可在网络数据库中搜索有关特定网络的信息。信息以静态分配的结构返回；如果需要保存，则必须复制该信息。

功能： struct netent * getnetbyname （const char * name ）

初步： MT不安全种族：netbyname env语言环境| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

的getnetbyname函数返回有关命名网络的信息name 。如果没有这样的网络，它将返回一个空指针。

功能： struct netent * getnetbyaddr （uint32_t net ，int type ）

初步： MT不安全种族：netbyaddr语言环境| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

的getnetbyaddr函数返回有关类型的网络的信息type带数字net 。您应该指定一个值AF_INET为了type Internet网络的论点。

getnetbyaddr如果没有这样的网络，则返回空指针。

您也可以使用setnetent ， getnetent和endnetent 。使用这些功能时请小心，因为它们不可重入。

函数： void setnetent （int stayopen ）

初步： MT不安全种族：有效的环境语言| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

此功能将打开并倒回网络数据库。

如果stayopen参数为非零，这会设置一个标志，以便后续对getnetbyname要么getnetbyaddr不会关闭数据库（通常会关闭）。如果您多次调用这些函数，可以避免每次调用重新打开数据库，从而提高效率。

函数： struct netent * getnetent （无效）

初步：不安全MT种族：netent种族：netentbuf env语言环境| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

此函数返回网络数据库中的下一个条目。如果没有更多条目，则返回空指针。

功能： 无效 endnetent （无效）

初步： MT不安全种族：有效的环境语言| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

此功能关闭网络数据库。
