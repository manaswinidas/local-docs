16.6.4服务数据库

跟踪“知名”服务的数据库通常是以下文件之一/etc/services或名称服务器中的等效项。您可以使用以下声明的实用程序netdb.h ，以访问服务数据库。

数据类型： structservnt

此数据类型保存有关服务数据库中条目的信息。它具有以下成员：

char *s_name

这是服务的“正式”名称。

char **s_aliases

这些是服务的备用名称，表示为字符串数组。空指针终止数组。

int s_port

这是服务的端口号。端口号以网络字节顺序给出；参见字节顺序 。

char *s_proto

这是与此服务一起使用的协议的名称。请参阅协议数据库 。

要获取有关特定服务的信息，请使用getservbyname要么getservbyport功能。信息以静态分配的结构返回；如果需要跨调用保存信息，则必须复制该信息。

函数： struct servent * getservbyname （const char * name ，const char * proto ）

初步： MT不安全种族：servbyname语言环境| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

的getservbyname函数返回有关命名服务的信息name使用协议proto 。如果找不到这样的服务，它将返回一个空指针。

此功能对服务器和客户端均有用。服务器使用它来确定应侦听的端口（请参阅侦听 ）。

函数： struct servent * getservbyport （int port ，const char * proto ）

初步： MT不安全种族：servbyport语言环境| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

的getservbyport函数返回有关端口服务的信息port使用协议proto 。如果找不到这样的服务，它将返回一个空指针。

您也可以使用以下方法扫描服务数据库setservent ， getservent和endservent 。使用这些功能时请小心，因为它们不可重入。

函数： void setservent （int stayopen ）

初步： MT不安全种族：服从语言环境| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

此功能将打开服务数据库以开始对其进行扫描。

如果stayopen参数为非零，这会设置一个标志，以便后续对getservbyname要么getservbyport不会关闭数据库（通常会关闭）。如果您多次调用这些函数，可以避免每次调用重新打开数据库，从而提高效率。

函数： struct servent * getservent （无效）

初步：不安全MT种族：仆人种族：serventbuf地区| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

此函数返回服务数据库中的下一个条目。如果没有更多条目，则返回空指针。

函数：void endservent（无效）

初步： MT不安全种族：服从语言环境| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

此功能关闭服务数据库。
