25.3.10标志argp_help功能

打电话时argp_help （请参阅Argp帮助 ）或argp_state_help （请参阅Argp帮助器函数 ），确切的输出由flags论点。这应该由以下任何标志组成，或一起使用：

ARGP_HELP_USAGE

Unix' Usage: ”消息，其中明确列出了所有选项。

ARGP_HELP_SHORT_USAGE

Unix' Usage: '消息，其中显示适当的占位符以指示选项的位置；用于显示非选项参数语法。

ARGP_HELP_SEE

一种 ' Try … for more help ' 信息; ' … '包含程序名称和' --help '。

ARGP_HELP_LONG

详细的选项帮助消息，其中提供了每个可用选项及其文档字符串。

ARGP_HELP_PRE_DOC

详细选项帮助前的argp解析器doc字符串部分。

ARGP_HELP_POST_DOC

详细选项帮助后的argp分析器doc字符串部分。

ARGP_HELP_DOC

(ARGP_HELP_PRE_DOC | ARGP_HELP_POST_DOC)

ARGP_HELP_BUG_ADDR

如果出现以下情况，则会显示一条消息，报告该程序的错误报告位置。 argp_program_bug_address变量包含此信息。

ARGP_HELP_LONG_ONLY

这将修改任何输出以反映ARGP_LONG_ONLY模式。

以下标志仅当与argp_state_help 。它们控制函数在打印输出后返回还是终止程序：

ARGP_HELP_EXIT_ERR

这将终止程序exit (argp_err_exit_status) 。

ARGP_HELP_EXIT_OK

这将终止程序exit (0) 。

以下标志是用于打印标准消息的基本标志的组合：

ARGP_HELP_STD_ERR

假设已打印出用于分析错误的错误消息，这将打印一条有关如何获取帮助的消息，并以错误终止程序。

ARGP_HELP_STD_USAGE

这将打印标准用法消息，并以错误终止程序。当没有其他合适的错误消息或没有可用的错误消息时，将使用此选项。

ARGP_HELP_STD_HELP

这会为“ --help '选项，并成功终止程序。
