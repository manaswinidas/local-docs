32.6文件系统容量限制

POSIX.1标准指定了许多描述文件系统限制的参数。系统可能对参数有固定的统一限制，但这不是通常的情况。在大多数系统上，不同的文件系统（对于某些参数，甚至是不同的文件）可能具有不同的最大限制。例如，如果使用NFS从其他计算机挂载某些文件系统，则很有可能会发生这种情况。

以下每个宏均在中定义limits.h仅当系统对相关参数有固定的统一限制时。如果系统允许不同的文件系统或文件具有不同的限制，则该宏未定义；使用pathconf要么fpathconf找出适用于特定文件的限制。参见Pathconf 。

每个参数还具有另一个宏，其名称以“ _POSIX '，它给出了在任何 POSIX系统上允许的限制的最小值。请参阅文件最小值 。

巨集： int LINK_MAX

给定文件名称的统一系统限制（如果有）。请参阅硬链接 。

巨集： int MAX_CANON

启用输入编辑后，输入行中文本量的统一系统限制（如果有）。请参阅“ 规范”或“不规范” 。

巨集： int MAX_INPUT

预先输入的字符总数的统一系统限制（如果有）。请参阅I / O队列 。

巨集： int NAME_MAX

文件名组件长度的统一系统限制（如果有），不包括终止空字符。

可移植性注：在某些系统上，GNU C库定义NAME_MAX ，但实际上并没有强制执行此限制。

巨集： int PATH_MAX

整个文件名长度的统一系统限制（如果有）（即，给系统调用指定的参数，例如open ），包括终止的空字符。

可移植性注意：即使GNU C库也不会强制执行此限制PATH_MAX被定义为。

巨集： int PIPE_BUF

可以原子地写入管道的字节数的统一系统限制（如果有）。如果多个进程同时写入同一管道，则来自不同进程的输出可能会交错插入此大小的块中。请参阅管道和FIFO 。

这些是某些相同信息的替代宏名称。

巨集： int MAXNAMLEN

这是BSD的名称NAME_MAX 。它在dirent.h 。

巨集： int FILENAME_MAX

该宏的值是一个整数常量表达式，表示文件名字符串的最大长度。它在stdio.h 。

不像PATH_MAX ，即使没有实际限制也定义了此宏。在这种情况下，其值通常为非常大的数字。在GNU / Hurd系统上总是如此。

使用注意：请勿使用FILENAME_MAX作为存储文件名的数组大小！您不可能将数组做得那么大！请改用动态分配（请参阅内存分配 ）。
