13.11.2获取AIO操作状态

如上一节中的功能文档中所述，必须有可能获取有关I / O请求状态的信息。当操作真正地异步执行时（如aio_read和aio_write与lio_listio当模式是LIO_NOWAIT ），有时需要知道特定请求是否已经终止，如果终止了，结果是什么。以下两个功能使您可以获得此类信息。

功能： int aio_error （const struct aiocb * aiocbp ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

此功能确定请求描述的错误状态struct aiocb指向的变量aiocbp 。如果请求尚未终止，则返回的值始终为EINPROGRESS 。请求终止后，该值aio_error如果请求成功完成，则返回为0 ，或者返回将存储在errno如果请求将使用完成，则为变量read ， write ， 要么fsync 。

该函数可以返回ENOSYS如果未实现。它也可能返回EINVAL如果aiocbp参数未引用返回状态未知的异步操作。

当源编译时_FILE_OFFSET_BITS == 64这个功能其实aio_error64因为LFS接口透明地替换了常规实现。

函数： int aio_error64 （const struct aiocb64 * aiocbp ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

该功能类似于aio_error唯一的区别是参数是对类型变量的引用struct aiocb64 。

当源编译时_FILE_OFFSET_BITS == 64该功能在名称下可用aio_error因此透明地替换了32位计算机上小文件的接口。

功能： ssize_t aio_return （struct aiocb * aiocbp ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

此函数可用于检索由指向的变量所描述的请求所执行的操作的返回状态。 aiocbp 。只要此请求的错误状态为aio_error是EINPROGRESS该函数的返回值是不确定的。

请求完成后，该函数就可以精确地使用一次以获取返回值。后续调用可能导致未定义的行为。传回值本身就是read ， write ， 要么fsync呼叫。

该函数可以返回ENOSYS如果未实现。它也可能返回EINVAL如果aiocbp参数未引用返回状态未知的异步操作。

当源编译时_FILE_OFFSET_BITS == 64这个功能其实aio_return64因为LFS接口透明地替换了常规实现。
