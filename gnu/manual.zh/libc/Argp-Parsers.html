25.3.3指定Argp解析器

第一个参数argp_parse函数是指向struct argp ，称为argp解析器 ：

数据类型： struct argp

该结构指定了如何解析给定的选项和参数集，可能与其他argp解析器结合使用。它具有以下字段：

const struct argp_option *options

指向向量的指针argp_option指定此argp解析器可以理解的选项的结构；如果根本没有选项，则可能为零。请参阅Argp选项向量 。

argp_parser_t parser

指向定义此解析器动作的函数的指针；对于每个解析的选项，以及在解析过程中其他明确定义的点，都将调用它。零值与始终返回的函数的指针相同ARGP_ERR_UNKNOWN 。请参阅Argp分析器函数 。

const char *args_doc

如果非零，则为描述此解析器调用哪些非选项参数的字符串。这仅用于打印“ Usage: ' 信息。如果包含换行符，则由它们分隔的字符串将被视为替代使用模式，并打印在单独的行上。第一行之后的行以' or: ' 代替 ' Usage: '。

const char *doc

如果不为零，则为一个字符串，其中包含要在一条长帮助消息中的选项之前和之后打印的多余文本，并且两个部分之间用垂直制表符（ '\v' ， '\013' ）字符。按照惯例，选项之前的文档只是说明程序功能的简短字符串。选项后打印的文档更详细地描述了行为。

const struct argp_child *children

指向向量的指针argp_child结构。该指针指定应将其他argp解析器与此对象组合。参见Argp儿童 。

char *(*help_filter)(int key, const char *text, void *input)

如果非零，则为指向过滤帮助消息输出的函数的指针。请参阅Argp帮助过滤 。

const char *argp_domain

如果非零，则使用此字符串描述的域来转换argp库中使用的字符串。如果为零，则使用当前的默认域。

在以上组中， options ， parser ， args_doc和doc通常只需要这些字段。如果将argp解析器定义为已初始化的C变量，则仅需要在初始化程序中指定使用的字段。由于C结构初始化的工作方式，其余的默认为零。大多数argp结构都采用了这种设计。最常用的字段在开头附近分组，未使用的字段则未指定。
