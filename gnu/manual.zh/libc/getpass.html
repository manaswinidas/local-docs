17.8读取密码短语

在读入密码短语时，最好避免在屏幕上显示该密码短语，以帮助保密。以下函数可以方便地处理此问题。

函数： char * getpass （const char * prompt ）

初步： MT不安全条款| AS不安全堆锁损坏| AC不安全术语锁损坏|请参阅POSIX安全概念 。

getpass输出prompt ，然后从终端读取一个字符串而不回显它。它尝试连接到真实终端， /dev/tty ，如果可能的话，鼓励用户不要在文件中放置明文密码短语；否则，它使用stdin和stderr 。 getpass还可以使用以下命令在终端上禁用INTR，QUIT和SUSP字符ISIG终端属性（请参阅本地模式 ）。终端在刷新前后getpass ，因此不会意外看到密码输入错误的字符。

在其他C库中， getpass可能只返回第一个PASS_MAX密码短语的字节。GNU C库没有限制，因此PASS_MAX未定义。

该函数的原型在unistd.h 。PASS_MAX将在limits.h 。

这种精确的操作集可能并不适合所有可能的情况。在这种情况下，建议用户自己写getpass替代。例如，一个非常简单的替代方法如下：


#include <termios.h>
#include <stdio.h>

ssize_t
my_getpass (char **lineptr, size_t *n, FILE *stream)
{
  struct termios old, new;
  int nread;

  /* Turn echoing off and fail if we can’t. */
  if (tcgetattr (fileno (stream), &old) != 0)
    return -1;
  new = old;
  new.c_lflag &= ~ECHO;
  if (tcsetattr (fileno (stream), TCSAFLUSH, &new) != 0)
    return -1;

  /* Read the passphrase */
  nread = getline (lineptr, n, stream);

  /* Restore terminal. */
  (void) tcsetattr (fileno (stream), TCSAFLUSH, &old);

  return nread;
}

替代品的参数与getline （请参阅“ 线路输入” ）；用户必须打印所需的任何提示。
