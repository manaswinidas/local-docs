30.5读取过程的角色

这里是对读取进程的用户ID和组ID的功能的详细说明，包括真实的和有效的。要使用这些功能，必须包含头文件sys/types.h和unistd.h 。

数据类型： uid_t

这是用于表示用户ID的整数数据类型。在GNU C库中，这是以下项的别名unsigned int 。

数据类型： gid_t

这是用于表示组ID的整数数据类型。在GNU C库中，这是以下项的别名unsigned int 。

函数： uid_t getuid （无效）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的getuid函数返回进程的真实用户ID。

函数： gid_t getgid （无效）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的getgid函数返回进程的真实组ID。

函数： uid_t geteuid （无效）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的geteuid函数返回该过程的有效用户ID。

函数： gid_t getegid （无效）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的getegid函数返回该进程的有效组ID。

函数： int getgroups （int count ，gid_t * groups ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的getgroups函数用于查询进程的补充组ID。取决于count这些组ID中的一个存储在数组中groups ;该函数的返回值是实际存储的组ID的数量。如果count小于补充组ID的总数，则getgroups返回值-1和errno被设定为EINVAL 。

如果count是零，那么getgroups只返回补充组ID的总数。在不支持补充组的系统上，该值始终为零。

这是使用方法getgroups读取所有补充组ID：

gid_t *
read_all_groups (void)
{
  int ngroups = getgroups (0, NULL);
  gid_t *groups
    = (gid_t *) xmalloc (ngroups * sizeof (gid_t));
  int val = getgroups (ngroups, groups);
  if (val < 0)
    {
      free (groups);
      return NULL;
    }
  return groups;
}
