12.13.2转换说明符选项

如果您定义“ %A '，如果模板包含' %+23A ' 要么 ' %-#A '？为了实现合理的含义，处理程序在被调用时必须能够获取模板中指定的选项。

这俩handler-function和arginfo-function接受一个指向struct printf_info ，其中包含有关在转换说明符实例中出现的选项的信息。该数据类型在头文件中声明printf.h 。

类型： struct printf_info

此结构用于传递有关出现在转换说明符实例中的选项的信息。 printf该说明符的处理程序和arginfo函数的模板字符串。它包含以下成员：

int prec

这是指定的精度。值是-1如果未指定精度。如果精度为' * '， printf_info传递给处理程序函数的结构包含从参数列表检索的实际值。但是传递给arginfo函数的结构包含一个值INT_MIN ，因为实际值未知。

int width

这是指定的最小字段宽度。值是0如果未指定宽度。如果字段宽度指定为“ * '， printf_info传递给处理程序函数的结构包含从参数列表检索的实际值。但是传递给arginfo函数的结构包含一个值INT_MIN ，因为实际值未知。

wchar_t spec

这是指定的转换说明符。它存储在结构中，以便您可以为多个字符注册相同的处理函数，但是仍然可以在调用处理函数时区分开它们。

unsigned int is_long_double

这是一个布尔值，如果“ L '，' ll '， 要么 ' q '指定了类型修饰符。对于整数转换，这表明long long int ，而不是long double用于浮点转换。

unsigned int is_char

这是一个布尔值，如果“ hh '指定了类型修饰符。

unsigned int is_short

这是一个布尔值，如果“ h '指定了类型修饰符。

unsigned int is_long

这是一个布尔值，如果“ l '指定了类型修饰符。

unsigned int alt

这是一个布尔值，如果“ # '标志已指定。

unsigned int space

这是一个布尔值，如果“ '标志已指定。

unsigned int left

这是一个布尔值，如果“ - '标志已指定。

unsigned int showsign

这是一个布尔值，如果“ + '标志已指定。

unsigned int group

这是一个布尔值，如果“ ' '标志已指定。

unsigned int extra

根据上下文，此标志具有特殊含义。用户定义的处理程序可以自由使用它，但是当从printf函数，此变量始终包含值0 。

unsigned int wide

如果流面向宽方向，则设置此标志。

wchar_t pad

这是用于将输出填充到最小字段宽度的字符。值是'0'如果“ 0 '标志已指定，并且' '除此以外。
