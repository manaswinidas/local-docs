14.9.4文件所有者

每个文件都有一个所有者 ，该所有者是系统上定义的注册用户名之一。每个文件还具有一个组 ，该组是已定义的组之一。文件所有者通常可用于显示编辑文件的人（特别是当您使用GNU Emacs进行编辑时），但其主要目的是用于访问控制。

文件所有者和组在确定访问权限中起着一定的作用，因为文件具有所有者的一组访问权限位，适用于属于该文件组的用户的另一组权限以及适用于其他所有用户的第三组位。有关如何根据此数据确定访问权限的详细信息，请参见访问权限 。

创建文件后，其所有者将被设置为创建该文件的过程的有效用户ID（请参阅Process Persona ）。根据存储文件的系统，可以将文件的组ID设置为进程的有效组ID或包含文件的目录的组ID。当您访问远程文件系统时，它会根据自己的规则运行，而不是根据您的程序在其上运行的系统运行。因此，无论您在哪种系统上运行程序，都必须准备好遇到两种行为。

您可以使用来更改现有文件的所有者和/或组所有者。 chown功能。这是原始的chown和chgrp外壳命令。

此函数的原型在中声明unistd.h 。

功能： int chown （const char * filename ，uid_t owner ，gid_t group ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的chown函数更改文件的所有者filename至owner ，以及其所有者group 。

在某些系统上更改文件的所有者将清除设置用户标识和设置组标识权限位。（这是因为这些位可能不适用于新所有者。）其他文件权限位未更改。

返回值为0成功与-1失败。除了常见的文件名错误（请参阅文件名错误 ）之外，以下内容errno为此功能定义了错误条件：

EPERM

此过程缺少进行请求的更改的权限。

只有特权用户或文件的所有者才能更改文件的组。在大多数文件系统上，只有特权用户才能更改文件所有者；如果您当前是所有者，则某些文件系统允许您更改所有者。当您访问远程文件系统时，遇到的行为是由实际保存文件的系统决定的，而不是由程序运行所在的系统决定的。
