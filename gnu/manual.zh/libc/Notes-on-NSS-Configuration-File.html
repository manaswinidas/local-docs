29.2.3关于NSS配置文件的说明

最后还有一些提示。NSS的实施并非完全无助，如果/etc/nsswitch.conf不存在。对于所有受支持的数据库，都有一个默认值，因此即使文件已损坏或丢失，通常也应该可以使系统运行。

为了hosts和networks数据库的默认值为dns [!UNAVAIL=return] files 。即，系统为DNS服务不可用作好了准备，但是如果可用，则返回的答案是确定的。

对于所有其他数据库，默认值为files除非GNU C库配置有--enable-obsolete-rpc选项，在这种情况下，默认值为nis [NOTFOUND=return] files 。

第二点是用户应尝试优化查找过程。不同的服务具有不同的响应时间。在本地文件上查找一个简单的文件可能很快，但是如果文件很长并且所需的条目位于文件的末尾附近，则可能要花费一些时间。在这种情况下，最好使用db允许快速本地访问大型数据集的服务。

通常情况是必须使用某些全局信息，例如NIS。因此，不可避免的是使用诸如nis等等。但是如果可能的话，应该避免这种慢速服务。
