25.3.2 Argp全局变量

这些变量使用户程序可以轻松实现“ --version '选项，并在'中提供一个错误报告地址--help的输出。这些默认情况下在argp中实现。

变量： const char * argp_program_version

如果由用户程序定义或设置为非零值，则' --version使用时会添加'选项argp_parse ，它将打印“ --version '字符串，后跟换行符并退出。例外情况是，如果ARGP_NO_EXIT标志被使用。

变量： const char * argp_program_bug_address

如果由用户程序定义或设置为非零值， argp_program_bug_address应该指向一个字符串，该字符串将在“ --help '选项，并在句子中插入' Report bugs to address. '。

变量： argp_program_version_hook

如果由用户程序定义或设置为非零值，则' --version使用时会添加'选项arg_parse ，它会打印程序版本并以零状态退出。如果ARGP_NO_HELP标志被使用。如果ARGP_NO_EXIT设置了flag时，将抑制或修改程序的退出行为，就像argp解析器将由其他程序使用时一样。

它应该指向具有这种签名类型的函数：

void print-version (FILE *stream, struct argp_state *state)

有关的解释，请参见Argp解析状态 。 state 。

此变量优先于argp_program_version ，如果程序的版本信息不容易用简单的字符串表示，则很有用。

变量： error_t argp_err_exit_status

这是由于解析错误而导致argp退出时使用的退出状态。如果未由用户程序定义或设置，则默认为： EX_USAGE从 。
