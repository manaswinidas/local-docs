24.4.3处理程序中的非本地控制传递

您可以使用以下命令从信号处理程序中进行非本地控制转移setjmp和longjmp设施（请参阅非本地出口 ）。

当处理程序执行非本地控制传输时，正在运行的程序部分将不会继续。如果程序的这一部分正在更新重要的数据结构，则该数据结构将保持不一致。由于程序不会终止，因此稍后可能会发现不一致。

有两种方法可以避免此问题。一种是阻止程序中更新重要数据结构的部分的信号。一旦关键更新完成，阻塞信号将延迟其传递，直到将其解除阻塞为止。请参阅阻止信号 。

另一种方法是在信号处理程序中重新初始化关键数据结构，或使其值保持一致。

这是一个相当示意性的示例，显示了一个全局变量的重新初始化。 

#include <signal.h>
#include <setjmp.h>

jmp_buf return_to_top_level;

volatile sig_atomic_t waiting_for_input;

void
handle_sigint (int signum)
{
  /* We may have been waiting for input when the signal arrived,
     but we are no longer waiting once we transfer control. */
  waiting_for_input = 0;
  longjmp (return_to_top_level, 1);
}



int
main (void)
{
  …
  signal (SIGINT, sigint_handler);
  …
  while (1) {
    prepare_for_command ();
    if (setjmp (return_to_top_level) == 0)
      read_and_execute_command ();
  }
}

char *
read_data ()
{
  if (input_from_terminal) {
    waiting_for_input = 1;
    …
    waiting_for_input = 0;
  } else {
    …
  }
