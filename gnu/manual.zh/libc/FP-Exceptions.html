20.5.1 FP异常

IEEE 754 标准定义了在计算过程中可能发生的五个异常 。每个都对应于一种特定类型的错误，例如溢出。

当发生异常时（在标准语言中引发异常时），可能会发生以下两种情况之一。默认情况下，该异常仅在浮点状态字中指出，并且程序继续运行，就好像什么都没发生一样。该操作将产生一个默认值，该值取决于异常（请参见下表）。您的程序可以检查状态字以找出发生了哪些异常。

另外，您可以启用陷阱的例外。在这种情况下，当引发异常时，您的程序将收到SIGFPE信号。该信号的默认操作是终止程序。有关如何更改信号效果的信息，请参见信号处理 。

IEEE 754 中定义的例外是：

' Invalid Operation '

如果给定的操作数对于要执行的操作无效，则会引发此异常。示例如下（请参阅IEEE 754 ，第7节 ）：

  1. 加法或减法： ∞-∞ 。（但是∞+∞=∞ ）。
  2. 乘法： 0·∞ 。
  3. 除法： 0/0或∞/∞ 。
  4. 余数： x REM y ，其中y为零或x为无限。
  5. 如果操作数小于零，则平方根。更一般而言，在其范围之外评估的任何数学函数都会产生此异常。
  6. 当数字不能以目标格式表示时（由于溢出，无穷大或NaN），将浮点数转换为整数或十进制字符串。
  7. 无法识别的输入字符串的转换。
  8. 当一个或另一个操作数为NaN时，通过涉及<或>的谓词进行比较。您可以通过使用无序比较功能来防止此异常。请参阅FP比较功能 。

如果没有捕获异常，则操作的结果为NaN。

' Division by Zero '

当有限的非零数除以零时会引发此异常。如果没有陷阱发生，则结果为+∞或-∞ ，具体取决于操作数的符号。

' Overflow '

只要结果不能以目标的精度格式表示为有限值，就会引发此异常。如果没有陷阱发生，则结果取决于中间结果的符号和当前的舍入模式（IEEE 754 ，第7.3节 ）：

  1. 四舍五入到最接近会以中间结果的符号将所有溢出都带到∞ 。
  2. 向0取整将所有溢出均带到最大可表示的有限数，中间结果为正负号。
  3. 朝-∞回合进行积极的溢出到最大可表示有限数量和负溢出到-∞。
  4. 朝向∞的舍入将负溢出带到最负的可表示有限数，将正溢出带到∞ 。

每当引发溢出异常时，也会引发不精确的异常。

' Underflow '

当中间结果太小而无法精确计算时，或者如果运算结果舍入到目标精度太小而无法归一化时，会引发下溢异常。

如果未安装任何用于下溢异常的陷阱，则仅在同时检测到准确性和准确性下降的情况下（通过下溢标志）才发出下溢信号。如果未安装陷阱处理程序，则操作将以不精确的小值继续进行；如果目标精度无法容纳较小的精确结果，则操作将继续以零进行操作。

' Inexact '

如果四舍五入的结果不准确（例如，当计算二的平方根时），或者结果溢出而没有溢出陷阱，则会发出此异常信号。
