3.2.6.5 Obstack函数和宏

根据编译器的不同，可以将使用obstack的接口定义为函数或宏。obstack工具适用于所有C编译器，包括ISO C 和传统C，但是如果您打算使用GNU C以外的编译器，则必须采取一些预防措施。

如果您使用的是老式的非ISO C 编译器，则实际上所有的堆栈“函数”都仅定义为宏。您可以像函数一样调用这些宏，但是不能以其他任何方式使用它们（例如，您不能使用它们的地址）。

调用宏需要采取特殊的预防措施：即，第一个操作数（obstack指针）可能不包含任何副作用，因为它可能会被多次计算。例如，如果您编写此代码：

obstack_alloc (get_obstack (), 4);

你会发现get_obstack可能被多次调用。如果您使用*obstack_list_ptr++作为obstack指针参数，由于增量可能会发生多次，因此您会得到非常奇怪的结果。

在ISO C ，每个函数都具有宏定义和函数定义。如果在不调用函数的情况下获取函数的地址，则使用函数定义。普通调用默认情况下使用宏定义，但是您可以请求函数定义，而不是通过在括号中写入函数名称，如下所示：

char *x;
void *(*funcp) ();
/* Use the macro.  */
x = (char *) obstack_alloc (obptr, size);
/* Call the function.  */
x = (char *) (obstack_alloc) (obptr, size);
/* Take the address of the function.  */
funcp = obstack_alloc;

对于标准库函数，这与ISO C 中存在的情况相同。请参阅宏定义 。

警告：在使用宏时，即使在ISO C ，也必须注意避免在第一个操作数中产生副作用的注意事项。

如果使用GNU C编译器，则无需采取这种预防措施，因为GNU C中的各种语言扩展都允许定义宏，以便每个参数仅计算一次。
