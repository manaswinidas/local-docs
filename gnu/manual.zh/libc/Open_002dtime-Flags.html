13.15.2开放时间标志

开放时间标志指定影响选项的选项open会表现。打开文件后，将不会保留这些选项。例外是O_NONBLOCK ，这也是一种I / O操作模式，因此已保存。有关如何调用的信息，请参见打开和关闭文件 。 open 。

开放时间标志指定两种类型的选项。

  • 文件名转换标志会影响open查找文件名以找到文件，以及是否可以创建文件。
  • 开放时间操作标志指定了额外的操作，这些操作open打开文件后，它将在文件上执行。

这是文件名转换标志。

巨集： int O_CREAT

如果设置，则将创建该文件（如果尚不存在）。

巨集： int O_EXCL

如果两者O_CREAT和O_EXCL设置好了open如果指定的文件已经存在，则失败。这样可以确保永远不会破坏现有文件。

的O_EXCL结合使用具有特殊含义O_TMPFILE ;见下文。

巨集： int O_DIRECTORY

如果设置，则如果给定名称不是目录名称，则打开操作将失败。的errno变量设置为ENOTDIR对于此错误情况。

巨集： int O_TMPFILE

如果指定了此标志，则open家族创建一个未命名的临时文件。在这种情况下， open系列功能（请参见打开和关闭文件 ）被解释为在其中创建临时文件的目录（从而确定为文件提供存储的文件系统）。的O_TMPFILE标志必须与O_WRONLY要么O_RDWR和mode参数是必需的。

以后可以使用以下名称为临时文件命名linkat ，将其转换为常规文件。这允许原子创建具有特定文件属性（模式和扩展属性）和文件内容的文件。如果出于安全原因不希望为文件指定名称，则O_EXCL标志可以与O_TMPFILE 。

并非所有内核都支持此打开标志。如果不支持此标志，则尝试创建未命名的临时文件失败，错误为EINVAL 。如果基础文件系统不支持O_TMPFILE旗帜EOPNOTSUPP错误是结果。

的O_TMPFILE flag是GNU扩展。

巨集： int O_NONBLOCK

这样可以防止open阻止“长时间”打开文件。这仅对某些类型的文件有意义，通常是诸如串行端口之类的设备。当它没有意义时，它是无害的并且被忽略。通常，打开调制解调器的端口会阻塞，直到调制解调器报告载波检测为止。如果O_NONBLOCK被指定open将立即返回，没有携带者。

请注意O_NONBLOCK作为I / O操作模式和文件名转换标志，此标志均过载。这意味着指定O_NONBLOCK在open还设置非阻塞I / O模式；请参阅操作模式 。要打开文件而不阻塞但执行阻塞的常规I / O，必须调用open与O_NONBLOCK设置然后调用fcntl关闭位。

巨集： int O_NOCTTY

如果命名文件是终端设备，请不要将其作为该过程的控制终端。请参阅“ 作业控制” ，以获取有关成为控制终端的含义的信息。

在GNU / Hurd系统和4.4 BSD上，打开文件绝不会使其成为控制终端， O_NOCTTY是零。但是，GNU / Linux系统和某些其他系统将非零值用于O_NOCTTY并在打开作为终端设备的文件时设置控制终端；为了便于携带，使用O_NOCTTY避免这种情况很重要。

以下三个文件名转换标志仅在GNU / Hurd系统上存在。

巨集： int O_IGNORE_CTTY

即使命名文件引用了该进程的现有控制终端设备，也不要将其识别为控制终端。对新文件描述符的操作将永远不会引起作业控制信号。请参阅作业控制 。

巨集： int O_NOLINK

如果命名文件是符号链接，请打开链接本身而不是它所引用的文件。 （ fstat在新文件描述符上将返回由以下信息返回的信息lstat在链接的名称上。）

巨集： int O_NOTRANS

如果命名文件是经过特殊翻译的，则不要调用翻译器。打开转换器本身看到的裸文件。

开放时间操作标志告诉open进行与打开文件无关的其他操作。将它们作为一部分的原因open而不是单独的电话是open能原子地做他们。

巨集： int O_TRUNC

将文件截断为零长度。此选项仅对常规文件有用，而对目录或FIFO等特殊文件不起作用。POSIX.1要求您打开文件进行写入才能使用O_TRUNC 。在BSD和GNU中，您必须具有写入文件的权限才能截断它，但无需打开以进行写入访问。

这是POSIX.1指定的唯一打开时间操作标志。没有充分的理由进行截断open ，而不是通过致电ftruncate然后。的O_TRUNC标记在Unix之前已存在ftruncate是发明的，为了向后兼容而保留。

其余的操作模式是BSD扩展。它们仅存在于某些系统上。在其他系统上，未定义这些宏。

巨集： int O_SHLOCK

获取文件的共享锁，就像flock 。请参阅文件锁 。

如果O_CREAT如果指定，则在创建文件时自动完成锁定。您可以确保没有其他进程会首先获得对新文件的锁定。

巨集： int O_EXLOCK

获取文件的排他锁，就像flock 。请参阅文件锁 。这是原子的O_SHLOCK 。
