A.5.2整数类型的范围

假设您需要存储一个整数值，范围从零到一百万。您可以使用哪种最小的类型？没有一般规则；它取决于C编译器和目标计算机。您可以使用“ MIN '和' MAX '中的宏limits.h确定哪种类型有效。

每个带符号的整数类型都有一对宏，这些宏给出了它可以容纳的最小和最大值。每个无符号整数类型都有一个这样的宏，以获取最大值。最小值当然是零。

这些宏的值都是整数常量表达式。' MAX '和' MIN的宏char和short int 类型的值具有类型int 。' MAX '和' MIN其他类型的'宏的值与该宏描述的类型相同，因此， ULONG_MAX有类型unsigned long int 。

SCHAR_MIN

这是可以用signed char 。

SCHAR_MAX
UCHAR_MAX

这些是可以用signed char 和unsigned char分别为 。

CHAR_MIN

这是可以用char 。等于SCHAR_MIN如果char已签名，否则为零。

CHAR_MAX

这是可以由a表示的最大值char 。等于SCHAR_MAX如果char已签名，或UCHAR_MAX除此以外。

SHRT_MIN

这是可以用signed short int 。在运行GNU C库的大多数计算机上， short整数是16位的数量。

SHRT_MAX
USHRT_MAX

这些是可以用signed short int 和unsigned short int分别为 。

INT_MIN

这是可以用signed int 。在运行GNU C库的大多数计算机上， int是32位数量。

INT_MAX
UINT_MAX

这些是分别可以由类型表示的最大值signed int 和类型unsigned int 。

这是可以用signed long long int 。在运行GNU C库的大多数计算机上， long long 整数是64位数量。
