21.4.6.2更加方便的解析时间和日期的方法

Unix标准定义了另一个用于解析日期字符串的函数。界面很奇怪，但是如果功能恰好适合您的应用程序就可以了。在多线程程序或库中使用此函数是有问题的，因为它返回指向静态变量的指针，并使用全局变量和全局状态（环境变量）。

变量： getdate_err

此变量的类型int包含上次未成功调用的错误代码getdate 。定义的值为：

1个

环境变量DATEMSK未定义或为null。

2

模板文件由DATEMSK环境变量无法打开。

3

无法获取有关模板文件的信息。

4

模板文件不是常规文件。

5

读取模板文件时发生I / O错误。

6

没有足够的内存来执行该功能。

7

模板文件不包含匹配的模板。

8

输入的日期无效，但否则将与模板匹配。其中包括2月31日之类的日期，以及不能在time_t变量。

功能： struct tm * getdate （const char * string ）

初步： MT不安全种族：getdate env语言环境| AS不安全堆锁| AC不安全锁内存fd |请参阅POSIX安全概念 。

界面getdate是函数解析字符串并返回值的最简单方法。 string是输入字符串，结果将返回到静态分配的变量中。

用户看不到有关如何处理字符串的详细信息。实际上，它们可以不受程序控制。识别哪些格式由环境变量命名的文件控制DATEMSK 。此文件应包含可传递给的有效格式字符串行strptime 。

的getdate函数会依次读取这些格式字符串，并尝试匹配输入字符串。使用与输入字符串完全匹配的第一行。

未通过格式字符串初始化的元素将保留在getdate函数调用。

认可的格式getdate与相同strptime 。请参阅上面的说明。只有几个扩展strptime行为：

  • 如果%Z格式给出的细分时间是基于匹配的时区的当前时间，而不是运行时环境的当前时区。

    注意 ：（当前）尚未实现。问题在于时区名称不是唯一的。如果假设给定的字符串采用固定时区（例如EST表示美国东海岸时间），则将其用于美国以外的其他国家/地区会失败。到目前为止，我们还没有找到好的解决方案。

  • 如果仅指定工作日，则所选日期取决于当前日期。如果当前工作日大于或等于tm_wday值将选择当前星期几，否则将选择下星期几。
  • 当只给出月份而不是年份时，使用类似的启发式方法。如果月份大于或等于当前月份，则使用当前年份。否则，它将延续到明年。如果未明确指定一个月，则假定该月的第一天。
  • 如果未通过格式设置适当的值，则使用当前的小时，分钟和秒。
  • 如果没有给出日期，则如果时间小于当前时间，则使用明天的日期。否则，采用今天的日期。

应当注意，模板文件中的格式不仅需要包含格式元素。以下是可能的格式字符串的列表（取自Unix标准）：

警告： getdate SUID程序中绝对不能使用该功能。原因很明显：使用DATEMSK您可以使用环境变量来打开任何文件，并且使用某些假输入（例如二进制文件），程序崩溃的可能性很高。

函数： int getdate_r （const char * string ，struct tm * tp ）

初步： MT安全环境语言环境| AS不安全堆锁| AC不安全锁内存fd |请参阅POSIX安全概念 。

的getdate_r函数是的可重入对应项getdate 。它不使用全局变量getdate_err表示错误，但返回错误代码。与错误代码中所述相同的错误代码getdate_err使用上面的文档，0表示成功。

此外， getdate_r将分解时间存储在type变量中struct tm由第二个参数指向，而不是在静态变量中指向。

Unix标准中未定义此功能。但是，它也可以在其他一些Unix系统上使用。
