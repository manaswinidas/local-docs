13.11.4取消AIO操作

当一个或多个请求被异步处理时，在某些情况下取消选定的操作可能很有用，例如，如果很明显写入的数据不再准确，则必须尽快重写。例如，假设有一个应用程序，该应用程序在必须将新的传入数据写入将由排队请求更新的文件中的情况下，将数据写入文件中。 POSIX AIO实现提供了这样的功能，但是该功能不能强制取消请求。由实施来决定是否可以取消操作。因此，使用此功能仅是提示。

函数： int aio_cancel （int fildes ，struct aiocb * aiocbp ）

初步： MT安全| AS不安全锁堆| AC不安全锁记忆|请参阅POSIX安全概念 。

的aio_cancel功能可用于取消一个或多个未完成的请求。如果aiocbp参数是NULL ，该函数会尝试取消所有将处理文件描述符的未完成请求fildes （即， aio_fildes成员是fildes ）。如果aiocbp不是NULL ， aio_cancel尝试取消由指向的特定请求aiocbp 。

对于成功取消的请求，应该发出有关请求终止的常规通知。即，取决于struct sigevent控制该对象的对象，什么也不会发生，不会发送信号或启动线程。如果无法取消该请求，则在执行该操作后将以通常的方式终止。

成功取消请求后，将调用aio_error对此请求的引用，因为参数将返回ECANCELED并打给aio_return将返回-1 。如果请求未取消且仍在运行，则错误状态仍为EINPROGRESS 。

该函数的返回值为AIO_CANCELED如果存在尚未终止且已成功取消的请求。如果还有一个或多个无法取消的请求，则返回值为AIO_NOTCANCELED 。在这种情况下aio_error必须用于找出哪些请求（可能是多个请求）（如果aiocbp是NULL ）未成功取消。如果所有请求当时都已终止aio_cancel被称为返回值是AIO_ALLDONE 。

如果执行过程中发生错误aio_cancel该函数返回-1并设置errno为以下值之一。

EBADF

文件描述符fildes无效。

ENOSYS

aio_cancel未实现。

当源编译时_FILE_OFFSET_BITS == 64 ，实际上这个功能aio_cancel64因为LFS接口透明地替换了常规实现。

函数： int aio_cancel64 （int fildes ，struct aiocb64 * aiocbp ）

初步： MT安全| AS不安全锁堆| AC不安全锁记忆|请参阅POSIX安全概念 。

该功能类似于aio_cancel唯一的区别是参数是对类型变量的引用struct aiocb64 。

当源编译时_FILE_OFFSET_BITS == 64 ，此功能在名称下可用aio_cancel因此透明地替换了32位计算机上小文件的接口。
