13.14文件描述符标志

文件描述符标志是文件描述符的其他属性。这些标志与特定的文件描述符相关联，因此，如果您从文件的单个打开创建了重复的文件描述符，则每个描述符都有其自己的标志集。

当前只有一个文件描述符标志： FD_CLOEXEC ，如果您使用任何一个exec…功能（请参阅执行文件 ）。

本节中的符号在头文件中定义fcntl.h 。

巨集： int F_GETFD

此宏用作command争论fcntl ，以指定它应返回与关联的文件描述符标志filedes论点。

的正常返回值fcntl使用此命令是一个非负数，可以解释为各个标志的按位“或”（除了当前仅使用一个标志）。

如有错误， fcntl返回-1 。下列errno为该命令定义了错误条件：

EBADF

的filedes参数无效。

巨集： int F_SETFD

此宏用作command争论fcntl ，以指定它应设置与filedes论点。这需要三分之一int用于指定新标志的参数，因此调用的形式为：

fcntl (filedes, F_SETFD, new-flags)

的正常返回值fcntl该命令的值是-1以外的未指定值，表示错误。标志和错误条件与F_GETFD命令。

以下宏定义为与文件描述符标志一起使用fcntl功能。该值是可用作位掩码值的整数常数。

巨集： int FD_CLOEXEC

该标志指定当exec函数被调用；请参阅执行文件 。分配文件描述符时（如open要么dup ），此位最初会在新文件描述符上清除，这意味着描述符将在新文件中保留到新程序中exec 。

如果要修改文件描述符标志，应使用以下命令获取当前标志F_GETFD并修改值。不要以为这里列出的标志是唯一实现的标志。您的程序可能会从现在开始运行数年，然后可能会存在更多标志。例如，这是设置或清除标志的功能FD_CLOEXEC而不更改任何其他标志：

/* Set the FD_CLOEXEC flag of desc if value is nonzero,
   or clear the flag if value is 0.
   Return 0 on success, or -1 on error with errno set. */

int
set_cloexec_flag (int desc, int value)
{
  int oldflags = fcntl (desc, F_GETFD, 0);
  /* If reading the flags failed, return error indication now. */
  if (oldflags < 0)
    return oldflags;
  /* Set just the flag we want to set. */
  if (value != 0)
    oldflags |= FD_CLOEXEC;
  else
    oldflags &= ~FD_CLOEXEC;
  /* Store modified flag word in the descriptor. */
  return fcntl (desc, F_SETFD, oldflags);
}
