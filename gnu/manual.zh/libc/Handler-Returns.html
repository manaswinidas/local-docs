24.4.1返回的信号处理程序

通常返回的处理程序通常用于信号，例如SIGALRM以及I / O和进程间通信信号。但是一个处理程序SIGINT在设置一个指示程序在方便时间退出的标志后，它也可能会正常返回。

从程序处理程序正常返回程序错误信号是不安全的，因为在程序错误后未定义处理程序函数返回时程序的行为。请参阅程序错误信号 。

通常返回的处理程序必须修改某些全局变量才能生效。通常，该变量是程序在正常运行期间定期检查的变量。其数据类型应为sig_atomic_t由于原子数据访问中描述的原因。

这是此类程序的一个简单示例。它执行循环的主体，直到发现一个SIGALRM信号到了。该技术很有用，因为它允许在循环退出之前，当信号到达完成时进行迭代。 


#include <signal.h>
#include <stdio.h>
#include <stdlib.h>

/* This flag controls termination of the main loop. */
volatile sig_atomic_t keep_going = 1;

/* The signal handler just clears the flag and re-enables itself. */
void
catch_alarm (int sig)
{
  keep_going = 0;
  signal (sig, catch_alarm);
}

void
do_stuff (void)
{
  puts ("Doing stuff while waiting for alarm....");
}

int
main (void)
{
  /* Establish a handler for SIGALRM signals. */
  signal (SIGALRM, catch_alarm);

  /* Set an alarm to go off in a little while. */
  alarm (2);

  /* Check the flag once in a while to see when to quit. */
  while (keep_going)
    do_stuff ();

  return EXIT_SUCCESS;
}
