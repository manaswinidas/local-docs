16.9.5.1发送数据

函数： ssize_t send （int socket ，const void * buffer ，size_t size ，int flags ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的send功能就像write ，但带有其他标志flags 。的可能值flags在套接字数据选项中描述。

此函数返回传输的字节数，或者-1失败。如果套接字是非阻塞的，则send （喜欢write ）可以在仅发送部分数据后返回。有关非阻塞模式的信息，请参见文件状态标志 。

但是请注意，成功的返回值仅表示已正确发送了消息，并不一定表示已正确接收了消息。

下列errno为此功能定义了错误条件：

EBADF

的socket参数不是有效的文件描述符。

EINTR

发送任何数据之前，该操作已被信号中断。请参阅中断的基元 。

ENOTSOCK

描述符socket不是套接字。

EMSGSIZE

套接字类型要求以原子方式发送消息，但是消息太大而无法发送。

EWOULDBLOCK

已在套接字上设置了非阻塞模式，并且写操作将阻塞。（一般send直到操作完成为止。）

ENOBUFS

内部缓冲区空间不足。

ENOTCONN

您从未连接过此插座。

EPIPE

该套接字已连接，但连接现在断开。在这种情况下， send产生一个SIGPIPE先发信号如果该信号被忽略或阻止，或者其处理程序返回，则send失败于EPIPE 。

此功能在多线程程序中定义为取消点，因此必须为此做准备，并确保即使取消了线程，也释放了分配的资源（如内存，文件描述符，信号量等）。
