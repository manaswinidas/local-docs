13.19中断驱动输入

如果您设定O_ASYNC文件描述符上的状态标志（请参阅文件状态标志 ）， SIGIO只要可以在该文件描述符上进行输入或输出，就发送信号。可以使用以下命令选择接收信号的过程或过程组： F_SETOWN命令给fcntl功能。如果文件描述符是套接字，则还会选择接收者SIGURG带外数据到达该套接字时传递的信号；请参阅带外数据 。 （ SIGURG在任何情况下被发送select会报告套接字具有“例外情况”。请参阅等待I / O。 ）

如果文件描述符对应于终端设备，则SIGIO信号被发送到终端的前台处理组。请参阅作业控制 。

本节中的符号在头文件中定义fcntl.h 。

巨集： int F_GETOWN

此宏用作command争论fcntl ，以指定它应获取有关其所属的流程或流程组的信息SIGIO信号已发送。（对于终端，这实际上是前台进程组ID，您可以使用该ID tcgetpgrp ;请参阅终端访问功能 。）

返回值被解释为一个进程ID。如果为负，则其绝对值为进程组ID。

下列errno为该命令定义了错误条件：

EBADF

的filedes参数无效。

巨集： int F_SETOWN

此宏用作command争论fcntl ，以指定应将流程或流程组设置为SIGIO信号已发送。此命令需要类型的第三个参数pid_t传递给fcntl ，因此调用的形式为：

fcntl (filedes, F_SETOWN, pid)

的pid参数应为进程ID。您还可以传递一个绝对值是进程组ID的负数。

的返回值fcntl如果发生错误，则此命令的值为-1 ；如果成功，则使用其他值。下列errno为该命令定义了错误条件：

EBADF

的filedes参数无效。

ESRCH

没有与之对应的过程或过程组pid 。
