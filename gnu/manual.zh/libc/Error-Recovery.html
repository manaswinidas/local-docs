12.16从错误中恢复

您可以使用以下命令明确清除错误和EOF标志： clearerr功能。

功能： void clearerr （文件* stream ）

初步： MT安全| AS安全AC不安全锁|请参阅POSIX安全概念 。

此功能清除流的文件结束和错误指示符stream 。

文件定位功能（请参阅文件定位 ）还清除了流的文件结束指示符。

函数： void clearerr_unlocked （FILE * stream ）

初步： MT-Safe竞赛：流| AS安全交流安全请参阅POSIX安全概念 。

的clearerr_unlocked功能相当于clearerr函数，但它不会隐式锁定流。

此函数是GNU扩展。

请注意，仅清除错误标志并重试失败的流操作是不正确的。写入失败后，自上次缓冲区刷新以来，任意数量的字符可能已提交给文件，而某些缓冲的数据可能已被丢弃。因此，仅重试会导致数据丢失或重复。

读取失败可能会使文件指针处于不合适的位置，以进行第二次尝试。在这两种情况下，您都应在重试之前寻求已知位置。

大多数可能发生的错误是无法恢复的-第二次尝试将始终以相同的方式再次失败。因此，通常最好放弃错误并将错误报告给用户，而不是安装复杂的恢复逻辑。

一个重要的例外是EINTR （请参阅中断基元 ）。许多流I / O实现会将其视为普通错误，这可能非常不便。您可以通过安装所有信号来避免这种麻烦SA_RESTART旗。

出于类似的原因，通常不建议在流的文件描述符上设置非阻塞I / O。
