12.21.1字符串流

的fmemopen和open_memstream函数允许您对字符串或内存缓冲区进行I / O。这些设施在stdio.h 。

函数： FILE * fmemopen （无效* buf ，size_t size ，const char * opentype ）

初步： MT安全| AS不安全堆锁| AC不安全的内存锁|请参阅POSIX安全概念 。

此函数打开一个流，该流允许由opentype参数，从参数指定的缓冲区读取或写入buf 。该数组必须至少为size个字节长。

如果您将空指针指定为buf论据fmemopen动态分配一个数组size长字节（与malloc ;请参阅无限制分配 ）。仅当您要先将内容写入缓冲区然后再次读回它们时，这才真正有用，因为您实际上无法获得指向缓冲区的指针（为此，请尝试open_memstream ，如下）。当关闭流时，释放缓冲区。

争论opentype与中的相同fopen （请参阅打开流 ）。如果opentype指定附加模式，然后将初始文件位置设置为缓冲区中的第一个空字符。否则，初始文件位置在缓冲区的开头。

刷新或关闭打开的要写入的流时，如果合适，则在缓冲区的末尾写入一个空字符（零字节）。您应该在其中添加一个额外的字节size论证这一点。尝试写超过size缓冲区的字节导致错误。

对于打开供读取的流，缓冲区中的空字符（零字节）不算作“文件结尾”。读操作仅在文件位置超前时才指示文件结束size个字节。因此，如果您想从以null终止的字符串中读取字符，则应提供字符串的长度作为size论点。

这是使用的示例fmemopen创建一个流以读取字符串：


#include <stdio.h>

static char buffer[] = "foobar";

int
main (void)
{
  int ch;
  FILE *stream;

  stream = fmemopen (buffer, strlen (buffer), "r");
  while ((ch = fgetc (stream)) != EOF)
    printf ("Got %c\n", ch);
  fclose (stream);

  return 0;
}

该程序产生以下输出：

Got f
Got o
Got o
Got b
Got a
Got r
功能： FILE * open_memstream （char ** ptr ，size_t * sizeloc ）

初步： MT安全| AS不安全堆| AC不安全的内存|请参阅POSIX安全概念 。

此函数打开用于写入缓冲区的流。缓冲区是动态分配的，并根据需要使用malloc 。关闭流后，您有责任使用以下缓冲区清理此缓冲区： free要么realloc 。请参阅无限制分配 。

当流关闭时fclose或冲洗fflush ，地点ptr和sizeloc更新为包含指向缓冲区及其大小的指针。这样存储的值仅在流上不再发生输出时才有效。如果执行更多输出，则必须再次刷新流以存储新值，然后再使用它们。

空字符写入缓冲区的末尾。此空字符不包含在存储在以下位置的大小值中sizeloc 。

您可以使用以下方法移动流的文件位置fseek要么fseeko （请参阅文件位置 ）。将文件位置移至已写入数据的末尾，将中间的空间填充为零。

这是使用的示例open_memstream ：


#include <stdio.h>

int
main (void)
{
  char *bp;
  size_t size;
  FILE *stream;

  stream = open_memstream (&bp, &size);
  fprintf (stream, "hello");
  fflush (stream);
  printf ("buf = `%s', size = %zu\n", bp, size);
  fprintf (stream, ", world");
  fclose (stream);
  printf ("buf = `%s', size = %zu\n", bp, size);

  return 0;
}

该程序产生以下输出：

buf = `hello', size = 5
buf = `hello, world', size = 12
