30.13.2查找一位用户

您可以使用以下方法在系统用户数据库中搜索有关特定用户的信息： getpwuid要么getpwnam 。这些函数在中声明pwd.h 。

功能： struct passwd * getpwuid （uid_t uid ）

初步： MT不安全种族：多语言环境| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

该函数返回一个指向静态分配结构的指针，该结构包含有关用户ID为的用户的信息uid 。后续的调用可能会覆盖此结构getpwuid 。

空指针值指示数据库中没有具有用户ID的用户uid 。

函数： int getpwuid_r （uid_t uid ，struct passwd * result_buf ，字符* buffer ，size_t buflen ，struct passwd ** result ）

初步： MT安全区域| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

该功能类似于getpwuid它返回有关用户ID为的用户的信息uid 。但是，它填充了用户指向的结构result_buf信息而不是使用静态缓冲区。首先buflen指向的附加缓冲区的字节buffer用于包含其他信息，通常是结果结构的元素所指向的字符串。

如果用户为ID uid找到后，指针返回result指向包含所需数据的记录（即， result包含值result_buf ）。如果未找到用户或发生错误，则指针返回result是一个空指针。该函数返回零或错误代码。如果缓冲buffer太小，无法包含所有必需的信息，错误代码ERANGE返回并errno被设定为ERANGE 。

功能： struct passwd * getpwnam （const char * name ）

初步： MT不安全种族：pwnam语言环境| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

此函数返回一个指向静态分配结构的指针，该结构包含有关用户名为的用户的信息name 。后续的调用可能会覆盖此结构getpwnam 。

空指针返回表示没有用户名name 。

函数： int getpwnam_r （const char * name ，struct passwd * result_buf ，字符* buffer ，size_t buflen ，struct passwd ** result ）

初步： MT安全区域| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

该功能类似于getpwnam它返回有关用户名是该用户的用户的信息name 。但是，就像getpwuid_r ，它将填充用户提供的缓冲区result_buf和buffer信息而不是使用静态缓冲区。

返回值与相同getpwuid_r 。
