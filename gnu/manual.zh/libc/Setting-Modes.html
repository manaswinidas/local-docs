17.4.3正确设置终端模式

设置终端模式时，应致电tcgetattr首先获取特定终端设备的当前模式，仅修改您真正感兴趣的那些模式，然后将结果存储在tcsetattr 。

简单地初始化一个struct termios结构到一组选定的属性，然后将其直接传递给tcsetattr 。您的程序可能会从现在开始在支持本手册中未介绍的成员的系统上运行数年。避免将这些成员设置为不合理的值的方法是避免更改它们。

此外，不同的终端设备可能需要不同的模式设置才能正常运行。因此，应避免将属性从一个终端设备盲目复制到另一台终端设备。

当成员包含独立标志的集合时， c_iflag ， c_oflag和c_cflag成员这样做，即使设置整个成员也是一个坏主意，因为特定的操作系统具有自己的标志。相反，您应该从成员的当前值开始，并仅更改其值在程序中重要的标志，而使其他标志保持不变。

这是一个如何设置一个标志的示例（ ISTRIP ） 在里面struct termios结构，同时适当保留结构中的所有其他数据：

int
set_istrip (int desc, int value)
{
  struct termios settings;
  int result;



  result = tcgetattr (desc, &settings);
  if (result < 0)
    {
      perror ("error in tcgetattr");
      return 0;
    }

  settings.c_iflag &= ~ISTRIP;
  if (value)
    settings.c_iflag |= ISTRIP;

  result = tcsetattr (desc, TCSANOW, &settings);
  if (result < 0)
    {
      perror ("error in tcsetattr");
      return 0;
   }
  return 1;
}
