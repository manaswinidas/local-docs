32.10应用程序容量限制

POSIX.2标准指定了您可以通过访问的某些系统限制sysconf适用于实用程序行为，而不是库或操作系统的行为。

GNU C库为这些限制定义了宏，并且sysconf如果您要求返回值；但是这些值没有传达任何有意义的信息。它们只是POSIX.2允许的最小值。

巨集： int BC_BASE_MAX

最大的价值obase那bc实用程序得到支持。

巨集： int BC_DIM_MAX

一个数组中的元素数量最多bc实用程序得到支持。

巨集： int BC_SCALE_MAX

最大的价值scale那bc实用程序得到支持。

巨集： int BC_STRING_MAX

一个字符串常量中的最大字符数bc实用程序得到支持。

巨集： int COLL_WEIGHTS_MAX

可以在定义语言环境的整理顺序中使用的最大权重数。

巨集： int EXPR_NEST_MAX

括号内可嵌套的最大表达式数expr效用。

巨集： int LINE_MAX

面向文本的POSIX.2实用程序可以支持的最大文本行。（如果使用的是这些实用程序的GNU版本，则除了可用虚拟内存所施加的限制外，没有其他实际限制，但是库无法告诉您这一点。）

巨集： int EQUIV_CLASS_MAX

可以分配给条目的最大权数LC_COLLATE类别“ order语言环境定义中的'关键字。GNU C库目前不支持语言环境定义。
