12.13.1注册新的转换

注册新输出转换的功能是register_printf_function ，在中声明printf.h 。

函数： int register_printf_function （int spec ，printf_function handler-function ，printf_arginfo_function arginfo-function ）

初步： MT不安全const：printfext | AS不安全堆锁| AC不安全的内存锁|请参阅POSIX安全概念 。

此函数定义转换说明符spec 。因此，如果spec是'Y' ，它定义了转换“ %Y '。您可以重新定义内置转化，例如“ %s '，但标记诸如' # '并输入' l '永远不能用作转换；呼唤register_printf_function对于那些字符没有作用。建议不要使用小写字母，因为ISO C标准警告说，将来的标准版本中可能会标准化其他小写字母。

的handler-function是由调用的函数printf和此转换出现在模板字符串中的朋友。有关如何定义要作为该参数传递的函数的信息，请参见定义输出处理程序 。如果指定空指针，则任何现有的处理函数spec已移除。

的arginfo-function是由调用的函数parse_printf_format当此转换出现在模板字符串中时。有关此信息，请参见解析模板字符串 。

注意：在2.0之前的GNU C库版本中， arginfo-function除非用户使用了parse_printf_format功能。这已经改变了。现在拨打任何printf当此格式说明符出现在格式字符串中时，函数将调用此函数。

返回值为0成功的基础上-1失败时（如果发生spec超出范围）。

可移植性注意：可以重新定义标准输出转换，但是强烈建议不要这样做，因为这可能会干扰假定转换效果符合相关语言标准的程序和编译器实现的行为。另外，合格的编译器不必保证程序中每个格式字符串中的每次此类转换都会调用为标准转换注册的函数。
