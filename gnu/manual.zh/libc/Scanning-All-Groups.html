30.14.3扫描所有组的列表

本节说明程序如何读取系统中所有组的列表，一次读取一组。此处描述的功能在中声明grp.h 。

您可以使用fgetgrent用于从特定文件读取组条目的功能。

功能： 结构组* fgetgrent （文件* stream ）

初步： MT不安全种族：fgrent | AS不安全的损坏锁| AC不安全的损坏锁|请参阅POSIX安全概念 。

的fgetgrent函数从中读取下一个条目stream 。它返回指向该条目的指针。该结构是静态分配的，并在随后的调用时被覆盖fgetgrent 。如果要保存信息，则必须复制结构的内容。

流必须与格式与标准组数据库文件相同的文件相对应。

函数： int fgetgrent_r （FILE * stream ，结构组* result_buf ，字符* buffer ，size_t buflen ，结构组** result ）

初步： MT安全|不安全AS损坏| AC不安全的损坏锁|请参阅POSIX安全概念 。

该功能类似于fgetgrent它从中读取下一个用户条目stream 。但是结果将返回到所指向的结构中result_buf 。首先buflen指向的附加缓冲区的字节buffer用于包含其他信息，通常是结果结构的元素所指向的字符串。

该流必须与格式与标准组数据库文件相同的文件相对应。

如果函数返回零result指向带有所需数据的结构（通常在result_buf ）。如果发生错误，则返回值为非零，并且result包含一个空指针。

扫描组数据库中所有条目的方法是setgrent ， getgrent和endgrent 。

函数： void setgrent （无效）

初步： MT不安全种族：格林特地区| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

此函数初始化要从组数据库读取的流。您可以通过调用此流getgrent要么getgrent_r 。

函数： 结构组* getgrent （无效）

初步：不安全MT种族：grent种族：grentbuf地区| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

的getgrent函数从初始化的流中读取下一个条目setgrent 。它返回指向该条目的指针。该结构是静态分配的，并在随后的调用时被覆盖getgrent 。如果要保存信息，则必须复制结构的内容。

函数： int getgrent_r （结构组* result_buf ，字符* buffer ，size_t buflen ，结构组** result ）

初步： MT不安全种族：格林特地区| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

该功能类似于getgrent因为它从初始化的流中返回下一个条目setgrent 。喜欢fgetgrent_r ，它将结果放入用户指向的用户提供的缓冲区中result_buf和buffer 。

如果函数返回零result包含一个指向数据的指针（通常等于result_buf ）。如果发生错误，则返回值为非零，并且result包含一个空指针。

功能： void endgrent （无效）

初步： MT不安全种族：格林特地区| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

此功能关闭由getgrent要么getgrent_r 。
