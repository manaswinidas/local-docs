16.2沟通方式

GNU C库包括对几种不同类型的套接字的支持，每种套接字具有不同的特性。本节介绍了受支持的套接字类型。此处列出的符号常量在sys/socket.h 。

巨集： int SOCK_STREAM

的SOCK_STREAM样式就像管道（请参见Pipes和FIFO ）。它通过与特定远程套接字的连接进行操作，并以字节流的形式可靠地传输数据。

在Connections中详细介绍了此样式的使用。

巨集： int SOCK_DGRAM

的SOCK_DGRAM style用于不可靠地发送单独寻址的数据包。是与之截然相反的SOCK_STREAM 。

每次将数据写入此类套接字时，该数据都会成为一个数据包。以来SOCK_DGRAM套接字没有连接，必须在每个数据包中指定收件人地址。

系统对您的数据传输请求的唯一保证是，它将尽最大努力传递您发送的每个数据包。在第四个和第五个数据包失败之后，第六个数据包可能成功。第七分组可以在第六分组之前到达，并且可以在第六分组之后第二次到达。

典型用途SOCK_DGRAM如果在合理的时间内未看到响应，则可以简单地重新发送数据包。

有关如何使用数据报套接字的详细信息，请参见数据报。

巨集： int SOCK_RAW

这种样式提供对底层网络协议和接口的访问。普通的用户程序通常不需要使用这种样式。
