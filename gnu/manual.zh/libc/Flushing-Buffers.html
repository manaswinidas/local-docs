12.20.2冲洗缓冲液

在缓冲流上刷新输出意味着将所有累积的字符传输到文件。在许多情况下，流中的缓冲输出会自动刷新：

  • 当您尝试执行输出并且输出缓冲区已满时。
  • 流关闭时。请参阅关闭流 。
  • 当程序通过调用终止时exit 。请参阅正常终止 。
  • 写入换行符时，如果流是行缓冲的。
  • 每当任何流上的输入操作实际从其文件中读取数据时。

如果要在另一时间刷新缓冲的输出，请致电fflush ，它在头文件中声明stdio.h 。

功能： int fflush （FILE * stream ）

初步： MT安全|不安全AS损坏| AC不安全锁损坏|请参阅POSIX安全概念 。

此功能导致任何缓冲输出stream传递到文件。如果stream是一个空指针，然后fflush导致刷新所有打开的输出流上的缓冲输出。

该函数返回EOF如果发生写错误，否则为零。

功能： int fflush_unlocked （FILE * stream ）

初步： MT-Safe竞赛：流|不安全AS损坏| AC不安全的腐败|请参阅POSIX安全概念 。

的fflush_unlocked功能相当于fflush函数，但它不会隐式锁定流。

的fflush函数可用于刷新当前打开的所有流。尽管这在某些情况下很有用，但它经常做得比必要的多，因为在需要终端输入且程序希望确保所有输出在终端上可见的情况下，它可能会做。但这意味着只需要刷新行缓冲流。Solaris为此专门引入了一项功能。它总是以某种形式在GNU C库中提供，但从未正式导出。

函数： void _flushlbf （void）

初步： MT安全|不安全AS损坏| AC不安全锁损坏|请参阅POSIX安全概念 。

的_flushlbf函数刷新当前打开的所有行缓冲流。

该函数在stdio_ext.h标头。

兼容性说明：众所周知，某些会损坏大脑的操作系统已完全固定在面向行的输入和输出上，因此刷新行缓冲流会导致写入换行符！幸运的是，这种“功能”似乎变得不那么普遍了。您无需担心GNU C库的问题。

在某些情况下，不刷新流的输出挂起而将其忘记就可能是有用的。如果传输成本高昂，并且不再需要输出，则这是合理的推理。在这种情况下，可以使用Solaris中引入的非标准功能，该功能可以在GNU C库中使用。

功能： void __fpurge （FILE * stream ）

初步： MT-Safe竞赛：流|不安全AS损坏| AC不安全的腐败|请参阅POSIX安全概念 。

的__fpurge函数导致流的缓冲区stream被清空。如果流当前处于读取模式，则缓冲区中的所有输入都将丢失。如果流处于输出模式，则缓冲的输出不会写入设备（或其他任何底层存储），并且会清除缓冲区。

该函数在stdio_ext.h 。
