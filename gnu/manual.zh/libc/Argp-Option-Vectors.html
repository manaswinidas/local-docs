25.3.4在Argp解析器中指定选项

的options在一个领域struct argp指向的向量struct argp_option结构，每个结构都指定argp解析器支持的选项。如果单个选项具有多个名称，则可以使用多个条目。这应该由所有字段中都为零的条目终止。请注意，当使用初始化的C数组作为选项时，编写{ 0 }足以实现这一目标。

数据类型： struct argp_option

此结构指定argp解析器可以理解的单个选项，以及如何解析和记录该选项。它具有以下字段：

const char *name

此选项的长名称，对应于长选项' --name ';如果此选项只有短名称，则此字段可以为零。要为一个选项指定多个名称，可以在该名称之后添加其他条目，并在其中添加OPTION_ALIAS标志集。请参阅Argp选项标志 。

int key

当前选项提供给选项解析器的整数键。如果key的值是可打印的ASCII字符（即， isascii (key)是真实的）， 还指定了一个简短的选项“ -char '，在哪里char是带有代码的ASCII字符key 。

const char *arg

如果非零，则这是与此选项关联的参数名称，必须提供该参数（例如，带有' --name=value ' 要么 ' -char value '语法），除非OPTION_ARG_OPTIONAL标志（请参阅Argp选项标志 ）已设置，在这种情况下可以提供它。

int flags

与该选项相关的标志，上面提到了其中一些标志。请参阅Argp选项标志 。

const char *doc

此选项的文档字符串，用于在帮助消息中打印。

如果两者name和key字段为零，此字符串将在常规选项列的左侧以选项卡式打印，使其用作组标题。这将是该组中的第一件事。在这种用法中，通常以' :字符。

int group

此选项的组标识。

在很长的帮助消息中，选项在每个组中按字母顺序排序，并且这些组以0、1、2，...， n ，- m ，…，-2，-1。

选项数组中此字段为0的每个条目都将继承前一个条目的组号，如果是第一个，则为零。如果是带有name和key字段均为零，默认为上一个条目+ 1。自动选项，例如“ --help '放入-1组。

请注意，由于C结构的初始化规则，通常无需指定此字段，因为0是正确的值。
