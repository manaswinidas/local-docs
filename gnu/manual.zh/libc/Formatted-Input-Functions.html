12.14.8格式化的输入函数

这是执行格式化输入的功能的说明。这些函数的原型在头文件中stdio.h 。

函数： int scanf （const char * template ，…）

初步： MT安全区域| AS不安全的损坏堆| AC不安全的内存锁损坏|请参阅POSIX安全概念 。

的scanf函数从流中读取格式化的输入stdin在模板字符串的控制下template 。可选参数是指向接收结果值的位置的指针。

返回值通常是成功分配的数量。如果在执行任何匹配之前检测到文件结束条件，包括与模板中的空格和文字字符匹配，则EOF返回。

函数： int wscanf （const wchar_t * template ，…）

初步： MT安全区域| AS不安全的损坏堆| AC不安全的内存锁损坏|请参阅POSIX安全概念 。

的wscanf函数从流中读取格式化的输入stdin在模板字符串的控制下template 。可选参数是指向接收结果值的位置的指针。

返回值通常是成功分配的数量。如果在执行任何匹配之前检测到文件结束条件，包括与模板中的空格和文字字符匹配，则WEOF返回。

函数： int fscanf （FILE * stream ，const char * template ，…）

初步： MT安全区域| AS不安全的损坏堆| AC不安全的内存锁损坏|请参阅POSIX安全概念 。

这个功能就像scanf ，除了从流中读取输入stream代替stdin 。

函数： int fwscanf （FILE * stream ，const wchar_t * template ，…）

初步： MT安全区域| AS不安全的损坏堆| AC不安全的内存锁损坏|请参阅POSIX安全概念 。

这个功能就像wscanf ，除了从流中读取输入stream代替stdin 。

功能： int sscanf （const char * s ，const char * template ，…）

初步： MT安全区域| AS不安全堆| AC不安全的内存|请参阅POSIX安全概念 。

就像scanf ，但字符取自以空值终止的字符串s而不是从流中。到达字符串的末尾被视为文件结束条件。

如果在重叠的对象之间进行复制，则该函数的行为不确定（例如，如果s也可以作为参数来接受在' %s '，' %S '， 要么 ' %[转换。

功能： int swscanf （const wchar_t * ws ，const wchar_t * template ，…）

初步： MT安全区域| AS不安全堆| AC不安全的内存|请参阅POSIX安全概念 。

就像wscanf ，但字符取自以空值终止的字符串ws而不是从流中。到达字符串的末尾被视为文件结束条件。

如果在重叠的对象之间进行复制，则该函数的行为不确定（例如，如果ws也可以作为参数来接受在' %s '，' %S '， 要么 ' %[转换。
