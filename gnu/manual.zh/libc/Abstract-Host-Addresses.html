16.6.2.1 Internet主机地址

IPv4 Internet主机地址是一个包含四个字节数据的数字。从历史上看，它们分为两部分， 网络内的网络号和本地网络地址号 。在1990年代中期，引入了无类别地址，从而改变了这种行为。由于某些函数隐式地期望使用旧的定义，因此我们首先描述基于类的网络，然后再描述无类地址。IPv6仅使用无类地址，因此以下段落不适用。

基于类的IPv4网络号由前一个，两个或三个字节组成；其余字节是本地地址。

IPv4网络号已在网络信息中心（NIC）中注册，并分为三类-A，B和C。单个计算机的本地网络地址号已向特定网络的管理员注册。

A类网络的单字节数字范围为0到127。A类网络数量很少，但是每个网络都可以支持大量主机。中型B类网络有两个字节的网络号，第一个字节的范围为128到191。C类网络最小。它们具有三字节的网络号，第一字节的范围为192-255。因此，Internet地址的前1个，2个或3个字节指定了一个网络。Internet地址的其余字节指定该网络内的地址。

A类网络0保留用于广播到所有网络。另外，每个网络中的主机号0保留用于广播到该网络中的所有主机。这些用途现在已过时，但出于兼容性原因，您不应使用网络0和主机号0。

A类网络127被保留用于环回；您可以随时使用Internet地址' 127.0.0.1 '指主机。

由于一台计算机可以是多个网络的成员，因此它可以具有多个Internet主机地址。但是，永远不应该有多台具有相同主机地址的机器。

Internet地址有四种形式的标准数字符号 ：

a.b.c.d

它分别指定地址的所有四个字节，并且是常用的表示形式。

a.b.c

地址的最后一部分， c ，被解释为2字节数量。这对于在B类网络中使用网络地址号指定主机地址很有用a.b 。

a.b

地址的最后一部分， b ，被解释为3字节数量。这对于使用网络地址号指定A类网络中的主机地址很有用a 。

a

如果只给出一部分，则直接对应于主机地址号。

在地址的每个部分中，都适用用于指定基数的常规C约定。换句话说，领先的“ 0x ' 要么 ' 0X '表示十六进制基数；领先的' 0 '表示八进制；否则假定为十进制基数。

无类地址

IPv4地址（也包括IPv6地址）现在被认为是无类的。 A，B和C类之间的区别可以忽略。相反，IPv4主机地址由一个32位地址和一个32位掩码组成。该掩码包含网络部分的设置位和主机部分的清除位。网络部分从左侧开始连续，其余位代表主机。结果，可以简单地将网络掩码指定为设置的位数。A，B和C类只是该一般规则的特例。例如，A类地址的网络掩码为“ 255.0.0.0 '或前缀长度为8。

无类IPv4网络地址以数字和点符号表示，并附加前缀长度和斜杠作为分隔符。例如，A类网络10被写为“ 10.0.0.0/8 '。

IPv6地址

IPv6地址包含128位（IPv4具有32位）的数据。主机地址通常写为以冒号分隔的八个16位十六进制数字。两个冒号用于缩写连续零的字符串。例如，IPv6回送地址' 0:0:0:0:0:0:0:1 '可以写成' ::1 '。
