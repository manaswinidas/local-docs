13.7在两个文件之间复制数据

提供了一种特殊功能，可以在同一文件系统上的两个文件之间复制数据。系统可以优化此类复制操作。这在网络文件系统上尤其重要，否则，该文件必须通过网络传输两次。

请注意，此功能仅复制文件数据，而不复制诸如文件许可权或扩展属性之类的元数据。

功能： 为ssize_t copy_file_range（INT inputfd ，off64_t * inputpos ，int outputfd ，off64_t * outputpos ，ssize_t length ，unsigned int flags ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

此功能最多可复制length文件描述符中的字节inputfd到文件描述符outputfd 。

该功能可以在当前文件位置（例如read和write ）和显式偏移量（如pread和pwrite ）。如果inputpos指针为空，文件位置inputfd用作复制操作的起点，并且文件位置在复制过程中前进。如果inputpos不为空，则*inputpos用作复制操作的起点，并且*inputpos以复制的字节数增加，但文件位置保持不变。类似的规则适用于outputfd和outputpos输出文件的位置。

的flags参数是当前保留的，并且必须为零。

的copy_file_range函数返回复制的字节数。可以小于指定的数量length如果输入文件包含的剩余字节少于length ，或者发生读取或写入失败。如果立即遇到输入文件的结尾，则返回值为零。

如果无法复制任何字节，要报告错误， copy_file_range返回值-1并设置errno 。下表列出了此功能的一些错误情况。

ENOSYS

内核未实现所需的功能。

EISDIR

至少一个描述符inputfd要么outputfd指目录。

EINVAL

至少一个描述符inputfd要么outputfd指非常规，非目录文件（例如套接字或FIFO）。

复制操作之前和之后的输入或输出位置超出了实现定义的限制。

的flags参数不为零。

EFBIG

新文件的大小将超过进程文件的大小限制。请参阅资源限制 。

复制操作之前和之后的输入或输出位置超出了实现定义的限制。如果未在32位计算机上使用大文件支持（LFS）打开文件，并且复制操作将创建的文件大于off_t可以代表。

EBADF

争论inputfd不是有效的文件描述符，无法读取。

争论outputfd不是可打开的有效文件描述符，或者outputfd已与O_APPEND 。

此外， copy_file_range可能因使用的错误代码而失败read ， pread ， write和pwrite 。

的copy_file_range功能是取消点。在取消的情况下，输入位置（文件位置或*inputpos ）是不确定的。
