24.6.1发信号给自己

进程可以使用raise功能。该函数在signal.h 。

函数： int 提高 （int signum ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的raise功能发送信号signum调用过程。如果成功，则返回零；如果失败，则返回非零值。关于失败的唯一原因是signum是无效的。

函数： int gsignal （int signum ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的gsignal函数与raise ;提供它仅是为了与SVID兼容。

一种方便的用途raise是重现您捕获的信号的默认行为。例如，假设您程序的用户键入了SUSP字符（通常C-z ;请参阅特殊字符 ）以向其发送交互式停止信号（ SIGTSTP ），并且您想在停止之前清理一些内部数据缓冲区。您可以这样设置：

#include <signal.h>

/* When a stop signal arrives, set the action back to the default
   and then resend the signal after doing cleanup actions. */

void
tstp_handler (int sig)
{
  signal (SIGTSTP, SIG_DFL);
  /* Do cleanup actions here. */
  …
  raise (SIGTSTP);
}

/* When the process is continued again, restore the signal handler. */

void
cont_handler (int sig)
{
  signal (SIGCONT, cont_handler);
  signal (SIGTSTP, tstp_handler);
}


/* Enable both handlers during program initialization. */

int
main (void)
{
  signal (SIGCONT, cont_handler);
  signal (SIGTSTP, tstp_handler);
  …
}

便携性说明： raise由ISO C 委员会发明。较旧的系统可能不支持它，因此使用kill可能更便携。请参阅发信号通知另一个进程 。
