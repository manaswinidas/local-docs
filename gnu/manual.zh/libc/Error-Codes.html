2.2错误代码

错误代码宏在头文件中定义errno.h 。它们全部扩展为整数常数值。其中一些错误代码不能在GNU系统上发生，但是可以在其他系统上使用GNU C库来发生。

巨集： int EPERM

“不允许操作。”只有文件（或其他资源）或具有特殊特权的进程的所有者才能执行该操作。

巨集： int ENOENT

“无此文件或目录。”对于在预期已经存在的上下文中引用的普通文件，这是一个“文件不存在”错误。

巨集： int ESRCH

“没有这样的过程。”没有任何进程与指定的进程ID匹配。

巨集： int EINTR

“系统调用中断。”发生异步信号，并阻止了呼叫的完成。发生这种情况时，您应该再次尝试呼叫。

您可以选择在信号处理后恢复功能，而不是失败EINTR ;请参阅中断的基元 。

宏： int EIO

“输入/输出错误。”通常用于物理读取或写入错误。

巨集： int ENXIO

“没有这样的设备或地址。”系统尝试使用由您指定的文件表示的设备，但找不到该设备。这可能意味着设备文件安装不正确，或者物理设备丢失或未正确连接到计算机。

巨集： int E2BIG

“参数列表太长。”当参数传递给使用以下命令之一执行的新程序时使用exec函数（请参阅执行文件 ）占用太多内存空间。在GNU / Hurd系统上永远不会出现这种情况。

巨集： int ENOEXEC

“执行格式错误。”无效的可执行文件格式。这种情况是由exec功能;请参阅执行文件 。

巨集： int EBADF

“错误的文件描述符。”例如，已关闭的描述符上的I / O或从描述符打开的描述符上的I / O仅用于写入（反之亦然）。

巨集： int ECHILD

“没有子进程。”当没有任何可操纵的进程时，该错误发生在应该操纵子进程的操作上。

巨集： int EDEADLK

“避免了资源僵局。”分配系统资源将导致死锁情况。系统不保证会注意到所有此类情况。此错误表示您很幸运，系统也注意到了；它可能会挂起。有关示例，请参见文件锁 。

巨集： int ENOMEM

“无法分配内存。”系统无法分配更多的虚拟内存，因为其容量已满。

宏： int EACCES

“没有权限。”文件权限不允许尝试的操作。

巨集： int EFAULT

“地址错误。”检测到无效的指针。在GNU / Hurd系统上，永远不会发生此错误。您会收到一个信号。

巨集： int ENOTBLK

“需要阻止设备。”在需要一个特殊文件的情况下，给出了不是块特殊文件的文件。例如，尝试将普通文件挂载为Unix中的文件系统会出现此错误。

巨集： int EBUSY

“设备或资源繁忙。”无法共享的系统资源已在使用中。例如，如果尝试删除作为当前已挂载文件系统根目录的文件，则会出现此错误。

巨集： int EEXIST

“文件已存在。”在仅指定一个新文件的上下文中指定了一个现有文件。

巨集： int EXDEV

“无效的跨设备链接。”检测到试图跨文件系统建立不正确的链接。这不仅发生在您使用link （请参阅“ 硬链接” ），而且在您使用rename （请参阅重命名文件 ）。

巨集： int ENODEV

“无此设备。”给期望特定类型设备的功能提供了错误的设备类型。

巨集： int ENOTDIR

“不是目录。”当需要目录时，指定的文件不是目录。

巨集： int EISDIR

“是目录。”您无法打开要写入的目录，也不能创建或删除指向该目录的硬链接。

巨集： int EINVAL

“无效的论点。”这用于指示将错误的参数传递给库函数时出现的各种问题。

巨集： int EMFILE

“打开的文件太多。”当前进程打开的文件太多，无法再打开。重复的描述符确实计入此限制。

在BSD和GNU中，打开文件的数量由通常可以增加的资源限制控制。如果收到此错误，则可能要增加RLIMIT_NOFILE限制或使其不受限制；请参阅资源限制 。

巨集： int ENFILE

“系统中打开的文件太多。”整个系统中有太多不同的文件开口。请注意，任何数量的链接通道都仅计为一个文件的打开。请参阅链接的频道 。在GNU / Hurd系统上永远不会发生此错误。

巨集： int ENOTTY

“设备不适当的ioctl。”I / O控制操作不当，例如尝试在普通文件上设置终端模式。

巨集： int ETXTBSY

“文本文件忙。”尝试执行当前正在打开以供写入的文件，或写入当前正在执行的文件。通常认为使用调试器运行程序可以打开程序进行编写，并且会导致此错误。（名称代表“文本文件繁忙”。）这在GNU / Hurd系统上不是错误；文本将根据需要复制。

巨集： int EFBIG

“文件过大。”文件大小将大于系统允许的大小。

巨集： int ENOSPC

“设备上没有剩余空间。”由于磁盘已满，对文件的写操作失败。

巨集： int ESPIPE

“非法寻求。”无效的查找操作（例如在管道上）。

宏： int EROFS

“只读文件系统。”试图修改只读文件系统上的内容。

巨集： int EMLINK

“链接太多。”单个文件的链接数将变得太大。 rename如果已重命名的文件已具有尽可能多的链接，则可能导致此错误（请参阅重命名文件 ）。

巨集： int EPIPE

“管道坏了。”没有过程从管道的另一端读取。每个返回此错误代码的库函数也会生成一个SIGPIPE信号;如果未处理或阻止，该信号将终止程序。因此，您的程序将永远不会真正看到EPIPE除非已处理或阻止SIGPIPE 。

巨集： int EDOM

“数值论点超出范围。”当自变量值不属于定义函数的域时，由数学函数使用。

巨集： int ERANGE

“数值结果超出范围。”当结果值由于上溢或下溢而无法表示时，由数学函数使用。

巨集： int EAGAIN

“资源暂时不可用。”如果稍后重试，则该呼叫可能会起作用。宏EWOULDBLOCK是的另一个名字EAGAIN ;它们在GNU C库中始终相同。

在几种不同情况下可能会发生此错误：

  • 在选择了非阻塞模式的对象上尝试了将阻塞的操作。再次尝试相同的操作将阻塞，直到某些外部条件允许读取，写入或连接（无论该操作如何）为止。您可以使用select找出何时可以进行手术；请参阅等待I / O。

    可移植性注意：在许多较旧的Unix系统中，此情况由表示。 EWOULDBLOCK ，这是一个明显的错误代码，与EAGAIN 。为了使您的程序具有可移植性，您应该检查两个代码并对其进行相同处理。

  • 暂时的资源短缺使操作无法进行。fork可以返回此错误。它表明短缺现象有望解决，因此您的程序可以稍后再次尝试调用，并且可能成功。延迟几秒钟再尝试一次，以留出时间让其他进程释放稀缺的资源，这可能是一个好主意。这种短缺通常相当严重，并且会影响整个系统，因此通常交互式程序应将错误报告给用户并返回其命令循环。
巨集： int EWOULDBLOCK

“操作会阻止。”在GNU C库中，这是别名EAGAIN （以上）。在每个操作系统上，这些值始终相同。

许多较旧的Unix系统中的C库都有EWOULDBLOCK作为单独的错误代码。

宏： int EINPROGRESS

“操作正在进行中。”在选择了非阻塞模式的对象上启动了无法立即完成的操作。某些必须始终阻塞的功能（例如connect ;参见连接 ）永不返回EAGAIN 。相反，他们返回EINPROGRESS表示操作已经开始，将需要一些时间。尝试在调用返回之前操纵对象EALREADY 。您可以使用select找出挂起操作何时完成的功能；请参阅等待I / O。

巨集： int EALREADY

“操作已经在进行中。”对已选择非阻止模式的对象的操作已经在进行中。

巨集： int ENOTSOCK

“非套接字上的套接字操作。”需要套接字时，指定的文件不是套接字。

巨集： int EMSGSIZE

“消息太长。”在套接字上发送的消息的大小大于支持的最大大小。

巨集： int EPROTOTYPE

“套接字的协议类型错误。”套接字类型不支持所请求的通信协议。

宏： int ENOPROTOOPT

“协议不可用。”您指定的套接字选项对于套接字使用的特定协议没有意义。请参阅套接字选项 。

巨集： int EPROTONOSUPPORT

“不支持协议。”套接字域不支持所请求的通信协议（可能是因为所请求的协议完全无效）。请参阅创建套接字 。

巨集： int ESOCKTNOSUPPORT

“不支持套接字类型。”不支持套接字类型。

巨集： int EOPNOTSUPP

“不支持该操作。”不支持您请求的操作。某些套接字功能并非对所有类型的套接字都有意义，而其他套接字功能可能未对所有通信协议都实现。在GNU / Hurd系统上，当对象不支持特定操作时，许多调用都会发生此错误。这是一般指示，表明服务器对该呼叫不知道要做什么。

巨集： int EPFNOSUPPORT

“不支持协议家族。”不支持您请求的套接字通信协议系列。

巨集： int EAFNOSUPPORT

“协议不支持地址族。”不支持为套接字指定的地址族。它与套接字上使用的协议不一致。请参阅套接字 。

巨集： int EADDRINUSE

“地址已在使用中。”所请求的套接字地址已被使用。请参阅套接字地址 。

巨集： int EADDRNOTAVAIL

“无法分配请求的地址。”请求的套接字地址不可用；例如，您试图给套接字指定一个与本地主机名不匹配的名称。请参阅套接字地址 。

巨集： int ENETDOWN

“网络已关闭。”套接字操作失败，因为网络已关闭。

巨集： int ENETUNREACH

“网络不可达。”套接字操作失败，因为包含远程主机的子网不可访问。

巨集： int ENETRESET

“重置时网络断开了连接。”由于远程主机崩溃，网络连接被重置。

巨集： int ECONNABORTED

“软件导致连接中止。”网络连接在本地中止。

巨集： int ECONNRESET

“连接被同级重置。”网络连接由于本地主机无法控制的原因而被关闭，例如由于远程计算机重新启动或不可恢复的协议冲突。

巨集： int ENOBUFS

“没有可用的缓冲区空间。”用于I / O操作的内核缓冲区已全部使用。在GNU中，此错误始终与ENOMEM ;您可能会从网络运营中获得收益。

巨集： int EISCONN

“传输端点已连接。”您试图连接已经连接的套接字。请参阅连接 。

巨集： int ENOTCONN

“传输端点未连接。”套接字未连接任何东西。当您尝试通过套接字传输数据而未先指定数据目的地时，会出现此错误。对于无连接套接字（对于数据报协议，例如UDP），您将获得EDESTADDRREQ代替。

巨集： int EDESTADDRREQ

“要求的目的地地址。”没有为套接字设置默认目标地址。当您尝试通过无连接套接字传输数据时，如果没有先使用connect 。

巨集： int ESHUTDOWN

“传输端点关闭后无法发送。”套接字已被关闭。

巨集： int ETOOMANYREFS

“参考文献过多：无法拼接。”

巨集： int ETIMEDOUT

“连接超时。”具有指定超时的套接字操作在超时期间未收到任何响应。

巨集： int ECONNREFUSED

“拒绝连接。”远程主机拒绝允许网络连接（通常是因为它没有运行所请求的服务）。

巨集： int ELOOP

“太多级别的符号链接。”查找文件名时遇到太多级别的符号链接。这通常表示符号链接的周期。

巨集： int ENAMETOOLONG

“文件名太长。”文件名太长（大于PATH_MAX ;请参阅文件限制 ）或主机名太长（在gethostname要么sethostname ;请参阅主机标识 ）。

巨集： int EHOSTDOWN

“主机掉线了。”用于请求的网络连接的远程主机已关闭。

巨集： int EHOSTUNREACH

“没有到主机的路由。”无法访问用于请求的网络连接的远程主机。

巨集： int ENOTEMPTY

“目录不为空。”目录不为空，预计目录为空。通常，当您尝试删除目录时，会发生此错误。

巨集： int EPROCLIM

“流程太多。”这意味着尝试超过新用户的每用户限制fork 。有关资源的详细信息，请参见资源限制。 RLIMIT_NPROC限制。

宏： int EUSERS

“用户太多。”文件配额系统混乱，因为用户太多。

巨集： int EDQUOT

“磁盘配额超标。”超出了用户的磁盘配额。

巨集： int ESTALE

“文件句柄过时。”这表明文件系统内部混乱，这是由于服务器主机上针对NFS文件系统的文件系统重新排列或其他文件系统的损坏而引起的。修复此状况通常需要卸载，可能需要修复并重新安装文件系统。

巨集： int EREMOTE

巨集： int ENOTSUP

“不支持。”当某些参数值有效，但它们要求的功能不可用时，函数将返回此错误。这可能意味着该功能根本没有实现特定的命令或选项值或标志位。对于在参数中给定的某些对象（例如文件描述符或端口）上运行的函数，它可能意味着仅该特定对象 （文件描述符，端口等）无法支持给定的其他参数。不同的文件描述符可能支持不同范围的参数值。

如果整个功能在实现中根本不可用，则返回ENOSYS代替。

巨集： int EILSEQ

“无效或不完整的多字节或宽字符。”解码多字节字符时，该函数出现无效或不完整的字节序列，或者给定的宽字符无效。

巨集： int EBACKGROUND

“后台处理操作不当。”在GNU / Hurd系统上，支持以下功能的服务器term当呼叫者不在终端的前台进程组中时，协议针对某些操作返回此错误。用户通常不会看到此错误，因为诸如read和write将其翻译成SIGTTIN要么SIGTTOU信号。有关流程组和这些信号的信息，请参见作业控制 。

巨集： int EDIED

“翻译死了。”在GNU / Hurd系统上，当文件被程序翻译并且翻译程序在启动时在连接到文件之前死亡时，打开文件会返回此错误。
