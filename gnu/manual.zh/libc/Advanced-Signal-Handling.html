24.3.2高级信号处理

的sigaction功能与以下功能具有相同的基本作用signal ：指定过程应如何处理信号。然而， sigaction提供更多的控制权，却以更多的复杂性为代价。特别是， sigaction允许您指定其他标志来控制何时生成信号以及如何调用处理程序。

的sigaction函数在signal.h 。

数据类型： struct sigaction

类型结构struct sigaction用于sigaction函数指定有关如何处理特定信号的所有信息。此结构至少包含以下成员：

sighandler_t sa_handler

用法与action争论signal功能。该值可以是SIG_DFL ， SIG_IGN ，或函数指针。请参阅基本信号处理 。

sigset_t sa_mask

这指定在处理程序运行时要阻止的一组信号。阻塞在Handler的阻塞中进行了说明。请注意，默认情况下，在启动其处理程序之前，已自动阻止传递的信号。不管其中的值是什么sa_mask 。如果不希望该信号在其处理程序中被阻止，则必须在该处理程序中编写代码以取消阻止它。

int sa_flags

这指定了可能影响信号行为的各种标志。这些在Sigaction的标志中有更详细的描述。

函数： int sigaction （int signum ，const struct sigaction * restrict action ，结构提示* restrict old-action ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的action参数用于为信号设置新动作signum ，而old-action参数用于返回有关先前与此信号相关联的动作的信息。（换一种说法， old-action具有与signal函数的返回值-您可以查看该信号有效的旧操作，然后在需要时将其恢复。）

要么action要么old-action可以是空指针。如果old-action是一个空指针，这只是抑制了有关旧操作的信息的返回。如果action是一个空指针，与信号关联的动作signum不变；这使您可以查询信号的处理方式，而无需更改该处理方式。

的返回值sigaction如果成功则为零，并且-1失败。下列errno为此功能定义了错误条件：

EINVAL

的signum参数无效，或者您试图捕获或忽略SIGKILL要么SIGSTOP 。
