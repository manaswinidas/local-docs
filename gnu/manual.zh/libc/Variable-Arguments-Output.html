12.12.9可变参数输出函数

功能vprintf和朋友被提供，以便您可以定义自己的可变参数printf类函数，它们使用与内置格式化输出函数相同的内部函数。

定义此类功能的最自然的方法是使用语言结构来表示“ printf并在前五个之后传递此模板以及我的所有论点。”但是用C语言无法做到这一点，而且很难提供一种方法，因为在C语言级别上，无法知道函数接收了多少个参数。

由于该方法是不可能的，因此我们提供了替代功能， vprintf系列，让您通过va_list描述“我在前五个之后的所有论点。”

当足以定义一个宏而不是一个实函数时，GNU C编译器提供了一种使用宏更轻松地执行此操作的方法。例如：

#define myprintf(a, b, c, d, e, rest...) \
	    printf (mytemplate , ## rest)

有关详细信息，请参见C预处理器中的 可变参数宏 。但这仅限于宏，根本不适用于实函数。

致电之前vprintf或本节中列出的其他功能，您必须调用va_start （请参见可变参数函数 ）初始化指向变量参数的指针。那你可以打电话va_arg获取您要处理的参数。这会使指针越过这些参数。

一旦你va_list指针指向您选择的参数，您可以调用vprintf 。该参数和传递给函数的所有后续参数由vprintf以及您分别指定的模板。

可移植性注意： va_list调用后不确定指针vprintf ，因此请勿使用va_arg打电话后vprintf 。相反，您应该致电va_end使指针退出服务。你可以打电话va_start再次从变量参数列表的开头开始获取参数。（或者，您可以使用va_copy制作一份va_list调用前的指针vfprintf ）呼唤vprintf不会破坏函数的参数列表，只会破坏传递给函数的特定指针。

这是显示如何使用的示例vfprintf 。此功能可将错误消息打印到流中stderr ，以及指示程序名称的前缀（有关错误码的说明，请参见错误消息） 。 program_invocation_short_name ）。 

#include <stdio.h>
#include <stdarg.h>

void
eprintf (const char *template, ...)
{
  va_list ap;
  extern char *program_invocation_short_name;

  fprintf (stderr, "%s: ", program_invocation_short_name);
  va_start (ap, template);
  vfprintf (stderr, template, ap);
  va_end (ap);
}

你可以打电话eprintf像这样：

eprintf ("file `%s' does not exist\n", filename);

在GNU C中，有一个特殊的结构可以用来让编译器知道函数使用了printf样式的格式字符串。然后，它可以检查每次调用函数时参数的数量和类型，并在参数与格式字符串不匹配时发出警告。例如，使用以下声明eprintf ：

void eprintf (const char *template, ...)
	__attribute__ ((format (printf, 1, 2)));

这告诉编译器eprintf使用如下格式的字符串printf （相对于scanf ;参见格式化输入 ）;格式字符串显示为第一个参数；满足格式的参数以第二个开头。有关更多信息，请参见《 使用GNU CC》中的“ 声明函数的属性” 。
