16.4接口命名

每个网络接口都有一个名称。它通常由与接口类型有关的几个字母组成，如果存在多个该类型的接口，则可以跟一个数字。例子可能是lo （环回接口）和eth0 （第一个以太网接口）。

尽管这样的名称对于人类来说很方便，但是当程序需要引用接口时必须使用它们是很笨拙的。在这种情况下，接口由其索引引用，该索引是任意分配的小正整数。

头文件中声明了以下函数，常量和数据类型net/if.h 。

常数： size_t IFNAMSIZ

该常数定义了保存接口名称所需的最大缓冲区大小，包括其终止的零字节。

函数： unsigned int if_nametoindex （const char * ifname ）

初步： MT安全| AS不安全锁| AC不安全锁fd |请参阅POSIX安全概念 。

该函数产生对应于特定名称的接口索引。如果不存在具有给定名称的接口，则返回0。

函数： char * if_indextoname （无符号int ifindex ，字符* ifname ）

初步： MT安全| AS不安全锁| AC不安全锁fd |请参阅POSIX安全概念 。

此函数将接口索引映射到其相应名称。返回的名称放在由指向的缓冲区中ifname ，至少必须是IFNAMSIZ字节长度。如果索引无效，则函数的返回值为空指针，否则为ifname 。

数据类型： struct if_nameindex

此数据类型用于保存有关单个接口的信息。它具有以下成员：

unsigned int if_index;

这是接口索引。

char *if_name

这是以空值终止的索引名称。

函数： struct if_nameindex * if_nameindex （无效）

初步： MT安全| AS不安全堆锁定/堆| AC不安全锁/护栏|请参阅POSIX安全概念 。

该函数返回一个数组if_nameindex结构，用于每个存在的接口。列表的末尾由具有接口0和空名称指针的结构指示。如果发生错误，此函数将返回空指针。

返回的结构必须使用if_freenameindex使用后。

函数： void if_freenameindex （结构if_nameindex * ptr ）

初步： MT安全| AS不安全堆| AC不安全的内存|请参阅POSIX安全概念 。

此函数释放先前调用所返回的结构if_nameindex 。
