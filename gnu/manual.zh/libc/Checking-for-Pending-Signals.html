24.7.6检查待处理信号

您可以随时通过以下方式找出哪些信号待处理sigpending 。该函数在signal.h 。

函数： int sigpending （sigset_t * set ）

初步： MT安全| AS-不安全锁/门| AC不安全锁/门|请参阅POSIX安全概念 。

的sigpending函数将有关待处理信号的信息存储在set 。如果有一个挂起的信号被阻止传递，则该信号是返回集的成员。（您可以使用以下方法测试特定信号是否是该信号集的成员： sigismember ;请参阅信号集 。）

返回值为0如果成功，并且-1失败。

测试信号是否未决通常不是有用的。几乎可以肯定的是，测试何时没有阻塞该信号是不好的设计。

这是一个例子。 

#include <signal.h>
#include <stddef.h>

sigset_t base_mask, waiting_mask;

sigemptyset (&base_mask);
sigaddset (&base_mask, SIGINT);
sigaddset (&base_mask, SIGTSTP);

/* Block user interrupts while doing other processing. */
sigprocmask (SIG_SETMASK, &base_mask, NULL);
…

/* After a while, check to see whether any signals are pending. */
sigpending (&waiting_mask);
if (sigismember (&waiting_mask, SIGINT)) {
  /* User has tried to kill the process. */
}
else if (sigismember (&waiting_mask, SIGTSTP)) {
  /* User has tried to stop the process. */
}

请记住，如果有一个特定的信号要处理，则与此同时到达的相同类型的其他信号可能会被丢弃。例如，如果SIGINT另一个信号挂起时SIGINT信号到达时，您的程序在取消阻止此信号时可能只会看到其中之一。

便携性注： sigpending该功能是POSIX.1中的新增功能。较旧的系统没有等效的功能。
