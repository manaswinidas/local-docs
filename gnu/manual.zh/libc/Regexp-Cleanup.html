10.3.6 POSIX Regexp匹配清除

完成使用编译后的正则表达式后，可以通过调用释放其使用的存储空间regfree 。

函数： void regfree （regex_t * compiled ）

初步： MT安全| AS不安全堆| AC不安全的内存|请参阅POSIX安全概念 。

呼唤regfree释放所有的存储空间*compiled指着。这包括regex_t本手册未记录的结构。

regfree不释放对象*compiled本身。

您应该始终释放空间regex_t与结构regfree在使用该结构编译另一个正则表达式之前。

什么时候regcomp要么regexec报告错误，可以使用该功能regerror将其转换为错误消息字符串。

函数： size_t regerror （整数errcode ，const regex_t *限制compiled ，char *限制buffer ，size_t length ）

初步： MT安全环境| AS不安全损坏的堆锁dlopen | AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

此函数为错误代码生成错误消息字符串errcode ，并将字符串存储在length内存字节起始于buffer 。为了compiled参数，提供相同的已编译正则表达式结构， regcomp要么regexec出现错误时正在使用。或者，您可以提供NULL对于compiled ;您仍然会收到有意义的错误消息，但是可能不那么详细。

如果错误消息不能放入length字节（包括终止的空字符），然后regerror截断它。那个字符串regerror商店即使被截断也总是以null终止。

的返回值regerror是存储整个错误消息所需的最小长度。如果小于length ，则错误消息没有被截断，您可以使用它。否则，您应该致电regerror再次使用更大的缓冲区。

这是一个使用regerror ，但总是为错误消息动态分配缓冲区：

char *get_regerror (int errcode, regex_t *compiled)
{
  size_t length = regerror (errcode, compiled, NULL, 0);
  char *buffer = xmalloc (length);
  (void) regerror (errcode, compiled, buffer, length);
  return buffer;
}
