13.15.1文件访问模式

文件访问模式允许将文件描述符用于读取，写入或两者。（在GNU / Hurd系统上，它们也不允许这些，并且允许将文件作为程序执行。）打开文件时选择访问模式，并且永远不会更改。

巨集： int O_RDONLY

打开文件以进行读取访问。

巨集： int O_WRONLY

打开文件以进行写访问。

巨集： int O_RDWR

打开文件以进行读写。

在GNU / Hurd系统（而非其他系统）上， O_RDONLY和O_WRONLY是可以按位进行“或”运算的独立位，并且对任一位进行设置或清除均有效。这意味着O_RDWR是相同的O_RDONLY|O_WRONLY 。允许的文件访问模式为零；它不允许进行输入或输出到文件的操作，但允许其他操作，例如fchmod 。在GNU / Hurd系统上，由于“只读”或“仅写”是用词不当， fcntl.h定义文件访问模式的其他名称。在编写特定于GNU的代码时，最好使用这些名称。但是大多数程序都希望可以移植到其他POSIX.1系统，而应该使用上面的POSIX.1名称。

巨集： int O_READ

打开文件进行读取。和...一样O_RDONLY ;仅在GNU上定义。

巨集： int O_WRITE

打开文件进行写入。和...一样O_WRONLY ;仅在GNU上定义。

巨集： int O_EXEC

打开文件执行。仅在GNU上定义。

使用以下命令确定文件访问模式fcntl ，您必须从检索到的文件状态标志中提取访问模式位。在GNU / Hurd系统上，您只需测试O_READ和O_WRITE标志字中的位。但是在其他POSIX.1系统中，读写访问模式不存储为不同的位标志。提取文件访问模式位的便携式方法是O_ACCMODE 。

巨集： int O_ACCMODE

该宏代表可以与文件状态标志值按位与的掩码，以产生代表文件访问模式的值。该模式将是O_RDONLY ， O_WRONLY ， 要么O_RDWR 。（在GNU / Hurd系统上，它也可以为零，并且永远不包含O_EXEC位。）
