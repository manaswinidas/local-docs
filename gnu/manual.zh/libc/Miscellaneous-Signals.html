24.2.7其他信号

这些信号用于各种其他目的。通常，它们不会影响您的程序，除非它明确将它们用于某些用途。

巨集： int SIGUSR1
巨集： int SIGUSR2

的SIGUSR1和SIGUSR2信号留给您使用，以使用任何您想要的方式。如果在接收信号的程序中为它们编写信号处理程序，则它们对于简单的进程间通信很有用。

默认操作是终止该过程。

巨集： int SIGWINCH

窗口大小更改。当终端驱动程序在屏幕上的行数和列数的记录更改时，这在某些系统（包括GNU）上会生成。默认操作是忽略它。

如果程序进行全屏显示，则应处理SIGWINCH 。信号到达时，它应获取新的屏幕尺寸并相应地重新格式化其显示。

巨集： int SIGINFO

信息请求。在4.4 BSD和GNU / Hurd系统上，当用户以规范模式键入STATUS字符时，此信号将发送到控制终端的前台进程组中的所有进程。请参阅信号字符 。

如果流程是流程组的负责人，则默认操作是打印一些有关系统以及流程正在执行的状态信息。否则默认为不执行任何操作。
