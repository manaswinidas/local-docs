3.2.3.6分配对齐的内存块

由返回的块地址malloc要么realloc在GNU系统中，总是8的倍数（在64位系统上是16的倍数）。如果您需要一个地址是其高2的倍数的块，请使用aligned_alloc要么posix_memalign 。 aligned_alloc和posix_memalign被声明在stdlib.h 。

函数： void * aligned_alloc （size_t alignment ，size_t size ）

初步： MT安全| AS不安全锁| AC不安全锁fd mem |请参阅POSIX安全概念 。

的aligned_alloc函数分配一个块size地址是的倍数的字节alignment 。的alignment必须是二的幂size必须是的倍数alignment 。

的aligned_alloc函数针对错误返回空指针并进行设置errno为以下值之一：

ENOMEM

没有足够的可用内存来满足请求。

EINVAL

alignment不是二的幂。

此函数是在ISO C11 引入的，因此，与现代非POSIX系统相比，它可以具有更好的可移植性。 posix_memalign 。

函数： void * memalign （size_t boundary ，size_t size ）

初步： MT安全| AS不安全锁| AC不安全锁fd mem |请参阅POSIX安全概念 。

的memalign函数分配一个块size地址是的倍数的字节boundary 。的boundary必须是两个的幂！功能memalign通过分配一个稍大的块，然后在指定边界上的块内返回一个地址来工作。

的memalign函数针对错误返回空指针并进行设置errno为以下值之一：

ENOMEM

没有足够的可用内存来满足请求。

EINVAL

boundary不是二的幂。

的memalign功能已过时， aligned_alloc要么posix_memalign应该改为使用。

函数： int posix_memalign （void ** memptr ，size_t alignment ，size_t size ）

初步： MT安全| AS不安全锁| AC不安全锁fd mem |请参阅POSIX安全概念 。

的posix_memalign功能类似于memalign函数，因为它返回一个缓冲区size字节对齐为的倍数alignment 。但这给参数增加了一个要求alignment ：该值必须是2的整数倍sizeof (void *) 。

如果函数成功分配了内存，则返回指向已分配内存的指针*memptr返回值为零。否则，该函数将返回指示问题的错误值。返回的可能的错误值是：

ENOMEM

没有足够的可用内存来满足请求。

EINVAL

alignment不是二的倍数的幂sizeof (void *) 。

POSIX 1003.1d中引入了此功能。尽管此功能已被aligned_alloc ，对于不支持ISO C11 旧POSIX系统而言，它更具移植性。

函数： void * valloc （size_t size ）

初步： MT不安全初始化| AS不安全初始化锁| AC不安全初始化锁fd mem |请参阅POSIX安全概念 。

使用valloc就像使用memalign并将页面大小作为第一个参数的值传递。它是这样实现的：

void *
valloc (size_t size)
{
  return memalign (getpagesize (), size);
}
