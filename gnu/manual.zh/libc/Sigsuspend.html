24.8.3使用sigsuspend

等待信号到达的干净可靠的方法是将其阻塞然后使用sigsuspend 。通过使用sigsuspend在循环中，您可以等待某些类型的信号，同时让其他类型的信号由其处理程序处理。

功能： int sigsuspend （const sigset_t * set ）

初步： MT不安全种族：sigprocmask /！bsd！linux | AS-不安全锁/门| AC不安全锁/门|请参阅POSIX安全概念 。

此功能将过程的信号掩码替换为set然后暂停该过程，直到传递了一个信号，该信号的作用是终止该过程或调用一个信号处理功能。换句话说，该程序将被有效地挂起，直到不是其中一个成员的信号之一set到达。

如果通过调用处理程序函数的信号的传递唤醒了该过程，并且处理程序函数返回，则sigsuspend也返回。

面具仍然存在set只要sigsuspend在等待。功能sigsuspend返回时始终恢复前一个信号掩码。

返回值和错误条件与相同pause 。

用sigsuspend ，您可以更换pause要么sleep上一节中的循环完全可靠：

sigset_t mask, oldmask;

…

/* Set up the mask of signals to temporarily block. */
sigemptyset (&mask);
sigaddset (&mask, SIGUSR1);

…

/* Wait for a signal to arrive. */
sigprocmask (SIG_BLOCK, &mask, &oldmask);
while (!usr_interrupt)
  sigsuspend (&oldmask);
sigprocmask (SIG_UNBLOCK, &mask, NULL);

最后一段代码有些棘手。这里要记住的关键是sigsuspend返回时，它将过程的信号掩码重置为原始值，即调用之前的值sigsuspend在这种情况下SIGUSR1信号再次被阻塞。第二次致电sigprocmask必须显式解除对此信号的阻塞。

另一点：您可能想知道为什么while循环是完全必要的，因为该程序显然只在等待一个SIGUSR1信号。答案是面具传给了sigsuspend允许通过传送其他种类的信号（例如，作业控制信号）唤醒该过程。如果该过程被未设置的信号唤醒usr_interrupt ，它只是再次暂停自身，直到“正确”的信号最终到达。

该技术需要更多的准备工作，但是对于您要使用的每种等待条件，只需要一次。实际等待的代码只有四行。
