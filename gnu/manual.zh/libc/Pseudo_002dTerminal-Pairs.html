17.9.2打开伪终端对

这些功能源自BSD，可单独使用libutil库，并在pty.h 。

函数： int openpty （int * amaster ，int * aslave ，字符* name ，const struct termios * termp ，const struct winsize * winp ）

初步： MT安全区域| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

该函数分配并打开一个伪终端对，并在其中返回主服务器的文件描述符*amaster ，以及从站的文件描述符*aslave 。如果论点name不是空指针，则从属伪终端设备的文件名存储在*name 。如果termp不是空指针，则从站的终端属性设置为结构中指定的那些termp指向（请参阅终端模式 ）。同样，如果winp不是空指针，则从站的屏幕大小设置为以下结构中指定的值： winp指着。

警告：使用openpty具有功能name没有设置为NULL 非常危险，因为它无法防止字符串溢出name 。您应该使用ttyname返回的文件描述符上的函数*slave而是查找从属伪终端设备的文件名。

函数： int forkpty （int * amaster ，字符* name ，const struct termios * termp ，const struct winsize * winp ）

初步： MT安全区域| AS-Unsafe dlopen插件堆锁| AC不安全损坏的锁fd mem |请参阅POSIX安全概念 。

此功能类似于openpty功能，但除此之外，派生一个新进程（请参阅创建进程 ），并使新打开的从属伪终端设备成为子进程的控制终端（请参见Controling Terminal ）。

如果操作成功，则将同时存在父进程和子进程，并且都forkpty返回值，但具有不同的值：它在子进程中返回值0 ，并在父进程中返回子进程的ID。

如果伪终端对的分配或进程创建失败， forkpty在父进程中返回值-1 。

警告： forkpty功能方面存在相同的问题name作为openpty 。
