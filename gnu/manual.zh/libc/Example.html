12.22.3使用方法fmtmsg和addseverity

这是一个简单的示例程序，用于说明本节中介绍的两个功能的使用。 


#include <fmtmsg.h>

int
main (void)
{
  addseverity (5, "NOTE2");
  fmtmsg (MM_PRINT, "only1field", MM_INFO, "text2", "action2", "tag2");
  fmtmsg (MM_PRINT, "UX:cat", 5, "invalid syntax", "refer to manual",
          "UX:cat:001");
  fmtmsg (MM_PRINT, "label:foo", 6, "text", "action", "tag");
  return 0;
}

第二次致电fmtmsg说明了此功能的用法，该用法通常出现在System V系统上，该系统大量使用此功能。在这里简要说明一下该系统如何在System V上工作似乎是值得的。的价值label栏位（ UX:cat ）表示错误发生在Unix程式中cat 。错误的解释如下， action参数是"refer to manual" 。如有必要，这里可以更具体。的tag如上所述，字段包含给定的字符串值label参数，以及唯一的ID（ 001在这种情况下）。对于GNU环境，此字符串可以包含对该程序的“信息”页面中相应节点的引用。

在不指定MSGVERB和SEV_LEVEL函数产生以下输出：

UX:cat: NOTE2: invalid syntax
TO FIX: refer to manual UX:cat:001

我们看到了消息的不同字段以及多余的胶水（冒号和TO FIX字符串）。但是，只有三个呼叫中的一个fmtmsg生产的产出。第一次呼叫不会打印任何内容，因为label参数的格式不正确。该字符串必须包含两个字段，并用冒号分隔（请参阅“ 打印格式化的消息” ）。第三fmtmsg由于类具有数值，因此调用未产生任何输出6没有定义。虽然是带有数值的类5也是默认情况下未定义的addseverity介绍它，并第二次致电fmtmsg产生以上输出。

当我们将程序的环境更改为包含SEV_LEVEL=XXX,6,NOTE运行时，我们得到不同的结果：

UX:cat: NOTE2: invalid syntax
TO FIX: refer to manual UX:cat:001
label:foo: NOTE: text
TO FIX: action tag

现在第三次致电fmtmsg产生了一些输出，我们看到了字符串NOTE来自环境变量的消息出现在消息中。

现在，我们可以通过指定我们感兴趣的字段来减少输出。如果我们另外设置环境变量MSGVERB值severity:label:action我们得到以下输出：

UX:cat: NOTE2
TO FIX: refer to manual
label:foo: NOTE
TO FIX: action

即，由text和tag参数fmtmsg消失了。另请注意，现在NOTE和NOTE2输出中的字符串。这是没有必要的，因为由于缺少文本，因此该行没有更多输出。
