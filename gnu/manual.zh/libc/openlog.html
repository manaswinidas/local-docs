18.2.1 openlog

本节中引用的符号在文件中声明syslog.h 。

功能： void openlog （const char * ident ，int option ，int facility ）

初步： MT安全| AS不安全锁| AC不安全锁fd |请参阅POSIX安全概念 。

openlog打开或重新打开与Syslog的连接，以准备提交消息。

ident是将来的任意标识字符串syslog调用将作为每条消息的前缀。这旨在标识消息的来源，并且人们通常将其设置为将提交消息的程序的名称。

如果ident为NULL，或者openlog如果未调用，则Syslog消息中使用的默认标识字符串将是程序名称，取自argv [0]。

请注意，字符串指针ident将由Syslog例程在内部保留。您一定不能释放内存ident指着。将引用传递给自动变量也很危险，因为离开范围将意味着结束变量的生命周期。如果要更改ident字符串，您必须调用openlog再次;覆盖由指向的字符串ident不是线程安全的。

您可以使Syslog例程删除对ident并通过调用返回默认字符串（程序名称取自argv [0]） closelog ：请参见closelog 。

特别是，如果您正在为共享库编写代码，那么该库可能会被加载然后卸载（例如，PAM模块），并且您使用openlog ，您必须致电closelog在可能卸载库的任何点之前，如本例所示：

#include <syslog.h>

void
shared_library_function (void)
{
  openlog ("mylibrary", option, priority);

  syslog (LOG_INFO, "shared library has been invoked");

  closelog ();
}

无需致电closelog ，未来的调用syslog如果共享库被卸载并且包含字符串的内存，则使用共享库的程序可能会崩溃"mylibrary"变得未映射。这是BSD syslog接口的限制。

openlog可能会或可能不会打开/dev/log插座，取决于option 。如果是这样，它将尝试将其打开并将其作为流套接字连接。如果那不起作用，它将尝试将其打开并将其作为数据报套接字连接。套接字具有“执行时关闭”属性，因此如果进程执行exec，内核将关闭该套接字。

你不用用openlog 。如果你打电话syslog没有打电话openlog ， syslog只是隐式地打开连接，并对其中的信息使用默认值ident和options 。

options是一个位字符串，其位由以下单个位掩码定义：

LOG_PERROR

如果开启， openlog设置连接，以便任何syslog在此连接上，除了将其消息提交给Syslog外，还将其消息写入调用过程的“标准错误”流。如果关闭， syslog不会将消息写入标准错误。

LOG_CONS

如果开启， openlog设置连接，以便syslog如果此连接未能向Syslog提交消息，则将消息写入系统控制台。如果关闭， syslog不会写入系统控制台（但是Syslog当然可能会将收到的消息写入控制台）。

LOG_PID

开启时openlog设置连接，以便syslog在此连接上，将呼叫进程的进程ID（PID）插入消息中。关闭时openlog不插入PID。

LOG_NDELAY

开启时openlog打开并连接/dev/log插座。关闭时，未来syslog呼叫必须打开并连接插座。

可移植性说明：在早期系统中，这一点的含义恰恰相反。

LOG_ODELAY

该位不执行任何操作。存在它是为了向后兼容。

如果还有其他地方options亮，结果不确定。

facility是此连接的默认功能代码。一种syslog在指定默认功能的此连接上，此功能使该功能与消息相关联。看到syslog获取可能的值。零值表示默认值，即LOG_USER 。

如果在致电时Syslog连接已经打开openlog ， openlog “重新打开”连接。重新打开就像打开一样，不同之处在于，如果您为默认设施代码指定零，则默认设施代码只是保持不变，并且如果您指定LOG_NDELAY并且套接字已经打开并已连接， openlog只是那样离开。
