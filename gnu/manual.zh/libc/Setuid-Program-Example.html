30.9 Setuid程序示例

这是显示如何设置程序以更改其有效用户ID的示例。

这是游戏程序的一部分，称为caber-toss操纵文件scores那只能由游戏程序本身写。该程序假定其可执行文件将使用setuid位设置并由与该用户相同的用户拥有。 scores文件。通常，系统管理员会设置一个帐户，例如games以此目的。

可执行文件给定模式4755 ，这样就可以ls -l '在它上产生如下输出：

-rwsr-xr-x   1 games    184422 Jul 30 15:17 caber-toss

setuid位在文件模式下显示为“ s '。

分数文件给定模式644 ，然后做一个“ ls -l '上显示：

-rw-r--r--  1 games           0 Jul 31 15:33 scores

程序的这些部分显示了如何设置更改后的用户ID。此程序是有条件的，因此，如果受支持，它将利用文件ID功能，否则将使用setreuid交换有效和真实的用户ID。 

#include <stdio.h>
#include <sys/types.h>
#include <unistd.h>
#include <stdlib.h>


/* Remember the effective and real UIDs. */

static uid_t euid, ruid;


/* Restore the effective UID to its original value. */

void
do_setuid (void)
{
  int status;

#ifdef _POSIX_SAVED_IDS
  status = seteuid (euid);
#else
  status = setreuid (ruid, euid);
#endif
  if (status < 0) {
    fprintf (stderr, "Couldn't set uid.\n");
    exit (status);
    }
}



/* Set the effective UID to the real UID. */

void
undo_setuid (void)
{
  int status;

#ifdef _POSIX_SAVED_IDS
  status = seteuid (ruid);
#else
  status = setreuid (euid, ruid);
#endif
  if (status < 0) {
    fprintf (stderr, "Couldn't set uid.\n");
    exit (status);
    }
}


/* Main program. */

int
main (void)
{
  /* Remember the real and effective user IDs.  */
  ruid = getuid ();
  euid = geteuid ();
  undo_setuid ();

  /* Do the game and record the score.  */
  …
}

注意第一件事main功能是将有效用户ID设置回真实用户ID。这样一来，在用户玩游戏时执行的任何其他文件访问都将使用真实的用户ID来确定权限。仅当程序需要打开成绩文件时，它才切换回文件用户ID，如下所示：

/* Record the score. */

int
record_score (int score)
{
  FILE *stream;
  char *myname;

  /* Open the scores file. */
  do_setuid ();
  stream = fopen (SCORES_FILE, "a");
  undo_setuid ();


  /* Write the score to the file. */
  if (stream)
    {
      myname = cuserid (NULL);
      if (score < 0)
        fprintf (stream, "%10s: Couldn't lift the caber.\n", myname);
      else
        fprintf (stream, "%10s: %d feet.\n", myname, score);
      fclose (stream);
      return 0;
    }
  else
    return -1;
}
