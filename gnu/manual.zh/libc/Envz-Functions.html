5.15.2 Envz函数

Envz向量只是argz向量，对每个元素的形式都有附加的约束。因此，也可以在它们上使用argz函数。

envz向量中的每个元素都是一个名称-值对，以'='字节;如果多个'='字节存在于元素中，第一个字节之后的字节被视为值的一部分，并像其他所有非字节一样对待'\0'个字节。

如果没有 '='元素中存在字节，该元素被视为“空”条目的名称，与带有空值的条目不同： envz_get将返回0如果给定名称为null的条目，则值为空的条目将导致值为"" ; envz_entry仍然会找到此类条目。空条目可以使用envz_strip功能。

与argz函数一样，可以分配内存（从而失败）的envz函数的返回类型为error_t ，然后返回0要么ENOMEM 。

这些功能在标准包含文件中声明envz.h 。

函数： char * envz_entry （const char * envz ，size_t envz_len ，const char * name ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的envz_entry函数在以下位置找到条目envz名称name ，并返回指向整个条目的指针，即以开头的argz元素name跟一个'='字节。如果没有该名称的条目， 0返回。

函数： char * envz_get （const char * envz ，size_t envz_len ，const char * name ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的envz_get函数在以下位置找到条目envz名称name （喜欢envz_entry ），并返回指向该条目的值部分的指针（ '=' ）。如果没有名称相同的条目（或只有一个空条目）， 0返回。

函数： error_t envz_add （字符** envz ，size_t * envz_len ，const char * name ，const char * value ）

初步： MT安全| AS不安全堆| AC不安全的内存|请参阅POSIX安全概念 。

的envz_add函数将条目添加到*envz （更新*envz和*envz_len ）的名称name和价值value 。如果同名条目已经存在envz ，它首先被删除。如果value是0 ，则新条目将是特殊的null类型条目（如上所述）。

函数： error_t envz_merge （字符** envz ，size_t * envz_len ，const char * envz2 ，size_t envz2_len ，int override ）

初步： MT安全| AS不安全堆| AC不安全的内存|请参阅POSIX安全概念 。

的envz_merge函数将每个条目添加到envz2至envz ，就像envz_add ，更新*envz和*envz_len 。如果override是真的，那么值envz2将取代同名的人envz ，否则不会。

在这方面，与其他条目一样对待空条目，因此在envz可以防止在envz2从被添加到envz如果override是错误的。

函数： void envz_strip （char ** envz ，size_t * envz_len ）

初步： MT安全| AS安全交流安全请参阅POSIX安全概念 。

的envz_strip函数从中删除所有空条目envz ，更新*envz和*envz_len 。

函数： void envz_remove （char ** envz ，size_t * envz_len ，const char * name ）

初步： MT安全| AS不安全堆| AC不安全的内存|请参阅POSIX安全概念 。

的envz_remove函数删除一个名为name从envz ，更新*envz和*envz_len 。
