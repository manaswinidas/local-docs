1引言

给定一个或多个模式， grep在输入文件中搜索与模式匹配的内容。当发现行中有匹配项时，它将行复制到标准输出（默认情况下），或产生您要求的带有选项的任何其他类型的输出。

虽然grep希望对文本进行匹配，除了可用内存以外，对输入行长没有限制，并且可以匹配一行中的任意字符。如果输入文件的最后一个字节不是换行符， grep默默地提供一个。由于换行符也是模式列表的分隔符，因此无法匹配文本中的换行符。

2.1.2匹配控制

-e patterns
--regexp=patterns

使用patterns作为一种或多种模式；内的换行符patterns将每个模式与下一个模式分开。如果此选项已多次使用或与-f （ --file ）选项，搜索给定的所有模式。（ -e由POSIX指定。）

-f file
--file=file

从中获取模式file ，每行一个。如果此选项已多次使用或与-e （ --regexp ）选项，搜索给定的所有模式。空文件包含零个模式，因此不匹配。（ -f由POSIX指定。）

-i
-y
--ignore-case

忽略大小写区别，以便仅大小写不同的字符彼此匹配。尽管当字母仅通过小写-大写对而不同时，这很简单，但是在其他情况下，行为没有指定。例如，大写字母“ S”在许多语言环境中都有一个不寻常的小写字母“ ſ”（Unicode字符U + 017F，拉丁文小写字母LONG S），并且即使这个不寻常字符与“ S”或“ s”匹配，也不确定大写将产生“ S”。另一个示例：小写的德语字母“ß”（U + 00DF，拉丁小写字母SHARP S）通常大写为两个字符的字符串“ SS”，但不匹配“ SS”，并且可能不匹配大写字母“ẞ”（U + 1E9E，拉丁大写字母大写S）即使将小写字母改成小写也可以。

-y是为了兼容性而提供的过时同义词。（ -i由POSIX指定。）

-v
--invert-match

反转匹配感，以选择不匹配的行。（ -v由POSIX指定。）

-w
--word-regexp

仅选择包含构成整个单词的匹配项的行。测试是匹配的子字符串必须位于行的开头，或者必须在非单词组成字符之前。同样，它必须在行的末尾，或后跟非单词组成字符。单词构成字符是字母，数字和下划线。如果以下情况，此选项无效-x也被指定。

因为-w选项可以匹配不以单词组成部分开头和结尾的子字符串，这与用'包围正则表达式不同\< '和' \> '。例如，尽管“ grep -w @ '匹配仅包含'的行@ '，' grep '\<@\>' '无法匹配任何行，因为' @ '不是单词组成。请参见反斜杠字符和特殊表达式 。

-x
--line-regexp

仅选择与整行完全匹配的那些匹配项。对于正则表达式模式，这就像用括号将每个模式括起来，然后用' ^ '和' $ '。（ -x由POSIX指定。）

2.1.3常规输出控制

-c
--count

抑制正常输出；而是为每个输入文件打印匹配行数。随着-v （ --invert-match ）选项，计算不匹配的行。（ -c由POSIX指定。）

--color[=WHEN]
--colour[=WHEN]

用转义序列将匹配的（非空）字符串，匹配行，上下文行，文件名，行号，字节偏移量和分隔符（用于字段和上下文行组）括起来，以在终端上以彩色显示它们。颜色由环境变量定义GREP_COLORS且默认为“ ms=01;31:mc=01;31:sl=:cx=:fn=35:ln=32:bn=32:se=36 '表示粗体红色匹配文本，洋红色文件名，绿色行号，绿色字节偏移量，青色分隔符以及其他默认终端颜色。不推荐使用的环境变量GREP_COLOR仍受支持，但其设置没有优先级；它默认为' 01;31 '（粗体红色），仅覆盖匹配文本的颜色。 WHEN是' never '，' always '， 要么 ' auto '。

-L
--files-without-match

抑制正常输出；而是打印每个输入文件的名称，通常不会从该文件中打印输出。每个文件的扫描在第一个匹配项时停止。

-l
--files-with-matches

抑制正常输出；而是打印通常会从中打印输出的每个输入文件的名称。每个文件的扫描在第一个匹配项时停止。（ -l由POSIX指定。）

-m num
--max-count=num

在第一个之后停止num选定的行。如果输入是来自常规文件的标准输入，并且num选择的行被输出， grep确保标准输入位于退出之前的最后选择的行之后，而不管尾随上下文行的存在。这使呼叫过程可以恢复搜索。例如，以下shell脚本使用了它：

while grep -m 1 PATTERN
do
  echo xxxx
done < FILE

但是以下内容可能不起作用，因为管道不是常规文件：

# This probably will not work.
cat FILE |
while grep -m 1 PATTERN
do
  echo xxxx
done

什么时候grep之后停止num选定的行，它将输出任何尾随的上下文行。当。。。的时候-c要么--count选项也被使用， grep输出的计数不大于num 。当。。。的时候-v要么--invert-match选项也被使用， grep输出后停止num不匹配的行。

-o
--only-matching

仅打印匹配行的匹配（非空）部分，并将每个这样的部分打印在单独的输出行上。输出行使用与输入相同的定界符，如果以下情况，定界符为空字节-z （ --null-data ）也可以使用（请参阅其他选项 ）。

-q
--quiet
--silent

安静;不要在标准输出中写任何东西。如果发现任何匹配项，即使检测到错误，也以零状态立即退出。另请参阅-s要么--no-messages选项。（ -q由POSIX指定。）

-s
--no-messages

禁止显示有关不存在或不可读文件的错误消息。可移植性说明：与GNU不同grep ，第7版Unix grep不符合POSIX，因为它缺少-q及其-s选项的行为类似于GNU grep的-q选项。 1 USG风格grep也缺乏-q但是它-s选项的行为类似于GNU grep的。可移植的shell脚本应该避免两者-q和-s并将标准和错误输出重定向到/dev/null代替。（ -s由POSIX指定。）

2.1.4输出线前缀控制

当要输出几个前缀字段时，顺序始终是文件名，行号和字节偏移，而与这些选项的指定顺序无关。

-b
--byte-offset

在输出的每一行之前，在输入文件中打印基于0的字节偏移量。如果-o （ --only-matching ），则打印出匹配零件本身的偏移量。

-H
--with-filename

打印每个匹配项的文件名。当要搜索多个文件时，这是默认设置。

-h
--no-filename

在输出中禁止文件名的前缀。当仅搜索一个文件（或仅标准输入）时，这是默认设置。

--label=LABEL

显示实际来自标准输入的输入作为来自文件的输入LABEL 。在实施诸如zgrep ;例如：

gzip -cd foo.gz | grep --label=foo -H something
-n
--line-number

在输出的每一行之前，在其输入文件中添加从1开始的行号。（ -n由POSIX指定。）

-T
--initial-tab

确保实际行内容的第一个字符位于制表位上，以便制表符的对齐看起来很正常。这对于将其输出前缀为实际内容的选项很有用： -H ， -n和-b 。这也可以在输出行号和字节偏移量之前加空格，以便单个文件中的行都始于同一列。

-Z
--null

输出零字节（ASCII NUL字符），而不是通常在文件名后的字符。例如， ' grep -lZ '在每个文件名之后输出一个零字节，而不是通常的换行符。即使存在包含诸如换行符之类的不寻常字符的文件名，此选项也可以使输出明确。此选项可与“ find -print0 '，' perl -0 '，' sort -z '和' xargs -0 '处理任意文件名，即使是包含换行符的文件名。

2.1.5上下文线控制

上下文线是匹配线附近的不匹配线。仅当使用以下选项之一时，才输出它们。无论如何设置这些选项， grep从不输出任何给定的行超过一次。如果-o （ --only-matching ）选项被指定，这些选项无效，并且在使用时给出警告。

-A num
--after-context=num

打印num匹配行之后的尾随上下文行。

-B num
--before-context=num

打印num匹配行之前的前导上下文行。

-C num
-num
--context=num

打印num前导和尾随输出上下文行。

--group-separator=string

什么时候-A ， -B要么-C正在使用中，打印string代替--组线之间。

--no-group-separator

什么时候-A ， -B要么-C在使用中，请勿在行组之间打印分隔符。

以下是一些有关如何grep选择要在前缀字段和行内容之间打印的分隔符：

  • 匹配行通常使用' : '作为前缀字段和实际行内容之间的分隔符。
  • 上下文（即不匹配）行使用' -代替。
  • 如果未指定上下文，则仅逐行输出匹配的行。
  • 如果指定了上下文，则输入中相邻的线将组成一个组，并逐个输出，而默认情况下，非相邻组之间会出现分隔符。
  • 默认的分隔符是“ --线可以使用上述选项更改其外观和外观。

2.1.6文件和目录选择

-a
--text

像对待文本一样处理二进制文件；这等效于“ --binary-files=text ' 选项。

--binary-files=type

如果文件的数据或元数据表明该文件包含二进制数据，则假定该文件的类型为type 。非文本字节表示二进制数据；这些是针对当前语言环境未正确编码的输出字节（请参阅环境变量 ），或者是-z （ --null-data ）选项未提供（请参阅其他选项 ）。

默认， type是' binary '和grep发现空输入二进制数据后抑制输出，并抑制包含编码错误的数据的输出行。当某些输出被抑制时， grep在任何输出之后都显示一条单行消息，表明二进制文件匹配。

如果type是' without-match '， 什么时候grep发现空输入二进制数据（假定文件的其余部分不匹配）；这相当于-I选项。

如果type是' text '， grep以二进制形式处理二进制数据；这相当于-a选项。

什么时候type是' binary '， grep可能会将非文本字节视为行终止符，即使没有-z （ --null-data ） 选项。这意味着选择“ binary ' 与 ' text '会影响模式是否与文件匹配。例如，当type是' binary '模式' q$ “可能匹配” q '立即后跟一个空字节，即使在type是' text '。相反，当type是' binary '模式' . '（句点）可能不匹配空字节。

警告： -a （ --binary-files=text ）选项可能会输出二进制垃圾，如果输出是终端并且终端驱动程序将其中的一些解释为命令，则二进制垃圾可能会带来不良影响。另一方面，在读取文本编码未知的文件时，使用它可能会有所帮助-a或设置“ LC_ALL='C'在环境中，即使匹配对直接显示也不安全，也可以找到更多匹配。

-D action
--devices=action

如果输入文件是设备，FIFO或套接字，请使用action处理它。如果action是' read '，所有设备的读取就像它们是普通文件一样。如果action是' skip '，设备，FIFO和套接字都被静默跳过。默认情况下，如果设备在命令行上或如果-R （ --dereference-recursive ）选项，如果递归地遇到它们，则跳过-r （ --recursive ）选项。此选项对通过标准输入读取的文件无效。

-d action
--directories=action

如果输入文件是目录，请使用action处理它。默认， action是' read '，这意味着目录的读取就像它们是普通文件一样（某些操作系统和文件系统不允许这样做，并且会导致grep打印每个目录的错误消息或静默跳过它们）。如果action是' skip '，目录被静默跳过。如果action是' recurse '， grep遵循命令行符号链接并跳过其他符号链接，以递归方式读取每个目录下的所有文件；这相当于-r选项。

--exclude=glob

跳过名称后缀与模式匹配的任何命令行文件glob ，使用通配符匹配；名称后缀可以是全名，也可以是' / '并且在非'之前/ '。递归搜索时，跳过基名称匹配的所有子文件glob ;基本名称是最后一个' / '。模式可以使用' * '，' ? '和' [ '...' ] '作为通配符，并且\逐个引用通配符或反斜杠字符。

--exclude-from=file

跳过名称与读取的任何模式匹配的文件file （使用如下所述的通配符匹配--exclude ）。

--exclude-dir=glob

跳过名称后缀与模式匹配的任何命令行目录glob 。递归搜索时，请跳过基名称匹配的所有子目录glob 。忽略任何多余的斜杠glob 。

-I

处理二进制文件，就好像它不包含匹配的数据一样；这等效于“ --binary-files=without-match ' 选项。

--include=glob

仅搜索名称匹配的文件glob ，使用如下所述的通配符匹配--exclude 。

-r
--recursive

对于每个目录操作数，以递归方式读取和处理该目录中的所有文件。遵循命令行上的符号链接，但跳过递归遇到的符号链接。请注意，如果未提供文件操作数，则grep将搜索工作目录。这与“ --directories=recurse ' 选项。

-R
--dereference-recursive

对于每个目录操作数，遵循所有符号链接，递归地读取和处理该目录中的所有文件。

2.1.7其他选项

--

分隔选项列表。以后的参数（如果有的话）将被视为操作数，即使它们以'开头- '。例如， ' grep PAT -- -file1 file2 '在名为的文件中搜索模式PAT -file1和file2 。

--line-buffered

在输出上使用行缓冲。这可能会导致性能下降。

-U
--binary

在区分文本和二进制I / O的平台上，在读取和写入用户终端以外的文件时，请使用后者，以便按原样读取和写入所有输入字节。这将覆盖默认行为，其中grep遵循操作系统的建议是使用文本还是二进制I / O。在MS-Windows上grep使用文本I / O，它读取回车符-换行符对作为换行符，并按Control-Z读取文件末尾，并写入换行符作为回车符-换行符对。

使用文字I / O时--byte-offset （ -b ）计数和--binary-files启发式应用于文本I / O处理后的输入数据。另外， --binary-files启发式方法不必同意--binary选项;也就是说，即使他们将数据视为文本，即使--binary被给予，反之亦然。请参阅文件和目录选择 。

此选项对不区分文本和二进制I / O的GNU和其他POSIX兼容平台无效。

4用法

这是调用GNU的示例命令grep ：

grep -i 'hello.*world' menu.h main.c

这列出了文件中的所有行menu.h和main.c包含字符串“ hello '后跟字符串' world ';这是因为 ' .* '匹配一行中的零个或多个字符。请参阅正则表达式 。的-i选择原因grep忽略大小写，使其与行' Hello, world! ”，否则将无法匹配。有关如何调用的更多详细信息，请参见调用 。 grep 。

以下是一些有关的常见问题和解答grep用法。

  1. 如何只列出匹配文件的名称？ 
    grep -l 'main' *.c

    列出当前目录中所有C文件的名称，其内容中提到“ main '。

  2. 如何递归搜索目录？ 
    grep -r 'hello' /home/gigi

    搜索“ hello在“ /home/gigi目录。要更好地控制搜索哪些文件，请使用find ， grep和xargs 。例如，以下命令仅搜索C文件：

    find /home/gigi -name '*.c' -print0 | xargs -0r grep -H 'hello'

    这与命令不同：

    grep -H 'hello' *.c

    只是寻找“ hello '在当前目录中所有名称以'结尾的文件中.c '。' find ...上面的'命令行与命令更相似：

    grep -rH --include='*.c' 'hello' /home/gigi
  3. 如果图案前导' - '？ 
    grep -e '--cut here--' *

    搜索与“ --cut here-- '。不带-e ， grep会尝试解析“ --cut here-- '作为选项列表。

  4. 假设我要搜索整个单词，而不是单词的一部分？ 
    grep -w 'hello' *

    仅搜索“ hello '就是全部的话；它不符合“ Othello '。要进行更多控制，请使用“ \< '和' \>以匹配单词的开头和结尾。例如：

    grep 'hello\>' *

    仅搜索以“ hello '，因此它与单词' Othello '。

  5. 如何在匹配行周围输出上下文？ 
    grep -C 2 'hello' *

    在每条匹配的线周围打印两行上下文。

  6. 我该如何强迫grep打印文件名？

    附加/dev/null ：

    grep 'eli' /etc/passwd /dev/null

    让您：

    /etc/passwd:eli:x:2098:1000:Eli Smith:/home/eli:/bin/bash

    或者，使用-H ，这是GNU扩展：

    grep -H 'eli' /etc/passwd
  7. 人们为什么在上使用奇怪的正则表达式ps输出？ 
    ps -ef | grep '[c]ron'

    如果将模式写成没有方括号，那么它不仅会匹配ps输出线cron ，而且ps输出线grep 。请注意，在某些平台上， ps将输出限制为屏幕的宽度； grep除可用内存外，对行的长度没有任何限制。

  8. 为什么grep报告“二进制文件匹配”？

    如果grep列出二进制文件中所有匹配的“行”，它可能会生成无用的输出，甚至可能破坏您的显示。所以GNU grep禁止显示看似二进制文件的文件输出。强制GNU grep要甚至从看似二进制的文件中输出行，请使用-a要么 ' --binary-files=text ' 选项。要消除“二进制文件匹配”消息，请使用-I要么 ' --binary-files=without-match ' 选项。

  9. 为什么不grep -lv '打印不匹配的文件名？

    ' grep -lv '列出了包含一个或多个不匹配行的所有文件的名称。要列出不包含匹配行的所有文件的名称，请使用-L要么--files-without-match选项。

  10. 我可以对“ | '，但“ AND”呢？ 
    grep 'paul' /etc/motd | grep 'franc,ois'

    查找同时包含“ paul '和' franc,ois '。

  11. 空模式为什么匹配每个输入行？

    的grep命令搜索包含与模式匹配的字符串的行。每行都包含空字符串，因此空模式会导致grep在每行上找到一个匹配项。这不是唯一的这种模式： ^ '，' $ '，' .* '，以及许多其他模式导致grep匹配每一行。

    要匹配空行，请使用“ ^$ '。要匹配空白行，请使用“ ^[[:blank:]]*$ '。要根本不匹配任何行，请使用命令' grep -f /dev/null '。

  12. 如何在标准输入和文件中搜索？

    使用特殊文件名“ - '：

    cat /etc/passwd | grep 'alain' - /etc/motd
  13. 如何以正则表达式表达回文？

    可以通过使用反向引用来完成；例如，可以用BRE编写4个字符的回文：

    grep -w -e '\(.\)\(.\).\2\1' file

    它与“雷达”或“公民”一词匹配。

    Guglielmo Bondioni提出了一个单一的RE，它使用9个子表达式和9个反向引用 查找长达19个字符的所有回文：

    grep -E -e '^(.?)(.?)(.?)(.?)(.?)(.?)(.?)(.?)(.?).?\9\8\7\6\5\4\3\2\1$' file

    注意，这是通过使用GNU ERE扩展来完成的。它可能无法移植到其他实现grep 。

  14. 为什么此反向引用失败？ 
    echo 'ba' | grep -E '(a)\1|b\1'

    这没有任何输出，因为第一个备用' (a)\1 '不匹配，因为没有' aa在输入中，因此， \1 '在第二个备用项中没有要引用的内容，这意味着它将永远不会匹配任何内容。（此示例中的第二个替代项只有在第一个替代项已匹配的情况下才能匹配，从而使第二个替代项成为多余。）

  15. 如何跨线匹配？

    标准grep无法执行此操作，因为它基本上是基于行的。因此，仅使用[:space:]字符类与您期望的方式不匹配换行符。

    与GNU grep选项-z （ --null-data ），每个输入和输出“行”均以空值结尾；请参阅其他选项 。因此，您可以在输入中匹配换行符，但是通常如果匹配，则输出整个输入，因此此用法通常与输出抑制选项（例如， -q ，例如：

    printf 'foo\nbar\n' | grep -z -q 'foo[[:space:]]\+bar'

    如果这还不够，您可以先将输入转换为grep ，或转到awk ， sed ， perl ，或旨在跨线操作的许多其他实用程序。

    名字grep来自Unix上的行编辑方式。例如， ed使用以下语法在屏幕上打印匹配行的列表：

5表现

通常grep是搜索文本的有效方法。但是，在某些情况下它可能会非常慢，并且它可以搜索大文件，即使对性能进行细微调整也可以显着帮助。虽然使用的算法grep是一个实施细节，可以随版本的不同而变化，了解其基本优点和缺点可以帮助您提高其性能。

的grep命令部分通过一组为提高效率而设计的自动机进行操作，部分通过较慢的匹配器进行操作，当快速匹配器遇到异常功能（例如反向引用）时，较慢的匹配器将接管工作。在可行的情况下，可以使用Boyer-Moore快速字符串搜索算法来匹配单个固定模式，而使用Aho-Corasick算法来匹配多个固定模式。

一般来说grep由于它可以避免多字节字符所需的特殊处理，因此可以在单字节语言环境中更有效地运行。如果您的模式也能正常工作，请设置LC_ALL更改为单字节语言环境可以极大地提高性能。设置 ' LC_ALL='C' '可能特别有效，因为grep针对该语言环境进行了调整。
