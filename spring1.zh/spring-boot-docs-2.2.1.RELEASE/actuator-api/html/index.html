该API文档描述了Spring Boot Actuators Web端点。

1。总览

在继续之前，您应该阅读以下主题：

  • 网址

  • 时间戳记

1.1。网址

默认情况下，所有Web端点在路径下方均可用/actuator带有以下形式的URL /actuator/{id} 。的/actuator基本路径可以通过使用management.endpoints.web.base-path属性，如以下示例所示：

management.endpoints.web.base-path=/manage

前面的application.properties示例将端点URL的形式从/actuator/{id}至/manage/{id} 。例如，URL info端点将变成/manage/info 。

1.2。时间戳记

端点消耗的所有时间戳（作为查询参数或在请求正文中）必须格式化为ISO 8601中指定的偏移日期和时间。

2。审核事件（ auditevents ）

的auditevents端点提供有关应用程序审核事件的信息。

2.1。检索审核事件

要检索审核事件，请GET请求/actuator/auditevents ，如以下基于curl的示例所示：

$ curl 'http://localhost:8080/actuator/auditevents?principal=alice&after=2019-11-06T21%3A32%3A23.405Z&type=logout' -i -X GET

前面的示例检索logout校长活动alice ，该时间发生在UTC时区的2017年11月7日09:37之后。产生的响应类似于以下内容：

HTTP/1.1 200 OK
Content-Type: application/vnd.spring-boot.actuator.v3+json
Content-Length: 121

{
  "events" : [ {
    "timestamp" : "2019-11-06T21:32:23.407Z",
    "principal" : "alice",
    "type" : "logout"
  } ]
}

2.1.1。查询参数

端点使用查询参数来限制其返回的事件。下表显示了受支持的查询参数：

参数 描述

after

将事件限制为在给定时间之后发生的事件。可选的。

principal

将事件限制为具有给定主体的事件。可选的。

type

将事件限制为具有给定类型的事件。可选的。

2.1.2。回应结构

该响应包含与查询匹配的所有审核事件的详细信息。下表描述了响应的结构：

路径 类型 描述

events

Array

一系列审计事件。

events.[].timestamp

String

事件发生的时间戳。

events.[].principal

String

触发事件的主体。

events.[].type

String

事件的类型。

4。缓存（ caches ）

的caches端点提供对应用程序缓存的访问。

4.1。检索所有缓存

要检索应用程序的缓存，请创建一个GET请求/actuator/caches ，如以下基于curl的示例所示：

$ curl 'http://localhost:8080/actuator/caches' -i -X GET

产生的响应类似于以下内容：

HTTP/1.1 200 OK
Content-Type: application/vnd.spring-boot.actuator.v3+json
Content-Length: 435

{
  "cacheManagers" : {
    "anotherCacheManager" : {
      "caches" : {
        "countries" : {
          "target" : "java.util.concurrent.ConcurrentHashMap"
        }
      }
    },
    "cacheManager" : {
      "caches" : {
        "cities" : {
          "target" : "java.util.concurrent.ConcurrentHashMap"
        },
        "countries" : {
          "target" : "java.util.concurrent.ConcurrentHashMap"
        }
      }
    }
  }
}

4.1.1。回应结构

该响应包含应用程序缓存的详细信息。下表描述了响应的结构：

路径 类型 描述

cacheManagers

Object

缓存管理器，由ID键控。

cacheManagers.*.caches

Object

按名称键入的应用程序上下文中的缓存。

cacheManagers.*.caches.*.target

String

本机缓存的全限定名称。

4.2。按名称检索缓存

要按名称检索缓存，请创建一个GET请求/actuator/caches/{name} ，如以下基于curl的示例所示：

$ curl 'http://localhost:8080/actuator/caches/cities' -i -X GET

前面的示例检索有关名为的缓存的信息cities 。产生的响应类似于以下内容：

HTTP/1.1 200 OK
Content-Type: application/vnd.spring-boot.actuator.v3+json
Content-Length: 113

{
  "target" : "java.util.concurrent.ConcurrentHashMap",
  "name" : "cities",
  "cacheManager" : "cacheManager"
}

4.2.1。查询参数

如果请求的名称足够特定以标识单个高速缓存，则不需要额外的参数。否则， cacheManager必须指定。下表显示了受支持的查询参数：

参数 描述

cacheManager

用于限定高速缓存的cacheManager的名称。如果缓存名称是唯一的，则可以省略。

4.2.2。回应结构

该响应包含请求的缓存的详细信息。下表描述了响应的结构：

路径 类型 描述

name

String

缓存名称。

cacheManager

String

缓存管理器名称。

target

String

本机缓存的全限定名称。

4.3。驱逐所有缓存

要清除所有可用的缓存，请执行DELETE请求/actuator/caches如以下基于curl的示例所示：

$ curl 'http://localhost:8080/actuator/caches' -i -X DELETE

4.4。按名称逐出缓存

要驱逐特定的缓存，请DELETE请求/actuator/caches/{name}如以下基于curl的示例所示：

$ curl 'http://localhost:8080/actuator/caches/countries?cacheManager=anotherCacheManager' -i -X DELETE
由于有两个缓存命名countries ， cacheManager必须提供以指定Cache应该清除。

4.4.1。请求结构

如果请求的名称足够特定以标识单个高速缓存，则不需要额外的参数。否则， cacheManager必须指定。下表显示了受支持的查询参数：

参数 描述

用于限定高速缓存的cacheManager的名称。如果缓存名称是唯一的，则可以省略。
