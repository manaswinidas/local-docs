可以使用systemPropertyVariables属性。以下套property1至test和property2至42：

<project>
  ...
  <build>
    <properties>
      <my.value>42</my.value>
    </properties>
    ...
    <plugins>
      ...
      <plugin>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-maven-plugin</artifactId>
        <version>2.2.1.RELEASE</version>
        <configuration>
          <systemPropertyVariables>
            <property1>test</property1>
            <property2>${my.value}</property2>
          </systemPropertyVariables>
        </configuration>
        ...
      </plugin>
      ...
    </plugins>
    ...
  </build>
  ...
</project>

如果该值为空或未定义（即 >），将系统属性设置为空String。Maven修剪pom中指定的值，因此无法通过此机制指定需要以空格开头或结尾的System属性：考虑使用jvmArguments代替。

任何String类型的Maven变量都可以作为系统属性传递。尝试传递任何其他Maven变量类型（例如List或一个URL变量）将导致按原义传递变量表达式（未求值）。

的jvmArguments参数优先于上述机制定义的系统属性。在以下示例中， property1是overridden ：

mvn spring-boot:run -Dspring-boot.run.jvmArguments="-Dproperty1=overridden"
