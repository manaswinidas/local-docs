Spring Boot使用在另一个jar文件中定义的自定义布局工厂对该项目的jar文件重新打包，该工厂作为对构建插件的依赖关系提供：

<project>
  ...
  <build>
    ...
    <plugins>
      ...
      <plugin>
        <groupId>${project.groupId}</groupId>
        <artifactId>${project.artifactId}</artifactId>
        <version>${project.version}</version>
        <executions>
          <execution>
            <id>repackage</id>
            <goals>
              <goal>repackage</goal>
            </goals>
            <configuration>
              <layoutFactory implementation="com.example.CustomLayoutFactory">
                <customProperty>value</customProperty>
              </layoutFactory>
            </configuration>
          </execution>
        </executions>
        <dependencies>
          <dependency>
            <groupId>com.example</groupId>
            <artifactId>custom-layout</artifactId>
            <version>0.0.1.BUILD-SNAPSHOT</version>
          </dependency>
        </dependencies>
        ...
      </plugin>
      ...
    </plugins>
    ...
  </build>
  ...
</project>

布局工厂是作为以下实现提供的LayoutFactory （从spring-boot-loader-tools中）在pom中显式指定。如果只有一个习惯LayoutFactory在插件的类路径上，它列在META-INF/spring.factories那么就无需在插件配置中显式设置它。

如果设置了显式布局，则总是会忽略布局工厂。
