Spring Boot CLI是一个命令行工具，如果您想快速开发Spring应用程序，可以使用它。它使您可以运行Groovy脚本，这意味着您具有类似Java的熟悉语法，而没有太多样板代码。您还可以引导一个新项目或为其编写自己的命令。

1。安装CLI

可以使用SDKMAN手动安装Spring Boot CLI（命令行界面）！（SDK Manager）或使用Homebrew或MacPorts（如果您是OSX用户）。有关全面的安装说明，请参见“入门”部分中的getting-started.html 。

一旦安装了CLI，就可以通过键入以下内容来运行它spring然后在命令行中按Enter。如果你跑spring没有任何参数，将显示一个简单的帮助屏幕，如下所示：

2.1。使用CLI运行应用程序

您可以使用以下命令编译并运行Groovy源代码run命令。Spring Boot CLI是完全独立的，因此您不需要任何外部Groovy安装。

以下示例显示了用Groovy编写的“ hello world” Web应用程序：

你好。
@RestController
class WebApplication {

    @RequestMapping("/")
    String home() {
        "Hello World!"
    }

}

要编译并运行该应用程序，请键入以下命令：

$ spring run hello.groovy

要将命令行参数传递给应用程序，请使用--将命令与“ spring”命令参数分开，如以下示例所示：

$ spring run hello.groovy -- --server.port=9000

要设置JVM命令行参数，可以使用JAVA_OPTS环境变量，如以下示例所示：

$ JAVA_OPTS=-Xmx1024m spring run hello.groovy
设定时JAVA_OPTS在Microsoft Windows上，请确保引用整个指令，例如set "JAVA_OPTS=-Xms256m -Xmx2048m" 。这样做可以确保将值正确传递给流程。

2.1.1。推导的“抓取”依赖性

Spring Boot进一步扩展了该技术，并尝试根据您的代码推断出哪些库可以“抢”。例如，由于WebApplication先前显示的代码使用@RestController注解，Spring Boot会获取“ Tomcat”和“ Spring MVC”。

2.1.2。推导“抓取”坐标

Spring Boot扩展了Groovy的标准@Grab通过让您指定不带组或版本的依赖项（例如， @Grab('freemarker') ）。这样做可以参考Spring Boot的默认依赖元数据来推断工件的组和版本。

默认元数据与您使用的CLI版本相关。仅当您移至新版本的CLI时，它才会更改，从而使您可以控制依赖项的版本何时更改。在附录中可以找到一张表格，其中显示了默认元数据中包含的依赖关系及其版本。

2.1.3。默认导入语句

为了帮助减少Groovy代码的大小，有几个import语句自动包含在内。注意前面的示例是如何引用的@Component ， @RestController和@RequestMapping无需使用标准名称或import陈述。

许多Spring注释无需使用即可工作import陈述。在添加导入之前，请尝试运行您的应用程序以查看失败的原因。

2.1.4。自动主要方法

与等效的Java应用程序不同，您无需包含public static void main(String[] args)您的方法Groovy脚本。一种SpringApplication是自动创建的，您的编译后的代码将充当source 。

2.1.5。自定义依赖管理

默认情况下，CLI使用在spring-boot-dependencies解决时@Grab依赖性。可以使用以下命令配置其他依赖项管理，该依赖项管理将覆盖默认的依赖项管理。 @DependencyManagementBom注解。注释的值应指定坐标（ groupId:artifactId:version ）的一个或多个Maven BOM。

例如，考虑以下声明：

@DependencyManagementBom("com.example.custom-bom:1.0.0")

前面的声明custom-bom-1.0.0.pom在下面的Maven存储库中com/example/custom-versions/1.0.0/ 。

指定多个BOM时，它们以声明它们的顺序应用，如以下示例所示：

@DependencyManagementBom(["com.example.custom-bom:1.0.0",
        "com.example.another-bom:1.0.0"])

前面的示例表明在another-bom覆盖中的依赖项管理custom-bom 。

您可以使用@DependencyManagementBom您可以使用的任何地方@Grab 。但是，为了确保依赖性管理的顺序一致，可以使用@DependencyManagementBom在您的应用程序中最多一次。

2.2。具有多个源文件的应用程序

您可以对所有接受文件输入的命令使用“ shell globbing”。这样可以使您从单个目录使用多个文件，如以下示例所示：

$ spring run *.groovy

2.3。打包您的应用程序

您可以使用jar命令将您的应用程序打包到一个独立的可执行jar文件中，如以下示例所示：

$ spring jar my-app.jar *.groovy

生成的jar包含通过编译应用程序产生的类以及所有应用程序的依赖项，以便随后可以使用来运行它java -jar 。jar文件还包含来自应用程序的类路径的条目。您可以使用以下命令添加和删除jar的显式路径--include和--exclude 。两者都用逗号分隔，并且都接受前缀“ +”和“-”形式，以表示应将其从默认值中删除。默认包括以下内容：

public/**, resources/**, static/**, templates/**, META-INF/**, *

默认排除项如下：

.*, repository/**, build/**, target/**, **/*.jar, **/*.groovy

类型spring help jar在命令行上了解更多信息。

2.5。使用嵌入式外壳

Spring Boot包含用于BASH和zsh Shell的命令行完成脚本。如果您不使用这两个外壳程序（也许您是Windows用户），则可以使用shell命令以启动集成外壳程序，如以下示例所示：

$ spring shell
Spring Boot (v2.2.1.RELEASE)
Hit TAB to complete. Type \'help' and hit RETURN for help, and \'exit' to quit.

在嵌入式外壳程序内部，您可以直接运行其他命令：

$ version
Spring CLI v2.2.1.RELEASE

嵌入式外壳支持ANSI颜色输出以及tab完成。如果您需要运行本机命令，则可以使用!字首。要退出嵌入式外壳，请按ctrl-c 。

2.6。将扩展添加到CLI

您可以使用以下命令将扩展添加到CLI install命令。该命令采用格式的一组或多组工件坐标group:artifact:version ，如以下示例所示：

$ spring install com.example:spring-boot-cli-extension:1.0.0.RELEASE

除了安装由您提供的坐标标识的工件之外，还将安装所有工件的依赖项。

要卸载依赖项，请使用uninstall命令。与install命令，它采用以下格式的一组或多组工件坐标： group:artifact:version ，如以下示例所示：

$ spring uninstall com.example:spring-boot-cli-extension:1.0.0.RELEASE

它将卸载由您提供的坐标及其依赖项标识的工件。

要卸载所有其他依赖项，可以使用--all选项，如以下示例所示：

$ spring uninstall --all

3。使用Groovy Beans DSL开发应用程序

Spring Framework 4.0原生支持beans{} “ DSL”（从Grails借来的），您可以使用相同的格式将bean定义嵌入Groovy应用程序脚本中。有时，这是包括外部功能（如中间件声明）的好方法，如以下示例所示：

@Configuration(proxyBeanMethods = false)
class Application implements CommandLineRunner {

    @Autowired
    SharedService service

    @Override
    void run(String... args) {
        println service.message
    }

}

import my.company.SharedService

beans {
    service(SharedService) {
        message = "Hello World"
    }
}

您可以将类声明与beans{}只要它们位于顶层，就可以放在同一文件中；或者，如果愿意，可以将bean DSL放在单独的文件中。

Spring Boot CLI使用Maven的依赖关系解析引擎Aether来解决依赖关系。CLI利用了在~/.m2/settings.xml配置以太。CLI遵循以下配置设置：

  • 后视镜

  • 伺服器

  • 代理人

有关更多信息，请参见Maven的设置文档 。
