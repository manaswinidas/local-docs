    共享类缓存诊断实用程序

    这些实用程序显示有关共享类缓存的内容的信息。通过将实用程序指定为以下选项的子选项来运行它们-Xshareclasses 。除非您通过添加以下命令指定缓存，否则这些实用程序将在默认缓存上运行name=子选项。

    printAllStats

    -Xshareclasses:printAllStats
-Xshareclasses:printAllStats,name=<cache_name>

    按时间顺序显示缓存的内容。您可以使用此输出来查看对缓存进行的更新的历史记录。对于分层缓存，某些信息仅针对顶层缓存显示，而某些信息针对所有组合的图层显示。要仅查看顶层缓存的信息，请使用printTopLayerStats=all 。

    输出中的每个条目都以VM ID开头，因此您可以查看哪个VM写入了相关数据。这是各种类型的缓存数据的示例条目，并带有说明：

    类路径

    此示例显示了一个包含4个条目的类路径：

    1: 0x2234FA6C CLASSPATH
        /myVM/Apps/application1.jar
        /myVM/Apps/application2.jar
        /myVM/Apps/application3.jar
        /myVM/Apps/application4.jar
    • 1 ：写入此数据的VM的ID。
    • 0x2234FA6C ：此数据的存储地址。
    • CLASSPATH ：写入的数据类型。

    ROM类

    此示例显示单个条目ROMClass ：

    1: 0x2234F7DC ROMCLASS: java/lang/Runnable at 0x213684A8
        Index 1 in class path 0x2234FA6C
    • 1 ：写入此数据的VM的ID。
    • 0x2234F7DC ：存储有关类的元数据的地址。
    • ROMCLASS ：存储的数据类型。
    • java/lang/Runnable ：类的名称。
    • 0x213684A8 ：存储该类的地址。
    • Index 1 ：从中加载类的类路径中的索引。
    • 0x2234FA6C ：存储该类的类路径的地址。

    过时的类标记为!STALE! 。还显示了存储类时使用的任何分区或修改上下文。

    AOT方法

    本示例显示一个AOT编译方法的条目：

    1: 0x540FBA6A AOT: loadConvert
        for ROMClass java/util/Properties at 0x52345174
    • 1 ：写入此数据的VM的ID。
    • 0x540FBA6A ：数据存储的地址。
    • AOT ：存储的数据类型。
    • loadConvert ：存储AOT编译代码的方法。
    • java/util/Properties ：包含方法的类。
    • 0x52345174 ：包含方法的类的地址。

    过时的方法标有!STALE! 。

    网址和令牌

    一种Token是传递给Java™的字符串SharedClassHelper API。这些数据类型的输出与类路径具有相同的格式，但只有一个条目。

    邮编条目缓存

    此示例显示了zip条目缓存的4个单独的条目：

    1: 0x042FE07C ZIPCACHE: luni-kernel.jar_347075_1272300300_1 Address: 0x042FE094 Size: 7898
1: 0x042FA878 ZIPCACHE: luni.jar_598904_1272300546_1 Address: 0x042FA890 Size: 14195
1: 0x042F71F8 ZIPCACHE: nio.jar_405359_1272300546_1 Address: 0x042F7210 Size: 13808
1: 0x042F6D58 ZIPCACHE: annotation.jar_13417_1272300554_1 Address: 0x042F6D70 Size: 1023
    • 1 ：写入此数据的VM的ID。
    • 0x042FE07C ：用于存储zip条目缓存的元数据的地址。
    • ZIPCACHE ：存储的数据类型。
    • luni-kernel.jar_347075_1272300300_1 ：zip条目缓存的名称。
    • 0x042FE094 ：数据存储的地址。
    • 7898 ：存储的数据大小，以字节为单位。

    JIT数据

    有关JIT数据的信息显示在JITPROFILE和JITHINT条目。例如：
    1: 0xD6290368 JITPROFILE: getKeyHash Signature: ()I Address: 0xD55118C0
for ROMClass java/util/Hashtable$Entry at 0xD5511640.
2: 0xD6283848 JITHINT: loadClass Signature: (Ljava/lang/String;)Ljava/lang/Class; Address: 0xD5558F98
for ROMClass com/ibm/oti/vm/BootstrapClassLoader at 0xD5558AE0.

    启动提示

    有关启动提示的信息显示在STARTUP HINTS KEY和STARTUP HINTS DETAIL 。例如：
    1: 0x000000002237C6E0 STARTUP HINTS KEY: -Xoptionsfile=jre\bin\compressedrefs\options.default -Xlockword:mode=default -Xjcl:jclse29 -Dcom.ibm.oti.vm.bootstrap.library.path=jre\bin\compressedrefs;jre\bin -Djava.home=jre -Djava.ext.dirs=jre\lib\ext -Duser.dir=bin -Djava.class.path=. -Dsun.java.launcher=SUN_STANDARD Address: 0x000000002237C700 Size: 96
STARTUP HINTS DETAIL Flags: 1 DATA1: 1677721 DATA2: 5033165

    printStats

    -Xshareclasses:printStats
-Xshareclasses:printStats,name=<cache_name>
-Xshareclasses:printStats=<data_type1>[+<data_type2>][...],name=<cache_name>

    显示有关缓存的摘要信息。对于分层缓存， -Xshareclasses:printStats显示有关顶层缓存的一些信息，以及有关所有合并层的摘要信息（仅字节和计数）。要仅查看顶层缓存的信息，请使用printTopLayerStats 。

    您可以使用以下方法请求有关存储在共享缓存中的特定数据类型的项目的更多详细信息： printStats= 。使用加号（+）分隔数据类型。例如使用printStats=romclass+url,name=myCache查看有关的信息ROMClass和URL缓存中所有层缓存中的项目称为Cache1 。有效数据类型如下（不区分大小写）：

    • help （显示有效数据类型的列表）
    • all （相当于printAllStats ）
    • classpath
    • url
    • token
    • romclass
    • rommethod
    • aot
    • jitprofile
    • jithint
    • zipcache
    • stale
    • startuphint

    然后使用以下消息之一显示缓存中类的行号信息：

    • Cache contains only classes with line numbers ：已为该共享缓存中存储的所有类启用VM行号处理（ -Xlinenumbers选项默认情况下处于启用状态）。如果原始类包含行号数据，则缓存中的所有类都包含行号。
    • Cache contains only classes without line numbers ： -Xnolinenumbers选项用于禁用此共享缓存中存储的所有类的VM行号处理，因此所有类都不包含行号。

    显示以下摘要数据：

    baseAddress和endAddress

    包含类的共享内存区域的边界地址。

    allocation pointer

    地址在哪里ROMClass当前正在高速缓存中分配数据。

    cache layer

    缓存统计信息所涉及的层号。

    cache size和free bytes

    cache size显示共享内存区域的总大小（以字节为单位），并且free bytes显示剩余的可用字节。

    softmx bytes

    缓存的软最大大小。有关更多信息，请参见-Xscmx 。

    ROMClass bytes

    高速缓存中类数据的字节数。

    AOT bytes

    高速缓存中AOT编译的代码的字节数。

    Reserved space for AOT bytes

    缓存中为AOT编译的代码保留的字节数。

    Maximum space for AOT bytes

    可以在缓存中存储的AOT编译代码的最大字节数。

    JIT data bytes

    缓存中存储的与JIT相关的数据的字节数。

    Reserved space for JIT data bytes

    缓存中为与JIT相关的数据保留的字节数。

    Maximum space for JIT data bytes

    可以在高速缓存中存储的与JIT相关的数据的最大字节数。

    Zip cache bytes

    存储在高速缓存中的zip条目高速缓存字节数。

    Startup hint bytes

    存储的用于描述启动提示的数据字节数。

    Data bytes

    VM存储的非类数据的字节数。

    Metadata bytes

    存储以描述高速缓存的类的数据的字节数。注意：此字段仅在顶层缓存输出中或未分层缓存时可用。

    Metadata % used

    元数据字节与类字节的比例，指示使用缓存空间的效率。显示的值确实考虑了Class debug area size 。

    Class debug area size

    类调试区域的大小（以字节为单位）。该区域保留用于存储LineNumberTable和LocalVariableTable类属性信息。

    Class debug area bytes used

    包含数据的类调试区域的大小（以字节为单位）。

    Class debug area % used

    包含数据的“类调试区域”的百分比。

    ROMClasses

    缓存中的类数。缓存存储ROMClasses （类数据本身，它是只读的）以及有关从中加载类的位置的信息。该信息以不同的方式存储，具体取决于Java SharedClassHelper用于存储类的API。有关更多信息，请参阅《 使用Java Helper API》 。

    AOT methods

    （可选） ROMClass可以编译方法并将AOT代码存储在缓存中。的AOT methods信息显示缓存中已为其编译AOT代码的方法的总数。此数字包括用于陈旧类的AOT代码。

    Classpaths ， URLs和Tokens

    缓存中的类路径，URL和令牌的数量。从一个存储的类SharedClassURLClasspathHelper与类路径一起存储。使用SharedClassURLHelper与URL存储在一起。使用SharedClassTokenHelper与令牌一起存储。大多数类加载器（包括引导程序和应用程序类加载器）都使用SharedClassURLClasspathHelper 。结果是最常见的是在缓存中看到类路径。
    存储的类路径，URL和令牌的数量由许多因素决定。例如，每次更新Classpath的元素时，例如.jar重建文件，将新的类路径添加到缓存。此外，如果使用分区或修改上下文，则它们与类路径，URL或令牌关联。为分区和修改上下文的每个唯一组合存储一个Classpath，URL或Token。有关更多信息，请参见SharedClassHelper分区和Modification上下文 。

    Zip caches

    条目缓存存储在共享缓存中的.zip文件数。

    Startup hints

    缓存中存储的启动提示数。用于启动JVM的每组唯一的命令行选项都有一个启动提示。

    Stale classes

    由于VM或Java应用程序更新，已被缓存代码标记为“可能过期”的类的数量。请参阅了解动态更新 。

    % Stale classes

    缓存中过期的类的百分比。

    Cache is XXX% full

    当前使用的缓存百分比。仅当未设置最大软尺寸时，才显示此行。该值的计算如下：
    % Full = (('Cache Size' - 'Free Bytes') * 100) / ('Cache Size')

    Cache is XXX% soft full

    当前使用的最大软尺寸的百分比。仅当设置了最大软尺寸时，才显示此行。的free bytes高速缓存中的统计信息表示软最大限制内的可用字节。该值的计算如下：
    % soft Full = (('Soft max bytes' - 'Free Bytes') * 100) / ('Soft max  bytes')

    有关最大软尺寸的更多信息，请参见-Xscmx 。

    Cache is accessible to current user

    当前用户是否可以访问缓存。

    printTopLayerStats

    将此实用程序与分层缓存一起使用。该实用程序的工作方式与printStats 。默认情况下，此实用程序显示有关顶层缓存的信息。要查看特定图层的统计信息，请使用layer=选项。例如，要显示2层缓存中第二层的统计信息，请运行printTopLayerStats,layer=1 。
