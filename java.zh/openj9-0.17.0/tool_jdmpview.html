    转储查看器（ jdmpview ）

    转储查看器是一个命令行工具，可让您检查从OpenJ9 VM生成的系统转储的内容。转储查看器使您可以从生成转储的时间开始查看Java™和本机信息。

    对于长时间运行的任务，转储查看器也可以以批处理模式运行。

    转储查看器对于诊断很有用OutOfMemoryError Java™应用程序中的异常。对于常规保护故障（GPF），系统异常终止和SIGSEV等问题，系统调试器（例如gdb （Linux））提供了更多信息。

    句法

    启动转储查看器

    jdmpview (-core | -zip ) [-notemp]

    输入选项 说明
    -core 指定转储文件。
    -zip 指定包含核心文件和关联的XML文件的压缩文件（由转储提取器生成（ jextract ）工具（在AIX®和Linux™系统上）。
    -notemp 默认情况下，当您使用-zip选项，将内容提取到临时目录，然后进行处理。使用-notemp选项以防止执行此提取步骤，并在内存中运行所有后续命令。

    注意： -core选项可以与-zip选项以在压缩文件中指定核心文件和XML文件。没有这些选项， jdmpview显示了多个上下文，一个上下文对应于它在压缩文件中标识的每个源文件。

    在z /OS®上，您可以将转储复制到HFS文件并将其作为输入提供给jdmpview ，或者您可以提供标准的MVS™数据集名称。例如：

    > jdmpview -core USER1.JVM.TDUMP.SSHD6.D070430.T092211
DTFJView version 4.28.3, using DTFJ version 1.11.28004
Loading image from DTFJ...

    MVS数据集名称可能包含美元符号（$）。包含美元符号的名称必须用单引号（'）引起来。例如：

    > jdmpview -core 'USER1.JVM.$TDUMP.SSH$D7.D141211.T045506'

    后jdmpview处理转储文件，会话开始，显示以下消息：

    For a list of commands, type "help"; for how to use "help", type "help help"
>

    如果您运行jdmpview在包含多个转储的压缩文件中使用该工具，该工具会检测并显示所有转储文件，无论这些文件是系统转储，Java转储还是堆转储。由于这种现象，当您开始时可能会显示多个上下文jdmpview 。要切换上下文，请键入context ，在哪里是您要调查的转储的上下文值。

    在z / OS上，系统转储可以包含多个地址空间，而一个地址空间可以包含多个VM实例。在这种情况下，上下文允许您选择转储文件中的地址空间和VM实例。以下z / OS示例显示了地址空间（ ASID ），其中两个JVM占用地址空间0x73 （上下文5和6）。当前上下文为5（ CTX:5> ），并带有星号。要在上下文6中查看JVM，可以通过指定进行切换context 6 。

    CTX:5> context
Available contexts (* = currently selected context) :

0 : ASID: 0x1 : No JRE : No JRE
1 : ASID: 0x3 : No JRE : No JRE
2 : ASID: 0x4 : No JRE : No JRE
3 : ASID: 0x6 : No JRE : No JRE
4 : ASID: 0x7 : No JRE : No JRE
*5 : ASID: 0x73 EDB: 0x83d2053a0 : JRE 1.8.0 z/OS s390x-64 build 20181117_128845 (pmz6480-20181120_01)
6 : ASID: 0x73 EDB: 0x8004053a0 : JRE 1.8.0 z/OS s390x-64 build 20181117_128845 (pmz6480-20181120_01)
7 : ASID: 0x73 EDB: 0x4a7bd9e8 : No JRE
8 : ASID: 0xffff : No JRE : No JRE

    如果您正在使用jdmpview要查看Java转储和堆转储，某些选项不会产生任何输出。例如，堆转储不包含info system命令，但确实包含info class命令。

    如果您正在查看堆上有大量对象的转储，则可以提高性能。 jdmpview确保您的系统有足够的可用内存，并且不需要将内存分页到磁盘。为此，请开始jdmpview通过指定-Xmx选项。使用-J通过选项-Xmx JVM的命令行选项。例如：

    jdmpview -J-Xmx<n> -core <core file>

    会话参数下显示了转储查看器会话可用的选项

    以批处理模式启动

    对于长期运行或日常工作， jdmpview可以在批处理模式下使用。

    您可以通过在命令末尾附加命令来运行单个命令而无需指定命令文件。 jdmpview命令行。例如：

    jdmpview -core mycore.dmp info class

    指定接受通配符参数的jdmpview命令时，必须将通配符替换为ALL防止外壳解释通配符。例如，在交互模式下，命令info thread *必须通过以下方式指定：

    jdmpview -core mycore.dmp info thread ALL

    批处理模式由以下命令行选项控制：

    选项 说明
    -cmd文件 包含一系列jdmpview命令的文件，这些命令将按顺序读取和运行。
    -字符集 -cmdfile中指定的命令的字符集（名称必须是java.nio.charset中定义的受支持字符集。字符集。例如，US-ASCII）
    -输出文件 记录命令生成的任何输出的文件。
    -覆盖 如果-outfile中指定的文件存在，则此选项将覆盖该文件。
    -附加 如果-outfile中指定的文件存在，则新的输出消息将附加到该文件的末尾。-append和-overwrite选项不能同时使用。

    命令文件可以有包含空格的空行，或者以//或＃开头的注释行。这些行被jdmpview忽略。示例命令文件：

    // commands.txt
info system
info proc

    要以批处理方式运行jdmpview，请使用以下命令文件，指定：

    jdmpview -outfile out.txt [-overwrite|-append] -cmdfile commands.txt -core <path to core file>

    当输出文件存在时，您需要指定-overwrite选项或-append选项。否则，将生成错误消息。

    处理输出

    您可以将命令输出重定向到文件，或将命令输出通过管道传输到另一个命令。

    要将jdmpview命令输出重定向到文件，请使用以下格式之一：

    command > <target_file>

    如果目标文件存在，则此重定向将覆盖其中的内容。

    command >> <target_file>

    如果目标文件存在，则此重定向会将输出附加到目标文件。

    哪里是文件名，可以包括文件的完整路径。

    管道jdmpview命令输出到另一个命令，请使用竖线（|）字符。例如：

    command | grep string

    您可以使用多个竖线字符将两个以上的命令链接在一起。

    以下命令可用于询问输出：

    • charsFrom
    • charsTo
    • grep
    • tokens

    使用CharsFrom

    使用charsFrom竖线字符后的命令，以排除结果行中指定模式之前的所有字符。

    charsFrom <options> pattern

    哪里 ：

    • -e要么-exclude ：从结果行中排除匹配的模式。默认情况下，匹配的模式包含在结果行中。
    • -keep ：保留不包含与模式匹配项的行。默认情况下，不包含匹配项的行将从结果中排除。
    • -i要么-ignoreCase ：忽略大小写。

    例如，以下命令显示包含模式的结果行jre ，并修剪每行以排除此模式之前的所有字符：

    > info mod | charsFrom jre
jre/lib/ppc64/libzip.so @ 0x0, sections:
jre/lib/ppc64/libdbgwrapper80.so @ 0x0, sections:
jre/lib/ppc64/libverify.so @ 0x0, sections:
jre/lib/ppc64/libjava.so @ 0x0, sections:
jre/lib/ppc64/compressedrefs/libjclse7b_28.so @ 0x0, sections:

    使用CharsTo

    使用CharsTo竖线字符后的命令，以将字符包括在结果行中，直到找到特定图案为止。

    charsTo <options> pattern

    哪里 ：

    • -include ：在结果行中包含匹配的模式。默认情况下，匹配的模式将从结果行中排除。
    • -keep ：保留不包含与模式匹配项的行。默认情况下，不包含匹配项的行将从结果中排除。
    • -i要么-ignoreCase ：忽略大小写。

    例如，以下命令显示包含模式“ @”的行，并修剪每行以从“ @”开始排除所有字符：

    > info mod | charsTo @
bin/java
/usr/lib64/gconv/UTF-16.so
/test/sdk/lib/ppc64le/libnet.so
/test/sdk/lib/ppc64le/libnio.so
/test/sdk/lib/ppc64le/libzip.so
/test/sdk/lib/ppc64le/libjsig.so
libjsig.so

    您也可以使用charsFrom和charsTo一起，以竖线字符分隔。例如，以下命令显示包含模式“ lib”的行，并修剪每行以排除此模式之前的所有字符以及该模式中的所有字符@ ：

    > info mod | charsFrom lib | charsTo @
lib/ppc64le/libzip.so
lib/ppc64le/libjsig.so
lib/ppc64le/libverify.so
lib/ppc64le/libjava.so
lib/ppc64le/compressedrefs/libj9jit29.so

    注意：如果charsFrom和charsTo一起使用，但只有一个模式在一行中匹配。此外，如果两个模式都在一行中匹配，则不会显示该行，但是charsTo模式出现在之前而不是之后charsFrom图案。

    使用grep

    使用grep竖线字符后的命令，以显示哪些行与指定的模式匹配。

    grep <options> pattern

    哪里 ：

    • -i ：忽略大小写。
    • -r ， -G ， 要么--regex注意：请使用java.utils.regex的Java文档中定义的正则表达式。模式类。
    • -b要么--block ：显示其中至少有一条线与图案匹配的线段。行块之间用空行分隔。
    • -A 或+ ：显示匹配行之后的最多行。例如grep -A 2模式或grep +2模式。
    • -B 或- ：最多显示匹配行之前的行。
    • -C 或+- ：显示匹配行之前和之后的最多行。
    • -v要么--invert-match ：与grep命令一起使用以显示与模式不匹配的行。这些选项等效于grep-命令。
    • -F要么--fixed-strings ：请勿将星号（*）视为通配符。将这些选项与-r ， -G ， 要么--regex选项。

    模式规则：

    • 模式中的星号（*）被视为通配符，除非您指定-F要么--fixed-strings选项。
    • 如果模式包含空格，则将模式用一对双引号（“）引起来。
    • 如果模式包含双引号，则将模式括在一对单引号（'）中。

    • 您可以使用以下格式指定多个子模式以进行匹配，但前提是您不使用-r ， -G ， 要么--regex选项：

      “ [pattern1 | pattern2 | ... | patternN]”

      需要在首尾加上双引号和括号（[]）。使用竖线字符分隔子图案。子模式中不允许使用引号和竖线。子模式的中间允许有空格，但前导和尾随空格将被修剪。

    • 使用grep命令以显示与模式不匹配的行。

    例如，以下命令显示实例的数量和堆的总大小java/lang/String类：

    > info class | grep java/lang/String              
94 7688 [Ljava/lang/String;            
1822 58304 java/lang/String               
1 16 java/lang/String$CaseInsensitiveComparator               
0 0 java/lang/String$UnsafeHelpers

    以下命令结合使用两个管道来显示实例的数量和堆的总大小java/lang/StringCoding.StringDecoder类：

    > info class | grep java/lang/String | grep -i decoder
1 48 java/lang/StringCoding$StringDecoder

    使用tokens

    使用tokens竖线字符后的命令，以隔离结果行中的指定标记。

    tokens [options] range[,range][..range]

    您可以按以下格式定义范围：

    • X
    • 设
    • 。

    一组规则适用于以下格式：

    • x或y可以前缀“-”。这意味着x或y从列表末尾开始倒数。例如，ay值-1表示列表中的最后一个标记，而-2表示列表中的倒数第二个标记。
    • x必须表示在y之前或与y相同位置的标记。

    以这种格式，如果省略x，则假定为'1'。如果省略y，则假定为-1。

    设置heapdump

    配置堆转储生成设置。
    set heapdump <options>

    哪里是：

    • phd ：将“堆转储”格式设置为“便携式堆转储”，这是默认设置。
    • txt ：将堆转储格式设置为经典。
    • file ：设置堆转储的目的地。
    • multiplefiles [on|off] ：如果将multifiles设置为on，则系统转储中的每个Java堆都将写入一个单独的文件。如果将multifiles设置为off，则所有Java堆都将写入同一文件。默认为关闭。

    设定记录

        set logging <options>

    配置日志记录设置，开始日志记录或停止日志记录。使用此参数可以将命令结果记录到文件中，其中是：

    • [on|off] ：打开或关闭登录。（默认：关闭）
    • file ：设置要登录的文件。该路径是相对于pwd命令返回的目录的，除非指定了绝对路径。如果在登录时设置了文件，则更改将在下次启动日志时生效。默认情况下未设置。
    • overwrite [on|off] ：打开或关闭指定日志文件的覆盖。禁用覆盖后，日志消息将附加到日志文件。启用覆盖后，设置set logging命令后将覆盖日志文件。（默认：关闭）
    • redirect [on|off] ：打开或关闭重定向到文件的功能，默认设置为off。当日志记录设置为开时：
    • 重定向的on值仅将非错误输出发送到日志文件。

    x / J [ | <0xaddr> ]

    显示有关特定对象或类的所有对象的信息。如果提供后，将显示所有静态字段及其值，然后显示该类的所有对象及其字段和值。如果提供了一个对象地址（以十六进制表示），则不会显示该对象类的静态字段。该对象的其他字段和值连同其地址一起打印。

    注意：此命令将忽略项目传递给它的项目数和单位大小x/命令。

    x / D < 0xaddr >

    在指定的地址上显示整数，并针对此转储文件所在的硬件体系结构进行了调整。例如，文件可能来自大端架构。

    此命令使用项目数量和单位大小传递给它x/命令。

    x / X < 0xaddr >

    显示指定地址的字节的十六进制值，已针对此转储文件所基于的硬件体系结构进行了调整。例如，文件可能来自大端架构。

    注意：此命令使用由项目传递给它的项目数和单位大小x/命令。

    x / K < 0xaddr >

    如果大小是由体系结构的指针大小定义的，则此参数显示内存每个部分的值。从指定地址开始，针对此转储文件所来自的硬件体系结构调整输出。它还会显示一个模块，该模块具有模块部分，并且如果指针指向该模块部分，则该模块与内存中该模块部分的起始位置会有偏移。如果未找到符号，则当指针指向当前地址的4KB（4096字节）的地址时，它将显示“ *”和当前地址的偏移量。尽管此命令可以在内存的任意部分上运行，但在引用堆栈帧的内存部分上可能更有用。要查找线程堆栈帧的内存部分，请使用info thread命令。

    注意：此命令使用由项目传递给它的项目数和单位大小x/命令。
