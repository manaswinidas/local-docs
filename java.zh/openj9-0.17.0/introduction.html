    Eclipse OpenJ9

    OpenJ9是一种高性能，可扩展的Java™虚拟机（VM）实施，完全符合Java虚拟机规范 。

    在运行时，VM解释由Java编译器编译的Java字节码。VM充当语言与底层操作系统和硬件之间的翻译器。Java程序需要特定的VM才能在特定的平台（例如Linux®，z /OS®或Windows™）上运行。

    该参考资料提供有关VM配置和调整选项的信息以及默认设置。

    配置系统

    为了正常运行，必须在操作系统级别上设置某些环境变量。根据您的系统环境，您可能还需要设置其他配置选项，以允许VM利用硬件和操作系统功能。阅读自定义系统以了解有关以下选项的更多信息：

    • 设置PATH和CLASSPATH环境变量。
    • 设置LIBPATH或LD_LIBRARY_PATH环境变量（AIX®和Linux）以指示在哪里可以找到共享库。
    • 在AIX和Linux系统上设置ulimit以确保操作系统为您的应用程序分配足够的资源。
    • 在z / OS系统上设置区域大小，BPXPRM参数和LanguageEnvironment®运行时选项。
    • 如果您的应用程序分配了大量内存并经常访问该内存，则可能要在系统上启用大页面支持。请参阅配置大页内存分配 。
    • 在AIX系统上配置动态LPAR支持。

    注意：在macOS®系统上，必须设置DYLD_LIBRARY_PATH环境变量以指示在哪里可以找到共享库。另外，为确保有足够的资源供您的应用程序使用，您可能需要通过使用以下命令增加系统限制launchctl ， ulimit ， 要么kern. 。有关更多说明，请参阅macOS版本的文档。

    性能调优

    OpenJ9配置为从一组默认选项开始，这些默认选项为具有典型工作负载的Java应用程序提供了最佳的运行时环境。但是，如果您的应用程序是非典型的，则可以通过调整OpenJ9 VM来提高性能。您还可以通过启用硬件功能或在应用程序代码中使用特定的API来提高性能。

    垃圾收集政策

    OpenJ9包含几种垃圾回收策略。要了解有关这些策略以及可以从中受益的应用程序工作负载类型的更多信息，请参阅垃圾回收 。

    班级数据共享

    您可以在运行的虚拟机之间共享类数据，这可以在创建缓存后减少虚拟机的启动时间。有关更多信息，请参见类数据共享 。

    本机数据操作

    如果您的Java应用程序操纵了本机数据，请考虑编写您的应用程序以利用Data Access Accelerator API中的方法。

    云优化

    要提高在容器中运行的应用程序的性能，请尝试设置以下调整选项：

    • 使用共享的类缓存（ -Xshareclasses -XX:SharedCacheHardLimit=200m -Xscmx60m ）与提前（AOT）编译，以缩短启动时间。有关更多信息，请参见类数据共享和AOT编译器 。
    • 使用-Xtune：virtualized选项，该选项将OpenJ9配置为典型的云部署，在该典型的云部署中，向VM guest虚拟机配置少量虚拟CPU，以最大程度地运行可运行的应用程序。启用后，OpenJ9会调整其内部进程以减少CPU消耗量并减少内存占用。这些更改的代价只是吞吐量损失很小。

    OpenJ9 VM会自动检测它何时在docker容器中运行，并使用一种机制来检测VM何时处于空闲状态。当检测到空闲状态时，OpenJ9会运行垃圾回收周期，并将可用内存页释放回操作系统。还压缩对象堆，以充分利用可用内存来进行进一步的应用程序处理。对于基于内存使用量收费的云服务，保持较小的占用空间可以节省成本。有关控制该过程的调整选项的更多信息，请参见-XX：IdleTuningMinIdleWaitTime 。

    加密操作

    OpenJDK默认使用内置的Java加密实现。但是，本机加密实现通常可提供更好的性能。OpenSSL是用于传输层安全性（TLS）和安全套接字层（SSL）协议的本地开放源代码加密工具包，它已经很好地建立并与许多企业应用程序一起使用。Digest，CBC，GCM和RSA算法当前支持OpenSSL V1.0.x和V1.1.x实现。ChaCha20和ChaCha20-Poly1305算法也支持OpenSSL V1.1.x实现。

    在Linux和AIX平台上，应该在系统路径上找到OpenSSL 1.0.x或1.1.x库。如果使用程序包管理器安装OpenSSL，系统路径将自动更新。在其他平台上，OpenSSL 1.1.x库当前与AdoptOpenJDK的二进制文件捆绑在一起。

    默认情况下，所有支持的算法均启用OpenSSL支持。如果要限制对特定算法的支持，则可以使用许多系统属性来调整实现。

    通过在命令行上设置以下系统属性，可以分别禁用每种算法：

    • 要关闭摘要 ，请设置-Djdk.nativeDigest=false
    • 要关闭ChaCha20和ChaCha20-Poly1305 ，请设置-Djdk.nativeChaCha20=false 。 注意： Java 8不支持这些算法
    • 要关闭CBC ，请设置-Djdk.nativeCBC=false
    • 要关闭GCM ，请设置-Djdk.nativeGCM=false
    • 要关闭RSA ，请设置-Djdk.nativeRSA=false

    您可以通过在命令行上设置以下系统属性来关闭所有算法：

    -Djdk.nativeCrypto=false

    要使用OpenJ9构建包含OpenSSL支持的OpenJDK版本，请遵循我们详细的构建说明中的步骤：

    • 具有OpenJ9的OpenJDK 8 。
    • 具有OpenJ9的OpenJDK 11 。
    • 具有OpenJ9的OpenJDK 13 。

    注意：如果你获得与OpenJ9构建的OpenJDK AdoptOpenJDK包括OpenSSL的或建立自己的版本，其中包括OpenSSL的支持，下面的确认申请按照许可证条款：

    • 本产品包括由OpenSSL Project开发的，用于OpenSSL Toolkit的软件。 （http://www.openssl.org/）。
    • 该产品包括由Eric Young（eay@cryptsoft.com）编写的加密软件。

    利用GPU

    OpenJ9提供CUDA4J API和com.ibm.gpu API ，使用它们可以开发可利用图形处理单元（GPU）处理来实现适当功能（例如代码页转换）的应用程序。您还可以通过指定JIT编译器来将某些处理任务卸载到GPU -Xjit:enableGPU命令行上的选项。启用后，JIT编译器会根据性能启发式方法确定何时卸载任务。

    仅在Windows（x86-64）和Linux（x86-64和IBM Power LE）系统上支持GPU处理。有关启用GPU处理的更多信息，请参阅利用图形处理单元 。

    运行时选项

    运行时选项在命令行上指定，包括系统属性，标准选项，非标准（ -X ）选项和-XX选项。有关运行时选项的详细列表，请参见OpenJ9命令行选项。

    默认设置

    如果您在运行时未在命令行上指定任何选项，则OpenJ9 VM会以定义其运行方式的默认设置启动。有关这些设置的更多信息，请参见OpenJ9 VM的默认设置 。

    使用Jlink

    在Java 11及更高版本上，您可以使用jlink实用程序来创建自定义OpenJ9运行时映像，该映像可让您优化映像大小。如果您不需要英语翻译，则可以删除翻译文件以进一步优化大小。您可以通过指定--exclude-files=**java_**.properties运行时的选项jlink 。默认英文java.properties文件不受影响。

    故障排除

    OpenJ9诊断组件包含广泛的功能来帮助确定问题。诊断数据是在默认条件下生成的，但也可以通过使用-Xdump选项启动VM或使用com.ibm.jvm.Dump API。您还可以使用-Xtrace选项跟踪Java应用程序，方法和VM操作。

    首先，请阅读诊断工具和数据 。
