在模块路径上启动Jetty的另一方式

Jetty模块还可以通过以下方式充当自动JPMS模块： Automatic-Module-Name罐子里的属性MANIFEST.MF文件。

这样可以从模块路径而不是类路径运行Jetty。

我们建议使用JDK 11或更高版本，因为JDK 11从JDK中删除了所有“企业”模块，因此，它保证了一个更稳定的平台来依赖您的应用程序。这些“企业”模块中的类与JDK 8捆绑在一起，并存在于JDK 9和JDK 10中的“企业”模块中。在JDK 11中，这些“企业”类在JDK中不可用（因为它们的对应模块已删除），或者它们存在于其他模块中。

因为Jetty或在Jetty中运行的应用程序需要其中一些“企业”类，所以最好为这些类使用稳定的源-在这种情况下，使用JDK 11或更高版本，并显式引用“企业”类为依赖关系，而不是假设它们与JDK捆绑在一起。

在模块路径上启动Jetty

要在模块路径而非类路径上启动Jetty，只需添加--jpms命令行选项，例如：

$ mkdir my-jetty-base
$ cd my-jetty-base
$ java -jar $JETTY_HOME/start.jar --add-to-start=http
INFO  : server          transitively enabled, ini template available with --add-to-start=server
INFO  : http            initialized in ${jetty.base}/start.ini
INFO  : threadpool      transitively enabled, ini template available with --add-to-start=threadpool
INFO  : Base directory was modified
$ java -jar $JETTY_HOME/start.jar --jpms

上面的示例创建了一个Jetty基本目录，并启用了http使用模块--add-to-start命令。然后，服务器使用以下命令在模块路径上启动Jetty --jpms选项。

[NOTE]
When running on the module-path using the `--jpms` option, the Jetty start mechanism will fork a second JVM passing it the right JVM options to run on the module-path.
You will have two JVMs running: one that runs `start.jar` and one that runs Jetty on the module-path.

如果您对在模块路径上运行Jetty的命令行的详细信息感兴趣，可以添加--dry-run选项：

$ java -jar $JETTY_HOME/start.jar --jpms --dry-run

这将使输出看起来像这样（为清晰起见，在各节中进行了细分）：

/opt/openjdk-11+28/bin/java
--module-path /opt/jetty/lib/servlet-api-3.1.jar:/opt/jetty/lib/jetty-schemas-3.1.jar:/opt/jetty/lib/jetty-http-9.4.13-SNAPSHOT.jar:...
--patch-module servlet.api=/opt/jetty/lib/jetty-schemas-3.1.jar
--module org.eclipse.jetty.xml/org.eclipse.jetty.xml.XmlConfiguration /opt/jetty/etc/jetty-threadpool.xml /opt/jetty/etc/jetty.xml ...

的--module-path选项指定Jetty罐子的列表。此列表取决于通过以下方式启用的Jetty模块--add-to-start命令。

的--patch-module该选项对于Servlet和JSP容器必须找到XML DTD和XML模式，以验证Web应用程序中存在的各种XML文件（例如web.xml和别的）。

的--module选项告诉JVM运行主类XmlConfiguration来自org.eclipse.jetty.xml模块，使用给定的XML文件作为程序参数。

JVM启动时，模块org.eclipse.jetty.xml被添加到JPMS 根模块中 ;所有其他自动的Jetty模块将被解析并添加到模块图中。不是模块的JAR文件，例如servlet-api-3.1.jar ，位于模块路径上，因此将由JVM设置为自动模块（因此，派生模块名称servlet.api对于这个罐子，由--patch-module上面的命令行选项）。

先进的JPMS配置

Web应用程序可能需要来自Servlet容器的其他服务，例如JDBC DataSource引用或JTA UserTransaction参考。

例如，对于JDBC，通常在JNDI中存储对连接池的引用。 DataSource （如com.zaxxer.hikari.HikariDataSource ）或对JDBC驱动程序的直接引用DataSource （ com.mysql.jdbc.jdbc2.optional.MysqlDataSource ）。Jetty需要能够实例化那些类，因此需要能够加载那些类及其所有超类，其中包括javax.sql.DataSource 。

当Jetty在类路径上运行时，可以通过使用自定义模块轻松实现：

mysql.mod。 

[description]
MySQL module

[lib]
lib/mysql/mysql-connector-java-*.jar

但是，在模块路径上运行时，情况却大不相同。

类javax.sql.DataSource在名为的JDK捆绑模块中java.sql ，它不是自动的（这是一个正确的JPMS模块），并且不在根模块集中。因为它不是自动模块，所以不会将其添加到模块图中，因此需要使用JVM命令行显式添加。 --add-modules 。

添加JPMS模块java.sql到模块图，您需要使用我们的以下方式修改自定义模块mysql.mod举个例子：

mysql.mod。 

[description]
MySQL module

[lib]
lib/mysql/mysql-connector-java-*.jar

[jpms]
add-modules: java.sql

新的[jpms]本节仅在通过以下方式在模块路径上启动Jetty时使用--jpms命令行选项。

假如说mysql-connector-java-*.jar是非JPMS模块化罐或自动JPMS模块化罐，Jetty启动机制将添加mysql-connector-java-*.jar到模块路径，并将添加JVM命令行选项--add-modules java.sql 。

如果mysql-connector-java-*.jar是名称正确的JPMS模块化jar（例如） com.mysql.jdbc ，则需要以这种方式将其显式添加到模块图中：

mysql.mod。 

[description]
MySQL module

[lib]
lib/mysql/mysql-connector-java-*.jar

[jpms]
add-modules: com.mysql.jdbc

JPMS模块java.sql不需要显式添加，因为这将是com.mysql.jdbc模块，因此会自动添加到模块图中。

的[jpms]部分具有以下格式：

[jpms]
add-modules: <module name>(,<module name>)*
patch-module: <module>=<file>(:<file>)*
add-opens: <module>/<package>=<target-module>(,<target-module>)*
add-exports: <module>/<package>=<target-module>(,<target-module>)*
add-reads: <module>=<target-module>(,<target-module>)*

在模块路径上启动Jetty的另一方式

上面的部分使用--jpms命令行选项以在模块路径上启动Jetty。获得相同结果的另一种方法是使用Jetty模块， $JETTY_BASE/modules/jpms.mod ，它指定您要使用JPMS运行（并可能添加一些JPMS特定的配置）。

jpms.mod。 

[ini]
--jpms

[jpms]
# Additional JPMS configuration.

的[ini]这部分相当于传递--jpms命令行选项。的[jpms]部分（另请参阅高级JPMS配置部分 ）允许您指定其他JPMS配置。

$ mkdir jetty-base-jpms
$ cd jetty-base-jpms
$ mkdir modules
# Copy the jpms.mod file above into the $JETTY_BASE/modules/ directory.
$ cp /tmp/jpms.mod modules/
# Add both the http and the jpms modules.
$ java -jar $JETTY_HOME/start.jar --add-to-start=http,jpms
# Jetty will start on the module-path.
$ java -jar $JETTY_HOME/start.jar
