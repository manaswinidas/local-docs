使用Jetty发行版时，您首先需要启用session-store-hazelcast-remote您的Jetty基础 模块 --add-to-start命令行上的参数。

$ java -jar ../start.jar --add-to-start=session-store-hazelcast-remote

这样做可以启用远程Hazelcast Session模块以及使其在服务器上运行所需的任何相关模块或文件。上面的示例使用的是新鲜的${jetty.base}没有启用任何其他功能。因为Hazelcast不是Eclipse Foundation提供的技术，所以系统会提示用户同意外部供应商（在这种情况下为Apache）的许可证。

当。。。的时候--add-to-start参数已添加到命令行，它启用了session-store-hazelcast-remote模块以及sessions和server模块，这是Hazelcast会话管理运行所必需的。它还下载了所需的特定于Hazelcast的jar文件，并创建了一个名为${jetty.base}/lib/hazelcast/安置他们。

除了将这些模块添加到服务器的类路径之外，它还向该服务器添加了一些ini配置文件。 ${jetty.base}/start.d目录。

注意

如果您具有Jetty自动下载的jar文件的更新版本，则可以将其放置在关联的${jetty.base}/lib/目录并使用--skip-file-validation=命令行选项以防止在启动服务器时出错。

配置Hazelcast远程属性

打开start.d/session-store-hazelcast-remote.ini将显示Hazelcast模块的所有可配置选项的列表：

# ---------------------------------------
# Module: session-store-hazelcast-remote
# Enables session data store in a remote Hazelcast Map
# ---------------------------------------
--module=session-store-hazelcast-remote

#jetty.session.hazelcast.mapName=jetty_sessions
#jetty.session.hazelcast.onlyClient=true
#jetty.session.hazelcast.configurationLocation=
jetty.session.hazelcast.scavengeZombies=false
#jetty.session.gracePeriod.seconds=3600
#jetty.session.savePeriod.seconds=0
jetty.session.hazelcast.mapName
Hazelcast中将存储会话的地图名称。
jetty.session.hazelcast.onlyClient
Hazelcast实例将在客户端模式下配置
jetty.session.hazelcast.configurationLocation
Hazelcast xml配置文件的路径
jetty.session.hazelcast.scavengeZombies
真假。 False默认。如果true ，jetty将使用hazelcast查询来查找不再在任何码头节点上使用且其到期时间已过的会话。如果启用此选项，并且会话存储从Webapp内部引用类或码头类的属性，则需要确保这些类在每个hazelcast实例上均可用。
jetty.session.gracePeriod.seconds
关闭集群之前，等待集群检查实际上已在整个集群中到期的等待其他节点的时间（以秒为单位）。
jetty.session.savePeriod.seconds = 0

默认情况下，每当最后一个并发请求离开会话时，该会话始终通过SessionDataStore ，即使会话中唯一更改的是其更新的上次访问时间。非零值表示SessionDataStore如果仅更改了访问时间，并且少于savePeriod自上次写入会话以来的秒数。

注意

配置中savePeriod如果您的持久性技术的写入速度非常慢/成本很高，则此功能很有用。在群集环境中，存在会话的上次访问时间在共享存储中的过期时间可能长达9天。 savePeriod秒。这允许一个节点可能会过早地终止会话，即使该会话正在被另一个节点使用。彻底考虑maxIdleTime设置会话的时间savePeriod势在必行-设置一个savePeriod大于maxIdleTime 。

注意使用scavengeZombies如果您的会话属性包含webapp内部的类（或jetty类），则需要将这些类放入所有hazelcast实例的类路径中。
