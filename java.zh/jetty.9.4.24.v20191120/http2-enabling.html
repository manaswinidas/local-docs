启用HTTP / 2

本节假设在使用Jetty基本目录的情况下编写。可以使用以下命令创建一个支持HTTP / 1，HTTPS / 1以及从webapps目录进行部署的演示Jetty基础：

$ JETTY_BASE=http2-demo
$ mkdir $JETTY_BASE
$ cd $JETTY_BASE
$ java -jar $JETTY_HOME/start.jar --add-to-start=http,https,deploy

上面的命令创建一个$JETTY_BASE目录称为http2-demo ，并初始化http,https和deploy模块（及其依赖项）以在端口8080（对于HTTP / 1）和8443（对于HTTPS / 1）上运行典型的Jetty服务器。请注意，HTTPS模块下载带有自签名证书的演示密钥库文件，该文件需要由证书颁发机构颁发的证书替换才能进行实际部署。

要将HTTP / 2添加到此演示库，只需启用http2使用以下命令的模块：

$ java -jar $JETTY_HOME/start.jar --add-to-start=http2

此命令不会创建新的连接器，而只是将HTTP / 2协议添加到现有的HTTPS / 1连接器中，因此它现在在端口8443上支持这两个协议。为此，它也可过渡地启用ALPN模块进行协议协商。启动服务器时，可以在信息日志中看到对每种协议的支持：

$ java -jar $JETTY_HOME/start.jar
...
2015-06-17 14:16:12.549:INFO:oejs.ServerConnector:main: Started ServerConnector@34c9c77f{HTTP/1.1,[http/1.1]}{0.0.0.0:8080}
2015-06-17 14:16:12.782:INFO:oejs.ServerConnector:main: Started ServerConnector@711f39f9{SSL,[ssl, alpn, h2, h2-17, http/1.1]}{0.0.0.0:8443}
...

此日志显示端口8080仅支持HTTP / 1.1（根据规范，该端口包括HTTP / 1.0支持），而端口8443支持SSL协议，并通过ALPN协商在多个HTTP / 2版本之间进行选择（h2和h2-17草案） ）和HTTP / 1.1。未显示的是HTTP / 1.1是默认的ALPN协议，因此，如果客户端连接时不会说ALPN，则将假定使用HTTP / 1.1。

现在可以将浏览器指向https://localhost:8443/如果它支持HTTP / 2，则将使用它（通常由地址栏中的闪电图标表示）。请注意，浏览器指向此服务器的URL开头为http://localhost:8080/仍会使用HTTP / 1.1，因为未在纯文本连接器上启用HTTP / 2。

可以在纯文本连接器上启用HTTP / 2，并使用以下命令重新启动服务器：

$ java -jar $JETTY_HOME/start.jar --add-to-start=http2c
$ java -jar $JETTY_HOME/start.jar
..
2015-06-17 14:16:12.549:INFO:oejs.ServerConnector:main: Started ServerConnector@6f32cd1e{HTTP/1.1,[http/1.1, h2c, h2c-17]}{0.0.0.0:8080}
2015-06-17 14:16:12.782:INFO:oejs.ServerConnector:main: Started ServerConnector@711f39f9{SSL,[ssl, alpn, h2, h2-17, http/1.1]}{0.0.0.0:8443}
..

当前没有主流浏览器支持纯文本HTTP / 2，因此8080端口将只能在特定客户端上使用HTTP / 2（例如curl ）使用升级机制或采用HTTP / 2。
