启用MongoDB会话

使用Jetty发行版时，您首先需要启用session-store-mongo您的Jetty基础 模块 --add-to-start命令行上的参数。

$ java -jar ../start.jar --create-startd
INFO : Base directory was modified

$ java -jar ../start.jar --add-to-start=session-store-mongo

这样做将启用MongoDB会话模块以及使其在服务器上运行所需的任何相关模块或文件。上面的示例使用的是新鲜的${jetty.base}没有启用任何其他功能。

由于MongoDB不是Eclipse Foundation提供的技术，因此系统会提示用户同意外部供应商（在本例中为Apache）的许可证。当。。。的时候--add-to-start参数已添加到命令行，它启用了session-store-mongo模块以及sessions和server模块，这是MongoDB会话管理运行所必需的。它还下载了所需的特定于Mongo的jar文件，并创建了一个名为${jetty.base}/lib/nosql/容纳它。

除了将这些模块添加到服务器的类路径之外，还向ini添加了一些ini配置文件。 ${jetty.base}/start.d目录。

注意

如果您具有Jetty自动下载的jar文件的更新版本，则可以将其放置在关联的${jetty.base}/lib/目录并使用--skip-file-validation=命令行选项以防止在启动服务器时出错。

配置MongoDB会话属性

打开start.d/session-store-mongo.ini将显示MongoDB模块的所有可配置选项的列表：

# ---------------------------------------
# Module: session-store-mongo
# Enables NoSql session management with a MongoDB driver.
# ---------------------------------------
--module=session-store-mongo

#jetty.session.mongo.dbName=HttpSessions
#jetty.session.mongo.collectionName=jettySessions
#jetty.session.gracePeriod.seconds=3600
#jetty.session.savePeriod.seconds=0

connection-type=address
#jetty.session.mongo.host=localhost
#jetty.session.mongo.port=27017

#connection-type=uri
#jetty.session.mongo.connectionString=mongodb://localhost
jetty.session.gracePeriod.seconds
关闭集群之前，等待集群检查实际上已在整个集群中到期的等待其他节点的时间（以秒为单位）。
jetty.session.savePeriod.seconds = 0

默认情况下，每当最后一个并发请求离开会话时，该会话始终通过SessionDataStore ，即使会话中唯一更改的是其更新的上次访问时间。非零值表示SessionDataStore如果仅更改了访问时间，并且少于savePeriod自上次写入会话以来的秒数。

注意

配置中savePeriod如果您的持久性技术的写入速度非常慢/成本很高，则此功能很有用。在群集环境中，存在会话的上次访问时间在共享存储中的过期时间可能长达9天。 savePeriod秒。这允许一个节点可能会过早地终止会话，即使该会话正在被另一个节点使用。彻底考虑maxIdleTime设置会话的时间savePeriod势在必行-设置一个savePeriod大于maxIdleTime 。

jetty.session.mongo.dbName
Mongo中用于存储Session集合的数据库名称。
jetty.session.mongo.collectionName
Mongo中用于保留所有会话的集合的名称。
jetty.session.gracePeriod.seconds
关闭集群之前，等待集群检查实际上已在整个集群中到期的等待其他节点的时间（以秒为单位）。
连接类型=地址

在利用与Mongo服务器的直接连接时使用。

jetty.session.mongo.host
远程Mongo实例的主机名或地址。
jetty.session.mongo.port
远程Mongo实例的端口号。
连接类型= uri

在将MongoURI用于安全连接时使用。

jetty.session.mongo.connectionString

定义MongoURI值的字符串，例如mongodb://[username:password@]host1[:port1][,host2[:port2],...[,hostN[:portN]]][/[database][?options]] 。有关如何格式化MongoURI字符串的更多信息，请参见mongo的官方文档。

注意

您只会使用一个 connection-type一次address要么uri 。如果两者都用于您的session-store-mongo.ini ，只有最后一个connection-type文件中配置的将被使用。默认情况下， connection-type的address用来。
