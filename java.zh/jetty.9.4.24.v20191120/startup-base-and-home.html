管理Jetty基地和Jetty主页

Jetty配送中的演示基地
宣布Jetty基地

除了可以从多个不同的分发位置管理多个Jetty实现之外，还可以在独立Jetty的二进制安装（称为${jetty.home} ），以及针对您特定环境的自定义设置（称为${jetty.base} ）。除了轻松管理多个服务器实例外，还可以进行快速，直接的Jetty升级。总应该只有一个 Jetty Home（Jetty的每个版本），但是可以有多个引用它的Jetty Base目录。

Jetty基地
  • 也称为${jetty.base}属性。
  • 这是您对Jetty发行版进行配置和自定义的位置。
Jetty之家
  • 也称为${jetty.home}属性。
  • 这是Jetty发行二进制文件，默认XML IoC配置和默认模块定义的位置。

重要

Jetty之家应始终被视为真理的标准。所有配置修改，更改和添加都应在相应的Jetty Base目录中进行。

可以从这两个目录位置解析潜在配置。当Jetty从它们启动流程配置时，如下所示：

首先检查Jetty基地
如果引用的配置存在，则相对于定义的Jetty基础，将使用它。
检查Jetty回家第二
如果存在引用的配置（相对于定义的Jetty主页），则使用它。
使用java.io。文件（字符串路径名）逻辑
最后，使用参考作为java.io.File(String pathname)参考，遵循该构造函数概述的默认解析规则。简而言之，引用将按原样使用，无论是相对的（相对于当前工作目录，也就是$ \ {user.dir}）还是绝对路径，甚至是网络引用（例如在Windows上和UNC路径的使用）。

有关使用start.jar启动的工作方式的更多详细信息，请参见使用start.jar：执行

如果你看${jetty.base}/start.ini您将看到类似于以下的布局。

[my-base]$ cat start.ini

# Enable security via jaas, and configure it
--module=jaas
jaas.login.conf=etc/login.conf

# Enable rewrite examples
--module=rewrite
etc/demo-rewrite-rules.xml

# Websocket chat examples needs websocket enabled
# Don't start for all contexts (set to true in test.xml context)
org.eclipse.jetty.websocket.jsr356=false
--module=websocket

# Create and configure the test realm
etc/test-realm.xml
demo.realm=etc/realm.properties

# Initialize module server
--module=server
threads.min=10
threads.max=200
threads.timeout=60000
jetty.dump.start=false
jetty.dump.stop=false

--module=deploy
--module=jsp
--module=ext
--module=resources
--module=client
--module=annotations

在这个例子中${jetty.base}/start.ini是Jetty的主要启动配置入口点。您将看到我们为Jetty启用了一些模块，指定了一些属性，还引用了一些Jetty IoC XML文件（即etc/demo-rewrite-rules.xml和etc/test-realm.xml文件）

当Jetty的start.jar解析start.ini ，它将遵循上面的解决规则 。

例如，对etc/demo-rewrite-rules.xml被发现于${jetty.base}/etc/demo-rewrite-rules.xml 。

宣布Jetty基地

Jetty发行的start.jar是管理此分离行为的组件。

Jettystart.jar和XML文件始终假定${jetty.home}和${jetty.base}在启动Jetty时定义。

您可以选择手动定义${jetty.home}和${jetty.base}目录，例如：

[jetty-distribution-9.4.24.v20191120]$ pwd
/home/user/jetty-distribution-9.4.24.v20191120

[jetty-distribution-9.4.24.v20191120]$ java -jar start.jar \
    jetty.home=/home/user/jetty-distribution-9.4.24.v20191120 \
    jetty.base=/home/user/my-base

2013-10-16 09:08:47.802:INFO:oejs.Server:main: jetty-9.4.24.v20191120
2013-10-16 09:08:47.817:INFO:oejdp.ScanningAppProvider:main: Deployment monitor [file:/home/user/my-base/webapps/] at interval 1
...

或者，您可以声明一个目录，然后让另一个目录被发现。

以下示例使用默认发现${jetty.home}通过使用where的父目录start.jar本身就是一个手动声明${jetty.base} 。

[jetty-distribution-9.4.24.v20191120]$ pwd
/home/user/jetty-distribution-9.4.24.v20191120

[jetty-distribution-9.4.24.v20191120]$ java -jar start.jar jetty.base=/home/user/my-base

2013-10-16 09:08:47.802:INFO:oejs.Server:main: jetty-9.4.24.v20191120
2013-10-16 09:08:47.817:INFO:oejdp.ScanningAppProvider:main: Deployment monitor [file:/home/user/my-base/webapps/] at interval 1
...

但是Jetty建议您始终从以下目录启动Jetty ${jetty.base}然后参考start.jar在你的{$jetty.home}远程地。

下面通过允许默认发现来演示这一点。 ${jetty.home}通过找到start.jar ，并使用user.dir的系统属性${jetty.base} 。

[jetty-distribution-9.4.24.v20191120]$ pwd
/home/user/jetty-distribution-9.4.24.v20191120

[jetty-distribution-9.4.24.v20191120]$ cd /home/user/my-base
[my-base]$ java -jar /home/user/jetty-distribution-9.4.24.v20191120/start.jar

2013-10-16 09:08:47.802:INFO:oejs.Server:main: jetty-9.4.24.v20191120
2013-10-16 09:08:47.817:INFO:oejdp.ScanningAppProvider:main: Deployment monitor [file:/home/user/my-base/webapps/] at interval 1
...

重要

注意user.dir系统属性，因为只有在JVM启动且许多第三方库（尤其是日志记录）使用此系统属性时才能安全地设置它。强烈建议您坐在所需目录中${jetty.base}启动Jetty时具有一致的行为并使用user.dir系统属性。
