HTTP / 2简介

Jetty HTTP / 2安全更新
码头HTTP / 2子项目

Jetty支持RFC 7540定义的HTTP / 2协议的客户端和服务器实现。

运行HTTP / 2的要求是JDK 8或更高版本，并且通常还具有ALPN支持（请参见第15章， 应用程序层协议协商（ALPN） ）。

通常，通过TLS（SSL）部署的服务器会通过TLS扩展应用程序层协议协商（ALPN）通告HTTP / 2协议。

注意

要通过TLS连接器在Jetty中使用HTTP / 2，您需要在启动类路径中添加ALPN启动jar。使用Jetty发行版的start.jar 模块系统时 ，此操作会自动完成，但否则必须直接进行配置。

Jetty HTTP / 2安全更新

在2019年中，发布了许多CVE ，警告易受攻击的HTTP / 2实现。这些CVE（CVE-2019-9511至CVE-2019-9518）通常围绕攻击者操纵和泛滥HTTP / 2服务器并创建拒绝服务（DOS）。这些漏洞已在Jetty 9.4.21中修复。

由于这些CVE，Jetty在Jetty 9.4.22中引入了新的可配置的拒绝服务（DOS）保护功能。

Jetty的HTTP / 2实现现在具有新的速率控制参数， jetty.http2.rateControl.maxEventsPerSecond ，对于所有ping，不良帧，设置帧，优先级更改等，每个连接默认为每秒20个事件。

码头HTTP / 2子项目

Jetty HTTP / 2实现由以下子项目组成（每个子项目都产生一个jar文件）：

  1. http2-common ：包含HTTP / 2 API和其他模块之间共享的部分实现。
  2. http2-hpack ：包含用于HTTP标头压缩的HTTP / 2 HPACK实现。
  3. http2-server ：提供HTTP / 2的服务器端实现。
  4. http2-client ：提供具有低级HTTP / 2 API的HTTP / 2客户端的实现，处理HTTP / 2流，帧等。
  5. http2-http-client-transport ：提供HTTP / 2传输的实现，用于HttpClient （请参阅第22章， HTTP客户端 ）。应用程序可以使用由以下人员提供的更高级别的API： HttpClient发送HTTP请求并接收HTTP响应，HTTP / 2传输将负责将它们转换为HTTP / 2格式（另请参见此博客条目 ）。
