启用Infinispan会话

使用Jetty发行版时，您首先需要启用session-store-infinispan-remote您的Jetty基础 模块 --add-to-start命令行上的参数。

$ java -jar ../start.jar --create-startd
INFO : Base directory was modified

$ java -jar ../start.jar --add-to-start=session-store-infinispan-remote

这样做可以启用远程Infinispan Session模块以及使其在服务器上运行所需的任何相关模块或文件。上面的示例使用的是新鲜的${jetty.base}没有启用任何其他功能。因为Infinispan不是Eclipse Foundation提供的技术，所以系统会提示用户同意外部供应商（在本例中为Apache）的许可证。

当。。。的时候--add-to-start参数已添加到命令行，它启用了session-store-infinispan-remote模块以及sessions和server这些模块是Infinispan会话管理运行所必需的。它还下载了所需的Infinispan特定的jar文件，并创建了一个名为${jetty.base}/lib/infinispan/安置他们。

除了将这些模块添加到服务器的类路径之外，它还向该服务器添加了一些ini配置文件。 ${jetty.base}/start.d目录。

注意

如果您具有Jetty自动下载的jar文件的更新版本，则可以将其放置在关联的${jetty.base}/lib/目录并使用--skip-file-validation=命令行选项以防止在启动服务器时出错。

配置Inifinspan远程属性

打开start.d/session-store-infinispan-remote.ini将显示JDBC模块的所有可配置选项的列表：

# ---------------------------------------
# Module: session-store-infinispan-remote
# Enables session data store in a remote Infinispan cache
# ---------------------------------------
--module=session-store-infinispan-remote

#jetty.session.infinispan.remoteCacheName=sessions
#jetty.session.infinispan.idleTimeout.seconds=0
#jetty.session.gracePeriod.seconds=3600
#jetty.session.savePeriod.seconds=0
jetty.session.infinispan.remoteCacheName
Infinispan中将存储会话的缓存的名称。
jetty.session.infinispan.idleTimeout.seconds
在infinispan删除会话条目之前，infinispan中的会话条目可以闲置（即未读取或写入）的时间（以秒为单位）。通常情况下，你不想为此设置一个值，只要你想Jetty处理所有会话过期（和调用任何SessionListeners）。但是，如果有可能将会话保留在infinispan中，但不再被任何Jetty节点引用（所谓的“僵尸”或“孤立”会话），那么您可能要使用此功能。您应该确保指定的秒数足够大，以免出现Jetty仍然引用会话但这种情况很少被infinispan访问并因此将其删除的情况。或者，您可以启用infinispan-remote-query模块，这将允许Jetty搜索infinispan会话缓存，以主动查找并正确（即调用任何SessionListeners）清除已失效的会话。
jetty.session.gracePeriod.seconds
关闭集群之前，等待集群检查实际上已在整个集群中到期的等待其他节点的时间（以秒为单位）。
jetty.session.savePeriod.seconds = 0
默认情况下，每当最后一个并发请求离开会话时，该会话始终通过SessionDataStore ，即使会话中唯一更改的是其更新的上次访问时间。非零值表示SessionDataStore如果仅更改了访问时间，并且少于savePeriod自上次写入会话以来的秒数。

注意

配置中savePeriod如果您的持久性技术的写入速度非常慢/成本很高，则此功能很有用。在群集环境中，存在会话的上次访问时间在共享存储中的过期时间可能长达9天。 savePeriod秒。这允许一个节点可能会过早地终止会话，即使该会话正在被另一个节点使用。彻底考虑maxIdleTime设置会话的时间savePeriod势在必行-设置一个savePeriod大于maxIdleTime 。

配置远程Infinispan查询模块

启用此模块将允许Jetty在infinispan中搜索不再被任何Jetty节点引用的过期会话。请注意，这是一个附加模块，可与session-store-infinispan-remote模块。

java -jar ../start.jar --add-to-start=infinispan-remote-query

没有与此模块关联的配置属性。

配置嵌入式Inifinspan集群

在测试过程中，运行Infinispan的进程内实例可能会有所帮助。要启用此功能，您首先需要启用session-store-infinispan-embedded您的Jetty基础 模块 --add-to-start命令行上的参数。

重要

配置Inifinspan嵌入式查询

类似于session-store-infinispan-remote模块， session-store-infinispan-embedded模块有一个辅助模块infinispan-embedded-query启用此功能后，Jetty将可以检测并适当清除infinispan中滞留的无效会话。

java -jar ../start.jar --add-to-start=infinispan-embedded-query

没有与此模块关联的配置属性。

Jetty-9.4.13的转换会话格式

从jetty-9.4.13开始，使用远程缓存（即使用hotrod）时，我们更改了序列化会话的格式。在版本9.4.13之前，我们使用了默认的Infinispan序列化，但是，它不能存储足够的信息以允许Jetty在所有情况下都能正确地反序列化会话属性。有关更多背景信息，请参见问题https://github.com/eclipse/jetty.project/issues/2919 。

我们提供了一个转换程序，该程序会将Infinispan中存储的所有会话转换为新格式。

重要

我们建议您在运行转换程序之前备份存储的会话。

如何使用转换器：

java -cp servlet-api-3.1.jar:jetty-util-9.4.13.jar:jetty-server-9.4.13.jar:infinispan-remote-9.1.0.Final.jar:jetty-infinispan-9.4.13.jar:[other classpath]  org.eclipse.jetty.session.infinispan.InfinispanSessionLegacyConverter

Usage:  InfinispanSessionLegacyConverter [-Dhost=127.0.0.1] [-Dverbose=true|false] <cache-name> [check]
类路径
必须包含servlet-api，jetty-util，jetty-server，jetty-infinispan和infinispan-remote jar。如果会话包含使用应用程序类的属性，则还需要将这些类放在类路径中。如果您的会话已通过身份验证，则可能还需要在类路径中包括jetty-security和jetty-http jars。
参量

不带任何参数使用时，将显示用法消息。当与cache-name参数执行转换。当两者一起使用时cache-name和check参数，检查会话是否被转换。

-幽灵
您可以选择提供系统属性以及远程Infinispan服务器的地址。默认为本地主机。
-详细
默认为false。如果为true，则输出有关失败的更全面的堆栈跟踪信息。对于诊断为什么不转换会话很有用。
缓存名称
包含您的会话的远程缓存的名称。这是强制性的。
校验
可选的check命令将验证会话是否已转换。转换后使用它。

要执行转换，只需运行InfinispanSessionLegacyConverter cache-name ，以及可选的host系统属性。下面的命令将尝试转换命名的缓存中的所有会话my-remote-cache在机器上myhost ，确保/my/custom/classes目录位于类路径上：

java -cp servlet-api-3.1.jar:jetty-util-9.4.13.jar:jetty-server-9.4.13.jar:infinispan-remote-9.1.0.Final.jar:jetty-infinispan-9.4.13.jar:/my/custom/classes  org.eclipse.jetty.session.infinispan.InfinispanSessionLegacyConverter -Dhost=myhost my-remote-cache

如果转换器无法转换会话，则将显示错误消息和stacktrace，转换将中止。失败的会话应该保持不变，但是在尝试进行转换之前，请谨慎备份缓存 。
