WebSocket事件

每个WebSocket都可以接收各种事件：

连接事件

向WebSocket指示升级已成功并且WebSocket现在已打开。

您会收到一个org.eclipse.jetty.websocket.api.Session引用此Open Event的特定会话的对象。

对于普通的WebSocket，重要的是要保留此会话并将其用于与远程端点进行通信。

对于无状态WebSocket，会话将在事件发生时传递到每个事件中，从而使您只有一个WebSocket实例可以为许多远程端点提供服务。

结束事件

WebSocket现在已关闭的指示。

每个关闭事件都会有一个状态码 （和可选的关闭原因消息）

正常的WebSocket关闭将通过关闭握手，本地端点和远程端点都将发送关闭帧以指示连接已关闭。

本地WebSocket可以通过向远程端点发出关闭帧来指示其关闭意愿，但是远程端点可以继续发送消息，直到发送关闭帧为止。这称为半开连接，需要注意的是，一旦本地端点发送了关闭帧，它就无法再写入WebSocket通信。

在异常关闭（例如，连接断开连接或连接超时）时，低级连接将被终止，而不会经历关闭握手，这仍将导致打开关闭事件（并可能导致相应的打开错误事件）。

错误事件
如果发生错误，则在实现期间，将通过此事件处理程序通知WebSocket。
消息事件

表示已收到完整的消息，并准备由WebSocket处理。

这可以是（UTF8）TEXT消息或原始BINARY消息。
