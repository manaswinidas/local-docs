用法

通过实际的远程IP控制对包装的处理程序的访问。通过白名单/黑名单提供控制，包括互联网地址和URI。此处理程序使用连接的真实Internet地址，而不使用转发的标头中报告的地址，因为此地址不易伪造。

通常，黑/白列表将在以下三种模式之一中使用：

  • 通过指定多个黑名单条目来阻止一些特定的IP / URL。
  • 通过指定几个白名单条目，仅允许某些特定的IP / URL。
  • 通过指定几个常规白名单条目来允许IP / URL的常规范围，然后通过几个特定的黑名单例外进一步完善这些条目。

空白列表被视为全部匹配。如果白名单中至少有一个条目，则请求必须与白名单条目匹配。始终会应用黑名单条目，因此即使某个条目与白名单匹配，黑名单条目也将覆盖它。

Internet地址可以指定为绝对地址，也可以指定为四个八位字节通配符规范（abcd）的组合，定义如下。

  • nnn-绝对值（0-255）

  • mmm-nnn-绝对值的范围，包括以下速记符号：

    • nnn-⇒nnn-255
    • -nnn⇒0-nnn
    • -⇒0-255
  • a，b，...-通配符规范列表

Internet地址规范使用“ |”与URI模式分开（管道）字符。URI模式遵循Servlet规范中的简单*前缀和后缀通配符（例如/、/foo、/foo/bar、/foo/bar/*、*.baz）。

处理程序的早期版本使用Internet地址前缀通配符规范来定义Internet地址的范围（例如127.，10.10。，172.16.1。）。他们还使用URI模式的第一个“ /”字符将其与Internet地址分开。在当前版本中，这两个功能均已弃用。

条目规范的示例包括：

  • 10.10.1.2-来自IP 10.10.1.2的所有请求
  • 10.10.1.2 | / foo / bar-从IP 10.10.1.2到URI / foo / bar的所有请求
  • 10.10.1.2 | / foo / *-从IP 10.10.1.2到以/ foo /开头的URI的所有请求
  • 10.10.1.2 | * .html-从IP 10.10.1.2到以.html结尾的URI的所有请求
  • 10.10.0-255.0-255-来自10.10.0.0/16子网中IP的所有请求
  • 10.10.0 -.- 255 | / foo / bar-从10.10.0.0/16子网中的IP到URI / foo / bar的所有请求
  • 10.10.0-3,1,3,7,15 | / foo / *-子网10.10.0.0/22中来自最后一个八位位组等于1,3,7,15的IP地址到以/ foo /开头的URI的所有请求

处理程序的早期版本使用Internet地址前缀通配符规范来定义Internet地址的范围（例如127.，10.10。，172.16.1。）。他们还使用URI模式的第一个“ /”字符将其与Internet地址分开。在当前版本中，这两个功能均已弃用。
