JConsole和Java Mission Control（JMX）是图形工具；它们使您可以使用JMX远程管理和监视服务器和Web应用程序的状态。按照下面给出的说明进行操作时，还请确保对可能使用的任何防病毒软件进行必要的更改，这些操作可能会阻止JConsole或JMC运行。

启动Jetty Standalone

启用支持的最简单方法是将JMX-Remote支持模块添加到您的{$jetty.base} 。

[mybase]$ java /opt/jetty-dist/start.jar --add-to-start=jmx-remote, jmx
INFO: jmx-remote      initialised in ${jetty.base}/start.ini
INFO: jmx             initialised in ${jetty.base}/start.ini

然后打开{$jetty.base}/start.ini （要么{$jetty.base}/start.d/jmx-remote.ini ）文件并编辑属性以适合您的需求：

#
# Initialize module jmx-remote
#
--module=jmx-remote
## JMX Configuration
## Enable for an open port accessible by remote machines
jetty.jmxrmihost=localhost
jetty.jmxrmiport=1099

要使用JConsole监视Jetty的服务器状态，请启动Jetty，然后通过键入以下内容启动JConsole jconsole在命令行上。

连接到服务器进程

重要

如果在可以连接的进程列表中没有看到您的Jetty进程，可以快速切换选项卡，或者关闭并重新打开新的“新建连接”对话框窗口。这将强制JConsole刷新列表，并识别您新启动的Jetty流程。

选择start.jar条目，然后单击“连接”按钮。一个新的JConsole窗口打开：

在此窗口中，您可以监视内存使用情况，线程使用情况，类加载和VM统计信息。您还可以执行诸如手动垃圾收集之类的操作。JConsole是一个非常强大且有用的工具。

使用JConsole管理Jetty对象

JConsole的MBean选项卡允许访问Java应用程序中的托管对象，包括JVM提供的MBean。如果您还想通过JConsole与Jetty JMX实现进行交互，则需要以JConsole可以访问的形式启动Jetty JMX。有关更多信息，请参见将JMX与Jetty一起使用 。
