我们的政策

我们喜欢看到人们为Jetty项目贡献补丁，而且过程相对简单。提交的要求不高，但对于Eclipse Foundation和开源项目的知识产权非常重要。以下是我们运作的一般流程。

  • 您必须已签署Eclipse贡献者协议。
  • 该协议必须使用与发起Git拉取请求相同的电子邮件地址。
  • 提交必须经过签名。

  • 发出拉取请求后，git-hook将验证电子邮件地址。

    • 如果结果是绿色复选框，则Jetty提交者可以查看拉取请求。
    • 如果结果是红色的X，则此时Jetty提交者绝对不能做任何事情来接受提交。
  • 这可能不是提交的最终形式，可能会来回一些，并且可能会要求您重新发出拉取请求。

因为我们在GitHub上，所以并不是所有的事情都特别相关，但是其中的关键是详细的。ECA 对于该过程至关重要 。

签署Eclipse贡献者协议（ECA）

Eclipse基金会有一项强有力的知识产权政策，该政策会详细跟踪贡献，以确保：

  1. 撰稿人是否100％是内容的作者？
  2. 贡献者是否有权将此内容贡献给Eclipse？
  3. 是根据项目许可（例如EPL）进行的出资吗？

无论提供方式如何，贡献者都需要签署Eclipse贡献者协议（有关更多说明，请参见Eclipse ECA FAQ ）。您可以在Contributing via Git上熟悉Eclipse Wiki页面。为了使任何Eclipse项目都接受请求请求，您必须完成此协议。

小费

登录到Eclipse主页 （如果尚未创建，则需要使用Eclipse Foundation创建帐户），单击“ Eclipse ECA”，然后填写表格。创建任何Git提交记录时，请确保使用相同的电子邮件地址 。

配置Git

GitHub上有大量有关如何与系统交互的质量文档，您将仅需配置user.email属性。在GitHub上查看以下指南以获取更多信息。

做出承诺

在为拉取请求进行提交时， 至关重要的是，您使用git commit -s选项。没有此签名，您的补丁将无法应用于Jetty存储库，因为它将被拒绝。

您可以在Github查看该指南以获取更多信息。

小费

想到这一点的一种方法是，当您在ECA上签名时，表明您可以自由地为日食做出贡献，但这并不意味着您所做的一切都可以被贡献。使用提交签名机制表示您的提交是在协议的主持下进行的。

如果请求请求是针对我们存储库中的特定问题，则提交消息的格式很重要。该消息应遵循“问题＃123 ”的形式。当Jetty项目运行发布时，我们有一个自动化的过程，该过程将扫描这种格式的提交以包含在VERSION.txt文件中。

> git commit -s -m "Issue #123 resolving the issue by adding widget"

拉取请求

拉取请求在很大程度上是一个GitHub流程，因此Github可以最好地解释它 。

我们的政策

我们竭诚欢迎对Jetty的贡献，并会尽力及时处理它们。尽管并非所有贡献都会被接受，但我们的承诺是与感兴趣的各方就他们关心的事情进行合作。考虑到这一点，我们只能与积极参与的各方一起处理请求请求。我们保留放弃请求者没有及时回复的请求的权利。

我们通常会按照以下时间表进行捐款：

  • 无效的提取请求-1周

    • 由于缺少捐助者协议或错误的电子邮件，这些拉取请求由于某些原因而没有遵循捐助要求。
    • 我们将尝试与拉取请求作者联系，以解决该问题，但是其中大部分是我们无法控制的，并且在提交者和Eclipse Foundation之间。
    • 如果一周后仍未收到来自投稿人的消息，我们将关闭拉取请求。

  • 有效拉取请求-2周

    • 这些提交请求在提交标题之后带有绿色的复选标记。
    • 如果拉取请求可以立即应用，我们将这样做。
    • 在这个问题上可能需要进行一些对话，在这种情况下，提交者将在请求请求中跟进作者。
    • 如果原始贡献者在2周内没有响应，我们可能会关闭提交。
    • 如果我们在提交中看到了价值，但提交人在两周后仍未答复，则我们可以自己修改提交。
