远程调试

如果您已将一个Web应用程序部署到Jetty中，则可以从调试的角度轻松地与其远程交互。基础知识是您必须使用其他参数启动远程JVM，然后在Eclipse中为有问题的Web应用程序启动远程调试会话。这很容易实现。

注意

本示例假定您将Web应用程序部署到码头分布中。

启动码头

假设您已将Web应用程序部署到码头，有两种不同的方法可以解决此问题：

通过命令行

像这样在命令行上添加所需的参数。

$ java -Xdebug -agentlib:jdwp=transport=dt_socket,address=9999,server=y,suspend=n -jar start.jar
通过start.ini

如果您要调试特定的码头分布并且不必记住命令行提示，则最好使用此方法。

  1. 编辑start.ini并取消注释--exec行，如果要将jvm选项添加到start.ini文件中，则这是必需的，因为jetty-start必须生成所需的类路径并派生新的jvm。

  2. 在命令行选项中添加上面提到的参数，以便您的start.ini如下所示：

    #===========================================================
# Configure JVM arguments.
# If JVM args are include in an ini file then --exec is needed
# to start a new JVM from start.jar with the extra args.
# If you wish to avoid an extra JVM running, place JVM args
# on the normal command line and do not use --exec
#-----------------------------------------------------------
--exec
-Xdebug
-agentlib:jdwp=transport=dt_socket,address=9999,server=y,suspend=n
# -Xmx2000m
# -Xmn512m
# -XX:+UseConcMarkSweepGC
# -XX:ParallelCMSThreads=2
# -XX:+CMSClassUnloadingEnabled
# -XX:+UseCMSCompactAtFullCollection
# -XX:CMSInitiatingOccupancyFraction=80
# -verbose:gc
# -XX:+PrintGCDateStamps
# -XX:+PrintGCTimeStamps
# -XX:+PrintGCDetails
# -XX:+PrintTenuringDistribution
# -XX:+PrintCommandLineFlags
# -XX:+DisableExplicitGC

    取消注释任何您希望在调试会话中使用的其他jvm环境选项。

  3. 不管选择哪个选项，您都应该在码头分销启动公司的顶部看到以下几行。

    Listening for transport dt_socket at address: 9999
