Deploying Web Applications

Jetty Demonstration Web Applications

Jetty server instances that configure the deploy module will have a web application deployer that hot deploys files found in the webapps directory. Standard WAR files and Jetty configuration files that are placed in the webapps directory are hot deployed to the server with the following conventions:

  • A directory called example/ is deployed as a standard web application if it contains a WEB-INF/ subdirectory, otherwise it is deployed as context of static content. The context path is /example (that is, http://localhost:8080/example/) unless the base name is ROOT (case insensitive), in which case the context path is /. If the directory name ends with ".d" it is ignored (but may be used by explicit configuration).
  • A file called example.war is deployed as a standard web application with the context path /example (that is, http://localhost:8080/example/). If the base name is ROOT (case insensitive), the context path is /.如果example.war和example/如果存在，则仅部署WAR（可以将目录用作解压缩位置）。
  • XML文件，例如example.xml被部署为上下文，其配置由XML定义。配置本身必须设置上下文路径。如果example.xml和example.war存在时，仅部署XML（可在其配置中使用WAR）。

如果您有标准的Web应用程序，则可以通过将其复制到webapps目录。

码头示范网络应用

demo-base / webapps目录包含以下可部署和辅助文件：

ROOT/
静态内容的目录，由于名称而被部署到根上下文。包含Jetty演示欢迎页面。
test.d
包含其他配置文件的目录test.xml注入额外的配置test.war 。
test.xml
上下文配置文件，用于配置和部署test.war.附加配置包括上下文路径，以及设置在test.d目录。
test.war
由配置和部署的演示Web应用程序test.xml 。
async-rest.war
Web应用程序演示了与eBay异步REST的过程，该演示基于文件名自动部署到/ async-rest。
test-jaas.war
一个使用JAAS进行身份验证的演示Web应用程序。
test-jaas.xml
上下文配置文件，用于配置test-jaas.war 。其他配置包括设置LoginService进行身份验证和授权。
test-jndi.war
一个演示Web应用程序，展示了JNDI的用法。
test-jndi.xml
上下文配置文件，用于配置test-jndi.war 。其他配置包括在命名空间中定义可以从Web应用程序引用的对象。
test-spec.war
一个演示Web应用程序，显示注释，片段， ServletContainerInitializers和其他Servlet Specification 3.0 / 3.1功能。
test-spec.xml
上下文配置文件，用于配置test-spec.war 。附加配置包括在命名空间中设置一些可由批注引用的对象。
javadoc-proxy.war
一个演示Web应用程序，它使用透明代理从Eclipse Jetty网站提供Jetty源Javadoc 。
example-moved.xml
一个演示上下文配置文件，显示如何使用MovedContextHandler从一条路径重定向到另一条路径。
