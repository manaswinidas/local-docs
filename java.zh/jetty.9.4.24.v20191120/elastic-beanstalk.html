警告

这是一个日趋老龄化的集成，类似的事情可能发生了足够的变化，这并不是直接有用的，但是如果有人想对其进行研究，它可以作为有用的起点。

Elastic Beanstalk是Amazon Web Services产品的一个组件，它使您可以基于几种可用的基准配置之一配置整个虚拟机，然后通过功能强大的配置系统对其进行自定义。尽管当前可用的默认产品基于针对Java社区的Tomcat，但我们使用该配置系统制定了基础知识，从而启用了Jetty。

总览

弹性beantalk具有非常强大的配置机制，因此该集成可有效地对tomcat配置进行重做并将其替换为使码头运行就位所需的钻头。以下是各种配置文件的作用以及beantalk上一般配置流程的逐步演练。

有一个.ebextensions beantalk应用程序中的目录（包含所有文件）需要配置和定制beantalk和应用程序组合。此目录中以.config结尾的文件将按字母顺序处理。

00-java7.config
将Java 7安装到beantalk环境中，并使其成为默认设置
10-tweak.config
不需要，但更改/opt/elasticbeanstalk目录易于阅读，使调试更加容易
11-jetty.config
将jetty9安装到/opt/jetty-9并删除不需要的分发文件
12-beanstalk.config
在许多配置文件中处理用jetty替换tomcat，配置日志记录并连接系统启动过程。您的某些文件.ebextensions目录被移动以替换/ opt / elasticbeanstalk下的文件。

如果你看着.ebextensions您的应用程序目录中，您还应该看到其他码头特定的xml和ini文件。最终的配置文件将处理这些问题，因为它们很大程度上是为您的应用程序定制的。

20-testapp.config
将应用程序特定的配置文件分层放置到码头安装中

我们的示例测试Web应用程序中的文件在此处为需要加载的库启用了各种选项，可以自定义正在部署的根应用程序，甚至可以像我们在码头发行演示中那样部署其他上下文。这也是用于连接特定于应用程序的东西的机制，例如，如果需要安装其他软件，制作自定义目录等。

Maven Bits

对此功能的支持利用Maven使事情变得容易，它由三个不同的模块组成。

码头豆茎覆盖
这是将码头插入普通beantalk设置所需的脚本的集合。该模块旨在提取到Webapp中，以使其能够与码头一起使用。
码头豆茎资源
这将生成由配置过程下载的文件的工件，并包含某些beantalk文件以及各种系统级码头配置文件（例如更新版）的替换文件jetty.sh的脚本/etc/init.d设定。
码头豆茎测试程序
一个示例Web应用程序，展示了如何将来自另一个Maven模块的war文件与jetty-beanstalk-overlay结合在一起以生成启用了beantalk的应用程序包。还包括一些示例，说明如何更改诸如定制的东西的码头配置start.ini文件。

注意

测试Web应用程序需要访问test-jetty-webapp的快照版本，因此，它实际上更多地是一个示例，说明如何使用为beantalk和jetty定制应用程序所需的位来对Webapp进行分层。

要实际使用这些工件，您当前必须克隆此git存储库并在本地构建它。一旦有了工件，您只需要复制jetty-beanstalk-testapp中的方法即可将配置应用于您的webapp。

重要

Bluepill用于管理应用程序服务器的启动和停止过程。这似乎是一个有问题的软件，其历史悠久，撰写本文时所使用的版本已过时。启动和停止（或重新启动）应用服务器时，您可能会看到错误消息，表明服务器超时获得响应或类似信息。这些是红色鲱鱼，我的经验是码头已经开始和停止得很好，所需的pid文件以非常及时的方式显示（在/var/run/jetty.pid ），请检查应用服务器是否已启动，但是请注意，这里还没有解决一个奇怪的问题。
