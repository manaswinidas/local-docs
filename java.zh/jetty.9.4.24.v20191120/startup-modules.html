标准Jetty Distribution随附了几个模块，这些模块在${jetty.home}/modules/ 。模块与Jetty XML文件进行交互以配置服务器的选项和参数，并且是Jetty发行版的主要配置方法。模块为实现提供了灵活性，其即插即用的特性使添加或删除服务器功能几乎毫不费力。

启用模块

小费

默认分发有一个混合${jetty.home}和${jetty.base}目录在哪里${jetty.home}和${jetty.base}指向相同的位置。强烈建议您了解Jetty Base与Jetty Home的区别，并充分利用此设置。

启用模块是一个简单的过程：只需添加--add-to-start=,,...etc.命令行上的语法。这样做将启用指定的模块和任何相关模块。

现在，您已经配置了功能正常的服务器，尽管未部署任何Web应用程序。此时，您可以将webapp（war文件）放入mybase/webapps/目录并启动Jetty。

Start.ini与Start.d

在上面的示例中，当模块被激活时，该模块文件的内容被添加到${jetty.base}/start.ini 。添加其他模块后，其内容将附加到此文件中。如果要将所有模块配置都放在一个文件中，这可能会很有用，但是对于具有许多模块的大型服务器实例，快速查找和进行更改或删除模块可能会带来挑战。

替代单身start.ini您可以选择将模块包含在一个文件中${jetty.base}/start.d目录。当以下情况激活模块start.d目录存在时，将被每个模块存储为单个文件。以下是一个新鲜的例子${jetty.base}这将创建一个start.d目录并激活几个模块。

[jetty.home]$ mkdir mybase
[jetty.home]$ cd mybase/
[mybase]$ java -jar ../start.jar --create-startd

[mybase]$ ls -all

drwxr-xr-x@ 26 staff  staff  884 Aug 29 15:16 ..
drwxr-xr-x   6 staff  staff  204 Aug 29 15:19 start.d

[mybase]$ java -jar ../start.jar --add-to-start=server,client,webapp,websocket

注意

不建议同时使用${jetty.base}/start.ini文件和一个${jetty.base}/start.d目录，并且这样做可能会导致问题。

配置模块

为服务器启用模块后，可以对其进行进一步配置以满足您的需求。这是通过编辑模块的相关ini文件来完成的。如果您的服务器设置使用的是集中式ini配置，则您将编辑${jetty.base}/server.ini文件。如果您选择管理自己的ini文件中的每个模块，则可以在${jetty.base}/start.d目录。

重要

重要的是不要在$JETTY_HOME/modules目录。$ JETTY_HOME应该始终保持真理的标准。如果要更改实际的模块文件（而不是其中的值） ini-template ），请修改其关联的ini文件在$JETTY_BASE/start.d目录或复制所需的模块文件，然后将其复制到$JETTY_BASE目录并在那里进行编辑。start.jar读取本地$JETTY_BASE/modules扫描之前的文件（如果存在） $JETTY_HOME 。

激活模块后，默认情况下会设置许多属性。要查看这些默认值，请打开关联的ini文件。ini文件中列出了相关的模块文件以及可以设置的任何属性。

以下是一个示例requestlog.ini文件：

# ---------------------------------------
# Module: requestlog
--module=requestlog

## Logging directory (relative to $jetty.base)
# jetty.requestlog.dir=logs

## File path
# jetty.requestlog.filePath=${jetty.requestlog.dir}/yyyy_mm_dd.request.log

## Date format for rollovered files (uses SimpleDateFormat syntax)
# jetty.requestlog.filenameDateFormat=yyyy_MM_dd

## How many days to retain old log files
# jetty.requestlog.retainDays=90

## Whether to append to existing file
# jetty.requestlog.append=true

## Whether to use the extended log output
# jetty.requestlog.extended=true

## Whether to log http cookie information
# jetty.requestlog.cookies=true

## Timezone of the log entries
# jetty.requestlog.timezone=GMT

## Whether to log LogLatency
# jetty.requestlog.loglatency=false

第一行命名被调用的模块文件（位于{$jetty.home/modules} ）。随后的行列出了可以更改的属性以及每个属性的描述。要编辑属性，请先删除该行，然后删除该行的注释#在行的开头，然后在之后进行更改=标志（例如更改true重视false ）。

禁用模块

禁用模块很容易。要禁用模块，请注释掉--module=关联的ini文件中的一行。删除与模块关联的ini文件是另一种选择，但可能并非在所有情况下都可行。

列出可用和活动模块

要查看可用的模块， 请使用--list-modules命令行参数。此命令还将向您显示启用了哪些模块。模块按以下值排序[tags]相关的部分.mod文件。如果在[tags]部分，它按列表中的第一个标签排序。

注意

默认情况下， --list-modules命令行参数显示所有不包含的模块internal在里面[tags]相关的部分.mod文件。如果您想查看所有模块，请使用--list-all-modules

搜索模块

自从引入模块系统以来，已经添加了许多新模块。结果，从整体上看模块列表可能有点让人不知所措。要缩小要选择的模块范围，可以按[tags]部分。请注意，当您以这种方式搜索时，所有包含您的条件的模块[tags]将显示包括内部模块的部分。要在搜索特定模块标签时过滤掉内部模块，只需添加-internal到命令行。例如，如果您只想查看日志记录模块（内部实现模块除外），则可以使用--list-modules=logging,-internal 。

[mybase]$ java -jar $JETTY_HOME/start.jar --list-modules=logging,-internal
