要将Web应用程序或WAR部署到不同的环境中，很可能需要自定义Web应用程序以使其与每种环境兼容。面临的挑战是在不更改webapp本身的情况下做到这一点。您可以使用jetty.xml文件，因为它不是webapp的一部分。但是有一些变化jetty.xml例如，无法完成对servlet初始化参数和上下文初始化参数的修改。使用webdefault.xml不是一个选择，因为Jetty适用webdefault.xml在应用程序本身之前访问Web应用程序WEB-INF/web.xml ，这意味着它无法覆盖webapp的`web.xml`中的值。

解决方法是override-web.xml 。它是一个web.xml Jetty应用于Web应用程序之后的文件WEB-INF/web.xml ，这意味着它可以覆盖值或添加新元素。这是使用Jetty XML语法在每个Web应用程序的基础上定义的。

使用override-web.xml

您可以指定override-web.xml用于位于Jetty webapps文件夹中的可部署xml文件中的单个Web应用程序。例如，如果您有一个名为MyApp的Web应用程序，则将放置一个名为myapp.xml在${jetty.base}/webapps其中包括overrideDescriptor的条目override-web.xml文件。

<Configure class="org.eclipse.jetty.webapp.WebAppContext">
  ...
  <!-- Set up the path to the custom override descriptor,
  relative to your $(jetty.home) directory or to the current directory -->
  <Set name="overrideDescriptor"><SystemProperty name="jetty.home" default="."/>/my/path/to/override-web.xml</Set>
  ...
</Configure>

代码中的等效项是：

import org.eclipse.jetty.webapp.WebAppContext;

    ...

    WebAppContext wac = new WebAppContext();
    ...
    //Set the path to the override descriptor, based on your $(jetty.home) directory
    wac.setOverrideDescriptor(System.getProperty("jetty.home")+"/my/path/to/override-web.xml");
    ...

另外，您可以使用类加载器（ Jetty Classloading ）获取覆盖描述符的路径作为资源。

使用Jetty Maven插件

使用标签如下：

<project>
    ...
    <plugins>
        <plugin>
            ...
            <artifactId>jetty-maven-plugin</artifactId>
            <configuration>
                <webAppConfig>
                  ...
                  <overrideDescriptor>src/main/resources/override-web.xml</overrideDescriptor>
                </webAppConfig>
            </configuration>
        </plugin>
        ...
    </plugins>
    ...
</project>
