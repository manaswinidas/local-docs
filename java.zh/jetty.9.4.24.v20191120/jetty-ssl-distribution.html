使用Jetty发行版时，启用SSL支持与激活适当的模块一样容易。Jetty支持默认的JSSE提供程序和Conscrypt提供程序作为SSL实现。

默认JSSE SSL配置

要获得默认的SSL支持，只需激活ssl 模块 ：

$ cd /path/to/mybase
$ java -jar ${JETTY_HOME}/start.jar --add-to-startd=ssl
INFO : server          initialised (transitively) in ${jetty.base}/start.d/server.ini
INFO : ssl             initialised in ${jetty.base}/start.d/ssl.ini
INFO : Base directory was modified
$ tree
.
├── etc
│   └── keystore
└── start.d
    ├── server.ini
    └── ssl.ini

当你打开start.d/ssl.ini ，您将在配置时看到准备使用的几个注释属性SslContextFactory基本。

要突出显示一些更常用的属性：

jetty.ssl.host
配置SSL / TLS连接器应侦听的接口。
码头
配置SSL / TLS连接器应侦听的端口。
jetty.httpConfig.securePort
如果Web应用程序需要重定向到相同资源的安全版本，则此端口是响应上报告的端口location行（如果您在Jetty前面有东西，例如负载均衡器或代理，则将其分开很有用）。
jetty.sslContext.keyStorePath
设置位置keystore您使用证书配置的。
jetty.sslContext.keyStorePassword
设置密码keystore 。

加密SSL配置

启用Conscrypt SSL与默认SSL一样简单-启用conscrypt和ssl 模块 ：

$ cd ${JETTY_HOME}
$ java -jar ../start.jar --add-to-start=ssl,conscrypt

无需其他Conscrypt配置。SSL特定参数，例如keyStorePath和keyStorePassword仍然可以按照上面的示例进行配置${JETTY_BASE}/start.d/ssl.ini文件。

客户端证书认证

要在Jetty发行版中启用客户端证书认证，您需要同时启用ssl和https模块。

$ cd /path/to/mybase
$ java -jar /path/to/jetty-dist/start.jar --add-to-startd=ssl,https

$ JETTY_BASE / start.d / ssl.ini。 

# Module: ssl
--module=ssl

jetty.ssl.host=0.0.0.0
jetty.ssl.port=8583
jetty.sslContext.keyStorePath=etc/keystore
jetty.sslContext.trustStorePath=etc/truststore
jetty.sslContext.keyStorePassword=OBF:
jetty.sslContext.keyManagerPassword=OBF:
jetty.sslContext.trustStorePassword=OBF:
# Enable client certificate authentication.
jetty.sslContext.needClientAuth=true

$ JETTY_BASE / start.d / https.ini。 

# Module: https
--module=https
