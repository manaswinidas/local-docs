为FastCGI配置Jetty

配置Jetty以将HTTP / 2代理到FastCGI

在本节中，您将看到如何配置Jetty通过FastCGI服务WordPress。

第一步是在服务器计算机上安装WordPress，例如在/var/www/wordpress 。有关如何安装WordPress的更多信息，请参阅WordPress安装指南 。

第二步是安装php-fpm并确保将其配置为侦听TCP套接字；通常将其配置为收听localhost:9000 。

第三步是安装Jetty，例如在/opt/jetty ，在以下调用$JETTY_HOME 。请参阅下载Jetty ，以获取有关如何安装Jetty的更多信息。

第四步是创建一个Jetty基本目录（请参阅管理Jetty Base和Jetty Home ），在下面调用$JETTY_BASE ，您可以在其中设置在Jetty中支持FastCGI所需的配置，并配置fcgi ， http和deploy模块，以便Jetty能够接受来自浏览器的HTTP请求，在FastCGI中对其进行转换，并将其代理到php-fpm ：

$ mkdir -p /usr/jetty/wordpress
$ cd /usr/jetty/wordpress
$ java -jar $JETTY_HOME/start.jar --add-to-start=fcgi,http,deploy

因此$JETTY_BASE=/usr/jetty/wordpress 。

第五步是部署Web应用程序，该应用程序将客户端请求的代理提供给FastCGI服务器， php-fpm 。通常，这是通过部署一个*.war文件在$JETTY_BASE/webapps目录。对于FastCGI，不需要开发任何Web应用程序-Jetty已经为您完成了所有工作。这样，您只需部署直接配置Web应用程序的Jetty上下文XML文件。将以下内容复制并粘贴为$JETTY_BASE/webapps/jetty-wordpress.xml ：

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE Configure PUBLIC "-//Jetty//Configure//EN" "http://www.eclipse.org/jetty/configure_9_3.dtd">
<Configure class="org.eclipse.jetty.servlet.ServletContextHandler">

    <New id="root" class="java.lang.String">
        <Arg>/var/www/wordpress</Arg>
    </New>

    <Set name="contextPath">/</Set>
    <Set name="resourceBase"><Ref refid="root" /></Set>
    <Set name="welcomeFiles">
        <Array type="string"><Item>index.php</Item></Array>
    </Set>

    <Call name="addFilter">
        <Arg>org.eclipse.jetty.fcgi.server.proxy.TryFilesFilter</Arg>
        <Arg>/*</Arg>
        <Arg>
            <Call name="of" class="java.util.EnumSet">
                <Arg><Get name="REQUEST" class="javax.servlet.DispatcherType" /></Arg>
            </Call>
        </Arg>
        <Call name="setInitParameter">
            <Arg>files</Arg>
            <Arg>$path /index.php?p=$path</Arg>
        </Call>
    </Call>

    <Call name="addServlet">
        <Arg>
            <New class="org.eclipse.jetty.servlet.ServletHolder">
                <Arg>default</Arg>
                <Arg>
                    <Call name="forName" class="java.lang.Class">
                        <Arg>org.eclipse.jetty.servlet.DefaultServlet</Arg>
                    </Call>
                </Arg>
                <Call name="setInitParameter">
                    <Arg>dirAllowed</Arg>
                    <Arg>false</Arg>
                </Call>
            </New>
        </Arg>
        <Arg>/</Arg>
    </Call>

    <Call name="addServlet">
        <Arg>org.eclipse.jetty.fcgi.server.proxy.FastCGIProxyServlet</Arg>
        <Arg>*.php</Arg>
        <Call name="setInitParameter">
            <Arg>proxyTo</Arg>
            <Arg>http://localhost:9000</Arg>
        </Call>
        <Call name="setInitParameter">
            <Arg>prefix</Arg>
            <Arg>/</Arg>
        </Call>
        <Call name="setInitParameter">
            <Arg>scriptRoot</Arg>
            <Arg><Ref refid="root" /></Arg>
        </Call>
        <Call name="setInitParameter">
            <Arg>scriptPattern</Arg>
            <Arg>(.+?\\.php)</Arg>
        </Call>
    </Call>

</Configure>

以上内容的说明：

  • Linne 6在此示例中指定WordPress安装目录/var/www/wordpress （如第一步所定义）。
  • 第9行指定了将在其中提供WordPress的上下文路径，在此示例中为根上下文路径/ 。
  • 第10行指定上下文的资源库，也设置为WordPress安装目录。这使Jetty可以直接从WordPress安装目录提供静态资源。
  • 第12行将欢迎文件指定为index.php ，这样在URI以开头的情况下，Jetty可以执行正确的重定向/字符。
  • 第15行指定了TryFilesFilter ，是Servlet过滤器，受Nginx提供的try_files功能启发。该过滤器尝试首先从文件系统提供资源，如果找不到该资源，则将请求转发为index.php?p=$path ，它将与下面定义的代理Servlet匹配。有关更多信息，请参考TryFilesFilter文档。
  • 第29行指定了Jetty的DefaultServlet提供静态内容，例如CSS文件，JavaScript文件等。 DefaultServlet将通过在第10行定义的上下文资源库中查找这些文件（请参见上文）。
  • 第47行指定了FastCGIProxyServlet ，一个Servlet，它将来自客户端的HTTP请求代理为对FastCGI服务器的FastCGI请求。
  • 第52行指定FastCGI服务器的TCP地址（ php-fpm ），其中HTTP请求作为FastCGI请求转发。
  • 第60行再次指定WordPress安装目录，以便FastCGIProxyServlet可以将此信息传递给FastCGI服务器。
  • 除了第49行中Servlet定义的标准URL映射之外，第64行还指定了一个正则表达式，该正则表达式与对此Servlet执行的请求URI匹配。有关更多信息，请参阅FastCGIProxyServlet文档。

最后一步是启动Jetty（请参阅第9章， 启动Jetty ）并导航到http://localhost:8080使用您的浏览器并享受WordPress：

$ cd $JETTY_BASE
$ java -jar /opt/jetty/start.jar

配置Jetty以将HTTP / 2代理到FastCGI

为了将Jetty配置为侦听来自启用HTTP / 2的客户端的HTTP / 2请求，并将它们作为FastCGI请求转发到FastCGI服务器，您需要启用http2模块，这将需要TLS连接器，并因此需要一个密钥库来读取TLS所需的密钥材料。

启用http2简单;除了上述已启用的模块之外， http2模块：

$ cd $JETTY_BASE
$ java -jar $JETTY_HOME/start.jar --add-to-start=http2

上面的命令添加了http2模块（及其依赖项）到现有模块，并使用默认的Jetty密钥库来提供TLS所需的密钥材料。您将要使用自己的密钥库以及自己的域的私有密钥和证书。

请记住，通过添加http2模块，将启动两个JVM：一个读取配置，另一个在引导类路径中具有ALPN引导启动jar，如配置HTTP / 2中所述 。

由于现在您的网站将在TLS上运行，因此您需要确保WordPress URL也已配置。如果您已按照上一节的步骤进行操作，则可以在以下位置访问WordPress网站http://localhost:8080 。您需要将其更改为https://localhost:8443从WordPress管理Web界面，或按照WordPress的说明进行操作，而无需使用管理Web界面。

因此，在HTTP / 2上使用Jetty运行WordPress所需的最少模块是： http2 ， http ， fcgi和deploy 。这些将在端口8080上为HTTP / 1.1设置明文连接器，并在端口8443上为HTTP / 2和HTTP / 1.1设置TLS连接器。

此时，您可以启动Jetty（请参阅第9章， 启动Jetty ），点击http://localhost:8080使用您的浏览器，并使用支持HTTP / 2的浏览器通过HTTP / 2享受WordPress：

$ cd $JETTY_BASE
$ java -jar $JETTY_HOME/start.jar

如果您没有启用HTTP / 2的浏览器，则WordPress仍可通过HTTP / 1.1使用。
