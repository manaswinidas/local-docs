连接器是Jetty通过各种机制接受网络连接的机制。配置连接器是配置以下各项的组合：

  • 连接器本身上的网络参数（例如：侦听端口）。
  • 连接器使用的服务（例如：执行程序，调度程序）。
  • 连接工厂实例化和配置用于接受连接的协议。

通常，除了设置监听端口并启用外，连接器只需要很少的配置。 X-Forwarded-For自定义（如果适用）。其他设置（包括构造自己的构造函数Jetty XML文件）仅用于专家配置。

启用模块后，它们会加载几个默认选项。可以通过编辑关联的模块ini文件来更改这些设置。 start.d目录（或其中的相关行server.ini如果您的实现不使用start.d ）。例如，如果我们检查http.ini归档在我们的start.d在上面创建的目录中，我们将看到以下设置：

# ---------------------------------------
# Module: http
# Enables a HTTP connector on the server.
# By default HTTP/1 is support, but HTTP2C can
# be added to the connector with the http2c module.
# ---------------------------------------
--module=http

### HTTP Connector Configuration

## Connector host/address to bind to
# jetty.http.host=0.0.0.0

## Connector port to listen on
# jetty.http.port=8080

## Connector idle timeout in milliseconds
# jetty.http.idleTimeout=30000

## Number of acceptors (-1 picks default based on number of cores)
# jetty.http.acceptors=-1

## Number of selectors (-1 picks default based on number of cores)
# jetty.http.selectors=-1

## ServerSocketChannel backlog (0 picks platform default)
# jetty.http.acceptorQueueSize=0

## Thread priority delta to give to acceptor threads
# jetty.http.acceptorPriorityDelta=0

## HTTP Compliance: RFC7230, RFC2616, LEGACY
# jetty.http.compliance=RFC7230

要更改这些设置，请取消注释该行（通过删除＃）并将属性更改为所需的值。例如，如果要将HTTP端口更改为5231，则可以按如下所示编辑行：

限制连接

Jetty还提供了限制与服务器和/或上下文的连接的方法。这由分发中的两个不同模块提供。

connectionlimit
对连接数施加限制。如果超过此限制，则新连接将暂停指定的时间（以毫秒为单位）。
acceptratelimit
限制接受新连接的速率。如果超过此限制，则新连接将暂停指定的时间（以毫秒为单位）。

与上面列出的模块一样，可以通过添加启用这些模块--add-to-start=到命令行。

进阶设定

Jetty主要使用一种称为ServerConnector的单一连接器类型。

在Jetty 9之前，连接器的类型既指定了协议，也指定了所使用的实现。例如，基于选择器的非阻塞I / O与阻塞I / O，或者SSL连接器与非SSL连接器。Jetty 9具有一个基于选择器的非阻塞I / O连接器，并且包含一个ConnectionFactories现在在连接器上配置协议。

标准的Jetty发行版随附以下用于创建和配置连接器的Jetty XML文件；阅读本节时，应检查它们：

jetty-http.xml
实例化一个ServerConnector接受HTTP连接（可以升级为WebSocket连接）。
jetty-ssl.xml
实例化一个ServerConnector接受SSL / TLS连接。单独而言，此连接器无法正常工作，并且还需要配置以下一个或多个文件以添加ConnectionFactories使连接器正常工作。
jetty-https.xml
添加一个HttpConnectionFactory到ServerConnector由配置jetty-ssl.xml两者结合在一起以提供对HTTPS的支持。
jetty-http-forwarded.xml
添加一个ForwardedRequestCustomizer到HTTP连接器以处理来自代理的转发样式标头。
jetty-http2.xml
添加一个Http2ServerConnectionFactory到ServerConnector由配置jetty-ssl.xml支持http2协议。
jetty-alpn.xml
添加一个ALPNServerConnectionFactory到ServerConnector由配置jetty-ssl.xml它允许一个SSL连接器支持ALPN扩展名用于选择每个连接要使用的协议的多个协议。

构造一个ServerConnector

服务一ServerConnector实例使用由构造函数注入设置，实例化后无法更改。许多服务的默认值可能为null或0，以便使用合理的默认值，因此对于大多数用途，仅需要将Server和连接工厂传递给连接器构造函数。在Jetty XML中（即，在jetty-http.xml ），您可以通过以下方式进行操作：

<New class="org.eclipse.jetty.server.ServerConnector">
  <Arg name="server"><Ref refid="Server" /></Arg>
  <Arg name="factories">
    <Array type="org.eclipse.jetty.server.ConnectionFactory">
      <!-- insert one or more factories here -->
    </Array>
  </Arg>
  <!-- set connector fields here -->
</New>

您可以看到构造一个ServerConnector在Javadoc中 。通常，默认值几乎适用于几乎所有部署。

网络设置

您可以通过在启动连接器之前调用连接器上的设置器来配置连接器网络设置。例如，您可以使用Jetty XML设置端口：

<New class="org.eclipse.jetty.server.ServerConnector">
  <Arg name="server"><Ref refid="Server" /></Arg>
  <Arg name="factories"><!-- insert one or more factories here --></Arg>

  <Set name="port">8080</Set>
</New>

Jetty XML中的值也可以被参数化，以便它们可以从属性文件中传递或在命令行上设置。因此，通常在Jetty XML中设置端口，但使用Property可自定义的元素：

<New class="org.eclipse.jetty.server.ServerConnector">
  <Arg name="server"><Ref refid="Server" /></Arg>
  <Arg name="factories"><!-- insert one or more factories here --></Arg>

  <Set name="port"><Property name="jetty.http.port" default="8080"/></Set>
</New>

HTTP配置

的HttpConfiguration类保存以下配置HttpChannel s，您可以为每个HTTP连接创建1：1，或在多路HTTP / 2连接上创建1：n。因此HttpConfiguration对象被注入到HTTP和HTTP / 2连接工厂中。为避免重复配置，标准Jetty发行版创建了通用HttpConfiguration实例在jetty.xml ，这是一个Ref然后用于jetty-http.xml ， jetty-https.xml和在jetty-http2.xml 。

HttpConfiguration的典型配置为：

<New id="httpConfig" class="org.eclipse.jetty.server.HttpConfiguration">
  <Set name="secureScheme">https</Set>
  <Set name="securePort"><Property name="jetty.ssl.port" default="8443" /></Set>
  <Set name="outputBufferSize">32768</Set>
  <Set name="requestHeaderSize">8192</Set>
  <Set name="responseHeaderSize">8192</Set>
</New>

此示例HttpConfiguration可以通过引用ID“ httpConfig “：

<Call name="addConnector">
  <Arg>
    <New class="org.eclipse.jetty.server.ServerConnector">
      <Arg name="server"><Ref refid="Server" /></Arg>
      <Arg name="factories">
        <Array type="org.eclipse.jetty.server.ConnectionFactory">
          <Item>
            <New class="org.eclipse.jetty.server.HttpConnectionFactory">
              <Arg name="config"><Ref refid="httpConfig" /></Arg>
            </New>
          </Item>
        </Array>
      </Arg>
      <!-- ... -->
    </New>
  </Arg>
</Call>

这都儿一样httpConfig被引用SecuredRedirectHandler重定向安全请求时。请注意，如果您的httpConfig不包括secureScheme要么securePort或没有HttpConfiguration目前，这些类型的安全请求将被返回403错误。

对于基于SSL的连接器（在jetty-https.xml和jetty-http2.xml ），常见的“ httpConfig “实例被用作创建具有ID的SSL特定配置的基础” sslHttpConfig “：

<New id="sslHttpConfig" class="org.eclipse.jetty.server.HttpConfiguration">
  <Arg><Ref refid="httpConfig"/></Arg>
  <Call name="addCustomizer">
    <Arg><New class="org.eclipse.jetty.server.SecureRequestCustomizer"/></Arg>
  </Call>
</New>

这增加了一个SecureRequestCustomizer它将SSL会话ID和证书信息添加为请求属性。

SSL上下文配置

HTTPS和HTTP / 2的SSL / TLS连接器需要证书才能建立安全连接。Jetty在标准JVM密钥库中保存证书，并在服务器上配置为密钥库和信任库。 SslContextFactory.Server被注入到SslConnectionFactory实例。使用与Jetty一起分发的密钥库（包含自签名测试证书）的示例在jetty-https.xml 。在配置SSL中阅读有关SSL密钥库的更多信息。

代理/负载平衡器连接配置

通常，连接器需要配置为接受来自中介（例如，反向代理和/或部署在服务器前的负载平衡器）的连接。在这种环境中，服务器上终止的TCP / IP连接不是源自客户端，而是来自中介，因此，远程IP和端口号可能在日志中错误地报告，并且在某些情况下，错误的服务器地址和端口可能会错误地报告。使用。

因此，中介通常实现几种事实上的标准之一，以将与终止于中介上的原始客户端连接有关的信息传达给服务器。码头支持X-Forwarded-For标头和代理协议机制，如下所述。

注意

Jetty发行版中的XML文件包含两个注释示例。 X-Forwarded-For和代理协议机制。使用这些示例时，建议不要编辑Jetty发行版中的XML。而是应将文件复制到Jetty基本目录中，然后进行修改。

X转发配置

的X-Forwarded-for标头和关联的标头是事实上的标准，其中中介程序将HTTP标头添加到它们转发来描述原始连接的每个请求中。这些标头可以由以下实例解释ForwardedRequestCustomizer可以添加到HttpConfiguration如下：

<New id="httpConfig" class="org.eclipse.jetty.server.HttpConfiguration">
  <Set name="outputBufferSize">32768</Set>
  <Set name="requestHeaderSize">8192</Set>
  <Set name="responseHeaderSize">8192</Set>

  <Call name="addCustomizer">
    <Arg><New class="org.eclipse.jetty.server.ForwardedRequestCustomizer"/></Arg>
  </Call>
</New>

代理协议

代理协议是HAProxy创建的事实上的标准，并由Amazon Elastic Cloud等环境使用。此机制独立于任何协议，因此可用于HTTP2，TLS等。有关客户端连接的信息将作为每个新建立的连接上的小数据帧发送。在Jetty中，此协议可以由ProxyConnectionFactory解析数据帧，然后实例化下一个ConnectionFactory在连接上具有已用有关原始客户端连接的数据定制的端点。连接工厂可以添加到任何ServerConnector并且应该是第一个ConnectionFactory 。

将工厂添加到HTTP连接器的示例如下所示：

<Call name="addConnector">
  <Arg>
    <New class="org.eclipse.jetty.server.ServerConnector">
      <Arg name="server"><Ref refid="Server" /></Arg>
      <Arg name="factories">
        <Array type="org.eclipse.jetty.server.ConnectionFactory">
          <Item>
            <New class="org.eclipse.jetty.server.ProxyConnectionFactory"/>
          </Item>
          <Item>
            <New class="org.eclipse.jetty.server.HttpConnectionFactory">
              <Arg name="config"><Ref refid="httpConfig" /></Arg>
            </New>
          </Item>
        </Array>
      </Arg>
      <Set name="host"><Property name="jetty.host" /></Set>
      <Set name="port"><Property name="jetty.http.port" default="80" /></Set>
    </New>
  </Arg>
</Call>
