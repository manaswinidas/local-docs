标题过滤器

信息
用法

信息

  • 班级名称：org.eclipse.jetty.servlets.HeaderFilter
  • Maven工件：org.eclipse.jetty：jetty-servlets
  • Javadoc： http : //www.eclipse.org/jetty/javadoc/9.4.24.v20191120/org/eclipse/jetty/servlets/HeaderFilter.html

用法

标头过滤器基于可选的包含/排除的路径规范，mime类型和/或HTTP方法列表，将标头设置或添加到每个响应。该过滤器在调用之前处理其配置的标头doFilter在过滤器链中。在此过滤器中配置的某些标头可能会被其他过滤器和/或处理请求的servlet覆盖。

必需的JAR

要使用标题过滤器，这些JAR文件必须在WEB-INF / lib中可用：

  • $ JETTY_HOME / lib / jetty-http.jar
  • $ JETTY_HOME / lib / jetty-servlets.jar
  • $ JETTY_HOME / lib / jetty-util.jar

样本配置

将配置放入Webapp的web.xml要么jetty-web.xml 。该过滤器将对每个响应执行以下操作：

  • 将X-Frame-Options标头设置为DENY。
  • 添加一个包含无缓存，无存储，必须重新验证的Cache-Control标头
  • 将Expires标头设置为大约未来一年。
  • 添加带有当前系统时间的Date标头。

注意

每个动作必须用逗号分隔。

<filter>
   <filter-name>HeaderFilter</filter-name>
   <filter-class>org.eclipse.jetty.servlets.HeaderFilter</filter-class>
   <init-param>
     <param-name>headerConfig</param-name>
     <param-value>
      set X-Frame-Options: DENY,
      "add Cache-Control: no-cache, no-store, must-revalidate",
      setDate Expires: 31540000000,
      addDate Date: 0
     </param-value>
   </init-param>
 </filter>

配置标题过滤器参数

下列init参数控制过滤器的行为：

includedPaths
可选的。包含的路径规范的逗号分隔值。
excludesPaths
可选的。以逗号分隔的排除路径规范的值。
includedMimeTypes
可选的。包含的mime类型的逗号分隔值。如果未在响应中设置内容类型，则会从请求URL末尾的扩展名中猜测出MIME类型。
excludeMimeTypes
可选的。逗号分隔的排除的MIME类型的值。如果未在响应中设置内容类型，则会从请求URL末尾的扩展名中猜测出MIME类型。
includedHttpMethods
可选的。包含的http方法的逗号分隔值。
excludeHttpMethods
可选的。逗号分隔的排除http方法的值。
headerConfig
要在标头上执行的动作的逗号分隔值。每个动作的语法是action headerName: headerValue 。

支持的标头操作：

  • set -原因集setHeader被响应
  • add -原因集addHeader被响应
  • setDate -原因setDateHeader在响应中被调用。
  • addDate -原因addDateHeader在响应中被调用。

如果setDate要么addDate用来， headerValue应为写入标头值之前要添加到当前系统时间的毫秒数。

如果过滤器配置同时包含和排除属性，则将其视为已排除。

路径规格规则：

  • 如果规格以^ ，则该规范假定为基于正则表达式的路径规范，并将与常规Java regex规则匹配。
  • 如果规格以/ ，则该规范假定为完全匹配或基于前缀的匹配的Servlet url-pattern规则路径规范。
  • 如果规格以*. ，则该规范假定为基于后缀的匹配的Servlet url-pattern规则路径规范。
  • 不支持所有其他语法。
