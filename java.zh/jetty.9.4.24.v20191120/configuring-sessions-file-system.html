持久会话：文件系统

启用文件系统会话
配置文件系统会话属性

注：坚持会话的本地文件系统不应该在集群环境中使用。

启用文件系统会话

使用Jetty发行版时，您首先需要启用session-store-file您的Jetty基础 模块 --add-to-start命令行上的参数。

$ java -jar ../start.jar --create-startd
INFO : Base directory was modified

$ java -jar ../start.jar --add-to-start=session-store-file
INFO  : server          transitively enabled, ini template available with --add-to-start=server
INFO  : sessions        transitively enabled, ini template available with --add-to-start=sessions
INFO  : session-store-file initialized in ${jetty.base}/start.d/session-store-file.ini
MKDIR : ${jetty.base}/sessions
INFO  : Base directory was modified

这样做将启用文件系统会话模块以及使其在服务器上运行所需的任何相关模块或文件。上面的示例使用的是新鲜的${jetty.base}没有启用任何其他功能。

当。。。的时候--add-to-start参数已添加到命令行，它启用了session-store-file模块以及sessions和server模块，这是文件系统会话管理运行所必需的。另外一个${jetty.base}/sessions目录已创建。默认情况下，会话文件将保存到此目录。

除了将这些模块添加到服务器的类路径之外，还向ini添加了一些ini配置文件。 ${jetty.base}/start.d目录。

注意

现在仅在请求时才加载会话数据。以前的功能，例如setLazyLoad已被删除。

配置文件系统会话属性

开场start.d/session-store-file.ini将显示文件系统会话模块的所有可配置选项的列表：

# ---------------------------------------
# Module: session-store-file
# Enables session persistent storage in files.
# ---------------------------------------
--module=session-store-file

jetty.session.file.storeDir=${jetty.base}/sessions
#jetty.session.file.deleteUnrestorableFiles=false
#jetty.session.savePeriod.seconds=0
jetty.session.storeDir
这定义了会话文件的存储位置。
jetty.session.file.deleteUnrestorableFiles
布尔值。如果设置为true，将删除不可读的文件：当清除程序定期（重新）尝试为会话加载损坏的信息以使其过期时，这将有助于防止重复记录同一错误。
jetty.session.savePeriod.seconds = 0

默认情况下，每当最后一个并发请求离开会话时，该会话始终通过SessionDataStore ，即使会话中唯一更改的是其更新的上次访问时间。非零值表示SessionDataStore如果仅更改了访问时间，并且少于savePeriod自上次写入会话以来的秒数。

注意

配置中savePeriod如果您的持久性技术的写入速度非常慢/成本很高，则此功能很有用。在群集环境中，存在会话的上次访问时间在共享存储中的过期时间可能长达9天。 savePeriod秒。这允许一个节点可能会过早地终止会话，即使该会话正在被另一个节点使用。彻底考虑maxIdleTime设置会话的时间savePeriod势在必行-设置一个savePeriod大于maxIdleTime 。
