Java WatchService

补救

JVM WatchService可以监视目录等对象的更改，然后更新其内容并将这些更改通知应用程序。此服务对于热部署等功能很有用。当检测到变化时， WatchService会输入一个“安静时间”，在等待更改（一个或多个更改）并完成之前，将通知更改应用程序。

示例：将新的war文件复制到/webapps 。的WatchService可以（取决于实现方式）看到文件已创建（已注册为事件！，并且文件的大小正在增长（另一个事件））。在安静的时间中，每个事件都在超时之后门控，然后再将汇总的事件发送到应用程序。

尽管某些操作系统（例如Windows）在此安静时间内具有本机价值，但并非所有操作系统都可以，尤其是OSX。从根本上讲，这是对JVM的FileSystem特定实现的限制，但已经引起了项目的注意。

补救

为了帮助补偿OSX等系统中的延迟，Jetty将非本机实现的默认值设置为5000ms。不建议使用小于5000ms的值，并且该值经常会失败。
