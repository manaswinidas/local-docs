Jetty HTTP客户端模块提供了易于使用的API和实用程序类，以执行HTTP（或HTTPS）请求。

Jetty的HTTP客户端是非阻塞且异步的。它提供了一个永不阻塞I / O的异步API，使其在线程利用率方面非常高效，非常适合诸如负载测试或并行计算之类的高性能场景。

但是，当您需要做的是GET请求资源时，Jetty的HTTP客户端还提供了一个同步API；一个编程接口，发出请求的线程将阻塞，直到请求/响应对话完成为止。

Jetty的HTTP客户端支持不同的传输方式 ：HTTP / 1.1，FastCGI和HTTP / 2。这表示HTTP请求的语义（即“ GET我资源/index.html ”）可以通过网络以不同的格式传送。最常见的默认格式是HTTP / 1.1。也就是说，Jetty的HTTP客户端可以使用FastCGI格式或新的HTTP / 2格式来承载相同的请求。

FastCGI传输在Jetty的FastCGI支持中大量使用，该支持使Jetty可以作为PHP的反向代理（就像Apache或Nginx一样），因此可以服务（例如）WordPress网站。

HTTP / 2传输允许Jetty的HTTP客户端使用HTTP / 2到启用HTTP / 2的网站执行请求，另请参阅Jetty的HTTP / 2支持 。

Jetty HTTP客户端提供的现成功能包括：

  • 重定向支持-自动遵循重定向代码，例如302或303。
  • Cookies支持-服务器发送的cookie被存储并根据匹配的请求发送回服务器。
  • 身份验证支持-支持HTTP“基本”和“摘要”身份验证，其他身份验证可插入。
  • 转发代理支持-HTTP代理和SOCKS4代理。

启动HttpClient

主类命名org.eclipse.jetty.client.HttpClient 。

你可以想到一个HttpClient实例作为浏览器实例。就像浏览器一样，它可以向不同的域发出请求，可以管理重定向，Cookie和身份验证，您可以使用代理对其进行配置，还可以为您提供对请求的响应。

为了使用HttpClient ，您必须实例化它，对其进行配置，然后再启动它：

// Instantiate HttpClient
HttpClient httpClient = new HttpClient();

// Configure HttpClient, for example:
httpClient.setFollowRedirects(false);

// Start HttpClient
httpClient.start();

您可以创建多个实例HttpClient ，但通常一个实例对于一个应用程序就足够了。有多个原因有很多HttpClient实例包括但不限于：

  • 您想要指定不同的配置参数（例如，一个实例配置有转发代理，而另一实例则没有）
  • 您希望两个实例的行为类似于两个不同的浏览器，因此具有不同的cookie，不同的身份验证凭据等。
  • 您想使用其他交通工具

当您创建一个HttpClient使用无参数构造函数的实例，您将只能执行纯HTTP请求，而不能执行HTTPS请求。

为了执行HTTPS请求，您应该先创建一个SslContextFactory.Client ，对其进行配置，然后将其传递给HttpClient构造函数。当使用SslContextFactory ， HttpClient将能够对任何域执行HTTP和HTTPS请求。

// Instantiate and configure the SslContextFactory
SslContextFactory.Client sslContextFactory = new SslContextFactory.Client();

// Instantiate HttpClient with the SslContextFactory
HttpClient httpClient = new HttpClient(sslContextFactory);

// Configure HttpClient, for example:
httpClient.setFollowRedirects(false);

// Start HttpClient
httpClient.start();

停止HttpClient

建议您在应用程序停止时也停止HttpClient您正在使用的一个或多个实例。

httpClient.stop();

正在停止HttpClient确保释放它所保存的内存（例如，身份验证凭据，Cookie等），并确保线程池和调度程序已正确停止，以允许由HttpClient退出。
