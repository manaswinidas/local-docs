WebSocket API
启用WebSocket

WebSocket是用于双向通信的新协议，该协议通过HTTP / 1.1升级启动，并提供基于TCP分层的基本消息框架。它基于低级框架协议，该协议以UTF-8 TEXT或BINARY格式传递消息。

WebSocket中的单个消息可以是任意大小（但是基础帧的单个帧限制为63位 ）。可以发送无限数量的消息。消息是按顺序发送的，基本协议不支持交错消息。

Jetty提供什么

Jetty提供了以下标准和规范的实现。

RFC-6455

WebSocket协议

我们支持已发布和最终规范的版本13。

Jetty使用高速公路测试套件测试其WebSocket协议实现。

重要

Jetty 7和Jetty 8支持WebSocket的早期草稿，但Jetty 9已删除了该支持。这意味着Jetty 9将不支持实现WebSocket早期草案的旧浏览器。 （例如Safari 5.0或Opera 12）

小费

是否想知道您定位的浏览器是否支持WebSocket？使用caniuse.com/websockets进行查找。

JSR-356

Java WebSocket API（ javax.websocket ）

这是用于WebSockets的官方Java API。

不稳定的标准和规格：

逐帧压缩

每帧压缩扩展。

Google / Chromium小组的早期扩展草案，它将提供WebSocket框架压缩。许多版本的Chrome / Chromium中都使用deflate算法进行perframe压缩。

Jetty对perframe-compression的支持基于04草案规范。该标准已被permessage-compression取代。

压迫感

每帧压缩扩展。

这是逐帧压缩的替代，将压缩切换为基于整个消息，而不是单个帧。
