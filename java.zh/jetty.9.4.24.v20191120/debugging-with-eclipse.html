使用Eclipse进行调试

与Eclipse链接
在Eclipse中

有很多选项可用于在Eclipse中调试应用程序。

如果尚未完成，请按照以下说明为远程调试准备应用程序： 启用远程调试

与Eclipse链接

接下来，我们需要将Eclipse项目与已部署的Web应用程序链接。

  1. 在Eclipse中，右键单击包含已部署到jetty的web应用程序的项目，然后选择Debug→Debug Configurations并创建Remote Java Application的新配置。确保选择的端口与在“ 启用远程调试”中添加的端口相同。

  2. 接下来，在Web应用程序中，您可以在Servlet中设置一个断点，当它被触发时，它将暂停远程jvm的处理线程，以等待从Eclipse实例调试命令。

  3. 指向您的远程调试配置的Jetty分布，在浏览器中访问该servlet，应将Eclipse实例转换为标准的Debug视图。

在Eclipse中

由于Jetty的嵌入非常简单，因此许多人选择创建一个小的main它们可以直接在Eclipse中启动的方法，以便更轻松地调试整个应用程序。开始使用此方法的最佳位置是浏览]和xref：embedded-examples []部分。

一旦定义了用于启动应用程序的主要方法，请右键单击源文件，然后选择Debug As→Java Application 。在Eclipse的“ 控制台”选项卡中，您应该看到应用程序启动，完成启动后，您应该能够配置断点并通过Web浏览器正常访问Jetty实例。

小费

您可以通过以下方式轻松配置日志记录jetty-logging.properties文件。如果此文件在您的类路径中，则Jetty将使用它来配置日志记录，我们在Jetty开发过程中广泛使用了这种方法，它使工作变得更加轻松。您可以在jetty-logging.properties文件部分中看到此操作。
