在未安装发行版的情况下使用Jetty

制备

的想法jetty-runner非常简单–使用单个jar文件和尽可能多的默认配置，直接从命令行运行webapp。当然，如果您的Web应用程序不那么简单， jetty-runner具有命令行选项，可用于自定义执行环境。

制备

您将需要jetty-runner罐：

  1. 下载jetty-runner jar在Maven Central可用。

部署简单上下文

假设我们有一个非常简单的Web应用程序，它不需要环境中的任何资源，也不需要除默认设置以外的任何配置。启动它就像执行以下操作一样简单：

> java -jar jetty-runner.jar simple.war

这将在端口8080上启动Jetty，并将webapp部署到/ 。

您的webapp不必卷入战争，您可以使用相同的方法部署作为目录的webapp：

> java -jar jetty-runner.jar simple

实际上，webapp不必是战争甚至目录，它可以只是描述您的webapp的Jetty上下文xml文件：

> java -jar jetty-runner.jar simple-context.xml

注意

使用上下文xml文件时，甚至不需要将要部署的应用程序作为完整的Webapp。它可以仅仅是一个Jetty 上下文 。

默认， jetty-runner实现所有配置类，以便用户可以使用尽可能少的配置来设置和部署新实例。如果仅希望实现某些配置类，则需要在webapp / context的上下文xml中定义它们。默认配置类为：

org.eclipse.jetty.webapp.WebInfConfiguration org.eclipse.jetty.webapp.WebXmlConfiguration org.eclipse.jetty.webapp.MetaInfConfiguration org.eclipse.jetty.webapp.FragmentConfiguration org.eclipse.jetty.webapp.JettyWebXmlConfiguration org.eclipse.jetty.plus.webapp.EnvConfiguration org.eclipse.jetty.plus.webapp.PlusConfiguration org.eclipse.jetty.annotations.AnnotationConfiguration

您可以在此处了解有关实现特定配置类的更多信息。

部署多个上下文

如果必须部署多个Web应用程序，只需在命令行上全部提供即可。您可以使用以下命令控制它们的上下文路径--path参数。这是部署2次Wars的示例（尽管其中一个或两个都可以是解压缩的目录）：

> java -jar jetty-runner.jar --path /one my1.war --path /two my2.war

如果您有描述Web应用程序的上下文xml文件，则可以在其中完全配置Web应用程序，因此无需使用命令行开关。只需提供上下文文件列表即可，如下所示：

> java -jar jetty-runner.jar my-first-context.xml my-second-context.xml my-third-context.xml

注意

命令行上使用的切换覆盖配置文件设置。因此，例如，您可以在上下文xml文件中设置webapp的上下文路径，然后使用--path切换以在命令行上覆盖它。

更改默认端口

默认情况下jetty-runner将在端口8080上侦听。您可以使用以下命令在命令行上轻松更改此设置--port命令。这是一个在端口9090上运行simple.war的示例：

> java -jar jetty-runner.jar --port 9090 simple.war

使用jetty.xml文件

除了使用命令行开关外，您还可以使用一个或多个命令行开关jetty.xml文件来配置您的Web应用程序的环境。这是一个示例，我们应用两个不同的jetty.xml文件：

> java -jar jetty-runner.jar --config jetty.xml --config jetty-https.xml simple.war

您可以使用--help开关：

> java -jar jetty-runner.jar --help

打印版本

打印出Jetty的版本，然后立即退出。

> java -jar jetty-runner.jar --version

配置请求日志

使Jetty编写具有给定名称的请求日志。如果文件带有前缀yyyy_mm_dd然后文件将自动翻转。请注意，为了更精细地配置请求日志 ，您将需要使用Jetty xml文件。

> java -jar jetty-runner.jar --log yyyy_mm_dd-requests.log  my.war

配置输出日志

将Jetty日志的输出重定向到命名文件。如果文件带有前缀yyyy_mm_dd然后文件将自动翻转。

> java -jar jetty-runner.jar --out yyyy_mm_dd-output.log my.war

配置HTTP接口

与Jetty独立服务器一样，默认设置是使连接器在计算机上的所有接口上侦听。您可以通过指定希望与特定接口一起使用的特定接口的名称或IP地址来控制它--host参数：

> java -jar jetty-runner.jar --host 192.168.22.19 my.war

配置HTTP端口

默认端口号是8080。要配置https连接器，请改用Jetty xml配置文件。

> java -jar jetty-runner.jar --port 9090  my.war

配置停止

您可以为Jetty配置端口号，以侦听stop命令，因此您可以从其他终端将其停止。这要求使用“秘密”密钥，以防止恶意或意外终止。使用--stop-port和--stop-key （要么-DSTOP.PORT=和-DSTOP.KEY=分别）参数作为jetty-runner ：

> java -jar jetty-runner.jar --stop-port 8181 --stop-key abc123

然后，要从其他终端停止Jetty，您需要提供相同的端口和密钥信息。为此，您将需要在本地安装Jetty， jetty-maven-plugin ， jetty-ant插件或自定义类。以下是使用Jetty安装执行停止的方法：

> java -jar start.jar -DSTOP.PORT=8181 -DSTOP.KEY=abc123 --stop

配置容器类路径

通过在本地安装Jetty，您可以将jar和类添加到容器的类路径中， {$jetty.base}/lib目录。随着jetty-runner ，您可以使用--lib ， --jar和--classes参数来实现相同的目的。

--lib添加包含要添加到容器类路径的jar的目录的位置。您可以添加1个或更多。这是配置2个目录的示例：

> java -jar jetty-runner.jar --lib /usr/local/external/lib --lib $HOME/external-other/lib my.war

--jar向容器类路径添加一个jar文件。您可以添加1个或更多。这是配置3个额外jar的示例：

> java -jar jetty-runner.jar --jar /opt/stuff/jars/jar1.jar --jar $HOME/jars/jar2.jar --jar /usr/local/proj/jars/jar3.jar  my.war

--classes添加包含要添加到容器classpath的类的目录的位置。您可以添加1个或更多。这是配置单个额外类目录的示例：

> java -jar jetty-runner.jar --classes /opt/stuff/classes my.war

注意

使用时--jar和/或--lib参数，默认情况下将不检查这些参数META-INF诸如META-INF/resources ， META-INF/web-fragment.xml ， 要么META-INF/taglib.tld 。如果需要检查这些jar文件，则需要在上下文xml文件中定义jar模式 。Jetty-Runner自动为jtsl taglib提供并添加合适的模式（此模式与标准Jetty发行版中的模式不同）。

收集统计

如果启用了统计信息收集，则可以通过浏览上下文来查看它们/stats 。您可以选择使用密码保护对该上下文的访问。这是启用统计信息的示例，没有密码保护：

> java -jar jetty-runner.jar --stats unsecure my.war

如果我们希望保护对/stats在上下文中，我们将提供包含身份验证和授权信息的Jetty领域配置文件的位置。例如，我们可以使用Jetty发行版中的以下示例领域文件：

jetty: MD5:164c88b302622e17050af52c89945d44,user
admin: CRYPT:adpexzg3FUZAk,server-administrator,content-administrator,admin
other: OBF:1xmk1w261u9r1w1c1xmq,user
plain: plain,user
user: password,user
# This entry is for digest auth.  The credential is a MD5 hash of username:realmname:password
digest: MD5:6e120743ad67abfbc385bc2bb754e297,user

假设我们已将其复制到本地目录中，则将其应用为

> java -jar jetty-runner.jar --stats realm.properties my.war
