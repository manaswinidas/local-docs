Jetty XML用法

使用jetty.xml
使用多个配置文件
在配置文件中设置参数

Jetty提供了基于XML的配置。它基于Java的Reflection API。中的类java.lang.reflect表示Java方法和类，以便您可以实例化对象并根据其名称和参数类型调用它们的方法。在后台，Jetty的XML配置解析器将XML元素和属性转换为反射调用。

使用jetty.xml

使用jetty.xml ，在运行Jetty时将其指定为配置文件。

 java -jar start.jar etc/jetty.xml

注意

如果在不指定配置文件的情况下启动Jetty，Jetty会自动查找并使用默认安装jetty.xml文件。因此java -jar start.jar相当于java -jar start.jar etc/jetty.xml 。

使用多个配置文件

您不仅限于一个配置文件；您可以在运行Jetty时使用多个配置文件，然后Jetty将配置相应的服务器实例。伺服器中的ID 标签指定您要配置的实例。配置文件中的每个服务器标识都在同一JVM中创建一个新的服务器实例。如果在多个配置文件中使用相同的ID，则这些配置都将应用于同一服务器。

在配置文件中设置参数

您可以使用系统属性在配置文件中设置参数（使用 ）或属性文件（使用 ）通过命令行传递。例如，此代码在jetty.xml允许在命令行上定义端口，如果未指定端口，则退回到“ 8080”：

  <Set name="port"><SystemProperty name="jetty.http.port" default="8080"/></Set>

然后，使用以下命令在运行Jetty时修改端口：

 java -Djetty.http.port=8888 -jar start.jar etc/jetty.xml

从命令行定义系统属性和属性文件的示例：

 java -Djetty.http.port=8888 -jar start.jar myjetty.properties etc/jetty.xml etc/other.xml
