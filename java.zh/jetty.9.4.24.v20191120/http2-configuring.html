配置HTTP / 2

在Jetty服务器中启用HTTP / 2模块不会创建特定于HTTP / 2的连接器，而是会将HTTP / 2连接工厂添加到现有连接器。因此，配置HTTP / 2是在连接器上配置公共属性和在连接工厂上配置HTTP / 2特定属性的组合。可以使用以下命令查看涉及的模块和XML文件：

$ java -jar $JETTY_HOME/start.jar --list-modules
    ...
    1) alpn-impl  <transitive>
    ...
    2) http            ${jetty.base}/start.d/http.ini
    2) ssl             ${jetty.base}/start.d/ssl.ini
    3) alpn            ${jetty.base}/start.d/alpn.ini
    3) http2c          ${jetty.base}/start.d/http2c.ini
    ...
    4) http2           ${jetty.base}/start.d/http2.ini
    5) https           ${jetty.base}/start.d/https.ini

$ java -jar $JETTY_HOME/start.jar --list-config
 ...
 ${jetty.home}/etc/jetty-ssl.xml
 ${jetty.home}/etc/jetty-ssl-context.xml
 ${jetty.home}/etc/jetty-alpn.xml
 ${jetty.home}/etc/jetty-http2c.xml
 ${jetty.home}/etc/jetty-http.xml
 ...
 ${jetty.home}/etc/jetty-http2.xml
 ${jetty.home}/etc/jetty-https.xml

可以在模块ini文件中设置与连接器关联的通用属性（主机，端口，超时等）。 start.ini如果--add-to-start已使用）： ${jetty.base}/start.d/http.ini和${jetty.base}/start.d/ssl.ini 。这些属性在关联的XML文件中实例化： ${jetty.home}/etc/jetty-http.xml ; ${jetty.home}/etc/jetty-ssl.xml ，并且SSL密钥库在实例化${jetty.home}/etc/jetty-ssl-context.xml 。

注意

如果您打算编辑XML文件，请确保将其复制到您的{$jetty.base}/etc/目录之前这样做。随之而来的是Jetty发行的XML文件不应该直接修改。

可以在模块ini文件中设置HTTP / 2特定的属性： ${jetty.base}/start.d/http2.ini和${jetty.base}/start.d/http2c.ini ，在关联的XML文件中实例化： ${jetty.home}/etc/jetty-http2.xml ; ${jetty.home}/etc/jetty-http2c.xml ， 分别。当前，HTTP / 2配置属性很少，默认值是合理的：

表16.1HTTP / 2配置属性

属性描述

jetty.http2.maxConcurrentStreams

单个HTTP / 2连接上允许的并发打开流的最大数量（默认为128）。较大的值会增加并行度，但会占用内存。

jetty.http2.initialSessionRecvWindow

新会话的初始接收流控制窗口大小（默认值为1048576）。较大的值可能允许更大的吞吐量，但是如果触发了TCP / IP流量控制，也可能会造成行阻塞。

jetty.http2.initialStreamRecvWindow

新流的初始接收流控制窗口大小（默认为524288）。较大的值可能允许更大的吞吐量，但是如果触发了TCP / IP流量控制，也可能会造成行阻塞。
