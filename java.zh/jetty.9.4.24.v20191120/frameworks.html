Jetty弹簧模块
使用Spring配置Jetty

您可以用代码或几乎任何IoC样式框架（包括Spring）来组装和配置Jetty。如果您要做的只是在您的春季使用中设置Jetty服务器，只需以下面的xml代码段为例。如果要用spring替换用于启动普通Jetty发行版的jetty-xml，可以这样做，但是当前它不会利用其余模块系统。

Jetty弹簧模块

可以通过模块机构通过Jetty分布实现Jetty弹簧模块的骨架。例如：

$ java -jar start.jar --add-to-start=spring

这（或可选的--add-to-start = spring命令）创建了一个${jetty.home}/lib/spring目录，并用Jetty-Spring集成罐填充它。它不提供弹簧罐及其依赖项。您将需要下载这些文件并将其放入Jetty的类路径中-您可以使用${jetty.home}/lib/spring为此，由spring.mod创建的目录。

使用Spring配置Jetty

通过Spring配置Jetty只需将API称为Spring bean。以下是模仿默认Jetty启动配置的示例。

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE beans PUBLIC "-//SPRING//DTD BEAN//EN" "http://www.springframework.org/dtd/spring-beans.dtd">

<!-- =============================================================== -->
<!-- Configure the Jetty Server with Spring                          -->
<!-- This file is the similar to jetty.xml, but written in spring    -->
<!-- XmlBeanFactory format.                                          -->
<!-- =============================================================== -->

<beans>
    <bean id="contexts" class="org.eclipse.jetty.server.handler.ContextHandlerCollection"/>
    <bean id="server" name="Main" class="org.eclipse.jetty.server.Server" init-method="start" destroy-method="stop">
        <constructor-arg>
            <bean id="threadPool" class="org.eclipse.jetty.util.thread.QueuedThreadPool">
                <property name="minThreads" value="10"/>
                <property name="maxThreads" value="50"/>
            </bean>
        </constructor-arg>
        <property name="connectors">
            <list>
                <bean id="connector" class="org.eclipse.jetty.server.ServerConnector">
                    <constructor-arg ref="server"/>
                    <property name="port" value="8080"/>
                </bean>
            </list>
        </property>
        <property name="handler">
            <bean id="handlers" class="org.eclipse.jetty.server.handler.HandlerCollection">
            <property name="handlers">
                    <list>
                        <ref bean="contexts"/>
                        <bean id="defaultHandler" class="org.eclipse.jetty.server.handler.DefaultHandler"/>
                    </list>
                </property>
            </bean>
        </property>
        <property name="beans">
            <list>
                <bean id="deploymentManager" class="org.eclipse.jetty.deploy.DeploymentManager">
                    <property name="contexts" ref="contexts"/>
                    <property name="appProviders">
                        <list>
                            <bean id="webAppProvider" class="org.eclipse.jetty.deploy.providers.WebAppProvider">
                                <property name="monitoredDirName" value="webapps"/>
                                <property name="scanInterval" value="1"/>
                                <property name="extractWars" value="true"/>
                            </bean>
                        </list>
                    </property>
                </bean>
            </list>
        </property>
    </bean>
</beans>
