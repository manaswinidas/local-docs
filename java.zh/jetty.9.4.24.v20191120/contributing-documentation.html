本文档是使用maven，git和asciidoc的组合制作的。我们欢迎任何人和所有人为本书的内容做出贡献。以下是有关如何获取和编写本书的信息，以及有关如何回馈本书的信息。

注意：对本文档的所有贡献均在EPLv1下，并且版权归Mort Bay所有。

工具类

你会需要：

吉特

该文档是Jetty项目的一部分，因此所有贡献都必须通过正常的Jetty贡献过程进行。

如果您熟悉SCM，并且可以从那里随意安装和使用git，则可以使用git的两种方法之一。否则，我们建议您使用github客户端本身，因为它将帮助处理与git有关的某些工作流程。所有捐款都经过签名，可以通过正常的提取请求流程提取到Jetty中。

专家3
我们使用可在Apache Maven中找到的maven 3构建文档。

渲染链

Jetty文档全部以asciidoc编写，用作原始格式。Maven构建使用asciidoctor-maven-plugin将所有.adoc文件处理为单个docbook文件，然后使用该文件生成最终输出。我们使用此中间步骤来主要生成分块的html，然后将其部署到Eclipse Jetty网站。但是，如果我们愿意，我们也可以使用此docbook输出生成pdf文件，epub图书或什至Eclipse帮助文件。

入门（cli）

首先，您需要获取文档项目的源代码。

克隆存储库：

$ git clone https://github.com/eclipse/jetty.project.git

现在，您将拥有一个包含所有码头信息的本地目录，包括码头文件。现在我们继续构建它。

$ cd jetty.project/jetty-documentation
$ mvn install

当maven运行时，您可能会看到很多文件正在下载。如果您不熟悉maven，那么您将看到maven设置执行环境以生成文档。此版本将首先生成docbook xml，然后通过docbkx-maven-plugin生成分块的html输出。下载是使此构建工作所需的所有Java依赖项。一段时间后，下载将停止，您应该看到asciidoctor-maven-plugin的执行以及docbkx-maven-plugin的执行。

[INFO] --- asciidoctor-maven-plugin:1.5.3:process-asciidoc (output-html) @ jetty-documentation ---
[INFO] Rendered /Users/jesse/src/projects/jetty/jetty-docs/src/main/asciidoc/index.adoc
[INFO]

[INFO] Processing input file: index.xml
[INFO] Applying customization parameters
[INFO] Chunking output.
[INFO] See /Users/jesse/src/projects/jetty/jetty-docs/target/docbkx/html/index for generated file(s)

一旦看到类似于此的消息，构建就完成了：

[INFO] ------------------------------------------------------------------------
[INFO] BUILD SUCCESS
[INFO] ------------------------------------------------------------------------
[INFO] Total time: 7.014s
[INFO] Finished at: Tue Oct 25 14:15:37 CDT 2011
[INFO] Final Memory: 14M/229M
[INFO] ------------------------------------------------------------------------

现在，您可以打开Web浏览器并浏览到html输出的第一页，以查看所产生的结果！通常，您可以通过File→Open File→来执行此操作，这将打开文件系统浏览屏幕，导航到您的jetty-documentation目录，然后进一步进入target / docbkx / html / index / index.html，这是生成的第一页文档。

小费

如果构建已损坏，请随时通知我们。

做出改变

现在，您已经建立了文档，现在您想要对其进行编辑并进行一些更改。现在，我们需要退后一步，看看git和github的工作方式。在上面的示例中，您直接从我们的规范文档存储库中进行了克隆。显然，我们不允许任何人立即访问该存储库，因此您必须自己创建一个存储库，然后发出后拉请求以建立文档业力。用英语，这意味着您将转到github中文档的网址：

https://github.com/eclipse/jetty.project

当您在此页面上时，您会看到一个名为Fork的小按钮，您可以单击该按钮，然后将您带回到github上的主页，您在该主页上有了新的存储库。当您结帐该存储库时，您可以自由地将所有您希望在Jetty文档中看到的更改全部付诸实践。您可以将其克隆到本地计算机上，并按照上述相同的方式进行构建。因此，让我们从一个小例子开始。在文档中找到您认为需要更改的段落。在本地结帐处找到并进行更改。现在按照流程将更改正确地推回Jetty。确保确保更改有效且构建未损坏，因此请确保运行maven并检查输出。然后提交更改。

$ git commit -s -m "Tweaked the introduction to fix a horrid misspelled word." src/main/asciidoc/quickstart/introduction/topic.xml

注意

为了使我们接受您对Jetty存储库的提交，您必须在文件中包含Eclipse CLA并签署您的提交。请查看补丁程序部分以获取更多信息。

这将在您的本地存储库中提交更改。然后，您可以将更改推送到github上的存储库。

$ git push

现在，您将看到一些输出，显示您的更改已传播到github上的存储库。实际上，如果您导航到文件列表顶部的该存储库，则应该在此处看到您的注释。成功，您的更改现在可以通知我们！如果单击提交消息本身，则会转到一个屏幕，该屏幕显示该提交中更改了哪些文件。右上角是“ 拉取请求”按钮。当您选择此选项并遵循工作流程时，我们将收到您的贡献通知，并可以在审查后将其应用于我们的git存储库。

而已！您已经成功地为Jetty项目的文档编制工作做出了贡献。经过足够多的此类贡献并建立了良好的社区业障，您可能会被要求加入我们，成为文档的提交者。

约定

以下是在此框架内开发文档时应遵循的约定列表。这些不是一成不变的，应随着我们的了解而更新。

通风的散文
每个句子应在其自己的行中，并在行末添加硬返回。Asciidoc渲染不会将此样式视为单独的行，而是可以正确生成段落。主要好处是，您可以轻松查看scm版本的文件之间的更改，这使得快速查看拉取请求变得微不足道。其他好处是能够注释掉段落中间的句子或在段落内移动句子。在您喜欢的编辑器中启用“换行”可以使操作起来更容易。
id的

对于能够生成可持久使用的url至关重要。没有健全的ID，各章和节将生成ID，这些ID植根于一些晦涩难懂的伏都教。使用这些e12c8673类型链接的url在各代文档中都不持久。这些ID需要在第二章和第二节以及您打算交叉链接的任何地方使用。

id值进入全局命名空间，因此它们在整个文档中必须是唯一的，否则最后一个示例将获胜，并且任何交叉链接都将到达那里。以下是ID的示例。

[[this-id-an-id]]
链接与外部参照
的link:当您要选择将在链接本身中呈现的文本时，通常应使用item链接文档。但是，如果您要链接到某个部分并希望将标题本身用作链接文本，请使用xref:标记，链接ID前面没有井号。
版本差异
通常，发行版中功能上的差异应归入嵌套部分，并使用诸如以下条件的标题： 或在版本中： 。
许可证块
每个adoc文件都应包含index.adoc文件中存在的许可证块，并且副本已添加到此页面的底部，以供参考。
