webdefault.xml

使用webdefault.xml

的webdefault.xml该文件使Web应用程序不必自己定义许多内部管理和特定于容器的元素web.xml文件。例如，您可以使用它来设置MIME类型的映射和JSP servlet映射。码头适用webdefault.xml在应用程序本身之前访问Web应用程序WEB-INF/web.xml ，这表示它无法覆盖网络应用内部的值web.xml 。它使用jetty.xml语法。通常，Jetty实例中的所有Web应用共享相同的内容非常方便webdefault.xml文件。但是，当然可以为各个Web应用程序提供不同的`webdefault.xml`文件。

的webdefault.xml文件位于$(jetty.home)/etc/webdefault.xml 。

使用webdefault.xml

您可以指定用于特定Web应用程序或所有Web应用程序的自定义配置文件。如果您未指定备用默认描述符，则$JETTY-HOME/etc/jetty-deploy.xml文件将配置码头以自动使用$JETTY_HOME/etc/webdefault.xml 。

注意

为了确保您的webdefault.xml文件已验证，您需要设置validateXml属性为true，如此处所述。

的webdefault.xml Jetty发行版随附的包含几个配置选项，例如init参数和servlet映射，并且分为几部分以方便导航。一些较常见的选项包括但不限于：

dirAllowed
如果为true，则在未找到欢迎文件的情况下返回目录列表。否则，显示403禁止。
预压缩
如果将其设置为以逗号分隔的文件扩展名列表，则这些文件将指示已知映射为MIME类型的压缩格式，该MIME类型可能会在请求Accept-Encoding标头中列出。如果设置为布尔True，则将使用默认的一组压缩格式，否则将不使用预压缩格式。
maxCacheSize
高速缓存的最大总大小，如果没有高速缓存，则为0。
maxCachedFileSize
要缓存的文件的最大大小。
maxCachedFiles
要缓存的最大文件数。

为一个WebApp创建自定义webdefault.xml

您可以指定一个自定义webdefault.xml对于该webapp的jetty.xml中的单个Web应用程序，如下所示：

<Configure class="org.eclipse.jetty.webapp.WebAppContext">
  ...
  <!-- Set up the absolute path to the custom webdefault.xml -->
  <Set name="defaultsDescriptor">/my/path/to/webdefault.xml</Set>
  ...
</Configure>

代码中的等效项是：

import org.eclipse.jetty.webapp.WebAppContext;

    ...

    WebAppContext wac = new WebAppContext();
    ...
    //Set up the absolute path to the custom webdefault.xml.
    wac.setDefaultsDescriptor("/my/path/to/webdefault.xml");
    ...

或者，您可以使用Jetty Classloading查找代表您的自定义资源webdefault.xml 。

为多个WebApp创建自定义webdefault.xml

如果要应用相同的自定义webdefault.xml到许多Web应用程序，请在jetty.xml中的$JETTY_HOME/etc/jetty-deploy.xml文件：

   <Set name="defaultsDescriptor"><Property name="jetty.home" default="." />/other/path/to/another/webdefault.xml</Set>

使用Jetty Maven插件

同样，在使用Jetty Maven插件时，您需要提供自定义的webdefault.xml您的webapp的文件，如下所示：

<project>
    ...
    <plugins>
        <plugin>
            ...
            <artifactId>jetty-maven-plugin</artifactId>
            <configuration>
                <webApp>
                  ...
                  <defaultsDescriptor>/my/path/to/webdefault.xml</defaultsDescriptor>
                </webApp>
            </configuration>
        </plugin>
        ...
    </plugins>
    ...
</project>
