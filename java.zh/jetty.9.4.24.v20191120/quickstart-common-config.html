创建一个新的码头基地

的demo-base前面描述的目录是jetty.base机制。通过Jetty基本目录，可以将服务器实例的配置和Web应用程序与Jetty发行版分开存储，从而可以在不造成干扰的情况下完成升级。Jetty的默认配置基于两个属性：

码头
该属性定义Jetty发行版的位置，其lib，默认模块和默认XML文件（通常是start.jar，lib等）。
码头基地
该属性定义Jetty服务器的特定实现的位置，其配置，日志和Web应用程序（通常是start.d / *。ini文件，日志和Web应用程序）。

重要

您的Jetty主目录应被视为真实标准，并且未经修改或更改。对配置的更改或添加应在Jetty Base目录中进行。

的jetty.home和jetty.base属性可以在命令行上显式设置，或者与以下命令一起使用时，可以从环境中推断出来：

> cd $JETTY_BASE
> java -jar $JETTY_HOME/start.jar

以下命令创建一个新的基本目录，同时启用HTTP连接器和Web应用程序部署程序模块，并复制要部署的演示Web应用程序：

> JETTY_BASE=/tmp/mybase
> mkdir $JETTY_BASE
> cd $JETTY_BASE
> java -jar $JETTY_HOME/start.jar

更改码头

您可以通过设置以下选项将Jetty配置为在其他端口上运行jetty.http.port命令行上的属性：

> cd $JETTY_BASE
> java -jar $JETTY_HOME/start.jar jetty.http.port=8081
...

服务器启动后，它将现在在端口上运行8081 。重要的是要注意，在命令行上设置属性仅对该服务器实例有效。要更改配置，以便服务器将始终在所需的端口上启动，您需要编辑start.d/http.ini

注意

按属性进行配置通过以下链进行：

  • 的start.d/http.ini文件是有效命令行的一部分，包含--module=http激活http模块的参数。
  • 的modules/http.mod文件定义了http模块，该模块指定了etc/jetty-http.xml配置文件及其使用的模板ini属性。
  • 的jetty.http.port属性由属性XML元素使用etc/jetty.http.xml注入ServerConnector实例与端口。

有关更多信息，请参见《 快速入门配置指南》和《 配置连接器》 。

为HTTPS和HTTP2添加SSL

的--add-to-start命令在ini文件中设置有效命令行，以运行支持HTTPS和HTTP2协议的ssl连接，如下所示：

  • 暂时启用ssl通过添加来配置SSL连接器（例如端口，密钥库等）的模块etc/jetty-ssl.xml和etc/jetty-ssl-context.xml到有效的命令行。
  • 暂时启用alpn通过添加在SSL连接器上配置协议协商的模块etc/jetty-alpn.xml到有效的命令行。
  • 创造start.d/https.ini通过添加配置SSL连接器上的HTTPS协议etc/jetty-https.xml到有效的命令行。
  • 创造start.d/http2.ini通过添加配置SSL连接器上的HTTP / 2协议etc/jetty-http2.xml到有效的命令行。
  • 检查是否存在etc/keystore文件，如果不存在，则下载演示密钥库文件。

更改码头HTTPS端口

您可以通过设置SSL连接器，使其在其他端口上运行jetty.ssl.port命令行上的属性：

> cd $JETTY_BASE
> java -jar $JETTY_HOME/start.jar jetty.ssl.port=8444

或者，可以将属性值添加到从命令行生成的有效命令行中。 start.ini文件或start.d/*.ini文件，具体取决于您的设置。有关更多信息，请参见Start.ini与Start.d部分。

更多start.jar选项

的工作start.jar是解释命令行， start.ini和start.d目录（和相关的.ini文件）以构建Java类路径以及属性和配置文件列表，以传递给Jetty XML配置机制的主类。的start.jar该机制有许多选项，这些选项在第9章， 启动码头管理部分中介绍，您可以使用以下命令摘要查看它们：

> java -jar $JETTY_HOME/start.jar --help
