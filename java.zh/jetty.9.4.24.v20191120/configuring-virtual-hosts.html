虚拟主机是在DNS中注册的IP地址的备用名称，以便多个域名将解析为共享服务器实例的相同IP。如果要提供给别名名称的内容不同 ，则需要为每个部署的上下文配置虚拟主机，以指示上下文将响应的名称。

通过调用以下命令在上下文中设置虚拟主机setVirtualHosts要么addVirtualHost该方法可以通过几种方式完成：

  • 在webapps目录中使用上下文XML文件（请参阅Jetty发行版中test.xml中的示例）。
  • 创建具有绑定的自定义部署程序 ，以针对同一环境中找到的所有上下文配置虚拟主机webapps目录。
  • 直接在嵌入式用法上调用API 。
  • 用一个WEB-INF/jetty-web.xml文件（现已弃用）。

虚拟主机名

Jetty支持以下样式的虚拟主机名：

www.hostname.com
完全限定的主机名。列出所有变体非常重要，因为网站可能同时从www.hostname.com和hostname.com接收流量
* .hostname.com
通配符合格的主机，将仅匹配一个级别的任意名称。* .foo.com将匹配www.foo.com和m.foo.com，但不匹配www.other.foo.com
10.0.0.2
可以给定IP地址作为虚拟主机名，以指示上下文应处理在该服务器端口上收到的未指定主机名（HTTP / 0.9或HTTP / 1.0）的请求。
@ConnectorName
连接器名称，严格来说，它不是虚拟主机，而是仅与在具有通过Connector.setName（String）设置了匹配名称的连接器上收到的请求匹配。
www.√integral.com
可以使用从Punycode / IDN转换器获得的Puny Code等效项将非ASCII和IDN域名设置为虚拟主机。例如，如果非ASCII域名www.√integral.com被提供给浏览器，然后它将发出使用域名的请求www.xn--integral-7g7d.com ，这是应该添加为虚拟主机名的名称。

虚拟主机配置示例

虚拟主机可以与ContextHandler的子类的任何上下文一起使用。让我们看一个示例，其中我们使用虚拟主机配置一个Web应用程序（由WebAppContext类表示）。您提供可访问Web应用程序的IP地址和名称的列表，例如：

  • 333.444.555.666
  • 127.0.0.1
  • www.blah.com
  • www.blah.net
  • www.blah.org

假设您有一个名为blah.war ，您希望上述所有名称和地址都在路径“ /blah ”。这是使用上下文XML文件配置虚拟主机的方法：

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE Configure PUBLIC "-//Jetty//Configure//EN" "http://www.eclipse.org/jetty/configure_9_3.dtd">

<Configure class="org.eclipse.jetty.webapp.WebAppContext">
  <Set name="contextPath">/blah</Set>
  <Set name="war"><Property name="jetty.webapps"/>blah.war</Set>
  <Set name="virtualHosts">
    <Array type="java.lang.String">
      <Item>333.444.555.666</Item>
      <Item>127.0.0.1</Item>
      <Item>www.blah.com</Item>
      <Item>www.blah.net</Item>
      <Item>www.blah.org</Item>
    </Array>
  </Set>
</Configure>

使用不同的虚拟主机集选择不同的上下文

通过为每个虚拟主机提供特定的虚拟主机列表，可以配置不同的上下文以对不同的虚拟主机做出响应。

例如，假设您的虚拟机具有以下DNS名称：

  • www.blah.com
  • www.blah.net
  • www.blah.org
  • www.other.com
  • www.other.net
  • www.other.org

假设您还有2个Web应用程序，其中一个名为blah.war您想从*.blah.*名字，一个叫other.war您只想从*.other.*名称。

使用准备contextXML文件的方法，每个Web应用程序都会产生以下内容：

对于blah网络应用程序：

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE Configure PUBLIC "-//Jetty//Configure//EN" "http://www.eclipse.org/jetty/configure_9_3.dtd">

<Configure class="org.eclipse.jetty.webapp.WebAppContext">
  <Set name="contextPath">/blah</Set>
  <Set name="war"><Property name="jetty.webapps"/>/blah.war</Set>
  <Set name="virtualHosts">
    <Array type="java.lang.String">
      <Item>www.blah.com</Item>
      <Item>www.blah.net</Item>
      <Item>www.blah.org</Item>
    </Array>
  </Set>
</Configure>

现在，这些URL可以解析为上下文（即blah.war ）：

  • http://www.blah.com/blah
  • http://www.blah.net/blah
  • http://www.blah.org/blah

对于other网络应用程序：

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE Configure PUBLIC "-//Jetty//Configure//EN" "http://www.eclipse.org/jetty/configure_9_3.dtd">

<Configure class="org.eclipse.jetty.webapp.WebAppContext">
  <Set name="contextPath">/other</Set>
  <Set name="war"><Property name="jetty.webapps"/>/other.war</Set>
  <Set name="virtualHosts">
    <Array type="java.lang.String">
      <Item>www.other.com</Item>
      <Item>www.other.net</Item>
      <Item>www.other.org</Item>
    </Array>
  </Set>
</Configure>

这些网址现在可以解析到其他上下文（即other.war ）：

  • http://www.other.com/other
  • http://www.other.net/other
  • http://www.other.org/other

使用不同的虚拟主机集在同一上下文路径中选择不同的上下文

在上一节中，我们设置了2个不同的上下文，分别从不同上下文路径中的不同虚拟主机提供服务。但是，不要求上下文路径必须是不同的：您可以对多个上下文使用相同的上下文路径，并使用虚拟主机来确定为给定请求提供服务的主机。

考虑一个示例，其中我们具有与以前相同的DNS名称集，并且具有相同的Web应用程序blah.war和other.war 。我们仍然想要blah.war响应主机名*.blah.* ，而我们仍然想要other.war响应*.other.*名称。但是，我们希望所有客户使用"/"上下文路径，无论以哪个上下文为目标。

换句话说，我们希望将以下所有URL映射到blah.war ：

  • http://www.blah.com/
  • http://www.blah.net/
  • http://www.blah.org/

同样，我们希望以下URL映射到other.war ：

  • http://www.other.com/
  • http://www.other.net/
  • http://www.other.org/

为此，我们只需使用相同的上下文路径/对于我们的每个Web应用程序，同时仍应用我们不同的虚拟主机名集。

对于foo webapp：

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE Configure PUBLIC "-//Jetty//Configure//EN" "http://www.eclipse.org/jetty/configure_9_3.dtd">

<Configure class="org.eclipse.jetty.webapp.WebAppContext">
  <Set name="contextPath">/</Set>
  <Set name="war"><Property name="jetty.webapps"/>/foo.war</Set>
  <Set name="virtualHosts">
    <Array type="java.lang.String">
      <Item>www.blah.com</Item>
      <Item>www.blah.net</Item>
      <Item>www.blah.org</Item>
    </Array>
  </Set>
</Configure>

对于酒吧webapp：

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE Configure PUBLIC "-//Jetty//Configure//EN" "http://www.eclipse.org/jetty/configure_9_3.dtd">

<Configure class="org.eclipse.jetty.webapp.WebAppContext">
  <Set name="contextPath">/</Set>
  <Set name="war"><Property name="jetty.webapps"/>/bar.war</Set>
  <Set name="virtualHosts">
    <Array type="java.lang.String">
      <Item>www.other.com</Item>
      <Item>www.other.net</Item>
      <Item>www.other.org</Item>
    </Array>
  </Set>
</Configure>
