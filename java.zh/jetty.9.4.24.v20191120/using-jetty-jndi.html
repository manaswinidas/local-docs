定义web.xml

您可以配置命名资源以在web.xml文件和从内部访问java:comp/env执行期间Webapp的命名环境。具体来说，您可以配置以下支持web.xml元素：

<env-entry/>
<resource-ref/>
<resource-env-ref/>

配置环境条目向您展示如何为以下项设置替代env-entry中的元素web.xml ，同时进行配置resource-refs和resource-env-refs讨论如何配置支持资源，例如javax.sql.DataSource 。

您也可以插入JTA javax.transaction.UserTransaction实施到Jetty中，以便Webapp可以查找java:comp/UserTransaction获取分布式事务管理器：请参阅配置XA事务 。

申报资源

您必须声明要绑定到Jetty环境中的对象，然后才能通过env-entry ， resource-ref和resource-env-refs在web.xml 。通过使用以下类型的声明来创建这些绑定：

org.eclipse.jetty.plus.jndi.EnvEntry
对于env-entry条目类型
org.eclipse.jetty.plus.jndi.Resource
对于所有其他类型的资源
org.eclipse.jetty.plus.jndi.Transaction
对于JTA经理
org.eclipse.jetty.plus.jndi.Link
对于一个之间的联系web.xml资源名称和命名条目

每种类型的声明都遵循相同的一般模式：

<New class="org.eclipse.jetty.plus.jndi.xxxx">
  <Arg><!-- scope --></Arg>
  <Arg><!-- name --></Arg>
  <Arg><!-- value --></Arg>
</New>

您可以将这些声明放入三个不同的文件中，具体取决于您的需要和所声明资源的范围 。

决定在哪里声明资源

您可以在三个位置定义命名资源：

jetty.xml
命名中定义的资源jetty.xml文件的作用域在任JVM级别或服务器级别。资源的类必须在Jetty容器级别可见。如果该资源的类仅存在于您的webapp内部，则必须在WEB-INF/jetty-env.xml文件。
WEB-INF / jetty-env.xml
命名资源WEB-INF/jetty-env.xml文件的作用域是文件所在的Web应用程序。尽管可以选择输入JVM或服务器作用域，但我们不建议这样做。此处定义的资源可能使用Webapp内部的类。这是特定于Jetty的机制。
上下文xml文件
上下文xml文件中的条目的作用域应在它们所应用的Webapp级别，但是如果您选择的话，可以提供不太严格的Server或JVM范围级别。与在jetty.xml文件中，与资源关联的类必须在容器的类路径上可见。

资源名称范围

Jetty中的资源命名按限制顺序从高到低属于三个不同的范围之一：

JVM范围

该名称在JVM实例中是唯一的，并且对所有应用程序代码可见。您用一个null资源声明的第一个参数。例如：

<New id="cf" class="org.eclipse.jetty.plus.jndi.Resource">
  <Arg></Arg>  <!-- empty arg -->
  <Arg>jms/connectionFactory</Arg>
  <Arg>
    <New class="org.apache.activemq.ActiveMQConnectionFactory">
       <Arg>vm://localhost?broker.persistent=false</Arg>
    </New>
  </Arg>
</New>
服务器范围

该名称对于Server实例是唯一的，并且仅对与该实例关联的代码可见。通过将Server实例作为资源声明的第一个参数引用来表示此作用域。例如：

<Configure id="Server" class="org.eclipse.jetty.Server">
  <New id="cf" class="org.eclipse.jetty.plus.jndi.Resource">
    <Arg><Ref refid="Server"/></Arg>  <!-- reference to Server instance -->
    <Arg>jms/connectionFactory</Arg>
    <Arg>
      <New class="org.apache.activemq.ActiveMQConnectionFactory">
        <Arg>vm://localhost?broker.persistent=false</Arg>
      </New>
    </Arg>
  </New>
</Configure>
Webapp范围

该名称对于WebAppContext实例是唯一的，并且仅对与该实例关联的代码可见。您可以通过引用WebAppContext实例作为资源声明的第一个参数。例如：

<Configure id='wac' class="org.eclipse.jetty.webapp.WebAppContext">
  <New id="cf" class="org.eclipse.jetty.plus.jndi.Resource">
    <Arg><Ref refid='wac'/></Arg> <!-- reference to WebAppContext -->
    <Arg>jms/connectionFactory</Arg>
    <Arg>
      <New class="org.apache.activemq.ActiveMQConnectionFactory">
        <Arg>vm://localhost?broker.persistent=false</Arg>
      </New>
    </Arg>
  </New>
</Configure>

可以绑定什么资源？

您可以将四种类型的对象绑定到Jetty JNDI参考中：

  • 一个普通的POJO实例。
  • 一个javax.naming。参考实例。
  • 实现javax.naming的对象实例。参考界面。
  • 名称中引用的名称之间的链接web.xml并在Jetty环境中引用。
