可以将Jetty的HTTP客户端配置为使用不同的传输方式来承载HTTP请求和响应的语义。

这意味着客户请求资源的意图/index.html使用GET该方法可以通过网络以不同格式携带。

HTTP客户端传输是负责将高级语义HTTP请求（例如“ GET resource /index.html”）转换为服务器可以理解的特定格式的组件（例如HTTP / 2），并且将服务器响应从特定格式（HTTP / 2）转换为应用程序可以使用的高级语义对象。

这样，应用程序不会知道所使用的实际协议。这使他们可以针对高级API编写逻辑，该API隐藏了网络上使用的特定协议的详细信息。

最常见的协议格式是HTTP / 1.1，这是一种基于文本的协议，其中的行由\r\n ：

GET /index.html HTTP/1.1\r\n
Host: domain.com\r\n
...
\r\n

但是，可以使用FastCGI（二进制协议）发出相同的请求：

x01 x01 x00 x01 x00 x08 x00 x00
x00 x01 x01 x00 x00 x00 x00 x00
x01 x04 x00 x01 xLL xLL x00 x00
x0C x0B  D   O   C   U   M   E
 N   T   _   U   R   I   /   i
 n   d   e   x   .   h   t   m
 l
...

同样，HTTP / 2是一个二进制协议，以一种不同的格式传输相同的信息。

HTTP / 1.1传输

HTTP / 1.1是默认传输。

// No transport specified, using default.
HttpClient client = new HttpClient();
client.start();

如果要自定义HTTP / 1.1传输，则可以显式配置HttpClient通过这种方式：

int selectors = 1;
HttpClientTransportOverHTTP transport = new HttpClientTransportOverHTTP(selectors);

HttpClient client = new HttpClient(transport, null);
client.start();

上面的示例可让您自定义NIO选择器的数量HttpClient将使用。

HTTP / 2传输

可以通过以下方式配置HTTP / 2传输：

HTTP2Client h2Client = new HTTP2Client();
h2Client.setSelectors(1);
HttpClientTransportOverHTTP2 transport = new HttpClientTransportOverHTTP2(h2Client);

HttpClient client = new HttpClient(transport, null);
client.start();

HTTP2Client是提供基于HTTP / 2概念的API的低级客户端，例如特定于HTTP / 2的会话 ， 流和帧 。

HttpClientTransportOverHTTP2用途HTTP2Client将高级语义HTTP请求（“ GET resource /index.html”）格式化为特定于HTTP / 2的格式。

FastCGI传输

可以通过以下方式配置FastCGI传输：

int selectors = 1;
String scriptRoot = "/var/www/wordpress";
HttpClientTransportOverFCGI transport = new HttpClientTransportOverFCGI(selectors, false, scriptRoot);

HttpClient client = new HttpClient(transport, null);
client.start();

为了使用FastCGI传输进行请求，您需要拥有FastCGI服务器，例如PHP-FPM （另请参见http://php.net/manual/zh/install.fpm.php ）。

Jetty的FastCGI支持主要使用FastCGI传输来提供PHP页面（例如WordPress）。
