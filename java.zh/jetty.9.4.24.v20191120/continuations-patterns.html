挂起恢复模式

当通过异步处理程序终止了异步等待之后，使用servlet和/或过滤器生成响应时，将使用suspend / resume样式。通常，请求属性用于传递结果并指示请求是否已被挂起。

void doGet(HttpServletRequest request, HttpServletResponse response)
{
     // if we need to get asynchronous results
     Object results = request.getAttribute("results");
     if (results==null)
     {
       final Continuation continuation = ContinuationSupport.getContinuation(request);

       // if this is not a timeout
       if (continuation.isExpired())
       {
         sendMyTimeoutResponse(response);
         return;
       }

       // suspend the request
       continuation.suspend(); // always suspend before registration

       // register with async service.  The code here will depend on the
       // the service used (see Jetty HttpClient for example)
       myAsyncHandler.register(new MyHandler()
       {
          public void onMyEvent(Object result)
          {
            continuation.setAttribute("results",results);
            continuation.resume();
          }
       });
       return; // or continuation.undispatch();
     }

     // Send the results
     sendMyResultResponse(response,results);
}

当响应需要Servlet容器的设施（例如，它使用Web框架）时，或者如果一个事件可以恢复许多请求，以便可以使用容器的线程池来处理每个请求，则此样式非常好。

暂停继续模式

当使用异步处理程序生成响应时，将使用suspend / complete样式：

void doGet(HttpServletRequest request, HttpServletResponse response)
{
     final Continuation continuation = ContinuationSupport.getContinuation(request);

     // if this is not a timeout
     if (continuation.isExpired())
     {
       sendMyTimeoutResponse(request,response);
       return;
     }

     // suspend the request
     continuation.suspend(); // response may be wrapped.

     // register with async service.  The code here will depend on the
     // the service used (see Jetty HttpClient for example)
     myAsyncHandler.register(new MyHandler()
     {
       public void onMyEvent(Object result)
       {
         sendMyResultResponse(continuation.getServletResponse(),results);
         continuation.complete();
       }
     });
}

当响应不需要Servlet容器的功能（例如，它不使用Web框架）并且事件将仅恢复一个连续时，这种样式非常好。如果要发送许多响应（例如，聊天室），则编写一个响应可能会阻止其他响应上的DOS。

例子

  • ChatServlet示例显示了如何使用暂停/恢复样式直接对聊天室进行编码（请参见使用Async Servlet的类似示例 ）。相同的原则适用于像Cometd这样的框架，该框架基于Continuations为此类应用程序提供了更丰富的环境。

  • QoSFilter使用挂起/恢复样式来限制过滤器中同时请求的数量。这可用于保护JDBC连接池或其他有限资源免受过多同时请求的侵害。

    如果接收到太多请求，则额外的请求会在信号量上等待一小段时间，然后才被挂起。当过滤器中的请求返回时，它们使用优先级队列来恢复挂起的请求。当计算机处于负载状态时，这使您的经过身份验证的用户或优先级用户可以更好地共享服务器资源。

  • DosFilter与QoSFilter相似，但是尽可能地从Web应用程序内部保护Web应用程序免受拒绝服务攻击。

    如果检测到来自一个源的太多请求，则这些请求将被挂起并生成警告。这是在假设攻击者可能以简单的阻止样式编写的情况下进行的，因此，通过挂起您可能会消耗他们的资源。DOS的真正保护只能通过网络设备（或优生产品:)来实现。

  • ProxyServlet使用暂停/完成样式和Jetty异步HTTP客户端来实现可伸缩的代理服务器（或透明代理）。
