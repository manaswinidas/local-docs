测试Jetty释放

测试CometD

要测试Jetty版本，请为要测试的每个版本完成以下步骤：

  1. 下载分阶段发布的版本：

     wget https://oss.sonatype.org/content/repositories/jetty-[reponumber]/org/eclipse/jetty/jetty-distribution/[jetty-version]/jetty-distribution-9.[jetty-minor-version].tar.gz
  2. 提取到您选择的目录。

  3. 开始Jetty：

     cd [installdir] ; java -jar start.jar
  4. 如果没有例外，请继续。否则，请调查。
  5. 在浏览器中打开http：// localhost：8080 / 。在示例部分中，单击“ Test Jetty Webapp”。您应该看到test.war网络应用程序。
  6. 通过所有测试，并验证一切正常。
  7. 在示例部分中，单击“ JAAS测试”，并验证一切正常。
  8. 在示例部分中，单击“ JNDI测试”，并验证一切正常。
  9. 在示例部分中，单击“ Servlet 3.1测试”，并验证一切正常。

  10. 验证热部署有效。

     cd [installdir] ;
 touch [pathToJettyDistribution]/webapps.demo/test.xml
  11. 验证test.war重新部署STDOUT 。

  12. 验证spdy示例webapp和spdy-proxy是否起作用

     cd jetty_src/jetty-spdy/spdy-example-webapp
 mvn jetty:run-forked
  13. 浏览到https：// localhost：8443并确认一切正常

  14. 使用CTRL + C停止服务器，然后以代理方式重新启动服务器：

     mvn -Pproxy jetty:run-forked
  15. 浏览到http：// localhost：8080和https：// localhost：8443并确认一切看起来都不错

测试CometD

  1. 克隆彗星D。

     clone git://github.com/cometd/cometd.git
 git clone git://github.com/cometd/cometd.git

  2. 编辑pom.xml并更新jetty-version.

        <properties>
        <project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
        <jetty-version>7.6.2.v20120308</jetty-version>
        <jetty-plugin-version>${jetty-version}</jetty-plugin-version>
        <slf4j.version>1.6.4</slf4j.version>
        <spring-version>3.1.0.RELEASE</spring-version>
    </properties>
    <repositories>
    <repository>
      <id>Jetty Staging</id>
      <url>https://oss.sonatype.org/content/repositories/jetty-988/</url>
    </repository>
  </repositories>

  3. 建立彗星：

     mvn clean install
  4. 耐心一点。
  5. 按照此处所述运行负载测试： http ://cometd.org/documentation/2.x/howtos/loadtesting。保留默认值，但请确保将客户端设置提高到1000。运行负载测试，直到“ JIT编译时间 ”接近于零（约1万次调用）。
