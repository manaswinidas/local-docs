热部署

Jetty允许通过监视目录中的更改来部署任意上下文或Web应用程序。如果将Web应用程序或上下文描述符添加到目录中，Jetty的DeploymentManager （DM）部署新上下文。如果触摸或更新了上下文描述符，则DM停止，重新配置并重新部署其上下文。如果删除了上下文，则DM将停止它并将其从服务器中删除。

可以通过配置来控制此行为WebAppProvider属性。

monitorDirName
扫描可能的可部署Web应用程序（或部署描述符XML文件）的目录。
scanInterval
两次扫描之间的秒数monitoredDirName 。值0禁用连续的热部署扫描，将仅在启动时部署Web应用程序。

此配置的默认位置在${jetty.home}/etc/jetty-deploy.xml文件。要将其修改为Jetty发行版的一部分，请首先启用deploy模块。启用后，您可以在任一$JETTY_BASE/start.d/deploy.ini要么$JETTY_BASE/start.ini文件，具体取决于您的实现的配置方式。

<?xml version="1.0"?>
# ---------------------------------------
# Module: deploy
# Enables webapplication deployment from the webapps directory.
# ---------------------------------------
--module=deploy

# Monitored directory name (relative to $jetty.base)
# jetty.deploy.monitoredDir=webapps
# - OR -
# Monitored directory path (fully qualified)
# jetty.deploy.monitoredPath=/var/www/webapps

# Defaults Descriptor for all deployed webapps
# jetty.deploy.defaultsDescriptorPath=${jetty.base}/etc/webdefault.xml

# Monitored directory scan period (seconds)
# jetty.deploy.scanInterval=1

# Whether to extract *.war files
# jetty.deploy.extractWars=true

有关更多配置详细信息，请参见了解默认WebAppProvider 。

另请参阅部署体系结构以获取详细的概念信息。
