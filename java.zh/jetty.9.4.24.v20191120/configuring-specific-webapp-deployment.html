配置特定的Web应用程序部署

Jetty可部署描述符XML文件
使用基本描述符文件
配置高级描述符文件

使用“自动Web应用程序部署”模型既快速又容易，但是有时您可能需要调整某些部署属性（例如，您要使用不基于文件名的上下文路径进行部署，或者想要定义特殊的仅用于此Web应用程序的数据库连接池）。您可以使用Jetty Deployable Descriptor XML File完成此类调整。

Jetty可部署描述符XML文件

Jetty支持通过XML文件部署Web应用程序，该文件将构建一个ContextHandler实例，然后可以部署。

使用基本描述符文件

在默认的Jetty安装中，Jetty会扫描其$JETTY_HOME/webapps上下文部署描述符文件的目录。要使用此类文件部署Web应用程序，只需将文件放在该目录中即可。

部署描述符文件本身是一个xml文件，用于配置WebAppContext类。对于基本安装，只需配置两个属性：

战争
Web应用程序文件（或目录）的文件系统路径
contextPath
用于Web应用程序的上下文路径

例如，这是一个部署文件的描述符文件/opt/myapp/myapp.war到上下文路径/wiki ：

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE Configure PUBLIC "-//Jetty//Configure//EN" "http://www.eclipse.org/jetty/configure_9_3.dtd">

<Configure class="org.eclipse.jetty.webapp.WebAppContext">
  <Set name="contextPath">/wiki</Set>
  <Set name="war">/opt/myapp/myapp.war</Set>
</Configure>

都SystemProperty和Property元素可以在描述符文件中使用。例如，如果系统属性设置为myapp.home=/opt/myapp ，前面的示例可以重写为：

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE Configure PUBLIC "-//Jetty//Configure//EN" "http://www.eclipse.org/jetty/configure_9_3.dtd">

<Configure class="org.eclipse.jetty.webapp.WebAppContext">
  <Set name="contextPath">/wiki</Set>
  <Set name="war"><SystemProperty name="myapp.home"/>/myapp.war</Set>
</Configure>

如果需要更改应用程序的原始路径，则仅需要更改系统属性。如果应用程序的版本经常更改，这很有用。

注意

为了确保您的web.xml文件已验证，您需要设置validateXml属性为true，如此处所述。

配置高级描述符文件

的官方文档WebAppContext类列出了可以设置的所有属性。以下是一些在描述符文件中配置高级选项的示例。

第一个示例告诉Jetty部署时不要扩展WAR文件。这可以帮助弄清楚用户不应对临时解压缩的WAR进行更改，因为此类更改不会持续存在，因此在下次Web应用程序部署时不会应用。

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE Configure PUBLIC "-//Jetty//Configure//EN" "http://www.eclipse.org/jetty/configure_9_3.dtd">

<Configure class="org.eclipse.jetty.webapp.WebAppContext">
  <Set name="contextPath">/wiki</Set>
  <Set name="war"><SystemProperty name="myapp.home"/>/myapp.war</Set>
  <Set name="extractWAR">false</Set>
</Configure>

下一个示例检索JavaEE Servlet上下文并在其上设置一个初始化参数。的setAttribute方法也可以用于设置Servlet上下文属性。但是，由于web.xml对于Web应用程序，在部署描述符之后， web.xml值会覆盖部署描述符中名称相同的属性。

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE Configure PUBLIC "-//Jetty//Configure//EN" "http://www.eclipse.org/jetty/configure_9_3.dtd">

<Configure class="org.eclipse.jetty.webapp.WebAppContext">
  <Set name="contextPath">/wiki</Set>
  <Set name="war"><SystemProperty name="myapp.home"/>/myapp.war</Set>
  <Get name="ServletContext">
     <Call name="setInitParameter">
       <Arg>myapp.config</Arg>
       <Arg><SystemProperty name="myapp.home">/config/app-config.xml</Arg>
    </Call>
  </Get>
</Configure>

以下示例设置了一个特殊的web.xml覆盖描述符。该描述符在Web应用程序的web.xml ，因此它可以覆盖名称相同的属性。当添加参数或其他Servlet映射而不打开打包的WAR文件时，此功能很有用。

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE Configure PUBLIC "-//Jetty//Configure//EN" "http://www.eclipse.org/jetty/configure_9_3.dtd">

<Configure class="org.eclipse.jetty.webapp.WebAppContext">
  <Set name="contextPath">/wiki</Set>
  <Set name="war"><SystemProperty name="myapp.home"/>/myapp.war</Set>
  <Set name="overrideDescriptor">/opt/myapp/overlay-web.xml</Set>
</Configure>

下一个示例不仅配置Web应用程序上下文，还配置应用程序可以使用的数据库连接池（请参阅数据源示例 ）。如果web.xml不包含对此数据源的引用，可以使用覆盖描述符机制（如上例所示）将其包括在内。

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE Configure PUBLIC "-//Jetty//Configure//EN" "http://www.eclipse.org/jetty/configure_9_3.dtd">

<Configure class="org.eclipse.jetty.webapp.WebAppContext">
  <Set name="contextPath">/wiki</Set>
  <Set name="war"><SystemProperty name="myapp.home"/>/myapp.war</Set>

  <New id="DSTest" class="org.eclipse.jetty.plus.jndi.Resource">
    <Arg></Arg>
    <Arg>jdbc/DSTest</Arg>
    <Arg>
      <New class="org.apache.commons.dbcp.BasicDataSource">
        <Set name="driverClassName">org.some.Driver</Set>
        <Set name="url">jdbc.url</Set>
        <Set name="username">jdbc.user</Set>
        <Set name="password">jdbc.pass</Set>
      </New>
    </Arg>
  </New>
</Configure>

您还可以在其中更改许多其他设置WebAppContext 。的Javadoc WebAppContext是很好的信息来源。另请参阅有关避免zip文件异常的文档，以获取有关以下内容的说明： WebAppContext这些设置可确定诸如在部署过程中是否自动打开战争包装或将Webapp的某些部分复制到临时位置之类的事情。
