配置HTTP / 2推送

HTTP / 2推送是一种机制，它允许服务器针对单个客户端请求将多个资源发送到客户端。这将减少检索组成网页的所有资源所需的往返次数，并可以显着缩短页面加载时间。

通过配置一个PushCacheFilter在里面web.xml ， 通过这种方式：

<?xml version="1.0" encoding="UTF-8"?>
<web-app
    xmlns="http://xmlns.jcp.org/xml/ns/javaee"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee http://xmlns.jcp.org/xml/ns/javaee/web-app_3_1.xsd"
    metadata-complete="true"
    version="3.1">

    ...
    <filter>
        <filter-name>PushFilter</filter-name>
        <filter-class>org.eclipse.jetty.servlets.PushCacheFilter</filter-class>
        <async-supported>true</async-supported>
    </filter>
    <filter-mapping>
        <filter-name>PushFilter</filter-name>
        <url-pattern>/*</url-pattern>
    </filter-mapping>
    ...

</web-app>

PushCacheFilter分析HTTP请求以获取到达Web应用程序的资源。其中一些请求包含HTTP Referer标头，指向先前已请求的资源。

这允许PushCacheFilter在树中组织资源，例如根index.html有两个孩子资源的资源， styles.css和application.js和styles.css有子资源background.png 。根资源称为主要资源，而后代资源称为辅助资源。

资源树是使用时间窗口构建的，因此当请求根资源时，只有在时间窗口内进行的后续请求才会添加到资源树中。还可以限制资源树的大小，以便限制与主要资源关联的辅助资源的数量。

默认情况下，仅资源路径 （不带查询字符串）用于将辅助资源与主要资源相关联，但是您可以配置PushCacheFilter考虑到查询字符串。

PushCacheFilter可以用以下配置init-params ：

  • associatePeriod ：时间窗（以毫秒为单位），在该时间窗内，对次要资源的请求将与主要资源相关联；默认为4000毫秒
  • maxAssociations ：可以与主要资源关联的辅助资源的最大数量；默认为16
  • hosts ：以逗号分隔的主机列表， Referer标题默认为Host标头
  • ports ：用逗号分隔的端口列表， Referer标题默认为Host标头
  • useQueryInKey ：一个布尔值，指示在将辅助资源与主要资源相关联时是否应考虑请求的查询字符串；默认为false
