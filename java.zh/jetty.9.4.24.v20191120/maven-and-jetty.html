使用Maven

在Maven中使用嵌入式Jetty
使用Jetty和Maven开发标准WebApp

Apache Maven是一个软件项目管理和理解工具。基于项目对象模型（POM）的概念，Maven可以从中央信息管理项目的构建，报告和文档。

它是构建Web应用程序项目的理想工具，并且此类项目可以使用jetty-maven-plugin轻松运行Web应用程序并节省开发时间。您还可以使用Maven来构建，测试和运行嵌入Jetty的项目。

注意

Maven和码头，Maven的插件的使用不是必需的。使用Maven进行Jetty的实现是一个受欢迎的选择，但是鼓励用户以适合他们需求的任何方式管理他们的项目。其他流行的工具包括Ant和Gradle。

首先，我们将看一个嵌入Jetty的非常简单的HelloWorld Java应用程序，然后是一个利用jetty-maven-plugin加快开发周期的简单Web应用程序。

在Maven中使用嵌入式Jetty

要了解针对Jetty进行构建和运行的基本操作，请先进行以下审查：

  • 嵌入码头
  • 码头HelloWorld示例

Maven使用约定而不是配置，因此最好使用Maven建议的项目结构。您可以使用原型来快速设置Maven项目，但是我们将为这个简单的教程示例手动设置结构：

> mkdir JettyMavenHelloWorld
> cd JettyMavenHelloWorld
> mkdir -p src/main/java/org/example

创建HelloWorld类

使用编辑器创建文件src/main/java/org/example/HelloWorld.java具有以下内容：

package org.example;

import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import javax.servlet.ServletException;
import java.io.IOException;
import org.eclipse.jetty.server.Server;
import org.eclipse.jetty.server.Request;
import org.eclipse.jetty.server.handler.AbstractHandler;

public class HelloWorld extends AbstractHandler
{
    public void handle(String target,
                       Request baseRequest,
                       HttpServletRequest request,
                       HttpServletResponse response)
        throws IOException, ServletException
    {
        response.setContentType("text/html;charset=utf-8");
        response.setStatus(HttpServletResponse.SC_OK);
        baseRequest.setHandled(true);
        response.getWriter().println("<h1>Hello World</h1>");
    }

    public static void main(String[] args) throws Exception
    {
        Server server = new Server(8080);
        server.setHandler(new HelloWorld());

        server.start();
        server.join();
    }
}

创建POM描述符

的pom.xml文件声明项目名称及其依赖项。使用编辑器创建文件pom.xml在里面JettyMavenHelloWorld具有以下内容的目录：

<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/maven-v4_0_0.xsd">

  <modelVersion>4.0.0</modelVersion>
  <groupId>org.example</groupId>
  <artifactId>hello-world</artifactId>
  <version>0.1-SNAPSHOT</version>
  <packaging>jar</packaging>
  <name>Jetty HelloWorld</name>

  <properties>
      <!-- Adapt this to a version found on
           https://repo1.maven.org/maven2/org/eclipse/jetty/jetty-maven-plugin/
        -->
      <jettyVersion>9.3.9.v20160517</jettyVersion>
  </properties>

  <dependencies>
    <dependency>
      <groupId>org.eclipse.jetty</groupId>
      <artifactId>jetty-server</artifactId>
      <version>${jettyVersion}</version>
    </dependency>
  </dependencies>

  <build>
    <plugins>
      <plugin>
        <groupId>org.codehaus.mojo</groupId>
        <artifactId>exec-maven-plugin</artifactId>
        <version>1.1</version>
        <executions>
          <execution><goals><goal>java</goal></goals></execution>
        </executions>
        <configuration>
          <mainClass>org.example.HelloWorld</mainClass>
        </configuration>
      </plugin>
    </plugins>
  </build>
</project>

使用Jetty和Maven开发标准WebApp

上一节演示了如何将Maven与嵌入Jetty的应用程序一起使用。现在，我们将研究如何使用Maven和Jetty开发标准的Webapp。首先创建Maven结构（如果愿意，可以使用maven webapp原型）：

> mkdir JettyMavenHelloWarApp
> cd JettyMavenHelloWebApp
> mkdir -p src/main/java/org/example
> mkdir -p src/main/webapp/WEB-INF

创建一个Servlet

使用编辑器创建文件src/main/java/org/example/HelloServlet.java具有以下内容：

package org.example;

import java.io.IOException;
import javax.servlet.ServletException;
import javax.servlet.http.HttpServlet;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;

public class HelloServlet extends HttpServlet
{
    protected void doGet(HttpServletRequest request, HttpServletResponse response) throws ServletException, IOException
    {
        response.setContentType("text/html");
        response.setStatus(HttpServletResponse.SC_OK);
        response.getWriter().println("<h1>Hello Servlet</h1>");
        response.getWriter().println("session=" + request.getSession(true).getId());
    }
}

您需要在部署描述符中声明此servlet，因此请创建文件src/main/webapp/WEB-INF/web.xml并添加以下内容：

<?xml version="1.0" encoding="UTF-8"?>
<web-app
   xmlns="http://xmlns.jcp.org/xml/ns/javaee"
   xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
   xsi:schemaLocation="http://xmlns.jcp.org/xml/ns/javaee http://xmlns.jcp.org/xml/ns/javaee/web-app_3_1.xsd"
   metadata-complete="false"
   version="3.1">

  <servlet>
    <servlet-name>Hello</servlet-name>
    <servlet-class>org.example.HelloServlet</servlet-class>
  </servlet>
  <servlet-mapping>
    <servlet-name>Hello</servlet-name>
    <url-pattern>/hello/*</url-pattern>
  </servlet-mapping>

</web-app>

创建POM描述符

的pom.xml文件声明项目名称及其依赖项。使用编辑器创建文件pom.xml在以下内容中JettyMavenHelloWarApp目录，尤其要注意jetty-maven-plugin的声明：

<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/maven-v4_0_0.xsd">

  <modelVersion>4.0.0</modelVersion>
  <groupId>org.example</groupId>
  <artifactId>hello-world</artifactId>
  <version>0.1-SNAPSHOT</version>
  <packaging>war</packaging>
  <name>Jetty HelloWorld WebApp</name>

  <properties>
      <jettyVersion>9.4.24.v20191120</jettyVersion>
  </properties>

  <dependencies>
    <dependency>
      <groupId>javax.servlet</groupId>
      <artifactId>javax.servlet-api</artifactId>
      <version>3.1.0</version>
      <scope>provided</scope>
    </dependency>
  </dependencies>

  <build>
    <plugins>
      <plugin>
        <groupId>org.eclipse.jetty</groupId>
        <artifactId>jetty-maven-plugin</artifactId>
        <version>${jettyVersion}</version>
      </plugin>
    </plugins>
  </build>

</project>

构建和运行Web应用程序

现在，您可以通过以下命令使用jetty-maven-plugin来构建和运行Web应用程序，而无需将其组装成战争：

> mvn jetty:run

您可以在以下位置看到静态和动态内容http://localhost:8080/hello

jetty-maven-plugin有很多配置选项可用来帮助您构建和运行Webapp。完整的参考资料位于配置Jetty Maven插件 。

建立WAR档案

您可以使用以下命令从项目中创建Web应用程序存档（WAR）文件：

> mvn package

产生的战争文件在target目录，并且可以部署在任何标准servlet服务器上，包括Jetty 。
