合并数据源
非池数据源

以下是为各种数据库配置JNDI数据源的示例。

注意

请阅读xref：jndi-configuration [中的]，以获取有关配置数据源的更多信息。

所有这些示例都对应一个resource-ref在web.xml 。

  <resource-ref>
     <description>My DataSource Reference</description>
     <res-ref-name>jdbc/DSTest</res-ref-name>
     <res-type>javax.sql.DataSource</res-type>
     <res-auth>Container</res-auth>
  </resource-ref>

这些示例假定所有数据源都在JVM范围内声明，但是如果需要，您可以使用其他范围。您可以在一个目录中配置所有JNDI资源。 jetty.xml文件，一个WEB-INF/jetty-env.xml文件或上下文XML文件。有关更多信息，请参见决定在哪里声明资源 。

重要

您必须为Jetty提供必要的库，以通过将相应的Jar放入实例化已配置的数据源。 JETTY_HOME/lib/ext 。

合并数据源

池化数据源启用连接池化，这使您可以重用现有连接，而不必创建与数据库的新连接。就内存分配和对数据库的请求速度而言，这非常高效。我们强烈建议在生产环境中使用此选项。

以下是我们过去使用过的合并数据源示例的列表：

  • 光明CP
  • 骨CP
  • c3p0
  • DBCP
  • Atomikos 3.3.2以上
  • 的MySQL
  • PostgreSQL的
  • DB2

光明CP

连接池，可从HikariCP下载处获得 。有关HikariCP的所有配置选项，请参见： HikariCP文档 。

  <New id="DSTest" class="org.eclipse.jetty.plus.jndi.Resource">
     <Arg></Arg>
     <Arg>jdbc/DSTest</Arg>
     <Arg>
       <New class="com.zaxxer.hikari.HikariDataSource">
         <Arg>
            <New class="com.zaxxer.hikari.HikariConfig">
               <Set name="minimumPoolSize">5</Set>
               <Set name="maximumPoolSize">20</Set>
               <Set name="dataSourceClassName">com.mysql.jdbc.jdbc2.optional.MysqlDataSource</Set>
               <Set name="username">jdbc.user</Set>
               <Set name="password">jdbc.pass</Set>
               <Call name="addDataSourceProperty">
                  <Arg>url</Arg>
                  <Arg>jdbc.url</Arg>
               </Call>
            </New>
         </Arg>
      </New>
    </Arg>
  </New>

骨CP

连接池，可从BoneCP下载获得 。此处介绍了BoneCP的所有配置选项： BoneCP API 。

  <New id="DSTest" class="org.eclipse.jetty.plus.jndi.Resource">
     <Arg></Arg>
     <Arg>jdbc/DSTest</Arg>
     <Arg>
       <New class="com.jolbox.bonecp.BoneCPDataSource">
         <Set name="driverClass">com.mysql.jdbc.Driver</Set>
         <Set name="jdbcUrl">jdbc.url</Set>
         <Set name="username">jdbc.user</Set>
         <Set name="password">jdbc.pass</Set>
         <Set name="minConnectionsPerPartition">5</Set>
         <Set name="maxConnectionsPerPartition">50</Set>
         <Set name="acquireIncrement">5</Set>
         <Set name="idleConnectionTestPeriod">30</Set>
      </New>
    </Arg>
  </New>

非池数据源

如果要在生产环境中进行部署，我们强烈建议使用Pooling DataSource。由于并非总是这样，因此我们在此处也列出了一些非池数据源的示例。

以下是非池数据源示例的列表：

  • SQL Server 2000
  • 甲骨文9i / 10g
  • PostgreSQL的
  • Sybase公司
  • DB2
