使用WebSocket注释

package examples.echo;

import org.eclipse.jetty.websocket.api.Session;
import org.eclipse.jetty.websocket.api.annotations.OnWebSocketMessage;
import org.eclipse.jetty.websocket.api.annotations.WebSocket;

/**
 * Example EchoSocket using Annotations.
 */
@WebSocket(maxTextMessageSize = 64 * 1024)
public class AnnotatedEchoSocket
{
    @OnWebSocketMessage
    public void onText(Session session, String message)
    {
        if (session.isOpen())
        {
            System.out.printf("Echoing back message [%s]%n", message);
            // echo the message back
            session.getRemote().sendString(message, null);
        }
    }
}

上面的示例是一个简单的WebSocket回显终结点，它将回显收到的所有TEXT消息。

此实现对Echo套接字使用无状态方法，因为在事件发生时将Session传递给Message事件。这将允许您重用AnnotatedEchoSocket的单个实例来使用多个端点。

您可以使用的注释：

@WebSocket

必需的类级别注释。

将此POJO标记为WebSocket。

该类不能是抽象的且公共的。

@OnWebSocketConnect

可选的方法级别注释。

将类中的一种方法标记为接收On Connect事件。

方法必须是公共的，不是抽象的，返回void，并且具有单个Session参数。

@OnWebSocket关闭

可选的方法级别注释。

将类中的一个方法标记为接收On Close事件。

方法签名必须是公共的，而不是抽象的，并且返回void。

方法参数：

  1. Session （可选的）
  2. int closeCode （需要）
  3. String closeReason （需要）
@OnWebSocketMessage

可选的方法级别注释。

在该类中最多将2个方法标记为接收On Message事件。

您可以使用1种方法处理TEXT消息，并使用1种方法处理BINARY消息。

方法签名必须是公共的，而不是抽象的，并且返回void。

短信的方法参数：

  • Session （可选的）

  • String text （需要）

    二进制消息的方法参数：

  • Session （可选的）
  • byte buf[] （需要）
  • int offset （需要）
  • int length （需要）
@OnWebSocketError

可选的方法级别注释。

将类中的一种方法标记为从WebSocket实现接收Error事件。

方法签名必须是公共的，而不是抽象的，并且返回void。

方法参数：

  1. Session （可选的）
  2. Throwable cause （需要）
@OnWebSocketFrame

可选的方法级别注释。

在升级握手期间声明的任何扩展处理了事件之后，将该类中的一个方法标记为从WebSocket实现接收Frame事件。

方法签名必须是公共的，而不是抽象的，并且返回void。

方法参数：

  1. Session （可选的）

  2. Frame （需要）

    接收到的帧将通过此方法进行通知，然后由Jetty处理，可能会导致另一个事件，例如“关闭”或“消息”。Jetty不会看到对框架的更改。
