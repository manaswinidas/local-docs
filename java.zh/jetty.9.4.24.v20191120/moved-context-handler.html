信息

  • Maven工件：org.eclipse.jetty：jetty-server
  • Javadoc： http : //www.eclipse.org/jetty/javadoc/9.4.24.v20191120/org/eclipse/jetty/server/handler/MovedContextHandler.html

用法

您可以使用MovedContextHandler重新定位或重定向已更改上下文路径和/或虚拟主机的上下文。

您可以将其配置为将旧URL 永久重定向到新URL，在这种情况下，Jetty会使用新URL向浏览器发送301的Http状态代码。或者，您可以将其设置为非永久性，在这种情况下，Jetty会发送302 Http状态代码以及新的URL。

此外，与任何其他上下文一样，您可以配置虚拟主机列表，这意味着此上下文仅响应对所列主机名之一的请求。

假设您在处部署了上下文/foo ，但现在您要在根上下文中进行部署/代替。

  • 首先，您在Jetty上重新配置和重新部署上下文。
  • 接下来，您需要一种方法来重定向所有已添加书签的浏览器/foo通往新道路。您在中创建一个新的上下文xml文件。 {$jetty/.base}/webapps并配置MovedContextHandler从进行重定向/foo至/ 。

下面是一个例子。这是一个永久重定向，也保留了pathinfo并在重定向上查询字符串：

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE Configure PUBLIC "-//Jetty//Configure//EN" "http://www.eclipse.org/jetty/configure_9_3.dtd">

<Configure class="org.eclipse.jetty.server.handler.MovedContextHandler">
  <Set name="contextPath">/foo</Set>
  <Set name="newContextURL">/</Set>
  <Set name="permanent">true</Set>
  <Set name="discardPathInfo">false</Set>
  <Set name="discardQuery">false</Set>

  <Set name="virtualHosts">
    <Array type="String">
          <Item>209.235.245.73</Item>
          <Item>127.0.0.73</Item>
          <Item>acme.org</Item>
          <Item>www.acme.org</Item>
          <Item>server.acme.org</Item>
    </Array>
  </Set>
</Configure>
