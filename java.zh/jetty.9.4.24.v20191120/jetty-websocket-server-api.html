Jetty WebSocket服务器API

Jetty WebSocketServlet
使用WebSocketCreator

Jetty提供了通过使用WebSocketServlet桥Servlet将WebSocket端点连接到Servlet路径规范的功能。

在内部，Jetty管理从HTTP升级到WebSocket以及从HTTP连接到WebSocket连接的迁移。

这仅在Jetty容器中运行时才有效（与过去的Jetty技术不同，您无法让Jetty WebSocket服务器功能在其他容器（如JBoss，Tomcat或WebLogic）中运行Jetty）。

Jetty WebSocketServlet

要通过WebSocketServlet将WebSocket连接到特定路径，您需要扩展org.eclipse.jetty.websocket.servlet。WebSocketServlet并指定应使用传入的Upgrade请求创建哪个WebSocket对象。

package examples;

import javax.servlet.annotation.WebServlet;

import org.eclipse.jetty.websocket.servlet.WebSocketServlet;
import org.eclipse.jetty.websocket.servlet.WebSocketServletFactory;

@SuppressWarnings("serial")
@WebServlet(name = "MyEcho WebSocket Servlet", urlPatterns = {"/echo"})
public class MyEchoServlet extends WebSocketServlet
{
    @Override
    public void configure(WebSocketServletFactory factory)
    {
        // set a 10 second timeout
        factory.getPolicy().setIdleTimeout(10000);

        // register MyEchoSocket as the WebSocket to create on Upgrade
        factory.register(MyEchoSocket.class);
    }
}

本示例将创建一个通过@WebServlet批注映射到的Servlet路径规范的Servlet。 "/echo" （或者您可以在WEB-INF/web.xml的Web应用程序）在遇到HTTP升级请求时将创建MyEchoSocket实例。

的WebSocketServlet.configure(WebSocketServletFactory factory)在此处放置WebSocket的特定配置。在示例中，我们指定了10秒钟的空闲超时，并使用默认的WebSocketCreator注册MyEchoSocket，即我们要在升级时创建的WebSocket类。

注意

配置websocket时，请考虑所有防火墙或路由器超时，这一点很重要。确保websocket配置低于防火墙或路由器。

使用WebSocketCreator

所有WebSocket都是通过您在WebSocketServletFactory中注册的任何WebSocketCreator创建的。

默认情况下，WebSocketServletFactory是一个简单的WebSocketCreator，能够创建单个WebSocket对象。使用WebSocketCreator.register(Class websocket)告诉WebSocketServletFactory应该实例化哪个类（确保它具有默认构造函数）。

如果您有一个更复杂的创建方案，则可能要提供自己的WebSocketCreator，该WebSocketCreator基于它创建的WebSocket，而WebSocket是基于UpgradeRequest对象中存在的信息进行的。

package examples;

import org.eclipse.jetty.websocket.servlet.ServletUpgradeRequest;
import org.eclipse.jetty.websocket.servlet.ServletUpgradeResponse;
import org.eclipse.jetty.websocket.servlet.WebSocketCreator;

public class MyAdvancedEchoCreator implements WebSocketCreator
{
    private MyBinaryEchoSocket binaryEcho;
    private MyEchoSocket textEcho;

    public MyAdvancedEchoCreator()
    {
        // Create the reusable sockets
        this.binaryEcho = new MyBinaryEchoSocket();
        this.textEcho = new MyEchoSocket();
    }

    @Override
    public Object createWebSocket(ServletUpgradeRequest req, ServletUpgradeResponse resp)
    {
        for (String subprotocol : req.getSubProtocols())
        {
            if ("binary".equals(subprotocol))
            {
                resp.setAcceptedSubProtocol(subprotocol);
                return binaryEcho;
            }
            if ("text".equals(subprotocol))
            {
                resp.setAcceptedSubProtocol(subprotocol);
                return textEcho;
            }
        }

        // No valid subprotocol in request, ignore the request
        return null;
    }
}

在这里，我们显示了一个WebSocketCreator，它将利用请求中的WebSocket子协议信息来确定应创建哪种WebSocket类型。

package examples;

import javax.servlet.annotation.WebServlet;

import org.eclipse.jetty.websocket.servlet.WebSocketServlet;
import org.eclipse.jetty.websocket.servlet.WebSocketServletFactory;

@SuppressWarnings("serial")
@WebServlet(name = "MyAdvanced Echo WebSocket Servlet", urlPatterns = {"/advecho"})
public class MyAdvancedEchoServlet extends WebSocketServlet
{
    @Override
    public void configure(WebSocketServletFactory factory)
    {
        // set a 10 second timeout
        factory.getPolicy().setIdleTimeout(10000);

        // set a custom WebSocket creator
        factory.setCreator(new MyAdvancedEchoCreator());
    }
}

当您需要自定义WebSocketCreator时，请使用WebSocketServletFactory.setCreator(WebSocketCreator creator) WebSocketServletFactory将使用您的创建者来处理此Servlet上所有传入的Upgrade请求。

WebSocketCreator的其他用途：

  • 控制WebSocket子协议的选择
  • 执行任何您认为重要的WebSocket起源。
  • 从传入请求获取HTTP标头
  • 获取Servlet HttpSession对象（如果存在）
  • 指定响应状态代码和原因
