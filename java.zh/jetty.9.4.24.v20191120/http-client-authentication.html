认证支持

Jetty的HTTP客户端支持BASIC和DIGEST RFC 7235定义的身份验证机制。

您可以在HTTP客户端实例中配置身份验证凭据，如下所示：

URI uri = new URI("http://domain.com/secure");
String realm = "MyRealm";
String user = "username";
String pass = "password";

// Add authentication credentials
AuthenticationStore auth = httpClient.getAuthenticationStore();
auth.addAuthentication(new BasicAuthentication(uri, realm, user, pass));

ContentResponse response = httpClient
        .newRequest(uri)
        .send()
        .get(5, TimeUnit.SECONDS);

Jetty的HTTP客户端针对服务器发出的质询测试身份验证凭据（请参阅此处有关安全密码混淆的部分 ），如果匹配，它将自动将正确的身份验证头发送到服务器以进行身份验证。如果身份验证成功，它将缓存结果，并将其重新用于相同域和匹配URI的后续请求。

成功匹配的HTTP对话如下：

Application  HttpClient                     Server
     |           |                             |
     |--- GET ---|------------ GET ----------->|
     |           |                             |
     |           |<-- 401 + WWW-Authenticate --|
     |           |                             |
     |           |--- GET + Authentication --->|
     |           |                             |
     |<-- 200 ---|------------ 200 ------------|

该应用程序未接收到与代码401有关的响应事件，这些事件由内部处理HttpClient产生与原始请求类似但正确的请求Authorization标题，然后将代码200的响应中继到应用程序。

成功的身份验证会被缓存，但是可以清除它们以便再次强制身份验证：

httpClient.getAuthenticationStore().clearAuthenticationResults();

可以通过以下方式抢占身份验证，以免由于服务器质询而导致额外的往返：

AuthenticationStore auth = httpClient.getAuthenticationStore();
URI uri = URI.create("http://domain.com/secure");
auth.addAuthenticationResult(new BasicAuthentication.BasicResult(uri, "username", "password"));

这样，对给定URI的请求将通过HttpClient立即与Authorization标头，则服务器应以200和资源内容作为响应，而不是401和质询。

也可以通过这种方式仅抢占单个请求的身份验证：

URI uri = URI.create("http://domain.com/secure");
Authentication.Result authn = new BasicAuthentication.BasicResult(uri, "username", "password")
Request request = httpClient.newRequest(uri);
authn.apply(request);
request.send();

另请参阅代理身份验证部分，以获取有关身份验证如何与HTTP代理一起工作的更多信息。
