异步servlet最初是与Jetty 6 Continuations一起引入的，Jetty 6 Continuations是Jetty特定的机制。从Jetty 7开始，Continuations API已扩展为通用API，可在任何servlet-3.0容器以及Jetty 6、7或8上异步工作。在任何Servlet 2.5容器中，继续操作也将以阻塞模式工作。

获得延续

ContinuationSupport工厂类可用于获取与请求关联的延续实例：

Continuation continuation = ContinuationSupport.getContinuation(request);

暂停请求

要挂起请求，可以在继续中调用suspend方法：

    void doGet(HttpServletRequest request, HttpServletResponse response)
      {
      ...
      // optionally:
      // continuation.setTimeout(long);
      continuation.suspend();
      ...
      }

请求的生命周期将扩展到从Servlet.service(...)方法和Filter.doFilter(...)电话。当这些分派方法返回时，暂挂的请求将不再提交，并且响应也不会发送到HTTP客户端。

一旦请求被挂起，就应该向异步服务注册该继续，以便在等待事件发生时，异步回调可以使用该继续。

该请求将被暂停，直到其中一个continuation.resume()要么continuation.complete()叫做。如果两者都不被调用，则继续将超时。超时应在挂起之前通过调用来设置continuation.setTimeout(long)如果未设置超时，则使用默认期限。如果没有超时侦听器继续执行或完成延续，则继续执行continuation.isExpired()真正。

暂停类似于Servlet 3.0 request.startAsync()方法。与码头6延续不同，暂停不会引发异常，并且该方法应正常返回。这样就可以在暂停后继续记录，并且不需要互斥体。如果需要一个异常（绕过不知道连续性并且可能尝试提交响应的代码），那么continuation.undispatch()可以通过抛出一个异常来调用以从当前调度中退出当前线程ContinuationThrowable 。

恢复请求

一旦发生异步事件，就可以恢复继续：

    void myAsyncCallback(Object results)
    {
    continuation.setAttribute("results",results);
    continuation.resume();
    }

当继续执行继续操作时，该请求将重新调度到servlet容器，几乎就像再次接收到该请求一样。但是，在重新调度期间， continuation.isInitial()方法返回false，并且异步处理程序设置的任何属性均可用。

继续简历类似于Servlet 3.0 AsyncContext.dispatch() 。

完成请求

作为恢复请求的替代方法，异步处理程序可以自己编写响应。写入响应后，处理程序必须通过调用complete方法来指示请求处理已完成：

    void myAsyncCallback(Object results)
    {
      writeResults(continuation.getServletResponse(),results);
      continuation.complete();
    }

调用complete之后，容器将计划要提交和刷新的响应。延续完成类似于Servlet 3.0 AsyncContext.complete() 。

继续听众

应用程序可以通过使用ContinuationListener来监视连续状态：

    void doGet(HttpServletRequest request, HttpServletResponse response)
    {
      ...

      Continuation continuation = ContinuationSupport.getContinuation(request);
      continuation.addContinuationListener(new ContinuationListener()
      {
        public void onTimeout(Continuation continuation) { ... }
        public void onComplete(Continuation continuation) { ... }
      });

      continuation.suspend();
      ...
    }

继续侦听器类似于Servlet 3.0 AsyncListener。
