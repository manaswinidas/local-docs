SYSCS_DIAG.STATEMENT_DURATION诊断表功能

您可以使用SYSCS_DIAG.STATEMENT_DURATION诊断表功能来分析有用的SQL语句的执行持续时间 。 derby.log文件或您指定的日志文件。

您还可以使用此诊断表功能来获取瓶颈在应用程序的JDBC代码中的位置的指示。

对于同时启用身份验证和SQL授权的数据库 ，只有数据库所有者才能访问此诊断表功能。有关更多信息，请参见《 Derby安全指南 》中的“配置用户身份验证”和“配置用户授权”。

要访问SYSCS_DIAG.STATEMENT_DURATION诊断表函数，必须使用SQL表函数语法。

例如：
SELECT * 
    FROM TABLE (SYSCS_DIAG.STATEMENT_DURATION()) 
    AS T1
其中T1是用户指定的表名，它是任何有效的标识符。
限制：对于每个事务ID，对于具有该事务ID的最后一条语句，均不返回一行。如果刚结束的事务修改了数据，则在提交或回滚后，事务ID在连接内更改。

您可以将日志文件名指定为SYSCS_DIAG.STATEMENT_DURATION诊断表功能的可选参数。指定日志文件名时，该文件名必须是一个表达式，其数据类型映射到Java字符串。

例如：
SELECT * 
    FROM TABLE (SYSCS_DIAG.STATEMENT_DURATION('somederby.log')) 
    AS T1
提示：默认情况下， Derby日志文件仅包含引导，关闭和错误消息。见derby.stream.error.logSeverityLevel财产和derby.language.logStatementText属性，以获取有关如何将更多信息打印到Derby日志文件的说明。然后，您可以使用SYSCS_DIAG.STATEMENT_DURATION诊断表功能来查询该信息。

返回的表具有下表中显示的列。

表格1。SYSCS_DIAG.STATEMENT_DURATION表函数返回的列
栏名 类型 长度 可空 内容

螺纹 VARCHAR 80 假 线程名称。
