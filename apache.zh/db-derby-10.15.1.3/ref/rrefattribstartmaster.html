startMaster = true属性

startMaster = true属性以主模式启动数据库的复制。

在指定此属性之前，必须彻底关闭主系统上的数据库，执行数据库到从属系统的文件系统副本，并指定startSlave = true属性。有关详细信息，请参阅《 Derby服务器和管理指南 》中“复制数据库”下的“启动和运行复制”主题。

如果用户指定startMaster = true时 ，master数据库已经启动，并且任何未记录的操作都在运行，则启动master的尝试将失败，并显示一条错误消息。

有关复制的更多信息，请参阅《 Derby服务器和管理指南 》中“复制数据库”下的其他主题。

与其他属性结合

您必须在连接URL中（在子协议中）或使用databaseName = nameofDatabase属性来指定数据库名称。

您必须将此属性与slaveHost = hostname属性一起指定。您也可以将这个属性与slavePort = portValue属性一起指定。如果未指定slavePort = portValue属性，则默认端口值为4851。

如果启用了身份验证，则还必须指定此属性以及user = userName和password = userPassword属性。如果同时启用了身份验证和SQL授权，则用户必须是数据库所有者。

您不得将本属性与本节中未提及的任何属性一起指定。

例子

-- start master using database name in subprotocol, default slave 
--  port, authorization
jdbc:derby:myDB;startMaster=true;slaveHost=elsewhere;user=mary;
   password=little88lamb
-- start master using databaseName attribute, non-default slave 
--  port, no security
jdbc:derby:;databaseName=myDB;startMaster=true;slaveHost=elsewhere;
   slavePort=4852
