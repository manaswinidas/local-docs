columnName

在SQL语法的许多地方，可以通过使用tableName或correlationName限定列的名称来表示列的名称。

在某些情况下，您不能使用tableName或correlationName限定columnName ，而必须使用simpleColumnName 。这些情况是：

  • 创建表（ CREATE TABLE语句 ）
  • 在游标中指定可更新的列
  • 在SELECT表达式中列的相关名称中（请参见selectExpression ）
  • 在tableExpression中的列的相关名称中（请参见tableExpression ）
您不能对可更新的列使用correlationName 。以这种方式使用correlationName将导致SQL异常。例如：
SELECT c11 AS col1, c12 AS col2, c13 FROM t1 FOR UPDATE of c11,c13
在此示例中， correlationName col1 FOR c11不允许，因为c11列在FOR UPDATE列中。您可以使用correlationName FOR c12因为它不在FOR UPDATE列表中。

句法

[ { tableName | correlationName } . ] SQLIdentifier

例

-- C.Country is a columnName qualified with a correlationName.
SELECT C.Country
FROM APP.Countries C
