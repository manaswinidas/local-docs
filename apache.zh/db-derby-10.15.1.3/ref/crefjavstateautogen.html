自动生成的密钥

JDBC的自动生成键功能提供了一种从索引的一部分或分配了默认值的列中检索值的方法。

Derby支持自动递增功能，该功能允许用户在表中创建列，数据库系统会为此表自动分配递增的整数值。用户可以调用Statement.getGeneratedKeys方法来检索此类列的值。此方法返回一个ResultSet对象，其中包含用于自动生成的键的列。

仅当最后一条语句是单行插入或更新语句时， Statement.getGeneratedKeys的Derby实现才返回有意义的结果。如果是多行插入或更新，则Statement.getGeneratedKeys将返回仅包含一行的结果集，即使它应为插入或更新的每一行返回一行。

在getGeneratedKeys返回的ResultSet对象上调用ResultSet.getMetaData会产生一个ResultSetMetaData对象，该对象类似于IDENTITY_VAL_LOCAL返回的对象。

用户可以通过将以下值之一作为第二个参数传递给Connection.prepareStatement ， Statement.execute或Statement.executeUpdate方法来指示应使自动生成的列可用于检索：

  • 指示应提供自动生成的密钥的常数。使用的特定常数是Statement.RETURN_GENERATED_KEYS 。
  • 应当使插入或更新的行中的列名的数组。如果阵列中的任何列名称都未指定自动递增列，则Derby将使用Derby嵌入式驱动程序引发错误。使用客户端驱动程序，当前会忽略一个元素列名称，并且返回的值对应于标识列。为了确保与将来的更改兼容，应用程序应确保所描述的列是标识列。如果列名对应于另一列或不存在的列，则将来的更改可能会导致返回另一列的值或引发异常。
  • 应当使插入或更新的行中的列的位置的数组可用。如果阵列中的任何列位置都与自动增量列不相关，则Derby将使用Derby嵌入式驱动程序引发错误。使用客户端驱动程序，当前忽略一个元素位置数组，并且返回的值对应于标识列。为了确保与将来的更改兼容，应用程序应确保所描述的列是标识列。如果位置对应于另一列或不存在的列，则将来的更改可能会导致返回另一列的值或引发异常。

例

假设我们有一个表TABLE1定义如下：

CREATE TABLE TABLE1 (C11 int, C12 int GENERATED ALWAYS AS IDENTITY)

以下三个代码段都将执行相同的操作：也就是说，它们将创建一个ResultSet ，其中包含的值C12插入到TABLE1中。

代码片段1：

Statement stmt = conn.createStatement(); 
stmt.execute(
    "INSERT INTO TABLE1 (C11) VALUES (1)",
    Statement.RETURN_GENERATED_KEYS);
ResultSet rs = stmt.getGeneratedKeys();

代码片段2：

Statement stmt = conn.createStatement(); 
String [] colNames = new String [] { "C12" };
stmt.execute(
    "INSERT INTO TABLE1 (C11) VALUES (1)",
    colNames);
ResultSet rs = stmt.getGeneratedKeys();

代码片段3：

Statement stmt = conn.createStatement(); 
int [] colIndexes = new int [] { 2 };
stmt.execute(
    "INSERT INTO TABLE1 (C11) VALUES (1)",
    colIndexes);
ResultSet rs = stmt.getGeneratedKeys();

如果没有迹象表明应该使自动生成的列可用于检索，则对Statement.getGeneratedKeys的调用将返回空ResultSet 。
