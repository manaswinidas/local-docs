encryptionKey =密钥属性

encryptionKey = key属性指定一个外部加密密钥。

该属性指定用于以下目的的外部键：
  • 加密新数据库
  • 配置现有的未加密数据库进行加密
  • 引导现有的加密数据库
您的应用程序必须提供加密密钥。加密密钥值必须是长度至少为16位（8字节）的十六进制字符串，并且必须包含偶数个数字。

有关数据加密的信息，请参阅《 Derby安全指南 》中的“配置数据库加密”。

与其他属性结合

创建新数据库时，必须将cryptoKey = key属性与create = true和dataEncryption = true属性结合使用。

当配置现有的未加密数据库进行加密时，必须将encryptionKey = key属性与dataEncryption = true属性结合使用。对于同时启用身份验证和SQL授权的现有未加密数据库，只有数据库所有者才能执行加密。有关更多信息，请参见《 Derby安全指南 》中的“配置用户身份验证”和“配置用户授权”。

可以将加密密钥=密钥属性与加密提供者= providerName ， 加密算法=算法和/或加密密钥长度=长度属性组合。

例子

-- create a new encrypted database
jdbc:derby:newDB;create=true;dataEncryption=true;
   encryptionKey=6162636465666768
-- configure an existing unencrypted database for encryption
jdbc:derby:salesdb;dataEncryption=true;encryptionKey=6162636465666768
-- boot an encrypted database
jdbc:derby:encryptedDB;encryptionKey=6162636465666768
