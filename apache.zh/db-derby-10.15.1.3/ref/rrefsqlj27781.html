MAX功能

MAX是一个聚合函数，用于评估一组行中表达式的最大值。

有关这些功能的更多信息，请参见聚合（设置功能） 。

MAX仅在计算结果为可索引数据类型的表达式上允许使用（特别是，在第二种表中，在数据类型分配和比较，排序和排序中，用Y标记的表达式“ Derby允许的比较”）。这意味着，MAX不能与用于BLOB，CLOB，LONG VARCHAR，LONG VARCHAR FOR BIT DATA，XML或用户定义类型的表达式一起使用。

句法

MAX ( [ DISTINCT | ALL ] expression )
DISTINCT和ALL限定词消除或保留重复项，但是这些限定词对MAX表达式没有影响。每个selectExpression仅允许一个DISTINCT聚合表达式。例如，不允许以下查询：
SELECT COUNT (DISTINCT flying_time), MAX (DISTINCT miles)
FROM Flights

该表达式可以包含多个列引用或表达式，但不能包含另一个聚合或子查询。它必须求值为内置数据类型。因此，您可以调用评估为内置数据类型的方法。（例如，一个返回java.lang的方法。整数或整数的值等于一个整数。）如果表达式的计算结果为NULL，则聚合将跳过该值。

类型的比较规则确定最大值。对于CHAR和VARCHAR，值末尾的空格数会影响对MAX求值的方式。例如，如果值“ z”和“ z”都存储在一列中，则您将无法控制以最大值返回哪个值，因为字符比较会忽略空格。

结果数据类型与其所操作的表达式相同（它将永远不会溢出）。

例子

-- find the latest date in the FlightAvailability table
SELECT MAX (flight_date) FROM FlightAvailability
-- find the longest flight originating from each airport,
-- but only when the longest flight is over 10 hours
SELECT MAX(flying_time), orig_airport
FROM Flights
GROUP BY orig_airport
HAVING MAX(flying_time) > 10
