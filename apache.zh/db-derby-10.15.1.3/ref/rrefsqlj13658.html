ORDER BY子句

ORDER BY子句是几个语句，表达式和子查询的可选元素。

它可以是以下元素：

  • SELECT语句
  • selectExpression
  • 值表达式
  • 标量子查询
  • 一个tableSubquery

也可以在INSERT语句或CREATE VIEW语句中使用它 。

ORDER BY子句允许您指定结果集中行的出现顺序。在子查询中，除非ORDER BY子句附带结果offset和fetch first子句中的一个或两个，或者与ROW_NUMBER函数一起使用，否则它是没有意义的，因为不能保证顺序保留在外部结果集中。允许将外部查询上的ORDER BY与子查询中的ORDER BY组合。

句法

ORDER BY { columnName | columnPosition | expression }
    [ ASC | DESC ]
    [ NULLS FIRST | NULLS LAST ]
    [ , columnName | columnPosition | expression 
        [ ASC | DESC ]
        [ NULLS FIRST | NULLS LAST ]
    ]*
columnName
指的是从selectItems在SELECT语句的基础查询中。您在ORDER BY子句中指定的columnName不必是SELECT列表。
columnPosition
一个整数，用于标识列中的列号selectItems在SELECT语句的基础查询中。columnPosition必须大于0且不大于结果表中的列数。换句话说，如果要按列排序，则必须在SELECT列表中指定该列。
表达
排序键表达式，例如数字，字符串和日期时间表达式。表达式也可以是行值表达式，例如scalarSubquery或case表达式。
ASC
指定结果应按升序返回。如果未指定顺序，则默认为ASC。
数据中心
指定结果应按降序返回。
首先为空
指定应在非NULL值之前返回NULL值。
倒数最后
指定应在非NULL值之后返回NULL值。

笔记

  • 如果指定了SELECT DISTINCT或SELECT语句包含GROUP BY子句，则ORDER BY列必须在SELECT列表中。
  • ORDER BY子句可防止SELECT语句成为可更新的游标。有关更多信息，请参见可更新游标和可更新ResultSet的要求 。
  • 如果未指定空顺序，则对空值的处理为：
    • 如果排序是ASC，则NULLS LAST
    • 如果排序为DESC，则先为NULL
  • 如果既未指定升序也未指定降序，并且也未指定空值排序，则将使用两个默认值，因此该顺序将以NULLS LAST升序。

使用相关名称的示例

如果在选择列表中指定了相关名称，则可以按相关名称对结果集进行排序。例如，要从CITIES数据库返回CITY_NAME和COUNTRY列中的所有条目，其中COUNTRY列具有相关名称NATION，请指定此SELECT语句：

SELECT CITY_NAME, COUNTRY AS NATION 
    FROM CITIES 
    ORDER BY NATION

使用数字表达式的示例

您可以按数字表达式对结果集进行排序，例如：
SELECT name, salary, bonus FROM employee 
   ORDER BY salary+bonus
在此示例中，salary和bonus列是DECIMAL数据类型。

使用函数的例子

您可以通过调用函数对结果集进行排序，例如：
SELECT i, len FROM measures 
   ORDER BY sin(i)

指定空顺序的示例

您可以使用null排序规范来指定NULL值的位置：
SELECT * FROM t1 ORDER BY c1 DESC NULLS LAST
