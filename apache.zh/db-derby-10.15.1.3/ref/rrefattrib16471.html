shutdown = true属性

如果您指定了databaseName，则shutdown = true属性将关闭指定的数据库 。（重新连接到数据库会重新启动数据库。）

对于同时启用身份验证和SQL授权的数据库 ，只有数据库所有者才能关闭该数据库。有关更多信息，请参见《 Derby安全指南 》中的“配置用户身份验证”和“配置用户授权”。

当且仅当您未指定databaseName时 ，才关闭整个Derby系统。

关闭单个数据库时，该属性允许Derby在数据库上执行最终检查点。

关闭系统时，该属性允许Derby在所有系统数据库上执行最终检查点，注销JDBC驱动程序（如果允许；请参见deregister = false ），并在JVM退出之前在JVM中关闭。成功关闭将始终导致SQLException ，该异常指示Derby已关闭并且没有连接。一旦关闭Derby ，您可以通过重新加载驱动程序来重新启动它。有关重新启动Derby的详细信息，请参阅《 Derby开发人员指南 》中的“关闭系统”。

检查点是指将所有数据和事务信息写入磁盘，以便在下一个连接时无需执行恢复。

该属性仅在将其嵌入应用程序中时才用于关闭整个系统。
注意：对具有shutdown = true属性的DriverManager的任何请求都会引发异常。

与其他属性结合

该属性可以与deregister = false属性结合使用。

关闭数据库时，如果启用了身份验证，则必须与user = userName和password = userPassword属性一起指定此属性。如果同时启用了身份验证和SQL授权，则用户必须是数据库所有者。

关闭Derby系统时，如果同时启用了身份验证和SQL授权，则必须与user = userName和password = userPassword属性一起指定此属性。

例子

-- shut down entire system
jdbc:derby:;shutdown=true
-- shut down salesDB (authentication not enabled)
jdbc:derby:salesDB;shutdown=true
-- shut down an in-memory database using the embedded driver
jdbc:derby:memory:myInMemDB;shutdown=true
-- shut down an in-memory database using the Network Server
jdbc:derby://localhost:1527/memory:myInMemDB;shutdown=true
