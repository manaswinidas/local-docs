大写和特殊字符

使用JDBC的类和方法，您可以将SQL语句作为字符串提交给Derby 。包含SQL语句的字符串允许的字符集是Unicode。

在这些字符串中，适用以下规则：
  • 双引号分隔在SQL中称为分隔标识符的特殊标识符 。
  • 单引号分隔字符串。
  • 在字符串中，要表示单引号或撇号，请使用两个单引号。（换句话说，单引号是单引号的转义字符。）

    双引号不需要转义符。要表示双引号，只需使用双引号。但是，请注意，在Java程序中，双引号需要反斜杠转义符。

    例：
    -- a single quotation mark is the escape character
-- for a single quotation mark

VALUES 'Joe''s umbrella'
-- in ij, you don't need to escape the double quotation marks
VALUES 'He said, "hello!"'

n = stmt.executeUpdate(
    "UPDATE aTable setStringcol = 'He said, \"hello!\"'");
  • SQL关键字不区分大小写。例如，您可以将关键字SELECT输入为SELECT，选择，选择或选择。
  • SQL样式的标识符不区分大小写（请参阅SQLIdentifier ），除非它们是定界的。
  • Java样式的标识符始终区分大小写。
  • *是selectExpression中的通配符。 请参阅*通配符 。它也可以是乘法运算符。在所有其他情况下，它是一个语法上的元符号，标记您可以重复0次或多次的项目。
  • 在LIKE运算符后面的字符串中使用时，％和_是字符通配符（使用转义字符进行转义时除外）。请参见布尔表达式 。
  • 根据SQL标准，注释可以是单行或多行。单行注释以两个破折号（-）开头，并以换行符结束。多行注释括在方括号中，以星号正斜杠（/ *）开头，以星号正斜杠（* /）结尾。请注意，方括号中的注释可能是嵌套的。在开始和结束注释字符序列之间的任何文本都将被忽略。
