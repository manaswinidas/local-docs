encryptionKeyLength =长度属性

encryptionKeyLength = length属性指定使用bootPassword = key属性对数据库进行加密时要生成的加密密钥中的位数。

有关详细信息，请参见bootPassword = key属性 。

默认的加密密钥长度为128。

仅在满足以下所有情况时，才需要指定encryptionKeyLength = length ：
  • 您正在指定非默认加密算法（默认为DES）。
  • 您指定的加密算法允许多个密钥长度。
  • 您要使用非默认密钥长度。

有关数据加密的信息，请参阅《 Derby安全指南 》中的“配置数据库加密”。

与其他属性结合

encryptionKeyLength = length属性可以与bootPassword = key属性或encryptionKey = key属性结合使用（尽管如果使用encryptionKey = key，则是多余的）。

EncryptionKeyLength = length属性也可以与encryptionProvider = providerName和/或encryptionAlgorithm = algorithm属性结合使用。

如果您将加密密钥长度 （ encryptionKeyLength = length）与加密密钥= key一起使用 ，则指定的密钥必须具有指定的长度。

例子

-- create a new, encrypted database
jdbc:derby:newDB;create=true;dataEncryption=true;
    encryptionKeyLength=192;encryptionAlgorithm=AES/CBC/NoPadding;
    bootPassword=Thursday
-- configure an existing unencrypted database for encryption
jdbc:derby:myDB;dataEncryption=true;
    encryptionKeyLength=168;encryptionAlgorithm=DESede/CBC/NoPadding;
    bootPassword=Wednesday
注意：如果指定的算法不支持指定的加密密钥长度，则Derby将返回异常。
