CHAR FOR BIT DATA数据类型

CHAR FOR BIT DATA类型允许您存储指定长度的字节字符串。对于字符串不合适的非结构化数据很有用。

句法

{ CHAR | CHARACTER }[(length)] FOR BIT DATA

length是一个无符号整数文字，以字节为单位指定长度。

CHAR FOR BIT DATA类型的默认长度为1，最大长度为254个字节。

JDBC元数据类型（java.sql。类型）

二进制

CHAR FOR BIT DATA存储固定长度的字节字符串。如果CHAR FOR BIT DATA值小于目标CHAR FOR BIT DATA，则将其填充为0x20字节值。

CHAR FOR BIT DATA和VARCHAR FOR BIT DATA值的比较是精确的。为了使两个位串相等，它们的长度必须完全相同 。（这与某些其他DBMS处理BINARY值的方式不同，但按照SQL中的规定工作。）

对VARCHAR FOR BIT DATA和CHAR FOR BIT DATA值（例如，级联）进行运算会产生VARCHAR FOR BIT DATA值。

例

CREATE TABLE t (b CHAR(2) FOR BIT DATA);
INSERT INTO t VALUES (X'DE');
SELECT *
FROM t;
-- yields the following output
B
-----
de20
