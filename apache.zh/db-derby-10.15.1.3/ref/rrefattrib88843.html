encryptionProvider = providerName属性

encryptionProvider = providerName属性指定数据加密的提供程序。

有关数据加密的信息，请参阅《 Derby安全指南 》中的“配置数据库加密”。

如果未指定此属性，则默认的加密提供程序是您所使用的JVM中包括的提供程序。

与其他属性结合

EncryptionProvider = providerName属性必须与dataEncryption = true属性以及bootPassword = key或encryptionKey = key属性结合使用。您也可以使用encryptionAlgorithm = algorithm属性来指定加密算法。

对于同时启用身份验证和SQL授权的现有未加密数据库，只有数据库所有者可以执行加密或重新加密。有关更多信息，请参见《 Derby安全指南 》中的“配置用户身份验证”和“配置用户授权”。

例子

-- create a new, encrypted database
jdbc:derby:encryptedDB;create=true;dataEncryption=true;
    encryptionProvider=com.sun.crypto.provider.SunJCE;
    encryptionAlgorithm=DESede/CBC/NoPadding;
    bootPassword=cLo4u922sc23aPe
-- configure an existing database for encryption 
jdbc:derby:salesdb;dataEncryption=true; 
    encryptionProvider=com.sun.crypto.provider.SunJCE; 
    encryptionAlgorithm=DESede/CBC/NoPadding; 
    bootPassword=cLo4u922sc23aPe
