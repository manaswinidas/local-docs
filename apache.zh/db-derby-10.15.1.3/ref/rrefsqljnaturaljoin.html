自然联接操作

NATURAL JOIN是一个JOIN操作，它基于要连接的两个表中的公共列为您创建一个隐式连接子句。公共列是在两个表中具有相同名称的列。

NATURAL JOIN可以是INNER联接，LEFT OUTER联接或RIGHT OUTER联接。默认值为INNER join。

如果在其中出现NATURAL JOIN操作的SELECT语句在选择列表中带有星号（*），则星号将扩展到以下列列表（按此顺序）：

  • 所有常用栏
  • 第一（左）表中的每一列都不是公共列
  • 第二个（右）表中的每一列都不是公共列

由表名限定的星号（例如COUNTRIES。*）将扩展到该表的非公共列的每一列。

如果在没有表名限定的情况下引用公共列，则如果联接是INNER JOIN或LEFT OUTER JOIN，则该列引用指向第一个（左）表中的列。如果它是RIGHT OUTER JOIN，则对公共列的不合格引用将指向第二个（右）表中的列。

句法

tableExpression NATURAL [ { LEFT | RIGHT } [ OUTER ] | INNER ] JOIN
{ 
    tableViewOrFunctionExpression |
    ( tableExpression )
}

例子

如果表COUNTRIES和CITIES具有两个名为COUNTRY和COUNTRY_ISO_CODE的公共列，则以下两个SELECT语句是等效的：

SELECT * FROM COUNTRIES NATURAL JOIN CITIES
SELECT * FROM COUNTRIES JOIN CITIES
    USING (COUNTRY, COUNTRY_ISO_CODE)

以下示例与上面的示例相似，但是它还保留了第一个（左）表中不匹配的行：

SELECT * FROM COUNTRIES NATURAL LEFT JOIN CITIES
