cryptoDatabase = true属性

cryptoDatabase = true属性将加密的数据库返回到未加密状态。

有关数据加密的信息，请参阅《 Derby安全指南 》中的“配置数据库加密”。

您必须先关闭数据库，然后再解密。尝试解密已引导的数据库无效。

为未加密的数据库指定此属性无效。

与其他属性结合

cryptoDatabase = true属性必须与bootPassword = key属性或encryptionKey = key属性结合使用。

对于同时启用身份验证和SQL授权的现有加密数据库，只有数据库所有者才能执行解密。有关更多信息，请参见《 Derby安全指南 》中的“配置用户身份验证”和“配置用户授权”。

解密数据库后，请确保检查SQLWarning 。只有在没有SQLWarning或SQLException的情况下，解密才能成功。

例子

-- decrypt a database
jdbc:derby:encryptedDB;decryptDatabase=true;bootPassword=cLo4u922sc23aPe
-- decrypt a database with authentication and SQL authorization enabled
jdbc:derby:salesdb;decryptDatabase=true;user=user1;password=mypass;
    bootPassword=cLo4u922sc23aPe
