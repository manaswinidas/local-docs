dataEncryption = true属性

dataEncryption = true属性为新数据库指定磁盘上的数据加密，或为加密配置现有的未加密数据库。

有关数据加密的信息，请参阅《 Derby安全指南 》中的“配置数据库加密”。

加密数据库后，可以通过指定cryptoDatabase = true属性将其返回到未加密状态。

与其他属性结合

dataEncryption = true属性必须与bootPassword = key属性或encryptionKey = key属性结合使用。

使用bootPassword = key或encryptionKey = key时 ，您还可以选择指定encryptionProvider = providerName ， encryptionAlgorithm = algorithm和/或encryptionKeyLength = length属性。

加密密钥和启动密码使用不同的存储机制，因此，例如，如果使用启动密码创建数据库，则在启动数据库时必须继续指定启动密码。您不能切换到加密密钥。您可以通过指定newBootPassword = newPassword或newEncryptionKey = key来更改引导密码或加密密钥 。

对于同时启用身份验证和SQL授权的现有未加密数据库，只有数据库所有者才能执行加密。有关更多信息，请参见《 Derby安全指南 》中的“配置用户身份验证”和“配置用户授权”。

对现有的未加密数据库进行加密之后，请确保检查SQLWarning 。仅在没有SQLWarning或SQLException的情况下加密才成功。

例子

-- encrypt a new database
jdbc:derby:encryptedDB;create=true;dataEncryption=true;
    bootPassword=cLo4u922sc23aPe
-- configure an existing unencrypted database for encryption 
jdbc:derby:salesdb;dataEncryption=true;bootPassword=cLo4u922sc23aPe
