startSlave = true属性

startSlave = true属性以从属模式启动数据库复制。

在指定此属性之前，必须完全关闭主系统上的数据库，然后执行数据库到从系统的文件系统副本。

startSlave = true属性执行以下操作：

  1. 部分引导指定的数据库
  2. 开始侦听指定的端口并接受主服务器的连接
  3. 挂起，直到主机连接到主机
  4. 向呼叫者报告启动状态（是否已经启动，如果没有启动，为什么不启动）
  5. 不断从主数据库接收事务日志的大块并将事务日志中的操作应用于从数据库

如果复制成功启动，则会引发错误代码XRE08的异常。因此， startSlave = true属性不会返回有效的连接。

有关更多信息，请参阅《 Derby服务器和管理指南 》中“复制数据库”下的主题。

与其他属性结合

您必须在连接URL中（在子协议中）或使用databaseName = nameofDatabase属性来指定数据库名称。

您可以将此属性与slaveHost = hostname属性一起指定。如果未指定slaveHost = hostname属性，则默认值为localhost 。

您也可以将这个属性与slavePort = portValue属性一起指定。如果未指定slavePort = portValue属性，则默认端口值为4851。

如果启用了身份验证，则还必须指定此属性以及user = userName和password = userPassword属性。如果同时启用了身份验证和SQL授权，则用户必须是数据库所有者。

您不得将本属性与本节中未提及的任何属性一起指定。

例子

-- start slave using database name in subprotocol, default slave host 
--    and port, authorization
jdbc:derby:myDB;startslave=true;user=mary;password=little88lamb
-- start slave using databaseName attribute, non-default slave host 
--   and port, no security
jdbc:derby:;databaseName=myDB;startSlave=true;slaveHost=localhost;
   slavePort=4852
