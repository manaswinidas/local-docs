traceLevel =值属性

如果启用了跟踪，则traceLevel = value属性指定Derby Network Client使用的跟踪级别。

该值为数字。如果未指定跟踪级别，则默认值为TRACE_ALL。

有关跟踪的更多信息，请参阅《 Derby服务器和管理指南 》中的“网络客户端跟踪”。有关与跟踪相关的其他属性，请参见traceFile = path ， traceFileAppend = true和traceDirectory = path 。

要指定多个跟踪级别，请使用以下技术之一：

  • 如果您正在使用ij工具，将十进制值加在一起并指定总和。例如，要跟踪PROTOCOL流和连接调用，请添加TRACE_PROTOCOL_FLOWS（64）和TRACE_CONNECTION_CALLS（1）的值。指定总和，值65。
  • 如果您正在运行JDBC程序，请执行以下一项操作：
    • 对两个或多个跟踪值使用按位OR运算符（|）。例如，要跟踪协议流和连接调用，请为traceLevel指定以下值：
      TRACE_PROTOCOL_FLOWS | TRACE_CONNECTION_CALLS
    • 使用带跟踪值的按位补运算符（〜）可以指定除特定跟踪以外的所有其他值。例如，要跟踪除协议流以外的所有内容，请为traceLevel指定以下值：
      ~TRACE_PROTOCOL_FLOWS

与其他属性结合

如果指定此属性，则还必须指定traceFile = path属性或traceDirectory = path属性。

例

-- enable tracing on a new database
jdbc:derby://localhost:1527/mydb;create=true;traceFile=trace.out;
    traceLevel=65
