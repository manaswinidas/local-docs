stopMaster = true属性

stopMaster = true属性在主系统上停止数据库复制。

如果网络连接正常，则此属性将停止从站消息发送到从属系统。然后，它关闭所有与复制相关的功能，而不关闭指定的数据库。

指定此属性的Derby实例必须是指定数据库的复制主数据库。

有关更多信息，请参阅《 Derby服务器和管理指南 》中“复制数据库”下的主题。

与其他属性结合

您必须在连接URL中（在子协议中）或使用databaseName = nameofDatabase属性来指定数据库名称。

如果启用了身份验证，则还必须指定此属性以及user = userName和password = userPassword属性。如果同时启用了身份验证和SQL授权，则用户必须是数据库所有者。

您不得将本属性与本节中未提及的任何属性一起指定。

例子

-- stop master using database name in subprotocol, authorization
jdbc:derby:myDB;stopMaster=true;user=mary;password=little88lamb
-- stop master using databaseName attribute, no security
jdbc:derby:;databaseName=myDB;stopMaster=true;
