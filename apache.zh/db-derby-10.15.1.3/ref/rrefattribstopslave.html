stopSlave = true属性

stopSlave = true属性停止从属系统上的数据库复制。

仅当主系统和从系统之间的网络连接断开时，才可以指定此连接URL属性。如果网络连接正常，则从系统仅接受来自主系统的命令，因此必须在主系统上指定stopMaster = true属性，以停止在主系统和从系统上进行复制。

指定此属性时，或者在执行stopMaster = true属性的过程中发送停止-从属消息时，将从主服务器接收到的所有事务日志块都写入磁盘。然后将关闭从属复制功能和数据库。

指定了此属性的Derby实例必须以复制从模式服务于指定的数据库。

有关更多信息，请参阅《 Derby服务器和管理指南 》中“复制数据库”下的主题。

与其他属性结合

您必须在连接URL中（在子协议中）或使用databaseName = nameofDatabase属性来指定数据库名称。

如果启用了身份验证，则还必须指定此属性以及user = userName和password = userPassword属性。当网络连接断开时，无法检查主数据库的授权，因此不会强制要求用户必须是数据库所有者。

您不得将本属性与本节中未提及的任何属性一起指定。

例子

-- stop slave from master using database name in subprotocol,
--   authorization
jdbc:derby:myDB;stopSlave=true;user=mary;password=little88lamb
-- stop slave using databaseName attribute, no security
jdbc:derby:;databaseName=myDB;stopSlave=true;
