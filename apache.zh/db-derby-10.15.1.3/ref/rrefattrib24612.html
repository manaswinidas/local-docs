设置数据库连接URL的属性

Derby允许您为其数据库连接URL提供属性列表，这是JDBC功能。

这些属性特定于Derby 。

通常，您在协议和子协议（在某些情况下，还包括子子协议）之后，以分号分隔的列表设置属性。有关如何设置属性的信息，请参阅Derby数据库连接URL的属性 。本节仅提供参考信息。

DriverManager.getConnection方法可以同时使用连接URL和Properties对象作为参数。如果在连接URL和“ 属性”对象中都指定了任何属性，则连接URL的属性将覆盖“属性”对象中的属性 。

注意：不会解析属性的正确性。如果您输入了错误的属性或相应的值，则将其忽略。
  • bootPassword =键属性
    bootPassword = key属性指定启动密码（加密密钥）。
  • 排序规则=排序规则属性
    collation = collation属性是一个可选属性，它指定排序规则是基于为数据库指定的语言环境还是基于Unicode代码点排序规则。
  • create = true属性
    create = true属性创建数据库连接URL Derby系统中指定的标准数据库，然后连接到该数据库。
  • createFrom = path属性
    createFrom = path属性使用指定位置的完整备份创建数据库。
  • databaseName = nameOfDatabase属性
    databaseName = nameOfDatabase属性指定连接的数据库路径名。
  • dataEncryption = true属性
    dataEncryption = true属性为新数据库指定磁盘上的数据加密，或为加密配置现有的未加密数据库。
  • cryptoDatabase = true属性
    cryptoDatabase = true属性将加密的数据库返回到未加密状态。
  • deregister = false属性
    如果将deregister = false属性设置为true（默认值），则在关闭后，该属性将从DriverManager中注销嵌入式JDBC驱动程序，以便可以对Derby类进行垃圾收集。
  • drop = true属性
    drop = true属性将删除数据库连接URL中指定的内存数据库。
  • encryptionKey =密钥属性
    encryptionKey = key属性指定一个外部加密密钥。
  • encryptionKeyLength =长度属性
    encryptionKeyLength = length属性指定使用bootPassword = key属性对数据库进行加密时要生成的加密密钥中的位数。
  • encryptionProvider = providerName属性
    encryptionProvider = providerName属性指定数据加密的提供程序。
  • encryptionAlgorithm =算法属性
    encryptionAlgorithm = algorithm属性指定数据加密算法。
  • failover = true属性
    failover = true属性停止从属系统上的数据库复制，并将从属数据库转换为普通数据库。
  • logDevice = logDirectoryPath属性
    logDevice = logDirectoryPath属性指定在数据库创建或还原期间要存储数据库日志的目录的路径。
  • newBootPassword = newPassword属性
    newBootPassword = newPassword属性为加密的数据库指定新的启动密码。
  • newEncryptionKey =密钥属性
    newEncryptionKey = key属性为加密的数据库指定新的外部加密密钥。
  • password = userPassword属性
    password = userPassword属性指定给定用户名的有效密码。
  • restoreFrom = path属性
    restoreFrom = path属性使用指定位置的完整备份来还原数据库。
  • resolveMessageText = false属性
    resolveMessageText = false属性（如果设置为true）（默认值），则从服务器检索每个SQLException的消息文本。
  • rollForwardRecoveryFrom = path属性
    rollForwardRecoveryFrom = path属性使用备份副本还原数据库，并使用已归档和活动日志执行前滚恢复。
  • securityMechanism =值属性
    securityMechanism = value属性指定用于客户端访问网络服务器的安全性机制。
  • shutdown = true属性
    如果您指定了databaseName，则shutdown = true属性将关闭指定的数据库。（重新连接到数据库会重新启动数据库。）
  • slaveHost =主机名属性
    slaveHost = hostname属性指定将用作数据库复制的从属系统。
  • slavePort = portValue属性
    slavePort = portValue属性指定从属系统将在数据库复制中使用的端口。
  • ssl = sslMode属性
    ssl = sslMode属性指定客户端的SSL模式。
  • startMaster = true属性
    startMaster = true属性以主模式启动数据库的复制。
  • startSlave = true属性
    startSlave = true属性以从属模式启动数据库复制。
  • stopMaster = true属性
    stopMaster = true属性在主系统上停止数据库复制。
  • stopSlave = true属性
    stopSlave = true属性停止从属系统上的数据库复制。
  • region = ll_CC属性
    boundary = ll_CC属性在数据库创建时将非默认语言环境与数据库相关联。
  • traceDirectory =路径属性
    traceDirectory = path属性指定Derby Network Client将JDBC跟踪信息发送到的目录。
  • traceFile =路径属性
    traceFile = path属性指定Derby Network Client将JDBC跟踪信息发送到的文件。
  • traceFileAppend = true属性
    traceFileAppend = true属性指定Derby Network Client应该将JDBC跟踪信息附加到跟踪文件。
  • traceLevel =值属性
    如果启用了跟踪，则traceLevel = value属性指定Derby Network Client使用的跟踪级别。
  • upgrade = true属性
    upgrade = true属性会将使用早期版本的Derby创建的数据库升级到当前版本的Derby，然后连接到该数据库。
  • user = userName属性
    user = userName属性指定系统的有效用户名以及密码。
  • 在不指定属性的情况下创建连接
    如果未指定任何属性，则必须指定一个databaseName。
