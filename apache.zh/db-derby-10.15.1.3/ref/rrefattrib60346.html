encryptionAlgorithm =算法属性

encryptionAlgorithm = algorithm属性指定数据加密算法。

指定算法时，请使用Java约定。例如：

algorithmName/feedbackMode/padding

Derby唯一允许的填充类型是NoPadding 。

如果未指定加密算法，则默认值为DES / CBC / NoPadding 。

有关数据加密的信息，请参阅《 Derby安全指南 》中的“配置数据库加密”。

与其他属性结合

encryptionAlgorithm = algorithm属性必须与dataEncryption = true属性以及bootPassword = key属性或encryptionKey = key属性结合使用。您还可以使用encryptionProvider = providerName属性来指定算法的加密提供程序。

对于同时启用身份验证和SQL授权的现有数据库，只有数据库所有者可以执行加密或重新加密。有关更多信息，请参见《 Derby安全指南 》中的“配置用户身份验证”和“配置用户授权”。

例子

-- encrypt a new database
jdbc:derby:encryptedDB;create=true;dataEncryption=true;
    encryptionProvider=com.sun.crypto.provider.SunJCE;
    encryptionAlgorithm=DESede/CBC/NoPadding;
    bootPassword=cLo4u922sc23aPe
-- configure an existing database for encryption 
jdbc:derby:salesdb;dataEncryption=true; 
    encryptionProvider=com.sun.crypto.provider.SunJCE; 
    encryptionAlgorithm=DESede/CBC/NoPadding; 
    bootPassword=cLo4u922sc23aPe
注意：如果指定的提供程序不支持指定的算法，则Derby将返回异常。
