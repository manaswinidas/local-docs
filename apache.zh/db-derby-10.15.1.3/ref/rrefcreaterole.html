CREATE ROLE语句

CREATE ROLE语句创建一个SQL角色。当数据库有许多用户时，角色对于管理特权很有用。

只有数据库所有者可以创建角色。

有关角色的更多信息，请参阅《 Derby安全指南 》中的“使用SQL角色”。

句法

CREATE ROLE roleName

在发出CREATE ROLE语句之前，请验证derby.database.sqlAuthorization属性是否设置为TRUE 。derby.database.sqlAuthorization属性启用SQL授权模式。

如果有一个使用该名称的用户，则无法创建该名称。尝试创建与现有用户名冲突的角色名会引发SQLException X0Y68。

如果用户名不受数据库所有者（或管理员）的控制，则最好为角色使用命名约定以减少与用户名冲突的可能性。

Derby试图避免用户名和角色名之间的名称冲突，但这并不总是可能的，因为Derby具有可插入的授权体系结构。例如，可能存在一个外部定义的用户，该用户尚未连接到数据库，创建任何模式对象或被授予任何特权。如果Derby知道用户名，它将禁止使用该名称创建角色。相应地，与角色同名的用户将被禁止连接。 创建角色时，将检查Derby内置用户的碰撞情况。

角色名称不能以前缀SYS开头（大小写标准化后）。该限制的目的是在以后为系统定义的角色保留名称空间。使用前缀SYS会引发SQLException 4293A。

您不能使用以下名称创建角色PUBLIC （大小写归一化之后）。 PUBLIC是保留的授权标识符。尝试使用名称创建角色PUBLIC引发SQLException 4251B。

创建角色的示例

CREATE ROLE purchases_reader;

无效的角色名称示例

CREATE ROLE public;   -- throws SQLException;
CREATE ROLE "PUBLIC"; -- throws SQLException;
CREATE ROLE sysrole;  -- throws SQLException;

使用命名约定创建角色的示例

以下示例使用约定为每个角色名称添加后缀_role 。

CREATE ROLE purchases_reader_role;
