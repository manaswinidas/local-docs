create = true属性

create = true属性创建在数据库连接URL Derby系统内指定的标准数据库，然后连接到该数据库。

如果无法创建数据库，则错误将显示在错误日志中，并且连接尝试将失败，并显示SQLException，指示找不到数据库。

如果数据库已经存在，则该属性创建到现有数据库的连接，并发出SQLWarning 。

如果在数据库调用发生后发生故障，则在创建时连接失败时，JDBC不会删除数据库。如果数据库连接URL使用create = true ，并且连接创建失败，请检查数据库目录。如果存在，请在下一次尝试创建数据库之前删除它及其内容。

数据库所有者

创建数据库后，当前的授权标识符将成为数据库所有者（请参阅user = userName属性）。如果同时启用了身份验证和SQL授权（请参阅《 Derby安全指南 》中的“配置用户身份验证”和“配置用户授权”），则只有数据库所有者才能关闭或删除数据库，对其进行加密，并使用新的启动密码对其进行重新加密。或新的加密密钥 ，或执行完整升级。如果未启用身份验证，并且不提供任何用户，则数据库所有者默认为“ APP”，这也是默认模式的名称（请参阅SET SCHEMA语句 ）。

与其他属性结合

您必须使用此属性指定一个databaseName （在数据库连接URL中的子协议或子子协议之后）或databaseName = nameofDatabase属性。

您可以将此属性与其他属性组合。要在创建数据库时指定语言环境，请使用region = ll_CC属性。

例子

创建文件系统数据库：

-- create a file system database
jdbc:derby:sampleDB;create=true
-- create a file system database using the databaseName attribute
jdbc:derby:;databaseName=newDB;create=true
-- create an in-memory database using the embedded driver
jdbc:derby:memory:myInMemDB;create=true
-- create an in-memory database using the databaseName attribute
jdbc:derby:;databaseName=memory:myInMemDB;create=true
-- create an in-memory database using the Network Server
jdbc:derby://localhost:1527/memory:myInMemDB;create=true

有关创建内存数据库的信息，请参见《 Derby开发人员指南 》中的“使用内存数据库”。
