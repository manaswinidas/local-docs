CAST功能

CAST函数将值从一种数据类型转换为另一种数据类型，并将数据类型提供给动态参数（？）或NULL值。

允许在任何允许表达式的地方使用CAST表达式。

句法

CAST ( [ expression | NULL | ? ]
    AS dataType )

您要将表达式转换为的数据类型是目标类型 。从中强制转换表达式的数据类型是源类型 。

笔记

在此讨论中， Derby SQL数据类型分类如下：
  • 合乎逻辑的
    • 布兰
  • 数字
    • 确切的数字（ SMALLINT ， INTEGER ， BIGINT ， DECIMAL ， NUMERIC ）
    • 近似数值（ FLOAT ， REAL ， DOUBLE PRECISION ）
  • 串
    • 字符串（ CLOB ， CHAR ， VARCHAR ， LONG VARCHAR ）
    • 位字符串（ BLOB ， CHAR用于BIT数据 ， VARCHAR用于BIT数据 ， LONG VARCHAR用于BIT数据 ）
  • 约会时间
    • 日期
    • 时间
    • 时间戳

逻辑类型之间的转换

可以将BOOLEAN值显式转换为任何字符串类型。结果为“ true”，“ false”或null。相反，可以将字符串类型转换为BOOLEAN。但是，如果字符串值不是'true'，'false'，'unknown'或null，则会引发错误。将“ unknown”强制转换为布尔值将导致返回空值。

来自数字类型的转换

数值类型可以转换为任何其他数值类型。如果目标类型不能无截断地表示非小数部分，则会引发异常。如果目标数字不能表示源数字的小数部分（小数位），则源将被自动截断以适合目标。例如，将763.1234强制转换为INTEGER可产生763。

位串之间的转换

位字符串可以转换为其他位字符串，但不能转换为字符串。转换为位字符串的字符串以尾随零填充以适合目标位字符串的大小。BLOB类型受到更多限制，并且需要显式强制转换。在大多数情况下，BLOB类型不能与其他类型进行强制转换：只能将BLOB强制转换为另一个BLOB，但是可以将其他位字符串类型强制转换为BLOB。

日期/时间值的转换

日期/时间值可以始终与TIMESTAMP相互转换。如果将DATE转换为TIMESTAMP，则所得TIMESTAMP的TIME分量始终为00:00:00。如果将TIME数据值转换为TIMESTAMP，则在执行CAST时DATE组件将设置为CURRENT_DATE的值。如果将TIMESTAMP转换为DATE，则TIME组件将被静默截断。如果将TIMESTAMP转换为TIME，则DATE组件将被静默截断。

XML值的转换

不能使用显式或隐式CAST将XML值转换为任何非XML类型。使用XMLSERIALIZE运算符将XML类型转换为字符类型。

例子

SELECT CAST (miles AS INT)
FROM Flights
-- convert timestamps to text
INSERT INTO mytable (text_column)
VALUES (CAST (CURRENT_TIMESTAMP AS VARCHAR(100)))
-- you must cast NULL as a data type to use it
SELECT airline
FROM Airlines
UNION ALL
VALUES (CAST (NULL AS CHAR(2)))
-- cast a double as a decimal
SELECT CAST (FLYING_TIME AS DECIMAL(5,2))
FROM FLIGHTS
-- cast a SMALLINT to a BIGINT
VALUES CAST (CAST (12 as SMALLINT) as BIGINT)
