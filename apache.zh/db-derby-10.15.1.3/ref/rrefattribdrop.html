drop = true属性

drop = true属性将删除数据库连接URL中指定的内存数据库。

如果成功，该属性将生成SQLException 08006。如果该数据库不存在，它将生成一个错误报告，指出找不到该数据库。

对于同时启用身份验证和SQL授权的数据库 ，只有数据库所有者才能删除该数据库。

删除数据库之前不必关闭数据库。

如果使用非内存数据库的数据库指定此属性，则Derby会生成SQLException XBM0I。

与其他属性结合

如果启用了身份验证，则必须与user = userName和password = userPassword属性一起指定此属性。如果同时启用了身份验证和SQL授权，则用户必须是数据库所有者。

例子

-- drop an in-memory database using the embedded driver
jdbc:derby:memory:myInMemDB;drop=true
-- drop an in-memory database using the Network Server
jdbc:derby://localhost:1527/memory:myInMemDB;drop=true
