SET ISOLATION语句

SET ISOLATION语句更改用户连接的隔离级别。

有效的隔离级别为SERIALIZABLE，REPEATABLE READ，READ COMMITTED和READ UNCOMMITTED。

发出此语句始终会提交当前事务。JDBC java.sql。Connection.setTransactionIsolation方法的行为与该命令几乎相同，但有一个例外：如果您使用的是嵌入式驱动程序，以及对java.sql的调用。Connection.setTransactionIsolation实际上不会更改隔离级别（也就是说，如果将隔离级别设置为其当前值），则不会提交当前事务。

有关隔离级别的信息，请参阅《 Derby开发人员指南 》中的“锁定，并发和隔离”。有关JDBC java.sql的信息。Connection.setTransactionIsolation方法，请参见java.sql。Connection.setTransactionIsolation方法 。

句法

SET [ CURRENT ] ISOLATION [ = ]
{
    UR | DIRTY READ | READ UNCOMMITTED |
    CS | READ COMMITTED | CURSOR STABILITY |
    RS |
    RR | REPEATABLE READ | SERIALIZABLE |
    RESET 
}

例

set isolation serializable;
