columnDefinition

simpleColumnName [ dataType ]
    [ columnLevelConstraint ]*
    [ [ WITH ] DEFAULT defaultConstantExpression
       | generatedColumnSpec 
       | generationClause 
    ]
    [ columnLevelConstraint ]*

dataType的语法在Data types中描述。仅当您指定generationClause时，才可以省略dataType 。如果省略dataType ，则生成的列的类型就是generationClause的类型。如果同时指定一个数据类型和generationClause的generationClause的类型必须分配给数据类型 。

CONSTRAINT子句中描述了columnLevelConstraint和tableLevelConstraint的语法。

列默认

对于默认值的定义， defaultConstantExpression是不引用任何表的表达式。它可以包括常量，日期时间特殊寄存器，当前模式，用户，角色和null：

defaultConstantExpression:
          NULL
        | CURRENT { SCHEMA | SQLID }
        | USER | CURRENT_USER | SESSION_USER | CURRENT_ROLE
        | DATE
        | TIME
        | TIMESTAMP
        | CURRENT DATE | CURRENT_DATE
        | CURRENT TIME | CURRENT_TIME
        | CURRENT TIMESTAMP | CURRENT_TIMESTAMP
        | literal

有关Derby的详细信息literal值，请参阅数据类型 。

一个中的值defaultConstantExpression必须在类型上与列兼容，但是defaultConstantExpression具有以下其他类型限制：

  • 如果指定USER，CURRENT_USER，SESSION_USER或CURRENT_ROLE，则该列必须是长度至少为8的字符列。
  • 如果指定CURRENT SCHEMA或CURRENT SQLID，则该列必须是长度至少为128的字符列。
  • 如果列是整数类型，则默认值必须是整数文字。
  • 如果列是十进制类型，则默认值的小数位数和精度必须在该列的范围内。
