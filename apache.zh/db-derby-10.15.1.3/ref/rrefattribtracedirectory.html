traceDirectory =路径属性

traceDirectory = path属性指定Derby Network Client将JDBC跟踪信息发送到的目录。

如果程序或会话具有多个连接，则网络客户端将为每个连接创建一个单独的文件。默认情况下，文件被命名为_driver_0 ， _driver_1 ， 等等。使用traceFile = path属性为跟踪文件指定文件名。

如果该目录不存在， Derby将发出错误消息。如果未指定绝对路径名，则假定目录相对于当前目录。

有关跟踪的更多信息，请参阅《 Derby服务器和管理指南 》中的“网络客户端跟踪”。有关与跟踪相关的其他属性，请参见traceFile = path ， traceFileAppend = true和traceLevel = value 。

与其他属性结合

您可以将此属性与其他属性组合。

例子

-- enable tracing on an existing database that will have multiple
--   connections
jdbc:derby://localhost:1527/mydb;traceDirectory=/home/mydir/mydbtracedir
-- specify a trace file name within the directory
jdbc:derby://localhost:1527/mydb;
   traceDirectory=/home/mydir/mydbtracedir;traceFile=trace.out
-- append to the default trace file
jdbc:derby://localhost:1527/mydb;
   traceDirectory=/home/mydir/mydbtracedir;traceFileAppend=true
