Derby数据库连接URL的属性

您可以为数据库连接URL提供可选的属性列表。

Derby将这些属性转换为属性，因此您还可以在传递给DriverManager.getConnection的Properties对象中设置属性。（您不能仅在传递给DriverManager.getConnection方法的对象中将这些属性设置为系统属性。）

如果在连接URL和“ 属性”对象中都指定了任何属性，则连接URL的属性将覆盖“属性”对象中的属性 。

这些属性特定于Derby ，并在“ 设置数据库连接URL的属性”中列出。

属性名称/值对将转换为属性，并添加到连接调用中提供的属性中。如果连接调用中未提供任何属性，则会创建一个属性集，其中仅包含从数据库连接URL获得的属性。
import java.util.Properties;

Connection conn = DriverManager.getConnection(
    "jdbc:derby:sampleDB;create=true");

/* setting an attribute in a Properties object */
Properties myProps = new Properties();
myProps.put("create", "true");
Connection conn = DriverManager.getConnection(
    "jdbc:derby:sampleDB", myProps);

/* passing user name and password */
Connection conn = DriverManager.getConnection(
    "jdbc:derby:sampleDB", "dba", "password");
注意：不会解析属性的正确性。如果您输入了错误的属性或相应的值，则将其忽略。（ Derby确实提供了一种用于解析属性正确性的工具。有关更多信息，请参阅《 Derby工具和实用程序指南》 。
