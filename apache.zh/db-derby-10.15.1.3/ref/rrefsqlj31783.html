FOR UPDATE子句

FOR UPDATE子句是SELECT语句的可选部分。

默认情况下，游标是只读的。FOR UPDATE子句指定游标应该是可更新的，并在编译期间强制执行检查以确保SELECT语句满足可更新游标的要求。有关可更新性的更多信息，请参见可更新游标和可更新ResultSet的要求 。

句法

FOR
{
    READ ONLY |
    FETCH ONLY |
    UPDATE [ OF simpleColumnName [ , simpleColumnName ]* ]
}

simpleColumnName引用基础查询的FROM子句中指定的表的可见名称。

可以指定FOR READ ONLY或其同义词FOR FOR FETCH ONLY来代替FOR UPDATE，以指示结果集不可更新。

注意：对于获得可更新的JDBC ResultSet，不是必须使用FOR UPDATE子句。只要用于生成JDBC ResultSet的语句满足可更新游标的要求，生成JDBC ResultSet的JDBC语句就具有并发模式就足够了ResultSet.CONCUR_UPDATABLE使ResultSet可更新。

即使正在更新索引中的列，优化器也可以使用索引。

有关索引如何影响性能的信息，请参见Tuning Derby 。

例

SELECT RECEIVED, SOURCE, SUBJECT, NOTE_TEXT FROM SAMP.IN_TRAY FOR UPDATE
