DECIMAL数据类型

DECIMAL提供一个精确的数字，其中精度和小数位数可以任意设置大小。

您可以指定精度（小数点左边和右边的位数总数）和小数位数（小数部分的位数）。所需的存储量取决于精度。

句法

{ DECIMAL | DEC } [(precision [, scale ])]

精度必须在1到31之间。比例尺必须小于或等于精度。

如果未指定比例，则默认比例为0。如果未指定精度，则默认精度为5。

只要不损失任何非小数精度，就可以尝试将数字值放入DECIMAL。从DECIMAL值截断尾随数字时， Derby会四舍五入。

例如：
-- this cast loses only fractional precision
values cast (1.798765 AS decimal(5,2));
1
--------
1.79
-- this cast does not fit
values cast (1798765 AS decimal(5,2));
1
--------
ERROR 22003: The resulting value is outside the range 
for the data type DECIMAL/NUMERIC(5,2).

与表达式中的其他数据类型混合时，结果数据类型将遵循表达式中数字类型提升中显示的规则。

另请参阅将一种数字数据类型的值存储在另一种数字数据类型的列中 。

当两个十进制值混合在表达式中时，结果值的小数位数和精度遵循小数位运算的小数位数中显示的规则。

对于BIGINT而言太大的整数常量被设为DECIMAL常量。

对应的编译时Java类型

java.math。大十进制

JDBC元数据类型（java.sql。类型）

十进制

例子

VALUES 123.456

VALUES 0.001
