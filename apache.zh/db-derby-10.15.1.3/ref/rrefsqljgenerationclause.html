世代条款

GENERATED ALWAYS AS ( valueExpression )

valueExpression是一个解析为单个值的表达式 ，此处有一些限制。有关表达式 s的更多信息，请参见SQL表达式 。

参考文献

generationClause可以引用表中其他未生成的列，但是不能引用任何已生成的列。generationClause不得引用另一个表中的列。

功能

如果功能满足以下要求，那么generationClause可以调用用户编码的功能：

  • 该函数不得读取或写入SQL数据。
  • 必须已将这些函数声明为DETERMINISTIC。
  • 该函数不得调用以下任何可能不确定的系统函数：
    • 当前日期
    • 当前时间
    • CURRENT_TIMESTAMP
    • 当前用户
    • 目前角色
    • 当前模式
    • 当前的SQLID
    • SESSION_USER

子查询

generationClause不得包含子查询。

外键

如果生成的列是引用另一个表的外键的一部分，则引用操作不得指定SET NULL或SET DEFAULT，并且更新规则不得指定ON UPDATE CASCADE。

例

CREATE TABLE employee
(
  employeeID           int,
  name                 varchar( 50 ),
  caseInsensitiveName  GENERATED ALWAYS AS( UPPER( name ) )
);
CREATE INDEX caseInsensitiveEmployeeName ON employee( caseInsensitiveName );
