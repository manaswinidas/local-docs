创建SYNONYM语句

CREATE SYNONYM语句为同一模式或其他模式中存在的表或视图提供了备用名称。

您还可以为其他同义词创建同义词，从而产生嵌套的同义词。在SELECT，INSERT，UPDATE，DELETE或LOCK TABLE语句中，可以使用同义词代替原始的限定表或视图名称。您可以为不存在的表或视图创建同义词，但是在可以使用同义词之前，必须存在目标表或视图。

同义词与表或视图共享相同的名称空间。您不能使用与相同模式中已经存在的表相同的名称来创建同义词。同样，您不能创建名称与已存在的同义词匹配的表或视图。

可以为创建同义词时不存在的表/视图定义同义词。如果表或视图不存在，您将收到警告消息（SQLSTATE 01522）。在DML语句中使用同义词时，必须存在被引用的对象。

您可以创建嵌套的同义词（另一个同义词的同义词），但是任何尝试创建导致循环引用的同义词的尝试都会返回错误消息（SQLSTATE 42916）。

不能在系统架构中定义同义词。所有以'SYS'开头的模式都被视为系统模式，并由Derby保留。

无法在临时表上定义同义词。尝试在临时表上定义同义词将返回错误消息（SQLSTATE XCL51）。

句法

CREATE SYNONYM synonymName FOR { viewName | tableName }

语句中的synonymName表示您为目标表或视图提供的同义词名称，而viewName或tableName表示目标表或视图的原始名称。

例

CREATE SYNONYM SAMP.T1 FOR SAMP.TABLEWITHLONGNAME
