使用foreignViews可选工具

的foreignViews可选工具为外部数据库中的所有用户表创建模式，表功能和便利视图。这对于批量导入外部数据很有用。

在运行之前foreignViews可选工具，请确保您的类路径包含Derby jar文件，包括derbytools.jar 。

您可以加载和卸载foreignViews通过使用工具SYSCS_UTIL.SYSCS_REGISTER_TOOL系统过程。有关此过程的信息，请参见《 Derby参考手册 》。

加载foreignViews工具，使用如下语句：

call syscs_util.syscs_register_tool( 'foreignViews', true,
    'foreignDatabaseURL', 'XYZ_' )

两个尾随参数具有以下含义：

  • foreignDatabaseURL是适合通过调用java.sql创建与外部数据库的连接的URL 。DriverManager.getConnection（） 。例如：
    'jdbc:derby:db3;user=fred;password=fredpassword'
  • 'XYZ_'是一个以该工具创建的所有架构的名称为前缀的字符串。该参数可以省略。如果省略该工具，则该工具将创建与外部数据库中的架构名称相同的架构。

例如，假设外部数据库具有两个模式， S1和S2 。 S1包含两个用户表， T1和T2 。S2包含两个用户表， U1和U2 。加载如上所示的工具将在Derby数据库中创建以下对象：

  • 图式XYZ_S1
  • 表功能XYZ_S1.T1 ，其内容为S1.T1从外部数据库
  • 表功能XYZ_S1.T2 ，其内容为S1.T2从外部数据库
  • 视图XYZ_S1.T1 ，它包装了相应的表函数
  • 视图XYZ_S1.T2 ，它包装了相应的表函数
  • 图式XYZ_S2
  • 表功能XYZ_S2.U1 ，其内容为S2.U1从外部数据库
  • 表功能XYZ_S2.U2 ，其内容为S2.U2从外部数据库
  • 视图XYZ_S2.U1 ，它包装了相应的表函数
  • 视图XYZ_S2.U2 ，它包装了相应的表函数

视图将隐藏表函数的参数。然后，可以使用以下SELECT语句填充本地模式：

insert into S1.T1 select * from XYZ_S1.T1
insert into S1.T2 select * from XYZ_S1.T2
insert into S2.U1 select * from XYZ_S2.U1
insert into S2.U2 select * from XYZ_S2.U2

完成批量导入外部数据后，可以按以下方式删除此模式，表函数和视图包：

call syscs_util.syscs_register_tool( 'foreignViews', false,
    'foreignDatabaseURL', 'XYZ_' )
