更新索引

创建索引后，用户可以对文本列执行批量重新索引。

如果启用了SQL授权，则数据库所有者是唯一可以重新索引另一个用户表中的文本列的帐户。以下过程为整个表中的列重新索引：

LUCENESUPPORT.UPDATEINDEX
(
    SCHEMANAME VARCHAR( 128 ),
    TABLENAME VARCHAR( 128 ),
    TEXTCOLUMN VARCHAR( 128 ),
    INDEXDESCRIPTORMAKER VARCHAR( 32672 )
)

前三个参数标识要重新索引的列。最后一个参数使您可以覆盖如何为文本建立索引以及如何解析查询。

此版本的luceneSupport该工具不支持增量索引数据。更新索引是一个批量操作，它将重新索引整个数据集。因此，本版本的luceneSupport该工具不适用于全文查询结果必须为最新的更新密集型应用程序。的luceneSupport该工具更适合昨天的全文查询结果足够好的应用程序：

  • 只读 ：分析静态文本数据的应用程序
  • 模糊 ：可以定期（例如每天一次）对文本数据进行批量重新索引的应用程序，并且可以容忍查询结果中的这种模糊性

例

-- reindex a column using a custom analyzer
CALL LUCENESUPPORT.UPDATEINDEX
(
    'ruth', 'poemView', 'poemText',
    'myapp.MyIndexDescriptor.makeMe',
);
