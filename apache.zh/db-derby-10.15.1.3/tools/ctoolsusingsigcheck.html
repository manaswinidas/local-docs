使用SignatureChecker

在运行之前SignatureChecker工具，请确保您的类路径包含Derby jar文件，包括derbytools.jar 。

在Java SE平台上，运行SignatureChecker如下所示的工具，其中connection-url-to-database是用于通过调用DriverManager.getConnection（）获得连接的连接URL：

java org.apache.derby.tools.SignatureChecker connection-url-to-database

或者，您可以使用以下命令调用该工具derbyrun.jar 。例如：

java -jar derbyrun.jar SignatureChecker "jdbc:derby:myDB"

在上面的示例中， SignatureChecker找到的工具与所有例程都匹配，但函数app.iDontExist和app.tabFuncDoesntExist除外 。如果该工具找不到与您的功能或过程之一匹配的内容，它将告诉您期望找到的签名。您需要通过以下方式之一来调整应用程序：

  • 方法：更改Java方法的签名，使其与SignatureChecker工具。
  • 例程：删除并重新创建函数或过程，以使其参数和返回类型根据Derby参考手册 “参数匹配”中描述的SQL Standard规则与Java方法匹配。
