dblook示例

以下示例说明了各种dblook实用程序选项可以从命令行指定。这些示例使用sample数据库。

注意：这些示例中显示的引号是命令参数的一部分，必须传递给dblook 。引号的传递方式取决于您使用的操作系统和命令行。在某些系统中，可能需要通过在引号之前使用正斜杠来使引号转义。例如：
"\"My Table"\"

状态消息将写入输出（或者-o filename （如果指定的话，或控制台）作为SQL脚本注释。这些状态消息用作标头，以显示正在或已经处理的数据库对象类型。 dblook效用。

将DDL写入控制台

您可以将DDL写入控制台中的所有内容。 sample数据库。在此示例中，数据库位于当前目录中。例如：

java org.apache.derby.tools.dblook -d jdbc:derby:sample

在dblook命令中包括错误和警告消息

将DDL写入控制台时，可以写入错误和警告消息。消息使用System.err 。例如：

java org.apache.derby.tools.dblook -d jdbc:derby:sample -verbose

将DDL写入文件

您可以将DDL写入名为myDB_DDL.sql对于其中的一切sample数据库。在此示例中，数据库和文件位于当前目录中。例如：

java org.apache.derby.tools.dblook -d jdbc:derby:sample -o myDB_DDL.sql

在dblook命令中指定目录路径

如果数据库或文件不在当前目录中，则必须指定目录路径。例如：

java org.apache.derby.tools.dblook -d 'jdbc:derby:c:\private\stuff\sample'
   -o 'C:\temp\newDB.sql'

在dblook命令中指定模式

您可以指定数据库的架构。要将DDL写入控制台，请针对sample数据库位于SAMP架构中的数据库，请使用以下命令：

java org.apache.derby.tools.dblook -d jdbc:derby:sample -z samp

指定远程数据库和主机

如果sample数据库位于SAMP模式上localhost:1527 ，您必须指定您的用户ID和密码。例如，使用以下命令将DDL写入控制台：

java org.apache.derby.tools.dblook
   -d 'jdbc:derby://localhost:1527/"C:\temp\sample";
   user=someusername;password=somepassword' -z samp

在dblook命令中指定数据库内的模式和表

您可以指定仅sample与SAMP和My Table表被写入控制台。例如：

java org.apache.derby.tools.dblook -d jdbc:derby:sample -z samp -t "My Table"

在dblook命令中指定多个表

您可以在dblook通过用空格分隔表名来命令。例如，对于sample与以下任一数据库相关联的数据库My Table表或STAFF表，使用以下命令：

java org.apache.derby.tools.dblook -d jdbc:derby:sample -t "My Table" staff

在没有语句定界符的情况下将DDL写入文件

为DDL中的所有对象编写DDL sample数据库到myDB_DDL.sql如果文件中没有语句定界符，则必须省略默认的分号。如果文件已经存在，则可以将DDL附加到输出文件中。例如：

java org.apache.derby.tools.dblook -d jdbc:derby:sample
   -o myDB_DDL.sql -td '' -append

从DDL中排除视图

要将DDL写入控制台中的所有对象sample数据库除视图外，请使用以下命令：

java org.apache.derby.tools.dblook -d jdbc:derby:sample -noview
