开始ij

Derby为Windows和UNIX环境中的用户提供了批处理和Shell脚本，可用于启动ij 。

通过调用适当的脚本，您可以开始ij并能够通过简单的命令进行连接。脚本位于bin Derby安装目录。您还可以自定义ij适合您环境的脚本。

如果将Derby用作客户端/服务器环境，请在连接到Derby数据库之前启动网络服务器。（有关详细信息，请参阅《 Derby服务器和管理指南 》中的“启动网络服务器”。）你可以开始了ij通过运行ij您的环境的脚本。请按照Derby入门中“设置环境”中的说明进行操作。 DERBY_HOME和JAVA_HOME环境变量并添加DERBY_HOME/bin到你的路。然后使用以下命令：

ij [-p propertyFile] [inputFile]
或者，将DERBY_HOME环境变量，然后使用以下命令之一：
(UNIX) java [options] -jar $DERBY_HOME/lib/derbyrun.jar ij
    [-p propertyFile] [inputFile]
(Windows) java [options] -jar %DERBY_HOME%\lib\derbyrun.jar ij
    [-p propertyFile] [inputFile]

请注意，您不能使用-cp论证或CLASSPATH要设置的环境变量CLASSPATH使用时的变量-jar参数开始ij工具。也就是说，如果您想运行ij具有自定义类路径的工具，那么您将无法使用-jar论点。相反，您必须使用完整的类名来启动ij工具（ java org.apache.derby.tools.ij ）。

您也可以运行ij通过设置CLASSPATH或MODULEPATH变量setNetworkServerCP要么setNetworkServerCP.bat脚本。请注意，您首先需要设置DERBY_HOME变量。跑步ij从CLASSPATH中，键入

java [options] org.apache.derby.tools.ij 
    [-p propertyFile] [inputFile]

跑步ij从MODULEPATH中，键入

(UNIX) java -p $MODULEPATH \
    -m org.apache.derby.tools/org.apache.derby.tools.ij \
    [options]
(Windows) java -p %MODULEPATH% ^
    -m org.apache.derby.tools/org.apache.derby.tools.ij ^
    [options]

命令行项目是：

  • java

    启动JVM。

  • options

    JVM使用的选项。您可以使用-D设置选项ij属性（请参阅使用properties启动ij ）或系统属性（例如Derby属性）。

  • propertyFile

    您可以用来设置的文件ij属性（而不是-D选项）。属性文件应采用以下格式创建的格式： java.tools.Properties.save方法，其格式与derby.properties文件。

  • inputFile

    从中读取命令的文件。的ij工具在文件末尾或exit命令。使用输入文件的原因ij在运行命令时打印出命令。如果您重新路由标准输入， ij不会打印出命令。如果您不提供输入文件， ij从标准输入读取。

有关的详细信息ij命令，请参阅ij命令和错误参考 。
