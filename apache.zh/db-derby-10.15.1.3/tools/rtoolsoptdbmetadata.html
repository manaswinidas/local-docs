使用databaseMetaData可选工具

的databaseMetaData可选工具创建与java.sql中的大多数方法相对应的函数和表函数。DatabaseMetaData接口。

在运行之前databaseMetaData可选工具，请确保您的类路径包含Derby jar文件，包括derbytools.jar 。

您可以加载和卸载databaseMetaData通过使用工具SYSCS_UTIL.SYSCS_REGISTER_TOOL系统过程。有关此过程的信息，请参见《 Derby参考手册 》。

加载databaseMetaData工具，使用以下语句：

call syscs_util.syscs_register_tool( 'databaseMetaData', true )

该命令在当前模式中创建元数据函数和表函数。函数和表函数的名称与相应的java.sql相同。他们包装的DatabaseMetaData方法。加载此工具后，您可以筛选和加入这些功能以创建功能强大的元数据查询。例如，以下查询列出了用户创建的表中所有列的列名和数据类型：

select t.table_schem, t.table_name, c.column_name, c.type_name
from table( getTables( null, '%', '%' ) ) t,
     table( getColumns( null, '%', '%', '%') ) c
where c.table_schem = t.table_schem
and c.table_name = t.table_name
and t.table_type = 'TABLE'
order by table_schem, table_name, column_name

一些DatabaseMetaData方法采用数组参数。由于这些参数不能表示为Derby类型，因此将消除这些参数。这意味着已删除了getTables（）和getUDTs（）的尾随类型参数。此外，以下DatabaseMetaData方法没有相应的元数据例程：

  • 由于Derby不提供java.sql的实现，因此消除了getRowIdLifetime（） 。RowIdLifetime 。
  • 由于Derby不支持重载，因此取消了getSchemas（） 。包括了更通用的getSchemas（String，String）方法。
  • 由于Derby不支持重载，因此取消了supportsConvert（） 。包括了更通用的supportsConvert（int，int） 。

完成元数据结果的连接后，可以按以下方式删除此函数包和表函数包：

call syscs_util.syscs_register_tool( 'databaseMetaData', false )
