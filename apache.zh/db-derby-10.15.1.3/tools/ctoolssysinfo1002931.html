使用sysinfo检查类路径

sysinfo提供一个论据（ -cp ），可以用来测试类路径。

java org.apache.derby.tools.sysinfo -cp 
[ [ embedded ][ server ][ client] [ tools ] [ anyClass.class ] ]

如果您的环境设置正确，该实用程序将显示指示成功的输出。

您可以提供可选参数-cp测试不同的环境。的可选参数-cp是：
  • embedded
  • server
  • client
  • tools
  • classname.class
如果您的类路径中缺少某些内容，则该实用程序会指出缺少的内容。例如，如果您忽略了包含名为的类的目录SimpleApp在您的类路径中，该实用程序将在发出以下命令行时（在一行中全部输入）进行指示：
$ java org.apache.derby.tools.sysinfo -cp embedded SimpleApp.class
FOUND IN CLASS PATH:

Derby embedded engine library (derby.jar)


NOT FOUND IN CLASS PATH:

user-specified class (SimpleApp)
(SimpleApp not found.)
