验证Derby系统配置

在开始使用Derby之前，应确认计算机已安装正确的软件，并且已设置环境变量。

要检查Derby系统配置：

  1. 验证java可执行文件（版本9或更高版本）在您的命令执行PATH中，方法是打开命令窗口并运行java -version命令。命令的输出如下所示：
    java version "9"
Java(TM) SE Runtime Environment (build 9+181)
Java HotSpot(TM) 64-Bit Server VM (build 9+181, mixed mode)
    您看到的输出可能与此处显示的输出不同，因为java -version命令输出特定于供应商的信息。如果命令产生错误或列出的版本不是9或更高版本，请在继续操作之前安装JDK。
  2. 验证是否已设置DERBY_HOME环境变量，并指向Derby 10.15安装的根目录。打开命令窗口，然后为您的系统运行适当的命令。 如果将Derby安装在UNIX上的/ opt / Derby_10目录中或Windows上的c：\ Derby_10目录中，则所使用的命令和预期的输出如下表所示。
    表格1。验证DERBY_HOME设置
    操作系统 命令 输出量
    UNIX（Korn Shell） echo $DERBY_HOME / opt / Derby_10
    视窗 echo %DERBY_HOME% C：\ Derby_10
    如果收到错误或看不到预期的输出，请确保已设置DERBY_HOME环境变量，如设置环境变量中所述 。
注意：您也可以使用sysinfo实用程序以验证是否正确设置了环境变量。如果环境变量设置正确，则sysinfo命令显示有关Java虚拟机（JVM）和已安装的Derby版本的信息。
