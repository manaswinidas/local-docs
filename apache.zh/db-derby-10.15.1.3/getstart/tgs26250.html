手动设置CLASSPATH / MODULEPATH环境变量

当您启动Java应用程序和JVM时，可以在操作系统中临时，永久或在运行时设置CLASSPATH和MODULEPATH环境变量。

类路径是JVM和其他Java应用程序运行程序所需的类库的列表。同样，modulepath是程序所需的Java模块的相应列表。Derby包含一些脚本，这些脚本可以设置类路径/模块路径来运行Derby工具。

如果要直接使用Java而不是使用脚本来调用Derby工具，则必须手动设置CLASSPATH环境变量或手动声明模块路径。

在大多数开发环境中，最好在要输入命令的命令行外壳中临时设置这些变量。

Derby在DERBY_HOME / bin目录中提供了几个脚本，以帮助您快速设置类路径/模块路径。这些脚本是：
setEmbeddedCP
在嵌入式模式下使用数据库引擎时，请使用setEmbeddedCP脚本设置CLASSPATH和MODULEPATH。
setNetworkServerCP
要启动网络服务器时，请使用setNetworkServerCP脚本设置CLASSPATH和MODULEPATH。
setNetworkClientCP
要使用网络客户端访问数据库时，请使用setNetworkClientCP脚本设置CLASSPATH和MODULEPATH。

要临时设置类路径和模块路径，请在每次打开新命令窗口时运行适合您的环境的脚本。

请注意，这些脚本在UNIX系统和Windows系统上的行为略有不同。在Windows系统上，在命令外壳中运行脚本将为外壳设置环境变量。

在UNIX系统上，您需要使用“ dot”或“ source”命令来确保脚本在调用shell的环境中运行。否则，在运行脚本时，它将为该脚本启动新的外壳程序。CLASSPATH和MODULEPATH变量在该Shell中设置。脚本完成后，该外壳程序退出，您将返回到您的外壳程序。CLASSPATH / MODULEPATH仅针对运行脚本的shell进行更改。Shell中的CLASSPATH和MODULEPATH变量未更改。

众所周知，UNIX Shell脚本可以在Bash Shell中成功运行，如果您使用其他UNIX Shell，则可能需要对其进行一些修改。

以下是运行脚本，然后显示脚本设置的类路径的示例：
  • 在UNIX上，使用“点”命令调用脚本，示例会话可能如下：
    sh-2.05b$ echo $CLASSPATH
sh-2.05b$ DERBY_HOME=/derby/db-derby-10.X.Y.0-bin
sh-2.05b$ . $DERBY_HOME/bin/setEmbeddedCP
sh-2.05b$ echo $CLASSPATH
/derby/db-derby-10.X.Y.0-bin/lib/derbyshared.jar:/derby/db-derby-10.X.Y.0-bin/lib/derby.jar:/derby/db-derby-10.X.Y.0-bin/lib/derbytools.jar:/derby/db-derby-10.X.Y.0-bin/lib/derbyoptionaltools.jar:
sh-2.05b$ java org.apache.derby.tools.ij
ij version 10.X
ij> quit;
  • 在UNIX上，使用“源”命令来调用脚本，示例会话可能如下：
    -bash-2.05b$ echo $CLASSPATH
-bash-2.05b$ export DERBY_HOME=/derby/db-derby-10.X.Y.0-bin
-bash-2.05b$ source $DERBY_HOME/bin/setEmbeddedCP
  • 在Windows上，示例会话可能如下：
    C:\derby\db-derby-10.X.Y.0-bin\bin>echo %CLASSPATH%
%CLASSPATH%
C:\derby\db-derby-10.X.Y.0-bin\bin>set DERBY_HOME=C:\derby\db-derby-10.X.Y.0-bin
C:\derby\db-derby-10.X.Y.0-bin\bin>setEmbeddedCP.bat
C:\derby\db-derby-10.X.Y.0-bin\bin>echo %CLASSPATH%
C:\derby\DB-DERBY-10.X.Y.0-bin\lib\derbyshared.jar;C:\derby\DB-DERBY-10.X.Y.0-bin\lib\derby.jar;C:\derby\DB-DERBY-10.X.Y.0-bin\lib\derbytools.jar;C:\derby\DB-DERBY-10.X.Y.0-bin\lib\derbyoptionaltools.jar;
C:\derby\db-derby-10.X.Y.0-bin\bin>java org.apache.derby.tools.ij
ij version 10.X
ij> quit;
