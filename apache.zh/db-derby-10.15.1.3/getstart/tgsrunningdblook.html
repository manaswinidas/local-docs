运行dblook

德比 dblook实用程序是一种数据定义语言（DDL）生成实用程序。

的dblook实用程序是一个简单的实用程序，它将用户指定数据库的DDL的全部或部分转储到控制台或文件中。然后可以将生成的DDL用于诸如重新创建数据库的全部或部分，查看数据库中对象的子集（例如，与特定表和模式有关的那些对象）或记录数据库模式的事情。 。

选择将用于运行dblook脚本。下表列出了这些方法。

跑dblook作为独立命令。 如果您是Java编程语言的新手，还是Derby的新手，请使用此方法。 在运行操作系统之前，请按照设置环境变量中的步骤进行操作。 dblook使用此方法的实用程序。

运行dblook命令行中的脚本使用：

dblook -d connectionURL [options]

的dblook脚本设置适当的环境变量（包括CLASSPATH）并运行dblook效用。
跑dblook使用位于以下目录中的jar文件dblook居住。 如果您不熟悉Derby ，但熟悉Java编程语言，请使用此方法。 您必须先设置DERBY_HOME环境变量，然后才能运行dblook使用此方法的实用程序。

在UNIX上，命令是：

java [options] -jar $DERBY_HOME/lib/derbyrun.jar dblook -d connectionURL [options]

在Windows上，命令为：

java [options] -jar %DERBY_HOME%\lib\derbyrun.jar dblook -d connectionURL [options]
跑dblook使用java命令。 如果您既熟悉Java编程语言，又熟悉Derby ，请使用此方法，并且已经添加了java可执行文件添加到命令执行路径。 您必须设置CLASSPATH。使用手动设置CLASSPATH / MODULEPATH环境变量中指定的步骤。然后在java命令。例如：
java org.apache.derby.tools.dblook -d connectionURL [options]
跑dblook使用java使用模块路径的命令。 如果您既熟悉Java编程语言又熟悉Derby ，请使用此方法，并且已经添加了java可执行文件添加到命令执行路径，并希望使用模块路径运行Derby。 您必须设置您的MODULEPATH。使用手动设置CLASSPATH / MODULEPATH环境变量中指定的步骤。然后在中指定工具模块和dblook类名称java命令。

在UNIX上，命令是：

java -p $MODULEPATH \
  -m org.apache.derby.tools/org.apache.derby.tools.dblook \
  -d connectionURL [options]

在Windows上，命令为：

java -p %MODULEPATH% ^
  -m org.apache.derby.tools/org.apache.derby.tools.dblook ^
  -d connectionURL [options]
