在没有安全管理器的情况下运行网络服务器

如果出于某种原因需要在Java安全保护之外运行应用程序，则可以覆盖网络服务器的安全管理器的默认安装。

警告：
通过向所有客户端开放服务器而不限制通过用户身份验证和安全策略的访问，会带来严重的安全风险。

使用-noSecurityManager选项强制网络服务器在没有安全管理器的情况下启动。例如：

java org.apache.derby.drda.NetworkServerControl start \
-h localhost -noSecurityManager
