使用SSL / TLS启动服务器

对于服务器SSL / TLS，需要生成服务器密钥对。如果服务器要进行客户端身份验证，则需要在信任存储区中安装客户端证书。

在密钥和证书处理中描述了这些操作。

使用属性激活服务器端的SSL derby.drda.sslMode （默认关闭）或-ssl服务器启动命令的选项。

使用基本SSL加密启动服务器

当SSL模式设置为basic ，服务器将仅接受SSL加密的连接。

属性javax.net.ssl.keyStore和javax.net.ssl.keyStorePassword需要设置适当的值。

例

java -Djavax.net.ssl.keyStore=serverKeyStore.key \
-Djavax.net.ssl.keyStorePassword=qwerty \
-jar derbyrun.jar server start -ssl basic

启动对客户端进行身份验证的服务器

服务器的SSL模式设置为peerAuthentication ，服务器除了对网络流量进行加密之外，还对其客户端身份进行身份验证。在这种情况下，服务器的信任库必须为每个要连接的客户端包含一个证书。

的javax.net.ssl.trustStore和javax.net.ssl.trustStorePassword除上述属性外，还需要设置。

服务器进行客户端身份验证时，请参阅使用SSL / TLS运行客户端以获取客户端设置。

例

java -Djavax.net.ssl.keyStore=serverKeyStore.key \
-Djavax.net.ssl.keyStorePassword=qwerty \
-Djavax.net.ssl.trustStore=serverTrustStore.key \
-Djavax.net.ssl.trustStorePassword=qwerty \
-jar derbyrun.jar server start -ssl peerAuthentication
