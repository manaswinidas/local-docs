网络服务器安全性

默认情况下， Derby网络服务器仅在本地主机上侦听。客户端必须使用localhost主机名进行连接。

默认情况下，客户端无法从另一台主机访问网络服务器。要启用来自其他主机的连接，请设置derby.drda.host属性，或使用-h中的选项java org.apache.derby.drda.NetworkServerControl start命令。

在以下示例中，服务器将仅在本地主机上侦听，并且客户端无法从其他主机访问服务器：

java org.apache.derby.drda.NetworkServerControl start

在以下示例中，服务器在主机上运行sampleserver.example.com并侦听其他主机的客户端。客户端必须在URL或数据源中将服务器指定为sampleserver.example.com ：

java org.apache.derby.drda.NetworkServerControl start \
-h sampleserver.example.com

要启动网络服务器以使其在所有接口上进行侦听，请以IP地址开头0.0.0.0 ，如以下示例所示：

java org.apache.derby.drda.NetworkServerControl start -h 0.0.0.0

以以下内容启动的服务器-h 0.0.0.0选项将侦听来自两者的客户端请求localhost和网络上的其他计算机。

但是，管理命令（例如， org.apache.derby.drda.NetworkServerControl shutdown ）只能在启动服务器的主机上运行，即使服务器是使用-h选项。
