搜索条件未知的查询

查询有时具有未知的搜索条件，例如在语句的WHERE子句包含仅在执行时而不是在编译时才知道的动态参数，或者在语句涉及联接时。

例如：

-- dynamic parameters
SELECT *
FROM Flights
WHERE orig_airport = ?

rollback
-- joins
SELECT * FROM Countries, Cities
WHERE Countries.country_ISO_code = Cities.country_ISO_code

-- complex search conditions
SELECT * FROM Countries
WHERE region = (select region from Countries where country = 'Spain')

在上面的SELECT语句中，优化器无法从索引中获取足够的有用信息，以了解特定访问路径将返回多少行。但是，通常可以通过查看表对特定WHERE子句的选择性来做出很好的猜测。

选择性是指针对特定WHERE子句将从表中返回的行的分数。优化程序将表中的行数乘以特定操作的选择性 。例如，如果特定搜索操作的选择性为.10，并且该表包含100行，则优化程序估计该操作将返回10行。（这并不确切；这只是一个很好的猜测。）
