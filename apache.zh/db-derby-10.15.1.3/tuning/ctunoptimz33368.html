什么是索引？

索引是一种数据库结构，可以快速查找表的一个或多个列中的数据。

例如，一个Flights表中的travelDB数据库具有三个索引：

  • 索引orig_airport列（称为OrigIndex ）
  • 索引dest_airport列（称为DestIndex ）
  • 强制执行primary key约束flight_id和segment_number列（具有系统生成的名称）

这意味着存在三个单独的结构，它们提供了进入Flights表。让我们看一下其中一种结构， OrigIndex 。

OrigIndex将每个值存储在orig_airport列，以及有关如何为每个值检索整个对应行的信息。

对于中的每一行Flights ，在中有一个条目OrigIndex包括orig_airport列和行本身的地址。条目按升序存储。 orig_airport价值观。

当索引包含多个列时，第一列是对条目进行排序的主要列。例如，（ flight_id ， segment_number ）首先由flight_id 。如果不止一个flight_id具有相同的值，然后按以下顺序对这些条目进行排序segment_number 。索引条目的摘录可能如下所示：

'AA1111' 1
'AA1111' 2
'AA1112' 1
'AA1113' 1
'AA1113' 2

索引有时只是有用的。当语句的WHERE子句正在寻找其值等于的行时，此特定索引很有用。 orig_airport是一些特定的值或值的范围。SELECT，UPDATE和DELETE都可以具有WHERE子句。

例如， OrigIndex对于以下语句很有帮助：

SELECT *
FROM Flights
WHERE orig_airport = 'SFO'

SELECT *
FROM Flights
WHERE orig_airport < 'BBB'

SELECT *
FROM Flights
WHERE orig_airport >= 'MMM'

DestIndex对于以下语句很有帮助：

SELECT *
FROM Flights
WHERE dest_airport = 'SCL'

主键索引（在flight_id和segment_number ）对于以下语句很有帮助：

SELECT *
FROM Flights
WHERE flight_id = 'AA1111'

SELECT *
FROM Flights
WHERE flight_id BETWEEN 'AA1111' AND 'AA1115'

SELECT *
FROM FlightAvailability AS fa, Flights AS fts
WHERE flight_date > CURRENT_DATE
AND fts.flight_id = fa.flight_id
AND fts.segment_number = fa.segment_number
