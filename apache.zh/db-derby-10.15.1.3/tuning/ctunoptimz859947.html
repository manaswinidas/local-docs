不同

当满足以下条件时，将元组过滤应用于DISTINCT。

  • SELECT列表完全由简单的列引用和常量组成。
  • 所有简单的列引用都来自同一张表，并且优化程序已选择有问题的表作为查询块中最外面的表。
  • 优化器已选择索引作为有关表的访问路径。
  • SELECT列表中的简单列引用以及表中具有等式谓词的任何简单列引用都是优化程序选择作为表访问路径的索引的前缀。
注意：列引用集必须是索引的有序前缀。

这是将应用元组过滤的最常见情况：

SELECT DISTINCT c1 FROM t1

相等谓词允许对以下内容进行元组过滤：

SELECT DISTINCT c2
FROM t1
WHERE c1 = 5

SELECT DISTINCT c2, c4
FROM t1
WHERE c1 = 5 and c3 = 7
-- the columns don't have to be in the
-- same order as the index
SELECT DISTINCT c2, c1
FROM t1
  • 快速DISTINCT扫描
    在以下情况下，执行DISTINCT时，Derby可以使用哈希表而不是排序器来消除重复项。
