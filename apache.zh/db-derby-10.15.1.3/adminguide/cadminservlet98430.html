使用Servlet接口远程管理Derby Network Server

您可以使用servlet接口来远程管理网络服务器。要使用servlet接口，必须在Application Server中注册servlet，并且derby.system.home必须是Application Server已知的。

重要： Servlet接口仅适用于测试目的。不应在生产中使用。

Derby Network Server的Web应用程序存档（WAR）文件， derby.war ，可用于$DERBY_HOME/lib 。该文件在相对路径中注册网络服务器的servlet。 /derbynet 。有关如何安装它的说明，请参阅Application Server的文档。

例如，如果derby.war安装在WebSphere Application Server中，其上下文根为derby ，服务器的URL为：

http://<server>[:port]/derby/derbynet
笔记：
  • Servlet引擎不属于网络服务器。
  • 当通过servlet接口启动网络服务器时，关闭应用程序服务器也将关闭网络服务器，因为两者都在同一JVM中运行。

Servlet采用以下可选配置参数：

主办
指定网络服务器要使用的主机名。请参阅下面的“安全注意事项”部分。
端口号
指定网络服务器要使用的端口号。
startNetworkServerOnInit
指定初始化Servlet时将启动网络服务器。
tracingDirectory
指定跟踪文件的位置。如果未指定跟踪目录，则将跟踪放置在derby.system.home 。

安全注意事项

有关网络服务器的一般安全注意事项，请参阅《 Derby安全指南》 。

的host参数允许配置主机名，该主机名将用于网络连接的侦听套接字。默认情况下，网络服务器将仅在环回地址上侦听请求，这意味着它将仅接受来自本地主机的连接。更改此值可能会使服务器暴露于外部连接，从而引起安全问题，因此在使用host参数，您应该在Java安全管理器下运行并启用用户身份验证。

本节描述Servlet页面。

  • 启动页面
    使用启动页面启动服务器。
  • 跑步页面
    如果网络服务器正在运行（无论是通过初始化Servlet还是以其他方式启动的），都会显示正在运行的页面。
  • 跟踪会话页面
    如果在运行页面上选择指定要跟踪的会话，那么将显示此页面。您必须输入会话ID。
  • 跟踪目录页面
    如果您选择在“运行”页面上更改跟踪目录，则会显示此页面。您必须输入跟踪目录。
  • 设置网络服务器参数
    如果单击线程参数按钮，则显示第一页。使用此页面设置新参数。
