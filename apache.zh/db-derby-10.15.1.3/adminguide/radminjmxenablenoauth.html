在没有身份验证或SSL的情况下启用远程JMX

以下简单示例使用Oracle JDK JVM在命令行上启动了启用了不安全的远程JMX管理和监视的Derby Network Server。

默认情况下，启用基于SSL的密码身份验证，但此处禁用了这些安全功能，以简化示例。

重要：建议不要在生产环境中禁用身份验证或SSL。
java -Dcom.sun.management.jmxremote.port=9999
-Dcom.sun.management.jmxremote.authenticate=false
-Dcom.sun.management.jmxremote.ssl=false 
-jar $DERBY_HOME/lib/derbyrun.jar server start

从命令行启动网络服务器时，除非您指定了以下选项，否则它将使用基本的默认安全策略自动安装安全管理器。 -noSecurityManager选项。您可能需要自定义此策略以使其适合您的需求。有关详细信息，请参见使用安全策略进行细粒度的授权 。
