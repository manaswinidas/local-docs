使用日志文件控制日志记录

网络服务器使用derby.log文件来记录遇到的问题。当属性derby.drda.logConnections被设定为true 。

有关此属性的信息，请参见derby.drda.logConnections属性。

的derby.log启动Derby服务器时，将创建文件。然后，网络服务器记录时间和版本。如果日志文件存在，则将其覆盖，除非该属性derby.infolog.append被设定为true 。有关此属性的信息，请参见《 Derby参考手册 》中的“ derby.infolog.append”。

当网络服务器记录连接时，它还会记录连接号；此日志消息被写入到derby.log文件并连接到网络服务器控制台。

  • 要打开连接日志，请使用以下命令（全部在一行上）： 
    java org.apache.derby.drda.NetworkServerControl 
  logconnections on [-h hostname] [-p portnumber]
  • 要关闭连接日志，请使用以下命令（全部在一行上）： 
    java org.apache.derby.drda.NetworkServerControl 
  logconnections off [-h hostname][-p portnumber]
有关以下内容的更多信息，请参见《 Derby开发人员指南》 。 derby.log文件。
