Derby网络服务器

Derby网络服务器提供了单个系统内或网络上与Derby数据库的多用户连接。

网络服务器使用标准的分布式关系数据库体系结构（DRDA）协议来接收和答复来自客户端的查询。使用Derby Network Client驱动程序通过Derby Network Server访问数据库。

网络服务器是针对连接到数据库的多个JVM的解决方案，与嵌入式方案不同，在嵌入式方案中，只有一个JVM作为系统的一部分运行。当Derby嵌入在单JVM应用程序中时，嵌入式JDBC驱动程序将调用本地Derby数据库。当Derby嵌入在服务器框架中时，服务器框架的连接软件会通过网络或Internet向多个客户端JDBC应用程序提供数据。

要运行Derby Network Server，您需要安装以下文件：
  • 在服务器端，安装derby.jar和derbynet.jar 。
  • 在客户端安装derbyclient.jar 。

有几种方法可以管理Derby Network Server，包括：

  • 通过命令行
  • 通过使用.bat和.ksh剧本
  • 使用您自己的Java程序（使用网络服务器API编写）
  • 通过设置网络服务器属性

默认情况下，启动网络服务器时将为安全管理器安装基本安全策略。该策略文件称为server.policy并且是derbynet.jar 。有关策略文件，授予权限和属性扩展的信息，请参见http://docs.oracle.com/javase/7/docs/technotes/guides/security/PolicyFiles.html和上的“默认策略实现和策略文件语法” http://docs.oracle.com/javase/7/docs/technotes/guides/security/PolicyGuide.html上的 “策略文件创建和管理”。

将网络服务器与预先存在的Derby应用程序一起使用说明了如何更改当前以嵌入式模式针对Derby运行的现有Java应用程序，以针对Derby Network Server运行。

管理Derby Network Server解释了如何使用命令行来管理Network Server，包括启动和停止它。

Derby Network Server高级主题包含面向Derby Network Server用户的高级主题。

由于所使用的JDBC驱动程序有所不同，因此在网络服务器框架中运行Derby时（而不是在用户应用程序中运行它）可能会遇到功能上的差异。有关嵌入式和网络服务器配置之间差异的完整列表，请参阅将网络服务器与预先存在的Derby应用程序一起使用 。
