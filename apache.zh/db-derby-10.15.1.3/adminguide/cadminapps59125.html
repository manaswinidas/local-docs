将网络服务器与预先存在的Derby应用程序一起使用

您必须修改当前以嵌入式模式针对Derby运行的Java应用程序，以便它们可与Derby Network Server一起使用。

本节中的主题讨论了这些更改。

  • 网络服务器和Java虚拟机（JVM）
    Derby Network Server与Java平台标准版（Java SE）9及更高版本兼容。
  • 安装所需的jar文件并将其添加到classpath / modulepath
    要使用网络服务器和网络客户端驱动程序，请将以下jar文件添加到服务器的classpath / modulepath中。
  • 启动网络服务器
    要启动网络服务器，可以调用脚本，JAR文件或类。
  • 关闭网络服务器
    要关闭网络服务器，您可以调用脚本，jar文件或类。
  • 获取系统信息
    您可以使用sysinfo实用程序获取有关网络服务器的信息，例如版本和当前属性值，Java信息以及Derby数据库服务器信息。
  • 使用网络客户端驱动程序访问网络服务器
    连接到网络服务器时，您的应用程序需要加载特定于网络服务器的驱动程序和连接URL。此外，如果使用身份验证，则必须指定用户名和密码。
  • 通过使用数据源对象访问网络服务器
    网络服务器支持一组Derby Network Client驱动程序数据源类。
  • XA和网络服务器
    Derby嵌入式驱动程序和网络服务器都提供XA支持。网络服务器提供DRDA 7级支持。支持XAMGR的DRDA客户端（例如Derby网络客户端）可以将XA请求发送到网络服务器。
  • 在网络服务器上使用Derby工具
    Derby工具ij和dblook在嵌入式模式和客户机/服务器模式下工作。
  • 在嵌入式模式下运行Derby与使用网络服务器之间的区别
    本节描述了在嵌入式模式下运行Derby与使用网络服务器之间的区别。
  • 设置端口号
    默认情况下，Derby Network Server侦听TCP / IP端口号1527。如果要使用其他端口号，则可以在启动网络服务器时在命令行上指定它。
