获取数据源

通常，您可以简单地使用javax.sql的接口。数据源 ， javax.sql。ConnectionPoolDataSource和javax.sql。XADataSource ，如以下示例所示。

import org.apache.derby.jdbc.EmbeddedConnectionPoolDataSource;
import org.apache.derby.jdbc.EmbeddedDataSource;
import org.apache.derby.jdbc.EmbeddedXADataSource; 

javax.sql.ConnectionPoolDataSource cpds = new EmbeddedConnectionPoolDataSource(); 
javax.sql.DataSource ds = new EmbeddedDataSource(); 
javax.sql.XADataSource xads = new EmbeddedXADataSource();

Derby为DataSource提供六个属性。这些属性在org.apache.derby.jdbc中。EmbeddedDataSource 。他们是：

  • 数据库名称

    必须设置此强制属性。它标识要访问的数据库。访问位于/ LOCAL1 /分贝/袋熊 ，呼叫setDatabaseName（“/ LOCAL1 /分贝/袋熊”）上的数据源对象数据库命名袋熊 。

  • 创建数据库

    可选的。设置一个属性，以在下次调用数据源对象的getConnection方法时创建数据库。字符串createString始终为“ create”（或可能为null）。（使用方法setDatabaseName（）定义数据库的名称。）

  • 关机数据库

    可选的。设置一个属性以关闭数据库。字符串shutDownString始终为“ shutdown”（或可能为null）。下次调用数据源对象的getConnection方法时，关闭数据库。

  • 数据源名称

    可选的。ConnectionPoolDataSource或XADataSource的名称。数据源对象未使用。仅用于提供信息。

  • 描述

    可选的。数据源的描述。数据源对象未使用。仅用于提供信息。

  • connectionAttributes

    可选的。特定于Derby的连接属性。有关属性的更多信息，请参见《 Derby参考手册 》。
