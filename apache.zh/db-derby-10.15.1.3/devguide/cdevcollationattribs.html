控制排序规则的数据库连接URL属性

创建Derby数据库时，您设置的属性确定与数据库中所有字符数据一起使用的排序规则。

下表显示了一些示例。

表格1。控制归类的数据库创建URL属性
示例创建URL 归类驱动
jdbc：derby：abcDB; create = true Unicode代码点排序规则（UCS_BASIC），这是Derby数据库的默认排序规则。
jdbc：derby：abcDB; create = true; territory = es_MX Unicode代码点排序规则（UCS_BASIC）。未设置collation = collation属性。
jdbc：derby：abcDB; create = true; collation = TERRITORY_BASED JVM的语言环境，因为没有设置region = ll_CC属性。
提示：要确定JVM的语言环境，请调用Locale.getDefault方法。
jdbc：derby：abcDB; create = true; territory = es_MX; collation = TERRITORY_BASED 地域= ll_CC属性。
jdbc：derby：abcDB; create = true; territory = es_MX; collation = TERRITORY_BASED：PRIMARY 整理强度为PRIMARY的region = ll_CC属性，使数据库不区分大小写（通常意味着仅将基本字母中的差异视为重要的）。
