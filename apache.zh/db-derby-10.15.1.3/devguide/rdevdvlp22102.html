数据库连接示例

本节中的示例使用在嵌入式环境中使用的连接URL的语法。

有关连接URL的参考信息，请参阅Derby参考手册中的 “具有嵌入式数据库的应用程序的数据库连接URL的语法”或Derby Server and Administration Guide中的“使用网络客户端驱动程序访问网络服务器” 。

  • jdbc：derby：db1

    打开与数据库db1的连接。 db1是位于系统目录中的目录。

  • jdbc：derby：london / sales

    打开到数据库london / sales的连接。 london是系统目录的子目录，而sales是该目录london的子目录。

  • jdbc：derby：/引用/短语/法语

    打开与数据库/ reference / phrases / french的连接 。

    在UNIX系统上，这将是目录的路径。在Windows系统上，如果当前驱动器为C ，则路径为C：\ reference \ phrases \ french 。

  • jdbc：derby：a：/ demo / sample

    打开与Windows系统上驱动器A （通常是软盘驱动器）上\ demo \ sample目录中存储的数据库的连接。

  • jdbc： 德比 ：c：/ databases / salesdb jdbc：derby：salesdb

    如果Derby系统的系统目录是C：\ databases ， 则这两个连接URL将连接到Windows平台上的相同数据库salesdb 。

  • jdbc：derby：support / bugsdb; create = true

    在系统目录中创建数据库support / bugsdb ，如果中间目录支持不存在，则自动创建该目录。

  • jdbc：derby：sample; shutdown = true

    关闭示例数据库。（未启用身份验证，因此不需要用户凭据。）

  • jdbc：derby：memory：myDB

    访问名为myDB的内存数据库。客户端连接URL的语法不同。有关详细信息，请参见使用内存数据库 。

  • jdbc：derby：classpath：myDB

    作为只读数据库访问myDB （直接在类路径的目录中）。

  • jdbc：derby：jar：（C：/dbs.jar）产品/水煮食品

    从JAR文件C的产品目录访问只读数据库boiledfood：/dbs.jar。

  • jdbc：derby：directory：myDB

    访问myDB ，它位于系统目录中。
