HTTP建议

不管编译和使用什么模块，Apache作为基本的Web服务器都遵循以下IETF建议：

RFC 1945 （信息性）
超文本传输协议（HTTP）是一种应用程序级别的协议，具有分布式，协作式超媒体信息系统所需的轻便性和速度。本文档记录了HTTP / 1.0。
RFC 2616 （标准跟踪）
超文本传输协议（HTTP）是分布式，协作式超媒体信息系统的应用程序级协议。本文档记录了HTTP / 1.1。
RFC 2396 （标准跟踪）
统一资源标识符（URI）是紧凑的字符串，用于标识抽象资源或物理资源。
RFC 4346 （标准跟踪）
TLS协议提供了Internet上的通信安全性。它提供加密功能，旨在防止窃听，篡改和伪造消息。

HTML建议

关于超文本标记语言，Apache遵循以下IETF和W3C建议：

RFC 2854 （信息性）
本文档总结了HTML开发的历史，并通过指向相关W3C建议来定义“ text / html” MIME类型。
HTML 4.01规范 （ 勘误 ）
该规范定义了超文本标记语言（HTML），这是万维网的发布语言。该规范定义了HTML 4.01，它是HTML 4的子版本。
HTML 3.2参考规范
超文本标记语言（HTML）是一种简单的标记语言，用于创建可从一个平台移植到另一个平台的超文本文档。HTML文档是SGML文档。
XHTML 1.1-基于模块的XHTML （ Errata ）
本建议书定义了一种新的XHTML文档类型，该类型基于XHTML模块化中定义的模块框架和模块。
XHTML 1.0可扩展超文本标记语言（第二版） （ 勘误 ）
该规范定义了第二版XHTML 1.0，将HTML 4重新格式化为XML 1.0应用程序以及对应于HTML 4定义的三个DTD。

认证方式

关于身份验证的不同方法，Apache遵循以下IETF建议：

RFC 2617 （标准跟踪）
“ HTTP / 1.0”包括基本访问身份验证方案的规范。

语言/国家/地区代码

以下链接记录了ISO以及其他语言和国家/地区代码信息：

ISO 639-2
ISO 639提供了两组语言代码，一组为两个字母的代码集（639-1），另一组为三个字母的代码集（ISO 639的此部分），用于表示语言名称。
ISO 3166-1
这些页面按照ISO 3166-1和相应的ISO 3166-1-alpha-2代码元素中的字母顺序记录了国家名称（英文的官方简称）。
BCP 47 （最新最佳实践）， RFC 3066
本文档描述了一种语言标签，用于在需要指示信息对象中使用的语言的情况下，如何注册要在该语言标签中使用的值以及与这些语言标签匹配的结构。
RFC 3282 （标准跟踪）
本文档定义了一个“ Content-language：”标头，用于在需要指示具有RFC 822之类的标头（例如MIME正文部分或Web文档）和“ Accept-Language：”标头的语言时使用的标头。用于希望表明自己对语言的偏爱的情况。
