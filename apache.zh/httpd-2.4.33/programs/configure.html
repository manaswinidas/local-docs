配置-配置源树

的configure该脚本可配置源代码树，以在特定平台上编译和安装Apache HTTP Server。各种选项允许根据您的个人需求编译服务器。

此脚本（包括在源发行版的根目录中）仅在Unix和类Unix系统上编译。对于其他平台，请参阅平台文档。

概要

您应该致电configure发行版根目录中的脚本。

./configure [OPTION]... [VAR=VALUE]...

分配环境变量（例如CC ， CFLAGS ...），将其指定为VAR=VALUE 。有关某些有用变量的说明，请参见下文 。

配置选项

以下选项会影响以下行为： configure本身。

-C
--config-cache
这是别名--cache-file=config.cache
--cache-file=FILE
测试结果将缓存在文件中FILE 。默认情况下禁用此选项。
-h
--help [short|recursive]
输出帮助并退出。与论点short仅显示特定于此软件包的选项。争论recursive显示所有包含的软件包的简短帮助。
-n
--no-create
的configure脚本可以正常运行，但不会创建输出文件。这对于在生成用于编译的makefile之前检查测试结果很有用。
-q
--quiet
不列印checking ...消息在配置过程中。
--srcdir=DIR
定义目录DIR成为源文件目录。默认目录是configure位于，或父目录。
--silent
和...一样--quiet
-V
- 版
显示版权信息并退出。

安装目录

这些选项定义安装目录。安装树取决于所选的布局。

--prefix=PREFIX
在以下位置安装与体系结构无关的文件PREFIX 。默认情况下，安装目录设置为/usr/local/apache2 。
--exec-prefix=EPREFIX
在中安装与体系结构相关的文件EPREFIX 。默认情况下，安装目录设置为PREFIX目录。

默认， make install将所有文件安装在/usr/local/apache2/bin ， /usr/local/apache2/lib等等。您可以指定安装前缀，而不是/usr/local/apache2使用--prefix ， 例如--prefix=$HOME 。

定义目录布局

--enable-layout=LAYOUT
配置源代码和构建脚本以根据布局假定安装树LAYOUT 。这使您可以分别指定Apache HTTP Server安装中每种文件类型的位置。的config.layout文件包含几个示例配置，您还可以根据示例创建自己的自定义配置。该文件中的不同布局分为...部分，并按以下名称命名FOO 。默认布局是Apache 。

微调安装目录

为了更好地控制安装目录，请使用以下选项。请注意，目录默认设置是由autoconf并被相应的布局设置覆盖。

--bindir=DIR
在以下位置安装用户可执行文件DIR 。用户可执行文件正在支持类似的程序htpasswd ， dbmmanage等对网站管理员有用的工具。默认DIR被设定为EPREFIX/bin 。
--datadir=DIR
在以下位置安装与架构无关的只读数据： DIR 。默认datadir被设定为PREFIX/share 。此选项由autoconf目前尚未使用。
--includedir=DIR
在以下位置安装C头文件DIR 。默认includedir被设定为EPREFIX/include 。
--infodir=DIR
在安装信息文档DIR 。默认infodir被设定为PREFIX/info 。该选项当前未使用。
--libdir=DIR
在安装目标代码库DIR 。默认libdir被设定为EPREFIX/lib 。
--libexecdir=DIR
在以下位置安装程序可执行文件（即共享模块） DIR 。默认libexecdir被设定为EPREFIX/modules 。
--localstatedir=DIR
在以下位置安装可修改的单机数据DIR 。默认localstatedir被设定为PREFIX/var 。此选项由autoconf目前尚未使用。
--mandir=DIR
将man文档安装在DIR 。默认mandir被设定为EPREFIX/man 。
--oldincludedir=DIR
在非gcc中安装C头文件DIR 。默认oldincludedir被设定为/usr/include 。此选项由autoconf目前尚未使用。
--sbindir=DIR
在以下位置安装系统管理员可执行文件DIR 。这些是服务器程序，例如httpd ， apachectl ， suexec等等，这些是运行Apache HTTP Server所必需的。默认sbindir被设定为EPREFIX/sbin 。
--sharedstatedir=DIR
在以下位置安装与架构无关的可修改数据： DIR 。默认sharedstatedir被设定为PREFIX/com 。此选项由autoconf目前尚未使用。
--sysconfdir=DIR
安装只读单机数据，例如服务器配置文件httpd.conf ， mime.types等DIR 。默认sysconfdir被设定为PREFIX/conf 。

系统类型

这些选项用于交叉编译Apache HTTP Server以在另一个系统上运行。在正常情况下，在同一系统上构建和运行服务器时，将不使用这些选项。

--build=BUILD
定义在其上构建工具的系统的系统类型。它默认为脚本的结果config.guess 。
--host=HOST
定义将在其上运行服务器的系统的系统类型。 HOST默认为BUILD 。
--target=TARGET
配置用于构建系统类型的编译器TARGET 。默认为HOST 。此选项由autoconf而对于Apache HTTP Server则不是必需的。

可选功能

这些选项用于微调HTTP服务器将具有的功能。

一般语法

通常，您可以使用以下语法来启用或禁用功能：

--disable-FEATURE
不包括FEATURE 。这和--enable-FEATURE=no 。
--enable-FEATURE[=ARG]
包括FEATURE 。的默认值ARG是yes 。
--enable-MODULE=shared
相应的模块将被构建为DSO模块。默认情况下，启用的模块是动态链接的。
--enable-MODULE=static
相应的模块将被静态链接。

注意

configure不会抱怨--enable-foo即使foo不存在，因此您需要仔细输入。

选择要编译的模块

大多数模块默认情况下已编译，必须显式禁用或使用关键字禁用few （看到--enable-modules ， --enable-mods-shared和--enable-mods-static以便进一步解释）或--enable-modules=none作为一个整体被删除。

其他模块默认情况下不编译，必须显式启用或使用关键字启用all要么reallyall可用。

要找出默认情况下编译的模块，请运行./configure -h要么./configure --help然后看下Optional Features 。假设您对mod_example1和mod_example2 ，您会看到以下内容：

Optional Features:
  ...
  --disable-example1     example module 1
  --enable-example2      example module 2
  ...

然后mod_example1默认情况下处于启用状态，您将使用--disable-example1不编译它。 mod_example2默认情况下处于禁用状态，您可以使用--enable-example2进行编译。

多处理模块

多处理模块 （MPM）实现服务器的基本行为。为了使服务器正常运行，单个MPM必须处于活动状态。可用的MPM列表显示在模块索引页面上 。

MPM可以构建为DSO，用于动态加载或与服务器静态链接，并且可以使用以下选项启用：

--with-mpm=MPM

选择服务器的默认MPM。如果MPM是作为DSO模块构建的（请参阅--enable-mpms-shared ），此伪指令选择将在默认配置文件中加载的MPM。否则，此伪指令选择唯一可用的MPM，它将被静态链接到服务器中。

如果省略此选项，则将使用操作系统的默认MPM 。

--enable-mpms-shared=MPM-LIST

启用MPM列表作为动态共享模块。这些模块之一必须使用LoadModule指示。

MPM-LIST是用引号括起来的MPM名称的空格列表。例如：

--enable-mpms-shared='prefork worker'

另外，您可以使用特殊关键字all ，它将选择在当前平台上支持动态加载的所有MPM，并将其构建为DSO模块。例如：

--enable-mpms-shared=all

第三方模块

要添加其他第三方模块，请使用以下选项：

--with-module=module-type:module-file[, module-type:module-file]

将一个或多个第三方模块添加到静态链接的模块列表中。模块源文件module-file将在modules/module-type Apache HTTP服务器源树的子目录。如果在那里找不到configure正在考虑module-file成为绝对文件路径，并尝试将源文件复制到module-type子目录。如果子目录不存在，将创建该子目录并使用标准填充Makefile.in 。

此选项对于添加由一个源文件组成的小型外部模块很有用。对于更复杂的模块，您应该阅读供应商的文档。

注意

如果要构建DSO模块而不是静态链接，请使用apxs 。

累积和其他选择

--enable-maintainer-mode
打开调试和编译时间警告，并加载所有已编译模块。
--enable-mods-shared=MODULE-LIST

定义要启用并构建为动态共享模块的模块列表。这意味着，必须使用LoadModule指示。

MODULE-LIST是用引号括起来的模块名称的空格分隔列表。模块名称不带前面的名称mod_ 。例如：

--enable-mods-shared='headers rewrite dav'

另外，您可以使用特殊关键字reallyall ， all ， most和few 。例如，

--enable-mods-shared=most

将编译大多数模块并将其构建为DSO模块，

--enable-mods-shared=few

只会编译非常基本的模块集。

默认设置为most 。

的LoadModule所选模块的指令将在主配置文件中自动生成。默认情况下，所有这些指令都将被注释掉，除了配置需要的模块或显式选择的模块--enable-foo论点。您可以通过激活或取消激活已加载的模块集来更改LoadModule指令在httpd.conf 。除此之外LoadModule可以通过configure选项激活所有内置模块的指令--enable-load-all-modules 。

--enable-mods-static=MODULE-LIST
此选项的行为类似于--enable-mods-shared ，但将静态链接给定的模块。这意味着，这些模块在运行时将始终存在httpd 。他们不需要加载LoadModule 。
--enable-modules=MODULE-LIST
此选项的行为类似于--enable-mods-shared ，还将动态链接给定的模块。特殊关键字none禁用所有模块的构建。
--enable-v4-mapped
允许IPv6套接字处理IPv4连接。
--with-port=PORT
这定义了哪个端口httpd会听。生成配置文件时使用此端口号httpd.conf 。默认值为80。
--with-program-name
定义备用可执行文件名称。默认是httpd 。

可选包装

这些选项用于定义可选软件包。

一般语法

通常，您可以使用以下语法来定义可选程序包：

--with-PACKAGE[=ARG]
使用包装PACKAGE 。的默认值ARG是yes 。
--without-PACKAGE
不要使用包装PACKAGE 。这和--with-PACKAGE=no 。此选项由autoconf但对Apache HTTP Server而言不是很有用。

具体套餐

--with-apr=DIR|FILE
Apache可移植运行时 （APR）是httpd源分发的一部分，将与HTTP服务器一起自动构建。如果要使用已安装的APR，则必须告诉configure通往的道路apr-config脚本。您可以将绝对路径和名称或目录设置为已安装的APR。 apr-config必须存在于此目录或子目录中bin 。
--with-apr-util=DIR|FILE
Apache可移植运行时实用程序（APU）是httpd源分发的一部分，将与HTTP服务器一起自动构建。如果要使用已安装的APU，则必须告诉configure通往的道路apu-config脚本。您可以为安装的APU设置绝对路径和名称或目录。 apu-config必须存在于此目录或子目录中bin 。
--with-ssl=DIR
如果mod_ssl已启用configure搜索已安装的OpenSSL。您可以将目录路径设置为SSL / TLS工具包。
--with-z=DIR
configure自动搜索已安装的zlib库，如果您的源配置需要一个（例如，当mod_deflate已启用）。您可以将目录路径设置为压缩库。

Apache HTTP Server的一些功能，包括mod_authn_dbm和mod_rewrite的DBM RewriteMap使用简单的键/值数据库快速查找信息。SDBM包含在APU中，因此该数据库始终可用。如果要使用其他数据库类型，请使用以下选项启用它们：

--with-gdbm[=path]
如果不path被指定configure将在通常的搜索路径中搜索GNU DBM安装的包含文件和库。一个明确的path会引发configure看path/lib和path/include有关文件。最后， path可以指定由冒号分隔的特定包含和库路径。
--with-ndbm[=path]
喜欢--with-gdbm ，但搜索新的DBM安装。
--with-berkeley-db[=path]
喜欢--with-gdbm ，但搜索Berkeley DB安装。

注意

DBM选项由APU提供，并传递到其配置脚本。使用已定义的已安装的APU时，它们无用--with-apr-util 。

您可以将不止一种DBM实现与HTTP服务器一起使用。每次都会在运行时配置中配置适当的DBM类型。

支持计划的选项

--enable-static-support
构建支持二进制文件的静态链接版本。这意味着，将集成所有必需的库来构建独立的可执行文件。否则，默认情况下将动态链接支持二进制文件。
--enable-suexec
使用此选项启用suexec ，它允许您为生成的进程设置uid和gid。除非您了解在服务器上运行suid二进制文件的所有安全隐患，否则不要使用此选项。 其他配置选项suexec 如下所述 。

使用以下选项可以创建单个支持程序的静态链接二进制文件：

--enable-static-ab
构建的静态链接版本ab 。
--enable-static-checkgid
构建的静态链接版本checkgid 。
--enable-static-htdbm
构建的静态链接版本htdbm 。
--enable-static-htdigest
构建的静态链接版本htdigest 。
--enable-static-htpasswd
构建的静态链接版本htpasswd 。
--enable-static-logresolve
构建的静态链接版本logresolve 。
--enable-static-rotatelogs
构建的静态链接版本rotatelogs 。

suexec配置选项

以下选项用于微调以下行为： suexec 。有关更多信息，请参见配置和安装suEXEC 。

--with-suexec-bin
这定义了通往suexec二进制默认为--sbindir （请参阅安装目录的微调 ）。
--with-suexec-caller
这定义了允许呼叫的用户suexec 。它应该与哪个用户相同httpd正常运行。
--with-suexec-docroot
这定义了目录树suexec允许可执行文件访问。默认值为--datadir/htdocs 。
--with-suexec-gidmin
将此定义为允许作为目标用户的最低GID suexec 。默认值为100。
--with-suexec-logfile
这定义了文件名suexec日志文件。默认情况下，日志文件名为suexec_log并位于--logfiledir 。
--with-suexec-safepath
定义环境变量的值PATH为启动的流程设置suexec 。默认值为/usr/local/bin:/usr/bin:/bin 。
--with-suexec-userdir
这定义了用户目录下的子目录，该子目录包含所有可执行文件， suexec允许访问。当您要使用此设置suexec以及用户特定的目录（由mod_userdir ）。默认是public_html 。
--with-suexec-uidmin
将此定义为允许作为目标用户的最低UID suexec 。默认值为100。
--with-suexec-umask
组umask用于由...启动的过程suexec 。默认为您的系统设置。

环境变量

有一些有用的环境变量可以覆盖以下选项： configure或帮助它查找具有非标准名称或位置的库和程序。

CC
定义用于编译的C编译器命令。
CFLAGS
设置要用于编译的C编译器标志。
CPP
定义要使用的C预处理程序命令。
CPPFLAGS
设置C / C ++预处理器标志，例如-Iincludedir如果标头位于非标准目录中includedir 。
LDFLAGS
设置链接器标志，例如-Llibdir如果您的库在非标准目录中libdir 。
