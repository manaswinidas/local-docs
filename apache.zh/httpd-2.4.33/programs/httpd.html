httpd-Apache超文本传输协议服务器

httpd是Apache超文本传输协议（HTTP）服务器程序。它被设计为作为独立的守护进程运行。像这样使用时，它将创建一个子进程或线程池来处理请求。

一般来说， httpd不应直接调用，而应通过调用apachectl在基于Unix的系统上或在Windows NT，2000和XP上作为服务， 在Windows 9x和ME上作为控制台应用程序 。

概要

httpd [ -d serverroot ] [ -f config ] [ -C directive ] [ -c directive ] [ -D parameter ] [ -e level ] [ -E file ] [ -k start|restart|graceful|stop|graceful-stop ] [ -h ] [ -l ] [ -L ] [ -S ] [ -t ] [ -v ] [ -V ] [ -X ] [ -M ] [ -T ]

在Windows系统上 ，可以使用以下附加参数：

httpd [ -k install|config|uninstall ] [ -n name ] [ -w ]

选件

-d serverroot
设置初始值ServerRoot指令serverroot 。可以通过配置文件中的ServerRoot指令来覆盖它。默认是/usr/local/apache2 。
-f config
使用文件中的指令config在启动时。如果config不以/开头，则视为相对于ServerRoot 。默认是conf/httpd.conf 。
-k start|restart|graceful|stop|graceful-stop
讯号httpd以启动，重新启动或停止。有关更多信息，请参见停止Apache httpd 。
-C directive
处理配置directive在读取配置文件之前。
-c directive
处理配置directive阅读配置文件后。
-D parameter
设置配置parameter可以与配置文件中的“节”以在服务器启动和重新启动时有条件地跳过或处理命令。也可以用来设置某些不太常见的启动参数，包括-DNO_DETACH （防止父母分叉）和-DFOREGROUND （防止家长打电话setsid()等）。
-e level
设置LogLevel至level服务器启动期间。这对于暂时增加错误消息的详细程度以在启动过程中发现问题很有用。
-E file
在服务器启动期间将错误消息发送给file 。
-h
输出可用命令行选项的简短摘要。
-l
输出编译到服务器中的模块列表。这不会列出使用LoadModule指示。
-L
输出静态模块提供的指令列表，以及预期的参数和指令有效的位置。共享模块提供的指令未列出。
-M
转储已加载的静态模块和共享模块的列表。
-S
显示从配置文件解析的设置（当前仅显示虚拟主机设置）。
-T （在2.3.8及更高版本中可用）
在启动/重新启动时跳过文档根检查。
-t
仅对配置文件运行语法测试。在这些语法分析测试之后，程序立即退出，返回码为0（语法确定）或返回码不等于0（语法错误）。如果-D DUMP _ VHOSTS如果同时设置，将打印虚拟主机配置的详细信息。如果-D DUMP _ MODULES设置后，将打印所有已加载的模块。
-v
打印版本httpd ，然后退出。
-V
打印版本和构建参数httpd ，然后退出。
-X
在调试模式下运行httpd。仅将启动一个工作服务器，并且服务器不会从控制台分离。

以下参数仅在Windows平台上可用：

-k install|config|uninstall
将Apache httpd安装为Windows NT服务；更改Apache httpd服务的启动选项；并卸载Apache httpd服务。
-n name
的name Apache httpd服务的信号。
-w
使控制台窗口在发生错误时保持打开状态，以便可以读取错误消息。
