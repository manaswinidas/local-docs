dbmmanage-以DBM格式管理用户身份验证文件

dbmmanage用于创建和更新用于存储用户名和密码的DBM格式文件，以通过HTTP进行HTTP用户的基本身份验证mod_authn_dbm 。可将Apache HTTP服务器上可用的资源限制为仅由以下用户创建的文件中列出的用户dbmmanage 。仅当用户名存储在DBM文件中时，才可以使用该程序。要使用平面文件数据库，请参见htpasswd 。

维护DBM密码数据库的另一种工具是htdbm 。

本手册页仅列出命令行参数。有关在中配置用户身份验证所需的指令的详细信息httpd请参阅httpd手册，该手册是Apache发行版的一部分，或者可以在http://httpd.apache.org/上找到。

概要

dbmmanage [ encoding ] filename add|adduser|check|delete|update username [ encpasswd [ group[,group...] [ comment ] ] ]

dbmmanage filename view [ username ]

dbmmanage filename import

选件

filename
DBM格式文件的文件名。通常不带扩展名.db ， .pag ， 要么.dir 。
username
执行操作的用户。的username可能不包含冒号（ : ）。
encpasswd
这是已加密的密码，可用于update和add命令。您可以使用连字符（ - ），如果您想获得提示输入密码的信息，请在此后填写字段。此外，使用update命令，一段（ . ）保持原始密码不变。
group
用户所属的组。组名不能包含冒号（ : ）。您可以使用连字符（ - ）（如果您不想将用户分配到组），而是填写评论字段。此外，使用update命令，一段（ . ）保持原始组不变。
comment
在这里您可以对用户发表不透明的评论，例如真实姓名，邮件地址或类似内容。服务器将忽略此字段。

编码方式

-d
crypt加密（默认，Win32，Netware除外）
-m
MD5加密（Win32，Netware上的默认设置）
-s
SHA1加密
-p
明文（ 不推荐 ）

指令

add
为添加一个条目username至filename使用加密的密码encpasswd 。

dbmmanage passwords.dat add rbowen foKntnEF3KSXA

adduser
要求输入密码，然后添加一个条目username至filename 。

dbmmanage passwords.dat adduser krietz

check
要求输入密码，然后检查是否username在filename密码是否与指定的密码匹配。

dbmmanage passwords.dat check rbowen

delete
删除username来自的条目filename 。

dbmmanage passwords.dat delete rbowen

import
读username:password条目（每行一个） STDIN并将它们添加到filename 。密码已经必须被加密。
update
与相同adduser命令，除了可以确保username已经存在于filename 。

dbmmanage passwords.dat update rbowen

view
仅显示DBM文件的内容。如果您指定一个username ，它仅显示特定记录。

dbmmanage passwords.dat view

虫子

应该意识到，存在许多不同的DBM文件格式，并且很可能在您的系统上存在一种以上格式的库。三个主要示例是SDBM，NDBM，GNU项目的GDBM和Berkeley DB 2。不幸的是，所有这些库都使用不同的文件格式，并且您必须确保filename格式与dbmmanage希望看到。 dbmmanage当前无法确定正在查看的DBM文件类型。如果以错误的格式使用它，将仅返回任何内容，或者使用不同的名称创建另一个DBM文件，或者在最坏的情况下，如果尝试写入该文件，可能会损坏DBM文件。

dbmmanage列出了DBM格式的首选项，由@AnyDBM::ISA程序开始附近的数组。由于我们更喜欢Berkeley DB 2文件格式，因此顺序dbmmanage将查找系统库是Berkeley DB 2，然后是NDBM，然后是GDBM，然后是SDBM。找到的第一个图书馆将是图书馆dbmmanage将尝试用于所有DBM文件事务。此订购与标准订购略有不同@AnyDBM::ISA Perl中的排序，以及简单代码使用的排序dbmopen()调用Perl，因此，如果您使用任何其他实用程序来管理DBM文件，则它们也必须遵循此首选项顺序。如果使用其他语言的程序（如C）访问这些文件，则必须采取类似的措施。

通常可以使用file大多数Unix系统附带的程序，以查看DBM文件的格式。
