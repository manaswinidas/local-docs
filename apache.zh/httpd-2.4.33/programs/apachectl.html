apachectl-Apache HTTP服务器控制接口

apachectl是Apache超文本传输协议（HTTP）服务器的前端。它旨在帮助管理员控制Apache的功能。 httpd守护程序。

的apachectl脚本可以在两种模式下运行。首先，它可以充当简单的前端httpd该命令仅设置任何必要的环境变量，然后调用httpd ，通过任何命令行参数传递。第二， apachectl可以充当SysV初始化脚本，采用简单的一词参数，例如start ， restart和stop ，并将其转换为适当的信号，以httpd 。

如果您的Apache安装使用非标准路径，则需要编辑apachectl脚本来设置适当的路径httpd二进制您还可以指定任何必要的httpd命令行参数。有关详细信息，请参见脚本中的注释。

的apachectl脚本成功返回0退出值，如果发生错误则返回> 0。有关更多详细信息，请查看脚本中的注释。

概要

在通过模式下进行操作时， apachectl可以接受所有可用的参数httpd二进制

apachectl [ httpd-argument ]

在SysV初始化模式下操作时， apachectl使用简单的一字命令，定义如下。

apachectl command

选件

此处仅定义SysV初始化样式选项。其他参数在httpd手册页。

start
启动Apache httpd守护程序。如果已在运行，则给出错误。这相当于apachectl -k start 。
stop
停止Apache httpd守护程序。这相当于apachectl -k stop 。
restart
重新启动Apache httpd守护程序。如果守护程序未在运行，它将启动。该命令自动检查配置文件，如下所示configtest在启动重新启动之前，请确保守护程序不会死亡。这相当于apachectl -k restart 。
fullstatus
显示来自的完整状态报告mod_status 。为此，您需要mod_status在服务器和基于文本的浏览器（例如lynx在您的系统上可用。可以通过以下方式设置用于访问状态报告的网址： STATUSURL脚本中的变量。
status
显示简短的状态报告。类似于fullstatus选项，除了省略当前正在处理的请求列表。
graceful
正常重启Apache httpd守护程序。如果守护程序未在运行，它将启动。这与正常重启不同，因为当前打开的连接不会中止。副作用是旧的日志文件不会立即关闭。这意味着，如果将其用于日志轮换脚本中，则可能需要相当长的延迟才能确保在处理旧日志文件之前将其关闭。该命令自动检查配置文件，如下所示configtest在启动重新启动以确保Apache不会死机之前。这相当于apachectl -k graceful 。
graceful-stop
优雅地停止Apache httpd守护程序。这与正常停止的不同之处在于，当前打开的连接不会中断。副作用是旧的日志文件不会立即关闭。这相当于apachectl -k graceful-stop 。
configtest
运行配置文件语法测试。它解析配置文件和报告Syntax Ok或有关特定语法错误的详细信息。这相当于apachectl -t 。

以下选项在早期版本中可用，但已被删除。

startssl
开始httpd如果使用SSL支持，则应编辑配置文件以包含相关指令，然后使用常规apachectl start 。
