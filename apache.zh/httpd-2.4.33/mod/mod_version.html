Apache模块mod_version

描述：版本相关配置
状态：延期
模块�标识符：版本模块
源文件：mod_version.c

摘要

该模块旨在用于必须处理不同httpd版本和不同配置的测试套件和大型网络中。它提供了一个新的容器<IfVersion> ，它允许灵活的版本检查，包括数字比较和正则表达式。

例子

<IfVersion 2.4.2>
    # current httpd version is exactly 2.4.2
</IfVersion>

<IfVersion >= 2.5>
    # use really new features :-)
</IfVersion>

参见下文了解更多可能性。

<IfVersion> 指令

描述：包含版本相关的配置
句法：<IfVersion [[！]操作员 ] 版本 > ...</ IfVersion>
内容：服务器配置，虚拟主机，目录，.htaccess
覆写：所有
状态：延期
模块：mod_version

<IfVersion>部分包含仅在httpd版本符合所需条件时才执行的配置指令。对于常规（数字）比较， version参数的格式为major [. minor [. patch ]] ，例如2.1.0或2.2 。 minor和patch是可选的。如果省略这些数字，则假定为零。可以使用以下数值运算符 ：

算子描述
=或== httpd版本相等
> httpd版本大于
> = httpd版本大于或等于
< httpd版本小于
<= httpd版本小于或等于

例

<IfVersion >= 2.3>
    # this happens only in versions greater or
    # equal 2.3.0.
</IfVersion>

除了数值比较，还可以将正则表达式与httpd版本进行匹配。有两种编写方法：

算子描述
=或== 版本的格式为/ regex /
〜 版本具有regex形式

例

<IfVersion = /^2.4.[01234]$/>
    # e.g. workaround for buggy versions
</IfVersion>

为了反转含义，可以在所有运算符的前面加上感叹号（ ! ）：

<IfVersion !~ ^2.4.[01234]$>
    # not for those versions
</IfVersion>

如果省略运算符 ，则假定为= 。
