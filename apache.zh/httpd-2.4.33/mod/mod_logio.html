Apache模块mod_logio

描述：记录每个请求的输入和输出字节
状态：延期
模块�标识符：logio_module
源文件：mod_logio.c

摘要

该模块记录每个请求接收/发送的输入和输出字节数。这些数字反映了网络上接收到的实际字节，然后考虑了请求和响应的标头和正文。计数是在输入的SSL / TLS之前和输出的SSL / TLS之后进行的，因此数字将正确反映加密所做的任何更改。

该模块需要mod_log_config 。

当KeepAlive连接与SSL一起使用时，SSL握手的开销反映在连接上第一个请求的字节数中。当按目录进行SSL重新协商时，字节与触发重新协商的请求相关联。

自定义日志格式

该模块添加了三个新的日志记录指令。通过将“ % ”指令放置在格式字符串中来记录请求本身的特征，这些指令在日志文件中由以下值代替：

格式字符串 描述
％一世 接收的字节（包括请求和标头）不能为零。
％O 发送的字节（包括标头）不能为零。
％S 传输（接收和发送）的字节（包括请求和标头）不能为零。这是％I和％O的组合。
在Apache 2.4.7和更高版本中可用
％^ FB 从请求到达到响应头的第一个字节之间的延迟（以微秒为单位）。仅在LogIOTrackTTFB设置为ON时可用。
在Apache 2.4.13和更高版本中可用

通常，功能的用法如下：

组合的I / O日志格式：
“％h％l％u％t \”％r \“％> s％b \”％{Referer} i \“ \”％{User-agent} i \“％I％O”

LogIOTrackTTFB 指令

描述：启用到第一个字节（TTFB）的时间跟踪
句法：LogIOTrackTTFB ON | OFF
默认：LogIOTrackTTFB关闭
内容：服务器配置，虚拟主机，目录，.htaccess
覆写：所有
状态：延期
模块：mod_logio
兼容性：Apache HTTP Server 2.4.13及更高版本

该指令配置此模块是否跟踪读取请求和写入响应标头的第一个字节之间的延迟。结果值可以用%^FB格式记录。
