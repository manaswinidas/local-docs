Apache模块mod_proxy_scgi

描述：用于mod_proxy SCGI网关模块
状态：延期
模块�标识符：proxy_scgi_module
源文件：mod_proxy_scgi.c
兼容性：在2.2.14版和更高版本中可用

摘要

该模块需要 mod_proxy服务。它提供对SCGI协议版本1的支持 。

因此，为了获得处理SCGI协议的能力，服务器中必须存在mod_proxy和mod_proxy_scgi 。

警告

在确保服务器安全之前，请勿启用代理。开放式代理服务器对您的网络和整个Internet都是危险的。

例子

请记住，为了使以下示例正常工作，必须启用mod_proxy和mod_proxy_scgi 。

简单的网关

ProxyPass /scgi-bin/ scgi://localhost:4000/

除上面列出的代理模块之外，平衡网关还需要mod_proxy_balancer和至少一个负载平衡器算法模块，例如mod_lbmethod_byrequests 。 mod_lbmethod_byrequests为mod_lbmethod_byrequests ，它将用于此示例配置。

平衡网关

ProxyPass "/scgi-bin/" "balancer://somecluster/"
<Proxy "balancer://somecluster">
    BalancerMember "scgi://localhost:4000"
    BalancerMember "scgi://localhost:4001"
</Proxy>

环境变量

除了控制mod_proxy行为的配置指令之外， 环境变量还可以控制SCGI协议提供程序：

proxy-scgi-pathinfo
默认情况下， mod_proxy_scgi既不会创建也不导出PATH_INFO环境变量。这允许后端SCGI服务器正确确定SCRIPT_NAME和Script-URI并符合RFC 3875第3.3节。相反，如果您需要mod_proxy_scgi为PATH_INFO生成“最佳猜测”，请设置此env-var。必须在SetEnv有效之前设置该变量。SetEnvIf可以用来代替： SetEnvIf Request_URI . proxy-scgi-pathinfo

ProxySCGIInternalRedirect 指令

描述：从后端启用或禁用内部重定向响应
句法：ProxySCGIInternalRedirect开|关| 标题名称
默认：ProxySCGIInternalRedirect开
内容：服务器配置，虚拟主机，目录
状态：延期
模块：mod_proxy_scgi
兼容性：标头功能在版本2.4.13和更高版本中可用

ProxySCGIInternalRedirect使后端可以在内部将网关重定向到其他URL。此功能起源于mod_cgi ，如果响应状态为OK （ 200 ）并且响应包含Location （或配置的备用标头）并且其值以斜杠（ / ）开头，则该模块在内部重定向响应。该值被解释为Apache httpd内部重定向到的新本地URL。

在这方面， mod_proxy_scgi与mod_cgi相同，除了可以关闭功能或指定使用Location以外的标头之外。

例

    ProxySCGIInternalRedirect Off

# Django and some other frameworks will fully qualify "local URLs"
# set by the application, so an alternate header must be used.
<Location /django-app/>
    ProxySCGIInternalRedirect X-Location
</Location>

ProxySCGISendfile 指令

描述：启用对X-Sendfile伪响应标头的评估
句法：ProxySCGISendfile开|关| 标题名称
默认：ProxySCGISendfile关
内容：服务器配置，虚拟主机，目录
状态：延期
模块：mod_proxy_scgi

ProxySCGISendfile指令使SCGI后端能够让文件直接由网关提供服务。这对于提高性能很有用sendfile可以使用sendfile或其他优化，如果文件来自后端套接字，则不可能。此外，文件内容不会发送两次。

ProxySCGISendfile参数确定网关行为：

关
不会进行任何特殊处理。
上
网关查找称为X-Sendfile的后端响应标头，并将该值解释为要提供的文件名。该头将从最终响应头中删除。这等效于ProxySCGISendfile X-Sendfile 。
还要别的吗
与On相似，但代替硬编码的标头名称X-Sendfile ，该参数用作标头名称。

例

# Use the default header (X-Sendfile)
ProxySCGISendfile On
    
# Use a different header
ProxySCGISendfile X-Send-Static
