例子

  1. 处理对/ foo / *的请求后的日志消息：
    <Location "/foo/">
��LogMessage "/foo/ has been requested"
</Location>
  2. 如果在子请求中处理了对/ foo / *的请求，则记录日志消息：
    <Location "/foo/">
��LogMessage "subrequest to /foo/" hook=type_checker "expr=-T %{IS_SUBREQ}"
</Location>
    默认的log_transaction钩子不用于子请求，因此我们必须使用其他钩子。
  3. 如果IPv6客户端导致请求超时的日志消息：
    LogMessage "IPv6 timeout from %{REMOTE_ADDR}" "expr=-T %{IPV6} && %{REQUEST_STATUS} = 408"
    注意expr=参数的双引号位置。
  4. 在请求的每个阶段中记录“ X-Foo”请求环境变量的值：
    <Location "/">
��LogMessage "%{reqenv:X-Foo}" hook=all
</Location>
    通过hook=all以及错误日志中的微秒时间戳，您还可以确定在请求处理的不同部分花费的时间。

LogMessage 指令

描述：将用户定义的消息记录到错误日志中
句法：LogMessage 消息 [hook = 钩子 ] [expr = 表达式 ]
默认：未设定
内容：目录
状态：实验性
模块：mod_log_debug

此伪指令导致将用户定义的消息记录到错误日志中。该消息可以使用ap_expr语法中的变量和函数。引用HTTP标头不会导致将标头名称添加到Vary标头中。消息记录在日志级别信息中。

挂钩指定在记录请求处理的哪个阶段之前。支持以下挂钩：

名称
翻译名称
type_checker
quick_handler
map_to_storage
check_access
check_access_ex
insert_filter
check_authn
check_authz
修正
处理程序
log_transaction

默认值为log_transaction 。还支持特殊值all ，从而导致在每个阶段记录一条消息。并非对于每个请求都执行所有钩子。

可选表达式允许在满足条件时限制消息。表达式语法的详细信息在ap_expr文档中进行了描述。引用HTTP标头不会导致将标头名称添加到Vary标头中。
