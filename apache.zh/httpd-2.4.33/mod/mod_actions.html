Apache模块mod_actions

描述：根据媒体类型或请求方法执行CGI脚本。
状态：基础
模块�标识符：actions_module
源文件：mod_actions.c

摘要

该模块有两个指令。使用Action指令，可以在每当请求某种MIME内容类型的文件时运行CGI脚本。Script指令可让您在请求中使用特定方法时运行CGI脚本。这使得执行处理文件的脚本变得更加容易。

行动 指令

描述：为特定的处理程序或内容类型激活CGI脚本
句法：动作动作类型 cgi-script [虚拟]
内容：服务器配置，虚拟主机，目录，.htaccess
覆写：文件信息
状态：基础
模块：mod_actions
兼容性：virtual修饰符和处理程序传递是在Apache 2.1中引入的

该指令添加了一个动作，当动作类型被请求触发时，它将激活cgi-script 。cgi-script是使用ScriptAlias或AddHandler指定为CGI脚本的资源的URL路径。动作类型可以是处理程序，也可以是MIME内容类型 。它使用标准CGI PATH_INFO和PATH_TRANSLATED环境变量发送所请求文档的URL和文件路径。使用REDIRECT_HANDLER变量传递用于特定请求的处理程序。

示例：MIME类型

# Requests for files of a particular MIME content type:
Action image/gif /cgi-bin/images.cgi

在此示例中，对MIME内容类型为image/gif文件的请求将由指定的cgi脚本/cgi-bin/images.cgi 。

示例：文件扩展名

# Files of a particular file extension
AddHandler my-file-type .xyz
Action my-file-type "/cgi-bin/program.cgi"

在此示例中，文件扩展名为.xyz的文件请求由指定的cgi脚本/cgi-bin/program.cgi 。

可选的virtual修饰符会关闭检查所请求的文件是否确实存在。例如，如果要在虚拟位置中使用Action指令，此功能将很有用。

<Location "/news">
    SetHandler news-handler
    Action news-handler "/cgi-bin/news.cgi" virtual
</Location>

脚本 指令

描述：为特定的请求方法激活CGI脚本。
句法：脚本方法 cgi-script
内容：服务器配置，虚拟主机，目录
状态：基础
模块：mod_actions

该指令添加了一个动作，当使用method 方法请求文件时，该动作将激活cgi-script 。cgi-script是使用ScriptAlias或AddHandler指定为CGI脚本的资源的URL路径。使用标准CGI PATH_INFO和PATH_TRANSLATED环境变量发送所请求文档的URL和文件路径。

可以使用任意方法名称。方法名称区分大小写 ，因此Script PUT和Script put具有两个完全不同的效果。

请注意， Script命令仅定义默认操作。如果调用了CGI脚本或其他能够在内部处理请求的方法的资源，它将这样做。还要注意，只有在存在查询参数的情况下，才会调用具有GET方法的Script （ 例如 foo.html？嗨）。否则，请求将正常进行。

# All GET requests go here
Script GET "/cgi-bin/search"

# A CGI PUT handler
Script PUT "/~bob/put.cgi"
