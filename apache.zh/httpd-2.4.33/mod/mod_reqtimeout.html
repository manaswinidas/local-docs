例子

  1. 等待10秒以接收请求，包括标头，等待30秒以接收请求正文：
    RequestReadTimeout header=10 body=30
  2. 至少等待10秒钟以接收请求正文。如果客户端发送数据，则每接收1000个字节将超时增加1秒，没有超时上限（由LimitRequestBody间接给出的限制除外）：
    RequestReadTimeout body=10,MinRate=1000
  3. 至少等待10秒钟以接收包含标头的请求。如果客户端发送数据，则每接收500个字节将超时增加1秒。但是，包括标头在内的请求不得超过30秒：
    RequestReadTimeout header=10-30,MinRate=500
  4. 通常，服务器应同时配置标头超时和主体超时。如果将通用配置用于http和https虚拟主机，则超时不应设置得太低：
    RequestReadTimeout header=20-40,MinRate=500 body=20,MinRate=500

RequestReadTimeout 指令

描述：设置超时值以接收来自客户端的请求标头和正文。
句法：RequestReadTimeout [header = 超时 [ -maxtimeout ] [，MinRate = rate ] [body = 超时 [ -maxtimeout ] [，MinRate = rate ]
默认：标头= 20-40，MinRate = 500正文= 20，MinRate = 500
内容：服务器配置，虚拟主机
状态：延期
模块：mod_reqtimeout
兼容性：在版本2.2.15和更高版本中可用；在版本2.3.14和更早版本中默认禁用。

该指令可以设置各种超时，以从客户端接收请求标头和请求正文。如果客户端未能在配置的时间内发送标头或正文，则会发送408 REQUEST TIME OUT错误。

对于SSL虚拟主机，标头超时值包括进行初始SSL握手所需的时间。如果将用户的浏览器配置为查询证书吊销列表，并且无法访问CRL服务器，则初始SSL握手可能要花费相当长的时间，直到浏览器放弃等待CRL为止。因此，对于SSL虚拟主机，标头超时值不应设置为非常低的值。主体超时值包括SSL重新协商所需的时间（如有必要）。

当使用AcceptFilter （通常在Linux和FreeBSD上就是这种情况），在至少接收到一个字节（或整个对httpready请求）之前，不会将套接字发送到服务器进程。使用RequestReadTimeout配置的标头超时仅在服务器进程收到套接字后才有效。

对于两种超时类型（标头或正文）中的每种，都有三种指定超时的方法：

  • 固定超时值 ：

    类型 = 超时

    分别读取所有请求标头或正文所允许的时间（以秒为单位）。值为0表示没有限制。

  • 禁用虚拟主机模块 ：

    标头= 0正文= 0

    这将完全禁用mod_reqtimeout 。

  • 接收数据时增加的超时值 ：

    类型 = 超时 ，MinRate = data_rate

    与上面相同，但是无论何时接收到数据，超时值都会根据指定的最小数据速率（以字节/秒为单位）增加。

  • 接收数据时增加的超时值，上限 ：

    类型 = 超时 -maxtimeout ，MinRate = data_rate

    与上面相同，但是超时不会增加到指定超时范围的第二个值以上。
