认证与授权

身份验证是您用来验证某人是否是他们所声称的身份的任何过程。授权是允许某人进入其想去的地方或获得其想要的信息的任何过程。

请参阅： 身份验证，授权

访问控制

访问控制是指基于任意准则限制或授予对资源的访问的过程。有多种不同的方法可以实现。

另请： 访问控制

使用CGI的动态内容

CGI（通用网关接口）为Web服务器定义了一种与外部内容生成程序进行交互的方式，这些程序通常称为CGI程序或CGI脚本。这是将动态内容放置在您的网站上的一种简单方法。本文档将介绍如何在Apache Web服务器上设置CGI，以及如何开始编写CGI程序。

请参阅： CGI：动态内容

.htaccess档案

.htaccess文件提供了一种基于目录进行配置更改的方法。包含一个或多个配置指令的文件放置在特定的文档目录中，这些指令适用于该目录及其所有子目录。

看到： .htaccess档案

HTTP / 2和httpd

HTTP / 2是世界上最成功的应用程序层协议HTTP的演进。它着重于在不更改HTTP语义的情况下更有效地利用网络资源。本指南说明了如何在httpd中实现HTTP / 2，并显示了基本的配置技巧和最佳实践。

请参阅： HTTP / 2指南

服务器端简介包括

SSI（服务器端包含）是放置在HTML页面中的指令，并在服务页面时在服务器上对其进行评估。它们使您可以将动态生成的内容添加到现有的HTML页面，而不必通过CGI程序或其他动态技术来提供整个页面。

请参阅： 服务器端包含（SSI）

每用户Web目录

在具有多个用户的系统上，可以使用以下命令允许每个用户在其主目录中拥有一个网站： UserDir指示。网址访问者http://example.com/~username/将从用户的主目录中获取内容” username ”，从UserDir指示。

请参阅： 用户Web目录（ public_html ）

反向代理指南

Apache httpd具有广泛的功能，可以作为反向代理服务器使用ProxyPass指令以及BalancerMember创建复杂的反向代理实现，以提供高可用性，负载平衡和故障转移，基于云的集群以及动态即时重新配置。

请参阅： 反向代理指南
