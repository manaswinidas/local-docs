8b。建立扩展的LDAP测试计划

在本节中，您将学习如何创建基本的测试计划以测试LDAP服务器。

由于扩展的LDAP采样器是高度可配置的，因此这也意味着需要花费一些时间来构建正确的测试计划。但是，您可以完全根据自己的需要进行调整。

您将创建1个用户，该用户在LDAP服务器上发送对九个测试的请求。另外，您将告诉用户一次运行他们的测试。因此，请求总数为（1个用户）x（9个请求）x（重复1次）= 9个 LDAP请求。要构建测试计划，您将使用以下元素：
线程组
添加LDAP扩展请求默认值 ，
添加LDAP请求 ，以及
添加侦听器以查看/存储测试结果

本示例假定LDAP服务器在ldap.test.com上可用。

对于经验不足的LDAP用户，我构建了一个小型LDAP教程 ，该教程简要说明了可用于构建复杂测试计划的几种LDAP操作。

在可分辨名称中使用LDAP特殊字符时要格外小心，在这种情况下（例如，您要在可分辨名称中使用+号），您需要通过在该字符之前添加“ \ ”符号来转义该字符。额外的例外：如果要在专有名称中添加\字符（在添加或重命名操作中），则需要使用4个反斜杠。

例子：

cn = dolf \ + smits
添加/搜索名称如cn = dolf + smits的条目
cn = dolf \\ smits
搜索名称为cn = dolf \ smits的条目
cn = c：\\\\ log.txt
添加名称如cn = c：\ log.txt的条目
8b.1添加用户

您要对每个JMeter测试计划进行的第一步是添加一个线程组元素。线程组告诉JMeter您要模拟的用户数量，用户应该发送请求的频率以及应该发送多少请求。

继续并通过以下方法添加线程组元素：首先选择“ 测试计划” ，单击鼠标右键获得“ 添加”菜单，然后选择“ 添加” →“ 线程（用户）” →“ 线程组” 。现在，您应该在“ 测试计划”下看到“ 线程组”元素。如果没有看到该元素，则通过单击“测试计划”元素来“扩展”测试计划树。


图8b.1。具有默认值的线程组


8b.2添加LDAP扩展请求默认值

首先选择“ LDAP Ext Users”元素。单击鼠标右键获得“ 添加”菜单，然后选择“ 添加” →“ 配置元素” →“ LDAP扩展请求默认值” 。然后，选择此新元素以查看其控制面板。

像大多数JMeter元素一样，“ LDAP扩展请求默认值”控制面板具有您可以修改的名称字段。在此示例中，将此字段保留为默认值。



图8b.2我们的测试计划的LDAP默认值

对于每个不同的操作，可以填写一些默认值。在所有情况下，填写默认值后，它将用于LDAP扩展请求。对于每个请求，您可以通过填写LDAP扩展请求采样器中的值来覆盖缺省值。如果没有输入测试所需的值，则测试将以无法预测的方式失败！


8b.3添加LDAP请求

在我们的测试计划中，我们要使用所有9个LDAP请求。

  1. 线程绑定
  2. 搜索测试
  3. 比较测试
  4. 单次绑定/解除绑定测试
  5. 添加测试
  6. 修改测试
  7. 重命名条目（moddn）
  8. 删除测试
  9. 线程解除绑定

JMeter按照将请求添加到树中的顺序发送请求。

添加请求始终始于：
将LDAP扩展请求添加到LDAP Ext Users元素（ 添加 → 采样器 → LDAP Ext请求 ）。然后，在树中选择LDAP Ext Request元素并编辑以下属性。

8b.3.1添加线程绑定请求

  1. 重命名元素：“ 1。线程绑定 “
  2. 选择“ 线程绑定 ”按钮。
  3. 在“服务器名”字段中输入LDAP服务器中的主机名值
  4. 在端口字段中输入来自LDAP服务器的端口号（ 636 ：基于SSL的ldap）
  5. （可选）在DN字段中输入baseDN，此baseDN将用作搜索，添加，删除等的起点。
    请注意，这必须是所有请求的最高共享级别，例如，当所有信息存储在ou = Users，dc = test，dc = com下时 ，可以在basedn中使用此值。
  6. （可选）输入要用于身份验证的用户的专有名称。当此字段保留为空时，将建立匿名绑定。
  7. （可选）输入要与之进行身份验证的用户的密码，空密码也将导致匿名绑定。
  8. （可选）输入与LDAP的连接超时值
  9. （可选）如果您通过SSL（ldaps）使用LDAP访问，请选中使用安全LDAP协议框。
  10. （可选）如果要客户端信任所有证书，请选中“ TrustAll”框


图8b.3.1。线程绑定示例


8b.3.2添加搜索请求

  1. 重命名元素：“ 2。搜索测试 “
  2. 选择“ 搜索测试 ”按钮。
  3. （可选）输入您要在其下执行搜索的搜索库，相对于在线程绑定请求中使用的basen。
    保留为空时，basedn用作搜索库，如果您要使用“ base-entry”或“ one-level”搜索（请参见下文），此文件非常重要。
  4. 输入searchfilter，任何合适的LDAP搜索过滤器都可以使用，但现在使用简单的名称，例如（sn = Doe）或（cn = *）
  5. （可选）在范围字段中输入范围，它具有三个选项：
    1. 基础对象搜索
      仅使用给定的搜索库，仅用于检查属性或存在性。
    2. 单级搜索
      仅使用给定搜索基础以下一级的搜索
    3. 子树搜索
      在给定基准以下的任何点搜索对象
  6. （可选）大小限制，指定返回的最大条目数，
  7. （可选）时间限制，指定SERVER可用于执行搜索的最大毫秒数。这不是应用程序将等待的最长时间。
    当从非常快的服务器通过非常慢的行返回非常大的返回值时，您可能必须等待一段时间才能完成搜索请求，但是此参数不会影响到此。
  8. （可选）您要在搜索答案中使用的属性。这可以用来限制答案的大小，尤其是当对象具有非常大的属性（例如jpegPhoto ）时。有三种可能性：
    1. 保留为空（默认设置也必须为空）。这将返回所有属性。
    2. 放入一个空值（ “” ），它将请求一个不存在的属性，因此实际上它不返回任何属性
    3. 输入属性，用分号分隔。它将仅返回请求的属性
  9. （可选）返回对象。Checked将返回所有java-object属性，它将这些属性添加到请求的属性中，如上所述。
    如果未选中，则将不返回任何Java对象属性。
  10. （可选）取消引用别名。选中表示将遵循引用，未选中表示不会。
  11. （可选）解析搜索结果吗？选中表示将在响应数据中获取所有结果，未选中表示不会。


图8b.3.2。搜索请求示例


8b.3.3添加比较请求

  1. 重命名元素：“ 3。比较测试 “
  2. 选择“ 比较 ”按钮。
  3. 相对于基础输入要在其上进行比较操作的对象的条目名称，例如“ cn = jdoe，ou = Users ”
  4. 输入比较过滤器，其格式必须为“ 属性=值 ”，例如“ mail=jdoe@test.com ”


图8b.3.3。比较例子


8b.3.4添加单个绑定/取消绑定

  1. 重命名元素：“ 4。单一绑定/解除绑定测试
  2. 选择“ 单一绑定/取消绑定 ”按钮。
  3. 输入您要用于身份验证的用户的全名。
    例如cn = jdoe，ou = Users，dc = test，dc = com当此字段保留为空时，将建立匿名绑定。
  4. 输入您要用于身份验证的用户的密码，空密码也将导致匿名绑定。
注意 ：单个绑定/解除绑定实际上是两个单独的操作，但不能轻易拆分！


图8b.3.4。单一绑定/解除绑定示例


8b.3.5添加添加请求

  1. 重命名元素：“ 5。添加测试 “
  2. 选择“ 添加 ”按钮。
  3. 输入要添加的对象相对于基础的专有名称。
  4. 在“ 添加测试 ”表中添加一行，填写属性和值。
    当您多次需要相同的属性时，只需添加新行，然后再次添加属性和其他值即可。
    必须指定所有必需的属性和值才能通过测试，请参见图片！
    （有时服务器添加属性“ objectClass = top ”，这可能会带来问题。


图8b.3.5。添加请求示例


8b.3.6添加修改请求

  1. 重命名元素：“ 6。修改测试 “
  2. 选择“ 修改测试 ”按钮。
  3. 输入要修改的对象相对于基础的专有名称。
  4. 使用“ 添加 ”按钮在“ 修改测试 ”表中添加一行。
  5. 您需要输入要修改的属性，（可选）值和操作码。该操作码的含义：
    加
    这意味着将把属性值（在这种情况下不是可选的）添加到该属性。
    当属性不存在时，将创建该属性并添加值
    当它存在并且定义为多值时，将添加新值。
    当它存在但单值时，它将失败。
    更换
    这将使用给定的新值覆盖属性（此处不可选）
    当属性不存在时，将创建该属性并添加值
    如果存在，则删除旧值，然后添加新值。
    删除
    如果未指定任何值，则将删除所有值
    给定值时，只会删除该值
    当给定值不存在时，测试将失败
  6. （可选）在“ 修改测试 ”表中添加更多修改。
    指定的所有修改都必须成功，以使修改测试通过。当一个修改失败时，将完全不进行任何修改，并且该条目将保持不变。


图8b.3.6。修改范例


8b.3.7添加重命名请求（moddn）

  1. 重命名元素：“ 7。重命名条目（moddn） “
  2. 选择“ 重命名条目 ”按钮。
  3. 在“ 旧条目名称 ”-字段中输入相对于baseDN的条目名称 。
    也就是说，如果要重命名“ cn = Little John Doe，ou = Users ”，并将baseDN设置为“ dc = test，dc = com ”，则需要输入“ cn = John Junior Doe，ou = Users”在旧的条目名称 -Field中。
  4. 在“ 新专有名称 ”-字段中输入相对于baseDN的条目的新名称 。
    当您仅更改RDN时，它将简单地重命名该条目
    当您还添加其他子树时（例如，从cn = john doe，ou = Users更改为cn = john doe，ou = oldusers） ，它将移动条目。您还可以移动完整的子树（如果您的LDAP服务器支持此功能！），例如ou = Users，ou = retired ， ou = oldusers，ou = users ，这会将完整的子树以及该子树中的所有退休人员移动到树上的新地方。


图8b.3.7。重命名示例


8b.3.8添加删除请求

  1. 重命名元素：“ 8。删除测试 “
  2. 选择“ 删除 ”按钮。
  3. 在Delete -Field中输入相对于baseDN的条目名称。
    也就是说，如果要删除“ cn = John Junior Doe，ou = Users，dc = test，dc = com ”，并且将baseDN设置为“ dc = test，dc = com ”，则需要输入“ cn = “ 删除”字段中的“ John Junior Doe，ou =用户 ”。


图8b.3.8。删除范例


8b.3.9添加取消绑定请求

  1. 重命名元素：“ 9。线程解除绑定 “
  2. 选择“ 线程解除绑定 ”按钮。只要关闭当前连接就足够了。系统已经知道了所需的信息


图8b.3.9。解除绑定的例子



8b.4添加侦听器以查看/存储测试结果

您需要添加到测试计划中的最后一个元素是侦听器。该元素负责将LDAP请求的所有结果存储在文件中，并提供数据的可视模型。选择“线程”组元素并添加一个“ 查看结果树” （“ 添加” →“ 侦听器” →“ 查看结果树” ）


图8b.4。查看结果树侦听器

在此侦听器中，您可以查看三个选项卡，即采样器结果，请求和响应数据。

  1. 采样器结果仅包含响应时间，返回码和返回消息
  2. 该请求提供了对所提出请求的简短描述，实际上，此处不包含任何相关信息。
  3. 响应数据包含已发送请求的完整详细信息以及接收到的答案的完整详细信息（以xml样式（自定义）给出）。 完整的描述可以在这里找到。
