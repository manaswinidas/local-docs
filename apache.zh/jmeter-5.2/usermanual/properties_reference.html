本文档介绍JMeter属性。jmeter.properties或reportgenerator.properties中存在的属性应在user.properties文件中设置。仅在重新启动JMeter后才考虑这些属性，因为通常在加载类时会解析它们。


19.1语言

19.3 SSL配置

SSL（Java）系统属性现在位于system.properties中
JMeter不再转换javax。 jmeter.properties中的 xxx属性条目进入系统属性。现在必须在system.properties文件或命令行中定义它们。system.properties文件提供了更大的灵活性。

参量

属性描述需要
https.sessioncontext.shared 默认情况下，SSL会话上下文现在是按线程创建的，而不是共享的。
可以通过将此属性设置为true来启用旧行为。默认为： false 		没有
https.default.protocol 请注意，https默认协议可能会因JVM版本而异。请参阅诊断TLS，SSL和HTTPS和错误58236 。默认HTTPS协议级别：
https.default.protocol=TLS
这可能需要更改为：
https.default.protocol=SSLv3
没有
https.socket.protocols 要启用的协议列表。如果发现目标服务器有问题，则可能只需要选择一个子集。
如果服务器不支持Socket版本协商， 则需要这样做 ，这可能导致错误： javax.net.ssl。SSLPeerUnverifiedException：对等方未通过身份验证或java.net。SocketException：连接重置 。
请参阅错误54759 ，例如：
https.socket.protocols=SSLv2Hello SSLv3 TLSv1
没有
https.cipherSuites 可以在HTTPS连接中使用的SSL密码套件的逗号分隔列表。可能需要使用密码套件的子集，以匹配预期的客户端行为或在有大量用户运行时减少JMeter中的加密开销。如果JVM不支持指定的密码套件，或者HTTPS服务器支持的密码套件不与该列表重叠，则可能会发生错误。请参阅《 JSSE参考指南》。
例如：

httpclient.reset_state_on_thread_group_iteration 开始新的线程组迭代时，重置HTTP状态。总之，“ true”表示下一次迭代与新用户相关联。 false表示下一次迭代与同一用户相关联。 真实涉及：
  • 关闭打开的连接
  • 重置SSL状态
默认值为： true 		没有
https.use.cached.ssl.context 控制是否允许在迭代之间重用缓存的SSL上下文。
将值设置为false可在每次迭代时重置SSL上下文。
默认值为： true

https.keyStoreStartIndex 与具有许多条目的密钥库一起使用的起始索引。
默认值是使用条目0 ，即第一个。
默认值： 0 		没有

您可以使用完整的类名，如下所示，也可以使用字符串“ System ”或“ CrossPlatform ”之一，这意味着JMeter将使用UIManager.get LookAndFeelClassName（）返回的相应字符串。
LAF可以被os.name覆盖（小写，用'_'代替）。

LAF属性查询的顺序

以Windows 10的os.name为例。

jmeter.laf.windows_10=javax.swing.plaf.metal.MetalLookAndFeel

jmeter.laf.windows=com.sun.java.swing.plaf.windows.WindowsLookAndFeel

通过设置系统LAF，Mac看起来更好看
jmeter.laf.mac=System
失败的话，可以通过以下方法定义JMeter的默认LAF：
jmeter.laf=System
如果没有定义上述jmeter.laf属性，则JMeter使用CrossPlatform LAF。这是因为CrossPlatform LAF通常看起来比System LAF好。有关详细信息，请参见错误52026 。

通过菜单选项>外观从JMeter GUI更改外观（LAF）时，应重新启动JMeter以确保更改完全有效。

19.6 JMX备份配置

参量

属性描述需要
jmeter.gui.action.save.backup_on_save 保存测试计划后，启用.jmx文件的自动备份。
启用后，将在保存.jmx之前将其备份到jmeter.gui.action.save.backup_directory属性指向的目录（请参见下文）。备份文件名是在保存jmx文件后构建的。例如，保存test-plan.jmx将在备份目录中创建test-plan-000012.jmx ，条件是最后创建的备份文件是test-plan-000011.jmx 。
默认值为true，表示已启用自动备份。
默认值为： true 		没有
jmeter.gui.action.save.backup_directory 设置在GUI中保存后将在其中创建JMX备份的备份目录路径。
如果未设置（默认设置），则将在JMeter基本安装的子目录中创建备份文件。如果设置并且目录不存在，将创建一个对应的目录。
默认值为： $ {JMETER_HOME} / backups 		没有
jmeter.gui.action.save.keep_backup_max_hours 设置自保存时间以来应保留备份文件的最长时间（以小时为单位）。
默认情况下，没有设置任何过期时间，这意味着我们将永远保留备份。
默认值： 0		 没有
jmeter.gui.action.save.keep_backup_max_count 设置应保留的最大备份文件数。默认情况下，将保留十个备份。

默认值为： 10 		没有

参量

属性描述需要

  1. 设置server_port = 1234
