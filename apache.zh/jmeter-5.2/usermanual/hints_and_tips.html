22提示和技巧

本节收集了各种提示和技巧，这些提示和技巧由“ JMeter用户”列表上的各种问题提出。如果您在这里找不到所需的内容，请查看JMeter Wiki 。另外，尝试搜索“ JMeter用户”列表；可能有人已经提供了解决方案。

22.1在线程之间传递变量

JMeter变量具有线程作用域。这是有意的，因此线程可以独立运行。但是，有时需要在相同或不同线程组的不同线程之间传递变量。

一种方法是改为使用属性。属性在所有JMeter线程之间共享，因此，如果一个线程设置了属性 ，则另一个线程可以读取更新后的值。

如果线程之间需要传递大量信息，请考虑使用文件。例如，您可以在一个线程中使用“ 将响应保存到文件侦听器或BeanShell PostProcessor”，并使用HTTP Sampler“ file： ”协议读取文件，并使用PostProcessor或BeanShell元素提取信息。

如果您可以在开始测试之前导出数据，那么最好将其存储在文件中，然后使用CSV数据集进行读取。


22.2启用调试日志记录

大多数测试元素包括调试日志记录。如果从GUI运行测试计划，请选择测试元素，然后使用“帮助”菜单启用或禁用日志记录。帮助菜单还具有显示GUI和测试元素类名称的选项。您可以使用它们来确定更改日志记录级别的正确属性设置。

有时，查看日志消息以调试JMeter中使用的动态脚本语言（如BeanShell或Apache Groovy）非常有用。您可以直接在JMeter GUI中查看日志消息，方法是：

  • 使用菜单选项 → 日志查看器 ，日志控制台将出现在界面底部
  • 或单击GUI右上角的“警告”图标
jmeter.properties
jmeter.loggerpanel.display=true
 
jmeter.loggerpanel.maxlength=80000

22.4在Linux或Windows上具有HiDPI屏幕的JMeter

Linux，MacOS或Windows上的Java中的Swing API当前不支持HiDPI模式。因此，使用HiDPI屏幕（小图标和字符）可能很难阅读JMeter。

您可以通过更改一些属性来改善JMeter在HiDPI屏幕上的显示：

jmeter.hidpi.mode
设置为true以激活“ 伪 ” -hidpi模式，从而允许增加某些UI元素的大小
jmeter.hidpi.scale.factor
设置为2.0以缩放某些UI元素的大小
jmeter.toolbar.icons.size
具有以下值： 22x22 （默认大小）， 32x32或48x48 （HiDPI的建议值）
jmeter.tree.icons.size
使用这些值：19×19（默认大小），24×24，32×32（对于HiDPI建议值）或48×48

另外，您可以通过更改以下属性来增加某些元素（如JSR223采样器）中文本区域的字体大小：

jsyntaxtextarea.font.family
设置为Hack以激活并更改字体及其大小
jsyntaxtextarea.font.size
设置为更大的值，例如28 （HiDPI的建议值）

这不是完整的HiDPI支持，只会影响
  • JMeter树节点
  • 工具栏中的图标
  • 表格内容
  • 字体大小到文本区域


22.5自动保存过程配置

从JMeter 3.0开始，JMeter会为每个保存的jmx文件自动最多保存十个备份。启用后，在保存jmx文件之前，它将备份到$ {JMETER_HOME} / backups子文件夹。备份文件以保存的jmx文件命名，并分配了一个自动递增的版本号，例如： test-plan-000001.jmx ， test-plan-000002.jmx ， test-plan-000003.jmx等。 -backup，将以下属性添加到user.properties 。

backup_on_save
要启用/禁用自动备份，请将以下属性设置为true / false （默认为true ）：
jmeter.gui.action.save.backup_on_save=false
backup_directory
备份目录也可以设置在其他位置。将jmeter.gui.action.save.backup_directory属性设置为所需目录的路径将导致备份文件存储在内部，而不是$ {JMETER_HOME} / backups文件夹中。如果指定的目录不存在，将创建该目录。保留此属性不变将导致使用$ {JMETER_HOME} / backups文件夹。 
jmeter.gui.action.save.backup_directory=/path/to/backups/dir
keep_backup_max_hours
您还可以配置自最近保存时间以来应保留备份文件的最长时间（以小时为单位）。默认情况下，将零过期时间设置为零，该时间指示JMeter永久保存备份文件。使用以下属性控制最大保留时间：
jmeter.gui.action.save.keep_backup_max_hours=0
keep_backup_max_count
您可以设置应保留的最大备份文件数。默认情况下，将保留10个备份。将此值设置为零将导致永不删除备份（除非将keep_backup_max_hours设置为非null值） 在选择时间到期后，将处理最大的备份文件选择，因此，即使将一年设置为到期时间，也仅保留keep_backup_max_count个最近的备份文件将被保留。 
jmeter.gui.action.save.keep_backup_max_count=10

22.5使用热键添加元素

当您使用JMeter进行大量脚本编写时，有一种方法可以使用键盘快捷键快速添加元素以测试计划。默认绑定是：

要更改这些绑定，请在jmeter.properties文件中找到“ gui.quick_ * ”属性，例如，建议将其替代内容放入user.properties文件中。


22.6浏览器渲染器未显示在“视图结果”树中

如果您使用的OpenJDK或Oracle Java版本高于8，则会注意到未显示Browser Renderer。这是因为未嵌入JavaFX。为了具有此元素，您需要遵循以下过程。

  • 请按照此文档为您的操作系统和Java版本安装Java FX。
    如果您不想阅读它，请执行以下必要步骤：
    • 转到https://gluonhq.com/products/javafx/并下载适用于您的Java版本和OS的运行时
    • 解压缩
    • 然后配置一个指向lib文件夹的变量：
      Linux / MacOSX：
      export PATH_TO_FX=path/to/javafx-sdk-XX/lib
      视窗：
      set PATH_TO_FX=path/to/javafx-sdk-XX/lib
  • 然后为Linux / MacOSX打开bin / jmeter文件，为Windows打开bin / jmeter.bat，找到JAVA9_OPTS变量并添加：
    Linux / MacOSX：
    --module-path $PATH_TO_FX --add-modules javafx.web,javafx.swing
    视窗：
    --module-path %PATH_TO_FX% --add-modules javafx.web,javafx.swing
