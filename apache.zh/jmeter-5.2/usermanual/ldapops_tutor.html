简短的LDAP操作教程

扩展的LDAP采样器旨在支持非常复杂的测试目的的测试。它旨在尽可能地支持LDAP操作。在这个简短的教程中，我将解释存在哪些LDAP操作以及它们的作用。对于每个操作，我将简要说明如何实现这些操作。
LDAP服务器是某种分层数据库，它们将对象（条目）存储在树中。树的最上层部分称为树的根。
例如。当树以dc = com开头时，根等于dc = com。
下一级别可以存在于根目录下，例如dc = test。该对象的全名（“专有名称”）为“ dc = test，dc = com”。
同样，可以通过在dc = test，dc = com下添加用户“ cn = admin”来达到以下级别。该对象的DN（专有名称）为“ cn = admin，dc = test，dc = com”。
相对专有名称（RDN）是DN的最后一部分，例如。 cn = admin。
对象的特征由objectClasses决定，可以将其视为属性的集合。
对象的类型由“结构对象类”确定，例如人，组织单位或国家。
属性包含对象的数据，例如邮件地址，名称，街道地址等。每个属性可以具有0、1或多个值。

1绑定操作

与LDAP服务器的任何联系都必须以绑定请求开始。 LDAP是一种状态依赖协议。如果不打开到LDAP服务器的会话，就无法发出其他请求。由于JAVA库中的某些特殊性，因此实现了2种不同的绑定操作。

1.1线程绑定

此绑定旨在打开与LDAP服务器的会话。任何测试计划都应将此操作用作会话的起点。对于每个线程（每个虚拟用户），将与LDAP服务器建立单独的连接，因此将执行单独的线程绑定。


1.2单次绑定/解除绑定

此绑定用于用户身份验证。需要经过身份验证的用户的适当开发的LDAP客户端，将使用给定的专有名称和密码执行绑定。此单次绑定/解除绑定操作就是为此目的。它建立自己的与LDAP服务器的单独连接，执行绑定操作，然后再次终止连接（通过发送取消绑定）。



2解除绑定

要关闭与LDAP服务器的连接，需要取消绑定操作。由于Single Bind / Unbind操作已经（隐式地）执行了取消绑定操作，因此仅需要执行Thread Unbind操作。该线程解除绑定只会关闭连接并清除其已使用的所有资源。


3比较

比较操作需要LDAP对象的完整可分辨名称，以及一个属性和该属性的值。它将简单地检查：“此对象确实具有此值的属性吗？”。典型用途是检查特定用户在给定组中的成员资格。


4搜索

搜索测试仅搜索符合给定搜索过滤器的所有对象，例如，所有具有“ employeeType = inactive”的人员或“所有具有userID等于user1的人员”


5添加

这只是将一个对象添加到LDAP目录中。当然，属性和distinguishedName的组合必须有效！


6修改

此操作会修改给定对象的一个或多个属性。它需要对象的可分辨名称，以及该属性的属性和新值。
添加可用三个版本，用于添加属性值
替换，用于用新值覆盖旧属性值
删除，从属性中删除值或删除属性中的所有值


7删除

该操作从LDAP服务器删除对象。它需要对象的专有名称。


8个modDN

此操作修改对象的专有名称（“移动”该对象）。
它有两种形式，只是重命名一个条目，然后指定一个新的RDN（相对专有名称，这是DN的最低部分）
例如，您可以将“ cn = admin，dc = test，dc = com”重命名为cn = administrator，dc = test，dc = com”
第二种方式是通过指定“新上级”来重命名（移动）完整的子树
例如，您可以通过指定新的rdn“ ou =退休人员”和“ dc = test，dc = com”的新上级
