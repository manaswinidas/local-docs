24.2支持的卷曲选项

-H，--header

：获取网页时使用的额外标题。

-X，-请求 ：指定与HTTP服务器通信时要使用的自定义请求方法。

--compressed ：使用curl支持的算法之一请求压缩响应，然后返回未压缩的文档。

-A，--user-agent ：指定用户代理字符串以发送到HTTP服务器。

-b，-cookie ：将数据作为cookie传递到HTTP服务器。

将POST请求中的指定数据发送到HTTP服务器。如果在同一命令行上多次使用此选项，则指定的数据段将与一个单独的“＆”字符合并。因此，使用'-d name = daniel -d skill = lousy'将生成POST块，看起来像'name = daniel＆skill = lousy'。
  • -d， -data --data-ascii ：使用@上传文件
  • -数据二进制
  • --data-raw ：这将完全按照指定的方式发布数据，而无需任何额外处理。如果以字符@开头数据，则其余部分应为文件名。
  • --data-urlencode ：与其他--data选项类似，此方法发布数据，但执行URL编码。
  • --data-raw ：这类似于--data发布数据，但不对@字符进行特殊解释。
这使curl可以模拟用户按下提交按钮的填充表单。
  • -F，--form ：使用@上传文件
  • --form-string

-u，--user ：指定用于服务器身份验证的用户和密码。

--basic，-digest ：告诉curl使用HTTP认证。

告诉curl在使用HTTPS获取文件时使用指定的客户端证书文件
  • -证书
  • --capath
  • -密码
  • --cert-status
  • --cert-type

-G，-- get：将发布数据放入url中，并使用get替换发布。

--no-keepalive ：在TCP连接上禁用keepalive消息的使用。

-e，--referer ：将“引用页面”信息发送到HTTP服务器。

-L，--location ：如果服务器报告所请求的页面已移动到其他位置，则此选项将使curl重新在新位置上重新请求。

-i， --include：在输出中包含HTTP标头。

-连接超时 ：连接服务器可能花费的最长时间（以秒为单位）。

--keepalive-time ：此选项设置在发送keepalive探测之前连接需要保持空闲的时间以及各个keepalive探测之间的时间。

-m，--max-time ：您允许整个操作进行的最长时间（以秒为单位）。

-x，--proxy <[protocol：//] [user：password @] proxyhost [：port]> ：使用指定的HTTP代理。如果未指定端口号，则假定为端口1080。

-U，--proxy-user ：指定用于代理身份验证的用户和密码。

-k，--insecure ：此选项显式允许curl执行“不安全”的SSL连接和传输。

--raw ：使用时，它将禁用内容的所有内部HTTP解码或传输编码，而是使它们以未更改的原始形式传递。

-I，--head ：仅获取HTTP标头。HTTP服务器具有命令HEAD，该命令用于获取文档的标头。

- 接口 ：使用指定的界面执行操作。您可以输入接口名称，IP地址或主机名。

--proxy-ntlm /-proxy-negotiate ：告诉curl与指定代理进行通信时使用HTTP基本/ ntlm /摘要身份验证。

--dns-服务器 ：通过DOH解析主机名。

- 解决 ：为特定的主机和端口对提供自定义地址。

-极限速率 ：指定您要卷曲使用的最大传输速率。

--max-redirs ：设置可以遵循的最大重定向数。

--noproxy ：以逗号分隔的不使用代理的主机列表（如果已指定）。

24.4例子

使用cookie： curl -X POST“ https://xxx.com” -b'username = Tom; password = 123456'

使用数据： curl -X POST“ https://xxx.com” --data'fname = a＆lname = b'

使用形式： curl -X POST“ https://xxx.com” -F'lname = a'-F'fname = b'-F'c = @ C：\ Test \ test.txt'

使用代理： curl'http://jmeter.apache.org/'-x'https：// aa：bb@example.com：8042'

使用自动翻译： curl“ https://xxx.com” -u'user：passwd'--basic

使用DNS： curl“ https://xxx.com” --dns-servers'0.0.0.0,1.1.1.1'
