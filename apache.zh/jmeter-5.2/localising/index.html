介绍

本文档介绍了用英语以外的其他语言为JMeter创建和维护翻译文本的过程。尽管默认英语显然不足以合理地明确地进行交流，但还是默认使用英语作为该项目的主要（或“默认”）语言，以向帝国的力量致敬：-)
因此，都市语言文本由软件开发人员维护，而其他项目提供者（在本文档中称为“翻译”）负责维护各省语言的文本。生成和维护后者的过程在本文档中称为“翻译”。

本文档假定您将使用i18nEdit作为编辑属性文件的工具，并且说明仅针对此软件，但这不是强制性的：如果您更喜欢使用其他工具（例如或vi） ，则该过程也应该可以正常工作或Emacs 。

本文档描述了6个过程：

  1. 获取当前文本[翻译]。
  2. 将当前文本提供给翻译人员[开发人员]。
  3. 下载并运行i18nEdit [所有人]。
  4. 翻译[translators]。
  5. 将您的翻译提交给项目[翻译者]。
  6. 合并新的翻译[committers]。


获取当前文本

如果您想帮助JMeter的翻译过程，请先阅读本文档。然后发送邮件至dev@jmeter.apache.org，说明您的意图。您需要的文件（* .properties和* .metaprop）包含在源归档中。但是，如果您遇到任何困难，项目贡献者之一将可以从SVN中获取当前文本并将其发送给您。您将收到需要解压缩到本地磁盘中的jar，zip，tar或tgz文件。

如果您熟悉SVN或很勇敢，请随意匿名连接到Apache SVN服务器并自己获取JMeter源代码，如http://jmeter.apache.org/svnindex.html中所述 -这些文件对于转换过程都在jmeter / src目录下。

解压缩文件或检出文件后，请确保在其中找到文件src / i18nedit.properties ：您需要知道从哪里开始使用i18nEdit。


提供当前文本给翻译人员

如果您有权访问JMeter的SVN存储库，并且想打包本地化所需的文件以发送到翻译器，则只需转到项目根目录上方的目录并发出以下命令：

tar czf jmeter-localisation.tgz `find jmeter/src -name "*.properties" -o -name "*.metaprops"`

当然，您也可以将整个jmeter目录发送给翻译器，但这会使他们的生活更轻松。


下载并运行i18nEdit

可以从https://sourceforge.net/projects/i18nedit/获得i18nEdit的运行时。下载二进制发行版（i18nedit-1.2.2.jar）并将其保存在本地。

要运行i18nEdit，只需确保PATH中有一个相当现代的Java运行时环境，然后转到保存i18nedit-1.2.2.jar的目录，然后发出以下命令：

java -jar i18nedit-1.2.2.jar
  1. 如果您以前从未运行过i18nEdit，请选择一种语言。本文档的其余部分假定您选择了UK English。
  2. 选择“ 项目 ”菜单，然后选择“ 打开项目… ”。
  3. 导航到jmeter / src / ，选择i18nedit.properties ，然后按“ 打开 ”按钮。
  4. 在打开的窗口中，选择“ 项目 ”菜单，然后选择“ 项目设置 ”。检查您的目标语言是否出现在“ 其他语言环境（ISO代码） ”字段的列表中。否则，请立即添加。按“ 保存 ”。


正在翻译

开始翻译之前，请选择“ 项目 ”菜单，然后选择“ 翻译设置 ”。选择工作模式“ 定向翻译（从源到目标） ”。在“ 源本地化 ”字段中输入“ en ”（不带引号）。在“ 目标本地化字段 ”中输入目标语言的ISO代码。

单击右侧面板中的可编辑字段之一（您的语言为“ Comment ”或“ Content ”）。按F2 。i18nEdit会将您带到需要引起您注意的第一个属性，这可能是因为它尚不存在翻译，或者因为自提供翻译以来英语文本已更改。如有必要，输入或修正文本，然后再次按F2键重复该过程。

i18nEdit的在线帮助非常有用：请仔细阅读以获取更多信息和提示。


将翻译提交到项目

翻译完成后，只需将jmeter / src中的整个文件打包在jar，zip，tar，tgz或类似文件中，然后将它们附加到JMeter错误报告中（请链接到JMeter主页中的 “已知错误”链接） 页面 ）。


合并新的翻译
  1. 将翻译器提交的文件解压缩到一个单独的目录中。
  2. 按照上面的下载和运行i18nEdit中所述启动i18nEdit。
  3. 如果翻译员使用的是新语言，请确保将其列在“项目设置”的“其他语言环境”字段中。
  4. 打开“ 团队 ”菜单，然后选择“将更改合并为集成商 ”。
  5. 在转换器提交的文件中输入src目录的路径。
  6. 选择翻译者的目标语言。
  7. 按“ 执行合并 ”。
  8. 关闭i18nEdit并照常提交到SVN（如果使用Eclipse，请记住要刷新项目）。
