建筑JMeter
  • Java 8兼容的JDK（Java开发套件）
  • 可选： Gradle安装
  • JMeter的源代码如下一节所示

获取源

可以从下载页面下载Apache JMeter的官方源代码发行版。

使用Gradle编译和打包JMeter

JMeter可以完全使用Gradle构建。基本命令是：

./gradlew build

请参阅通过./gradlew任务 （或./gradlew任务--all ）可用任务的列表，以了解其他可以使用的任务。有关可用任务的更多详细信息，可以在gradle.md中找到。

通过IntelliJ IDEA打开项目

您需要IntelliJ 2018.3.1或更高版本。

  • 使用IntelliJ IDEA打开build.gradle.kts文件，然后选择“以项目打开”
  • 确保选中“为每个源集创建单独的模块”
  • 确保已选择“使用默认的gradle包装器”
  • 在“文件已存在”对话框中，选择“是”以覆盖
  • 在“打开项目”对话框中，选择“删除现有项目并导入”

使用Eclipse编译和打包JMeter

选项1：通过Eclipse的“导入Gradle项目”向导导入Eclipse项目

文件 → 导入... 现有的Gradle项目

选项2：使用Gradle任务设置Eclipse项目

./gradlew eclipse
文件 → 导入 → 现有项目到工作区


为JMeter做贡献

我们爱贡献

如果您花一些时间为该项目做出贡献，我们将非常感谢您。如果您有时间花在项目上，则可以从“ 问题”页面中选择现有的增强功能或错误。
您也可以为翻译做出贡献，请参阅《 JMeter本地化》（《翻译指南》） 。

提交补丁

如果您想为JMeter进行错误修复或增强做出贡献，请遵循以下步骤：

检查你的补丁

在提交补丁之前，请确保您执行以下操作：
通过运行以下命令检查补丁是否已编译并遵循选项卡空间策略：

./gradlew check

通过运行以下命令检查补丁程序是否不会破坏JUnit测试：

./gradlew test

使用GIT创建PR

  • 打开bugzilla问题，请参见问题页面
  • 前叉Apache JMeter镜像
  • 在本地克隆您的分叉存储库：
    git clone https://github.com/yourid/jmeter/
  • 使用例如bugzilla id创建分支：
    git branch BUGID
  • 签出新分支：
    git checkout BUGID
  • 在此提交您的修复程序：
    git commit -m 'Fix to BUGID' list of files
  • 请避免在PR中创建合并提交。我们接受小改动，合并提交更难审核
  • 将JMeter存储库添加为上游：
    git remote add upstream http://www.github.com/apache/jmeter
  • 推它 ：
    git push origin BUGID
  • 创建一个PR，并在bugzilla问题中将其链接

不同的操作系统对于行尾标记具有不同的默认值。典型配置是Windows的CRLF和macOS和GNU / Linux的LF。
建议通过core.autocrlf的适当设置来遵循该配置。对于Windows
git config --global core.autocrlf true
，并且适用于macOS和GNU / Linux设置
git config --global core.autocrlf input
由于.gitattributes ，Git将自动识别存储库中的文本文件，并且Git会将文本文件的行尾转换为适当的平台本机格式（根据core.autocrlf ）
无论开发人员工作站的配置如何，某些文件（例如* .sh或* .bat ）都具有预定义的行尾策略。

使用补丁创建PR

  • 打开bugzilla问题，请参见问题页面
  • 签出Apache JMeter源代码
  • 编码您的修复
  • 通过右键单击Eclipse项目来创建补丁，然后选择Team → Create Patch…。
  • 将补丁附加到bugzilla问题


自动化构建

自动化（每晚）构建

作为开发过程的一部分，JMeter项目可以访问各种持续集成（CI）服务器版本。构建输出对于测试代码库的最新更改很有用。

请注意，这些构建没有经过任何质量检查，只能用于开发测试。有关更多信息，请参阅开发人员的夜间构建页面。


建筑附加组件

建筑附加组件

如果您只想构建附加组件，则无需构建JMeter。只需下载二进制归档文件，然后将jar添加到类路径中，或使用Maven工件来构建您的附件。您可能还需要下载源代码，以便IDE可以使用它。

请参阅源代码树中的extras / addons *文件以获取一些建议
