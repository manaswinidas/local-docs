问题追踪器

JMeter使用Bugzilla进行问题跟踪，即报告错误和请求增强功能。

在创建新问题之前，请通过搜索Bugzilla检查问题是否已经报告。还值得首先在JMeter用户邮件列表中进行检查；其他人可能已经有解决方案。


支持问题

请不要使用Bugzilla提出问题。这不是一个支持论坛。相反，请订阅 JMeter用户邮件列表并在此处询问。用户邮件列表具有更大的受众群体，您更有可能迅速获得答案。


已知错误和增强功能
  • Bugzilla-所有打开的错误和增强功能
  • Bugzilla-未解决的错误（增强功能除外）
  • Bugzilla-仅增强功能


要求增强

在大多数情况下，值得首先在邮件列表上进行讨论。Bugzilla可以很好地跟踪进度并提供补丁，但是对于较长的讨论却不方便。

如果尚未这样做，则需要首先使用Bugzilla主页的顶部的“ 新帐户 ”链接注册一个帐户： https : //bz.apache.org/bugzilla/ 。

在注册之前，请确保您已阅读并理解帐户创建页面上的信息。

登录后，单击“提交错误 ”，然后从列表中选择JMeter。请将严重性设置为“ 增强 ”。

请确保您详细描述了增强功能。如有必要，请提供示例用例。

如果要提供代码补丁，还请提供测试用例和有关如何使用新功能的文档（最好是作为文档补丁）。


提出问题

首先检查该问题尚未报告。如果报告错误，您确定它确实是JMeter中的错误，而不仅仅是对JMeter工作方式的误解吗？

如果尚未这样做，则需要首先使用Bugzilla主页的顶部的“ 新帐户 ”链接注册一个帐户： https : //bz.apache.org/bugzilla/ 。

在注册之前，请确保您已阅读并理解帐户创建页面上的信息。

登录后，您有2个选择：

  • 在屏幕左下方，单击“ 在“ JMeter”产品中提交新的错误 ”链接
  • 在屏幕的左上方，单击“ 新建 ”链接，然后从列表中选择JMeter。


错误报告所需的信息

请确保您提供了足够的信息，以供其他人有效地使用该报告。使用下面的清单来指导您。

  • JMeter版本
  • Java版本（从java -version输出）
  • 作业系统版本
  • jmeter.log文件（不太可能包含敏感信息，但请在上传前进行检查）
  • JMX文件（如果有的话）（首先编辑所有敏感信息），提供简化的测试计划（使用Debug Sampler ）将确保BUG的修复比没有它时更快。
  • JTL文件（如果相关）（可能需要编辑敏感信息）
  • 对于可疑的错误，请描述您做了什么，发生了什么以及这与您预期发生的情况有何不同。每次都会发生吗？
  • 将您自己添加到抄送列表中，以在JMeter团队需要更多信息时得到通知（在这种情况下，错误将标记为NEEDINFO ）
  • 当某个错误的市场名为NEEDINFO时 ，请尽快提供所需的信息，以便可以理解并修复该错误。请注意，如果在团队需要更多信息之后没有提供任何信息，并且错误无法再现，则错误将以WORKSFORME的形式关闭。提供所需信息后，您随时可以重新打开它。
  • 准确选择重要级别，“ 增强”表示不是错误，而其他则表示是错误
  • 如果您要提供补丁来修复错误，请确保该补丁采用统一的diff格式。如果使用Eclipse，请将补丁程序根目录设置为“ Project ”，而不是难以应用的默认“ Workspace ”。
  • 可以按原样提供新的源文件。请确保它们具有标准的Apache许可标头（与其他JMeter文件相同）。请不要使用@author标记（更改文件中将提供积分）。
  • 对于新功能的补丁程序，请尽可能提供文档补丁程序。组件记录在xdocs / usermanual / component_reference.xml中 。

另请参阅以下Bug编写准则 ，以及Bugzilla帐户创建页面上注明的条款和条件。
