不兼容的更改

  • HTTP（S）测试脚本记录器现在将数字添加到名称的末尾，而以前它是将其添加到开头。见错误63450
  • 当将XPath断言与返回布尔值的XPath表达式一起使用时， 如果没有匹配项无效则返回True，并且始终返回true，请参见Bug 63455
  • XML解析现在拒绝不安全的XML，这会影响以下功能：
    • XMLAssertion
    • XMLSchemAssertion
    • XPath功能
    • XPath 1和2提取器
    • XPath 1和2断言

已知问题和解决方法

  • “一次”控制器在线程组或循环控制器下的行为正确，但否则其行为不一致（或明确指定）。
  • 绿色框左侧显示的数字是活动线程数/线程总数，线程总数仅适用于本地运行的测试，否则将显示0 （请参见Bug 55510 ）。
  • 请注意，在某些Windows系统下，您可能会遇到以下警告：
    java.util.prefs.WindowsPreferences
WARNING: Could not open/create prefs root node Software\JavaSoft\Prefs at root 0
x80000002. Windows RegCreateKeyEx(…) returned error code 5.
    解决方法是以管理员身份运行JMeter，它将为您创建注册表项，然后您可以以普通用户身份重新启动JMeter，并且不再有警告。
  • 您可能会遇到以下错误：
    java.security.cert.CertificateException: Certificates does not conform to algorithm constraints
    如果您使用SSL证书（本身或信任链中的一个SSL证书），使用MD2的签名算法（例如md2WithRSAEncryption ）或大小小于1024位的SSL证书在网站上运行HTTPS请求。此错误与Java 8+中提高的安全性有关。
    为了允许您执行HTTPS请求，您可以通过编辑Java jdk.certpath.disabledAlgorithms属性来降低Java安装的安全性。根据具体情况，删除MD2值或大小限制。
    此属性在此文件中：
    JAVA_HOME/jre/lib/security/java.security
    有关详细信息，请参见错误56357 。
  • 在Mac OSX下，由于对数字的镜像作用，“聚合图”将显示错误的值。这是由于已知的Java错误引起的，请参见Bug JDK-8065373。此修补程序是使用JDK8_u45或更高版本。
  • 查看结果树可能无法在HTML渲染器下显示一些HTML代码，请参见Bug 54586 。这是由于已知的Java错误导致无法解析行/列属性中的“ px ”单位。请参见Bug JDK-8031109。此修补程序是使用JDK9 b65或更高版本。
  • 使用Mac OSX上的Java 7完全无法通过键盘（ SHIFT + 上/下 ）进行JTable选择。这是由于已知的Java错误JDK-8025126所致，该修复程序是使用JDK 8 b132或更高版本。
  • 从Java 11 开始，不赞成使用 JavaScript实现Nashorn 。如果您使用的是基于Nashorn的JavaScript，则Java将发出以下弃用警告。 
    Warning: Nashorn engine is planned to be removed from a future JDK release
    要使这些警告静音，请在Java参数中添加-Dnashorn.args =-no-deprecation-warning 。可以通过设置环境变量JVM_ARGS来实现 
    export JVM_ARGS="-Dnashorn.args=--no-deprecation-warning"
