创建功能

描述

的CREATE FUNCTION语句用于在Spark中创建临时或永久函数。临时功能的作用域是会话级别，其中在持久目录中创建了永久功能并将其提供给所有会话。在指定的资源USING子句在首次执行时可用于所有执行者。除SQL接口外，spark允许用户使用Scala，Python和Java API创建自定义的用户定义的标量和聚合函数。有关更多信息，请参考scalar_functions和聚合函数 。

句法

参量

OR REPLACE
如果指定，将重新加载该功能的资源。这对于拾取对该函数的实现所做的任何更改主要有用。此参数与以下项互斥IF NOT EXISTS并且不能一起指定。
TEMPORARY
指示要创建的功能范围。什么时候TEMPORARY如果指定，则创建的函数有效，并且在当前会话中可见。对于此类功能，目录中没有进行任何永久性输入。
IF NOT EXISTS
如果指定，则仅在不存在时创建函数。如果指定的功能已经存在于系统中，则功能创建成功（不引发错误）。此参数与以下项互斥OR REPLACE并且不能一起指定。
function_name
指定要创建的功能的名称。函数名称可以可选地用数据库名称限定。

句法： [database_name.]function_name
class_name
指定为要创建的函数提供实现的类的名称。实现类应扩展以下基类之一：
  • 应该延长UDF要么UDAF在org.apache.hadoop.hive.ql.exec包。
  • 应该延长AbstractGenericUDAFResolver ， GenericUDF ， 要么GenericUDTF在org.apache.hadoop.hive.ql.udf.generic包。
  • 应该延长UserDefinedAggregateFunction在org.apache.spark.sql.expressions包。
resource_locations
指定包含功能实现及其依赖关系的资源列表。

句法： USING { { (JAR | FILE ) resource_uri} , ...}
