迁移指南：PySpark（Spark上的Python）

  • 从PySpark 2.4升级到3.0
  • 从PySpark 2.3升级到2.4
  • 从PySpark 2.3.0升级到2.3.1及更高版本
  • 从PySpark 2.2升级到2.3
  • 从PySpark 1.4升级到1.5
  • 从PySpark 1.0-1.2升级到1.3

请注意，此迁移指南介绍了PySpark特定的项目。将PySpark迁移到更高版本时，可以应用许多SQL迁移项目。请参阅《 迁移指南：SQL，数据集和数据框架》 。

从PySpark 2.4升级到3.0

  • 从Spark 3.0开始，PySpark需要使用0.23.2或更高版本的Pandas版本才能使用与Pandas相关的功能，例如toPandas ， createDataFrame来自Pandas DataFrame等

  • 从Spark 3.0开始，PySpark需要PyArrow版本为0.12.1或更高版本才能使用与PyArrow相关的功能，例如pandas_udf ， toPandas和createDataFrame带有“ spark.sql.execution.arrow.enabled = true”等。

  • 在PySpark中，当创建一个SparkSession与SparkSession.builder.getOrCreate() ，如果存在SparkContext ，构建者正在尝试更新SparkConf现有的SparkContext具有为建造者指定的配置，但是SparkContext被所有人共享SparkSession s，因此我们不应该更新它们。从3.0开始，构建器开始不更新配置。这与2.3及更高版本中的Java / Scala API相同。如果要更新它们，则需要先更新它们，然后再创建SparkSession 。

  • 在PySpark中，启用箭头优化后，如果箭头版本高于0.11.0，则箭头在转换熊猫时可以执行安全的类型转换。在序列化期间将序列化为Arrow数组。当检测到不安全的类型转换（如溢出）时，箭头将引发错误。设置spark.sql.execution.pandas.arrowSafeTypeConversion为true可以启用它。默认设置为false。下表说明了PySpark对于Arrow版本的行为：
    PyArrow版本 整数溢出 浮点截断
    版本<0.11.0 引发错误 默默地允许
    版本> 0.11.0，arrowSafeTypeConversion = false 无声溢出 默默地允许
    版本> 0.11.0，arrowSafeTypeConversion = true 引发错误 引发错误
  • 自Spark 3.0起， createDataFrame(..., verifySchema=True)验证LongType以及在PySpark中。先前， LongType未验证并导致None如果值溢出。为了恢复这种行为， verifySchema可以设置为False禁用验证。

从PySpark 2.3升级到2.4

  • 在PySpark中，启用箭头优化后，以前toPandas当无法使用Arrow优化时失败，而createDataFrame来自Pandas DataFrame的支持允许回退到非优化。现在，两者toPandas和createDataFrame来自Pandas DataFrame的默认情况下允许后备，可以通过以下方式将其关闭spark.sql.execution.arrow.fallback.enabled 。

从PySpark 2.3.0升级到2.3.1及更高版本

  • 从2.3.1版开始，箭头功能包括pandas_udf和toPandas() / createDataFrame()与spark.sql.execution.arrow.enabled调成True ，已标记为实验性。这些仍在发展中，目前不建议在生产中使用。

从PySpark 2.2升级到2.3

  • 在PySpark中，如果要使用与Pandas相关的功能，例如Pandas 0.19.2或更高版本， toPandas ， createDataFrame来自Pandas DataFrame等

  • 在PySpark中，已将Pandas相关功能的时间戳值行为更改为尊重会话时区。如果要使用旧的行为，则需要设置配置spark.sql.execution.pandas.respectSessionTimeZone至False 。有关详细信息，请参见SPARK-22395 。

  • 在PySpark中， na.fill()要么fillna还接受布尔值，并用布尔值替换空值。在以前的Spark版本中，PySpark只会忽略它并返回原始的Dataset / DataFrame。

  • 在PySpark中， df.replace不允许省略value什么时候to_replace不是字典。先前， value在其他情况下可以省略，并具有None默认情况下，这违反直觉且容易出错。

从PySpark 1.4升级到1.5

  • Python中将字符串解析为列现在支持使用点（ . ）以限定列或访问嵌套值。例如df['table.column.nestedField'] 。但是，这意味着，如果您的列名包含任何点，那么您现在必须使用反引号将它们转义（例如， table.`column.with.dots`.nested ）。

  • PySpark中的DataFrame.withColumn方法支持添加新列或替换具有相同名称的现有列。

Python数据类型不再单例
