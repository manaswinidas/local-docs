GroovyDoc是负责从您的代码生成文档的工具。它的行为类似于Java世界中的Javadoc工具，但能够处理groovy和java文件。该发行版附带两种生成文档的方式： 从命令行或从Apache Ant 。其他构建工具（例如Maven或Gradle）也提供Groovydoc的包装器。

2。groovydoc Ant任务

的groovydoc Ant任务允许从Ant构建生成groovydocs。

2.1。必需的taskdef

假设您需要的所有普通jar都在my.classpath中 （这将是groovy-VERSION.jar ， groovy-ant-VERSION.jar ， groovy-groovydoc-VERSION.jar以及您可能正在使用的任何模块和传递依赖项），您需要在调用groovydoc任务之前在build.xml中的某个位置声明此任务。

<taskdef name         = "groovydoc"
         classname    = "org.codehaus.groovy.ant.Groovydoc"
         classpathref = "my.classpath"/>

2.4。自定义模板

的groovydoc Ant任务支持自定义模板，但是需要两个步骤：

  1. 定制的groovydoc类

  2. 一个新的groovydoc任务定义

2.4.1。自定义Groovydoc类

第一步需要您扩展Groovydoc类，如以下示例所示：

package org.codehaus.groovy.tools.groovydoc;

import org.codehaus.groovy.ant.Groovydoc;

/**
 * Overrides GroovyDoc's default class template - for testing purpose only.
 */
public class CustomGroovyDoc extends Groovydoc {

    @Override
    protected String[] getClassTemplates() {
        return new String[]{"org/codehaus/groovy/tools/groovydoc/testfiles/classDocName.html"};
    }
}

您可以覆盖以下方法：

  • getClassTemplates用于类级模板

  • getPackageTemplates用于包级模板

  • getDocTemplates用于顶层模板

您可以在以下位置找到默认模板的列表org.codehaus.groovy.tools.groovydoc.gstringTemplates.GroovyDocTemplateInfo类。

2.4.2。使用自定义groovydoc任务

编写完课程后，使用它只是重新定义groovydoc任务：

<taskdef name           = "groovydoc"
         classname      = "org.codehaus.groovy.ant.CustomGroovyDoc"
         classpathref   = "path_to_groovy_all"/>

请注意，模板自定义按原样提供。API可能会更改，因此您必须将此视为易碎功能。
