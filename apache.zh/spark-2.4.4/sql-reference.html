资料类型

Spark SQL和DataFrames支持以下数据类型：

  • 数值类型
    • ByteType ：表示1个字节的有符号整数。数字范围是从-128至127 。
    • ShortType ：表示2字节有符号整数。数字范围是从-32768至32767 。
    • IntegerType ：表示4字节有符号整数。数字范围是从-2147483648至2147483647 。
    • LongType ：表示8字节有符号整数。数字范围是从-9223372036854775808至9223372036854775807 。
    • FloatType ：表示4字节单精度浮点数。
    • DoubleType ：表示8字节的双精度浮点数。
    • DecimalType ：表示任意精度的带符号十进制数字。内部支持java.math.BigDecimal 。一种BigDecimal由任意精度的整数无标度值和32位整数标度组成。
  • 字符串类型
    • StringType ：表示字符串值。
  • 二进制类型
    • BinaryType ：表示字节序列值。
  • 布尔型
    • BooleanType ：表示布尔值。
  • 日期时间类型
    • TimestampType ：代表包含年，月，日，小时，分钟和秒的字段值的值。
    • DateType ：代表包含年，月，日字段值的值。
  • 复杂类型
    • ArrayType(elementType, containsNull) ：代表由一系列元素组成的值，其类型为elementType 。 containsNull用于指示元素中是否包含ArrayType价值可以有null价值观。
    • MapType(keyType, valueType, valueContainsNull) ：表示包含一组键值对的值。键的数据类型由keyType值的数据类型由valueType 。为一个MapType值，不允许有键null价值观。valueContainsNull用于指示a的值MapType价值可以有null价值观。
    • StructType(fields) ：代表值，其结构由一系列StructField s（ fields ）。
      • StructField(name, dataType, nullable) ：代表一个字段StructType 。字段名称由表示name 。字段的数据类型由表示dataType 。 nullable用于指示此字段的值是否可以具有null价值观。

包中包含Spark SQL的所有数据类型org.apache.spark.sql.types 。您可以通过以下方式访问它们

import org.apache.spark.sql.types._
在Spark存储库中的“ examples / src / main / scala / org / apache / spark / examples / sql / SparkSQLExample.scala”中找到完整的示例代码。

NaN语义学

处理非数字（NaN）时要特别处理float要么double与标准浮点语义不完全匹配的类型。特别：

  • NaN = NaN返回true。
  • 在聚合中，所有NaN值都分组在一起。
  • NaN在连接键中被视为普通值。
  • NaN值按升序排在最后，大于任何其他数值。

算术运算

对数字类型执行的操作（除decimal ）不会检查是否溢出。这意味着，如果某个操作导致溢出，则结果与Java / Scala程序中相同的操作返回的结果相同（例如，如果2个整数的总和大于可表示的最大值，则结果为负数） ）。
