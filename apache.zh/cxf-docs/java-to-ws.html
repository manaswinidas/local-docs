Java到WS

名称

java2ws -使用Web服务端点的实现（SEI）类和关联的类型类来生成WSDL文件，包装bean，服务器端代码以启动Web服务和客户端代码。

概要

   java2ws -databinding <jaxb or aegis> -frontend <jaxws or simple> 
           -wsdl -wrapperbean -client -server -ant -o <output-file>
           -d <resource-directory> -classdir <compile-classes-directory> 
           -cp <class-path> -soap12 -t <target-namespace> 
           -beans <ppathname of the bean definition file>* 
           -address <port-address> -servicename <service-name> 
           -portname <port-name> -createxsdimports -h -v -verbose 
           -quiet {classname}

描述

java2ws使用Web服务端点的实现（SEI）类和关联的类型类来生成WSDL文件，包装bean，服务器端代码以启动Web服务和客户端代码。

选件

下表介绍了用于管理代码生成过程的选项。

选项

解释

-? ， -h ， -help

显示此实用程序的联机帮助并退出。

-o

指定生成的WSDL文件的名称。

-数据绑定

指定数据绑定（宙斯盾或jaxb）。对于jaxws前端，默认值为jaxb；对于简单前端，默认值为aegis。

-前端

指定要使用的前端。支持jaxws和简单的前端。

-wsdl

指定生成WSDL文件。

-wrapperbean

指定生成包装器和故障Bean

-客户

指定生成客户端代码

-服务器

指定生成服务器端代码

-蚂蚁

指定生成Ant build.xml脚本

-cp

指定目录和zip / jar文件的SEI和类型类搜索路径。

-soap12

指定生成的WSDL将包括SOAP 1.2绑定。

-t

指定要在生成的WSDL文件中使用的目标名称空间。

-服务名称

指定生成的服务元素的name属性的值。

-v

显示工具的版本号。

-冗长

在代码生成过程中显示注释。

-安静

在代码生成过程中禁止注释。

-s

生成的源文件（包装器bean，故障bean，客户端或服务器端代码）所在的目录。

-classdir

生成的源编译到的目录。如果未指定，则不编译文件。

-端口名

指定要在生成的wsdl中使用的端口名。

-地址

指定端口地址。

-豆子

指定定义其他Spring Bean的文件的路径名，以自定义数据绑定配置。

-createxsdimports

输出模式可分隔文件，并使用导入来加载它们，而不是将它们内联到wsdl中。

-d

资源文件所在的目录，wsdl文件将默认放置在该目录中

班级名称

指定SEI类的名称。

您必须包括classname论点。所有其他参数都是可选的，可以按任何顺序列出。该工具将搜索并加载服务端点类和类型类。确保这些课程在CLASSPATH或通过-cp旗。如果未指定任何“ -wsdl，-wrapperbean，-client和-server”标志，则java2ws将不生成任何内容。

例子

java2ws -wsdl -d ./resource org.apache.hello.Greeter
java2ws -cp ./tmp org.apache.hello.Greeter -wsdl
java2ws -o hello.wsdl -wsdl org.apache.hello.Greeter
java2ws -client -server -s ./src org.apache.hello.Greeter
java2ws -wrapperbean -classdir ./classes org.apache.hello.Greeter

在Ant中使用java2ws

可以将java2ws命令包装在Ant目标中，如下所示：

<?xml version="1.0"?>
<project name="cxf java2ws" basedir=".">   
   <property name="cxf.home" location ="/usr/myapps/cxf-trunk"/>
   <property name="build.classes.dir" location ="${basedir}/build/classes"/>

   <path id="cxf.classpath">
      <pathelement location="${build.classes.dir}"/>
      <fileset dir="${cxf.home}/lib">
         <include name="*.jar"/>
      </fileset>
   </path>
      
   <target name="cxfJavaToWS">
      <java classname="org.apache.cxf.tools.java2ws.JavaToWS" fork="true">
         <arg value="-wsdl"/>
         <arg value="-o"/>
         <arg value="hello.wsdl"/>
         <arg value="service.Greeter"/>
         <classpath>
            <path refid="cxf.classpath"/>
         </classpath>
      </java>
   </target>
</project>

请确保为以下选项设置“ fork = true”属性任务如上所示。另外，请记住将每个单词或标志保留在命令行选项中元素（例如，不要使用 ，但将它们分成两部分元素）。

尽管我们建议使用Maven，但请参阅CXF发行版中的antbuild示例 ，以获取使用Ant创建CXF项目的示例。
