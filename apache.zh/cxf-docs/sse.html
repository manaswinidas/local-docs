介绍

JAX-RS 2.1引入了对服务器发送事件（SSE）的支持。

从3.2.5版本开始

由于社区的贡献，SSE的实现在3.2.5版本中进行了重大修改，并且不依赖于任何其他传输或库。

组态

在大多数情况下，可以通过SseProvidersExtension或SseTransportCustomizationExtension自动激活SSE提供程序。但是，如果需要手动添加SSE支持，则可以通过包含SseFeature来完成，例如：

final JAXRSServerFactoryBean factory = RuntimeDelegate.getInstance().createEndpoint(<application>, JAXRSServerFactoryBean.class);
factory.setFeatures(Arrays.asList(new SseFeature()));
...
return factory.create();

其他特性

有几个上下文属性可以将Apache CXF SSE实现微调到特定的上下文。

属性描述默认
org.apache.cxf.sse.sink.buffer.size

默认情况下，SSE事件是分批调度的，而不是一一清除。
如果缓冲区溢出，将不再接受任何事件（因此，将返回错误）。
可以使用此属性控制缓冲区的大小（如果默认值不合适）。

10000

OSGi

对于OSGi容器（如Apache Karaf）内部的部署， Apache CXF提供了专用的cxf-sse功能（取决于cxf-http和cxf-jaxrs ）。

karaf@root()> feature:install cxf-sse

一旦安装，从属捆绑包就可以通过在Blueprint配置中指定所需的传输来从SSE功能中受益。

<blueprint xmlns="http://www.osgi.org/xmlns/blueprint/v1.0.0"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:cxf="http://cxf.apache.org/blueprint/core"
       xmlns:jaxrs="http://cxf.apache.org/blueprint/jaxrs"

       xsi:schemaLocation="http://www.osgi.org/xmlns/blueprint/v1.0.0 http://www.osgi.org/xmlns/blueprint/v1.0.0/blueprint.xsd
                           http://cxf.apache.org/blueprint/core http://cxf.apache.org/schemas/blueprint/core.xsd
                           http://cxf.apache.org/jaxrs http://cxf.apache.org/schemas/blueprint/jaxrs.xsd">

    <bean id="sseFeature" class="org.apache.cxf.jaxrs.sse.SseFeature" />

    <cxf:bus>
        <cxf:features>
            <cxf:logging />
        </cxf:features>
    </cxf:bus>

    <jaxrs:server id="sseSampleService" address="/">
        <jaxrs:serviceBeans>
            <ref component-id="..." />
        </jaxrs:serviceBeans>
        <jaxrs:providers>
            <ref component-id="..." />
        </jaxrs:providers>
        <jaxrs:features>
            <ref component-id="sseFeature" />
        </jaxrs:features>
 </jaxrs:server>
</blueprint>

3.2.5之前的版本

尽管SSE通常在HTTP传输之上工作，但是CXF中的SSE实现使用基于Atmosphere框架的专用协议。此传输仅在服务器端需要（客户端通过常规HTTP进行工作），并且与HTTP传输完全兼容。

组态

有几种方法可以配置SSE传输（因此，可以配置SSE支持）。最直接的方法是将init参数“ transportId”传递给所涉及的CXF servlet，例如：

final CXFNonSpringServlet cxfServlet = new CXFNonSpringServlet();
final ServletHolder holder = new ServletHolder(cxfServlet);
holder.setInitParameter(CXFCdiServlet.TRANSPORT_ID, SseHttpTransportFactory.TRANSPORT_ID);
...

在这种情况下， JAXRSServerFactoryBean实例还应该在构造期间指定SSE传输，例如：

final JAXRSServerFactoryBean factory = RuntimeDelegate.getInstance().createEndpoint(<application>, JAXRSServerFactoryBean.class);
factory.setTransportId(SseHttpTransportFactory.TRANSPORT_ID);
...
return factory.create();

进阶设定

从Apache CXF 3.2.3版本开始，可以通过总线属性“ org.apache.cxf.preferred.transport.id ”（也可以作为AbstractTransportFactory使用），通过其他方式配置传输，包括SSE 。PREFERRED_TRANSPORT_ID常量）。将此属性设置为首选传输后，有问题的CXF servlet将选择它而不是使用默认的servlet。SSE实现受益于此，并且在许多情况下，可以省略在servlet级别上设置“ transportId” init参数。

OSGi

对于OSGi容器（如Apache Karaf）内部的部署， Apache CXF提供了专用的cxf-sse功能（取决于cxf-http和cxf-jaxrs ）。

karaf@root()> feature:install cxf-sse

一旦安装，从属捆绑包就可以通过在Blueprint配置中指定所需的传输方式而受益于SSE功能。

<blueprint xmlns="http://www.osgi.org/xmlns/blueprint/v1.0.0"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:cxf="http://cxf.apache.org/blueprint/core"
       xmlns:jaxrs="http://cxf.apache.org/blueprint/jaxrs"

       xsi:schemaLocation="http://www.osgi.org/xmlns/blueprint/v1.0.0 http://www.osgi.org/xmlns/blueprint/v1.0.0/blueprint.xsd
                           http://cxf.apache.org/blueprint/core http://cxf.apache.org/schemas/blueprint/core.xsd
                           http://cxf.apache.org/jaxrs http://cxf.apache.org/schemas/blueprint/jaxrs.xsd">

    <cxf:bus>
        <cxf:features>
            <cxf:logging />
        </cxf:features>
    </cxf:bus>

    <jaxrs:server id="sseSampleService" address="/" transportId="http://cxf.apache.org/transports/http/sse">
        <jaxrs:serviceBeans>
            <ref component-id="..." />
        </jaxrs:serviceBeans>
        <jaxrs:providers>
            <ref component-id="..." />
        </jaxrs:providers>
    </jaxrs:server>

</blueprint>
