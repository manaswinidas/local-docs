不同类型的服务

CXF支持三种主要服务类型：

  • SOAP-此页面总结了用于创建SOAP服务的选项。
  • REST-ful-REST支持在这里描述。

Java的JAX-WS批注服务

JAX-WS API包含一组注释 ，这些注释使您可以使用带注释的类来构建服务。这些服务基于包含一组操作的单个类。

这是一个简单的例子：

@WebService
public class Hello {
  public String sayHi(String name) {
    return "Hello " + name;
  }
}

JAX-WS还包含更多注释，例如：

  • @WebMethod-允许您自定义操作名称，将操作排除在服务之外，等等
  • @WebParam-允许您自定义参数的名称，名称空间，方向（IN或OUT）等
  • @WebResult-允许您自定义Web服务调用的返回值

数据通过JAXB数据绑定从XML整理到Java，反之亦然。

服务通过以下两种方式之一发布：

  • JAX-WS标准端点API
  • CXF的XML配置格式-即

WSDL的JAX-WS批注服务

如果您已有用于服务的WSDL，或者希望先编写WSDL然后生成类，则CXF提供了许多工具来帮助您完成此任务。

WSDL2Java工具将从您的WSDL生成一个JAX-WS批注的服务和服务器存根。您可以通过以下三种方式之一运行它：

  • 命令行
  • Maven插件
  • 使用WSDL2Java API

请注意，CXF通常将WSDL支持限制为WSI-BP，而不是完整的WSDL 1.1规范。

还有一个简单的前端 ，可让您创建服务而无需使用Java批注，而使用XML配置文件。

JAX-WS提供者

JAX-WS Provider允许您创建在消息级别工作的服务-与带注释类的操作级别相反。具有单个操作“调用”，该操作接收消息有效负载（即SOAP主体）或整个消息本身（即SOAP信封）。

这是一个简单的例子：

@WebServiceProvider
public class HelloProvider {
  public Source invoke(Source request) {
    return ....;
  }
}

服务通过以下两种方式之一发布：

  • JAX-WS标准端点API
  • CXF的XML配置格式-即

Java脚本

CXF提供了一个Javascript模块，该模块允许您使用Java Rhino库以Javascript构建服务。这样的优点之一是您可以使用E4X与XML进行更轻松的交互。有关更多信息，请参见JavaScript页面。
