Maven POM信息

要在Maven中使用CXF，您需要在POM中声明CXF依赖关系。 CXF groupId为“ org.apache.cxf”。这是一个小例子：

<properties>
  <cxf.version>2.2.3</cxf.version>
</properties>

<dependencies>
	<dependency>
		<groupId>org.apache.cxf</groupId>
		<artifactId>cxf-rt-frontend-jaxws</artifactId>
		<version>${cxf.version}</version>
	</dependency>
	<dependency>
		<groupId>org.apache.cxf</groupId>
		<artifactId>cxf-rt-transports-http</artifactId>
		<version>${cxf.version}</version>
	</dependency>
        <!-- Jetty is needed if you're are not using the CXFServlet -->
	<dependency>
		<groupId>org.apache.cxf</groupId>
		<artifactId>cxf-rt-transports-http-jetty</artifactId>
		<version>${cxf.version}</version>
	</dependency>
</dependencies>

其他依赖

根据您对CXF的使用情况，您可能需要引入其他依赖关系-mvn安装过程通常会明确您所缺少的内容。这是可能需要的其他CXF工件的详尽列表：

cxf-rt-core
cxf-rt-frontend-simple
cxf-rt-frontend-jaxws
cxf-rt-databinding-aegis
cxf-rt-transports-local
cxf-rt-transports-http
cxf-rt-transports-http-jetty
cxf-rt-transports-jms
cxf-rt-management
cxf-common-utilities

Maven快照存储库

要使用每晚更新的最新非发行版本的CXF（不建议用于生产使用），请将CXF版本更改为所需的-SNAPSHOT版本，并将Apache快照存储库添加到存储库和pluginRepositories部分：

<repositories>
   ...other repos...
   <repository>
      <id>apache-snapshots</id>
      <name>Apache SNAPSHOT Repository</name>
      <url>http://repository.apache.org/snapshots/</url>
      <snapshots>
         <enabled>true</enabled>
      </snapshots>
   </repository>
</repositories>

<pluginRepositories>
   ...other repos...
   <pluginRepository>
      ...same repo as above...
   </pluginRepository>
</pluginRepositories>

需要在插件存储库部分中添加内容，因为使用该条目下载了用于WSDL2Java，Java2WS等任务的cxf-codegen-plugin。
