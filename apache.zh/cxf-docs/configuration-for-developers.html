CXF配置如何工作

CXF通过多种方式支持配置-API，Spring bean定义以及WSDL中嵌入的配置。组件的配置通常会像这样分解：
1。为您的Spring配置编写模式。可选：如果您希望在WSDL中允许配置的摘要，请创建可被Spring配置和WSDL引用的模式类型。
2。为＃1中的XSD生成JAXB类型
3。使用JAXB生成的类型在bean上创建配置属性
4。为Spring编写BeanDefinitionParser

配置规则

在CXF中编写与配置相关的代码时，请遵循以下准则。

1。请记住所有用户。
我们有三种担心的配置方案。
1。Spring 2.0配置文件
2。API用户
3。嵌入在WSDL中的配置

 

2。不要将配置对象隐藏在私有字段中，而是通过API使其可用
请记住，许多用户不会接触XML配置。示例：用户创建一个客户端，并且他们想要更改代理服务器。我们应该使它尽可能简单，而不是强迫他们使用XML或浏览一堆对象。

 

3。谨慎使用新的名称空间
当用户为组件编写配置时，他们不必担心该组件位于一个名称空间还是另一个名称空间中。例如，不必将JMS配置划分为许多不同的名称空间，而只需使用一个名称空间，因此用户只需要在其配置文件中添加一个名称空间即可。

 

4。编写XML模式时，以小写字母开头的元素和属性
而不是“ ”，我们更喜欢“ ”

 

5，不要创建多余的架构名称
如果您正在编写JMS传输模式，请不要将类型命名为“ jmsConduit”。而是将其命名为“管道”。模式类型将已经在JMS命名空间中，因此“ jmsConduit”是多余的。例： 并不是

 

6。如果您的组件不需要通过WSDL进行配置，请将您的组件编写为Bean，然后为其编写一个Spring 2.0模式。
