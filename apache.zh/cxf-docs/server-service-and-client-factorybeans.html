总览

CXF有四个主要类可帮助表示服务：

类

功能

服务

服务包含服务模型，该模型是服务及其操作的抽象表示。它还包含诸如正在使用的数据绑定，调用程序，执行程序，服务属性，拦截器等信息。

终点

端点表示可以接收消息的端点。它的主要功能是保存特定于该端点的拦截器（例如SOAP拦截器），并为发生的错误提供侦听器。客户端和服务器都使用Endpoint类。

客户

客户端是一种客户端实现，用于管理客户端上的消息流。

服务器

服务器代表CXF内部的服务器端点。它提供了挂钩来启动/停止服务器并访问端点。

CXF还包括一些工厂，以帮助创建客户端和服务器：

类

功能

实作

AbstractServiceFactoryBean

此类负责根据WSDL或类文件创建服务模型。它还设置了基本属性，例如数据绑定或基本拦截器。它通常隐藏在Client / ServerFactoryBean之后。

ReflectionServiceFactoryBean，JaxWsServiceFactoryBean

ServerFactoryBean

创建服务器端端点。

ServerFactoryBean，JaxWsServerFactoryBean

ClientFactoryBean

创建一个客户端端点。

ClientFactoryBean，JaxWsClientFactoryBean

ClientProxyFactoryBean

在客户端周围创建Java代理

ClientProxyFactoryBean，JaxWsProxyFactoryBean

有关如何使用这些bean的更多信息，请参见Simple Frontend页面或javadocs。

ServiceFactoryBeans和AbstractServiceConfiguration

CXF中的主要服务工厂bean是ReflectionServiceFactoryBean。它提供了一种将类映射到服务的方法。如果您有兴趣控制此映射的发生方式，则可以扩展并提供自己的服务配置。大多数服务配置都是通过这些配置完成的。例如，在JAX-WS前端中，JaxWsServiceConfiguration通过检查JAX-WS批注来控制如何映射服务。

要添加自己的服务配置：

MyServiceConfiguration config = new AbstractServiceConfiguration() {
... // your implementation
};
ReflectionServiceFactoryBean serviceFactory = new ReflectionServiceFactoryBean();
serviceFactory.getServiceConfigurations().add(0, config);

ServerFactoryBean svrFactory = new ServerFactoryBean();
svrFactory.setServiceFactory(serviceFactory);
...
Server svr = svrFactory.create();
