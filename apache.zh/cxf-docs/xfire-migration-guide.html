我应该迁移吗？

XFire用户应考虑迁移到CXF。希望开始新的Web服务项目的任何人都应该认真考虑CXF。目前最主要的障碍是成为文档。在接下来的几周内，随着我们朝着2.0最终版本的迈进，我们将增加很多文档。

CXF为XFire用户提供了许多新功能：

  • 改进的WSDL支持
  • 改进的JAX-WS支持
  • 改进的JMS传输
  • Maven插件
  • Spring 2.0语法支持
  • 改进的WS- *支持
  • 清理了用于构建服务的API
  • 更容易使用扩展点
  • RESTful服务支持
  • 支持“裸” XML绑定，其中不包含SOAP信封

谁不应该迁移

在几个小区域中，我们尚未复制所有XFire功能。即JiBX数据绑定。我们看到的不迁移到CXF的唯一其他原因是Java 1.4的要求。

服务工厂

XFire包括用于构建服务的服务器服务工厂。CXF还包括服务工厂，但是对该概念进行了一些改进以使其更易于使用。CXF现在包括生成服务器的ServerFactoryBeans，生成客户端的ClientFactoryBeans和生成客户端代理的ClientProxyFactoryBeans的概念。

CXF现在包括以下类：

  • ServerFactoryBean-这从类创建服务器端点
  • ClientProxyFactoryBean-这从类创建客户端代理
  • JaxWsServerFactoryBean-这从类或WSDL创建一个JAX-WS服务器端点
  • JaxWsProxyFactoryBean-这从类创建一个JAX-WS客户端代理

请记住，与XFire相比，CXF的工作方式存在一些差异是很重要的：

  • CXF使用bean工厂方法。这意味着每个端点只有一个（Server / ClientProxy）FactoryBean。在XFire中，这些bean可以重复使用。
  • CXF要求您指定端点地址，除非它在WSDL中找到。如果您使用的是HTTP，则有一些不错的好处。
    • 如果您尚未设置Servlet传输，它将自动为您启动嵌入式Jetty实例。在XFire中，这要求您使用XFireHttpServer类，从而导致其他工作。
    • 这使您可以更轻松地将服务映射到不同的URL。在XFire中，URL映射到服务的方式更加静态。
  • 为服务指定端点地址时，请务必记住，在使用sServlet时，无需指定主机名。假设您将CXFServlet设置为侦听“ http：// localhost / services”。如果将端点地址指定为“ / myservice”，则所有到达“ http：// localhost / services / myservice”的请求都将转到您的服务。
  • 现在，JAXB是默认的数据绑定。任何时候要使用“宙斯盾”，都必须明确声明。

示例ObjectServiceFactory迁移

这是在XFire中使用ObjectServiceFactory的示例：

ObjectServiceFactory osf = new ObjectServiceFactory();
Service service = osf.create(MyServiceInterface.class);
service.setInvoker(new BeanInvoker(new MyServiceImpl());

这将相当于CXF：

ServerFactoryBean sf = new ServerFactoryBean();
sf.getServiceFactory().setDataBinding(new AegisDatabinding();
sf.setServiceBean(new MyServiceImpl());
sf.setServiceClass(MyServiceInterface.class);
sf.setAddress("http://localhost:8080/myservice");
sf.create();

AnnotationServiceFactory迁移示例

这是在XFire中使用AnnotationServiceFactory的示例：

AnnotationServiceFactory osf = new AnnotationServiceFactory();
Service service = osf.create(MyServiceInterface.class);
service.setInvoker(new BeanInvoker(new MyServiceImpl());

这将相当于CXF：

JaxWsServerFactoryBean sf = new JaxWsServerFactoryBean ();
sf.getServiceFactory().setDataBinding(new AegisDatabinding());
sf.setServiceClass(MyServiceImpl.class);
sf.setAddress("http://localhost:8080/myservice");
sf.create();

示例JAXWSServiceFactory迁移

这是在XFire中使用JAXWSServiceFactory的示例：

JAXWSServiceFactory sf = new JAXWSServiceFactory();
Service service = sf.create(MyServiceImpl.class);
service.setInvoker(new BeanInvoker(new MyServiceImpl());

这将相当于CXF：

JaxWsServerFactoryBean sf = new JaxWsServerFactoryBean();
sf.setServiceClass(MyServiceImpl.class);
sf.setAddress("http://localhost:8080/myservice");
sf.create();

HTTP和Servlet设置

CXF支持广泛的选项，用于通过HTTP部署服务。 XFire在HTTP URL和服务名称之间创建了一个静态映射-即，可以从http：// localhost / services / HelloService访问名为“ HelloService”的服务 。另一方面，CXF允许您控制发布服务的URL。此地址通过ServerFactoryBeans上的setAddress（）调用或通过XML配置提供给CXF。

XFire的另一个改进是CXF现在可以透明地检测是否正在使用CXFServlet。如果是这样，您的服务将在该位置提供。如果不是，则CXF将自动启动嵌入式Jetty实例（前提是您的类路径中包含cxf-rt-transports-http-jetty模块）。不再需要对XFireHttpServer的调用！

要设置CXF Servlet，请阅读“ Servlet传输”页面上的方法。

要使用嵌入式Jetty实例，不需要额外的工作。只需遵循以上ServiceFactory示例。

services.xml

XFire包括通过services.xml文件部署服务的支持。在CXF中，此文件名为cxf.xml，并将由CXFServlet加载

XFire services.xml示例：

<beans xmlns="http://xfire.codehaus.org/config/1.0">

<service xmlns:t="urn:my:namespace">
	<name>testservice</name>
	<serviceClass>org.example.TestService
	</serviceClass>
	<implementationClass>org.codehaus.xfire.spring.TestServiceImpl</implementationClass>
	<namespace>urn:my:namespace</namespace>
	<serviceFactory>org.codehaus.xfire.jaxws.JAXWSServiceFactory</serviceFactory>

	<properties>
		<property key="myKey">value</property>
        </properties>

	<inHandlers>
		<handler handlerClass="org.codehaus.xfire.spring.TestHandler"/>
	</inHandlers>
</service>

</beans>

CXF示例：

<beans xmlns="http://www.springframework.org/schema/beans"
      xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
      xmlns:jaxws="http://cxf.apache.org/jaxws"
      xsi:schemaLocation="
http://www.springframework.org/schema/beans
http://www.springframework.org/schema/beans/spring-beans.xsd
http://cxf.apache.org/jaxws
http://cxf.apache.org/schemas/jaxws.xsd">
 <import resource="classpath:META-INF/cxf/cxf.xml" />
 <import resource="classpath:META-INF/cxf/cxf-extension-soap.xml"/>
 <import resource="classpath:META-INF/cxf/cxf-servlet.xml" />

  <jaxws:endpoint address="http://localhost/testService"
    serviceName="t:testService"
    xmlns:t="urn:my:namespace">
    <jaxws:implementor>
      <bean class="org.example.TestServiceImpl"/>
    </jaxws:implementor>
    <jaxws:properties>
      <entry key="foo" value="bar"/>
    </jaxws:properties>
    <jaxws:inInterceptors>
      <bean class="org.example.ExampleInterceptor"/>
    </jaxws:inInterceptors>
  </jaxws:endpoint>

</beans>

这里有一些重要的区别注意事项：

  • 其中许多属性都是可选的，例如服务名称
  • 如果您使用的是JAX-WS，则无需指定serviceClass，因为服务应使用@ WebService.endpointInterface属性进行注释。
  • XFire处理程序的等效项是CXF内部的拦截器 。它们具有改进的API！

上面的示例适用于JAX-WS / JSR181服务。对于那些没有注释的开发服务，以下是“ 简单前端 ”示例

<beans xmlns="http://www.springframework.org/schema/beans"
      xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
      xmlns:simple="http://cxf.apache.org/simple"
      xsi:schemaLocation="
http://www.springframework.org/schema/beans
http://www.springframework.org/schema/beans/spring-beans.xsd
http://cxf.apache.org/simple
http://cxf.apache.org/schemas/simple.xsd">

  <simple:server address="http://localhost/testService"
    serviceClass="org.example.TestService">
    <simple:serviceBean>
      <bean class="org.example.TestServiceImpl"/>
    </simple:serviceBean>
    <simple:properties>
      <entry key="foo" value="bar"/>
    </jaxws:properties>
    <simple:inInterceptors>
      <bean class="org.example.ExampleInterceptor"/>
    </simple:inInterceptors>
  </simple:server>

</beans>
