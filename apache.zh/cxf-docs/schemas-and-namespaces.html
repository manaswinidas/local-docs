模式和命名空间的使用

CXF在运行时中广泛使用XML模式：其核心包括标准模式的JAXB映射，例如http://schemas.xmlsoap.org/wsdl/和http://www.w3.org/2006/03/addressing/ws -addr.xsd 。 CXF还为WSDL扩展定义了自己的模式，例如命名空间http://cxf.apache.org/transports/jms中的jms-address元素，以及Spring和Blueprint（OSGi）配置的模式，例如http：/ /cxf.apache.org/jaxws和http://cxf.apache.org/blueprint/jaxws命名空间。

为了避免在构建期间通过网络访问资源，CXF会维护这些架构的本地副本。但是，从逻辑上讲，第三方以及CXF模式在schemaLocation属性和类似位置中由其公共URI引用。CXF代码生成器使用目录文件将它们映射到主干中的实际位置，有关示例，请参阅api模块中的pom.xml文件。

请注意，如果您使用的是Spring，它将使用每个JAR的META-INF / spring.schemas文件（ 此处为示例）中的信息来验证Spring配置文件。验证对于诊断CXF配置中的错误非常有帮助，但在运行时也很昂贵。应用程序正常运行后，可以通过设置系统属性来禁用验证：-Dorg.apache.spring.validation.mode=VALIDATION_NONE

下表包括您需要在Spring配置文件的schemaLocation属性中指定的CXF配置模式的URI，以便Spring的验证解析器可以验证该文件。从理论上讲，您不必关心文件的实际位置：Spring通常可以从类路径中检测它们的位置。而且，如果要检查模式文件的内容（以二进制分发），则应该能够使用其URI进行检查。您还可以使用下表中的类路径位置在主干中找到架构的原始版本。

例

以下是有效的Spring配置文件的示例。使用上表，为schemaLocation属性设置正确的值没有任何魔力！

<beans xmlns="http://www.springframework.org/schema/beans"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xmlns:http-conf="http://cxf.apache.org/transports/http/configuration"
    xmlns:jaxws="http://cxf.apache.org/jaxws"
    xsi:schemaLocation="
http://cxf.apache.org/transports/http/configuration http://cxf.apache.org/schemas/configuration/http-conf.xsd
http://cxf.apache.org/jaxws http://cxf.apache.org/schemas/jaxws.xsd
http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd">

    <http-conf:conduit name="{http://cxf.apache.org/hello_world_soap_http}SoapPort.http-conduit">
        <http-conf:client DecoupledEndpoint="http://localhost:9999/decoupled_endpoint"/>
    </http-conf:conduit>

    <jaxws:client name="{http://cxf.apache.org/hello_world_soap_http}SoapPort" createdFromAPI="true">
        <jaxws:conduitSelector>
            <bean class="org.apache.cxf.endpoint.DeferredConduitSelector"/>
        </jaxws:conduitSelector>
    </jaxws:client>
</beans>

开发人员注意事项：如果定义自己的配置模式，请将其放置在资源目录的schemas子目录中，然后将http://cxf.apache.org/和模式相对于资源目录的路径结合起来以形成系统ID，并在模块的META-INF目录中的spring.schemas文件中添加类似于以下内容的行，以使Spring知道后者（请注意转义的：字符）：

http\://cxf.apache.org/schemas/wsdl/http-conf.xsd=schemas/wsdl/http-conf.xsd
