将应用程序从Celtix迁移到CXF非常简单。两者都遵循JAX-WS映射，因此大量的应用程序代码将不需要任何修改。

以下是一些注意事项：

  • 软件包名称从org.objectweb.celtix.*至org.apache.cxf.* 。如果您在应用程序中使用了任何Celtix特定的非JAX-WS API（例如，使用Bus ），那么您必须更改导入的软件包名称。
  • 没有Bus.init()要么Bus.getCurrent()方法。在CXF中，您可以使用以下代码的具体实现来创建总线BusFactory.createBus()方法，然后将新总线设置为默认总线，如下所示。
SpringBusFactory bf = new SpringBusFactory();
    Bus bus = bf.createBus("ws_rm.xml");
    bf.setDefaultBus(bus);

稍后，要搭乘巴士，您可以致电：

Bus bus = bf.getDefaultBus();
  • 在您的课堂上，改变celtix.jar至cxf-manifest.jar ;此jar包含清单条目，以提取CXF运行时中所有其他所需的jar。
  • 如果你打电话getPort()在客户端，那么必须指定WSDL协定的目标名称空间作为端口名称的名称空间，如下所示：
HelloWorld helloWorld = helloWorldService.getPort(
                        new QName("http://www.my/wsdl/target/namespace", "SOAPOverHTTPEndpoint"),
                                    HelloWorld.class );

在Celtix中，您可以将端口名称的命名空间保留为空字符串“”。但是，如果您在CXF中执行此操作，则对getPort()将失败。后一种行为更符合JAX-WS规范（请参阅JAX-WS规范的4.2.3节）。
