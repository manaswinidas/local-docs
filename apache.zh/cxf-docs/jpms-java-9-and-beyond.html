介绍

随Java 9发行版一起引入的主要功能之一是JPMS ：Java平台模块系统。到目前为止，尚未将Apache CXF模块作为Java模块构建/发布，但是已经开始朝这个方向开展工作。

自动模块名称

从3.2.5版本开始，所有Apache CXF模块都包含Automatic-Module-Name清单条目，以便提供稳定，方便的方式从模块化（基于JPMS）Java应用程序的描述符中引用它们。模块列表及其自动模块名称如下所示。

遵循Stephen Colebourne的建议（ http://blog.joda.org/2017/04/java-se-9-jpms-module-naming.html ）并根据包名称采用了命名方案。

用法示例

这是从模块化Java应用程序通过module-info.java描述符使用Apache CXF SSE功能的快速示例。

module sample.sse {
    ...
    
    requires org.apache.cxf.frontend.jaxrs;
    requires org.apache.cxf.transport.http;
    requires org.apache.cxf.rs.sse;
    requires com.fasterxml.jackson.jaxrs.json;
    requires com.fasterxml.jackson.annotation;
    requires io.reactivex.rxjava2;
    
    requires transitive java.ws.rs;
    requires transitive org.reactivestreams;
    
    requires javax.servlet.api;
    requires jetty.server;
    requires jetty.servlet;
    requires jetty.util;
    
    requires java.xml.bind;
}
