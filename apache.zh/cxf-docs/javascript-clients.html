JavaScript客户端

JavaScript客户端支持是CXF 2.1的功能。

CXF提供了一种生成JavaScript代码的功能，该JavaScript代码可以通过公共XMLHttpRequest对象与Web服务进行通信。该对象存在于现代浏览器中。CXF还包括一个可插入Rhino的实现（尽管在撰写本文时，此打包并不方便）。

简介：为什么要这样（在浏览器中）？

如果您正在编写恰好使用JavaScript编写的应用程序，并且希望它与Web服务进行通信，那么这个问题的答案可能不言而喻。更为有趣的问题是：“为什么要通过浏览器的SOAP绑定与Web服务对话？”

历史上，备受关注已经支付给绑定和协议是不适合浏览器-服务器的通信SOAP。关于这一点的文章通常是从一个共同的想法开始的：“处理JavaScript中的完整XML / SOAP消息传递的复杂性可能不切实际。'

CXF JavaScript客户端模块的作者不是第一个，或者绝对不是唯一考虑这一想法的人。有几种具有不同级别的功能和许可证的替代方案。如果您熟悉SOAP Web服务，那么与采用REST等其他框架相比，构建和理解与标准文档/文字或RPC /文字服务的JavaScript绑定相关的代码可能会更容易。

另一方面，如果您对使用REST或类似的东西感到满意，请不要被本讨论所冒犯。CXF很乐意容纳。

能力是什么？

CXF JavaScript客户端工具是代码生成器。给定具有SOAP绑定的Web服务的抽象模型，它将产生一组与之对话的JavaScript类。您可以将其视为“ WSDL to JavaScript”。“不过，CXF 不强迫你使用WSDL文件生成一个JavaScript客户端。CXF支持以下所有功能：

  • wsdl2js-读取WSDL文件，编写JavaScript。
  • java2js-使用CXF前端之一（简单或JAX-WS）和受支持的数据绑定之一（Aegis或JAXB）“读取” Java代码，并编写JavaScript。
  • 动态JavaScript-使用JavaScript代码响应形式为http：// host：port / port_and_or_service？js的URL以与服务进行通信。

动态模式很方便：它避免了生成和管理JavaScript文件的需要。但是，您可能会发现生成一些JavaScript文件并阅读它们以了解编码内容会有所帮助。

作为代码生成器，JavaScript客户端生成器与JAXB或JAX-WS并行。它定义了从Web服务的抽象模型到JavaScript对象的映射。与JAXB和JAX-WS不同，没有委员会对“ JavaScript绑定”进行标准化。'CXF绑定可能并不符合每个人的口味。

CXF JavaScript绑定仅与接口和操作有关。它完全忽略了“服务”和“端口”。程序员应该知道服务的URL，仅此而已。
