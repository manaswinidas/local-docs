使用RequestDispatcherProvider

RequestDispatcherProvider是一个JAXRS MessageBodyWriter，它可以重定向到JSP页面，命名的或默认的servlet。它可用于提供给定资源类的所有响应，或者仅使用classResources映射属性将其限制为提供一组有限的类。

从CXF 2.5.0和2.4.4开始，也可以指定仅对具有匹配URI的请求的响应进行处理。

目前，此提供程序已静态配置为支持text / html内容类型，但可以根据需要轻松配置为支持其他内容类型。

除了“ resourcePath”和“ dispatcherName”属性外，还可以设置“ scope”属性，该属性具有两个可能的值：“ request”和“ session”，其中“ request”为默认值。它影响JSP代码检索由RequestDispatcherProvider传递给它的参数的方式。如果它是“请求”范围，则所有参数均设置为当前HTTP请求的属性。如果是会话作用域，则将它们设置为当前HTTP会话的属性。

RequestDispatcherProvider设置以下参数：

  • JAXRS方法响应对象。此参数的名称可以是此对象的简单类名（小写），也可以是使用该对象的完全限定的类名从beanNames映射属性检索的值。
  • 在方法执行期间已初始化的所有路径，查询和矩阵参数
  • 从当前UriInfo获得的“ absolute.path”，“ base.path”和“ relative.path”

这里有些例子。假设我们已经部署了book.war Web应用程序。

<jaxrs:server id="bookservice1" address="/bookstore1">
    <jaxrs:serviceBeans>
      <bean class="org.apache.cxf.systest.jaxrs.BookStoreDispatch"/>
    </jaxrs:serviceBeans>		  
    <jaxrs:providers>
       <ref bean="dispatchProvider"/>
    </jaxrs:providers> 
</jaxrs:server>

<bean id="dispatchProvider" class="org.apache.cxf.jaxrs.provider.RequestDispatcherProvider">
      <property name="resourcePath" value="/book.html"/>
</bean>

上面将响应重定向到默认的book.html页面，该页面直接在/ webapps / book文件夹中可用。通常，这样做是为了向客户端返回一些静态确认。例如，考虑一个给定的JAX-RS方法已处理的POST表单请求，现在唯一需要做的就是返回HTML确认视图。请注意，如果资源方法未返回任何自定义对象，则不会调用JAX-RS MessageBodyWriters-在静态确认的情况下不需要此自定义对象，因此对于RequestDispatcherProvider而言，能够重定向到book.html的人应该简单地引入EmptyConfirmation bean，没有属性，并从资源方法返回它。

这是另一个示例（省略jaxrs：server声明以确保酿造性）：

<bean id="dispatchProvider" class="org.apache.cxf.jaxrs.provider.RequestDispatcherProvider">
      <property name="resourcePath" value="/book.jsp"/>
</bean>

与上一个示例的唯一区别是，将“ /book.jsp”委派给创建视图的任务。这是一个更有趣的示例，我们假定资源方法返回一个“ org.bar”的实例。书”豆：

@Path("/books")
public Resource {
    @GET
    @Produces({"text/html", "application/xml"})
    @Path("{id}")
    public Book getBook(@PathParam("id") String id) {
        // return the book
    }
}

请注意，就编写JAX-RS资源代码而言，非介入式RequestDispatcherProvider的方式如何，您只需照常在@Produces中列出受支持的媒体类型即可。

RequestDispatcherProvider将使Book实例作为名为“ org.bar”的HttpServletRequest属性可用。默认情况下为“图书”。可以自定义。如果将“ beanName”属性设置为例如“ book”，则book.jsp将作为“ book”属性访问Book实例。如果您说2种资源方法返回不同bean类的实例（可能用于不同的视图处理程序），则可以使用beanNames map属性，例如：

<bean id="dispatchProvider" class="org.apache.cxf.jaxrs.provider.RequestDispatcherProvider">
      <property name="classResources">
          <map>
             <entry key="org.bar.Book"  value="/book.jsp"/>
             <entry key="org.bar.Customer"  value="/customer.jsp"/>
          </map>
      </property>
      <property name="beanNames">
          <map>
             <entry key="org.bar.Book"  value="book"/>
             <entry key="org.bar.Customer"  value="customer"/>
          </map>
      </property>
</bean>

上面的配置说，“ book.jsp”资源将通过作为“ book”属性访问它的一个Book实例，并通过一个“ customer”属性来获取“ customer.jsp”（一个Customer的实例）来处理Book的一个实例。注意，在这种情况下，您不需要使用“ beanNames”属性，除非您有一个同时处理Book和Customer的（jsp）资源，否则，可以使用更简单的“ beanName”属性。

默认情况下，除了使响应类的实例作为HttpServletRequest属性可用之外，RequestDispatcherProvider还将使所有Path，Query和Matrix参数作为HttpServletRequest参数可用（相对于属性）。例如，给定上面的代码片段，一个名为“ id”的HttpServletRequest参数（表示一个@PathParam（“ id”））可用于视图处理程序，以及所有其他查询和矩阵参数。
请注意，通过将boolean saveParametersAsAttributes属性设置为true，可以将RequestDispatcherProvider配置为将所有这些请求参数另存为HttpServletRequest属性。

现在，想象这样一个场景：我们有两种资源方法返回ReservationStatus bean。第一种方法返回成功的确认或使用Response.seeOther（...）重定向到处理失败的保留的方法。因此，这两种方法都返回相同的ReservationStatus bean，但我们将有两个分别处理成功和失败保留的视图。这是一种管理方式：

<bean id="reserveRegistrationViews" class="org.apache.cxf.jaxrs.provider.RequestDispatcherProvider">
         <property name="resourcePaths">
            <map>
              <entry key="/reservations/reserve/complete" value="/forms/reservationConfirm.jsp"/>
              <entry key="/reservations/reserve/failure" value="/forms/reservationFailure.jsp"/>
            </map>
         </property>
         <property name="beanName" value="data"/>
</bean>

鉴于在两种情况下都返回了相同的ReservationStatus bean，实际上是原始请求URI片段，用于匹配哪个视图处理程序将处理给定的ReservationStatus，例如，对请求URI的响应“ http：// localhost：8080 / reservations / reserve / complete”将由“ /forms/reservationConfirm.jsp”处理。

请注意，RequestDispatcherProvider具有'dispatcherName'属性-在重定向到已命名的servlet（例如MyServlet）时，该属性很方便，包括
例如“ jsp”或“ default”，特别是当处理给定调用的CXFServlet具有uri模式（通常为通配符）时，它也可能捕获重定向请求，请参阅下一节以了解更多信息。

接下来，想象一个这样的场景：我们有一个单一的资源方法来接受一些数据，并且响应视图将根据请求处理的状态而有所不同。在这种情况下，使用枚举是最有效的选择：

package resource;

public class Status {
    UPDATE_SUCCESS,
    UPDATE_FAILURE
}

@Path("/")
public class Resource {

  @PUT
  @Produces("text/html")
  public Response update(SomeData data) {
     if (update(data)) {
         return Response.ok(Status.UPDATE_SUCCESS).build();
     } else {
         return Response.ok(Status.UPDATE_FAILURE).build();  
     }
  } 
}

接下来，您可能只有一个JSP处理程序，它将检查它是否为Status。UPDATE_SUCCESS或状态。UPDATE_FAILURE并相应地格式化响应。在这种情况下，基本的RequestDispatcherProvider配置将执行以下操作：

<bean id="dispatchProvider" class="org.apache.cxf.jaxrs.provider.RequestDispatcherProvider">
      <property name="resourcePath" value="/updateStatus.jsp"/>
</bean>

另外，您可能拥有专门的视图处理程序来处理特定的状态，在这种情况下，您可以：

<bean id="reserveRegistrationViews" class="org.apache.cxf.jaxrs.provider.RequestDispatcherProvider">
         <property name="classResources">
            <map>
              <entry key="resource.Status.UPDATE_SUCCESS" value="/forms/updateSuccess.jsp"/>
              <entry key="resource.Status.UPDATE_FAILURE" value="/forms/updateFailure.jsp"/>
            </map>
         </property>
</bean>

或者，从CXF 2.7.1开始，

<beans xmlns="http://www.springframework.org/schema/beans"
      xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
      xmlns:util="http://www.springframework.org/schema/util"
      xsi:schemaLocation="
http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd
http://www.springframework.org/schema/util http://www.springframework.org/schema/util/spring-util.xsd">

<bean id="reserveRegistrationViews" class="org.apache.cxf.jaxrs.provider.RequestDispatcherProvider">
         <property name="enumResources">
            <map>
              <entry 
                 <key>
                    <util:constant static-field="resource.Status.UPDATE_SUCCESS"/>
                 </key> 
                 <value>/forms/updateSuccess.jsp</value>
              </entry>
              <entry 
                 <key>
                    <util:constant static-field="resource.Status.UPDATE_FAILURE"/>
                 </key> 
                 <value>/forms/updateFailure.jsp</value>
              </entry> 
            <map>
         </property>
</bean>
</beans>

会有所帮助。

从CXF 2.6.1开始，可以配置提供程序以检查当前类是否具有关联的视图处理程序，例如：

<bean id="viewHandler" class="org.apache.cxf.jaxrs.provider.RequestDispatcherProvider">
         <property name="dispatcherName" value=jsp""/>
         <property name="useClassNames" value="true"/>
</bean>

例如，给定一个简单的类名称，例如“ BookInfo”，RequestDispatcherProvider将检查“ /WEB-INF/bookInfo.jsp”处理程序是否可用。提供商可能会扩展为根据需要检查更多的位置。

RequestDispatcherProvider还检查出站消息上的“ redirect.resource.path”属性。如果设置了此属性，则它将尝试查找在给定路径上可用的RequestDispatcher。

从CXF 3.0.4开始，将提供一个新属性“ includeResource”：如果将此属性设置为true，则将使用RequestDispatcher.include（）而不是RequestDispatcher.forward（）。

最后，可以使用“ ServletContextPath”属性将其他一些ServletContext（与当前属性相反）用于RequestDispatcher查找。如果设置，则将使用当前的ServletContext.getContext（servletContextPath）来获取所需的ServletContext。

记录重定向

要使RequestDispatcherProvider记录有关重定向的信息，请设置'logRedirects'属性：

<bean id="reserveRegistrationViews" class="org.apache.cxf.jaxrs.provider.RequestDispatcherProvider">
         <property name="logRedirects" value="true"/>
         <!-- other properties as needed --> 
</bean>

您将看到这样的日志记录条目：

23-Jul-2012 11:26:13 org.apache.cxf.jaxrs.provider.RequestDispatcherProvider logRedirection

INFO: Setting an instance of "oauth2.common.ConsumerRegistration" as HttpServletRequest attribute "newClient" and redirecting the response to "/forms/registerAppConfirm.jsp"

使用CXFServlet

请参阅“ Servlet传输”页面上的“重定向”部分。

请注意，如该页所述，CXFServlet和JAXRS RequestDispatcherProvider可以在执行重定向请求时有效地协同工作。

如果CXFServlet URI模式与RequestDispatcherProvider重定向到的资源URI不匹配，则无需担心。

如果您让CXFServlet侦听“ /”（从而有效地捕获所有请求）并且还想使用RequestDispatcherProvider，则请确保还设置了“ dispatcherName”属性，例如：

<bean id="dispatchProvider" class="org.apache.cxf.jaxrs.provider.RequestDispatcherProvider">
    <property name="dispatcherName" value="jsp"/>
    <property name="resourcePath" value="/WEB-INF/jsp/test.jsp"/>
</bean>

如果重定向到的资源可以公开（即，从/ WEB-INF中移出），则替代选项（而不是向RequestDispatcherProvider添加'dispatcherName'属性，并且仍然让CXFServlet侦听'/'）是配置这两个RequestDispatcherProvider和CXFServlet重定向到诸如“ /jsp/test.jsp”之类的资源。

另外，如果您有许多公共视图处理程序，那么不必在每个调度程序bean上设置“ dispatcherName”属性，则只能在CXFServlet上将其设置一次：

<servlet>
        <servlet-name>RESTServlet</servlet-name>
        <servlet-class>org.apache.cxf.transport.servlet.CXFServlet</servlet-class>
        <init-param>
    	   <param-name>redirects-list</param-name>
	   <param-value>/forms/.*</param-value>
	</init-param>
	<init-param>
    	   <param-name>redirect-servlet-name</param-name>
	   <param-value>jsp</param-value>
	</init-param>
	<load-on-startup>1</load-on-startup>
    </servlet>
    <servlet-mapping>
        <servlet-name>RESTServlet</servlet-name>
        <url-pattern>/*</url-pattern>
    </servlet-mapping>

在这里，我们有所有针对/ form / *的请求（例如，/ form / book.jsp等）都由jsp处理程序处理。

自订重新导向

可以从RequestDispatcherProvider借用一些代码，并在拦截器或自定义调用程序中从CXF执行自定义重定向，如果您尝试这样做，则还需要设置AbstractHTTPDestination。REQUEST_REDIRECTED属性，在当前输入消息上具有“ true”值。
