CXF 2.4.x中的更改

以下内容适用于CXF 2.3.x及更早版本。 从CXF 2.4.0开始，扩展由CXF内部自动加载，您无需导入所有的cxf-extension-*。xml文件。您只需要导入classpath：META-INF / cxf / cxf.xml。

您可以将CXF嵌入现有的Spring应用程序中。由于所有XML配置文件都是Spring xml文件，因此这两种方法应该等效。

CXF包含配置各种CXF模块的Spring配置文件。您将需要将它们导入到您的应用程序中。这是一个导入核心CXF组件，SOAP绑定和Servlet传输的示例：

<beans xmlns="http://www.springframework.org/schema/beans"
	xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
	xmlns:jaxws="http://cxf.apache.org/jaxws"
	xsi:schemaLocation="
http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans-2.0.xsd
http://cxf.apache.org/jaxws http://cxf.apache.org/schema/jaxws.xsd">

	<import resource="classpath:META-INF/cxf/cxf.xml" />
	<import resource="classpath:META-INF/cxf/cxf-extension-soap.xml" />
	<import resource="classpath:META-INF/cxf/cxf-servlet.xml" />
  ...
</beans>

要包括所有CXF模块，可以使用cxf-all或通配符表达式：

<beans xmlns="http://www.springframework.org/schema/beans"
	xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
	xmlns:jaxws="http://cxf.apache.org/jaxws"
	xsi:schemaLocation="
http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans-2.0.xsd
http://cxf.apache.org/jaxws http://cxf.apache.org/schema/jaxws.xsd">

	<import resource="classpath:META-INF/cxf/cxf.xml" />
	<import resource="classpath*:META-INF/cxf/cxf-extension-*.xml" />
  ...
</beans>

唯一的cxf-all模块将不包括servlet传输。如果要包括servlet传输，则必须专门指定它。这是因为它将告诉CXF不再使用在“ classpath：META-INF / cxf / cxf-extension-http-jetty.xml” xml文件中指定的独立HTTP传输，并且您可能并不总是希望这样做这个。

该邮件列表线程提供了有关CXF Spring文件的更多信息，这些文件可能需要导入到web.xml中。
