如果要使用JavaScript客户端生成器，则首要决定是使用工具生成JavaScript文件还是使用?？js URL处理程序。使用这些工具有一些优点：您可以轻松阅读JavaScript并查看生成的API，并且无需使用运行时CPU时间来生成JavaScript。另一方面，您必须处理JavaScript文件。一个更严重的问题适用于代码优先服务。确保使用与服务在运行时建立的配置完全相同的配置来配置java2js可能很棘手。

无论哪种情况，JavaScript客户端生成器中唯一可配置的项目是名称空间URI到JavaScript对象名称前缀的映射。在撰写本文时，您可以在端点的Spring配置（或API）中配置它，也可以将其配置为java2js的命令行选项。我们希望对其进行修改以与WSDL / Spring架构扩展一起使用，因此，早期采用者现在可能更喜欢使用默认映射。

如果您打算使用？js URL处理程序，您必须将其配置到总线上。如果使用cxf.xml或cxf-servlet.xml来配置服务，则？js URL处理程序将与所有其他总线扩展一起被包括在内。如果构建自己的Spring配置，则必须在bean配置中包括META-INF / cxf / cxf-extension-javascript-client.xml。如果您很难配置总线，请添加org.apache.cxf.javascript。JavascriptQueryHandlerRegistry到总线，它将注册处理程序。
