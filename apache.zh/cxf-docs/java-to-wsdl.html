概要

java2wsdl [-?|-help|-h][-o <output-file>][-cp <class-path>][-soap12][-t <target-namespace>][-servicenam <seservice-name>][-v][-verbose|-quiet][-s <source-directory>]
          [-classdir <compile-classes-directory>][-portname <port-name>][-createxsdimports][-d <output-directory>] { classname }

描述

java2wsdl使用已编译的Web服务端点的实现（SEI）类和关联的类型类来生成WSDL文件。

注意：java2wsdl仅适用于当前生产的CXF 2.0.x系列。对于即将推出的2.1版本，请改用java2ws。

例

java2wsdl org.apache.hello_world_soap_http。招待员

java2wsdl -cp ./tmp org.apache.hello_world_soap_http。招待员

java2wsdl -o hello.wsdl org.apache.hello_world_soap_http。招待员

java2wsdl -o hello.wsdl -t http://cxf.apache.org org.apache.hello_world_soap_http。招待员

（有关Apache Ant和Maven的用法，请参见下文。）

争论

下表回顾了用于管理WSDL文件生成的参数。

选项

解释

-？

显示此实用程序的联机帮助。

-救命

-H

-o

指定生成的WSDL文件的名称。

-cp

指定目录和zip / jar文件的SEI和类型类搜索路径。

-soap12

指定生成的WSDL将包括SOAP 1.2绑定。

-t

指定要在生成的WSDL文件中使用的目标名称空间。

-服务名称

指定生成的服务元素的name属性的值。

-v

显示工具的版本号。

-冗长

在代码生成过程中显示注释。

-安静

在代码生成过程中禁止注释。

-s

放置生成的源文件的目录（通常以jaxws模式），它将生成包装器bean和故障bean

-classdir

生成的源编译到的目录。如果未指定，则不编译文件。

-端口名

指定要在生成的wsdl中使用的端口名。

-createxsdimports

输出模式可分隔文件，并使用导入来加载它们，而不是将它们内联到wsdl中。

-d

输出文件所在的目录。

班级名称

指定SEI类的名称。请注意，这是已编译的类，而不是对其源文件的引用。

您必须包括classname论点。所有其他参数都是可选的，可以按任何顺序列出。该工具将搜索并加载服务端点类和类型类。确保这些课程在CLASSPATH或通过-cp旗。

在Ant中使用java2wsdl

可以将java2wsdl命令包装在Ant目标中，如下所示：

<?xml version="1.0"?>
<project name="cxf java2wsdl" basedir=".">   
   <property name="cxf.home" location ="/usr/myapps/cxf-2.0.1"/>
   <property name="build.classes.dir" location ="${basedir}/build/classes"/>

   <path id="cxf.classpath">
      <pathelement location="${build.classes.dir}"/>
      <fileset dir="${cxf.home}/lib">
         <include name="*.jar"/>
      </fileset>
   </path>
      
   <target name="cxfJavaToWSDL">
      <java classname="org.apache.cxf.tools.java2wsdl.JavaToWSDL" fork="true">
         <arg value="-o"/>
         <arg value="hello.wsdl"/>
         <arg value="service.Greeter"/>
         <classpath>
            <path refid="cxf.classpath"/>
         </classpath>
      </java>
   </target>
</project>

请确保为以下选项设置“ fork = true”属性任务如上所示。另外，请记住将每个单词或标志保留在命令行选项中元素（例如，不要使用 ，但将它们分成两部分元素）。
