Spring Boot CXF JAX-WS入门

特征

使用“ / services / *” URL模式注册CXFServlet，以服务CXF JAX-WS端点。

设定

JAX-WS入门
<dependency>
    <groupId>org.apache.cxf</groupId>
    <artifactId>cxf-spring-boot-starter-jaxws</artifactId>
    <version>3.1.12</version>
</dependency>

附加配置

使用“ cxf.path ”属性来自定义CXFServlet URL模式

使用“ cxf.servlet.init ”映射属性来自定义CXFServlet属性，例如“ services-list-path”（默认在“ / services”可用）等。

如果需要，可以使用Spring ImportResource批注导入类路径上可用的现有JAX-WS上下文。

API文档

JAX-WS端点支持WSDL。

服务注册中心发布

将JAX-WS端点发布到诸如Netflix Eureka Registry之类的知名服务注册表中，可以类似于注册JAX-RS端点并在JAX-RS Spring Boot Scan演示中所示的方法。

例子

考虑以下配置实例：

JAX-WS配置
package sample.ws;

import javax.xml.ws.Endpoint;
import org.apache.cxf.Bus;
import org.apache.cxf.jaxws.EndpointImpl;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import sample.ws.service.HelloPortImpl;


@Configuration
public class WebServiceConfig {
    @Autowired
    private Bus bus;

    @Bean
    public Endpoint endpoint() {
        EndpointImpl endpoint = new EndpointImpl(bus, new HelloPortImpl());
        endpoint.publish("/Hello");
        return endpoint;
    }
}

 

与此配置一起使用CXF JAX-WS入门程序就足以启用将响应SOAP请求URI的CXF JAX-WS端点，例如

“ http：// localhost：8080 / services / Hello”。

另请参阅JAX-WS Spring Boot演示。

Spring Boot CXF JAX-RS入门

特征

使用“ / services / *” URL模式注册CXF Servlet，以服务CXF JAX-RS端点。

（可选）自动发现JAX-RS根资源和提供程序，并创建JAX-RS端点。

请注意， 本节介绍了在SpringBoot应用程序中使用CXF JAX-RS客户端。

设定

JAX-RS入门
<dependency>
    <groupId>org.apache.cxf</groupId>
    <artifactId>cxf-spring-boot-starter-jaxrs</artifactId>
    <version>3.1.12</version>
</dependency>

附加配置

使用“ cxf.path ”属性来自定义CXFServlet URL模式。

使用“ cxf.servlet.init ”映射属性来自定义CXFServlet属性，例如“ services-list-path”（默认在“ / services”可用）等。

使用“ cxf.jaxrs.server.path ”属性来自定义JAX-RS服务器端点地址（默认为“ /”）。

可以自动发现以JAX-RS @Path和@Provider注释的JAX-RS根资源和提供程序，以及以CXF @Provider注释的本机CXF提供程序。

使用“ cxf.jaxrs.component-scan ”属性从自动发现的JAX-RS根资源和提供程序创建一个JAX-RS端点，这些资源和提供程序被标记为Spring组件（用Spring @Component注释或从@Bean方法创建并返回） ）。

使用“cxf.jaxrs.component扫描 - 包 ”属性来限制自动发现的弹簧部件的被接受为JAX-RS资源和提供程序类。它设置给定bean实例的类必须位于的包的逗号分隔列表。注意，如果设置了此属性，则仅在给定的bean是单例时才有效。它可以与“ cxf.jaxrs.component-scan - beans ”属性一起使用或替代。此属性从CXF 3.1.11开始可用。

使用“ cxf.jaxrs.component-scan - beans ”属性来限制将哪些自动发现的Spring组件接受为JAX-RS资源或提供程序类。它设置了一个以逗号分隔的可接受bean名称的列表-仅当其bean名称在此列表中时，自动发现的组件才会被接受。属性-它可以一起或作为替代“ 套餐 cxf.jaxrs.component扫描 ”使用。此属性从CXF 3.1.11开始可用。

使用“ cxf.jaxrs.classes-scan ”属性从自动发现的JAX-RS根资源和提供程序类创建JAX-RS端点。此类不必使用Spring @Component进行注释。该属性需要带有一个“ cxf.jaxrs.classes-scan-packages ”属性，该属性设置要扫描的软件包的逗号分隔列表。

请注意，尽管“ cxf.jaxrs.component-scan ”和“ cxf.jaxrs.classes-scan ”是互斥的，但“ cxf.jaxrs.component-scan ”可以与“ cxf.jaxrs.classes-scan-packages ”一起使用”属性，以启用自动发现可能标记为Spring组件的JAX-RS资源和提供者的功能。

如果需要，可以使用Spring ImportResource批注导入类路径上可用的现有JAX-RS上下文，而不是自动发现资源。

API文档

昂首阔步

有关如何在SpringBoot中启用Swagger2Feature以及如何自动激活Swagger UI的信息，请参阅CXF Swagger2Feature文档 。

瓦德

如果运行时类路径上有cxf-rt-rs-service-description模块可用，则CXF会自动加载WADL提供程序。

服务注册中心发布

JAX-RS Spring Boot Scan演示中显示了将JAX-RS端点发布到诸如Netflix Eureka Registry之类的知名服务注册表中。

例子

手动配置

考虑以下配置实例：

JAX-RS配置
package sample.rs.service;
import java.util.Arrays;

import org.apache.cxf.Bus;
import org.apache.cxf.endpoint.Server;
import org.apache.cxf.jaxrs.JAXRSServerFactoryBean;
import org.apache.cxf.jaxrs.swagger.Swagger2Feature;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.context.annotation.Bean;

import sample.rs.service.hello1.HelloServiceImpl1;
import sample.rs.service.hello2.HelloServiceImpl2;

@SpringBootApplication
public class SampleRestApplication {
    @Autowired
    private Bus bus;

    public static void main(String[] args) {
        SpringApplication.run(SampleRestApplication.class, args);
    }
 
    @Bean
    public Server rsServer() {
        JAXRSServerFactoryBean endpoint = new JAXRSServerFactoryBean();
        endpoint.setBus(bus);
        endpoint.setAddress("/");
        // Register 2 JAX-RS root resources supporting "/sayHello/{id}" and "/sayHello2/{id}" relative paths
        endpoint.setServiceBeans(Arrays.<Object>asList(new HelloServiceImpl1(), new HelloServiceImpl2()));
        endpoint.setFeatures(Arrays.asList(new Swagger2Feature()));
        return endpoint.create();
    }
}

 

与此配置一起使用CXF JAX-RS入门程序足以启用将响应HTTP请求URI之类的CXF JAX-RS端点，例如

“ http：// localhost：8080 / services / sayHello / ApacheCxfUser ”。上面的代码还使Swagger文档可从“ http：// localhost：8080 / services / swagger.json ”获得。

另请参阅JAX-RS Spring Boot演示。

 

自动配置

如果启用了自动发现，则可以简化“手动配置”部分中显示的“ Spring Boot应用程序”示例。

例如，给定以下application.yml属性：

应用属性
 cxf:
  jaxrs:
    component-scan: true
    classes-scan-packages: org.apache.cxf.jaxrs.swagger

 

该应用程序变得简单：

 

JAX-RS自动配置
package sample.rs.service;
import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class SampleScanRestApplication {
    public static void main(String[] args) {
        SpringApplication.run(SampleScanRestApplication.class, args);
    }
}

 

此应用程序将启用CXF JAX-RS端点，该端点将响应HTTP请求URI，例如

“ http：// localhost：8080 / services / sayHello / ApacheCxfUser ”。上面的代码还使Swagger文档可从“ http：// localhost：8080 / services / swagger.json ”获得。

另请参阅JAX-RS Spring Boot Scan演示。
