CXF提供了几个自定义注释，可用于配置和自定义CXF运行时。

org.apache.cxf.feature。特征

@Features批注用于添加Feature 。请参阅FeaturesList以获取我们“开箱即用”的功能列表，但是您可以轻松创建自己的功能。但是，在许多情况下，这些功能本身都具有注释，可以使用这些注释并提供对配置的更大控制。

org.apache.cxf.interceptor。在拦截器中
org.apache.cxf.interceptor。OutInterceptors，
org.apache.cxf.interceptor。OutFaultInterceptors，
org.apache.cxf.interceptor。错误拦截器

将拦截器添加到用于处理消息的各种链中。有关更多详细信息，请参见拦截器 。

org.apache.cxf.annotations。WSDLDocumentation
org.apache.cxf.annotations。WSDLDocumentationCollection（从2.3开始）

对于从Java接口/代码派生WSDL的“ java优先”方案，这些注释允许将wsd：documentation元素添加到生成的wsdl中的各个位置。

例如：

@WebService
@WSDLDocumentationCollection(
    {
        @WSDLDocumentation("My portType documentation"),
        @WSDLDocumentation(value = "My top level documentation",
                           placement = WSDLDocumentation.Placement.TOP),
        @WSDLDocumentation(value = "My binding doc",
                           placement = WSDLDocumentation.Placement.BINDING)
    }
)
public interface MyService {

    @WSDLDocumentation("The docs for echoString")
    String echoString(String s);

}

org.apache.cxf.annotations。SchemaValidation（从2.3开始）

打开消息的SchemaValidation。默认情况下，出于性能原因，CXF不会针对架构验证消息。通过启用验证，可以更轻松地确定消息与模式不匹配的问题。这些注释可以全局应用于SEI，也可以在SEI和实现中的方法级别应用。

 

在将模式验证应用于客户端和服务器上的传入消息
请求（自2.7.14、3.0.3、3.1开始 ）将架构验证应用于请求消息。此注释将导致验证将应用于传出客户端消息和传入服务器消息。
出将模式验证应用于客户端和服务器上的传出消息
响应（自2.7.14、3.0.3和3.1开始）将架构验证应用于响应消息。此注释将使验证应用于传入的客户端消息和传出的服务器消息。
都将模式验证应用于客户端和服务器上的传入和传出消息
没有禁用所有架构验证。
@WebService
@SchemaValidation(type = SchemaValidationType.BOTH)
public interface MyService {
    Foo validateBoth(Bar data);

    @SchemaValidation(type = SchemaValidationType.NONE)
    Foo validateNone(Bar data);

    @SchemaValidation(type = SchemaValidationType.IN)
    Foo validateIn(Bar data);

    @SchemaValidation(type = SchemaValidationType.OUT)
    Foo validateOut(Bar data);

    @SchemaValidation(type = SchemaValidationType.REQUEST)
    Foo validateRequest(Bar data);

    @SchemaValidation(type = SchemaValidationType.RESPONSE)
    Foo validateResponse(Bar data);
}

 

org.apache.cxf.annotations。数据绑定（自2.2.4开始）

设置与服务关联的DataBinding类。默认情况下，CXF假定您正在使用JAXB数据绑定。但是，CXF支持不同的数据绑定，例如XMLBean，Aegis，SDO，甚至可能更多。可以使用此批注代替配置来选择数据绑定类。

@DataBinding(org.apache.cxf.sdo.SDODataBinding.class)
public interface MyService {
    public commonj.sdo.DataObject echoStruct(
        commonj.sdo.DataObject struct
    );
}

org.apache.cxf.annotations。记录（自2.3版开始）

打开端点的日志记录。可用于控制所记录内容的大小限制以及位置。它支持以下属性：

限制

设置大小限制，在该大小限制后，日志中的消息将被截断。默认值为64K

inLocation

设置记录传入消息的位置。可以是，， 或文件：URL。默认为

outLocation

设置记录外发消息的位置。可以是，， 或文件：URL。默认是

@Logging(limit=16000, inLocation="<stdout>")
public interface MyService {

    String echoString(String s);

}

org.apache.cxf.annotations。GZIP（从2.3版开始）

启用在线数据的GZIP压缩。支持的属性：

阈

不压缩邮件的阈值

力

强制进行GZIP压缩，而不是通过Accept-Encoding标头进行协商

GZIP是经过协商的增强功能。来自客户端的初始请求不会被压缩，但是会添加Accept标头，如果服务器支持，响应将被压缩，随后的任何请求都会被压缩。

org.apache.cxf.annotations。FastInfoset（从2.3开始）

启用在线数据的FastInfoset。支持的属性：

力

强制使用fastinfoset而不是进行谈判。默认为假

FastInfoset是经过协商的增强功能。来自客户端的初始请求将不在fastinfoset中，但是将添加Accept标头，并且如果服务器支持，则响应将在fastinfoset中，并且随后的请求都将在fastinfoset中。

org.apache.cxf.annotations。端点属性
org.apache.cxf.annotations。EndpointProperties（从2.3开始）

将属性添加到端点。诸如WS-Security相关的东西之类的东西可以通过端点属性进行配置。传统上，这些设置是通过spring config中的元素设置的，但是这些注释允许将这些属性配置到代码中。

@WebService
@EndpointProperties(
    {
       @EndpointProperty(key = "my.property", value="some value"),
       @EndpointProperty(key = "my.other.property", value="some other value"),
    })
public interface MyService {
    String echoString(String s);
}

org.apache.cxf.annotations。政策
org.apache.cxf.annotations。政策（自2.3版开始）

用于将WS-Policy片段附加到服务或操作。该策略支持以下属性：

乌里

要求包含策略定义的文件的位置

includeInWSDL

生成wsdl时是否在生成的WSDL中包括策略。默认为true

放置

指定放置策略的位置

faultClass

如果放置为FAULT，则指定该策略将应用于哪个故障

使用自定义Spring配置时，您需要导入META-INF / cxf / cxf-extension-policy.xml

@Policies({
    @Policy(uri = "annotationpolicies/TestInterfacePolicy.xml"),
    @Policy(uri = "annotationpolicies/TestImplPolicy.xml",
            placement = Policy.Placement.SERVICE_PORT),
    @Policy(uri = "annotationpolicies/TestPortTypePolicy.xml", 
            placement = Policy.Placement.PORT_TYPE)
}
)
@WebService
public static interface TestInterface {
    @Policies({
        @Policy(uri = "annotationpolicies/TestOperationPolicy.xml"),
        @Policy(uri = "annotationpolicies/TestOperationInputPolicy.xml", 
                placement = Policy.Placement.BINDING_OPERATION_INPUT),
        @Policy(uri = "annotationpolicies/TestOperationOutputPolicy.xml", 
                placement = Policy.Placement.BINDING_OPERATION_OUTPUT),
        @Policy(uri = "annotationpolicies/TestOperationPTPolicy.xml", 
                placement = Policy.Placement.PORT_TYPE_OPERATION),
        @Policy(uri = "annotationpolicies/TestOperationPTInputPolicy.xml", 
                placement = Policy.Placement.PORT_TYPE_OPERATION_INPUT),
        @Policy(uri = "annotationpolicies/TestOperationPTOutputPolicy.xml", 
                placement = Policy.Placement.PORT_TYPE_OPERATION_OUTPUT)
    }
    )
    int echoInt(int i);
}

org.apache.cxf.annotations。UseAsyncMethod（从2.6.0版开始）

在JAX-WS服务实现对象上使用，以将方法标记为首选方法的“异步”版本而不是同步版本。使用JAX-WS，服务默认使用同步方法，这些方法要求从该方法返回返回值。通过使用@UseAsyncMethod批注标记方法，如果传输支持，则CXF将调用采用AsyncHandler对象的异步版本，并且服务可以在响应准备就绪时调用该处理程序。如果传输不支持CXF继续，则将按常规方式调用同步方法。

    @UseAsyncMethod
    public String greetMeSometime(String me) {
        LOG.info("Executing operation greetMeSometime synchronously");
        System.out.println("Executing operation greetMeSometime synchronously\n");
        return "How are you " + me;
    }

    public Future<?>  greetMeSometimeAsync(final String me, 
                                           final AsyncHandler<GreetMeSometimeResponse> asyncHandler) {
        LOG.info("Executing operation greetMeSometimeAsync asynchronously");
        System.out.println("Executing operation greetMeSometimeAsync asynchronously\n");
        final ServerAsyncResponse<GreetMeSometimeResponse> r 
            = new ServerAsyncResponse<GreetMeSometimeResponse>();
        new Thread() {
            public void run() {
                GreetMeSometimeResponse resp = new GreetMeSometimeResponse();
                resp.setResponseType("How are you " + me);
                r.set(resp);
                System.out.println("Responding on background thread\n");
                asyncHandler.handleResponse(r);
            }
        } .start();
        
        return r; 
    }
