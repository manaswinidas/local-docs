添加SOAP 1.1绑定

使用wsdltosoap

要使用wsdltosoap生成SOAP 1.1绑定， 请使用以下命令：

wsdl2soap [[-?] | [-help] | [-h]] {-iport-type-name} [-bbinding-name] [-doutput-directory] [-ooutput-file]
[-nsoap-body-namespace] [-style (document/rpc)] [-use (literal/encoded)] [-v ] [[ -verbose ] |  [ -quiet ]] wsdlurl

该命令具有以下选项：

选项

解释

-？

显示此实用程序的联机帮助。

-救命

 

-H

 

-导入类型名称

指定应为其生成绑定的portType元素。

-绑定名称

指定生成的SOAP绑定的名称。

-d输出目录

指定放置生成的WSDL文件的目录。

-o输出文件

指定生成的WSDL文件的名称。

-nsoap-body-namespace

当样式为RPC时，指定SOAP主体名称空间。

样式（文档/ RPC）

指定在SOAP绑定中使用的编码样式（文档或RPC）。默认值为文档。

使用（文字/编码）

指定在SOAP绑定中使用的绑定使用（编码或文字）。默认值为文字。

-v

显示工具的版本号。

-冗长

在代码生成过程中显示注释。

-安静

在代码生成过程中禁止注释。

wsdlurl

包含portType元素定义的WSDL文件的路径和名称。

的-i port-type-name和wsdlurl参数是必需的。如果-style rpc指定了参数， -n soap-body-namspace参数也是必需的。所有其他参数都是可选的，可以按任何顺序列出。

wsdltosoap不支持文档/编码的SOAP绑定的生成。

例

如果您的系统有一个接订单的接口，并且提供了一个处理订单的单一操作，则它将在WSDL文档中定义，类似于以下所示。

订购系统界面
<?xml version="1.0" encoding="UTF-8"?>
<definitions name="widgetOrderForm.wsdl"
    targetNamespace="http://widgetVendor.com/widgetOrderForm"
    xmlns="http://schemas.xmlsoap.org/wsdl/"
    xmlns:soap="http://schemas.xmlsoap.org/wsdl/soap/"
    xmlns:tns="http://widgetVendor.com/widgetOrderForm"
    xmlns:xsd="http://www.w3.org/2001/XMLSchema"
    xmlns:xsd1="http://widgetVendor.com/types/widgetTypes"
    xmlns:SOAP-ENC="http://schemas.xmlsoap.org/soap/encoding/">

<message name="widgetOrder">
  <part name="numOrdered" type="xsd:int"/>
</message>
<message name="widgetOrderBill">
  <part name="price" type="xsd:float"/>
</message>
<message name="badSize">
  <part name="numInventory" type="xsd:int"/>
</message>

<portType name="orderWidgets">
  <operation name="placeWidgetOrder">
    <input message="tns:widgetOrder" name="order"/>
    <output message="tns:widgetOrderBill" name="bill"/>
    <fault message="tns:badSize" name="sizeFault"/>
  </operation>
</portType>
...
</definitions>

为orderWidgets生成的SOAP绑定如下所示。

绑定orderWidgets
<binding name="orderWidgetsBinding" type="tns:orderWidgets">
  <soap:binding style="document" transport="http://schemas.xmlsoap.org/soap/http"/>
    <operation name="placeWidgetOrder">
      <soap:operation soapAction="" style="document"/>
      <input name="order">
        <soap:body use="literal"/>
      </input>
      <output name="bill">
        <soap:body use="literal"/>
      </output>
      <fault name="sizeFault">
        <soap:body use="literal"/>
      </fault>
  </operation>
</binding>

此绑定指定使用文档/文字消息样式发送消息。

将SOAP标头添加到SOAP 1.1绑定中

总览

SOAP标头是通过添加定义的soap:header您的默认SOAP 1.1绑定的元素。的soap:header元素是的可选子元素input ， output和fault绑定元素。SOAP标头成为父消息的一部分。通过指定消息和消息部分来定义SOAP头。每个SOAP标头只能包含一个消息部分，但是您可以根据需要插入任意数量的SOAP标头。

句法

SOAP Header Syntax中显示了定义SOAP头的语法 。的message的属性soap:header是消息的限定名称，将从中获取插入到标题中的部分。的part attribute是插入SOAP标头中的消息部分的名称。因为SOAP头始终是文档样式，所以必须使用元素来定义插入SOAP头中的WSDL消息部分。消息和part属性一起完全描述了要插入SOAP标头中的数据。

SOAP标头语法
<binding name="headwig">
  <soap:binding style="document"
                transport="http://schemas.xmlsoap.org/soap/http"/>
    <operation name="weave">
      <soap:operation soapAction="" style="document"/>
      <input name="grain">
        <soap:body .../>
        <soap:header message="QName" part="partName"/>
      </input>
...
</binding>

以及强制性message和part属性soap:header也支持namespace ， use和encodingStyle属性。这些可选属性的功能相同soap:header就像他们为soap:body 。

在正文和标题之间拆分消息

插入SOAP标头中的消息部分可以是合同中的任何有效消息部分。它甚至可以是用作SOAP正文的父消息的一部分。因为您不太可能希望在同一条消息中发送两次信息，所以SOAP绑定提供了一种用于指定插入SOAP正文中的消息部分的方法。

的soap:body元素具有可选属性， parts ，它以空格分隔的零件名称列表。什么时候parts定义后，仅列出的消息部分被插入到SOAP主体中。然后，您可以将其余部分插入SOAP标头中。

使用部分父消息定义SOAP头时，CXF会自动为您填充SOAP头。

例

使用SOAP标头进行SOAP 1.1绑定显示了Ordering System Interface中显示的orderWidgets服务的修改版本。此版本已修改，因此每个订单都有一个xsd:base64binary放置在请求和响应的SOAP标头中的值。SOAP标头定义为keyVal部分来自widgetKey信息。在这种情况下，您将负责在应用程序逻辑中添加SOAP标头，因为它不是输入或输出消息的一部分。

SOAP 1.1与SOAP标头的绑定
<?xml version="1.0" encoding="UTF-8"?>
<definitions name="widgetOrderForm.wsdl"
    targetNamespace="http://widgetVendor.com/widgetOrderForm"
    xmlns="http://schemas.xmlsoap.org/wsdl/"
    xmlns:soap="http://schemas.xmlsoap.org/wsdl/soap/"
    xmlns:tns="http://widgetVendor.com/widgetOrderForm"
    xmlns:xsd="http://www.w3.org/2001/XMLSchema"
    xmlns:xsd1="http://widgetVendor.com/types/widgetTypes"
    xmlns:SOAP-ENC="http://schemas.xmlsoap.org/soap/encoding/">

<types>
  <schema targetNamespace="http://widgetVendor.com/types/widgetTypes"
           xmlns="http://www.w3.org/2001/XMLSchema"
           xmlns:wsdl="http://schemas.xmlsoap.org/wsdl/">
    <element name="keyElem" type="xsd:base64Binary"/>
  </schema>
</types>

<message name="widgetOrder">
  <part name="numOrdered" type="xsd:int"/>
</message>
<message name="widgetOrderBill">
  <part name="price" type="xsd:float"/>
</message>
<message name="badSize">
  <part name="numInventory" type="xsd:int"/>
</message>
<message name="widgetKey">
  <part name="keyVal" element="xsd1:keyElem"/>
</message>

<portType name="orderWidgets">
  <operation name="placeWidgetOrder">
    <input message="tns:widgetOrder" name="order"/>
    <output message="tns:widgetOrderBill" name="bill"/>
    <fault message="tns:badSize" name="sizeFault"/>
  </operation>
</portType>

<binding name="orderWidgetsBinding" type="tns:orderWidgets">
  <soap:binding style="document" transport="http://schemas.xmlsoap.org/soap/http"/>
    <operation name="placeWidgetOrder">
      <soap:operation soapAction="" style="document"/>
      <input name="order">
        <soap:body use="literal"/>
        <soap:header message="tns:widgetKey" part="keyVal"/>
      </input>
      <output name="bill">
        <soap:body use="literal"/>
        <soap:header message="tns:widgetKey" part="keyVal"/>
      </output>
      <fault name="sizeFault">
        <soap:body use="literal"/>
      </fault>
  </operation>
</binding>
...
</definitions>

您可以使用SOAP Header修改SOAP 1.1绑定，以使标头值成为输入和输出消息的一部分，如带有SOAP Header的orderWidgets的SOAP 1.1 Binding中所示。在这种情况下keyVal是输入和输出消息的一部分。在里面soap:body元素的parts属性指定keyVal不要插入体内。但是，它被插入到SOAP标头中。

具有SOAP标头的orderWidget的SOAP 1.1绑定
<?xml version="1.0" encoding="UTF-8"?>
<definitions name="widgetOrderForm.wsdl"
    targetNamespace="http://widgetVendor.com/widgetOrderForm"
    xmlns="http://schemas.xmlsoap.org/wsdl/"
    xmlns:soap="http://schemas.xmlsoap.org/wsdl/soap/"
    xmlns:tns="http://widgetVendor.com/widgetOrderForm"
    xmlns:xsd="http://www.w3.org/2001/XMLSchema"
    xmlns:xsd1="http://widgetVendor.com/types/widgetTypes"
    xmlns:SOAP-ENC="http://schemas.xmlsoap.org/soap/encoding/">

<types>
  <schema targetNamespace="http://widgetVendor.com/types/widgetTypes"
           xmlns="http://www.w3.org/2001/XMLSchema"
           xmlns:wsdl="http://schemas.xmlsoap.org/wsdl/">
    <element name="keyElem" type="xsd:base64Binary"/>
  </schema>
</types>

<message name="widgetOrder">
  <part name="numOrdered" type="xsd:int"/>
  <part name="keyVal" element="xsd1:keyElem"/>
</message>
<message name="widgetOrderBill">
  <part name="price" type="xsd:float"/>
  <part name="keyVal" element="xsd1:keyElem"/>
</message>
<message name="badSize">
  <part name="numInventory" type="xsd:int"/>
</message>

<portType name="orderWidgets">
  <operation name="placeWidgetOrder">
    <input message="tns:widgetOrder" name="order"/>
    <output message="tns:widgetOrderBill" name="bill"/>
    <fault message="tns:badSize" name="sizeFault"/>
  </operation>
</portType>

<binding name="orderWidgetsBinding" type="tns:orderWidgets">
  <soap:binding style="document" transport="http://schemas.xmlsoap.org/soap/http"/>
    <operation name="placeWidgetOrder">
      <soap:operation soapAction="" style="document"/>
      <input name="order">
        <soap:body use="literal" parts="numOrdered"/>
        <soap:header message="tns:widgetOrder" part="keyVal"/>
      </input>
      <output name="bill">
        <soap:body use="literal" parts="bill"/>
        <soap:header message="tns:widgetOrderBill" part="keyVal"/>
      </output>
      <fault name="sizeFault">
        <soap:body use="literal"/>
      </fault>
  </operation>
</binding>
...
</definitions>
