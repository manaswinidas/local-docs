使用方式

总览

Dispatch对象具有两种使用模式 ：

  • 讯息模式
  • 消息有效负载模式（有效负载模式）

您指定的使用方式Dispatch对象确定将详细信息量传递给用户级别的代码。

讯息模式

在消息模式下 ， Dispatch对象可以处理完整的消息。完整的消息包括任何绑定特定的头和包装器。例如，与需要SOAP消息的服务进行交互的消费者需要提供Dispatch对象的invoke()方法一个完全指定的SOAP消息。的invoke()方法还将返回完全指定的SOAP消息。使用者代码负责完成和读取SOAP消息的标头和SOAP消息的信封信息。

当您希望使用JAXB对象时，消息模式不是理想的选择。

您指定一个Dispatch对象通过提供值来使用消息模式java.xml.ws.Service.Mode.MESSAGE创建Dispatch对象时。

有效负载模式

在有效负载模式 （也称为消息有效负载模式）下， Dispatch对象仅适用于消息的有效负载。例如，一个Dispatch在有效负载模式下工作的对象仅适用于SOAP消息的主体。绑定层处理任何绑定级别的包装程序和标头。从返回结果时invoke()绑定级别的包装器和标头已被剥离，仅保留邮件的正文。

当使用不使用特殊包装的绑定时，例如XML绑定，有效负载模式和消息模式将提供相同的结果。

您指定一个Dispatch对象通过提供值使用有效负载模式java.xml.ws.Service.Mode.PAYLOAD当创建Dispatch宾语。

资料类型

总览

Dispatch对象，因为它们是低级对象，未针对与高级使用者API使用相同的JAXB生成类型进行优化。 Dispatch对象可与以下类型的对象一起使用：

  • javax.xml.transform.Source
  • javax.xml.soap.SOAPMessage
  • javax.activation.DataSource
  • 杰克斯

使用Source对象

一种Dispatch对象可以接受并返回从对象派生的对象javax.xml.transform.Source接口。源对象是保存XML文档的低级对象。每Source实现提供了访问存储的XML文档并操纵其内容的方法。以下对象实现了Source接口：

  • DOMSource
  • SAXSource
  • StreamSource

    使用时Source开发人员负责确保所有必需的绑定特定包装器都添加到消息中。例如，与需要SOAP消息的服务进行交互时，开发人员必须确保将所需的SOAP信封添加到传出请求中，并且SOAP信封的内容正确。

使用SOAPMessage对象

Dispatch对象可以使用javax.xml.soap.SOAPMessage当满足以下条件时，对象将被拒绝：

  • 的Dispatch对象正在使用SOAP绑定。
  • 的Dispatch对象正在使用消息模式。

使用DataSource对象

Dispatch对象可以使用实现javax.activation.DataSource满足以下条件时，界面：

  • 的Dispatch对象正在使用HTTP绑定。
  • 的Dispatch对象正在使用消息模式。

DataSource对象提供了一种机制，用于处理来自各种来源（包括URL，文件和字节数组）的MIME类型的数据。

使用JAXB对象

而Dispatch对象旨在用作允许您处理原始消息的低级API，也允许您使用JAXB对象。要使用JAXB对象Dispatch对象必须通过JAXBContext知道如何对使用中的JAXB对象进行封送。的JAXBContext通过时Dispatch对象已创建。

您可以传递由JAXBContext对象作为参数invoke()方法。您还可以将返回的消息强制转换为JAXBContext宾语。

与...合作Dispatch对象

程序

要使用Dispatch对象来调用远程服务，请执行以下操作：

  1. 创建一个Dispatch宾语。
  2. 构造一个请求消息。
  3. 打电话给适当的人invoke()方法。
  4. 解析响应消息。

创建一个Dispatch宾语

创建一个Dispatch对象执行以下操作：

  1. 创建一个Service代表对象wsdl:service定义服务的元素Dispatch对象将进行调用。
  2. 创建Dispatch使用对象Service对象的createDispatch()方法。 
    public Dispatch<T> createDispatch(QName portName, java.lang.Class<T> type, Service.Mode mode)
throws WebServiceException;

    如果您使用的是JAXB对象，则方法签名为createDispatch()是：

    public Dispatch<T> createDispatch(QName portName, javax.xml.bind.JAXBContext context, Service.Mode mode)
throws WebServiceException;

下表描述了用于createDispatch() 。

参数

描述

portName

指定的QName wsdl:port表示服务提供商的元素Dispatch对象将进行调用。

类型

指定对象使用的对象的数据类型Dispatch宾语。

模式

指定使用方式Dispatch宾语。

下面的代码创建了一个Dispatch适用的对象DOMSource有效负载模式下的对象。

package com.mycompany.demo;

import javax.xml.namespace.QName;
import javax.xml.ws.Service;

public class Client {
  public static void main(String args[]) {
    QName serviceName = new QName("http://org.apache.cxf", "stockQuoteReporter");
    Service s = Service.create(serviceName);

    QName portName = new QName("http://org.apache.cxf", "stockQuoteReporterPort");
    Dispatch<DOMSource> dispatch = s.createDispatch(portName,
                                                  DOMSource.class,
                                                  Service.Mode.PAYLOAD);
    ...
  }
}

构造请求消息

使用时Dispatch对象请求必须从头开始。开发人员负责确保消息传递到Dispatch对象匹配目标服务提供商可以处理的请求。这需要有关服务提供商使用的消息的精确知识，以及它需要的头信息（如果有）。

可以由定义消息的WSDL文档或XMLSchema文档提供此信息。尽管服务提供商的差异很大，但可以遵循一些准则：

  • 请求的根元素基于的name属性值wsdl:operation对应于被调用的操作的元素。

    如果所调用的服务使用doc / literal裸消息，则请求的根元素将基于请求的name属性的值。 wsdl:part所引用的元素wsdl:operation元件。

  • 请求的根元素将是名称空间限定的。
  • 如果被调用的服务使用rpc / literal消息，则请求中的顶级元素将不是名称空间限定的。

    顶级元素的子级可能是名称空间限定的。可以肯定的是，您需要检查其架构定义。

  • 如果被调用的服务使用rpc / literal消息，则任何顶级元素都不能为null。
  • 如果所调用的服务使用doc / literal消息，则消息的架构定义将确定是否有任何元素符合名称空间要求。

有关服务如何使用XML消息的更多信息，请参见WS-I Basic Profile。

同步调用

对于进行同步调用以生成响应的使用者，可以使用Dispatch对象的invoke()方法如下所示。

T invoke(T msg) throws WebServiceException;

传递给的响应和请求的类型invoke()方法是在创建Dispatch对象时确定的。例如，如果您创建了一个Dispatch对象使用createDispatch(portName, SOAPMessage.class, Service.Mode.MESSAGE)响应和请求都将是SOAPMessage对象。

使用JAXB对象时，响应和请求可以是提供的任何类型JAXBContext对象可以编组和取消编组。同样，响应和请求可以是不同的JAXB对象。

下面的代码使用以下命令在远程服务上进行同步调用DOMSource宾语。

// Creating a DOMSource Object for the request
DocumentBuilder db = DocumentBuilderFactory.newDocumentBuilder();
Document requestDoc = db.newDocument();
Element root = requestDoc.createElementNS("http://org.apache.cxf/stockExample", "getStockPrice");
root.setNodeValue("DOW");
DOMSource request = new DOMSource(requestDoc);

// Dispatch disp created previously
DOMSource response = disp.invoke(request);

异步调用

Dispatch对象还支持异步调用。与第4章中讨论的高级异步API一样， Dispatch对象可以同时使用轮询方法和回调方法。

使用轮询方法时， invokeAsync()方法返回一个Response可以定期轮询以查看响应是否到达的对象。

Response <T> invokeAsync(T msg) throws WebServiceException;

使用回调方法时， invokeAsync()方法需要AsyncHandler返回时处理响应的实现。

Future<?> invokeAsync(T msg, AsyncHandler<T> handler) throws WebServiceException;

与同步invoke()方法，响应的类型和请求的类型在创建时确定Dispatch宾语。

单向调用

当请求未生成响应时，您可以使用Dispatch对象的invokeOneWay() 。

void invokeOneWay(T msg) throws WebServiceException;

在以下情况下确定用于打包请求的对象类型： Dispatch对象已创建。例如，如果Dispatch使用以下对象创建对象createDispatch(portName, DOMSource.class, Service.Mode.PAYLOAD)该请求将被打包到DOMSource宾语。

使用JAXB对象时，响应和请求可以是提供的任何类型JAXBContext对象可以编组和取消编组。同样，响应和请求可以是不同的JAXB对象。

下面的代码使用JAXB对象对远程服务进行了单向调用。

// Creating a JAXBContext and an Unmarshaller for the request
JAXBContext jbc = JAXBContext.newInstance("com.mycompany.StockExample");
Unmarshaller u = jbc.createUnmarshaller();

// Read the request from disk
File rf = new File("request.xml");
GetStockPrice request = (GetStockPrice)u.unmarshal(rf);

// Dispatch disp created previously
disp.invokeOneWay(request);

操作确定

当使用Dispatch带有WSDL文件的客户端，将在以下情况之一下设置操作名称。

  • javax.xml.ws.handler。MessageContext。使用操作QName设置请求上下文的WSDL_OPERATION属性。
  • 寻址功能已启用（在总线中或在端点处），并且有一个与负载元素匹配的操作。
  • 请求上下文的“ find.dispatch.operation”属性设置为布尔值。为TRUE，并且有一个与有效负载元素匹配的操作。默认情况下未设置此属性。确定正确的操作将影响处理，例如设置正确的SOAPAction或WS-Addressing的Action。
