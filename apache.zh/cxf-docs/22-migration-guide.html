API变更

宙斯盾

Aegis类型系统中的Schema构建已从使用JDom更改为直接使用XmlSchema。这样可以避免3跳JDOM-> DOM-> XmlSchema性能损失，但是需要更改任何自定义Aegis类型对象。

早已弃用的org.codehaus.xfire.aegis.type.java5批注已删除。从XFire迁移应用程序时，您需要进行更新以使用JAXB批注或org.apache.cxf.aegis.type.java5中的等效项。

JAX-RS

JAX-RS运行时/ api已从0.8更新到JAX-RS 1.0级别。为0.8编写的某些代码可能需要修改才能在1.0级别运行。

运行时更改

政策引擎

策略引擎现在默认为打开而不是关闭。因此，如果您正在使用包含WS-Policy断言的WSDL，则现在将默认评估那些断言。如果需要将其关闭，请更新您的配置以包括：

<cxf:bus>
    <cxf:features>
        <p:policies enabled="false"/>
    </cxf:features>
</cxf:bus>

注意：仅当cxf-rt-policy模块可用或使用完整的cxf捆绑罐时，这才适用。如果您使用的是Maven模块，但未引入cxf-rt-policy构件，则策略引擎将不可用。

配置变更

JMS

useJms11标志的默认值已更改为true，因此CXF将在可能的情况下使用JMS 1.1 API。如果使用的是JMS 1.0提供程序，则需要调整配置以将其设置为false，以强制使用1.0 API。

工具变更

Maven WSDL2Java插件

现在，将代码生成到其中的sourceRoot变量的默认位置为“ $ {project.build.directory} / generated-sources / cxf”，以匹配插件的maven最佳实践。
