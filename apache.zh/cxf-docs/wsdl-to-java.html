WSDL到Java

名称

wsdl2java -获取WSDL文档并生成带有完整注释的Java代码，从中可以实现服务。

概要

Usage : wsdl2java -fe|-frontend <front-end-name> -db|-databinding <data-binding-name> 
-wv <wsdl-version> -p <[wsdl-namespace =]package-name>* -sn <service-name> 
-b <binding-file-name>* -reserveClass <class-name>* -catalog <catalog-file-name> 
-d <output-directory> -compile -classdir <compile-classes-directory> -impl -server
-client -clientjar <jar-file-name> -all -autoNameResolution -allowElementReferences|-aer<=true> 
-defaultValues<=class-name-for-DefaultValueProvider> -ant 
-nexclude <schema-namespace [= java-package-name]>* -exsh <(true, false)> -noTypes
-dns <(true, false> -dex <(true, false)> -validate -keep 
-wsdlLocation <wsdlLocation> -xjc<xjc-arguments>* -asyncMethods<[=method1,method2,...]>* 
-bareMethods<[=method1,method2,...]>* -mimeMethods<[=method1,method2,...]>* -noAddressBinding 
-faultSerialVersionUID <fault-serialVersionUID> -exceptionSuper <exceptionSuper> 
-mark-generated -suppress-generated-date -maxExtensionStackDepth <maxExtensionStackDepth>
-h|-?|-help -version|-v -verbose|-V -quiet|-q|-Q 
-wsdlList <wsdlurl>

描述

wsdl2java提取 WSDL文档，并生成完全注释的Java代码，从中可以实现服务。WSDL文档必须具有有效的portType元素，但不需要包含绑定元素或服务元素。使用可选参数，您可以自定义生成的代码。另外，wsdl2java可以生成基于Ant的makefile来构建您的应用程序。

选件

下表回顾了用于验证WSDL文件的选项。

选项

解释

-? ， -h ，-help

显示此实用程序的联机帮助并退出。

-fe frontend-name

指定前端。默认值为JAXWS。当前仅支持JAXWS前端和“ jaxws21”前端来生成与JAX-WS 2.1兼容的代码。

-db databinding-name

指定数据绑定。默认为jaxb。当前支持jaxb，xmlbeans，sdo（sdo-static和sdo-dynamic）和jibx。

-wv wsdl-version

指定wsdl版本。默认值为WSDL1.1。当前仅支持WSDL1.1版本。

-p [ wsdl-namespace= ] PackageName

指定零个或多个用于生成代码的程序包名称。（可选）指定WSDL名称空间到程序包名称的映射。

-sn service-name

用于生成的代码的WSDL服务名称。

-b binding-name

指定JAXWS或JAXB绑定文件或XMLBeans上下文文件。使用多个-b标志来指定多个条目。

-catalog catalog-file-name

指定目录文件以映射导入的wsdl / schema

-d output-directory

指定将生成的代码文件写入的目录。

-compile

编译生成的Java文件。

-classdir compile-class-dir

指定将编译的类文件写入的目录。

-client

为客户主线生成起点代码。

-clientjar jar-file-name

生成包含所有客户端类和wsdl的jar文件；定义-clientJar时，指定的wsdlLocation将不起作用。

-server

生成服务器主线的起点代码。

-impl

为实现对象生成起点代码。

-all

生成所有起点代码：类型，服务代理，服务接口，服务器主线，客户端主线，实现对象和一个Ant build.xml文件。

-ant

指定生成蚂蚁build.xml脚本。

-autoNameResolution

自动解决命名冲突，而无需使用绑定自定义项。

-defaultValues=[DefaultValueProvider impl]

指定为impl和客户端生成默认值。您还可以提供自定义默认值提供程序。默认提供程序是RandomValueProvider

-nexclude schema-namespace [=java-packagename]

生成代码时，请忽略指定的WSDL模式名称空间。可以多次指定此选项。另外，可以选择指定排除的名称空间中描述的类型所使用的Java包名称。

-exsh (true/false)

启用或禁用对隐式SOAP头的处理（即，在wsdl：binding中定义的SOAP头，但在wsdl：portType部分中未定义）。处理SOAP标头需要在类路径上可用的SOAP绑定jar，这在CXF 2.4.x和更早版本中不是默认的。您可能需要向cxf-rt-binding-soap添加依赖项，此标志才能起作用。默认为false。

-dns (true/false)

启用或禁用默认名称空间包名称映射的加载。默认值为true，并且将启用http://www.w3.org/2005/08/addressing=org.apache.cxf.ws.addressing命名空间包映射。

-dex (true/false)

启用或禁用默认排除名称空间映射的加载。默认为true。

-validate

启用在生成代码之前验证WSDL的功能。

-keep

指定代码生成器将不覆盖任何先前存在的文件。您将负责解决所有由此产生的编译问题。

-wsdlLocation wsdlLocation

指定@WebServiceClient批注的wsdlLocation属性的值。

-xjc

指定使用逗号分隔的参数列表，这些列表在使用JAXB数据绑定时直接传递给XJC处理器。可用的XJC插件列表可以使用获取-xjc-X 。

-noAddressBinding

为了与CXF 2.0兼容，此标志指示代码生成器生成较旧的CXF专有WS-Addressing类型，而不是与JAX-WS 2.1兼容的WS-Addressing类型。

-v

显示工具的版本号。

-verbose

在代码生成过程中显示注释。

-quiet

在代码生成过程中禁止注释。

-exceptionSuper

从wsdl：fault元素生成的故障bean的超类（默认为java.lang。例外）

-reserveClass classname

与-autoNameResolution一起使用，为wsdl-to-java定义一个类名，该类名在生成类时不使用。对多个类别多次使用此选项。

-allowElementReferences<=true>

（或-aer）如果为true，则在使用包装样式映射时，请忽略JAX-WS 2.2规范的2.3.1.2（v）节中给出的禁止元素引用的规则。

-asyncMethods=foo,bar,...

随后生成的Java类方法的列表，以允许客户端异步调用，类似于JAX-WS绑定文件中的enableAsyncMapping。

-bareMethods=foo,bar,...

随后生成的具有包装样式的Java类方法的列表（请参见下文），类似于JAX-WS绑定文件中的enableWrapperStyle。

-mimeMethods=foo,bar,...

随后生成的用于启用mime：content映射的Java类方法的列表，类似于JAX-WS绑定文件中的enableMIMEContent。

-faultSerialVersionUID

如何生成故障异常的suid。使用NONE，TIMESTAMP，FQCN或特定数字。默认为“无”。

-mark-generated

将@Generated批注添加到生成的类。

-suppress-generated-date禁止在生成的文件中写入当前时间戳（自CXF 3.2.2版开始）
-maxExtensionStackDepth解析架构扩展时允许的最大堆栈深度。默认值为5。（自CXF 3.3.4起 ）

wsdlurl

用于生成代码的WSDL文件的路径和名称。

您必须将WSDL文档的绝对或相对路径指定为最后一个参数。

例子

wsdl2java HelloWorld.wsdl

wsdl2java -p com.mycompany.greeting Greeting.wsdl

wsdl2java -client HelloWorld.wsdl

（有关Apache Ant和Maven的用法，请参见下文。）

将wsdl2java与Ant结合使用

wsdl2java命令可以包装在Ant目标中，如下所示：

<?xml version="1.0"?>
<project name="cxf wsdl2java" basedir=".">   
   <property name="cxf.home" location ="/usr/myapps/cxf-2.5.1"/>

   <path id="cxf.classpath">
      <fileset dir="${cxf.home}/lib">
         <include name="*.jar"/>
      </fileset>
   </path>
      
   <target name="cxfWSDLToJava">
      <java classname="org.apache.cxf.tools.wsdlto.WSDLToJava" fork="true">
         <arg value="-client"/>
         <arg value="-d"/>
         <arg value="src"/>
         <arg value="MyWSDL.wsdl"/>
         <classpath>
            <path refid="cxf.classpath"/>
         </classpath>
      </java>
   </target>
</project>

请确保为以下选项设置“ fork = true”属性任务如上所示。另外，请记住将每个单词或标志保留在命令行选项中元素（例如，不要使用 ，但将它们分成两部分元素）。

尽管我们建议使用Maven，但请参阅CXF发行版中的antbuild示例 ，以获取使用Ant创建CXF项目的示例。

JAXWS定制

可以使用定制绑定文件来定制由wsdl2java创建的缺省JAX-WS前端。有关示例，请参见samples / jaxws_async中的async_binding.xml文件-如果在运行wsdl2java时指定，它将在SEI中生成异步方法。

问：如果要更改生成的SEI名称怎么办？

答：我们没有命令行选项可以执行此操作，但是您可以使用如下代码段这样的绑定文件来实现此目标

<bindings
    xmlns:xsd="http://www.w3.org/2001/XMLSchema"
    xmlns:wsdl="http://schemas.xmlsoap.org/wsdl/"
    wsdlLocation="hello_world.wsdl"
    xmlns="http://java.sun.com/xml/ns/jaxws">
    <bindings node="wsdl:definitions/wsdl:portType[@name='GreeterSE']">
	<class name="GreeterSEI"/>
    </bindings>
</bindings>

问：如何将绑定文件传递给wsdl2java？

答：如果您通过命令行工具使用wsdl2java：

wsdl2java HelloWorld.wsdl -b my_binding.xml

对于Ant，请按照上面的示例说明如何将“ -b”和“ my_binding.xml”添加为arg元素。

对于Maven，请参见cxf-codegen-plugin

问：如何将xsd：dateTime映射到java.util。日期？
嗯，人们不喜欢XMLGregorianCalendar，它是xsd：dateTime（以及xsd：time和xsd：date）的默认映射，
我们可以使用jaxws定制来更改默认映射，这是一些示例绑定文件
如果您在wsdl中包含架构，则可以使用以下绑定文件：

<jaxws:bindings wsdlLocation="YOUR_WSDL_LOCATION"
          xmlns:jaxws="http://java.sun.com/xml/ns/jaxws"
          xmlns:xs="http://www.w3.org/2001/XMLSchema"
          xmlns:jxb="http://java.sun.com/xml/ns/jaxb"
          xmlns:wsdl="http://schemas.xmlsoap.org/wsdl/">
  <jaxws:bindings  node="wsdl:definitions/wsdl:types/xs:schema[@targetNamespace='THE_NAMESPACE_OF_YOUR_SCHEMA']">
      <jxb:globalBindings xmlns:jxb="http://java.sun.com/xml/ns/jaxb" xmlns:xs="http://www.w3.org/2001/XMLSchema">
        <jxb:javaType name="java.util.Date" xmlType="xs:dateTime"
                      parseMethod="org.apache.cxf.xjc.runtime.DataTypeAdapter.parseDateTime"
                      printMethod="org.apache.cxf.xjc.runtime.DataTypeAdapter.printDateTime"/>
      </jxb:globalBindings>
  </jaxws:bindings>
</jaxws:bindings>

这需要附加的依赖项：

<dependency>
   <groupId>org.apache.cxf.xjc-utils</groupId>
   <artifactId>cxf-xjc-runtime</artifactId>
</dependency>


如果要使用java.util。日历，只需更改org.apache.cxf.xjc.runtime。将DataTypeAdapter转换为javax.xml.bind。DatatypeConverter，并将名称值更改为“ java.util。日历”

如果您的架构超出了wsdl，请尝试以下示例：

<jxb:bindings version="2.0"
              xmlns:jxb="http://java.sun.com/xml/ns/jaxb"
              xmlns:xs="http://www.w3.org/2001/XMLSchema">
  <jxb:bindings schemaLocation="file:<path><name>.xsd" node="/xs:schema">
    <jxb:globalBindings xmlns:jxb="http://java.sun.com/xml/ns/jaxb" xmlns:xs="http://www.w3.org/2001/XMLSchema">
      <jxb:javaType name="java.util.Date" xmlType="xs:dateTime" 
                    parseMethod="org.apache.cxf.xjc.runtime.DataTypeAdapter.parseDateTime" 
                    printMethod="org.apache.cxf.xjc.runtime.DataTypeAdapter.printDateTime"/>
    </jxb:globalBindings>
  </jxb:bindings>
</jxb:bindings>


问：如何将生成的Web服务方法调用从包装器样式切换到非包装器样式（反之亦然）？

答：创建一个外部绑定文件并设置其值根据需要更改为true或false：

<jaxws:bindings
    xmlns:xsd="http://www.w3.org/2001/XMLSchema"
    xmlns:wsdl="http://schemas.xmlsoap.org/wsdl/"
    wsdlLocation="your.wsdl"
    xmlns="http://java.sun.com/xml/ns/jaxws"
    xmlns:jaxws="http://java.sun.com/xml/ns/jaxws">
    <enableWrapperStyle>false</enableWrapperStyle>
</jaxws:bindings>

或者，您可以将此指令直接嵌入WSDL文件中，作为wsdl：portType的直接子代：

<wsdl:portType name="MyWebServicePortType">
    <jaxws:bindings xmlns:jaxws="http://java.sun.com/xml/ns/jaxws">
        <jaxws:enableWrapperStyle>false</jaxws:enableWrapperStyle>
        ... other binding settings if needed ...
    </jaxws:bindings>
    <wsdl:operation name="sayHello">
    ...
</wsdl:portType>

注意： JAX-WS 2.1规范中定义的“包装器样式”和“非包装器样式”的含义可能是违反直觉的。包装器样式表示请求消息中的每个数据元素都有其自己的Java参数，而非包装器样式表示包含所有数据元素的单个Java对象充当Web服务方法调用的唯一参数。（有关示例，请参见规范的图2.2。）另外， 请注意包装器样式并非总是可用 ，必须满足规范第2.3.1.2节（“包装器样式”）中指定的WSDL标准，否则将仅生成非包装器样式。

问：JAXWS定制绑定文件还能更改什么？

答：您可以在JAX-WS规范的第8章中找到自定义项的完整列表。
