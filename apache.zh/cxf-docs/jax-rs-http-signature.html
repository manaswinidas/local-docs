介绍

自CXF 3.3.0起，CXF添加了对HTTP签名草案规范的支持。这提供了除XML安全性和JOSE之外的提供消息完整性的替代方法，并且实际上提供了一种更好的消息完整性度量，因为它允许在签名中包含HTTP标头，包括HTTP方法和路径。

Maven依赖

使用CXF的HTTP签名实现需要以下依赖项。

<dependency>
  <groupId>org.apache.cxf</groupId>
  <artifactId>cxf-rt-rs-security-http-signature</artifactId>
  <version>3.3.2</version>
</dependency>

组态

CXF在客户端和服务端都支持HTTP签名创建和验证。通过消化有效负载并将结果插入“摘要”标头中来支持有效负载完整性，该标头也由HTTP签名进行签名。

提供者

要在CXF中启用HTTP签名，必须将以下提供程序之一添加到客户端或端点：

  • 客户端/服务出站签名创建：org.apache.cxf.rs.security.httpsignature.filters。CreateSignatureInterceptor
  • 客户端入站签名验证：org.apache.cxf.rs.security.httpsignature.filters。VerifySignatureClientFilter
  • 服务入站签名验证：org.apache.cxf.rs.security.httpsignature.filters。验证签名过滤器

例如在代码中：

WebClient配置
CreateSignatureInterceptor signatureFilter = new CreateSignatureInterceptor();

String address = "http://localhost:" + PORT + "/httpsig/bookstore/books";
WebClient client =
    WebClient.create(address, Collections.singletonList(signatureFilter), busFile.toString());

在春季：

弹簧配置
<bean id="httpSignatureVerifier" class="org.apache.cxf.rs.security.httpsignature.filters.VerifySignatureFilter">
    <property name="messageVerifier" ref="messageVerifier"/>
</bean>
    
<jaxrs:server address="http://localhost:${testutil.ports.jaxrs-httpsignature}/httpsig">
   <jaxrs:serviceBeans>
       <ref bean="serviceBean"/>
   </jaxrs:serviceBeans>
   <jaxrs:providers>
       <ref bean="httpSignatureVerifier"/>
   </jaxrs:providers>
</jaxrs:server>

细粒度配置

除了添加所需的提供程序（参见上文）之外，我们还需要配置它们（例如，要使用的密钥，要签名的标头等）。有两种选择方法。在本节中，我们将研究直接配置提供程序，这将允许更精细的配置。有关使用配置属性从密钥库检索密钥的替代方法，请参见下文。

对于出站签名，我们需要使用MessageSigner实例配置CreateSignatureInterceptor提供程序。MessageSigner包含许多不同的构造函数，可以根据所需的功能使用它们。至少，我们需要提供PrivateKey实例来对消息进行签名，并提供规范中定义的“密钥ID”。我们还可以提供签名算法名称-如果未指定，则默认为“ rsa-sha256”。同样，我们可以提供安全提供程序名称，默认为“ SunRsaSign”。

我们还可以提供要签名的HTTP标头列表。默认情况下，它会签名CXF提供给它的所有HTTP标头。在客户端，它还将通过“（request-target）”对HTTP方法和路径进行签名。

这是测试的示例：

CreateSignatureInterceptor signatureFilter = new CreateSignatureInterceptor();
KeyStore keyStore = KeyStore.getInstance("JKS");
keyStore.load(ClassLoaderUtils.getResourceAsStream("keys/alice.jks", this.getClass()),
    "password".toCharArray());
PrivateKey privateKey = (PrivateKey)keyStore.getKey("alice", "password".toCharArray());
assertNotNull(privateKey);

MessageSigner messageSigner = new MessageSigner(keyId -> privateKey, "alice-key-id");
signatureFilter.setMessageSigner(messageSigner);

String address = "http://localhost:" + PORT + "/httpsig/bookstore/books";
WebClient client =
    WebClient.create(address, Collections.singletonList(signatureFilter), busFile.toString());
client.type("application/xml").accept("application/xml");

为了进行签名验证，我们需要为VerifySignatureClientFilter和VerifySignatureFilter实例提供一个MessageVerifier实例。至少，我们需要使用KeyProvider实例配置MessageVerifier，该实例是一个接口，提供给定消息中存在的“密钥ID”，以验证签名所需的密钥。根据MessageSigner，我们还可以指定所需的签名算法以及安全提供程序。它的默认值与上述MessageSigner记录的值相同。我们还可以指定必须签名的HTTP标头列表。除此列表外，默认行为是要求对“摘要”标头进行签名（除非使用GET或HEAD的HTTP方法进行服务请求，并且除非服务响应的状态为204或不是“ OK” ），以及客户端请求的“（请求目标）”标头。可以通过将MessageVerifier的boolean addDefaultRequiredHeaders属性设置为false来禁用此默认行为。

这是测试的示例：

<bean id="publicKeyProvider" class="org.apache.cxf.systest.jaxrs.security.httpsignature.CustomPublicKeyProvider"/>
<bean id="messageVerifier" class="org.apache.cxf.rs.security.httpsignature.MessageVerifier">
    <constructor-arg>
        <ref bean="publicKeyProvider"/>
    </constructor-arg>
    <constructor-arg>
        <util:list>
            <value>(request-target)</value>
        </util:list>
    </constructor-arg>
</bean>
<bean id="httpSignatureVerifier" class="org.apache.cxf.rs.security.httpsignature.filters.VerifySignatureFilter">
    <property name="messageVerifier" ref="messageVerifier"/>
</bean>
    
<jaxrs:server address="http://localhost:${testutil.ports.jaxrs-httpsignature}/httpsig">
    <jaxrs:serviceBeans>
        <ref bean="serviceBean"/>
    </jaxrs:serviceBeans>
    <jaxrs:providers>
        <ref bean="httpSignatureVerifier"/>
    </jaxrs:providers>
</jaxrs:server>

配置属性

如上一节所述，配置MessageSigner和MessageVerifier实例的一种替代方法是改为使用以下安全配置属性。这些属性依赖于从存储在本地文件系统中的密钥库中获取用于签名消息的私钥以及用于验证消息的公钥。请注意，“密钥ID”将被忽略，我们将使用配置属性中定义的公共密钥来验证消息。

以下配置属性可用于通过各种过滤器配置HTTP签名。请注意，它们大部分与JAX-RS JOSE共享。

配置标签默认描述
rs.security.keystore
要使用的Java KeyStore对象。如果要动态传递KeyStore对象，则使用此配置标签。

rs.security.keystore.type

JKS

密钥库类型。

rs.security.keystore.password
访问密钥库所需的密码。
rs.security.keystore.alias
与要使用的密钥相对应的密钥库别名。
rs.security.keystore.file
密钥库文件的路径。
rs.security.key.password
访问私钥（在密钥库中）所需的密码。
rs.security.key.password.provider
对PrivateKeyPasswordProvider实例的引用，该实例用于检索访问密钥的密码。
rs.security.signature.out.properties

用于创建Compact或JSON签名的签名属性文件。如果未指定，则回退到“ rs.security.signature.properties”。

rs.security.signature.in.properties

用于Compact或JSON签名验证的签名属性文件。如果未指定，则回退到“ rs.security.signature.properties”。

rs.security.signature.properties
用于Compact或JSON签名创建/验证的签名属性文件。
rs.security.signature.algorithmrsa-sha256要使用的签名算法。
rs.security.http.signature.key.id
签名密钥ID。这是出站端的必需配置选项。
rs.security.http.signature.out.headers所有标头，包括“（（请求目标）”）

字符串值列表，该值与将在出站请求中签名的HTTP标头列表相对应。

rs.security.http.signature.in.headers客户请求的“摘要”和“（请求目标）”。

字符串值列表，该值与必须在入站请求中签名的HTTP标头列表相对应。默认情况下，客户请求必须签名“（request-target）”。另外，除非是GET / HEAD请求，否则客户请求必须签署“摘要”。服务响应必须为除204之外的所有“正常”状态代码签名“摘要”。

rs.security.http.signature.digest.algorithmSHA-256消化有效负载时要使用的摘要算法。

这是一个Java示例：

List<Object> providers = new ArrayList<>();
providers.add(new CreateSignatureInterceptor());
providers.add(new VerifySignatureClientFilter());
String address = "http://localhost:" + PORT + "/httpsigresponse/bookstore/books";
WebClient client = WebClient.create(address, providers, busFile.toString());
client.type("application/xml").accept("application/xml");

Map<String, Object> properties = new HashMap<>();
properties.put("rs.security.signature.out.properties",
            "org/apache/cxf/systest/jaxrs/security/httpsignature/alice.httpsig.properties");
properties.put("rs.security.signature.in.properties",
            "org/apache/cxf/systest/jaxrs/security/httpsignature/bob.httpsig.properties");
WebClient.getConfig(client).getRequestContext().putAll(properties);

其中“ alice.httpsig.properties”如下所示：

rs.security.keystore.type=jks
rs.security.keystore.password=password
rs.security.keystore.alias=alice
rs.security.keystore.file=keys/alice.jks
rs.security.key.password=password
rs.security.http.signature.key.id=alice-key-id

这是一个春季示例：

<jaxrs:server address="http://localhost:${testutil.ports.jaxrs-httpsignature}/httpsigresponseprops">
    <jaxrs:serviceBeans>
        <ref bean="serviceBean"/>
    </jaxrs:serviceBeans>
    <jaxrs:providers>
        <bean class="org.apache.cxf.rs.security.httpsignature.filters.VerifySignatureFilter" />
        <bean class="org.apache.cxf.rs.security.httpsignature.filters.CreateSignatureInterceptor" />
    </jaxrs:providers>
    <jaxrs:properties>
        <entry key="rs.security.signature.in.properties" 
               value="org/apache/cxf/systest/jaxrs/security/httpsignature/alice.httpsig.properties" />
        <entry key="rs.security.signature.out.properties" 
               value="org/apache/cxf/systest/jaxrs/security/httpsignature/bob.httpsig.properties" />
    </jaxrs:properties>
</jaxrs:server>
