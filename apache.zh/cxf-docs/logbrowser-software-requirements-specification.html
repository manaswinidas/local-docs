介绍

该规范记录了日志浏览器项目的系统级别要求。本规范的目的是提供系统概述，包括定义，目标，目的，上下文和主要功能。

该规范分为以下几节：

  • 简介 ，向读者介绍日志浏览器的规范。
  • 总体描述 ，它提供了日志浏览器的简要，高级描述，包括其定义，业务目标，业务目标，上下文和功能。
  • 特殊要求 ，它根据用例模型指定功能系统要求，用例模型由每个外部用例和用例路径组成。

目的

该SRS描述了日志浏览器1.0版的软件和非功能要求。该文档供Apache CXF社区使用，该社区将实现并验证系统的正确功能。除非另有说明，否则此处指定的所有要求都是最高优先级，并且已针对1.0版进行了提交。

Apache CXF是一个开放源代码服务框架。这是一个非常高级的库，其中包含许多其他工具和插件。但是，对于不是开发人员的人员（例如管理员，测试人员等）来说，维护和管理工具的数量仍然不足。理想的解决方案是创建一组Web应用程序来执行管理任务-例如浏览日志，浏览请求/响应等等

该项目的目标是从根本上改善应用程序的查看日志方式。感谢您的支持，Apache CXF库将更加有用，并且维护起来也更加容易。

用户利益：

  • 全新的日志查看方式-比旧的基于文本的样式熟悉；
  • 通过网络浏览器（从任何软件或硬件平台）轻松访问；
  • 改善日常工作；

产品范围

该产品必须易于最终用户使用。为了能够广泛访问应用程序，新系统将设计为Web应用程序。

但是，该产品不是独立的应用程序。它必须与现有应用程序集成-需要一些配置过程。只要本文档中指定的Web浏览器仍然可用，就将设计使用日志浏览器。

该产品还将包括项目站点上的用户指南。

总体描述

产品观点

在某些方面，该应用程序是保存在服务器上的下一代旧的基于文本的查看服务器日志文件（或几个文件）。

日志浏览器是基于服务器的应用程序，带有基于Web的客户端，用于最终用户访问。服务器部分将建立AtomPullBean和BootstrapStorage组件。 AtomPullBean组件负责收集和发布日志条目。鉴于BootstrapStorage组件负责存储用户的设置。所有这些组件都可以在您的软件中使用-它们不是独立的应用程序。

日志浏览器将能够订阅许多终结点，这些终结点提供了完整的日志列表。假设您的应用程序发布了两个端点。第一个通道包含任何活动的所有条目，但是第二个通道仅包含来自DAO层的条目。

选择指定的端点后，用户将可以搜索，在页面之间导航或查看条目详细信息。所有这些功能将在同一文档的后面定义和描述。实际的用户界面将在“ 软件设计说明”部分中进行描述。

产品功能

该应用程序提供的主要功能：

  • 管理端点列表（添加，删除，编辑）；
  • 按词组，日期范围或入门级别（信息，调试，警告，错误）搜索；
  • 在条目页面之间导航。有链接：下一页，上一页，第一页和最后一页；
  • 选择后显示条目详细信息；

用户类别和特征

目标用户是测试人员和管理员。

操作环境

建议的解决方案将在客户端/服务器模型中实现。搜索和信息存储的所有处理都将在服务器端完成。另外，设置将存储在本地的cookie文件中，以提高性能。但是，服务器将存储设置的备份以进行恢复；

  • 服务器 -您的应用程序必须使用Java平台开发，并且应正确配置以启用日志浏览器。配置说明将在项目现场提供。 BootstrapStorage组件将用户的设置存储在外部数据源中，例如XML文件。因此，运行时环境具有足够的可用磁盘空间非常重要。默认AtomPullBean组件存储有限数量的日志条目。开发人员必须实施ReadWriteStorage接口。如果需要将条目存储在磁盘上；
  • 客户端 -基于Web的客户端与大多数操作系统兼容，并且需要可正常安装Firefox 1.0，Internet Explorer 6，Safari 2.0，Opera 9.0或更高版本。客户端计算机应匹配或超过为与客户端应用程序一起使用的Web浏览器指定的系统要求；

设计和实施约束

在为日志浏览器项目实施合适的解决方案时，一些问题限制了我们的选择。由于具有相同的原始策略 ，因此无法从其他主机（除了提供静态内容的主机）订阅端点。只要有可能，该应用程序应仅保留键盘输入设备的全部可用性。

用户文件

最终用户的文档将可从项目站点获得。还将提供在线教程。应用程序将包含指向文档页面的链接。

假设和依赖

我们假设对应用程序的任何使用都会在完全符合此规范的环境中发生。该应用程序将在满足或超过“ 操作环境”部分中列出的要求的客户端计算机上使用。该应用程序的服务器组件将在匹配或超过“ 操作环境”部分中为服务器列出的要求的计算机系统上运行 。

软件系统属性

本部分指定了与用例模型中记录的特定功能要求无关的所需系统质量因子。

  • 可用性和可靠性 -服务器内置在您的应用程序中。只要您的应用程序正在运行，服务器就应该可用；
  • 可重用性 -产品应该易于扩展-我们考虑添加请求/响应浏览支持。将来，日志浏览器将成为下一类管理工具的基础。
  • 安全性 -必须有一个选项将端点设置为安全。通过使用WSSE UserName HTTP，对该信息的访问必须是私有的；
  • 可维护性 -可以毫无问题地进行对最新版本的更新。用户设置应使用以下方式远程保存BootstrapStorage 。另外BootstrapStorage应该将信息存储在外部数据源中以便于备份；
  • 可移植性 -基于Web的客户端应可在任何平台上运行。唯一的限制是使用一种受支持的Web浏览器（在Operating Environment中进行了介绍 ）。另一方面，您的软件必须使用Java平台进行开发，以便与AtomPullBean和BootstrapStorage组件;
  • 性能 -应用程序的性能将取决于客户端计算机可以访问的Internet访问的速度和类型。响应应足够快，以免打断用户的思维流程；
