在撰写本文时，生成器和运行时存在许多重大限制。

需要XMLHttpRequest

该实用程序已经与记录在案的XMLHttpRequest对象进行了测试。基于Mozilla的浏览器和Internet Explorer的最新版本支持此对象。该代码包括对ActiveXObject（“ MSXML2。XMLHTTP.6.0“），但我们并未积极测试此后备广告。如果需要使用Internet Explorer的较早版本，则可以自行调整和测试实用程序。但是，我们很乐意接受这方面的贡献。

绑定：仅Soap 1.1

生成器仅支持SOAP 1.1。Soap 1.2支持可能很快就会到来。其他绑定具有可疑的价值。

不认证

假设，浏览器中的XMLHttpRequest对象支持身份验证。实用程序不会提供此支持。

巴洛克式架构

生成器中不支持使用“序列”以外的粒子的复杂类型。不支持属性。如果您是先进行模式编码，或者使用的是非常复杂的JAXB注释，则可以提出其他粒子，并且生成器将拒绝为这些类型生成代码。

没有跨脚本

如其他页面所述，该实用程序不包含任何在某些环境中启用交叉脚本的常用技巧。如果需要交叉脚本，则需要找到另一个工具包或改编JavaScript代码。
