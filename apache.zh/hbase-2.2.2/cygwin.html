介绍

Apache HBase（TM）是一个分布式的，面向列的存储，以Google的BigTable为模型。Apache HBase基于Hadoop构建于MapReduce和分布式文件系统实现之上 。所有这些项目都是开源的，并且是Apache Software Foundation的一部分 。

作为分布式的大型平台，Hadoop和HBase项目主要关注用于生产安装的* nix 环境 。但是，这两个项目都是用Java开发的，可以跨平台完全移植 ，因此也可以移植到Windows操作系统 。为了便于开发，这些项目依靠Cygwin在Windows上具有类似于* nix的环境来运行Shell脚本。

目的

本文档介绍了使用Cygwin作为用于测试和开发的多合一单节点安装在Windows上运行Apache HBase的复杂性 。另一方面，《 HBase 概述和快速入门》指南在解释如何在更复杂的部署方案中设置HBase方面大有帮助。

安装

要在Windows上运行Apache HBase，需要3种技术： Java，Cygwin和SSH 。以下各段详细介绍了上述每种技术的安装。

爪哇

HBase取决于Java平台标准版6发行版 。因此，目标系统必须至少具有Java Runtime Environment（JRE）；但是，如果该系统也将用于开发，则首选Jave开发套件（JDK）。您可以从Sun的下载页面下载最新版本。安装是一个简单的GUI向导，可指导您完成整个过程。

西格温

Cygwin可能是该解决方案堆栈中最奇怪的技术。它提供了一个动态链接库，可以模拟Windows上大多数* nix环境。最重要的是，提供了一堆最常用的* nix工具。DLL与工具结合在一起，在Windows上形成了一个非常类似于* nix的环境。

对于安装，Cygwin提供了setup.exe实用程序 ，该实用程序可跟踪目标系统上所有已安装组件的版本，并提供用于从Cygwin的镜像站点安装或更新所有内容的机制。

为了支持安装， setup.exe实用程序在目标系统上使用2个目录。Cygwin的根目录（默认为C：\ cygwin） ，在最终的Cygwin安装中将变为/ 。和本地软件包目录（例如C：\ cygsetup ，这是setup.exe在安装软件包之前将软件包存储在其中的缓存）。缓存不得与Cygwin根目录位于同一文件夹。

执行以下步骤来安装Cygwin，《 Cygwin用户指南》 第二章详细介绍了这些步骤：

  1. 确保您在目标系统上具有管理员特权。
  2. 选择并创建您的Root和Local Package目录。一个好的建议是使用C：\ cygwin \ root和C：\ cygwin \ setup文件夹。
  3. 下载setup.exe实用程序并将其保存到本地程序包目录。
  4. 运行setup.exe实用程序，
    1. 选择从Internet安装选项，
    2. 选择您的Root和Local Package文件夹
    3. 并选择合适的镜子。
    4. 暂时不要选择任何其他软件包，因为我们现在仅想安装Cygwin。
    5. 等待下载并安装
    6. 完成安装
  5. （可选）您现在还可以在“开始”菜单中添加一个快捷方式，以指向“ 本地程序包”文件夹中的setup.exe实用程序。
  6. 添加CYGWIN_HOME系统范围的环境变量，该变量指向您的Root目录。
  7. 将％CYGWIN_HOME％\ bin添加到PATH环境变量的末尾。
  8. 更改环境变量后，重新启动系统，否则操作系统将无法找到Cygwin实用程序。
  9. 通过运行新创建的快捷方式或Root文件夹中的Cygwin.bat命令来测试安装。您应该最终在运行Bash shell的终端窗口中。通过发出以下命令来测试外壳：
    1. cd /应该带您到Cygwin的root目录；
    2. LS命令，该命令应列出当前目录中的所有文件和文件夹。
    3. 使用exit命令结束终端。
  10. 需要时，要卸载 Cygwin，您只需删除“ 根和本地软件包”目录以及在安装过程中创建的快捷方式 。

SSH协议

HBase（和Hadoop）依靠SSH进行进程间/节点通信并启动远程命令 。SSH将通过Cygwin在目标系统上配置，该系统支持将Cygwin程序作为Windows服务运行！

  1. 重新运行setup.exe 实用程序 。
  2. 保留所有参数不变，使用“ 下一步”按钮跳过向导，直到显示“ 选择软件包”面板。
  3. 最大化窗口，然后单击“ 查看”按钮切换到列表视图，该视图在Package上按顺序进行排序，从而使查找所需的软件包更加容易。
  4. 单击状态字（通常为Skip ），选择以下软件包，以将其标记为安装。使用“ 下一步”按钮下载并安装软件包。
    1. OpenSSH的
    2. tcp_wrappers
    3. diffutils
    4. zlib
  5. 等待安装完成并完成安装。

HBase的

从网站下载最新版本的Apache HBase。由于Apache HBase可分发文件只是压缩的归档文件，因此安装就像解压缩归档文件一样简单，因此它最终位于其最终安装目录中。注意，必须在Cygwin中安装HBase，一个好的目录建议是使用/ usr / local / （或Windows slang中的[ 根目录] \ usr \ local ）。您应该以/ usr / local / hbase-结尾在Cygwin中安装。

这样就完成了安装。我们继续进行配置。

组态

剩下3个部分需要配置： Java，SSH和HBase本身。以下各段详细解释了eacht主题。

爪哇

通常，在shell脚本（例如* nix和Windows）中要记住的一件重要事情是，由于需要转义和引用这些字符和字符串，包含空格的路径名称的管理，操作和组装非常困难。因此，我们尝试远离路径名中的空格。 * nix环境可以通过使用符号链接非常轻松地帮助我们。

  1. 通过使用以下命令并替换您选择的Java环境的名称，在/ usr / local中创建指向Java主目录的链接：
    LN -s /cygdrive/c/Program\ Files/Java/<jre name> /usr/local/<jre name>
  2. 通过将目录更改为Java文件夹CD / usr / local /来测试Java安装并发出命令./bin/java -version 。这应该输出所选JRE的版本。

配置SSH非常复杂，但主要是默认情况下将其作为Windows服务启动的问题。

  1. 在Windows Vista及更高版本上，通过右键单击a使用以管理员身份运行的快捷方式，确保以提升的特权运行Cygwin shell。
  2. 首先，我们必须确保某些重要文件的权限正确。使用下面的命令。您可以通过对不同文件使用LS -L命令来验证所有权限。另外，请注意外壳中的自动完成功能在这种情况下非常方便。
    1. chmod + r / etc / passwd使密码文件对所有人可读
    2. chmod u + w / etc / passwd使密码文件对所有者可写
    3. chmod + r / etc / group使组文件对所有人可读
    1. chmod u + w / etc / group使组文件对所有者可写
    1. chmod 755 / var使var文件夹对所有者可写，并且对所有人可读并可执行
  3. 使用您喜欢的编辑器编辑/etc/hosts.allow文件（为什么外壳中没有VI！）并确保以下两行位于PARANOID行之前：
    1. ALL：本地主机127.0.0.1/32：允许
    2. ALL：[:: 1] / 128：允许
  4. 接下来，我们必须使用脚本ssh-host-config 配置SSH 。
    1. 如果此脚本要求覆盖现有的/ etc / ssh_config ，请回答yes 。
    2. 如果此脚本要求覆盖现有的/ etc / sshd_config ，请回答yes 。
    3. 如果此脚本要求使用特权分离，请回答yes 。
    4. 如果此脚本要求将sshd作为服务安装，请回答yes 。确保您以管理员身份启动Shell！
    5. 如果此脚本要求CYGWIN值，则只需默认为ntsec 。
    6. 如果此脚本要求创建sshd帐户，请回答yes 。
    7. 如果此脚本要求使用其他用户名作为服务帐户，请回答否，因为默认值就足够了。
    8. 如果此脚本要求创建cyg_server帐户，请回答yes 。输入该帐户的密码。
  5. 使用net start sshd或cygrunsrv --start sshd 启动SSH服务 。请注意， cygrunsrv是使进程作为Windows服务运行的实用程序。确认您看到一条消息，指出CYGWIN sshd服务已成功启动。
  6. 使用以下命令协调Windows和Cygwin 用户帐户 ：
    1. mkpasswd -cl> / etc / passwd
    2. mkgroup --local> / etc / group
  7. 测试 SSH的安装：
    1. 打开一个新的Cygwin终端
    2. 使用命令whoami验证您的userID
    3. 发出ssh localhost连接到系统本身
      1. 出示服务器指纹后回答是
      2. 出现提示时输入密码
      3. 在远程会话中测试一些命令
      4. exit命令应该带您回到Cygwin中的第一个shell
    4. 退出应终止Cygwin外壳。

HBase的

如果以前的所有配置均能正常工作，我们只需要对HBase配置文件进行一些修补即可在Windows / Cygwin上正确解析。此处引用的所有文件和路径都从HBase [ 安装目录]作为工作目录开始。
  1. HBase使用./conf/ hbase-env.sh来配置其对运行时环境的依赖关系。复制并取消注释以下几行，使其位于原始行的下方，并进行更改以适合您的环境。他们应阅读如下内容：
    1. 导出JAVA_HOME = / usr / local /
    2. 导出HBASE_IDENT_STRING = $ HOSTNAME，因为这很可能不会包含空格。
  2. HBase使用./conf/ hbase-default.xml文件进行配置。由于JVM在Windows上运行，因此某些属性无法解析到现有目录。这是使用Cygwin时要记住的主要问题：在外壳中，所有路径都是* nix相似的，因此相对于根/ 。但是，要在Windows中使用的每个参数都会自行处理，因此必须是Windows设置，因此类似C：\ -。更改配置文件中的以下属性，并在必要时调整路径以符合您自己的安装：
    1. hbase.rootdir必须读取例如file：/// C：/ cygwin / root / tmp / hbase / data
    2. hbase.tmp.dir必须读取C：/ cygwin / root / tmp / hbase / tmp
    3. hbase.zookeeper.quorum必须读取127.0.0.1，因为出于某些原因， 本地主机似乎无法在Cygwin上正确解析。
  3. 确保已配置的hbase.rootdir和hbase.tmp.dir 目录存在并且具有正确的权限 ，例如通过在它们上发出chmod 777 。

这将结束使用Cygwin在Windows上安装和配置Apache HBase的过程。因此，现在该进行测试了 。

  1. 启动Cygwin 终端 （如果尚未启动）。
  2. 使用CD / usr / local / hbase-将目录更改为HBase 安装 ，最好使用自动补全功能。
  3. 使用命令./bin/start-hbase.sh 启动HBase
    1. 当提示您接受SSH指纹时，请回答yes 。
    2. 出现提示时，提供您的密码。也许多次。
    3. 命令完成后，HBase服务器应该已经启动。
    4. 但是，可以肯定的是，请检查./logs目录中的日志是否有任何异常。
  4. 接下来，我们使用命令./bin/hbase shell 启动HBase shell。
  5. 我们运行一些简单的测试命令
    1. 使用命令create'test'，'data'创建一个简单的表
    2. 使用命令列表验证表是否存在
    3. 使用例如将数据插入表中
      put 'test', 'row1', 'data:1', 'value1'
put 'test', 'row2', 'data:2', 'value2'
put 'test', 'row3', 'data:3', 'value3'
    4. 使用命令scan'test'列出表中的所有行，该命令应列出先前插入的所有行。注意如何在不更改架构的情况下添加3个新列！
    5. 最后，我们通过发出disable'test'，然后是drop'test' ，并通过list进行验证来摆脱该表， list应该给出一个空列表。
  6. 退出退出 外壳
  7. 要停止HBase服务器，请发出./bin/stop-hbase.sh命令。并等待它完成！！！终止进程可能会损坏磁盘上的数据。
  8. 如有问题 ，
    1. 验证./logs目录中的HBase日志。
    2. 尝试解决问题
    3. 在论坛或IRC（ #hbase@freenode.net ）上获取帮助。人们非常积极，乐于提供帮助！
    4. 停止，重新启动并重新测试服务器。

结论

现在您的HBase服务器正在运行， 开始编码并在这个特殊但可扩展的数据存储上构建下一个杀手级应用程序！
