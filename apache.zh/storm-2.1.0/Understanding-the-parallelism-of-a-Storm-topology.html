了解风暴拓扑的并行性

构成运行拓扑的原因是：工作流程，执行程序和任务

Storm区分了以下三个主要实体，这些主要实体用于在Storm集群中实际运行拓扑：

  1. 工人流程
  2. 执行器（线程）
  3. 任务

这是它们之间关系的简单说明：

工作进程执行拓扑的子集。辅助进程属于特定的拓扑，并且可以为该拓扑的一个或多个组件（喷嘴或螺栓）运行一个或多个执行程序。正在运行的拓扑由在Storm群集内的许多计算机上运行的许多此类进程组成。

执行程序是由工作进程产生的线程。它可能针对同一组件（喷嘴或螺栓）运行一项或多项任务。

一个任务执行实际的数据处理-您在代码中实现的每个喷嘴或螺栓在整个集群中执行的任务数量一样多。在拓扑的整个生命周期中，组件的任务数量始终是相同的，但是组件的执行程序（线程）的数量会随着时间而变化。这意味着以下条件成立： #threads ≤ #tasks 。默认情况下，任务数设置为与执行程序数相同，即Storm将在每个线程中运行一个任务。

配置拓扑的并行性

请注意，在Storm的术语中，“并行性”专门用于描述所谓的并行性提示 ，这意味着组件的执行程序（线程）的初始数量。尽管在本文档中，我们从更一般的意义上使用术语“并行性”来描述如何不仅可以配置执行程序的数量，还可以配置Storm拓扑的工作进程的数量和任务的数量。我们将特别指出在Storm的常规狭义定义中使用“并行”时。

以下各节概述了各种配置选项以及如何在代码中进行设置。但是，设置这些选项的方法不只一种，并且表中仅列出了其中一些。Storm当前具有以下优先级的配置设置 ： defaults.yaml < storm.yaml <特定于拓扑的配置<特定于内部组件的配置<特定于外部组件的配置。

工作进程数

  • 描述：在集群中的机器之间为拓扑创建多少个工作进程。
  • 配置选项： TOPOLOGY_WORKERS
  • 如何在您的代码中设置（示例）：
    • Config＃setNumWorkers

执行程序数（线程）

  • 描述： 每个组件可产生多少执行程序 。
  • 配置选项：无（通过parallelism_hint参数setSpout要么setBolt ）
  • 如何在您的代码中设置（示例）：
    • TopologyBuilder＃setSpout（）
    • TopologyBuilder＃setBolt（）
    • 请注意，从Storm 0.8开始， parallelism_hint参数现在指定执行者的初始数量（不是任务！）对于那个螺栓。

任务数

  • 描述： 每个组件要创建多少个任务。
  • 配置选项： TOPOLOGY_TASKS
  • 如何在您的代码中设置（示例）：
    • ComponentConfigurationDeclarer＃setNumTasks（）

这是示例代码片段，用于在实践中显示这些设置：

topologyBuilder.setBolt("green-bolt", new GreenBolt(), 2)
               .setNumTasks(4)
               .shuffleGrouping("blue-spout");

在上面的代码中，我们将Storm配置为运行螺栓GreenBolt初始数量为两个执行器和四个相关任务。Storm将为每个执行程序（线程）运行两个任务。如果您未明确配置任务数，Storm默认将为每个执行程序运行一个任务。

正在运行的拓扑示例

下图显示了简单拓扑在操作中的外观。拓扑结构由三部分组成：一个喷嘴称为BlueSpout还有两个螺栓GreenBolt和YellowBolt 。组件被链接为BlueSpout将其输出发送到GreenBolt ，然后将其自身的输出发送到YellowBolt 。

的GreenBolt根据上面的代码片段配置，而BlueSpout和YellowBolt仅设置并行提示（执行程序的数量）。以下是相关代码：

Config conf = new Config();
conf.setNumWorkers(2); // use two worker processes

topologyBuilder.setSpout("blue-spout", new BlueSpout(), 2); // set parallelism hint to 2

topologyBuilder.setBolt("green-bolt", new GreenBolt(), 2)
               .setNumTasks(4)
               .shuffleGrouping("blue-spout");

topologyBuilder.setBolt("yellow-bolt", new YellowBolt(), 6)
               .shuffleGrouping("green-bolt");

StormSubmitter.submitTopology(
        "mytopology",
        conf,
        topologyBuilder.createTopology()
    );

当然，Storm附带了其他配置设置来控制拓扑的并行性，包括：

  • TOPOLOGY_MAX_TASK_PARALLELISM ：此设置为可以为单个组件生成的执行程序的数量设置了上限。它通常用于测试期间，以限制在本地模式下运行拓扑时产生的线程数。您可以通过Config＃setMaxTaskParallelism（）设置此选项。

如何更改正在运行的拓扑的并行性

Storm的一个漂亮功能是您可以增加或减少工作进程和/或执行程序的数量，而无需重新启动集群或拓扑。这样做的行为称为重新平衡。

您有两个选项可以重新平衡拓扑：

  1. 使用Storm Web UI重新平衡拓扑。
  2. 如下所述使用CLI工具风暴重新平衡。

这是使用CLI工具的示例：

## Reconfigure the topology "mytopology" to use 5 worker processes,
## the spout "blue-spout" to use 3 executors and
## the bolt "yellow-bolt" to use 10 executors.

$ storm rebalance mytopology -n 5 -e blue-spout=3 -e yellow-bolt=10
