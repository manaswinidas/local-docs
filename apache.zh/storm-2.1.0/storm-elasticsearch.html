Storm Elasticsearch集成

Storm Elasticsearch螺栓和三叉戟状态

EsIndexBolt，EsPercolateBolt和EsState允许用户将数据从风暴直接流式传输到Elasticsearch。有关详细说明，请参阅以下内容。

EsIndexBolt（org.apache.storm.elasticsearch.bolt。EsIndexBolt）

EsIndexBolt将元组直接流到Elasticsearch。元组以指定的索引和类型组合进行索引。用户应确保EsTupleMapper可以从输入元组中提取“源”，“索引”，“类型”和“ id”。“索引”和“类型”用于标识目标索引和类型。“源”是JSON格式的文档，将在Elasticsearch中建立索引。

EsConfig esConfig = new EsConfig(clusterName, new String[]{"localhost:9300"});
EsTupleMapper tupleMapper = new DefaultEsTupleMapper();
EsIndexBolt indexBolt = new EsIndexBolt(esConfig, tupleMapper);

EsPercolateBolt（org.apache.storm.elasticsearch.bolt。EsPercolateBolt）

EsPercolateBolt将元组直接流到Elasticsearch。元组用于将渗滤请求发送到指定的索引和类型组合。用户应确保EsTupleMapper可以从输入元组中提取“源”，“索引”，“类型”。“索引”和“类型”用于标识目标索引和类型。“源”是JSON格式字符串的文档，该文档将在percolate请求中发送到Elasticsearch。

EsConfig esConfig = new EsConfig(clusterName, new String[]{"localhost:9300"});
EsTupleMapper tupleMapper = new DefaultEsTupleMapper();
EsPercolateBolt percolateBolt = new EsPercolateBolt(esConfig, tupleMapper);

如果存在非空的渗滤液响应，则EsPercolateBolt将发出带有原始来源和渗滤液的元组。匹配每个渗滤液。在PercolateResponse中匹配。

EsState（org.apache.storm.elasticsearch.trident。EsState）

Elasticsearch Trident状态也遵循与EsBolts类似的模式。它以EsConfig和EsTupleMapper作为参数。

EsConfig esConfig = new EsConfig(clusterName, new String[]{"localhost:9300"});
EsTupleMapper tupleMapper = new DefaultEsTupleMapper();

StateFactory factory = new EsStateFactory(esConfig, tupleMapper);
TridentState state = stream.partitionPersist(factory, esFields, new EsUpdater(), new Fields());

EsLookupBolt（org.apache.storm.elasticsearch.bolt。EsLookupBolt）

EsLookupBolt向Elasticsearch执行get请求。为此，需要满足三个依赖关系。除了通常的EsConfig外，还必须提供其他两个依赖项：ElasticsearchGetRequest用于将传入的元组转换为将针对Elasticsearch执行的GetRequest。EsLookupResultOutput用于声明输出字段，并将GetResponse转换为由螺栓发出的值。

传入的元组传递给提供的GetRequest创建者，执行结果传递给Elasticsearch客户端。然后，螺栓使用提供程序输出适配器（EsLookupResultOutput）将GetResponse转换为要发出的值。螺栓的用户还可以通过输出适配器（EsLookupResultOutput）指定输出字段。

EsConfig esConfig = createEsConfig();
ElasticsearchGetRequest getRequestAdapter = createElasticsearchGetRequest();
EsLookupResultOutput output = createOutput();
EsLookupBolt lookupBolt = new EsLookupBolt(esConfig, getRequestAdapter, output);

EsConfig（org.apache.storm.elasticsearch.common。EsConfig）

提供的组件（螺栓，状态）将EsConfig用作构造函数arg。

EsConfig esConfig = new EsConfig(clusterName, new String[]{"localhost:9300"});

要么

Map<String, String> additionalParameters = new HashMap<>();
additionalParameters.put("client.transport.sniff", "true");
EsConfig esConfig = new EsConfig(clusterName, new String[]{"localhost:9300"}, additionalParameters);

EsConfig参数

精氨酸 描述 类型
clusterName Elasticsearch集群名称 字符串（必填）
节点 String数组中的Elasticsearch节点，每个元素应遵循{host}：{port}模式 字符串数组（必填）
AdditionalParameters 其他Elasticsearch Transport Client配置参数 地图（可选）

EsTupleMapper（org.apache.storm.elasticsearch.common。EsTupleMapper）

为了将元组存储到Elasticsearch或从Elasticsearch中渗入元组，我们需要定义用于哪些字段。用户需要通过实施来定义自己的EsTupleMapper 。Storm-elasticsearch提供默认映射器org.apache.storm.elasticsearch.common.DefaultEsTupleMapper ，它从相同的字段提取其源，索引，类型，id值。您可以参考DefaultEsTupleMapper的实现，以了解如何实现自己的实现。

关于Apache Storm

Apache Storm与任何排队系统和任何数据库系统集成。Apache Storm的喷口抽象使集成新排队系统变得容易。同样，将Apache Storm与数据库系统集成很容易。
