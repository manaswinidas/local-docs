Storm Hive集成

Hive提供了流API，该API允许将数据连续写入Hive。传入的数据可以以小批记录的形式连续提交到现有的Hive分区或表中。提交数据后，它立即对所有配置单元查询可见。有关Hive Streaming API的更多信息https://cwiki.apache.org/confluence/display/Hive/Streaming+Data+Ingest

借助Hive Streaming API，HiveBolt和HiveState允许用户将数据从Storm直接流式传输到Hive。要使用Hive流API，用户需要创建ORC格式的存储桶表。下面的例子

  create table test_table ( id INT, name STRING, phone STRING, street STRING) partitioned by (city STRING, state STRING) stored as orc tblproperties ("orc.compress"="NONE");

HiveBolt（org.apache.storm.hive.bolt。HiveBolt）

HiveBolt将元组直接流到Hive。元组是使用Hive Transactions编写的。HiveBolt将要流式传输到的分区可以创建，也可以预先创建，如果缺少，HiveBolt可以选择创建分区。来自元组的字段映射到表列。用户应确保元组字段名称与表列名称匹配。

DelimitedRecordHiveMapper mapper = new DelimitedRecordHiveMapper()
            .withColumnFields(new Fields(colNames));
HiveOptions hiveOptions = new HiveOptions(metaStoreURI,dbName,tblName,mapper);
HiveBolt hiveBolt = new HiveBolt(hiveOptions);

RecordHiveMapper

此类将元组字段名称映射到Hive表列名称。有两种实现方式

  • DelimitedRecordHiveMapper（org.apache.storm.hive.bolt.mapper。DelimitedRecordHiveMapper）
  • JsonRecordHiveMapper（org.apache.storm.hive.bolt.mapper。JsonRecordHiveMapper）
   DelimitedRecordHiveMapper mapper = new DelimitedRecordHiveMapper()
            .withColumnFields(new Fields(colNames))
            .withPartitionFields(new Fields(partNames));
    or
   DelimitedRecordHiveMapper mapper = new DelimitedRecordHiveMapper()
            .withColumnFields(new Fields(colNames))
            .withTimeAsPartitionField("YYYY/MM/DD");
精氨酸 描述 类型
withColumnFields 元组中要映射到表列名称的字段名称 字段（必填）
withPartitionFields 元组中的字段名称可以映射到配置单元表分区 领域
withTimeAsPartitionField 用户可以选择系统时间作为配置单元表中的分区 字串。日期格式

HiveOptions（org.apache.storm.hive.common。HiveOptions）

HiveBolt将HiveOptions用作构造函数arg。

  HiveOptions hiveOptions = new HiveOptions(metaStoreURI,dbName,tblName,mapper)
                                .withTxnsPerBatch(10)
                                .withBatchSize(1000)
                                .withIdleTimeout(10)

HiveOptions参数

精氨酸 描述 类型
metaStoreURI 蜂巢元存储URI（可以在hive-site.xml中找到） 字符串（必填）
dbName 数据库名称 字符串（必填）
tblName 表名 字符串（必填）
映射器 Mapper类将元组字段名称映射到表列名称 DelimitedRecordHiveMapper或JsonRecordHiveMapper（必填）
withTxnsPerBatch Hive将诸如HiveBolt之类的流客户端授予一批事务而不是单笔事务。此设置配置每个事务批处理所需的事务数。单个批次中来自所有事务的数据最终存储在单个文件中。HiveBolt将在批处理中的每个事务中最多写入batchSize事件。此设置与batchSize结合使用，可以控制每个文件的大小。请注意，最终，Hive将透明地将这些文件压缩为更大的文件。 整数 。默认值100
withMaxOpenConnections 仅允许此数量的打开连接。如果超过此数量，则关闭最近最少使用的连接。 整数 。默认值100
withBatchSize 单个Hive事务中写入Hive的最大事件数 整数。默认值15000
withCallTimeout （以毫秒为单位）Hive和HDFS I / O操作（例如openTxn，写入，提交，中止）的超时。 整数。默认值10000
withHeartBeatInterval （以秒为单位）发送给Hive的连续心跳之间的间隔，以防止未使用的事务过期。将此值设置为0以禁用心跳。 整数。默认240
withAutoCreatePartitions HiveBolt将自动创建要流式传输的必要Hive分区。 布尔值。默认为true
使用KerberosPrinicipal Kerberos用户主体，用于访问安全的Hive 串
使用KerberosKeytab 用于访问安全配置单元的Kerberos密钥表 串
withTickTupleInterval （以秒为单位）如果> 0，则Hive Bolt将定期刷新交易批次。建议启用此功能以避免在等待批处理填充时元组超时。 整数。默认0

HiveState（org.apache.storm.hive.trident。HiveTrident）

Hive Trident状态也遵循与HiveBolt类似的模式，该模式将HiveOptions作为arg。

   DelimitedRecordHiveMapper mapper = new DelimitedRecordHiveMapper()
            .withColumnFields(new Fields(colNames))
            .withTimeAsPartitionField("YYYY/MM/DD");

   HiveOptions hiveOptions = new HiveOptions(metaStoreURI,dbName,tblName,mapper)
                                .withTxnsPerBatch(10)
                                .withBatchSize(1000)
                                .withIdleTimeout(10)

   StateFactory factory = new HiveStateFactory().withOptions(hiveOptions);
   TridentState state = stream.partitionPersist(factory, hiveFields, new HiveUpdater(), new Fields());

关于Apache Storm

Apache Storm与任何排队系统和任何数据库系统集成。Apache Storm的喷口抽象使集成新排队系统变得容易。同样，将Apache Storm与数据库系统集成很容易。
