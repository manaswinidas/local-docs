本页说明0.7.0及更低版本的多语言协议。协议在0.7.1版中进行了更改。

风暴多语言协议

ShellBolt

通过ShellBolt类实现对多种语言的支持。此类实现IBolt接口，并使用Java的ProcessBuilder类实现用于通过外壳执行脚本或程序的协议。

输出栏位

输出字段是拓扑的Thrift定义的一部分。这意味着在Java中使用多语言时，您需要创建一个扩展ShellBolt，实现IRichBolt并在其中声明字段的螺栓。 declareOutputFields 。您可以在概念上了解更多

协议序言

通过执行的脚本或程序的STDIN和STDOUT实现简单的协议。简单的字符串和JSON编码的数据的混合与过程交换，从而使几乎所有语言的支持成为可能。

包装你的东西

要在群集上运行ShellBolt，必须将脱壳的脚本放在resources/将jar内的目录提交给主目录。

但是，在本地计算机上进行开发或测试期间，资源目录仅需要位于类路径中。

协议书

注意：*该协议的两端都使用行读取机制，因此请确保从输入中剪掉换行符并将其附加到输出中。*所有JSON输入和输出都由包含“ end”的一行终止。*下面的要点是从脚本编写者的STDIN和STDOUT的角度编写的。

  • 您的脚本将由Bolt执行。
  • STDIN：表示路径的字符串。这是一个PID目录。您的脚本应在此目录中创建一个名为pid的空文件。例如，PID为1234，因此将在目录中创建一个名为1234的空文件。该文件可让主管知道PID，以便以后可以关闭进程。
  • STDOUT：您的PID。这不是JSON编码的，只是一个字符串。ShellBolt会将PID记录到其日志中。
  • STDIN：（JSON）Storm配置。各种设置和属性。
  • STDIN：（JSON）拓扑上下文
  • 其余的发生在while（true）循环中
  • STDIN：元组！这是一个JSON编码的结构，如下所示：
{
    // The tuple's id
    "id": -6955786537413359385,
    // The id of the component that created this tuple
    "comp": 1,
    // The id of the stream this tuple was emitted to
    "stream": 1,
    // The id of the task that created this tuple
    "task": 9,
    // All the values in this tuple
    "tuple": ["snow white and the seven dwarfs", "field2", 3]
}
  • STDOUT：螺栓的结果，JSON编码。这可以是一系列的ack，fail，emission和/或log。发射看起来像：
{
    "command": "emit",
    // The ids of the tuples this output tuples should be anchored to
    "anchors": [1231231, -234234234],
    // The id of the stream this tuple was emitted to. Leave this empty to emit to default stream.
    "stream": 1,
    // If doing an emit direct, indicate the task to sent the tuple to
    "task": 9,
    // All the values in this tuple
    "tuple": ["field1", 2, 3]
}

ack看起来像：

{
    "command": "ack",
    // the id of the tuple to ack
    "id": 123123
}

失败看起来像：

{
    "command": "fail",
    // the id of the tuple to fail
    "id": 123123
}

一个“日志”将在工作日志中记录一条消息。看起来像：

{
    "command": "log",
    // the message to log
    "msg": "hello world!"

}
  • STDOUT：当螺栓完成发射/确认/失败并准备好进行下一个输入时，自身会单独发出“同步”信号

同步

注意：此命令不是JSON编码的，它以简单字符串的形式发送。

这使父级螺栓知道脚本已完成处理并且可以用于另一个元组。
