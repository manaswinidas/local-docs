动态日志级别设置

我们添加了使用Storm UI和Storm CLI为正在运行的拓扑设置日志级别设置的功能。

日志级别设置的应用方式与您期望的log4j相同，因为我们所做的只是告诉log4j设置您提供的记录器级别。如果您设置了父记录器的日志级别，则子记录器将开始使用该级别（除非子记录器具有更高的限制级别）。如果工作人员应自动重置日志级别，则可以选择提供超时（DEBUG模式除外，在UI中是必需的）。

此还原操作是使用轮询机制触发的（每30秒一次，但这是可配置的），因此您应该希望超时是您提供的值加上0到设置值之间的任何值。

使用Storm UI

为了设置级别，请单击正在运行的拓扑，然后在“拓扑操作”部分中单击“更改日志级别”。

接下来，提供记录器名称，选择所需的级别（例如WARN），以及以秒为单位的超时（如果不需要则为0）。然后点击“添加”。

要清除日志级别，请单击“清除”按钮。这会将日志级别恢复为添加设置之前的级别。日志级别行将从用户界面中消失。

虽然重置日志级别存在延迟，但是首先将日志级别设置为立即（或尽快将消息从UI / CLI通过雨云和Zookeeper传递给工作人员）。

使用CLI

使用CLI，发出命令：

./bin/storm set_log_level [topology name] -l [logger name]=[LEVEL]:[TIMEOUT]

例如：

./bin/storm set_log_level my_topology -l ROOT=DEBUG:30

将ROOT记录器设置为DEBUG 30秒。

./bin/storm set_log_level my_topology -r ROOT

清除ROOT记录器动态日志级别，将其重置为其原始值。

关于Apache Storm

Apache Storm与任何排队系统和任何数据库系统集成。Apache Storm的喷口抽象使集成新排队系统变得容易。同样，将Apache Storm与数据库系统集成很容易。
