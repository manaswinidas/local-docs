本地模式

本地模式可模拟过程中的Storm集群，对于开发和测试拓扑非常有用。在本地模式下运行拓扑类似于在群集上运行拓扑。

要以本地模式运行拓扑，您有两个选择。最常见的选择是使用storm local代替storm jar

这将建立一个本地模拟集群，并强制所有与nimbus的交互都经过该模拟集群，而不是进入单独的过程。默认情况下，此过程将运行该过程20秒钟，然后再拆除整个群集。您可以通过添加一个--local-ttl命令行选项，用于设置应运行的秒数。

程式化

如果您想进行一些自动化测试，但又没有真正启动风暴集群，则可以在内部使用相同的类storm local做。

为此，您首先需要提取访问这些类所需的依赖项。对于Java API，您应依赖storm-server作为一个test依赖性。

要创建进程内群集，只需使用LocalCluster类。例如：

import org.apache.storm.LocalCluster;

...

try (LocalCluster cluster = new LocalCluster()) {
    //Interact with the cluster...
}

然后，您可以使用submitTopology上的方法LocalCluster宾语。就像StormSubmitter上的相应方法一样 ， submitTopology具有名称，拓扑配置和拓扑对象。然后，您可以使用killTopology以拓扑名称为参数的方法。

许多Nimbus API也可以通过LocalCluster获得。

DRPC

DRPC也可以在本地模式下运行。这是在本地模式下运行上述示例的方法：

try (LocalDRPC drpc = new LocalDRPC();
     LocalCluster cluster = new LocalCluster();
     LocalTopology topo = cluster.submitTopology("drpc-demo", conf, builder.createLocalTopology(drpc))) {

    System.out.println("Results for 'hello':" + drpc.execute("exclamation", "hello"));
}

首先，您创建一个LocalDRPC宾语。该对象模拟过程中的DRPC服务器，就像LocalCluster模拟进行中的Storm集群。然后您创建LocalCluster在本地模式下运行拓扑。 LinearDRPCTopologyBuilder具有用于创建本地拓扑和远程拓扑的单独方法。在本地模式下LocalDRPC对象没有绑定到任何端口，因此拓扑需要知道与之通信的对象。这就是为什么createLocalTopology接受LocalDRPC对象作为输入。

启动拓扑后，可以使用以下命令执行DRPC调用： execute方法开启LocalDRPC 。

由于在try块作用域结束时使用的所有对象都是AutoCloseable实例，因此拓扑被终止，群集将关闭，而drpc服务器也将关闭。

Clojure API

Storm还提供用于测试的Clojure API。

这篇博客文章谈到了这一点，但是有点过时了。要获得此功能，您需要包括storm-clojure-test依赖性。这本身会带来很多风暴，这些风暴本不应该与您的拓扑打包在一起，请确保仅是测试依赖项。

使用IDE调试拓扑

本地模式的一个很好的用例之一是能够使用IDE逐步完成螺栓和喷口的代码执行。您可以在命令行上通过添加--java-debug选项，后跟传递给jdwp的参数。这使得使用以下命令启动本地集群很简单-agentlib:jdwp=打开。

从IDE本身运行时，可以通过调用来修改代码运行LocalCluster.withLocalModeOverride

public static void main(final String args[]) {
    LocalCluster.withLocalModeOverride(() -> originalMain(args), 10);
}

或者，您也可以修改IDE以运行“ org.apache.storm。“ LocalCluster”（而不是您的主类）在启动时，并将类的名称作为参数传递给它。这也会触发本地模式，这是什么storm local在幕后做。

本地模式的通用配置

您可以在此处查看完整的配置列表。

  1. 配置TOPOLOGY_MAX_TASK_PARALLELISM ：此配置为单个组件产生的线程数设置了上限。通常，生产拓扑具有许多并行性（数百个线程），这在尝试以本地模式测试拓扑时会带来不合理的负载。此配置可让您轻松控制该并行性。
  2. 配置TOPOLOGY_DEBUG ：设置为true时，每次从任何喷嘴或螺栓发射出元组时，Storm都会记录一条消息。这对于调试非常有用。一种

storm local topology.jar <MY_MAIN_CLASS> -c topology.debug=true
