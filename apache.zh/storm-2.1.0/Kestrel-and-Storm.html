风暴与红est

本页说明如何使用Storm消耗Kestrel群集中的项目。

初赛

风暴

本教程使用来自storm-kestrel项目和storm-starter项目的示例。建议您克隆这些项目并按照示例进行操作。阅读设置开发环境和创建新的Storm项目以设置计算机。

红est

假定您能够按照此处所述在本地运行Kestrel服务器。

Kestrel服务器和队列

单个Kestrel服务器具有一组队列。Kestrel队列是一个非常简单的消息队列，它在JVM上运行，并使用内存缓存协议（具有某些扩展）与客户端进行通信。有关详细信息，请查看Storm- kestrel项目中提供的KestrelThriftClient类的实现。

严格按照FIFO（先进先出）原则对每个队列进行排序。为了跟上性能，项目被缓存在系统内存中；但是，只有前128MB保留在内存中。停止服务器时，队列状态存储在日志文件中。

此外，可以在此处找到详细信息。

红est是：*快速*小*耐用*可靠

举例来说，Twitter使用红隼作为消息传递基础结构的骨干描述这里 。

将项目添加到红est

首先，我们需要一个可以将项目添加到Kestrel队列的程序。以下方法利用了storm- kestrel中的KestrelClient实现。它将句子添加到从包含五个可能句子的数组中随机选择的Kestrel队列中。

    private static void queueSentenceItems(KestrelClient kestrelClient, String queueName)
            throws ParseError, IOException {

        String[] sentences = new String[] {
                "the cow jumped over the moon",
                "an apple a day keeps the doctor away",
                "four score and seven years ago",
                "snow white and the seven dwarfs",
                "i am at two with nature"};

        Random _rand = new Random();

        for(int i=1; i<=10; i++){

            String sentence = sentences[_rand.nextInt(sentences.length)];

            String val = "ID " + i + " " + sentence;

            boolean queueSucess = kestrelClient.queue(queueName, val);

            System.out.println("queueSucess=" +queueSucess+ " [" + val +"]");
        }
    }

从红est中删除项目

此方法从队列中将项目出队，而不会将其删除。

    private static void dequeueItems(KestrelClient kestrelClient, String queueName) throws IOException, ParseError
             {
        for(int i=1; i<=12; i++){

            Item item = kestrelClient.dequeue(queueName);

            if(item==null){
                System.out.println("The queue (" + queueName + ") contains no items.");
            }
            else
            {
                byte[] data = item._data;

                String receivedVal = new String(data);

                System.out.println("receivedItem=" + receivedVal);
            }
        }

此方法使队列中的项目出队，然后将其删除。

    private static void dequeueAndRemoveItems(KestrelClient kestrelClient, String queueName)
    throws IOException, ParseError
         {
            for(int i=1; i<=12; i++){

                Item item = kestrelClient.dequeue(queueName);


                if(item==null){
                    System.out.println("The queue (" + queueName + ") contains no items.");
                }
                else
                {
                    int itemID = item._id;


                    byte[] data = item._data;

                    String receivedVal = new String(data);

                    kestrelClient.ack(queueName, itemID);

                    System.out.println("receivedItem=" + receivedVal);
                }
            }
    }

将项目连续添加到红Add

这是我们的最终方案，以便不断一句项添加到一个名为sentence_queue本地运行的红隼服务器的队列中运行。

为了停止它，请在控制台中键入右括号char']'，然后按Enter。

    import java.io.IOException;
    import java.io.InputStream;
    import java.util.Random;

    import org.apache.storm.spout.KestrelClient;
    import org.apache.storm.spout.KestrelClient.Item;
    import org.apache.storm.spout.KestrelClient.ParseError;

    public class AddSentenceItemsToKestrel {

        /**
         * @param args
         */
        public static void main(String[] args) {

            InputStream is = System.in;

            char closing_bracket = ']';

            int val = closing_bracket;

            boolean aux = true;

            try {

                KestrelClient kestrelClient = null;
                String queueName = "sentence_queue";

                while(aux){

                    kestrelClient = new KestrelClient("localhost",22133);

                    queueSentenceItems(kestrelClient, queueName);

                    kestrelClient.close();

                    Thread.sleep(1000);

                    if(is.available()>0){
                     if(val==is.read())
                         aux=false;
                    }
                }
            } catch (IOException e) {
                // TODO Auto-generated catch block
                e.printStackTrace();
            }
            catch (ParseError e) {
                // TODO Auto-generated catch block
                e.printStackTrace();
            } catch (InterruptedException e) {
                // TODO Auto-generated catch block
                e.printStackTrace();
            }

            System.out.println("end");

        }
    }

使用KestrelSpout

此拓扑使用KestrelSpout从Kestrel队列中读取句子，将句子拆分成其组成词（Bolt：SplitSentence），然后为每个单词发出它之前看到该单词的次数（Bolt：WordCount）。保证消息处理中详细描述了如何处理数据。

    TopologyBuilder builder = new TopologyBuilder();
    builder.setSpout("sentences", new KestrelSpout("localhost",22133,"sentence_queue",new StringScheme()));
    builder.setBolt("split", new SplitSentence(), 10)
                .shuffleGrouping("sentences");
    builder.setBolt("count", new WordCount(), 20)
            .fieldsGrouping("split", new Fields("word"));

执行

首先，以生产或开发模式启动本地Kestrel服务器。

然后，请等待大约5秒钟，以避免ConnectException。

现在执行程序以将项目添加到队列并启动Storm拓扑。启动程序的顺序并不重要。

如果使用TOPOLOGY_DEBUG运行拓扑，则应该看到在拓扑中发出了元组。
