指标报告API v2

Apache Storm 1.2版引入了新的指标系统，用于报告内部统计信息（例如，确认，失败，发出，传输，破坏者队列指标等），以及用于用户定义指标的新API。

新的指标系统基于Dropwizard指标 。

用户定义的指标

为了允许用户定义自定义指标，以下方法已添加到TopologyContext类，其实例传递到spout的open()方法和螺栓prepare()方法：

public Timer registerTimer(String name)

public Histogram registerHistogram(String name)

public Meter registerMeter(String name)

public Counter registerCounter(String name)

public Gauge registerGauge(String name, Gauge gauge)

API文档： 计时器 ， 直方图 ， 仪表 ， 计数器 ， 量规

这些方法中的每一个都需要name用作标识符的参数。注册度量标准后，Storm将添加其他信息，例如主机名，端口，拓扑ID等，以形成唯一的度量标准标识符。例如，如果我们注册一个名为myCounter如下：

    Counter myCounter = topologyContext.registerCounter("myCounter");

发送给指标报告者的名称将扩展为：

   storm.topology.{topology ID}.{hostname}.{component ID}.{task ID}.{worker port}-myCounter

附加信息允许对整个集群中的组件实例的指标进行唯一标识。

重要说明：为了确保可以可靠地解析度量标准名称，任何.名称组件中的字符将被下划线（ _ ）字符。例如，主机名storm.example.com将显示为storm_example_com在指标名称中该字符替换*不适用于用户提供的name参数。

示例：元组埋头螺栓

以下示例是一个简单的螺栓实现，它将报告螺栓接收的运行总计元组：

public class TupleCountingBolt extends BaseRichBolt {
    private Counter tupleCounter;
    @Override
    public void prepare(Map stormConf, TopologyContext context, OutputCollector collector) {
        this.tupleCounter = context.registerCounter("tupleCount");
    }

    @Override
    public void execute(Tuple input) {
        this.tupleCounter.inc();
    }
}

指标报告器配置

为了使指标有用，必须将其报告 ，换句话说，将其发送到可以使用和分析它们的地方。这很简单，例如将它们写入日志文件，将它们发送到时间序列数据库或通过JMX公开它们。

从Storm 1.2.0开始，支持以下指标报告器

  • 控制台记者（ org.apache.storm.metrics2.reporters.ConsoleStormReporter ）：将指标报告给System.out 。
  • CSV报表器（ org.apache.storm.metrics2.reporters.CsvStormReporter ）：将指标报告到CSV文件。
  • 石墨记者org.apache.storm.metrics2.reporters.GraphiteStormReporter ）：将指标报告给Graphite服务器。
  • JMX记者（ org.apache.storm.metrics2.reporters.JmxStormReporter ）：通过JMX公开指标。

指标报告器在storm.yaml文件。默认情况下，Storm将收集指标但不“报告”或将收集的指标发送到任何地方。要启用指标报告，请添加一个storm.metrics.reporters到storm.yaml并配置一个或多个记者。

以下示例配置设置了两个报告器：Graphite Reporter和Console Reporter：

storm.metrics.reporters:
  # Graphite Reporter
  - class: "org.apache.storm.metrics2.reporters.GraphiteStormReporter"
    daemons:
        - "supervisor"
        - "nimbus"
        - "worker"
    report.period: 60
    report.period.units: "SECONDS"
    graphite.host: "localhost"
    graphite.port: 2003

  # Console Reporter
  - class: "org.apache.storm.metrics2.reporters.ConsoleStormReporter"
    daemons:
        - "worker"
    report.period: 10
    report.period.units: "SECONDS"
    filter:
        class: "org.apache.storm.metrics2.filters.RegexFilter"
        expression: ".*my_component.*emitted.*"

每个报告器部分都以一个class代表报告程序实现的全限定类名称的参数。的daemons部分确定报告程序将应用于哪个守护程序（在示例Graphite Reporter中配置为报告所有Storm守护程序的指标，而控制台报告程序仅报告工作程序和拓扑指标）。

许多报告程序的实现是按计划进行的 ，这意味着它们定期报告度量标准。报告间隔由report.period和report.period.units参数。

还可以为报告者配置一个可选的过滤器，该过滤器确定要报告哪些指标。风暴包括org.apache.storm.metrics2.filters.RegexFilter过滤器，该过滤器使用正则表达式来确定要报告的指标。可以通过实施org.apache.storm.metrics2.filters.StormMetricFilter接口：

public interface StormMetricsFilter extends MetricFilter {

    /**
     * Called after the filter is instantiated.
     * @param config A map of the properties from the 'filter' section of the reporter configuration.
     */
    void prepare(Map<String, Object> config);

   /**
    *  Returns true if the given metric should be reported.
    */
    boolean matches(String name, Metric metric);

}
