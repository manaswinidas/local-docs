仅在实际的Storm群集上才需要本机依赖项。在本地模式下运行Storm时，Storm使用纯Java消息系统，因此您无需在开发计算机上安装本机依赖项。

安装ZeroMQ和JZMQ通常很简单。但是，有时人们会遇到autoconf的问题并得到奇怪的错误。如果您遇到任何问题，请给Storm邮件列表发送电子邮件，或者在freenode上的＃storm-user会议室获得帮助。

Storm已通过ZeroMQ 2.1.7进行了测试，这是您安装的推荐ZeroMQ版本。您可以在此处下载ZeroMQ版本。安装ZeroMQ应该看起来像这样：

wget http://download.zeromq.org/zeromq-2.1.7.tar.gz
tar -xzf zeromq-2.1.7.tar.gz
cd zeromq-2.1.7
./configure
make
sudo make install

JZMQ是ZeroMQ的Java绑定。 JZMQ没有任何发行版（我们正在与之合作），因此，如果始终从master分支进行安装，则存在回归的风险。为防止发生回归，您应该从经过测试可与Storm一起使用的此fork安装。安装JZMQ应该看起来像这样：

#install jzmq
git clone https://github.com/nathanmarz/jzmq.git
cd jzmq
./autogen.sh
./configure
make
sudo make install

要使JZMQ构建正常工作，您可能需要执行以下一项或全部操作：

  1. 适当设置JAVA_HOME环境变量
  2. 安装Java开发包（有关Mac OSX用户的更多信息，请参见此处 ）
  3. 在机器上升级autoconf
  4. 按照此博客文章中的说明进行操作

如果在运行时遇到任何错误./configure ， 此线程可以提供解决方案。

关于Apache Storm

Apache Storm与任何排队系统和任何数据库系统集成。Apache Storm的喷口抽象使集成新排队系统变得容易。同样，将Apache Storm与数据库系统集成很容易。
