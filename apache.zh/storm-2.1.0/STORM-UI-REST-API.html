Storm UI REST API

Storm UI守护程序提供了一个REST API，使您可以与Storm集群进行交互，其中包括检索指标数据和配置信息以及管理操作（例如启动或停止拓扑）。

资料格式

REST API返回JSON响应并支持JSONP。客户端可以传递回调查询参数以将JSON包装在回调函数中。

使用UI REST API

注意：建议忽略JSON响应中未记录的元素，因为Storm的未来版本可能不再 支持这些元素。

REST API基本URL

REST API是Storm的UI守护进程的一部分（由storm ui ），因此与Storm UI在同一主机和端口上运行（UI守护程序通常与Nimbus守护程序在同一主机上运行）。端口由配置ui.port ，它设置为8080默认情况下（请参见defaults.yaml ）。

API的基本URL因此为：

http://<ui-host>:<ui-port>/api/v1/...

您可以使用以下工具curl与REST API交谈：

# Request the cluster configuration.
# Note: We assume ui.port is configured to the default value of 8080.
$ curl http://<ui-host>:8080/api/v1/cluster/configuration

在安全环境中模拟用户

在安全的环境中，经过身份验证的用户可以模拟其他用户。要模拟用户，呼叫者必须通过doAsUser参数或标头，其值设置为需要执行请求的用户。请参阅SECURITY.MD以了解有关如何设置模拟ACL和授权的更多信息。其余的API使用与nimbus相同的配置和ACL。

例子：

 1. http://ui-daemon-host-name:8080/api/v1/topology/wordcount-1-1425844354\?doAsUser=testUSer1
 2. curl 'http://localhost:8080/api/v1/topology/wordcount-1-1425844354/activate' -X POST -H 'doAsUser:testUSer1'

/ api / v1 / supervisor（GET）

按ID返回主管的摘要，或按主机返回所有主管的摘要。

例子：

 1. By host: http://ui-daemon-host-name:8080/api/v1/supervisor?host=supervisor-daemon-host-name
 2. By id: http://ui-daemon-host-name:8080/api/v1/supervisor?id=f5449110-1daa-43e2-89e3-69917b16dec9-192.168.1.1

请求参数：

参数 值 描述
ID 串。主管编号 如果指定，请使用ID来响应主管和工人统计信息。请注意，当指定id时，host参数将被忽略。
主办 串。主机名 如果指定，对主持人中的所有主管和工作人员统计信息进行响应（通常只有一个）
系统 串。值1或0。默认值0 控件包括响应的sys stats部分

响应字段：

领域 值 描述
主管 数组 主管摘要数组
工作人员 数组 工人摘要数组
schedulerDisplayResource 布尔型 是否显示调度程序资源信息

每个主管的定义如下：

领域 值 描述
ID 串 主管ID
主办 串 主管的主机名
正常运行时间 串 显示主管运行了多长时间
正常运行时间 整数 显示主管运行多长时间（以秒为单位）
slotTotal 整数 该主管的工作人员总数
已使用 整数 该主管使用的工作位置数量
schedulerDisplayResource 布尔型 是否显示调度程序资源信息
总记忆 双 该主管的总存储容量
总CPU 双 该主管的CPU总容量
使用过的记忆 双 该主管的已用内存容量
用过的CPU 双 该主管使用的CPU容量

每个工人的定义如下：

分析和调试GET操作

/ api / v1 / topology / / profiling / start / / （获取）

要求在具有超时的工作程序上启动分析器。返回状态并链接到工作人员的探查器工件。用和替换适当的值。

参数 值 描述
ID 字符串（必填） 拓扑ID
主机端口 字符串（必填） 工人编号
超时 字符串（必填） 剖析器超时几分钟后停止

响应字段：

领域 值 描述
ID 串 工作人员编号
状态 串 回应状态
超时 串 请求超时
转储链接 串 链接到工作分析器文档的logviewer URL。

例子：

1. http://ui-daemon-host-name:8080/api/v1/topology/wordcount-1-1446614150/profiling/start/10.11.1.7:6701/10
2. http://ui-daemon-host-name:8080/api/v1/topology/wordcount-1-1446614150/profiling/start/10.11.1.7:6701/5
3. http://ui-daemon-host-name:8080/api/v1/topology/wordcount-1-1446614150/profiling/start/10.11.1.7:6701/20

样本回复：

{
   "status": "ok",
   "id": "10.11.1.7:6701",
   "timeout": "10",
   "dumplink": "http:\/\/10.11.1.7:8000\/dumps\/wordcount-1-1446614150\/10.11.1.7%3A6701"
}

{"topologyOperation":"activate","topologyId":"wordcount-1-1420308665","status":"success"}

停用拓扑。代替。

参数 值 描述

样本响应：

{"topologyOperation":"deactivate","topologyId":"wordcount-1-1420308665","status":"success"}

/ api / v1 / topology / / rebalance / （POST）

重新平衡拓扑。代替和。

参数 值 描述
ID 字符串（必填） 拓扑ID
等待时间 字符串（必填） 重新平衡发生之前的等待时间
rebalanceOptions 杰森（可选） 拓扑重新平衡选项

示例rebalanceOptions json：

{"rebalanceOptions" : {"numWorkers" : 2, "executors" : {"spout" :4, "count" : 10}}, "callback" : "foo"}

例子：

curl  -i -b ~/cookiejar.txt -c ~/cookiejar.txt -X POST  
-H "Content-Type: application/json" 
-d  '{"rebalanceOptions": {"numWorkers": 2, "executors": { "spout" : "5", "split": 7, "count": 5 }}, "callback":"foo"}' 
http://localhost:8080/api/v1/topology/wordcount-1-1420308665/rebalance/0

样本响应：

{"topologyOperation":"rebalance","topologyId":"wordcount-1-1420308665","status":"success"}

/ api / v1 / topology / / kill / （POST）

杀死拓扑。代替和。

参数 值 描述
ID 字符串（必填） 拓扑ID
等待时间 字符串（必填） 重新平衡发生之前的等待时间

警告：较小的等待时间（0-5秒）可能会增加触发STORM-112中报告的错误的可能性，这可能会导致代理Supervisor守护程序。

样本响应：

{"topologyOperation":"kill","topologyId":"wordcount-1-1420308665","status":"success"}

API错误

如果有任何错误，API将返回500个HTTP状态代码。

DRPC REST API

如果DRPC使用http或https端口配置，它将公开REST端点。（有关操作，请参阅设置Storm集群 ）

在所有这些命令中:func是DRPC函数， :args是它的参数。唯一的区别在于这些参数的提供方式。在所有情况下，响应都在响应的主体中。

在所有情况下，DRPC都没有状态，因此，如果您的请求超时或导致错误，请重试该请求，但最好使用指数补偿，以避免在DRPC服务器上执行DDOS。

/ drpc /：func（开机自检）

在这种情况下:args对drpc的请求都在帖子的正文中。

/ drpc /：func /：args（获取）

在这种情况下:args作为URL本身的一部分提供。URL长度受许多工具的限制，因此，如果长度超过100个字符，建议改用POST选项。

/ drpc /：func（获取）

在极少数情况下:args DRPC命令可能不需要。如果不:args节在DRPC请求中给出，并且为空字符串""将用于参数。
