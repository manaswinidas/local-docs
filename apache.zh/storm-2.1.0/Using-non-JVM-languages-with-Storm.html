关于以非JVM语言实现DSL的注意事项

正确的开始位置是src / storm.thrift。由于Storm拓扑只是Thrift结构，而Nimbus是Thrift守护程序，因此您可以使用任何语言创建和提交拓扑。

当为喷口和螺栓创建Thrift结构时，喷口或螺栓的代码在ComponentObject结构中指定：

union ComponentObject {
  1: binary serialized_java;
  2: ShellComponent shell;
  3: JavaObject java_object;
}

对于非JVM DSL，您需要使用“ 2”和“ 3”。ShellComponent使您可以指定脚本来运行该组件（例如，您的python代码）。JavaObject允许您为组件指定本机Java喷嘴和螺栓（Storm将使用反射来创建该喷嘴或螺栓）。

有一个“ storm shell”命令将有助于提交拓扑。它的用法是这样的：

storm shell resources/ python topology.py arg1 arg2

然后，storm shell将资源/打包到一个jar中，将该jar上传到Nimbus，并按如下所示调用您的topology.py脚本：

python topology.py arg1 arg2 {nimbus-host} {nimbus-port} {uploaded-jar-location}

然后，您可以使用Thrift API连接到Nimbus并提交拓扑，并将{uploaded-jar-location}传递到SubmitTopology方法中。供参考，这是submitTopology定义：

void submitTopology(1: string name, 2: string uploadedJarLocation, 3: string jsonConf, 4: StormTopology topology)
    throws (1: AlreadyAliveException e, 2: InvalidTopologyException ite);

关于Apache Storm

Apache Storm与任何排队系统和任何数据库系统集成。Apache Storm的喷口抽象使集成新排队系统变得容易。同样，将Apache Storm与数据库系统集成很容易。
