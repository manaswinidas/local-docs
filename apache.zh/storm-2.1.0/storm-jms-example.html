Storm JMS集成

Storm JMS拓扑示例

storm-jms源代码包含一个示例项目（在“ examples”目录中），该项目使用JMS Spout和JMS Bolt组件构建多螺栓/多喷嘴拓扑（如下所示）。

绿色组件代表storm-jms组件的实例。白色组件代表“标准”风暴螺栓（在这些示例中，这些螺栓是GenericBolt它只是记录有关它接收和发出的元组的信息）。

灰色箭头表示JMS消息，而黑色箭头表示Storm元组对象的流。

JMS交易和保证处理

该示例设置为“事务性”，这意味着JMS Spout将使用Storm的保证的处理能力来确定是否应确认JMS消息。拓扑中的每个螺栓将锚定到它接收的每个元组。如果每个螺栓成功处理并固定了链中的每个元组，则将确认原始JMS消息，并且基础JMS实现将不会尝试重新传递消息。如果螺栓无法处理/确认元组，则将不会确认JMS消息，并且JMS实现将使该消息排队以便重新交付。

数据流

该拓扑包含两个链：一个链起源于连接到队列的JMS Spout，另一链起源于连接到主题的JMS Spout。

链＃1

  1. “ JMS队列出口”从队列接收JMS消息对象，并向“中间螺栓”发出一个元组
  2. “中间螺栓”向“最终螺栓”和“ JMS主题螺栓”发出一个元组，并确认收到的元组。
  3. “最终螺栓”接收元组并简单地将其固定，它不会发出任何东西。
  4. “ JMS主题螺栓”接收一个元组，从该元组的值构造一个JMS消息，并将该消息发布到JMS主题。
  5. 如果“ JMS主题螺栓”成功发布了JMS消息，它将确认该元组。
  6. 如果链中的所有螺栓均已加紧并确认原始JMS消息，则“ JMS队列喷口”将收到通知。如果链中的一个或多个螺栓无法固定一个元组，则“ JMS队列喷口”将不确认JMS消息。

链＃2

  1. “ JMS主题喷口”从主题接收JMS消息，并向“另一个螺栓”发出一个元组。
  2. “另一个螺栓”接收并固定元组。
  3. “ JMS主题喷口”确认JMS消息。

构建示例拓扑

$ cd storm-jms
$ mvn clean install

在本地运行示例拓扑

该示例对JMS使用ApacheMQ 5.4.0。从Apache ActiveMQ下载下载并安装它。

不需要特定的配置，只需启动ActiveMQ：

$ [ACTIVEMQ_HOME]/bin/activemq

从中运行示例拓扑examples目录：

$ mvn exec:java

拓扑运行时，它将连接到ActiveMQ并创建以下JMS目标：

backtype.storm.contrib.example.queue
backtype.storm.contrib.example.topic

将消息发布到backtype.storm.contrib.example.queue队列：

  1. 打开ActiveMQ队列管理控制台： http：// localhost：8161 / admin / queues.jsp
  2. 点击发送到链接backtupe.storm.example.queue队列条目。
  3. 在“发送JMS消息”表单上，选择“持久传递”复选框，为消息正文输入一些文本，然后单击“发送”。

关于Apache Storm

Apache Storm与任何排队系统和任何数据库系统集成。Apache Storm的喷口抽象使集成新排队系统变得容易。同样，将Apache Storm与数据库系统集成很容易。
