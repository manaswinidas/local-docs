罐

句法：storm jar topology-jar-path class ...

运行的主要方法class具有指定的参数。风暴罐和配置~/.storm被放在类路径上。配置该过程，以便StormSubmitter将在以下位置上传jar topology-jar-path提交拓扑时。

当您想运送应用程序jar中未包含的其他jar时，可以将它们传递给--jars用逗号分隔的字符串选项。例如，--jars“ your-local-jar.jar，your-local-jar2.jar”将加载your-local-jar.jar和your-local-jar2.jar。当您要运送Maven工件及其传递依赖项时，可以将它们传递给--artifacts用逗号分隔的字符串。您还可以排除一些依赖项，例如您在maven pom中所做的事情。请在排除项目之后添加用“ ^”分隔的字符串的排除项目。例如， --artifacts "redis.clients:jedis:2.9.0,org.apache.kafka:kafka_2.10:0.8.2.2^org.slf4j:slf4j-log4j12"将加载jedis和kafka工件以及所有传递依赖项，但从kafka中排除slf4j-log4j12。

当您需要从Maven Central以外的其他地方获取工件时，可以使用逗号分隔的字符串将远程存储库传递给--artifactRepositories选项。储存库格式为“ ^ ”。因为URL允许各种字符，所以将'^'用作分隔符。例如，--artifactRepositories“ jboss-repository ^ http://repository.jboss.com/maven2,HDPRepo^http://repo.hortonworks.com/content/groups/public/ ”将添加JBoss和HDP存储库以进行依赖解析器。

这两个选项的完整示例在这里：./bin/storm jar example/storm-starter/storm-starter-topologies-*.jar org.apache.storm.starter.RollingTopWords blobstore-remote2 remote --jars "./external/storm-redis/storm-redis-1.1.0.jar,./external/storm-kafka-client/storm-kafka-client-1.1.0.jar" --artifacts "redis.clients:jedis:2.9.0,org.apache.kafka:kafka-clients:1.0.0^org.slf4j:slf4j-api" --artifactRepositories "jboss-repository^http://repository.jboss.com/maven2,HDPRepo^http://repo.hortonworks.com/content/groups/public/"

当您传递jar和/或工件选项时，StormSubmitter会在提交拓扑时将它们上传，并且它们将被包含在运行该类的进程以及该拓扑的工作程序的类路径中。

本地

句法：storm local topology-jar-path class ...

本地命令的行为就像storm jar除了将拓扑提交给集群以外，它将以本地模式运行集群。这意味着风暴守护程序的嵌入式版本将在与拓扑相同的过程中运行30秒，然后自动关闭。这样，拓扑的类路径将被扩展为包括运行这些守护程序所需的所有内容。

sql

句法：storm sql sql-file topology-name

将SQL语句编译为Trident拓扑并将其提交给Storm。

--jars和--artifacts和--artifactRepositories jar可用的选项也适用于sql命令。请参考“帮助罐”以了解如何使用--jars和--artifacts以及--artifactRepositories选项。通常，您需要传递这些选项，因为您需要将数据源设置为sql（在许多情况下是外部存储设备）。

杀

句法：storm kill topology-name [-w wait-time-secs]

用名称杀死拓扑topology-name 。Storm将首先在拓扑消息超时期间停用拓扑的喷口，以允许当前正在处理的所有消息完成处理。风暴将关闭工人并清理他们的状态。您可以使用-w标志覆盖Storm在停用和关闭之间等待的时间。

启用

句法：storm activate topology-name

激活指定拓扑的喷口。

停用

句法：storm deactivate topology-name

停用指定拓扑的喷嘴。

重新平衡

句法：storm rebalance topology-name [-w wait-time-secs] [-n new-num-workers] [-e component=parallelism]*

有时您可能希望散布拓扑工作程序的运行位置。例如，假设您有一个10节点集群，每个节点运行4个工作线程，然后再向集群添加10个节点。您可能希望Storm分散正在运行的拓扑的工作线程，以便每个节点运行2个工作线程。一种方法是取消拓扑并重新提交，但是Storm提供了“重新平衡”命令，该命令提供了一种更简便的方法。

重新平衡将首先在消息超时的时间内停用拓扑（通过-w标志可覆盖），然后在集群中均匀地重新分配工作线程。然后，拓扑将返回到其先前的激活状态（因此，停用的拓扑仍将被停用，并且激活的拓扑将返回到被激活的状态）。

rebalance命令还可以用于更改正在运行的拓扑的并行性。使用-n和-e开关分别更改组件的工作程序数或执行程序数。

代表

不推荐使用：在将来的发行版中可能会删除此子命令。

句法：storm repl

在类路径上打开带有风暴罐和配置的Clojure REPL。对于调试很有用。

类路径

句法：storm classpath

在运行命令时打印风暴客户端使用的类路径。

server_classpath

句法：storm server_classpath

打印风暴守护程序使用的类路径。

局部价值

句法：storm localconfvalue conf-name

打印出的值conf-name在本地Storm配置中。本地Storm配置是~/.storm/storm.yaml与中的配置合并defaults.yaml 。

远程配置

句法：storm remoteconfvalue conf-name

打印出的值conf-name在集群的Storm配置中。集群的Storm配置是$STORM-PATH/conf/storm.yaml与中的配置合并defaults.yaml 。此命令必须在群集计算机上运行。

雨云

句法：storm nimbus

启动nimbus守护程序。该命令应在daemontools或monit之类的工具的监督下运行。有关更多信息，请参见设置Storm集群 。

主管

句法：storm supervisor

启动主管守护程序。该命令应在daemontools或monit之类的工具的监督下运行。有关更多信息，请参见设置Storm集群 。

ui

句法：storm ui

启动UI守护程序。UI提供了一个Storm集群的Web界面，并显示了有关运行拓扑的详细统计信息。该命令应在daemontools或monit之类的工具的监督下运行。有关更多信息，请参见设置Storm集群 。

drpc

句法：storm drpc

启动DRPC守护程序。该命令应在daemontools或monit之类的工具的监督下运行。有关更多信息，请参见分布式RPC 。

drpc客户端

句法：storm drpc-client [options] ([function argument]*)|(argument*)

提供了一种非常简单的发送DRPC请求的方法。如果一个-f提供了arguments，以设置函数名称，所有参数均视为函数的参数。如果没有给出函数，则参数必须是一对函数参数。

注意：这不是真正用于生产用途。这主要是因为解析结果可能会很痛苦。

创建一个实际的DRPC客户端仅需几行，因此对于生产，请这样做。

Config conf = new Config();
try (DRPCClient drpc = DRPCClient.getConfiguredClient(conf)) {
  //User the drpc client
  String result = drpc.execute(function, argument);
}

例子

storm drpc-client exclaim a exclaim b test bar

这将提交3个单独的DRPC请求。 1. function =“ exclaim” args =“ a” 2. function =“ exclaim” args =“ b” 3. function =“ test” args =“ bar”

storm drpc-client -f exclaim a b

这将提交2个单独的DRPC请求。 1. function =“ exclaim” args =“ a” 2. function =“ exclaim” args =“ b”

Blobstore

句法：storm blobstore cmd

list [KEY ...]-列出当前在Blob存储中的Blob

cat [-f FILE]键-读取Blob，然后将其写入文件或STDOUT（需要读取访问权限）。

创建[-f文件] [-a ACL ...] [-复制系数NUMBER]键-创建一个新的blob。内容来自FILE或STDIN。 ACL的形式为[uo]：[用户名]：[r-] [w-] [a-]可以是逗号分隔的列表。

更新[-f FILE]键-更新Blob的内容。内容来自FILE或STDIN（需要写访问权限）。

Delete KEY-从Blob存储中删除条目（需要写访问权限）。

set-acl [-s ACL]键-ACL的形式为[uo]：[用户名]：[r-] [w-] [a-]可以是逗号分隔的列表（需要管理员访问权限）。

复制--read KEY-用于读取Blob的复制因子。

复制-更新-复制因子NUMBER KEY，其中NUMBER> 0。它用于更新Blob的复制因子。

例如，以下将使用存储在data.tgz中的数据创建mytopo：data.tgz密钥。用户alice将具有完全访问权限，bob将具有读/写访问权限，其他所有人将具有读访问权限。

风暴blobstore创建mytopo：data.tgz -f data.tgz -au：alice：rwa，u：bob：rw，o :: r

有关更多信息，请参见Blobstore（Distcahce） 。

开发人员

句法：storm dev-zookeeper

使用“ dev.zookeeper.path”作为其本地目录并使用“ storm.zookeeper.port”作为其端口，启动一个新的Zookeeper服务器。这仅用于开发/测试，启动的Zookeeper实例未配置为在生产中使用。

错误

句法：storm get-errors topology-name

从正在运行的拓扑中获取最新的错误。返回的结果包含错误组件的组件名称和组件错误的键值对。结果以json格式返回。

心跳

句法：storm heartbeats [cmd]

list PATH-列出当前在ClusterState中的PATH下的心跳节点。获取PATH-在PATH中获取心跳数据

kill_workers

句法：storm kill_workers

杀死在该主管上运行的工人。此命令应在主管节点上运行。如果集群以安全模式运行，则用户需要在节点上具有管理员权限才能成功杀死所有工作线程。

清单

句法：storm list

列出正在运行的拓扑及其状态。

日志查看器

句法：storm logviewer

启动日志查看器守护程序。它提供了用于查看风暴日志文件的Web界面。该命令应在daemontools或monit之类的工具的监督下运行。

有关更多信息，请参见设置Storm集群。（ http://storm.apache.org/documentation/Setting-up-a-Storm-cluster ）

监控

句法：storm monitor topology-name [-i interval-secs] [-m component-id] [-s stream-id] [-w [emitted | transferred]]

交互式监视给定拓扑的吞吐量。可以指定poll-interval，component-id，stream-id，watch-item [emitted |缺省情况下，轮询间隔为4秒。所有组件ID都将列出； stream-id为'default';监视项目是“发射”的；

节点健康检查

句法：storm node-health-check

在本地主管上运行健康检查。

起搏器

句法：storm pacemaker

启动Pacemaker守护程序。该命令应在daemontools或monit之类的工具的监督下运行。

有关更多信息，请参见设置Storm集群。（ http://storm.apache.org/documentation/Setting-up-a-Storm-cluster ）

set_log_level

句法：storm set_log_level -l [logger name]=[log level][:optional timeout] -r [logger name] topology-name

动态更改拓扑日志级别

其中日志级别是以下之一：ALL，TRACE，DEBUG，INFO，WARN，ERROR，FATAL，OFF，超时时间为整数秒。

例如./bin/storm set_log_level -l ROOT = DEBUG：30拓扑名称

将根记录器的级别设置为DEBUG 30秒

./bin/storm set_log_level -l com.myapp = WARN拓扑名称

将com.myapp记录器的级别设置为WARN 30秒

./bin/storm set_log_level -l com.myapp = WARN -l com.myOtherLogger = ERROR：123拓扑名称

将com.myapp记录程序的级别无限期设置为WARN，并将com.myOtherLogger设置为ERROR持续123秒

./bin/storm set_log_level -r com.myOtherLogger拓扑名称

清除设置，重设回原始级别

可以构造jar并上传到nimbus以使用非JVM语言

例如：storm shell resources/ python topology.py arg1 arg2

将一组新的凭据上传到正在运行的拓扑

版

句法：storm version

打印此Storm发行版的版本号。

管理员

句法：storm admin [options]

storm admin命令提供对多种操作的访问权限，这些操作可以帮助管理员调试或修复集群。

remove_corrupt_topologies -该命令应在与nimbus运行相同的用户的nimbus节点上运行。它将直接进入zookeeper + blobstore，并查找由于缺少blob而似乎已损坏的拓扑。它将杀死那些拓扑。

zk_cli [options] -此命令将启动一个zookeeper cli，它指向以nimbus用户身份登录的风暴zookeeper实例。用户nimbus的运行方式应在nimbus服务器上运行。

  • -s --server ：设置要使用的连接字符串，默认为风暴连接字符串。
  • -t --time-out ：设置要使用的超时，默认为风暴Zookeeper超时。
  • -w --write ：允许写入，默认为只读，我们不想引起问题。
  • -n --no-root ：不要在默认的连接字符串上包括风暴根。
  • -j --jaas ：包括在使用ZK进行身份验证时应使用的jaas文件，默认为java.security.auth.login.config conf。

creds -打印拓扑的凭证密钥。

救命

句法：storm help [command]

打印一条帮助消息或可用命令列表

关于Apache Storm

Apache Storm与任何排队系统和任何数据库系统集成。Apache Storm的喷口抽象使集成新排队系统变得容易。同样，将Apache Storm与数据库系统集成很容易。
