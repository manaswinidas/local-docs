介绍

接收器组件负责接收群集消息。您可能会通过配置注意到，消息的接收和消息的发送是两个不同的组件，这与许多其他框架不同，但是这样做有充分的理由，可以将消息发送方式的逻辑分离开来。如何接收消息。
接收器非常类似于Tomcat连接器，它是传入集群消息的线程池的基础。接收器很简单，但是此处管理所有用于传入流量的套接字设置。

阻塞与非阻塞接收器

接收器支持非阻塞， org.apache.catalina.tribes.transport.nio.NioReceiver以及屏蔽org.apache.catalina.tribes.transport.bio.BioReceiver 。最好使用非阻塞接收器，以使您的群集增长而不会遇到线程不足的情况。
使用非阻塞接收器使您可以以非常有限的线程数来服务大量消息。通常，规则是对于小型群集，群集中的每个节点使用1个线程，然后根据您的消息频率和硬件，您会发现以一定数量达到峰值的最佳线程数。

属性

共同属性

属性描述
className接收器组件的实现。有两种实现方式， org.apache.catalina.tribes.transport.nio.NioReceiver和org.apache.catalina.tribes.transport.bio.BioReceiver 。
的org.apache.catalina.tribes.transport.nio.NioReceiver是首选的实现
address侦听传入流量的地址（网络接口）。与绑定地址相同。默认值为auto并翻译成java.net.InetAddress.getLocalHost().getHostAddress() 。
direct可能的值为true要么false 。如果希望接收者在从套接字读取数据时使用直接字节缓冲区，则设置为true。
port传入数据的侦听端口。默认值为4000 。为避免端口冲突，接收器将自动绑定到以下范围内的空闲端口： port <= bindPort < port+autoBind因此，例如，如果端口为4000，并且autoBind设置为10，则接收器将在4000-4009范围内的第一个可用端口上打开服务器套接字。
autoBind默认值为100 。如果您希望自动避免端口冲突，请使用此值，群集接收器将尝试打开服务器上的服务器套接字。 port属性端口，然后工作autoBind次数。
securePort安全监听端口。此端口已启用SSL。如果省略此属性，则不会打开SSL端口。未设置默认值，这意味着没有SSL套接字可用。
udpPortUDP侦听端口。如果省略此属性，则不会打开UDP端口。未设置默认值，这意味着没有UDP侦听器可用。
selectorTimeout轮询超时的值（以毫秒为单位） NioReceiver 。在较早版本的JDK上，存在一些错误，这些错误现在应该清除，选择器永远不会醒来。默认值是很高5000毫秒。
maxThreads接收器线程池中的最大线程数。默认值为6相对于群集中的节点数，正在交换的消息数以及运行的硬件，调整此值。更高的值并不意味着更高的效率，请根据您自己的测试结果调整此值。
minThreads接收器启动时要创建的最小线程数。默认值为6
maxIdleTime空闲线程终止之前的最大毫秒数。默认值为60000毫秒。
ooBInline套接字OOBINLINE选项的布尔值。可能的值为true要么false 。
rxBufSize接收套接字上的接收器缓冲区大小。值以字节为单位，默认值为43800个字节。
txBufSize接收套接字上的发送缓冲区大小。值以字节为单位，默认值为25188个字节。
udpRxBufSize数据报套接字上的接收缓冲区大小。默认值为25188个字节。
udpTxBufSize数据报套接字上的发送缓冲区大小。默认值为43800个字节。
soKeepAlive套接字SO_KEEPALIVE选项的布尔值。可能的值为true要么false 。
soLingerOn布尔值，用于确定是否使用SO_LINGER套接字选项。可能的值为true要么false 。默认值为true 。
soLingerTime设置SO_LINGER套接字选项时间值。该值以秒为单位。默认值为3秒。
soReuseAddress套接字SO_REUSEADDR选项的布尔值。可能的值为true要么false 。
tcpNoDelay套接字TCP_NODELAY选项的布尔值。可能的值为true要么false 。默认值为true
timeout在套接字上设置SO_TIMEOUT选项。该值以毫秒为单位，默认值为3000毫秒。
useBufferPool布尔值是否使用缓存的共享缓冲池org.apache.catalina.tribes.io.XByteBuffer对象。如果设置为true，则用于在通道上传递消息的XByteBuffer将在请求结束时回收。这意味着通道中的拦截器必须在org.apache.catalina.tribes.ChannelInterceptor#messageReceived方法已退出。
