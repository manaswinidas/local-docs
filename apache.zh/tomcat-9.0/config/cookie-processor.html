介绍

CookieProcessor元素表示将接收到的Cookie标头解析为的组件javax.servlet.http.Cookie通过可访问的对象HttpServletRequest.getCookies()并转换javax.servlet.http.Cookie通过以下方式添加到响应中的对象HttpServletResponse.addCookie()返回到客户端的HTTP标头。

CookieProcessor元素可以嵌套在Context组件内。如果不包括，将自动创建默认实现。

属性

共同属性

CookieProcessor的所有实现均支持以下属性：

属性描述
className

使用的Java类名称。此类必须实现org.apache.tomcat.util.http.CookieProcessor接口。如果未指定，将使用标准值（定义如下）。

标准实施

CookieProcessor的标准实现是org.apache.tomcat.util.http.Rfc6265CookieProcessor 。

此cookie处理器基于RFC6265，并进行了以下更改以支持更好的互操作性：

  • cookie八位字节中允许使用0x80到0xFF值，以支持在HTML 5使用的cookie值中使用UTF-8。
  • 对于没有值的Cookie，由于某些浏览器未发送该名称，因此名称后不需要'='。

RFC 6265 Cookie处理器通常比旧版Cookie解析器宽松。特别是：

  • ' = '和' / Cookie值中始终允许使用'个字符。
  • 始终仅使用名称cookie。
  • Cookie标头始终保留。

RFC 6265 Cookie处理器支持以下附加属性。

属性描述
sameSiteCookies

启用设置同站点Cookie属性。

如果值是none那么就不会设置相同站点的Cookie属性。这是默认值。

如果值是lax那么浏览器仅在相同站点请求和跨站点顶级GET请求中发送cookie。

如果值是strict那么浏览器将阻止在任何跨站点请求中发送Cookie。

旧版Cookie处理器-org.apache.tomcat.util.http。旧版Cookie处理器

这是基于RFC6265，RFC2109和RFC2616的旧版Cookie解析器。它对Cookie规范进行了严格的解释。由于浏览器的各种互操作性问题，默认情况下并非所有严格的行为都处于启用状态，如果需要，可以使用其他选项进一步放松此cookie处理器的行为。

属性描述
allowEqualsInValue

如果是这样true Tomcat将允许“ =解析未引用的Cookie值时的'个字符。如果false ，包含“ = “将在“ =遇到，则Cookie值的其余部分将被删除。

如果未设置，则符合规范的默认值false将会被使用。

allowHttpSepsInV0

如果是这样true Tomcat允许在cookie名称和值中使用HTTP分隔符。

如果未指定，则默认规范兼容值false将会被使用。

allowNameOnly

如果是这样true Tomcat将只允许使用名称cookie（带或不带尾部的' = '）在解析Cookie标头时。如果false ，仅名称的cookie将被删除。

如果未设置，则符合规范的默认值false将会被使用。

alwaysAddExpires

如果是这样true Tomcat始终会向SetCookie标头添加一个expires参数，即使对于版本大于零的cookie也是如此。这是为了解决一个已知的IE6和IE7错误，该错误导致我忽略SetCookie标头中的Max-Age参数。

如果org.apache.catalina.STRICT_SERVLET_COMPLIANCE被设定为true ，此设置的默认值为false ，否则默认值为true 。

forwardSlashIsSeparator

如果是这样true Tomcat将对待正斜杠字符（' / '）作为处理Cookie标头时的HTTP分隔符。如果org.apache.catalina.STRICT_SERVLET_COMPLIANCE被设定为true ，此设置的默认值为true ，否则默认值为false 。

sameSiteCookies

启用设置同站点Cookie属性。

如果值是none那么就不会设置相同站点的Cookie属性。这是默认值。

如果值是lax那么浏览器仅在相同站点请求和跨站点顶级GET请求中发送cookie。

如果值是strict那么浏览器将阻止在任何跨站点请求中发送Cookie。

嵌套组件

CookieProcessor内不能嵌套任何元素。

特殊功能

CookieProcessor元素没有任何特殊功能。
