介绍

Tomcat Host Manager应用程序使您可以在Tomcat中创建，删除和管理虚拟主机。最佳做法是此指南随附以下文档：

  • 虚拟主机操作方法，以获取有关虚拟主机的更多信息。
  • 主机容器 ，以获取有关虚拟主机的基础xml配置和属性描述的更多信息。
  • 主机管理器应用程序-文本界面，提供命令的完整说明。

Tomcat Host Manager应用程序是Tomcat安装的一部分，默认情况下可使用以下上下文使用： /host-manager 。您可以通过以下方式使用主机管理器：

  • 利用图形用户界面，可从以下位置访问： {server}:{port}/host-manager/html 。
  • 使用一组适合脚本编写的最小HTTP请求。您可以通过以下方式访问此模式： {server}:{port}/host-manager/text 。

两种方式都可以使您添加，删除，启动和停止虚拟主机。可以通过使用persist命令。本文档重点介绍HTML界面。有关图形界面的更多信息，请参阅Host Manager App-文本界面 。

配置Manager应用程序访问

以下说明使用$CATALINA_HOME引用基本的Tomcat目录。例如，它是安装Tomcat的目录。 C:/tomcat9 ， 要么/usr/share/tomcat9 。

主机管理器应用程序要求用户具有以下角色之一：

  • admin-gui -将此角色用于图形Web界面。
  • admin-script -在脚本Web界面中使用此角色。

要允许访问主机管理器应用程序的HTML界面，请为Tomcat用户授予适当的角色，或创建一个具有正确角色的新角色。例如打开${CATALINA_BASE}/conf/tomcat-users.xml并输入以下内容：

<user username="test" password="chang3m3N#w" roles="admin-gui"/>

无需其他设置。现在访问{server}:{port}/host-manager/html ，您就可以使用创建的凭据登录。

请注意，如果您使用DataSourceRealm ， JDBCRealm ， 要么JNDIRealm机制，分别在数据库或目录服务器中添加适当的角色。

接口说明

该界面分为六个部分：

  • 消息 -显示成功和失败消息。
  • 主机管理器 -提供基本的主机管理器操作，例如列表和帮助。
  • 主机名 -提供虚拟主机名的列表，并使您可以操作它们。
  • 添加虚拟主机 -使您可以添加新的虚拟主机。
  • 持久配置 -使您能够持久保留当前的虚拟主机。
  • 服务器信息 -有关Tomcat服务器的信息。

信息

显示有关您执行的上一个主机管理器命令成功或失败的信息：

  • 成功：显示OK ，然后可能会显示一条成功消息。
  • 失败：显示FAIL ，然后显示错误消息。

请注意，Tomcat服务器的控制台可能会显示有关每个命令的更多信息。

房东经理

主机管理器部分使您能够：

  • 列出虚拟主机 -刷新当前配置的虚拟主机的列表。
  • HTML Host Manager帮助 -文档链接。
  • 主机管理器帮助 -文档链接。
  • 服务器状态 -指向Manager应用程序的链接。请注意，您的用户必须具有足够的权限才能访问该应用程序。

主机名

主机名部分包含当前配置的虚拟主机名的列表。它使您能够：

  • 查看主机名
  • 查看主机名别名
  • 执行基本命令，即start ， stop和remove 。

添加虚拟主机

使用“添加虚拟主机”部分，您可以使用图形界面添加虚拟主机。有关每个属性的描述，请参阅Host Manager App-文本界面文档。请注意，通过此接口添加的任何配置都是非持久性的。

持续配置

“持久配置”部分使您可以将当前配置持久保存到server.xml文件中。

默认情况下禁用此功能。要启用此选项，您必须配置StoreConfigLifecycleListener听众第一。为此，请将以下侦听器添加到server.xml中 ：

<Listener className="org.apache.catalina.storeconfig.StoreConfigLifecycleListener"/>

配置侦听器后，单击“ 全部”使配置持久化。

服务器信息

提供有关当前正在运行的Tomcat实例，JVM和基础操作系统的基本信息。
