总览

介绍

本规范的目的是为管理应用程序定义高级要求，这些要求可用于管理正在运行的Tomcat容器的操作。应支持对支持的管理功能的各种访问方法 ，以满足各种要求：

  • 作为可编写脚本的Web应用程序 -现有Manager Web应用程序提供了一个简单的基于HTTP的界面，用于通过完全通过请求URI表示的命令来管理Tomcat。这在希望使用可生成HTTP事务的工具编写管理命令脚本的环境中很有用。
  • 作为基于HTML的Web应用程序 -使用HTML演示文稿来提供类似于GUI的用户界面，以便人类与管理功能进行交互。
  • 作为基于SOAP的Web服务 -用于管理Tomcat的操作命令可作为使用SOAP消息格式的Web服务使用。
  • 作为Java管理扩展（JMX）命令 -通过JMX API提供了用于管理Tomcat的操作命令，以便集成到使用它们的管理控制台中。
  • 其他远程访问API-其他远程访问API（例如JINI，RMI和CORBA）也可以用于访问管理功能。

在上述所有访问方法的基础上，假设实际操作是直接在相应的Catalina组件上执行的（例如，调用Deployer.deploy()部署新Web应用程序的方法），或通过可在所有访问方法之间共享的“业务逻辑”层。这种方法最大程度地减少了以后添加新管理功能的成本-只需添加相应的业务逻辑功能，然后为所有所需的访问方法向其编写适配器。

该功能规范的当前状态为“ 建议” 。尚未在TOMCAT-DEV邮件列表上进行讨论和同意。

外部规格

此功能的实现取决于以下外部规范：

  • Java IDL （用于CORBA，包含在JDK中）
  • Java管理扩展
  • 远程方法调用 （包含在JDK中）

实施要求

此功能的实现应符合以下要求：

  • 在最大可行的范围内，所有管理功能以及支持它们的访问方法应可移植地在Tomcat本身运行的所有平台上运行。
  • 在默认的Tomcat发行版中，应禁用所有管理功能。系统管理员必须专门启用所需的访问方法（例如，通过将具有特定角色的用户名/密码添加到Tomcat用户的数据库中）。
  • 管理功能应通过直接调用相应的Catalina API来实现，或通过独立于用于启动它的访问方法的业务逻辑层来实现。
  • 通用业务逻辑组件应在程序包中实现org.apache.catalina.admin 。
  • 通用业务逻辑组件应作为标准Catalina构建过程的一部分进行构建，并在Catalina类加载器中可见。
  • 每种访问方法所需的Java组件应在以下组件的子包中实现： org.apache.catalina.admin 。
  • 构建脚本应将每种访问方法都视为可选方法，以便仅当在构建时存在相应的必需API时才构建该访问方法。
  • 应该可以保存正在运行的Tomcat容器的配置状态，以便在关闭并重新启动容器时可以重现此状态。
  • 对于这些应用程序而言，用于启动和关闭整个Tomcat容器的管理命令超出了范围 。假定其他（通常是特定于平台的）机制将用于容器启动和关闭。

功能性

受管理对象的属性

管理应用程序的功能要求是根据“ 管理对象”指定的， 此处列出了其定义和详细属性。通常，受管理对象与Catalina体系结构中的组件相对应，但是出于以下原因，此处分别定义了这些对象：

  • 管理应用程序可能不会公开基础组件的每个可能的配置方面。
  • 在某些情况下，一个管理对象（从管理操作的角度来看）是由多个Catalina组件实现的，并且其详细程度更高。
  • 有必要与组件本身分开表示组件的配置信息（例如，为了存储该配置信息以供以后使用）。
  • 没有相应的组件实例时，有必要表示配置信息（例如，默认上下文）。
  • 当实现为Java类时，被管理对象将包括用于管理操作的方法，这些方法与对应的实际组件所执行的操作不对应。

假定读者熟悉Catalina的整体组件体系结构。有关更多信息，请参见相应的开发人员文档。区分用作管理对象和管理对象的名称Components ，使用了不同的字体演示。许多属性的默认值列在[方括号]中。

支持的行政运作

可用的管理操作根据相应的“管理对象”（如上定义）进行描述，其方式与请求这些操作的访问方法无关。通常，此类操作仅在特定受管对象的上下文中相关（并且很可能会通过对相应受管对象类的方法调用来实现），因此它们是基于当前“集中”的受管对象来组织的。此处记录了可用的支持的操作。

上述整个行政操作套件应通过为人机交互设计的Web应用程序提供。此外，此Web应用程序应符合以下要求：

  • 必须使用上面“外部技术”中描述的servlet，JSP和MVC框架技术来实现。
  • 提示和错误消息必须可国际化为多种语言。
  • 渲染的HTML必须与Netscape Navigator（4.7或更高版本）和Internet Explorer（5.0或更高版本）兼容。
