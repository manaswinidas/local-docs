验证Geronimo版本

由Apache Geronimo Project分发的所有正式代码发布版均由该版本的发布管理器签名。PGP签名和MD5散列随发行版一起提供。

您应该直接从Apache软件基金会而不是从我们的镜像下载PGP签名和MD5哈希。这是为了确保签名文件的完整性。但是，建议您从我们的镜像下载版本。（我们的下载页面将您指向发布的镜像和签名的官方网站，因此自动为您完成。）

检查签名

以下示例详细说明了签名交互的工作方式。在此示例中，假定您已经下载了geronimo-tomcat-j2ee-1.1.tar.gz （发布）和geronimo-tomcat-j2ee-1.1.tar.gz.asc （分离的签名）。

本示例使用GNU Privacy Guard 。任何符合OpenPGP的程序均应成功运行。

首先，我们将检查分离的签名geronimo-tomcat-j2ee-1.1.tar.gz.asc反对我们的释放geronimo-tomcat-j2ee-1.1.tar.gz 。

% gpg geronimo-tomcat-j2ee-1.1.tar.gz.asc
gpg: Signature made Mon Jun 26 15:26:36 2006 AUSEST using DSA key ID A46C4CA1
gpg: Can't check signature: public key not found

我们的本地系统中没有发布经理的公钥（A46C4CA1）。现在，您需要从密钥服务器检索公共密钥。一种流行的服务器是pgpkeys.mit.edu （具有Web界面 ）。公钥服务器链接在一起，因此您应该可以连接到任何密钥服务器。

% gpg --keyserver pgpkeys.mit.edu --recv-key A46C4CA1
gpg: requesting key A46C4CA1 from hkp keyserver pgpkeys.mit.edu
gpg: trustdb created
gpg: key A46C4CA1: public key "Matt Hogstrom " imported
gpg: Total number processed: 1
gpg: imported: 1

在此示例中，您现在已收到称为“ Matt Hogstrom ”的实体的公共密钥。 但是，您无法验证此密钥是否由称为Matt Matt Hogstrom的人创建。但是，让我们尝试再次验证发布签名。

% gpg geronimo-tomcat-j2ee-1.1.tar.gz.asc
gpg: Signature made Mon Jun 26 15:25:36 2006 AUEST using DSA key ID A46C4CA1
gpg: Good signature from "Matt Hogstrom "
gpg: checking the trustdb
gpg: no ultimately trusted keys found
gpg: WARNING: This key is not certified with a trusted signature!
gpg: There is no indication that the signature belongs to the owner.
Fingerprint: 9056 B710 F1E3 3278 0DE7 AF34 CBAE BE39 A46C 4CA1

在这一点上，签名是好的，但是我们不信任此密钥。好的签名意味着该文件未被篡改。但是，由于公用密钥密码学的性质，您需要另外验证密钥A46C4CA1是由真正的 Matt Hogstrom创建的。

任何攻击者都可以创建公钥并将其上传到公钥服务器。然后，他们可以创建由该假密钥签名的恶意发行版。然后，如果您尝试验证此损坏版本的签名，则它将成功，因为该密钥不是“真实”密钥。因此，您需要验证此密钥的真实性。

验证密钥的真实性

您可以从我们的网站下载Apache Geronimo开发人员的公共密钥，也可以从公共PGP密钥服务器中检索它们（请参见上文）。但是，导入这些密钥不足以验证签名的完整性。如果发行版验证良好，则需要验证该密钥是由Apache Geronimo Project的官方代表创建的。

验证的关键步骤是确认公用密钥的密钥指纹。

% gpg --fingerprint A46C4CA1
pub 1024D/A46C4CA1 2006-01-05
Key fingerprint = 9056 B710 F1E3 3278 0DE7 AF34 CBAE BE39 A46C 4CA1
uid Matt Hogstrom
sub 2048g/2FD8C3E0 2006-01-05

验证密钥的良好起点是与政府签发的多张照片身份确认书进行面对面交流。但是，每个人都可以有自己的标准来确定钥匙的真实性。有些人通过电话阅读密钥签名（语音验证）感到满意。有关确定哪种信任级别最适合您的更多信息，请阅读GNU隐私手册中有关验证公共密钥环上其他密钥的部分。

大多数Apache Geronimo开发人员都尝试对彼此的密钥进行签名（通常使用面对面验证）。因此，为了进入信任网，您只需要在我们的信任网中验证一个人即可。（提示：我们所有开发人员的密钥都在KEYS文件中。）

由于开发人员通常很忙，因此您可能不会立即在愿意面对面见面的人中找到成功（他们可能因为忙碌而无法回复您的电子邮件！）。如果附近没有开发商，或者找不到合适的人，请发送电子邮件至您要验证的钥匙地址。他们可能能够找到愿意验证其密钥或安排替代验证机制的人。

进入信任网后，在验证发行签名后，应该会看到以下内容。

% gpg geronimo-tomcat-j2ee-1.1.tar.gz.asc
gpg: Signature made Mon Jun 26 15:25:36 2006 AUEST using DSA key ID A46C4CA1
gpg: Good signature from "Matt Hogstrom "
