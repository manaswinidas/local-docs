运行GShell并获得帮助

一个简单的启动脚本/.bat文件位于/bin目录，在哪里是服务器的安装目录。

  1. 在linux / Unix / Solaris上，执行/bin/gsh.sh
  2. 在Windows上，执行\ bin \ gsh.bat

注意 ：请勿使用--secure选项（在Geronimo 2.1.0.1或更高版本中受支持）启动GShell。它仅在GShell命令中受支持，而在脚本中不受支持。

帮助 （或？ ）仅command命令将显示当前环境中可用的所有GShell命令。要获取有关任何特定命令的帮助信息，请使用--help （ -h ）选项。这是一个例子：

deploy/list-modules --help

要么

deploy/list-modules -h

您可以使用exit或quit命令退出GShell。

注意： Windows用户， 请在GShell命令中使用正斜杠“ /”代替传统的反斜杠“ \”作为目录分隔符。使用反斜杠可能会导致在处理命令时出错。

设置和取消设置变量

set命令可用于设置变量，并遵循以下语法：

set <variable1>=string
set <variable2>="A string separated by space"

variable1是一个不包含空格或特殊字符的变量，因此不需要添加引号。这是一个例子：

set username=system
set password=manager
deploy/connect -u $username -w $password

set newstring="two strings"
echo $newstring

unset命令用于取消以前的设置。

unset <variable>

连接到已经运行的服务器

GShell允许您在远程服务器上运行一系列命令。为此，您首先需要连接到远程服务器。deploy / connect命令可用于连接到已经在运行的Geronimo实例。

deploy/connect -u <user> -w <password> -s <server hostname> -p <port> --secure

有关此命令的选项的信息，请参见常规选项 。

deploy / disconnect命令可用于与已连接的服务器断开连接。由于一次只能连接一个服务器实例，因此不需要其他选项来指定要从哪个服务器断开连接。如果尝试连接到第二个服务器实例，请先使用此命令断开连接。

保存用于当前连接的用户名和密码

使用Gshell，您可以在连接到正在运行的服务器后保存凭据。只需使用deploy / login指定您的用户名和密码，就不会为重复输入证书而烦恼。该命令的行为与deploy的 login命令选项相同。

deploy/login -u <user> -w <password> -s <server hostname> -p <port>

有关此命令的选项的信息，请参见常规选项 。

部署模块

deploy / deploy命令可用于将模块部署到先前已连接的服务器，如连接到已经运行的服务器中所述 。如果没有现有连接可用，那么deploy / deploy命令将首先建立连接，然后执行特定命令。部署后，模块将通过Geronimo中的模块ID进行标识。deploy / deploy命令具有以下语法：

deploy/deploy <module> <deployment plan> -u <user> -w <password> -s <server hostname> -p <port> -t <target1;target2> -i --secure

模块文件可以是以下之一：

  • J2EE企业应用程序归档文件（EAR）文件
  • J2EE Web应用程序存档（WAR）文件
  • J2EE Enterprise JavaBean存档（JAR）文件
  • J2EE Java资源存档（RAR）文件

如果WAR文件的部署计划不在WEB-INF目录中，则必须在命令中的模块之后指定其位置。

选项

用法

描述

-i或--inPlace

-一世

可用于从您实际正在开发应用程序的目录中指定就地部署。

-t或--targets

-t

可用于指定模块应部署到的存储库目标。您可以列出目标以获取Geronimo服务器上可用的目标的列表。

如果提供了-i选项，则需要提供到应用程序的路径来代替模块的位置。有关此命令的其他选项的信息，请参见常规选项 。

重新部署模块

deploy / redeploy命令用于将模块的较新版本部署到已经部署了较旧模块的服务器上。它的功能类似于部署/部署，但缺少就地部署选项。
deploy / redeploy命令具有以下语法：

deploy/redeploy <module> <deployment plan> <module_id> -u <user> -w <password> -s <server hostname> -p <port> --secure

如果未指定module_id ，则将使用提供的计划（或模块内部的计划）来确定要重新部署的实际配置。通过使用现有模块ID重新部署计划，您可以修改正在运行的模块的配置，而无需进行中间的取消部署。有关此命令的选项的信息，请参见常规选项 。

取消部署模块

deploy / undeploy命令用于从服务器中正确删除模块。取消部署后，除非再次使用deploy命令，否则无法再次启动该模块。必须为您要取消部署的模块提供模块ID。

deploy/undeploy -u <user> -w <password> -s <server hostname> -p <port> <module_id> --secure

有关此命令的选项的信息，请参见常规选项 。

分配模块

deploy / distribute命令的工作原理与deploy / deploy完全相同，只是模块一旦部署到服务器中就不会启动，并且每次服务器启动时都不会标记为已启动，因此该命令不会生效。可以使用与deploy / deploy相同的方式和相同的选项来发出命令。-t选项可用于指定模块应分配到的存储库目标。您可以列出目标以获取Geronimo服务器上可用的目标的列表。有关此命令的其他选项的信息，请参见常规选项 。

deploy/distribute <module> <deployment plan>  -u <user> -w <password> -s <server hostname> -p <port> -i -t <target1;target2> --secure

启动模块

deploy / start命令启动未运行的先前部署的模块，并使用以下语法：

deploy/start -u <user> -w <password> -s <server hostname> -p <port> <module_id> --secure

有关此命令的选项的信息，请参见常规选项 。

停止模块

deploy / stop命令停止正在运行的模块，并使用以下语法：

deploy/stop -u <user> -w <password> -s <server hostname> -p <port> <module_id> --secure

可以使用与deploy / start相同的选项来发出命令。

重新启动模块

deploy / restart命令ommand重新启动已经运行的模块或先前停止的模块。

deploy/restart -u <user> -w <password> -s <server hostname> -p <port> <module_id> --secure

可以使用与deploy / start相同的选项来发出命令。

清单模块

deploy / list-modules命令列出活动服务器上的可用模块，并使用以下语法：

部署/列出模块-u -w -s -p filterOptions-安全

哪里

创建一个新实例

deploy / new-instance命令可用于从当前实例创建新的服务器实例，并使用以下语法：

连接到远程Gshell

该remote / rsh可用于在远程Gshell服务器上执行gshell命令。

remote/rsh tcp://<REMOTEIP>:<PORT> <GShellCommand>

其中REMOTEIP是运行远程GShell服务器的IP地址， PORT是远程Gshell服务器上的侦听端口，而GShellCommand是可以在任何GShell环境中执行的命令。

启动远程GShell

remote / rsh-server命令可用于启动远程GShell会话，以侦听端口以接受来自外部地址的请求。

remote/rsh-server tcp://<LOCALIP>:<PORT>

其中LOCALIP是GShell服务器的IP地址， PORT是侦听端口，您可以指定任何未占用的端口号。

监控集群心跳

启用基于插件的Farming时， cluster / heartbeat命令可用于监视集群心跳。

cluster/heartbeat -f <Regex>

其中Regex是用于过滤显示的心跳数据的正则表达式。除-f选项外，还可以使用--filter 。

从类生成WSDL文件

jaxws / java2ws命令可用于从Web服务端点的实现（SEI）类和关联的类型类生成WSDL文件，包装bean，服务器端代码和客户端代码。它使用以下语法：

jaxws/java2ws -databinding <jaxb or aegis> -frontend <jaxws or simple> -wsdl -wrapperbean -client -server -ant -wrapperbean -o 
<output-file> -d <resource-directory> -s <source-directory> -classdir <compile-classes-directoty> -cp <class-path> -soap12 -t 
<target-namespace> -beans <pathname of the bean definition file> -servicename <service-name> -portname <port-name> -address 
<address> -createxsdimports -h -v -verbose -quiet <classname>

下表描述了这些选项：

从类生成JAX-WS工件

jaxws / wsgen命令可用于从Java类为JAX-WS应用程序生成必要的可移植工件。与jaxws / java2ws不同，此命令仅在需要时才生成WSDL文件。它使用以下语法：

jaxws/wsgen -classpath <path> -cp <path> -d <directory> -extension -help -keep -r <directory> -verbose -version -wsdl[:protocol]
 -servicename <name> -portname <name>

从WSDL生成Java类

jaxws / wsdl2java命令可用于从WSDL创建Java SEI类，并使用以下语法：

jaxws/wsdl2java -fe <front-end-name>* -db <data-binding-name>* -wv <wsdl-version> -p <[wsdl-namespace = ]package-name>* -sn 
<service-name> -b <bindling-file-name> -catalog <catalog-file-name> -d <output-directory> -compile -classdir <compile-classes-directory> -impl -server -client 
-all -autoNameResolution -defaultValues<=class-name-for-DefaultValueProvider> -ant -nexclude <schema-namespace [=java=package-
name]>* -exsh <<true,false>> -dns <<true,false>> -dex <<true,false>> -validate -keep -wsdllocation <wsdlLocation> -xjc 
<xjc-arguments> -noAddressBinding -h -v -verbose -quiet <wsdlurl>

从WSDL生成JAX-WS工件

jaxws/wsimport -b <path> -B <jaxbOption> -catalog <file> -d <directory> -extension -help -httpproxy:<host>:<port> -keep -p <pkg>
-quiet -s <directory> -target <version> -verbose -version -wsdllocation <location>

-B
