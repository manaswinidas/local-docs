企业Java Bean已成为Java EE规范的基石之一。作为经过Java EE 5认证的应用服务器，Apache Geronimo在OpenEJB EJB容器的帮助下广泛支持EJB。尽管可以使用标准Java对象来包含您的业务逻辑和业务数据，但是使用EJB可以解决许多使用简单Java对象的问题，例如可伸缩性，生命周期管理和状态管理。在本文中，您将看到如何扩展初始数据库应用程序并将其用于Enterprise Java Beans后端的本地和远程引用的应用程序客户端。该应用程序使用内置的Apache Derby作为其数据库。使用本文来学习如何简化您的企业应用程序开发过程。

银行应用程序具有两种类型的应用程序客户端，即“银行远程应用程序”和“银行Web应用程序”。这些客户机中的每一个都演示了如何分别在远程和本地接口中引用Enterprise Java Bean。这两个客户端都引用了一个公共的业务层，该层已经通过会话和实体Bean的帮助实现了。无状态会话Bean充当业务实体和应用程序客户端之间的业务服务接口。应用程序层的所有业务实体都是通过实体Bean来实现的。

阅读本文之后，您应该能够充分利用Geronimo的EJB功能，例如定义Enterprise Java Bean，管理它们之间的关系以及通过不同类型的客户端引用EJB。

本文分为以下部分：

EJB功能概述

EJB实现可能因一家供应商而异。以下是Apache Geronimo支持的主要功能的列表。

  • 有状态和无状态会话Bean
  • 实体豆
  • 消息驱动bean（MDB）
  • 使用RMI-IIOP或JAXRPC的互操作性
  • 能够将无状态会话bean和MDB公开为Web服务
  • 支持通过Web服务发送和接收消息
  • 轻松置备和热部署EJB和基于JMX的Web服务
  • 从外部CORBA对象访问EJB
