在Tomcat中配置虚拟主机主题介绍了如何在Geronimo的虚拟主机中定义虚拟主机。 config.xml文件，以便可以在应用程序部署计划侧以最小的配置将应用程序部署到特定的主机（ 虚拟主机 ）。但是，需要在服务器级别定义虚拟主机（ config.xml ）仅适用于带有Tomcat发行版的Geronimo。

排队

对于Apache Geronimo的Jetty发行版，虚拟主机的配置非常简单。

您只需要定义标签在应用程序的部署计划中，并确保主机名（虚拟主机）可以由客户端解析。实际上，Geronimo服务器端不需要其他配置。

在本主题中，以“ 快速入门-急躁的Apache Geronimo”部分介绍的HelloWorld Web示例应用程序为例。您可以使用以下步骤来创建HelloWorld应用程序，并使用Jetty在Geronimo中定义虚拟主机：

  1. 创建一个app_home目录位于文件系统中的某个位置，从现在开始，我们将引用此位置为 。
  2. 创建一个HelloWorld.jsp文件在目录并将以下内容粘贴到文件中。 HelloWorld.jsp 基本的HelloWorld JSP

    您好，GENERIMO世界！

    在$ {datetime}
  3. 创建一个WEB-INF目录下 。
  4. 创建一个web.xml文件在\WEB-INF目录并将以下内容粘贴到文件中。 web.xml HelloWorld.jsp
  5. 创建一个geronimo-web.xml文件在\WEB-INF目录并将以下内容粘贴到文件中。 geronimo-web.xml sample.applications HelloWorldApp 2.1 war / hello virtualhost1.com
    如果将此部署计划与“ 快速入门-Apache Geronimo急用”部分中描述的计划进行比较，您会注意到仅添加了一行，即virtualhost1.com 。

这就是使用Jetty在Geronimo中定义虚拟主机所需要的全部。

要部署此应用程序，可以使用部署程序工具中的--inPlace选项，这样就无需打包应用程序或将其复制到其他任何地方。运行以下命令：

\ bin \ deploy-用户系统-密码管理器部署--inPlace

以下信息将显示在您的命令窗口中：

稳定的 D：\ geronimo-jetty6-jee5-2.1 \ bin>部署-用户系统-密码管理器使用GERONIMO_BASE部署--inPlace \ HelloWorld_2.1：D：\ geronimo-jetty6-jee5-2.1使用GERONIMO_HOME：D：\ geronimo-jetty6-jee5-2.1使用GERONIMO_TMPDIR：D：\ geronimo-jetty6-jee5-2.1 \ var \ temp使用JRE_HOME：C：\ Java \ jdk1.5.0_06 \\ jre部署的sample.applications / HelloWorldApp / 2.1 / war @ http：// hcunico：8080 /你好

请注意，即使屏幕上的成功部署消息显示该应用程序已部署到http：// XYZ：8080 / hello主机，该应用程序也只能通过先前在标记在geronimo-web.xml部署计划。
