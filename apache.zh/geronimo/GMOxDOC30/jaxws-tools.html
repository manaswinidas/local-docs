jaxws-tools脚本可用于生成JAX-WS Web服务中使用的可移植工件。例如，可移植工件，例如服务端点接口（SEI）类，服务类，JAXB生成的值类型等。

该命令具有以下语法：

jaxws-tools toolName [toolOptions] ...

哪里toolName可以是：

  • #wsgen-从Java生成可移植工件
  • #wsimport-从WSDL生成可移植工件

jaxws-tools命令也可以通过使用java -jar命令来启动：

java -Djava.endorsed.dirs=lib/endorsed -jar bin/jaxws-tools.jar toolName [toolOptions] ...

韦斯根

jaxws工具使用Sun提供的wsgen工具从Java类生成可移植工件。wsgen工具具有以下语法：

jaxws-tools wsgen [options] <SEI>
  • -classpath
    指定在何处查找输入类文件。
  • -d
    指定放置生成的输出文件的位置。
  • -保持
    保留生成的文件。
  • -r
    指定放置资源文件（如WSDL）的位置。
  • -s
    指定放置生成的源文件的位置。
  • -wsdl
    生成WSDL文件。
  • -服务名称
    指定要在生成的WSDL中使用的服务名称（与-wsdl选项结合使用）。
  • -端口名
    指定要在生成的WSD中使用的端口名称（与-wsdl选项结合使用）。

例：

jaxws-tools wsgen -d output -keep -wsdl -classpath . foo.BarService

导入

jaxws-tools使用Sun提供的wsimport工具从WSDL生成可移植工件。 wsimport具有以下语法：

jaxws-tools wsimport [options] <WSDL_URI>
  • -d
    指定放置生成的输出文件的位置。
  • -保持
    保留生成的文件。
  • -p
    指定目标软件包。
  • -s
    指定放置生成的源文件的位置。
  • -wsdllocation
    指定的@ WebService.wsdlLocation和@ WebServiceClient.wsdlLocation值。

例：

jaxws-tools wsimport -d output -keep http://localhost:8080/foo/Bar?wsdl
