总览

在Geronimo中部署资源适配器需要Geronimo计划。这可能是一个外部计划，也可能打包为“ META-INF / geronimo-ra.xml ”打包在资源适配器rar文件中（而不是rar文件中的jar文件）。部署计划与ra.xml Java EE部署计划结合使用，以将JCA连接器RAR部署到Geronimo应用程序服务器。它用于指定已部署模块的moduleId，任何第三方依赖项，出站连接池参数，覆盖的config-properties，管理对象（例如jms目标）和其他GBean。

打包

geronimo-ra.xml部署计划可以打包如下：

  1. 嵌入JAR文件中。在这种情况下，必须将geronimo-ra.xml文件放在JAR的/ META-INF目录中，该目录与ra.xml文件所在的位置相同。

  2. 与JAR文件分开维护。在这种情况下，必须将文件的路径提供给适当的Geronimo部署程序（例如，命令行或控制台）。请注意，在这种情况下，文件名可以命名为geronimo-ra.xml以外的其他名称，但必须遵循相同的架构。

  3. 嵌入到应用程序EAR文件中，并由 EAR部署计划的元素。

架构图

geronimo-ra.xml部署计划由位于Geronimo主安装目录的/ schema /子目录中的geronimo-connector-1.2.xsd模式定义。此处简要描述了geronimo-connector-1.2.xsd模式：

http://geronimo.apache.org/schemas-3.0/docs/geronimo-connector-1.2.xsd.html
