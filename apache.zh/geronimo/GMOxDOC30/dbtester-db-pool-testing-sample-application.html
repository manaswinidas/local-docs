访问应用程序服务器特定的功能并在J2EE应用程序中使用它们将使其比仅从其中访问J2EE功能更强大。它使您能够将扩展写入应用程序服务器。

该示例应用程序列出了Geronimo中定义的所有数据库连接池。您可以选择这些连接池中的任何一个，并针对数据库测试连接性，并列出现有的架构和表。此外，用户可以查看每个列出的表的记录。

阅读本文之后，您应该能够从应用程序访问Geronimo特定的资源，并以有效的方式使用它们。

本文分为以下几节：

应用概述

本文介绍的示例应用程序将帮助您测试Geronimo服务器中部署的数据库连接池。可以将其视为Geronimo控制台的扩展，因为当前版本不包含在部署数据库池后测试与数据库池的连接的功能。

下图说明了应用程序流程。

该应用程序的欢迎页面用作布告栏，其中显示了Geronimo应用程序服务器中部署的数据库连接池的列表。用户可以从第一页直接测试那些连接池。如果该特定的连接池需要用户名和密码来建立连接，请在出现的弹出窗口中输入这些详细信息。数据库模式列表以及与连接池关联的表将显示在“模式和表”页面中。可以从那里访问每个表的内容。

申请内容

库存应用程序由以下软件包列表组成。

  • org.apache.geronimo.samples.dbtester.beans
    • DBManagerBean-处理大多数应用程序逻辑（包括对Geronimo Kernel的访问）的应用程序的心脏。
  • org.apache.geronimo.samples.dbtester.web
    • ContentTableServlet-获取数据库表的内容并将其传递给演示文稿
      层。
    • ListTablesServlet-获取与数据库池关联的模式和表的列表。

用法

该应用程序可从http：// localhost：8080 / dbtester获得 。请注意，如果使用-Pit运行maven，则获得的示例服务器将不会显示任何数据，因为它们不包含任何数据库池。更为现实的观点是在完整的geronimo服务器上安装插件或应用程序。

摘要

本文向您展示了如何从J2EE应用程序访问与Geronimo相关的功能。您按照分步说明构建，部署和测试示例应用程序，以详细说明这些功能。该示例应用程序可用作Geronimo中部署的数据库连接池的测试器。
