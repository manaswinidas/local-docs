在Harmony上运行Geronimo

本文介绍了使用Apache Harmony作为JVM运行Geronimo v2.1的方式和已知问题。

一般说明

本文是使用以下环境编写的：

Geronimo v2.1 Jetty完整J2EE版本（ geronimo-jetty6-javaee5-2.1-SNAPSHOT-bin.zip ）
和谐快照R617157（2008年1月31日），Windows / ia32 / msvc 1310，调试版本
Microsoft Windows XP Professional Service Pack 2，32位，英语

对于其他版本和不同的环境，其行为可能有所不同。

尚未解决的问题

以下是需要解决的所有未解决问题的简短摘要：

  • HARMONY-5191 – Harmony没有SSL实施（已调查，已变通 ）。
  • HARMONY-5462 – 和声不支持-javaagent VM选项（已调查，已解决 ）。
  • HARMONY-4014 – Geronimo占用了所有可用的CPU资源（已调查，已解决 ）。
  • HARMONY-5523 – Geronimo Console错误地显示了内存使用信息（已调查，不是很重要）。
  • 应用程序无法正常终止–按下Ctrl-C启动从未完成的关机过程（未经调查）。

Geronimo配置

建议您备份自己的var/config/config.xml文件编辑之前–如果您想在Sun Java上运行Geronimo安装，或者在编辑时出错并损坏且无法修复，则可能需要原始文件。

配置服务器以对JNDI使用Harmony RMI Registry提供程序

和谐将JNDI 提供者置于org.apache.harmony.jndi包树，不在com.sun.jndi就像Sun Java一样，因此您必须将Geronimo指向正确的方向才能找到并正确访问RMI Registry提供程序。

为此，编辑var/config/config.xml文件，调整NamingProperties GBean配置，如下所示：

xml solid org.apache.harmony.jndi.provider rmi：// $ {ServerHostname}：$ {NamingPort + PortOffset}

使用TLS代替SSL

Harmony没有SSL实现（请参阅HARMONY-5191 ），因此必须将以下语句添加到var/config/config.xml指示Geronimo使用TLS而不是SSL的文件 ：

XML 固态 TLS ...

更改默认密钥库类型

Geronimo使用Harmony不支持的JKS密钥库，有关详细信息，请参见GERONIMO-3757 。因此，您必须生成另一个密钥库，即PKCS12，BKS（ Harmony的默认值）或Harmony支持的另一种类型，并指示Geronimo使用它。

要将现有的Geronimo JKS密钥库转换为PKCS12类型，可以使用简单的Java程序，例如JKStoPKCS12.java 。该程序需要在Sun Java上运行，因为它同时支持JKS和PKCS12密钥库。如果需要BKS密钥库，则需要再次转换您的PKCS12密钥库，并在Harmony上运行PKCS12toBKS.java –因为Sun Java不支持BKS，而Harmony不支持JKS。

另外，您可以使用本文附带的预生成的PKCS12或BKS密钥库： geronimo.pkcs12 ， geronimo.bks 。

要指示Geronimo使用备用密钥库，请将密钥库文件复制到var/security/keystores/目录并将以下内容添加到var/config/config.xml文件：

xml 固态 var / security / keystores / geronimo.pkcs12 PKCS12

另外，您也可以只覆盖var/security/keystores/geronimo-default ，那么您可以省略keystorePath标记在配置中。但是，不建议这样做，因为如果需要在Sun Java上运行Geronimo安装，则可能需要原始密钥库。如果您决定覆盖，请不要忘记备份原始密钥库。万一您没有备份并且需要原始密钥库，则可以使用本文所附的密钥库，以防万一： geronimo.jks 。

定制服务器启动

您可以编辑Geronimo启动脚本，也可以从命令行运行服务器，但是无论哪种情况，都应考虑以下调整：

1。和谐不接受-javaagent选项（ HARMONY-5462 ），因此您必须忽略它。或者，您可以删除bin/jpa.jar Geronimo发行版中的文件-这样-javaagent Geronimo启动脚本将省略该选项。

2。Geronimo v2.1需要访问Internet才能部署SPECjAppServer2004之类的应用程序–它试图从http://java.sun.com站点获取XML模式。因此，如果要部署这样的应用程序，并且在防火墙后面，则应添加适当的-Dhttp.proxyHost=和-Dhttp.proxyPort= Geronimo命令行的选项。此问题是由OPENEJB-700错误引起的，在修复了该错误并将其传播到新版本的Geronimo之后应该消失。

3。SPECjAppServer2004和其他应用程序可能需要针对EJB的JNDI命名约定的其他规范，例如： -Dopenejb.jndiname.failoncollision=true -Dopenejb.jndiname.format={ejbName }

4。和谐通常需要更多的内存。因此请确保添加相应的-Xms和-Xmx选项。

您的最终启动行可能如下所示：

固体

java -Xms256M -Xmx512M -Djava.endorsed.dirs=lib/endorsed -Dopenejb.jndiname.failoncollision=true -Dopenejb.jndiname.format={ejbName} -Dhttp.proxyHost=your.proxy -Dhttp.proxyPort=8080 -jar bin/server.jar

配置部署程序以将Harmony RMI Registry提供程序用于JNDI

Geronimo Deployer不使用var/config/config.xml文件（请参见上文 ），因此需要通过系统属性提供JNDI配置，如下所示：

固体

java -Djava.naming.factory.url.pkgs=org.apache.harmony.jndi.provider -jar bin/deployer.jar ...

使用DRLVM

您应该使用DRLVM （ 和谐 VM）来运行Geronimo 。

Harmony还支持IBM J9 VM，但是Geronimo不能使用它，因为Geronimo依赖java.util.concurrent需要正确实施的软件包sun.misc.Unsafe类中缺少IBM J9 VM。
