总览

排队

Java EE应用程序的Geronimo部署计划通常打包为一个JAR文件，称为“ geronimo-application-client.xml ”。

geronimo-application-client.xml部署计划是一个可选文件，但通常在部署使用Geronimo客户端应用程序容器的客户端应用程序JAR文件时使用。它用于指定已部署模块的moduleId，任何第三方依赖项，回调处理程序定义，任何消息目标，资源，其他GBean，以及对任何外部GBean，EJB，服务或资源的引用。

打包

geronimo-application-client.xml部署计划可以如下打包：

  1. 嵌入JAR文件中。在这种情况下，必须将geronimo-application-client.xml文件放在JAR的/ META-INF目录中。

  2. 与JAR文件分开维护。在这种情况下，必须将文件的路径提供给适当的Geronimo部署程序（例如，命令行或控制台）。请注意，在这种情况下，文件名可以使用geronimo-application-client.xml之外的其他名称，但必须遵循相同的架构。

  3. 嵌入到应用程序EAR文件中，并由 EAR部署计划的元素。

架构图

geronimo-application-client.xml部署计划由位于Geronimo主安装目录的/ schema /子目录中的geronimo-application-client-2.0.xsd模式定义。此处简要描述了geronimo-application-client-2.0.xsd模式：

http://geronimo.apache.org/schemas-3.0/docs/geronimo-application-client-2.0.xsd.html
