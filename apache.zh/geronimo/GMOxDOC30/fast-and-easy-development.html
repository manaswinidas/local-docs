Geronimo Eclipse插件（GEP）在Geronimo和Web工具平台（WTP）之间提供了集成。使用此插件，用户将能够使用WTP中的功能在Geronimo上创建，部署和调试应用程序。
排队

本节提供了使用Geronimo中的GEP开发和运行应用程序的快速教程。

此开发人员快速入门指南分为以下几节：

取得软件

Geronimo Eclipse插件（GEP）需要以下必备软件（所有这些都是特定于平台的）：

  • Sun JDK 6.0（J2SE 6.0）
  • 面向Java EE开发人员的Eclipse IDE

可以从Sun的Java SE Downloads站点下载Sun JDK 6.0。可以从Eclipse Downloads站点下载适用于Java EE开发人员的Eclipse IDE。下载Eclipse IDE for Java EE Developers归档文件并将其解压缩到您选择的目录（例如C：\ eclipse）。归档文件将解压缩到您指定目录（例如，C：\ eclipse \ eclipse）下名为eclipse的目录，称为从现在开始。确保PATH中的JDK是Sun JDK 6.0。通过运行/ eclipse启动Eclipse。

可以使用Geronimo Eclipse插件来安装Geronimo服务器，或者可以选择手动安装。要手动安装它，请从http://geronimo.apache.org/downloads.html下载Geronimo 3.0M1服务器，然后将归档文件解压缩到您选择的目录中（例如C：\ g）。归档文件将解压缩到目录geronimo-tomcat7-javaee5-3.0M1 （例如C：\ g \ geronimo-tomcat7-javaee6-3.0M1）或geronimo-jetty8-javaee6-3.0M1，具体取决于您是否下载了Geronimo 3.0M1使用Tomcat或Jetty作为Web容器进行分发。我们将这个目录称为从现在开始。

安装Geronimo Eclipse插件

可以通过创建指向http://www.apache.org/dist/geronimo/eclipse/updates/的新远程站点，通过Eclipse更新管理器来安装GEP。请按照以下步骤安装GEP：

  1. 通过运行/ eclipse启动Eclipse。
  2. 打开Eclipse Update Manager，如下所示：
    • 选择帮助 ， 软件更新 ， 查找并安装 。
    • 选择搜索要安装的新功能，然后单击下一步 。
  3. 创建一个Eclipse远程更新站点，如下所示：
    • 单击新建远程站点...按钮。



    • 在名称字段中输入Geronimo Eclipse Update Site（或其他合适的唯一名称）。
    • 在URL字段中输入 Geronimo Eclipse Update站点的URL http://www.apache.org/dist/geronimo/eclipse/updates 。
    • 单击确定 。
  4. 在要包括在搜索选择框中的站点中仅选择Geronimo Eclipse Update Site，然后单击Finish 。
  5. 选择一个镜像，然后根据提示单击“ 确定” 。
  6. 在搜索结果中 ，展开Geronimo Eclipse更新站点和Geronimo WTP服务器适配器 。
  7. 选择Geronimo v3.0服务器适配器3.0M1 ，然后单击下一步 。
  8. 阅读并接受任何许可协议，然后单击“ 下一步” 。
  9. 单击完成以下载所选功能。
  10. 单击全部安装 。
  11. 单击完成以完成安装。
  12. 单击“ 是”重新启动Eclipse，以使更改生效。

可以在以下位置找到GEP安装说明的最新版本： 安装Geronimo Eclipse插件 。

验证GEP安装

要验证GEP安装是否成功，请开始新的Eclipse安装，然后从工作区中选择“ 帮助 -> 关于Eclipse平台 ”。在出现的面板中，您会在4-6个Eclipse图标中的任意位置看到Eclipse图标，这些图标代表安装的各种Eclipse项目和/或组件。您还必须看到Geronimo图标，如下所示。如果没有，则说明未正确安装。如果出现了Geronimo图标，则很好地表明Eclipse和GEP已正确安装，您可以放心地继续下一步。



定义新的服务器运行时

请按照以下步骤定义新的Geronimo v3.0M1Server运行时：

  1. 使用以下之一启动“ 定义新服务器”对话框：
    • 在Java EE透视图中的“ 服务器”视图中：右键单击，选择“ 新建” ，然后选择“ 服务器” 。
    • 从任何角度看：选择File ， New ， Other ， Server 。
  2. 选择Apache Geronimo v3.0M1 Server ，然后单击Next 。
  3. 选择一个6.0 JRE（单击“ 已安装的JRE”以添加任何新的JRE，并在完成后返回此对话框）。
  4. 如果已经安装了要与GEP一起使用的Geronimo v3.0M1，请输入目录名称或浏览到然后单击下一步 。跳过接下来的三个步骤，然后转到选择源存档。
  5. 输入Application Server安装目录的目录 ，在“ 下载选项”下选择Web容器，然后单击“ 下载并安装” 。 重要

    为选择“ 下载选项”，必须存在为Application Server安装目录指定的目录。

  6. 单击“ 确定”进行安装确认。这将安装选定的Geronimo v3.0M1服务器，并相应地填写“ 应用程序服务器安装目录”字段。
  7. 单击下一步 。
  8. 选择包含Geronimo源的归档文件的位置，然后单击Next 。仅当您打算调试Geronimo源代码时才需要。
  9. 如有必要，修改主机名 ， 管理员ID ， 管理员密码 ， Web连接器端口和RMI命名端口 ，然后单击完成 。通常，您不需要为新安装的Geronimo服务器更改这些字段的默认值。

这样就完成了新的Geronimo Server运行时的定义。新添加的服务器将出现在“ 服务器”视图中。可以通过选择服务器并单击“ 启动服务器”按钮来启动服务器 。



打包您的应用程序

如果您的应用程序尚未在Eclipse工作区中，请使用以下步骤导入应用程序归档文件：

  1. 选择“ 文件” ，“ 导入”以启动“ 选择”对话框。
  2. 根据您的应用程序类型，如下选择：
    • 对于Web应用程序，选择Web ， WAR文件 。
    • 对于EAR应用程序，选择J2EE ， EAR文件 。
    • 对于EJB应用程序，选择EJB ， EJB JAR文件 。
    • 对于Aries应用程序，选择OSGi ， OSGi应用程序（EBA）或OSGi复合包（CBA） 。
  3. 单击下一步 。

    其余步骤基于Web应用程序。

  4. 键入WAR文件的名称，或通过单击Browse来选择WAR文件，输入/编辑项目名称，然后选择Apache Geronimo v3.0M1 Runtime作为Target Runtime。
  5. 单击完成以完成导入。

可以在“ 项目资源管理器”视图中查看导入的应用程序。如果归档文件中还没有GEP，则GEP将创建一个Geronimo特定的部署计划，在本例中为WebContent / WEB-INF下的geronimo-web.xml。您可能需要编辑部署计划以映射应用程序使用的所有资源。

部署您的应用

要在Geronimo上部署应用程序，请遵循以下步骤：

  1. 在“ 服务器”视图中，右键单击位于本地主机上的Apache Geronimo v3.0M1服务器，然后选择添加和删除项目 。
  2. 在“ 添加和删除项目”对话框中，在“ 可用项目”下选择项目 ，然后单击“ 添加”按钮。现在，该项目将列在“已配置的项目”下 。
  3. 点击完成 。

当服务器启动或调用发布时，将部署该应用程序。

运行你的应用程序

为了运行该应用程序，请按照以下步骤操作：

  1. 在“ 项目资源管理器”视图中，右键单击项目，依次选择“运行方式” ， “在服务器上运行” 。
  2. 选择选择现有服务器，然后在localhost选择Apache Geronimo v3.0M1服务器 。
  3. 点击完成 。

如果服务器尚未启动，它将启动服务器并运行该应用程序。该应用程序的欢迎页面将在Eclipse中的浏览器中打开。
