该向导为导入现有的BEA WebLogic数据库池提供了两种不同的选择：

  • 第一个是通过提供配置文件（例如， config.xml ）。如果选择此选项，向导将转换尽可能多的字段，然后将要求您手动输入失败的转换字段。您必须手动输入的值之一是数据库密码。
  • 如果在同一台计算机上同时安装了Apache Geronimo服务器和BEA WebLogic服务器，则第二种选择是可行的。您可以直接指定WebLogic安装路径和域目录。这种选择的优点是能够直接读取数据库密码。

除了您选择的替代方法之外，您仍然需要告诉Geronimo数据库驱动程序jar在哪里。对于此特定示例， 存储库 portlet用于安装PointBase驱动程序jar。

以下是在Geronimo中安装PointBase数据库驱动程序jar的步骤：

  1. 找到PointBase客户端驱动程序jar pbclient44.jar 。该文件位于weblogic81\common\eval\pointbase\lib目录。复制此文件并将其重命名pbclient-4.4.0.jar 。
  2. 使用存储库 portlet安装PointBase驱动程序jar。在管理控制台中，单击“ JAR存储库”以访问该Portlet。单击浏览，然后指向数据库驱动程序jar。将Group：更改为PointBase ，并在默认情况下保留其余字段。然后点击安装 。你会看见PointBase/pbclient/4.4.0/jar在存储库条目列表顶部附近列出。

在Geronimo管理控制台中，单击“ 数据库池”链接。在数据库池 Portlet中，单击从WebLogic 8.1导入 。下图说明了导入向导。

对于此特定示例，重点是导入现有BEA WebLogic数据库池的第二种方法。默认示例域是在WebLogic服务器中创建的，默认情况下还包括所有示例应用程序。该域位于\ user_projects \ domains \ examples目录中。

在导入向导的第一个屏幕上（如上图所示），为Domain目录路径：和weblogic81 / server / lib路径：输入以下值，然后单击Next 。

  • 域目录路径：\user_projects\domains\examples
  • weblogic81 /服务器/ lib路径：\weblogic81\server\lib

下图所示的步骤2显示了您指定的WebLogic域中已识别的数据库池的列表。这些数据库池可以导入到Apache Geronimo中。

请注意，如果要成功测试连接，则尝试从中导入数据库池的WebLogic域必须正在运行。对于此示例，单击列表中第二个数据库池的Confirm and Deploy链接，该链接对应于examples-dataSource-demoPool 。

在接下来的步骤中，选择您刚刚在Geronimo存储库中创建的Driver JAR ： 。

请注意，数据库密码已被识别。单击“ 测试连接” ，您将看到类似于下图的确认。点击部署 。

下一页返回到步骤2。这次它将显示可用于导入的其余数据库池，还将显示您刚导入的数据库池的导入状态。此时，您可以单击“ 跳过剩余池”以退出导入向导。

现在，您可以在数据库池portlet中看到刚导入的数据库池。
