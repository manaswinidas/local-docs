列出目录var下的所有文件，提供一般概述以及指向完整文档的链接

/ var / activemq /

  • /conf/activemq.xml/
  • /数据/

/ var / catalina

Tomcat Web容器用作主目录的文件系统目录，用于保存整个容器通用的文件。该目录配置为名为的属性catalinaHome在GBean上的Tomcat容器上org.apache.geronimo.tomcat.TomcatContainer 。此属性的默认值指定为var/catalina 。

  • /工作/

    一个目录，其中包含从JavaServer页面编译的Java类。

  • / logs /

    包含访问日志的目录。

  • /根/

    包含用于初始欢迎页面的Web模块的目录。

  • / conf /

    包含web.xml文件的目录，该文件定义默认的Web模块资源，例如默认的servlet，JSP编译器，会话超时，mime类型的映射以及欢迎文件列表。

/ var / config

一个目录，其中包含启动时Geronimo使用的配置文件。

  • artifact_aliases.properties使用此文件指示您要用一个模块替换另一个模块。
  • client_artifact_aliases.properties
  • config-substitutions.properties将变量及其替换值放在此文件中。
  • config.xml有关服务器启动期间要加载的模块的信息。
  • jsr88-configurer-config.xml Java EE部署API使用的配置文件
  • offline-deployer-config.xml offline-deployer使用的配置文件。

/ var / derby /

包含本地Derby数据库文件的目录。

/ var / log

在服务器运行时包含配置文件和日志文件的目录。查看Apache log4j以获取有关每个文件中的定义的详细信息。

  • client-log4j.properties客户端日志的配置文件
  • deployer-log4j.properties部署程序日志的配置文件
  • server-log4j.properties服务器日志的配置文件

/ var / security

  • groups.properties组列表的配置文件
  • users.properties用户列表和组的配置文件

/ var /共享

Geronimo sharedlib配置的默认目录。您可以使用这些目录轻松地在多个应用程序之间共享.jar文件或类。将jar或类放在这些目录中，并在sharedlib配置的应用程序部署计划中声明一个依赖项。例如：

plan.xml

/ var / tmp

JVM使用此临时目录进行临时文件存储。JVM被配置为在geronimo.sh（bat）脚本中将其用作其java.io.tmpdir。出于安全原因，Geronimo被配置为使用此临时目录，而不是其默认目录。临时目录必须存在，Tomcat才能正常工作。

/ var / txlog

  • howl_1.log
  • howl_2.log
    事务管理器日志，其中包含准备好的事务和已提交的事务的条目。在事务管理器恢复期间，这些将被匹配，而不匹配的将完成其事务提交。
