配置JAX-WS引擎

排队

Apache Geronimo带有两个JAX-WS引擎： Apache Axis2和Apache CXF 。

Geronimo-Jetty程序集默认使用CXF作为其JAX-WS引擎，而Geronimo-Tomcat程序集使用Axis2。

有两种方法可以配置应使用的JAX-WS引擎：

  • ＃设置系统属性
  • ＃更改配置文件

设置系统属性

要将Axis2配置为Geronimo的JAX-WS引擎，请将org.apache.geronimo.jaxws.provider系统属性设置为axis2 。例如：

在Windows系统上：set GERONIMO_OPTS=-Dorg.apache.geronimo.jaxws.provider=axis2
在类似Unix的系统上：export GERONIMO_OPTS=-Dorg.apache.geronimo.jaxws.provider=axis2

要将CXF配置为Geronimo的JAX-WS引擎，请将org.apache.geronimo.jaxws.provider系统属性设置为cxf 。例如：

在Windows系统上：set GERONIMO_OPTS=-Dorg.apache.geronimo.jaxws.provider=cxf
在类似Unix的系统上：export GERONIMO_OPTS=-Dorg.apache.geronimo.jaxws.provider=cxf

更改配置文件

要将Axis2配置为Geronimo的JAX-WS引擎，请修改/var/config/config.xml通过以下方式归档：

  1. 除去condition属性，并将load =“ false”属性添加到org.apache.geronimo.configs / cxf-deployer // car模块。
  2. 除去condition属性，并将load =“ true”属性添加到org.apache.geronimo.configs / axis2-deployer // car模块。

要将CXF配置为Geronimo的JAX-WS引擎，请修改/var/config/config.xml通过以下方式归档：

  1. 除去condition属性，并将load =“ false”属性添加到org.apache.geronimo.configs / axis2-deployer // car模块。
  2. 除去condition属性，并将load =“ true”属性添加到org.apache.geronimo.configs / cxf-deployer // car模块。

SAAJ引擎

默认情况下，CXF将使用Sun的带有Java附件API的SOAP（SAAJ）实现。该实现需要Sun JDK。对于无法使用Sun JDK的平台，有一些选项：

  1. 切换到Axis2 SAAJ实现。在Geronimo中，可以通过设置org.apache.geronimo.saaj.provider系统属性来配置应使用哪个SAAJ提供程序。要使用Axis2 SAAJ实现，请将org.apache.geronimo.saaj.provider系统属性设置为axis2 。例如：

    在Windows系统上：set GERONIMO_OPTS=-Dorg.apache.geronimo.saaj.provider=axis2
    在类似Unix的系统上： export GERONIMO_OPTS=-Dorg.apache.geronimo.saaj.provider=axis2

  2. 将Sun XML解析器jar添加到类路径。下载： https : //maven-repository.dev.java.net/nonav/repository/com.sun.xml.parsers/jars/jaxp-ri-1.4.2.jar并将其放在/lib/endorsed/目录。

    请注意，这些选项尚未经过测试。
