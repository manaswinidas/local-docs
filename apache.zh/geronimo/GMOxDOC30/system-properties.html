下表列出了Geronimo服务器支持的各种系统级属性。这些属性可以使用GERONIMO_OPTS环境变量或使用etc/system.properties配置文件。

要使用GERONIMO_OPTS环境变量设置给定的系统属性，请使用以下命令：

  • 在Windows系统上， 设置GERONIMO_OPTS = -D =
  • 在Linux / Unix系统上， 导出GERONIMO_OPTS = -D =

常规系统属性

选项类

值

org.apache.geronimo.deployment。LenientMFCP

真假

假

指定宽大处理与严格清单类路径的解释。将true指定为更宽松的处理，例如忽略缺少的jar和不符合规范的引用。

org.apache.geronimo.deployment.util。DeploymentUtil.jarUrlRewrite

真假

假

防止在部署期间锁定归档，而不是返回jar URL，而是将内容写入临时文件并返回该文件的URL。

org.apache.geronimo.gbean。无代理

真假

真正

指定是否使用快速代理访问GBean。如果不是，则使用反射。

org.apache.geronimo.kernel.config。马沙勒

 

org.apache.geronimo.kernel.config。序列化配置Marshaler

指定配置封送处理程序类。

org.apache.geronimo.kernel.config。MPCLSearchOption

安全，优化

安全

指定MultiParentClassLoader的classLoaderSearchMode。

指定JACC策略提供者的类名。必须是java.security的子类。政策。

org.apache.geronimo.config.file

 

 

指定服务器配置文件的位置。

org.apache.geronimo.config.substitutions.file

 

 

指定服务器配置替换文件的位置。

org.apache.geronimo.config.substitution.prefix

 

org.apache.geronimo.config.substitution

指定用于指定配置替换的前缀。用户可以指定一个（键，值）以覆盖在配置替换文件中设置的属性。例如，可以通过设置覆盖诸如hostName = localhost的条目-Dorg.apache.geronimo.config.substitution.hostName=foo当前缀是org.apache.geronimo.config.substitution 。

org.apache.geronimo.home.dir

org.apache.geronimo.server.dir

 

 

指定服务器实例的基本目录。

org.apache.geronimo.server.name

 

 

指定服务器实例的名称，用于在以下情况下将基本目录构造为服务器安装的子目录org.apache.geronimo.server.dir未指定。

org.apache.geronimo.repository.boot.path

 

 

指定服务器存储库的路径。

org.apache.geronimo.tomcat。GeronimoStandardContext.allowLinking

真假

假

该选项仅适用于Tomcat程序集。
如果此标志的值为true，则将在Web应用程序内部允许符号链接，指向Web应用程序基本路径之外的资源。
如果未指定，则标志的默认值为false。

org.apache.geronimo.webapplication.stage

开发，生产

生产

如果设置为development Web模块的内容将提取到磁盘上。这使工具能够对应用程序进行更快的更新。

org.apache.geronimo.equinox.useURLClassLoader

真假

假

如果为true，则模块类加载器将扩展URLClassLoader。

OSGi应用程序系统属性

选项类

值

默认

描述

ariesApplicationResolver.returnOptionalResources

真假

假

控制在OSGi应用程序安装期间是否应提供可选捆绑软件。

org.apache.geronimo.aries.failApplicationOnStartError

真假

假

如果为true，并且OSGi应用程序的捆绑软件之一无法启动，则将取消部署整个OSGi应用程序。如果为false，则即使发生捆绑包启动错误，OSGi应用程序也会保留。

org.apache.geronimo.aries.resolveApplicationOnDeploy

真假

真正

控制在安装过程中是否解析OSGi应用程序。如果为false，则每次启动OSGi应用程序都会对其进行解析。

org.apache.geronimo.osgi.web.extender.synchronous

真假

假

支持Web应用程序捆绑包（WAB）的同步部署。
当量与在同步-部署头MANIFEST.MF WAB的文件。
