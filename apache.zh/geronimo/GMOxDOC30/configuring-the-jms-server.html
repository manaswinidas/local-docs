要配置JMS服务器，请在左侧的“ 控制台导航”菜单上选择“ JMS服务器 ”。您将在服务器中获得JMS代理的列表，并在JMS服务器管理器 Portlet中获得它们的状态。您还将获得显示所有JMS网络连接器的JMS网络侦听器 portlet。

下图说明了这两个portlet和选项。

添加一个新的JMS代理

要添加新的JMS代理，请单击链接添加JMS代理 。在结果页面中，将为您提供ActiveMQ代理的模板。下图说明了模板：

它包含Geronimo的一些自定义配置。强烈建议您基于模板文件添加自己的配置，而不是从其他位置复制它。有关配置步骤，请参见以下示例：

  1. 输入新代理的名称，在此示例中为testborker 。
  2. 默认情况下，将占位符保留在配置文件中。Geronimo将在运行时将其替换为实际值。这是一个示例： $ {activemq.brokerName}
  3. 在大多数情况下，您只需要为新的borker添加新的连接器。取消注释连接器的元素，然后更换*在您的计算机中具有可用的端口号。确保保留占位符样式；否则，由于端口冲突，某些高级功能（例如群集）可能无法正常工作。 xml 添加连接器后，该部分应类似于以下示例： xml
  4. 单击Save ，您将返回到JMS Server Manager porlet。如果未发生任何错误，系统将提示您已成功创建新代理。JMS网络侦听器 portlet将在新的JMS代理中显示连接器。

在创建新代理后，将创建一些文件和文件夹：

  • 文件夹var/activemq/data/brokerName将创建以保存代理的数据。
  • 名为的文件brokerName.xml将在文件夹中创建var/activemq/conf来保存您的代理配置文件。

其中brokerName是您为新代理指定的名称。

为代理添加SSL传输连接器

Openwire是ActiveMQ的默认传输方式。在Geronimo中，可以通过使用SSL传输到客户端来连接到代理。使用以下步骤来启用SSL传输：

  1. 在代理的部署计划中启用SSL连接器部分。 XML文件
  2. 将您打算用于ssl验证的密钥库文件（例如test.ks）复制到/var/activemq/data/$brokerName/目录。
  3. 更新计划中的sslContext。 XML文件
