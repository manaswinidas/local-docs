配置基于HTTP标头的身份验证
编辑页面 浏览空间 添加页面 添加新闻

本章介绍使用CA服务器（例如Siteminder ）验证HTTP头传递的身份验证信息来实现单点登录的过程。

单一登录是一种为服务器提供访问控制的方法。它使您只能进行一次身份验证，并可以访问多个软件系统的资源。换句话说，对于使用Single Sign-on的同一安全领域，希望向服务器进行身份验证的用户代理可能只需要这样做一次。

在这种身份验证过程中，CA服务器将对附加到HTTP标头的信息进行交叉检查。如果该信息与安全领域定义的内容一致，则将验证要认证的客户端的身份。

使用基于HTTP标头的身份验证的应用程序必须按以下方式配置其部署描述符：

摘自部署描述符
<login-config>
      <auth-method>GENERIC</auth-method>
      <realm-name>TestPropsRealm</realm-name>
</login-config>

哪里

  • TestPropsRealm ： TestPropsRealm是用于认证的安全领域的名称。

使用Siteminder

著名的CA服务器是Siteminder ，它通过在HTTP请求上设置特定的标头来帮助向应用程序提供信息。默认情况下，它使用SM_USER标头传递用户名，该用户名稍后由GenericHttpHeaderLoginmodule WebSphere Application Server Community Edition中的类。

您可以为基于HTTP标头的身份验证配置安全领域。以下TestPropsRealm.xml文件是一个部署计划，用于在应用程序服务器上创建Properties文件安全领域 。使用此安全领域的应用程序可以实现单点登录。

摘录自TestPropsRealm.xml
<module xmlns="http://geronimo.apache.org/xml/ns/deployment-1.2">
    <environment>
        <moduleId>
            <groupId>console.realm</groupId>
            <artifactId>TestPropsRealm</artifactId>
            <version>1.0</version>
            <type>car</type>
        </moduleId>
        <dependencies>
            <dependency>
                <groupId>org.apache.geronimo.framework</groupId>
                <artifactId>j2ee-security</artifactId>
                <type>car</type>
            </dependency>
        </dependencies>
    </environment>
    <gbean name="TestSqlRealm" class="org.apache.geronimo.security.realm.GenericSecurityRealm" 
                 xsi:type="dep:gbeanType" xmlns:dep="http://geronimo.apache.org/xml/ns/deployment-1.2" 
                 xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
        <attribute name="realmName">TestPropsRealm</attribute>
        <reference name="ServerInfo">
            <name>ServerInfo</name>
        </reference>
        <xml-reference name="LoginModuleConfiguration">
            <log:login-config xmlns:log="http://geronimo.apache.org/xml/ns/loginconfig-2.0">
                <log:login-module control-flag="REQUIRED" wrap-principals="false">
                    <log:login-domain-name>TestPropsRealm</log:login-domain-name>
                    <log:login-module-class>org.apache.geronimo.security.realm.providers.GenericHttpHeaderPropertiesFileLoginModule</log:login-module-class>
                    <log:option name="groupsURI">var/security/demo_groups.properties</log:option>
					<log:option name="headerNames">SM_USER</log:option>
					<log:option name="authenticationAuthority">Siteminder</log:option>
                </log:login-module>
            </log:login-config>
        </xml-reference>
    </gbean>
</module>

哪里

  • GenericHttpHeaderPropertiesFileLoginModule ：是一个类，该类使Generic Http Header可以用于通过Properties文件安全领域进行身份验证。使用LDAP领域或数据库（SQL）领域时， GenericHttpHeaderLdapLoginModule和GenericHttpHeaderSqlLoginmodule分别使用。
  • var/security/demo_groups.properties ：表示demo_groups.properties文件是安全领域的属性文件。
  • SM_USER ： SM_USER是将用户名传递给服务器进行身份验证的标头名称。
  • Siteminder ：Siteminder是单点登录系统。

同样，您可以为LDAP领域或数据库（SQL）领域配置部署计划，以使用Siteminder进行单点登录。
