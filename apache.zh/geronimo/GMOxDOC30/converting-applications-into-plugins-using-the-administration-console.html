使用管理控制台将应用程序转换为插件
编辑页面 浏览空间 添加页面 添加新闻

本主题将带您使用Apache Geronimo管理控制台进行插件转换，导出和导入。我们使用了教程部分中的现有教程。有关应用程序开发和部署的信息，请参见Stateless Session Bean教程。然后，使用本教程中的插图从Apache Geronimo管理控制台导出和导入Java EE工件。

澄清度

本教程中提到的Application客户端实际上是一个Web Application客户端。

通过控制台将应用程序转换为插件

在本节中，我们将使用管理控制台直接将现有应用程序转换为插件。

  1. 启动管理控制台，然后在导航面板上的“ 应用程序 ”下选择“ 插件 ”。





  2. 在“ 创建Geronimo插件”下 ，从下拉菜单中选择jdbc / userds数据库池。





  3. 然后，点击导出插件 。





  4. 在下一页上，将所有值保留为默认值，然后单击“ 保存插件数据” 。





  5. 在下一个屏幕上，单击“完成”将资源池存档直接转换为插件。





  6. 同样，将ApplicationClient和StatelessSessionEJB转换为插件。


通过控制台导出和导入插件

本教程使用Apache Geronimo V2.2。

将应用程序导出为插件

  1. 启动管理控制台，然后在导航面板上的“ 应用程序 ”下选择“ 插件 ”。





  2. 在Create Geronimo Plugin下 ，从下拉菜单中选择jdbc / userds数据库池，如图所示。





  3. 然后单击“ 导出插件” 。





  4. 在下一页上，将所有值保留为默认值，然后单击“ 保存插件数据” 。





  5. 在下一个屏幕上，点击导出插件 。





  6. 弹出一个窗口，提示您保存插件。单击确定 。





  7. 同样，将ApplicationClient和StatelessSessionEJB导出为插件。

  6. 将存储库命名为myplugins ，并将完整名称命名为file：/ C：/ myplugins / *。点击*添加存储库 。





    这样就完成了Geronimo中存储库的添加。

导入插件

  1. 在开始导入导出的插件之前，我们应该从服务器上卸载EJB jar，Application Client WAR和数据库池。可以如后续附图中所示。











  2. 完成此操作后，从下拉列表中选择myplugins存储库。





  3. 单击显示所选存储库中的插件 。





  4. 现在，从前面的步骤中，我们看到Application Client将StatelessSessionEJB作为依赖项，而该依赖项又对数据库池具有依赖项。因此，如果我们导入Application Client插件，则应自动导入其他两个插件。选择Application Client插件，然后单击Install 。





  5. 下一个屏幕显示与我们的Application Client插件相关的各种工件。点击安装 。





  6. 完成此操作后，确认提示数据库池和无状态会话EJB已成功安装。





    完成此操作后，您可以重新测试应用程序功能的功能。
