在本节中，我们将重点介绍使用数据库来验证和检索用户名和密码。

在此示例中，我们使用内置的Derby数据库创建了一个名为SecurityDatabase的新数据库。以下步骤总结了创建数据库和表，加载一些样本数据以及创建连接池所执行的过程。配置数据库池部分中介绍了有关如何定义数据库连接池的详细说明。

创建数据库并加载样本数据

  • 在左侧的Console Navigation菜单中，单击DB Manager 。
  • 在创建数据库：字段中输入SecurityDatabase ，然后点击创建 。
  • 从“ 使用数据库：”下拉菜单中选择SecurityDatabase数据库，输入以下命令，然后单击“运行SQL” 。 创建表用户（用户名varchar（15），密码varchar（15））;创建表组（用户名varchar（15），组名varchar（15））;插入用户值（'userone'，'p1'）;插入用户值（'usertwo'，'p2'）;插入用户值（'userthree'，'p3'）;插入组值（“ userone”，“ admin”）；插入值组（'usertwo'，'admin'）;插入值组（'userthree'，'user'）;

创建连接池

  • 在左侧的Console Navigation菜单中，单击Database Pools 。
  • 单击使用Geronimo数据库池向导 。
  • 输入SecurityDatabasePool作为数据库池名称。
  • 从数据库池类型下拉菜单中选择Derby Embedded XA ，然后单击Next 。
  • 从驱动程序JAR滚动框中，选择org.apache.geronimo.configs / system-database / 2.1.1-SNAPSHOT / car 。
  • 将数据库用户名和密码留空 。
  • 输入SecurityDatabase作为数据库名称。
  • 点击部署 。

添加一个新的安全领域

要创建新的安全领域，请从“ 安全领域” Portlet中单击“ 添加新的安全领域 ” 。

在“ 安全领域的名称：”字段中输入derby_security_realm ，然后从“ 领域类型：”下拉菜单中选择“ 数据库（SQL）领域” ，然后单击“ 下一步” 。

以下屏幕配置登录模块。您需要填写的前两个字段可能因一种数据库类型而异。在这种情况下，我们使用嵌入式Derby数据库，因此User和Group select SQL的内容应如下所示：

用户SELECT SQL： select username, password from users where username=?
组SELECT SQL： select username, groupname from groups where username=?

输入用于检索用户和组的SQL语句后，需要从“ 数据库池”下拉菜单中选择在上一步中创建的数据库连接池。添加所需的值，如下所示，然后单击“ 下一步” 。

数据库池： SecurityDatabasePool

下一步将允许您启用审核以监视通过此领域的登录尝试。在此步骤中，您还可以基于在指定时间范围内失败的日志记录尝试次数来配置帐户锁定。如果启用“ 存储密码” ，则它将允许领域将用户的密码存储在“主题”中的私人证书中。如果启用了Naming Credential ，则除了用户密码外，此选项还将使用私有凭据来存储用户名。

至此，您已经配置了这个新的安全领域，下一步我将对其进行测试，然后进行部署。单击测试登录 。

输入要从数据库中检索到的有效用户名和密码，然后单击“ 下一步” 。

您应该收到一条登录成功的确认消息，单击“ 部署领域”以将该配置加载到服务器。

现在，您有了一个新的，完全配置的安全领域，该领域从Derby数据库中的构建中检索用户名和密码。

如果您在首次尝试验证此领域时遇到错误，则很可能会看到SQL Exception: Failed to start database ...终端和日志中的错误。这是Derby的一个已知问题，您将需要重新启动Geronimio，以便新数据库可以正常通信。
