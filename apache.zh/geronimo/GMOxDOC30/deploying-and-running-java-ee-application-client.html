部署和运行Java EE应用程序客户端
编辑页面 浏览空间 添加页面 添加新闻

您可以创建在Apache Geronimo应用程序客户端容器中运行而不是在本地Java环境中运行的应用程序客户端。Geronimo客户端应用程序容器的主要目的是向客户端程序提供一组标准的服务，而不必强制客户端应用程序指定依赖项列表（换句话说，易于开发）。客户端代码可以通过标准JNDI查找来查找Geronimo组件，例如资源，EJB和连接器。本示例创建了最简单的应用程序客户机，该客户机可以与Geronimo服务器进行通信并将消息放置在Geronimo控制台中。

  1. 创建一个包含这些文件的简单示例的客户端jar文件。客户端将在Geronimo控制台上写一条消息“ CLIENT RUNNING ...”： client/MainClient.java 
    package client;

public class MainClient {

    /**
     * @param args ignored
     */
    public static void main(String[] args){
        System.out.println ("CLIENT RUNNING...");  
    }
}
  2. 创建一个提供客户端显示名称的应用程序客户端部署描述符： META-INF/application-client.xml 
    <?xml version="1.0"?>
<application-client xmlns="http://java.sun.com/xml/ns/j2ee" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
 xsi:schemaLocation="http://java.sun.com/xml/ns/j2ee [http://java.sun.com/xml/ns/j2ee/applicationclient_1_4.xsd]" version="1.4">
    <display-name>EXAMPLE Client</display-name>
</application-client>
  3. 创建一个描述部署模块Id和类型的Geronimo应用程序客户端部署计划： META-INF/geronimo-application-client.xml您需要定义两个模块，一个用于客户端，一个用于服务器端。EXAMPLEClientServer是始终在服务器上运行的组件， EXAMPLEClientClient是要从以下位置在客户端运行的组件client.jar 。 
    <?xml version="1.0"?>
<application-client xmlns="http://geronimo.apache.org/xml/ns/j2ee/application-client-2.0" xmlns:dep="http://geronimo.apache.org/xml/ns/deployment-1.2">
    <dep:client-environment>
        <dep:moduleId>
            <dep:groupId>JEE5</dep:groupId>
            <dep:artifactId>EXAMPLEClient</dep:artifactId>
            <dep:version>2.1</dep:version>
            <dep:type>car</dep:type>
        </dep:moduleId>
    </dep:client-environment>

    <dep:server-environment>
        <dep:moduleId>
            <dep:groupId>JEE5</dep:groupId>
	    <dep:artifactId>EXAMPLEClientServer</dep:artifactId>
	    <dep:version>2.1</dep:version>
	    <dep:type>car</dep:type>
        </dep:moduleId>
    </dep:server-environment>
    
</application-client>
  4. 创建一个清单，该清单告诉Java jar util首先运行哪个类。META-INF/MANIFEST.MF 
    Manifest-Version: 1.0
Main-Class: client.MainClient
\# blank line
    MANIFEST文件末尾必须包含Main-Class和空白行。
  5. 创建一个打包客户端jar的bat文件，取消部署客户端/服务器应用程序的任何较旧版本，然后部署客户端应用程序。此bat文件创建J2EE应用程序客户端jar文件（此处名为app_client.jar 。）和应用程序bin文件夹中的关联文件（在这种情况下， D:\workspace2\GERONIMO_APPLICATION\bin ）。运行java client/MainClient.java创建一个类文件。jar命令-M参数表明该jar文件必须使用MANIFEST.MF文件。创建jar文件后，验证这是否正确。
jar cfM app_client.jar -C D:\workspace2\GERONIMO_APPLICATION\bin .
java -jar %GERONIMO_HOME%/bin/deployer.jar --user system --password manager --host localhost undeploy JEE5/EXAMPLEClient/2.1/car
java -jar %GERONIMO_HOME%/bin/deployer.jar --user system --password manager --host localhost undeploy JEE5/EXAMPLEClientServer/2.1/car
java -jar %GERONIMO_HOME%/bin/deployer.jar --user system --password manager deploy app_client.jar
pause

部署之后，您将在Geronimo Web Console中看到两个组件： Applications- > App Clients ：

JEE5/EXAMPLEClient/2.1/car    	   state: stopped
JEE5/EXAMPLEClientServer/2.1/car   state: running

您可以使用以下命令运行应用程序客户端：

%GERONIMO_HOME%/bin/client JEE5/EXAMPLEClient/2.1/car

您应该在Geronimo控制台中注意到应用程序客户机消息。
