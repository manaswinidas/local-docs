部署：部署模块

该命令用于将模块部署到服务器。部署后，模块将通过服务器中的模块ID进行标识。它具有以下语法：

部署模块 generaloptions commandoptions

哪里

module是命令所必需的模块文件，并且可以是以下Java应用程序之一：

  • Java EE企业应用程序归档文件（EAR）文件
  • Java EE Web应用程序存档（WAR）文件
  • Java EE Enterprise JavaBean归档（JAR）文件
  • Java EE Java资源存档（RAR）文件
  • Web Archive Bundle（WAB）文件
  • 企业捆绑包（EBA）文件

部署计划是Java EE模块的可选计划文件。如果Java EE归档文件的部署计划不在WEB-INF目录中，则必须在命令中的模块之后指定其位置。有关更多详细信息，请参见创建部署计划 。
命令选项包括：

- 安全

使用安全通道

-i或--inPlace

将存档或目录添加并启动到活动服务器，而无需将存档或目录复制到服务器目录树中

-t或--targets

指定要部署到的目标存储库。请参阅列表目标 ，以获取服务器上可用存储库的名称。

部署模块命令示例

使用此语法来部署WAR文件。

部署模块d：/HelloWorld.war

使用此语法来部署WAR文件，该文件的部署计划位于WEB-INF目录的外部。

服务器> deploy：deploy-module d：/HelloWorld.war d：/geronimo-web.xml

回到顶部

重新部署模块 generaloptions commandoptions

哪里

module是命令所必需的模块文件，并且可以是以下Java应用程序之一：

  • Java EE企业应用程序归档文件（EAR）文件
  • Java EE Web应用程序存档（WAR）文件
  • Java EE Enterprise JavaBean归档（JAR）文件
  • Java EE Java资源存档（RAR）文件

部署：安装捆绑

该命令用于将OSGi捆绑软件或通用jar库文件安装到服务器中（jar文件将转换为OSGi捆绑软件），并且捆绑信息将记录在etc/startup.properties文件。它具有以下语法：

安装束 generaloptions commandoptions

哪里

命令选项包括：

--groupId

定义分发包的组ID。

--startLevel

指定捆绑包的开始级别。

- 开始

指定安装后是否启动捆绑软件。默认情况下，捆绑包未启动。

捆绑软件文件是要安装的捆绑软件的标准名称。

install-bundle命令示例

使用此语法来安装OSGi捆绑软件。如果文件是jar库文件，则服务器会将库文件转换为OSGi捆绑包。

服务器> deploy：install-bundle --groupId myCompany --start --startLevel 100 d：/HelloWorld-1.0.jar
