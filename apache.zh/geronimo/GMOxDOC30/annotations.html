您可以通过部署计划来配置在Geronimo中安装的每个模块，无论是服务，应用程序，资源等。这些部署计划是基于XML架构的XML文件，其中包含特定应用程序模块或组件的配置详细信息。Java EE规范定义了标准部署描述符，例如web.xml，application.xml等。INLINE

Java EE 6规范还允许直接在Java代码中使用注释，以减轻与创建Java EE 5应用程序相关的开发负担。

Java EE规范中的注释是一种新的元数据功能，其中许多Java实体（例如类，字段，包，方法等）都可以使用注释来修饰。这些声明可以减少并且在许多情况下完全消除使用部署计划的必要性。如果同时提供了部署计划中的注释和声明，则部署plna中的声明将优先。这允许更改Java EE 6模块的部署特性，而不必更改代码。

Geronimo作为Java EE 6认证的应用程序服务器，支持Java EE 6规范中的所有注释。下面列出并描述了这些注释以及相应的Java EE 6规范，以及它们可以替换的部署计划XML元素（如果适用）：

这是Geronimo支持的所有Java EE 6注释的列表：
