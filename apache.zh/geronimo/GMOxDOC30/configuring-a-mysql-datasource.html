配置MySQL数据源
编辑页面 浏览空间 添加页面 添加新闻

本文向您展示了如何在Geronimo中配置mysql数据源。Geronimo嵌入了同时支持本地和XA事务的mysql资源适配器。您需要按照说明安装mysql JDBC驱动程序。

有两种方法可以配置mysql数据源：

  • ＃创建数据源并将其部署为独立的JEE连接器模块
  • ＃将数据源定义嵌入到应用程序部署计划中

创建数据源并将其部署为独立的JEE连接器模块

  1. 请参阅附件sample-mysql-xa-ds-plan-standalone.xml以创建独立的Java EE连接器部署计划。
  2. 使用管理控制台将mysql jdbc驱动程序安装到Geronimo存储库，并确保其模块ID与示例计划中定义的模块ID相同。在此示例中，mysql JDBC驱动程序的模块ID为mysql/mysql-connector-java/5.1.7/jar 。
  3. 使用命令行部署数据源。这是用于部署mysql xa数据源的示例命令：
    <geronimo_home>/bin/deploy.sh -u system -p manager deploy 
<GERONIMO_HOME>/repository/org/tranql/tranql-connector-mysql-xa/1.2/tranql-connector-mysql-xa-1.2.rar sample-mysql-xa-ds-plan-standalone.xml
    哪里是Geronimo的安装目录。
  4. 现在，您可以使用其jndi名称为java：comp / env / jdbc / mysqlxads的数据源。

将数据源定义嵌入到应用程序部署计划中

  1. 请参阅附件sample-mysql-xa-ds-plan-embeded.xml以创建应用程序部署计划，并根据您的应用程序替换Web模块和ejb模块。
  2. 使用管理控制台将mysql jdbc驱动程序安装到Geronimo存储库，并确保其模块ID与示例计划中定义的模块ID相同。在此示例中，mysql JDBC驱动程序的模块ID为mysql/mysql-connector-java/5.1.7/jar 。
  3. 使用命令行或管理控制台通过部署计划来部署应用程序。例如，使用以下命令行：
    <geronimo_home>/bin/deploy.sh -u system -p manager deploy yourapp.ear sample-mysql-xa-ds-plan-embeded.xml
    哪里是Geronimo的安装目录。
  4. 现在，您可以使用其jndi名称为java：comp / env / jdbc / mysqlxads的数据源。

事务配置元素如下所示：

<connectionmanager>
  <xa-transaction>
     <transaction-caching/>
  </xa-transaction>
  ...
</connectionmanager>

对于本地交易：

<connectionmanager>
   <local-transaction/>
   ...
</connectionmanager>
