部署期间如何执行SQL语句

排队

本文是关于如何在例如数据源的部署期间执行一些清理或初始化SQL语句的。

以下是如何完成此操作的示例。

以下是此示例中使用的依赖项：

  • my-sql.jar包含要执行的.sql文件（与MySQL无关，它只是一个包含.sql文件的jar文件，仅此而已）。

怎么运行的

我们必须在连接器部署描述符中指定一个GBean。该gbean将在部署期间执行。但是它需要以下信息：

  1. 要在其上执行的数据源的名称
  2. .sql文件的路径
  3. 一个将作为布尔值的测试SQL查询。如果查询返回零条记录，则将执行sql语句（例如，您可能要创建一些表）
xml $ {pom.groupId} $ {pom.artifactId} $ {version} 汽车 org.apache.geronimo.configs 系统数据库 汽车 org.apache.geronimo.sample my-sql $ {monitoringConsoleVersion} jar javax.sql。数据源 jdbc / myDS true MyDB 10 0 SELECT t.tablename FROM SYS.SYSTABLES吨，其中较低的（t.tablename）= 'myTable的' META-INF /数据库/德比/ createTables.sql JDBC / MyDS
