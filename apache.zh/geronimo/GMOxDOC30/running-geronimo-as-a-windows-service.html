本文将向您展示如何配置Apache Geronimo服务器以作为Microsoft Windows服务运行。当前，有两个选项可将服务器作为Windows服务运行：

1个

Apache Commons守护程序

软件先决条件

Microsoft .NET Framework 2.0+版。例如，可以从以下URL下载版本2.0：
http://www.microsoft.com/downloads/details.aspx?familyid=0856eacb-4362-4b0d-8edd-aab15c5e04f5&displaylang=en

安装Apache Commons守护程序

要使用Apache Commons Daemon将Apache Geronimo服务器设置为作为Microsoft Windows服务运行，您需要安装procrun插件。使用管理控制台或部署程序工具安装插件：插件的名称为“ Windows Service Wrapper ”。

使用Apache Commons守护程序

安装插件后，您可以通过运行以下命令将Apache Geronimo服务器安装为Windows服务：

\bin\service_pr.bat install

例如：

固定 D：\ geronimo-tomcat7-javaee6-3.0 \ bin> service_pr.bat安装已安装服务“ geronimosrv”。


这将使用默认参数为名称为geronimosrv的服务器创建Geronimo服务。您可以使用Windows'启动/停止Geronimo服务。 Services工具或通过使用\bin\geronimosrvw.exe工具。

默认情况下，Geronimo服务配置为“ 手动 ”启动类型。使用Windows' Services工具或geronimosrvw.exe命令将其配置为“自动”（如果需要）。

默认的管理员帐户“ system”和密码“ manager”用于关闭服务。如果更改了默认管理员帐户，请使用geronimosrvw.exe工具（在“关闭”选项卡中）更改此设置。

service_pr.bat脚本不会读取JAVA_OPTS和GERONIMO_OPTS环境变量。如果需要添加用于启动/停止服务的任何选项，则可以使用geronimosrvw.exe工具将它们添加到“启动”和/或“关闭”选项卡中的命令行参数中。您也可以使用geronimosrv.exe命令执行此操作。

有关如何使用geronimosrv.exe和geronimosrvw.exe命令的更多帮助，请参阅Apache Commons Daemon手册 。

要删除Apache Geronimo服务，请运行以下命令：

\bin\service_pr.bat remove

例如：

固定 D：\ geronimo-tomcat7-javaee6-3.0 \ bin> service_pr.bat remove已删除服务'geronimosrv'。

Java服务包装器

将需要Java Service Wrapper作为实现此配置的附加软件。使用服务包装器可以使您对正在配置的服务进行其他“控制”，例如ping服务并在服务关闭时采取措施。

软件先决条件

要将Apache Geronimo服务器设置为作为Microsoft Windows服务运行，您肯定需要一个Java Service Wrapper。在此示例中，我们使用了Java Service Wrapper 3.2.3，可以从以下URL下载：

http://sourceforge.net/projects/wrapper/

对于此示例，我们还使用Windows XP上运行的Apache Geronimo v3.0。您可以从以下URL下载Apache Geronimo二进制文件：

http://geronimo.apache.org/downloads.html

配置包装器

当我们尝试将所有包装器文件保留在Geronimo的目录结构中时，请按照以下示例中的说明创建配置文件，并将其放置在/ var / config目录中。

该服务包装程序允许三种不同的集成方法，这取决于是否使用相同的类来启动和停止服务器。

对于Geronimo，我们将使用最简单的集成方法，因为调用System.exit（）时服务器可以干净地关闭。对于此集成，包装器的主类设置为WrapperSimpleApp和org.apache.geronimo.cli.daemon.DaemonCLI class指定为启动服务器的主要类。目前，没有传递任何额外的参数来启动服务器，但是可以通过添加其他参数来指定它们wrapper.app.parameter价值观。包装器将通过调用System.exit（）来停止服务器。

如前所述，在本示例中，我们尝试在Geronimo目录结构中定位所有与包装器相关的文件。从上面显示的wrapper.conf中可以看到，大多数目录引用都是相对于Geronimo的安装目录而言的。您只需要更换和使用适合您环境的设置。

接下来，下载g_service.bat批处理文件并将其放在/ bin目录中。该批处理文件从命令行运行该服务，而无需实际安装该服务。在测试和调试配置时，这特别有用。

现在，您应该可以从命令行将Geronimo作为服务运行。只需运行以下命令：

/bin/g_service.bat

下一步是实际安装该服务，以便可以在Windows启动时将其拾取并与其他现有服务一起进行管理。

安装/删除服务

提供了以下两个批处理文件来安装和删除Geronimo作为Windows服务。下载Install_Geronimo_NT.bat和Uninstall_Geronimo_NT.bat并将其放置在/bin目录。请注意，可能需要对批处理文件进行一些修改。

要将Apache Geronimo作为MS Windows服务安装，请运行以下命令：

/bin/Install_Geronimo_NT.bat

您将获得一个类似于此示例的确认屏幕。

固定 D：\ geronimo-tomcat7-javaee6-3.0 \ bin> Install_Geronimo_NT.bat包装器|已安装Apache Geronimo v3.0 Server。

要删除Apache Geronimo服务，请运行以下命令：

/bin/Uninstall_Geronimo_NT.bat

您将获得一个类似于此示例的确认屏幕。

固定 D：\ geronimo-tomcat7-javaee6-3.0 \ bin> Uninstall_Geronimo_NT.bat包装器|删除了Apache Geronimo v3.0 Server。
