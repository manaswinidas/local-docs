应用概述

此样本演示了Servlet 3.0规范中的模块化和可插入性功能，该规范也称为Web片段。Web片段就像Web应用程序的一部分，它在其部署描述符中几乎消耗与Web应用程序所有相同的元素，但是，Web片段的部署描述符必须命名为web-fragment.xml放进去META-INF/片段的目录。并且顶层元素必须是 。您可以更新Web片段，而不会对Web应用程序产生任何影响。

在Web应用程序中，所有配置文件（例如web.xml和web-fragment.xml已加载并扫描。您可以指定Web应用程序使用的订购要求要么元件。该示例仅显示了元件。

申请内容

该示例应用程序由以下软件包和类列表组成。

org.apache.geronimo.samples.javaee6.webfragment.fragment1是Web应用程序的第一个片段

  • FilterFragment1.java是fragment1中对/ messageRecord的请求的筛选器类。
  • QueryAll.java显示您可以购买的所有项目。
  • META-INF / web-fragment.xml是fragment1的部署描述符。

org.apache.geronimo.samples.javaee6.webfragment.fragment2是Web应用程序的第二个片段

  • FilterFragment2.java是fragment2中对/ messageRecord的请求的筛选器类。
  • BuyRecordItem.java执行实际的购买操作。
  • ShoppingCart.java将您购买的商品放入购物车。
  • META-INF / web-fragment.xml是fragment2的部署描述符。

org.apache.geronimo.samples.javaee6.webfragment.fragment3是Web应用程序的第一个片段

  • FilterFragment3.java是fragment3中对/ messageRecord的请求的筛选器类。
  • Payment.java计算您应该支付的金额。
  • ServletListener.java是fragment3中对/ Payment servlet的所有请求的侦听器类。
  • META-INF / web-fragment.xml是fragment3的部署描述符。

org.apache.geronimo.samples.javaee6.webfragment是Web应用程序的主体

  • WelcomePage.java显示欢迎页面和示例应用程序的简要介绍。
  • messageRecord.java在购买过程中显示所有筛选器消息和侦听器消息。

应用实施

geronimo-web.xml指定该模块的信息以及该Web应用程序的URL

有关项目的信息（例如模块的唯一标识，依赖项）在内部进行了描述标签。最好给模块一个唯一的标识，以便以后可以由其他一些可部署的应用程序引用它。该模块在org.apache.geronimo.samples组中。在指定的路径标签将是此Web应用程序的入口。因此，您可以从http：// ： / webfragment-javaee6访问此Web应用程序。

xml geronimo-web.xml org.apache.geronimo.samples webfragment-javaee6 $ {version} 汽车 / webfragment-javaee6

web.xml定义了Webfragment应用程序的欢迎页面以及由片段组成的资源的顺序。使用意味着必须首先处理web.xml，必须顺序处理fragment3和fragment2。 元素可确保片段1之前的其他Web片段正在处理。而fragment1是要处理的最后一个。

xml web.xml ... 的index.html WelcomePage WelcomePage 1 WelcomePage / WelcomePage / * 30 fragment3 fragment2 片段1

WelcomePage.java是一个Servlet组件，它将请求发送到fragment1中的另一个Servlet QueryAll 。

web-fragment.xml使用与web.xml中相同的元素，除了顶级元素必须是 。而metadata-complete =“ true”表示将不处理此片段内的类文件中的任何注释。 fragment1必须与webfragment应用程序的web.xml中的一致。

构建Web应用程序

一旦所有源都签出，下一步就是构建webfragment-javaee6 。它需要Maven 2来构建二进制文件。

从< webfragment-javaee6_home>目录运行以下命令。

mvn全新安装

此过程将花费几分钟。二进制文件将在相应的目标目录中生成。

部署Web应用程序

部署示例应用程序非常简单，因为我们将使用Geronimo Console。

  1. 从“控制台导航”面板向下滚动到Deploy New 。
  2. 将webfragment-javaee6-war / target文件夹中的webfragment-javaee6-war-3.0-SNAPSHOT.war 加载到“ 归档”输入框中。
  3. 按安装按钮以在服务器中部署应用程序。

测试Web应用程序

该应用程序位于http：// localhost：8080 / webfragment-javaee6 /

输入您要购买的ID和数量，然后单击“ 添加到购物车”

单击转到付款以查看应支付的费用以及过滤器或侦听器类记录的所有消息。
