本文向您展示了如何在开发过程中使用Geronimo Eclipse插件（GEP）创建安全领域。

GEP为各种类型的安全领域配置（例如属性文件领域，SQL领域和LDAP领域）提供了易于使用的向导。

在本文中，我们使用示例Dynamic Web Project和SQL领域进行演示。当然，您可以根据需要选择项目或领域类型。

创建一个SQL领域

  1. 双击geronimo-web.xml在您的应用程序/ META-INF目录下的文件中，在Geronimo Deployment Plan Editor中将其打开，您将看到该应用程序的常规信息页面。
  2. 选择“ 安全性”选项卡，然后打开“ 安全领域”部分以显示已定义的安全领域。
    • 添加 -启动用于创建安全领域的向导。
    • 删除 -删除现有的安全领域。
    • 编辑 -重新配置现有的安全领域。
  3. 单击添加以启动安全领域向导，并填写必填字段。在此示例中，我们选择SQL Realm 。
    • 领域名称用于指定将用于应用程序的用户身份验证的安全领域的名称。
    • 领域类型是登录模块的类型。
  4. 单击“ 下一步”输入用于用户和组验证的SQL语句。
    • 选择SQL ：输入SQL语句以从数据库中检索用户和组信息。
    • 摘要配置 ：指定用户密码的摘要算法和编码配置。
  5. 输入数据库池名称或JDBC URL以连接到存储凭据的数据库。
  6. 单击“ 完成”以完成向导，您将看到“ 安全领域”部分中列出了该领域和相应的LoginModule类。
  7. 在工具栏中，单击“ 完成”以保存更改。

现在，您已经为简单的Web应用程序创建了一个新的安全领域。您可以单击“ 源”选项卡以查看示例部署计划。
