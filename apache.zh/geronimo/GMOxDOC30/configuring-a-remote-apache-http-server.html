配置远程Apache HTTP服务器
编辑页面 浏览空间 添加页面 新增新闻

在撰写本文时，Geronimo管理控制台中提供的Apache mod_jk配置 portlet在识别已经安装的Web应用程序时出现了一些不一致之处。作为一种解决方法，本文介绍了两种配置远程Apache HTTP服务器的替代方法。

Apache Geronimo有两种选择，具体取决于您决定使用的Web容器。您可以在Tomcat或Jetty发行版之间进行选择。如果要将Apache HTTP服务器（HTTPd）放在Geronimo的前面，则必须对其进行配置，使其可以将客户端请求转发到Geronimo。

正如我们已经提到的，本文介绍了两种如何配置Apache HTTPd来将客户端请求转发到Geronimo的方法，Geronimo通过使用“内置” mod_proxy模块或配置Jakarta Tomcat Connector mod_jk将其配置为反向代理。与Apache Tomcat源一起提供。

  • 在Apache HTTPd中配置反向代理时，该配置将适用于Geronimo的任何发行版（Tomcat或Jetty），而无需在Geronimo端进行任何其他配置。
  • 在配置Jakarta Tomcat连接器时，将需要其他配置。将根据需要解决许多新术语，但是请记住，本文的范围是使用远程HTTPd配置两层方案。

基于这两种选择，本文分为以下几节：

  • 将Apache HTTPd配置为反向代理（mod_proxy）
  • 使用Jakarta Tomcat Connector（mod_jk）配置Apache HTTPd

为了为该配置示例树立共同的基础，以下是经过测试的版本的列表：

  • Apache Geronimo v1.2
  • Apache HTTPd v2.2.3
  • mod_jk-apache-2.2.3.so
  • Windows XP SP2
