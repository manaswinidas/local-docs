JavaServer Faces的基础
编辑页面 浏览空间 添加页面 添加新闻

本文将带您了解Java Server Faces的基础知识。稍后，我们将提供示例，这些示例将详细说明Java Server Faces的特性和功能。

Java Server Faces（以下称为JSF）是Java EE 5领域中基于事件的开发框架。它是用于为Web应用程序构建用户界面的框架。
JSF使用现有的用户界面和Web层机制，但没有将应用程序绑定到特定的标记语言，协议和客户端设备。它封装了各种UI组件功能，使其能够通过开发所需的自定义标签将数据呈现给各种客户端设备。JavaServer Faces技术提供了用于渲染到HTML客户端的自定义渲染器和JSP自定义标签库，从而允许Java EE应用程序的开发人员在其应用程序中使用JavaServer Faces技术。

JSF遵循MVC架构，遵循MVC架构的Web应用程序最好使用JSF开发。
让我们简要地了解什么是MVC架构。任何企业应用程序都具有业务组件，演示组件和控制器组件。通常，小型应用程序会将所有组件紧密地耦合在一起。对一个组件的任何更改都会影响其他组件。随着应用程序大小的增加，调试和修改应用程序变得更加困难和复杂。为了解决这个问题，我们需要改进应用程序的体系结构设计。

Model（M），View（V）和Controller（C）是改进应用程序设计的第一步。

  • 模型 -处理应用程序的数据和逻辑。它处理应用程序的业务组件。
  • 视图 -负责用户界面。它处理必须显示给用户的应用程序输出。它处理表示组件。
  • 控制器 -处理来自用户的输入，并根据用户输入修改模型或视图。控制器根据用户输入和模型操作的结果选择必须显示的下一个视图。

通过遵循MVC架构，可以在模型，视图和架构之间进行功能划分。该模型具有以下优点：

  • 有助于减少代码中的重复
  • 现有代码的维护变得更加容易
  • 提供一个中央位置来控制所有用户视图-控制器
  • 可以添加任何新的用户视图，然后将其连接到控制器
  • 调试应用程序变得非常容易
  • 更改模型对View和Controller的影响最小

现在回到JSF，JSF技术清楚地将表示逻辑与应用程序逻辑分开。在应用程序开发状态期间，可以创建和指定几个开发角色。JSF技术是通过JSR-127下的Java Community Process开发的。

JSF具有以下作为主要组件：

  1. 用户界面组件-这些是基于JavaBeans构建的有状态对象。每个UI组件都有关联的事件，属性和方法。这些组件位于服务器端而不是客户端（例如swing组件）。在JSF中，为浏览器呈现了UI组件。对于任何用户请求组件，输出HTML，然后将HTML呈现并作为GUI元素显示给用户。
  2. 导航-JSF定义了非常有效的导航系统。对于给定的页面，导航规则定义什么是有效结果以及为每个输出加载哪个页面。
  3. 渲染器-此组件用于渲染UI组件。该组件可以根据不同类型的浏览器（例如，Netscape，Internet Explorer，Mozilla，Firefox）提供不同的呈现选项。渲染器将用户输入转换为组件可以理解的值，反之亦然。
  4. 转换器-转换器将数据从一种类型转换为另一种类型。在JSF中进行转换的用户将输入转换为组件，反之亦然。
  5. 验证程序-验证程序验证用户输入并验证输入是否为可接受的格式。在常规的Web开发中，验证是通过脚本语言（如Java Script）来进行的，这给应用程序带来了相当大的开销。JSF提供了控制器机制来验证输入。验证是在UI组件级别以及Bean级别完成的。
  6. JavaBeans-在JSF中，Java Beans定义UI组件和模型之间的交互。它们在JSF中扮演控制器的角色。我们可以说Backing Bean控制着表单以及与表单相关联的组件。
  7. 国际化和本地化-JSF使用Java Server页面标准标记库（JSTL）中的国际化标记。

这就完成了给出有关JSF架构设计和Java Server Faces可用的各种组件的基本思想的内容。

JSF技术提供

  • 用于表示用户界面组件和管理其状态，处理在其上触发的事件，验证用户输入的输入，定义页面导航，支持国际化和可访问性的API。
  • 一个定制标记库，用于在JSP中开发JSF接口。
