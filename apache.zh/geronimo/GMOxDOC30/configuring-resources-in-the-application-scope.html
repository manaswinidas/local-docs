在本节中，我们将讨论安装和启动服务器时已在服务器中部署并运行的配置。

连接池

Apache Geronimo附带了嵌入式Derby数据库和ActiveMQ消息代理。还有连接到Derby和activeMQ的连接池，这些连接池配置为在已安装的服务器中运行。以下各节讨论已安装的服务器中已在运行的各种此类配置。

嵌入式Derby数据库连接池

Apache Geronimo附带了嵌入式Derby数据库。Derby库位于服务器存储库中，网址为/repository/org/apache/derby 。默认情况下，Derby数据库按名称SystemDatabase创建并与数据库相关的文件存储在/var/derby/SystemDatabase 。除此之外，默认情况下，服务器会在SystemDatabase配置名称org.apache.geronimo.configs/system-database/2.1/car 。数据库连接池的名称是SystemDatasource 。配置工件存储在/repository/org/apache/geronimo/configs/system-database 。用于数据库连接池的部署计划如下。

启动服务器后，正在运行的数据库连接池SystemDatasource可以在管理控制台上从console Navigation => Services => Database pools 。用于部署上述数据库连接池的资源适配器是tranql-connector-derby-embed-xa-1.3.rar 。上面的计划实际上是出站资源适配器的部署计划。如果上述计划与rar文件，则将xml内容放入META-INF/geronimo-ra.xml档案。

嵌入式ActiveMQ资源适配器

默认情况下，连接到嵌入式activemq消息代理的JMS资源适配器已部署并在apache geronimo服务器中运行。这是一个出站jms资源适配器，用于配置连接工厂和两个消息队列。资源适配器的配置名称为org.apache.geronimo.configs/activemq-ra/2.1/car 。资源适配器的工件存储在/repository/org/apache/geronimo/configs/activemq-ra 。部署计划如下。

用于部署上述计划的资源适配器是/repository/org/apache/geronimo/modules/geronimo-activemq-ra/2.1 。启动服务器后，可以在管理控制台上从以下位置查找正在运行的资源适配器： Console Navigation => Services => JMS Resource 。我们还可以在管理控制台上观察资源适配器部署的连接工厂和队列。

安全

Java EE应用程序可能包含几个组件，这些组件可以部署到JEE5服务器中的不同容器中，例如WEB容器，EJB容器，WebServices容器。这种部署允许相互交互的多层应用程序执行给定的用户任务。可以通过适当选择身份验证机制并设计授权级别或角色来保护多层JEE5应用程序。如果应用程序组件使用声明性安全性管理，则在相应的JEE5部署描述符中声明身份验证和授权方面。声明的安全角色或级别通过安全领域映射到geronimo部署计划中的实际安全角色或级别。在Apache Geronimo中，安全领域抽象了应用程序组件的身份验证和授权方面。身份验证和授权共同支持对应用程序各个组件的访问控制。

根据所选的认证系统，将在安全领域中选择并配置JAAS登录模块。JAAS登录模块连接到相应的用户/组存储库，并执行认证和检索授权信息。Geronimo服务器提供登录模块，这些模块连接到不同类型的用户/组存储库。这些是PropertiesFileLoginModule ， LDAPLoginModule ， SQLLoginModule和CertificatePropertiesFileLoginModule 。

例如，当用户登录geronimo管理控制台时，Geronimo使用geronimo-admin安全领域对用户进行身份验证。安全领域的部署计划如下。

上面的XML文档的默认名称空间是http://geronimo.apache.org/xml/ns/deployment-1.2 。没有名称空间前缀的XML元素属于默认名称空间。

上面的安全领域部署在两个属性文件上/var/security/users.properties和var/security/groups.properties包含使用以下信息的用户/组信息org.apache.geronimo.security.realm.providers.PropertiesFileLoginModule 。Geronimo管理控制台是一个使用上述安全领域进行用户身份验证的Web应用程序。

安全领域部署计划是一个XML文件，该文件将http://geronimo.apache.org/xml/ns/deployment-1.2模式用于moduleId ，依赖项和安全领域GBean配置。XML文件使用http://geronimo.apache.org/xml/ns/loginconfig-2.0模式进行登录模块配置。所有XML模式文件(.xsd)位于/schema目录。
