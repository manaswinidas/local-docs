在Apache Geronimo中使用Eclipse XML工具

本教程将演示如何使用Eclipse XML工具来生成Geronimo特定的部署计划。

我们确实鼓励用户利用GEP进行架构验证，因为GEP已将这些架构文件集成到其分发包中。本文中描述的步骤仅供您参考。
在Eclipse v3.5（Galileo）中使用Geronimo Deployment Plan Editor时，您可能会遇到某些XML验证问题，这是WTP中Xerces功能的已知缺陷 。请尝试在Eclipse v3.5中使用以下变通办法来避免该问题：

  1. 在工具栏中，选择Windows- > 首选项
  2. 展开XML部分，然后选择XML文件 -> 验证
  3. 取消选中“ 遵守所有XML模式位置”选项，然后保存所有更改。

要运行本教程，至少需要您安装以下必备软件。

  • Sun JDK 5.0+（J2SE 1.5）
  • Eclipse 3.4.1（Eclipse Ganymede SR1发行版）
  • Apache Geronimo v2.2

开发环境部分中提供了有关安装Eclipse的详细信息。本教程分为以下几节：

为geronimo模式配置Eclipse

  1. 可以在/ schema中找到Geronimo模式文件。将所有这些模式文件复制到一个名为geronimoschema的文件夹中。这些文件的内容可能因发行版本而有所不同。
  2. 将以下XML目录文件添加到此文件夹并压缩。这会给你一个geronimoschema.zip文件。 geronimoXmlCatalog.xml目录
  3. 启动Eclipse并选择New-> Project ，如图所示。





  4. 在下一个屏幕上，选择General-> Project 。选择下一步。





  5. 将项目命名为geronimoschema并选择Finish 。





  6. 在“项目资源管理器”下，右键单击geronimoschema项目，然后选择“ 导入” 。然后选择General-> Archive File 。选择下一步 。





  7. 在下一个屏幕上浏览到geronimoschema.zip在步骤2中创建。完成后，选择完成 。





  8. 在geronimoschema项目下，您可以看到所有模式定义。





  9. 接下来，从Eclipse菜单中选择Window-> Preferences 。





  10. 在下一个窗口中，选择XML- > XML Catalog 。选择用户指定的条目 ，然后单击添加 。





  11. 在下一个窗口中，单击“ 位置”旁边的图标，然后选择“ 工作区” 。


  12. 选择GeronimoSchema.xmlcatalog 。选择确定 。





  13. 指定密钥，例如geronimoschema 。





  14. 在下一个屏幕上，您可以看到列出的Geronimoschema 。


在应用程序开发中使用配置

  1. 对于下一步，我们将考虑无状态会话Bean项目。
  2. 考虑以上教程中的“ 很少有更多配置”部分，您必须在其中为数据库池添加依赖项元素。
  3. 在StatelessSessionEJB中，选择ejbModule-> META-INF-> openejb-jar.xml 。右键单击并选择打开方式-> XML编辑器 。





  4. 获得openejb-jar.xml后，选择“ 设计”选项卡。





  5. 您可以通过双击每个字段来编辑字段。


  6. 完成后，打开“源”选项卡，您可以看到已将依赖项元素添加到openejb-jar.xml。





这就完成了将Eclipse xml工具用于Apache Geronimo的说明。
