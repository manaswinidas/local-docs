什么是REST？

REST的首字母缩写是Representational State Transfer ，它基本上意味着每个唯一的URI都是某个对象的表示以及在其上执行的操作。

URI和URL

本教程通篇使用URI（统一资源标识符）。如果让您感觉更好，请将其删除并使用URL。

在Web服务世界中，代表性状态转移（REST）是一个关键的设计习惯，它包含无状态的客户端-服务器体系结构，在该体系结构中，Web服务被视为资源，并且可以通过其URI进行标识。想要使用这些资源的Web服务客户端可以获取使用HTTP GET的内容，使用HTTP POST或HTTP UPDATE修改内容或使用HTTP DELETE删除资源。

REST-建筑风格，不是标准

REST不是标准。您不会看到W3C推出REST规范。因为REST只是一种建筑风格。您可以通过了解样式来设计Web服务。

RESTful Web服务

遵循REST原则的系统通常称为RESTful。RESTful Web服务就是其中之一，与SOAP Web Services相比，它简化了Web服务的开发。如今，用于XML Web服务的Java API（JAX-WS）为构建和部署RESTful Web服务提供了基本支持。但是，即将发布的RESTful Web服务Java API（JAX-RS）规范将提供对Java平台的RESTful Web服务的全面支持。

XML消息级别

在RESTful Web服务中，应用程序通常在XML消息级别工作，从而释放了充分利用Web服务的潜力。尽管这增加了客户端应用程序的复杂性，但不利的是，与轻松开发服务相比，这几乎没有什么障碍。JAX-WS提供了一个通用的Provider和Dispatch接口，以使用低级API访问消息有效负载。

REST的优势

  • Web服务是完全无状态的
  • 重量轻
  • 防火墙没有安全问题
  • 可读的结果
  • 相对轻松并减少开发服务的时间。
安全问题

由于防火墙不了解SOAP消息的含义，因此它们从不让那些消息通过，而REST消息则没有此问题，因为它们仅使用HTTP标准中指定的内容。

如果创建Web服务非常流行，那么REST无疑是开发Web服务的一种时髦方式。

全球资讯网

万维网是RESTful设计的关键示例。它大部分符合REST原则。REST本身从与Internet兼容的当前设计中汲取了大部分设计。

开发REST的工具

由于缺乏支持REST开发的公司，这是REST绊脚石的领域。由于REST服务易于构建，因此理论上可以使用任何符合HTTP的工具来开发它们。在Java中，JAX-WS规范为REST服务的开发和使用提供了足够的支持。
