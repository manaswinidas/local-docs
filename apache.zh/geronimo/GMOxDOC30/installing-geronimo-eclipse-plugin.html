{scrollbar}我们将使用Eclipse Galileo和Sun JDK 6.0+作为插图。可以使用以下选项来安装Geronimo Eclipse插件（GEP）：

排队

本页介绍如何安装当前版本的GEP（GEP 3.0-M1）。

有关安装以前版本的信息，请参阅此页面 。

请注意，v3.0-M1 Geronimo服务器适配器功能为v1.1.x，v2.0.x，v2.1.x，v2.2和v3.0-M1 Geronimo服务器提供了服务器适配器。支持v1.1.x服务器的原因是使v1.1.x服务器适配器与最新版本的eclipse兼容。但是，由于Geronimo 1.1尚未正式支持JDK 6，因此不建议在JDK 6中使用v1.1.x服务器适配器。请参阅GERONIMODEVTOOLS-578中的要求。

如果打算将GEP升级到最新版本，则必须首先从Eclipse卸载现有版本。有关更多详细信息，请参见卸载GEP 。

先决条件Prereqs

  • Java SE 1.6.0
  • 适用于Java EE开发人员的Eclipse IDE（Helios（Eclipse 3.6））
    • Eclipse Helios下载
  • Apache Geronimo 3.0-M1服务器
    • Apache Geronimo下载

可以从Sun的J2SE 6.0下载站点下载Sun JDK 6.0。可以从Eclipse Downloads站点下载Java EE开发人员的Eclipse IDE。下载Eclipse IDE for Java EE Developers归档文件并将其解压缩到您选择的目录（例如C：\ eclipse）。存档将被提取到您指定目录（例如C：\ eclipse \ eclipse）下名为eclipse的目录，称为从现在开始。确保PATH中的JDK是Sun JDK 6.0。通过运行启动Eclipse /eclipse 。

要安装Geronimo服务器，请从http://www.apache.org/dist/geronimo/下载Geronimo 3.0-M1服务器，然后将归档文件解压缩到您选择的目录中（例如C：\ g）。归档文件将解压缩到geronimo-tomcat6-javaee5-3.0-M1目录 （例如C：\ g \ geronimo-tomcat7-javaee5-3.0-M1）或geronimo-jetty7-javaee6-3.0-M1目录中，具体取决于您是否下载了Geronimo 3.0-M1发行版，以Tomcat或Jetty作为Web容器。我们将这个目录称为从现在开始。

使用Eclipse更新管理器安装GEP

提供了使用Eclipse Update Manager安装Apache Geronimo Eclipse插件的视频桌面演示。

  1. 安装完所有先决条件后。从启动Eclipse /eclipse 。默认情况下，Eclipse将以Java EE透视图启动。
  2. 从菜单中选择帮助->安装新软件...。





  3. 在下一个屏幕上，单击添加 。




  4. 将以下站点复制并粘贴到“ 添加站点”窗口，然后选择“ 确定” 。 html http://apache.org/dist/geronimo/eclipse/updates/
  5. 展开Geronimo WTP服务器适配器 ，然后选择Geronimo v3.0服务器适配器 ，然后单击下一步 。





  6. 等到Eclipse完成对需求和依赖项的解决。查看详细信息，然后单击完成 。



  7. 查看许可证并接受协议。



  8. 在Eclipse解决依赖关系时，您必须等待几秒钟。





  9. 选择您信任的项目，然后选择确定 。您可以通过单击details来查看这些证书的详细信息 。





  10. 一旦安装了GEP，将提示您重新启动。选择是 。

在WTP中安装GEP

  1. 安装完所有#Prerequisites后 ，从启动Eclipse。 /eclipse 。默认情况下，Eclipse将以Java EE透视图启动。
  2. 如图所示，选择“ 服务器”选项卡，然后选择“ 新建”->“服务器” 。





  3. 在下一个屏幕上，选择“ 下载其他服务器适配器” 。





  4. 选择Geronimo v3.0服务器适配器，然后选择下一步 。





  5. 接受许可协议，然后选择完成。





  6. 下一个屏幕将要求确认。选择确定 。完成后，将下载并安装Geronimo V3.0服务器适配器。





  7. 安装适配器后，将提示您重新启动服务器。选择是。




从版本3.4开始，Eclipse采用了一个称为P2的新更新管理器。更新管理器无法卸载P2未安装的插件。结果，使用此方法，eclipse 3.4中的eclipse更新管理器无法卸载GEP。有关详细信息，请参见Bug245469。

手动安装

使用updatesite.zip

  1. 从http://apache.org/dist/geronimo/eclipse/3.0-M1/下载软件包geronimo-eclipse-plugin-3.0_M1-updatesite.zip 。
  2. 完成后，将包解压缩到计算机上的目录中。
  3. 请遵循＃ 使用Eclipse更新管理器安装GEP中的说明 。在步骤5中添加更新站点时，选择Local ，然后导航到上一步中解压缩updatesite.zip软件包的目录。

使用deployable.zip

  1. 从http://apache.org/dist/geronimo/eclipse/3.0-M1/下载软件包geronimo-eclipse-plugin-3.0_M1-deployable.zip 。
  2. 完成后，将软件包解压缩到eclipse安装目录中。您可能会收到有关文件替换的提示。全选。
  3. 使用-clean选项重新启动eclipse。

验证GEP安装

  1. 选择帮助->关于Eclipse平台。





  2. 您可以在About Eclipse Platform中看到一个G图标。这确认了GEP的成功安装。单击图标。





  3. 在下一个屏幕上，您可以查看详细信息-许可证，插件详细信息，并设置要显示的表的列。


卸载GEP 卸载 GEP

GEP可以通过与安装GEP相似的方式进行卸载。

卸载在Helios上无法正常运行，尤其是对于GEP。它会卸载GEP，但是您不能重新安装它，因为它实际上并没有被卸载。在Helios中重新安装GEP的唯一方法是重新安装Helios本身！有关详细信息，请参见Eclipse Bugzilla 244935 ，希望很快能修复！
