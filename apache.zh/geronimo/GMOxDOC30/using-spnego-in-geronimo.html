在Geronimo中使用SPNEGO
编辑页面 浏览空间 添加页面 新增新闻

在Geronimo中使用简单和受保护的GSS-API协商机制（SPNEGO） ，HTTP用户只能在其桌面上登录和认证一次，然后他们才能从Geronimo服务器接收自动认证。

  • 先决条件
  • 程序
    • 设置域控制器机器
    • 设置客户端应用程序机器
      • 在Microsoft Internet Explorer浏览器中启用SPNEGO身份验证
      • 在Firefox中启用SPNEGO身份验证
    • 设置Geronimo服务器
  • 几个非常重要的注意事项

先决条件

使用SPNEGO需要三种不同的机器：

  • 运行Active Directory域控制器和关联的Kerberos密钥分发中心（KDC）的Microsoft Windows 2000或Windows 2003 Server
  • 具有Internet浏览器（例如Microsoft Internet Explorer或Firefox浏览器）的域成员
  • 运行Geronimo的服务器平台

请注意，客户端，Microsoft Active Directory域控制器和Geronimo服务器上的时钟必须同步到五分钟之内，并且它们必须位于同一域内。

程序

设置域控制器机器

  1. 在活动目录中创建一个用户帐户。确保您创建的用户是唯一的，并且未在“计算机或域控制器”中列出。该帐户最终将映射到Kerberoes服务主体名称（SPN）。
  2. 使用命令setspn将用户帐户映射到SPN。通常，SPN看起来像HTTP / 。确保您没有相同的SPN映射到多个Microsoft用户帐户。如果将同一SPN映射到多个用户帐户，则Web浏览器客户端可以向Geronimo服务器发送NT LAN管理器（NTLM）身份验证请求而不是SPNEGO令牌。有关该命令的更多用法，请参见Windows 2003技术参考（setspn命令） 。

    setspn-一个HTTP / test.xyz.com测试用户。

    哪里
    • testuser是在步骤1中创建的用户帐户
    • HTTP / test.xyz.com是映射到testuser的唯一SPN， test.xyz.com是Geronimo服务器的主机名。
  3. 创建Kereros密钥表文件（ krb5.keytab ），并使用ktpass命令将文件从域控制器复制到Geronimo服务器，从而使文件可用于Geronimo服务器。有关该命令的更多用法，请参见Windows 2003技术参考（ktpass命令） 。

    ktpass -out c：\ winnt \ krb5.keytab -princ HTTP/test.xyz.com@XYZ.COM -mapUser testuser -mapOp set -pass testuser123 -crypto RC4-HMAC-NT -pType KRB5_NT_PRINCIPAL

    哪里
    • HTTP/test.xyz.com@XYZ.COM是用户登录名和领域名称的串联，该名称必须为大写。
    • testuser是用于映射的用户帐户。
    • testuser123是用户testuser的密码。

设置客户端应用程序机器

在客户机上，Web浏览器负责为Geronimo服务器生成SPNEGO令牌供用户使用。对浏览器执行以下配置。请注意，只能通过Geronimo服务器的域名来访问Geronimo服务器上的资源，并且客户机必须是Domain的成员。

在Microsoft Internet Explorer浏览器中启用SPNEGO身份验证

  1. 在Internet Explorer窗口中，单击工具> Internet选项>安全选项卡。
  2. 选择本地Intranet图标，然后单击站点 。
  3. 确保在“ 本地Intranet”窗口中选中所有复选框，然后单击“ 高级”按钮。
  4. 将Geronimo服务器的URI名称（例如_ http：//test.xyz.com_）添加到列表网站中，以便可以启用单一登录（SSO），然后单击“ 确定”完成此步骤并关闭“ 本地”。 Intranet窗口。
  5. 在“ Internet选项”窗口上，单击“ 高级”选项卡，然后转到“ 安全设置” 。确保选中“ 启用集成Windows身份验证（需要重新启动）”复选框，然后单击“ 确定”关闭所有窗口。
  6. 重新启动您的Microsoft Internet Explorer以激活配置。

在Firefox中启用SPNEGO身份验证

  1. 在Firefox浏览器的URL地址栏中，键入about：config ，然后按Enter键。
  2. 在以下窗口中，在“ 过滤器”中键入network.nego 。
  3. 双击network.negotiate-auth.trusted-uris ，然后在弹出窗口中添加http：//，https：//，然后单击“ 确定”关闭窗口。
  4. 双击network.negotiate-auth.delegation-ruis ，然后在弹出窗口中添加http：//，https：//，然后单击“ 确定”关闭该窗口。
  5. 重新启动Firefox以激活配置。

设置Geronimo服务器

  1. 复制Keroes密钥表文件krb5.keytab到Geronimo Server的目录之一。该文件是在设置域控制器机器期间创建的。
  2. 创建一个名为Kerbeores的基本配置文件，名称为krb5.ini为了将SPNEGO用于服务器。这些文件应存储在本地服务器上，并使用以下密钥列表定义Kerberoes密钥分发中心（KDC）名称和SPNEGO身份验证的领域设置。
    krb5.ini
     
[libdefaults]
  default_realm = XYZ.COM
     default_keytab_name = FILE:c:\winnt\krb5.keytab
     default_tkt_enctypes = rc4-hmac,des-cbc-md4,des-cbc-crc
     default_tgs_enctypes = rc4-hmac,des-cbc-md4,des-cbc-crc
     forwardable=true 
[realms]
  XYZ.COM = {
        kdc = domaincontroller.xyz.com:88
        default_domain = xyz.com   
        }
[domain_realm]
  xyz.com= XYZ.COM
  .xyz.com = XYZ.COM
  3. 使用以下密钥对配置JVM属性，以确保JVM成功读取Kerberoes配置。

    设置JAVA_OPTS = -Djava.security.krb5.conf = C：\ winnt \ krb5.ini -Dcom.ibm.security.jgss.debug = all -Dcom.ibm.security.krb5。Krb5Debug = all -Djavax.security.auth.useSubjectCredsOnly = false

  4. 按照以下步骤为Geronimo服务器创建系统范围的领域。该示例代码是SPNEGO和.properties文件领域的组合，以使身份验证依靠。SPNEGO身份验证失败后，属性将进入领域。如果不需要，可以删除.properties文件领域。
    spnego_properties_realm.xml
    <module xmlns="http://geronimo.apache.org/xml/ns/deployment-1.2">
    <environment>
        <moduleId>
            <groupId>console.realm</groupId>
            <artifactId>SpnegoTest</artifactId>
            <version>1.0</version>
            <type>car</type>
        </moduleId>
        <dependencies>
            <dependency>
                <groupId>org.apache.geronimo.framework</groupId>
                <artifactId>j2ee-security</artifactId>
                <type>car</type>
            </dependency>
        </dependencies>
    </environment>
    <gbean name="SpnegoTest" class="org.apache.geronimo.security.realm.GenericSecurityRealm" xsi:type="dep:gbeanType" 
                 xmlns:dep="http://geronimo.apache.org/xml/ns/deployment-1.2" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
        <attribute name="realmName">SpnegoTest</attribute>
        <reference name="ServerInfo">
            <name>ServerInfo</name>
        </reference>
        <xml-reference name="LoginModuleConfiguration">
            <log:login-config xmlns:log="http://geronimo.apache.org/xml/ns/loginconfig-2.0">
                <log:login-module control-flag="SUFFICIENT" wrap-principals="false">
                    <log:login-domain-name>SpnegoTest</log:login-domain-name>
                    <log:login-module-class>org.apache.geronimo.security.realm.providers.SpnegoLoginModule</log:login-module-class>
                    <log:option name="targetName">http/test.xyz.com</log:option>
					<log:option name="ldapUrl">ldap://domaincontroller.xyz.com:389</log:option>
					<log:option name="ldapLoginName">testuser</log:option>
					<log:option name="ldapLoginPassword">testuser123</log:option>
					<log:option name="searchBase">DC=xyz,DC=com</log:option>
                </log:login-module>
                <log:login-module control-flag="SUFFICIENT" wrap-principals="false">
                    <log:login-domain-name>demo-properties-realm</log:login-domain-name>
                    <log:login-module-class>org.apache.geronimo.security.realm.providers.PropertiesFileLoginModule</log:login-module-class>
                    <log:option name="usersURI">var/security/demo_users.properties</log:option>
                    <log:option name="groupsURI">var/security/demo_groups.properties</log:option>
                </log:login-module>
            </log:login-config>
        </xml-reference>
    </gbean>
</module>
  5. 配置应用程序的部署计划，以确保正确调用SPNEGO领域。请参阅下面的示例代码以供参考。
    geronimo-web.xml
    <?xml version="1.0" encoding="UTF-8"?>
<web:web-app xmlns:app="http://geronimo.apache.org/xml/ns/j2ee/application-2.0"    xmlns:client="http://geronimo.apache.org/xml/ns/j2ee/application-client-2.0" 
        xmlns:conn="http://geronimo.apache.org/xml/ns/j2ee/connector-1.2" xmlns:dep="http://geronimo.apache.org/xml/ns/deployment-1.2" 
        xmlns:ejb="http://openejb.apache.org/xml/ns/openejb-jar-2.2" xmlns:name="http://geronimo.apache.org/xml/ns/naming-1.2" 
        xmlns:pers="http://java.sun.com/xml/ns/persistence" xmlns:pkgen="http://openejb.apache.org/xml/ns/pkgen-2.1" 
        xmlns:sec="http://geronimo.apache.org/xml/ns/security-2.0" xmlns:web="http://geronimo.apache.org/xml/ns/j2ee/web-2.0.1">
    <dep:environment>
        <dep:moduleId>
            <dep:groupId>com.mycompany.samples</dep:groupId>
            <dep:artifactId>security-demo</dep:artifactId>
            <dep:version>3.0</dep:version>
            <dep:type>war</dep:type>
        </dep:moduleId>
        <dep:dependencies/>
        <dep:hidden-classes>
            <dep:filter>
	        org.apache.geronimo.security.realm.providers.SpnegoLoginModule
	    </dep:filter>
        </dep:hidden-classes>
        <dep:non-overridable-classes/>
    </dep:environment>
    <web:context-root>/demo</web:context-root>
    <web:security-realm-name>SpnegoTest</web:security-realm-name>
    <sec:security>
        <sec:role-mappings>
            <sec:role role-name="content-administrator">
                <sec:principal class="org.apache.geronimo.security.realm.providers.GeronimoGroupPrincipal" name="Domain Admins"/>
		<sec:principal class="org.apache.geronimo.security.realm.providers.GeronimoUserPrincipal" name="testuser@TEST.XYZ.COM"/>
            </sec:role>
            <sec:role role-name="Guest-administrator">
           <sec:principal class="org.apache.geronimo.security.realm.providers.GeronimoGroupPrincipal" name="Domain Admins"/>
             </sec:role>
        </sec:role-mappings>
    </sec:security>
</web:web-app>
  6. 配置部署描述符，以确保您的应用程序使用SPNEGO身份验证以及Geronimo服务器支持的相应领域提供程序。
    web.xml的摘录
    <?xml version="1.0" encoding="ISO-8859-1"?>
...
   <login-config>
      <auth-method>SPNEGO</auth-method>
      <realm-name>SpnegoTest</realm-name>
      ...
   </login-config>

几个非常重要的注意事项

  • 如果要使用geronimo配置Spnego，请确保在Web应用程序中使用Basic作为身份验证机制。
  • 提供的领域是2个登录模块的组合，可以通过geronimo管理控制台轻松创建。
  • 在为Spnego登录模块创建安全领域时，您只需要指定一个选项即可，其形式为“ targetName = http / ”。看一下示例领域。这将使您对要使用的选项有所了解。
  • 创建2个登录模块时，请确保选择足够的控制标志。
  • 确保按照“设置Active Directory域控制器”的＃2中的定义，仅将一个用户映射到SPN。
