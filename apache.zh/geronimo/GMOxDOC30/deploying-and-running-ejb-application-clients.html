部署和运行EJB应用程序客户端

本教程将引导您使用Eclipse和Geronimo配置，开发和部署企业应用程序。要运行本教程，至少需要您安装以下必备软件。

  1. Sun JDK 6.0+（J2SE 1.6）
  2. 面向特定平台的Java EE开发人员的Eclipse IDE
  3. Apache Geronimo Eclipse插件2.1.x
  4. Apache Geronimo服务器2.1.x

    本教程使用了Geronimo 2.1.x版，Java 1.5运行时和Eclipse Ganymede，但可以使用其他版本（例如Geronimo 2.2版，Java 1.6，Eclipse Europa）

开发环境部分中提供了有关安装Eclipse的详细信息。本教程分为以下几节：

  • 设置Eclipse IDE以进行EJB应用程序开发
  • 开发EJB本地接口，EJB远程接口和Bean类
  • 开发Java客户端
  • 部署并运行

让我们简要了解此应用程序。该应用程序将带您创建一个简单的无状态会话EJB。稍后，我们将开发Java应用程序客户端来访问此EJB。 EJB开发将使用Java EE5中引入的注释。

设置Eclipse IDE以进行EJB应用程序开发

  1. 启动Eclipse并选择Window-> Open Perspective-> Other-> JavaEE并选择OK


这样就完成了用于EJB应用程序开发的Eclispe IDE的设置。

开发EJB本地接口，EJB远程接口和Bean类

  1. 右键单击ejb软件包，然后创建一个新的Remote Interface CountryCapital ，如图所示：




  2. 输入如图所示的字段，然后选择“ 完成” 。




  3. 添加如下代码，如下所示
    CountryCapital.java
    package ejb;

import javax.ejb.Remote;
@Remote
public interface CountryCapital {
	public String capitalName(String countryName);
}
    这里@Remote是用于将接口声明为远程接口的注释。
  4. 同样，创建一个Local接口CountryCapitalLocal 。将以下代码添加到接口。
    CountryCapitalLocal.java
    package ejb;

import javax.ejb.Local;
@Local
public interface CountryCapitalLocal {
	public String capitalName(String countryName);

}

    类似地，在此代码中，我们具有@Local注释，该注释将该接口声明为local。

    在“ 远程 ”和“ 本地”接口中，我们仅具有一种业务方法的声明。因此，只有此方法对应用程序客户端可见。如果在Bean类中实现，则除capitalName之外的方法将是Bean类专用的，并且对应用程序客户端不可见。

  5. 下一步是创建一个Bean类，该类将实现要执行的Business方法。右键单击ejb包并创建一个新类。




  6. 输入类名称为CountryCapitalBean 。




  7. 使用以下代码填充bean类
    CountryCapitalBean.java
    package ejb;

import javax.ejb.Stateless;
@Stateless
public class CountryCapitalBean implements CountryCapital,CountryCapitalLocal{
	public String capitalName(String countryName)
	{
		String capital=new String("No such country");
		if (countryName.equalsIgnoreCase("India"))
		{
			capital="New Delhi";
		}
		if (countryName.equalsIgnoreCase("United States Of America"))
		{
			capital="Washington DC";
		}
		if (countryName.equalsIgnoreCase("China"))
		{
			capital="Beijing";
		}

		return capital;

	}

}

这样就完成了EJB应用程序的开发。

开发Java客户端

  1. 选择文件->新建->项目-> JavaEE->应用程序客户端项目 。选择下一步 。




  2. 在“ 新建项目”窗口中，将项目名称命名为ApplicationClient，然后选择“ 下一步” 。




  3. 在“ Java设置”窗口上。选择“ 项目”选项卡，然后单击“ 添加” 。




  4. 选中SimpleEJB复选框，然后选择确定 。




  5. 如图所示，这会将SimpleEJB项目添加到Application Client的构建路径。选择完成 。




    为什么我们要添加EJB项目来构建路径？

    这是因为构建路径用于查找Application Client源代码引用的类，并且这些类对于Application Client源代码的编译是必需的。


  6. 右键单击ApplicationClient项目，然后创建一个新的Java类，如下图所示：




  7. 将包名称命名为appclient ，并将类名称命名为ApplicationClient 。选择完成 。

  8. 填充ApplicationClient.java具有以下代码的类：
    ApplicationClient.java
    package appclient;

import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.util.Properties;
import ejb.CountryCapital;
import javax.naming.Context;
import javax.naming.InitialContext;

public class ApplicationClient {
	 public static void main(String [] args)
	 {
		 String capital=new String();
		 try{
			 Properties prop=new Properties();
			 prop.put(Context.INITIAL_CONTEXT_FACTORY, "org.apache.openejb.client.RemoteInitialContextFactory");
				prop.put("java.naming.provider.url", "ejbd://localhost:4201");
	 Context context = new InitialContext(prop);

	 CountryCapital myejb =
	 (CountryCapital)context.lookup("CountryCapitalBeanRemote");
	 System.out.println("Give the name of a country");
	 BufferedReader in= new BufferedReader(new InputStreamReader(System.in));
	 String str="";
	 str=in.readLine();
	 capital=myejb.capitalName(str);
	 System.out.println(capital);
		 }
		 catch(Exception e)
		 {
			 e.printStackTrace();
		 }
	 }


}

    让我们尝试理解以下代码片段：
    代码片段
    Properties prop=new Properties();
prop.put(Context.INITIAL_CONTEXT_FACTORY, "org.apache.openejb.client.RemoteInitialContextFactory");
prop.put("java.naming.provider.url", "ejbd://localhost:4201");
Context context = new InitialContext(prop);
CountryCapital myejb =(CountryCapital)context.lookup("CountryCapitalBeanRemote");

    上面的代码建议使用哪个InitialContextFactory来创建InitialContext。在这里我们正在使用org.apache.openejb.client.RemoteInitialContextFactory创建InitialContext。在这种情况下，因为EJB服务器是命名服务提供者，并且具有在端口4201上运行的命名服务。因此，我们需要指定EJB服务器的位置。在这种情况下ejbd://localhost:4201 。完成此操作后，便有了上下文，该上下文使我们能够查找和获取对象。

    为什么查找名称为CountryCapitalBeanRemote？

    这将在部署和运行部分中讨论。


  9. 下一步是添加org.apache.openejb.client.RemoteInitialContextFactory类到构建路径。这是因为应用程序客户端需要此类来创建InitialContext，如上所述。
  10. 右键单击Project Explorer下的ApplicationClient项目。选择属性 。




  11. 选择Java构建路径 。在Java Build Path下，选择Libraries-> Add External JARs 。




  12. 浏览到\repository\org\apache\openejb\openejb-client\3.0并选择jar openejb-client-3.0.jar 。选择确定 。

这样就完成了应用程序客户端开发。

部署并运行

本节将带您完成EJB应用程序的部署。稍后，我们将使用一些示例输入来运行应用程序客户端。

  1. 在Java EE透视图中 ，右键单击Geronimo服务器，然后选择Add and Remove Projects...。
  2. 您的工作区中应该有一个可部署项目。选择全部添加>> ，然后选择完成 。
  3. 一旦部署成功打开/var/log/geronimo.log并找到以下内容：
    部署SimpleEJB之后，来自geronimo.log的片段
    18:16:39,750 INFO  [startup] Jndi(name=CountryCapitalBeanLocal) --> Ejb(deployment-id=SimpleEJB/CountryCapitalBean)
18:16:39,750 INFO  [startup] Jndi(name=CountryCapitalBeanRemote) --> Ejb(deployment-id=SimpleEJB/CountryCapitalBean)
    现在， OpenEJB容器可以识别 “ 远程 ”和“ 本地”查找名称。在本教程中， SimpleEJB应用程序和Java Application Client在不同的JVM实例下运行，因此我们需要使用CountryCapitalBeanRemote作为远程访问的查找名称。
  4. 在Eclipse下，右键单击Application Client项目，然后以Java Application的身份运行。
