如何从源代码构建Geronimo Eclipse插件
编辑页面 浏览空间 添加页面 新增新闻

要从源代码构建Geronimo Eclipse插件（GEP），您需要安装以下先决条件。

  • Apache Maven 2.2.1+
  • Subversion 1.5.4+
  • Sun JDK 6.0+下载
  • 将Maven和Subversion的bin目录添加到系统变量Path中 。

    有用的信息

    您还可以通过修改/conf/settings.xml并修改以下标记来更改m2repo的默认位置
    C：\ m2repo

    • 获取源代码
    • 建筑资源
    • 其他Maven选项
      • 跳过测试

    获取源代码

  1. 在根目录中创建目录gep。

    有用的信息

    强烈建议您为源代码下载使用一个小的目录名称。有时由于路途遥远而出现问题。

  2. 打开命令提示符并更改之前创建的gep目录。运行以下命令以开始下载源。 
    svn co http://svn.apache.org/repos/asf/geronimo/devtools/eclipse-plugin/trunk trunk



  3. Subversion将开始下载源代码，并且命令窗口将显示进度信息。





  4. 成功下载后，您可以看到下载源的修订号。





建筑资源

  1. 如下修改JVM的堆设置。

    需要设置以下环境变量以正确配置JVM以满足内存要求。
    MAVEN_OPTS =“-Xms512m -Xmx512m -XX：MaxPermSize = 256m”

    在Windows上，使用以下命令来设置环境变量（请注意，不应使用双引号）：
    设置MAVEN_OPTS = -Xmx512m -XX：MaxPermSize = 128m

  2. 转到已下载源的主干目录，然后运行以下命令
    mvn clean install



  3. 完成后，各种依赖项将下载到您的m2存储库。如果您是第一次构建GEP，则可能需要花费大量时间。



  4. 成功构建后，您可以在命令提示符下看到“ Build Success”消息。





    您无需手动下载Eclipse或Web工具项目即可构建。在构建过程中会自动为您处理。您可以在以下位置找到下载内容
    \ eclipse-下载

    跳过测试

    有时您的构建可能会由于测试用例失败而失败。您可以通过运行以下命令来跳过测试
    mvn clean -Dmaven.test.skip=true -Dmaven.itest.skip=true install

    如果要针对GEP源代码运行集成测试，请使用mvn clean install -Ptestsuite来构建GEP。但是请注意，此命令可能会导致GEP构建失败，因为Maven中央存储库中不再有许多GEP依赖的jar文件，并且另一个原因是trunk中的源代码还不稳定。
