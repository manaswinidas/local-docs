Geronimo中的JMS集群
编辑页面 浏览空间 添加页面 新增新闻

Activeon组件使用蓝图服务来处理Geronimo中的JMS集群。您可以将代理配置为集群，并且如果JMS代理使用主从功能关闭，则JMS请求可以故障转移到另一个代理。

先决条件

确保系统模块org/apache/geronimo/activemq-ra/3.0/car在服务器启动期间加载。然后更新config-substitutions.properties文件在/var/config目录以在服务器停止时为每个ActiveMQ节点指定IP地址或主机名。

config-substitutions.properties
ActiveMQHostname=hostname/IP

JMS集群方案

根据ActiveMQ文档，可以使用不同类型的主/从配置：

  • 纯主奴隶
  • 共享文件系统
  • JDBC主从

在Geronimo服务器中，所有这些配置都是使用蓝图服务来处理的。您需要更新的内容activemq.xml中activemq-broker-blueprint-3.0.car下/repository/org/apache/geronimo/configs/activemq-broker-blueprint/3.0目录根据您选择的方案。更简单的方法是解压缩activemq-broker-blueprint-3.0.car并在修改后重新打包。

对于Geronimo中的每个方案，请参见以下配置

纯主奴隶

在这种情况下，您必须明确指定主节点和从节点，并手动重新启动发生故障的主节点。

主节点

在主节点上，您只需通过使用brokerName属性指定当前节点是主节点，如下所示。

activemq.xml
...
       <cm:property name="serverHostname" value="masterIP"/>
...

从节点

因为在“纯主/从”方案中每个主节点只有一个从属节点，所以从属节点必须知道主节点的URI，并且还必须使用brokerName属性将其标记为从属节点。

activemq.xml
...
       <cm:property name="serverHostname" value="slaveIP"/>
...

客户端连接

JMS客户端使用failover：//协议定位集群中的代理。请参见以下示例：

故障转移：//（tcp：// masterIP：61616，tcp：// slaveIP：61616）？randomize =假

共享文件系统

在这种情况下，您必须使用共享文件系统来提供代理的高可用性和自动发现主/从节点。共享文件夹必须允许不同的从站具有写权限。

每个节点

在每个节点上，将共享目录配置为代理使用代理的位置。 persistenceAdapter元素如下：

activemq.xml
...
       <cm:property name="serverHostname" value="broker1"/>
...
       <amq:persistenceAdapter>
            <amq:amqPersistenceAdapter directory="/sharedFileSystem/sharedBrokerData"/>
       </amq:persistenceAdapter>
...

注意：

  • 对于Linux节点上的共享文件系统，必须首先安装共享目录。
  • 对于Windows节点上的共享文件系统，可以使用以下路径： //ipAddress/sharedFolder在配置中。
  • 在每个节点上， broker1应该替换为当前节点的确切IP地址。

客户端连接

JMS客户端使用failover：//协议在群集中定位代理。请参见以下示例：

故障转移：//（tcp：// broker1：61616，tcp：// broker2：61616，tcp：// broker3：61616）？randomize =假

JDBC主从

在这种情况下，您必须使用共享数据库作为持久性引擎和自动恢复。

每个节点

在每个节点上，使用以下命令配置共享数据库池jdbcPersistenceAdapter元素如下。我们以一个远程Oracle数据库服务器为例：

activemq.xml
...
     <cm:property name="serverHostname" value="broker1"/>
...
     <amq:persistenceAdapter>
         <amq:jdbcPersistenceAdapter dataSource="#oracle-ds"/>
     </amq:persistenceAdapter>
...
     <bean id="oracle-ds" class="org.apache.commons.dbcp.BasicDataSource" destroy-method="close">
      <property name="driverClassName" value="oracle.jdbc.driver.OracleDriver"/>
      <property name="url" value="jdbc:oracle:thin:@dbServer:1521:AMQDB"/>
      <property name="username" value="scott"/>
      <property name="password" value="tiger"/>
      <property name="maxActive" value="200"/>
      <property name="poolPreparedStatements" value="true"/>
    </bean>
...

注意：

  • 对于数据库服务器， 应将dbServer替换为数据库服务器的实际IP地址。
  • 在每个节点上， broker1应该替换为当前节点的确切IP地址。
