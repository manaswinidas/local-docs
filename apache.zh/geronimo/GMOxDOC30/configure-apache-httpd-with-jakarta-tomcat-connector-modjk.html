使用Jakarta Tomcat Connector（mod_jk）配置Apache HTTPd
编辑页面 浏览空间 添加页面 添加新闻

HTTP服务器和Geronimo之间的通信也可以通过AJP连接器来完成。默认情况下，两个Apache Geronimo发行版（Jetty和Tomcat）都已经预定义了一个侦听端口8009的AJP13连接器。

Jakarta Tomcat连接器mod_jk模块作为来自Apache Tomcat源的连接器提供。Jetty和Tomcat都与此连接器完全兼容。当您下载Tomcat源代码时，此模块可用，但是您也可以单独下载它。有关系统的正确版本，请参见http://tomcat.apache.org/download-connectors.cgi 。

除了mod_jk之外 ，您还需要一个workers.properties文件，也随Apache Tomcat源发行版一起提供。该属性文件告诉mod_jk插件如何连接到Geronimo服务器。有关配置Jakarta Tomcat连接器的所有可用选项的详细说明，请访问http://tomcat.apache.org/connectors-doc/config/workers.html 。

出于实用的目的，模块mod_jk-apache-2.2.3.so为Windows和workers.properties都包含在本文中的附件部分。

配置Apache HTTPd

从Tomcat网站下载适合您平台的mod_jk 。对于此特定示例，将其重命名为mod_jk.so并将其复制到\modules目录。从Apache Tomcat源下载并提取worker.properties文件（或者，从“ 附件”部分下载文件）到\conf目录。

编辑httpd.conf文件位于\conf目录以加载Jakarta Tomcat连接器mod_jk模块。在末尾添加以下行httpd.conf文件。

摘录自httpd.conf
LoadModule jk_module modules/mod_jk.so			
# Loads the Jakarta Tomcat Connector module

JkWorkersFile <httpd_home>\conf\workers.properties
# Tells the module the location of the workers.properties file

JkLogFile     <httpd_home>\logs\mod_jk.log
# Specifies the location for this module's specific log file

JkLogLevel    info
# Sets the module's log level to info

JkMount /console/* ajp13
# Sets a mount point from a context to a Tomcat worker. In this case will allow access (forward the request) to the console.

JkMount会将/ console /后面的任何内容映射到worker ajp13 。名称ajp13在workers.properties file ，将在以下部分中进行描述。您可能需要添加更多的JkMount指令，具体取决于您希望通过远程HTTPd访问的应用程序。

在此示例中，仅出于演示目的而启用了控制台 。在生产环境中，您不希望从其他网络（通常是Internet）访问控制台 。具有控制台可访问性代表着很大的安全隐患。

规则是，所有内容都应具有受限的访问权限。通常，将在HTTP和应用程序服务器之间放置一个防火墙（取决于拓扑），并且您应该仅映射使应用程序从另一端运行所需的最少资源。

配置worker.properties

的workers.properties文件除其他外，还告诉HTTPd Geronimo服务器在哪里，要使用的AJP版本以及Geronimo监听的端口。

编辑workers.properties文件位于\conf目录以匹配您的环境。以下示例摘自workers.properties文件，其中包含您应关注的变量。

对workers.properties的最低要求
worker.ajp13.type=ajp13
# Sets the version of AJP used. The AJP listeners defined in Geronimo are AJP v13.

worker.ajp13.host=localhost
# Specifies the location of the Geronimo server. Use default localhost for single-tier scenarios. Specify the
# hostname of the Geronimo server for multi-tier environments.

worker.ajp13.port=8009
# Both Tomcat and Jetty come with a predefined AJP13 listener on port 8009

在此示例中，请注意工作人员的名称是如何定义的。查看变量定义worker.ajp13 ，其中ajp13是您先前在httpd.conf文件。

最后，停止并重新启动Apache HTTPd，以确保加载了这些更改。

测试中

为了测试此配置，请确保Geronimo和HTTPd都已启动并正在运行。

  1. 通过访问http：// localhost：8080 / console来检查Geronimo的连接性，您应该看到Geronimo管理控制台。
  2. 通过访问http：// localhost检查HTTPd连接，您应该看到Apache HTTPd欢迎页面。
  3. 通过访问http：// localhost / console /检查HTTPd-Geronimo请求转发，您应该被重定向到Geronimo管理控制台。请注意，URL的末尾有一个“ /”，如果不包含此/，将导致Jakarta Tomcat Connector模块触发“ 未找到”错误。
