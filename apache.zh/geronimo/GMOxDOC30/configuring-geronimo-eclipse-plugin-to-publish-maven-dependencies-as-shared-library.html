配置Geronimo Eclipse插件以将Maven依赖项发布为共享库

为了配置GEP以将Maven依赖项发布为共享库，您将需要安装Maven Integration for Eclipse 。要安装m2eclipse插件，请按照http://m2eclipse.codehaus.org/上给出的说明进行操作。使用Web应用程序并添加了commons-dbcp库的依赖关系来说明配置步骤。

  1. 按照开发Hello World Web应用程序中的说明创建一个HelloWorld Dynamic Web项目。
  2. 启用对Web应用程序项目的Maven 2支持，如下所示：
    • 在Project Explorer视图中，右键单击HelloWorld ，选择Maven- > Enable Dependency Management 。
    • 保持默认选项不变，然后单击Finish 。
  3. 要添加对commons-dbcp库的依赖关系：
    • 在Project Explorer视图中，右键单击HelloWorld ，选择Maven- > Add Dependency 。
    • 输入commons-dbcp作为查询，在Search Results中选择1.3-SNAPSHOT-commons-dbcp-1.3- .. ，然后单击OK 。
  4. 在Web应用程序的内部使用commons-dbcp库中的类index.jsp ：
    • 双击HelloWorld- > WebContent- > index.jsp在JSP编辑器中将其打开。
    • 在以下代码之前添加以下代码
标签。
<％= org.apache.commons.dbcp。DriverConnectionFactory.class.getName（）％>
  • 按Ctrl + S保存更改。

    确保org.apache.commons.dbcp。解决了DriverConnectionFactory类，没有类型未解决的错误。

  • 如下所示在服务器编辑器中启用就地共享库支持：
    • 在“ 服务器”视图中，双击位于本地主机上的Apache Geronimo v2.1 Server，以打开服务器编辑器。
    • 在服务器编辑器的“ 测试环境”下 ，选择“ 启用就地共享库支持” 。



    • 按Ctrl + S保存更改。
  • 要添加对org.apache.geronimo.configs / sharedlib // car的依赖geronimo-web.xml ：
    • 双击HelloWorld- > Web内容 -> WEB-INF- > geronimo-web.xml ，在Geronimo Deployment Plan Editor中将其打开。
    • 在“ 常规”选项卡中，选择“ 将运行时依赖项添加到Geronimo的共享库” 。



    • 按Ctrl + S保存文件。

      在“ 源”选项卡中，将添加对geronimo / sharedlib // car的依赖关系，如下所示：

      geronimo sharedlib 车

      这是由于Geronimo部署计划编辑器中的当前错误所致。从geronimo更改为org.apache.geronimo.configs ，然后按Ctrl + S保存更改。

  • 要运行HelloWorld应用程序：
    • 右键单击HelloWorld ，单击运行方式 -> 在服务器上运行 。
    • 在本地主机上选择Apache Geronimo v2.1 Server，然后单击Finish 。这将运行该应用程序，并且欢迎页面将在Eclipse中的浏览器中打开。



    • 使用以下步骤来验证是否确实将maven依赖项添加为共享库：

    1. 验证文件HelloWorld.eclipse.jar在下创建/var/shared/lib 。
    2. 验证Manifest.mf HelloWorld.eclipse.jar中的文件包含以下内容： 清单版本：1.0类路径：文件：/ E：/m2repo/commons-pool/commons-pool/1.3/commons-pool -1.3.jar文件：/ E： /m2repo/commons-dbcp/commons-dbcp/1.3-SNAPSHOT/commo ns-dbcp-1.3-SNAPSHOT.jar

      清单类路径中使用的文件路径可能会根据maven2存储库的位置而有所不同。
