如何打开Geronimo Eclipse插件的错误
编辑页面 浏览空间 添加页面 添加新闻

JIRA在GERONIMODEVTOOLS项目中跟踪了Geronimo Eclipse插件（GEP）问题。
有关GEP的问题可以在Geronimo 邮件列表中提出 。
Eclipse Platform或WTP问题应在Eclipse Bugzilla下打开。
与Eclipse，WTP或任何Eclipse Project有关的其他问题应在Eclipse Newsgroups上提出。

在打开错误之前，搜索Apache Geronimo 邮件列表始终是一件好事。您可以搜索以下Geronimo邮件列表的归档文件Geronimo Apache Geronimo用户 Apache Geronimo Developer 。如果您确定遇到的问题是错误，请继续执行以下步骤。

Apache Geronimo和Geronimo Eclipse插件（GEP）的每个发行版都有许多错误修复。如果您使用的是Geronimo / Geronimo Eclipse插件的旧版本，请尝试使用最新的可用版本，并查看是否遇到相同的问题。

搜索GERONIMODEVTOOLS JIRA问题跟踪数据库，以查看是否已报告您的错误，或者甚至已修复了该错误。您也许可以将其他信息添加到以前报告的错误中。

  • 报告GEP错误

报告GEP错误

  1. 启动报告GEP错误 。
  2. 完成后，您将必须登录。如果您没有一个帐户，则可以创建一个帐户。这将帮助您跟踪所报告错误的进度。
  3. 完成后，选择“ 创建新问题” 。
  4. 在下一个屏幕上，选择Project作为Geronimo Devtools ，对于问题，您可以签出针对问题的帮助下拉框。
  5. 在下一页上，选择您遇到问题的版本。如果您使用的是Geronimo / Geronimo Eclipse插件的旧版本，建议您使用最新版本并尝试解决相同的问题。
  6. 选择您希望修复该问题的版本。
  7. 正确提及环境，其中可能包括您的操作系统，Java版本，Geronimo版本，GEP版本。如果这是构建失败，则还要指定您的Maven版本。
  8. 请提供描述，作为迈向发现错误的一系列步骤。
  9. 您还可以附加应用程序，测试用例以重新创建问题，截图等。
