在里面geronimo-web.xml文件，应用程序部署程序映射安全角色，ejb名称，数据库资源，JMS资源等。 web.xml到服务器中部署的相应实体。除此之外，如果有任何特定于Web容器的配置（例如Tomcat或Jetty特定），则取决于应用程序的需求，所有这些设置也在此处配置。如果Web应用程序依赖于服务器中运行的任何第三方库或其他服务，则所有这些依赖项都将在计划中声明。某些Web应用程序要求的类加载要求不同于默认的类加载行为。的geronimo-web.xml允许应用程序部署者也配置它。还有许多其他配置可以通过geronimo-web.xml取决于Web应用程序的需求。以下各节简要说明了geronimo-web.xml可用于配置Web容器和Web应用程序。

Web应用程序的样本计划

例如，以下web.xml和geronimo-web.xml是分别连接到DB2上部署的数据源并从表中检索数据的Web应用程序的部署描述符和Geronimo部署计划。

web.xml
<?xml version="1.0" encoding="ISO-8859-1"?>
<web-app xmlns="http://java.sun.com/xml/ns/javaee"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://java.sun.com/xml/ns/javaee
         http://java.sun.com/xml/ns/javaee/web-app_2_5.xsd" version="2.5">
    
    <resource-ref>
        <res-ref-name>jdbc/DataSource</res-ref-name>
        <res-type>javax.sql.DataSource</res-type>
        <res-auth>Container</res-auth>
        <res-sharing-scope>Shareable</res-sharing-scope>
    </resource-ref>
    
    <welcome-file-list>
        <welcome-file>jsp/EMPdemo.jsp</welcome-file>
    </welcome-file-list>
</web-app>

上面的XML文档的默认名称空间是http://java.sun.com/xml/ns/javaee 。没有名称空间前缀的XML元素属于默认名称空间。

在Servlet 2.5规范中，许多声明是通过web.xml也可以通过Servlet和JSP中的相应注释来完成。同时提供注释和web.xml时，web.xml中的声明优先于注释。

Web模块使用其JNDI名称连接到后端数据源jdbc/DataSource如在声明web.xml 。

在geronimo-web.xml ， 添加了一个元素以使资源档案（RAR）对您的应用程序可见，然后在中提供了映射。

geronimo-web.xml
<web-app xmlns="http://geronimo.apache.org/xml/ns/j2ee/web-2.0.1"
          xmlns:naming="http://geronimo.apache.org/xml/ns/naming-1.2"
          xmlns:sec="http://geronimo.apache.org/xml/ns/security-2.0"
          xmlns:sys="http://geronimo.apache.org/xml/ns/deployment-1.2">

    <sys:environment>
        <sys:moduleId>
            <sys:groupId>samples</sys:groupId>
            <sys:artifactId>EmployeeDemo</sys:artifactId>
            <sys:version>2.5</sys:version>
            <sys:type>war</sys:type>
        </sys:moduleId>
        
        <sys:dependencies>
            <sys:dependency>
                <sys:groupId>samples</sys:groupId>
                <sys:artifactId>EmployeeDatasource</sys:artifactId>
                <sys:version>2.5</sys:version>
                <sys:type>rar</sys:type>
            </sys:dependency>
        </sys:dependencies>
    </sys:environment>
    
    <context-root>/EmployeeDemo</context-root>
        <naming:resource-ref>
        <naming:ref-name>jdbc/DataSource</naming:ref-name>
        <naming:resource-link>jdbc/EmployeeDatasource</naming:resource-link>
    </naming:resource-ref>
</web-app>

上面的XML文档的默认名称空间是http://geronimo.apache.org/xml/ns/j2ee/web-2.0.1 。没有名称空间前缀的XML元素属于默认名称空间。

观察部署计划中出于各种目的使用的各种XML标记和相应的名称空间。

.. ：这些元素提供moduleid配置和依赖关系。moduleId元素提供Web模块的配置名称。因此，在部署Web模块时，会为其指定配置名称samples/EmployeeDemo/2.5/war 。依赖项元素提供了Web模块所依赖的配置和第三方库。这些配置和库将通过类加载器层次结构提供给Web模块。在这种情况下，Web模块取决于samples/EmployeeDatasource/2.5/rar这是连接到后端DB2数据库的已部署数据源的配置。数据源部署数据库连接池(javax.sql.Datasource)有名字jdbc/EmployeeDatasource 。

.. ：用于提供Web应用程序Web上下文根的XML元素。

.. ：这些元素有助于我们配置资源引用。在这种情况下，数据源参考jdbc/DataSource映射到jdbc/EmployeeDatasource 。

Web应用程序中特定于容器的配置

在某些情况下，您将需要为Web应用程序提供特定于容器的设置。WASCE服务器仅包含Tomcat Web容器，因此此处仅显示Tomcat特定的设置。

container-config元素

的元素用于使用以下格式保存特定于容器的Web应用程序设置。不需要任何Tomcat配置元素，但是如果指定，它们必须遵循以下示例中的顺序。


<container-config>
    <tomcat xmlns="http://geronimo.apache.org/xml/ns/web/tomcat/config-1.0">
        <!-- Begin Tomcat configuration elements -->
        <host>host</host>
        <cross-context/>
        <disable-cookies/>
        <valve-chain>valve</valve-chain>
        <listener-chain>listener</listener>
        <tomcat-realm>realm</tomcat-realm>
        <manager>manager</manager>
        <cluster>cluster</cluster>
        <!-- End Tomcat configuration elements -->
    </tomcat>
</container-config>

哪里

  • host是在虚拟主机的initParams属性的name参数中定义的主机的名称 Web应用程序将部署到的位置。有关配置虚拟主机的更多信息，请参见配置虚拟主机参考。
  • 表示此Web应用程序将向其他应用程序分发请求，并且应启用此功能。如果要禁用跨上下文分派请求，请不要指定此标记。
  • 表示此Web应用程序将不使用cookie，因此应禁用此功能。如果要启用cookie，请不要指定此标签。
  • valve是a的名称， 它定义了阀链中的第一个阀。有关定义阀链的更多信息，请参见阀链参考。
  • listener是a的名称， 它定义生命周期事件侦听器链中的第一个侦听器。有关定义生命周期侦听器链的更多信息，请参见生命周期侦听器参考。
  • 域是Tomcat域的名称用于此Web应用程序的身份验证和授权。如果未指定，则服务器将使用Tomcat配置中定义的领域。
  • manager是Tomcat管理器的名称用于管理Web应用程序资源。
  • cluster是Tomcat集群的名称 Web应用程序将部署到的位置。有关定义Tomcat群集的更多信息，请参见《 群集和服务器场参考》。
