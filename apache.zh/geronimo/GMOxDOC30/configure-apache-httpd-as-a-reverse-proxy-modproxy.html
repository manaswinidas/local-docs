将Apache HTTPd配置为反向代理（mod_proxy）
编辑页面 浏览空间 添加页面 添加新闻

这是将请求从Apache HTTPd重新路由到Geronimo服务器（或您可能拥有的任何其他服务器）的最简单方法。要使用此功能，您需要启用一些特定的模块并向HTTPd配置添加几行。这些步骤描述如下：

  1. 打开httpd.conf文件位于\conf目录。
  2. 查找以下LoadModule指令，并通过删除该行开头的＃取消注释它们。 
    LoadModule proxy_module modules/mod_proxy.so
LoadModule proxy_http_module modules/mod_proxy_http.so
  3. 在底部httpd.conf文件，添加以下行以启用重新路由。 
    ProxyPass /console http://localhost:8080/console
ProxyPass /images http://localhost:8080/images
ProxyPassreverse / http://localhost:8080/

您可能需要根据应用程序的要求添加更多的ProxyPass指令。最后一个ProxyPassreverse伪指令捕获来自Geronimo服务器的响应并屏蔽URL，因为该URL将由Apache HTTPd直接响应，从而隐藏了Geronimo服务器的身份或位置。

在此示例中，仅出于演示目的而启用了控制台 。在生产环境中，您不希望从其他网络（通常是Internet）访问控制台 。具有控制台可访问性代表着很大的安全隐患。

规则是，所有内容都应具有受限的访问权限。通常，在HTTP和应用程序服务器之间放置一个防火墙（取决于拓扑），并且您应该仅映射使应用程序从另一端运行所需的最少资源。

如果HTTPd和Geronimo服务器都在同一台机器上，则可以使用localhost用于重定向。当服务器位于不同的机器上时，您将必须指定Geronimo服务器的URL。

作为此配置的结果，当您将浏览器指向http：// locahost / console时 ，请求将被重定向到http：// localhost：8080 / console 。此选项使您可以重新路由URL和端口。

注意：如果您更喜欢使用mod_proxy_ajp而不是mod_proxy_http ，则需要加载proxy_ajp_module在httpd.conf并将路由端口更改为8009（默认），如下所示：

LoadModule proxy_module modules/mod_proxy.so
LoadModule proxy_ajp_module modules/mod_proxy_ajp.so
...
ProxyPass /console ajp://localhost:8009/console
ProxyPass /images ajp://localhost:8009/images
ProxyPassreverse / ajp://localhost:8009/
