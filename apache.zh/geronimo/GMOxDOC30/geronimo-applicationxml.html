排队

企业应用程序的Geronimo部署计划通常打包为EAR文件，称为“ geronimo-application.xml ”。

geronimo-application.xml部署计划与application.xml JAVA EE部署计划结合使用，以部署由Web应用程序WAR，EJB JAR，客户端应用程序JAR，JCA连接器JAR组成的企业应用程序（s）和资源适配器模块RAR（s）到Geronimo应用程序服务器。geronimo-application.xml部署计划是一个可选文件，但通常在部署EAR文件时使用。它用于为已部署的模块，任何第三方依赖项，安全性选项和服务指定moduleId。

打包

geronimo-application.xml部署计划可以如下打包：

  1. 嵌入EAR文件中。在这种情况下，必须将geronimo-application.xml文件放置在EAR的/ META-INF目录中，该目录与application.xml文件必须位于同一位置。

  2. 与EAR文件分开维护。在这种情况下，必须将文件的路径提供给适当的Geronimo部署程序（例如，命令行或控制台）。请注意，在这种情况下，文件名可以使用geronimo-application.xml以外的其他名称，但必须遵循相同的架构。

架构图

geronimo-application.xml部署计划由位于Geronimo主安装目录的/ schema /子目录中的geronimo-application-2.0.xsd模式定义。此处简要描述了geronimo-application-2.0.xsd模式：

http://geronimo.apache.org/schemas-2.1/docs/geronimo-application-2.0.xsd.html
