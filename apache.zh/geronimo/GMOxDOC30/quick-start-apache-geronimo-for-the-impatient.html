快速入门-急躁的Apache Geronimo
编辑页面 浏览空间 添加页面 添加新闻

如果您急于启动并运行Apache Geronimo，那么本文可以向您展示在短时间内下载，构建（必要时）和运行服务器的基本步骤。

您只需要5到10分钟即可完成这些步骤。

取得软件

当前有以下两种方法来获取发行的软件：

  • 从Apache SVN信息库检索源代码。
  • 从Apache Geronimo网站下载二进制文件。

本节仅显示如何从Apache Geronimo网站下载二进制文件，因为与从Apache SVN信息库检索源代码相比，这种方式更快，更容易。

先决条件

在安装Geronimo之前，必须安装Java Runtime Environment（JRE）或Java Development Kit（JDK）。仅wsgen工具需要JDK。

安装Java SE 6

Apache Geronimo v3.0-M1是未经认证的里程碑发行版，其中包含一些Java EE 6功能并支持OSGi编程模型。有关如何下载和安装最新的Java SE 6 JRE或JDK的更多信息，请访问此处 。

Geronimo 3.0-M1是具有里程碑意义的发行版，仅应用于了解即将推出的带有Java EE 6和OSGi / Aries支持的Geronimo 3.0发行版，而不应用于任何类型的生产用途。

下载二进制文件

您可以访问http://geronimo.apache.org/downloads.html下载可用的软件包（二进制文件和源代码）。您必须根据计划在其上安装和运行Apache Geronimo的平台选择安装文件。

Geronimo v3.0-M1具有以下三种装配配置：

  • Web Container版本：Java EE 6 Web配置文件（未认证）
  • 最小（Little-G）：Web容器，事务和连接器功能
  • 框架：Geronimo内核核心功能

在本节中，我们使用带有Tomcat Web容器的发行版进行演示。有关这三种组件配置之间的区别的更多信息，请参见“ 获取和安装Geronimo” 。

您可以通过单击网站上的链接来下载与您的操作系统兼容的压缩格式（.zip或.tar.gz）的安装文件。要安装Apache Geronimo，请将二进制文件解压缩到硬盘上的新目录中，例如d：/ geronimo。在本节的其余部分，此目录称为 。

启动服务器

要启动Apache Geronimo：

  1. 在命令行界面中，转到/ bin目录。
  2. 输入以下命令：
    • ./geronimo run

该命令在命令行界面而不是在后台启动服务器。启动服务器后，您将看到以下信息。

部署和测试应用程序

在此示例中，我们不花时间在打包应用程序上。相反，我们使用--inPlace用于部署应用程序的部署工具的功能。您可以使用此功能从任何目录部署应用程序。

部署应用程序：

  1. 在命令行界面中，转到/ bin目录。
  2. 输入以下命令：
deploy --user system --password manager deploy --inPlace <app_home>

注意：作为一种快速的选择，此Hello World示例中的三个文件已打包到HelloWorld.war中并附加到此页面。部署它：

  1. 将HelloWorld.war下载到在您的系统上。
  2. 通过以下方式部署：
deploy --user system --password manager deploy <war_home>/HelloWorld.war

成功部署应用程序后，您会看到以下消息：

在启动服务器的命令行界面中，按CTRL + C停止服务器。
