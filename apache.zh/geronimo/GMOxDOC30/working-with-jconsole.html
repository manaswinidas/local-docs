使用JConsole

JConsole是兼容Java Management eXtension（JMX）的GUI工具，可用于连接到正在运行的Geronimo服务器实例。在JConsole中，您将能够监视JVM内存使用情况，线程堆栈跟踪，已加载的类和VM信息以及Geronimo MBean。

在JConsole中监视Geronimo

要使JConsole可以访问Geronimo，您可以简单地停用密码验证和安全套接字层（SSL）加密：

  1. 如果使用IBM JDK 6运行服务器，则必须添加以下Java选项以禁用密码认证和SSL连接。请注意，该命令应在一行上键入。
    • 在Windows系统上
    set JAVA_OPTS=-Dcom.sun.management.jmxremote.port=<port> -Dcom.sun.management.jmxremote.authenticate=false
-Dcom.sun.management.jmxremote.ssl=false
    • 在Linux系统上
    export JAVA_OPTS="-Dcom.sun.management.jmxremote.port=<port> -Dcom.sun.management.jmxremote.authenticate=false  
-Dcom.sun.management.jmxremote.ssl=false"
    哪里

    是一个免费端口，您要在该端口上启用此连接。


  2. 启动服务器。
  3. 然后，运行命令jconsole从\bin目录。
  4. 在“ JConsole的新建连接 ”对话框中，在“ 远程过程”字段中输入连接信息。
    • :用于监视本地服务器的整体信息，例如内存，线程和VM摘要。
    • service:jmx:rmi:///jndi/rmi://:1099/JMXConnector用于监视“ MBeans”选项卡中的Geronimo信息。在这种情况下， MBean以外的选项卡均处于非活动状态。
      哪里
      • 如果要监视本地服务器，则可以为localhost；如果监视远程计算机可以通过以下方式访问Geronimo服务器，则可以为主机名： ping命令。
      • 是您在Java选项中指定的端口号。
      • 1099是Geronimo的默认命名端口。
  5. 然后单击“ 确定” 。请注意，当您使用IBM SDK或Sun SDK时，信息可能会有所不同。

在启用安全JMX连接的情况下，在JConsole中监视Geronimo

安全套接字层（SSL）上的密码身份验证可用于确保未经授权的人员无法控制或监视您的服务器。在JConsole中连接到安全JMX服务器之前，必须执行一些配置步骤以禁用非安全JMX服务器并启动jmx-security模块。

  1. 启用Geronimo服务器的JMX安全模块 。
  2. 然后，打开命令窗口并设置环境变量。
    • 在Windows系统上
    set GERONIMO_HOME=<Geronimo_HOME>
    • 在Linux系统上
    export GERONIMO_HOME=<Geronimo_HOME>
    哪里是Geronimo服务器的安装目录
  3. 跑/bin/jconsole通过以下命令。请注意，该命令已分成几行以提高可读性。该命令必须在一行上键入。
    • 在Windows系统上
    jconsole \-J-D-Dorg.apache.geronimo.keyStoreTrustStorePasswordFile=%myDir%/KeystoreTrustSotrePasswordFile.key
    • 在Linux系统上
    jconsole \-J-D-Dorg.apache.geronimo.keyStoreTrustStorePasswordFile=$myDir/KeystoreTrustSotrePasswordFile.key
    哪里

    _myDir_是您信任的密钥库文件的位置。默认情况下，文件内容来自/var/config/config-substitutions.properties文件。有关更多详细信息，请参阅创建用于SSL身份验证的密钥库文件 。
  4. 在“ 连接到 JConsole的代理 ”对话框中，单击“ 高级” ，然后输入以下信息：
    • JMX URL ：service:jmx:rmi:///jndi/rmi://:1099/JMXSecureConnector
    • 用户名 ：系统（或您设置的用户名）
    • 密码 ：管理员（或您设置的密码）
      哪里
      • 如果要监视本地服务器，则可以为localhost；如果监视远程计算机可以通过以下方式访问Geronimo服务器，则可以为主机名： ping命令。
      • 1099是Geronimo的默认命名端口。
  5. 然后单击“ 确定” 。 JConsole将连接到Geronimo MBeans管理界面。切换到“ 均值”选项卡以获取服务器范围的信息。在这种情况下， MBean以外的选项卡均处于非活动状态。请注意，当您使用IBM SDK或Sun SDK时，信息可能会有所不同。
