Geronimo安装的主目录

  • / bin /

    存放用户实用程序脚本的目录

  • /部署/

    用于热部署的目录。该目录是由Geronimo插件创建的org.apache.geronimo.configs/hot-deployer//car加载插件时。默认， org.apache.geronimo.deployment.hot.DirectoryMonitor每2000毫秒扫描一次目录，以查找新的归档文件或展开的应用程序。您可以通过覆盖一个名为gbean的gbean来更改默认的热部署目录和轮询间隔org.apache.geronimo.deployment.hot.DirectoryHotDeployer如下：

    ...
<gbean gbeaninfo="org.apache.geronimo.deployment.hot.DirectoryHotDeployer"
       name="=org.apache.geronimo.configs/hot-deployer/2.2-SNAPSHOT/car?ServiceModule=org.apache.geronimo.configs/hot-deployer/2.2-SNAPSHOT/car,j2eeType=GBean,name=NewHotDeployer">
        <attribute name="path">NewHotDeploy/</attribute>
        <attribute name="pollIntervalMillis">6000</attribute>
        <reference name="ServerInfo">
                <pattern>
                    <groupId>org.apache.geronimo.framework</groupId>
                    <artifactId>j2ee-system</artifactId>
                    <version>2.2-SNAPSHOT</version>
                    <type>car</type>
                    <name>ServerInfo</name>
                </pattern>
         </reference>
         <reference name="ConfigManager"/>
    </gbean>
...
  • /等等/

    用于配置Gshell命令的目录

  • /农业/

    种植后用于保留应用程序的目录

  • / lib /

    该目录包含用于内核启动的仅框架库

  • /存储库/

    在本地保存工件和各种类型的依赖项的目录

  • /模式/

    包含XML模式文件的目录，该文件涵盖Geronimo部署计划以及通用元素和专用配置语法。查看了解部署计划以了解每种架构的用法。

  • / var /

    该目录包含Geronimo在其操作过程中向其读取/写入数据的文件
