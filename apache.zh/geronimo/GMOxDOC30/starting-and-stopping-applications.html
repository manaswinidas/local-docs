启动和停止应用程序
编辑页面 浏览空间 添加页面 新增新闻

有几种替代方法可用于启动和停止应用程序：

  • 通过Geronimo管理控制台
  • 通过部署工具
  • 通过GShell

通过Geronimo管理控制台

作为命令行选项的替代，您可以使用Geronimo管理控制台来更改模块的状态。打开Geronimo管理控制台，然后导航到左侧控制台导航菜单上的应用程序 。根据要更改其状态的应用程序类型，选择Application EAR或Web App WAR 。在此示例中，我们使用HelloWorld.war作为示例应用程序，因此我们将选择Web App WARs 。

我们在“ 部署和取消部署应用程序”部分中使用了此portlet，以通过使用Geronimo管理控制台来安装和删除应用程序。您也可以从同一portlet更改应用程序的状态。Portlet中的可用命令根据每个应用程序的状态而变化。如果应用程序处于运行状态（已启动），那么将显示“ 停止”和“ 重新启动 ”命令。如果应用程序已停止 ，则仅显示启动命令。始终显示“ 卸载”命令，而与应用程序的状态无关。要更改应用程序的状态，请单击命令（启动或停止）。

通过部署工具

Deployer工具具有许多命令。在“ 部署和取消部署应用程序”部分中，我们主要研究了部署和取消部署 ，以及非常简短的列表模块 。本节重点介绍list-modules命令，以检索要更改其状态的模块的名称。本节还介绍了两个新命令start和stop 。

list-module命令接受以下树参数：

  • --all ：默认情况下，未指定其他选项时使用。它列出了所有可用的模块。
  • --started ：此选项仅列出正在运行的模块。
  • --stopped ：此选项仅列出未运行的模块。

您也可以在命令末尾指定目标 。很少使用此选项，因为在大多数情况下，每台服务器上仅定义一个配置存储。仅列出指定目标上的模块。如果未指定目标，则列出所有目标上的模块。

此list-module命令对于标识特定模块的状态和模块ID本身很有用。您可以使用此命令来标识HelloWorld应用程序并更改其状态。

在命令行窗口中，从\bin目录列出所有模块：

deploy --user system --password manager list-modules

您将收到服务器上安装的所有模块的列表。请注意，已启动的模块由左侧的+号区分，并首先显示在列表中。

找出 ” + sample.applications/HelloWorldApp/2.0/war @ http://hcunico:8080/hello ”模块。该模块的状态为start ，因为左侧有一个+ 。或者，您可以运行以下命令以仅列出已启动的模块：

deploy --user system --password manager list-modules --started

要停止“ sample.applications/HelloWorldApp/2.0/war模块，键入以下命令：

deploy --user system --password manager stop sample.applications/HelloWorldApp/2.0/war

您将收到与以下消息相似的确认消息，表明模块已停止。

通过GShell

您还可以使用GShell deploy / start命令来管理应用程序的状态。同样， 部署/停止可用于停止正在运行的应用程序，而部署/重新启动可用于重新启动已启动或已停止的应用程序。有关这些命令及其选项的更多信息，请参见Geronimo GShell命令 。
