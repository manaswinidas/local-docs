使用GEP部署和调试应用程序

排队

Geronimo Eclipse插件是用于将应用程序部署和测试到Geronimo服务器的工具。

在尝试部署应用程序之前，请确保已定义了Geronimo服务器。

在Eclipse集成开发环境（IDE）中创建或导入Java EE项目时，请将先前定义的Geronimo服务器运行时指定为目标运行时。该操作将服务器的类库添加到项目的构建路径。

注意

如果使用Eclipse部署资产，建议您也使用Eclipse取消部署和重新部署应用程序。例如，如果您在Eclipse中部署应用程序，然后使用管理控制台或控制台取消部署该应用程序deploy命令，Eclipse将不会检测到更改，并且视图将显示资产仍在部署中。如果您遇到这种情况，则可以通过除去已发布到服务器但在Eclipse框架之外未部署的应用程序来更正视图。

本地部署

使用此过程将应用程序部署到本地服务器。

  1. 在Java EE透视图中，选择“ 项目”视图，然后右键单击要部署的Java EE项目。选择运行方式，在服务器上运行
  2. 在“在服务器上运行”面板上，如果有现有服务器，请保留“ 选择现有服务器”选项并选择服务器。如果未定义Geroimo服务器，请选择“ 手动定义新服务器”选项，然后选择服务器。
  3. 点击完成 。Geronimo Eclipse插件将很快部署应用程序。如果未启动服务器，则在服务器初始化后，Geronimo Eclipse插件将启动服务器并部署Java EE应用程序。

删除已发布到服务器的资产

将资产发布到服务器后，如果要从服务器中将其删除，则必须使用该工具的“ 添加/删除项目”选项。如果仅删除资产而不删除其项目，则资产将保持部署在服务器上。

  1. 在Java EE透视图中，选择“ 服务器”视图。
  2. 在“ 服务器”面板上，右键单击部署资产的服务器。
  3. 在出现的上下文菜单上，单击“ 添加/删除项目” 。
  4. 在选择面板上，单击要取消部署的资产，然后单击< /strong="">按钮将其从右移到左列表。点击完成 。<>

远程部署

您可以使用上面讨论的相同步骤将Java EE资产部署到远程服务器，但是还有其他注意事项。

  1. 确保可以通过ping命令联系目标系统。
  2. 如果系统和目标服务器之间存在防火墙，则必须将其配置为允许HTTP和RMI请求都在系统和服务器主机之间流动。安装服务器后，初始HTTP端口为8080 ，初始RMI端口为1099 。如果目标服务器已配置为使用其他端口，则必须将防火墙配置为使用这些端口。
  3. 在目标服务器上，找到RemoteDeployHostname归因于\var\config\config-substitutions.properties ，然后将该值更改为目标服务器主机的名称或IP地址。
  4. 定义本地服务器，然后将Eclipse中的主机名更改为远程服务器的主机名。这是必需的，因为Eclipse框架必须使用本地服务器中的类库。您将无法使用Eclipse启动，停止或重新启动远程服务器。您不能使用Eclipse在调试模式下启动远程服务器。通常，此限制使使用本地服务器开发和调试Java EE资产，然后在需要通用服务器将资产与其他开发人员的资产集成时切换到远程服务器更加方便。
  5. 调用deploy命令时，远程服务器必须正在运行。当您部署或刷新Java EE资产时，这些文件将在网络上复制，另存为临时文件，然后进行部署。

在已经运行的服务器上调试远程应用程序

在正在运行的服务器上调试远程应用程序之前，必须使用geronimo.sh/bat在jpda调试器下启动服务器：

geronimo jpda运行

要么

geronimo jpda开始

使用此过程将远程应用程序（在此示例中为WAR文件）附加到以调试模式启动的已经运行的服务器。

  1. 在Eclipse菜单栏上，单击File and Import 。在“ 导入”面板上，展开“ Web” ，选择“ WAR文件” ，然后单击“ 下一步” 。
  2. 指定WAR文件目录和目标运行时。单击下一步 。
  3. 如果需要，导入Web库。点击完成 。
  4. 左键单击主工具栏上调试图标后面的向下箭头，然后选择“ 调试配置...” 。
  5. 选择“ 远程Java应用程序” ，然后单击“新建”按钮以创建新的调试配置。
  6. 填写有关服务器的信息：
    • 名称：根据需要
    • 项目：要在服务器上调试的WAR文件
    • 主机：服务器的主机名
    • 端口：5005（如果更改了服务器的调试端口，则为适当的值。）
  7. 点击调试 。
