Eclipse故障排除

在Eclipse错误日志视图中查找错误

如果Eclipse报告了任何错误，则可以在Eclipse的“ 错误日志”视图中查看有关该错误的详细信息。发生错误时，只有Eclipse Galileo会将这个视图放置在其他视图的前面。在Eclipse Europa或Ganymede中，您必须执行以下步骤才能找到此视图：

  1. 单击窗口 ，然后单击显示视图->其他 。
  2. 展开常规 ，然后单击错误日志 。
  3. 单击确定 。将显示“错误日志”视图。

对Java EE资产部署进行故障排除

Eclipse 3.5上的应用程序部署计划的共存并导致错误

如果在此部署计划中同时使用和元素geronimo-application.xml对于您的EAR，Eclipse 3.5报告一个错误。这是因为Eclipse 3.5中的XML编辑器无法识别命名空间中的多个别名。但是，即使Eclipse 3.5报告计划中的错误，该应用程序仍可以成功地部署在服务器上。有关此问题的更多信息，请参见Eclipse错误 。
为了防止Eclipse报告此类错误，请遵循以下步骤：

  1. 单击窗口->首选项 。
  2. 展开XML ，然后展开XML文件 。
  3. 选择验证 。取消选择“ 兑现所有XML模式位置” 。
  4. 点击应用 。
  5. 关闭部署计划，然后重新打开它。Eclipse不会再报告这种类型的错误。

使用注释时找不到托管的Bean异常

如果您使用过，可能会发生这种情况@ManagedBean在您的应用程序中添加注解，但根据JSF 2.0规范，只有那些包含faces-config.xml在其/META-INF文件夹将被扫描。您需要手动添加faces-config.xml文件在/META-INF您的应用程序中是否有jar文件/WEB-INF/lib/ jar文件中具有@ManagedBean批注定义的目录。
