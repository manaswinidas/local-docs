管理OSGi捆绑包
编辑页面 浏览空间 添加页面 添加新闻

您可以使用OSGI Bundles portlet来安装和管理软件包。在管理控制台上，从左侧导航面板的“ 资源”文件夹中选择“ OSGI Bundles” portlet，以访问该portlet。

本文档分为以下几节：

  • 安装OSGi套件
  • 管理OSGi捆绑包的状态
  • 查看不同组的捆绑包
  • 查看OSGi捆绑包
  • 查看OSGi捆绑软件的信息

安装OSGi套件

您可以使用管理控制台上的OSGI Bundles portlet通过以下步骤安装OSGi Bundle：

  1. 单击浏览以指定要安装的捆绑软件的位置。
  2. （可选）在“ 开始级别”字段中输入一个非负整数，以设置捆绑软件的开始级别。默认情况下，捆绑包的开始级别设置为60。
  3. （可选）如果要在安装后启动捆绑软件，请选中“ 启动”复选框。
  4. 点击安装 。

每个OSGi捆绑包都与一个启动级别相关联，该启动级别使服务器能够控制捆绑包的相对启动和停止顺序。只有启动级别小于或等于服务器框架的活动启动级别的捆绑软件才必须处于活动状态。通常，启动级别较小的捆绑包往往会更早启动。

管理OSGi捆绑包的状态

OSGI Bundles portlet在安装区域下的表中按不同组显示已安装的OSGi Bundle及其状态。您可以启动，停止和卸载此表中列出的捆绑软件。

该表包括以下部分：

  1. ID ：此列显示每个捆绑包唯一的捆绑包ID。表中的捆绑包是根据其ID的升序排序的。此列出顺序符合捆绑商品的开始顺序。起始级别较小的束往往具有较小的ID号。
  2. 符号名称 ：此列显示每个捆绑软件的符号名称。该属性由捆绑软件中MANIFEST.MF文件中的Bundle-SymbolicName标头设置。
  3. 版本 ：此列显示每个捆绑软件的版本，该版本由捆绑软件中MANIFEST.MF文件中的Bundle-Version标头设置。
  4. 状态 ：此列显示每个捆绑软件的状态。
    • 启动的捆绑包显示为Active 。您可以使用“ 停止”按钮停止分发包。
    • 停止的捆绑包显示为已解决 。您可以使用“ 开始”按钮启动捆绑软件。
  5. 实用程序 ：此列显示您可以执行的操作，以查看每个捆绑软件的不同信息。

查看不同组的捆绑包

默认情况下，打开OSGi portlet时，将列出所有已安装的捆绑包。但是，您可以选择仅显示特定的捆绑包组：

  • 选择WAB仅列出Web捆绑包；
  • 选择“ 蓝图”仅列出蓝图包；
  • 选择系统仅列出系统捆绑，系统捆绑的开始级别必须小于50；
  • 选择配置以列出Geronimo系统模块使用的所有配置包；

您还可以通过在Portlet上使用按符号名搜索功能来根据包的符号名过滤包。

查看OSGi捆绑包

您可以使用“ 查找软件包”来查看特定软件包及其有线束的信息。输入框支持通配符。

单击软件包以查看导出软件包和导入软件包的捆绑软件。

查看OSGi捆绑软件的信息

以下是“ 实用程序”列中列出的操作，您可以执行这些操作来查看每个捆绑软件的信息：

  • 单击查看清单以查看捆绑软件的MANIFEST.MF文件的内容。该文件包含标头，这些标头指定服务器框架正确安装和激活捆绑软件所需的信息。此类信息包括：导入包，导出包，Bundle-SymbolicName，Bundle-Version等。
  • 单击查看有线捆绑软件，以查看那些导入的软件包来自哪些捆绑软件，以及哪些捆绑软件正在使用当前捆绑软件的导出软件包。
  • 单击查看服务以查看捆绑软件正在使用的服务以及捆绑软件已在Framework中注册的服务。

有关OSGi捆绑包的更多信息，请参见OSGi规格 。
