配置DB2数据源
编辑页面 浏览空间 添加页面 新增新闻

本文向您展示了如何在Apache Geronimo中配置DB2数据源。

在这种情况下，您将为DB2数据库创建一个连接池，以连接到该数据库，您需要定义至少两个驱动程序jar文件。它们包括JDBC驱动程序本身和相应的许可证文件。根据您的实现，您将需要不同的许可证文件。

当您没有在服务器上安装驱动程序，需要多个驱动程序文件或两者都安装时，本文中介绍的步骤对于部署任何其他数据源均有效。

本文为部署连接池提供了两种选择：使用Geronimo管理控制台和使用命令行选项。

将DB2驱动程序添加到存储库

要将DB2驱动程序和许可证添加到Geronimo存储库，需要将文件放在特定的目录结构中。通常，您将有两个或三个文件添加到存储库。这些文件是：

  • db2jcc.jar-这是实际的DB2通用JDBC驱动程序jar文件。
  • db2jcc_license_cu.jar-这是标准的DB2通用JDBC驱动程序许可证文件，允许访问Linux，UNIX和Windows服务器的DB2通用数据库。
  • db2jcc_license_cisuz.jar-这是用于z / OS和iSeries的DB2（DB2 ESE和DB2 Connect）JDBC驱动程序许可证，除了这些服务器的标准许可证之外，还应该使用。

这些文件位于\java目录。有关DB2 JDBC驱动程序和许可证的其他信息，请访问位于以下URL的DB2信息中心：

http://publib.boulder.ibm.com/infocenter/db2luw/v9/index.jsp?topic=/com.ibm.db2.udb.doc/ad/t0010264.htm

要在Geronimo中使用这些文件，请根据下表重命名（复制和重命名）文件。在这种情况下，您将不使用db2jcc_license_cisuz.jar，因为它用于Z / OS和iSeries。

原名

更名

db2jcc.jar

db2jcc-9.5.jar

db2jcc_license_cu.jar

db2jcc_license_cu-9.5.jar

这意味着对于这种特殊情况，将使用DB2 v9.5。

此时，您已经准备好以两种方式将这些文件添加到Geronimo存储库中。您可以使用任何图形或命令行工具来复制文件并创建必要的目录，也可以使用Geronimo管理控制台并将驱动程序和许可证添加到公共库中。请参阅以下详细信息。

使用命令行

在以下目录下创建以下目录结构\repository目录，然后将适当的文件复制到相应目录。

  • com/ibm/db2/db2jcc/9.5
    并将db2jcc-9.5.jar文件复制到该目录中。
  • com/ibm/db2/db2jcc_license_cu/9.5
    并将db2jcc_license_cisuz-9.5.jar文件复制到该目录中。

使用Geronimo管理控制台

要使用管理控制台，Apache Geronimo必须正在运行。通过将浏览器指向以下URL来访问Geronimo管理控制台：

http：// localhost：8080 / console

  1. 输入system作为用户名，输入manager作为密码，然后单击Login 。
  2. 单击服务->存储库 portlet。
  3. 单击浏览，然后选择要安装的第一个文件。在这种情况下，您可以先安装db2jcc-9.5.jar 。
  4. 默认情况下将建议一组值，将Group：设置为com.ibm.db2 ，其余设置为默认，然后单击Install 。
  5. 对db2jcc_license_cisuz-9.5.jar重复前面的两个步骤。

安装了驱动程序和许可证文件后，您现在可以创建一个新的数据库连接池。

使用Geronimo管理控制台中的向导创建数据库连接池

在Geronimo管理控制台中，选择“ 数据库池”，然后通过单击“ 使用Geronimo数据库池向导”创建一个新池。

  1. 在数据库池向导中输入池名称DB2_ds和数据库类型DB2（DataDirect） ，然后单击Next 。请参见下图中的插图。


  2. 选择com/ibm/db2/db2jcc/9.5/jar和com.ibm/db2/db2jcc_license_cu/9.5/jar通过按键盘上的Ctrl键从列表中选择Driver JAR 。然后输入剩余的连接信息，如下图所示。对于此示例，使用缺省的db2admin用户名和密码，并通过DB2控制中心创建一个SAMPLE数据库。组localhost作为服务器名称 。


  3. 点击部署 。您将在数据库池 portlet中列出DB2_ds 。


使用命令行部署数据库连接池

作为向导的替代方法，您可以手动创建部署计划，然后使用基于命令行的部署程序工具进行部署。要使用此选项，请创建db2-plan.xml文件，并将以下示例的内容复制到该文件中。

db2-plan.xml部署计划
<?xml version="1.0" encoding="UTF-8"?>
<connector xmlns="http://geronimo.apache.org/xml/ns/j2ee/connector-1.2">
    <dep:environment xmlns:dep="http://geronimo.apache.org/xml/ns/deployment-1.2">
        <dep:moduleId>
            <dep:groupId>console.dbpool</dep:groupId>
            <dep:artifactId>DB2_ds</dep:artifactId>
            <dep:version>1.0</dep:version>
            <dep:type>car</dep:type>
        </dep:moduleId>
        <dep:dependencies>
            <dep:dependency>
                <dep:groupId>com.ibm.db2</dep:groupId>
                <dep:artifactId>db2jcc</dep:artifactId>
                <dep:version>9.5</dep:version>
                <dep:type>jar</dep:type>
            </dep:dependency>
            <dep:dependency>
                <dep:groupId>com.ibm.db2</dep:groupId>
                <dep:artifactId>db2jcc_license_cu</dep:artifactId>
                <dep:version>9.5</dep:version>
                <dep:type>jar</dep:type>
            </dep:dependency>
        </dep:dependencies>
    </dep:environment>
    <resourceadapter>
        <outbound-resourceadapter>
            <connection-definition>
                <connectionfactory-interface>javax.sql.DataSource</connectionfactory-interface>
                <connectiondefinition-instance>
                    <name>DB2_ds</name>
                    <config-property-setting name="TraceFile"/>
                    <config-property-setting name="UserName">db2admin</config-property-setting>
                    <config-property-setting name="DatabaseName">sample</config-property-setting>
                    <config-property-setting name="Password">db2admin</config-property-setting>
                    <config-property-setting name="ServerName">localhost</config-property-setting>
                    <connectionmanager>
                        <local-transaction/>
                        <single-pool>
                            <max-size>10</max-size>
                            <min-size>0</min-size>
                            <match-one/>
                        </single-pool>
                    </connectionmanager>
                </connectiondefinition-instance>
            </connection-definition>
        </outbound-resourceadapter>
    </resourceadapter>
</connector>

在里面部分，您可以找到moduleId标识正在部署的资源或组件；在管理控制台中moduleId在“ 数据库池” Portlet的“ 组件名称”列中显示。

之后moduleId是依赖项的定义。在这种情况下，您可以找到三个与DB2 JDBC驱动程序和许可证jar有关的块。此计划中的最后一个“大”块是在其中定义了连接参数，例如驱动程序，用户和密码，连接URL等。

部署数据源

要部署刚刚创建的DB2数据源，请从\ bin目录运行以下命令。

deploy --user system --password manager deploy \db2-plan.xml ..\repository\org\tranql\tranql-connector-db2-xa\1.4\tranql-connector-db2-xa-1.4.rar
