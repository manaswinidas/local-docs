总览

排队

Geronimo使用OpenEJB容器提供EJB服务。随着Java EE的出现，可以在EJB类本身中使用注释来声明EJB容器服务，例如事务管理，安全性，生命周期管理。但是，仍然可以通过使用ejb-jar.xml文件来提供EJB部署描述符。

同时提供注释和ejb-jar.xml文件时， ejb-jar.xml文件优先于注释。

EJB应用程序特定于Geronimo的部署计划（通常打包为EJB JAR文件）被称为“ openejb-jar.xml ”。openejb-jar.xml部署计划与ejb-jar.xml Java EE部署计划结合使用，以将企业应用程序部署到Geronimo应用程序服务器。openejb-jar.xml部署计划是一个可选文件，但通常在部署EJB JAR文件时使用。它用于将在openejb-jar.xml部署计划中声明的角色和资源（例如，安全角色，EJB名称，数据库资源，JMS资源等）映射到服务器中部署的相应实体。另外，如果需要任何特定于EJB容器的配置，那么这些设置也会在此部署计划中进行配置。如果EJB模块依赖于服务器中运行的任何第三方库或其他服务，则所有这些第三方库和服务都在openejb-jar.xml文件中指定。某些EJB应用程序要求的类加载要求不同于默认的类加载行为。openejb-jar.xml文件允许应用程序部署者也进行配置。还有许多其他配置可以通过openejb-jar.xml文件取决于EJB应用程序的需求。

打包

可以将Geronimo特定的openejb-jar.xml部署计划打包如下：

  1. 嵌入到EJB JAR文件中。在这种情况下，必须将openejb-jar.xml文件放在JAR的/ META-INF目录中，该目录与ejb-jar.xml文件所在的位置相同。

  2. 与EJB JAR文件分开维护。在这种情况下，在部署EJB JAR文件时，必须将文件的路径提供给适当的Geronimo部署程序（例如，命令行或控制台）。请注意，在这种情况下，文件名可以使用openejb-jar.xml以外的名称，但必须遵循相同的架构。还要注意，如果将EJB JAR文件嵌入到企业应用程序EAR文件中，这将不起作用（请参见下文）。

  3. 嵌入到企业应用程序EAR文件中：在一种情况下，高级元素可以嵌入EAR文件的geronimo-application.xml文件中的EJB JAR文件之外

  4. 嵌入到企业应用程序EAR文件中：在另一种情况下，可以将实际的openejb-jar.xml文件嵌入EJB JAR文件之外，并放置在EAR中的其他位置，并在geronimo-application.xml部署中用alt-dd元素引用计划文件。

EJB实体Bean

EJB实体Bean设置可以包括远程客户端使用的JNDI名称，CMP设置以及CORBA配置和解析引用。

EJB会话Bean
