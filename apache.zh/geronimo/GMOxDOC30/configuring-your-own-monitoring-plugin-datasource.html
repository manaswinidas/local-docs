可以捕获服务器的基本信息并将其存储在Geronimo支持的数据库中。Geronimo管理节点利用MBean收集给定时间（也称为快照）的服务器统计信息。这些快照均与JSR-77兼容。这样，管理员可以监视服务器状态。

默认情况下，数据源用于连接Apache Derby数据库（其原因是因为Geronimo使用的默认数据库也是Derby）。但是，由于数据源都打包到单独的插件中，因此任何人都可以编写自己的数据源部署描述符，将其打包到插件中，然后使用它代替默认提供的插件。

为了对监控插件使用自己的数据库，必须确保以下各项：

  • 活动数据库的数据源必须命名为jdbc / ActiveDS
  • 存档数据库的数据源必须命名为jdbc / ArchiveDS
  • 模块ID必须是org.apache.geronimo.plugins.monitoring / agent-ds // car，因为插件中对此模块ID有依赖性

原始数据源的部署描述符

这是用于监控插件的数据源的原始部署描述符。 \repository\org\apache\geronimo\plugins\monitoring\agent-ds\\agent-ds-.car\META-INF 。

META-INF/database/derby是的道路createTables.sql包装在JAR文件中\repository\org\apache\geronimo\plugins\monitoring\agent-sql\\agent-sql-.jar ，这是数据源的依赖项。

部署描述符末尾的两个gbean指向一个SQL文件createTables.sql为了在它们不存在的条件下创建必要的表。这两个表的元素描述如下：

  • 豆类
    • id （主键）：MBean的ID。
    • mbeanName ：MBean的名称。
    • statsNameList ：用逗号分隔的字符串，代表MBean的统计名称。
  • 统计
    • id （主键）：统计信息的ID。
    • mbeanId （外键）：MBean的ID。
    • snapshot_time ：两次快照之间的时间间隔。
    • statsValueList ：用逗号分隔的字符串，表示MBean的统计值。

一个例子

在此示例中，我们将用DB2 Express-C替换Geronimo 2.2中的缺省Apache Derby插件数据源。

先决条件

在开始替换监视插件数据源之前，我们应该在Geronimo上安装DB2 Express-C数据库及其驱动程序JAR包，然后为Geronimo配置DB2 Express-C数据源以连接到该数据库。

  1. 下载并安装DB2 Express-C。有关下载和安装此数据库的详细信息，请参见DB2 Express-C 。
  2. 在Geronimo上安装驱动程序JAR软件包。有关详细步骤，请参见将DB2驱动程序添加到存储库 。
  3. 为DB2 Express-C部署数据源。有关详细信息，请参见Geronimo管理控制台中的使用向导创建数据库连接池 。

替换默认的监视插件数据源

现在我们有了一个DB2监视插件数据源，我们必须为该数据源创建部署描述符。

  1. 创建部署计划db2-agent-ds.xml根据默认plan.xml 。 xml db2-agent-ds.xml org.apache.geronimo.plugins.monitoring db2-agent-ds 2.2 car org.apache.geronimo.plugins.monitoring agent-sql 2.2 jar com.ibm.db2 db2jcc 9.5.0 jar com。 ibm.db2 db2jcc_license_cu 9.5.0 jar javax.sql。数据源 jdbc / ActiveDS db2admin db2manager 50000 localhost activedb 4 10 0 javax.sql。数据源 jdbc / ArchiveDS db2admin db2manager 50000 本地主机 archdb 4 10 0 从SYS.SYSTABLES中选择t.tablename下级（t.tablename）='statistics'META -INF / database / derby / createTables.sql jdbc / ActiveDS从SYS.SYSTABLES中选择t.tablename下级（t.tablename ）='statistics'META -INF / database / derby / createTables.sql jdbc / ArchiveDS Monitoring-runas-realm false monitoring-runas-domain org.apache.geronimo.security.credentialstore。RunAsLoginModule org.apache.geronimo.security.realm.providers。GeronimoGroupPrincipal 管理员 您必须对db2-agent-ds.xml ：
    • 使用DB2 Express-C所需的驱动程序更新部分。
    • 更改数据源的用户名和密码，并添加ServerName和portnumber设置。
    • jdbc / ActiveDS的DatabaseName是activedb ，并且jdbc / ArchiveDS的DatabaseName为archdb 。
  2. 同样，创建db2-mconsole-ds.xml 。该文件中的DatabaseName是mcdb 。
  3. 在DB2 Express-C中创建相应的数据库： activedb ， archdb和mcdb 。
  4. 更改var\config\config.xml Geronimo，以防止在启动时启动以下模块。这可以通过添加一个Load="false"每个模块。这将防止以下配置更改引起冲突。
    • org.apache.geronimo.plugins.monitoring / agent-ds / 2.2 / car
    • org.apache.geronimo.plugins.monitoring / agent / 2.2 / car
    • org.apache.geronimo.plugins.monitoring / mconsole-ds / 2.2 / car
    • org.apache.geronimo.plugins.monitoring / mconsole-tomcat / 2.2 / car
  5. 启动服务器并使用deploy命令部署计划。 deploy.bat部署.. \ repository \ org \ tranql \ tranql-connector-db2-xa \ 1.3 \ tranql-connector-db2-xa-1.3.rar ：\ db2-agent-ds.xml deploy.bat部署.. \ repository \ org \ tranql \ tranql-connector-db2-xa \ 1.3 \ tranql-connector-db2-xa-1.3.rar ：\ db2-mconsole-ds.xml其中是放置计划的目录。
  6. 关闭服务器并进行更改var\config\artifact_aliases.properties文件替换默认模块。这样，Geronimo将在启动时加载DB2数据源，而不是缺省的。 org.apache.geronimo.plugins.monitoring / agent-ds / 2.2 / car = org.apache.geronimo.plugins.monitoring / db2-agent-ds / 2.2 / car org.apache.geronimo.plugins.monitoring / mconsole-ds /2.2/car=org.apache.geronimo.plugins.monitoring/db2-mconsole-ds/2.2/car
  7. 在var\config\config.xml ，负载org.apache.geronimo.plugins.monitoring/agent/2.2/car和org.apache.geronimo.plugins.monitoring/mconsole-tomcat/2.2/car在启动时。这可以通过删除Load="false"每个模块。
