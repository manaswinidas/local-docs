服务器使用Apache Maven项目提示的相同目录结构和命名约定，将公共Java库保留在存储库中。如果您的Java EE资产依赖于存储库中尚未存在的库，则可以通过管理控制台或使用deployer命令更新存储库以包括新库，并在Java EE资产的部署计划中定义依赖项。

本主题介绍如何使用管理控制台将Java库添加到Geronimo存储库。

使用新库更新存储库

要将档案或jar添加到存储库，可以通过在左侧的“ 控制台导航”菜单上选择“ 服务器”>“存储库”来使用“存储库查看器” Portlet。下图所示的Repository Viewer portlet显示了安装在服务器存储库中的工件。存储库的布局与Apache Maven使用的布局相同，这使得可以轻松地复制文件。

这是“ 资源库查看器” Portlet上每个字段的含义：

  • 文件是存档的实际位置。
  • group是一个组标识符，通常是开源项目的名称（例如commons-logging或log4j ），或者是与提供该库的组织的Java包前缀匹配的目录树（例如org / apache / derby ） 。
  • 工件是库的文件名前缀。
  • 版本是文件中库的版本标识符，从而使存储库可以毫无歧义地保存同一库的多个版本。
  • 类型是文件的类型，通常是jar 。

在部署计划中定义依赖项

为了避免尝试在每个jvm生命周期内多次加载同一本机库的问题，您需要将用于加载本机库（例如许多db驱动程序）的代码放入一个classloader插件中，该插件可以在需要的所有数据源插件之间共享类。

要在应用程序中使用工件，请以列表中的第一个工件为例，您需要添加一个下的元素在里面应用程序的部署计划中的元素。以下是部署计划的摘要：
