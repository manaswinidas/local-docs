从Eclipse构建Geronimo
编辑页面 浏览空间 添加页面 添加新闻

该文档是关于如何构建Geronimo eclipse项目的。为了便于讨论，开发操作系统是Windows。请根据您的环境调整这些说明。

先决条件

在开始之前，您必须安装以下软件：

  1. 按照“安装Eclipse”中的指南安装Eclipse 。
  2. 安装Maven 2.2.1或更高版本。
  3. 安装Subversion 。
  4. 安装适用于Eclipse的Maven插件 。您可以从Maven Integration for Eclipse更新站点安装此插件。

创建Eclipse项目

您可以导入工作空间中的所有Geronimo项目或子项目。但是，一种更有效的方法是仅导入要使用的模块。

使用多个模块

建议您首次在Eclipse中使用Geronimo构建时导入所有模块。

通过从Geronimo 顶层目录调用以下命令，为Geronimo子项目生成Eclipse项目：

mvn -o eclipse:eclipse

注意：

第一次，您必须忽略“ -o”参数，以便它将下载maven-eclipse-plugin。

mvn eclipse:eclipse

通过从每个模块目录运行上述命令，可以仅导入工作空间中的几个模块。在这种情况下，如果不使用mvn重建修改后的模块，则一个模块中所做的更改将对其他模块不可见。

有用的信息

可以编辑.classpath文件，以便一个模块中所做的更改对工作区中的所有其他模块立即可见。这仅适用于高级用户。

使用单个模块

如果计划修改单个geronimo模块，则可以通过运行以下命令将其仅导入工作空间中：

cd modules\a_module  or  cd maven-plugins\a_plugin
mvn -o eclipse:eclipse

启动Eclipse

在为Geronimo创建Eclipse项目之后，是时候将这些项目导入Eclipse工作区了。

  1. 单击文件 > 导入 > 常规 > 现有项目到工作区 。
  2. 在#Creating Eclipse Projects步骤中，根据您如何创建项目来选择一个或多个项目 。
  3. 单击窗口>首选项> Java>构建路径>类路径变量 ，然后单击新建按钮。
  4. 输入M2_REPO并将其设置到您的maven2存储库目录。在Linux上，该目录通常位于
    ~/.m2/repository
    在窗户上通常
    "%USERPROFILE%\.m2\repository"

注意 ：例如，如果您正在使用构建器模块。 geronimo-tomcat6-builder，必须将xmlbeans类添加到构建路径。

  1. 右键单击构建器项目，然后选择“ 构建路径”>“配置构建路径”>“ Java Build PAth”>“添加类文件夹” 。
  2. 从选择中检查target / clover / classes文件夹。
  3. 为了防止Eclipse将.svn目录复制到项目的输出文件夹，请单击窗口>首选项> Java>编译器>建筑>输出文件夹 ，然后追加.svn 到*过滤资源字段。

在Eclipse中构建Geronimo源代码

您可以通过右键单击项目并选择mvn选项来构建每个Geronimo模块。

信息：

  • 如果计划为项目贡献代码或补丁，则必须将eclipse配置为不使用选项卡。选择窗口>首选项> Java>代码样式>格式化程序> Java约定[built-in] ，然后点击编辑 。在“缩进”下，将“制表符”策略选择为“仅空格”。输入此配置文件的新名称，然后单击“ 确定” 。
  • 您可能要通过导入Geronimo格式化程序和代码模板文件来配置Geronimo开发环境。
    • 单击窗口>首选项> Java>代码样式>格式器>导入 ，然后指定Geronimo-template-formatter.xml的本地路径。
    • 单击窗口>首选项> Java>代码样式>代码模板>导入 ，然后指定Geronimo-template-code.xml的路径。
