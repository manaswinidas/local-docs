配置多个存储库
编辑页面 浏览空间 添加页面 新增新闻
{scrollbar}

关于Geronimo javaee6和最低发行版捆绑包。GERONIMO-6270及其子任务完成了对多个存储库的完全支持。此支持在2012年3月1日之后的最新快照中提供，并将在3.0-beta-2标准版本中提供。本文档介绍了在应用GERONIMO-6270中的更改后如何设置多个存储库。

对于以前的版本，如果org.ops4j.pax.url.mvn.defaultRepositories属性设置正确，如本文档所述。

排队

具有多个存储库的优点是它们可以驻留在不同的服务器实例上。

您可以在不同的服务器实例上创建多个存储库，在单个服务器实例上部署一个存储库，并在单个服务器实例上的多个存储库上部署应用程序。

为了在单个Geronimo安装中支持多个实例，建议每个实例都有自己的本地存储库以部署到（例如GERONIMO_SERVER/repository ）。主共享存储库（ GERONIMO_HOME/repository ）在多实例配置中应保持只读状态。

本主题包括以下信息：

在单个服务器实例上创建多个存储库

您可以在Geronimo上创建多个存储库。您可以添加第二个存储库来部署应用程序，并将Geronimo保留在其默认存储库中。

在默认存储库之外创建存储库

要在默认存储库之外添加第二个存储库，请执行以下步骤：

  1. 启动Geronimo。
  2. 在下面创建目录repo2 ， 例如， /repo2 。
  3. 建立档案repo2.xml在下面/repo2目录，例如： solid repo2.xml org.example.configs myrepo 2.2 car org.apache.geronimo.framework j2ee-system car repo2 / false ServerInfo Repo2
  4. 部署repo2.xml在命令提示符下：
    deploy(.bat) deploy /repo2/repo2.xml
    。
    要验证存储库是否已成功部署，可以使用以下命令列出当前服务器上的目标：
    deploy(.bat) list-targets ，您会看到以下信息： 可用目标：org.apache.geronimo.framework / j2ee-system / 2.2 / car？ServiceModule = org.apache.geronimo.framework / j2ee-system / 2.2 / car，j2eeType = ConfigurationStore，name = Local org.example.configs / myrepo / 2.2 / car？ServiceModule = org.example.configs / myrepo / 2.2 / car，j2eeType = ConfigurationStore，name = Local2
  5. 更新etc/org.ops4j.pax.url.mvn.cfg文件，将新的第二个存储库添加到org.ops4j.pax.url.mvn.defaultRepositories属性
    物业org.ops4j.pax.url.mvn.defaultRepositories维护以逗号分隔的存储库列表，供运行时服务器使用。
    格式为： file: < / absolute / path / to / repo2 >@snapshots
    并且，如果使用了变量，则刚部署的新存储库将定义为： file：$ {org.apache.geronimo.server.dir}/repo2@snapshots并将结果属性设置为如下所示： org .ops4j.pax.url.mvn.defaultRepositories =文件：$ {karaf.home}/repository@snapshots，文件：$ {org.apache.geronimo.server.dir}/repo2@snapshots第一个定义的存储库是默认存储库，主要的GERONIMO_HOME / repository ，第二个是刚刚创建的新的GERONIMO_HOME / repo2存储库。

将应用程序部署到存储库

这是将示例jsp应用程序部署到存储库repo2的示例。

  1. 通过以下命令列出当前服务器上的目标： deploy(.bat) list-targets ，然后您会看到以下信息： 可用目标：org.example.configs / myrepo / 2.2 / car？ServiceModule = org.example.configs / myrepo / 2.2 / car，j2eeType = ConfigurationStore，name = Local2 org.apache.geronimo.configs / j2ee-system / 2.2 / car？ServiceModule = org.apache.geronimo.configs / j2ee-system / 2.2 / car，j2eeType = ConfigurationStore，name = Local
  2. 使用以下部署命令将应用程序部署到名为“ Local2”的资源库repo2：
  • 在Windows系统上：
    deploy.bat deploy --targets Local2 /applications/geronimo-jsp-examples/target/geronimo-jsp-examples-2.2.war
  • 在非Windows系统上：
    部署部署--targets Local2 /applications/geronimo-jsp-examples/target/geronimo-jsp-examples-2.2.war
    您的样本目录在哪里。然后，将jsp示例部署到新的存储库repo2。显示以下消息： 部署的org.apache.geronimo.applications.examples / geronimo-jsp-examples / 2.2 / war @ / jsp-examples
  1. 通过http：// <主机>：<端口> / jsp-examples访问该应用程序。
  1. 要从存储库repo2中取消部署应用程序，请运行以下deploy（.bat）命令：
  • 在Windows系统上：
    deploy.bat取消部署“ org.apache.geronimo.applications.examples / geronimo-jsp-examples / 2.2 / war”
  • 在非Windows系统上：
    取消部署“ org.apache.geronimo.applications.examples / geronimo-jsp-examples / 2.2 / war” 模块org.apache.geronimo.applications.examples / geronimo-jsp-examples / 2.2 / war的加载。org.apache.geronimo.applications.examples / geronimo-jsp-examples / 2.2 / war模块已卸载。未部署的org.apache.geronimo.applications.examples / geronimo-jsp-examples / 2.2 / war

在多个服务器实例上创建多个存储库

创建多个服务器实例，然后启动服务器实例。有关创建服务器实例的信息，请参阅运行多个Geronimo实例 。然后，您需要在这些服务器实例上创建几个存储库。您可以通过将应用程序部署到这些服务器实例并从这些服务器实例取消部署应用程序来测试存储库。

为服务器instance2创建存储库

创建多个服务器实例后，您可以创建存储库并将存储库部署到不同的服务器实例。有关如何在服务器实例上创建和部署存储库的信息，请参见以下示例，该示例创建repo22并将其部署在server2上：

  1. 在下面创建目录repo22 /server2 ， 例如， /server2/repo22 。
  2. 建立档案repo22.xml在下面/server2/repo22目录。有关文件的详细信息，请参阅创建多个存储库 。
  3. 部署repo22.xml通过从命令提示符处运行以下命令：
  • 使用非Windows：
    导出GERONIMO_SERVER = server2
    deploy -port instance2_RMI_port deploy $GERONIMO_HOME/server2/repo22/repo22.xml
  • 使用Windows
    设置GERONIMO_SERVER“ server2”
    deploy.bat -port instance2_RMI_port deploy %GERONIMO_HOME%/server2/repo22/repo22.xml

您可以按照相同的步骤为服务器instance3创建存储库。

将示例应用程序部署到存储库中或从存储库中取消部署

要测试您创建的存储库，您可以将示例应用程序部署到存储库中，或从存储库中取消部署。

  1. 记下先前创建的存储库repo22的存储库名称。我们假设名称为Local22。
  2. 使用以下deploy（.bat）命令将应用程序部署到存储库repo22中：
  • 在非Windows系统上：
    导出GERONIMO_SERVER = server2
    deploy -port instance2_RMI_port deploy --targets Local22 /applications/geronimo-jsp-examples/target/geronimo-jsp-examples-2.2.war
  • 在Windows系统上：
    设置GERONIMO_SERVER“ server2”
    deploy.bat -port instance2_RMI_port deploy --targets Local22 /applications/geronimo-jsp-examples/target/geronimo-jsp-examples-2.2.war
  1. 使用以下命令从repo22存储库中取消部署jsp应用程序：
  • 在非Windows系统上：
    导出GERONIMO_SERVER = server2
    部署-port instance2_RMI_port取消部署org.apache.geronimo.applications.examples / geronimo-jsp-examples / 2.2 / war
  • 在Windows系统上：
    设置GERONIMO_SERVER“ server2”
    deploy.bat -port instance2_RMI_port取消部署org.apache.geronimo.applications.examples / geronimo-jsp-examples / 2.2 / war

您可以按照相同的步骤将应用程序部署到服务器instance3上的存储库或从中取消部署应用程序。
