安全配置在哪里？

在普通的Geronimo服务器中，基本安全配置分为两个插件， j2ee-security和server-security-config 。您不太可能希望更改的部分（例如jacc提供程序和密钥库管理器）位于j2ee-security中。您几乎肯定要更改的部分在服务器安全性配置中。例如，管理控制台的玩具属性文件安全领域位于server-security-config中。

所以我有一个企业范围的身份验证系统...。如何为我的所有应用程序设置它？

您想用自己的Geronimo插件（请参阅“ 管理插件” ）替换server-security-config，该插件包含针对您的安全设置（例如ldap）定制的安全领域，并包括所需的任何密钥库。要用您的插件替换server-security-config的所有用法，请在您的插件中包含一个artifact-alias元素geronimo-plugin.xml文件。

与任何geronimo插件一样，您可以通过在geronimo存储库中安装jar并将它们作为geronimo计划中的依赖项列出，将任何jar包含在插件的类加载器中。car-maven-plugin可以用于使geronimo依赖项与maven依赖项相同，并使插件安装还安装所有需要的jar。

谁需要整个企业范围？我希望我的应用包含自己的安全设置！

您还可以在您的应用程序的geronimo计划中包括安全领域配置，密钥库和凭证存储。只需将gbean配置放在javaee特定配置的末尾即可。在这种情况下，您可能不想删除标准的server-security-config，因为删除它会阻止管理控制台或mejb的启动。
