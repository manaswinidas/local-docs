如何在Eclipse中开发Geronimo Eclipse插件
编辑页面 浏览空间 添加页面 新增新闻
  1. 使用Maven构建GEP。有关详细信息，请参见如何从源代码构建Geronimo Eclipse插件 。现在可以将项目导入到Eclipse工作空间中。
  2. 启动Eclipse Helios SR1的全新安装（有关详细信息，请参阅此页面 ）。
  3. 请遵循以下步骤完成用于GEP开发的Eclipse配置：
    • 将Java Installed JRE设置为Java 6
    • 设置ClassPath变量
    • 设定目标平台
    • 将源代码导入您的Eclipse工作区
    • 运行和调试配置

将Java Installed JRE设置为Java 6

  1. 选择窗口->首选项
  2. 选择Java-> Installed JRE 。
  3. 确保选择了Java 6 JRE。
  4. 选择Java-> Compiler 。
  5. 确保选择了Compiler合规性级别1.6。
    • 这样做避免了大约8个错误，例如：
      The method getServerPluginManager() of type GeronimoServer must override a superclass method
      错误，并且应该匹配我们使用的Java 6 JRE。

设置ClassPath变量

  1. 如图所示，选择Window-> Preferences 。





  2. 选择Java-> Build Path-> Classpath Variables 。选择新建。





  3. 输入您的m2repo的位置。我的m2repo位于C：/ m2repo 。选择确定。 
    *Name:* M2_REPO
*Path:* C:\m2repo



  4. 完成后，您可以看到M2_REPO已添加到ClassPath变量列表中。


设定目标平台

Eclipse帮助中有关目标平台的摘录中说：“目标平台是指您正在开发和测试插件的平台。目标平台包括目标的位置，构成目标的插件列表，目标环境和启动参数。”
在配置用于GEP开发的目标平台时，需要执行以下步骤。

  1. 选择窗口->首选项 。





  2. 在下一个屏幕上，选择Plug-in Development-> Target Platform 。浏览到/ eclipse / eclipse 。选择应用。
    • 注意： Helios SR1中的菜单会更改。对Helios SR1使用以下步骤：
    1. 单击添加...目标平台
    2. 下一个
    3. 加...
    4. 选择目录，单击下一步
    5. 将/ eclipse / eclipse放在适当的位置。（您可以使用浏览...）
    6. 点击完成
    7. Give是一个类似于M2_REPO> / eclipse / eclipse的名称 ，然后单击Finish。
    8. 确保选择了新平台。



  3. 选择确定。这将为我们的GEP开发建立目标平台。


  4. 如果您要使用http://www.ibm.com/developerworks/rational/downloads/10/rationaldevtoolsforosgiapplications.html中提供的免费OSGi工具，
    然后从该页面上的链接（当前为http://public.dhe.ibm.com/ibmdl/export/pub/software/rational/OSGiAppTools ）下载插件，
    并按位置将它们添加到您的目标平台。

将源代码导入您的Eclipse工作区

下一步是将插件的源代码导入您的工作区。这是您将一直使用的代码。

  1. 右键单击Project Explorer下的，然后选择Import-> Import 。





  2. 在下一个窗口中，选择General-> Existing Projects into Workspace 。选择下一步 。





  3. 在导入项目窗口中，浏览到/ trunk 。选择完成。





    如果出现一些错误，请尝试修复这些错误（将其视为使用GEP的第一步！ ），或者使用Geronimo邮件列表 。有关详细信息，请参见如何为Geronimo Eclipse插件打开错误 。

运行和调试配置

在开始运行和删除代码之前，请设置调试和运行配置。

  1. 选择运行->运行配置 。





  2. 在“ 运行配置”窗口上，选择“ Eclipse应用程序”，然后选择左上角的“ 新建”按钮。





  3. 将新配置命名为GEP_DEVELOPMENT 。另外，将工作空间位置名称从runtime-New_location更改为runtime-GEP_Development 。选择应用。





  4. 接下来，为您的GEP_Development运行配置选择Arguments选项卡，并如下设置VM参数
    -Xms256m -Xmx256m -XX:MaxPermSize=128m



  5. 接下来，选择“跟踪”选项卡，然后选择最上面的4个罐子。参见下图。选择“应用并运行”。





  6. 这将启动一个新的Eclipse窗口，不过是Eclipse + 。您可以修改代码，并且每次可以启动Eclipse Configuration来测试代码中的修改。





  7. 要在Eclipse中调试代码更改到Debug Perspective，然后选择Run-> Debug Configurations 。选择调试。
