排队

cxf-tools脚本用于生成Web服务

。该命令具有以下语法：

cxf工具

java2ws或wsdl2java在哪里。

java2ws工具使用以下语法：

java2ws-数据绑定- 前端 -wsdl -wrapperbean- 客户端-服务器-ant -wrapperbean -o -d -s -classdir -cp -soap12 -t -beans -servicename -portname -address
-createxsdimports -h -v-详细-安静

下表回顾了用于管理代码生成过程的参数。

选项

用法

描述

-help或-h

-help或-h

可用于获取帮助信息。

-数据绑定

-数据绑定

可用于指定数据绑定（jaxb或aegis）。默认情况下，对于jaxws前端，它是jaxb；对于简单的前端，它是aegis。

-前端

-前端

可以使用tp指定前端。支持Jaxws和简单的前端。

-wsdl

-wsdl

可用于生成WSDL文件。

-客户

-客户

可用于生成客户端代码。

-服务器

-服务器

可用于生成服务器端代码。

-蚂蚁

-蚂蚁

可用于生成蚂蚁build.xml脚本。

-wrapperbean

-wrapperbean

可用于生成包装器和故障Bean。

-o

-o

可用于指定生成的WSDL文件的名称。

-d

-d

可以用于指定资源文件所在的目录。默认情况下，wsdl文件将放置在此目录中。

-s

-s

可以用来指定所生成的源文件（包装器bean，故障bean，客户端或服务器端代码）所在的目录。

-classdir

-classdir

可用于指定将生成的源编译到的目录。如果未指定，则不编译文件。

-cp

-cp

可用于指定目录和zip / jar文件的SEI和类型类搜索路径。

-soap12

-soap12

可以用来指示生成的WSDL将包括SOAP 1.2绑定。

-t

-t

可用于在生成的WSDL文件中指定目标名称空间。

-豆子

-豆子

可用于指定生成的bean定义文件的路径和名称。

-服务名称

-服务名称

可用于指定生成的服务元素的name属性的值。

-端口名

-端口名

可用于指定在生成的WSDL中使用的端口名称。

-地址

-地址

可用于指定端口地址。

-createxsdimports

-createxsdimports

可用于输出模式以分离文件并通过导入来加载它们，而不是将其内联到WSDL中。

-v

-v

可以用来获取版本号。

-冗长

-冗长

可用于在代码生成过程中显示注释。

-安静

-安静

可用于在代码生成过程中取消注释。

可用于指定SEI类的名称。

wsdl2java工具使用以下语法：

wsdl2java -fe * -db * -wv -p <[ wsdl- 命名空间=]程序包名称> * -sn -b -d -compile -classdir -impl -server -client -all -autoNameResolution -defaultValues <= class-name -for-DefaultValueProvider> -ant -nexclude * -exsh < > -dns -dex < > -validate -keep -wsdllocation -xjc -noAddressBinding -h -v -verbose -quiet

下表回顾了用于管理代码生成过程的参数。

选项

用法

描述

-help或-h

-help或-h

可用于获取帮助信息。

-fe

-fe

可用于指定前端。默认情况下，它是JAXWS前端。当前仅支持JAXWS前端。

-D b

-D b

可用于指定数据绑定。默认情况下是jaxb。当前仅支持JAXB数据绑定。

-wv

-wv

可以用于指定wsdl版本。默认情况下，它是WSDL 1.1。当前仅支持WSDL 1.1版本。

-p

-p <[wsdl-namespace =]软件包名称> *

可用于为生成的代码指定零个或多个包名称。

-sn

-sn

可用于为生成的代码指定WSDL服务名称。

-b

-b

可以用于指定零个或多个JAXWS或JAXB绑定文件。您可以使用空格分隔多个条目。

-目录

-目录

可用于指定映射导入的wsdl / schema的目录文件。

-d

-d

可用于指定将生成的代码文件写入的目录。

-编译

-编译

可用于编译生成的Java文件。

-classdir

-classdir

可以用于指定将编译的类文件写入的目录。

-impl

-impl

可用于生成实现对象的起点代码。

-客户

-客户

可用于生成客户端主线的起点代码。

-服务器

-服务器

可用于生成服务器主线的起点代码。

-所有

-所有

可用于生成所有起点代码：类型，服务代理，服务接口，服务器主线，客户端主线，实现对象和一个Ant build.xml文件。

-autoNameResolution

-autoNameResolution

可用于自动解决命名冲突而无需绑定自定义项。

-默认值

-defaultValues = [DefaultValueProvider隐含]

可用于生成impl和客户端的默认值。您还可以提供自定义默认值提供程序。默认提供程序是RandomValueProvider。

-蚂蚁

-蚂蚁

可用于生成蚂蚁build.xml文件。

-排除

-nexclude *

可用于在生成代码时忽略指定的WSDL模式名称空间。可以多次指定此选项。也可以指定排除的名称空间中描述的类型所使用的Java软件包名称。Java包名称是可选的。

-exsh

-exsh < >

可用于启用或禁用隐式SOAP头的处理（在wsdl：binding中定义的SOAP头，但在wsdl：portType部分中未定义。）默认情况下为false。

-DNS

-dns < >

可用于启用或禁用默认名称空间包名称映射的加载。默认为true并且

http://www.w3.org/2005/08/addressing=org.apache.cxf.ws.addressing

名称空间包映射将被启用。

-dex

-dex < >

可用于启用或禁用默认的排除名称空间映射的加载。默认为true。

-验证

-验证

可用于在生成代码之前启用对WSDL的验证。

-保持

-保持

可以用来指示代码生成器不会覆盖任何现有文件。您将负责解决所有由此产生的编译问题。

-wsdlLocation

-wsdllocation

可以用于指定@WebServiceClient批注的wsdlLocation属性的值。

-xjc

-xjc

可以用于指定使用逗号分隔的参数列表，这些列表在使用JAXB数据绑定时直接传递给XJC处理器。可用的XJC插件列表可以使用获取-xjc-X 。

-noAddressBinding

-noAddressBinding

可用于指导代码生成器生成较早的CXF专有WS-Addressing类型，而不是与JAX-WS 2.1兼容的WS-Addressing类型。

-v

-v

可以用来获取版本号。

-冗长

-冗长

可用于在代码生成过程中显示注释。

-退出

-安静

可用于在代码生成过程中取消注释。

可用于在生成代码时指定WSDL文件的路径和名称。
