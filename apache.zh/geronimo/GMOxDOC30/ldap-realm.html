本文档介绍了在Apache Geronimo v2.1中配置LDAP领域的一些替代方法。在先前版本的Apache Geronimo中，ApacheDS与Geronimo一起提供。为了减少服务器占用空间并利用Geronimo插件体系结构，LDAP服务器当前作为可选插件安装可用，而不是与Geronimo Server捆绑在一起用于2.1。话虽如此，但ApacheDS并不是配置LDAP域的必要条件。但是，对于本文档而言，使用本地LDAP服务器将是切实可行的。

本文档分为以下几节：

  • 安装ApacheDS插件（可选）
    • 从Geronimo管理控制台安装插件
    • 从命令行安装插件
  • 部署LDAP领域
    • 使用Geronimo管理控制台
    • 使用命令行部署程序
      • LDAP Realm部署计划

安装ApacheDS插件（可选）

如果您选择由ApacheDS提供并在Geronimo中运行LDAP服务，则可以通过安装插件来快速安装它。Geronimo的ApacheDS 1.0插件基于Apache Directory v1.5.1，可以从命令行或管理控制台安装。

从Geronimo管理控制台安装插件

  • 登录到Geronimo管理控制台。
  • 单击插件 。
  • 单击更新存储库列表 （如果您已经指向http://geronimo.apache.org/plugins/geronimo-2.1，则为可选）。
  • 单击“ 显示所选存储库中的插件” 。
  • 单击Apache Geronimo-Apache目录服务器插件1.0 LDAP链接，它应显示为可安装。
  • 一直滚动到页面底部，然后单击“ 安装” 。

安装后，该插件将自动启动。

从命令行安装插件

在命令行窗口中，将目录更改为/ bin并运行以下命令：

  • deploy search-plugins 。
  • 从列表中选择适当的存储库，在此示例中，选择1. http://geronimo.apache.org/plugins/geronimo-2.1/ 。
  • 从以下列表中确定Apache Geronimo-Apache Directory Server Plugin (1.0) ，此选项将显示在LDAP具有ID号的类别。
  • 输入LDAP插件的ID号（例如78 ），然后按Enter 。

安装后，该插件将自动启动。

部署LDAP领域

与往常一样，在这种情况下，您可以选择从管理控制台或命令行部署资产，例如安全领域。

使用Geronimo管理控制台

  • 登录到Geronimo管理控制台。
  • 单击安全领域 portlet。
  • 单击添加新安全领域 。
  • 输入“安全领域名称”的值： 。
  • 从“ 领域类型：”下拉菜单中选择“ LDAP领域” ，然后单击“ 下一步” 。
  • 输入与您的LDAP配置相对应的设置，然后单击下一步 。
  • 根据需要选择“高级设置”。
  • 要验证设置是否正确，请单击“ 测试登录名” 。
  • 输入已知的用户名和密码，然后单击下一步 。
  • 您应该会收到一条确认消息，表明连接成功，成功登录了1个主体
  • 单击“ 部署领域” 。

LDAP Realm将部署，您应该能够从Security Realms portlet中看到它。从那里您将能够进一步编辑领域。

请注意，为了成功连接和测试此领域，您的LDAP服务器必须已经配置/预加载了适当的数据。换句话说，您在此处输入的内容与服务器中的内容匹配。LDAP服务器不附带任何预加载的样本数据。

使用命令行部署程序

如前所述，部署此配置的另一种方法是通过命令行中的Deployer工具 。通过这种方式进行部署，您将必须提供一个部署计划，其中已包含所有配置信息，这次将没有交互式菜单。

部署本身非常简单，一旦您解决了部署计划，请从/ bin目录运行以下命令：

deploy --user system --password manager deploy /ldap_realm_plan.xml

LDAP Realm部署计划

我们提到，使用命令行部署程序时，您将需要提供部署计划。本部分的目的是向您显示示例部署计划，有关其他详细信息，请参阅ldap-sample-app-LDAP示例应用程序部分。

示例LDAP Realm部署计划
<module xmlns="http://geronimo.apache.org/xml/ns/deployment-1.2">
    <environment>
        <moduleId>
            <groupId>console.realm</groupId>
            <artifactId>LDAP_Test_Realm</artifactId>
            <version>1.0</version>
            <type>car</type>
        </moduleId>
        <dependencies>
            <dependency>
                <groupId>org.apache.geronimo.framework</groupId>
                <artifactId>j2ee-security</artifactId>
                <type>car</type>
            </dependency>
        </dependencies>
    </environment>
    <gbean name="LDAP_Test_Realm" class="org.apache.geronimo.security.realm.GenericSecurityRealm" 
				xsi:type="dep:gbeanType" xmlns:dep="http://geronimo.apache.org/xml/ns/deployment-1.2" 
				xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
        <attribute name="realmName">LDAP_Test_Realm</attribute>
        <reference name="ServerInfo">
            <name>ServerInfo</name>
        </reference>
        <xml-reference name="LoginModuleConfiguration">
            <log:login-config xmlns:log="http://geronimo.apache.org/xml/ns/loginconfig-2.0">
                <log:login-module control-flag="REQUIRED" wrap-principals="false">
                    <log:login-domain-name>LDAP_Test_Realm</log:login-domain-name>
                    <log:login-module-class>org.apache.geronimo.security.realm.providers.LDAPLoginModule</log:login-module-class>
                    <log:option name="connectionUsername">uid=admin,ou=system</log:option>
                    <log:option name="userSearchSubtree">false</log:option>
                    <log:option name="roleSearchMatching">uniqueMember={0}</log:option>
                    <log:option name="roleBase">ou=users,ou=system</log:option>
                    <log:option name="connectionPassword">secret</log:option>
                    <log:option name="roleName">cn</log:option>
                    <log:option name="initialContextFactory">com.sun.jndi.ldap.LdapCtxFactory</log:option>
                    <log:option name="roleSearchSubtree">false</log:option>
                    <log:option name="connectionURL">ldap://localhost:10389</log:option>
                    <log:option name="userBase">ou=users,ou=system</log:option>
                    <log:option name="userSearchMatching">uid={0}</log:option>
                    <log:option name="authentication">simple</log:option>
                </log:login-module>
            </log:login-config>
        </xml-reference>
    </gbean>
</module>
