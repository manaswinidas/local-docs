Java EE应用程序客户端部署计划

Java EE应用程序客户端模块在客户端容器中运行，并且还可以访问服务器环境。通常，将创建Java EE客户端应用程序来管理服务器中正在运行的企业应用程序。客户端模块在单独的JVM中运行，并连接到企业应用程序资源，但是可以以标准Java EE方式访问所有应用程序资源。

Java EE客户端模块需要application-client.xml作为部署描述符，而geronimo-application-client.xml作为部署计划。在里面application-client.xml ，则在时声明了所需的ejb名称，安全角色名称，资源名称等。 geronimo-application-client.xml ，声明的名称将映射到服务器中的实际资源。

以下是Java EE应用程序客户端模块的部署描述符，该描述符查找ejb并在其上调用方法。ejb将印度卢比（Rs。）转换为美元（$）。客户端向ejb发送一个双倍的值，即印度卢比。ejb返回等值的美元作为双值。

application-client.xml

上面的XML文档的默认名称空间是http://java.sun.com/xml/ns/javaee 。没有名称空间前缀的XML元素属于默认名称空间。因此，在上述XML文档中，所有XML元素都属于默认名称空间。

应用程序客户端声明ejb名称ejb/Converter通过> .. 元素。
