解决Web服务问题

部署期间WSDL生成问题

当部署不具有相关WSDL的JAX-WS Web服务时，应用程序服务器将自动生成该服务的WSDL。通过Axis2作为JAX-WS提供程序，应用程序服务器使用Sun的wsgen生成WSDL的工具。有时wsgen可能无法生成WSDL（例如，由于无效的注释）。在大多数情况下， wsgen工具将记录到geronimo.log文件。但是，在某些情况下，可能不会记录输出，因此可能需要启用额外的日志记录才能查看日志的完整输出。 wsgen工具。启用完整日志记录wsgen工具将以下行添加到server-log4j.properties文件：

log4j.logger.org.apache.geronimo.jaxws.builder =调试

wsgen的输出将出现在geronimo.log文件。

使用CXF部署Web服务时出现AccessControlException

使用CXF部署JAX-WS Web服务时，您可能会看到以下异常：

造成原因：java.security。AccessControlException：访问被拒绝（javax.xml.ws。WebServicePermission publishEndpoint），位于java.security。java.security上的AccessControlContext.checkPermission（AccessControlContext.java:264）。org.apache.cxf.jaxws上的AccessController.checkPermission（AccessController.java:427）。org.apache.cxf.jaxws上的EndpointImpl.checkPublishPermission（EndpointImpl.java:389）。org.apache.cxf.jaxws上的EndpointImpl.doPublish（EndpointImpl.java:248）。org.apache.cxf.jaxws上的EndpointImpl.publish（EndpointImpl.java:201）。位于sun.reflect的EndpointImpl.publish（EndpointImpl.java:394）。sun.reflect处的NativeMethodAccessorImpl.invoke0（本机方法）。在sun.reflect处调用NativeMethodAccessorImpl.invoke（NativeMethodAccessorImpl.java:39）。java.lang.reflect上的DelegatingMethodAccessorImpl.invoke（DelegatingMethodAccessorImpl.java:25）。Method.invoke（Method.java:585）...

根据JSR-109规范，应用服务器应该阻止通过Endpoint API发布新的Web Service端点。但是，可以通过将org.apache.cxf.jaxws.checkPublishEndpointPermission系统属性设置为false来禁用此功能。例如，设置以下属性后，启动服务器：

导出JAVA_OPTS = -Dorg.apache.cxf.jaxws.checkPublishEndpointPermission = false

使用Web Service客户端通过证书访问Web Service

对于Web Service客户端，在使用client-cert时，可能需要请求服务器将证书添加到Geronimo服务器中的受信任密钥存储中。同时，您需要获取服务器的证书，然后将其添加到本地密钥库中（Geronimp管理控制台可用于导出服务器的证书）。

在启动服务器之前，您还需要设置以下属性。

导出GERONIMO_OPTS = -Djavax.net.ssl.trustStore = YOUR_CLIENT_TRUSTED_KEY_STORE -Djavax.net.ssl.trustStorePassword = YOUR_CLIENT_TRUSTED_KEY_STORE_PASSWORD -Djavax.net.ssl.keyStore = YOUR_CLIENT_KEY_STORE.DStore-Djavax。
