CVE-2010-2227：Apache Tomcat远程拒绝服务和信息泄露漏洞

Tomcat Web容器包含一个漏洞，该漏洞可能使Geronimo服务器受到远程拒绝服务攻击，并可能泄露有关Geronimo服务器上运行的应用程序的信息。可以在以下位置找到此漏洞的详细信息：

  • http://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2010-2227

如果您选择不升级到Apache Geronimo 2.1.7，另一种解决方法是使用Tomcat的更新版本手动修补服务器。修补现有发行版的说明可以在这里找到：

  • Geronimo Geronimo 2.1.x和2.2.x CVE-2010-2227 Apache Tomcat远程拒绝服务补丁说明

JIRA： GERONIMO-5387
影响：2.1.1-2.1.6

请访问2.1.6发行说明页面，以获取所有包含的JIRA的详细信息。

Geronimo服务器：

CVE-2010-1632和CVE-2010-2076：Axis2和CXF HTTP绑定启用了基于DTD的XML攻击。

在Axis2和CXF Web服务运行时中都发现了一个漏洞，该漏洞可使攻击者确定目标服务器上文件的存在并可能提取目标文件的内容。这会影响所有包括Axis2或CXF运行时的Geronimo程序集，尤其是javaee5 Jetty和Tomcat程序集。可以在以下Axis2和CXF安全警报中找到这些漏洞的详细信息：

  • https://svn.apache.org/repos/asf/axis/axis2/java/core/security/CVE-2010-1632.pdf
  • https://svn.apache.org/repos/asf/cxf/trunk/security/CVE-2010-2076.pdf

Apache Geronimo 2.1.6发行版包含Axis2 1.3和Axiom 1.2.5的修补程序以及对CXF 2.1.13的升级。

如果您选择不升级到Apache Geronimo 2.1.6，则另一种解决方法是禁用Web服务运行时或使用运行时的更新版本手动修补服务器。可以在以下位置找到有关禁用Web服务运行时或修补现有发行版的说明：

  • Geronimo 2.1.x CVE-2010-1632补丁说明

JIRA： GERONIMO-5383
影响：2.1.2.1.5

Geronimo服务器：

CVE-2010-1622：Spring框架执行任意代码

Spring框架提供了一种使用客户端提供的数据来更新对象属性的机制。该机制允许攻击者修改用于加载对象的类加载器的属性（通过“ class.classloader”）。这可能导致任意命令执行，因为例如，攻击者可以修改类加载器使用的URL，以指向由攻击者控制的位置。可以在以下位置找到此漏洞的详细信息：

  • http://www.securityfocus.com/archive/1/511877/30/0/threaded

Apache Geronimo 2.1.6版本包括对Spring Framework v2.5.6的升级。SEC02。

当前，由于这种利用，在Geronimo服务器中没有已知的公开信息，但是使用包含的Spring Framework版本的应用程序可能会受到攻击。如果您选择不升级到Apache Geronimo 2.1.6，另一种解决方法是使用Spring Framework的更新版本手动修补服务器。修补现有发行版的说明可以在这里找到：

  • Geronimo Geronimo 2.1.x和2.2.x Spring Framework SEC02补丁说明

JIRA： GERONIMO-5387
影响：2.1.2.1.5

Geronimo管理控制台：

CVE-2008-5518：Apache Geronimo Web管理控制台目录遍历漏洞。

在Windows上运行Apache Geronimo服务器时，在多个Portlet（包括服务/存储库，嵌入式DB / DB Manager和Security / Keystore）中发现了一个漏洞。此问题可能允许远程攻击者上载任何目录中的任何文件。这会影响所有完整的JavaEE Geronimo程序集或其他发行版，其中包括管理Web控制台（包括Apache Geronimo 2.1.3之前的版本）。另一种解决方法（如果您选择不升级到Apache Geronimo 2.1.4）则是停止或取消部署服务器中的管理Web控制台应用程序。
图片来源：Apache Geronimo项目要感谢Digital Security Research Group（dsecrg.com）负责任地报告此问题并协助我们验证修补程序。

JIRA： GERONIMO-4597
影响：2.1.2.1.3

CVE-2009-0038：Apache Geronimo Web管理控制台XSS漏洞

在Apache Geronimo管理控制台和相关实用程序中发现了各种链接的和存储的跨站点脚本（XSS）漏洞。使用此漏洞，攻击者可以窃取管理员的Cookie，然后以管理员身份进行身份验证或执行某些管理操作。例如，用户可以在某些URL或各种portlet中的几个输入字段中注入XSS。这会影响所有完整的JavaEE Geronimo程序集或其他发行版，其中包括管理Web控制台（包括Apache Geronimo 2.1.3之前的版本）。另一种解决方法（如果您选择不升级到Apache Geronimo 2.1.4）则是停止或取消部署服务器中的管理Web控制台应用程序。
图片来源：Apache Geronimo项目要感谢Digital Security Research Group（dsecrg.com）和Marc Schoenefeld（红帽安全响应团队）负责任地报告了此问题并帮助我们验证了修补程序。

JIRA： GERONIMO-4597
影响：2.1.2.1.3

CVE-2009-0039：Apache Geronimo Web管理控制台XSRF漏洞

在Apache Geronimo Web管理控制台中发现了各种跨站点请求伪造（XSRF或CSRF）漏洞。利用这些问题，一旦用户通过身份验证并获得与服务器的有效会话，远程攻击者就可以使用可预测的URL请求执行某些管理操作，例如更改Web管理密码，上传应用程序等。这会影响所有完整的JavaEE Geronimo程序集或其他发行版，其中包括管理Web控制台（包括Apache Geronimo 2.1.3之前的版本）。另一种解决方法（如果您选择不升级到Apache Geronimo 2.1.4）则是停止或取消部署服务器中的管理Web控制台应用程序。
图片来源：Apache Geronimo项目要感谢Digital Security Research Group（dsecrg.com）负责任地报告此问题并协助我们验证修补程序。

JIRA： GERONIMO-4597
影响：2.1.2.1.3
