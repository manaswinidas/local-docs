欢迎使用Apache Geronimo

Apache Geronimo是一个开源服务器运行时，它集成了最佳的开源项目，以创建满足企业开发人员和系统管理员需求的Java / OSGi服务器运行时。

我们最受欢迎的发行版以前是经过完全认证的Java EE 6应用程序服务器运行时。

现在，我们将重点重新放在提供JavaEE / JakartaEE库和Microprofile实现上。

请注意，Apache Geronimo Project不再积极维护Geronimo Application Server！

如果您要寻找替代产品，请查看Apache TomEE Application Server项目。

但是，我们仍在积极地提供可重用的Java EE组件。它们被广泛使用并且仍在积极维护中！

我们的一些指导原则是：

  • 使用方便。
  • 构建根据Apache软件许可协议分发的服务器。
  • 提供满足开发人员，管理员和系统集成商需求的运行时。
  • 与可用的最佳开源工具（如Eclipse）集成。
  • 提供我们软件的频繁发布，以便用户可以体验最新功能并访问最新的错误修复程序。
  • 建立一个社区，其中包含创建复杂的运行时和可工具化的基础架构所需的多个学科。

Apache Geronimo v3.0.0发布
Xia Forrest发表于2012年7月13日

2012年7月13日

Apache Geronimo项目很高兴宣布Apache Geronimo v3.0.0服务器可用。此版本是3.0-beta-1之后的版本3的正式版本。它仅包括Tomcat程序集。除了自3.0-beta-1起实现Java EE 6兼容性外，Geronimo v3.0.0发行版还增加了对所有Java EE 6 Web概要文件技术的Java 7支持。请参阅3.0.0发行说明中的详细信息。

几个要点是：

  • Tomcat升级到7.0.27
  • OSGi升级到4.3
  • BVal升级到0.4
  • 将Derby更新到版本10.8.2.2
  • 支持OSGi应用程序的类热插拔
