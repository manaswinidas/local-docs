Apache Geronimo 2.2.x漏洞

该页面列出了Apache Geronimo 2.2的维护版本或临时版本中修复的所有安全漏洞。Apache Geronimo团队或提供此修复程序的从属项目对每个漏洞都给予了安全影响等级-请注意，该等级不是统一的，并且因项目而异。我们还会列出已知会影响该漏洞的Apache Geronimo的版本，并且在尚未验证该漏洞的地方列出带有问号的版本。

请将对这些漏洞的评论或更正发送到Geronimo Security邮件列表 。

  • Apache Geronimo 2.2.1


在Geronimo 2.2.1中修复

CVE-2011-5034和CVE-2011-4858- “通过哈希算法冲突的多种实现拒绝服务”已通过GERONIMO-6253修复。

请访问2.2.1发行说明页面，以获取所有包含的JIRA的详细信息。

Geronimo服务器：

CVE-2010-1632和CVE-2010-2076：Axis2和CXF HTTP绑定启用了基于DTD的XML攻击。

在Axis2和CXF Web服务运行时中都发现了一个漏洞，该漏洞可使攻击者确定目标服务器上文件的存在并可能提取目标文件的内容。这会影响所有包含Axis2或CXF运行时的Geronimo程序集，尤其是javaee5 Jetty和Tomcat程序集。可以在以下Axis2和CXF安全警报中找到这些漏洞的详细信息：

  • https://svn.apache.org/repos/asf/axis/axis2/java/core/security/CVE-2010-1632.pdf
  • https://svn.apache.org/repos/asf/cxf/trunk/security/CVE-2010-2076.pdf

CXF漏洞已在CXF 2.1.10中修复。Axis2漏洞将在Axis2 1.5.2和Axiom 1.2.9中修复。

如果您选择不升级到Apache Geronimo 2.2.1，另一种解决方法是禁用Web服务运行时或使用运行时的更新版本手动修补服务器。可以在以下位置找到有关禁用Web服务运行时或修补现有发行版的说明：

  • Geronimo 2.2.x CVE-2010-1632补丁说明

JIRA： GERONIMO-5383
影响：2.2

Geronimo服务器：

CVE-2010-1622：Spring框架执行任意代码

Spring框架提供了一种使用客户端提供的数据来更新对象属性的机制。该机制允许攻击者修改用于加载对象的类加载器的属性（通过“ class.classloader”）。这可能导致任意命令执行，因为例如，攻击者可以修改类加载器使用的URL，以指向由攻击者控制的位置。可以在这里找到此漏洞的详细信息：

  • http://www.securityfocus.com/archive/1/511877/30/0/threaded

当前，由于这种利用，在Geronimo服务器中没有已知的公开信息，但是使用包含的Spring Framework版本的应用程序可能很容易受到攻击。Apache Geronimo 2.2.1发行版包括对Spring Framework v2.5.6的升级。SEC02修复此问题。

如果您选择不升级到Apache Geronimo 2.2.1，另一种解决方法是使用Spring Framework的更新版本手动修补服务器。修补现有发行版的说明可以在这里找到：

  • Geronimo Geronimo 2.1.x和2.2.x Spring Framework SEC02补丁说明

JIRA： GERONIMO-5387
影响：2.2

Geronimo服务器：

CVE-2010-2227：Apache Tomcat远程拒绝服务和信息泄露漏洞

Tomcat Web容器包含一个漏洞，该漏洞可能使Geronimo服务器受到远程拒绝服务攻击，并可能泄露有关Geronimo服务器上运行的应用程序的信息。可以在这里找到此漏洞的详细信息：

  • http://cve.mitre.org/cgi-bin/cvename.cgi?name=CVE-2010-2227

Apache Geronimo 2.2.1发行版包括对Tomcat 2.0.29的升级，以解决此问题。

如果您选择不升级到Apache Geronimo 2.2.1，另一种解决方法是使用Tomcat的更新版本手动修补服务器。修补现有发行版的说明可以在这里找到：

  • Geronimo Geronimo 2.1.x和2.2.x CVE-2010-2227 Apache Tomcat远程拒绝服务补丁说明

JIRA： GERONIMO-5387
影响：2.2
