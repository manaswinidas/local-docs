Geronimo群集概述-10,000英尺...

集群是将企业JEE实现与其他组件区分开来的关键功能之一。因此，这是Apache Geronimo的一项重要要求。

通过使用集群技术为JEE部署提供可扩展且高度可用的平台，Apache Geronimo将能够与现有的商业产品平等竞争。

本文档将开始以下任务：

  • 枚举Geronimo层中的集群要求
  • 从这些抽象出共同点
  • 提供给我们的编目软件，可用于实现它们
  • 建议集群架构

“集群”是一种架构，它通过布置多个更小，更便宜，更不可靠的资源而不是单个大型，昂贵且极其可靠的资源来实现可伸缩性和高可用性。

通过解耦对共享资源的依赖关系来确保可伸缩性，以便相关任务可以在群集中的许多机器（节点）上同时运行，而不会互相干扰。如果您的体系结构实现了这一目标，则可以通过添加更多节点来扩展规模以服务更多的用户。

高可用性是通过冗余实现的。如果一个不太可靠的节点发生故障，则您将故障转移到下一个。这样，系统的可用性就不是总和，而是其组成节点的可用性的乘积。

状态在集群中的存在，通过成为共享竞争点（许多任务可能需要同时读取/写入同一状态）和失败点（如果状态保存在易碎的资源（例如内存）中，丢失了，状态也是如此，即不再存在
可用）。

对状态进行分区可以帮助恢复可伸缩性，并且可以制作和维护状态的多个副本（复制）来规避争用问题。两种解决方案都导致其他较小的问题，例如，确保进程在与其状态相同的分区中运行，并确保状态的多个副本之间视图的一致性等。其他解决方案和其他问题比比皆是。

集群中状态的不同用途的数量排除了单一有效解决方案的可能性。因此，我们需要为Geronimo中的每个用例设计解决方案。

我们将枚举并检查这些用例。

兴趣小组：朱尔斯·戈斯内尔，扬·巴特尔，杰夫·金纳德，大卫·詹克斯，...

Web层中的集群有两个实现点：

HTTP负载均衡器：

  • 我们的解决方案应与任何负载均衡器一起使用
  • 我们可以期望LB支持某种形式的亲和力（粘性会话或持久性会话）。

HttpSession：

  • 大量潜在的大对象（层中的数据多于一个节点中可以容纳的数量）-需要分区缓存
  • 通常经常写
  • 通常经常阅读
  • 通常是单个消费者/客户
  • 无交易
  • 一次只能复制一个HttpSession的副本
  • 多线程
  • 悲观
  • 建议的展示-WADI

注意：其他一些解决方案不仅可以对HttpSession进行集群。规范仅要求存储在HttpSession中的数据是可分发的。

EJB层（如Web）中的集群有两个实现点：

银行

  • 大量潜在的大对象（层中的数据多于一个节点中可以容纳的数量）-需要分区缓存
  • 交易性的
  • 通常经常写

  • 潜在的多个消费者

  • 需要分布式数据库支持的缓存
  • 由于映射到数据库，因此不需要分区缓存，因此只需卸载数据即可
  • 实体bean的分布式缓存： http : //www.mail-archive.com/dev%40geronimo.apache.org/msg15697.html

部署方式

http://jcp.org/en/jsr/detail?id=088

  • 潜在的大对象（也许我们可以只发送引用）
  • 层内容的并集小于单节点容量
  • 很少写（ 未部署）
  • 经常阅读（调用）
  • 无交易
  • 没有唯一性约束吗？（我们应该如何创建异构部署？）
  • 直接进行1->所有复制的主要候选对象
  • 建议的展示-ActiveSpace

建议-“部署集”-共享/实施同构部署的一组节点。请参阅“同类型部署与异类部署”。

建议-将应用程序部署在一个节点上，它通过其http服务器形成指向该应用程序的url，并将此链接复制到其部署集中的其他服务器。该集合中的每个成员均会接收链接，拉出应用程序并进行部署。

建议-是否可以同步和串行进行复制，以便在将应用程序部署在每个节点上时，可以在分发继续到下一个节点之前检查其完整性？

WADI / ActiveSpace协同

WADI和ActiveSpace是免费技术。这里有API收敛和代码重用的范围。如果两者可以共存于同一个API之后，我们也许可以完全从使用者那里抽象出Cache impl，从而提供更大的灵活性，并允许我们针对特定问题更紧密地定制解决方案。

建议的构建基块

ActiveMQ

网址： http ： //incubator.apache.org/activemq/
兴趣小组：James Strachan，Hiram Chirino，...

Geronimo的JMS实现具有可插拔的传输，包括对等方：///协议，该协议允许集群中的对等方直接相互发送消息，而无需中央代理。

ActiveCluster

网址： http ： //activecluster.codehaus.org/
兴趣小组：James Strachan，Hiram Chirino，...

提供基本群集功能（特别是成员资格更改通知）以及1-> all和1-> 1消息传递的API。另外，此API的各种形式，最显着的是使用ActiveMQ。

回到顶部

活动空间

网址： http ： //activespace.codehaus.org/
兴趣小组：James Strachan，Hiram Chirino，...

提供两个抽象，一个JCache样式的Cache（复制的和/或事务的）和一个JavaSpaces样式的空间，用于分布式计算。
