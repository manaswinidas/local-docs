http://www.w3.org/WAI/是可访问性的一个很好的起点。 W3C早在http://www.w3.org/TR/WCAG10/上发布了一个指南。而且（很长一段时间）正在开发更新的版本http://www.w3.org/TR/WCAG20/ 。

这是一个简短的版本，突出显示了与Geronimo的管理控制台最相关的准则。

Web辅助功能指南（简短版本）
1。提供听觉和视觉内容的等效替代方法。

  • 为IMG，APPLET元素设置有意义的ALT属性。
  • 如果使用服务器端图像映射，请提供等效的文本链接。

2。不要仅仅依靠颜色。

  • 确保所有带有颜色的信息也可以不带颜色使用。

3。使用标记和样式表，并正确执行。

  • 使用样式表来控制布局和表示。
  • 在标记语言属性值和样式表属性值中使用相对单位，而不是绝对单位。例如，在CSS中，请使用'em'或百分比长度，而不是'pt'或'cm'。

4。创建可以正常转换的表。

  • 对于数据表，请标识行标题和列标题。例如，使用TD标识数据单元，使用TH标识标头。
  • 除非将表格线性化，否则请勿使用表格进行布局。改用CSS。

5，确保包含新技术的页面正常转换。

  • 确保关闭或不支持脚本，小程序或其他编程对象时，页面可用。如果不可能，请在其他可访问页面上提供等效信息。

6。提供上下文和方向信息。

  • 为每帧命名，以方便帧识别和导航。例如，在HTML中，在FRAME元素上使用“ title”属性。
  • 将标签与其控件明确关联。例如，在HTML中使用LABEL及其“ for”属性。

7。通过键盘界面使所有功能均可用。

8。帮助用户避免错误，并轻松纠正确实发生的错误。
