为新的Geronimo版本创建插件存储库的步骤
编辑页面 浏览空间 添加页面 新增新闻

警告：这些步骤尚未验证。经过验证后，请删除此评论

如果新服务器版本（n）支持先前版本（n-1）中的插件：

  1. 在site / trunk / docs / plugins / n-1 / geronimo-plugins.xml中为当前版本（n）添加新元素
  2. （可选的）
    • 将新的服务器版本（完整）发布到Maven存储库
    • 更新site / trunk / docs / plugins / n-1 / geronimo-plugins.xml以包括新服务器版本中的所有插件
  3. 更新geronimo网站

如果新的服务器版本（xy） 不支持以前版本的插件：

  1. （假定）分支服务器版本，并在其poms中设置版本（xy）
  2. 在服务器/分支/ xy / configs下的所有geronimo-plugin.xml文件中将元素更新为xy
  3. 在server / branches / xy / assemblies / * / src / main / var / config / config.xml中更新PluginInstallerGBean的存储库列表引用
  4. 构建新服务器版本并将其（完整）发布到Maven存储库
  5. 在site / trunk / docs / plugins / xy / geronimo-plugins.xml中为发布创建新的插件目录
    • 目前，这是通过将server / branches / xy / configs下的geronimo-plugin.xml文件串联并进行必要的调整手动创建的
  6. 在site / trunk / docs / plugins / plugin-repository-list-xytxt中为xy版本创建一个新的存储库列表
  7. 更新geronimo网站
