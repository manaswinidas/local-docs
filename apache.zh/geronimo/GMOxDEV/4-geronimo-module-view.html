Geronimo核心

ActiveMQ

ActiveMQ是消息总线的Apache实现。它支持JMS 1.1，并且与Geronimo分开开发。可以在此处找到更多信息。重要的是要注意该子系统，围绕ActiveMQ的Geronimo中的代码只是包装器（特别是GBean）。此子系统中包含三个模块。ge-activemq-rar模块包含ActiveMQ RAR（来自ActiveMQ项目），geronimo-activemq-gbean模块包含围绕ActiveMQ的GBean包装器。geronimo-activemq-gbean-management包含用于从外部访问子系统的接口。仅在ActiveMQ子系统之外使用这些接口可以在将来使用另一个MQ解决方案代替ActiveMQ。控制台是如何在外部（特别是外部可用接口）使用此子系统的一个示例。控制台通过geronimo-activemq-gbean-management中的接口访问子系统，该接口通过GBean检索。

阿帕奇

geronimo-directory提供对Geronimo中嵌入的LDAP服务器的访问。嵌入式LDAP服务器由ApacheDS项目提供，并用于Geronimo附带的一些演示应用程序。

应用客户端

Application Client子系统处理来自Geronimo实例外部存在的独立应用程序的调用。一个示例是对应用程序服务器的CORBA调用。

杰罗尼莫系统

geronimo系统模块包含运行Geronimo所需的许多核心代码。在内部，Geronimo使用GBean而不是JMX MBean来处理模块之间的通信。在GBean和MBean之间转换的代码与MBean服务器（基本上只是将调用委派给Geronimo Kernel）一起在此模块中。该模块中包含的其他内容是启动Geronimo服务器的代码和运行时日志记录信息。

杰罗尼莫交易

该模块是Geronimo对Java Transaction API规范的实现。该规范本质上具有4个必需的接口：UserTransaction，Transaction，TransactionManager和XAResource。在geronimo-transaction模块中可以找到这四个接口的实现。应用程序服务器将Transaction（TransactionImpl）的实现用于事务感知处理。Web容器（Jetty和Tomcat）使用UserTransaction实现（GeronimoUserTransaction）在已部署的Web应用程序中使用。当应用程序服务器自动管理应用程序的事务时（例如，通过某些EJB操作），将使用TransactionManager。最后，XAResource实现（通过Geronimo中的NamedXAResource使用）实现了X / Open CAE Specification事务标准。

聚类

Geronimo使用Codehaus的WADI软件来帮助其集群Geronimo服务器实例。geronimo集群模块包含更多与WADI无关的通用代码，例如，群集节点和分布式会话的接口。其他模块包含利用WADI将请求分发到服务器的代码。有关WADI的更多信息，请参见此处 。

J2EE管理模型

该子系统是Geronimo对Sun的J2EE管理模型（JSR 77）的实现。该子系统的目的是为Geronimo的特定实现提供通用接口。例如，在geronimo-management模块中，有一个称为JMSBroker的接口，由ActiveMQBroker实现。

贾斯

JAAS是Sun创建的身份验证和授权规范。Geronimo已经实现了该规范，供Geronimo中部署的应用程序使用。该子系统中有两个模块。geronimo-security模块包含实现JAAS规范的代码。

Java邮件

JavaMail是Sun提出的一种规范，用于抽象化使用Java发送电子邮件的一些细节。Geronimo包装了这些标准JavaMail类，以便可以通过GBeans访问和管理它们。javax.mail是一个如何包装它们的示例。会话课程。Session类管理配置和授权信息。在Geronimo中，可以通过org.apache.geronimo.mail访问Session类。geronimo-mail模块中的MailGBean。JavaMail还使用JavaBeans激活框架（JAF）。JavaBeans激活框架旨在帮助识别JavaBeans中各种属性的内容。对于JavaMail框架，它用于MIME类型。Geronimo扩展了JAF以处理来自MIME类型的数据，这些类位于geronimo-activation模块中。

J2EE连接器体系结构（JCA）

J2EE连接器体系结构是与供应商无关的规范，用于与企业信息系统（例如企业资源计划（ERP）系统）进行连接和通信。geronimo-connector模块中提供了Geronimo的实现。关联的geronimo-connector-builder模块用于将XML部署计划转换为运行时连接器。

Java命名和目录接口（JNDI）

该子系统包含Java命名和目录接口的实现。geronimo命名模块包含用于处理JNDI查找并将所有这些类通过GBean公开给Geronimo服务器其余部分的所有类。geronimo-naming-builder读取部署计划，并在运行的JNDI上下文中创建新条目。

Java持久性API（JPA）

Java Persistence API是添加到EJB 3.0的规范，以使EJB的持久化更加容易。Geronimo使用Apache的JPA的OpenJPA实现。关于它的更多信息可以在这里找到。关于JPA的Geronimo中代码的目的是将JPA代码包装在GBean中。在geronimo-persistence-jpa10模块中，PersistenceUnitGBean类允许通过GBean访问OpenJPA例程。geronimo-persistence-jpa10-builder模块在运行时根据XML配置文件创建持久性GBean。

共享

共享子系统包含许多其他子系统使用的模块，以及Geronimo随附的各种应用程序服务器实用程序和演示应用程序使用的代码。

杰罗尼莫·德比

Apache Derby是Geronimo附带的基于Java的关系数据库。它的内存占用空间小，可以将其嵌入应用程序中。在Geronimo中，Derby用于很多方面。首先，Geronimo附带了一些使用数据库的示例应用程序，而不是要求安装外部数据库，其中包括Derby。Derby还用于帮助创建可搜索的日志文件。geronimo-derby模块包装Derby并将其公开为GBean。

杰罗尼莫计时器

该模块为Geronimo创建了类似cron的系统。它利用java.util。计时器类可安排执行各种任务。它还允许任务是事务性的。

Geronimo转换器

这是一个小模块，其中包含将数据源配置从BEA Weblogic和JBoss Application Server中使用的格式转换为Geronimo的代码。控制台仅将其用于创建新的数据源。

geronimo升级

geronimo升级模块包含有助于从一个版本的Geronimo迁移到另一个版本的代码。此处的实用程序代码将部署计划的一个版本迁移到另一个版本。

geronimo-jmx远程处理

该模块处理与MBeanServer的远程连接。它通过JAAS处理身份验证，并将包含对GBean / MBean服务器当前正在运行的实例的引用。

网络容器

Jetty和Tomcat都是Web容器，可用于Geronimo。这些Web容器不是由Geronimo开发人员维护的，但是对于Jetty和Tomcat服务器的许多操作都有GBean包装器。这些子系统的代码旨在使容器适应Geronimo环境。

网页服务

用于在Geronimo内部部署Web服务的模块。Geronimo包含Axis的支持，Axis是SOAP的一种实现，可以在这里找到。geronimo-webservice和geronimo-webservice-builder模块更为通用。通用模块由Web容器使用，而不是直接使用Axis（因此，应用程序服务器的其余部分不会与Axis绑定）。
