本文介绍如何使用Open Terracotta项目对Geronimo HTTP Session Web应用程序数据进行集群。

打开兵马俑概述

Open Terracotta是一个支持透明集群的OSS项目。该项目目前有三个可用的软件包：

  • 集群POJO（Terracotta DSO软件包）
  • Terracotta HTTP会话群集（Terracotta会话包）
  • Terracota for Clustering Spring（Terracotta春季套餐）

本文将利用Terracotta会话包在Geronimo环境中对HTTP会话进行集群。

以下是Open Terracotta技术的一些亮点：

  • 应用程序开发人员不再需要为集群环境编写应用程序
  • 集群在运行时通过JVM级别的集群（JVM的字节码检测）透明地完成
  • Terracotta服务器负责在群集中的所有应用程序服务器之间透明地复制/维护状态
  • 群集成员似乎是一个大型全局JVM的一部分
  • Terracotta群集不需要“数据序列化”
  • 开发人员定义（在xml配置文件中）要集群的对象。对于Terracotta Sessions软件包，这包括标识要聚类的上下文根以及要放置在HttpSession中的数据类型。
  • Terracotta支持ACID（原子，一致，隔离，持久）合规性（集群成员之间的事务行为）
  • Open Terracotta支持跨集群成员的延迟加载

请注意，Open Terracotta不是Apache项目，根据Apache许可证当前不可用，而是根据Terracotta公共许可证提供。您应确定许可是否满足您的需求。

有关Open Terracotta项目的更多信息，请访问： http ： //wiki.terracotta.org/confluence/display/orgsite/Home

文章概述

本文介绍如何使用Open Terracotta对简单的Web应用程序的HttpSession数据进行聚类。所需步骤包括：

  • 安装Terracotta服务器
  • 安装Geronimo Server
  • 安装Terracotta客户端
  • 更新Terracotta配置文件
  • 启动Terracotta服务器
  • 启动Terracotta客户端和Geronimo Server
  • 在每个Geronimo集群成员上安装Web应用程序
  • 设置负载平衡和故障转移
  • 测试负载平衡和故障转移
  • 使用Terracotta管理控制台

安装Terracotta服务器installterracotta

需要将Terracotta安装在一台将用作Terracotta服务器的计算机上 。Terracotta服务器将负责维护会话状态，并确保它正确反映在每个Geronimo集群成员中。

下载Terracotta Sessions捆绑包并运行安装程序。本示例需要“ Open Terracotta 2.2.1 Stable 2”或更高版本。

安装Geronimo Server installgeronimo

需要在要集群的每台机器上安装Geronimo Application Server。本文中的示例将在两台单独的机器（集群成员）上安装Geronimo。

从Geronimo网站下载包含Tomcat Web容器的Geronimo映像，然后将其解压缩到每个集群成员上。此示例需要Geronimo 1.1.1和Tomcat Web容器。Terracotta尚不支持Jetty Web容器。

请注意，必须在要集群的每台机器上安装相同版本的Geronimo。Terracotta当前不支持混合版本的Geronimo。

安装Terracotta客户端installterracottaclient

需要在包含Geronimo集群成员的每台机器上安装Terracotta客户端。当前，相同的安装包包括Terracotta服务器和客户端。因此，可以在具有共同安装的同一台机器上运行Terracotta Server和Terracotta Client。

下载Terracotta Sessions捆绑包 ，然后在每个集群成员上运行安装程序。本示例需要“ Open Terracotta 2.2.1 Stable 2”或更高版本。

更新Terracotta配置文件configtc

tc-config.xml文件标识要群集的上下文根以及要放置在HttpSession中的数据类型。

本示例将使用Terracotta发行版随附的Terracotta配置文件。特定文件为：C：\ Terracotta \ terracotta-2.2.1 \ sessions \ sandbox \ tomcat5.5 \ tc-config.xml

更改tc-config.xml文件，使其类似于以下示例。请注意，下面的tc-config.xml示例仅显示要更新的xml项。该xml文件的其他部分应保持不变。请记住，将以下示例中的IP地址更改为Terracotta服务器的地址。

来自tc-config.xml 数据/服务器数据 日志/服务器日志的 xml 固体 摘录 ... SessionExample ... servlets-examples-terracotta

启动Terracotta Server starttc

启动Terracotta服务器，指定刚刚更新的tc-config.xml文件的位置。

  • cd C：\ Terracotta \ terracotta-2.2.1 \ sessions \ bin
  • start-tc-server.bat -fc：\ Terracotta \ terracotta-2.2.1 \ sessions \ sandbox \ tomcat5.5 \ tc-config.xml

启动Terracotta Client和Geronimo Server startgeronimo

Open Terracotta支持多种JVM，并且每一个都需要在启动Geronimo服务器时在bootclasspath中提供一个唯一的启动jar。下面的示例使用的是Terracotta Sessions软件包中提供的Windows JVM。尽管可能需要创建其他启动jar，但也可以使用其他JVM。另外，以下脚本中的某些启动参数是通过在c：\ Terracotta \ terracotta-2.2.1 \ sessions \ bin \目录中发出dso-env脚本获得的。您应该在系统上发出此命令以找到适合您的环境的参数。

以下示例中的IPAddress需要替换为Terracotta服务器的IP地址（或主机名）。该值将用于确定哪个Terracotta服务器将向Terracotta客户端提供其tc-config.xml文件。或者，可以省略-Dtc.config =“ IPAddress：9510”，尽管您需要在每个集群成员上更新tc-config.xml（例如c：\ Terracotta \ terracotta-2.2.1 \ sessions \ sandbox \ tomcat5.5 \ tc-config.xml）。查看Terracotta文档以获取更多详细信息。

这是Windows计算机上的启动示例：

启动Geronimo solid c：\ Terracotta \ terracotta-2.2.1 \ jre \ bin \ java.exe -Xbootclasspath / p：“ C：\ Terracotta \ terracotta-2.2.1 \ common \ lib \ dso-boot \ dso-boot- hotspot_win32_150_09.jar“ -Dtc.install-root =” C：\ Terracotta \ terracotta-2.2.1“ -Dtc.config =” IPAddress：9510“ -jar c：\ geronimo-1.1.1-TERRACOTTA \ geronimo-1.1。 1 \ bin \ server.jar

在每个Geronimo集群成员installapp上安装Web应用程序

使用Geronimo管理控制台将适当的示例应用程序安装到每个Geronimo安装中。

这里是示例应用程序和部署计划的链接。

  • servlets-examples-cluster-node1.war-集群成员1的Web应用程序
  • servlets-examples-cluster-node2.war-集群成员2的Web应用程序
  • servlets-examples-geronimo-plan.xml -Geronimo通用部署计划（针对两个集群成员）

请注意，上述两个应用程序是相同的，除了指示服务器名称的字段（例如，Server1，Server2）。这种区别在测试时将很有用。

  • 可以在每台机器上的http：// localhost：8080 / console上访问Geronimo管理控制台。
  • Geronimo管理控制台的默认ID和密码是id = system password = manager。
  • 从GUI选择“ Applications-> Deploy New”，然后按照面板上的说明安装示例应用程序

现在，通过使用您喜欢的Web浏览器访问应用程序来测试应用程序是否已正确部署：

http：// YourHost1：8080 / servlets-examples-terracotta / servlet / SessionExample
http：// YourHost2：8080 / servlets-examples-terracotta / servlet / SessionExample

如果此方法正常运行，现在是时候设置负载均衡器了。

设置负载平衡和故障转移installwebserver

现在，您可以设置负载均衡器了。我们建议在此示例中使用Apache HTTP服务器和mod_jk。

  • 安装Apache HTTP服务器-说明和下载位于http://httpd.apache.org/
  • 安装Apache mod_jk-请参阅http://tomcat.apache.org/tomcat-5.5-doc/balancer-howto.html

mod_jk的配置提示：

worker.list =负载均衡器，状态
worker.node1.port = 8009
worker.node1.host = _your.first.cluster.member.host.name_or_IP_address
worker.node1.type = ajp13
worker.node1.lbfactor = 1

worker.node2.port = 8009
worker.node2.host = _your.second.cluster.member.host.name_or_IP_address
worker.node2.type = ajp13
worker.node2.lbfactor = 1

worker.loadbalancer.type = lb
worker.loadbalancer.balance_workers = node1，node2
worker.loadbalancer.sticky_session = 1
worker.status.type =状态

/ servlets-examples-terracotta =负载均衡器
/ servlets-examples-terracotta / * = loadbalancer

测试负载平衡和故障转移测试

正确获得Apache HTPP Server和mod_jk设置后。您可以通过使用喜欢的Web浏览器请求以下URL来测试负载平衡和故障转移。当然，用实际的主机名或IP地址替换YourHost。

http：// YourHost / servlets-examples-terracotta / servlet / SessionExample

请注意，输出页面包含正在处理请求的服务器的ID。在浏览器窗口中，填写适当的输入字段，然后单击“提交”按钮。在购物车中添加更多物品。请注意，浏览器指示哪个服务器（Server1或Server2）正在为您的请求提供服务。没有粘性会话支持，您可能会看到负载均衡器在两个集群成员之间交替路由请求。如果您正确设置了所有内容，则浏览器请求在两台服务器之间移动后，会话中应该存在购物车数据。对于粘性会话，您可以通过停止当前正在为该会话提供服务的geronimo服务器并查看下一个http请求将故障转移到其余集群成员中来测试故障转移。来自先前请求的httpsession数据应被恢复并显示在刷新的浏览器窗口中。

使用Web浏览器进行测试时，请确保在测试用例之间擦除cookie和缓存的页面。测试时，浏览器缓存可能会引起混乱。

使用Terracotta管理控制台tcadmin

Terracotta管理控制台还可用于验证会话数据是否已在Terracotta服务器中正确复制。

发出以下命令

  • cd c：\ Terracotta \ terracotta-2.2.1 \ sessions \ bin
  • tc-admin.bat
  • 指定端口9520并选择“连接”
  • 向下浏览资源树，您将能够从浏览器中看到httpSession数据！

调查项目（可能在不久的将来成为常见问题解答）

  • 研究是否存在精简的Terracotta会话下载映像，该映像不包含Session Configurator，JDK，示例等。
  • 研究是否存在精简的Terracotta会话下载映像，该映像仅包含Terracotta Client。
  • 调查Open Terracotta是否计划支持Jetty Web容器。
  • 调查粘性会话支持。
  • 研究Open Terracotta何时支持Geronimo的EJB集群。
  • 研究Open Terracotta将何时支持IBM JDK。
