在Eclipse中开发Geronimo
编辑页面 浏览空间 添加页面 添加新闻

以下说明将使您可以将Geronimo作为Eclipse项目进入Eclipse 3.2工作区。您将能够使用eclipse进行代码更改并运行单元测试。为了便于讨论，开发操作系统是Windows。请根据您的环境调整这些说明。

先决条件

安装构建Apache Geronimo所需的软件，并按照说明构建Geronimo。

创建Eclipse项目

您可以导入工作空间中的所有geronimo项目或子项目。但是，一种更有效的方法是仅导入要使用的模块。

使用单个模块

如果计划修改单个geronimo模块，则可以通过运行以下命令将其仅导入工作空间中：

cd modules\a_module   or   cd maven-plugins\a_plugin
mvn -o eclipse:eclipse

使用多个模块

建议用户首次导入所有模块。
通过从顶部目录调用以下命令，为geronimo子项目生成Eclipse项目：

mvn -o eclipse:eclipse

注意：

第一次，您必须忽略“ -o”参数，以便它将下载maven-eclipse-plugin。

mvn eclipse:eclipse

通过从每个模块目录运行上述命令，可以仅导入工作空间中的几个模块。在这种情况下，如果不使用mvn重建修改后的模块，则一个模块中所做的更改将对其他模块不可见。

有用的信息

可以编辑.classpath文件，以便一个模块中所做的更改对工作区中的所有其他模块立即可见。这仅适用于高级用户。

启动Eclipse

  • 使用以下命令将项目导入到Eclipse工作区：
    File --> Import --> +General --> Existing Projects into Workspace
    在#Creating Eclipse Projects步骤中，根据您如何创建项目来选择一个或多个项目 。
  • 设置Maven类路径变量M2_REPO，以通过单击Eclipse中的“新建”按钮来告诉Eclipse Maven存储库在哪里。
    Window --> Preferences --> Java --> Build Path --> Classpath Variables
    输入M2_REPO并将其设置为您的maven2存储库目录。在Linux上，该目录通常位于
    ~/.m2/repository
    在窗户上通常
    "%USERPROFILE%\.m2\repository"
    。
  • 如果您正在使用构建器模块，即geronimo-tomcat6-builder，则必须将xmlbeans类添加到构建路径。
    右键单击构建器项目，然后通过执行以下操作选择“添加类文件夹”：
    <Project> -->  Build Path --> Configure Build Path --> Java Build PAth --> Add Class Folder
    从选择中检查target / clover / classes文件夹。
  • 为了防止Eclipse将.svn目录复制到项目的输出文件夹，请执行以下操作：
    使用展开“输出文件夹”设置
    Window --> Preferences --> Java --> Compiler --> Building --> Output folder
    将* .svn附加到“过滤的资源”字段。

代码格式化

  • 如果计划为项目贡献代码或补丁，请通过以下操作将eclipse配置为不使用选项卡：
    Window --> Preferences --> Java --> Code Style --> Formatter --> Java Conventions [built-in]
    点击“编辑”。在“缩进”下，将“制表符”策略选择为“仅空格”。输入此配置文件的新名称，然后单击“确定”。

Eclipse IDE配置文件

您可能要导入以下文件，然后单击几次即可配置环境。

Window --> Preferences --> Java --> Code Style --> Formatter --> Import
用于Geronimo的Eclipse代码样式格式化程序配置文件
Window --> Preferences --> Java --> Code Style --> Code Templates --> Import

用于Geronimo的Eclipse代码样式代码模板
