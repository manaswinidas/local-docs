遮盖密码
编辑页面 浏览空间 添加页面 添加新闻

（适用于geronimo 2.0.2和更高版本）

几个geronimo配置文件和登录模块后备存储可以包含密码，并且一些用户希望在不同程度上掩盖这些密码。我使用“晦涩”一词，因为geronimo需要阅读它们才能起作用，因此任何具有文件系统访问权限geronimo安装的人都将能够弄清楚究竟是什么密钥可以对它们进行解码。除非我们需要启动geronimo的凭据，否则无法消除将密钥存储在像geronimo安装一样不受保护的地方的方法。

密码存储在两种地方。其中之一是geronimo配置，例如部署计划，var / config / config.xml文件以及可能的var / config / config-substitutions.properties（当前未用于此目的）。目前仅var / config / config.xml被遮盖。另一个是登录模块后备存储，例如用于属性文件和sql登录模块的存储。当前，ldap登录模块不支持这种晦涩的样式。请注意，这些后备存储区也可以使用密码哈希，在此不做讨论。geronimo配置文件不能使用密码哈希，因为geronimo需要使用这些密码来访问各种受保护的资源，而不仅仅是将提供的密码与存储的密码进行比较。

简而言之，此模糊方法适用于config.xml中名为“ password”的gbean属性以及存储在属性文件和sql登录模块后备存储中的密码。

这样可以防止某人从这些文件之一中逐字复制密码，并使您的密码保持可用（即使您在本地丢失密码，该密钥也位于svn中），但当然任何人都可以查找该密钥并解码该密码。

如果您想使自己更加晦涩难懂，则可能会丢失密钥，并使所有密码完全无法使用。

通常，要安装其他遮盖策略，您需要编写一个实现org.apache.geronimo.util的GBean。加密接口（在geronimo-util模块中）。启动时，它将在EncryptionManager中注册并重新加密所有现有的加密密码，并将其用于以后的所有密码加密/解密。

我们提供了一个这样的gbean org.apache.geronimo.system.util。geronimo系统模块中的ConfiguredEncryption。gbean配置需要包括密钥的位置和对ServerInfo的引用。该位置将使用ServerInfo相对于服务器位置进行解析。如果缺少密钥，将使用当前时间播种的SecureRandom创建密钥。与默认设置一样，它使用AES。
