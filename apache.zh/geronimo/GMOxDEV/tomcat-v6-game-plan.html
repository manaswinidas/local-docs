本页概述了将tomcat v6集成到Geronimo2.0中的游戏计划

阶段1-更新规格

Tomcat v6具有以下规格依赖性

  • geronimo-servlet_2.5_spec
    • 已经在geronimo / specs / trunk中
  • geronimo-annotation_1.0_spec
    • 已经在geronimo / specs / trunk中
  • geronimo-jsp_2.1_spec
    • 从https://svn.apache.org/repos/asf/tomcat/tc6.0.x/trunk/java/javax/servlet/jsp/复制
  • 杰罗尼莫尔
    • 从https://svn.apache.org/repos/asf/tomcat/tc6.0.x/trunk/java/javax/el/复制

将所有这些规范发布到Maven存储库中

第2阶段-更新模块和配置

  • 请求tomcat开发团队在maven存储库中发布6.0.2 beta版本
    • http://issues.apache.org/bugzilla/show_bug.cgi?id=41093
  • 在本地构建tomcat v6.0.2 beta，并在临时M2存储库中发布
    • http://people.apache.org/~pmcmahan/maven2/
    • 使用groupid“ tomcat6”避免Maven错误（我怀疑MNG2340）
    • 用过的groupid org / apache / tomcat
  • 模块/ geronimo-tomcat
    • 将存储库http://people.apache.org/~pmcmahan/maven2/添加到pom
      • 这将在tomcat发布v6工件时删除
    • 更新pom以使用tomcat 6.0.2 beta罐子
    • 删除过时的群集（oagtomcat.cluster）
    • 更新src / main / resources / META-INF / geronimo-tomcat / var / catalina中的资源
    • 更新src / main / resources / META-INF / geronimo-dependency.xml中的依赖项
    • 更新测试用例
  • 模块/ geronimo-tomcat-builder
    • 删除对集群支持的过时引用
    • 更新测试用例
  • 配置/ tomcat
    • 将存储库http://people.apache.org/~pmcmahan/maven2/添加到pom
      • 这将在tomcat发布v6工件时删除
  • configs / webconsole-tomcat
    • 从pom.xml和plan.xml删除对jasper运行时的依赖
  • configs / jee5-specs
    • 使用Servlet 2.5，JSP 2.1，EL 1.0，注释1.0
  • configs / tomcat-deployer
    • 更新pom.xml以使用新的web25-builder
  • 组件/ geronimo-tomcat-j2ee
    • 重命名为geronimo-tomcat-jee5
    • 从jetty6-jee程序集中同步pom.xml和config.xml
  • 管理控制台的粗略测试
  • 测试用例
    • 向geronimo-tomcat添加servlet 2.5的新单元测试
    • 从命令行和管理控制台测试2.5 WAR的部署
    • 测试tomcat-minimal和jetty6-jee5组件的启动
