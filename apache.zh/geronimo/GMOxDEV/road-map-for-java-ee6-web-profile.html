Java EE 6 Web Profile的基础

Java EE 6 Web概要文件是针对支持“典型” Web应用程序的小型服务器的一种指定的最低配置。Web概要文件是最低规范，因此可以免费添加附加服务或允许应用程序提供附加服务的实现。网络配置文件的必需元素是：

  • Java EE 6（JSR-316）
  • Servlet 3.0（JSR-315）
  • JavaServer页面（JSP）2.2（JSR-245）
  • 表达语言（EL）2.2（JSR-245）
  • 其他语言1.0的调试支持（JSR-45）
  • JavaServer Pages（JSTL）1.2（JSR-52）的标准标记库
  • JavaServer Faces（JSF）2.0（JSR-314）
  • Java Platform 1.1（JSR-250）的通用注释
  • 企业JavaBeans（EJB）3.1 Lite（JSR-318）
  • Java Transaction API（JTA）1.1（JSR-907）
  • Java持久性API（JPA）2.0（JSR-317）
  • Java 1.0的依赖注入（JSR-330）
  • Java EE平台1.0的上下文和依赖注入（JSR-299）
  • Bean验证1.0（JSR-303）
  • 托管Beans 1.0（JSR-316）
  • 拦截器1.1（JSR-318）

下面详细介绍了每种方法的当前状态。对于其中的某些组件，Geronimo已经处于当前级别，对于其他组件，我们将需要查找更新的版本。希望这只是我们已经使用过的组件的新版本，但这可能还需要直接将某些元素作为Geronimo的一部分进行开发。

另外，Geronimo规范树包含这些不同接口的API类版本，因此我们可能需要开发API版本的新版本。

作为开发这些工具的第一步，我们可能希望使用Geronimo中当前使用的Java EE 5组件创建一个插件组，作为Web配置文件的一个实施例。这将提供一个平台，用于试验如何为较小的占用空间配置服务器以及如何将其他服务添加到最小配置文件配置中。另外，随着兼容Java EE 6的实现可用，我们可以创建其他插件，以允许将较新的版本交换到配置中。

Java EE 6的所有概要文件都需要新的“ java：global”，“ java：app”和“ java：module”命名空间。以及@Resource和@EJB已更新为具有新的“ lookup”属性。

OpenEJB的

Java EE 6活动的状态

像网络配置文件本身一样，这似乎描述了一个最小的功能集，但是如果需要的话，实现可以提供比最小功能更多的功能。一旦实现3.1功能集，这似乎相当容易实现。3.1功能的某些部分已经在OpenEJB中实现。

OpenEJB中尚未完成的主要EJB 3.1功能是异步接口，基于调度的计时器和无接口视图。在这三个中，EJB Lite仅需要无接口视图。最近几个月对EJB 3.1规范的更改需要进行一些更新，这些更新主要围绕部署描述符，附加注释，Singleton bean生命周期调整以及对官方javax.ejb.embedded的支持。EJBContainer API基于现有的OpenEJB InitialContext /嵌入式API。完成无接口视图和EJB Lite概要文件所需的更新所需时间的粗略估计分别为2-3周和4-6周。异步调用/接口的粗略估计为3-5周，基于计划的计时器支持4-?。基于时间表的计时器支持是最大的任务，它基于Cron和Quartz等事实上的标准，但又有足够的区别，以致无法按原样使用任何现有的库。实际完成日期取决于在其他优先级上花费的时间。

Geronimo中要完成的一项主要的新Java EE 6 / EJB / Servlet特性是能够将EJB包含在.war文件中，这是Java EE 6 Web Profile的一部分。Web应用程序中的所有EJB都共享Web应用程序的类加载器和Web应用程序的java：comp / env命名空间。还需要更新Geronimo部署过程，以将所有Web应用程序视为EJB支持该功能的潜在来源。粗略估计是戴维·布列文斯（David Blevins）和戴维·詹克斯（David Jencks）的时间大概是3-4周。

附加条款

OpenEJB的嵌入式版本能够处理从Lite配置文件版本到完整功能（减去CORBA集成（由Geronimo处理））不同子系统的动态配置。看来Geronimo方面需要做一些工作来创建配置文件/插件，这些配置文件/插件将有选择地启动将OpenEJB堆栈的不同部分与配置集成在一起的GBean。当前，Geronimo EJB配置是一个全有或全无的主张。

当前正在asf / openejb / trunk / openejb3 / api / ejb31-api-experimental /中开发的EJB 3.1 API库，计划在规范关闭后将其移至asf / geronimo / specs / trunk / geronimo-ejb_3.1_spec。目前的截止日期定为2009年9月。
