过时的内容

此页面的内容过时，可能包含或可能不包含相关，有用或正确的信息。

这是从2006年4月的Geronimo用户邮件列表中确定的用户要求。

（X）表示该项目的请求数量。

规格/功能

  • 用于Geronimo的JDK 5 （12）
  • JEE 5 （4）
    • 企业JavaBeans 3.0 （5）
    • JAX-WS支持（4）
    • Java Servlet 2.5
    • Java ServerPages 2.1
    • Java ServerPages标准标记库
    • Java持久性体系结构（这是Java EE的一部分吗？）
    • Java交易API（JTA）
    • J2EE管理
    • J2EE连接器架构
  • GBean的改进（文档，生命周期，依赖项，深度注入等）（2）
  • 动态查询（2）
  • javax.persistence
  • 带注释的会话bean。
  • 支持JSR-168（Portlet API）
  • 服务混合
  • Maven2支持
  • 服务/守护程序包装器
  • 删除openejb-jar.xml的要求？
  • 管理控制台中更好的数据库工具
  • 配置管理，可以在不具有某些当前模块的情况下制作生产版本（例如，OpenEJB或ActiveMQ），
    有些最终用户没有足够的资源（内存，磁盘）来运行功能齐全的Geronimo，并且他们不需要
    J2EE堆栈的功能
  • 持续支持码头
  • 一流的HttpSession集群。
  • 与Apache mod_JK / mod_ajp的更多集成。如果请求将继续通过一个池直到到达，那就太好了
    在具有该Web应用程序的服务器上。
  • 大规模部署工具，允许Geronimos初期提到的服务器“克隆”和推出。
  • 全球JNDI ENC
  • 集成或提供Struts Frameworks，JSF，Pluto等东西作为插件。

工具类

  • NetBeans支持（4）
  • JDK 5，用于启动和运行Geronimo Eclipse插件（2）
  • Eclipse插件改进（确实不错，但认为可能是
    更好）
  • Eclipse迷你路线图（来自Sachin）
    • 直接从工作空间运行资源，因此不会在每次发布时构建耳朵并重新打包
    • 更好地控制运行时/服务器向导，发布过程和服务器管理
    • 继续构建UI，以便为G部署计划提供一整套“表单编辑器”
    • 修改计划时，可以在Common Navigator之类的工具中实时查看反映的更改
    • 源视图和使用模型的视图之间的完全同步
    • 复制/粘贴，撤消/重做支持
    • WTP中还计划改进服务器工具框架，以使其更容易对定义运行时的内容进行更多控制。
  • 改进的IDE集成，不仅与Eclipse集成，还包括对Netbeans，IntelliJ等的支持。 。

易用性

  • 部署（6）
    • 更少的部署要求（更简单的计划，更多的默认设置等）
    • 简化部署（生成geronimo部署计划的某些方法？
    • 部署EAR文件的简便方法
    • 部署时进行更多应用程序验证
    • 更好的重新部署，以防止请求在重新部署期间遇到该服务器时失败。
  • 更强大的文本配置
  • 从Tomcat到Geronimo的迁移路径
  • 构建Web服务的快捷方式
  • 功能齐全的控制台（自动执行包括创建部署计划在内的所有任务）关注易用性，这是一般人所需要的。我需要运行我的应用程序。我不想担心1001渔获物和解决方法或程序。

处理

  • 发行频率更高，并包含更多的用户反馈（较小的发行版本相对于较大的发行版本每年仅出现4次） （7）
  • Geronimo认证合作伙伴计划（包括Jetspeed-2成为会员）。（2）
  • 需要快速将JIRA分配给所有者（太多未分配的JIRA）
  • 解决双重Wiki情况（动荡，汇合）
