部署小程序

您可以使用runApplet 部署工具包脚本的功能。的runApplet该功能确保客户机上存在所需的Java Runtime Environment（JRE）软件的最低版本，然后运行该applet。的runApplet函数生成HTML 标记提供的信息。

注意：在尝试查看网页源时，根据浏览器的类型，您可能无法查看Deployment Toolkit脚本生成的HTML。要查看生成的HTML，请在加载HTML页面后尝试保存它，或使用Firebug（Mozilla Firefox附加组件）之类的工具。

您可以通过将部署选项指定为应用程序的属性和参数来部署小程序。 标签。您还可以在Java网络启动协议（JNLP）文件中指定部署选项，以利用高级功能。有关此协议的更多信息，请参见Java网络启动协议主题。

注意： 

如果客户端没有所需的最低JRE软件版本，则Deployment Toolkit脚本会将浏览器重定向到http://www.java.com允许用户下载最新的JRE软件。在某些平台上，可能需要重定向用户，然后才能查看包含小程序的网页。

的参数runApplet根据您是否使用JNLP，功能会有所不同。仅当客户端计算机上存在下一代Java Plug-in软件（Java平台标准版6更新10版本中引入了下一代Java Plug-in软件）时，使用JNLP部署的Applet才能运行。

下一节说明如何使用runApplet HTML页面中显示小程序的功能。描述了以下使用场景：

  • 将部署选项指定为属性和参数名称/值对
  • 使用jnlp_href参数以在JNLP文件中指定部署选项
  • 指定属性和参数名称/值对以及一个JNLP文件 （使applet可以在旧的和下一代Java Plug-in软件上运行）

功能签名： runApplet: function(attributes, parameters, minimumVersion)

参数：

  • attributes –生成的属性的名称和值标签
  • parameters –的名称和值生成的标签标签
  • minimumVersion –运行此小程序所需的JRE软件的最低版本

    • 用法：

    • 将部署选项指定为属性和参数名称/值对

      作为名称-值对传递的属性和参数将作为属性写出并嵌套生成的标签标签。以这种方式部署的Applet可以由旧的Java Plug-in软件运行。

      // launch the Java 2D applet on JRE version 1.6.0
// or higher with one parameter (fontSize)
<script src=
    "https://www.java.com/js/deployJava.js"></script>
<script>
    var attributes = {code:'java2d.Java2DemoApplet.class',
        archive:'Java2Demo.jar', width:710, height:540};
    var parameters = { fontSize:16, permissions:'sandbox' };
    var version = '1.6';
    deployJava.runApplet(attributes, parameters, version);
</script>

      打开DeployUsingNameValuePairs.html在浏览器中查看Java2D小程序。

      注意：如果看不到小程序正在运行，则至少需要安装Java SE Development Kit（JDK）7版本。
    • 使用jnlp_href参数以在JNLP文件中指定部署选项

      属性和参数（ jnlp_href （在这种情况下）作为名称-值对传递的属性将作为属性写出并嵌套生成的标签标签。以这种方式部署的Applet只能由下一代Java Plug-in软件运行。最好将小程序的宽度和高度指定为以下属性：

      <script src="https://www.java.com/js/deployJava.js"></script>
<script> 
    var attributes = { code:'java2d.Java2DemoApplet', width:710, height:540 }; 
    var parameters = { jnlp_href: 'java2d.jnlp' }; 
    deployJava.runApplet(attributes, parameters, '1.6'); 
</script>

      打开DeployUsingJNLP.html在浏览器中查看Java2D小程序。

      注意：如果看不到小程序正在运行，则至少需要安装Java SE Development Kit（JDK）6 update 10发行版。
    • 指定属性和参数名称/值对以及 JNLP文件

      仅当最终用户在其浏览器上运行下一代Java插件软件时，使用JNLP部署的Applet才会运行。如果您还希望小程序也可以在旧的Java插件软件上运行，请使用属性和参数名称/值对以及 JNLP文件指定部署选项。

          
<script src="https://www.java.com/js/deployJava.js"></script>
<script>  
    var attributes = {code:'java2d.Java2DemoApplet.class', 
            archive:'Java2Demo.jar', width:710, height:540}; 
    var parameters = { fontSize:16, jnlp_href:'java2d.jnlp' }; 
    var version = '1.6' ; 
    deployJava.runApplet(attributes, parameters, version);      
</script>

      • 如果某些部署选项在属性名称-值对和JNLP文件中具有不同的值，那么以下准则会有所帮助：

      • 指定width和height作为属性名称/值对（不在JNLP文件中）。
      • 指定参数，例如image和boxbgcolor作为参数名称/值对（不在JNLP文件中）。在applet启动过程的早期，需要这些参数。
      • 在JNLP文件中，保留codebase属性为空或指定绝对URL。当。。。的时候codebase属性保留为空，默认为包含JNLP文件的目录。
      • 如果使用JNLP文件启动了applet，则code ， codebase和archive属性取自JNLP文件。如果这些属性也分别指定为属性名称-值对，则将忽略属性名称-值对。

      打开DeployUsingNameValuePairsAndJNLP.html在浏览器中查看Java2D小程序。

      注意：如果看不到小程序正在运行，则至少需要安装Java SE Development Kit（JDK）7版本。

      下载 “ 运行小程序”示例的源代码以进行进一步试验。
