小程序可以做什么和不能做什么

当用户访问包含小程序的页面时，Java小程序会加载到客户端上。Java小程序背后的安全模型的设计目标是保护用户免受恶意小程序的侵害。

小程序可以是沙箱小程序，也可以是特权小程序。沙箱小程序在仅允许一组安全操作的安全沙箱中运行。特权小程序可以在安全沙箱外部运行，并且具有访问客户端的广泛功能。

未签名的小程序仅限于安全沙箱，并且仅在用户接受小程序时运行。由来自公认的证书颁发机构的证书签名的小程序可以仅在沙箱中运行，也可以请求在沙箱外部运行的权限。在这两种情况下，用户都必须接受小程序的安全证书，否则，小程序将无法运行。

建议您使用Java网络启动协议（JNLP）启动小程序，以利用扩展的功能并改善用户体验。请参阅部署Applet ，以获取有关Applet部署的逐步说明。

建议您将小程序部署到Web服务器，甚至进行测试。要在本地运行小程序，请将小程序添加到例外站点列表，该列表可从Java控制面板的“安全性”选项卡进行管理。

在本主题中，我们将讨论applet的安全限制和功能。

沙盒小程序

沙箱小程序仅限于安全沙箱，并且可以执行以下操作：

  • 他们可以建立与主机和端口的网络连接。协议必须匹配，并且如果使用域名来加载小程序，则必须使用域名而不是IP地址来连接回主机。
  • 他们可以使用showDocument的方法java.applet.AppletContext类。
  • 他们可以在同一页面上调用其他小程序的公共方法。
  • 从本地文件系统（从用户目录中的目录CLASSPATH ）没有通过网络加载的applet的任何限制。
  • 他们可以读取安全的系统属性。请参阅系统属性以获取安全系统属性的列表。
  • 通过使用JNLP启动时，沙盒小程序还可以执行以下操作：
    • 他们可以在客户端上打开，读取和保存文件。
    • 他们可以访问共享的系统范围的剪贴板。
    • 他们可以访问打印功能。
    • 他们可以在客户端上存储数据，决定应如何下载和缓存小程序，等等。有关使用JNLP API开发小程序的更多信息，请参见JNLP API。

沙盒小程序无法执行以下操作：

  • 他们无法访问客户端资源，例如本地文件系统，可执行文件，系统剪贴板和打印机。
  • 他们无法连接到任何第三方服务器（源于该服务器的服务器以外的任何服务器）或从中检索资源。
  • 他们无法加载本机库。
  • 他们无法更改SecurityManager。
  • 他们无法创建ClassLoader。
  • 他们无法读取某些系统属性。请参阅系统属性以获取禁止的系统属性的列表。

特权小程序

特权小程序没有对沙箱小程序施加的安全限制，并且可以在安全沙箱外部运行。

注意： JavaScript代码被视为未签名的代码。当从HTML页面中的JavaScript代码访问特权小程序时，该小程序将在安全沙箱中执行。这意味着特权小程序的行为本质上类似于沙盒小程序。

有关如何使用小程序的信息，请参阅Rich Internet Applications中的安全性 。
