开发Java Web Start应用程序

使用基于组件的体系结构设计的软件可以很容易地开发和部署为Java Web Start应用程序。考虑带有基于Swing的图形用户界面（GUI）的Java Web Start应用程序的示例。通过基于组件的设计，可以使用较小的构建块或组件来构建GUI。以下一般步骤用于创建应用程序的GUI：

  • 创建一个MyTopJPanel属于的子类的类JPanel 。将应用程序的GUI组件布置在MyTopJPanel类。
  • 创建一个名为MyApplication那是JFrame类。
  • 在里面main的方法MyApplication类，实例化MyTopJPanel类并将其设置为JFrame 。

以下各节通过使用“动态树演示”应用程序来更详细地探讨这些步骤。如果您不熟悉Swing，请参阅使用Swing 创建GUI以了解有关使用Swing GUI组件的更多信息。

单击下面的启动按钮以启动动态树演示应用程序。

注意：如果看不到示例正在运行，则可能需要在浏览器中启用JavaScript解释器，以便Deployment Toolkit脚本能够正常运行。

创建应用程序

对于具有基于Swing的GUI的应用程序，创建一个类，该类是以下子类的子类javax.swing.JFrame 。

实例化您的顶部JPanel类并将其设置为JFrame在应用程序的main方法。的main的方法DynamicTreeApplication类调用createGUI AWT Event Dispatcher线程中的方法。

package webstartComponentArch;

import javax.swing.JFrame;

public class DynamicTreeApplication extends JFrame {
    public static void main(String [] args) {
        DynamicTreeApplication app = new DynamicTreeApplication();
        app.createGUI();
    }

    private void createGUI() {
        //Create and set up the content pane.
        DynamicTreePanel newContentPane = new DynamicTreePanel();
        newContentPane.setOpaque(true); 
        setContentPane(newContentPane);
        setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
        pack();
        setVisible(true);
    }    
}

将核心功能与最终部署机制分离的好处

创建应用程序的另一种方法是只删除抽象层（单独的顶部JPanel ），并布置应用程序的所有控件main方法本身。直接在应用程序的界面中创建GUI的缺点main方法是，如果您以后选择将功能部署为applet，将会更加困难。

在“动态树演示”示例中，核心功能分为以下几个部分： DynamicTreePanel类。现在放下DynamicTreePanel分成一个JApplet并将其部署为applet。

因此，为了保持可移植性并保持部署选项开放，请遵循本主题中所述的基于组件的设计。
