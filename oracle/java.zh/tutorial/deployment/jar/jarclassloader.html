JarClassLoader类

JarClassLoader类扩展了java.net。URLClassLoader 。顾名思义， URLClassLoader设计用于加载通过搜索一组URL访问的类和资源。 URL可以引用目录或JAR文件。

除了子类化URLClassLoader之外 ， JarClassLoader还利用其他两个与JAR相关的新API中的功能，即java.util.jar包和java.net。JarURLConnection类。在本节中，我们将详细研究JarClassLoader的构造函数和两个方法。

JarClassLoader构造函数

构造函数采用java.net的实例。URL作为参数。传递给此构造函数的URL将在JarClassLoader中的其他地方使用，以查找要从中加载类的JAR文件。

public JarClassLoader(URL url) {
    super(new URL[] { url });
    this.url = url;
}

URL对象将传递给超类URLClassLoader的构造函数，该构造函数将URL []数组而不是单个URL实例作为参数。

getMainClassName方法

一旦使用捆绑了JAR的应用程序的URL构造了JarClassLoader对象，就需要一种方法来确定JAR文件中的哪个类是应用程序的入口点。这就是getMainClassName方法的工作：

public String getMainClassName() throws IOException {
    URL u = new URL("jar", "", url + "!/");
    JarURLConnection uc = (JarURLConnection)u.openConnection();
    Attributes attr = uc.getMainAttributes();
    return attr != null
                   ? attr.getValue(Attributes.Name.MAIN_CLASS)
                   : null;
}

您可能从上一课中还记得，由JAR文件清单的Main-Class标头指定了JAR捆绑的应用程序的入口点。为了了解getMainClassName如何访问Main-Class标头值，让我们详细看一下该方法，并特别注意它使用的新JAR处理特性：

JarURLConnection类和JAR URL

getMainClassName方法使用java.net指定的JAR URL格式。JarURLConnection类。JAR文件的URL的语法如下例所示：

jar:http://www.example.com/jarfile.jar!/

终结！/分隔符表示URL引用了整个JAR文件。分隔符后面的所有内容均指代特定的JAR文件内容，如本例所示：

jar:http://www.example.com/jarfile.jar!/mypackage/myclass.class

getMainClassName方法的第一行是：

URL u = new URL("jar", "", url + "!/");

该语句构造一个表示JAR URL的新URL对象，并附加!！/分隔符，用于创建JarClassLoader实例的URL。

java.net。JarURLConnection类

此类表示应用程序和JAR文件之间的通信链接。它具有访问JAR文件清单的方法。 getMainClassName的第二行是：

JarURLConnection uc = (JarURLConnection)u.openConnection();

在此语句中，在第一行中创建的URL实例打开一个URLConnection 。然后将URLConnection实例强制转换为JarURLConnection，以便它可以利用JarURLConnection的JAR处理功能。

获取清单属性：java.util.jar。属性

将JarURLConnection打开到JAR文件后，可以使用JarURLConnection的getMainAttributes方法访问JAR文件清单中的头信息。此方法返回java.util.jar的实例。Attributes ，一个映射JAR文件中的标头名称的类及其关联的字符串值。getMainClassName中的第三行创建一个Attributes对象：

Attributes attr = uc.getMainAttributes();

要获取清单的Main-Class标头的值， getMainClassName的第四行调用Attributes.getValue方法：

return attr != null
               ? attr.getValue(Attributes.Name.MAIN_CLASS)
               : null;

方法的参数， 属性。名称。MAIN_CLASS ，指定它是您想要的Main-Class标头的值。（ 属性。Name类还提供了MANIFEST_VERSION ， CLASS_PATH和SEALED之类的静态字段，用于指定其他标准清单标头。）

invokeClass方法

我们已经看到JarURLClassLoader如何在JAR捆绑的应用程序中识别主类。最后考虑的方法JarURLClassLoader.invokeClass ，使该主类可以被调用以启动JAR捆绑的应用程序：

public void invokeClass(String name, String[] args)
    throws ClassNotFoundException,
           NoSuchMethodException,
           InvocationTargetException
{
    Class c = loadClass(name);
    Method m = c.getMethod("main", new Class[] { args.getClass() });
    m.setAccessible(true);
    int mods = m.getModifiers();
    if (m.getReturnType() != void.class || !Modifier.isStatic(mods) ||
        !Modifier.isPublic(mods)) {
        throw new NoSuchMethodException("main");
    }
    try {
        m.invoke(null, new Object[] { args });
    } catch (IllegalAccessException e) {
        // This should not happen, as we have disabled access checks
    }
}

invokeClass方法带有两个参数：应用程序的入口点类的名称和传递给入口点类的main方法的字符串参数数组。首先，加载主类：

Class c = loadClass(name);

loadClass方法是从java.lang继承的。ClassLoader 。

加载主类后，将使用java.lang.reflect包的反射API将参数传递给该类并启动它。您可以参考The Reflection API上的教程以了解反射。
