使用RowId对象

注意 ：MySQL和Java DB当前不支持RowId JDBC接口。因此，没有JDBC教程示例可用来演示本节中描述的功能。

一种RowId对象表示数据库表中某一行的地址。但是请注意， ROWID type不是标准的SQL类型。 ROWID值很有用，因为它们通常是访问单行的最快方法，并且是表中行的唯一标识。但是，您不应使用ROWID值作为表的主键。例如，如果您从表中删除特定行，则数据库可能会重新分配其行。 ROWID值插入到稍后插入的行中。

涵盖以下主题：

  • 检索RowId对象
  • 使用RowId对象
  • RowId有效期

检索RowId对象

检索一个java.sql.RowId对象，方法是调用接口中定义的getter方法ResultSet和CallableStatement 。的RowId返回的对象是一个不变的对象，您可以将其作为行的唯一标识符用于后续引用。以下是调用ResultSet.getRowId方法：

java.sql.RowId rowId_1 = rs.getRowId(1);

使用RowId对象

您可以设置一个RowId对象作为参数化中的参数PreparedStatement宾语：

Connection conn = ds.getConnection(username, password);
PreparedStatement ps = conn.prepareStatement(
    "INSERT INTO BOOKLIST" +
    "(ID, AUTHOR, TITLE, ISBN) " +
    "VALUES (?, ?, ?, ?)");
ps.setRowId(1, rowId_1);

您还可以使用特定的内容更新列RowId可更新的对象ResultSet宾语：

ResultSet rs = ...
rs.next();
rs.updateRowId(1, rowId_1);

一种RowId对象值通常不能在数据源之间移植，并且在以下情况下使用set或update方法时应将其视为特定于数据源的PreparedStatement和ResultSet对象。因此，建议不要使用RowId来自的对象ResultSet连接到一个数据源的对象，然后尝试使用相同的对象RowId无关对象ResultSet连接到不同数据源的对象。

RowId有效期

一种RowId只要未删除所标识的行并且该对象的生存期，该对象就有效RowId对象在该数据源所指定的生存期的范围内RowId 。

确定寿命RowId数据库或数据源的对象，调用方法DatabaseMetaData.getRowIdLifetime 。它返回a的值RowIdLifetime枚举数据类型。以下方法JDBCTutorialUtilities.rowIdLifeTime返回的生存期RowId对象：

public static void rowIdLifetime(Connection conn)
    throws SQLException {

    DatabaseMetaData dbMetaData = conn.getMetaData();
    RowIdLifetime lifetime = dbMetaData.getRowIdLifetime();

    switch (lifetime) {
        case ROWID_UNSUPPORTED:
            System.out.println("ROWID type not supported");
            break;

        case ROWID_VALID_FOREVER:
            System.out.println("ROWID has unlimited lifetime");
            break;

        case ROWID_VALID_OTHER:
            System.out.println("ROWID has indeterminate lifetime");
            break;

        case ROWID_VALID_SESSION:
            System.out.println(
                "ROWID type has lifetime that " +
                "is valid for at least the " +
                "containing session");
            break;

        case ROWID_VALID_TRANSACTION:
            System.out.println(
                "ROWID type has lifetime that " +
                "is valid for at least the " +
                "containing transaction");
            break;
    }
}
