使用行集对象

一个JDBC RowSet对象保存表格数据的方式比结果集更灵活，更易于使用。

甲骨文定义了五个RowSet界面的一些更流行的用途RowSet ，并且标准参考适用于这些RowSet接口。在本教程中，您将学习如何使用这些参考实现。

这些版本的RowSet提供接口和它们的实现是为了方便程序员。程序员可以自由编写自己的版本javax.sql.RowSet接口，以扩展这五个的实现RowSet接口，或编写自己的实现。但是，许多程序员可能会发现标准参考实现已经满足了他们的需求，并将照常使用它们。

本节向您介绍RowSet接口以及扩展该接口的以下接口：

  • JdbcRowSet
  • CachedRowSet
  • WebRowSet
  • JoinRowSet
  • FilteredRowSet

涵盖以下主题：

  • RowSet对象可以做什么？
  • 行集对象的种类

RowSet对象可以做什么？

所有RowSet对象是从ResultSet接口并因此共享其功能。是什么使JDBC RowSet对象的特殊之处在于它们添加了以下新功能：

  • 充当JavaBeans组件
  • 添加滚动性或可更新性

充当JavaBeans组件

所有RowSet对象是JavaBeans组件。这意味着他们具有以下优势：

  • 物产
  • JavaBeans通知机制

物产

所有RowSet对象具有属性。属性是具有相应的getter和setter方法的字段。属性公开给构建器工具（例如，IDE JDveloper和Eclipse附带的那些工具），使您可以直观地操作bean。有关更多信息，请参阅JavaBeans路径中的“ 属性”课程。

JavaBeans通知机制

RowSet对象使用JavaBeans事件模型，在该模型中，发生某些事件时会通知已注册的组件。对所有人RowSet对象，三个事件触发通知：

  • 光标移动
  • 更新，插入或删除行
  • 整个改变RowSet内容

事件的通知去所有侦听器 ，组件已实施RowSetListener界面，并将自己添加到RowSet三个事件中的任何一个发生时要通知的对象的组件列表。

侦听器可以是GUI组件，例如条形图。如果条形图正在跟踪数据RowSet对象，只要数据发生更改，侦听器就希望知道新的数据值。因此，侦听器将实现RowSetListener定义发生特定事件时将执行的操作的方法。然后，还必须将侦听器添加到RowSet对象的侦听器列表。以下代码行注册条形图组件bg与RowSet宾语rs 。

rs.addListener(bg);

现在bg每次光标移动，行更改或所有rs获取新数据。

添加滚动性或可更新性

一些DBMS不支持可以滚动（可滚动）的结果集，而某些DBMS不支持可以更新（可更新）的结果集。如果该DBMS的驱动程序未添加滚动或更新结果集的功能，则可以使用RowSet反对这样做。一种RowSet默认情况下，对象是可滚动且可更新的，因此通过填充RowSet带有结果集内容的对象，您可以有效地使结果集可滚动和可更新。

行集对象的种类

一种RowSet对象被视为已连接或已断开。一个连接 RowSet对象使用JDBC驱动程序建立与关系数据库的连接，并在整个生命周期内维护该连接。断开连接 RowSet对象建立与数据源的连接仅是为了从ResultSet对象或将数据写回到数据源。从数据源读取数据或将数据写入数据源后， RowSet对象与其断开连接，从而成为“断开连接”。在其使用寿命的大部分时间内， RowSet对象与其数据源没有连接，并且独立运行。接下来的两节将告诉您连接或断开连接意味着什么RowSet对象可以做到。

连接的行集对象

只有一种标准RowSet实现是一个连接RowSet宾语： JdbcRowSet 。始终连接到数据库， JdbcRowSet对象最类似于ResultSet对象，通常用作包装器，以使其本来是不可滚动且只读的ResultSet对象可滚动和可更新。

作为JavaBeans组件， JdbcRowSet例如，可以在GUI工具中使用object对象选择JDBC驱动程序。一种JdbcRowSet可以通过这种方式使用object对象，因为它实际上是获得与数据库的连接的驱动程序的包装。

断开连接的行集对象

其他四个实现已断开连接RowSet实现。断线RowSet对象具有连接的所有功能RowSet对象以及它们具有仅可用于断开连接的附加功能RowSet对象。例如，不必维护与数据源的连接就可以断开连接RowSet物体远比JdbcRowSet对象或ResultSet宾语。断线RowSet对象也可序列化，并且可序列化和轻量级的结合使它们成为通过网络发送数据的理想选择。它们甚至可以用于向PDA和移动电话等瘦客户端发送数据。
