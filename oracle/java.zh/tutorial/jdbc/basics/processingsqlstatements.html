使用JDBC处理SQL语句

通常，要使用JDBC处理任何SQL语句，请按照以下步骤操作：

  1. 建立连接。
  2. 创建一个语句。
  3. 执行查询。
  4. 处理ResultSet宾语。
  5. 关闭连接。

此页面使用以下方法， CoffeesTables.viewTable ，从教程示例中演示了这些步骤。该方法输出表的内容COFFEES 。此方法将在本教程的后面部分中详细讨论：

public static void viewTable(Connection con, String dbName)
    throws SQLException {

    Statement stmt = null;
    String query = "select COF_NAME, SUP_ID, PRICE, " +
                   "SALES, TOTAL " +
                   "from " + dbName + ".COFFEES";
    try {
        stmt = con.createStatement();
        ResultSet rs = stmt.executeQuery(query);
        while (rs.next()) {
            String coffeeName = rs.getString("COF_NAME");
            int supplierID = rs.getInt("SUP_ID");
            float price = rs.getFloat("PRICE");
            int sales = rs.getInt("SALES");
            int total = rs.getInt("TOTAL");
            System.out.println(coffeeName + "\t" + supplierID +
                               "\t" + price + "\t" + sales +
                               "\t" + total);
        }
    } catch (SQLException e ) {
        JDBCTutorialUtilities.printSQLException(e);
    } finally {
        if (stmt != null) { stmt.close(); }
    }
}

建立连接

首先，与您要使用的数据源建立连接。数据源可以是DBMS，旧文件系统或具有相应JDBC驱动程序的其他数据源。此连接由一个Connection宾语。有关更多信息，请参见建立连接 。

创建报表

一种Statement是表示SQL语句的接口。你执行Statement对象，它们生成ResultSet对象，这是代表数据库结果集的数据表。你需要一个Connection创建一个对象Statement宾语。

例如， CoffeesTables.viewTable创建一个Statement具有以下代码的对象：

stmt = con.createStatement();

有三种不同的语句：

  • Statement ：用于实现不带参数的简单SQL语句。
  • PreparedStatement ：（扩展Statement ）用于预编译可能包含输入参数的SQL语句。有关更多信息，请参见使用预准备语句 。
  • CallableStatement: （扩展PreparedStatement ）用于执行可能包含输入和输出参数的存储过程。有关更多信息，请参见存储过程 。

执行查询

要执行查询，请调用execute来自的方法Statement例如以下内容：

  • execute ：退货true如果查询返回的第一个对象是ResultSet宾语。如果查询可以返回一个或多个查询，请使用此方法ResultSet对象。检索ResultSet通过重复调用从查询返回的对象Statement.getResultSet 。
  • executeQuery ：返回一个ResultSet宾语。
  • executeUpdate ：返回一个整数，该整数表示受SQL语句影响的行数。如果您使用的是这种方法INSERT ， DELETE ， 要么UPDATE SQL语句。

例如， CoffeesTables.viewTable执行了Statement具有以下代码的对象：

ResultSet rs = stmt.executeQuery(query);

有关更多信息，请参见从结果集中检索和修改值 。

处理ResultSet对象

您可以在ResultSet通过游标的对象。请注意，此游标不是数据库游标。此光标是一个指针，它指向ResultSet宾语。最初，光标位于第一行之前。您调用在ResultSet用于移动光标的对象。

例如， CoffeesTables.viewTable反复调用该方法ResultSet.next将光标向前移动一行。每次打电话next ，该方法在光标当前所在的行中输出数据：

try {
    stmt = con.createStatement();
    ResultSet rs = stmt.executeQuery(query);
    while (rs.next()) {
        String coffeeName = rs.getString("COF_NAME");
        int supplierID = rs.getInt("SUP_ID");
        float price = rs.getFloat("PRICE");
        int sales = rs.getInt("SALES");
        int total = rs.getInt("TOTAL");
        System.out.println(coffeeName + "\t" + supplierID +
                           "\t" + price + "\t" + sales +
                           "\t" + total);
    }
}
// ...

有关更多信息，请参见从结果集中检索和修改值 。

关闭连接

当您使用完Statement ，调用方法Statement.close立即释放正在使用的资源。当您调用此方法时， ResultSet对象已关闭。

例如方法CoffeesTables.viewTable确保Statement在方法结束时关闭对象，无论任何情况SQLException通过将其包装在finally块：

} finally {
    if (stmt != null) { stmt.close(); }
}

JDBC抛出SQLException与数据源交互期间遇到错误时。有关更多信息，请参见处理SQL异常 。

在Java SE版本7和更高版本中可用的JDBC 4.1中，可以使用try-with-resources语句自动关闭Connection ， Statement和ResultSet对象，无论是否SQLException被抛出。自动资源声明包括try声明和一个或多个声明的资源。例如，您可以修改CoffeesTables.viewTable这样它Statement对象自动关闭，如下所示：

public static void viewTable(Connection con) throws SQLException {

    String query = "select COF_NAME, SUP_ID, PRICE, " +
                   "SALES, TOTAL " +
                   "from COFFEES";

    try (Statement stmt = con.createStatement()) {

        ResultSet rs = stmt.executeQuery(query);

        while (rs.next()) {
            String coffeeName = rs.getString("COF_NAME");
            int supplierID = rs.getInt("SUP_ID");
            float price = rs.getFloat("PRICE");
            int sales = rs.getInt("SALES");
            int total = rs.getInt("TOTAL");
            System.out.println(coffeeName + ", " + supplierID +
                               ", " + price + ", " + sales +
                               ", " + total);
        }
    } catch (SQLException e) {
        JDBCTutorialUtilities.printSQLException(e);
    }
}

以下语句是try -with-resources语句，它声明一个资源， stmt ，则当try块终止：

try (Statement stmt = con.createStatement()) {
    // ...
}
