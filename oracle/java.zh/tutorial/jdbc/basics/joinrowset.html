使用JoinRowSet对象

一种JoinRowSet实现使您可以创建一个SQL JOIN之间RowSet对象没有连接到数据源时。这很重要，因为它节省了必须创建一个或多个连接的开销。

涵盖以下主题：

  • 创建JoinRowSet对象
  • 添加行集对象
  • 管理匹配列

的JoinRowSet接口是的子接口CachedRowSet接口，从而继承了CachedRowSet宾语。这意味着JoinRowSet对象已断开连接RowSet对象并且可以运行而不必始终连接到数据源。

创建JoinRowSet对象

一种JoinRowSet对象充当SQL的持有人JOIN 。以下代码行显示了如何创建一个JoinRowSet宾语：

JoinRowSet jrs = new JoinRowSetImpl();

变量jrs直到什么都没有RowSet对象被添加到其中。

注意 ：或者，您可以使用JoinRowSet JDBC驱动程序的实现。但是， RowSet接口将与参考实现不同。这些实现将具有不同的名称和构造函数。例如，Oracle JDBC驱动程序的实现JoinRowSet接口被命名oracle.jdbc.rowset.OracleJoinRowSet 。

添加行集对象

任何RowSet可以将对象添加到JoinRowSet对象，只要它可以是SQL的一部分JOIN 。一种JdbcRowSet可以添加始终连接到其数据源的对象，但是通常它构成一个对象的一部分JOIN通过直接使用数据源而不是成为数据源的一部分JOIN通过添加到JoinRowSet宾语。提供一个点JoinRowSet实施是为了使断开连接成为可能RowSet成为对象的一部分JOIN关系。

咖啡馆的Coffee Break连锁店的所有者想要获得他从Acme，Inc.购买的咖啡的清单。为此，所有者必须从两个表中获取信息， COFFEES和SUPPLIERS 。在数据库世界之前RowSet技术，程序员会将以下查询发送到数据库：

String query =
    "SELECT COFFEES.COF_NAME " +
    "FROM COFFEES, SUPPLIERS " +
    "WHERE SUPPLIERS.SUP_NAME = Acme.Inc. " +
    "and " +
    "SUPPLIERS.SUP_ID = COFFEES.SUP_ID";

在世界RowSet技术，您无需发送查询到数据源就可以实现相同的结果。你可以加RowSet包含两个表中数据的对象JoinRowSet宾语。然后，因为所有相关数据都在JoinRowSet对象，您可以对其执行查询以获取所需的数据。

以下代码片段来自JoinSample.testJoinRowSet创建两个CachedRowSet对象coffees用表中的数据填充COFFEES和suppliers用表中的数据填充SUPPLIERS 。的coffees和suppliers对象必须与数据库建立连接以执行其命令并填充数据，但是在此之后，它们不必再次重新连接即可形成一个数据库。 JOIN 。

coffees = new CachedRowSetImpl();
coffees.setCommand("SELECT * FROM COFFEES");
coffees.setUsername(settings.userName);
coffees.setPassword(settings.password);
coffees.setUrl(settings.urlString);
coffees.execute();

suppliers = new CachedRowSetImpl();
suppliers.setCommand("SELECT * FROM SUPPLIERS");
suppliers.setUsername(settings.userName);
suppliers.setPassword(settings.password);
suppliers.setUrl(settings.urlString);
suppliers.execute();

管理匹配列

看着SUPPLIERS表格中，您可以看到Acme，Inc.的标识号为101。桌子上的咖啡COFFEES供应商识别号码为101的是Columbian和Colombian_Decaf。由于两个表都有列，因此可以合并来自两个表的信息SUP_ID in common 。在JDBC中RowSet技术， SUP_ID ，其上的列JOIN基于，称为匹配列 。

每RowSet对象已添加到JoinRowSet对象必须具有匹配列，在该列上JOIN基于。设置匹配列的方法有两种RowSet宾语。第一种方法是将match列传递给JoinRowSet方法addRowSet ，如以下代码行所示：

jrs.addRowSet(coffees, 2);

这行代码添加了coffeesCachedRowSet到jrs对象并设置第二列coffees （ SUP_ID ）作为匹配列。该行代码也可以使用列名而不是列号。

jrs.addRowSet(coffees, "SUP_ID");

这一点， jrs只有coffees在里面。下一个RowSet对象已添加到jrs将必须能够形成一个JOIN与coffees ，这是真的suppliers因为两个表都具有列SUP_ID。以下代码行添加suppliers至jrs并将列SUP_ID设置为匹配列。

jrs.addRowSet(suppliers, 1);

现在jrs包含一个JOIN之间coffees和suppliers所有者可以从中获取Acme，Inc.提供的咖啡的名称。因为该代码未设置JOIN ， jrs拥有一个内部JOIN，这是默认值。换句话说，在jrs合并一行coffees和一行suppliers 。它保持列在coffees加上列suppliers对于其中的值COFFEES.SUP_ID列与中的值匹配SUPPLIERS.SUP_ID 。以下代码打印出Acme，Inc.提供的咖啡的名称，其中StringsupplierName等于Acme, Inc.请注意，这是可能的，因为该列SUP_NAME ，来自suppliers和COF_NAME ，来自coffees ，现在都包含在JoinRowSet宾语jrs 。

System.out.println("Coffees bought from " + supplierName + ": ");

while (jrs.next()) {
    if (jrs.getString("SUP_NAME").equals(supplierName)) {
        String coffeeName = jrs.getString(1);
        System.out.println("     " + coffeeName);
    }
}

这将产生类似于以下内容的输出：

Coffees bought from Acme, Inc.:
     Colombian
     Colombian_Decaf

的JoinRowSet接口提供用于设置类型的常量JOIN将会形成，但是目前唯一实现的类型是JoinRowSet.INNER_JOIN 。
