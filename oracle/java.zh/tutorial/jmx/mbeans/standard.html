标准MBean

本节提供了一个简单的标准MBean的示例。

通过编写称为SomethingMBean还有一个叫做Something实现该接口。接口中的每个方法都定义MBean中的属性或操作。默认情况下，每个方法都定义一个操作。属性和操作是遵循某些设计模式的方法。一个标准的MBean由一个MBean接口和一个类组成。MBean接口列出了所有公开的属性和操作的方法。该类实现此接口，并提供所检测资源的功能。

以下各节检查标准MBean和管理MBean的简单的启用JMX技术的代理（JMX代理）的示例。

MBean接口

基本MBean接口的示例， HelloMBean ，如下所示：


package com.example; 
 
public interface HelloMBean { 
 
    public void sayHello(); 
    public int add(int x, int y); 
    
    public String getName(); 
     
    public int getCacheSize(); 
    public void setCacheSize(int size); 
}

按照约定，MBean接口采用实现它的Java类的名称，后缀为MBean添加。在这种情况下，该接口称为HelloMBean 。的Hello下一节将介绍实现此接口的类。

根据JMX规范，除了可以由MBean管理的应用程序调用的命名和类型操作之外，MBean接口还包括可读且可能可写的命名和类型属性。的HelloMBean接口声明了两个操作：Java方法add()和sayHello() 。

HelloMBean声明两个属性： Name是只读字符串，并且CacheSize是可以读取和写入的整数。声明了Getter和setter方法以允许托管应用程序访问并可能更改属性值。根据JMX规范的定义， getter是任何不返回void并且名称开头为的公共方法get 。获取器使管理器能够读取属性的值，该属性的类型是返回对象的类型。设置器是任何采用单个参数且名称以开头的公共方法set 。设置器使管理者可以在属性中写入新值，该属性的类型与参数的类型相同。

这些操作和属性的实现在以下部分中显示。

MBean实施

的Hello接下来的Java类实现HelloMBean MBean接口：

package com.example; 
 
public class Hello ... 
    implements HelloMBean { 
    public void sayHello() { 
        System.out.println("hello, world"); 
    } 
     
    public int add(int x, int y) { 
        return x + y; 
    } 
     
    public String getName() { 
        return this.name; 
    }  
     
    public int getCacheSize() { 
        return this.cacheSize; 
    } 
     
    public synchronized void setCacheSize(int size) {
        ...
    
        this.cacheSize = size; 
        System.out.println("Cache size now " + this.cacheSize); 
    } 
    ...
     
    private final String name = "Reginald"; 
    private int cacheSize = DEFAULT_CACHE_SIZE; 
    private static final int 
        DEFAULT_CACHE_SIZE = 200; 
}

简单明了Hello类提供了由声明的操作和属性的定义HelloMBean 。的sayHello()和add()操作非常简单，但实际操作可以根据需要简单或复杂。

获得方法Name属性并获取并设置CacheSize属性也被定义。在此示例中， Name属性值永不变。但是，在实际情况下，此属性可能会随着托管资源的运行而改变。例如，该属性可能表示统计信息，例如正常运行时间或内存使用情况。在这里，属性仅仅是名称Reginald 。

呼叫setCacheSize方法使您可以更改CacheSize属性从其声明的默认值200开始。在实际情况下，更改CacheSize属性可能需要执行其他操作，例如丢弃条目或分配新条目。本示例仅打印一条消息以确认高速缓存大小已更改。但是，可以定义更复杂的操作，而不是简单地调用println() 。

随着Hello如此定义的MBean及其接口，现在可以用来管理它们表示的资源，如以下部分所示。

创建一个JMX代理来管理资源

一旦MBean检测到资源，该资源的管理就由JMX代理执行。

JMX代理的核心组件是MBean服务器。MBean服务器是其中注册了MBean的受管对象服务器。JMX代理还包括一组用于管理MBean的服务。请参阅API文档，以了解MBeanServer接口，以获取MBean服务器实现的详细信息。

的Main后面的类表示基本的JMX代理：

package com.example; 
 
import java.lang.management.*; 
import javax.management.*; 
 
public class Main { 
 
    public static void main(String[] args) 
        throws Exception { 
     
        MBeanServer mbs = ManagementFactory.getPlatformMBeanServer(); 
        ObjectName name = new ObjectName("com.example:type=Hello"); 
        Hello mbean = new Hello(); 
        mbs.registerMBean(mbean, name); 
          
        ...
     
        System.out.println("Waiting forever..."); 
        Thread.sleep(Long.MAX_VALUE); 
    } 
}

JMX代理Main首先，获取通过平台创建和初始化的MBean服务器，方法是调用getPlatformMBeanServer()的方法java.lang.management.ManagementFactory类。如果平台尚未创建MBean服务器，则getPlatformMBeanServer()通过调用JMX方法自动创建MBean服务器MBeanServerFactory.createMBeanServer() 。的MBeanServer实例由Main被称为mbs 。

下一个， Main为将创建的MBean实例定义一个对象名称。每个JMX MBean必须具有一个对象名称。对象名称是JMX类的实例ObjectName并且必须符合JMX规范定义的语法。即，对象名称必须包含一个域和一个键属性列表。在定义的对象名称中Main ，该域是com.example （包含示例MBean的程序包）。此外，密钥属性声明此对象的类型为Hello 。

一个实例Hello对象，命名mbean ， 被建造。的Hello对象名为mbean然后在MBean服务器中注册为MBean mbs与对象名称name ，方法是将对象和对象名称传递给对JMX方法的调用MBeanServer.registerMBean() 。

随着Hello在MBean服务器中注册的MBean， Main只是等待执行管理操作Hello 。在此示例中，这些管理操作正在调用sayHello()和add() ，并获取和设置属性值。

运行标准MBean示例

检查了示例类之后，您现在可以运行该示例。在此示例中，使用JConsole与MBean进行交互。

要运行该示例，请按照下列步骤操作：

  1. 保存捆绑的JMX API示例类， jmx_examples.zip ，到您的工作目录， work_dir 。
  2. 通过在终端窗口中使用以下命令来解压缩样本类包。 
    unzip jmx_examples.zip
  3. 从内部编译示例Java类work_dir目录。 
    javac com/example/*.java
  4. 如果您正在运行Java Development Kit（JDK）版本6，请启动Main使用以下命令的应用程序。 
    java com.example.Main

    如果您运行的JDK版本早于版本6，则需要启动Main指定以下选项的应用程序，以公开应用程序进行监视和管理。

    java -Dcom.sun.management.jmxremote example.Main

    确认Main正在等待某事发生。

  5. 在同一台计算机上的另一个终端窗口中启动JConsole。 
    jconsole

    将显示“新建连接”对话框，其中列出了可以连接的正在运行的JMX代理。

  6. 在“新建连接”对话框中，选择com.example.Main从列表中，然后单击连接。

    显示平台当前活动的摘要。

  7. 单击“ MBeans”选项卡。

    此面板显示当前在MBean服务器中注册的所有MBean。

  8. 在左侧框架中，展开com.example MBean树中的节点。

    您看到示例MBean Hello由创建和注册的Main 。如果单击Hello ，您会在MBean树中看到其关联的“属性和操作”节点。

  9. 展开的“属性”节点Hello MBean树中的MBean。

    由MBean属性定义的Hello显示班级。

  10. 更改值CacheSize属性为150。

    在您开始的终端窗口中Main ，将生成此属性更改的确认。

  11. 展开Hello MBean树中的MBean。

    声明的两个操作Hello MBean， sayHello()和add()可见。

  12. 调用sayHello()通过单击操作sayHello按钮。

    JConsole对话框通知您该方法已成功调用。消息“ hello world”在终端窗口中生成，其中Main在跑。

  13. 提供两个整数add()添加操作并单击add按钮。

    答案显示在“ JConsole”对话框中。

  14. 要关闭JConsole，请选择连接->退出。
