课程：通知

JMX API定义了一种机制，使MBean能够生成通知，例如，以信号通知状态更改，检测到的事件或问题。

要生成通知，MBean必须实现接口NotificationEmitter或扩展NotificationBroadcasterSupport 。要发送通知，您需要构造一个类的实例javax.management.Notification或子类（例如AttributeChangedNotification ），并将实例传递给NotificationBroadcasterSupport.sendNotification 。

每个通知都有一个来源。源是生成通知的MBean的对象名称。

每个通知都有一个序列号。当订单很重要时，此数字可用于订购来自同一来源的通知，并且存在以错误的顺序处理通知的风险。序列号可以为零，但是对于给定MBean的每个通知，序列号最好递增。

的Hello 标准MBean中的 MBean实现实际上实现了通知机制。但是，为简单起见，本课程中省略了此代码。的完整代码Hello如下：

package com.example;

import javax.management.*;

public class Hello
        extends NotificationBroadcasterSupport
        implements HelloMBean {

    public void sayHello() {
        System.out.println("hello, world");
    }

    public int add(int x, int y) {
        return x + y;
    }

    public String getName() {
        return this.name;
    }

    public int getCacheSize() {
        return this.cacheSize;
    }

    public synchronized void setCacheSize(int size) {
        int oldSize = this.cacheSize;
        this.cacheSize = size;

        System.out.println("Cache size now " + this.cacheSize);

        Notification n = new AttributeChangeNotification(this,
                                sequenceNumber++, System.currentTimeMillis(),
                                "CacheSize changed", "CacheSize", "int",
                                oldSize, this.cacheSize);

        sendNotification(n);
    }

    @Override
    public MBeanNotificationInfo[] getNotificationInfo() {
        String[] types = new String[]{
            AttributeChangeNotification.ATTRIBUTE_CHANGE
        };

        String name = AttributeChangeNotification.class.getName();
        String description = "An attribute of this MBean has changed";
        MBeanNotificationInfo info = 
                new MBeanNotificationInfo(types, name, description);
        return new MBeanNotificationInfo[]{info};
    }
    
    private final String name = "Reginald";
    private int cacheSize = DEFAULT_CACHE_SIZE;
    private static final int DEFAULT_CACHE_SIZE = 200;
    private long sequenceNumber = 1;
}

这个Hello MBean实现扩展了NotificationBroadcasterSupport类。 NotificationBroadcasterSupport实现NotificationEmitter接口。

操作和属性的设置方式与标准MBean示例中的设置方式相同，不同之处在于： CacheSize属性的setter方法现在定义的值为oldSize 。此值记录CacheSize设置操作之前的属性值。

通知是根据实例构造的， n ，属于JMX类AttributeChangeNotification ，扩展了javax.management.Notification 。该通知是在setCacheSize()从以下信息中获取方法。此信息传递给AttributeChangeNotification作为参数。

  • 通知来源的对象名称，即Hello MBean，由this
  • 序列号，即sequenceNumber ，将其设置为1并逐渐增加
  • 时间戳记
  • 通知消息的内容
  • 在这种情况下，已更改的属性的名称，CacheSize
  • 更改的属性类型
  • 在这种情况下，旧的属性值oldSize
  • 在这种情况下，新的属性值this.cacheSize

通知n然后传递给NotificationBroadcasterSupport.sendNotification()方法。

最后， MBeanNotificationInfo实例被定义为描述由MBean为给定类型的通知生成的不同通知实例的特征。在这种情况下，发送的通知类型为AttributeChangeNotification通知。

运行MBean通知示例

再一次，您将使用JConsole与Hello MBean，这次发送和接收通知。此示例需要Java SE平台的版本6。

  1. 如果还没有保存，请保存jmx_examples.zip进入你的work_dir目录。
  2. 通过在终端窗口中使用以下命令来解压缩样本类包。 
    unzip jmx_examples.zip
  3. 从内部编译示例Java类work_dir目录。 
    javac com/example/*.java
  4. 开始Main应用。 
    java com.example.Main

    确认Main等待发生的事情。

  5. 在同一台计算机上的另一个终端窗口中启动JConsole。 
    jconsole

    将显示“新建连接”对话框，其中列出了可以连接的正在运行的JMX代理。

  6. 在“新建连接”对话框中，选择com.example.Main从列表中，然后单击连接。

    显示平台当前活动的摘要。

  7. 单击“ MBeans”选项卡。

    此面板显示当前在MBean服务器中注册的所有MBean。

  8. 在左侧框架中，展开com.example MBean树中的节点。

    您看到示例MBean Hello由创建和注册的Hello 。如果单击Hello ，您会在MBean树中看到其Notifications节点。

  9. 展开的“通知”节点Hello MBean树中的MBean。

    请注意该面板是空白的。

  10. 单击订阅按钮。

    当前收到的通知数（0）显示在“通知”节点标签中。

  11. 展开的“属性”节点Hello MBean树中的MBean，然后更改CacheSize属性为150。

    在您开始的终端窗口中Main ，将显示此属性更改的确认。请注意，“通知”节点中显示的接收到的通知数已更改为1。

  12. 再次展开Hello MBean树中的MBean。

    显示通知的详细信息。

  13. 要关闭JConsole，请选择连接->退出。
