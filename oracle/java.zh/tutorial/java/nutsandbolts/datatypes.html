原始数据类型

Java编程语言是静态类型的，这意味着必须先声明所有变量，然后才能使用它们。正如您已经看到的，这涉及到声明变量的类型和名称：

int gear = 1;

这样做会告诉您的程序，存在一个名为“ gear”的字段，该字段保存数值数据，并且其初始值为“ 1”。变量的数据类型决定了它可能包含的值，以及可能对其执行的操作。此外int Java编程语言还支持其他七个原始数据类型 。基本类型由语言预先定义，并由保留关键字命名。基本值不与其他基本值共享状态。Java编程语言支持的八种原始数据类型是：

  • 字节 ： byte数据类型是8位带符号的二进制补码整数。最小值为-128，最大值为127（含）。的byte数据类型对于在大阵列中节省内存非常有用，而内存节省实际上很重要。它们也可以代替int它们的限制可帮助您澄清代码；变量范围有限的事实可以作为文档的一种形式。

  • 简短的 ： short数据类型是16位带符号的二进制补码整数。最小值为-32,768，最大值为32,767（含）。与byte ，则适用相同的准则：您可以使用short在实际需要节省内存的情况下，在大型阵列中节省内存。

  • int ：默认情况下， int数据类型是32位带符号的二进制补码整数，最小值为-2 31 ，最大值为2 31 -1。在Java SE 8和更高版本中，您可以使用int表示无符号32位整数的数据类型，其最小值为0，最大值为2 32 -1。使用Integer类来使用int数据类型为无符号整数。有关更多信息，请参见“数字类”部分。静态方法，例如compareUnsigned ， divideUnsigned等已添加到Integer类支持无符号整数的算术运算。

  • 长 ： long数据类型是64位二进制补码整数。带符号的long的最小值为-2 63 ，最大值为2 63 -1。在Java SE 8和更高版本中，您可以使用long表示无符号64位长的数据类型，其最小值为0，最大值为2 64 -1。当您需要的值范围比以下提供的值宽时，请使用此数据类型int 。的Long类还包含类似的方法compareUnsigned ， divideUnsigned等支持无符号长整数的算术运算。

  • 浮点数 ： float数据类型是单精度32位IEEE 754浮点。其值的范围超出了本文的讨论范围，但在Java语言规范的“ 浮点类型，格式和值”部分中进行了指定。与建议有关byte和short ， 用一个float （代替double ），如果您需要将内存保存在大的浮点数数组中。永远不要将这种数据类型用于精确值，例如货币。为此，您将需要使用java.math。改为使用BigDecimal类。 数字和字符串封面BigDecimal以及Java平台提供的其他有用的类。

  • 双重 ： double数据类型是双精度64位IEEE 754浮点。其值的范围超出了本文的讨论范围，但在Java语言规范的“ 浮点类型，格式和值”部分中进行了指定。对于十进制值，此数据类型通常是默认选择。如上所述，永远不要将这种数据类型用于精确值，例如货币。

  • 布尔值 ： boolean数据类型只有两个可能的值： true和false 。将此数据类型用于跟踪真/假条件的简单标志。这种数据类型代表一小部分信息，但是它的“大小”并不是精确定义的。

  • 字符 ： char数据类型是单个16位Unicode字符。最小值为'\u0000' （或0），且最大值为'\uffff' （或65,535含）。

除了上面列出的八种原始数据类型之外，Java编程语言还通过java.lang提供了对字符串的特殊支持。字符串类。用双引号括住您的字符串会自动创建一个新的String宾语;例如， String s = "this is a string"; 。 String对象是不可变的 ，这意味着对象一旦创建就无法更改。的String从技术上讲，类不是原始数据类型，但是考虑到语言对该类的特殊支持，您可能会倾向于将其视为此类。您将了解有关String 简单数据对象中的类

默认值

声明字段时，不一定总是需要分配值。编译器会将已声明但未初始化的字段设置为合理的默认值。一般而言，此默认值为零或null ，具体取决于数据类型。但是，通常认为依赖于此类默认值是不好的编程风格。

下表总结了上述数据类型的默认值。

数据类型 默认值（用于字段）
字节 0
短 0
整型 0
长 0升
浮动 0.0分
双 0.0天
烧焦 '\ u0000'
字符串（或任何对象） 空值
布尔值 假

局部变量略有不同。编译器永远不会将默认值分配给未初始化的局部变量。如果无法在声明它的地方初始化本地变量，请确保在尝试使用它之前给它分配一个值。访问未初始化的局部变量将导致编译时错误。

文字

您可能已经注意到new初始化基本类型的变量时不使用关键字。基本类型是语言中内置的特殊数据类型。它们不是从类创建的对象。文字是固定值的源代码表示；文字直接在您的代码中表示，无需计算。如下所示，可以将文字分配给基本类型的变量：

boolean result = true;
char capitalC = 'C';
byte b = 100;
short s = 10000;
int i = 100000;

整数字面量

整数文字类型为long如果以字母结尾L要么l ;否则是类型int 。建议您使用大写字母L因为小写字母l很难与数字区分开1 。

整数类型的值byte ， short ， int和long可以从创建int文字。类型值long超出范围int可以从创建long文字。整数文字可以通过以下数字系统表示：

  • 十进制：以10为底的数字，由0到9组成；这是您每天使用的号码系统
  • 十六进制：基数16，其数字由数字0到9和字母A到F组成
  • Binary：基数2，其数字由数字0和1组成（您可以在Java SE 7和更高版本中创建二进制文字）

对于通用编程，十进制可能是您将要使用的唯一数字系统。但是，如果需要使用其他数字系统，则以下示例显示正确的语法。前缀0x表示十六进制， 0b表示二进制：

// The number 26, in decimal
int decVal = 26;
//  The number 26, in hexadecimal
int hexVal = 0x1a;
// The number 26, in binary
int binVal = 0b11010;

浮点文字

浮点文字是类型float如果以字母结尾F要么f ;否则它的类型是double它可以选择以字母结尾D要么d 。

浮点类型（ float和double ）也可以使用E或e（用于科学计数法），F或f（32位浮点文字）和D或d（64位双精度文字；这是默认值，并且按照惯例被省略）表示。

double d1 = 123.4;
// same value as d1, but in scientific notation
double d2 = 1.234e2;
float f1  = 123.4f;

字符和字符串文字

类型文字char和String可以包含任何Unicode（UTF-16）字符。如果您的编辑器和文件系统允许，则可以在代码中直接使用此类字符。如果没有，则可以使用“ Unicode转义”，例如'\u0108' （带有回旋音的大写字母C），或"S\u00ED Se\u00F1or" （西班牙语中的SíSeñor）。始终使用“单引号” char文字和“双引号” String文字。Unicode转义序列可以在程序的其他地方（例如，在字段名称中）使用，而不仅仅是char要么String文字。

Java编程语言还支持一些特殊的转义序列，用于char和String文字： \b （退格键）， \t （标签）， \n （换行）， \f （换页）， \r （回车）， \" （双引号）， \' （单引号），和\\ （反斜杠）。

还有一个特别的null可用作任何引用类型的值的文字。 null可以分配给任何变量，原始类型变量除外。您几乎无能为力null超越测试其存在的价值。因此， null通常在程序中用作标记，以指示某些对象不可用。

最后，还有一种特殊的文字，称为类文字 ，它是通过使用类型名称并附加“ .class" ;例如， String.class 。这是指对象（类型Class ）代表类型本身。

在数字文字中使用下划线字符

在Java SE 7和更高版本中，任意数量的下划线字符（ _ ）可以出现在数字文字中数字之间的任何位置。例如，此功能使您可以使用。分隔数字文字中的数字组，这可以提高代码的可读性。

例如，如果您的代码包含许多数字，则可以使用下划线字符将数字分成三组，类似于使用逗号或空格这样的标点符号作为分隔符的方式。

以下示例显示了在数字文字中使用下划线的其他方式：

long creditCardNumber = 1234_5678_9012_3456L;
long socialSecurityNumber = 999_99_9999L;
float pi =  3.14_15F;
long hexBytes = 0xFF_EC_DE_5E;
long hexWords = 0xCAFE_BABE;
long maxLong = 0x7fff_ffff_ffff_ffffL;
byte nybbles = 0b0010_0101;
long bytes = 0b11010010_01101001_10010100_10010010;

您只能在数字之间添加下划线；您不能在以下位置放置下划线：

  • 在数字的开头或结尾
  • 与浮点文字中的小数点相邻
  • 之前F要么L后缀
  • 在需要一串数字的位置

下面的示例演示数字文字中的有效和无效的下划线位置（突出显示）：

// Invalid: cannot put underscores
// adjacent to a decimal point
float pi1 = 3_.1415F;
// Invalid: cannot put underscores 
// adjacent to a decimal point
float pi2 = 3._1415F;
// Invalid: cannot put underscores 
// prior to an L suffix
long socialSecurityNumber1 = 999_99_9999_L;

// OK (decimal literal)
int x1 = 5_2;
// Invalid: cannot put underscores
// At the end of a literal
int x2 = 52_;
// OK (decimal literal)
int x3 = 5_______2;

// Invalid: cannot put underscores
// in the 0x radix prefix
int x4 = 0_x52;
// Invalid: cannot put underscores
// at the beginning of a number
int x5 = 0x_52;
// OK (hexadecimal literal)
int x6 = 0x5_2; 
// Invalid: cannot put underscores
// at the end of a number
int x7 = 0x52_;
