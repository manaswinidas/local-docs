数字类

使用数字时，大多数时候，您会在代码中使用基本类型。例如：

int i = 500;
float gpa = 3.65f;
byte mask = 0xff;

但是，有理由使用对象代替基元，并且Java平台为每种基元数据类型提供了包装器类。这些类将“原始”包装在对象中。通常情况下，包装是由编译器完成的，如果你使用了原始的其中一个对象预计，编译器盒为你的包装类的原始。同样，如果在期望使用原语时使用数字对象，则编译器会为您解包该对象。有关更多信息，请参见自动装箱和拆箱。

所有数字包装器类都是抽象类的子类Number ：

注意：的其他四个子类Number在此不讨论。 BigDecimal和BigInteger用于高精度计算。 AtomicInteger和AtomicLong用于多线程应用程序。

您可能会出于以下三个原因使用Number对象而不是原始对象：

  1. 作为期望对象的方法的参数（通常在处理数字集合时使用）。
  2. 使用类定义的常量，例如MIN_VALUE和MAX_VALUE ，提供数据类型的上限和下限。
  3. 使用类方法在其他原始类型之间进行值转换，在字符串之间进行转换以及在数字系统（十进制，八进制，十六进制，二进制）之间进行转换。
