作为超类的对象

的Object类，在java.lang包，位于类层次结构树的顶部。每个类别都是该类别的直接或间接的后代Object类。您使用或编写的每个类都继承了的实例方法Object 。您不需要使用任何这些方法，但是，如果您选择使用这些方法，则可能需要使用特定于您的类的代码覆盖它们。从继承的方法Object本节中讨论的是：

  • protected Object clone() throws CloneNotSupportedException
    创建并返回此对象的副本。
  • public boolean equals(Object obj)
    指示其他某个对象是否与此对象“相等”。
  • protected void finalize() throws Throwable
    当垃圾回收时，由垃圾回收器在对象上调用
    集合确定不再有对该对象的引用
  • public final Class getClass()
    返回对象的运行时类。
  • public int hashCode()
    返回对象的哈希码值。
  • public String toString()
    返回对象的字符串表示形式。

的notify ， notifyAll和wait方法Object所有这些都在同步程序中独立运行的线程的活动中发挥作用，这将在后面的课程中进行讨论，此处不再赘述。这些方法有五种：

  • public final void notify()
  • public final void notifyAll()
  • public final void wait()
  • public final void wait(long timeout)
  • public final void wait(long timeout, int nanos)
注意：许多这些方法都有一些细微的方面，尤其是clone方法。

clone（）方法

如果某个类或其超类之一实现了Cloneable界面，您可以使用clone()从现有对象创建副本的方法。要创建克隆，请编写：

aCloneableObject.clone();

Object此方法的实现检查对象是否位于clone()被调用实现Cloneable接口。如果对象不存在，则该方法将引发CloneNotSupportedException例外。异常处理将在以后的课程中介绍。目前，您需要知道clone()必须声明为

protected Object clone() throws CloneNotSupportedException

要么：

public Object clone() throws CloneNotSupportedException

如果你要写一个clone()覆盖其中一个的方法Object 。

如果物体在哪clone()被调用确实实现了Cloneable接口， Object的实施clone()方法创建与原始对象相同类的对象，并将新对象的成员变量初始化为具有与原始对象的相应成员变量相同的值。

使类可克隆的最简单方法是添加implements Cloneable到班级的声明。然后您的对象可以调用clone()方法。

对于某些类，默认行为是Object的clone()方法效果很好。但是，如果对象包含对外部对象的引用，请说ObjExternal ，您可能需要覆盖clone()获得正确的行为。否则，改变ObjExternal由一个对象创建的对象也将在其克隆中可见。这意味着原始对象及其克隆不是独立的，要分离它们，您必须重写clone()以便克隆对象并 ObjExternal 。然后原始对象引用ObjExternal该克隆引用了一个克隆ObjExternal ，因此对象及其克隆是真正独立的。

equals（）方法

的equals()方法比较两个对象是否相等并返回true如果它们相等。的equals()中提供的方法Object类使用身份运算符（ == ），以确定两个对象是否相等。对于原始数据类型，这将给出正确的结果。但是，对于对象，不是。的equals()提供的方法Object测试对象引用是否相等，即是否比较的对象是完全相同的对象。

要测试两个对象在等效意义上是否相等 （包含相同的信息），必须覆盖equals()方法。这是一个例子Book覆盖的类equals() ：

public class Book {
    ...
    public boolean equals(Object obj) {
        if (obj instanceof Book)
            return ISBN.equals((Book)obj.getISBN()); 
        else
            return false;
    }
}

考虑一下这段代码，该代码测试了Book平等课程：

// Swing Tutorial, 2nd edition
Book firstBook  = new Book("0201914670");
Book secondBook = new Book("0201914670");
if (firstBook.equals(secondBook)) {
    System.out.println("objects are equal");
} else {
    System.out.println("objects are not equal");
}

该程序显示objects are equal即使firstBook和secondBook引用两个不同的对象。之所以认为它们相等，是因为所比较的对象包含相同的ISBN号。

您应该始终覆盖equals()如果标识运算符不适合您的班级，请使用此方法。

注意：如果您覆盖equals() ，您必须覆盖hashCode()也一样

finalize（）方法

的Object类提供了一个回调方法， finalize() ， 可以在对象变成垃圾时在其上调用。 Object的实现finalize()什么都不做-您可以覆盖finalize()进行清理，例如释放资源。

的finalize()方法可能会由系统自动调用，但是何时调用，即使调用它，也不确定。因此，您不应依赖此方法为您进行清理。例如，如果您在执行I / O之后没有在代码中关闭文件描述符，则您期望finalize()要为您关闭它们，您可能用完了文件描述符。

getClass（）方法

您不能覆盖getClass 。

的getClass()方法返回一个Class对象，该对象具有可用于获取有关类的信息的方法，例如其名称（ getSimpleName() ），其超类（ getSuperclass() ）及其实现的接口（ getInterfaces() ）。例如，以下方法获取并显示对象的类名称：

void printClassName(Object obj) {
    System.out.println("The object's" + " class is " +
        obj.getClass().getSimpleName());
}

的Class类，在java.lang包，具有大量方法（超过50种）。例如，您可以测试以查看该类是否为注释（ isAnnotation() ），界面（ isInterface() ）或枚举（ isEnum() ）。您可以看到对象的字段是（ getFields() ）或其方法是（ getMethods() ）， 等等。

hashCode（）方法

传回的值hashCode()是对象的哈希码，它是对象的内存地址（十六进制）。

根据定义，如果两个对象相等，则它们的哈希码也必须相等。如果您覆盖equals()方法，您可以更改两个对象的相等方式， Object的实现hashCode()不再有效。因此，如果您覆盖equals()方法，您还必须覆盖hashCode()方法。

toString（）方法

您应该始终考虑覆盖toString()类中的方法。

的Object的toString()方法返回一个String对象的表示形式，这对于调试非常有用。的String对象的表示形式完全取决于对象，这就是为什么需要覆盖的原因toString()在你的课上。

您可以使用toString()随着System.out.println()显示对象的文本表示形式，例如的实例Book ：

System.out.println(firstBook.toString());

适当覆盖toString()方法，打印一些有用的内容，例如：

ISBN: 0201914670; The Swing Tutorial; A Guide to Constructing GUIs, 2nd Edition
