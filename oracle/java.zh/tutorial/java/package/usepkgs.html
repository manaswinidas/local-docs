使用包成员

组成程序包的类型称为程序包成员 。

要使用public软件包成员必须从其软件包外部执行以下操作之一：

  • 用其完全限定的名称称呼该会员
  • 导入包成员
  • 导入会员的整个包裹

每种情况都适用于不同的情况，如以下各节所述。

用其合格名称引用软件包成员

到目前为止，本教程中的大多数示例都通过简单名称来引用类型，例如Rectangle和StackOfInts 。如果您要编写的代码与该成员位于同一软件包中，或者已导入该成员，则可以使用该成员的简单名称。

但是，如果您尝试使用来自其他软件包的成员，而该软件包尚未导入，则必须使用成员的全限定名，其中包括软件包名称。这是Rectangle在中声明的类graphics上一个示例中的包。

graphics.Rectangle

您可以使用此限定名来创建一个实例graphics.Rectangle ：

graphics.Rectangle myRect = new graphics.Rectangle();

合格的名称可以不经常使用。然而，当重复使用名称时，重复输入名称变得乏味，并且代码变得难以阅读。或者，您可以导入成员或其包，然后使用其简单名称。

导入包裹会员

要将特定成员导入当前文件，请在import文件开头的语句，在任何类型定义之前，但在package声明（如果有）。这是导入方式Rectangle来自的班级graphics在上一节中创建的包。

import graphics.Rectangle;

现在您可以参考Rectangle类的简单名称。

Rectangle myRectangle = new Rectangle();

如果您只使用来自graphics包。但是，如果您从包中使用许多类型，则应导入整个包。

导入整个程序包

要导入特定软件包中包含的所有类型，请使用import带星号的语句(*)通配符。

import graphics.*;

现在，您可以在graphics以其简单名称打包。

Circle myCircle = new Circle();
Rectangle myRectangle = new Rectangle();

中的星号import语句只能用于指定包中的所有类，如下所示。它不能用于匹配包中类的子集。例如，以下代码与“ graphics开头的包A 。

// does not work
import graphics.A*;

而是，它生成一个编译器错误。随着import语句，通常只导入一个包成员或整个包。

注意：另一种较不常见的形式import允许您导入封闭类的公共嵌套类。例如，如果graphics.Rectangle类包含有用的嵌套类，例如Rectangle.DoubleWide和Rectangle.Square ，您可以导入Rectangle使用以下两个语句及其嵌套类。 
import graphics.Rectangle;
import graphics.Rectangle.*;
请注意，第二个导入语句不会导入Rectangle 。

另一种较不常见的形式import 静态导入语句 ，将在本节末尾讨论。

为了方便起见，Java编译器会为每个源文件自动导入两个完整的软件包：（1） java.lang包和（2）当前包（当前文件的包）。

程序包的明显层次结构

最初，程序包看起来是分层的，但不是。例如，Java API包含一个java.awt包装，一个java.awt.color包装，一个java.awt.font包，以及许多其他以java.awt 。然而java.awt.color包装， java.awt.font包装等java.awt.xxxx软件包不包含在java.awt包。前缀java.awt （Java抽象窗口工具包）用于许多相关的程序包，以使关系显而易见，但不显示包含关系。

输入java.awt.*导入所有类型的java.awt包，但不导入 java.awt.color ， java.awt.font或任何其他java.awt.xxxx包。如果您打算在中使用类和其他类型java.awt.color以及那些java.awt ，您必须同时导入两个软件包及其所有文件：

import java.awt.*;
import java.awt.color.*;

名字歧义

如果一个包中的成员与另一个包中的成员共享其名称，并且两个包均已导入，则必须使用其合格名称来引用每个成员。例如， graphics包定义了一个名为Rectangle 。的java.awt包装中还包含一个Rectangle类。如果两者graphics和java.awt已经被导入，以下是含糊的。

Rectangle rect;

在这种情况下，您必须使用会员的全限定名来准确指出Rectangle你想要的课。例如，

graphics.Rectangle rect;

静态导入语句

在某些情况下，您需要经常访问一两个类的静态final字段（常量）和静态方法。一遍又一遍地加上这些类的名称可能会导致代码混乱。静态import语句为您提供了一种导入要使用的常量和静态方法的方法，因此您无需在其类名之前添加前缀。

的java.lang.Math类定义了PI常数和许多静态方法，包括计算正弦，余弦，正切，平方根，最大值，最小值，指数等的方法。例如，

public static final double PI 
    = 3.141592653589793;
public static double cos(double a)
{
    ...
}

通常，要使用另一个类中的这些对象，请在类名之前添加前缀，如下所示。

double r = Math.cos(Math.PI * theta);

您可以使用静态导入语句来导入java.lang的静态成员。数学，这样您就无需在类名前添加前缀， Math 。的静态成员Math可以单独导入：

import static java.lang.Math.PI;

或作为一个组：

import static java.lang.Math.*;

导入静态成员后，可以直接使用它们而无需进行资格鉴定。例如，先前的代码片段将变为：

double r = cos(PI * theta);

显然，您可以编写自己的类，这些类包含经常使用的常量和静态方法，然后使用静态import语句。例如，

import static mypackage.MyConstants.*;
注意：非常谨慎地使用静态导入。过度使用静态导入会导致代码难以阅读和维护，因为代码的读者不会知道哪个类定义了特定的静态对象。通过正确使用静态导入，可以消除类名重复，从而使代码更具可读性。
