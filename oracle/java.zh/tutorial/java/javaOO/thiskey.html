使用此关键字

在实例方法或构造函数中， this是对当前对象的引用—当前对象的方法或构造函数正在被调用。您可以使用以下方法从实例方法或构造函数中引用当前对象的任何成员： this 。

使用this与田野

使用的最常见原因this关键字是因为字段被方法或构造函数参数遮盖。

例如， Point班级是这样写的

public class Point {
    public int x = 0;
    public int y = 0;
        
    //constructor
    public Point(int a, int b) {
        x = a;
        y = b;
    }
}

但是它可能是这样写的：

public class Point {
    public int x = 0;
    public int y = 0;
        
    //constructor
    public Point(int x, int y) {
        this.x = x;
        this.y = y;
    }
}

构造函数的每个参数都隐藏了对象的字段之一-构造函数内部x是构造函数第一个参数的本地副本。要参考Point领域x ，构造函数必须使用this.x 。

使用this与构造函数

在构造函数中，您还可以使用this关键字，以调用同一类中的另一个构造函数。这样做称为显式构造函数调用 。这是另一个Rectangle类，其实现与“ 对象”部分中的实现不同。

public class Rectangle {
    private int x, y;
    private int width, height;
        
    public Rectangle() {
        this(0, 0, 1, 1);
    }
    public Rectangle(int width, int height) {
        this(0, 0, width, height);
    }
    public Rectangle(int x, int y, int width, int height) {
        this.x = x;
        this.y = y;
        this.width = width;
        this.height = height;
    }
    ...
}

此类包含一组构造函数。每个构造函数都会初始化矩形的一些或所有成员变量。构造函数为参数未提供其初始值的任何成员变量提供默认值。例如，无参数构造函数创建一个1x1 Rectangle在坐标0,0。两参数构造函数调用四参数构造函数，传入宽度和高度，但始终使用0,0坐标。和以前一样，编译器根据参数的数量和类型确定要调用的构造函数。

如果存在，则另一个构造函数的调用必须是该构造函数的第一行。
