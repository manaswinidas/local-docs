类型推断

类型推断是Java编译器查看每个方法调用和相应声明以确定使调用适用的类型参数的能力。推论算法确定参数的类型，以及确定结果是否被分配或返回的类型（如果可用）。最后，推理算法尝试找到与所有参数一起使用的最具体的类型。

为了说明最后一点，在下面的示例中，推论确定传递给pick方法的第二个参数的类型为Serializable ：

static <T> T pick(T a1, T a2) { return a2; }
Serializable s = pick("d", new ArrayList<String>());

类型推断和通用方法

通用方法为您引入了类型推断，使您可以像调用普通方法一样调用通用方法，而无需在尖括号之间指定类型。考虑以下示例， BoxDemo ，这需要Box类：

public class BoxDemo {

  public static <U> void addBox(U u, 
      java.util.List<Box<U>> boxes) {
    Box<U> box = new Box<>();
    box.set(u);
    boxes.add(box);
  }

  public static <U> void outputBoxes(java.util.List<Box<U>> boxes) {
    int counter = 0;
    for (Box<U> box: boxes) {
      U boxContents = box.get();
      System.out.println("Box #" + counter + " contains [" +
             boxContents.toString() + "]");
      counter++;
    }
  }

  public static void main(String[] args) {
    java.util.ArrayList<Box<Integer>> listOfIntegerBoxes =
      new java.util.ArrayList<>();
    BoxDemo.<Integer>addBox(Integer.valueOf(10), listOfIntegerBoxes);
    BoxDemo.addBox(Integer.valueOf(20), listOfIntegerBoxes);
    BoxDemo.addBox(Integer.valueOf(30), listOfIntegerBoxes);
    BoxDemo.outputBoxes(listOfIntegerBoxes);
  }
}

以下是此示例的输出：

Box #0 contains [10]
Box #1 contains [20]
Box #2 contains [30]

通用方法addBox定义一个名为的类型参数U 。通常，Java编译器可以推断通用方法调用的类型参数。因此，在大多数情况下，您不必指定它们。例如，调用通用方法addBox ，您可以使用类型见证人来指定type参数，如下所示：

BoxDemo.<Integer>addBox(Integer.valueOf(10), listOfIntegerBoxes);

另外，如果省略类型见证人，则Java编译器会自动（从方法的参数中）推断出type参数为Integer ：

BoxDemo.addBox(Integer.valueOf(20), listOfIntegerBoxes);

泛型类的类型推断和实例化

您可以使用一组空的类型参数来替换调用通用类的构造函数所需的类型参数（ <> ），只要编译器可以从上下文推断出类型实参即可。这对尖括号被非正式地称为菱形 。

例如，考虑以下变量声明：

Map<String, List<String>> myMap = new HashMap<String, List<String>>();

您可以用一组空的类型参数（ <> ）代替构造函数的参数化类型：

Map<String, List<String>> myMap = new HashMap<>();

注意，要在泛型类实例化期间利用类型推断的优势，必须使用菱形。在以下示例中，编译器生成未经检查的转换警告，因为HashMap()构造函数是指HashMap原始类型，不是Map>类型：

Map<String, List<String>> myMap = new HashMap(); // unchecked conversion warning

泛型和非泛型类的类型推断和泛型构造函数

请注意，构造函数在通用类和非通用类中都可以是通用的（换句话说，声明自己的形式类型参数）。考虑以下示例：

class MyClass<X> {
  <T> MyClass(T t) {
    // ...
  }
}

考虑以下类的实例化MyClass ：

new MyClass<Integer>("")

该语句创建参数化类型的实例MyClass ;该语句明确指定类型Integer对于形式类型参数， X ，属于一般类别MyClass 。请注意，该通用类的构造函数包含一个形式类型参数， T 。编译器推断类型String对于形式类型参数， T ，属于该通用类的构造函数（因为该构造函数的实际参数是String宾语）。

Java SE 7之前的发行版中的编译器能够推断泛型构造函数的实际类型参数，类似于泛型方法。但是，如果您使用菱形符号，则Java SE 7及更高版本中的编译器可以推断出实例化的泛型类的实际类型参数。 <> ）。考虑以下示例：

MyClass<Integer> myObject = new MyClass<>("");

在此示例中，编译器推断类型Integer对于形式类型参数， X ，属于一般类别MyClass 。推断类型String对于形式类型参数， T ，该泛型类的构造函数。

注意：必须注意，推理算法仅使用调用参数，目标类型，并且可能使用明显的预期返回类型。推理算法不使用程序后面的结果。

目标类型

Java编译器利用目标类型来推断通用方法调用的类型参数。表达式的目标类型是Java编译器期望的数据类型，具体取决于表达式出现的位置。考虑方法Collections.emptyList ，其声明如下：

static <T> List<T> emptyList();

考虑以下赋值语句：

List<String> listOne = Collections.emptyList();

该语句期望一个实例List ;此数据类型是目标类型。因为方法emptyList返回类型的值List ，编译器推断type参数T必须是值String 。这适用于Java SE 7和8。或者，您可以使用类型见证人并指定T如下：

List<String> listOne = Collections.<String>emptyList();

但是，在这种情况下，这不是必需的。但是，在其他情况下这是必要的。请考虑以下方法：

void processStringList(List<String> stringList) {
    // process stringList
}

假设您要调用该方法processStringList一个空列表。在Java SE 7中，以下语句不会编译：

processStringList(Collections.emptyList());

Java SE 7编译器生成类似于以下内容的错误消息：

List<Object> cannot be converted to List<String>

编译器需要type参数的值T所以从价值开始Object 。因此，调用Collections.emptyList返回类型的值List
