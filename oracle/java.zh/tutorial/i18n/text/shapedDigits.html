将拉丁数字转换为其他Unicode数字

默认情况下，当文本包含数字值时，这些值使用拉丁（欧洲）数字显示。当首选其他Unicode数字形状时，请使用java.awt.font.NumericShaper类。的NumericShaper API使您能够以Unicode数字形状显示内部表示为ASCII值的数字值。

以下代码段来自ArabicDigits示例，显示了如何使用NumericShaper将拉丁数字转换为阿拉伯数字的实例。决定整形动作的线为粗体。

ArabicDigitsPanel(String fontname) {
    HashMap map = new HashMap();
    Font font = new Font(fontname, Font.PLAIN, 60);
    map.put(TextAttribute.FONT, font);
    map.put(TextAttribute.NUMERIC_SHAPING,
        NumericShaper.getShaper(NumericShaper.ARABIC));

    FontRenderContext frc = new FontRenderContext(null, false, false);
    layout = new TextLayout(text, map, frc);
}

// ...

public void paint(Graphics g) {
    Graphics2D g2d = (Graphics2D)g;
    layout.draw(g2d, 10, 50);
}

的NumericShaper提取阿拉伯数字实例并将其放入HashMap为了TextLayout.NUMERIC_SHAPING属性键。哈希图被传递给TextLayout实例。在渲染文本后paint方法，数字以所需的脚本显示。在此示例中，拉丁数字0到9被绘制为阿拉伯数字。

前面的示例使用NumericShaper.ARABIC常数以检索所需的整形器，但是NumericShaper类提供了许多语言的常量。这些常数定义为位掩码，并称为NumericShaper 基于位掩码的常量 。

基于枚举的范围常数

指定一组特定数字的另一种方法是使用NumericShaper.Range枚举类型（枚举）。在Java SE 7发行版中引入的该枚举还提供了一组常量 。尽管使用不同的机制定义了这些常量， NumericShaper.ARABIC位掩码在功能上等效于NumericShaper.Range.ARABIC枚举，并且有一个对应的getShaper每种常量类型的方法：

  • getShaper(int singleRange)
  • getShaper(NumericShaper.Range singleRange)

的ArabicDigitsEnum该示例与ArabicDigits示例相同，除了它使用NumericShaper.Range枚举以指定语言脚本：

ArabicDigitsEnumPanel(String fontname) {
    HashMap map = new HashMap();
    Font font = new Font(fontname, Font.PLAIN, 60);
    map.put(TextAttribute.FONT, font);
    map.put(TextAttribute.NUMERIC_SHAPING,
        NumericShaper.getShaper(NumericShaper.Range.ARABIC));
    FontRenderContext frc = new FontRenderContext(null, false, false);
    layout = new TextLayout(text, map, frc);
}

都getShaper方法接受singleRange参数。无论使用哪种常量类型，都可以指定脚本特定数字的范围。基于位掩码的常量可以使用OR操作数，或者您可以创建一组NumericShaper.Range枚举。下面显示了如何使用每种常量类型定义范围：

NumericShaper.MONGOLIAN | NumericShaper.THAI |
NumericShaper.TIBETAN
EnumSet.of(
    NumericShaper.Range.MONGOLIAN,
    NumericShaper.Range.THAI,
    NumericShaper.Range.TIBETAN)

您可以查询NumericShaper对象来确定它支持使用哪个范围getRanges基于位掩码的整形器的方法或getRangeSet枚举整形器的方法。

注意： 

您可以使用传统的基于位掩码的常量，也可以使用Range基于枚举的常量。在决定使用哪种方法时，请考虑以下因素：

  • 的Range API需要JDK 7或更高版本。
  • 的Range与位掩码API相比，API涵盖了更多的Unicode范围。
  • 位掩码API的速度比Range API。

根据语言上下文渲染数字

的ArabicDigits该示例旨在将成形器用于特定语言，但是有时必须根据语言上下文来呈现数字。例如，如果数字前面的文本使用泰语脚本，则首选泰数字。如果文字以藏文显示，则以藏文数字为佳。

您可以使用getContextualShaper方法：

  • getContextualShaper（int范围）
  • getContextualShaper（int range，int defaultContext）
  • getContextualShaper（设置范围）
  • getContextualShaper（设置范围，NumericShaper。范围defaultContext）

前两种方法使用位掩码常量，后两种使用枚举常量。接受defaultContext参数使您可以指定在文本前显示数值时使用的初始成形器。如果未定义默认上下文，则使用拉丁形状显示任何前导数字。

的ShapedDigits示例显示了成型器的工作方式。显示五个文本布局：

  1. 第一种布局不使用整形器；所有数字均显示为拉丁文。
  2. 第二种布局将所有数字塑造为阿拉伯数字，而与语言环境无关。
  3. 第三种布局使用使用阿拉伯数字的上下文整形器。默认上下文定义为阿拉伯语。
  4. 第四种布局使用使用阿拉伯数字的上下文整形器，但是整形器未指定默认上下文。
  5. 第五种布局采用上下文整形器，该整形器使用ALL_RANGES位掩码，但整形器未指定默认上下文。

以下代码行显示了成形器（如果使用）的定义方式：

  1. 不使用整形器。
  2. NumericShaper arabic = NumericShaper.getShaper(NumericShaper.ARABIC);
  3. NumericShaper contextualArabic = NumericShaper.getContextualShaper(NumericShaper.ARABIC, NumericShaper.ARABIC);
  4. NumericShaper contextualArabicASCII = NumericShaper.getContextualShaper(NumericShaper.ARABIC);
  5. NumericShaper contextualAll = NumericShaper.getContextualShaper(NumericShaper.ALL_RANGES);

见ShapedDigits.java进一步实现细节的示例。
