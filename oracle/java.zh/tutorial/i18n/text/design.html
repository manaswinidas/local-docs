设计注意事项

要编写可使用任何脚本无缝运行于任何语言的代码，请记住以下几点。

考虑 原因
避免使用char数据类型。 避免使用char原始数据类型或使用char数据类型，因为使用该数据类型的代码不适用于补充字符。对于采用char类型参数，使用相应的int方法（如果有）。例如，使用Character.isDigit(int)方法而不是Character.isDigit(char)方法。
使用isValidCodePoint验证代码点值的方法。 代码点定义为int数据类型，它允许代码点值的有效范围之外的值（从0x0000到0x10FFFF）。出于性能原因，将代码点值作为参数的方法不会检查参数的有效性，但是您可以使用isValidCodePoint方法来检查值。
使用codePointCount字符计数方法。 的String.length()方法返回代码单位的数量，即16位char字符串中的值。如果字符串包含补充字符，则该计数可能会引起误解，因为它不会反映出代码点的真实数量。要准确计算字符数（包括补充字符），请使用codePointCount方法。
使用String.toUpperCase(int codePoint)和String.toLowerCase(int codePoint)方法而不是Character.toUpperCase(int codePoint)要么Character.toLowerCase(int codePoint)方法。 而Character.toUpperCase(int)和Character.toLowerCase(int)方法确实可以处理代码点值，有些字符不能一对一转换。例如，小写的德文字符ß转换为大写字母时，会变成两个字符SS。同样，小希腊Sigma字符也取决于字符串中的位置。的Character.toUpperCase(int)和Character.toLowerCase(int)方法无法处理此类情况；然而String.toUpperCase和String.toLowerCase方法可以正确处理这些情况。
删除字符时要小心。 调用时StringBuilder.deleteCharAt(int index)要么StringBuffer.deleteCharAt(int index)索引指向补充字符的方法，仅该字符的前半部分（前半部分char值）被删除。首先，调用Character.charCount字符上的方法以确定是一还是两个char值必须删除。
反转序列中的字符时要小心。 调用时StringBuffer.reverse()要么StringBuilder.reverse()在包含补充字符的文本的方法中，高和低代理对被反转，这将导致错误的代理对，甚至可能是无效的代理对。
