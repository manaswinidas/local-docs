课程：国际化服务提供商

国际化的服务提供商启用了与语言环境相关的数据和服务的插件。由于可以插入与语言环境相关的数据和服务，因此第三方能够在应用程序中提供大多数对语言环境敏感的类的实现。 java.text和java.util包。

服务是一组编程接口和类，它们提供对特定应用程序功能或特性的访问。服务提供者接口（SPI）是服务定义的一组公共接口和抽象类。服务提供商实施SPI。服务提供商使您能够创建可扩展的应用程序，您可以在不修改其原始代码库的情况下对其进行扩展。您可以使用新的插件或模块来增强其功能。有关服务提供商和可扩展应用程序的更多信息，请参见创建可扩展应用程序 。

您可以使用服务提供商进行国际化，以提供以下区域设置敏感类的自定义实现：

  • BreakIterator对象
  • Collator对象
  • 语言代码，国家/地区代码和变体名称Locale类
  • 时区名称
  • 货币符号
  • DateFormat对象
  • DateFormatSymbols对象
  • NumberFormat对象
  • DecimalFormatSymbols对象

相应的SPI都包含在java.text.spi和在java.util.spi套餐：

java.util.spi java.text.spi
  • CurrencyNameProvider
  • LocaleServiceProvider
  • TimeZoneNameProvider
  • BreakIteratorProvider
  • CollatorProvider
  • DateFormatProvider
  • DateFormatSymbolsProvider
  • DecimalFormatSymbolsProvider
  • NumberFormatProvider

例如，如果您想提供一个NumberFormat针对新语言环境的对象，请实施java.text.spi.NumberFormatProvider类并实现以下方法：

  • getCurrencyInstance(Locale locale)
  • getIntegerInstance(Locale locale)
  • getNumberInstance(Locale locale)
  • getPercentInstance(Locale locale)
Locale loc = new Locale("da", "DK");
NumberFormat nf = NumberFormatProvider.getNumberInstance(loc);

这些方法首先检查Java运行时环境是否支持请求的语言环境。如果是这样，则该方法将使用该支持。否则，这些方法将调用getAvailableLocales适当接口的已安装提供程序的方法，以查找支持请求的语言环境的提供程序。

有关如何使用服务提供商进行国际化的深入示例，请参阅安装自定义资源束作为扩展 。本节说明如何实施ResourceBundleControlProvider界面，使您可以使用任何自定义ResourceBundle.Control类，而无需对应用程序的源代码进行任何其他更改。
