安装自定义资源包作为扩展

“ 定制资源束加载 ”部分显示了如何更改资源束的加载方式。这涉及从班级派生一个新班级ResourceBundle.Control ，然后通过调用以下方法检索资源包：

ResourceBundle getBundle(
  String baseName,
  Locale targetLocale,
  ResourceBundle.Control control)

参数control是您的实现ResourceBundle.Control 。

的java.util.spi.ResourceBundleControlProvider界面使您可以更改以下方法加载资源束的方式：

ResourceBundle getBundle(
  String baseName,
  Locale targetLocale)

请注意，此版本的ResourceBundle.getBundle方法不需要的实例ResourceBundle.Control类。 ResourceBundleControlProvider是服务提供商接口（SPI）。SPI使您能够创建可扩展的应用程序，这些应用程序可以轻松扩展而无需修改其原始代码库。有关更多信息，请参见创建可扩展应用程序 。

要使用SPI，您首先需要通过实现SPI这样的方式来创建服务提供商ResourceBundleControlProvider 。实施SPI时，请指定其将如何提供服务。的服务ResourceBundleControlProvider SPI提供的是获得适当的ResourceBundle.Control应用程序调用方法时的实例ResourceBundle.getBundle(String baseName, Locale targetLocale) 。您将服务提供程序与Java扩展机制打包为已安装的扩展。运行应用程序时，不要在类路径中命名扩展名。运行时环境查找并加载这些扩展。

已安装的实现ResourceBundleControlProvider SPI替代默认值ResourceBundle.Control类（定义默认的捆绑包加载过程）。因此， ResourceBundleControlProvider界面使您可以使用任何自定义ResourceBundle.Control类，而无需对应用程序的源代码进行任何其他更改。此外，该接口使您无需引用任何自定义即可编写应用程序ResourceBundle.Control类。

的RBCPTest.java该示例显示了如何实施ResourceBundleControlProvider界面以及如何将其打包为已安装的扩展程序。此示例，打包在zip文件中RBCPTest.zip ，由以下文件组成：

  • src
    • java.util.spi.ResourceBundleControlProvider
    • RBCPTest.java
    • rbcp
      • PropertiesResourceBundleControl.java
      • PropertiesResourceBundleControlProvider.java
      • XMLResourceBundleControl.java
      • XMLResourceBundleControlProvider.java
    • resources
      • RBControl.properties
      • RBControl_zh.properties
      • RBControl_zh_CN.properties
      • RBControl_zh_HK.properties
      • RBControl_zh_TW.properties
      • XmlRB.xml
      • XmlRB_ja.xml
  • lib
    • rbcontrolprovider.jar
  • build ：包含打包在其中的所有文件rbcontrolprovider.jar以及类文件RBCPTest.class
  • build.xml

以下步骤显示了如何重新创建文件的内容RBCPTest.zip ，如何RBCPTest示例作品，以及如何运行它：

  1. 创建ResourceBundle的实现。控制类。
  2. 实现ResourceBundleControlProvider接口。
  3. 在您的应用程序中，调用方法ResourceBundle.getBundle。
  4. 通过创建配置文件来注册服务提供商。
  5. 将提供程序，其必需的类和配置文件打包到JAR文件中。
  6. 运行RBCPTest程序。

1。创建ResourceBundle的实现。控制类。

的RBCPTest.java该示例使用两种实现ResourseBundle.Control ：

  • PropertiesResourceBundleControlProvider.java ：一样ResourceBundle.Control在“ 自定义资源包加载”中定义的实现。

  • XMLResourceBundleControl.java ： 这个ResourceBundle.Control实现使用方法加载基于XML的包Properties.loadFromXML 。

XML属性文件

如使用属性文件支持ResourceBundle一节中所述，属性文件是简单的文本文件。它们在每一行上包含一个键/值对。XML属性文件就像属性文件一样：它们包含键值对，但它们具有XML结构。以下是XML属性文件XmlRB.xml ：

<?xml version="1.0" encoding="UTF-8"?>

<!DOCTYPE properties [
<!ELEMENT properties ( comment?, entry* ) >
<!ATTLIST properties version CDATA #FIXED "1.0">
<!ELEMENT comment (#PCDATA) >
<!ELEMENT entry (#PCDATA) >
<!ATTLIST entry key CDATA #REQUIRED>
]>

<properties>
    <comment>Test data for RBCPTest.java</comment>
    <entry key="type">XML</entry>
</properties>

以下是等效的属性文本文件：

# Test data for RBCPTest.java
type = XML

所有XML属性文本文件都具有相同的结构：

  • 指定文档类型定义（DTD）的DOCTYPE声明：DTD定义XML文件的结构。注意 ：可以在XML属性文件中使用以下DOCTYPE声明：

    <!DOCTYPE properties SYSTEM "http://java.sun.com/dtd/properties.dtd">

    系统URI（ http://java.sun.com/dtd/properties.dtd ）在导出或导入属性时无法访问；它是一个字符串，唯一标识XML属性文件的DTD。

  • 根元素 ：此元素包含所有其他元素。

  • 任意数量元素：这些用于评论。

  • 任意数量元素：使用属性key指定密钥；指定之间的键值标签。

见Properties类，以获取有关XML属性文件的更多信息。

2。实现ResourceBundleControlProvider接口。

此界面包含一种方法， ResourceBundle.Control getControl(String baseName)方法。参数baseName是资源束的名称。在方法定义中getBundle ，指定的实例ResourceBundle.Control应当以资源束的名称返回。

的RBCPTest该示例包含的两个实现ResourceBundleControlProvider接口， PropertiesResourceBundleControlProvider.java和XMLResourceBundleControlProvider.java 。方法PropertiesResourceBundleControlProvider.getBundle返回的实例PropertiesResourceBundleControl资源束的基本名称是否以resources.RBControl （在此示例中，所有资源文件都包含在包中resources ）：



package rbcp;

import java.util.ResourceBundle;
import java.util.spi.ResourceBundleControlProvider;

public class PropertiesResourceBundleControlProvider
    implements ResourceBundleControlProvider {
    static final ResourceBundle.Control PROPERTIESCONTROL =
        new PropertiesResourceBundleControl();

    public ResourceBundle.Control getControl(String baseName) {
        System.out.println("Class: " + getClass().getName() + ".getControl");
        System.out.println("    called for " + baseName);

        // Throws a NPE if baseName is null.
        if (baseName.startsWith("resources.RBControl")) {
            System.out.println("    returns " + PROPERTIESCONTROL);
            return PROPERTIESCONTROL;
        }
        System.out.println("    returns null");
        System.out.println();
        return null;
    }
}

同样，方法XMLResourceBundleControlProvider.getControl返回的实例XMLResourceBundleControl资源束的基本名称是否以resources.Xml 。

注意 ：您可以创建ResourceBundleControlProvider返回一个实例的接口PropertiesResourceBundleControl要么XMLResourceBundleControl取决于基本名称。

3。在您的应用程序中，调用方法ResourceBundle.getBundle。

班级RBCPTest使用方法检索资源包ResourceBundle.getBundle ：


import java.io.*;
import java.net.*;
import java.util.*;

public class RBCPTest {
    public static void main(String[] args) {
        ResourceBundle rb = ResourceBundle.getBundle(
            "resources.XmlRB", Locale.ROOT);
        String type = rb.getString("type");
        System.out.println("Root locale. Key, type: " + type);
        System.out.println();

        rb = ResourceBundle.getBundle("resources.XmlRB", Locale.JAPAN);
        type = rb.getString("type");
        System.out.println("Japan locale. Key, type: " + type);
        System.out.println();

        test(Locale.CHINA);
        test(new Locale("zh", "HK"));
        test(Locale.TAIWAN);
        test(Locale.CANADA);
    }

    private static void test(Locale locale) {
        ResourceBundle rb = ResourceBundle.getBundle(
            "resources.RBControl", locale);
        System.out.println("locale: " + locale);
        System.out.println("    region: " + rb.getString("region"));
        System.out.println("    language: " + rb.getString("language"));
        System.out.println();
    }
}

请注意，没有实现ResourceBundle.Control要么ResourceBundleControlProvider出现在此类中。因为ResourceBundleControlProvider界面使用Java扩展机制，运行时环境查找并加载这些实现。然而， ResourceBundleControlProvider与Java扩展机制一起安装的实现和其他服务提供者使用ServiceLoader类。使用此类意味着您必须向配置文件注册服务提供商，这将在下一步中进行描述。

4。通过创建配置文件来注册服务提供商。

配置文件的名称是提供程序实现的接口或类的标准名称。配置文件包含提供程序的完全限定的类名。文件java.util.spi.ResourceBundleControlProvider包含的完全限定名称PropertiesResourceBundleControlProvider和XMLResourceBundleControlProvider ，每行一个名称：

rbcp.XMLResourceBundleControlProvider
rbcp.PropertiesResourceBundleControlProvider

5，将提供程序，其必需的类和配置文件打包到JAR文件中。

编译源文件。从包含文件的目录build.xml ，运行以下命令：

javac -d build src/java.* src/rbcp/*.java

此命令将编译包含在src目录并将类文件放在build目录。在Windows上，请确保使用反斜杠（ \ ）分隔目录和文件名。

创建一个JAR文件，其中包含以下目录结构中的已编译类文件，资源文件和配置文件：

  • META-INF
    • services
      • java.util.spi.ResourceBundleControlProvider
  • rbcp
    • PropertiesResourceBundleControl.class
    • PropertiesResourceBundleControlProvider.class
    • XMLResourceBundleControl.class
    • XMLResourceBundleControlProvider.class
  • resources
    • RBControl.properties
    • RBControl_zh.properties
    • RBControl_zh_CN.properties
    • RBControl_zh_HK.properties
    • RBControl_zh_TW.properties
    • XmlRB.xml
    • XmlRB_ja.xml

注意配置文件java.util.spi.ResourceBundleControlProvider必须打包在目录中/META-INF/services 。本示例将这些文件打包在JAR文件中rbcontrolprovider.jar在里面lib目录。

有关创建JAR文件的更多信息，请参见JAR文件中的打包程序 。

或者，下载并安装Apache Ant ，这是一个使您能够自动化构建过程的工具，例如编译Java文件和创建JAR文件。确保Apache Ant可执行文件位于您的PATH环境变量，以便您可以从任何目录运行它。安装Apache Ant之后，请按照以下步骤操作：

  1. 编辑档案build.xml并改变${JAVAC}到Java编译器的完整路径名， javac和${JAVA}到Java运行时可执行文件的完整路径名， java 。

  2. 从包含文件的同一目录中运行以下命令build.xml ：

    ant jar

    此命令编译Java源文件，并将它们以及所需的资源和配置文件打包到JAR文件中rbcontrolprovider.jar在里面lib目录。

6。运行RBCPTest程序。

在命令提示符处，从包含以下内容的目录中运行以下命令： build.xml文件：

java -Djava.ext.dirs=lib -cp build RBCPTest

该命令假定以下内容：

  • 包含RBCPTest示例的已编译代码的JAR文件位于目录中lib 。
  • 编译的类， RBCPTest.class ， 在里面build目录。

或者，使用Apache Ant并从包含以下内容的目录中运行以下命令： build.xml文件：

ant run

安装Java扩展程序时，通常将扩展程序的JAR文件放入lib/ext您的JRE目录。但是，此命令使用system属性指定包含Java扩展的目录java.ext.dirs 。

Class: rbcp.XMLResourceBundleControlProvider.getControl

Root locale. Key, type: XML

Class: rbcp.XMLResourceBundleControlProvider.getControl
    called for resources.XmlRB
    returns rbcp.XMLResourceBundleControl@16c1857
Japan locale. Key, type: Value from Japan locale

该程序成功获取了一个实例XMLResourceBundleControl并访问属性文件XmlRB.xml和XmlRB_ja.xml 。
