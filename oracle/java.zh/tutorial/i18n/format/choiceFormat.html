处理复数

如果复数形式和单数形式都可能，则消息中的单词可能会有所不同。随着ChoiceFormat在课堂上，您可以将数字映射到单词或短语，从而构造语法正确的消息。

在英语中，单词的复数形式和单数形式通常是不同的。当您构造引用数量的消息时，这可能会带来问题。例如，如果您的消息报告磁盘上的文件数，则可能有以下变化：

There are no files on XDisk.
There is one file on XDisk.
There are 2 files on XDisk.

解决此问题的最快方法是创建一个MessageFormat像这样的模式：

There are {0,number} file(s) on {1}.

不幸的是，上述模式导致语法错误：

There are 1 file(s) on XDisk.

只要您使用ChoiceFormat类。在本节中，您将通过逐步执行一个示例程序来学习如何处理消息中的复数形式。 ChoiceFormatDemo 。该程序还使用MessageFormat类，将在上一节“ 处理复合消息”中进行讨论 。

1。定义消息模式

首先，确定消息中的变量：

接下来，将消息中的变量替换为参数，创建可应用于MessageFormat宾语：

There {0} on {1}.

磁盘名称的参数，由表示{1} ，很容易处理。你就像对待其他任何东西一样String一个变量MessageFormat图案。此参数与参数值数组中索引1处的元素匹配。（请参阅步骤7。 ）

处理论点{0}更复杂，原因有两个：

  • 该参数替换的短语随文件数量的不同而不同。要在运行时构造此短语，您需要将文件数映射到特定的String 。例如，数字1将映射到String包含短语is one file 。的ChoiceFormat类允许您执行必要的映射。
  • 如果磁盘包含多个文件，则该短语包括一个整数。的MessageFormat类可让您在短语中插入数字。

2。创建一个ResourceBundle

由于必须翻译消息文本，因此请将其隔离在ResourceBundle ：

ResourceBundle bundle = ResourceBundle.getBundle(
    "ChoiceBundle", currentLocale);

该示例程序支持ResourceBundle与属性文件。的ChoiceBundle_en_US.properties包含以下几行：

pattern = There {0} on {1}.
noFiles = are no files
oneFile = is one file
multipleFiles = are {2} files

此属性文件的内容显示了如何构造和格式化消息。第一行包含用于MessageFormat 。（请参阅步骤1。 ）其他行包含将替换参数的短语{0}在模式中。的短语multipleFiles键包含参数{2} ，将被替换为数字。

这是属性文件的法语版本，ChoiceBundle_fr_FR.properties

pattern = Il {0} sur {1}.
noFiles = n'y a pas de fichiers
oneFile = y a un fichier
multipleFiles = y a {2} fichiers

3。创建消息格式化程序

在这一步中，您将实例化MessageFormat并设置它Locale ：

MessageFormat messageForm = new MessageFormat("");
messageForm.setLocale(currentLocale);

4。创建选择格式器

的ChoiceFormat对象可让您根据double数字，特定String 。范围double数字，以及String它们映射到的对象在数组中指定：

double[] fileLimits = {0,1,2};
String [] fileStrings = {
    bundle.getString("noFiles"),
    bundle.getString("oneFile"),
    bundle.getString("multipleFiles")
};

ChoiceFormat映射中的每个元素double数组到元素中的String具有相同索引的数组。在示例代码中，0映射到String通过调用返回bundle.getString("noFiles") 。巧合的是，索引与fileLimits数组。如果代码已设置fileLimits[0]到七ChoiceFormat将数字7映射到fileStrings[0] 。

您指定double和String实例化时的数组ChoiceFormat ：

ChoiceFormat choiceForm = new ChoiceFormat(fileLimits, fileStrings);

5，套用图案

还记得您在步骤1中构建的模式吗？现在是时候从ResourceBundle并将其应用于MessageFormat宾语：

String pattern = bundle.getString("pattern");
messageForm.applyPattern(pattern);

6。分配格式

在此步骤中，您将分配给MessageFormat反对ChoiceFormat在第4步中创建的对象：

Format[] formats = {choiceForm, null, NumberFormat.getInstance()};
messageForm.setFormats(formats);

的setFormats方法分配Format消息模式中参数的对象。您必须调用applyPattern调用之前的方法setFormats方法。下表显示了Format数组对应于消息模式中的参数：

的Format的数组ChoiceFormatDemo程序

数组元素 模式参数
choiceForm {0}
null {1}
NumberFormat.getInstance() {2}

7。设置参数并设置消息格式

在运行时，程序将变量分配给传递给参数的参数数组。 MessageFormat宾语。数组中的元素对应于模式中的参数。例如， messageArgument[1]映射到模式参数{1} ，这是一个String包含磁盘名称。在上一步中，程序分配了一个ChoiceFormat反对论点{0}模式。因此分配给messageArgument[0]确定哪个String的ChoiceFormat对象选择。如果messageArgument[0]大于或等于2， String包含短语are {2} files替换参数{0}在模式中。分配给的号码messageArgument[2]将代替模式参数{2} 。这是尝试此代码的代码：

Object[] messageArguments = {null, "XDisk", null};

for (int numFiles = 0; numFiles < 4; numFiles++) {
    messageArguments[0] = new Integer(numFiles);
    messageArguments[2] = new Integer(numFiles);
    String result = messageForm.format(messageArguments);
    System.out.println(result);
}

8。运行演示程序

将程序显示的消息与ResourceBundle第2步。请注意ChoiceFormat对象选择正确的短语， MessageFormat对象用于构造适当的消息。输出ChoiceFormatDemo程序如下：

currentLocale = en_US
There are no files on XDisk.
There is one file on XDisk.
There are 2 files on XDisk.
There are 3 files on XDisk.

currentLocale = fr_FR
Il n'y a pas des fichiers sur XDisk.
Il y a un fichier sur XDisk.
Il y a 2 fichiers sur XDisk.
Il y a 3 fichiers sur XDisk.
