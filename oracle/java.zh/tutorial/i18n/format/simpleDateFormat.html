自定义格式

版本说明：此日期和时间部分使用java.util包中的日期和时间API。JDK 8版本中提供的java.time API提供了全面的日期和时间模型，该模型对java.util类进行了重大改进。在Date Time跟踪中描述了java.time API。“ 传统日期时间代码”页面可能会特别令人感兴趣。

上一节“ 使用预定义的格式 ”介绍了DateFormat类。在大多数情况下，这些预定义格式已足够。但是，如果您要创建自己的自定义格式，则可以使用SimpleDateFormat类。

以下代码示例演示了SimpleDateFormat类。您可以在名为的文件中找到示例的完整源代码。 SimpleDateFormatDemo 。

关于图案

当您创建一个SimpleDateFormat对象，您指定一个模式String 。模式的内容String确定日期和时间的格式。有关模式语法的完整说明，请参见“ 日期格式模式语法”中的表格。

以下代码根据模式设置日期和时间的格式String传递给SimpleDateFormat构造函数。的String由...返回format方法包含要显示的格式化日期和时间。

Date today;
String output;
SimpleDateFormat formatter;

formatter = new SimpleDateFormat(pattern, currentLocale);
today = new Date();
output = formatter.format(today);
System.out.println(pattern + " " + output);

模式和语言环境

的SimpleDateFormat类是对语言环境敏感的。如果实例化SimpleDateFormat没有Locale参数，它将根据默认格式设置日期和时间Locale 。模式和Locale确定格式。对于相同的模式， SimpleDateFormat如果日期和时间的格式可能不同， Locale不同。

在下面的示例代码中，模式被硬编码在创建SimpleDateFormat宾语：

Date today;
String result;
SimpleDateFormat formatter;

formatter = new SimpleDateFormat("EEE d MMM yy", currentLocale);
today = new Date();
result = formatter.format(today);
System.out.println("Locale: " + currentLocale.toString());
System.out.println("Result: " + result);

当。。。的时候currentLocale设置为不同的值，前面的代码示例生成以下输出：

Locale: fr_FR
Result: mar. 30 juin 09
Locale: de_DE
Result: Di 30 Jun 09
Locale: en_US
Result: Tue 30 Jun 09

非字母的字符被视为带引号的文本。也就是说，即使它们没有用单引号引起来，它们也将出现在格式化的文本中。

您指定的符号字母数量也决定了格式。例如，如果“ zz”模式导致“ PDT”，那么“ zzzz”模式将生成“ Pacific Daylight Time”。下表总结了这些规则：

文本 > = 4 完整形式
数 最少位数 较短的数字用零填充（一年中，如果“ y”的计数为2，则该年份将被截断为2位数字）
文字和数字 1-2 数字形式
