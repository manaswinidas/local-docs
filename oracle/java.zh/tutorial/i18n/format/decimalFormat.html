自定义格式

您可以使用DecimalFormat类，用于将十进制数字格式化为区域设置特定的字符串。此类允许您控制前导和尾随零，前缀和后缀，分组（千位）分隔符和十进制分隔符的显示。如果您要更改格式符号，例如小数点分隔符，则可以使用DecimalFormatSymbols与DecimalFormat类。这些类在数字格式设置方面提供了很大的灵活性，但是它们会使您的代码更复杂。

下面的文字使用了一些示例来说明DecimalFormat和DecimalFormatSymbols类。本材料中的代码示例来自一个名为的示例程序。 DecimalFormatDemo 。

构建模式

您指定的格式属性DecimalFormat带有图案String 。该模式确定格式化后的数字是什么样。有关模式语法的完整说明，请参见数字格式模式语法 。

以下示例通过传递模式来创建格式化程序String到DecimalFormat构造函数。的format方法接受double值作为参数，并以String ：

DecimalFormat myFormatter = new DecimalFormat(pattern);
String output = myFormatter.format(value);
System.out.println(value + " " + pattern + " " + output);

下表描述了前几行代码的输出。的value是数字，一个double ，将其格式化。的pattern是个String指定格式设置属性。的output ，这是一个String ，代表格式化的数字。

区域设置敏感格式

前面的示例创建了一个DecimalFormat默认对象Locale 。如果你想要一个DecimalFormat非默认对象Locale ，您实例化一个NumberFormat然后将其投射到DecimalFormat 。这是一个例子：

NumberFormat nf = NumberFormat.getNumberInstance(loc);
DecimalFormat df = (DecimalFormat)nf;
df.applyPattern(pattern);
String output = df.format(value);
System.out.println(pattern + " " + output + " " + loc.toString());

运行前面的代码示例将得到以下输出。第二列中的格式化数字随Locale ：

###,###.###      123,456.789     en_US
###,###.###      123.456,789     de_DE
###,###.###      123 456,789     fr_FR

到目前为止，这里讨论的格式化模式都遵循美国英语的惯例。例如，在模式###，###。##中，逗号是千位分隔符，句点表示小数点。如果您的最终用户没有接触到此约定，则该约定很好。但是，某些应用程序（例如电子表格和报告生成器）允许最终用户定义自己的格式设置模式。对于这些应用程序，最终用户指定的格式化模式应使用本地化的表示法。在这些情况下，您需要调用applyLocalizedPattern上的方法DecimalFormat宾语。

更改格式符号

您可以使用DecimalFormatSymbols类来更改出现在由format方法。这些符号包括小数点分隔符，分组分隔符，减号和百分号等。

下一个示例演示了DecimalFormatSymbols通过对数字应用奇怪的格式进行分类。异常格式是调用的结果setDecimalSeparator ， setGroupingSeparator和setGroupingSize方法。

DecimalFormatSymbols unusualSymbols = new DecimalFormatSymbols(currentLocale);
unusualSymbols.setDecimalSeparator('|');
unusualSymbols.setGroupingSeparator('^');

String strange = "#,##0.###";
DecimalFormat weirdFormatter = new DecimalFormat(strange, unusualSymbols);
weirdFormatter.setGroupingSize(4);

String bizarre = weirdFormatter.format(12345.678);
System.out.println(bizarre);

运行时，此示例以奇怪的格式打印数字：

1^2345|678

数字格式模式语法

您可以按照以下BNF图指定的规则设计自己的数字格式格式：

pattern    := subpattern{;subpattern}
subpattern := {prefix}integer{.fraction}{suffix}
prefix     := '\\u0000'..'\\uFFFD' - specialCharacters
suffix     := '\\u0000'..'\\uFFFD' - specialCharacters
integer    := '#'* '0'* '0'
fraction   := '0'* '#'*

下表解释了上图中使用的表示法：

符号 描述
X* 0个或更多X实例
(X | Y) X或Y
X..Y 从X到Y的任何字符，包括
S - T S中的字符，T中的字符除外
{X} X是可选的

在前面的BNF图中，第一个子模式指定正数的格式。第二个子模式是可选的，它指定负数的格式。

尽管在BNF图中未指出，但逗号可能会出现在整数部分内。

在子模式中，可以使用特殊符号指定格式。下表描述了这些符号：

符号 描述
0 一个数字
＃ 一个数字，零表示不存在
。 小数点分隔符的占位符
， 分组分隔符的占位符
Ë 以指数格式分隔尾数和指数
; 分隔格式
-- 默认负前缀
％ 乘以100并显示为百分比
？ 乘以1000并显示为千分之一
¤ 货币符号；由货币符号代替；如果翻倍，则用国际货币符号代替；如果存在于模式中，则使用货币十进制分隔符代替十进制分隔符
X 前缀或后缀中可以使用任何其他字符
' 用于在前缀或后缀中引用特殊字符
