BCP 47扩展

Java SE 7版本符合IETF BCP 47标准，该标准支持将扩展添加到Locale 。任何单个字符都可以用来表示扩展名，但是有两个预定义的扩展名代码： 'u'指定Unicode语言环境扩展名 ，以及'x'指定私有用途扩展名 。

Unicode语言环境扩展由Unicode 公共语言环境数据存储库（CLDR）项目定义。它们用于指定非特定于语言的信息，例如日历或货币。专用扩展名可用于指定任何其他信息，例如平台（例如Windows，UNIX或Linux）或发行信息（例如6u23或JDK 7）。

扩展名被指定为键/值对，其中键是单个字符（通常是'u'要么'x' ）。格式正确的值具有以下格式：

SUBTAG ('-' SUBTAG)*

格式如下：

SUBTAG = [0-9a-zA-Z]{2,8}    For key='u'
SUBTAG = [0-9a-zA-Z]{1,8}    For key='x'

请注意，私有用途扩展名允许使用单个字符的值。但是，在Unicode语言环境扩展中，值的最小值为2个字符。

扩展字符串不区分大小写，但是Locale类将所有键和值映射为小写。

Unicode语言环境扩展

如前所述，Unicode语言环境扩展由'u'关键代码或UNICODE_LOCALE_EXTENSION不变。值本身也由键/类型对指定。合法值在Unicode网站上的“ 键/类型定义”表中定义 。密钥代码由两个字母字符指定。下表列出了Unicode语言环境扩展键：

关键代码 描述
钙 日历算法
合作 整理类型
K a 整理参数
铜 货币类型
nu 数字类型
VA 普通变体类型
注意： 

指定Unicode语言环境扩展名（例如数字格式）并不能保证基础平台的语言环境服务将满足该请求。

下表显示了Unicode语言环境扩展的键/类型对的一些示例。

键/类型对 描述
佛教徒 泰国佛教日历
拼音 拉丁语拼音订购
cu-usd 美元
核蛋白 日本财务数字
tz-aldav 欧洲/安道尔

以下字符串使用Linux平台订购的电话簿样式表示德国国家/地区的德语语言环境。此示例还包含一个名为"email" 。

de-DE-u-email-co-phonebk-x-linux

下列Locale方法可用于访问有关Unicode语言环境扩展的信息。使用先前的德国语言环境示例描述了这些方法。

  • getUnicodeLocaleKeys() –返回Unicode语言环境键代码；如果语言环境中没有，则返回空集。对于德语示例，这将返回一个包含单个字符串的集合"co" 。
  • getUnicodeLocaleType(String) –返回与Unicode语言环境键代码关联的Unicode语言环境类型。调用中getUnicodeLocaleType("co")对于德国的示例将返回字符串"phonebk" 。
  • getUnicodeLocaleAttributes() –返回与此语言环境关联的Unicode语言环境属性集（如果有）。在德语示例中，这将返回一个包含单个字符串的集合"email" 。

私人用途扩展

专用扩展名，由'x'关键代码或PRIVATE_USE_EXTENSION常数，可以是任意值，只要该值格式正确即可。

以下是可能的私人用途扩展的示例：

x-jdk-1-7
x-linux
