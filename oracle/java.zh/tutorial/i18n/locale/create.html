创建语言环境

有几种方法可以创建Locale宾语。无论使用哪种技术，创建都可以像指定语言代码一样简单。但是，您可以通过设置区域（也称为“国家/地区”）和变体代码来进一步区分语言环境。如果使用的是JDK 7发行版或更高版本，则还可以指定脚本代码和Unicode语言环境扩展。

创建广告素材的四种方法Locale对象是：

  • Locale.Builder类
  • Locale建设者
  • Locale.forLanguageTag工厂方法
  • Locale常数
版本说明： Locale.Builder类和forLanguageTag Java SE 7版本中增加了该方法。

LocaleBuilder类

的Locale.Builder实用程序类可用于构造一个Locale符合IETF BCP 47语法的对象。例如，要指定法语和加拿大的国家/地区，您可以调用Locale.Builder构造函数，然后按如下所示链接setter方法：

Locale aLocale = new Locale.Builder().setLanguage("fr").setRegion("CA").build();

下一个示例创建Locale美国和英国的英语对象：

Locale bLocale = new Locale.Builder().setLanguage("en").setRegion("US").build();
Locale cLocale = new Locale.Builder().setLanguage("en").setRegion("GB").build();

最后的示例创建一个Locale俄语的对象：

Locale dLocale = new Locale.Builder().setLanguage("ru").setScript("Cyrl").build();

Locale建设者

可以使用三个构造函数Locale用于创建一个类Locale宾语：

  • Locale(String language)
  • Locale(String language, String country)
  • Locale(String language, String country, String variant)

以下示例创建Locale对象是加拿大的法语，美国和英国的英语以及俄语的对象。

aLocale = new Locale("fr", "CA");
bLocale = new Locale("en", "US");
cLocale = new Locale("en", "GB");
dLocale = new Locale("ru");

无法在Locale JDK 7之前的版本中的对象。

forLanguageTag工厂方法

如果您的语言标签字符串符合IETF BCP 47标准，则可以使用forLanguageTag(String)工厂方法，在Java SE 7发行版中引入。例如：

Locale aLocale = Locale.forLanguageTag("en-US");
Locale bLocale = Locale.forLanguageTag("ja-JP-u-ca-japanese");

Locale常数

为了您的方便Locale class提供某些语言和国家/地区的常量 。例如：

cLocale = Locale.JAPAN;
dLocale = Locale.CANADA_FRENCH;

当您指定语言常数时， Locale未定义。接下来的三个语句创建等效项Locale对象：

j1Locale = Locale.JAPANESE;
j2Locale = new Locale.Builder().setLanguage("ja").build();
j3Locale = new Locale("ja");

的Locale以下三个语句创建的对象也是等效的：

j4Locale = Locale.JAPAN;
j5Locale = new Locale.Builder().setLanguage("ja").setRegion("JP").build();
j6Locale = new Locale("ja", "JP");

语言代码是符合ISO 639标准的两个或三个小写字母。您可以在http://www.loc.gov/standards/iso639-2/php/code_list.php上找到ISO 639代码的完整列表。

有三种方法可以检索脚本的脚本信息。 Locale ：

  • getScript() –返回4个字母的脚本代码Locale宾语。如果没有为语言环境定义脚本，则返回一个空字符串。
  • getDisplayScript() –返回适合显示给用户的语言环境脚本的名称。如果可能，该名称将被本地化为默认语言环境。因此，例如，如果脚本代码为“ Latn”，则对于英语语言环境，返回的diplay脚本名称将为字符串“ Latin”。
  • getDisplayScript(Locale) –返回指定的显示名称Locale如果可能的话，本地化。

变体代码

可选的variant代码可用于进一步区分您的Locale 。例如，变体代码可用于指示区域代码未涵盖的辩证差异。

版本说明：在Java SE 7发行版之前，有时使用变体代码来识别并非特定于语言或地区的差异。例如，它可能已被用来识别Windows或UNIX等计算平台之间的差异。在IETF BCP 47标准下，不建议使用此方法。

要定义与您的环境相关的非语言特定变体，请使用扩展机制，如BCP 47扩展中所述 。

从符合IETF BCP 47标准的Java SE 7版本开始，变体代码专门用于表示定义一种语言或其方言的其他变体。IETF BCP 47标准对变体子标签施加了语法限制。您可以在http://www.iana.org/assignments/language-subtag-registry上看到变体代码列表（搜索变体 ）。

例如，Java SE使用变体代码来支持泰语。按照惯例， NumberFormat的对象th和th_TH语言环境将使用常见的阿拉伯数字形状或阿拉伯数字来格式化泰文数字。但是， NumberFormat为了th_TH_TH语言环境使用泰国数字形状。摘录自ThaiDigits.java证明了这一点：

String outputString = new String();
Locale[] thaiLocale = {
             new Locale("th"),
             new Locale("th", "TH"),
             new Locale("th", "TH", "TH")
         };

for (Locale locale : thaiLocale) {
    NumberFormat nf = NumberFormat.getNumberInstance(locale);
    outputString = outputString + locale.toString() + ": ";
    outputString = outputString + nf.format(573.34) + "\n";
}
