在NetBeans IDE中运行教程示例

Swing教程和部署教程中的示例也作为NetBeans项目提供，可以在NetBeans IDE（一个免费的集成开发环境，可以用来简化应用程序开发）中打开和运行这些项目。

此页面提供有关以下任务的信息：

  • 下载，安装和启动NetBeans IDE
  • 打开一个示例项目
  • 编译和运行项目

下载，安装和启动NetBeans IDE

获得NetBeans IDE的最简单方法是从NetBeans下载页面下载适用于您的操作系统的NetBeans IDE安装程序。

然后，您可以运行安装程序并在安装向导中完成一系列简短的步骤来设置IDE。您可以从下载页面上链接的安装说明中找到有关安装过程的更多详细信息。

然后，可以通过以下方式之一启动IDE：

  • 在Microsoft Windows系统上，可以使用“开始”菜单中的NetBeans IDE项。
  • 在Solaris OS和Linux系统上，通过导航到IDE的bin目录并键入./netbeans来执行IDE启动器脚本。
  • 在Mac OS X系统上，单击NetBeans IDE应用程序图标。

打开一个示例项目

在NetBeans IDE中打开项目包括以下步骤：

  • 下载一个教程示例或整个示例集。各个示例索引页（例如，“ 使用Swing组件：示例”索引页）上的“ NetBeans IDE项目示例”列中提供了各个示例。您可以将所有示例下载为Java Tutorial捆绑包的一部分。请参阅教程首页上的“下载最新的Java SE教程”链接。
  • 如果您下载了单个项目，请在系统上解压缩该项目。如果您下载了整个教程的zip文件或所有示例的zip文件，请解压缩该文件。
  • 在NetBeans IDE中，选择“文件”>“打开项目”。在文件选择器中，选择项目，然后单击“打开项目”。

打开后，该项目将显示在IDE的“项目”窗口中。展开项目的节点以显示项目的结构。该项目包括以下主要子节点：

  • 源码包 。包含代表您的源文件的Java包的节点。展开这些节点以访问项目的.java源文件。
  • 图书馆 。包含代表应用程序所依赖的库的节点。对于Swing教程中的示例，此处仅列出了JDK。

要打开文件，请在“项目”窗口中双击文件的节点。打开后，文件将显示在“项目”窗口右侧的“源代码编辑器”中。

“项目”窗口表示项目的“逻辑”结构，该结构与系统上文件的布局不对应。这种逻辑结构突出显示了您最需要访问的项目部分。“项目”窗口不显示已编译的.class文件。如果要查看整个项目在磁盘上的物理表示方式，请单击“文件”选项卡。

编译和运行项目

对于通过浏览器访问的应用程序，在编译项目之前，在JAR文件清单中添加Codebase和Permissions属性。在“文件”选项卡中，展开项目文件夹以查看子文件夹和文件。双击manifest.mf打开清单文件，并将Codebase属性设置为从中加载应用程序的位置。将Permissions属性设置为应用程序所需的权限级别。有关更多信息，请参见《 Java平台标准版部署指南》中的“用于安全性的JAR文件清单属性 ”。

要编译整个项目，请在“项目”窗口中右键单击该项目的节点，然后选择“生成项目”。

将打开“输出”窗口，并在编译进度中显示输出。

如果构建输出以以下语句结束BUILD FAILED ，则您的代码中可能存在语法错误。错误在“输出”窗口中报告为超链接文本。您双击这样的超链接，以导航到错误的来源。然后，您可以修复错误，然后再次选择“生成项目”。

在生成项目时，将生成字节码.class文件。您可以通过打开“文件”窗口并展开项目的文件夹来查看在何处生成新文件。

要运行您的项目，请在“项目”窗口中右键单击该项目的节点，然后选择“运行项目”。

注意：您还可以在主菜单中使用“构建主项目”和“运行主项目”命令。但是，这些命令仅在当前指定为“主”项目的项目上起作用。通过在“项目”窗口中右键单击其节点，然后选择“设置主项目”，可以将项目设为主项目。
