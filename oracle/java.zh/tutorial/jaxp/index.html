Trail：用于XML处理的Java API（JAXP）

XML处理XML API（JAXP）路径通过JAXP应用程序示例介绍了XML处理Java API（JAXP）1.4技术。

在阅读本教程之前

要充分利用XML处理Java API（JAXP）教程中的信息，您应该了解以下技术：

  • Java编程语言及其开发环境。
  • 可扩展标记语言（XML）
  • 由万维网联盟（W3C）DOM工作组定义的文档对象模型（DOM）。
  • XML的简单API（SAX），由XML-DEV邮件列表的成员合作开发。

假定对DOM和SAX有一些先验知识。本教程仅详细检查JAXP API特有的代码。

JAXP简介简要介绍了JAXP技术，包括其用途和主要功能。

XML的简单API引入了JAXP技术中使用的概念XML的简单API（SAX）：何时使用SAX，如何解析XML文件，如何实现SAX验证，如何运行SAX解析器以及如何处理词汇事件。提供了更多信息的链接。

文档对象模型介绍了文档对象模型（DOM）所使用的树结构，并向您展示了如何使用DOM函数创建节点，删除节点，更改节点内容以及遍历节点层次结构。

可扩展样式表语言转换包括有关如何将文档对象模型编写为XML文件以及如何从任意数据文件生成DOM以便将其转换为XML的信息。

用于XML的流API专注于基于流的Java技术，事件驱动的拉解析API，用于读取和写入XML文档。StAX使您能够创建双向XML解析器，这些解析器速度快，相对容易编程并且占用的内存少。

JAXP 1.5和“新属性”引入了已添加到7u40和JDK8中的属性。

处理限制讨论了JAXP实施限制，包括在7u45中添加的三个限制。
