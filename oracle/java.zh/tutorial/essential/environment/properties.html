物产

属性是作为键/值对管理的配置值。在每对中，键和值都是String价值观。键标识并用于检索值，就像变量名用于检索变量的值一样。例如，能够下载文件的应用程序可能使用名为“ download.lastDirectory”的属性来跟踪上次下载所使用的目录。

要管理属性，请创建java.util.Properties 。此类提供以下方法：

  • 将键/值对加载到Properties流中的对象
  • 从其键检索值，
  • 列出密钥及其值，
  • 枚举键，以及
  • 将属性保存到流中。

有关流的介绍，请参阅“ 基本I / O”课程中的“ I / O 流 ”部分。

Properties延伸java.util.Hashtable 。从中继承的一些方法Hashtable支持以下操作：

  • 测试以查看是否存在特定的键或值Properties宾语，
  • 获取当前键/值对的数量，
  • 删除密钥及其值，
  • 将键/值对添加到Properties清单，
  • 枚举值或键，
  • 通过其键检索值，以及
  • 找出是否Properties对象为空。
安全注意事项：对属性的访问必须经过当前安全经理的批准。假定本节中的示例代码段位于独立的应用程序中，默认情况下，这些应用程序没有安全管理器。小程序中的相同代码可能无法运行，具体取决于运行它的浏览器。有关小应用程序安全性限制的信息，请参阅Java小应用程序课程中的小应用程序可以做什么和不能做什么 。

的System类保持一个Properties定义当前工作环境配置的对象。有关这些属性的更多信息，请参见系统属性 。本节的其余部分说明如何使用属性来管理应用程序配置。

应用程序生命周期中的属性

下图说明了一个典型的应用程序如何通过以下方式管理其配置数据： Properties对象在其执行过程中。

  • Starting Up
    在应用程序启动时，将发生前三个框中给出的操作。首先，应用程序将知名位置的默认属性加载到Properties宾语。通常，默认属性会与默认属性一起存储在磁盘上的文件中。 .class以及该应用程序的其他资源文件。
    接下来，应用程序创建另一个Properties对象并加载自上次运行应用程序以来保存的属性。许多应用程序按用户存储属性，因此在此步骤中加载的属性通常位于该应用程序在用户主目录中维护的特定目录中的特定文件中。最后，应用程序使用默认的和记住的属性进行初始化。
    这里的关键是一致性。应用程序必须始终将属性加载并保存到相同位置，以便下次执行时可以找到它们。
  • Running
    在执行应用程序期间，用户可能会在“首选项”窗口中更改某些设置，并且Properties对象将更新以反映这些更改。如果要在以后的会话中记住用户的更改，则必须将其保存。
  • Exiting
    退出后，应用程序会将属性保存到其众所周知的位置，以在下次启动应用程序时再次加载。

设置属性对象

以下Java代码执行上一部分中描述的前两个步骤：加载默认属性和加载记住的属性：

. . .
// create and load default properties
Properties defaultProps = new Properties();
FileInputStream in = new FileInputStream("defaultProperties");
defaultProps.load(in);
in.close();

// create application properties with default
Properties applicationProps = new Properties(defaultProps);

// now load properties 
// from last invocation
in = new FileInputStream("appProperties");
applicationProps.load(in);
in.close();
. . .

首先，应用程序设置一个默认值Properties宾语。如果未在其他位置显式设置值，则此对象包含要使用的属性集。然后，load方法从磁盘上名为的文件中读取默认值defaultProperties 。

接下来，应用程序使用其他构造函数创建第二个Properties宾语， applicationProps ，其默认值包含在defaultProps 。检索属性时，默认值起作用。如果找不到该属性applicationProps ，然后搜索其默认列表。

最后，代码将一组属性加载到applicationProps来自一个名为appProperties 。该文件中的属性是上次调用时从应用程序中保存的属性，如下一节所述。

保存属性

下面的示例使用编写上一示例的应用程序属性Properties.store 。默认属性无需每次都保存，因为它们永远不会更改。

FileOutputStream out = new FileOutputStream("appProperties");
applicationProps.store(out, "---No Comment---");
out.close();

的store方法需要写入的流，以及在输出顶部用作注释的字符串。

获取物业信息

一旦应用程序设置好了Properties对象，应用程序可以查询对象以获取有关其包含的各种键和值的信息。应用程序从Properties启动后的对象，以便它可以根据用户的选择进行初始化。的Properties类有几种获取属性信息的方法：

  • contains(Object value)和containsKey(Object key)
    退货true如果值或键在Properties宾语。 Properties从继承这些方法Hashtable 。因此他们接受Object论据，但仅String应该使用值。
  • getProperty(String key)和getProperty(String key, String default)
    返回指定属性的值。第二个版本提供了默认值。如果找不到密钥，则返回默认值。
  • list(PrintStream s)和list(PrintWriter w)
    将所有属性写入指定的流或写入器。这对于调试很有用。
  • elements() ， keys()和propertyNames()
    返回一个Enumeration包含键或值（如方法名所示） Properties宾语。的keys方法仅返回对象本身的键；的propertyNames方法还返回默认属性的键。
  • stringPropertyNames()
    喜欢propertyNames ，但返回一个Set ，并且仅返回键和值均为字符串的属性的名称。请注意Set对象不受支持Properties对象，因此一个更改不会影响另一个。
  • size()
    返回键/值对的当前数量。

设定属性

用户在执行过程中与应用程序的交互可能会影响属性设置。这些变化应体现在Properties对象，以便在应用程序退出（并调用store方法）。以下方法可以更改Properties宾语：

  • setProperty(String key, String value)
    将键/值对放入Properties宾语。
  • remove(Object key)
    删除与键关联的键/值对。
注意：上述某些方法在Hashtable ，因此接受键和值参数类型不是String 。一律使用String键和值的，即使该方法允许其他类型。也不要调用Hashtable.set要么Hastable.setAll上Properties对象;总是使用Properties.setProperty 。
