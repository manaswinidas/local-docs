路径和类路径

本节说明如何使用PATH和CLASSPATH Microsoft Windows，Solaris和Linux上的环境变量。有关最新信息，请查阅Java开发套件（JDK）软件包的安装随附的安装说明。

安装软件后，JDK目录将具有以下所示的结构。

的bin目录包含编译器和启动器。

更新PATH环境变量（Microsoft Windows）

您可以很好地运行Java应用程序，而无需设置PATH环境变量。或者，您可以选择将其设置为方便。

设置PATH环境变量，如果您希望能够方便地运行可执行文件（ javac.exe ， java.exe ， javadoc.exe （等等），而无需键入命令的完整路径。如果不设置PATH变量，则每次运行时都需要指定可执行文件的完整路径，例如：

C:\Java\jdk1.7.0\bin\javac MyClass.java

的PATH环境变量是由分号（ ; ）。Microsoft Windows在PATH目录，从左到右。你应该只有一个bin路径中一次的JDK目录（忽略第一个之后的目录），因此，如果已经存在一个目录，则可以更新该特定条目。

以下是一个例子PATH环境变量：

C:\Java\jdk1.7.0\bin;C:\Windows\System32\;C:\Windows\;C:\Windows\System32\Wbem

设置PATH环境变量是永久的，因此它将在重启后仍然存在。永久更改PATH变量，请使用“控制面板”中的“ 系统”图标。确切的过程因Windows版本而异：

Windows XP
  1. 选择开始 ，然后选择控制面板 。双击系统 ，然后选择高级选项卡。
  2. 单击环境变量 。在“ 系统变量 ”部分中，找到PATH环境变量并选择它。点击编辑 。如果PATH环境变量不存在，请单击New 。
  3. 在“ 编辑系统变量 （或新系统变量 ）”窗口中，指定PATH环境变量。单击确定 。单击确定关闭所有剩余的窗口。
Windows Vista：
  1. 在桌面上，右键单击“ 我的电脑”图标。
  2. 从上下文菜单中选择“ 属性 ”。
  3. 单击“ 高级”选项卡（在Vista中为“ 高级系统设置”链接）。
  4. 单击环境变量 。在“ 系统变量 ”部分中，找到PATH环境变量并选择它。点击编辑 。如果PATH环境变量不存在，请单击New 。
  5. 在“ 编辑系统变量 （或新系统变量 ）”窗口中，指定PATH环境变量。单击确定 。单击确定关闭所有剩余的窗口。
Windows 7的：
  1. 在桌面上，右键单击“ 计算机”图标。
  2. 从上下文菜单中选择“ 属性 ”。
  3. 单击高级系统设置链接。
  4. 单击环境变量 。在“ 系统变量 ”部分中，找到PATH环境变量并选择它。点击编辑 。如果PATH环境变量不存在，请单击New 。
  5. 在“ 编辑系统变量 （或新系统变量 ）”窗口中，指定PATH环境变量。单击确定 。单击确定关闭所有剩余的窗口。
注意：您可能会看到一个PATH从“控制面板”进行编辑时，环境变量类似于以下内容：
%JAVA_HOME%\bin;%SystemRoot%\system32;%SystemRoot%;%SystemRoot%\System32\Wbem
用百分号括起来的变量（ % ）是现有的环境变量。如果在“控制面板”的“ 环境变量”窗口中列出了这些变量之一（例如JAVA_HOME ），然后可以编辑其值。如果未出现，则它是操作系统定义的特殊环境变量。例如， SystemRoot是Microsoft Windows系统文件夹的位置。要获取环境变量的值，请在命令提示符下输入以下内容。（此示例获取SystemRoot环境变量）：
echo %SystemRoot%

更新PATH变量（Solaris和Linux）

您可以运行JDK，而无需设置PATH变量，也可以选择将其设置为方便。但是，如果您希望能够运行可执行文件，则应设置path变量（ javac ， java ， javadoc （等等），而无需键入命令的完整路径。如果不设置PATH变量，则每次运行时都需要指定可执行文件的完整路径，例如：

% /usr/local/jdk1.7.0/bin/javac MyClass.java

要找出路径是否正确设置，请执行：

% java -version

这将打印java工具，如果可以找到它。如果版本太旧或出现错误java：Command not found ，则路径设置不正确。

要永久设置路径，请在启动文件中设置路径。

对于C shell（ csh ），编辑启动文件(~/.cshrc ）：

set path=(/usr/local/jdk1.7.0/bin $path)

对于bash ，编辑启动文件（ ~/.bashrc ）：

PATH=/usr/local/jdk1.7.0/bin:$PATH
export PATH

对于ksh ，启动文件由环境变量命名， ENV 。设置路径：

PATH=/usr/local/jdk1.7.0/bin:$PATH
export PATH

对于sh ，编辑配置文件（ ~/.profile ）：

PATH=/usr/local/jdk1.7.0/bin:$PATH
export PATH

然后加载启动文件，并通过重复执行以下步骤验证路径是否已设置java命令：

对于C shell（ csh ）：

% source ~/.cshrc
% java -version

对于ksh ， bash ， 要么sh ：

% . /.profile
% java -version

检查CLASSPATH变量（所有平台）

的CLASSPATH变量是一种告诉应用程序（包括JDK工具）在哪里寻找用户类的方法。（属于JRE，JDK平台和扩展名的类应通过其他方式定义，例如引导类路径或扩展名目录。）

指定类路径的首选方法是使用-cp命令行开关。这允许CLASSPATH为每个应用程序单独设置，而不会影响其他应用程序。设置CLASSPATH可能很棘手，应谨慎执行。

类路径的默认值为“。”，这意味着仅搜索当前目录。指定CLASSPATH变量或-cp命令行开关将覆盖此值。

检查是否CLASSPATH在Microsoft Windows NT / 2000 / XP上设置的，执行以下操作：

C:> echo %CLASSPATH%

在Solaris或Linux上，执行以下操作：

% echo $CLASSPATH

如果CLASSPATH如果未设置，则将得到CLASSPATH：未定义变量错误（Solaris或Linux）或仅是％CLASSPATH％ （Microsoft Windows NT / 2000 / XP）。

修改CLASSPATH ，请使用与PATH变量。

使用类路径通配符可以包含以下内容的整个目录： .jar类路径中的文件，而无需分别明确命名它们。有关更多信息，包括对类路径通配符的解释，以及有关如何清理CLASSPATH环境变量，请参阅设置类路径技术说明。
