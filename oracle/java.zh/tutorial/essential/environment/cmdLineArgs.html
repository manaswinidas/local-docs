命令行参数

Java应用程序可以从命令行接受任意数量的参数。这允许用户在启动应用程序时指定配置信息。

用户在调用应用程序时输入命令行参数，并在要运行的类的名称后指定它们。例如，假设有一个名为Sort对文件中的行进行排序。对名为的文件中的数据进行排序friends.txt ，用户将输入：

java Sort friends.txt

启动应用程序时，运行时系统将通过以下参数将命令行参数传递给应用程序的main方法： String s。在上一个示例中，命令行参数传递给了Sort包含单个数组的数组中的应用程序String ： "friends.txt" 。

回显命令行参数

的Echo该示例在一行上单独显示其每个命令行参数：


public class Echo {
    public static void main (String[] args) {
        for (String s: args) {
            System.out.println(s);
        }
    }
}

以下示例显示用户如何运行Echo 。用户输入以斜体显示。

java Echo Drink Hot Java
Drink
Hot
Java

请注意，该应用程序显示每个单词- Drink ， Hot和Java -仅靠一条线。这是因为空格字符分隔了命令行参数。具有Drink ， Hot和Java如果将其解释为单个参数，则用户可以通过将其用引号引起来来加入它们。

java Echo "Drink Hot Java"
Drink Hot Java

解析数字命令行参数

如果应用程序需要支持数字命令行参数，则必须转换为String代表数字（例如“ 34”）到数字值的参数。这是一个将命令行参数转换为int ：

int firstArg;
if (args.length > 0) {
    try {
        firstArg = Integer.parseInt(args[0]);
    } catch (NumberFormatException e) {
        System.err.println("Argument" + args[0] + " must be an integer.");
        System.exit(1);
    }
}

parseInt抛出一个NumberFormatException如果格式为args[0]无效。全部Number类- Integer ， Float ， Double ，依此类推-有parseXXX转换一个String代表其类型的对象的数字。
