模式类的方法

到目前为止，我们仅使用测试工具来创建Pattern最基本形式的对象。本节探讨了高级技术，例如使用标志创建模式以及使用嵌入式标志表达式。它还探讨了一些我们尚未讨论的其他有用方法。

用标志创建图案

的Pattern类定义备用compile接受一组影响模式匹配方式的标志的方法。flags参数是一个位掩码，可以包括以下任何公共静态字段：

  • Pattern.CANON_EQ启用规范对等。当指定此标志时，只有且仅当其完整规范分解匹配时，两个字符才会被视为匹配。表达方式"a\u030A" ，例如，将匹配字符串"\u00E5"指定此标志时。默认情况下，匹配不考虑规范对等。指定此标志可能会导致性能下降。
  • Pattern.CASE_INSENSITIVE启用不区分大小写的匹配。默认情况下，不区分大小写的匹配假定只匹配US-ASCII字符集中的字符。可以通过将UNICODE_CASE标志与该标志一起指定来启用Unicode感知的不区分大小写的匹配。不区分大小写的匹配也可以通过嵌入式标志表达式启用(?i) 。指定此标志可能会导致轻微的性能损失。
  • Pattern.COMMENTS允许空格和模式中的注释。在此模式下，空格将被忽略，并且嵌入的注释以#直到行尾都将被忽略。也可以通过嵌入式标志表达式启用注释模式(?x) 。
  • Pattern.DOTALL启用dotall模式。在dotall模式下，表达式.匹配任何字符，包括行终止符。默认情况下，此表达式不匹配行终止符。还可以通过嵌入式标志表达式启用Dotall模式(?s) 。（s是“单行”模式的助记符，在Perl中被称为。）
  • Pattern.LITERAL启用模式的文字分析。指定此标志后，指定模式的输入字符串将被视为文字字符序列。输入序列中的元字符或转义序列将没有特殊含义。旗帜CASE_INSENSITIVE和UNICODE_CASE与该标志一起使用时，保留它们对匹配的影响。其他标志变得多余。没有用于启用文字分析的嵌入式标志字符。
  • Pattern.MULTILINE启用多行模式。在多行模式下，表达式^和$分别在行终止符或输入序列的结尾之后或之前匹配。默认情况下，这些表达式仅在整个输入序列的开头和结尾匹配。也可以通过嵌入式标志表达式启用多行模式(?m) 。
  • Pattern.UNICODE_CASE启用支持Unicode的大小写折叠。如果指定了此标志，则不区分大小写的匹配（由CASE_INSENSITIVE标志，以与Unicode标准一致的方式完成。默认情况下，不区分大小写的匹配假定只匹配US-ASCII字符集中的字符。还可以通过嵌入式标志表达式启用支持Unicode的大小写折叠(?u) 。指定此标志可能会导致性能下降。
  • Pattern.UNIX_LINES启用UNIX行模式。在此模式下，只有'\n'行终止符被识别为. ， ^和$ 。UNIX行模式也可以通过嵌入式标志表达式启用(?d) 。

在以下步骤中，我们将修改测试工具， RegexTestHarness.java创建不区分大小写的匹配模式。

首先，修改代码以调用备用版本compile ：

Pattern pattern = 
Pattern.compile(console.readLine("%nEnter your regex: "),
Pattern.CASE_INSENSITIVE);

然后编译并运行测试工具以获取以下结果：

 
Enter your regex: dog
Enter input string to search: DoGDOg
I found the text "DoG" starting at index 0 and ending at index 3.
I found the text "DOg" starting at index 3 and ending at index 6.

如您所见，字符串文字“ dog”无论大小写都匹配这两种情况。要编译带有多个标志的模式，请使用按位OR运算符“ | ”。为了清楚起见，以下代码示例对正则表达式进行了硬编码，而不是从Console ：

 
pattern = Pattern.compile("[az]$", Pattern.MULTILINE | Pattern.UNIX_LINES);

您还可以指定一个int变量：

 
final int flags = Pattern.CASE_INSENSITIVE | Pattern.UNICODE_CASE;
Pattern pattern = Pattern.compile("aa", flags);

嵌入式标志表达式

也可以使用嵌入式标志表达式来启用各种标志。嵌入式标志表达式是两个参数版本的的替代方法compile ，并在正则表达式本身中指定。以下示例使用原始测试工具， RegexTestHarness.java带有嵌入式标志表达式(?i)启用不区分大小写的匹配。

 
Enter your regex: (?i)foo
Enter input string to search: FOOfooFoOfoO
I found the text "FOO" starting at index 0 and ending at index 3.
I found the text "foo" starting at index 3 and ending at index 6.
I found the text "FoO" starting at index 6 and ending at index 9.
I found the text "foO" starting at index 9 and ending at index 12.

再一次，无论大小写，所有比赛都成功。

对应于的嵌入式标志表达式Pattern的公开字段如下表所示：

不变 等效嵌入式标志表达
Pattern.CANON_EQ 没有
Pattern.CASE_INSENSITIVE (?i)
Pattern.COMMENTS (?x)
Pattern.MULTILINE (?m)
Pattern.DOTALL (?s)
Pattern.LITERAL 没有
Pattern.UNICODE_CASE (?u)
Pattern.UNIX_LINES (?d)

使用matches(String,CharSequence)方法

的Pattern类定义了一个方便matches允许您快速检查给定输入字符串中是否存在模式的方法。与所有公共静态方法一样，您应调用matches通过其类名，例如Pattern.matches("\\d","1"); 。在此示例中，该方法返回true ，因为数字“ 1”与正则表达式匹配\d 。

使用split(String)方法

的split方法是一种用于收集位于已匹配模式两侧的文本的好工具。如下图所示SplitDemo.java ， split方法可以提取“ one two three four five “来自字符串” one:two:three:four:five “：


import java.util.regex.Pattern;
import java.util.regex.Matcher;

public class SplitDemo {

    private static final String REGEX = ":";
    private static final String INPUT =
        "one:two:three:four:five";
    
    public static void main(String[] args) {
        Pattern p = Pattern.compile(REGEX);
        String[] items = p.split(INPUT);
        for(String s : items) {
            System.out.println(s);
        }
    }
}

OUTPUT:

one
two
three
four
five

为简单起见，我们匹配了字符串文字冒号（ : ），而不是复杂的正则表达式。由于我们仍在使用Pattern和Matcher对象，则可以使用split获取位于任何正则表达式两侧的文本。这是同一个例子SplitDemo2.java ，修改为按数字拆分：


import java.util.regex.Pattern;
import java.util.regex.Matcher;

public class SplitDemo2 {

    private static final String REGEX = "\\d";
    private static final String INPUT =
        "one9two4three7four1five";

    public static void main(String[] args) {
        Pattern p = Pattern.compile(REGEX);
        String[] items = p.split(INPUT);
        for(String s : items) {
            System.out.println(s);
        }
    }
}

OUTPUT:

one
two
three
four
five

其他实用程序方法

您可能会发现以下方法也很有用：

  • public static String quote(String s)返回文字模式String对于指定的String 。此方法产生一个String可以用来创建一个Pattern那会匹配String s好像它是一个文字模式。输入序列中的元字符或转义序列将没有特殊含义。
  • public String toString()返回String此模式的表示。这是从中编译此模式的正则表达式。

模式方法中的等效项java.lang.String

正则表达式支持还存在于java.lang.String通过几种模仿行为的方法java.util.regex.Pattern 。为了方便起见，下面列出了其API的主要摘录。

  • public boolean matches(String regex) ：判断此字符串是否与给定的正则表达式匹配。这种形式的方法的调用str.matches(regex)产生与表达式完全相同的结果Pattern.matches(regex, str) 。
  • public String[] split(String regex, int limit) ：将字符串拆分为给定正则表达式的匹配项。这种形式的方法的调用str.split(regex, n)产生与表达式相同的结果Pattern.compile(regex).split(str, n)
  • public String[] split(String regex) ：将字符串拆分为给定正则表达式的匹配项。此方法的工作原理与您使用给定表达式且限制参数为零的两参数拆分方法相同。尾随空字符串不包括在结果数组中。

还有一种替换方法，可以替换一个CharSequence和另外一个：

  • public String replace(CharSequence target,CharSequence replacement) ：使用指定的文字替换序列替换该字符串中与文字目标序列匹配的每个子字符串。替换从字符串的开头到结尾进行，例如，将字符串“ aaa”中的“ aa”替换为“ b”将得到“ ba”而不是“ ab”。
