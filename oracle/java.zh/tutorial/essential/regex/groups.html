捕获组

在上一节中 ，我们看到了量词如何一次附加到一个字符，字符类或捕获组。但是直到现在，我们还没有详细讨论捕获组的概念。

捕获组是一种将多个字符视为一个单元的方法。通过将要分组的字符放在一组括号内来创建它们。例如正则表达式(dog)创建一个包含字母的单个组"d" "o"和"g" 。输入字符串中与捕获组匹配的部分将保存在内存中，以供以后通过反向引用调用（如下面在“反向引用”部分中所述 ）。

编号

如中所述Pattern API中，捕获组通过从左到右计数其开括号来编号。在表达中((A)(B(C))) ，例如，有四个这样的组：

  1. ((A)(B(C)))
  2. (A)
  3. (B(C))
  4. (C)

要找出表达式中有多少个组，请调用groupCount匹配器对象的方法。的groupCount方法返回一个int显示匹配器模式中存在的捕获组的数量。在这个例子中groupCount将返回数字4 ，显示该模式包含4个捕获组。

还有一个特殊的组，组0，它始终代表整个表达式。该组不包括在报告的总数中groupCount 。团体始于(?是不捕获文字的纯非捕获组 ，也不计入该组的总数。（稍后，您将在Pattern Class的Methods部分中看到非捕获组的示例。）

重要的是要了解组的编号方式，因为有些Matcher方法接受int指定一个特定的组号作为参数：

  • public int start(int group) ：返回给定组在上一次匹配操作期间捕获的子序列的起始索引。
  • public int end (int group) ：返回上一个匹配操作期间给定组捕获的最后一个字符的索引加一个子序列的索引。
  • public String group (int group) ：返回在上一次匹配操作期间给定组捕获的输入子序列。

反向引用

输入字符串中与捕获组匹配的部分将保存在内存中，以供以后通过backreference调用。反引用在正则表达式中指定为反斜杠（ \ ）后跟一个数字，指示要调出的组的号码。例如，表达式(\d\d)定义一个与行中的两位数字匹配的捕获组，稍后可以通过反向引用在表达式中调用该捕获组\1 。

要匹配任意2位数字，然后匹配完全相同的两位数字，您可以使用(\d\d)\1作为正则表达式：

 
Enter your regex: (\d\d)\1
Enter input string to search: 1212
I found the text "1212" starting at index 0 and ending at index 4.

如果您更改最后两位数字，则匹配将失败：

 
Enter your regex: (\d\d)\1
Enter input string to search: 1234
No match found.

对于嵌套捕获组，反引用的工作方式完全相同：指定一个反斜杠，后跟要调用的组的编号。
