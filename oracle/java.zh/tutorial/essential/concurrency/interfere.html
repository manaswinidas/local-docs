线程干扰

考虑一个简单的类，叫做Counter


class Counter {
    private int c = 0;

    public void increment() {
        c++;
    }

    public void decrement() {
        c--;
    }

    public int value() {
        return c;
    }

}

Counter旨在使每次调用increment将加1 c ，并且每次调用decrement将减去1 c 。但是，如果Counter从多个线程引用了对象，线程之间的干扰可能会阻止这种情况按预期发生。

当在不同线程中运行但作用于相同数据的两个操作交错时，就会发生干扰。这意味着这两个操作由多个步骤组成，并且步骤顺序重叠。

在以下实例上执行操作似乎不太可能Counter交织，因为两个操作都在c是单个简单的语句。但是，即使是简单的语句也可以由虚拟机转换为多个步骤。我们将不检查虚拟机采取的具体步骤-足以知道单个表达式c++可以分解为三个步骤：

  1. 检索的当前值c 。
  2. 将检索到的值增加1。
  3. 将增加的值存储回c 。

表达方式c--可以用相同的方式分解，除了第二步递减而不是递增。

假设线程A调用increment大约在线程B调用的同时decrement 。如果初始值c是0 ，他们的交错动作可能会遵循以下顺序：

  1. 线程A：检索c。
  2. 线程B：检索c。
  3. 线程A：增加检索值；结果是1。
  4. 线程B：递减检索值；结果是-1。
  5. 线程A：将结果存储在c中； c现在是1。
  6. 线程B：将结果存储在c中； c现在为-1。

线程A的结果丢失，被线程B覆盖。这种特殊的交织只是一种可能。在不同的情况下，可能是线程B的结果丢失了，或者根本没有错误。由于它们是不可预测的，因此可能很难检测和修复线程干扰错误。
