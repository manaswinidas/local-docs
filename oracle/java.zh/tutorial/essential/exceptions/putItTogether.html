全部放在一起

前几节介绍了如何构建try ， catch和finally的代码块writeList中的方法ListOfNumbers类。现在，让我们遍历代码并研究可能发生的情况。

当所有组件放在一起时， writeList方法如下所示。

public void writeList() {
    PrintWriter out = null;

    try {
        System.out.println("Entering" + " try statement");

        out = new PrintWriter(new FileWriter("OutFile.txt"));
        for (int i = 0; i < SIZE; i++) {
            out.println("Value at: " + i + " = " + list.get(i));
        }
    } catch (IndexOutOfBoundsException e) {
        System.err.println("Caught IndexOutOfBoundsException: "
                           +  e.getMessage());
                                 
    } catch (IOException e) {
        System.err.println("Caught IOException: " +  e.getMessage());
                                 
    } finally {
        if (out != null) {
            System.out.println("Closing PrintWriter");
            out.close();
        } 
        else {
            System.out.println("PrintWriter not open");
        }
    }
}

如前所述，此方法的try街区有三种不同的出口可能性；这是其中两个。

  1. 中的代码try语句失败并引发异常。这可能是IOException由new FileWriter陈述或IndexOutOfBoundsException由错误的索引值引起for环。
  2. 一切都成功了， try语句正常退出。

让我们来看看writeList方法在这两种退出可能性之间。

方案1：发生异常

创建一个FileWriter可能由于多种原因而失败。例如， FileWriter抛出IOException如果程序无法创建或写入指示的文件。

什么时候FileWriter抛出IOException ，运行时系统立即停止执行try块;正在执行的方法调用未完成。然后，运行时系统开始在方法调用堆栈的顶部搜索适当的异常处理程序。在此示例中，当IOException发生， FileWriter构造函数位于调用堆栈的顶部。然而FileWriter构造函数没有适当的异常处理程序，因此运行时系统将检查下一个方法- writeList method —在方法调用堆栈中。的writeList方法具有两个异常处理程序：一个用于IOException还有一个IndexOutOfBoundsException 。

运行系统检查writeList的处理程序，以它们在try声明。第一个异常处理程序的参数是IndexOutOfBoundsException 。这与引发的异常类型不匹配，因此运行时系统将检查下一个异常处理程序- IOException 。这与引发的异常类型匹配，因此运行时系统结束了对适当异常处理程序的搜索。现在，运行时已经找到了合适的处理程序，其中的代码catch块被执行。

异常处理程序执行后，运行时系统将控制权传递给finally块。中的代码finally块执行，而不管其上方捕获的异常如何。在这种情况下， FileWriter从来没有打开过，不需要关闭。之后finally块完成执行后，程序将在finally块。

这是来自ListOfNumbers当IOException被抛出。

Entering try statement
Caught IOException: OutFile.txt
PrintWriter not open

以下清单中的粗体代码显示了在这种情况下执行的语句：

public void writeList() {
   PrintWriter out = null;

    try {

        System.err.println("Caught IndexOutOfBoundsException: "
                           + e.getMessage());
                                 
    } catch (IOException e) {
        System.err.println("Caught IOException: " + e.getMessage());
    } finally {
        if (out != null) {
            System.out.println("Closing PrintWriter");
            out.close();
        }

        }
    }

方案2：try块正常退出

在这种情况下，所有语句在try阻止成功执行并且不引发异常。执行落到最后try块，然后运行时系统将控制权传递给finally块。因为一切都成功了， PrintWriter当控制到达时打开finally块，关闭PrintWriter 。同样， finally块完成执行后，程序将在finally块。

这是来自ListOfNumbers没有异常时抛出的程序。

Entering try statement
Closing PrintWriter

以下示例中的粗体代码显示了在这种情况下执行的语句。

public void writeList() {
    PrintWriter out = null;
    try {
        System.out.println("Entering try statement");
        out = new PrintWriter(new FileWriter("OutFile.txt"));
        for (int i = 0; i < SIZE; i++)
            out.println("Value at: " + i + " = " + list.get(i));
                  
    } catch (IndexOutOfBoundsException e) {
        System.err.println("Caught IndexOutOfBoundsException: "
                           + e.getMessage());

    } catch (IOException e) {
        System.err.println("Caught IOException: " + e.getMessage());
                                 
    } finally {
        if (out != null) {
            System.out.println("Closing PrintWriter");
            out.close();
        } 
        else {
            System.out.println("PrintWriter not open");
        }
    }
}
