如何引发异常

在捕获异常之前，某些地方的某些代码必须抛出一个。任何代码都可能引发异常：您的代码，其他人编写的程序包中的代码，例如Java平台附带的程序包或Java运行时环境。不管引发异常的原因是什么，它总是与throw声明。

您可能已经注意到，Java平台提供了许多异常类。所有的类都是Throwable类，并且所有这些都允许程序区分在程序执行期间可能发生的各种类型的异常。

您还可以创建自己的异常类，以表示您编写的类中可能发生的问题。实际上，如果您是程序包开发人员，则可能必须创建自己的异常类集，以允许用户将程序包中可能发生的错误与Java平台或其他程序包中发生的错误区分开。

您还可以创建链接的异常。有关更多信息，请参见链接异常部分。

抛出声明

所有方法都使用throw语句引发异常。的throw语句需要一个参数：一个可抛出对象。可抛对象是以下对象的任何子类的实例Throwable类。这是一个例子throw声明。

throw someThrowableObject;

让我们看一下throw上下文中的语句。下列pop方法是从实现通用堆栈对象的类中获取的。该方法从堆栈中删除顶部元素，并返回对象。

public Object pop() {
    Object obj;

    if (size == 0) {
        throw new EmptyStackException();
    }

    obj = objectAt(size - 1);
    setObjectAt(size - 1, null);
    size--;
    return obj;
}

的pop方法检查是否有任何元素在堆栈上。如果堆栈为空（其大小等于0 ）， pop实例化一个新的EmptyStackException对象（ java.util ）并将其抛出。本章的“ 创建异常类”部分介绍了如何创建自己的异常类。目前，您只需要记住，您只能抛出从java.lang.Throwable类。

请注意， pop方法不包含throws条款。 EmptyStackException不是已检查的异常，因此pop不需要声明可能会发生。

错误类别

当Java虚拟机中发生动态链接故障或其他硬故障时，虚拟机将抛出Error 。简单的程序通常不会捕获或抛出Error s。

例外类别

大多数程序会抛出并捕获从Exception类。一个Exception表示发生了问题，但这不是严重的系统问题。您编写的大多数程序都会抛出并捕获Exception相对于Error s。

Java平台定义了Exception类。这些后代指示可能发生的各种异常。例如， IllegalAccessException表示找不到特定方法，并且NegativeArraySizeException表示程序试图创建一个具有负大小的数组。

一Exception子类， RuntimeException保留用于指示错误使用API的异常。运行时异常的一个示例是NullPointerException ，当方法尝试通过来访问对象的成员时发生null参考。未检查的异常—争议部分讨论了为什么大多数应用程序不应引发运行时异常或子类的原因RuntimeException 。
