格式化

实现格式化的流对象是以下任一实例PrintWriter ，字符流类或PrintStream ，字节流类。

注意：唯一PrintStream您可能需要的对象是System.out和System.err 。（有关这些对象的更多信息，请参见命令行中的I / O。 ）当您需要创建格式化的输出流时，请实例化PrintWriter ，不是PrintStream 。

像所有字节流和字符流对象一样， PrintStream和PrintWriter实施一套标准的write简单字节和字符输出的方法。另外，两者PrintStream和PrintWriter实现将内部数据转换为格式化输出的相同方法集。提供了两种格式设置：

  • print和println以标准方式格式化单个值。
  • format根据格式字符串格式化几乎所有数量的值，并提供许多用于精确格式化的选项。

的print和println方法

调用中print要么println使用适当的值转换值后，输出单个值toString方法。我们可以在Root例：

public class Root {
    public static void main(String[] args) {
        int i = 2;
        double r = Math.sqrt(i);
        
        System.out.print("The square root of ");
        System.out.print(i);
        System.out.print(" is ");
        System.out.print(r);
        System.out.println(".");

        i = 5;
        r = Math.sqrt(i);
        System.out.println("The square root of " + i + " is " + r + ".");
    }
}

这是输出Root ：

The square root of 2 is 1.4142135623730951.
The square root of 5 is 2.23606797749979.

的i和r变量格式化两次：第一次使用重载形式的代码print ，这是Java编译器自动生成的第二次转换代码， toString 。您可以通过这种方式设置任何值的格式，但是您对结果没有太多控制权。

的format方法

的format方法根据格式字符串格式化多个参数。格式字符串由嵌入格式说明符的静态文本组成；除格式说明符外，格式字符串的输出保持不变。

格式字符串支持许多功能。在本教程中，我们将仅介绍一些基础知识。有关完整说明，请参见format string syntax在API规范中。

的Root2示例使用单个格式设置两个值format调用：

public class Root2 {
    public static void main(String[] args) {
        int i = 2;
        double r = Math.sqrt(i);
        
        System.out.format("The square root of %d is %f.%n", i, r);
    }
}

这是输出：

The square root of 2 is 1.414214.

与本示例中使用的三个类似，所有格式说明符均以%并以1或2个字符的转换结尾，该转换指定要生成的格式化输出的类型。这里使用的三种转换是：

  • d将整数值格式化为十进制值。
  • f将浮点值格式化为十进制值。
  • n输出平台特定的行终止符。

以下是一些其他转换：

  • x将整数格式化为十六进制值。
  • s将任何值格式化为字符串。
  • tB将整数格式化为特定于语言环境的月份名称。

还有许多其他转换。

注意： 

除了%%和%n ，所有格式说明符必须与参数匹配。如果没有，则抛出异常。

在Java编程语言中， \n转义始终生成换行符（ \u000A ）。不要使用\n除非您特别需要换行符。要获得适用于本地平台的正确的行分隔符，请使用%n 。

除转换外，格式说明符还可以包含几个其他元素，以进一步自定义格式输出。这是一个例子Format ，它使用所有可能的元素。

public class Format {
    public static void main(String[] args) {
        System.out.format("%f, %1$+020.10f %n", Math.PI);
    }
}

这是输出：

3.141593, +00000003.1415926536

附加元素都是可选的。下图显示了较长的说明符如何分解为元素。

格式说明符的元素。

元素必须按照显示的顺序出现。从右侧开始，可选元素为：

  • 精密度 。对于浮点值，这是格式化值的数学精度。对于s和其他常规转换，这是格式化值的最大宽度；如果需要，该值将被右截断。
  • 宽度 。格式化值的最小宽度；如果需要，将填充该值。默认情况下，该值用空格左填充。
  • 标志指定其他格式选项。在里面Format例如， +标志指定数字应始终以符号格式化，并且0标志指定0是填充字符。其他标志包括- （右侧垫）和, （格式编号，带有特定于语言环境的千位分隔符）。请注意，某些标志不能与某些其他标志或某些转换一起使用。
  • 参数索引使您可以显式匹配指定的参数。您还可以指定<与前面的说明符匹配相同的参数。因此，该示例可能会说：System.out.format("%f, %<+020.10f %n", Math.PI);
