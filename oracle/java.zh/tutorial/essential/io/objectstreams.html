对象流

正如数据流支持原始数据类型的I / O一样，对象流也支持对象的I / O。大多数（但不是全部）标准类支持其对象的序列化。那些确实实现了标记接口的Serializable 。

对象流类是ObjectInputStream和ObjectOutputStream 。这些类实现ObjectInput和ObjectOutput ，它们是的子接口DataInput和DataOutput 。这意味着数据流中涵盖的所有原始数据I / O方法也都在对象流中实现。因此，对象流可以包含原始值和对象值的混合。的ObjectStreams例子说明了这一点。 ObjectStreams创建与相同的应用程序DataStreams ，进行了一些更改。首先，现在的价格BigDecimal对象，以更好地表示分数值。第二个Calendar对象被写入数据文件，指示发票日期。

如果readObject()没有返回预期的对象类型，尝试将其强制转换为正确的类型可能会抛出ClassNotFoundException 。在这个简单的示例中，这不可能发生，因此我们不会尝试捕获异常。相反，我们通过添加以下内容来通知编译器我们已意识到该问题ClassNotFoundException到main方法throws条款。

复杂对象的输出和输入

的writeObject和readObject方法很容易使用，但是它们包含一些非常复杂的对象管理逻辑。对于像Calendar这样的仅封装原始值的类，这并不重要。但是许多对象包含对其他对象的引用。如果readObject要从流中重构对象，它必须能够重构原始对象所引用的所有对象。这些其他对象可能有自己的引用，依此类推。在这个情况下， writeObject遍历整个对象引用网络，并将该网络中的所有对象写入流。因此，单次调用writeObject可能导致将大量对象写入流中。

如下图所示，其中writeObject被调用以编写名为的单个对象。该对象包含对对象b和c的引用，而b包含对d和e的引用。调用中writeobject(a)写不只是一个 ，但所有的对象必须重建一个 ，所以在这个网络中的其他四个对象也被写入。当a被读回readObject ，其他四个对象也将被读回，并且所有原始对象引用都将保留。

多个引用对象的I / O

您可能想知道如果同一流上的两个对象都包含对单个对象的引用会发生什么。回读它们是否都引用一个对象？答案是“是”。一个流只能包含一个对象的一个副本，尽管它可以包含对该对象的任意数量的引用。因此，如果您将一个对象显式地写入流两次，则实际上仅写入了两次引用。例如，如果以下代码编写了一个对象ob两次到流：

Object ob = new Object();
out.writeObject(ob);
out.writeObject(ob);

每writeObject必须由一个readObject ，因此读回流的代码将如下所示：

Object ob1 = in.readObject();
Object ob2 = in.readObject();

这导致两个变量， ob1和ob2 ，它们是对单个对象的引用。

但是，如果将单个对象写入两个不同的流，则该对象将被有效地复制-读回两个流的单个程序将看到两个不同的对象。
