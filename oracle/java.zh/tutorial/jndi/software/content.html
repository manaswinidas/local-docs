LDAP设定

以下是构建访问LDAP目录服务器的Java应用程序所涉及的步骤。

  1. 安装Java平台软件。
  2. 获得前面讨论的Directory Server软件。
  3. 使用所需的架构配置Directory Server。为了使用本教程中的示例，需要在服务器上配置特殊的架构 。
  4. 用所需的内容填充目录服务器。为了使用本教程中的示例，需要在服务器上填充特殊内容 。
  5. 编写一个JNDI应用程序以访问Directory，对Directory Server进行编译和运行以获取所需的结果。在下一课中将介绍JNDI示例。

前一部分介绍了前两个步骤。本课的其余部分讨论步骤3和步骤4的一部分。在下一课中将介绍涉及编写JNDI应用程序的第五步，该课程显示了如何编写JNDI应用程序以对目录执行各种操作。

设置目录或指示程序与现有目录进行通信后，您希望在其中找到什么样的信息？

该目录可以视为由名称到对象的绑定组成。也就是说，目录中的每个对象都有一个对应的名称。您可以通过查找对象名称来检索目录中的对象。

属性也存储在目录中。目录中的对象除具有名称外，还具有一组可选的属性。您可以要求目录提供对象的属性，也可以要求目录搜索具有某些属性的对象。

步骤3：目录架构

模式指定目录可能包含的对象的类型。本教程使用条目填充目录，其中一些条目需要特殊的架构定义。要容纳这些条目，必须首先关闭服务器中的架构检查，或者将本教程随附的架构文件添加到服务器。这两个任务通常由目录服务器的管理员执行。

本教程带有两个必须安装的架构文件：

  • Schema for Java objects
  • Schema for CORBA objects

这些文件的格式是一种正式描述，可能无法直接复制并粘贴到服务器配置文件中。具体而言，根据RFC 2252描述属性语法。

不同的目录服务器具有不同的配置其架构的方式。本教程包括一些用于在目录服务器上安装Java和CORBA模式的工具，这些工具允许通过LDAP修改它们的模式。以下是工具可执行的任务列表。

  1. Create Java Schema
  2. Create CORBA Schema

请遵循随附的说明README file运行这些程序。

注意：Windows Active Directory。 Active Directory通过使用内部格式来管理其架构。要更新架构，可以按照Active Directory的说明使用Active Directory管理控制台管理单元ADSIEdit或CreateJavaSchema实用程序。

步骤4：为本教程提供目录内容

在此跟踪的示例中，显示的结果反映了如何使用配置文件设置LDAP目录（ tutorial.ldif ）。如果使用的是现有服务器或设置不同的服务器，则可能会看到不同的结果。加载配置文件之前（ tutorial.ldif ）进入目录服务器，您必须按照说明更新服务器的架构，或者可以使用ldapadd或ldapmodify命令（如果在UNIX系统上可用）。

例如，使用ldapmodify可以做到（通过为主机名，管理员DN（-D选项）和密码插入适当的值）：

ldapmodify -a -c -v -h hostname -p 389\
        -D "cn=Administrator, cn=users, dc=xxx, dc=xxx"\
        -w passwd -f tutorial.ldif

安装说明：访问控制。 不同的目录服务器对访问控制的处理方式不同。本教程中的一些示例对目录进行更新。另外，您在本教程中安装的名称空间部分可能具有读取访问限制。因此，您需要采取特定于服务器的操作，以使目录可读和/或可更新，以使这些示例正常工作。对于Oracle目录服务器添加的建议的ACI条目sunds.aci.ldif文件添加到dn：o = JNDITutorial条目，以使整个目录可读和可更新。或者，您可以更改示例，以便它们对目录进行身份验证。安全课程中详细介绍了如何执行此操作。

安装说明：命名空间设置。 中的条目tutorial.ldif文件为根命名上下文使用专有名称（DN）“ o = JNDITutorial”。如果尚未将目录服务器配置为具有“ o = JNDITutorial”作为根命名上下文，则尝试导入tutorial.ldif将会失败。解决此问题的最简单方法是将现有根命名上下文的DN添加到tutorial.ldif文件中的每个“ dn：”行。例如，如果您的服务器已经具有根命名上下文“ dc = imc，dc = org”，则应更改该行

dn: o=JNDITutorial

至

dn: o=JNDITutorial, dc=imc, dc=org

对文件中以“ dn：”开头的每一行进行此更改。然后，在本教程的所有示例中，无论在何处使用“ o = JNDITutorial”，都应改为使用“ o = JNDITutorial，dc = imc，dc = org”。

安装说明：文件格式。 根据所使用的操作系统平台，可能需要编辑tutorial.ldif，以便它包含该平台的正确换行符。例如，如果您发现tutorial.ldif包含Windows样式的换行符（CRLF），并且正在将该文件导入到在UNIX平台上运行的目录服务器中，则需要编辑该文件并将CRLF替换为LF。此问题的症状是目录服务器拒绝tutorial.ldif中的所有条目。

安装说明：Windows Active Directory。

  1. 根命名上下文不会是“ o = janditutorial”。格式为“ dc = x，dc = y，dc = z”。您需要遵循前面的“ 命名空间设置”说明。
  2. 使用Active Directory管理控制台管理单元ADSIEdit将“ inetOrgPerson”和“ groupOfUniqueNames”对象类的对象类以及相关属性添加到Active Directory架构中。 “ groupOfUniqueNames”在RFC 2256中定义，“ inetOrgPerson”在RFC 2798中定义 。
  3. 默认情况下，Active Directory中不允许使用本教程使用的某些层次关系。要启用这些关系，请使用Active Directory管理控制台管理单元ADSIEdit来添加它们。 
    objectclass: organizationalUnit
possible superiors: domainDNS
                    inetOrgPerson
                    organizaton
                    organizationalPerson
                    organizationalUnit
                    person
                    top

objectclass: groupOfUniqueNames
possible superiors: top

objectclass: inetOrgPerson
possible superiors: container
                    organizationalPerson
                    person
                    top
  4. 从tutorial.ldif中的Mark Twain条目中删除两个“ sn”属性之一。Active Directory将“ sn”定义为单值属性，与RFC 2256相反。
  5. 使用ldifde命令行实用程序来加载修改后的tutorial.ldif文件。 
    # ldifde -i -v -k -f tutorial.ldif
  6. 大多数示例都假定已将目录设置为允许未经身份验证的读取和更新访问。您的Active Directory设置可能不允许您执行此操作。请参阅访问控制安装说明。
  7. 读取条目有时会产生比教程中显示的更多的属性，因为Active Directory通常会返回一些内部属性。
  8. 条目的创建可能需要指定其他特定于Active Directory的属性或使用其他对象类。
