存储和读取对象

应用程序和服务可以通过不同的方式使用目录来存储和定位对象：

  • 存储对象本身（的副本）。
  • 存储对对象的引用。
  • 存储描述对象的属性。

一般而言，Java对象的序列化形式以可用于联系对象的地址信息的紧凑表示形式包含对象的状态和对象的引用。“ 查找对象”课程中提供了一些示例。对象的属性是用于描述对象的属性。属性可能包括地址和/或状态信息。

这三种使用方式中的哪一种取决于正在构建的应用程序/系统，以及与共享目录中存储的对象的其他应用程序和系统进行交互的方式。另一个因素是服务提供商和基础目录服务提供的支持。

以编程方式，将对象存储在目录中时，所有应用程序都使用以下方法之一：

  • Context.bind（）
  • DirContext.bind（）
  • Context.rebind（）
  • DirContext.rebind（）

应用程序将要存储的对象传递给这些方法之一。然后，根据服务提供者支持的对象类型，将对象转换为基础目录服务可接受的表示形式。

本课程显示如何在对象存储后将可序列化的对象存储在目录中，而无论实际上存储了哪种类型的信息，您都可以简单地使用lookup（）从目录中获取对象的副本。

您不仅可以使用lookup（）来获取对象，还可以在列出上下文时以及在搜索上下文或其子树时获取对象。在所有这些情况下，都可能涉及对象工厂 。对象工厂在JNDI教程中进行了详细讨论。

要存储以下对象类型，请参阅JNDI教程：

  • 引用对象和JNDI 引用
    “ 添加，替换或删除绑定”课程中的bind（）示例使用可引用对象。
  • 具有属性的对象（ DirContext ）
  • RMI（Java远程方法调用）对象（包括使用IIOP的对象）
  • CORBA对象

继续之前：要成功运行这些示例，必须关闭服务器中的架构检查或添加the Java schema本教程随附的服务器。此任务通常由目录服务器的管理员执行。有关更多信息，请参见软件设置课程。

Windows Active Directory： Context.rebind（）和DirContext.rebind（）不适用于Active Directory，因为这些方法通过读取要更新的条目的属性，删除条目，然后添加包含已修改条目的新条目来起作用属性。Active Directory返回一些用户无法设置的属性，从而导致最后的添加步骤失败。此问题的解决方法是使用DirContext.getAttributes（）获取并保存要保留的属性。然后，删除该条目，并使用DirContext.bind（）将其与保存的属性（以及要添加的任何其他属性）一起添加回去。
