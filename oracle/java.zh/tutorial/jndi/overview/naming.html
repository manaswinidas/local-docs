命名包

javax.naming包包含用于访问命名服务的类和接口。

语境

javax.naming包定义了一个Context接口，它是用于查找，绑定/解除绑定，重命名对象以及创建和销毁子上下文的核心接口。

抬头
最常用的操作是lookup（） 。您提供lookup（）要查找的对象的名称，它返回绑定到该名称的对象。
绑定
listBindings（）返回名称到对象绑定的枚举。绑定是一个元组，其中包含绑定对象的名称，对象的类的名称以及对象本身。
清单
list（）与listBindings（）相似，不同之处在于它返回名称的枚举，其中包含对象的名称和对象的类的名称。 list（）对于想要发现有关上下文中绑定对象的信息但不需要所有实际对象的应用程序（如浏览器）很有用。尽管listBindings（）提供了所有相同的信息，但它可能是更昂贵的操作。
名称
名称是代表通用名称的接口-零个或多个组件的有序序列。命名系统使用此接口来定义遵循其命名惯例的名称，如“ 命名和目录概念”课程中所述。
参考文献
对象以不同的方式存储在命名和目录服务中。引用可能是对象的非常紧凑的表示形式。

JNDI定义了引用类来表示引用。参考包含有关如何构造对象副本的信息。JNDI将尝试将从目录中查找到的引用转换为它们所代表的Java对象，以便JNDI客户端错觉目录中存储的是Java对象。

初始环境

在JNDI中，所有命名和目录操作都是相对于上下文执行的。没有绝对的根源。因此，JNDI定义了InitialContext ，它为命名和目录操作提供了起点。一旦有了初始上下文，就可以使用它来查找其他上下文和对象。

例外情况

JNDI定义了用于在执行命名和目录操作过程中引发的异常的类层次结构。该类层次结构的根是NamingException 。对处理特定异常感兴趣的程序可以捕获异常的相应子类。否则，它们应该捕获NamingException 。
