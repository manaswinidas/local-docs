搜索结果

在DirContext界面中使用搜索方法时，将返回NamingEnumeration 。NamingEnumeration中的每个项目都是一个SearchResult ，其中包含以下信息：

  • 名称
  • 宾语
  • 班级名称
  • 属性

名称

每个SearchResult包含满足搜索过滤条件的LDAP条目的名称。您可以使用getName（）获得条目的名称。此方法返回相对于目标上下文的LDAP条目的组合名称 。目标上下文是name参数解析到的上下文。用LDAP的话来说，目标上下文是搜索的基础对象 。这是一个例子。

NamingEnumeration answer = ctx.search("ou=NewHires", 
    "(&(mySpecialKey={0}) (cn=*{1}))",  // Filter expression
    new Object[]{key, name},                // Filter arguments
    null);                                  // Default search controls

在此示例中，目标上下文是由“ ou = NewHires”命名的上下文。答案中SearchResult中的名称是相对于“ ou = NewHires”的 。例如，如果getName（）返回“ cn = J。Duke” ，则其相对于ctx的名称将为“ cn = J。Duke，ou = NewHires” 。

如果使用SearchControls执行搜索。SUBTREE_SCOPE或SearchControls。OBJECT_SCOPE和目标上下文本身满足搜索过滤器的要求，因此返回的名称将为“”（空名称），因为这是相对于目标上下文的名称。

这还不是全部。如果搜索涉及引用（请参见JNDI教程 ）或取消引用别名（请参见JNDI教程 ），则相应的SearchResult将具有与目标上下文无关的名称。相反，它们将是直接引用条目的URL。要确定getName（）返回的名称是相对名称还是绝对名称，请使用isRelative（） 。如果此方法返回true ，则名称相对于目标上下文；如果返回false ，则名称是一个URL。

如果名称是URL，并且您需要使用该URL，则可以将其传递给理解URL的初始上下文（请参见JNDI教程 ）。

如果需要获取条目的完整DN，则可以使用NameClassPair.getNameInNamespace（） 。

宾语

如果执行搜索以请求返回条目的对象（使用true调用SearchControls.setReturningObjFlag（） ），则SearchResult将包含一个代表条目的对象。要检索此对象，请调用getObject（） 。如果是java.io。先前将Serializable ， Referenceable或Reference对象绑定到该LDAP名称，然后使用条目中的属性来重建该对象（请参见JNDI教程中的示例 ）。否则，该条目的属性将用于创建代表LDAP条目的DirContext实例。在这两种情况下，LDAP提供程序都将在对象上调用DirectoryManager.getObjectInstance（）并返回结果。

班级名称

如果执行搜索请求返回条目的对象，则从返回的对象派生类名。如果搜索请求的属性包括对LDAP条目的“ javaClassName”属性的检索，则类名就是该属性的值。否则，类名称为“ javax.naming.directory”。DirContext” 。类名称是从getClassName（）获得的。

属性

执行搜索时，可以通过向search（）方法之一提供参数，或者使用SearchControls.setReturningAttributes（）设置搜索控件来选择返回属性。如果未明确指定任何属性，则将返回所有LDAP条目的属性。要指定不返回任何属性，必须传递一个空数组（ new String [0] ）。

要检索LDAP条目的属性，请在SearchResult上调用getAttributes（） 。

响应控制

有关如何检索搜索结果的响应控件的详细信息，请参见“ JNDI教程”的“ 控件和扩展”课程。
