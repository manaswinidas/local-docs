创建

有几种创建连接的方式。最常见的方法是从创建初始上下文开始。使用LDAP服务提供程序创建InitialContext ， InitialDirContext或InitialLdapContext时 ，将立即与Context中命名的目标LDAP服务器建立连接。PROVIDER_URL属性。每次创建初始上下文时，都会创建一个新的LDAP连接。有关如何更改此行为的信息，请参见“ 池化”部分。

如果属性值包含多个URL，则依次尝试每个URL，直到使用一个URL创建成功的连接为止。然后将属性值更新为成功的URL。有关如何使用URL列表创建初始上下文的示例，请参见JNDI教程 。

创建连接的其他三种直接方法。

  1. 通过将URL作为名称参数传递给初始上下文。将LDAP或LDAPS URL作为名称参数传递给初始上下文时，URL中的信息用于创建与LDAP服务器的新连接，而不管初始上下文实例本身是否与LDAP服务器建立连接。实际上，初始上下文可能未连接到任何服务器。有关如何将URL用作名称的更多信息，请参见JNDI教程 。
  2. 创建连接的另一种方法是使用Reference 。将包含LDAP或LDAPS URL的引用传递给NamingManager.getObjectInstance（）或DirectoryManager.getObjectInstance（）时 ，将使用URL中指定的信息来创建新连接。
  3. 最后，当手动或自动执行引荐时，引荐中的信息将用于创建新的连接。有关引用的信息，请参见JNDI教程 。

共享连接

从一个Context实例派生的Context实例和NamingEnumeration共享相同的连接，直到对Context实例之一进行更改使共享不再可能。例如，如果您从初始上下文调用Context.lookup（） ， Context.listBindings（）或DirContext.search（）并获取其他Context实例，则所有这些Context实例将共享同一连接。

这是an example 。

// Create initial context
DirContext ctx = new InitialDirContext(env);

// Get a copy of the same context
Context ctx2 = (Context)ctx.lookup("");

// Get a child context
Context ctx3 = (Context) ctx.lookup("ou=NewHires");

在此示例中， ctx ， ctx2和ctx3将共享相同的连接。

无论Context实例如何存在，共享都是完成的。例如，通过引用获得的Context实例将共享与引用相同的连接。

更改与连接相关的Context实例的环境属性（例如用户的主体名称或凭据）时，在其上进行这些更改的Context实例将获得其自己的连接（如果共享连接）。这是从这个背景下衍生的实例在未来上下文实例将分享这个新的连接。先前共享旧连接的上下文实例不受影响（也就是说，它们继续使用旧连接）。

这是an example使用两个连接。

// Create initial context (first connection)
DirContext ctx = new InitialDirContext(env);

// Get a copy of the same context
DirContext ctx2 = (DirContext)ctx.lookup("");

// Change authentication properties in ctx2
ctx2.addToEnvironment(Context.SECURITY_PRINCIPAL, 
    "cn=C. User, ou=NewHires, o=JNDITutorial");
ctx2.addToEnvironment(Context.SECURITY_CREDENTIALS, "mysecret");

// Method on ctx2 will use new connection
System.out.println(ctx2.getAttributes("ou=NewHires"));

ctx2最初与ctx共享相同的连接。但是，更改其主体和密码属性后，将无法再使用ctx的连接。LDAP提供程序将自动为ctx2创建一个新连接。

同样，如果使用LdapContext.reconnect（）更改Context实例的连接控件，则在共享共享的情况下， Context实例将获得其自己的连接。

如果没有共享Context实例的连接（即，没有从其派生Context ），则对其环境的更改或连接控件将不会导致创建新的连接。取而代之的是，与该连接有关的任何更改都将应用于现有连接。

创建超时

并非所有连接创建都成功。如果LDAP提供程序无法在一定的超时时间内建立连接，它将中止连接尝试。默认情况下，此超时时间是网络（TCP）超时值，大约为几分钟。要更改超时时间，请使用“ com.sun.jndi.ldap.connect.timeout”环境属性。此属性的值是整数的字符串表示形式，该整数表示连接超时（以毫秒为单位）。

这是an example 。

// Set up environment for creating initial context
Hashtable env = new Hashtable(11);
env.put(Context.INITIAL_CONTEXT_FACTORY, 
    "com.sun.jndi.ldap.LdapCtxFactory");
env.put(Context.PROVIDER_URL, "ldap://localhost:389/o=JNDITutorial");

// Specify timeout to be 5 seconds
env.put("com.sun.jndi.ldap.connect.timeout", "5000");

// Create initial context
DirContext ctx = new InitialDirContext(env);

// do something useful with ctx

在此示例中，如果无法在5秒钟内创建连接，则将引发异常。

如果上下文。PROVIDER_URL属性包含多个URL，提供程序将为每个URL使用超时。例如，如果有3个URL，并且将超时指定为5秒，则提供程序总共将等待最多15秒。

有关此属性如何影响连接池的信息，请参见“ 连接池”部分。
