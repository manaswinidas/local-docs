编写数据报客户端和服务器

本节中的示例包括两个应用程序：客户端和服务器。服务器通过数据报套接字连续接收数据报包。服务器接收的每个数据报包都指示客户端请求报价。当服务器接收到数据报时，它会通过将包含单行“当下报价”的数据报包发送回客户端来进行回复。

此示例中的客户端应用程序非常简单。它向服务器发送一个数据报包，指示客户端希望接收该时刻的报价。然后，客户端等待服务器发送数据报文包作为响应。

有两个类实现服务器应用程序： QuoteServer和QuoteServerThread 。单个类实现客户端应用程序： QuoteClient 。

让我们研究这些类，从包含以下内容的类开始main服务器应用程序的方法。 使用服务器端应用程序包含的applet版本QuoteClient类。

QuoteServer类

的QuoteServer类（在此处完整显示）包含一个方法： main报价服务器应用程序的方法。的main方法只是创建一个新QuoteServerThread对象并启动它：

import java.io.*;

public class QuoteServer {
    public static void main(String[] args) throws IOException {
        new QuoteServerThread().start();
    }
}

的QuoteServerThread类实现报价服务器的主要逻辑。

的QuoteServerThread类

创建后， QuoteServerThread创建一个DatagramSocket在端口4445（任意选择）上。这是DatagramSocket服务器通过它与所有客户端进行通信。

public QuoteServerThread() throws IOException {
    this("QuoteServer");
}

public QuoteServerThread(String name) throws IOException {
    super(name);
    socket = new DatagramSocket(4445);

    try {
        in = new BufferedReader(new FileReader("one-liners.txt"));
    }   
    catch (FileNotFoundException e){
        System.err.println("Couldn't open quote file.  Serving time instead.");
    }
}

请记住，某些端口专用于知名服务，您不能使用它们。如果指定正在使用的端口，则创建DatagramSocket将失败。

构造函数还会打开一个BufferedReader在一个名为one-liners.txt 包含引号列表。文件中的每个引号本身都是一行。

现在，对于有趣的部分QuoteServerThread ： 它的run方法。的run方法覆盖run在里面Thread类并提供线程的实现。有关线程的信息，请参见定义和启动线程 。

的run方法包含一个while只要文件中有更多的引号，循环就会继续。在循环的每次迭代过程中，线程等待一个DatagramPacket到达DatagramSocket 。该数据包指示来自客户端的请求。应客户的要求， QuoteServerThread从文件中获取报价，并将其放在DatagramPacket并通过DatagramSocket给要求它的客户。

让我们首先看一下接收来自客户端的请求的部分：

byte[] buf = new byte[256];
DatagramPacket packet = new DatagramPacket(buf, buf.length);
socket.receive(packet);

第一条语句创建一个字节数组，然后用于创建一个字节数组。 DatagramPacket 。的DatagramPacket由于用于创建套接字的构造函数，它将用于从套接字接收数据报。此构造函数仅需要两个参数：一个包含客户端特定数据的字节数组和该字节数组的长度。当构造一个DatagramPacket发送过来DatagramSocket ，还必须提供数据包目标的Internet地址和端口号。稍后我们讨论服务器如何响应客户端请求时，您将看到此信息。

上一个代码片段中的最后一条语句从套接字接收数据报（从客户端接收的信息被复制到数据包中）。接收方法永远等待，直到接收到数据包为止。如果未接收到任何数据包，则服务器不会继续执行任何操作，而是等待。

现在假设服务器已从客户端收到报价请求。现在服务器必须响应。run方法中的这段代码构造了响应：

String dString = null;
if (in == null)
    dString = new Date().toString();
else
    dString = getNextQuote();
buf = dString.getBytes();

如果由于某种原因未打开报价文件，则in等于null。在这种情况下，报价服务器将提供一天中的时间。否则，报价服务器从已打开的文件中获取下一个报价。最后，代码将字符串转换为字节数组。

现在run方法通过将该响应发送到客户端DatagramSocket使用此代码：

InetAddress address = packet.getAddress();
int port = packet.getPort();
packet = new DatagramPacket(buf, buf.length, address, port);
socket.send(packet);

此代码段中的前两个语句分别从客户端接收的数据报包中获取Internet地址和端口号。Internet地址和端口号指示数据报包的来源。这是服务器必须发送其响应的位置。在此示例中，数据报包的字节数组不包含任何相关信息。数据包本身的到来指示来自客户端的请求，可以在数据报数据包中指示的Internet地址和端口号上找到该请求。

第三条语句创建一个新的DatagramPacket用于通过数据报套接字发送数据报消息的对象。你可以说新的DatagramPacket由于用于创建数据的构造函数，因此旨在通过套接字发送数据。该构造函数需要四个参数。前两个参数与用于创建接收数据报的构造函数所需的参数相同：一个字节数组，其中包含从发送方到接收方的消息以及此数组的长度。接下来的两个参数是不同的：Internet地址和端口号。这两个参数是数据报包目标的完整地址，必须由数据报的发送方提供。代码的最后一行发送DatagramPacket在路上。

服务器从报价文件中读取所有报价后， while循环终止，并且run方法清理：

socket.close();

QuoteClient类

的QuoteClient类为该类实现了客户端应用程序QuoteServer 。此应用程序向QuoteServer ，等待响应，并在收到响应后将其显示到标准输出。让我们详细看一下代码。

的QuoteClient类包含一个方法， main客户端应用程序的方法。顶部main方法声明了几个局部变量供其使用：

int port;
InetAddress address;
DatagramSocket socket = null;
DatagramPacket packet;
byte[] sendBuf = new byte[256];

首先， main方法处理用于调用的命令行参数QuoteClient应用：

if (args.length != 1) {
    System.out.println("Usage: java QuoteClient <hostname>");
    return;
}

的QuoteClient应用程序需要一个命令行参数： QuoteServer在跑。

接下来， main方法创建一个DatagramSocket ：

DatagramSocket socket = new DatagramSocket();

客户端使用不需要端口号的构造函数。这个构造函数只是绑定了DatagramSocket到任何可用的本地端口。客户端绑定到哪个端口都没有关系，因为DatagramPacket包含寻址信息。服务器从服务器获取端口号DatagramPacket并将其响应发送到该端口。

接下来， QuoteClient程序向服务器发送请求：

byte[] buf = new byte[256];
InetAddress address = InetAddress.getByName(args[0]);
DatagramPacket packet = new DatagramPacket(buf, buf.length, 
                                address, 4445);
socket.send(packet);

代码段获取在命令行上命名的主机的Internet地址（大概是运行服务器的计算机的名称）。这个InetAddress和端口号4445（服务器用于创建其端口号的端口号）。 DatagramSocket ）然后用于创建DatagramPacket指定该Internet地址和端口号。因此DatagramPacket将交付给报价服务器。

请注意，该代码会创建一个DatagramPacket一个空字节数组。字节数组为空，因为此数据报包只是向服务器请求信息。服务器需要知道的所有信息（发送回复的地址和端口号）自动成为数据包的一部分。

接下来，客户端从服务器获取响应并显示它：

packet = new DatagramPacket(buf, buf.length);
socket.receive(packet);
String received = new String(packet.getData(), 0, packet.getLength());
System.out.println("Quote of the Moment: " + received);

为了获得服务器的响应，客户端会创建一个“接收”数据包并使用DatagramSocket receive方法，用于接收来自服务器的回复。接收方法等待，直到发往客户端的数据报包通过套接字。请注意，如果服务器的答复由于某种原因丢失，则由于数据报模型的无保证策略，客户端将永远等待。通常，客户端会设置一个计时器，这样它就不会永远等待答复。如果没有答复到达，计时器将关闭，客户端将重新传输。

当客户端收到服务器的答复时，客户端使用getData方法从数据包中检索该数据。然后，客户端将数据转换为字符串并显示。

运行服务器和客户端

成功编译服务器和客户端程序后，即可运行它们。您必须先运行服务器程序。只需使用Java解释器并指定QuoteServer班级名称。

服务器启动后，即可运行客户端程序。记住要使用一个命令行参数运行客户端程序： QuoteServer在跑。

客户端发送请求并收到服务器的响应后，您应该看到类似于以下的输出：
