Java教程是为JDK 8编写的。此页面中描述的示例和实践没有利用更高版本中引入的改进。

Trail：自定义网络

Java平台之所以受到高度重视，部分原因是它适合编写使用Internet和World Wide Web资源并与之交互的程序。实际上，与Java兼容的浏览器充分利用了Java平台的这种功能，可以在Internet上传输和运行小程序。

本教程将带您了解编写可在Internet上使用的Java应用程序和小程序的复杂性。

网络概述有两个部分。第一部分描述了您可能已经在使用的Java平台的联网功能，而没有意识到您正在使用网络。第二篇简要概述了网络，以使您熟悉在阅读如何使用URL，套接字和数据报之前应了解的术语和概念。

“使用URL”讨论了Java程序如何使用URL访问Internet上的信息。URL（统一资源定位符）是Internet上资源的地址。您的Java程序可以使用URL连接到网络并通过网络检索信息。本课提供了URL的更完整定义，并向您展示了如何创建和解析URL，如何打开与URL的连接以及如何读取和写入该连接。

《关于套接字的全部》介绍了如何使用套接字，以便您的程序可以与网络上的其他程序进行通信。套接字是网络上运行的两个程序之间双向通讯链接的一个端点。本课向您展示客户端如何连接到标准服务器Echo服务器并通过套接字与之通信。然后，它向您介绍完整的客户端/服务器示例的详细信息，该示例向您展示如何实现客户端/服务器对的客户端和服务器端。

《关于数据报》的所有内容将逐步指导您使用数据报进行通信的简单客户端/服务器示例。然后，它挑战您改用多播套接字重写示例。

对网络参数的编程访问说明了为什么您可能要访问网络接口参数以及如何进行访问。它提供了有关如何列出分配给计算机的所有IP地址以及其他有用信息（例如，接口是否正在运行）的示例。

使用Cookie讨论了如何使用Cookie在客户端和服务器之间创建会话，以及如何在HTTP URL连接中利用Cookie。

安全注意事项： 

请注意，通过网络进行的通信必须经过当前安全管理器的批准。 安全管理器描述了什么是安全管理器以及它如何影响您的应用程序。有关JDK提供的安全功能的一般信息，请参阅Java SE中的安全功能 。

以下课程中涉及URL，套接字和数据报的示例程序是独立的应用程序，默认情况下没有安全管理器。如果将这些应用程序转换为applet，则它们可能无法通过网络进行通信，具体取决于运行它们的浏览器或查看器。请参阅小程序可以做什么和不能做什么，以获取关于小程序的安全限制的信息。
