编写套接字的服务器端

本节介绍如何编写服务器以及与之配套的客户端。客户端/服务器对中的服务器提供Knock Knock笑话。敲敲笑话受到孩子们的青睐，通常是破坏双关语的工具。他们像这样去：

服务器 ：“敲敲！”
客户 ：“谁在那里？”
服务器 ：“ Dexter”。
客户 ：“糊精是谁？”
服务器 ：“德克斯特大厅，冬青枝。”
客户 ：“格兰”。

该示例由两个独立运行的Java程序组成：客户端程序和服务器程序。客户端程序由单个类实现， KnockKnockClient ，与EchoClient上一节中的示例。服务器程序由两个类实现： KnockKnockServer和KnockKnockProtocol 。 KnockKnockServer ，类似于EchoServer ，包含main服务器程序的方法，并执行侦听端口，建立连接以及从套接字读取和写入套接字的工作。班级KnockKnockProtocol讲笑话。它跟踪当前的笑话，当前状态（发送敲敲声，发送线索等），并根据当前状态返回笑话的各种文本。该对象实现了协议，即客户端和服务器已同意使用的通信语言。

下一节将详细介绍客户端和服务器中的每个类，然后向您展示如何运行它们。

敲敲服务器

本节介绍实现Knock Knock服务器程序的代码， KnockKnockServer 。

服务器程序首先创建一个新的ServerSocket对象以侦听特定端口（请参见以下代码段中以粗体显示的语句）。运行此服务器时，请选择尚未专用于某些其他服务的端口。例如，此命令启动服务器程序KnockKnockServer以便它监听端口4444：

java KnockKnockServer 4444

服务器程序创建ServerSocket一个对象try -with-resources语句：

int portNumber = Integer.parseInt(args[0]);

try ( 
    ServerSocket serverSocket = new ServerSocket(portNumber);
    Socket clientSocket = serverSocket.accept();
    PrintWriter out =
        new PrintWriter(clientSocket.getOutputStream(), true);
    BufferedReader in = new BufferedReader(
        new InputStreamReader(clientSocket.getInputStream()));
) {

ServerSocket是一个java.net类提供了客户端/服务器套接字连接的服务器端的独立于系统的实现。的构造函数ServerSocket如果无法侦听指定端口（例如，该端口已在使用），则会引发异常。在这种情况下， KnockKnockServer别无选择，只能退出。

如果服务器成功绑定到其端口，则ServerSocket对象创建成功，服务器继续下一步-接受来自客户端的连接（ try -with-resources语句）：

clientSocket = serverSocket.accept();

的accept方法等待客户端启动并请求此服务器的主机和端口上的连接。（假设您运行了服务器程序KnockKnockServer在名为knockknockserver.example.com ）在此示例中，服务器在第一个命令行参数指定的端口号上运行。当请求并成功建立连接时，accept方法将返回一个新的Socket绑定到同一本地端口并且其远程地址和远程端口设置为客户端地址的对象。服务器可以通过此新客户端与客户端进行通信Socket并继续侦听原始客户端上的客户端连接请求ServerSocket该程序的特定版本不侦听更多的客户端连接请求。但是，在“ 支持多个客户端”中提供了该程序的修改版本。

服务器成功建立与客户端的连接后，它将使用以下代码与客户端进行通信：

try (
    // ...
    PrintWriter out =
        new PrintWriter(clientSocket.getOutputStream(), true);
    BufferedReader in = new BufferedReader(
        new InputStreamReader(clientSocket.getInputStream()));
) {
    String inputLine, outputLine;
            
    // Initiate conversation with client
    KnockKnockProtocol kkp = new KnockKnockProtocol();
    outputLine = kkp.processInput(null);
    out.println(outputLine);

    while ((inputLine = in.readLine()) != null) {
        outputLine = kkp.processInput(inputLine);
        out.println(outputLine);
        if (outputLine.equals("Bye."))
            break;
    }

此代码执行以下操作：

  1. 获取套接字的输入和输出流，并在其上打开读取器和写入器。
  2. 通过写入套接字来启动与客户端的通信（以粗体显示）。
  3. 通过读取和写入套接字与客户端进行通信（ while环）。

步骤1已经很熟悉了。步骤2以粗体显示，值得一提。上面的代码段中的粗体语句启动与客户端的对话。该代码创建了一个KnockKnockProtocol对象-跟踪当前笑话，笑话中当前状态等的对象。

之后KnockKnockProtocol创建后，代码调用KnockKnockProtocol的processInput获取服务器发送给客户端的第一条消息的方法。对于此示例，服务器首先说的是“敲门！敲！“接下来，服务器将信息写入PrintWriter连接到客户端套接字，从而将消息发送到客户端。

步骤3编码为while环。只要客户端和服务器之间还有话要说，服务器就会从套接字读取和写入套接字，从而在客户端和服务器之间来回发送消息。

服务器通过“敲门！敲！”，因此之后服务器必须等待客户端说“谁在那儿？“因此， while循环迭代从输入流读取的内容。的readLine方法等待，直到客户端通过向其输出流（服务器的输入流）中写入内容来做出响应。当客户端响应时，服务器会将客户端的响应传递给KnockKnockProtocol反对并询问KnockKnockProtocol对象以寻求适当的答复。服务器使用对println的调用，立即通过连接到套接字的输出流将答复发送给客户端。如果服务器的响应是从KnockKnockServer对象是“再见”。这表明客户端不再需要任何笑话，并且退出循环。

Java运行时会自动关闭输入和输出流，客户端套接字和服务器套接字，因为它们已在try -with-resources语句。

敲敲协议

的KnockKnockProtocol类实现客户端和服务器用于通信的协议。此类跟踪客户端和服务器在其对话中的位置，并提供服务器对客户端语句的响应。的KnockKnockProtocol对象包含所有笑话的文本，并确保客户端对服务器的语句给出正确的响应。让客户说“请谁打扰？”，当服务器说“敲！敲！”

所有客户机/服务器对都必须具有某种协议，通过它们它们可以相互通信。否则，来回传递的数据将毫无意义。您自己的客户端和服务器使用的协议完全取决于它们完成任务所需的通信。

敲敲客户

的KnockKnockClient类实现与KnockKnockServer 。 KnockKnockClient基于EchoClient上一节中的“ 读取和写入套接字”中的程序 ，您应该有点熟悉。但是无论如何，我们还是会仔细研究该程序，并根据服务器中发生的情况查看客户端中发生的情况。

当启动客户端程序时，服务器应该已经在运行并且正在侦听端口，等待客户端请求连接。因此，客户端程序要做的第一件事是打开一个套接字，该套接字连接到在指定的主机名和端口上运行的服务器：

String hostName = args[0];
int portNumber = Integer.parseInt(args[1]);

try (
    Socket kkSocket = new Socket(hostName, portNumber);
    PrintWriter out = new PrintWriter(kkSocket.getOutputStream(), true);
    BufferedReader in = new BufferedReader(
        new InputStreamReader(kkSocket.getInputStream()));
)

创建其套接字时， KnockKnockClient该示例使用第一个命令行参数的主机名，即网络上运行服务器程序的计算机的名称KnockKnockServer 。

的KnockKnockClient在创建其套接字时，示例将第二个命令行参数用作端口号。这是一个远程端口号 ，即服务器计算机上的端口号 ，并且是该端口号。 KnockKnockServer在听。例如，以下命令运行KnockKnockClient举例knockknockserver.example.com作为运行服务器程序的计算机的名称KnockKnockServer和4444作为远程端口号：

java KnockKnockClient knockknockserver.example.com 4444

客户端的套接字绑定到任何可用的本地端口，即客户端计算机上的端口。请记住，服务器也会获得一个新的套接字。如果您运行KnockKnockClient在上一个示例中使用命令行参数作为示例，然后将此套接字绑定到运行该计算机的计算机上的本地端口号4444。 KnockKnockClient例。服务器的套接字和客户端的套接字已连接。

接下来是while实现客户端和服务器之间通信的循环。服务器先讲话，所以客户端必须先听。客户端通过从附加到套接字的输入流中读取数据来完成此操作。如果服务器确实讲话，则说“再见”。客户端退出循环。否则，客户端将文本显示到标准输出，然后从用户输入标准输入的内容中读取响应。用户键入回车符后，客户端将通过附加到套接字的输出流将文本发送到服务器。

while ((fromServer = in.readLine()) != null) {
    System.out.println("Server: " + fromServer);
    if (fromServer.equals("Bye."))
        break;

    fromUser = stdIn.readLine();
    if (fromUser != null) {
        System.out.println("Client: " + fromUser);
        out.println(fromUser);
    }
}

当服务器询问客户是否希望听到另一个笑话时，通信结束，客户说“否”，服务器说“再见”。

客户端会自动关闭其输入和输出流以及套接字，因为它们是在try -with-resources语句。

运行程序

您必须首先启动服务器程序。为此，就像使用任何其他Java应用程序一样，使用Java解释器运行服务器程序。将服务器程序侦听的端口号指定为命令行参数：

java KnockKnockServer 4444

接下来，运行客户端程序。请注意，您可以在网络上的任何计算机上运行客户端。它不必与服务器在同一台计算机上运行。将运行该计算机的计算机的主机名和端口号指定为命令行参数。 KnockKnockServer服务器程序：

java KnockKnockClient knockknockserver.example.com 4444

如果太快，则可能在服务器没有机会初始化自身并开始侦听端口之前启动客户端。如果发生这种情况，您将看到来自客户端的堆栈跟踪。如果发生这种情况，只需重新启动客户端即可。

如果在第一个客户端连接到服务器时尝试启动第二个客户端，则第二个客户端将挂起。下一节“ 支持多个客户端 ”讨论了支持多个客户端。

成功建立客户端与服务器之间的连接后，屏幕上将显示以下文本：

Server: Knock! Knock!

现在，您必须以以下方式回应：

Who's there?

客户端回显您键入的内容，并将文本发送到服务器。服务器以其曲目中众多敲门笑话之一的第一行作为响应。现在，您的屏幕应包含以下内容（您键入的文本以粗体显示）：

Server: Knock! Knock!
Who's there?
Client: Who's there?
Server: Turnip

现在，您以以下方式回复：

Turnip who?

客户端再次回显您键入的内容，并将文本发送到服务器。服务器以打孔线响应。现在您的屏幕应包含以下内容：

Server: Knock! Knock!
Who's there?
Client: Who's there?
Server: Turnip
Turnip who?
Client: Turnip who?
Server: Turnip the heat, it's cold in here! Want another? (y/n)

如果您想听到另一个笑话，请输入y 。如果不是，请输入n 。如果键入y ，则服务器再次以“敲门！敲！“如果您键入n ，服务器将显示“ Bye。”，从而导致客户端和服务器退出。

如果您在任何时候输入有误， KnockKnockServer对象捕获到它，并且服务器以类似以下消息响应：

Server: You're supposed to say "Who's there?"!

然后服务器再次开始玩笑：

Server: Try again. Knock! Knock!

请注意KnockKnockProtocol宾语是关于拼写和标点的，但不是大写的。

支持多个客户

为了保持KnockKnockServer示例简单，我们将其设计为侦听和处理单个连接请求。但是，多个客户端请求可以进入同一端口，因此可以进入同一端口ServerSocket 。客户端连接请求在端口排队，因此服务器必须顺序接受连接。但是，服务器可以通过使用线程（每个客户端连接每个线程一个线程）同时为它们提供服务。

这样的服务器中逻辑的基本流程是：

while (true) {
    accept a connection;
    create a thread to deal with the client;
}

线程根据需要读取和写入客户端连接。

修改KnockKnockServer这样它可以同时为多个客户端提供服务。我们的解决方案由两个类组成： KKMultiServer和KKMultiServerThread 。 KKMultiServer永远循环，监听客户端上的客户端连接请求ServerSocket 。收到请求后， KKMultiServer接受连接，创建一个新的KKMultiServerThread对象来处理它，将它从接受返回的套接字交给它，然后启动线程。然后，服务器返回侦听连接请求。的KKMultiServerThread对象通过读取和写入套接字与客户端进行通信。运行新的Knock Knock服务器KKMultiServer然后连续运行多个客户端。
