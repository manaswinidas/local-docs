读写套接字

让我们看一个简单的示例，该示例说明程序如何使用以下命令建立与服务器程序的连接： Socket类，然后客户端如何通过套接字向服务器发送数据和从服务器接收数据。

该示例程序实现了一个客户端， EchoClient ，它连接到回显服务器。回显服务器从其客户端接收数据并将其回显。这个例子EchoServer实现一个回显服务器。（或者，客户端可以连接到任何支持Echo协议的主机。）

的EchoClient示例创建一个套接字，从而获得与回显服务器的连接。它从用户在标准输入流上读取输入，然后通过将文本写入套接字将文本转发到回显服务器。服务器通过套接字将输入回显到客户端。客户端程序读取并显示从服务器传递回去的数据。

请注意EchoClient例如，它既可以读写套接字，也可以从套接字读取数据，从而向回显服务器发送数据并从其接收数据。

让我们遍历该程序并研究有趣的部分。以下语句try -with-resources语句中EchoClient例子很关键。这些行在客户端和服务器之间建立套接字连接，并打开一个PrintWriter和一个BufferedReader在套接字上：

String hostName = args[0];
int portNumber = Integer.parseInt(args[1]);

try (
    Socket echoSocket = new Socket(hostName, portNumber);
    PrintWriter out =
        new PrintWriter(echoSocket.getOutputStream(), true);
    BufferedReader in =
        new BufferedReader(
            new InputStreamReader(echoSocket.getInputStream()));
    BufferedReader stdIn =
        new BufferedReader(
            new InputStreamReader(System.in))
)

中的第一条陈述try -with resources语句创建一个新的Socket对象并命名echoSocket 。的Socket此处使用的构造函数要求计算机的名称和您要连接的端口号。该示例程序将第一个命令行参数用作计算机的名称（主机名），将第二个命令行参数用作端口号。当您在计算机上运行该程序时，请确保您使用的主机名是您要连接的计算机的标准IP名称。例如，如果您的回显服务器正在计算机上运行echoserver.example.com它正在侦听端口号7，请首先从计算机运行以下命令echoserver.example.com如果您想使用EchoServer示例作为您的回显服务器：

java EchoServer 7

之后，运行EchoClient以下命令的示例：

java EchoClient echoserver.example.com 7

的第二条陈述try -with resources语句获取套接字的输出流并打开一个PrintWriter在上面。同样，第三条语句获取套接字的输入流并打开一个BufferedReader在上面。该示例使用读取器和写入器，以便可以在套接字上写入Unicode字符。

要将数据通过套接字发送到服务器， EchoClient示例需要写到PrintWriter 。为了获得服务器的响应， EchoClient从中读取BufferedReader宾语stdIn ，这是在try -with资源声明。如果您还不熟悉Java平台的I / O类，则不妨阅读Basic I / O。

该程序的下一个有趣部分是while环。循环一次从标准输入流中读取一行，并通过将其写入到PrintWriter连接到插座：

String userInput;
while ((userInput = stdIn.readLine()) != null) {
    out.println(userInput);
    System.out.println("echo: " + in.readLine());
}

最后的陈述while循环从中读取一行信息BufferedReader连接到插座。的readLine方法等待，直到服务器将信息回显到EchoClient 。什么时候readline返回， EchoClient将信息打印到标准输出。

的while循环继续进行，直到用户键入输入结束字符为止。那就是EchoClient此示例从用户读取输入，将其发送到Echo服务器，从服务器获取响应，然后显示它，直到到达输入结束为止。（您可以通过按Ctrl-C来键入输入结束符。）的while循环然后终止，Java运行时会自动关闭连接到套接字和标准输入流的读取器和写入器，并关闭与服务器的套接字连接。Java运行时会自动关闭这些资源，因为它们是在try -with-resources语句。Java运行时以与创建资源相反的顺序关闭这些资源。（这很好，因为在关闭套接字本身之前，应先关闭连接到套接字的流。）

该客户端程序简单明了，因为回显服务器实现了简单的协议。客户端将文本发送到服务器，然后服务器将其回显。当您的客户端程序与更复杂的服务器（例如HTTP服务器）通信时，您的客户端程序也将更加复杂。但是，基本知识与该程序基本相同：

  1. 打开插座。
  2. 打开输入流，然后将输出流输出到套接字。
  3. 根据服务器协议从流中读取和写入流。
  4. 关闭流。
  5. 关闭插座。

客户端与客户端之间仅第3步有所不同，具体取决于服务器。其他步骤大致相同。
