运行示例程序

关于安全性的注意事项

服务器和客户端程序在安装了安全管理器的情况下运行。在运行这两个程序时，需要指定一个安全策略文件，以便为代码授予运行它所需的安全权限。这是与服务器程序一起使用的示例策略文件 ：

grant codeBase "file:/home/ann/src/" {
    permission java.security.AllPermission;
};

这是与客户端程序一起使用的示例策略文件 ：

grant codeBase "file:/home/jones/src/" {
    permission java.security.AllPermission;
};

对于这两个示例策略文件，所有权限都授予该程序的本地类路径中的类，因为本地应用程序代码是受信任的，但没有权限授予从其他位置下载的代码。因此，计算引擎服务器限制了它执行的任务（其代码不被信任并且可能是敌对的）执行任何需要安全权限的操作。示例客户的Pi任务不需要任何权限即可执行。

在此示例中，服务器程序的策略文件被命名为server.policy ，并且客户端程序的策略文件名为client.policy 。

启动服务器

在启动计算引擎之前，您需要启动RMI注册表。RMI注册表是一种简单的服务器端引导程序命名工具，使远程客户端可以获取对初始远程对象的引用。可以从rmiregistry命令。执行之前rmiregistry ，您必须确保要在其中运行的shell或窗口rmiregistry要么没有CLASSPATH环境变量设置或具有CLASSPATH环境变量，不包含要下载到远程对象的客户端的任何类的路径。

要在服务器上启动注册表，请执行rmiregistry命令。该命令不产生任何输出，通常在后台运行。对于此示例，注册表在主机上启动mycomputer 。

Microsoft Windows （使用javaw如果start不可用）：

start rmiregistry

Solaris OS或Linux ：

rmiregistry &

默认情况下，注册表在端口1099上运行。要在其他端口上启动注册表，请在命令行上指定端口号。别忘了取消您的设置CLASSPATH环境变量。

Microsoft Windows ：

start rmiregistry 2001

Solaris OS或Linux ：

rmiregistry 2001 &

一旦注册表启动，就可以启动服务器。您需要确保compute.jar文件和远程对象实现类在您的类路径中。启动计算引擎时，需要使用java.rmi.server.codebase属性，可以通过网络访问服务器的类。在此示例中，可供下载的服务器端类是Compute和Task接口，这些接口在compute.jar文件在public_html\classes用户目录ann 。计算引擎服务器在主机上启动mycomputer ，启动注册表的主机。

Microsoft Windows ：

java -cp c:\home\ann\src;c:\home\ann\public_html\classes\compute.jar
     -Djava.rmi.server.codebase=file:/c:/home/ann/public_html/classes/compute.jar
     -Djava.rmi.server.hostname=mycomputer.example.com
     -Djava.security.policy=server.policy
        engine.ComputeEngine

Solaris OS或Linux ：

java -cp /home/ann/src:/home/ann/public_html/classes/compute.jar
     -Djava.rmi.server.codebase=http://mycomputer/~ann/classes/compute.jar
     -Djava.rmi.server.hostname=mycomputer.example.com
     -Djava.security.policy=server.policy
        engine.ComputeEngine

以上java命令定义以下系统属性：

  • 的java.rmi.server.codebase属性指定的位置，基本代码URL，从这个服务器类始发的定义可以下载。如果代码库指定了目录层次结构（与JAR文件相对），则必须在代码库URL的末尾添加斜杠。
  • 的java.rmi.server.hostname属性指定要在此Java虚拟机中导出的远程对象的存根中放入的主机名或地址。此值是客户端在尝试传达远程方法调用时使用的主机名或地址。默认情况下，RMI实现使用服务器的IP地址，如java.net.InetAddress.getLocalHost API。但是，有时此地址不适用于所有客户端，并且标准主机名会更有效。为确保RMI为可从所有潜在客户端路由的服务器使用主机名（或IP地址），请设置java.rmi.server.hostname属性。
  • 的java.security.policy属性用于指定包含要授予的权限的策略文件。

启动客户端

注册表和计算引擎运行后，您可以启动客户端，并指定以下内容：

  • 客户服务其班级的位置（ Pi类） java.rmi.server.codebase属性
  • 的java.security.policy属性，用于指定安全策略文件，该文件包含您打算授予各种代码的权限
  • 作为命令行参数，服务器的主机名（以便客户端知道在何处可以找到服务器的主机名）。 Compute远程对象）以及要在其中使用的小数位数计算

在另一台主机（名为的主机）上启动客户端mysecondcomputer （例如），如下所示：

Microsoft Windows ：

java -cp c:\home\jones\src;c:\home\jones\public_html\classes\compute.jar
     -Djava.rmi.server.codebase=file:/c:/home/jones/public_html/classes/
     -Djava.security.policy=client.policy
        client.ComputePi mycomputer.example.com 45

Solaris OS或Linux ：

java -cp /home/jones/src:/home/jones/public_html/classes/compute.jar
     -Djava.rmi.server.codebase=http://mysecondcomputer/~jones/classes/
     -Djava.security.policy=client.policy
        client.ComputePi mycomputer.example.com 45

请注意，在命令行上设置了类路径，以便解释器可以找到客户端类和包含接口的JAR文件。另请注意， java.rmi.server.codebase指定目录层次结构的属性以斜杠结尾。
