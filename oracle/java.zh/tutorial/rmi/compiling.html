编译示例程序

在部署了诸如计算引擎之类的服务的实际场景中，开发人员可能会创建一个Java Archive（JAR）文件，其中包含Compute和Task用于实现服务器类和使用客户端程序的接口。接下来，开发人员（也许是接口JAR文件的相同开发人员）将编写Compute接口该服务并将其部署在客户端可用的计算机上。客户端程序的开发人员可以使用Compute和Task接口，包含在JAR文件中，并独立开发使用Compute服务。

在本节中，您将学习如何设置JAR文件，服务器类和客户端类。您会看到客户的Pi类将在运行时下载到服务器。另外， Compute和Task接口将在运行时从服务器下载到注册表。

本示例将接口，远程对象实现和客户端代码分为三个包：

  • compute – Compute和Task介面
  • engine – ComputeEngine实现类
  • client – ComputePi客户代码和Pi任务执行

首先，您需要构建接口JAR文件以提供给服务器和客户端开发人员。

构建接口类的JAR文件

首先，您需要在compute打包，然后构建包含其类文件的JAR文件。假设该用户waldo编写了这些接口并将源文件放置在目录中c:\home\waldo\src\compute在Windows或目录上/home/waldo/src/compute在Solaris OS或Linux上。给定这些路径，您可以使用以下命令来编译接口并创建JAR文件：

Microsoft Windows ：

cd c:\home\waldo\src
javac compute\Compute.java compute\Task.java
jar cvf compute.jar compute\*.class

Solaris OS或Linux ：

cd /home/waldo/src
javac compute/Compute.java compute/Task.java
jar cvf compute.jar compute/*.class

的jar由于以下原因，该命令显示以下输出： -v选项：

added manifest
adding: compute/Compute.class(in = 307) (out= 201)(deflated 34%)
adding: compute/Task.class(in = 217) (out= 149)(deflated 31%)

现在，您可以分发compute.jar文件发送给服务器和客户端应用程序的开发人员，以便他们可以使用这些接口。

使用以下命令构建服务器端或客户端类后javac编译器，如果其他Java虚拟机需要动态下载这些类中的任何一个，则必须确保将其类文件放置在可通过网络访问的位置。在此示例中，对于Solaris OS或Linux，此位置为/home/user/public_html/classes因为许多Web服务器都允许访问用户的public_html通过HTTP URL构造的目录http://host/~user/ 。如果您的Web服务器不支持此约定，则可以在Web服务器的层次结构中使用其他位置，也可以使用文件URL。文件URL采用以下形式file:/home/user/public_html/classes/在Solaris OS或Linux上以及file:/c:/home/user/public_html/classes/在Windows上。您还可以根据需要选择其他类型的URL。

类文件的网络可访问性使RMI运行时可以在需要时下载代码。RMI并未定义自己的代码下载协议，而是使用Java平台支持的URL协议（例如HTTP）来下载代码。请注意，不需要使用完整的重量级Web服务器来提供这些类文件。例如，可以在以下位置找到一个简单的HTTP服务器，该服务器提供使类可用于通过HTTP在RMI中下载的功能。 。
另请参见“ 远程方法调用主页” 。

构建服务器类

的engine软件包仅包含一个服务器端实现类， ComputeEngine ，实现远程接口Compute 。

假设该用户ann ，开发者ComputeEngine类，已放置ComputeEngine.java在目录中c:\home\ann\src\engine在Windows或目录上/home/ann/src/engine在Solaris OS或Linux上。她正在部署类文件供客户下载到她的子目录中public_html目录， c:\home\ann\public_html\classes在Windows上或/home/ann/public_html/classes在Solaris OS或Linux上。该位置可通过某些Web服务器访问，例如http://host:port/~ann/classes/ 。

的ComputeEngine类取决于Compute和Task接口，包含在compute.jar JAR文件。因此，您需要compute.jar构建服务器类时，将文件放在类路径中。假设compute.jar文件位于目录中c:\home\ann\public_html\classes在Windows或目录上/home/ann/public_html/classes在Solaris OS或Linux上。给定这些路径，您可以使用以下命令来构建服务器类：

Microsoft Windows ：

cd c:\home\ann\src
javac -cp c:\home\ann\public_html\classes\compute.jar
    engine\ComputeEngine.java

Solaris OS或Linux ：

cd /home/ann/src
javac -cp /home/ann/public_html/classes/compute.jar
    engine/ComputeEngine.java

的存根类ComputeEngine实现Compute接口，指的是Task接口。因此，这两个接口的类定义需要可通过网络访问，以便存根可以被其他Java虚拟机（例如注册表的Java虚拟机）接收。客户端Java虚拟机将在其类路径中具有这些接口，因此实际上不需要下载其定义。的compute.jar文件在public_html目录可以达到这个目的。

现在，计算引擎已准备就绪，可以进行部署。您可以立即执行此操作，也可以等到构建客户端之后再执行。

建立客户端类

的client程序包包含两个类， ComputePi ，主要客户程序，以及Pi ，客户的实施Task接口。

假设该用户jones ，客户端类的开发者放置了ComputePi.java和Pi.java在目录中c:\home\jones\src\client在Windows或目录上/home/jones/src/client在Solaris OS或Linux上。他正在为计算引擎部署类文件，以将其下载到他的子目录中public_html目录， c:\home\jones\public_html\classes在Windows上或/home/jones/public_html/classes在Solaris OS或Linux上。该位置可通过某些Web服务器访问，例如http://host:port/~jones/classes/ 。

客户端类取决于Compute和Task接口，包含在compute.jar JAR文件。因此，您需要compute.jar生成客户端类时，将文件放在类路径中。假设compute.jar文件位于目录中c:\home\jones\public_html\classes在Windows或目录上/home/jones/public_html/classes在Solaris OS或Linux上。给定这些路径后，您可以使用以下命令来构建客户端类：

Microsoft Windows ：

cd c:\home\jones\src
javac -cp c:\home\jones\public_html\classes\compute.jar
    client\ComputePi.java client\Pi.java
mkdir c:\home\jones\public_html\classes\client
cp client\Pi.class
    c:\home\jones\public_html\classes\client

Solaris OS或Linux ：

cd /home/jones/src
javac -cp /home/jones/public_html/classes/compute.jar
    client/ComputePi.java client/Pi.java
mkdir /home/jones/public_html/classes/client
cp client/Pi.class
    /home/jones/public_html/classes/client

只有Pi类需要放置在目录中public_html\classes\client因为只有Pi该类需要可用于下载到计算引擎的Java虚拟机。现在，您可以运行服务器，然后运行客户端。
