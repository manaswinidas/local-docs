设计远程接口

计算引擎的核心是协议，该协议使任务可以提交给计算引擎，计算引擎可以运行这些任务，并将这些任务的结果返回给客户端。该协议在计算引擎支持的接口中表示。下图说明了此协议的远程通信。

每个接口都包含一个方法。计算引擎的远程接口， Compute ，使任务可以提交给引擎。客户端界面， Task,定义计算引擎如何执行提交的任务。

的compute.Compute接口定义了远程访问部分，即计算引擎本身。这是Compute接口：

package compute;

import java.rmi.Remote;
import java.rmi.RemoteException;

public interface Compute extends Remote {
    <T> T executeTask(Task<T> t) throws RemoteException;
}

通过扩展接口java.rmi.Remote ， Compute interface将自身标识为一个接口，该接口的方法可以从另一个Java虚拟机调用。实现此接口的任何对象都可以是远程对象。

作为远程接口的成员， executeTask method是一种远程方法。因此，必须将此方法定义为能够抛出java.rmi.RemoteException 。RMI系统从远程方法调用中抛出此异常，以表明发生通信故障或协议错误。一种RemoteException是一个已检查的异常，因此任何调用远程方法的代码都需要通过捕获该异常或在该异常的声明中对其进行处理。 throws条款。

计算引擎所需的第二个接口是Task接口，这是参数的类型executeTask中的方法Compute接口。的compute.Task接口定义了计算引擎与其需要完成的工作之间的接口，从而提供了开始工作的方式。这是Task接口：

package compute;

public interface Task<T> {
    T execute();
}

的Task接口定义单个方法， execute ，它没有参数，也不会引发异常。因为接口不扩展Remote ，此界面中的方法无需列出java.rmi.RemoteException在其throws条款。

的Task接口具有类型参数， T ，代表任务计算的结果类型。该界面的execute方法返回计算结果，因此其返回类型为T 。

的Compute接口的executeTask方法反过来返回执行结果Task实例传递给它。就这样executeTask方法有自己的类型参数， T ，将其自己的返回类型与传递的结果类型相关联Task实例。

RMI使用Java对象序列化机制按值在Java虚拟机之间传输对象。对于被视为可序列化的对象，其类必须实现java.io.Serializable标记界面。因此，实现Task接口还必须实现Serializable ，用于任务结果的对象类也必须如此。

一个任务可以运行各种任务Compute对象，只要它们是Task类型。实现此接口的类可以包含任务计算所需的任何数据以及计算所需的任何其他方法。

这就是RMI如何使这种简单的计算引擎成为可能。因为RMI可以假设Task对象是用Java编程语言编写的， Task RMI将根据需要将计算引擎之前未知的对象下载到计算引擎的Java虚拟机中。此功能使计算引擎的客户端可以定义要在服务器计算机上运行的新任务，而无需在该计算机上显式安装代码。

计算引擎，由ComputeEngine类，实现Compute界面，通过调用其界面可以将不同的任务提交给它executeTask方法。这些任务是使用任务的execute方法和结果返回到远程客户端。
