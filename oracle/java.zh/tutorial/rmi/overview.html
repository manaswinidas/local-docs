RMI应用程序概述

RMI应用程序通常包含两个单独的程序，即服务器和客户端。一个典型的服务器程序会创建一些远程对象，使对这些对象的引用可访问，并等待客户端调用这些对象上的方法。一个典型的客户端程序获取对服务器上一个或多个远程对象的远程引用，然后调用它们上的方法。RMI提供了一种机制，服务器和客户端通过该机制进行通信并来回传递信息。这样的应用程序有时称为分布式对象应用程序 。

分布式对象应用程序需要执行以下操作：

  • 找到远程对象。 应用程序可以使用各种机制来获取对远程对象的引用。例如，应用程序可以使用RMI的简单命名功能RMI注册表注册其远程对象。或者，应用程序可以传递和返回远程对象引用，作为其他远程调用的一部分。
  • 与远程对象进行通信。 远程对象之间的通信详细信息由RMI处理。对程序员而言，远程通信看起来类似于常规的Java方法调用。
  • 为传递的对象加载类定义。 因为RMI使对象能够来回传递，所以它提供了用于加载对象的类定义以及传输对象的数据的机制。

下图描述了使用RMI注册表获取对远程对象的引用的RMI分布式应用程序。服务器调用注册表以将名称与远程对象关联（或绑定）。客户端在服务器的注册表中通过其名称查找远程对象，然后在其上调用一个方法。该图还显示RMI系统使用现有的Web服务器在需要时从服务器到客户端以及从客户端到服务器为对象加载类定义。

动态代码加载的优点

RMI的中心和独特功能之一是，如果未在接收方的Java虚拟机中定义对象的类，则它可以下载该类的定义。以前只能在单个Java虚拟机中使用的对象的所有类型和行为，都可以传输到另一个可能是远程的Java虚拟机。RMI按对象的实际类传递对象，因此将对象发送到另一个Java虚拟机时，它们的行为不会改变。此功能允许将新类型和行为引入远程Java虚拟机，从而动态扩展应用程序的行为。此跟踪中的计算引擎示例使用此功能将新行为引入分布式程序。

远程接口，对象和方法

像任何其他Java应用程序一样，使用Java RMI构建的分布式应用程序由接口和类组成。接口声明方法。这些类实现在接口中声明的方法，也许还声明其他方法。在分布式应用程序中，某些实现可能驻留在某些Java虚拟机中，而不能驻留在其他虚拟机中。具有可以在Java虚拟机之间调用的方法的对象称为远程对象 。

通过实现具有以下特征的远程接口 ，对象将变为远程：

  • 远程接口扩展了接口java.rmi.Remote 。
  • 接口的每个方法都声明java.rmi.RemoteException在其throws子句，以及任何特定于应用程序的例外。

当对象从一个Java虚拟机传递到另一Java虚拟机时，RMI将其与非远程对象区别对待。RMI不会在接收的Java虚拟机中复制实现对象，而是传递远程对象的远程存根 。存根充当远程对象的本地代表或代理，对于客户端而言，它基本上是远程引用。客户端在本地存根上调用方法，该存根负责对远程对象执行方法调用。

远程对象的存根实现与远程对象实现的相同的一组远程接口。使用此属性，可以将存根转换为远程对象实现的任何接口。但是， 只有在远程接口中定义的那些方法才可以从接收方Java虚拟机中调用。

使用RMI创建分布式应用程序

使用RMI开发分布式应用程序涉及以下一般步骤：

  1. 设计和实现分布式应用程序的组件。
  2. 编译源。
  3. 使班级网络可访问。
  4. 启动应用程序。

设计和实现应用程序组件

首先，确定您的应用程序体系结构，包括哪些组件是本地对象以及哪些组件可远程访问。此步骤包括：

  • 定义远程接口。 远程接口指定客户端可以远程调用的方法。客户端编程为远程接口，而不是这些接口的实现类。此类接口的设计包括确定将用作这些方法的参数和返回值的对象类型。如果这些接口或类尚不存在，则还需要定义它们。
  • 实现远程对象。 远程对象必须实现一个或多个远程接口。远程对象类可以包括仅本地可用的其他接口和方法的实现。如果要将任何本地类用作这些方法中任何一个的参数或返回值，则也必须实现它们。
  • 实施客户。 在定义了远程接口之后，包括在部署了远程对象之后，都可以在任何时候实现使用远程对象的客户端。

编译源

与任何Java程序一样，您可以使用javac编译器编译源文件。源文件包含远程接口，其实现，任何其他服务器类和客户端类的声明。

注意：对于Java Platform Standard Edition 5.0之前的版本，需要额外的步骤来建立存根类，方法是使用rmic编译器。但是，此步骤不再是必需的。

使班级可访问网络

在此步骤中，您可以通过网络访问某些类定义，例如，远程接口及其相关类型的定义以及需要下载到客户端或服务器的类的定义。通常通过Web服务器通过网络访问类定义。

开始申请

启动应用程序包括运行RMI远程对象注册表，服务器和客户端。

构建通用计算引擎

本教程着重于一个简单但功能强大的分布式应用程序，称为计算引擎 。计算引擎是服务器上的远程对象，可从客户端获取任务，运行任务并返回任何结果。任务在运行服务器的计算机上运行。这种类型的分布式应用程序可以使许多客户端计算机使用功能特别强大的计算机或具有专用硬件的计算机。

计算引擎的新颖之处在于，在编写或启动计算引擎时无需定义其运行的任务。可以随时创建新的任务，然后将其分配给要运行的计算引擎。任务的唯一要求是其类实现特定的接口。RMI系统可以将完成任务所需的代码下载到计算引擎。然后，计算引擎使用运行计算引擎的计算机上的资源来运行任务。

Java平台的动态特性实现了执行任意任务的能力，Java平台的动态特性通过RMI扩展到网络。 RMI将任务代码动态加载到计算引擎的Java虚拟机中，并且无需事先了解实现该任务的类就可以运行该任务。这种具有动态下载代码能力的应用程序通常称为基于行为的应用程序 。此类应用程序通常需要完整的启用代理的基础结构。使用RMI，此类应用程序是Java平台上分布式计算的基本机制的一部分。
