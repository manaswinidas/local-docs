JAXB示例

以下各节描述如何使用JAXB RI捆绑包中包含的示例应用程序。可从http://jaxb.java.net获得JAXB RI捆绑包。下载并安装JAXB RI捆绑软件。这些示例位于jaxb-ri-install / samples /目录中。这些示例演示并基于JAXB的关键功能和概念。按照显示的顺序执行以下步骤。

阅读完本节后，您应该对JAXB感到足够自在，可以：

  • 从XML模式生成JAXB Java类
  • 使用派生于模式的JAXB类来解组和封送Java应用程序中的XML内容
  • 使用模式派生的JAXB类创建Java内容树
  • 在解组期间和运行时验证XML内容
  • 自定义JAXB模式到Java绑定

本文档描述了三组示例：

  • 基本示例（Modify Marshal，Unmarshal Validate）展示了基本的JAXB概念，例如使用默认设置和绑定来解组，编组和验证XML内容。
  • 定制示例（Customize Inline，Datatype Converter，External Customize）展示了各种定制XML模式到Java对象的默认绑定的方法。
  • java-to-schema示例显示了如何使用注释将Java类映射到XML模式。
注意：基本示例和定制示例基于“采购订单”业务情景。每使用一个XML文档，PO.xml而 ，编写针对XML模式，po.xsd中 。这些文档来自David C. Fallside编辑的W3C XML Schema Part 0：Primer 。

基本和自定义示例目录包含几个基本文件：

  • po.xsd是XML模式，用作JAXB绑定编译器的输入，并从中生成基于模式的JAXB Java类。对于Customize Inline和Datatype Converter示例，此文件包含内联绑定自定义项。
  • po.xml是包含示例XML内容的Purchase Order XML文件，在每个示例中，它都是解编入Java内容树的文件。在每个示例中，该文件几乎相同。在内容上存在细微差别，以突出显示不同的JAXB概念。
  • Main.java是每个示例的主要Java类。
  • build.xml是为您提供方便的Ant项目文件。使用Ant工具自动生成，编译和运行模式派生的JAXB类。在各个示例中， build.xml文件有所不同。
  • 内联自定义示例中的MyDatatypeConverter.java是用于提供自定义数据类型转换的Java类。
  • “外部定制”示例中的binding.xjb是一个外部绑定声明文件，该文件传递给JAXB绑定编译器以自定义默认的JAXB绑定。

JAXB编译器选项

JAXB XJC模式绑定编译器将源XML模式转换或绑定到Java编程语言中的一组JAXB内容类。在JAXB RI束xjc.sh在Solaris / Linux和xjc.bat在Windows上：编译器类，XJC，提供作为。xjc类包含在JDK类库中（在tools.jar中）。

xjc.sh和xjc.bat都使用相同的命令行选项。您可以通过不带任何选项或使用-help开关调用脚本来显示快速使用说明。语法如下：

xjc [-options ...] <schema file/URL/dir/jar>... [-b >bindinfo<] ...

如果指定了dir ，将编译目录中的所有模式文件。如果指定了jar ，将编译/META-INF/sun-jaxb.episode绑定文件。

xjc命令行选项如下：

-nv
不要对输入模式或多个模式执行严格的验证。默认情况下， xjc在处理之前会对源模式进行严格验证。注意，这并不意味着绑定编译器将不执行任何验证；它只会执行不太严格的验证。
-延期
默认情况下，XJC绑定编译器严格执行JAXB规范的“兼容性”一章中概述的规则。在默认（严格）模式下，您还仅限于仅使用规范中定义的绑定定制。通过使用-extension开关，可以使用JAXB供应商扩展。
-b 文件
指定一个或多个要处理的外部绑定文件。（每个绑定文件必须具有自己的-b开关。）外部绑定文件的语法很灵活。您可能有一个包含多个模式自定义项的绑定文件，也可以将这些自定义项分解为多个绑定文件。另外，在命令行上模式文件和绑定文件的顺序无关紧要。
-d 目录
默认情况下，XJC绑定编译器在当前目录中生成Java内容类。使用此选项可以指定备用输出目录。该目录必须已经存在； XJC绑定编译器不会为您创建一个。
-p 包
指定备用输出目录。默认情况下，XJC绑定编译器将在当前目录中生成Java内容类。输出目录必须已经存在； XJC绑定编译器不会为您创建一个。
-http 代理代理
指定HTTP / HTTPS代理。格式为[ 用户 [： 密码 ] @] proxyHost [： proxyPort ] 。参考实现仍然支持旧的-host和-port选项，以实现向后兼容，但已弃用了它们。
-httpproxyfile f
类似于-httpproxy选项，但在文件中使用参数来保护密码。
-classpath arg
指定在哪里可以找到客户端使用的客户端应用程序类文件和定制。
-目录 文件
指定目录文件以解析外部实体引用。支持TR9401，XCatalog和OASIS XML目录格式。有关更多信息，请参见XML实体和URI解析器文档或检查catalog-resolver示例应用程序。
-只读
强制XJC绑定编译器将生成的Java源标记为只读。默认情况下，XJC绑定编译器不会对其生成的Java源文件进行写保护。
-npa
禁止将软件包级别的注释生成到** / package-info.java中 。使用此开关会使生成的代码将这些注释内部化为其他生成的类。
-无标题
禁止使用时间戳生成文件头。
-target（2.0 | 2.1）
表现得像XJC 2.0或2.1，并生成不使用任何XJC 2.2功能的代码。
-enableIntrospection
启用布尔值获取器/设置器的正确生成，以启用Bean自省api。
-contentForWildcard
为具有多个xs：any派生元素的类型生成content属性。
-xmlschema
将输入模式视为W3C XML模式（默认）。如果未指定此开关，则将输入模式视为W3C XML模式。
-冗长
在编译器输出中要格外详细。
-安静
禁止编译器输出，例如进度信息和警告。
-救命
显示编译器开关的简要摘要。
-版
显示编译器版本信息。
-完整版本
显示编译器的完整版本信息。
-Xinject代码
将指定的Java代码片段注入到生成的代码中。
-Xlocator
为生成的代码启用源位置支持。
-Xsync方法
生成与synchronized关键字访问方法。
标记生成
用-@javax.annotation标记生成的代码。 生成的注释。
-episode 文件
生成情节文件以进行单独编译。

JAXB模式生成器选项

JAXB模式生成器schemagen为Java类中引用的每个命名空间创建一个模式文件。可以使用平台的bin目录中的适当schemagen shell脚本来启动模式生成器。模式生成器仅处理Java源文件。如果您的Java源引用了其他类，则必须从系统CLASSPATH环境变量访问这些源；否则，生成架构时将发生错误。无法控制生成的模式文件的名称。

您可以通过不带任何选项的脚本调用或使用-help选项来显示快速使用说明。语法如下：

schemagen [-d path] 
    [java-source-files]

-d path选项指定处理器生成的和javac生成的类文件的位置。

物品类别

ObjectFactory类

在ObjectFactory.java中 ：

    • 字符串常量create 。
    • 如果Java内容接口嵌套在另一个接口中，则所有外部Java类名称。
    • Java内容接口的名称。
