排序

的sort算法重新排序List因此，其元素根据排序关系按升序排列。提供了两种形式的操作。简单形式需要List并根据其元素的自然顺序对其进行排序 。如果您不熟悉自然排序的概念，请阅读“ 对象排序”部分。

的sort操作使用快速和稳定的稍微优化的合并排序算法：

  • 快速 ：保证运行n log(n)时间并在几乎排序的列表上运行得更快。经验测试表明，它与高度优化的快速排序一样快。通常认为，快速排序比合并排序要快，但它不稳定且不能保证n log(n)性能。
  • 稳定 ：它不会重新排列相等的元素。如果您对不同属性重复对同一列表进行排序，这很重要。如果邮件程序的用户按邮寄日期对收件箱进行排序，然后按发件人对邮件收件箱进行排序，则用户自然希望来自给定发件人的当前连续邮件列表将（仍）按邮寄日期进行排序。仅当第二种情况稳定时，才能保证这一点。

下列trivial program按字典顺序（字母顺序）打印其参数。


import java.util.*;

public class Sort {
    public static void main(String[] args) {
        List<String> list = Arrays.asList(args);
        Collections.sort(list);
        System.out.println(list);
    }
}

让我们运行程序。

% java Sort i walk the line

产生以下输出。

[i, line, the, walk]

例行数据处理

的Collections该类提供了五种算法来对List对象，所有这些都非常简单：

  • reverse —颠倒元素中元素的顺序List 。
  • fill —覆盖List具有指定值。此操作对于重新初始化List 。
  • copy —接受两个参数，一个目的地List和一个来源List ，然后将源元素复制到目标中，从而覆盖其内容。目的地List必须至少与来源一样长。如果更长，则目标中的其余元素List不受影响。
  • swap —交换元素中指定位置的元素List 。
  • addAll -将所有指定的元素添加到Collection 。要添加的元素可以单独指定或作为数组指定。

正在搜寻

的binarySearch算法搜索排序后的指定元素List 。该算法有两种形式。第一需要List和要搜索的元素（“搜索关键字”）。此表格假设List根据其元素的自然顺序以升序排序。第二种形式是Comparator除了List和搜索键，并假定List根据指定的顺序升序排列Comparator 。的sort算法可用于对List打电话之前binarySearch 。

两种形式的返回值都相同。如果List包含搜索关键字，则返回其索引。如果不是，则返回值为(-(insertion point) - 1) ，其中插入点是将值插入到List ，或者第一个元素的索引大于值或list.size()如果所有元素List小于指定值。这个公认的丑陋公式保证了返回值将是>= 0当且仅当找到搜索键时。结合布尔值基本上是一种技巧(found)和一个整数(index)变成一个int返回值。

以下成语，可用于两种形式的binarySearch操作，查找指定的搜索关键字，并将其插入适当的位置（如果尚不存在）。

int pos = Collections.binarySearch(list, key);
if (pos < 0)
   l.add(-pos-1, key);

  • frequency —计算指定元素在指定集合中出现的次数
  • disjoint -确定是否两个Collections不相交也就是说，它们是否不包含共同点

寻找极致价值

的min和max算法分别返回指定元素中包含的最小和最大元素Collection 。这两种操作都有两种形式。简单的形式只需要一个Collection并根据元素的自然顺序返回最小（或最大）元素。第二种形式是Comparator除了Collection并根据指定的值返回最小（或最大）元素Comparator 。
