地图实施

Map实现分为通用，专用和并发实现。

通用地图实现

三大通用Map实现是HashMap ， TreeMap和LinkedHashMap 。如果你需要SortedMap操作或按键排序Collection -view迭代，使用TreeMap ;如果您想要最大速度并且不关心迭代顺序，请使用HashMap ;如果您想靠近- HashMap性能和插入顺序迭代，使用LinkedHashMap 。在这方面， Map类似于Set 。同样，“ 设置实施”部分中的所有其他内容也适用于Map实现。

LinkedHashMap提供两种无法使用的功能LinkedHashSet 。当您创建一个LinkedHashMap ，您可以根据密钥访问权限而不是插入顺序进行订购。换句话说，仅查找与键关联的值会将键带到地图的末尾。也， LinkedHashMap提供removeEldestEntry方法，可以重写此方法，以强加一种策略，以便在将新映射添加到地图时自动删除陈旧的映射。这使得实现自定义缓存非常容易。

例如，此覆盖将使地图最多可以扩展到100个条目，然后每次添加新条目时都会删除最旧的条目，从而保持100个条目的稳定状态。

private static final int MAX_ENTRIES = 100;

protected boolean removeEldestEntry(Map.Entry eldest) {
    return size() > MAX_ENTRIES;
}

专用地图实现

共有三种特殊的Map实现- EnumMap ， WeakHashMap和IdentityHashMap 。 EnumMap ，在内部实现为array ，是一种高性能Map与枚举键配合使用的实现。此实现结合了安全性的丰富性和安全性Map与阵列的速度接近。如果要将枚举映射到值，则应始终使用EnumMap优先于数组。

WeakHashMap是Map接口，仅存储对其键的弱引用。仅存储弱引用时，如果不再在其外部引用其键，则可以对键值对进行垃圾回收。 WeakHashMap 。此类提供了利用弱引用功能的最简单方法。这对于实现“类似注册表的”数据结构很有用，在该结构中，当任何线程都无法再访问其键时，条目的实用性就会消失。

IdentityHashMap是基于身份的Map基于哈希表的实现。此类对于保留拓扑的对象图转换（例如序列化或深度复制）很有用。要执行这样的转换，您需要维护一个基于身份的“节点表”，以跟踪已经看到了哪些对象。基于身份的映射还用于在动态调试器和类似系统中维护对象到元信息的映射。最后，基于身份的映射在阻止“欺骗攻击”中很有用equals方法，因为IdentityHashMap从不调用equals其键上的方法。此实现的另一个好处是它速度很快。
