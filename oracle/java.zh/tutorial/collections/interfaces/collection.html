收集界面

一种Collection表示一组称为其元素的对象。的Collection接口用于传递需要最大通用性的对象集合。例如，按照惯例，所有通用集合实现都有一个构造函数，该构造函数采用Collection论点。此构造函数称为转换构造函数 ，它初始化新集合以包含指定集合中的所有元素，无论给定集合的子接口或实现类型如何。换句话说，它允许您转换集合的类型。

例如，假设您有一个Collection c ，可能是List ， 一种Set或另一种Collection 。这个习语创造了一个新的ArrayList （实施List接口），最初包含中的所有元素c 。

List<String> list = new ArrayList<String>(c);
或者-如果您使用的是JDK 7或更高版本，则可以使用菱形运算符：
List<String> list = new ArrayList<>(c);

的Collection接口包含执行基本操作的方法，例如int size() ， boolean isEmpty() ， boolean contains(Object element) ， boolean add(E element) ， boolean remove(Object element)和Iterator iterator() 。

它还包含对整个集合进行操作的方法，例如boolean containsAll(Collection c) ， boolean addAll(Collection c) ， boolean removeAll(Collection c) ， boolean retainAll(Collection c)和void clear() 。

数组操作的其他方法（例如Object[] toArray()和 T[] toArray(T[] a)也存在。

在JDK 8和更高版本中， Collection界面还公开了方法Stream stream()和Stream parallelStream() ，用于从基础集合中获取顺序或并行流。（有关使用流的更多信息，请参阅名为“ 聚合操作”的课程。）

的Collection介面会根据您的期望Collection代表一组对象。它具有告诉您集合中有多少个元素的方法（ size ， isEmpty ），检查给定对象是否在集合中的方法（ contains ），从集合中添加和删除元素的方法（ add ， remove ），以及在集合上提供迭代器的方法（ iterator ）。

的add通常对method进行足够的定义，以使它对于允许重复的集合和不允许重复的集合有意义。它保证了Collection调用完成后将包含指定的元素，并返回true如果Collection因通话而改变。同样， remove方法旨在从中删除指定元素的单个实例。 Collection ，假设它包含以开头并返回的元素true如果Collection结果被修改了。

遍历集合

有三种遍历集合的方法：（1）使用聚合操作（2） for-each构造和（3）通过使用Iterator s。

综合运营

在JDK 8和更高版本中，对集合进行迭代的首选方法是获取流并对其执行聚合操作。聚合操作通常与lambda表达式结合使用，以使编程使用更少的代码来更具表现力。以下代码依次遍历一组形状并打印出红色对象：

myShapesCollection.stream()
.filter(e -> e.getColor() == Color.RED)
.forEach(e -> System.out.println(e.getName()));

同样，您可以轻松地请求并行流，如果集合足够大并且您的计算机具有足够的核心，这可能很有意义：

myShapesCollection.parallelStream()
.filter(e -> e.getColor() == Color.RED)
.forEach(e -> System.out.println(e.getName()));

使用此API收集数据的方式有很多。例如，您可能要转换Collection至String对象，然后加入它们，并用逗号分隔：

    String joined = elements.stream()
    .map(Object::toString)
    .collect(Collectors.joining(", "));

或将所有雇员的薪金相加：

int total = employees.stream()
.collect(Collectors.summingInt(Employee::getSalary)));

这些只是您可以使用流和聚合操作执行的几个示例。有关更多信息和示例，请参阅名为“ 聚合操作”的课程。

Collections框架一直提供许多所谓的“批量操作”作为其API的一部分。这些方法包括对整个集合进行操作的方法，例如containsAll ， addAll ， removeAll等等。请勿将这些方法与JDK 8中引入的聚合操作混淆。新的汇总操作与现有的批量操作之间的主要区别（ containsAll ， addAll等）是旧版本都是mutative的 ，这意味着它们都修改了基础集合。相反，新的聚合操作不会修改基础集合。使用新的聚合操作和lambda表达式时，必须注意避免突变，以免将来出现问题，如果以后从并行流中运行代码。

每个构造

的for-each构造允许您使用来简洁地遍历集合或数组for循环-请参见for语句 。以下代码使用for-each构造以在单独的行上打印出集合的每个元素。

for (Object o : collection)
    System.out.println(o);

迭代器

一个Iterator是一个对象，使您可以遍历集合并根据需要有选择地从集合中删除元素。你得到一个Iterator通过调用其集合iterator方法。以下是Iterator接口。

public interface Iterator<E> {
    boolean hasNext();
    E next();
    void remove(); //optional
}

的hasNext方法返回true如果迭代中包含更多元素，并且next方法返回迭代中的下一个元素。的remove方法删除由返回的最后一个元素next从底层Collection 。的remove每次调用只能调用一次next并在违反此规则时引发异常。

注意Iterator.remove是在迭代过程中修改集合的唯一安全方法；如果在迭代进行过程中以任何其他方式修改了基础集合，则行为不确定。

使用Iterator而不是for-each需要时构造：

  • 删除当前元素。的for-each构造隐藏了迭代器，因此您无法调用remove 。因此， for-each构造不适用于过滤。
  • 并行迭代多个集合。

以下方法向您展示了如何使用Iterator过滤任意Collection —即遍历集合，删除特定元素。

static void filter(Collection<?> c) {
    for (Iterator<?> it = c.iterator(); it.hasNext(); )
        if (!cond(it.next()))
            it.remove();
}

这个简单的代码是多态的，这意味着它适用于任何 Collection不管实施。该示例说明了使用Java Collections Framework编写多态算法有多么容易。

集合接口批量操作

批量操作对整个操作执行Collection 。您可以使用基本操作来实现这些速记操作，尽管在大多数情况下，此类实现的效率较低。以下是批量操作：

  • containsAll —退货true如果目标Collection包含指定中的所有元素Collection 。
  • addAll -添加指定元素中的所有元素Collection到目标Collection 。
  • removeAll -从目标中删除Collection所有包含在指定元素中的元素Collection 。
  • retainAll -从目标中删除Collection所有未包含在指定元素中的元素Collection 。也就是说，它仅保留目标中的那些元素Collection也包含在指定的Collection 。
  • clear -从中删除所有元素Collection 。

的addAll ， removeAll和retainAll方法全部返回true如果目标Collection在执行操作的过程中被修改。

作为批量操作功能的简单示例，请考虑以下惯用法来删除指定元素的所有实例， e ，来自Collection ， c 。

c.removeAll(Collections.singleton(e));

更具体地说，假设您要删除所有null来自一个元素Collection 。

c.removeAll(Collections.singleton(null));

这个习惯用法Collections.singleton ，这是一个静态工厂方法，它返回不可变的Set仅包含指定的元素。

收集接口阵列操作

的toArray提供方法作为在期望输入数组上的集合和旧API之间的桥梁。数组操作允许Collection被翻译成一个数组。没有参数的简单形式创建一个新的数组Object 。更为复杂的形式允许调用者提供一个数组或选择输出数组的运行时类型。

例如，假设c是一个Collection 。以下代码段转储了以下内容c放入新分配的Object其长度等于中的元素数c 。

Object[] a = c.toArray();

假设c已知仅包含字符串（也许是因为c是类型Collection ）。以下代码段转储了以下内容c放入新分配的String其长度等于中的元素数c 。

String[] a = c.toArray(new String[0]);
