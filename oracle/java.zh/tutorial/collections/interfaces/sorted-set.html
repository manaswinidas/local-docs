SortedSet接口

一种SortedSet是一个Set保持其元素升序排列，根据元素的自然顺序或根据Comparator提供于SortedSet创建时间。除了正常Set操作， SortedSet界面提供以下操作：

  • Range view —允许对排序集进行任意范围的操作
  • Endpoints —返回排序集中的第一个或最后一个元素
  • Comparator access —返回Comparator （如果有）用于对集合进行排序

的代码SortedSet界面如下。

public interface SortedSet<E> extends Set<E> {
    // Range-view
    SortedSet<E> subSet(E fromElement, E toElement);
    SortedSet<E> headSet(E toElement);
    SortedSet<E> tailSet(E fromElement);

    // Endpoints
    E first();
    E last();

    // Comparator access
    Comparator<? super E> comparator();
}

设定作业

的操作SortedSet继承自Set在排序集和普通集上的行为相同，但有两个例外：

  • 的Iterator由...返回iterator操作按顺序遍历排序后的集合。
  • 传回的阵列toArray按顺序包含排序后的集合的元素。

虽然介面无法保证，但toString Java平台的方法SortedSet实现按顺序返回包含排序集中的所有元素的字符串。

标准构造

按照惯例，所有通用Collection实现提供了一个采用Collection ; SortedSet实现也不例外。在TreeSet ，此构造方法创建一个实例，该实例根据其元素的自然顺序对其进行排序。这可能是一个错误。最好动态检查以查看指定的集合是否是一个SortedSet实例，如果是，则对新实例进行排序TreeSet根据相同的标准（比较器或自然排序）。因为TreeSet采用了它所采用的方法，还提供了一个构造函数SortedSet并返回一个新的TreeSet包含根据相同标准排序的相同元素。请注意，确定调用这两个构造函数中的哪一个（以及是否保留排序标准）的是参数的编译时类型，而不是其运行时类型。

SortedSet根据惯例，实现还提供了一个Comparator并返回一个空集，按照指定的顺序排序Comparator 。如果null传递给此构造函数，它返回一个集合，该集合根据元素的自然顺序对其进行排序。

范围视图操作

的range-view操作有点类似于由List界面，但有很大的不同。即使直接修改了支持排序的集合，排序的集合的范围视图仍然有效。这是可行的，因为排序集的范围视图的端点是元素空间中的绝对点，而不是后备集合中的特定元素（如列表）。一种range-view排序后的集合实际上只是一个窗口，该窗口指向集合中任何位于元素空间指定部分中的部分。的变化range-view写回支持排序的集，反之亦然。因此，可以使用range-view长时间在排序的集合上，与range-view在清单上。

排序集提供三个range-view操作。首先， subSet ，有两个端点，例如subList 。端点是对象，而不是索引，它们必须与排序集中的元素相当，使用Set的Comparator或其元素的自然顺序，无论哪个Set用于订购本身。喜欢subList ，范围是半开的，包括低端，但不包括高端。

因此，以下代码行告诉您之间有多少个单词"doorbell"和"pickle" ， 包含"doorbell"但不包括"pickle" ，包含在SortedSet的字符串dictionary ：

int count = dictionary.subSet("doorbell", "pickle").size();

以类似的方式，下面的一线删除所有以字母开头的元素f 。

dictionary.subSet("f", "g").clear();

可以使用类似的技巧来打印表格，以告诉您每个字母以多少个单词开头。

for (char ch = 'a'; ch <= 'z'; ) {
    String from = String.valueOf(ch++);
    String to = String.valueOf(ch);
    System.out.println(from + ": " + dictionary.subSet(from, to).size());
}

假设您要查看一个包含两个端点的封闭时间间隔 ，而不是一个开放时间间隔。如果元素类型允许计算元素空间中给定值的后继元素，则只需请求subSet从lowEndpoint至successor(highEndpoint) 。虽然不是很明显，但是字符串的后继者s在String的自然顺序是s + "\0" - 那是， s与一个null附加字符。

因此，下面的一句告诉你"doorbell"和"pickle"包括门铃和泡菜，都包含在词典中。

count = dictionary.subSet("doorbell", "pickle\0").size();

可以使用类似的技术来查看一个开放时间间隔 ，该时间间隔既不包含端点，也不包含端点。的开放间隔视图lowEndpoint至highEndpoint是从successor(lowEndpoint)至highEndpoint 。使用以下内容计算之间的字数"doorbell"和"pickle" ，排除两者。

count = dictionary.subSet("doorbell\0", "pickle").size();

的SortedSet界面还包含两个range-view操作- headSet和tailSet ，两者都需要一个Object论点。前者返回支持的初始部分的视图SortedSet ，最多但不包括指定的对象。后者返回支持的最后部分的视图SortedSet ，从指定的对象开始，一直到支持的结尾SortedSet 。因此，以下代码允许您将字典视为两个不相交的字典volumes （ a-m和n-z ）。

SortedSet<String> volume1 = dictionary.headSet("n");
SortedSet<String> volume2 = dictionary.tailSet("n");

端点操作

的SortedSet接口包含返回排序集中的第一个和最后一个元素的操作，不足为奇first和last 。除了明显的用途外， last允许解决以下缺陷SortedSet接口。您想做的一件事SortedSet是要进入内部Set并向前或向后迭代。从内部前进很容易：只需获得一个tailSet并遍历它。不幸的是，没有简单的方法可以退后。

以下成语获取小于指定对象的第一个元素o在元素空间中。

Object predecessor = ss.headSet(o).last();

这是从已排序集合内部的某个点向后退一个元素的好方法。可以重复使用它来向后迭代，但这效率很低，需要对返回的每个元素进行查找。

比较器访问器

的SortedSet接口包含一个称为的访问器方法comparator返回Comparator用于对集合进行排序，或者null如果该集合是根据其元素的自然顺序排序的。提供此方法是为了使排序后的集合可以以相同的顺序复制到新的排序后的集合中。它由SortedSet构造函数如前所述 。
