Java教程是为JDK 8编写的。此页面中描述的示例和实践没有利用更高版本中引入的改进。

对问题和练习的答案：

问题

  1. 问题：在本课开始时，您了解了核心集合接口被组织为两个不同的继承树。特别是一个接口不被认为是真实的Collection ，因此位于其自己的树的顶部。该接口的名称是什么？
    回答：Map

  2. 问题：集合框架中的每个接口都使用语法，告诉您它是通用的。当您声明一个Collection实例，指定将包含的对象类型有什么好处？
    答：指定类型可以使编译器在编译时验证您放入集合中的对象类型是否正确，从而减少了运行时的错误。

  3. 问题：哪个接口表示不允许重复元素的集合？
    回答： Set

  4. 问题：什么接口构成集合层次结构的根？
    回答： Collection

  5. 问题：哪个接口表示可能包含重复元素的有序集合？
    回答： List

  6. 问题：哪个接口代表在处理之前保存元素的集合？
    回答： Queue

  7. 问题：哪个接口代表将键映射到值的类型？
    回答： Map

  8. 问题：哪个接口代表双端队列？
    回答： Deque

  9. 问题：列举三种不同的方法来迭代a的元素List 。
    答：您可以遍历一个List使用流，增强for语句或迭代器。

  10. 问题：对或错：聚合操作是修改基础集合的变异操作。
    答：错。聚合操作不会使基础集合发生变异。实际上，在调用集合的聚合操作时，您必须小心不要使它变异。如果将来在某个时候将流更改为并行流，这样做可能会导致并发问题。

练习题

  1. 练习：编写一个程序，以随机顺序打印其参数。不要复制参数数组。演示如何使用流和传统的for语句增强版来打印元素。
    回答：
    
import java.util.*;

public class Ran {

    public static void main(String[] args) {
        
        // Get and shuffle the list of arguments
        List<String> argList = Arrays.asList(args);
        Collections.shuffle(argList);

        // Print out the elements using JDK 8 Streams
        argList.stream()
        .forEach(e->System.out.format("%s ",e));

        // Print out the elements using for-each
        for (String arg: argList) {
            System.out.format("%s ", arg);
        }

        System.out.println();
    }
}
  2. 练习： FindDups实例并对其进行修改以使用SortedSet代替Set 。指定一个Comparator因此在对集合元素进行排序和标识时将忽略这种情况。
    回答：
    import java.util.*;

public class FindDups {
    public static void main(String[] args) {
        Set<String> s = new HashSet<String>();
        for (String a : args)
               s.add(a);
               System.out.println(s.size() + " distinct words: " + s);
    }
}
  3. 练习：编写一个需要List并适用String.trim每个元素。
    回答：
    增强型for声明不允许您修改List 。使用实例Iterator类允许您删除元素，但不能替换现有元素或添加新元素。那离开ListIterator ：
    import java.util.*;

public class ListTrim {
    static void listTrim(List<String> strings) {
        for (ListIterator<String> lit = strings.listIterator(); lit.hasNext(); ) {
            lit.set(lit.next().trim());
        }
    }

    public static void main(String[] args) {
        List<String> l = Arrays.asList(" red ", " white ", " blue ");
        listTrim(l);
        for (String s : l) {
            System.out.format("\"%s\"%n", s);
        }
    }
}
  4. 练习：考虑四个核心接口， Set ， List ， Queue和Map 。对于以下四个分配中的每个分配，指定四个核心接口中的哪个最适合，并说明如何使用它来实现分配。
    答案：
    • Whimsical Toys Inc（WTI）需要记录其所有雇员的姓名。每个月，将从这些记录中随机选择一名员工，以获取免费玩具。
      用一个List 。通过选择一个数字来选择一个随机的员工0和size()-1 。
    • WTI决定将每个新产品都命名为一个员工，但只会使用名字，每个名称只能使用一次。准备唯一的名字列表。
      用一个Set 。实现此接口的集合不允许同一元素多次输入。
    • WTI决定只希望使用最受欢迎的玩具名称。计算每个名字的雇员人数。
      用一个Map ，其中键是名字，并且每个值都是具有该名字的员工人数的计数。
    • WTI为当地的长曲棍球队获取季票，由员工共享。为此受欢迎的运动创建一个等待列表。
      用一个Queue 。调用add()将员工添加到候补名单中，以及remove()删除它们。
