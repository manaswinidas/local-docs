列表界面

一种List是命令Collection （有时称为序列 ）。列表可能包含重复的元素。除了从继承的操作Collection ， List界面包含以下操作：

  • Positional access —根据元素在列表中的数字位置操作元素。这包括诸如get ， set ， add ， addAll和remove 。
  • Search —在列表中搜索指定的对象并返回其数字位置。搜索方法包括indexOf和lastIndexOf 。
  • Iteration —延伸Iterator语义以利用列表的顺序性质。的listIterator方法提供了这种行为。
  • Range-view — sublist方法对列表执行任意范围的操作 。

Java平台包含两个通用List实现。 ArrayList ，通常是效果更好的实现，并且LinkedList在某些情况下可以提供更好的性能。

收款业务

从继承的操作Collection假设您已经熟悉他们，那么所有人都会做您期望他们做的事情。如果您不熟悉Collection ，现在是阅读“收集接口”部分的好时机。的remove操作始终从列表中删除第一次出现的指定元素。的add和addAll操作总是将新元素追加到列表的末尾 。因此，以下成语将一个列表连接到另一个列表。

list1.addAll(list2);

这是这个习语的非破坏性形式，会产生第三种List由第二个列表组成，该列表附加到第一个之后。

List<Type> list3 = new ArrayList<Type>(list1);
list3.addAll(list2);

请注意，成语以其非破坏性形式利用了ArrayList的标准转换构造函数。

这是一个示例（JDK 8及更高版本），将一些名称聚合为一个List ：

List<String> list = people.stream()
.map(Person::getName)
.collect(Collectors.toList());

如Set接口， List加强了对equals和hashCode方法使两个List可以比较对象的逻辑相等性，而无需考虑其实现类。二List如果对象包含相同顺序的相同元素，则对象相等。

位置访问和搜索操作

基础的positional access操作是get ， set ， add和remove 。（ set和remove操作将返回正在被覆盖或删除的旧值。）其他操作（ indexOf和lastIndexOf ）返回列表中指定元素的第一个或最后一个索引。

的addAll操作插入指定的所有元素Collection从指定位置开始。元素按指定元素返回的顺序插入Collection的迭代器。此调用是的位置访问类似物Collection的addAll操作。

这是一种交换两个索引值的小方法List 。

public static <E> void swap(List<E> a, int i, int j) {
    E tmp = a.get(i);
    a.set(i, a.get(j));
    a.set(j, tmp);
}

当然，有很大的不同。 这是一个多态算法：它在任意位置交换两个元素List ，无论其实现类型如何。这是另一个使用前面的多态算法swap方法。

public static void shuffle(List<?> list, Random rnd) {
    for (int i = list.size(); i > 1; i--)
        swap(list, i - 1, rnd.nextInt(i));
}

此算法包含在Java平台的Collections类，使用指定的随机性源随机排列指定的列表。这有点微妙：它从底部开始运行列表，反复将随机选择的元素交换到当前位置。与大多数天真的改组尝试不同，它是公平的 （假设无偏的随机性来源，所有排列均以相同的可能性发生）且速度快 （要求精确list.size()-1掉期）。以下程序使用此算法以随机顺序在其参数列表中打印单词。

import java.util.*;

public class Shuffle {
    public static void main(String[] args) {
        List<String> list = new ArrayList<String>();
        for (String a : args)
            list.add(a);
        Collections.shuffle(list, new Random());
        System.out.println(list);
    }
}

实际上，可以使该程序更短，更快。的Arrays类有一个称为的静态工厂方法asList ，它允许将数组视为List 。此方法不复制数组。的变化List写入数组，反之亦然。结果列表不是通用的List实现，因为它没有实现（可选） add和remove操作：数组不可调整大小。占（某人）的便宜Arrays.asList并调用库的版本shuffle ，它使用默认的随机性来源，您将获得以下内容tiny program其行为与以前的程序相同。

import java.util.*;

public class Shuffle {
    public static void main(String[] args) {
        List<String> list = Arrays.asList(args);
        Collections.shuffle(list);
        System.out.println(list);
    }
}

迭代器

如您所料， Iterator由...返回List的iterator操作以正确的顺序返回列表的元素。 List还提供了更丰富的迭代器，称为ListIterator ，它允许您沿任一方向遍历列表，在迭代过程中修改列表并获得迭代器的当前位置。

三种方法ListIterator继承自Iterator （ hasNext ， next和remove ）在两个接口中做完全相同的事情。的hasPrevious和previous操作与...完全相同hasNext和next 。前一个操作引用了（隐式）光标之前的元素，而后一个操作引用了光标之后的元素。的previous操作将光标向后移动，而next向前移动。

这是用于向后遍历列表的标准用法。

for (ListIterator<Type> it = list.listIterator(list.size()); it.hasPrevious(); ) {
    Type t = it.previous();
    ...
}

注意参数listIterator在前面的习语中。的List界面有两种形式listIterator方法。不带参数的表单返回一个ListIterator位于列表的开头；带有int参数返回一个ListIterator定位在指定的索引处。索引是指最初调用会返回的元素next 。最初致电previous将返回其索引为index-1 。在长度清单中n ， 有n+1的有效值index ，来自0至n ， 包括的。

直观地讲，游标始终位于两个元素之间，即通过调用以下元素将返回的元素previous以及通过致电返回的那个next 。的n+1有效index值对应于n+1元素之间的间隙，从第一个元素之前的间隙到最后一个元素之后的间隙。 下图显示了包含四个元素的列表中五个可能的光标位置。

致电next和previous可以混合使用，但您必须小心一点。首次致电previous返回与上次调用相同的元素next 。同样，第一个调用next经过一系列的呼叫previous返回与上次调用相同的元素previous 。

毫不奇怪， nextIndex方法返回元素的索引，该元素的索引将由后续对的调用返回next和previousIndex返回元素的索引，该元素的索引将由随后的调用返回previous 。这些调用通常用于报告发现某物的位置或记录其位置。 ListIterator这样另一个ListIterator可以创建具有相同位置的广告。

也不奇怪，由...返回的数字nextIndex总是比返回的数字大一previousIndex 。这暗示了两种边界情况的行为：（1）调用previousIndex当光标在初始元素返回之前-1和（2） nextIndex当光标在最后一个元素返回之后list.size() 。为了使所有这些具体，下面是一个可能的实现List.indexOf 。

public int indexOf(E e) {
    for (ListIterator<E> it = listIterator(); it.hasNext(); )
        if (e == null ? it.next() == null : e.equals(it.next()))
            return it.previousIndex();
    // Element not found
    return -1;
}

请注意indexOf方法返回it.previousIndex()即使它正向遍历列表。原因是it.nextIndex()将返回我们将要检查的元素的索引，并且我们想返回我们刚刚检查的元素的索引。

的Iterator介面提供remove删除由返回的最后一个元素的操作next来自Collection 。对于ListIterator ，此操作将删除由返回的最后一个元素next要么previous 。的ListIterator界面提供了两个其他操作来修改列表- set和add 。的set方法覆盖由返回的最后一个元素next要么previous与指定的元素。以下多态算法使用set用一个指定值替换所有出现的值。

public static <E> void replace(List<E> list, E val, E newVal) {
    for (ListIterator<E> it = list.listIterator(); it.hasNext(); )
        if (val == null ? it.next() == null : val.equals(it.next()))
            it.set(newVal);
}

本示例中唯一棘手的问题是val和it.next 。您需要特殊情况val的价值null防止NullPointerException 。

的add方法在当前光标位置之前将一个新元素插入列表。在下面的多态算法中说明了此方法，以用指定列表中包含的值序列替换所有出现的指定值。

public static <E> 
    void replace(List<E> list, E val, List<? extends E> newVals) {
    for (ListIterator<E> it = list.listIterator(); it.hasNext(); ){
        if (val == null ? it.next() == null : val.equals(it.next())) {
            it.remove();
            for (E e : newVals)
                it.add(e);
        }
    }
}

范围查看操作

的range-view操作subList(int fromIndex, int toIndex) ，返回一个List此列表中索引范围为fromIndex （包括） toIndex ，独家。这个半开范围反映了典型的for环。

for (int i = fromIndex; i < toIndex; i++) {
    ...
}

由于长期的观点意味着，返回List由备份List在哪subList之所以称为前者，是因为前者的变化反映在后者中。

此方法消除了对显式范围操作（数组通常存在的那种范围）的需要。任何期望List可以通过传递一个用作范围操作subList查看而不是整体List 。例如，以下成语从List 。

list.subList(fromIndex, toIndex).clear();

可以构造类似的惯用法来搜索范围内的元素。

int i = list.subList(fromIndex, toIndex).indexOf(o);
int j = list.subList(fromIndex, toIndex).lastIndexOf(o);

请注意，前面的惯用语会返回找到的元素在subList ，而不是后备索引List 。

任何可在List ， 如那个replace和shuffle示例，与List由...返回subList 。

这是一个多态算法，其实现使用subList从甲板上伸出一只手。也就是说，它返回一个新的List （“手”）包含从指定末尾取出的指定数量的元素List （“平台”）。手中返回的元素将从牌组中删除。

public static <E> List<E> dealHand(List<E> deck, int n) {
    int deckSize = deck.size();
    List<E> handView = deck.subList(deckSize - n, deckSize);
    List<E> hand = new ArrayList<E>(handView);
    handView.clear();
    return hand;
}

请注意，此算法从牌组末端移开了手。对于许多常见List实现，例如ArrayList ，从列表末尾删除元素的性能明显好于从开头删除元素的性能。

以下是a program使用dealHand结合方法Collections.shuffle从普通的52张牌中产生手牌。该程序采用两个命令行参数：（1）发牌数和（2）每手牌数。

import java.util.*;

public class Deal {
    public static void main(String[] args) {
        if (args.length < 2) {
            System.out.println("Usage: Deal hands cards");
            return;
        }
        int numHands = Integer.parseInt(args[0]);
        int cardsPerHand = Integer.parseInt(args[1]);
    
        // Make a normal 52-card deck.
        String[] suit = new String[] {
            "spades", "hearts", 
            "diamonds", "clubs" 
        };
        String[] rank = new String[] {
            "ace", "2", "3", "4",
            "5", "6", "7", "8", "9", "10", 
            "jack", "queen", "king" 
        };

        List<String> deck = new ArrayList<String>();
        for (int i = 0; i < suit.length; i++)
            for (int j = 0; j < rank.length; j++)
                deck.add(rank[j] + " of " + suit[i]);
    
        // Shuffle the deck.
        Collections.shuffle(deck);
    
        if (numHands * cardsPerHand > deck.size()) {
            System.out.println("Not enough cards.");
            return;
        }
    
        for (int i = 0; i < numHands; i++)
            System.out.println(dealHand(deck, cardsPerHand));
    }
  
    public static <E> List<E> dealHand(List<E> deck, int n) {
        int deckSize = deck.size();
        List<E> handView = deck.subList(deckSize - n, deckSize);
        List<E> hand = new ArrayList<E>(handView);
        handView.clear();
        return hand;
    }
}

虽然subList操作非常强大，使用时必须格外小心。的语义List由...返回subList如果将元素添加到支持或从支持中删除，则变得不确定List除了通过返回以外的任何方式List 。因此，强烈建议您使用List由...返回subList仅作为瞬态对象-在背衬上执行一个或一系列范围操作List 。您使用的时间越长subList实例，您通过修改支持而损害它的可能性就越大List直接或通过另一个subList宾语。请注意，修改子列表的子列表并继续使用原始子列表是合法的（尽管不能同时使用）。

列表算法

大部分的多态算法Collections该课程专门适用于List 。拥有所有这些算法可轻松处理列表。这是这些算法的摘要，将在“ 算法”部分中详细介绍。

  • sort —排序一个List使用合并排序算法，该算法可提供快速，稳定的排序。（ 稳定排序是一种不会对相等元素进行重新排序的排序。）
  • shuffle —随机排列a中的元素List 。
  • reverse —颠倒元素中元素的顺序List 。
  • rotate —旋转List按指定的距离。
  • swap —交换元素中指定位置的元素List 。
  • replaceAll —用一个指定值替换所有出现的值。
  • fill —覆盖List具有指定值。
  • copy —复制源List到目的地List 。
  • binarySearch —搜索有序元素List使用二进制搜索算法。
  • indexOfSubList —返回一个的第一个子列表的索引List那等于另一个。
  • lastIndexOfSubList —返回一个的最后一个子列表的索引List那等于另一个。
