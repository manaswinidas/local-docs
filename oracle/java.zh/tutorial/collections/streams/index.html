课程：综合运营

注意 ：为了更好地理解本节中的概念，请查阅Lambda表达式和方法参考一节 。

您使用什么收藏？您不只是将对象存储在集合中并将它们保留在那里。在大多数情况下，您使用集合来检索存储在其中的项目。

再次考虑Lambda Expressions部分中描述的场景。假设您正在创建一个社交网络应用程序。您想创建一个功能，使管理员可以对满足特定条件的社交网络应用程序成员执行任何类型的操作，例如发送消息。

和以前一样，假设此社交网络应用程序的成员由以下内容表示Person类：

public class Person {

    public enum Sex {
        MALE, FEMALE
    }

    String name;
    LocalDate birthday;
    Sex gender;
    String emailAddress;
    
    // ...

    public int getAge() {
        // ...
    }

    public String getName() {
        // ...
    }
}

以下示例显示集合中包含的所有成员的名称roster与for-each循环：

for (Person p : roster) {
    System.out.println(p.getName());
}

下面的示例打印集合中包含的所有成员roster但是使用合计操作forEach ：

roster
    .stream()
    .forEach(e -> System.out.println(e.getName());

尽管在此示例中，使用聚合操作的版本比使用for-each循环的版本长，但是您将看到使用批量数据操作的版本对于更复杂的任务将更加简洁。

涵盖以下主题：

  • 管道和流
  • 集合操作和迭代器之间的差异

在示例中找到本节中描述的代码摘录BulkDataOperationsExamples 。

管道和流

管道是一系列聚合操作。下面的示例打印集合中包含的男性成员roster包含聚合操作的管道filter和forEach ：

roster
    .stream()
    .filter(e -> e.getGender() == Person.Sex.MALE)
    .forEach(e -> System.out.println(e.getName()));

将此示例与以下示例进行比较，该示例显示集合中包含的男性成员roster与for-each循环：

for (Person p : roster) {
    if (p.getGender() == Person.Sex.MALE) {
        System.out.println(p.getName());
    }
}

管道包含以下组件：

  • 源：可以是集合，数组，生成器函数或I / O通道。在此示例中，源是集合roster 。

  • 零次或多次中间操作 。中间操作，例如filter ，产生新的流。

    流是一系列元素。与集合不同，它不是存储元素的数据结构。相反，流通过管道从源携带值。本示例从集合创建流roster通过调用该方法stream 。

    的filter操作返回一个新流，该流包含与其谓词匹配的元素（此操作的参数）。在此示例中，谓词是lambda表达式e -> e.getGender() == Person.Sex.MALE 。它返回布尔值true如果gender对象领域e有价值Person.Sex.MALE 。因此， filter在此示例中，操作返回一个包含集合中所有男性成员的流roster 。

  • 终端操作 。终端操作，例如forEach会产生非流式结果，例如原始值（如double值），集合或forEach ，根本没有价值。在此示例中， forEach操作是lambda表达式e -> System.out.println(e.getName()) ，它调用方法getName在对象上e 。（Java运行时和编译器推断出对象的类型e是Person ）

以下示例计算集合中包含的所有男性成员的平均年龄roster包含聚合操作的管道filter ， mapToInt和average ：

double average = roster
    .stream()
    .filter(p -> p.getGender() == Person.Sex.MALE)
    .mapToInt(Person::getAge)
    .average()
    .getAsDouble();

的mapToInt操作返回一个新的类型流IntStream （这是仅包含整数值的流）。该操作将其参数中指定的功能应用于特定流中的每个元素。在这个例子中，函数是Person::getAge ，这是一个方法参考，返回成员的年龄。（或者，您可以使用lambda表达式e -> e.getAge() ）因此， mapToInt在此示例中，操作返回一个流，其中包含集合中所有男性成员的年龄roster 。

的average操作计算类型流中包含的元素的平均值IntStream 。它返回一个类型的对象OptionalDouble 。如果流中不包含任何元素，则average操作返回的空实例OptionalDouble ，并调用该方法getAsDouble抛出一个NoSuchElementException 。JDK包含许多终端操作，例如average通过组合流的内容返回一个值。这些操作称为归约操作 ;有关更多信息，请参见减少部分。

集合操作和迭代器之间的差异

汇总操作，例如forEach ，看起来像是迭代器。但是，它们有几个基本差异：

  • 他们使用内部迭代 ：聚合操作不包含类似next指示他们处理集合的下一个元素。随着内部委派 ，您的应用决定了它遍历集合了什么 ，但是JDK决定如何遍历集合。通过外部迭代 ，您的应用程序可以确定迭代哪个集合以及如何迭代它。但是，外部迭代只能顺序地迭代集合的元素。内部迭代没有此限制。它可以更轻松地利用并行计算的优势，并行计算包括将问题分解为子问题，同时解决这些问题，然后将解决方案的结果组合到子问题中。有关更多信息，请参见并行性一节。

  • 它们处理流中的元素 ：聚合操作从流中而不是直接从集合中处理元素。因此，它们也称为流操作 。

  • 它们支持将行为作为参数 ：您可以将lambda表达式指定为大多数聚合操作的参数。这使您可以自定义特定聚合操作的行为。
