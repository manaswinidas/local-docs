减少

“ 汇总操作 ”部分描述了以下操作流程，该流程计算了集合中所有男性成员的平均年龄roster ：

double average = roster
    .stream()
    .filter(p -> p.getGender() == Person.Sex.MALE)
    .mapToInt(Person::getAge)
    .average()
    .getAsDouble();

JDK包含许多终端操作（例如average ， sum ， min ， max和count ），通过组合流的内容返回一个值。这些操作称为归约操作 。JDK还包含归约操作，这些操作返回集合而不是单个值。许多归约运算执行特定任务，例如查找值的平均值或将元素分组为类别。但是，JDK为您提供了通用的归约操作reduce和collect ，本节将详细介绍。

本节涵盖以下主题：

  • 流减少方法
  • Stream.collect方法

您可以在示例中找到本节中描述的代码摘录ReductionExamples 。

流减少方法

的Stream.reduce该方法是通用的归约运算。考虑以下管道，该管道计算集合中男性成员的年龄总和roster 。它使用Stream.sum还原操作：

Integer totalAge = roster
    .stream()
    .mapToInt(Person::getAge)
    .sum();

将此与以下管道进行比较，该管道使用Stream.reduce计算相同值的运算：

Integer totalAgeReduce = roster
   .stream()
   .map(Person::getAge)
   .reduce(
       0,
       (a, b) -> a + b);

的reduce此示例中的operation具有两个参数：

  • identity ：如果流中没有元素，则identity元素既是reduce的初始值，也是默认结果。在此示例中，标识元素为0 ;这是年龄总和的初始值，如果集合中不存在成员，则为默认值roster 。

  • accumulator ：累加器函数具有两个参数：约简的部分结果（在此示例中，是到目前为止所有已处理整数的总和）和流的下一个元素（在此示例中，是整数）。它返回一个新的部分结果。在此示例中，accumulator函数是一个lambda表达式，该表达式将两个Integer值并返回一个Integer值：

    (a, b) -> a + b

的reduce操作始终返回新值。但是，累加器函数每次处理流元素时也会返回一个新值。假设您想将流的元素简化为一个更复杂的对象，例如集合。这可能会妨碍您的应用程序的性能。如果你的reduce操作涉及将元素添加到集合中，然后每次累加器函数处理一个元素时，它都会创建一个包含该元素的新集合，这效率很低。取而代之，更新现有集合会更有效。您可以使用Stream.collect方法，下一节将介绍。

Stream.collect方法

不像reduce方法，该方法在处理元素时始终会创建一个新值， collect方法修改或变异现有值。

考虑如何查找流中的平均值。您需要两个数据：值的总数和这些值的总和。但是，就像reduce方法和所有其他还原方法， collect方法仅返回一个值。您可以创建一个包含成员变量的新数据类型，该成员变量跟踪值的总数和这些值的总和，例如以下类， Averager ：

class Averager implements IntConsumer
{
    private int total = 0;
    private int count = 0;
        
    public double average() {
        return count > 0 ? ((double) total)/count : 0;
    }
        
    public void accept(int i) { total += i; count++; }
    public void combine(Averager other) {
        total += other.total;
        count += other.count;
    }
}

以下管道使用Averager类和collect计算所有男性成员平均年龄的方法：

Averager averageCollect = roster.stream()
    .filter(p -> p.getGender() == Person.Sex.MALE)
    .map(Person::getAge)
    .collect(Averager::new, Averager::accept, Averager::combine);
                   
System.out.println("Average age of male members: " +
    averageCollect.average());

的collect此示例中的operation使用三个参数：

  • supplier ：供应商具有工厂职能；它构造了新的实例。为了collect操作，它将创建结果容器的实例。在此示例中，它是Averager类。
  • accumulator ：累加器功能将流元素合并到结果容器中。在此示例中，它修改了Averager通过增加结果容器count变量加一并加到total member variable成员变量stream元素的值，它是代表男性成员年龄的整数。
  • combiner ：组合器函数接受两个结果容器并合并其内容。在此示例中，它修改了Averager通过增加结果容器count由变量count另一个的成员变量Averager实例并添加到total成员变量另一个的值Averager实例的total成员变量。

请注意以下几点：

  • 供应商是一个lambda表达式（或方法引用），而不是像reduce操作。
  • 累加器和合并器函数不返回值。
  • 您可以使用collect并行流操作；有关更多信息，请参见并行性一节。（如果您运行collect并行流的方法，那么只要合并器函数创建新对象（例如Averager本示例中的对象。因此，您不必担心同步。）

尽管JDK为您提供了average用于计算流中元素平均值的操作，您可以使用collect操作和自定义类（如果您需要从流的元素中计算多个值）。

的collect操作最适合收藏。以下示例将男性成员的姓名与collect操作：

List<String> namesOfMaleMembersCollect = roster
    .stream()
    .filter(p -> p.getGender() == Person.Sex.MALE)
    .map(p -> p.getName())
    .collect(Collectors.toList());

这个版本的collect操作采用一种类型的参数Collector 。此类封装了用作参数的函数collect需要三个参数（供应商，累加器和合并器函数）的运算。

的Collectors类包含许多有用的归约运算，例如将元素累积到集合中并根据各种标准对元素进行汇总。这些归约运算返回该类的实例Collector ，因此您可以将它们用作collect操作。

本示例使用Collectors.toList操作，将流元素累积到的新实例中List 。与大多数操作一样Collectors类， toList运算子传回的实例Collector ，而不是集合。

以下示例将集合的成员分组roster按性别：

Map<Person.Sex, List<Person>> byGender =
    roster
        .stream()
        .collect(
            Collectors.groupingBy(Person::getGender));

的groupingBy操作返回一个映射，该映射的键是应用指定为其参数的lambda表达式（称为分类函数 ）得出的值。在此示例中，返回的地图包含两个键， Person.Sex.MALE和Person.Sex.FEMALE 。键的对应值是List包含流元素的流元素，这些流元素在由分类功能处理时对应于键值。例如，对应于键的值Person.Sex.MALE是...的实例List包含所有男性成员。

以下示例检索集合中每个成员的名称roster并按性别将其分组：

Map<Person.Sex, List<String>> namesByGender =
    roster
        .stream()
        .collect(
            Collectors.groupingBy(
                Person::getGender,                      
                Collectors.mapping(
                    Person::getName,
                    Collectors.toList())));

的groupingBy在此示例中，运算具有两个参数，一个分类函数和一个实例Collector 。的Collector参数称为下游收集器 。这是一个收集器，Java运行时将其应用于另一个收集器的结果。因此，这groupingBy操作使您可以应用collect方法List由创建的值groupingBy操作员。此示例适用于收集器mapping ，它应用了映射功能Person::getName流中的每个元素。因此，结果流仅包含成员名称。像此示例一样，包含一个或多个下游收集器的管道称为多级归约 。

以下示例检索每个性别成员的总年龄：

Map<Person.Sex, Integer> totalAgeByGender =
    roster
        .stream()
        .collect(
            Collectors.groupingBy(
                Person::getGender,                      
                Collectors.reducing(
                    0,
                    Person::getAge,
                    Integer::sum)));

的reducing操作采用三个参数：

  • identity ： 如Stream.reduce操作中，如果流中没有任何元素，则identity元素既是reduce的初始值，也是默认结果。在此示例中，标识元素为0 ;这是年龄总和的初始值，如果不存在成员，则为默认值。
  • mapper ： reducing操作将此映射器函数应用于所有流元素。在此示例中，映射器检索每个成员的年龄。
  • operation ：操作功能用于减少映射值。在此示例中，操作功能添加Integer价值观。

以下示例检索每种性别的成员的平均年龄：

Map<Person.Sex, Double> averageAgeByGender = roster
    .stream()
    .collect(
        Collectors.groupingBy(
            Person::getGender,                      
            Collectors.averagingInt(Person::getAge)));
