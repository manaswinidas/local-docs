如何使用模型

大多数Swing组件都有模型。一个按钮 （ JButton ），例如，有一个模型（ ButtonModel对象），用于存储按钮的状态-它的键盘助记符是什么，是否已启用，选中或按下等等。某些组件具有多个模型。一个列表 （ JList ），例如，使用ListModel保存列表的内容，以及ListSelectionModel跟踪列表的当前选择。

您通常不需要了解组件使用的模型。例如，使用按钮的程序通常直接处理JButton对象，根本不处理ButtonModel宾语。

为什么会存在模型？最大的原因是它们为您提供了确定数据存储和检索方式的灵活性。例如，如果您正在设计一个电子表格应用程序，该应用程序在一个稀疏填充的表中显示数据，则可以创建自己的表模型，并为此进行优化。

模型也有其他好处。它们意味着数据不会在程序的数据结构和Swing组件的数据结构之间复制。此外，模型会自动将更改传播到所有感兴趣的侦听器，从而使GUI易于与数据保持同步。例如，要将项目添加到列表中，可以调用列表模型上的方法。当模型的数据更改时，模型会将事件触发到JList和其他任何注册的侦听器，并且相应地更新了GUI。

尽管Swing的模型体系结构有时被称为模型视图控制器（MVC）设计，但实际上并非如此。Swing组件通常是通过外观提供的单个UI对象实现的，因此视图和控制器是不可分割的。Swing模型体系结构更准确地描述为可分离的模型体系结构 。如果您想了解有关Swing模型体系结构的更多信息，请参阅《 Swing连接 概述 》中的Swing 体系结构概述 。

示例：转换器

本节以称为转换器的示例为例，该应用程序可以连续转换公制和美制单位之间的距离测量值。您可以运行Converter （ 下载JDK 7或更高版本 ）。或者，要自己编译并运行示例，请参考示例索引 。

如下图所示，Converter具有两个滑块，每个滑块都与一个文本字段相关联。滑块和文本字段都显示相同的数据-距离-但使用两个不同的度量单位。

对于此程序而言，重要的是确保只有一个模型可以控制数据的值。有多种方法可以实现此目的。我们通过遵循顶部滑块的模型来做到这一点。底部滑块的模型（自定义类的实例，称为FollowerRangeModel ）将所有数据查询转发到顶部滑块的模型（称为的自定义类的实例ConverterRangeModel ）。每个文本字段都通过侦听值更改的事件处理程序与其滑块保持同步，反之亦然。注意确保顶部滑块的模型对显示的距离拥有最终决定权。

当我们开始实现自定义滑块模型时，我们首先看了How to Use Sliders的API部分。它告知我们，所有滑块数据模型都必须实现BoundedRangeModel接口。的BoundedRangeModel API文档告诉我们该接口具有一个名为的实现类DefaultBoundedRangeModel 。的API文档DefaultBoundedRangeModel表明这是通用的实现BoundedRangeModel 。

我们没有使用DefaultBoundedRangeModel直接将其存储为整数，因为Converter使用浮点数据。因此，我们实施了ConverterRangeModel作为...的子类Object 。然后我们实施FollowerRangeModel作为...的子类ConverterRangeModel 。
