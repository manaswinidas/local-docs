如何使用分层窗格

分层窗格是一个Swing容器，它提供了第三个维度来定位组件： 深度 ，也称为Z order 。将组件添加到分层窗格时，可以将其深度指定为整数。数字越高，组件越接近容器内的“顶部”位置。如果组件重叠，则“较近”的组件将以较低的深度绘制在组件顶部。在相同深度处的组件之间的关系取决于它们在深度内的位置。

注意： 

AWT容器具有一个API，可让您操纵组件Z顺序 。有关更多信息，请参阅AWT Focus Specification 。

每个具有根窗格的 Swing容器-例如JFrame ， JApplet ， JDialog ， 要么JInternalFrame —自动具有分层的窗格。大多数程序没有显式使用根窗格的分层窗格，因此本节将不进行讨论。您可以在提供概述的The Root Pane和具有更多详细信息的The Layered Pane中找到有关它的信息。本节告诉您如何创建自己的分层窗格，以及如何在可以使用常规Swing容器的地方使用它。

Swing提供了两个分层的窗格类。首先， JLayeredPane ，是根窗格使用的类，并且是本节中的示例使用的类。第二， JDesktopPane ，是JLayeredPane专门用于保存内部框架的子类。有关使用示例JDesktopPane ，请参阅如何使用内部框架 。

这是来自的代码LayeredPaneDemo.java创建分层窗格：

layeredPane = new JLayeredPane();
layeredPane.setPreferredSize(new Dimension(300, 310));
layeredPane.setBorder(BorderFactory.createTitledBorder(
                                    "Move the Mouse to Move Duke"));
layeredPane.addMouseMotionListener(new MouseMotionAdapter() {
    ...
});

该代码使用JLayeredPane的唯一构造函数-无参数构造函数-创建分层窗格。其余代码使用从超类继承的方法为分层窗格提供首选的大小和边框，并为其添加鼠标移动侦听器。鼠标移动侦听器仅响应鼠标移动而移动Duke图像。尽管此处未显示代码，但是示例将分层窗格添加到框架的内容窗格中。

稍后我们将向您展示，您可以使用add方法。在将组件添加到分层窗格中时，您可以指定组件的深度，也可以指定其在其深度内的位置。演示程序中的分层窗格包含六个标签-五个彩色标签和第六个显示Duke图像的标签。如程序所示，组件的深度及其在该深度内的位置都可以动态更改。

添加组件并设置组件深度

这是示例程序中的代码，该代码将彩色标签添加到分层窗格中：

for (int i = 0; i < ...number of labels...; i++) {
    JLabel label = createColoredLabel(...);
    layeredPane.add(label, new Integer(i));
    ...
}

您可以找到实施createColoredLabel程序源代码中的方法。它只是创建一个不透明JLabel初始化为背景色，边框，一些文本和大小。

该示例程序使用了两个参数的add方法。第一个参数是要添加的组件，第二个参数是Integer对象，指定深度。该程序使用for循环迭代变量以指定深度。实际值无关紧要。重要的是深度的相对值，以及在程序中您如何使用每个深度是一致的。

注意： 

如果使用根窗格的分层窗格，请确保使用其深度约定。有关详细信息，请参见分层窗格 。该部分说明如何修改LayeredPaneDemo使用根窗格的分层窗格。通过修改，您可以看到拖动的Duke图像与控制面板中的组合框的关系。

从示例程序中可以看到，如果组件重叠，则深度较大的组件位于深度较小的组件的顶部。要动态更改组件深度，请使用setLayer方法。在该示例中，用户可以通过从组合框中进行选择来更改Duke的图层。这里是actionPerformed在组合框上注册的动作侦听器的方法：

public void actionPerformed(ActionEvent e) {
    int position = onTop.isSelected() ? 0 : 1;
    layeredPane.setLayer(dukeLabel,
                         layerList.getSelectedIndex(),
                         position);
}

的setLayer此处使用的method具有三个参数：要设置深度的组件，新深度以及该深度内的位置。 JLayeredPane有两个参数的版本setLayer只需要组件和新深度。该方法将组件置于其深度的底部位置。

注意事项： 

将组件添加到分层窗格时，请使用Integer 。使用时setLayer要更改组件的图层，请使用int 。您可能会认为，如果您使用int而不是Integer与add方法，编译器会抱怨或您的程序将抛出非法的参数异常。但是编译器什么也没说，这导致了常见的分层窗格问题 。您可以在本节末尾使用API表来检查参数的类型并返回处理图层的方法的值。

设置组件在其深度内的位置

以下代码创建显示Duke图像的标签，然后将标签添加到分层窗格中。

final ImageIcon icon = createImageIcon("images/dukeWaveRed.gif");
...
dukeLabel = new JLabel(icon);
...
dukeLabel.setBounds(15, 225,
                    icon.getIconWidth(),
                    icon.getIconHeight());
...
layeredPane.add(dukeLabel, new Integer(2), 0);

这段代码使用了三个参数的版本add方法。第三个参数指定Duke标签在其深度内的位置，该位置确定组件与相同深度处其他组件的关系。

位置用int在-1和（ n -1）之间，其中n是深度处的分量数。与图层编号不同，位置编号越小，组件在其深度内的位置越高。使用-1与使用n -1相同；它指示最底部的位置。使用0表示组件应位于其深度内的最高位置。如下图所示，除-1以外，位置编号越小表示深度内的位置越高。

组件在其层中的位置可以动态更改。在示例中，您可以使用复选框来确定Duke标签是否在其深度的顶部。这是actionPerformed在复选框上注册的动作侦听器的方法：

public void actionPerformed(ActionEvent e) {
    if (onTop.isSelected())
        layeredPane.moveToFront(dukeLabel);
    else
        layeredPane.moveToBack(dukeLabel);
}

当用户选中复选框时， moveToFront方法将Duke移动到前面（位置0）。并且当用户取消选择复选框时，Duke随即移至后面moveToBack方法。您也可以使用setPosition方法或三参数版本setLayer更改组件的位置。

在分层窗格中布置组件

默认情况下，分层窗格没有布局管理器。这意味着您通常必须编写代码来确定放置在分层窗格中的组件的位置和大小。

该示例使用setBounds设置每个标签的大小和位置的方法：

dukeLabel.setBounds(15, 225,
                    icon.getIconWidth(),
                    icon.getIconHeight());
...
label.setBounds(origin.x, origin.y, 140, 140);

当用户四处移动鼠标时，程序将调用setPosition更改杜克的职位：

dukeLabel.setLocation(e.getX()-XFUDGE, e.getY()-YFUDGE);

尽管默认情况下分层窗格没有布局管理器，但是您仍然可以将布局管理器分配给分层窗格。Java平台提供的所有布局管理器都将组件排列在一起，就好像它们都在一层上一样。这是先前演示的一个版本，该示例将分层窗格的布局管理器设置为的实例。 GridLayout ，使用该布局管理器来布局六个彩色标签。

您可以在以下位置找到该程序的代码LayeredPaneDemo2.java 。您可以运行LayeredPaneDemo2 （ 下载JDK 7或更高版本 ）。如果要编译示例，请查阅示例索引以获取所有必需文件的列表。

许多程序使用中间容器（例如面板）及其布局管理器在同一层上布置组件，但使用绝对定位在不同层上布置组件。有关绝对定位的更多信息，请参见在没有布局管理器的情况下进行操作（绝对定位） 。
