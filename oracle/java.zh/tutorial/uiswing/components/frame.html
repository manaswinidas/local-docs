如何制作框架（主窗口）

框架是带有标题和边框的顶级窗口。边框的大小包括为边框指定的任何区域。边界区域的尺寸可以使用getInsets方法。由于边框区域包括在框架的整体大小中，因此边框有效地遮盖了框架的一部分，将可用于渲染和/或显示子组件的区域限制为矩形的左上角位置为(insets.left ， insets.top) ，大小为width - (insets.left + insets.right)通过height - (insets.top + insets.bottom) 。

作为框架实例实现的框架JFrame class是具有装饰（例如边框，标题）并支持关闭或图标化窗口的按钮组件的窗口。具有GUI的应用程序通常至少包含一帧。小程序有时也使用框架。

要创建一个依赖于另一个窗口的窗口（例如，当另一个窗口被图标化时消失），请使用dialog代替frame. 。要使一个窗口出现在另一个窗口中，请使用内部框架 。

创建和显示框架

这是由FrameDemo演示应用程序。您可以在以下位置找到源代码FrameDemo.java 。您可以运行FrameDemo （ 下载JDK 7或更高版本 ）。

下列FrameDemo代码显示了如何创建和设置框架。

JFrame frame = new JFrame("FrameDemo");

//2. Optional: What happens when the frame closes?
frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);

//3. Create components and put them in the frame.
//...create emptyLabel...
frame.getContentPane().add(emptyLabel, BorderLayout.CENTER);

//4. Size the frame.
frame.pack();

//5. Show it.
frame.setVisible(true);

指定窗户装饰

默认情况下，窗口装饰由本机窗口系统提供。但是，您可以要求外观为框架提供装饰。您还可以指定框架完全没有窗户装饰，可以单独使用该功能，也可以提供自己的装饰，或者使用全屏独占模式 。

除了指定谁提供窗户装饰之外，您还可以指定使用哪个图标表示窗户。此图标的确切使用方式取决于窗口系统或提供窗口装饰的外观。如果窗口系统支持最小化，则使用图标表示最小化的窗口。大多数窗口系统或外观也会在窗口装饰中显示图标。典型的图标大小为16x16像素，但是某些窗口系统使用其他大小。

以下快照显示了三个相同的框架，除了它们的窗口装饰。从每帧中按钮的外观可以看出，这三个按钮均使用Java外观。第一种使用窗口系统提供的装饰，碰巧是Microsoft Windows，但也很可能是运行Java平台的任何其他系统。第二种和第三种使用Java外观提供的窗口装饰。第三帧使用Java外观窗口装饰，但具有自定义图标。

//Create the frame.

//Set the frame icon to an image loaded from a file.

响应窗口关闭事件

默认情况下，当用户关闭屏幕上的框架时，该框架是隐藏的。尽管不可见，但框架仍然存在，程序可以使其再次可见。如果您想要不同的行为，则需要注册一个处理窗口关闭事件的窗口侦听器，或者需要使用setDefaultCloseOperation方法。您甚至都可以做。

争论setDefaultCloseOperation必须是以下值之一，其中前三个是在WindowConstants介面（由JFrame ， JInternalPane和JDialog ）：

DO_NOTHING_ON_CLOSE
当用户请求关闭窗口时，不要执行任何操作。相反，该程序可能应该使用一个窗口侦听器，该侦听器在其内部执行其他操作windowClosing方法。
HIDE_ON_CLOSE （默认为JDialog和JFrame ）
用户关闭窗口时将其隐藏。这会将窗口从屏幕上删除，但使其可显示。
DISPOSE_ON_CLOSE （默认为JInternalFrame ）
当用户关闭窗口时，隐藏并丢弃该窗口。这将从屏幕上删除该窗口并释放它使用的所有资源。
EXIT_ON_CLOSE （在JFrame类）
使用以下命令退出应用程序System.exit(0) 。仅建议用于应用程序。如果在applet中使用， SecurityException可能会抛出。

注意： 

DISPOSE_ON_CLOSE可以得到类似于以下结果EXIT_ON_CLOSE如果只有一个窗口在屏幕上。更精确地，当处置Java虚拟机（VM）中的最后一个可显示窗口时，VM可以终止。有关详细信息，请参见AWT线程问题 。

在任何窗口侦听器处理窗口关闭事件之后，将执行默认的关闭操作。因此，例如，假定您指定默认的关闭操作是处理框架。您还实现了一个窗口侦听器，该侦听器测试该帧是否为最后一个可见帧，如果是，则保存一些数据并退出应用程序。在这种情况下，当用户关闭框架时，将首先调用窗口侦听器。如果它没有退出应用程序，则将执行默认的关闭操作（框架的处置）。

有关处理关闭窗口事件的更多信息，请参见如何编写窗口侦听器 。除了处理关闭窗口的事件外，窗口侦听器还可以对其他窗口状态更改（例如图标化和激活）做出反应。

框架API

下表列出了常用的JFrame构造函数和方法。您可能要调用的其他方法由java.awt.Frame ， java.awt.Window和java.awt.Component类，从中JFrame下降。

因为每个JFrame对象具有根窗格，框架支持在框架子项之前插入输入和绘画行为，将子项放置在不同的“层”上，以及支持Swing菜单栏。这些主题在使用顶级容器中介绍，并在如何使用根窗格中进行了详细说明。

使用框架的API分为以下几类：

  • 创建和设置框架
  • 设置窗口大小和位置
  • 与根窗格相关的方法
