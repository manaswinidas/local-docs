使用顶层容器

如前所述，Swing提供了三个通常有用的顶级容器类： JFrame ， JDialog和JApplet 。使用这些类时，应牢记以下事实：

  • 为了在屏幕上显示，每个GUI组件必须是包含层次结构的一部分。包含层次结构是一棵以顶级容器为根的组件树。我们将向您展示一下。
  • 每个GUI组件只能包含一次。如果某个组件已经在容器中，并且您尝试将其添加到另一个容器中，则该组件将从第一个容器中删除，然后添加到第二个容器中。
  • 每个顶层容器都有一个内容窗格，通常来说，该窗格包含（直接或间接）该顶层容器的GUI中的可见组件。
  • 您可以选择将菜单栏添加到顶层容器。按照惯例，菜单栏位于顶级容器内，但位于内容窗格之外。某些外观（例如Mac OS外观）使您可以选择将菜单栏放置在更适合外观的其他位置，例如屏幕顶部。

注意：尽管JInternalFrame模仿JFrame ，内部框架实际上不是顶级容器。

顶级容器和容纳层次结构

每个使用Swing组件的程序都至少具有一个顶级容器。此顶层容器是包含层次结构的根-包含所有出现在顶层容器中的Swing组件的层次结构。

通常，具有基于Swing GUI的独立应用程序至少具有一个包含层次结构，其中包含JFrame作为其根源。例如，如果一个应用程序具有一个主窗口和两个对话框，则该应用程序将具有三个包含层次结构，并因此具有三个顶级容器。一个密闭等级具有一个JFrame作为其根，其他两个都有一个JDialog对象为根。

基于Swing的applet至少具有一个包含层次结构，而其中一个正是由一个JApplet宾语。例如，弹出对话框的小程序具有两个包含层次结构。浏览器窗口中的组件位于以JApplet宾语。该对话框具有一个以JDialog宾语。

将组件添加到内容窗格

这是前面的示例用来获取框架内容窗格并为其添加黄色标签的代码：

frame.getContentPane().add(yellowLabel, BorderLayout.CENTER);

如代码所示，您可以通过调用以下代码找到顶级容器的内容窗格： getContentPane方法。默认的内容窗格是一个简单的中间容器，它继承自JComponent ，并且使用BorderLayout作为其布局经理。

自定义内容窗格很容易-例如，设置布局管理器或添加边框。但是，这里只有一个小陷阱。的getContentPane方法返回一个Container对象，不是JComponent宾语。这意味着，如果您想利用内容窗格的JComponent功能，您需要强制转换返回值或创建自己的组件作为内容窗格。我们的示例通常采用第二种方法，因为它更干净一些。我们有时采取的另一种方法是简单地向内容窗格中添加自定义组件，以完全覆盖内容窗格。

请注意，默认布局管理器用于JPanel是FlowLayout ;您可能需要更改它。

要使组件成为内容窗格，请使用顶级容器的setContentPane方法。例如：

//Create a panel and add components to it.
JPanel contentPane = new JPanel(new BorderLayout());
contentPane.setBorder(someBorder);
contentPane.add(someComponent, BorderLayout.CENTER);
contentPane.add(anotherComponent, BorderLayout.PAGE_END);


topLevelContainer.setContentPane(contentPane);
注意： 

为方便起见， add方法及其变体， remove和setLayout已被覆盖以转发给contentPane有必要的。这意味着你可以写

frame.add(child);

然后孩子将被添加到contentPane.

请注意，只有这三种方法可以执行此操作。这意味着getLayout()将不会返回带有setLayout() 。

添加菜单栏

从理论上讲，所有顶级容器都可以容纳菜单栏。但是，实际上，菜单栏通常仅出现在框架和小程序中。要将菜单栏添加到顶级容器，请创建一个JMenuBar对象，用菜单填充它，然后调用setJMenuBar 。的TopLevelDemo使用以下代码将菜单栏添加到其框架：

frame.setJMenuBar(greenMenuBar);

有关实现菜单和菜单栏的更多信息，请参见如何使用菜单 。

根窗格

每个顶级容器都依赖于一个称为根窗格的隐式中间容器。根窗格管理内容窗格和菜单栏，以及其他几个容器。通常，您不需要了解根窗格即可使用Swing组件。但是，如果您需要拦截鼠标单击或在多个组件上绘制，则应该熟悉根窗格。

这是根窗格提供给框架（以及其他所有顶层容器）的组件的列表：

我们已经告诉过您有关内容窗格和可选菜单栏的信息。根窗格添加的其他两个组件是分层窗格和玻璃窗格。分层窗格包含菜单栏和内容窗格，并允许对其他组件进行Z排序。玻璃窗格通常用于拦截发生在顶层容器上的输入事件，也可以用于在多个组件上绘制。

有关更多详细信息，请参见如何使用根窗格 。
