如何使用进度条

有时，在程序中运行的任务可能需要一段时间才能完成。一个用户友好的程序向用户提供一些指示，表明任务正在执行，任务可能需要花费多长时间以及已经完成了多少工作。指示工作或进度的一种方法是使用动画图像。

指示工作的另一种方法是使用Cursor类和Component定义的setCursor方法。例如，以下代码使光标移到上方时显示等待光标container （包括其中包含的所有未指定游标的组件）：

container.setCursor(Cursor.getPredefinedCursor(Cursor.WAIT_CURSOR));

有时您无法立即确定长时间运行的任务的长度，或者该任务可能长时间停留在相同的完成状态。您可以通过将进度条置于不确定模式来显示没有可衡量进度的工作。 不确定模式下的进度条显示动画，以指示正在进行工作。进度条可以显示更多有意义的信息后，应立即将其切换回默认的确定模式。在Java外观中，不确定的进度条如下所示：

Swing提供了三个类来帮助您使用进度条：

JProgressBar
一个可视组件，以图形方式显示总任务已完成的数量。有关信息和使用典型进度条的示例，请参见使用确定的进度条。使用不确定模式部分介绍如何在知道任务范围之前为进度条设置动画以显示活动。
ProgressMonitor
不是可见的组件。而是，此类的一个实例监视任务的进度，并在必要时弹出一个对话框 。有关详细信息和使用进度监视器的示例，请参见如何使用进度监视器 。
ProgressMonitorInputStream
输入流带有附加的进度监视器，该监视器监视流中的读取。您可以像Basic I / O中描述的任何其他输入流一样使用此流的实例。您可以通过以下方式获取信息流的进度监控器： getProgressMonitor并按照如何使用进度监视器中所述进行配置。

在看到进度条和进度监视器后， 决定使用进度条还是进度监视器可以帮助您确定哪种适合您的应用程序。

以下代码来自ProgressBarDemo.java ，创建并设置进度条：

//Where member variables are declared:
JProgressBar progressBar;
...
//Where the GUI is constructed:
progressBar = new JProgressBar(0, task.getLengthOfTask());
progressBar.setValue(0);
progressBar.setStringPainted(true);

创建进度条的构造函数设置进度条的最小值和最大值。您也可以使用setMinimum和setMaximum 。此程序中使用的最小值和最大值为0，且该任务的长度是许多程序和任务的典型长度。但是，进度条的最小值和最大值可以是任何值，甚至可以是负值。该代码段还将进度条的当前值设置为0。

致电setStringPainted使进度条在其范围内以文本形式显示已完成任务的百分比。默认情况下，进度条显示其返回的值getPercentComplete方法格式化为百分比，例如33％ 。或者，您可以通过调用以下命令将默认值替换为其他字符串setString 。例如，

if (/*...half way done...*/)
    progressBar.setString("Half way there!");

当用户单击Start时 ，内部类的实例Task创建并执行。

public void actionPerformed(ActionEvent evt) {
    startButton.setEnabled(false);
    setCursor(Cursor.getPredefinedCursor(Cursor.WAIT_CURSOR));
    done = false;
    task = new Task();
    task.addPropertyChangeListener(this);
    task.execute();
}

Task是的子类javax.swing.SwingWorker 。的Task实例为以下三个重要事项ProgressBarDemo ：

  1. 实例调用doInBackground在单独的线程中。这是长时间运行的任务实际执行的位置。使用后台线程而不是事件分发线程可以防止任务运行时用户界面冻结。
  2. 后台任务完成后，实例将调用done事件调度线程中的方法。
  3. 该实例维护一个绑定属性， progress ，即进行更新以指示任务的进度。的propertyChange每次调用方法progress变化。

有关更多信息，请参见Swing中的并发中的 Worker线程和SwingWorker 。 SwingWorker 。

中的后台任务ProgressBarDemo通过以随机间隔报告随机进度来模拟真实任务。的propertyChange方法响应任务的变化progress通过更新进度条设置属性：

public void propertyChange(PropertyChangeEvent evt) {
    if (!done) {
        int progress = task.getProgress();
        progressBar.setValue(progress);
        taskOutput.append(String.format(
                "Completed %d%% of task.\n", progress));
    }

后台任务完成后，任务的done方法重置进度条：

public void done() {
    //Tell progress listener to stop updating progress bar.
    done = true;
    Toolkit.getDefaultToolkit().beep();
    startButton.setEnabled(true);
    setCursor(null); //turn off the wait cursor
    progressBar.setValue(progressBar.getMinimum());
    taskOutput.append("Done!\n");
}

请注意done方法设置done到true ，防止propertyChange从进一步更新进度条。这是必要的，因为对progress财产可能在之后发生done被调用。

使用不确定模式

在ProgressBarDemo2设置不确定模式，直到实际进度开始：

public void propertyChange(PropertyChangeEvent evt) {
    if (!done) {
        int progress = task.getProgress();
        if (progress == 0) {
            progressBar.setIndeterminate(true);
            taskOutput.append("No progress yet\n");
        } else {
            progressBar.setIndeterminate(false); 
            progressBar.setString(null);
            progressBar.setValue(progress);
            taskOutput.append(String.format(
                    "Completed %d%% of task.\n", progress));
        }
    }
}

代码中的其他更改与字符串显示有关。显示字符串的进度条可能比不显示字符串的高，并且作为演示设计者，我们任意决定该进度条仅在处于默认确定模式时才显示字符串。但是，我们要避免进度条在更改模式时更改高度时可能导致的布局丑陋。因此，代码留给了setStringPainted(true)但添加了一个呼叫setString("")这样就不会显示任何文字。稍后，当进度条从不确定模式切换到确定模式时，将调用setString(null)使进度条显示其默认字符串。

我们未进行的一项更改是删除对progressBar.setValue来自progress事件处理程序。该调用不会造成任何危害，因为不确定的进度条不使用其value属性，除非可以在状态字符串中显示它。实际上，最好使进度条的数据保持最新，因为某些外观可能不支持不确定模式。

  2. 按下开始按钮。请注意，只要按下按钮，进度条就会开始动画显示，然后切换回确定模式（例如ProgressBarDemo）。

如何使用进度监视器

现在，让我们重写ProgressBarDemo以使用进度监视器而不是进度条。这是新的演示程序ProgressMonitorDemo的图片：

尝试这个： 
  1. 单击启动按钮，以使用Java™Web Start （ 下载JDK 7或更高版本 ）运行ProgressMonitor演示。另外，要自己编译并运行示例，请参考示例索引 。
  2. 按下开始按钮。一定时间后，程序将显示一个进度对话框。
  3. 单击确定按钮。请注意，即使对话框消失了，任务也会继续。
  4. 启动另一个任务。对话框弹出后，单击“ 取消”按钮。对话框消失，任务停止。

进度监视器无法再次使用，因此每次启动新任务时都必须创建一个新监视器。每次用户使用“ 开始”按钮启动新任务时，该程序都会创建一个进度监视器。

这是创建进度监视器的语句：

progressMonitor = new ProgressMonitor(ProgressMonitorDemo.this,
                                      "Running a Long Task",
                                      "", 0, task.getLengthOfTask());

该代码使用ProgressMonitor是创建监视器并初始化几个参数的唯一构造函数：

  • 第一个参数为进度监视器弹出的对话框提供父组件。
  • 第二个参数是一个字符串，它描述了被监视任务的性质。该字符串显示在对话框中。有关此参数的详细信息，请参见Progress Monitoring API 。
  • 第三个参数是提供可更改状态注释的另一个字符串。该示例使用空字符串指示对话框应为可更改的状态注释留出空间，但该注释最初为空。如果您提供null对于此参数，对话框中省略了注释。该示例在每次progress属性更改。它同时更新监视器的当前值：
    int progress = task.getProgress();
String message = String.format("Completed %d%%.\n", progress);
progressMonitor.setNote(message);
progressMonitor.setProgress(progress);
taskOutput.append(message);
  • 最后两个参数分别为对话框中显示的进度栏提供最小值和最大值。

默认情况下，进度监视器将等待最少500毫秒，然后再决定是否弹出对话框。它还等待进度超过最小值。如果计算出要完成该任务将花费2000毫秒以上，则会显示进度对话框。要调整最短等待时间，请调用setMillisToDecidedToPopup 。要调整显示对话框所需的最短进度，请调用setMillisToPopup 。

由于本示例使用进度监视器这一简单事实，它添加了使用进度条的程序版本中不存在的功能：用户可以通过单击对话框上的“ 取消”按钮来取消任务。这是示例中的代码，用于检查用户是否取消了任务或任务是否正常退出：

if (progressMonitor.isCanceled() || task.isDone()) {
    progressMonitor.close();
    Toolkit.getDefaultToolkit().beep();
    if (progressMonitor.isCanceled()) {
        task.cancel(true);
        taskOutput.append("Task canceled.\n");
    } else {
        taskOutput.append("Task completed.\n");
    }
    startButton.setEnabled(true);
}

请注意，进度监视器本身不会取消任务。它提供了GUI和API，以使程序可以轻松执行此操作。

决定使用进度条还是进度监视器

如果出现以下情况，请使用进度条 ：

  • 您希望对进度条的配置有更多控制。如果直接使用进度条，则可以将其设置为不确定状态，使其垂直显示，提供要显示的字符串，在其上注册更改侦听器，并为其提供有界范围模型以控制进度条的最小值，最大值和当前值。
  • 该程序需要显示其他组件以及进度条。
  • 您需要多个进度条。对于某些任务，您需要监视多个参数。例如，除了成功安装了多少文件之外，安装程序还可能监视磁盘空间使用情况。
  • 您需要重新使用进度条。进度条可以重复使用；进度监视器不能。一旦进度监视器决定显示（或不显示）对话框，进度监视器将无法再次执行该对话框。

在以下情况下，请使用进度监视器 ：

  • 您需要一种简单的方法来在对话框中显示进度。
  • 正在运行的任务是次要任务，用户可能对任务的进度不感兴趣。进度监视器为用户提供了一种在任务仍在运行时使其退出对话框的方法。
  • 您希望取消任务的简便方法。进度监视器为用户提供了一个GUI来取消任务。您所要做的就是呼叫进度监视器的isCanceled找出用户是否按下“ 取消”按钮的方法。
  • 您的任务在运行时会定期显示一条短消息。进度监视器对话框提供了setNote方法，以便任务可以提供有关其操作的更多信息。例如，安装任务可能会报告每个文件在安装时的名称。
  • 该任务可能不会花费很长时间才能完成。您可以确定正在运行的任务在什么时候需要花费足够长的时间来保证让用户知道它。如果任务在您设置的时间范围内完成，进度监视器将不会弹出对话框。

如果您决定使用进度监视器， 并且正在监视的任务正在从输入流中读取，请使用ProgressMonitorInputStream类。

进度监控API

下表列出了使用进度条和进度监视器的常用API。因为JProgressBar是的子类JComponent ，您可能会调用的其他方法JProgressBar在JComponent类中列出。注意ProgressMonitor是的子类Object并且不是视觉组件。

用于监视进度的API分为以下几类：

  • 创建进度条
  • 设置或获取进度栏的约束/值
  • 控制进度条的外观
  • 创建进度监视器
  • 配置进度监视器
  • 终止进度监视器
