如何使用清单

一种JList向用户显示一组项目，显示在一个或多个列中以供选择。列表可以包含许多项，因此它们通常放在滚动窗格中 。

除了列表之外，以下Swing组件还向用户提供了多个可选项目： 组合框 ， 菜单 ， 表格以及复选框或单选按钮组 。要显示分层数据，请使用树 。

建立模型

有三种创建列表模型的方法：

  • DefaultListModel-几乎所有工作都由您完成。此页面中的示例使用DefaultListModel 。
  • AbstractListModel-您管理数据并调用“射击”方法。对于这种方法，必须子类化AbstractListModel并实施getSize和getElementAt从继承的方法ListModel接口。
  • ListModel-您管理一切。

初始化列表

这是来自的代码ListDialog.java创建并设置其列表：

list = new JList(data); //data has type Object[]
list.setSelectionMode(ListSelectionModel.SINGLE_INTERVAL_SELECTION);
list.setLayoutOrientation(JList.HORIZONTAL_WRAP);
list.setVisibleRowCount(-1);
...
JScrollPane listScroller = new JScrollPane(list);
listScroller.setPreferredSize(new Dimension(250, 80));

代码将数组传递给列表的构造函数。数组中填充了从另一个对象传入的字符串。在我们的示例中，字符串恰好是男孩的名字。

其他JList构造函数可让您初始化一个列表Vector或来自粘附在ListModel接口。如果使用数组或向量初始化列表，则构造函数将隐式创建默认列表模型。默认列表模型是不可变的-您无法添加，删除或替换列表中的项目。要创建可以单独更改其项目的列表，请将列表的模型设置为可变列表模型类的实例，例如的实例。 DefaultListModel 。您可以在创建列表时设置列表的模型，也可以通过调用setModel方法。有关示例，请参见向列表添加项目和从列表中删除项目 。

致电setSelectionMode指定用户可以选择多少个项目，以及它们是否必须是连续的；下一节将向您详细介绍选择模式。

致电setLayoutOrientation使列表在多列中显示其数据。价值JList.HORIZONTAL_WRAP指定列表在包装到新行之前应从左至右显示其项目。另一个可能的值是JList.VERTICAL_WRAP ，它指定将数据换行到新列之前从上至下（通常）显示数据。下图显示了这两种包装方式以及默认方式， JList.VERTICAL 。

HORIZONTAL_WRAP VERTICAL_WRAP VERTICAL

结合致电setLayoutOrientation ，调用setVisibleRowCount(-1)使列表显示屏幕上可用空间中可能的最大项目数。的另一种常见用法setVisibleRowCount是向列表的滚动窗格指定列表希望显示多少行。

在列表中选择项目

列表使用的实例ListSelectionModel管理其选择。默认情况下，列表选择模型允许一次选择项目的任何组合。您可以通过调用setSelectionMode列表中的方法。例如，两者ListDialog和ListDemo将选择模式设置为SINGLE_SELECTION （由ListSelectionModel ），以便只能选择列表中的一项。下表描述了三种列表选择模式：

模式 描述
SINGLE_SELECTION


 一次只能选择一项。用户选择一个项目时，首先取消选择任何先前选择的项目。
SINGLE_INTERVAL_SELECTION


 可以选择多个连续的项目。当用户开始新的选择范围时，首先取消选择任何先前选择的项目。
MULTIPLE_INTERVAL_SELECTION 


 默认值。可以选择任何项目组合。用户必须明确取消选择项目。

无论您的列表使用哪种选择方式，只要选择更改，列表都会触发列表选择事件。您可以通过将列表选择侦听器添加到列表中来处理这些事件， addListSelectionListener方法。列表选择侦听器必须实现一种方法： valueChanged 。这里是valueChanged侦听器中的方法ListDemo ：

public void valueChanged(ListSelectionEvent e) {
    if (e.getValueIsAdjusting() == false) {

        if (list.getSelectedIndex() == -1) {
        //No selection, disable fire button.
            fireButton.setEnabled(false);

        } else {
        //Selection, enable the fire button.
            fireButton.setEnabled(true);
        }
    }
}

可以从单个用户操作（例如单击鼠标）生成许多列表选择事件。的getValueIsAdjusting方法返回true如果用户仍在操纵选择。该特定程序仅对用户操作的最终结果感兴趣，因此valueChanged该方法仅在以下情况下执行某些操作getValueIsAdjusting退货false 。

由于列表处于单选模式，因此该代码可以使用getSelectedIndex获取刚刚选择的项目的索引。 JList当选择模式允许选择多个项目时，提供了其他方法来设置或获取选择。如果需要，您可以侦听列表的列表选择模型上的事件，而不是列表本身。 ListSelectionDemo是一个示例，显示了如何在列表选择模型上侦听列表选择事件，并允许您动态更改列表的选择模式。

向列表添加项目和从列表中删除项目

我们之前显示的ListDemo示例具有一个列表，其内容可以更改。您可以在以下位置找到ListDemo的源代码ListDemo.java 。这是ListDemo代码，它创建一个可变的列表模型对象，将初始项放入其中，然后使用列表模型创建列表：

listModel = new DefaultListModel();
listModel.addElement("Jane Doe");
listModel.addElement("John Smith");
listModel.addElement("Kathy Green");


list = new JList(listModel);

这个特定的程式使用DefaultListModel ，是Swing提供的类。尽管有类名，但列表中没有DefaultListModel除非您的程序明确地这样做。如果DefaultListModel不适合您的需求，您可以编写一个自定义列表模型，该模型必须遵守ListModel接口。

以下代码段显示了actionPerformed Fire按钮上注册的动作侦听器的方法。粗体代码行将删除列表中的选定项目。如果列表现在为空，则方法中的其余行将禁用“启动”按钮，如果列表不为空，则进行另一选择。

public void actionPerformed(ActionEvent e) {
    int index = list.getSelectedIndex();
    listModel.remove(index);

    int size = listModel.getSize();

    if (size == 0) { //Nobody's left, disable firing.
        fireButton.setEnabled(false);

    } else { //Select an index.
        if (index == listModel.getSize()) {
            //removed item in last position
            index--;
        }

        list.setSelectedIndex(index);
        list.ensureIndexIsVisible(index);
    }
}

这里是actionPerformed 雇用按钮和文本字段共享的动作侦听器的方法：

public void actionPerformed(ActionEvent e) {
    String name = employeeName.getText();

    //User did not type in a unique name...
    if (name.equals("") || alreadyInList(name)) {
        Toolkit.getDefaultToolkit().beep();
        employeeName.requestFocusInWindow();
        employeeName.selectAll();
        return;
    }

    int index = list.getSelectedIndex(); //get selected index
    if (index == -1) { //no selection, so insert at beginning
        index = 0;
    } else {           //add after the selected item
        index++;
    }

    listModel.insertElementAt(employeeName.getText(), index);

    //Reset the text field.
    employeeName.requestFocusInWindow();
    employeeName.setText("");

    //Select the new item and make it visible.
    list.setSelectedIndex(index);
    list.ensureIndexIsVisible(index);
}

此代码使用列表模型的insertElementAt在当前选择项之后插入新名称的方法，如果没有选择项，则在列表的开头插入新名称。如果您只想添加到列表的末尾，则可以使用DefaultListModel的addElement方法代替。

每当将项目添加到列表中，从列表中删除或在列表中进行修改时，列表模型都会触发列表数据事件。有关如何监听这些事件的信息，请参阅如何编写列表数据监听器 。该部分包含的示例类似于ListDemo ，但添加了在列表中向上或向下移动项目的按钮。

编写自定义单元格渲染器

列表使用称为单元格渲染器的对象来显示其每个项目。默认的单元格渲染器知道如何显示字符串和图标，并显示Object s通过调用toString 。如果要更改默认渲染器显示图标或字符串的方式，或者想要的行为与toString ，您可以实现自定义单元格渲染器。请按照以下步骤为列表提供自定义单元格渲染器：

  • 写一个实现ListCellRenderer接口。
  • 创建您的课程的实例，然后调用列表的setCellRenderer使用实例作为参数。

我们没有提供带有自定义单元格渲染器的列表的示例，但确实有带有自定义渲染器的组合框的示例—组合框使用与列表相同类型的渲染器。请参阅“ 提供自定义渲染器”中描述的示例。

列表API

下表列出了常用的JList构造函数和方法。您最有可能在JList对象是诸如setPreferredSize它的超类提供的。有关常用继承方法的表，请参见JComponent API 。

列表的大部分操作由其他对象管理。列表中的项目由列表模型对象管理，选择由列表选择模型对象管理，大多数程序将列表放在滚动窗格中以处理滚动。在大多数情况下，您无需担心模型，因为JList根据需要创建它们，然后与它们隐式交互JList的便捷方法。

也就是说，使用列表的API分为以下几类：

  • 初始化列表数据
  • 显示清单
  • 管理列表的选择
