如何使用文件选择器

文件选择器提供了一个GUI，用于浏览文件系统，然后从列表中选择文件或目录，或者输入文件或目录的名称。要显示文件选择器，通常使用JFileChooser用于显示包含文件选择器的模式对话框的 API。呈现文件选择器的另一种方法是添加的实例JFileChooser到一个容器。

注意： 

如果您打算将程序作为沙箱Java Web Start应用程序分发，请不要使用JFileChooser API，您应该使用JNLP API提供的文件服务。这些服务- FileOpenService和FileSaveService -不仅为在受限环境中选择文件提供支持，而且还要照顾实际打开和保存文件的情况。在JWSFileChooserDemo中提供了使用这些服务的示例 。可以在Java Web Start课程中找到有关使用JNLP API的文档。

单击启动按钮，以使用Java™Web Start运行JWSFileChooserDemo（ 下载JDK 7或更高版本 ）。另外，要自己编译并运行示例，请参考示例索引 。


当使用JWSFileChooserDemo例如，请小心不要丢失所需的文件。每当您单击保存按钮并选择一个现有文件时，此演示都会弹出“文件已存在”对话框，并要求您替换文件。接受请求将覆盖文件。

本节的其余部分讨论如何使用JFileChooser API。一种JFileChooser对象仅显示用于选择文件的GUI。您的程序负责对所选文件执行某些操作，例如打开或保存它。有关如何读取和写入文件的信息，请参考基本I / O。

的JFileChooser API使您可以轻松打开和保存对话框。外观的类型决定了这些标准对话框的外观以及它们之间的区别。在Java外观中，保存对话框的外观与打开的对话框相同，不同之处在于对话框窗口上的标题和批准操作的按钮上的文本。这是Java外观中的标准打开对话框的图片：

  2. 单击打开文件按钮。浏览文件选择器，选择一个文件，然后单击对话框的“打开”按钮。

  4. 在源文件中FileChooserDemo.java ，将文件选择模式更改为仅目录模式。（搜索DIRECTORIES_ONLY并取消注释包含它的行。）然后编译并再次运行该示例。您将只能查看和选择目录，而不能查看普通文件。

弹出一个标准的打开对话框只需要两行代码：

//Create a file chooser
final JFileChooser fc = new JFileChooser();
...
//In response to a button click:
int returnVal = fc.showOpenDialog(aComponent);

的论点showOpenDialog方法指定对话框的父组件。父组件会影响对话框的位置以及对话框所依赖的框架。例如，Java外观将对话框直接放在父组件上。如果父组件位于框架中，则对话框取决于该框架。当框架最小化时，此对话框消失，而当框架最大化时，此对话框重新出现。

默认情况下，之前未显示的文件选择器将显示用户主目录中的所有文件。您可以使用以下任一方法指定文件选择器的初始目录： JFileChooser的其他构造函数，或者您可以使用setCurrentDirectory方法。

致电showOpenDialog出现在actionPerformed “打开文件”按钮的操作侦听器的方法：

public void actionPerformed(ActionEvent e) {
    //Handle open button action.
    if (e.getSource() == openButton) {
        int returnVal = fc.showOpenDialog(FileChooserDemo.this);

        if (returnVal == JFileChooser.APPROVE_OPTION) {
            File file = fc.getSelectedFile();
            //This is where a real application would open the file.
            log.append("Opening: " + file.getName() + "." + newline);
        } else {
            log.append("Open command cancelled by user." + newline);
        }
   } ...
}

的showXxxDialog方法返回一个整数，该整数指示用户是否选择了文件。根据您使用文件选择器的方式，通常足以检查返回值是否为APPROVE_OPTION然后不更改任何其他值。要获取所选文件（或目录，如果您将文件选择器设置为允许目录选择），请调用getSelectedFile文件选择器上的方法。此方法返回的实例File 。

该示例获取文件的名称，并在日志消息中使用它。您可以在File对象，例如getPath ， isDirectory ， 要么exists获取有关文件的信息。您还可以调用其他方法，例如delete和rename以某种方式更改文件。当然，您可能还想通过使用Java平台提供的阅读器或编写器类之一来打开或保存文件。有关使用读取器和写入器读取数据并将其写入文件系统的信息，请参见基本I / O。

示例程序使用相同的实例JFileChooser类以显示标准保存对话框。这次程序调用showSaveDialog ：

int returnVal = fc.showSaveDialog(FileChooserDemo.this);

通过使用相同的文件选择器实例来显示其打开和保存对话框，该程序可获得以下好处：

  • 选择器会记住两次使用之间的当前目录，因此打开和保存版本会自动共享同一当前目录。
  • 您只需要自定义一个文件选择器，并且自定义既适用于打开版本又适用于保存版本。

最后，该示例程序具有注释掉的代码行，可让您更改文件选择模式。例如，以下代码行使文件选择器只能选择目录，而不能选择文件：

fc.setFileSelectionMode(JFileChooser.DIRECTORIES_ONLY);

使用文件选择器执行自定义任务

如您所见， JFileChooser类提供了showOpenDialog显示打开对话框的方法和showSaveDialog显示保存对话框的方法。

该类还有另一种方法， showDialog ，用于在对话框中显示自定义任务的文件选择器。在Java外观中，此对话框与其他文件选择器对话框的唯一区别是对话框窗口上的标题和批准按钮上的标签。这是来自的代码FileChooserDemo2弹出“附加”任务的文件选择器对话框：

JFileChooser fc = new JFileChooser();
int returnVal = fc.showDialog(FileChooserDemo2.this, "Attach");

第一个参数showDialog方法是对话框的父组件。第二个参数是String同时提供对话框窗口标题和批准按钮标签的对象。

同样，文件选择器对所选文件不执行任何操作。该程序负责实现为其创建文件选择器的自定义任务。

过滤文件列表

默认情况下，文件选择器显示其检测到的所有文件和目录，隐藏文件除外。程序可以将一个或多个文件过滤器应用于文件选择器，以便选择器仅显示一些文件。文件选择器调用过滤器的accept每个文件确定是否应显示的方法。文件过滤器根据诸如文件类型，大小，所有权等条件来接受或拒绝文件。过滤器影响文件选择器显示的文件列表。用户可以输入任何文件的名称，即使它没有显示。

JFileChooser支持三种不同的过滤。将按照此处列出的顺序检查过滤器。例如，应用程序控制的过滤器仅查看内置过滤器接受的那些文件。

内置过滤
过滤是通过文件选择器上的特定方法调用来设置的。当前，唯一可用的内置过滤器用于隐藏文件，例如在UNIX系统上其名称以句点（。）开头的文件。默认情况下，不显示隐藏文件。呼叫setFileHidingEnabled(false)显示隐藏文件。
应用控制过滤
应用程序确定显示哪些文件。创建一个自定义子类FileFilter ，实例化它，并将实例用作setFileFilter方法。已安装的过滤器显示在用户可选择的过滤器列表中。文件选择器仅显示过滤器接受的那些文件。
用户可选择的过滤
文件选择器GUI提供了可供用户选择的过滤器列表。当用户选择过滤器时，文件选择器仅显示该过滤器接受的文件。 FileChooserDemo2将自定义文件过滤器添加到用户可选择的过滤器列表中：
fc.addChoosableFileFilter(new ImageFilter());
默认情况下，用户可选择的过滤器列表包括“全部接受”过滤器，使用户可以查看所有非隐藏文件。本示例使用以下代码禁用“全部接受”过滤器：
fc.setAcceptAllFileFilterUsed(false);
我们的自定义文件过滤器在ImageFilter.java并且是的子类FileFilter 。的ImageFilter类实现getDescription返回“ Just Images”的方法-将字符串放入用户可选择的过滤器列表中。 ImageFilter还实现了accept方法，以便它接受所有目录以及具有.png ， .jpg ， .jpeg ， .gif ， .tif ， 要么.tiff文件扩展名。 
public boolean accept(File f) {
    if (f.isDirectory()) {
        return true;
    }

    String extension = Utils.getExtension(f);
    if (extension != null) {
        if (extension.equals(Utils.tiff) ||
            extension.equals(Utils.tif) ||
            extension.equals(Utils.gif) ||
            extension.equals(Utils.jpeg) ||
            extension.equals(Utils.jpg) ||
            extension.equals(Utils.png)) {
                return true;
        } else {
            return false;
        }
    }

    return false;
}
通过接受所有目录，此筛选器允许用户在文件系统中浏览。如果此方法中省略了粗线，则用户将被限制在初始化选择器的目录中。

前面的代码示例使用getExtension方法和几个字符串常量Utils.java ，如下所示：

public class Utils {

    public final static String jpeg = "jpeg";
    public final static String jpg = "jpg";
    public final static String gif = "gif";
    public final static String tiff = "tiff";
    public final static String tif = "tif";
    public final static String png = "png";

    /*
     * Get the extension of a file.
     */  
    public static String getExtension(File f) {
        String ext = null;
        String s = f.getName();
        int i = s.lastIndexOf('.');

        if (i > 0 &&  i < s.length() - 1) {
            ext = s.substring(i+1).toLowerCase();
        }
        return ext;
    }
}

自定义文件视图

在Java外观中，选择器的列表显示每个文件的名称，并显示一个小图标，表示该文件是真实文件还是目录。您可以通过创建以下内容的自定义子类来自定义此文件视图 FileView并使用该类的实例作为setFileView方法。该示例使用一个自定义类的实例，该实例在ImageFileView.java ，作为文件选择器的文件视图。

fc.setFileView(new ImageFileView());

的ImageFileView类别针对上述图像过滤器接受的每种图像类型显示不同的图标。

的ImageFileView类覆盖了定义的五个抽象方法FileView如下。

String getTypeDescription(File f)
返回文件类型的描述。这是ImageFileView方法的实现：
public String getTypeDescription(File f) {
    String extension = Utils.getExtension(f);
    String type = null;

    if (extension != null) {
        if (extension.equals(Utils.jpeg) ||
            extension.equals(Utils.jpg)) {
            type = "JPEG Image";
        } else if (extension.equals(Utils.gif)){
            type = "GIF Image";
        } else if (extension.equals(Utils.tiff) ||
                   extension.equals(Utils.tif)) {
            type = "TIFF Image";
        } else if (extension.equals(Utils.png)){
            type = "PNG Image";
        }
    }
    return type;
}
Icon getIcon(File f)
返回表示文件或其类型的图标。这是ImageFileView方法的实现：
public Icon getIcon(File f) {
    String extension = Utils.getExtension(f);
    Icon icon = null;

    if (extension != null) {
        if (extension.equals(Utils.jpeg) ||
            extension.equals(Utils.jpg)) {
            icon = jpgIcon;
        } else if (extension.equals(Utils.gif)) {
            icon = gifIcon;
        } else if (extension.equals(Utils.tiff) ||
                   extension.equals(Utils.tif)) {
            icon = tiffIcon;
        } else if (extension.equals(Utils.png)) {
            icon = pngIcon;
        }
    }
    return icon;
}
String getName(File f)
返回文件名。此方法的大多数实现应返回null表示应该弄清楚外观。另一个常见的实现回报f.getName() 。
String getDescription(File f)
返回文件的描述。目的是更具体地描述单个文件。此方法的常见实现返回null表示应该弄清楚外观。
Boolean isTraversable(File f)
返回目录是否可遍历。此方法的大多数实现应返回null表示应该弄清楚外观。某些应用程序可能希望阻止用户进入某种类型的目录，因为它表示复合文档。的isTraversable方法永不返回true对于非目录。

提供配件

中的自定义文件选择器FileChooserDemo2有一个配件。如果当前选择的项目是PNG，JPEG，TIFF或GIF图像，则附件组件将显示图像的缩略图。否则，附件组件为空。除了预览器以外，附件组件最常见的用途可能是面板上带有更多控件，例如在功能之间切换的复选框。

该示例调用setAccessory建立实例的方法ImagePreview类，在ImagePreview.java ，作为选择者的辅助组件：

fc.setAccessory(new ImagePreview(fc));

从继承的任何对象JComponent类可以是附件组件。该组件应具有在文件选择器中看起来不错的首选大小。

当用户在列表中选择一个项目时，文件选择器将触发属性更改事件。每当选择更改时，带有附件组件的程序必须注册以接收这些事件以更新附件组件。在示例中， ImagePreview对象本身会注册这些事件。这样会将与附件组件有关的所有代码都归为一类。

这是示例的实现propertyChange方法，这是在触发属性更改事件时调用的方法：

//where member variables are declared
File file = null;
...
public void propertyChange(PropertyChangeEvent e) {
    boolean update = false;
    String prop = e.getPropertyName();

    //If the directory changed, don't show an image.
    if (JFileChooser.DIRECTORY_CHANGED_PROPERTY.equals(prop)) {
        file = null;
        update = true;

    //If a file became selected, find out which one.
    } else if (JFileChooser.SELECTED_FILE_CHANGED_PROPERTY.equals(prop)) {
        file = (File) e.getNewValue();
        update = true;
    }

    //Update the preview accordingly.
    if (update) {
        thumbnail = null;
        if (isShowing()) {
            loadImage();
            repaint();
        }
    }
}

如果SELECTED_FILE_CHANGED_PROPERTY是更改的属性，此方法获得一个File文件选择器中的对象。的loadImage和repaint方法使用File对象以加载图像并重新绘制附件组件。
