JApplet提供的功能

因为JApplet是顶级Swing容器，每个Swing小程序都有一个根窗格。根窗格的存在最明显的效果是支持添加菜单栏以及使用内容窗格的需要。

如使用顶层容器中所述 ，每个顶层容器（例如JApplet有一个内容窗格。内容窗格使Swing小程序与常规小程序在以下方面有所不同：

  • 您将组件添加到Swing小程序的内容窗格中，而不是直接添加到小程序中。将组件添加到内容窗格向您展示了如何。
  • 您在Swing小程序的内容窗格上而不是直接在小程序上设置布局管理器。
  • Swing小程序的内容窗格的默认布局管理器是BorderLayout 。这不同于默认的布局管理器Applet ，这是FlowLayout 。
  • 您不应该将绘画代码直接放在JApplet宾语。有关如何在小程序中执行自定义绘制的示例，请参见执行自定义绘制。

小程序中的线程

应该在事件调度线程上创建，查询和操纵Swing组件，但是浏览器不会从该线程调用applet“里程碑”方法。因此，里程碑方法- init ， start ， stop和destroy -应该使用SwingUtilities方法invokeAndWait （或者，如果合适， invokeLater ），以便在事件调度线程上执行引用Swing组件的代码。有关这些方法和事件调度线程的更多信息，请参见Swing中的Concurrency 。

这是一个例子init方法：

public void init() {
    //Execute a job on the event-dispatching thread:
    //creating this applet's GUI.
    try {
        javax.swing.SwingUtilities.invokeAndWait(new Runnable() {
            public void run() {
                createGUI();
            }
        });
    } catch (Exception e) {
        System.err.println("createGUI didn't successfully complete");
    }
}

private void createGUI() {
    JLabel label = new JLabel(
                       "You are successfully running a Swing applet!");
    label.setHorizontalAlignment(JLabel.CENTER);
    label.setBorder(BorderFactory.createMatteBorder(1,1,1,1,Color.black));
    getContentPane().add(label, BorderLayout.CENTER);
}

的invokeLater方法不适用于此实现，因为它允许init在初始化完成之前返回，这可能会导致applet问题难以调试。

在Swing小程序中使用图像

的Applet类提供了getImage将图像加载到applet中的方法。的getImage方法创建并返回一个Image表示已加载图像的对象。由于Swing组件使用Icon而不是Image s指图片，Swing applet倾向于不使用getImage 。相反，Swing applet创建以下实例ImageIcon —从图像文件加载的图标。 ImageIcon具有节省代码的优势：它可以自动处理图像跟踪。请参阅如何使用图标以获取更多信息。

杜克（Duke）做车轮的动画需要17张不同的图片。小程序使用一个ImageIcon每张图片并将其加载到init方法。由于图像加载时间可能很长，因此图标会在由SwingWorker宾语。这是代码：

public void init() {
    ...
    imgs = new ImageIcon[nimgs];
    (new SwingWorker<ImageIcon[], Object>() {
        public ImageIcon[] doInBackground() {
            //Images are numbered 1 to nimgs,
            //but fill array from 0 to nimgs-1.
            for (int i = 0; i < nimgs; i++) {
                imgs[i] = loadImage(i+1);
            }
            return imgs;
        }
        ...
    }).execute();

}
...
protected ImageIcon loadImage(int imageNum) {
    String path = dir + "/T" + imageNum + ".gif";
    int MAX_IMAGE_SIZE = 2400;  //Change this to the size of
                                 //your biggest image, in bytes.
    int count = 0;
    BufferedInputStream imgStream = new BufferedInputStream(
       this.getClass().getResourceAsStream(path));
    if (imgStream != null) {
        byte buf[] = new byte[MAX_IMAGE_SIZE];
        try {
            count = imgStream.read(buf);
            imgStream.close();
        } catch (java.io.IOException ioe) {
            System.err.println("Couldn't read stream from file: " + path);
            return null;
        }
        if (count <= 0) {
            System.err.println("Empty file: " + path);
            return null;
        }
        return new ImageIcon(Toolkit.getDefaultToolkit().createImage(buf));
    } else {
        System.err.println("Couldn't find file: " + path);
        return null;
    }
}

的loadImage方法为指定的动画帧加载图像。它使用getResourceAsStream方法，而不是通常的方法getResource获取图像的方法。结果代码不是很漂亮，但是getResourceAsStream比getResource用于将JAR文件中的图像加载到使用Java Plug-in™软件执行的applet中。有关更多详细信息，请参见将图像加载到小程序中 。

JApplet API

下表列出了有趣的方法JApplet添加到applet API。它们使您可以访问根窗格提供的功能。您可能使用的其他方法由Component和Applet类。请参阅组件方法以获取常用列表Component方法和Java Applet，以帮助使用Applet方法。

方法 目的
setContentPane（容器）
容器getContentPane（）		 设置或获取小程序的内容窗格。内容窗格包含小程序的可见GUI组件，并且应该是不透明的。
无效setRootPane（JRootPane）
JRootPane getRootPane（）		 创建，设置或获取小程序的根窗格。根窗格管理小程序的内部，包括内容窗格，玻璃窗格等。
无效setJMenuBar（JMenuBar）
JMenuBar getJMenuBar（）		 设置或获取小程序的菜单栏，以管理该小程序的菜单集。
无效setGlassPane（Component）
组件getGlassPane（）		 设置或获取小程序的玻璃窗格。您可以使用玻璃窗格来拦截鼠标事件。
无效setLayeredPane（JLayeredPane）
JLayeredPane getLayeredPane（）		 设置或获取小程序的分层窗格。您可以使用小程序的分层窗格将组件放置在其他组件之上或之下。
