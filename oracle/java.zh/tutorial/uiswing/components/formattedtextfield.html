如何使用格式化的文本字段

带格式的文本字段为开发人员提供了一种方法，可以指定可以在文本字段中键入的有效字符集。具体来说， JFormattedTextField类将格式程序和对象值添加到从继承的功能中JTextField类。格式化程序将字段的值转换为它显示的文本，并将文本转换为字段的值。

使用Swing提供的格式化程序，您可以设置格式化的文本字段以本地化格式键入日期和数字。另一种格式化程序使您可以使用字符掩码来指定可在字段中每个位置键入的字符集。例如，您可以指定用于以特定格式键入电话号码的掩码，例如（XX）X-XX-XX-XX-XX。

如果格式化的文本字段的可能值具有明显的顺序，请改用微调器 。微调器默认情况下使用带格式的文本字段，但添加了两个按钮，使用户可以选择序列中的值。

使用格式化文本字段的另一种替代方法或附件是在该字段上安装输入验证程序 。当组件几乎失去键盘焦点时，将调用组件的输入验证程序。输入验证程序使您可以检查组件的值是否有效，并可以选择更改它或停止转移焦点。

设置并获取字段的价值

使用格式化的文本字段时，请记住以下几点：

格式化文本字段的文本及其值是两个不同的属性，该值通常滞后于文本。

文本属性由JTextField类。此属性始终反映该字段显示的内容。值属性，由JFormattedTextField类，可能无法反映该字段中显示的最新文本。在用户键入时，text属性会更改，但是value属性不会更改，直到提交更改为止。

更准确地说，可以使用以下任一方法来设置格式化文本字段的值： setValue方法或commitEdit方法。的setValue方法将值设置为指定的参数。从技术上讲，论点可以是任何Object ，但格式化程序需要能够将其转换为字符串。否则，文本字段将不显示任何实质性信息。

的commitEdit方法将值设置为格式化程序确定的由字段文本表示的任何对象。的commitEdit发生以下任一情况时，将自动调用方法：

  • 当用户按下焦点区域时按下Enter键。
  • 默认情况下，当字段失去焦点时，例如，当用户按下Tab键将焦点更改为另一个组件时。您可以使用setFocusLostBehavior当字段失去焦点时指定不同结果的方法。
注意： 

一些格式化程序可能会不断更新该值，从而使失去焦点变得毫无意义，因为该值始终与文本指定的值相同。

设置格式化文本字段的值时，该字段的文本将更新以反映该值。值如何确切地表示为文本取决于字段的格式化程序。

请注意，尽管JFormattedTextField类继承了setText的方法JTextField上课，您通常不打电话给setText格式化文本字段的方法。如果这样做，则字段的显示会相应更改，但是值不会更新（除非字段的格式化程序不断更新它）。

要获取格式化的文本字段的当前值，请使用getValue方法。如有必要，您可以通过调用commitEdit之前的方法getValue 。因为getValue方法返回一个Object ，则需要将其强制转换为用于字段值的类型。例如：

Date enteredDate = (Date)dateField.getValue();

要检测格式化的文本字段的值中的更改，可以在格式化的文本字段上注册属性更改侦听器，以侦听对“ value”属性的更改。属性更改侦听器取自FormattedTextFieldDemo例：

//The property change listener is registered on each
//field using code like this:
//    someField.addPropertyChangeListener("value", this);

/** Called when a field's "value" property changes. */
public void propertyChange(PropertyChangeEvent e) {
    Object source = e.getSource();
    if (source == amountField) {
        amount = ((Number)amountField.getValue()).doubleValue();
    } else if (source == rateField) {
        rate = ((Number)rateField.getValue()).doubleValue();
    } else if (source == numPeriodsField) {
        numPeriods = ((Number)numPeriodsField.getValue()).intValue();
    }

    double payment = computePayment(amount, rate, numPeriods);
    paymentField.setValue(new Double(payment));
}

指定格式

的Format类提供了一种格式化区域设置敏感信息的方式，例如日期和数字。源于InternationalFormatter类，例如DateFormatter和NumberFormatter类，使用Format对象以在字段的文本和值之间转换。您可以获得一个Format对象，方法是通过调用DateFormat要么NumberFormat类，或使用其中之一SimpleDateFormat构造函数。

注意： 

第三种常用的格式化程序类， MaskFormatter ，不是从InternationalFormatter类，不使用格式。的MaskFormatter 使用MaskFormatter进行了讨论。

您可以在创建广告格式时自定义某些格式Format对象和其他对象（通过格式特定的API）。例如， DecimalFormat继承自的对象NumberFormat并且通常通过其工厂方法返回，可以使用setMaximumFractionDigits和setNegativePrefix方法。有关使用的信息Format对象，请参阅国际化跟踪的格式化课程。

将自定义格式与带格式的文本字段相关联的最简单方法是使用JFormattedTextField构造函数Format作为争论。您可以在前面创建的代码示例中看到此关联amountField和rateField对象。

使用MaskFormatter

的MaskFormatter类实现了一个格式化程序，该格式化程序确切指定在字段文本的每个位置哪些字符有效。例如，以下代码创建了一个MaskFormatter允许用户键入五位数的邮政编码：

zipField = new JFormattedTextField(
                    createFormatter("#####"));
...
protected MaskFormatter createFormatter(String s) {
    MaskFormatter formatter = null;
    try {
        formatter = new MaskFormatter(s);
    } catch (java.text.ParseException exc) {
        System.err.println("formatter is bad: " + exc.getMessage());
        System.exit(-1);
    }
    return formatter;
}

指定格式化程序并使用格式化程序工厂

指定格式化程序时，请记住，每个格式化程序对象一次最多只能被一个格式化的文本字段使用。每个字段至少应具有一个与其关联的格式化程序，该格式化程序可随时使用。

您可以通过几种方式指定格式化文本字段要使用的格式化程序：

  • 使用JFormattedTextField构造函数Format论点。
    将自动创建使用指定格式的字段格式化程序。
  • 使用JFormattedTextField构造函数JFormattedTextField.AbstractFormatter论点。
    指定的格式化程序用于该字段。
  • 设置没有指定格式，格式化程序或格式化程序工厂的格式化文本字段的值。
    默认格式器工厂使用该字段的值类型作为指导，将格式器分配给该字段。如果值为Date ，格式化程序是DateFormatter 。如果值为Number ，格式化程序是NumberFormatter 。其他类型导致以下情况的实例DefaultFormatter 。
  • 使格式化的文本字段使用返回自定义格式器对象的格式器工厂。
    这是最灵活的方法。当您要将一个以上的格式化程序与一个字段相关联或添加一种新的格式化程序以用于多个字段时，此功能很有用。前一种用法的示例是一个字段，该字段以某种方式解释用户的键入，但以另一种方式显示值（当用户未键入时）。后一种用法的一个示例是带有自定义类值的多个字段，例如， PhoneNumber 。您可以将字段设置为使用格式化程序工厂，该工厂返回电话号码的专用格式化程序。

您可以通过使用带有格式化程序工厂参数的构造函数来创建字段，或通过调用字段的格式化程序工厂来设置该字段setFormatterFactory现场方法。要创建格式器工厂，通常可以使用以下实例DefaultFormatterFactory类。一种DefaultFormatterFactory对象使您可以指定在编辑值，未编辑值或具有空值时返回的格式化程序。

粗体代码突出显示了对DefaultFormatterFactory构造函数。构造函数的第一个参数指定用于格式化文本字段的默认格式化程序。第二个参数指定显示格式化程序，该字段在没有焦点时使用。第三个参数指定编辑格式化程序，该字段在具有焦点时使用。该代码不使用第四个参数，但是如果使用了第四个参数，则第四个参数将指定空格式器，该格式器在字段的值为null时使用。因为未指定空格式化程序，所以当该值为空时将使用默认格式化程序。
