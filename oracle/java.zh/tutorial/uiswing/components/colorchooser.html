如何使用颜色选择器

使用JColorChooser类，使用户可以从调色板中进行选择。颜色选择器是一个组件，您可以将其放置在程序GUI中的任何位置。的JColorChooser API还使打开包含颜色选择器的对话框 （无论是否模态对话框 ）变得容易。

这是使用颜色选择器设置标题中文本颜色的应用程序的图片：

尝试这个： 
  • 单击启动按钮，以使用Java™Web Start （ 下载JDK ）运行ColorChooser演示。另外，要自己编译并运行示例，请参考示例索引 。

该程序的源代码在ColorChooserDemo.java 。

颜色选择器由“ 选择文本颜色 ”框中的所有内容组成。这就是Java Look＆Feel中标准颜色选择器的外观。它包含两个部分，一个选项卡式窗格和一个预览面板。选项卡式窗格中的三个选项卡选择选择器面板 。选项卡式窗格下方的预览面板显示当前选定的颜色。

这是示例中的代码，它创建了一个JColorChooser实例并将其添加到容器中：

public class ColorChooserDemo extends JPanel ... {
    public ColorChooserDemo() {
        super(new BorderLayout());
        banner = new JLabel("Welcome to the Tutorial Zone!",
                            JLabel.CENTER);
        banner.setForeground(Color.yellow);
        . . .
        tcc = new JColorChooser(banner.getForeground());
        . . .
        add(tcc, BorderLayout.PAGE_END);
    }

的JColorChooser上一个代码片段中的构造函数需要一个Color参数，指定选择者最初选择的颜色。如果未指定初始颜色，则显示颜色选择器Color.white 。见Color有关可以使用的颜色常量列表的API文档 。

颜色选择器使用的实例 ColorSelectionModel包含和管理当前选择。每当用户在颜色选择器中更改颜色时，颜色选择模型就会触发更改事件。该示例程序向颜色选择模型注册了一个更改侦听器，以便它可以更新窗口顶部的标题。以下代码注册并实现更改侦听器：

tcc.getSelectionModel().addChangeListener(this);
. . .
public void stateChanged(ChangeEvent e) {
    Color newColor = tcc.getColor();
    banner.setForeground(newColor);
}

有关更改侦听器和更改事件的一般信息，请参见如何编写更改侦听器 。

像示例程序中使用的那样，一种基本的颜色选择器对于许多程序就足够了。但是，颜色选择器API允许您通过为颜色选择器提供自己设计的预览面板，向其添加自己的选择器面板或从颜色选择器中删除现有的选择器面板来自定义颜色选择器。此外， JColorChooser类提供了两种方法，可轻松在对话框中使用颜色选择器。

这是ColorChooserDemo2的图片：

该程序通过以下方式自定义横幅文本颜色选择器：

  • 删除预览面板
  • 删除所有默认选择器面板

该程序还添加了一个在对话框中弹出颜色选择器的按钮，您可以使用该按钮设置横幅背景色。

在对话框中显示颜色选择器

的JColorChooser class提供了两个类方法，可轻松在对话框中使用颜色选择器。ColorChooserDemo2使用以下方法之一， showDialog ，当用户单击“ 显示颜色选择器...”按钮时显示背景颜色选择器 。这是示例中的单行代码，它在对话框中显示背景颜色选择器：

Color newColor = JColorChooser.showDialog(
                     ColorChooserDemo2.this,
                     "Choose Background Color",
                     banner.getBackground());

第一个参数是对话框的父级，第二个参数是对话框的标题，第三个参数是最初选择的颜色。

该对话框在以下三种情况下消失：用户选择一种颜色并单击“ 确定”按钮，用户使用“ 取消”按钮取消该操作，或者使用框架控件关闭该对话框。如果用户选择一种颜色， showDialog方法返回新颜色。如果用户取消操作或关闭窗口，则该方法返回null 。这是示例中的代码，该代码根据的返回值更新横幅背景颜色showDialog ：

if (newColor != null) {
    banner.setBackground(newColor);
}

对话框创建者showDialog是模态的。如果要使用非模式对话框，可以使用JColorChooser的createDialog创建对话框的方法。此方法还允许您为对话框窗口中的“ 确定”和“ 取消”按钮指定动作侦听器。使用JDialog的show方法以显示由此方法创建的对话框。有关使用此方法的示例，请参见“ 如何使用表”部分中的“ 指定其他编辑器 ”。

卸下或装回预览面板

默认情况下，颜色选择器显示预览面板。ColorChooserDemo2使用以下代码行删除文本颜色选择器的预览面板：

tcc.setPreviewPanel(new JPanel());

有效地删除了预览面板，因为JPanel没有大小，也没有默认视图。要将预览面板设置回默认值，请使用null作为论点setPreviewPanel 。

要提供自定义预览面板，您还可以使用setPreviewPanel 。您传递给方法的组件应继承自JComponent ，指定合理的尺寸并提供当前颜色的自定义视图。为了在用户更改颜色选择器中的颜色时得到通知，预览面板必须注册为颜色选择器的颜色选择模型上的更改侦听器， 如前所述 。

创建自定义选择器面板

默认颜色选择器提供五个选择器面板：

  • 色板—用于从色板集合中选择颜色。
  • HSV —用于使用“色相饱和度值”颜色表示法选择颜色。在JDK 7之前，由于色相-饱和度-亮度，它被称为HSB。
  • HSL —用于使用“色调饱和度-亮度”颜色表示法选择颜色。这是JDK 7中的新功能。
  • RGB —用于使用红-绿-蓝颜色模型选择颜色。
  • CMYK-用于使用原色或四色模型选择颜色。这是JDK 7中的新功能。

您可以通过添加您自己设计的选择器面板来扩展默认颜色选择器addChooserPanel ，也可以通过删除选择面板来限制它removeChooserPanel 。

如果要删除所有默认选择器面板并添加一个或多个自己的选择器面板，可以通过一次调用来完成setChooserPanels 。ColorChooserDemo2使用此方法用以下实例替换默认选择器面板： CrayonPanel ，一个自定义选择器面板。这是打给setChooserPanels从该示例：

//Override the chooser panels with our own.
AbstractColorChooserPanel panels[] = { new CrayonPanel() };
tcc.setChooserPanels(panels);

代码很直接：它创建了一个包含CrayonPanel 。接下来的代码调用setChooserPanels将数组的内容设置为颜色选择器的选择器面板。

CrayonPanel是的子类AbstractColorChooserPanel并覆盖其超类中定义的五个抽象方法：

void buildChooser()
创建包含选择器面板的GUI。该示例创建四个切换按钮（每个蜡笔一个），并将它们添加到选择器面板。
void updateChooser()
每当显示选择器面板时，都会调用此方法。此方法的实现选择代表当前选定颜色的切换按钮。 
public void updateChooser() {
    Color color = getColorFromModel();
    if (Color.red.equals(color)) {
        redCrayon.setSelected(true);
    } else if (Color.yellow.equals(color)) {
        yellowCrayon.setSelected(true);
    } else if (Color.green.equals(color)) {
        greenCrayon.setSelected(true);
    } else if (Color.blue.equals(color)) {
        blueCrayon.setSelected(true);
    }
}
String getDisplayName()
返回选择器面板的显示名称。名称在选择器面板的选项卡上使用。这是例子getDisplayName方法：
public String getDisplayName() {
    return "Crayons";
}
Icon getSmallDisplayIcon()
返回一个小图标来表示此选择器面板。目前尚未使用。将来的颜色选择器版本可能会使用此图标或较大的图标来表示显示屏中的该选择器面板。此方法的示例实现返回null 。
Icon getLargeDisplayIcon()
返回一个大图标来表示此选择器面板。目前尚未使用。将来的颜色选择器版本可能会使用此图标或较小的图标来代表显示屏中的该选择器面板。此方法的示例实现返回null 。
