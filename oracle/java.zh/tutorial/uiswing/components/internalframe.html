如何使用内部框架

随着JInternalFrame你可以显示一个类JFrame另一个窗口中的类似窗口。通常，您将内部框架添加到桌面窗格中。反过来，桌面窗格可能会用作桌面的内容窗格。 JFrame 。桌面窗格是的实例JDesktopPane ，这是的子类JLayeredPane添加了用于管理多个重叠内部框架的API。

您应该仔细考虑是将程序的GUI基于框架还是内部框架。从内部框架切换到框架，反之亦然，这不一定是简单的任务。通过对框架和内部框架进行试验，您可以了解选择一个与另一个之间的权衡取舍。

内部框架与常规框架

使用内部框架的代码在许多方面与使用常规Swing框架的代码相似。由于内部框架具有根窗格，因此请为JInternalFrame与为JFrame 。 JInternalFrame还提供了其他API，例如pack ，这使其类似于JFrame 。

注意： 

就像常规框架一样，您必须调用setVisible(true)要么show()在内部框架上显示它。内部框架直到您显式使其可见后才会出现。

但是，内部框架不是窗口或顶级容器，这使其与框架不同。例如，您必须向容器添加内部框架（通常是JDesktopPane ）;内部框架不能是容纳层次结构的根。同样，内部框架不会生成窗口事件。而是，导致框架触发窗口事件的用户操作导致内部框架触发内部框架事件。

由于内部框架是通过与平台无关的代码实现的，因此它们添加了框架无法提供的某些功能。这样的功能之一就是内部框架比框架对您的状态和功能有更多的控制。您可以以编程方式图标化或最大化内部框架。您也可以在内部框架的标题栏中指定要显示的图标。您甚至可以指定内部框架是否具有窗口装饰以支持调整大小，图标化，关闭和最大化。

另一个功能是内部框架旨在在桌面窗格中工作。的JInternalFrame API包含诸如moveToFront仅当内部框架的容器是分层窗格（例如JDesktopPane 。

使用内部框架的规则

如果您使用JFrame和其他的Swing组件，那么您已经知道如何使用内部框架。以下列表总结了使用内部框架的规则。有关更多信息，请参见如何制作框架和JComponent类 。

您必须设置内部框架的尺寸。
如果未设置内部框架的大小，则内部框架的大小将为零，因此永远不可见。您可以使用以下方法之一设置大小： setSize ， pack ， 要么setBounds 。
通常，应设置内部框架的位置。
如果未设置内部框架的位置，则内部框架将位于0,0（其容器的左上角）处。您可以使用setLocation要么setBounds指定内部框架相对于其容器的左上点的方法。
要将组件添加到内部框架，请将它们添加到内部框架的内容窗格。
这就像JFrame情况。有关详细信息，请参见将组件添加到内容窗格 。
内部框架的对话框应使用JOptionPane要么JInternalFrame ，不JDialog 。
要创建一个简单的对话框，您可以使用JOptionPaneshowInternalXxxDialog方法，如如何制作对话框中所述 。
您必须将内部框架添加到容器。
如果您不将内部框架添加到容器中（通常是JDesktopPane ），则不会出现内部框架。
你需要打电话show要么setVisible在内部框架上。
默认情况下，内部框架是不可见的。您必须调用setVisible(true)要么show()使它们可见。
内部框架触发内部框架事件，而不触发窗口事件。
处理内部框架事件与处理窗口事件几乎相同。有关更多信息，请参见如何编写内部框架侦听器 。

性能提示： 

当台式机具有许多内部框架时，用户可能会注意到移动它们似乎很慢。拖动轮廓是避免此问题的一种方法。使用轮廓拖动，在拖动内部框架时，仅在内部鼠标的当前位置绘制内部框架的轮廓。内部框架的内部不会在新的位置重新绘制，直到拖动停止。默认行为（称为实时拖动）是在移动内部框架时连续对其进行重新定位和重新绘制；如果桌面上有许多内部框架，这可能会很慢。

使用JDesktopPane方法setDragMode *指定轮廓拖动。例如：

desktop.setDragMode(JDesktopPane.OUTLINE_DRAG_MODE);

内部框架API

下表列出了常用的JInternalFrame构造函数和方法，以及一些JDesktopPane提供。除了本节中列出的API外， JInternalFrame从其超类继承有用的API， JComponent ， Component和Container 。有关这些类的方法列表，请参见JComponent类。

喜欢JInternalFrame ， JDesktopPane从JComponent ，因此提供了JComponent Class中描述的方法。因为JDesktopPane延伸JLayeredPane ，它还支持The Layered Pane API中描述的方法。

使用内部框架的API分为以下几类：

  • 创建内部框架
  • 向内部框架添加组件
  • 指定内部框架的可见性，大小和位置
  • 在内部框架上执行窗口操作
  • 控制窗户装饰和功能
  • 使用JDesktopPane API
