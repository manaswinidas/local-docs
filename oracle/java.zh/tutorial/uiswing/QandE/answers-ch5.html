答案：编写事件监听器

使用本课程的表格， 组件方法部分和事件侦听器方法部分来完成这些问题和练习。

问题

问题1：当某个特定组件出现在屏幕上时，您将实现哪种监听器来得到通知？哪种方法可以告诉您此信息？
答案1：您要注册一个ComponentListener在组件上。的componentShown方法。第一次显示窗口或取消定义窗口时调用此方法。

问题2：当用户通过按Enter键完成文本字段的编辑后，您将实现什么监听器通知？在将每个字符键入文本字段时，您将实现什么监听器来得到通知？请注意，您不应实现通用的键侦听器，而应实现特定于文本的侦听器。
答案2：要在用户按下Enter键时收到通知，您需要注册一个ActionListener在文本字段上；的actionPerformed当用户键入Enter时，将调用此方法。请注意，Enter字符不属于结果字符串。要在输入每个字符时收到通知，您需要注册一个DocumentListener在文本字段的Document 。的insertUpdate然后在键入每个字符时调用方法。请注意，这不是实现输入验证的正确方法。对于这种行为，您应该查看“ 如何使用Focus子系统”中的“ 输入验证API”部分。

问题3：当微调器的值更改时，您将实现哪种侦听器通知？您将如何获得微调器的新价值？
答案3：要在值更改时收到通知，您可以注册一个ChangeListener在微调器上。您将通过事件源获取新值。 stateChanged方法。以下代码段显示了如何完成此操作：

public void stateChanged(ChangeEvent e) {
    JSpinner mySpinner = (JSpinner)(e.getSource());
    SpinnerNumberModel model = (SpinnerNumberModel)(mySpinner.getModel());
    Number currentValue = model.getNumber();
    ...
}

问题4：焦点子系统的默认行为是使用焦点遍历键，例如Tab和Shift Tab。假设您要在应用程序的一个组件中防止此行为。您将如何实现？
答案4：您致电setFocusTraversalKeysEnabled(false)在该特定组件上。请注意，然后必须手动处理遍历。有关更多信息，请参见如何编写密钥侦听器和如何使用Focus子系统 。
