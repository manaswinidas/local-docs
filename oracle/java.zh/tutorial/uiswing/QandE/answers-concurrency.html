答案：Swing中的并发

问题

问题1：对于以下每个任务，请指定应在哪个线程中执行以及为什么执行该线程。
答1：

  • 初始化GUI。事件调度线程；与GUI框架的大多数交互都必须在此线程上进行。
  • 加载大文件。工作线程。在事件分发线程上执行此任务将阻止处理GUI事件，从而“冻结” GUI，直到任务完成为止。在初始线程上执行此任务会导致创建GUI的延迟。
  • 调用中javax.swing.JComponent.setFont更改组件的字体。事件调度线程。与大多数Swing方法一样，调用它并不安全setFont从任何其他线程。
  • 调用中javax.swing.text.JTextComponent.setText更改组件的文本。该方法记录为线程安全的，因此可以从任何线程中调用。

问题2：在上一个问题中提到的任何任务中，一个线程都不是首选线程。为该线程命名，并解释其用途如此有限的原因。
答案2：初始线程在事件分发线程上启动第一个GUI任务。之后，Swing程序主要由GUI事件驱动，该GUI事件触发事件分配线程和辅助线程上的任务。通常，初始线程不做任何事情。

问题3： SwingWorker有两个类型参数。说明如何使用这些类型参数，以及为什么它们通常无关紧要。
答案3：类型参数指定最终结果的类型（以及返回值的返回类型） doInBackground方法）和中间结果的类型（以及publish和process ）。许多后台任务无法提供最终或临时结果。

练习题

问题1：修改Flipper使其在“硬币翻转”之间暂停5秒。如果用户单击“取消”，则抛硬币循环立即终止。
答案1：查看源代码Flipper2 。修改后的程序在中央添加了一个延迟doInBackground环：

protected Object doInBackground() {
    long heads = 0;
    long total = 0;
    Random random = new Random();
    while (!isCancelled()) {
        try {
            Thread.sleep(5000);
        } catch (InterruptedException e) {
            //Cancelled!
            return null;
        }
        total++;
        if (random.nextBoolean()) {
            heads++;
        }
        publish(new FlipPair(heads, total));
    }
    return null;
}

的try ... catch原因doInBackground如果线程正在休眠时收到中断，则返回。调用中cancel带有true确保在取消任务时发送中断。
