如何在对话框中使用模态

Java™SE 6解决了平台早期版本中出现的模式问题。新的模式模型使开发人员可以确定或限制对话框的模式阻止。

在继续使用新的模式模型之前，请查看以下术语：

  • 对话框-带有标题和边框的顶级弹出窗口，通常需要用户输入某种形式。对话框可以是模式对话框，也可以是非模式对话框。有关对话框的更多信息，请参见“ 如何制作对话框”页面中的对话框概述 。
  • 模态对话框—阻止输入到应用程序中其他一些顶级窗口的对话框，但以该对话框作为其所有者创建的窗口除外。模态对话框将捕获窗口焦点，直到窗口焦点关闭为止，通常是响应按钮的按下。
  • 无模式对话框-一个对话框，使您可以在显示该对话框时与其他窗口一起使用。

在Java SE 6中，模式对话框和无模式对话框的行为都已更改，因此它们不仅总是出现在其父窗口以及所有阻止的窗口的顶部。

Java SE 6支持以下模式类型：

  • 无模式类型 -无模式对话框在可见时不会阻止任何其他窗口。
  • 文档模式类型 —文档模式对话框会阻止同一文档中的所有窗口，但其子层次结构中的窗口除外。在这种情况下，文档是共享公共祖先的窗口层次结构，称为文档根，这是没有所有者的最接近的祖先窗口。
  • 应用程序模式类型 -应用程序模式对话框将阻止来自同一应用程序的所有窗口，但其子层次结构中的窗口除外。如果在浏览器环境中启动了多个小程序，则允许浏览器将它们视为单独的应用程序或单个应用程序。此行为取决于实现。
  • 工具箱模式类型 -工具箱模式对话框会阻止在同一工具箱中运行的所有窗口，但其子层次结构中的窗口除外。如果启动了多个小程序，则所有小程序都将使用相同的工具箱运行。因此，从小程序显示的工具箱模式对话框可能会影响其他小程序以及嵌入该工具箱Java运行时环境的浏览器实例的所有窗口。

此外，您可以设置模态排除模式：

  • 排除模式 -可以将任何顶级窗口标记为不被模式对话框阻止。此属性使您可以设置模式排除模式。的Dialog.ModalExclusionType枚举指定可能的模式排除类型。
注意：新的模式模型没有实现系统模式，该系统模式会在模式对话框处于活动状态时阻止显示在桌面上的所有应用程序（包括Java应用程序）。

尝试这个： 
  1. 单击启动按钮，以使用Java™Web Start （ 下载JDK 7或更高版本 ）运行ModalityDemo。另外，要自己编译并运行示例，请参考示例索引 。
  2. 将出现以下对话框：
    • 书1（父级画框）
    • 书籍2（父系）
    • 反馈（父框架）
    • 经典作品（不含镜架）
  3. 切换到“书本1”框架并选择书的传记标题，然后选择“确定”。
  4. 传记标题将显示在对话框的标题中。在文本字段中输入名称，例如“ John”。
  5. 切换到Book 1框架，然后将标题更改为Funny Tale，然后选择OK。由于输入名称的对话框是无模式的 ，因此您可以轻松切换到其父框架。
    注意：无模式对话框标题已更改为“滑稽故事”。
  6. 在无模式对话框中选择确定。
  7. 出现童话文档模式对话框。
  8. 在文本字段中输入一些文本。请注意，它由您在无模式对话框中输入的名称签名。
  9. 切换到无模式对话框，然后尝试更改您的名称。您将无法执行此操作，因为文档模式对话框会阻止其父层次结构中的所有窗口。
  10. 对Book 2父框架执行相同的操作序列（步骤3-9）。
  11. 尝试切换到其他对话框。您会注意到，可以切换到“经典”框架或“反馈”框架，也可以切换到“书本1”框架或“书本2”框架的对话框。
  12. 切换到“反馈”父框架。选择评价自己。
  13. 出现确认对话框。尝试切换到其他对话框。因为标准确认对话框是应用程序模式对话框，并且阻止了同一应用程序中的所有窗口，所以您只能切换到“经典”对话框。但是，您会注意到，您可以在“古典”框架中选择自己喜欢的古典作者。该框架是通过使用APPLICATION_EXCLUDE模态排除类型，可防止所有顶级窗口被任何应用程序模态对话框阻止。

以下代码段显示了如何创建不同模式类型的对话框：

//The Book 1 parent frame
f1 = new JFrame("Book 1 (parent frame)");

...

//The modeless dialog box
d2 = new JDialog(f1);

...
        
//The document-modal dialog box
d3 = new JDialog(d2, "", Dialog.ModalityType.DOCUMENT_MODAL);

...

        //The Book2 parent frame
f4 = new JFrame("Book 2 (parent frame)");

...

//The modeless dialog box
d5 = new JDialog(f4);

...

//The document-modality dialog box
d6 = new JDialog(d5, "", Dialog.ModalityType.DOCUMENT_MODAL);
        
...

//The excluded frame
f7 = new JFrame("Classics (excluded frame)");
f7.setModalityExclusionType(Dialog.ModalExclusionType.APPLICATION_EXCLUDED);
        
...

//The Feedback parent frame and Confirm Dialog box
f8 = new JFrame("Feedback (parent frame)");
...

JButton b8 = new JButton("Rate yourself");
b8.addActionListener(new ActionListener() {
    public void actionPerformed(ActionEvent e) {
        JOptionPane.showConfirmationDialog(null,
                                           "I really like my book",
                                           "Question (application-modal dialog)", 
                                           JOptionPane.Yes_NO_OPTION,
                                           JOptionPane.QUESTION_MESSAGE); 
    }
});

模态API

的JDialog使用类构造函数可以创建各种模式类型的对话框。

建设者 目的
JDialog（对话框所有者） 创建一个具有指定模式的无模式对话框Dialog所有者，但没有标题。
JDialog（对话框所有者，布尔模式） 创建一个具有指定对话框Dialog所有者和模态。
JDialog（对话框所有者，字符串标题） 创建一个具有指定模式的无模式对话框Dialog所有者和头衔。
JDialog（对话框所有者，字符串标题，布尔模式） 创建一个具有指定对话框Dialog所有者，标题和方式。
JDialog（对话框所有者，字符串标题，布尔模式，GraphicsConfiguration gc） 创建一个具有指定对话框Dialog所有者，标题，形式和图形配置。
JDialog（框架所有者） 创建一个无模式对话框，不带指定标题Frame所有者。如果所有者的值为null，则将共享的隐藏框架设置为对话框的所有者。
JDialog（窗口所有者，字符串标题，对话框。ModalityType modalityType） 创建一个具有指定对话框Window所有者，标题和方式。
