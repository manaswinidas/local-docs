如何创建启动画面

几乎所有现代应用程序都有启动屏幕。通常，初始屏幕用于以下目的：

  • 广告产品
  • 向用户指示应用程序正在长时间启动
  • 提供每次访问仅需要一次的信息

Swing和抽象窗口工具包（AWT）的Java基础类使开发人员能够在Java技术应用程序中创建初始屏幕。但是，由于启动屏幕的主要目的是向用户提供有关应用程序启动的反馈，因此，应用程序启动与启动屏幕弹出之间的延迟应该最小。在弹出初始屏幕之前，应用程序必须加载并初始化Java™虚拟机（JVM），AWT，Swing以及有时还依赖于应用程序的库。所产生的几秒钟的延迟使基于Java™技术的启动屏幕的使用变得不那么理想。

幸运的是，Java™SE 6提供了一种解决方案，允许应用程序甚至在虚拟机启动之前就更早地显示启动屏幕。Java应用程序启动器能够解码图像并将其显示在简单的非装饰窗口中。

初始屏幕可以显示任何gif ， png ， 要么jpeg图像，具有透明度，半透明性和动画效果。下图显示了以动画形式开发的Java应用程序初始屏幕的示例gif文件。

的SplashScreen类用于关闭初始屏幕，更改初始屏幕图像，获取图像位置或大小以及在初始屏幕中绘画。应用程序无法创建此类的实例。只能在此类中创建一个实例，并且可以使用getSplashScreen()静态方法。如果应用程序在启动时未通过命令行或清单文件选项创建启动画面，则getSplashScreen方法返回null。

通常，开发人员希望将初始屏幕图像保留在屏幕上，并在图像上显示一些内容。初始屏幕窗口具有带alpha通道的覆盖表面，可以使用传统方法访问该表面Graphics2D接口。

以下代码段显示了如何获取SplashScreen对象，然后如何使用createGraphics()方法：

...
        final SplashScreen splash = SplashScreen.getSplashScreen();
        if (splash == null) {
            System.out.println("SplashScreen.getSplashScreen() returned null");
            return;
        }
        Graphics2D g = splash.createGraphics();
        if (g == null) {
            System.out.println("g is null");
            return;
        }
...

在中找到演示的完整代码SplashDemo.java文件。

注意： 

SplashDemo应用程序使用固定的坐标来显示叠加信息。这些坐标取决于图像，并针对每个启动画面分别计算。

本机启动屏幕可以通过以下方式显示：

  • 命令行参数
  • 具有指定清单选项的Java™存档（JAR）文件

如何使用命令行参数显示启动画面

要从命令行显示启动画面，请使用-splash:命令行参数。此参数是显示初始屏幕的Java应用程序启动器选项：

java -splash:<file name> <class name>
尝试这个： 
  1. 编译SplashDemo.java文件。
  2. 保存splash.gif中的图片images目录。
  3. 使用以下参数从命令行运行应用程序：
    java -splash:images/splash.gif SplashDemo
  4. 等待直到初始屏幕已完全显示。
  5. 出现应用程序窗口。要关闭窗口，请从弹出菜单中选择File | Exit或单击X。
  6. 将初始屏幕名称更改为不存在的图像，例如， nnn.gif 。运行应用程序，如下所示：
    java -splash:images/nnn.gif SplashDemo
  7. 您将看到以下输出字符串：
    SplashScreen.getSplashScreen() returned null

如何使用JAR文件显示启动画面

如果您的应用程序打包在JAR文件中，则可以使用SplashScreen-Image清单文件中的选项显示初始屏幕。将图像放在JAR文件中，并在选项中指定路径，如下所示：

Manifest-Version: 1.0
Main-Class: <class name>
SplashScreen-Image: <image name>
尝试这个： 
  1. 编译SplashDemo.java文件。
  2. 保存splash.gif中的图片images目录。
  3. 准备splashmanifest.mf文件如下：
    Manifest-Version: 1.0
Main-Class: SplashDemo
SplashScreen-Image: images/splash.gif
  4. 使用以下命令创建一个JAR文件：
    jar cmf splashmanifest.mf splashDemo.jar SplashDemo*.class images/splash.gif
    有关JAR文件的更多信息，请参阅“ JAR文件中的打包程序”页面中的“ 使用JAR文件 ” 。
  5. 运行应用程序：
    java -jar splashDemo.jar
  6. 请等待，直到初始屏幕已完全显示。
  7. 出现应用程序窗口。要关闭窗口，请从弹出菜单中选择File | Exit或单击X。

开机画面API

的SplashScreen类不能用于创建初始屏幕。在此类中创建的单个实例只能存在。

方法 目的
getSplashScreen（） 返回SplashScreen用于Java启动初始屏幕控制的对象。
createGraphics（） 创建图形上下文（作为Graphics2D对象）作为初始屏幕重叠图像，使您可以在初始屏幕上进行绘制。
getBounds（） 将初始屏幕窗口的边界返回为Rectangle 。
关（） 关闭启动画面并释放所有关联的资源。
