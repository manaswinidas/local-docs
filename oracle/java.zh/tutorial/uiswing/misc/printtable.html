如何打印表格

的JTable类提供了对打印表的支持。的JTable打印API包括一些方法，这些方法使您可以实现基本和高级打印任务。对于常见的打印任务，当您只需要打印表格时，请使用print直接方法。的print方法有几种形式，各种参数集。此方法准备您的表，获取对应的Printable对象，并将其发送到打印机。

如果默认执行Printable对象不符合您的需求，您可以通过覆盖getPrintable包装默认值的方法Printable甚至完全取代它。

打印表格的最简单方法是调用print没有参数的方法。请参见下面的代码示例。

try {
    boolean complete = table.print();
    if (complete) {
        /* show a success message  */
        ...
    } else {
        /*show a message indicating that printing was cancelled */
        ...
    }
} catch (PrinterException pe) {
    /* Printing failed, report to the user */
    ...
}

当您致电print没有参数的方法，将显示一个打印对话框，然后在FIT_WIDTH模式没有页眉或页脚。下面的代码示例显示了print具有完整参数集的方法签名。

boolean complete = table.print(JTable.PrintMode printMode,
                               MessageFormat headerFormat,
                               MessageFormat footerFormat, 
                               boolean showPrintDialog,
                               PrintRequestAttributeSet attr,
                               boolean interactive,
                               PrintService service);

当您致电print带有所有参数的方法，您可以明确选择打印功能，例如打印模式，页眉和页脚文本，打印属性，目标打印服务，以及是否显示打印对话框，以及是否以交互方式打印-互动地。要确定最适合您需要的参数，请参阅下面的可用功能说明。

的JTable打印API提供以下功能：

  • 交互式或非交互式打印
  • 显示打印对话框
  • 将页眉或页脚（或两者）添加到打印版式
  • 选择打印模式
  • 自动布局和分页

交互式或非交互式打印

在交互模式下，将在打印期间显示带有中止选项的进度对话框。这是进度对话框的示例。

该对话框使用户可以跟踪打印进度。进度对话框是模式对话框，这意味着，尽管对话框显示在屏幕上，但用户无法与表格进行交互。重要的是您的表在打印时保持不变，否则打印行为将不确定。但是，交互式打印不会阻止其他开发人员的代码更改表格。例如，还有另一个线程使用SwingUtilities.invokeLater方法。因此，为确保正确的打印行为，您应确保自己的代码不会在打印过程中修改表。

或者，您可以非交互式地打印表格。在这种模式下，打印立即在事件分发线程上开始，并完全阻止所有要处理的事件。一方面，此模式可以安全地防止表格发生任何更改，直到完成打印为止。另一方面，此模式完全剥夺了用户与GUI的任何交互。因此，仅当从具有不可见GUI的应用程序进行打印时，才建议非交互式打印。

打印对话框

您可以显示一个标准的打印对话框，该对话框允许用户执行以下操作：

  • 选择打印机
  • 指定份数
  • 更改打印属性
  • 开始打印之前取消打印
  • 开始列印

您可能会注意到，“打印”对话框未指定打印输出中的总页数。这是因为表打印实现使用Printable在打印之前，不知道API和页面总数。

将页眉或页脚（或两者）添加到打印版式

页眉和页脚由MessageFormat参数。这些参数允许对页眉和页脚进行本地化。阅读有关的文档MessageFormat类，因为某些字符（例如单引号）是特殊的，需要避免。页眉和页脚都居中。您可以使用{0}插入页码。

MessageFormat footer = new MessageFormat("Page - {0}");

由于在打印之前输出的总页数未知，因此无法指定编号格式，例如“第1页，共5页”。

列印模式

打印模式负责缩放输出并将其分布在页面上。您可以通过以下方式之一打印表格：

  • PrintMode.NORMAL
  • PrintMode.FIT_WIDTH

在里面NORMAL模式，表格以其当前大小打印。如果列不适合页面，则它们会根据表格的内容分布在其他页面上ComponentOrientation 。在里面FIT_WIDTH模式下，如果需要，表的大小较小，以适合每一页上的所有列。请注意，宽度和高度都按比例缩放以提供相同纵横比的输出。在两种模式下，行依次分布在多个页面上，并且页面上的行越多越好。

自动布局和分页

随着使用JTable打印API时，您无需注意布局和分页。您只需要为print方法，例如打印模式和页脚文本格式（如果要在页脚中插入页码）。如前所述，您可以在页脚中指定页码，方法是： "{0}"在给定的字符串中MessageFormat页脚参数。在打印输出中，{0}将被当前页码替换。

TablePrintDemo2示例

的TablePrintDemo2示例基于之前的演示，并且具有相同的界面。唯一的区别在于打印输出。如果您更仔细地查看TablePrintDemo1的打印结果，您可能会注意到对勾和X标记模糊。的TablePrintDemo2示例显示了如何自定义表格以使图像在表格打印输出中更具区别性。在此演示中，已覆盖getTableCellRendererComponent方法找出表格是否正在打印，并返回更清晰的黑白图像。如果未打印表格，它将返回您可以在屏幕上看到的彩色图像。

单击启动按钮以使用Java™Web Start （ 下载JDK 7或更高版本 ）运行TablePrintDemo2。另外，要自己编译并运行示例，请参考示例索引 。


的isPaintingForPrint方法中定义的JComponent与我们在屏幕上看到的内容相比，class允许我们自定义打印的内容。自定义单元格渲染器的代码，摘自TablePrintDemo2.java ，在下面列出。这段程式码根据传回的值选择要使用的影像isPaintingForPrint方法。

    /**
     * A custom cell renderer that extends TablePrinteDemo1's renderer, to instead
     * use clearer black and white versions of the icons when printing.
     */
    protected static class BWPassedColumnRenderer extends PassedColumnRenderer {
            public Component getTableCellRendererComponent(JTable table,
                                                           Object value,
                                                           boolean isSelected,
                                                           boolean hasFocus,
                                                           int row,
                                                           int column) {

            super.getTableCellRendererComponent(table, value, isSelected,
                                                hasFocus, row, column);

            /* if we're currently printing, use the black and white icons */
            if (table.isPaintingForPrint()) {
                boolean status = (Boolean)value;
                setIcon(status ? passedIconBW : failedIconBW);
            } /* otherwise, the superclass (colored) icons are used */

            return this;
        }
    }

该程序的完整代码可以在下面找到TablePrintDemo3.java 。在此演示中， JTable该类被称为FancyPrintingJTable 。这个FancyPrintingJTable类覆盖getPrintable返回一个自定义可打印对象的方法，该对象用其自己的装饰以及页眉和页脚包装默认可打印对象。这是实施getPrintable方法。

public Printable getPrintable(PrintMode printMode,
                              MessageFormat headerFormat,
                              MessageFormat footerFormat) {

     MessageFormat pageNumber = new MessageFormat("- {0} -");

     /* Fetch the default printable */
     Printable delegate = super.getPrintable(printMode, null, pageNumber);

     /* Return a fancy printable that wraps the default */
     return new FancyPrintable(delegate);
}

的FancyPrintable类负责将默认可打印对象包装到另一个可打印对象中，并设置剪贴板图像。实例化此类的实例时，它将加载组装剪贴板图像所需的图像，计算剪贴板图像所需的面积，计算表格的缩小区域，将表格打印到较小的区域，然后组装并打印表格。剪贴板图像。

表格打印API

本节列出了JTable允许您打印表格的类。

方法 目的
布尔值print（）
布尔值print（printMode）
布尔值print（printMode，MessageFormat，MessageFormat）
布尔值打印（printMode，MessageFormat，MessageFormat，布尔值，PrintRequestAttributeSet，布尔值）
布尔值打印（printMode，MessageFormat，MessageFormat，布尔值，PrintRequestAttributeSet，布尔值，PrintService）		 在不带参数的情况下调用时，将显示一个打印对话框，然后在FIT_WIDTH模式，没有页眉或页脚文本。退货true如果用户继续打印并且false如果用户取消打印。

每当一个PrintService省略参数，将使用默认打印机。
