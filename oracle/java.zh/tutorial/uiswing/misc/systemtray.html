如何使用系统托盘

系统托盘是桌面的专用区域，用户可以在其中访问当前正在运行的程序。在各种操作系统上，可能会以不同的方式引用此区域。在Microsoft Windows上，系统托盘称为“任务栏状态区域”，而在GNU网络对象模型环境（GNOME）桌面上则称为“通知区域”。在K Desktop Environment（KDE）上，此区域称为系统托盘。但是，在每个系统上，托盘区域由桌面上运行的所有应用程序共享。

的java.awt.SystemTray Java™SE版本6中引入的类代表了台式机的系统托盘。可以通过调用静态文件来访问系统托盘SystemTray.getSystemTray()方法。在调用此方法之前，请使用静态方法isSupported()检查系统托盘是否受支持。如果此平台不支持系统托盘，则isSupported()方法返回false。如果应用程序尝试调用getSystemTray()在这种情况下，该方法将抛出一个java.lang.UnsupportedOperationException 。

应用程序无法创建SystemTray类。只能在此类中创建一个实例，并且可以使用getSystemTray()方法。

系统托盘包含一个或多个托盘图标，可使用add(java.awt.TrayIcon)方法。当不再需要它们时，可以将它们删除。 remove(java.awt.TrayIcon)方法。

注意： add()方法可以抛出AWTException如果操作系统或Java运行时确定无法将图标添加到系统托盘中。例如， AWTException如果桌面上不存在系统托盘，X-Window桌面将抛出该图像。

的TrayIcon类功能超出了托盘中显示的图标。它还包括文本工具提示，弹出菜单，气球消息以及与之关联的一组侦听器。一种TrayIcon对象生成各种鼠标事件，并支持添加相应的侦听器以接收这些事件的通知。的TrayIcon类处理一些事件本身。例如，默认情况下，在任务栏图标上单击鼠标右键时，它将显示指定的弹出菜单。双击后， TrayIcon对象生成一个ActionEvent启动一个应用程序。当鼠标指针悬停在任务栏图标上时，将显示工具提示。图标图像会自动调整大小以适合在纸盘上为图像分配的空间。

SystemTray API

在其中仅创建一个实例SystemTray类可以存在。

方法 目的
加 将托盘图标添加到系统托盘。添加后，托盘图标在系统托盘中变得可见。没有指定图标在托盘中的显示顺序-它取决于平台和实现。
getSystemTray 得到SystemTray代表桌面托盘区域的实例。此方法始终为每个应用程序返回相同的实例。在某些平台上，可能不支持系统托盘。使用isSupported()检查系统托盘是否受支持的方法。
isSupported 返回有关当前平台是否支持系统托盘的信息。除了显示任务栏图标外，对系统托盘的最小支持还包括一个弹出菜单（请参阅TrayIcon.setPopupMenu(PopupMenu)方法）或动作事件（请参见TrayIcon.addActionListener(ActionListener) ）。

TrayIcon API

一种TrayIcon对象表示可以添加到系统托盘的托盘图标。一种TrayIcon对象可以具有工具提示（文本），图像，弹出菜单以及与之关联的一组侦听器。

方法 目的
setImageAutoSize 设置自动调整大小属性。自动调整大小确定纸盘图像是否自动调整大小以适合为纸盘上的图像分配的空间。默认情况下，auto-size属性设置为false 。
setPopupMenu 为此设置弹出菜单TrayIcon宾语。如果弹出null ，则不会与此菜单关联任何弹出菜单TrayIcon宾语。
setToolTip 为此设置工具提示字符串TrayIcon宾语。当鼠标悬停在图标上时，将自动显示工具提示。将工具提示设置为null删除任何工具提示文本。显示时，工具提示字符串在某些平台上可能会被截断；可能显示的字符数取决于平台。
