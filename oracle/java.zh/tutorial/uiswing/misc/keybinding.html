如何使用键绑定

的JComponent类支持键绑定，作为对用户键入的单个键作出响应的一种方式。以下是一些适当的键绑定示例：

  • 您正在创建一个自定义组件，并希望支持对其的键盘访问。
    例如，您可能希望组件在获得焦点并且用户按下Space键时做出反应。
  • 您要覆盖现有键绑定的行为。
    例如，如果您的应用程序通常以特定的方式对按F2键作出反应，则您可能希望它执行其他操作或忽略按键。
  • 您想为现有操作提供新的键绑定。
    例如，您可能强烈感到Control-Shift-Insert应该执行粘贴操作。

您通常不需要直接使用键绑定。助记符（在所有按钮，选项卡式窗格以及JLabel ）和加速器（受菜单项支持）。您可以在“ 启用键盘操作 ”部分中找到助记符和加速器的内容。

键绑定的替代方法是使用键侦听器 。按键侦听器在键盘输入的底层接口中占有一席之地，但是对于响应各个按键，按键绑定更为合适，并且往往会导致代码维护更加容易。如果要在组件没有焦点时激活键绑定，则键侦听器也很困难。按键绑定的一些优点是：它们可以自我记录，考虑容纳层次结构，鼓励可重用的代码块（ Action对象），并允许轻松删除，自定义或共享操作。而且，它们使更改动作绑定的键变得容易。动作的另一个优点是它们具有启用状态，该状态提供了一种简便的方法来禁用该动作而不必跟踪其附加到哪个组件。

按键绑定如何工作

提供的键绑定支持JComponent依靠InputMap和ActionMap类。输入地图结合击键到动作名称和动作映射指定的动作对应于每个操作名。从技术上讲，您不需要在地图中使用动作名称；您可以将任何对象用作地图的“键”。但是，按照约定，您使用一个字符串来命名一个动作。

每InputMap / ActionMap有一个通常来自用户界面的父项。每当外观改变时，都会重置父级。这样，开发人员指定的任何绑定都不会在外观更改时丢失。

每JComponent有一个动作图和三个输入图。输入映射对应于以下焦点情况：

JComponent.WHEN_FOCUSED
该组件具有键盘焦点。的WHEN_FOCUSED输入映射通常在组件没有子级时使用。例如，按钮使用WHEN_FOCUSED地图。
这些绑定仅在组件具有焦点时才有效。
JComponent.WHEN_ANCESTOR_OF_FOCUSED_COMPONENT
该组件包含（或是）具有焦点的组件。此输入映射通常用于复合组件，该组件的实现取决于子组件。例如， JTable使用以下命令进行所有绑定WHEN_ANCESTOR_OF_FOCUSED_COMPONENT因此，如果用户正在编辑，例如，向上箭头键仍会更改所选单元格。
JComponent.WHEN_IN_FOCUSED_WINDOW
组件的窗口要么具有焦点，要么包含具有焦点的组件。此输入映射通常用于助记符或加速器，无论焦点在窗口中的什么位置，它们都必须处于活动状态。

当用户键入密钥时， JComponent键事件处理代码通过一个或多个输入映射进行搜索，以找到该键的有效绑定。找到绑定后，它将在操作映射中查找相应的操作。如果启用了操作，则绑定有效并且执行该操作。如果已禁用，则继续搜索有效的绑定。

如果该键存在多个绑定，则仅使用找到的第一个有效绑定。输入映射按以下顺序检查：

  1. 重点组件的WHEN_FOCUSED输入地图。
  2. 重点组件的WHEN_ANCESTOR_OF_FOCUSED_COMPONENT输入地图。
  3. 的WHEN_ANCESTOR_OF_FOCUSED_COMPONENT输入焦点组件的父级的映射，然后是其父级的父级的映射，依此类推，从而继续包含层次结构。注意：将跳过禁用组件的输入映射。
  4. 的WHEN_IN_FOCUSED_WINDOW搜索聚焦窗口中所有已启用组件的输入映射。由于搜索组件的顺序是不可预测的，请避免重复WHEN_IN_FOCUSED_WINDOW绑定！

让我们考虑一下在两种典型的键绑定情况下发生的情况：一个按钮对Space键作出反应，以及一个带有默认按钮的框架对Enter键作出反应。

在第一种情况下，假设用户在按下JButton具有键盘焦点。首先，将事件通知给按钮的键侦听器。关键听众的假设没有消耗事件（通过调用consume上的方法KeyEvent ）按钮的WHEN_FOCUSED请参考输入图。找到绑定是因为JButton使用该输入映射将Space绑定到动作名称。在按钮的动作图中查找动作名称，然后actionPerformed该操作的方法被调用。的KeyEvent被消耗，并且处理停止。

在第二种情况下，假设当焦点位于具有默认按钮（使用JRootPane setDefaultButton方法）。无论关注的组件是什么，都将首先通知其关键侦听器。假设它们都不消耗关键组件的关键事件WHEN_FOCUSED请参考输入图。如果它没有与键的绑定，或者禁用了与键绑定的操作，则焦点组件的WHEN_ANCESTOR_OF_FOCUSED_COMPONENT咨询输入映射，然后（如果未找到绑定或禁用绑定到键的操作） WHEN_ANCESTOR_OF_FOCUSED_COMPONENT包含层次结构中每个组件祖先的输入映射。最终，根窗格的WHEN_ANCESTOR_OF_FOCUSED_COMPONENT搜索输入图。由于该输入映射具有Enter的有效绑定，因此将执行该操作，从而导致单击默认按钮。

如何制作和删除按键绑定

这是指定组件应对F2键做出反应的示例：

component.getInputMap().put(KeyStroke.getKeyStroke("F2"),
                            "doSomething");
component.getActionMap().put("doSomething",
                             anAction);
//where anAction is a javax.swing.Action

如前面的代码所示，要获取组件的操作图，请使用getActionMap方法（继承自JComponent ）。要获取输入地图，您可以使用getInputMap(int)方法，其中整数是其中之一JComponent.WHEN_*FOCUSED*前面列表中显示的常量。或者，在通常情况下，常数为JComponent.WHEN_FOCUSED ，您可以使用getInputMap没有参数。

要将条目添加到其中一张地图，请使用put方法。您可以使用KeyStroke对象，您可以使用KeyStroke.getKeyStroke(String)方法。您可以找到创建Action （以放置动作图）中的“ 如何使用动作” 。

这是一个稍微复杂的示例，该示例指定组件应该对Space键做出反应，就像用户单击鼠标一样。

component.getInputMap().put(KeyStroke.getKeyStroke("SPACE"),
                            "pressed");
component.getInputMap().put(KeyStroke.getKeyStroke("released SPACE"),
                            "released");
component.getActionMap().put("pressed",
                             pressedAction);
component.getActionMap().put("released",
                             releasedAction);
//where pressedAction and releasedAction are javax.swing.Action objects

要使组件忽略其通常响应的键，可以使用特殊操作名称“ none”。例如，以下代码使组件忽略F2键。

component.getInputMap().put(KeyStroke.getKeyStroke("F2"),
                            "none");
注意： 

前面的代码不会阻止相关WHEN_ANCESTOR_OF_FOCUSED_COMPONENT和WHEN_IN_FOCUSED_WINDOW输入地图以搜索F2键绑定。为了防止这种搜索，您必须使用有效的动作而不是“ none”。例如：

Action doNothing = new AbstractAction() {
    public void actionPerformed(ActionEvent e) {
        //do nothing
    }
};
component.getInputMap().put(KeyStroke.getKeyStroke("F2"),
                            "doNothing");
component.getActionMap().put("doNothing",
                             doNothing);
