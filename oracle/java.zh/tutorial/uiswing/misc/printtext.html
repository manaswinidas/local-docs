如何列印文字

的JTextComponent该类提供了对打印文本文档的支持。的JTextComponent API包含一些方法，这些方法使您可以实现基本和高级打印任务。支持的格式包括HTML，RTF和纯文本。对于常见的打印任务，例如仅打印文本文档，请使用print直接方法。的print方法有几种形式，各种参数集。该方法准备您的文本文件，获取对应的Printable对象，并将其发送到打印机。

如果默认执行Printable对象不符合您的需求，您可以通过覆盖getPrintable包装默认值的方法Printable甚至完全取代它。

打印文本组件最简单的方法是调用print没有参数的方法。请参见下面的代码示例。

try {
    boolean complete = textComponent.print();
    if (complete) {
        /* show a success message  */
        ...
    } else {
        /*show a message indicating that printing was cancelled */
        ...
    }
} catch (PrinterException pe) {
    /* Printing failed, report to the user */
    ...
}

当您致电print没有参数的方法，将显示一个打印对话框，然后以无页眉或页脚的方式交互式打印您的文本组件。下面的代码示例显示了print具有完整参数集的方法签名。

boolean complete = textComponent.print(MessageFormat headerFormat,
                                       MessageFormat footerFormat, 
                                       boolean showPrintDialog,
                                       PrintService service
                                       PrintRequestAttributeSet attributes,
                                       boolean interactive);

当您致电print带有所有参数的方法，您可以显式选择打印功能，例如页眉和页脚文本，打印属性，目标打印服务，以及是否显示打印对话框以及是交互式还是非交互式打印。要确定最适合您需要的参数，请参阅下面的可用功能说明。

的JTextComponent打印API提供以下功能：

  • 交互式或非交互式打印
  • 显示打印对话框
  • 将页眉或页脚（或两者）添加到打印版式
  • 自动布局和分页

交互式或非交互式打印

在交互模式下，将在打印期间显示带有中止选项的进度对话框。这是进度对话框的示例。

该对话框允许用户跟踪打印进度。当print在事件分配线程上调用方法，否则在非模态上调用。重要的是您的文档在打印时保持不变，否则打印行为是不确定的。的print方法可确保您的文档不会被更改，并在打印期间禁用该组件。

如果您致电print非交互模式下事件分发线程上的方法，则所有事件（包括重绘）都将被阻止。这就是为什么只建议对具有不可见GUI的应用程序在EDT上进行非交互式打印的原因。

打印对话框

您可以显示一个标准的打印对话框，该对话框允许用户执行以下操作：

  • 选择打印机
  • 指定份数
  • 更改打印属性
  • 开始打印之前取消打印
  • 开始列印

您可能会注意到，“打印”对话框未指定打印输出中的总页数。这是因为文本打印实现使用Printable在打印之前，不知道API和页面总数。

将页眉或页脚（或两者）添加到打印版式

页眉和页脚由MessageFormat参数。这些参数允许对页眉和页脚进行本地化。阅读有关的文档MessageFormat类（例如单引号）是特殊的，需要避免。页眉和页脚都居中。您可以使用插入页码{0} 。

MessageFormat footer = new MessageFormat("Page - {0}");

由于在打印之前输出的总页数未知，因此无法指定编号格式，例如“第1页，共5页”。

自动布局和分页

随着使用JTextComponent打印API时，您无需注意布局和分页。布局和分页都是自动完成的。文档内容的格式被设置为适合页面大小，并分布在多个页面上。您只需要指定适当的页脚文本格式即可print如果您要在页脚中插入页码，请使用此方法。如前所述，您可以在页脚中指定页码，方法是： "{0}"在给定的字符串中MessageFormat页脚参数。在打印输出中，{0}将被当前页码替换。

每当网络启动的应用程序尝试打印时，Java Web Start都会打开一个安全对话框，询问用户打印许可，除非已在系统设置中授予此许可。要继续打印，用户必须接受请求。

为“打印”按钮注册了一个动作侦听器。当用户单击“打印”按钮时， actionPerformed方法调用print方法，它将启动打印任务。打印任务是SwingWorker宾语。以下代码示例显示了PrintingTask类已实现。

private class PrintingTask extends SwingWorker<Object, Object> {
    private final MessageFormat headerFormat;
    private final MessageFormat footerFormat;
    private final boolean interactive;
    private volatile boolean complete = false;
    private volatile String message;
        
    public PrintingTask(MessageFormat header, MessageFormat footer,
                        boolean interactive) {
        this.headerFormat = header;
        this.footerFormat = footer;
        this.interactive = interactive;
    }
        
    @Override
    protected Object doInBackground() {
        try {
            complete = text.print(headerFormat, footerFormat,
                    true, null, null, interactive);
            message = "Printing " + (complete ? "complete" : "canceled");
        } catch (PrinterException ex) {
            message = "Sorry, a printer error occurred";
        } catch (SecurityException ex) {
            message =
                "Sorry, cannot access the printer due to security reasons";
        }
        return null;
    }
        
    @Override
    protected void done() {
        message(!complete, message);
    }
}

以下代码示例显示了print方法从GUI组件获取一组选定选项，然后创建一个实例。 PrintingTask类，并执行打印。

private void print(java.awt.event.ActionEvent evt) {
        MessageFormat header = createFormat(headerField);
        MessageFormat footer = createFormat(footerField);
        boolean interactive = interactiveCheck.isSelected();
        boolean background = backgroundCheck.isSelected();

        PrintingTask task = new PrintingTask(header, footer, interactive);
        if (background) {
            task.execute();
        } else {
            task.run()
        }
    }

粗体代码展示了PrintingTask的方法取决于background参数的值。每当用户喜欢在后台线程上打印时， execute调用方法，该方法调度打印任务以在后台线程上执行。否则run方法在EDT上执行打印任务。

由于打印大文档是一项耗时的任务，因此建议在后台线程上执行打印。

您可以在printSelectedPages方法。调用时，此方法首先获得选择打印的页面数量。以下代码示例显示了printSelectedPages方法创建一个Runnable每个页面的对象，然后在单独的线程上打印当前页面。

for (int i = 0; i < n; i++) {
    final PageItem item = (PageItem) pages.getElementAt(i);
    // This method is called from EDT.  Printing is a time-consuming
    // task, so it should be done outside EDT, in a separate thread.
    Runnable printTask = new Runnable() {
        public void run() {
            try {
                item.print(
                        // Two "false" args mean "no print dialog" and
                        // "non-interactive" (ie, batch-mode printing).
                                null, null, false, printService, null, false);

文字列印API

本节列出了JTextComponent允许您打印文本文档的类。

方法 目的
布尔值print（）
布尔值打印（MessageFormat，MessageFormat）
布尔值打印（MessageFormat，MessageFormat，布尔值，PrintRequestAttributeSet，布尔值，PrintService）		 在不带参数的情况下调用时，显示一个打印对话框，然后以交互方式打印此文本组件，而没有页眉或页脚文本。退货true如果用户继续打印并且false如果用户取消打印。
当与两个MessageFormat参数，显示一个打印对话框，然后以指定的页眉和页脚文本交互式地打印此文本组件。
当使用一整套参数调用时，将根据指定的参数打印此文本组件。他们俩MessageFormat参数指定页眉和页脚文本。第一个布尔参数定义是否显示打印对话框。另一个布尔参数指定是否以交互方式打印。使用其他两个参数可以指定打印属性和打印服务。
每当一个PrintService省略参数，将使用默认打印机。
Printable getPrintable（MessageFormat，MessageFormat） 返回一个Printable用于打印文本组件的对象。重写此方法以获得自定义的Printable对象。您可以将一个Printable对象包装到另一个对象中，以获得复杂的报告和文档。
