合成器示例

在名为A GroupLayout Example的课程中， GroupLayout用于创建一个名为“查找”的搜索对话框。创建对话框的程序， Find.java ，并以“海洋”为主题使用了跨平台（“金属”）外观：

本课程使用外部XML文件创建与Synth相同的对话框。这是清单SynthDialog.java文件。

SynthDialog.java与...完全相同Find.java除了initLookAndFeel()方法，此方法已更改为使用Synth外观和名为的外部文件synthDemo.xml 。这是新的initLookAndFeel()方法：

private static void initLookAndFeel() {
   SynthLookAndFeel lookAndFeel = new SynthLookAndFeel();
       
   // SynthLookAndFeel load() method throws a checked exception
   // (java.text.ParseException) so it must be handled
   try {
      lookAndFeel.load(SynthDialog.class.getResourceAsStream("synthDemo.xml"),
                                                               SynthDialog.class);
      UIManager.setLookAndFeel(lookAndFeel);
   } 
            
   catch (ParseException e) {
      System.err.println("Couldn't get specified look and feel ("
                                   + lookAndFeel
                                    + "), for some reason.");
      System.err.println("Using the default look and feel.");
      e.printStackTrace();
   }
}

XML文件

XML文件， synthDemo.xml ，从绑定到所有区域的样式开始。这样做是一种很好的做法，以确保没有绑定样式的区域将包含某些内容。这种样式使所有区域都以不透明的颜色绘制背景。它还设置默认字体和默认颜色。
笔记： 

1。颜色定义必须在元素内部。这允许根据状态改变颜色。中的元素backingStyle没有属性，因此适用于所有区域，无论其状态如何。如果区域具有其他状态，则将状态合并并赋予在文件中稍后显示的状态定义的优先级。

2。字体定义不是一个元素中，因为当有状态的改变字体不应该改变（很多组件取决于它们的字体大小，以及字体的变化可能会无意中导致在大小组件的变化）。
