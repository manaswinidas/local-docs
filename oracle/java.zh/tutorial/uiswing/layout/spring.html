如何使用SpringLayout

注意：本课程涵盖手工编写布局代码，这可能会很困难。如果您对学习布局管理的所有细节都不感兴趣，则可以使用GroupLayout布局管理器与构建器工具相结合，以布置您的GUI。 NetBeans IDE就是这样一种构建器工具。否则，如果您想手工编码而又不想使用GroupLayout ， 然后GridBagLayout建议将其作为下一个最灵活，功能最强大的布局管理器。

如果您对使用JavaFX创建GUI感兴趣，请参阅使用JavaFX中的布局 。

的SpringLayout JDK 1.4版中添加了类，以支持GUI构建器中的布局。 SpringLayout是一个非常灵活的布局管理器，可以模仿其他布局管理器的许多功能。 SpringLayout但是，它是非常低级的，因此您真的应该只将它与GUI构建器一起使用，而不是尝试手动编写spring布局管理器。

本节从一个简单的示例开始，显示创建第一个Spring布局时需要记住的所有内容-忘记它们会发生什么！稍后，它将介绍实用程序方法，使您可以将组件布置在几种不同类型的网格中。

弹簧布局如何工作

弹簧布局通过定义组件边缘之间的方向关系或约束来完成其工作。例如，您可以定义一个组件的左边缘与另一组件的右边缘之间的距离为固定距离（例如5像素）。

在一个SpringLayout ，每条边的位置仅取决于另一条边的位置。如果随后添加了约束以为边创建新的绑定，则先前的绑定将被丢弃，并且该边仍然依赖于单个边。

与许多布局管理器不同， SpringLayout不会自动设置它管理的组件的位置。如果您手动编码使用的GUI SpringLayout ，请记住通过限制西/东和北/南位置来初始化零部件位置。根据您使用的约束，您可能还需要显式设置容器的大小。

组件定义边缘属性，这些属性通过Spring实例。每个弹簧具有四个属性- 最小值 ， 首选值和最大值 ，以及其实际（当前） 值 。与每个组件关联的弹簧被收集到一个SpringLayout.Constraints宾语。

的实例Spring类具有表征其行为的三个属性：最小值，首选值和最大值。这些属性中的每一个都可能涉及基于一系列规则定义其第四值属性。

的实例Spring可以将其可视化为一种机械弹簧，它在压缩或拉伸远离其首选值时会提供校正力。该力被建模为距最佳值的距离的线性函数，但具有两个不同的常数-一个用于压缩力，一个用于拉伸力。这些常数由弹簧的最小值和最大值指定，这样，处于最小值的弹簧会产生与处于最大值时所产生的力相等且相反的力。因此，优选值与最小值之间的差表示弹簧可以被压缩的容易程度。其最大值和首选值之间的差异表明Spring可以扩展。

基于此， SpringLayout可以可视化为一组对象，这些对象在其边缘通过一组弹簧连接。

在此示例中，我们将指定组件应显示在单个行中，并且它们之间应有5个像素。以下代码指定标签的位置：

//Adjust constraints for the label so it's at (5,5).
layout.putConstraint(SpringLayout.WEST, label,
                     5,
                     SpringLayout.WEST, contentPane);
layout.putConstraint(SpringLayout.NORTH, label,
                     5,
                     SpringLayout.NORTH, contentPane);

首先putConstraint调用指定标签的左（西）边应与其容器的左边相距5像素。这将转换为x坐标5。第二putConstraint调用会在标签的顶部（北边）与其容器之间建立相似的关系，从而使y坐标为5。

这是设置文本字段位置的代码：

//Adjust constraints for the text field so it's at
//(<label's right edge> + 5, 5).
layout.putConstraint(SpringLayout.WEST, textField,
                     5,
                     SpringLayout.EAST, label);
layout.putConstraint(SpringLayout.NORTH, textField,
                     5,
                     SpringLayout.NORTH, contentPane);

首先putConstraint调用使文本字段的左（西）边与标签的右（东）边相距5像素。第二putConstraint call就像第一个代码片段中的第二个调用一样，并且具有将组件的y坐标设置为5的相同效果。

在这种情况下，弹簧布局选择为文本字段提供所有额外的空间。尽管似乎弹簧布局对标签和文本字段的处理方式不同，但是弹簧布局对Swing或AWT组件没有特殊的了解。它依赖于组件的最小，首选和最大大小属性的值。下一节讨论弹簧布局如何使用这些属性，以及为什么它们会导致空间分布不均匀。

弹簧和零件尺寸

一种SpringLayout对象自动安装Spring s为每个组件的高度和宽度SpringLayout控件。这些弹簧实质上是组件的盖getMinimumSize ， getPreferredSize和getMaximumSize方法。所谓的“覆盖”是指不仅用这些方法中的适当值初始化弹簧，而且弹簧跟踪这些值。例如， Spring表示组件宽度的对象是一种特殊的弹簧，它仅将其实现委托给组件的相关大小方法。这样，随着元件特性的变化，弹簧与尺寸方法保持同步。

当一个组件的getMaximumSize和getPreferredSize方法返回相同的值， SpringLayout将此解释为意味着不应拉伸组件。 JLabel和JButton是通过这种方式实现的组件的示例。因此，SpringDemo3示例中的标签不会拉伸。

的getMaximumSize一些组件的方法，例如JTextField ，返回值Integer.MAX_VALUE其最大尺寸的宽度和高度，表示该组件可以增长到任何尺寸。因此，当放大SpringDemo3窗口时， SpringLayout将所有多余的空间分配给可以增长的唯一弹簧-决定文本字段大小的弹簧。

正如前面的摘要所暗示的， SpringLayout和SpringLayout.Constraints倾向于使用不同的约定来描述弹簧。的SpringLayout API使用边来定义其约束。弹簧连接边缘以在它们之间建立线性关系。边缘由组件使用以下常量定义：

  • SpringLayout.NORTH指定组件边界矩形的顶部边缘。
  • SpringLayout.SOUTH指定组件边界矩形的底边。
  • SpringLayout.EAST指定组件边界矩形的右边缘。
  • SpringLayout.WEST指定组件边界矩形的左边缘。
  • SpringLayout.BASELINE指定组件的基线。
  • SpringLayout.HORIZONTAL_CENTER指定组件边界矩形的水平中心。
  • SpringLayout.VERTICAL_CENTER指定组件边界矩形的垂直中心。

边缘与Spring对象SpringLayout.Constraints类知道边，但是只有Spring对象具有以下属性：

  • X
  • ÿ
  • 宽度
  • 高度

每Constraints对象在其弹簧和它们表示的边缘之间保持以下关系：

 west = x
north = y
 east = x + width
south = y + height

如果您感到困惑，请不要担心。下一部分将介绍实用程序方法，您可以使用它们来完成一些常见的布局任务，而无需了解任何有关spring layout API的知识。

网格实用程序

因为SpringLayout类是为GUI构建器创建的，为布局设置单独的弹簧可能很麻烦，而手工编写代码很麻烦。本节介绍了几种方法，可用来安装在网格中布置一组组件所需的所有弹簧。这些方法模仿了GridLayout ， GridBagLayout和BoxLayout类。

两种方法，称为makeGrid和makeCompactGrid ，定义于SpringUtilities.java 。两种方法都可以通过将组件分为行和列并使用Spring.max制作宽度或高度弹簧的方法，该弹簧使行或列的大小足以容纳其中的所有组件。在里面makeCompactGrid方法相同的宽度或高度弹簧分别用于特定列或行中的所有组件。在里面makeGrid相比之下，容器中的每个组件都共享宽度和高度弹簧，从而迫使它们的大小相同。此外，工厂方法由Spring用于创建不同种类的弹簧，包括依赖于其他弹簧的弹簧。

让我们看看这些方法的作用。我们的第一个示例，在源文件中实现SpringGrid.java ，在文本字段中显示一堆数字。中心文本字段比其他文本字段宽得多。就像GridLayout ，拥有一个大单元将迫使所有单元都同样大。单击启动按钮以使用Java™Web Start （ 下载JDK 7或更高版本 ）运行SpringGrid。另外，要自己编译并运行示例，请参考示例索引 。
