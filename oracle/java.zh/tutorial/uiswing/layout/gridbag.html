如何使用GridBagLayout

注意：本课程涵盖手工编写布局代码，这可能会很困难。如果您对学习布局管理的所有细节都不感兴趣，则可以使用GroupLayout布局管理器与构建器工具相结合，以布置您的GUI。 NetBeans IDE就是这样一种构建器工具。否则，如果您想手工编码而又不想使用GroupLayout ， 然后GridBagLayout建议将其作为下一个最灵活，功能最强大的布局管理器。

如果您对使用JavaFX创建GUI感兴趣，请参阅使用JavaFX中的布局 。

GridBagLayout是Java平台提供的最灵活（也是最复杂）的布局管理器之一。一种GridBagLayout将组件放置在行和列的网格中，从而允许指定的组件跨越多个行或列。并非所有行都必须具有相同的高度。同样，并非所有列都必须具有相同的宽度。实质上， GridBagLayout将组件放置在网格中的矩形（单元）中，然后使用组件的首选大小来确定单元的大小。

下图显示了前面小程序的网格。如您所见，网格具有三行三列。第二行中的按钮跨所有列；第三行中的按钮跨越右两列。

如果您放大窗口，如下图所示，您将注意到包含Button 5的最下面一行获得了所有新的垂直空间。新的水平空间在所有列之间平均分配。调整大小的行为基于程序分配给程序中各个组件的权重。 GridBagLayout 。您还将注意到，每个组件都占据了所有可用的水平空间，但是（如按钮5所示）却没有占据所有可用的垂直空间。该行为也由程序指定。

程序指定其组件的大小和位置特征的方式是通过为每个组件指定约束 。在组件上设置约束的首选方法是使用Container.add变体，将其传递给GridBagConstraints对象，如以下各节所示。

以下各节说明可以设置的约束并提供示例。

指定约束

以下代码是使用GridBagLayout 。您将在下一节中看到更详细的示例。

JPanel pane = new JPanel(new GridBagLayout());
GridBagConstraints c = new GridBagConstraints();

//For each component to be added to this container:
//...Create the component...
//...Set instance variables in the GridBagConstraints instance...
pane.add(theComponent, c);

您可能从上面的示例中猜到了，可以重复使用相同的示例GridBagConstraints实例具有多个组件，即使组件具有不同的约束。但是，建议您不要重复使用GridBagConstraints ，因为如果您忘记为每个新实例重置字段，则很容易导致引入细微的错误。

注意：以下讨论假定GridBagLayout控制具有从左到右组件方向的容器。

您可以设置以下内容GridBagConstraints实例变量：

gridx ，gridy
在组件的左上方指定行和列。最左边的列有地址gridx=0第一行有地址gridy=0 。使用GridBagConstraints.RELATIVE （默认值）以指定将组件放置在（对于gridx ）或正下方（ gridy ）即将添加到容器中的组件。我们建议指定gridx和gridy每个组件的值，而不仅仅是使用GridBagConstraints.RELATIVE ;这往往会导致更可预测的布局。
gridwidth ，gridheight
指定列数（对于gridwidth ）或行（用于gridheight ）在组件的显示区域中。这些约束指定组件使用的单元数量， 而不是组件使用的像素数量。预设值为1。使用GridBagConstraints.REMAINDER指定组件是其行中的最后一个（用于gridwidth ）或列（用于gridheight ）。使用GridBagConstraints.RELATIVE指定该组件是其行中倒数第二个组件（对于gridwidth ）或列（用于gridheight ）。我们建议指定gridwidth和gridheight每个组件的值，而不仅仅是使用GridBagConstraints.RELATIVE和GridBagConstraints.REMAINDER ;这往往会导致更可预测的布局。

注意： GridBagLayout不允许组件跨越多行，除非组件位于最左列或您指定了正数gridx和gridy组件的值。

fill
当组件的显示区域大于组件的请求大小时使用，以确定是否以及如何调整组件的大小。有效值（定义为GridBagConstraints常量）包括NONE （默认）， HORIZONTAL （使组件足够宽，可以水平填充其显示区域，但不要更改其高度）， VERTICAL （使组件足够高以垂直填充其显示区域，但不要更改其宽度），以及BOTH （使组件完全填充其显示区域）。
ipadx ，ipady
指定内部填充：要添加到组件大小的数量。默认值为零。组件的宽度将至少为其最小宽度加上ipadx*2像素，因为填充适用于组件的两侧。同样，组件的高度将至少为其最小高度加ipady*2像素。
insets
指定组件的外部填充-组件与其显示区域的边缘之间的最小空间。该值指定为Insets宾语。默认情况下，每个组件都没有外部填充。
anchor
当组件小于其显示区域时使用，以确定放置组件的位置（在区域内）。有效值（定义为GridBagConstraints常数）是CENTER （默认）， PAGE_START ， PAGE_END ， LINE_START ， LINE_END ， FIRST_LINE_START ， FIRST_LINE_END ， LAST_LINE_END和LAST_LINE_START 。

这是在默认的从左到右组件方向的容器中如何解释这些值的图片。

FIRST_LINE_START PAGE_START FIRST_LINE_END
LINE_START 中央 LINE_END
LAST_LINE_START PAGE_END LAST_LINE_END
版本说明： PAGE_*和*LINE_*常量在1.4中引入。以前的版本需要以罗盘点命名的值。例如， NORTHEAST指示显示区域的右上部分。我们建议您改用新的常量，因为它们使本地化更容易。
weightx ，weighty
指定权重是一门艺术，可能会对组件的外观产生重大影响GridBagLayout控件。权重用于确定如何在列之间分配空间（ weightx ）和行之间（ weighty ）;这对于指定调整大小的行为很重要。

除非您至少指定一个非零值weightx要么weighty ，所有组件都在其容器的中央聚集在一起。这是因为当权重为0.0（默认值）时， GridBagLayout在其单元格网格和容器边缘之间放置任何多余的空间。

通常，权重的极限值设置为0.0和1.0：必要时使用中间的数字。较大的数字表示组件的行或列应获得更多的空间。对于每一列，权重与最高weightx为该列中的某个组件指定的值，每个多列组件的权重都会以某种方式在该组件所在的列之间分配。同样，每一行的权重与最高weighty为该行中的组件指定。多余的空间倾向于流向最右边的列和最下面的行。

范例说明

单击启动按钮，以使用Java™Web Start （ 下载JDK 7或更高版本 ）运行GridBagLayoutDemo。另外，要自己编译并运行示例，请参考示例索引 。


以下代码创建了GridBagLayout及其管理的组件您可以在以下位置找到整个源文件GridBagLayoutDemo.java 。

JButton button;
pane.setLayout(new GridBagLayout());
GridBagConstraints c = new GridBagConstraints();
if (shouldFill) {
                //natural height, maximum width
                c.fill = GridBagConstraints.HORIZONTAL;
}

button = new JButton("Button 1");
if (shouldWeightX) {
                   c.weightx = 0.5;
}
c.fill = GridBagConstraints.HORIZONTAL;
c.gridx = 0;
c.gridy = 0;
pane.add(button, c);

button = new JButton("Button 2");
c.fill = GridBagConstraints.HORIZONTAL;
c.weightx = 0.5;
c.gridx = 1;
c.gridy = 0;
pane.add(button, c);

button = new JButton("Button 3");
c.fill = GridBagConstraints.HORIZONTAL;
c.weightx = 0.5;
c.gridx = 2;
c.gridy = 0;
pane.add(button, c);

button = new JButton("Long-Named Button 4");
c.fill = GridBagConstraints.HORIZONTAL;
c.ipady = 40;      //make this component tall
c.weightx = 0.0;
c.gridwidth = 3;
c.gridx = 0;
c.gridy = 1;
pane.add(button, c);

button = new JButton("5");
c.fill = GridBagConstraints.HORIZONTAL;
c.ipady = 0;       //reset to default
c.weighty = 1.0;   //request any extra vertical space
c.anchor = GridBagConstraints.PAGE_END; //bottom of space
c.insets = new Insets(10,0,0,0);  //top padding
c.gridx = 1;       //aligned with button 2
c.gridwidth = 2;   //2 columns wide
c.gridy = 2;       //third row
pane.add(button, c);

本示例使用一个GridBagConstraints所有组件的实例GridBagLayout可以管理，但是在现实生活中，建议您不要重复使用GridBagConstraints ，因为如果您忘记为每个新实例重置字段，则很容易导致引入细微的错误。在将每个组件添加到容器之前，代码会在代码集中设置（或重置为默认值）适当的实例变量。 GridBagConstraints宾语。然后，将组件添加到其容器中，并指定GridBagConstraints对象作为第二个参数add方法。

例如，使按钮4更高，该示例具有以下代码：

c.ipady = 40;

在设置下一个组件的约束之前，代码将重置ipady设为默认值：

c.ipady = 0;

如果组件的显示区域大于组件本身，那么您可以使用来指定组件在显示区域中的下落。 GridBagConstraints.anchor约束。的anchor约束的值可以是绝对值（北，南，东，西等），也可以是相对方向的（在页面开头，行尾，第一行开头等等），或相对于组件的基线。有关的可能值的完整列表anchor约束，包括相对基线值，请参阅API文档以获取GridBagConstraints.anchor 。您可以在上面的代码摘录中看到，按钮5通过在以下位置设置锚点来指定应将其显示在显示区域的末尾GridBagConstraints.PAGE_END 。

注意：本教程的示例用于以不同的方式指定约束对象，您可能也会在其他程序中看到这种方式。而不是使用add方法，我们的示例用于调用setConstraints上的方法GridBagLayout宾语。例如：
GridBagLayout gridbag = new GridBagLayout();
pane.setLayout(gridbag);
...
gridbag.setConstraints(button, c);
pane.add(button);
但是，我们建议您使用Container.add方法，因为它比您使用的代码更干净setConstraints 。
