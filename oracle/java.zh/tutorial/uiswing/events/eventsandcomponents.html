Swing组件支持的侦听器

您可以通过查看可以在组件上注册的事件侦听器的种类来判断组件可以触发哪些事件。例如， JComboBox类定义了以下侦听器注册方法：

  • addActionListener
  • addItemListener
  • addPopupMenuListener

因此，组合框除了继承自其继承的侦听器方法外，还支持操作，项目和弹出菜单侦听器JComponent 。

Swing组件支持的侦听器分为两类：

  • 所有Swing组件都支持的侦听器
  • Swing组件支持的其他侦听器

所有Swing组件都支持的侦听器

因为所有Swing组件都来自AWT Component类，您可以在任何Swing组件上注册以下侦听器：

组件监听器
侦听组件的大小，位置或可见性的变化。
重点听众
侦听组件是否获得键盘焦点。
关键听众
监听按键；按键事件仅由具有当前键盘焦点的组件触发。
鼠标监听器
侦听鼠标单击，鼠标按下，鼠标释放以及鼠标移入或移出组件的绘图区域。
鼠标动作侦听器
侦听鼠标光标在组件上的位置变化。
鼠标滚轮监听器
侦听鼠标滚轮在组件上的移动。
层次侦听器
侦听对已更改事件的组件包含层次结构的更改。
层次边界侦听器
侦听对移动和调整大小事件的组件包含层次结构的更改。

所有Swing组件均来自AWT Container类，但其中许多不用作容器。因此，从技术上讲，任何Swing组件都可以触发容器事件 ，该事件会通知侦听器组件已添加到容器或从容器中删除。但是，实际上，只有容器（例如面板和框架）和复合组件（例如组合框）通常会引发容器事件。

JComponent提供了对另外三种侦听器类型的支持。您可以注册一个祖先侦听器 ，以在组件的容器祖先添加到容器或从容器中移除，隐藏，显示或移动时得到通知。此侦听器类型是在层次结构侦听器之前的实现详细信息。

其他两种侦听器类型是Swing组件符合JavaBeans规范的一部分。除其他外，这意味着Swing组件支持绑定和受约束的属性，并将属性的更改通知给侦听器。属性更改侦听器侦听绑定属性的更改，并由多个Swing组件（例如， 格式化的文本字段 ）用于跟踪组件绑定属性上的更改。同样，构建器工具使用属性更改侦听器和否决更改侦听器来侦听受约束属性的更改。有关更多信息，请参阅JavaBeans路径中的“ 属性”课程。
