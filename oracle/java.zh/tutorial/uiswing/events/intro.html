事件侦听器简介

如果您已经阅读了任何组件操作指南页面，则可能已经了解事件侦听器的基础知识。

让我们看一下最简单的事件处理示例之一。它称为Beeper，它具有一个单击按钮时会发出哔哔声的按钮。

单击启动按钮以使用Java™Web Start运行Beeper（ 下载JDK 7或更高版本 ）。或者，要自己编译并运行示例，请参考示例索引 。


您可以在以下位置找到整个程序Beeper.java 。以下是实现按钮事件处理的代码：

public class Beeper ... implements ActionListener {
    ...
    //where initialization occurs:
        button.addActionListener(this);
    ...
    public void actionPerformed(ActionEvent e) {
        ...//Make a beep sound...
    }
}

的Beeper类实现ActionListener接口，其中包含一种方法： actionPerformed 。以来Beeper贯彻ActionListener ， 一种Beeper对象可以注册为按钮触发的动作事件的侦听器。一旦Beeper已使用ButtonaddActionListener方法， Beeper的actionPerformed每次单击按钮时都会调用方法。

一个更复杂的例子

您在前面的示例中最简单地看到了事件模型，该模型非常强大且灵活。任意数量的事件侦听器对象都可以侦听任意数量的事件源对象中的各种事件。例如，一个程序可能为每个事件源创建一个侦听器。或者，一个程序可能只有一个侦听器，可以监听所有来源的所有事件。一个程序甚至可以从单个事件源中为单个事件提供多个侦听器。

可以注册多个侦听器，以从特定来源通知特定类型的事件。同样，同一侦听器可以侦听来自不同对象的通知。

每个事件由一个对象表示，该对象提供有关事件的信息并标识事件源。事件源通常是组件或模型，但是其他类型的对象也可以是事件源。

每当您要检测来自特定组件的事件时，请首先检查该组件的操作方法部分。组件操作指南部分的列表在此处 。操作方法部分提供了处理您最可能关心的事件的示例。例如，在“ 如何使用颜色选择器”中 ，您将找到一个编写更改侦听器以跟踪颜色选择器中颜色何时变化的示例。

下面的示例演示事件侦听器可以在多个对象上注册，并且同一事件可以发送到多个侦听器。该示例包含两个事件源（ JButton实例）和两个事件监听器。事件侦听器之一（称为的类的实例MultiListener ）从两个按钮监听事件。收到事件时，它将事件的“操作命令”（设置为按钮标签上的文本）添加到顶部文本区域。第二个事件侦听器（称为Eavesdropper ）仅在其中一个按钮上监听事件。收到事件时，它将操作命令添加到底部文本区域。

尝试这个： 
  1. 单击启动按钮，以使用Java™Web Start运行MultiListener（ 下载JDK 7或更高版本 ）。或者，要自己编译并运行示例，请参考示例索引 。
  2. 单击“ 等等等等”按钮。只有MultiListener注册对象以收听此按钮。
  3. 单击您不说！ 按钮。这俩MultiListener对象和Eavesdropper对象被注册以收听此按钮。

您可以在以下位置找到整个程序MultiListener.java 。以下是实现按钮事件处理的代码：

public class MultiListener ... implements ActionListener {
    ...
    //where initialization occurs:
        button1.addActionListener(this);
        button2.addActionListener(this);

        button2.addActionListener(new Eavesdropper(bottomTextArea));
    }

    public void actionPerformed(ActionEvent e) {
        topTextArea.append(e.getActionCommand() + newline);
    }
}

class Eavesdropper implements ActionListener {
    ...
    public void actionPerformed(ActionEvent e) {
        myTextArea.append(e.getActionCommand() + newline);
    }
}

在上面的代码中， MultiListener和Eavesdropper实施ActionListener界面，并使用JButtonaddActionListener方法。这两类的实现actionPerformed方法很相似：它们只是将事件的action命令添加到文本区域。
