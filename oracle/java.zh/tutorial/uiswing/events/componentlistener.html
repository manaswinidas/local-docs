如何编写组件侦听器

组件侦听器是用于接收组件事件的侦听器接口。组件是具有图形表示的对象，该对象可以显示在屏幕上并可以与用户进行交互。组件的一些示例是典型图形用户界面的按钮，复选框和滚动条。

对处理组件事件感兴趣的类要么实现此接口及其包含的所有方法，要么扩展抽象的ComponentAdapter类，使其仅覆盖感兴趣的方法。然后使用该组件的addComponentListener方法向该组件注册从该类创建的侦听器对象。当组件的大小，位置或可见性更改时，将调用侦听器对象中的相关方法，并将ComponentEvent传递给它。

一个或多个组件事件由Component组件被隐藏，可见，移动或调整大小之后的对象。

组件隐藏事件和组件显示事件仅是由于调用了Component的setVisible方法。例如，一个窗口可能被缩小为一个图标（图标化），而没有触发任何组件隐藏事件。

要编写一个简单的组件侦听器程序，请遵循以下步骤：

  • 声明一个实现Component侦听器的类。例如：
    public class ComponentEventDemo ... implements ComponentListener
  • 确定要捕获事件的组件。例如：窗格，标签，复选框等。
  • 将组件侦听器添加到已标识的组件。例如：
    ....
label.addComponentListener(this);
.....
checkbox.addComponentListener(this);
....
panel.addComponentListener(this);
...
frame.addComponentListener(this);
  • 最后，通过使用组件侦听器的四种方法来捕获这些组件的不同事件，如下所示：
    public void componentHidden(ComponentEvent e) {
        displayMessage(e.getComponent().getClass().getName() + " --- Hidden");
    }

    public void componentMoved(ComponentEvent e) {
        displayMessage(e.getComponent().getClass().getName() + " --- Moved");
    }

    public void componentResized(ComponentEvent e) {
        displayMessage(e.getComponent().getClass().getName() + " --- Resized ");            
    }

    public void componentShown(ComponentEvent e) {
        displayMessage(e.getComponent().getClass().getName() + " --- Shown");

    }

以下示例演示了组件事件。该窗口包含具有标签和复选框的面板。复选框控制标签是否可见。每当窗口，面板，标签或复选框触发组件事件时，文本区域都会显示一条消息。

尝试这个： 
  1. 单击启动按钮以使用Java™Web Start运行ComponentEventDemo（ 下载JDK 7或更高版本 ）。或者，要自己编译并运行示例，请参考示例索引 。
  2. 出现窗口时，将触发一个或多个组件显示的事件。
  3. 单击复选框以隐藏标签。
    标签会触发组件隐藏事件。面板触发组件移动和组件调整大小的事件。该复选框将触发一个组件移动的事件。
  4. 再次单击复选框以显示标签。
    标签会触发组件显示的事件。面板触发组件移动和组件调整大小的事件。该复选框将触发一个组件移动的事件。
  5. 图标化然后取消图标化窗口。
    您不会获得隐藏组件或显示的事件。如果希望收到有关图标化事件的通知，则应使用窗口侦听器或窗口状态侦听器。
  6. 调整窗口大小。
    您会从所有四个组件（标签，复选框，面板和框架）中看到组件调整大小（可能还有组件移动）的事件。如果框架和面板的布局管理器没有使每个组件都尽可能宽，则面板，标签和复选框的大小将不会调整。

组件侦听器API

ComponentListener接口

所有这些方法也都在适配器类中， ComponentAdapter 。

方法 目的
componentHidden（ComponentEvent） 在被侦听的组件被隐藏后，由于setVisible方法被调用。
componentMoved（ComponentEvent） 在侦听组件相对于其容器移动之后调用。例如，如果移动了一个窗口，则该窗口会触发一个组件移动的事件，但它包含的组件不会触发。
componentResized（ComponentEvent） 在侦听的组件的大小（矩形边界）更改之后调用。
componentShown（ComponentEvent） 在被侦听的组件由于可见的结果而变得可见之后调用setVisible方法被调用。

ComponentEvent类

方法 目的
组件getComponent（） 返回引发事件的组件。您可以使用它代替getSource方法。
