如何编写文档监听器

Swing文字组件使用Document代表其内容。文档内容以任何方式更改时，都会发生文档事件。您将文档侦听器附加到文本组件的文档，而不是文本组件本身。有关更多信息，请参见实施文档过滤器 。

下面的示例演示了两个纯文本组件上的文档事件。

尝试这个： 
  1. 单击启动按钮以使用Java™Web Start （ 下载JDK 7或更高版本 ）运行DocumentEventDemo。另外，要自己编译并运行示例，请参考示例索引 。
  2. 键入窗口左上方的文本字段或该文本字段下方的文本区域。
    每个键入的字符都会触发一个文档事件。
  3. 使用退格键删除文本。
    每个键入的Backspace键都会触发一个文档事件。
  4. 选择文本，然后通过键入退格键或使用键盘命令（例如， CTRL-X （切）。
    整个删除都会触发一个文档事件。
  5. 使用键盘命令将文本从一个文本组件复制到另一个文本组件CTRL-C （副本）和CTRL-V （糊）。
    无论粘贴文本的长度如何，都会为整个粘贴操作触发一个文档事件。如果在发布粘贴命令之前在目标文本组件中选择了文本，则会触发其他文档事件，因为首先删除了所选文本。

您可以在以下位置找到该演示的代码DocumentEventDemo.java 。这是演示的文档事件处理代码：

public class DocumentEventDemo ... {
    ...//where initialization occurs:
    textField = new JTextField(20);
    textField.addActionListener(new MyTextActionListener());
    textField.getDocument().addDocumentListener(new MyDocumentListener());
    textField.getDocument().putProperty("name", "Text Field");

    textArea = new JTextArea();
    textArea.getDocument().addDocumentListener(new MyDocumentListener());
    textArea.getDocument().putProperty("name", "Text Area");
    ...

class MyDocumentListener implements DocumentListener {
    String newline = "\n";
 
    public void insertUpdate(DocumentEvent e) {
        updateLog(e, "inserted into");
    }
    public void removeUpdate(DocumentEvent e) {
        updateLog(e, "removed from");
    }
    public void changedUpdate(DocumentEvent e) {
        //Plain text components do not fire these events
    }

    public void updateLog(DocumentEvent e, String action) {
        Document doc = (Document)e.getDocument();
        int changeLength = e.getLength();
        displayArea.append(
            changeLength + " character" +
            ((changeLength == 1) ? " " : "s ") +
            action + doc.getProperty("name") + "." + newline +
            "  Text length = " + doc.getLength() + newline);
    }
}

文档侦听器不应修改文档的内容；更改已在通知侦听器时完成。而是编写一个覆盖insertString要么remove方法，或两者兼而有之。有关详细信息，请参见侦听文档中的更改 。

文档监听器API

DocumentListener接口

DocumentListener没有适配器类。

方法 目的
changedUpdate（DocumentEvent） 当侦听文档中某些文本的样式更改时调用。此类事件仅从StyledDocument - 一种PlainDocument不会触发这些事件。
insertUpdate（DocumentEvent） 在将文本插入到收听的文档中时调用。
removeUpdate（DocumentEvent） 从侦听文档中删除文本时调用。

DocumentEvent接口

每个文档事件方法都传递一个实现DocumentEvent接口。通常，这是DefaultDocumentEvent ，定义于AbstractDocument 。

方法 目的
文件getDocument（） 返回触发事件的文档。请注意DocumentEvent接口不继承EventObject 。因此，它不会继承getSource方法。
int getLength（） 返回更改的长度。
int getOffset（） 返回文档中第一个字符已更改的位置。
ElementChange getChange（元素） 返回有关文档中哪些元素已更改以及如何更改的详细信息。 ElementChange是在DocumentEvent接口。
EventType getType（） 返回发生的更改的类型。 EventType是在DocumentEvent枚举文档上可能发生的更改的界面：插入文本，删除文本和更改文本样式。
