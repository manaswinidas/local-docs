有中期结果的任务

对于后台任务，在仍在运行时提供临时结果通常很有用。该任务可以通过调用SwingWorker.publish 。此方法接受可变数量的参数。每个参数必须是由以下参数指定的类型SwingWorker的第二个类型参数。

收集提供的结果publish ，覆盖SwingWorker.process该方法将从事件分配线程中调用。多次调用的结果publish通常是为单次调用而累积的process 。

让我们来看看Flipper.java示例使用publish提供中期结果。单击启动按钮运行Flipper使用Java™Web Start （ 下载JDK 7或更高版本 ）。或者，要自己编译并运行示例，请参考示例索引 。


该程序测试了java.util.Random通过产生一系列随机boolean后台任务中的值。这相当于掷硬币。由此得名Flipper 。为了报告其结果，后台任务使用类型为的对象FlipPair

private static class FlipPair {
    private final long heads, total;
    FlipPair(long heads, long total) {
        this.heads = heads;
        this.total = total;
    }
}

的heads栏位是随机值的次数true ;的total字段是随机值的总数。

后台任务由以下实例表示FlipTask ：

private class FlipTask extends SwingWorker<Void, FlipPair> {

由于该任务不会返回最终结果，因此第一类型参数是什么都没有关系。 Void用作占位符。任务调用publish每次“抛硬币”之后：

@Override
protected Void doInBackground() {
    long heads = 0;
    long total = 0;
    Random random = new Random();
    while (!isCancelled()) {
        total++;
        if (random.nextBoolean()) {
            heads++;
        }
        publish(new FlipPair(heads, total));
    }
    return null;
}

（ isCancelled方法将在下一节中讨论。）因为publish被频繁调用，很多FlipPair值可能会在process在事件分配线程中调用； process只对每次报告的最后一个值感兴趣，使用它来更新GUI：

protected void process(List<FlipPair> pairs) {
    FlipPair pair = pairs.get(pairs.size() - 1);
    headsText.setText(String.format("%d", pair.heads));
    totalText.setText(String.format("%d", pair.total));
    devText.setText(String.format("%.10g", 
            ((double) pair.heads)/((double) pair.total) - 0.5));
}

如果Random是公平的，显示在devText应该越来越接近于0 Flipper运行。

注意： setText用于Flipper实际上是其规范中定义的“线程安全”。这意味着我们可以省去publish和process并直接从工作线程中设置文本字段。我们选择忽略这一事实是为了提供一个简单的演示SwingWorker中期业绩。
