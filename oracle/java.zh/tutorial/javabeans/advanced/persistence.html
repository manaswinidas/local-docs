Bean持久性

当bean的属性，字段和状态信息保存到存储中并从存储中检索时，它具有持久性。组件模型提供了一种持久性机制，该机制使组件的状态能够存储在非易失性位置中，以便以后进行检索。

使持久化成为可能的机制称为序列化 。对象序列化是指将对象转换为数据流并将其写入存储。然后，使用该bean的任何applet，应用程序或工具都可以通过反序列化来“重构”它。然后将对象还原到其原始状态。

例如，Java应用程序可以序列化Microsoft Windows计算机上的“框架”窗口，序列化的文件可以通过电子邮件发送到Solaris计算机，然后Java应用程序可以将“框架”窗口还原到计算机上已存在的确切状态。 Microsoft Windows计算机。

然后，使用该bean的任何applet，应用程序或工具都可以通过反序列化 “重构”它。

所有的豆必须坚持下去。为了持久化，您的bean必须通过实现以下任一方法来支持序列化java.io.Serializable （在API参考文档中）界面，或java.io.Externalizable （在API参考文档中）接口。这些界面为您提供自动序列化和自定义序列化的选择。如果某个类的继承层次结构中的任何类实现Serializable要么Externalizable ，则该类可序列化。

可序列化的类

只要该类或父类实现了java.io.Serializable接口。可序列化类的示例包括Component ， String ， Date ， Vector和Hashtable 。因此， Component类，包括Applet ，可以序列化。不支持序列化的著名类包括Image ， Thread ， Socket和InputStream 。尝试序列化这些类型的对象将导致NotSerializableException 。

Java对象序列化API自动将可序列化对象的大多数字段序列化到存储流。这包括基本类型，数组和字符串。API不会序列化或反序列化标记为瞬态或静态的字段。

控制序列化

您可以控制bean进行的序列化级别。控制序列化的三种方法是：

  • 自动序列化，由Serializable接口。Java序列化软件可序列化整个对象，瞬时字段和静态字段除外。
  • 定制的序列化。通过使用标记，有选择地排除您不想序列化的字段transient （要么static ）修饰符。
  • 定制的文件格式，由Externalizable接口及其两种方法。Bean以特定的文件格式编写。

默认序列化：可序列化的接口

的Serializable接口使用Java对象序列化工具提供自动序列化。 Serializable不声明任何方法；它充当标记，告诉对象序列化工具您的bean类是可序列化的。标记您的班级Serializable表示您正在告诉Java虚拟机（JVM），您已确保您的类将与默认序列化一起使用。以下是有关使用Serializable接口：

  • 实现的类Serializable必须可以访问超类型的无参数构造函数 。当从一个对象“重构”一个对象时，将调用此构造函数。 .ser文件。
  • 您不需要实施Serializable在您的课程中，如果已经在超类中实现的话。
  • 除静态字段和瞬态字段外，所有其他字段均已序列化。使用transient修饰符，用于指定您不想序列化的字段，并指定不可序列化的类。

使用瞬态关键字的选择性序列化

排除字段中序列化的字段Serializable对象用标记字段transient修饰符。

transient int status;

默认序列化将不会序列化transient和static领域。

选择性序列化：writeObject和readObject

如果您的可序列化类包含以下两种方法之一（签名必须准确），则默认的序列化将不会发生。

private void writeObject(java.io.ObjectOutputStream out)
    throws IOException;
private void readObject(java.io.ObjectInputStream in)
    throws IOException, ClassNotFoundException;

您可以通过编写自己的实现来控制序列化更复杂的对象的方式writeObject和readObject方法。实行writeObject当您需要更好地控制要序列化的内容时，需要序列化默认序列化无法处理的对象，或者需要将数据添加到不是对象数据成员的序列化流中时。实行readObject重建您编写的数据流writeObject 。

可外部化的界面

使用Externalizable当您需要完全控制bean的序列化时（例如，在写入和读取特定文件格式时），可以使用接口。要使用Externalizable接口需要实现两种方法： readExternal和writeExternal 。实现的类Externalizable必须有一个无参数的构造函数。
