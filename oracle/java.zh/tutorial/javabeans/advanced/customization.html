Bean定制

定制提供了一种在应用程序构建器中修改Bean的外观和行为的方式，从而可以满足您的特定需求。Bean开发人员可以使用几种自定义级别，以使其他开发人员可以从Bean的潜在功能中获得最大的收益。

以下链接对于了解属性编辑器和定制器很有用：

  • PropertyEditor接口
  • PropertyEditorSupport类
  • PropertyEditorManager类
  • Customizer接口
  • BeanInfo接口

可以在设计时使用符合Bean的构建器工具自定义Bean的外观和行为。定制bean有两种方法：

  • 通过使用属性编辑器 。每个bean属性都有其自己的属性编辑器。NetBeans GUI构建器通常在“属性”窗口中显示bean的属性编辑器。与特定属性类型关联的属性编辑器可编辑该属性类型。
  • 通过使用定制器 。定制程序使您可以对Bean定制进行完全的GUI控制。如果属性编辑器不实用或不适用，则使用定制程序。与与属性关联的属性编辑器不同，定制器与Bean关联。

物业编辑

属性编辑器是用于自定义特定属性类型的工具。在“属性”窗口中激活属性编辑器。该窗口确定属性的类型，搜索相关的属性编辑器，并以相关的方式显示属性的当前值。

属性编辑器必须实现PropertyEditor介面，提供指定属性应如何在属性表中显示的方法。下图表示“属性”窗口，其中包含myBean1特性：

您可以通过单击属性条目来开始编辑这些属性的过程。单击大多数这些条目将弹出单独的面板。例如，要设置foreground要么background使用带有颜色选择的选择框，或按“ ...”按钮以使用标准的ColorEditor窗口。点击toolTipText属性将打开一个StringEditor窗口。

支持班PropertyEditorSupport提供默认的实现PropertyEditor接口。通过将属性编辑器子类化为PropertyEditorSupport ，您可以简单地覆盖所需的方法。

要在“属性”窗口中显示当前属性值“样本”，您需要覆盖isPaintable返回true 。然后，您必须覆盖paintValue将当前属性值绘制在属性表的矩形中。这是如何做ColorEditor贯彻paintValue ：

public void paintValue(java.awt.Graphics gfx, java.awt.Rectangle box) {
    Color oldColor = gfx.getColor();
    gfx.setColor(Color.black);
    gfx.drawRect(box.x, box.y, box.width-3, box.height-3);
    gfx.setColor(color);
    gfx.fillRect(box.x+1, box.y+1, box.width-4, box.height-4);
    gfx.setColor(oldColor);
}

要支持自定义属性编辑器，请重写另外两个方法。覆写supportsCustomEditor返回true，然后覆盖getCustomEditor返回自定义编辑器实例。 ColorEditor.getCustomEditor退货this 。

除此之外PropertyEditorSupport类保持一个PropertyChangeListener列表，并在更改绑定属性时向这些侦听器触发属性更改事件通知。

属性编辑器如何与属性相关联

通过以下方式发现属性编辑器并将其与给定属性相关联：

  • 通过以下方式显式关联BeanInfo宾语。标题属性的编辑器使用以下代码行设置：
    pd.setPropertyEditorClass(TitleEditor.class);
  • 通过以下方式明确注册java.beans.PropertyEditorManager.registerEditor方法。此方法有两个参数：bean类类型和与该类型关联的编辑器类。
  • 名称搜索。如果一个类没有明确关联的属性编辑器，则PropertyEditorManager通过以下方式搜索该类的属性编辑器：
    • 在完全限定的类名称后附加“编辑器”。例如，对于my.package.ComplexNumber类，属性编辑器经理将搜索my.package.ComplexNumberEditor类。
    • 在类名称后附加“编辑器”并搜索类路径。

客制化

您已经了解到，构建器工具为您创建自己的属性编辑器提供了支持。对于复杂的，具有工业强度的咖啡豆，视觉构建者还应满足哪些其他需求？通常不希望在单个（有时是巨大的）属性表上显示bean的所有属性。如果关于bean类型的单一根选择导致一半的属性无关紧要怎么办？JavaBeans规范提供了用户定义的定制程序，通过它可以为bean属性定义比属性编辑器更高级别的定制。

使用bean Customizer时 ，可以完全控制如何配置或编辑bean。定制程序是专门针对Bean定制的应用程序。有时，属性不足以表示Bean的可配置属性。在需要复杂的指令来更改Bean的情况下，以及在属性编辑器过于原始而无法实现Bean定制的情况下，都使用定制程序。

所有定制程序必须：

  • 延伸java.awt.Component或其子类别之一。
  • 实施java.beans.Customizer接口这意味着实现注册方法PropertyChangeListener目标bean发生更改时，在这些侦听器上触发对象更改和触发属性更改事件。
  • 实现默认的构造函数。
  • 通过以下方式将定制器与其目标类相关联BeanInfo.getBeanDescriptor 。
