合成图形

的AlphaComposite类封装了各种合成样式，这些样式决定了如何呈现重叠的对象。一个AlphaComposite也可以使用一个Alpha值来指定透明程度：Alpha = 1.0完全不透明，Alpha = 0.0完全透明（透明）。 AlphaComposite支持下表中所示的大多数标准Porter-Duff合成规则。

合成规则 描述
源上（ SRC_OVER ）
 如果要渲染的对象（源）中的像素与先前渲染的像素（目标）具有相同的位置，则源像素将在目标像素上渲染。
来源输入（ SRC_IN ）
 如果源像素和目标像素重叠，则仅渲染重叠区域中的源像素。
源输出（ SRC_OUT ）
 如果源像素和目标像素重叠，则仅渲染重叠区域之外的源像素。重叠区域中的像素被清除。
目的地（ DST_OVER ）
 如果源像素和目标像素重叠，则仅渲染重叠区域之外的源像素。重叠区域中的像素不变。
目的地（ DST_IN ）
 如果源和目标中的像素重叠，则来自源的Alpha应用于重叠区域中的目标像素。如果alpha = 1.0，则重叠区域中的像素不变。如果alpha为0.0，则清除重叠区域中的像素。
目的地输出（ DST_OUT ）
 如果源和目标中的像素重叠，则来自源的Alpha应用于重叠区域中的目标像素。如果alpha = 1.0，则清除重叠区域中的像素；如果alpha为0.0，则重叠区域中的像素不变。
清除（ CLEAR ）
 如果源和目标中的像素重叠，则清除重叠区域中的像素。

一个新的AlphaComposite ac是通过调用构造的AlphaComposite.getInstance并指定所需的合成规则。

AlphaComposite ac =
  AlphaComposite.getInstance(AlphaComposite.SRC);

当选择其他合成规则或Alpha值时， AlphaComposite.getInstance再次被调用，新的AlphaComposite分配给ac 。除每个像素的alpha值外，还会应用所选的alpha，并将其作为第二个参数传递给AlphaComposite 。 getInstance 。

ac = AlphaComposite.getInstance(getRule(rule), alpha);

通过传递AlphaComposite反对Graphics 2DsetComposite 。将对象渲染为BufferedImage然后将其复制到屏幕上，因此可以在Graphics2D的上下文BufferedImage ：

BufferedImage buffImg = new BufferedImage(w, h,
                        BufferedImage.TYPE_INT_ARGB);
Graphics2D gbi = buffImg.createGraphics();
...
gbi.setComposite(ac);
