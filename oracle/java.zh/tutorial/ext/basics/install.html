已安装的扩展

安装的扩展是Java Runtime Environment（JRE™）软件的lib / ext目录中的JAR文件。顾名思义，JRE是Java开发套件的运行时部分，包含平台的核心API，但没有诸如编译器和调试器之类的开发工具。JRE可以单独使用，也可以作为Java开发工具包的一部分使用。

JRE是JDK软件的严格子集。JDK软件目录树的子集如下所示：

JRE由图中高亮框中的那些目录组成。无论您的JRE是独立的还是JDK软件的一部分，运行时环境都会自动将JRE目录的lib / ext中的任何JAR文件都视为扩展名。

由于已安装的扩展扩展了平台的核心API，因此请谨慎使用。它们很少适合单个或少量应用程序使用的接口。

此外，由于已安装扩展定义的符号在所有Java进程中都是可见的，因此应注意确保所有可见的符号都遵循适当的“反向域名”和“类层次结构”约定。例如， com.mycompany。MyClass 。

从Java 6开始，扩展JAR文件也可以放置在独立于任何特定JRE的位置，以便扩展可以由系统上安装的所有JRE共享。在Java 6之前， java.ext.dirs的值指向单个目录，但是从Java 6开始，它是目录列表（例如CLASSPATH ），用于指定在其中搜索扩展名的位置。路径的第一个元素始终是JRE的lib / ext目录。第二个元素是JRE外部的目录。此其他位置允许扩展JAR文件安装一次，并由该系统上安装的多个JRE使用。位置因操作系统而异：

  • Solaris™操作系统： / usr / jdk / packages / lib / ext
  • Linux： / usr / java / packages / lib / ext
  • Microsoft Windows： ％SystemRoot％\ Sun \ Java \ lib \ ext

请注意，放置在上述目录之一中的已安装扩展名扩展了该系统上每个 JRE（Java 6或更高版本）的平台。

一个简单的例子

让我们创建一个简单的已安装扩展。我们的扩展包含一个类RectangleArea ，该类计算矩形的面积：

public final class RectangleArea {
    public static int area(java.awt.Rectangle r) {
        return r.width * r.height;
    }
}

此类只有一个方法area ，它使用java.awt的实例。矩形并返回矩形的面积。

假设您要使用名为的应用程序测试RectangleArea AreaApp ：

import java.awt.*;

public class AreaApp {
    public static void main(String[] args) {
        int width = 10;
        int height = 5;

        Rectangle r = new Rectangle(width, height);
        System.out.println("The rectangle's area is " 
                           + RectangleArea.area(r));
    }
}

此应用程序实例化一个10 x 5个矩形，然后使用RectangleArea.area方法打印出矩形区域。

在没有扩展机制的情况下运行AreaApp

让我们首先回顾一下如何运行AreaApp无需使用扩展机制的应用程序。我们假设RectangleArea类捆绑在一个名为area.jar的JAR文件中。

当然， RectangleArea类不是Java平台的一部分，因此您需要将area.jar文件放在类路径中才能运行AreaApp没有运行时异常。例如，如果area.jar位于目录/ home / user中 ，则可以使用以下命令：

java -classpath .:/home/user/area.jar AreaApp

此命令中指定的类路径既包含当前目录（包含AreaApp.class） ，又包含包含RectangleArea包的JAR文件的路径。通过运行以下命令，您将获得所需的输出：

The rectangle's area is 50

使用扩展机制运行AreaApp

现在让我们看看你将如何运行AreaApp通过使用RectangleArea类作为扩展。

要使RectangleArea类成为扩展名，请将文件area.jar放在JRE的lib / ext目录中。这样做会自动使RectangleArea成为已安装扩展的状态。

安装area.jar作为扩展，您可以运行AreaApp无需指定类路径：

java AreaApp

由于您使用area.jar作为已安装的扩展程序，因此运行时环境将能够找到并加载RectangleArea即使您没有在类路径上指定它，也可以使用class。同样，系统上任何用户所运行的任何小程序或应用程序都将能够找到并使用RectangleArea类。

如果系统上安装了多个JRE（Java 6或更高版本），并且希望RectangleArea类可用作所有这些的扩展，而不是将其安装在特定JRE的lib / ext目录中，则将其安装在系统范围内的位置。例如，在运行Linux的系统上，将area.jar安装在目录/ usr / java / packages / lib / ext中 。然后， AreaApp可以使用该系统上安装的不同JRE运行，例如，如果将不同的浏览器配置为使用不同的JRE。
