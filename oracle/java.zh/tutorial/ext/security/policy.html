设置扩展权限

如果安全管理器已生效，则必须满足以下条件才能使任何软件（包括扩展软件）执行对安全敏感的操作：

  • 扩展名中对安全敏感的代码必须包装在PrivilegedAction宾语。
  • 由安全管理器实施的安全策略必须向扩展授予适当的权限。默认情况下，已安装的扩展将被授予所有安全权限，就好像它们是核心平台API的一部分一样。安全策略授予的权限仅适用于包装在PrivilegedAction实例。

让我们通过一些示例更详细地研究每种情况。

使用PrivilegedAction类

假设您要修改RectangleArea上一课的扩展示例中的class类将矩形区域写入文件而不是stdout。但是，写入文件是对安全性敏感的操作，因此，如果您的软件要在安全性管理器下运行，则需要将代码标记为特权。您需要执行两个步骤：

  1. 您需要将执行安全敏感操作的代码放置在run类型对象的方法java.security.PrivilegedAction 。
  2. 你必须用那个PrivilegedAction对象作为调用中的参数doPrivileged的方法java.security.AccessController 。

如果我们将这些准则应用于RectangleArea类，我们的类定义如下所示：

import java.io.*;
import java.security.*;

public final class RectangleArea {
    public static void
    writeArea(final java.awt.Rectangle r) {
        AccessController.
          doPrivileged(new PrivilegedAction() {
            public Object run() {
                try { 
                    int area = r.width * r.height;
                    String userHome = System.getProperty("user.home");
                    FileWriter fw = new FileWriter( userHome + File.separator
                        + "test" + File.separator + "area.txt");
                    fw.write("The rectangle's area is " + area);
                    fw.flush();
                    fw.close();
                } catch(IOException ioe) {
                    System.err.println(ioe);
                }
                return null;
            }
        });
    }
}

此类中的单个方法， writeArea ，计算矩形的面积，并将该面积写入一个名为area.txt在里面test用户主目录下的目录。

处理输出文件的安全敏感语句放置在run的新实例的方法PrivilegedAction 。（注意run要求Object实例将被返回。返回的对象可以是null ）新的PrivilegedAction然后在调用时将实例作为参数传递给AccessController.doPrivileged 。

有关使用的更多信息doPrivileged ，请参阅JDK™文档中的Privileged Blocks API 。

将安全性敏感的代码包装在PrivilegedAction以这种方式实现对象是使扩展能够执行对安全敏感的操作的首要要求。第二个要求是：使安全管理器授予特权代码适当的权限。

使用安全策略指定权限

运行时生效的安全策略由策略文件指定。默认策略由文件设置lib/security/java.policy在JRE软件中。

该策略文件通过使用授权条目将安全特权分配给软件。该策略文件可以包含任意数量的授权条目。缺省策略文件具有此授予条目用于已安装的扩展：

grant codeBase "file:${{java.ext.dirs}}/*" {
    permission java.security.AllPermission;
};

此项指定目录中指定的文件file:${{java.ext.dirs}}/*将被授予称为java.security.AllPermission 。（请注意，从Java 6开始， java.ext.dirs指的是类路径的目录路径，每个目录都可以包含已安装的扩展名。）不难猜测java.security.AllPermission授予已安装的扩展所有可能授予的安全性特权。

然后，默认情况下，已安装的扩展没有安全限制。扩展软件可以执行安全敏感的操作，就好像没有安装安全管理器一样，前提是安全敏感代码包含在以下示例中： PrivilegedAction作为参数传递给doPrivileged呼叫。

要限制授予扩展的特权，您需要修改策略文件。要拒绝所有扩展的所有特权，您只需删除上面的授予条目即可。

并非所有权限都像java.security.AllPermission默认授予。删除默认授权条目后，您可以为特定权限输入新的授权条目，包括：

  • java.awt.AWTPermission
  • java.io.FilePermission
  • java.net.NetPermission
  • java.util.PropertyPermission
  • java.lang.reflect.ReflectPermission
  • java.lang.RuntimePermission
  • java.security.SecurityPermission
  • java.io.SerializablePermission
  • java.net.SocketPermission

JDK文档中的 “ 权限”提供了有关每个权限的详细信息。让我们看看使用RectangleArea作为扩展所需的内容。

的RectangleArea.writeArea方法需要两个权限：一个用于确定用户主目录的路径，另一个用于写入文件。假设RectangleArea类捆绑在文件中area.jar ，您可以通过将此条目添加到策略文件中来授予写特权：

grant codeBase "file:${java.home}/lib/ext/area.jar" {
    permission java.io.PropertyPermission "user.home",
        "read";
    permission java.io.FilePermission
        "${user.home}${/}test${/}*", "write";
};

的codeBase "file:${java.home}/lib/ext/area.jar"此条目的一部分保证此条目指定的任何权限仅适用于area.jar 。的java.io.PropertyPermission允许访问属性。第一个论点， "user.home" ，命名属性和第二个参数， "read" ，表示可以读取该属性。（另一种选择是"write" ）

的java.io.FilePermission允许访问文件。第一个论点， "${user.home}${/}test${/}*" ，表示area.jar被授予访问该文件中所有文件的权限test用户主目录中的目录。（注意${/}是与平台无关的文件分隔符。）第二个参数表示所授予的文件访问权仅用于写入。（第二个参数的其他选择是"read" ， "delete"和"execute" ）

签名扩展

您可以使用策略文件，通过要求扩展由可信实体签名来对授予扩展的权限进行其他限制。（有关签名和验证JAR文件的评论，请参阅本教程中的“ 签名JAR文件”课程。）

要允许对扩展或其他软件进行签名验证以及授予权限，策略文件必须包含密钥库条目 。密钥库条目指定在验证中使用哪个密钥库。密钥库条目具有以下形式

keystore "keystore_url";

URL keystore_url是绝对的或相对的。如果是相对的，则URL是相对于策略文件的位置的。例如，要使用keytool使用的默认密钥库，请将此条目添加到java.policy中。

keystore "file://${user.home}/.keystore";

为了表明必须对扩展名进行签名才能获得安全性特权，请使用signedBy领域。例如，以下条目指示扩展名area.jar仅由密钥库中别名为Robert和Rita的用户签名的用户才被授予列出的特权：

grant signedBy "Robert,Rita",
    codeBase "file:${java.home}/lib/ext/area.jar" {
        permission java.io.PropertyPermission
            "user.home", "read";
        permission java.io.FilePermission
            "${user.home}${/}test${/}*", "write";
};

如果codeBase字段被省略，如以下“授予”中所示，该权限被授予由“ Robert”或“ Rita”签名的任何软件，包括已安装或下载的扩展名：

grant signedBy "Robert,Rita" {
    permission java.io.FilePermission "*", "write";  
};

有关策略文件格式的更多详细信息，请参见JDK文档中“ 安全体系结构规范”的 3.3.1节。
