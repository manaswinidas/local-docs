检索和解析方法修饰符

方法声明中可能包含一些修饰符：

  • 访问修饰符： public ， protected和private
  • 修饰符仅限于一个实例：static
  • 禁止值修改的修饰符：final
  • 需要覆盖的修饰符：abstract
  • 防止再进入的修饰剂：synchronized
  • 指示以另一种编程语言实现的修饰符：native
  • 修饰符强制严格的浮点行为：strictfp
  • 注解

的MethodModifierSpy示例列出了具有给定名称的方法的修饰符。它还显示方法是合成的（由编译器生成的），可变变量的方法还是桥接的方法（由编译器生成以支持通用接口的）。


import java.lang.reflect.Method;
import java.lang.reflect.Modifier;
import static java.lang.System.out;

public class MethodModifierSpy {

    private static int count;
    private static synchronized void inc() { count++; }
    private static synchronized int cnt() { return count; }

    public static void main(String... args) {
	try {
	    Class<?> c = Class.forName(args[0]);
	    Method[] allMethods = c.getDeclaredMethods();
	    for (Method m : allMethods) {
		if (!m.getName().equals(args[1])) {
		    continue;
		}
		out.format("%s%n", m.toGenericString());
		out.format("  Modifiers:  %s%n",
			   Modifier.toString(m.getModifiers()));
		out.format("  [ synthetic=%-5b var_args=%-5b bridge=%-5b ]%n",
			   m.isSynthetic(), m.isVarArgs(), m.isBridge());
		inc();
	    }
	    out.format("%d matching overload%s found%n", cnt(),
		       (cnt() == 1 ? "" : "s"));

        // production code should handle this exception more gracefully
	} catch (ClassNotFoundException x) {
	    x.printStackTrace();
	}
    }
}

注意Method.isVarArgs()退货true对于Class.getConstructor() 。这表明方法声明如下所示：

public Constructor<T> getConstructor(Class<?>... parameterTypes)

不像这样：

public Constructor<T> getConstructor(Class<?> [] parameterTypes)

请注意，输出为String.compareTo()包含两种方法。在中声明的方法String.java ：

public int compareTo(String anotherString);

第二种合成或编译器生成的桥接方法。这是因为String实现参数化的接口Comparable 。在类型擦除期间，继承方法的参数类型Comparable.compareTo()从java.lang.Object至java.lang.String 。由于参数类型为compareTo方法中Comparable和String删除后不再匹配，则不会发生覆盖。在所有其他情况下，这会产生编译时错误，因为未实现接口。桥接方法的添加避免了此问题。

Method贯彻java.lang.reflect.AnnotatedElement 。因此，任何带有java.lang.annotation.RetentionPolicy.RUNTIME可能会被检索到。有关获取注释的示例，请参见检查类修饰符和类型 。
