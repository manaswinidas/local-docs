使用定序器方法

的Sequencer界面提供了几种类别的方法：

  • 从MIDI文件或Sequence对象，并将当前加载的音序数据保存到MIDI文件。
  • 与磁带录音机的传输功能类似的方法，用于停止和开始播放和录制，启用和禁用特定轨道上的录制以及在当前播放或录制位置中穿梭Sequence 。
  • 用于查询和设置对象的同步和定时参数的高级方法。一种Sequencer可能会以不同的节奏演奏，有些Tracks静音，并处于与其他对象的各种同步状态。
  • 注册“侦听器”对象的高级方法，这些对象在Sequencer处理某些类型的MIDI事件。

不管哪个Sequencer您将要调用的方法，第一步是获取一个Sequencer系统中的设备，并将其保留供程序使用。

获取音序器

应用程序不会实例化Sequencer ;毕竟， Sequencer只是一个接口。相反，就像Java Sound API的MIDI软件包中的所有设备一样， Sequencer通过静态访问MidiSystem宾语。如访问MIDI系统资源中所述，以下内容MidiSystem方法可用于获取默认值Sequencer ：

static Sequencer getSequencer()

以下代码片段获取默认值Sequencer ，获取所需的任何系统资源并使其可运行：

Sequencer sequencer;
// Get default sequencer.
sequencer = MidiSystem.getSequencer(); 
if (sequencer == null) {
    // Error -- sequencer device is not supported.
    // Inform user and return...
} else {
    // Acquire resources and make operational.
    sequencer.open();
}

的调用open保留音序器设备供程序使用。想象共享一个音序器没有多大意义，因为它一次只能播放一个音序。完成音序器的使用后，您可以通过调用使其对其他程序可用close 。

可以如访问MIDI系统资源中所述获得非默认音序器。

加载序列

从系统获得音序器并保留后，您需要加载音序器应播放的数据。有三种典型的方法可以完成此操作：

  • 从MIDI文件读取音序数据
  • 通过从其他设备（例如MIDI输入端口）接收MIDI消息来实时记录
  • 通过将曲目添加到空序列并添加来以编程方式“从头开始”构建它MidiEvent那些轨道的对象

现在，我们将研究获取序列数据的第一种方法。（下面分别在“ 记录和保存序列”和“ 编辑序列”下介绍了其他两种方法。）第一种方法实际上包含两种略有不同的方法。一种方法是将MIDI文件数据输入到InputStream然后您可以通过以下方式直接读取音序器Sequencer.setSequence(InputStream) 。使用这种方法，您无需显式创建Sequence宾语。实际上， Sequencer实现甚至可能不会创建一个Sequence在幕后，因为某些音序器具有内置的机制，可直接处理文件中的数据。

另一种方法是创建一个Sequence明确地。如果要在播放之前编辑序列数据，则需要使用此方法。使用这种方法，您可以调用MidiSystem's重载方法getSequence 。该方法能够从InputStream ， 一种File或URL 。该方法返回一个Sequence然后可以加载到Sequencer播放。扩展之前的代码摘录，这是获取Sequence来自的对象File并将其加载到我们的sequencer ：

try {
    File myMidiFile = new File("seq1.mid");
    // Construct a Sequence object, and
    // load it into my sequencer.
    Sequence mySeq = MidiSystem.getSequence(myMidiFile);
    sequencer.setSequence(mySeq);
} catch (Exception e) {
   // Handle error and/or return
}

喜欢MidiSystem'sgetSequence方法， setSequence可能会抛出一个InvalidMidiDataException —并且，对于InputStream变体IOException -如果遇到任何麻烦。

播放序列

启动和停止Sequencer使用以下方法完成：

    void start()

和

    void stop()

的Sequencer.start方法开始播放序列。请注意，播放将从序列中的当前位置开始。使用以下命令加载现有序列setSequence如上所述的方法，将定序器的当前位置初始化到序列的最开始。的stop方法停止定序器，但不会自动倒回当前Sequence 。开始停止Sequence无需重置位置，只需从当前位置恢复播放序列即可。在这种情况下， stop方法已用作暂停操作。但是，有各种各样Sequencer开始播放之前将当前序列位置设置为任意值的方法。（我们将在下面讨论这些方法。）

如前所述， Sequencer通常有一个或多个Transmitter对象，通过它发送MidiMessages到一个Receiver 。通过这些Transmitters那一个Sequencer扮演Sequence ，通过发出适当的时间MidiMessages对应于MidiEvents当前包含Sequence 。因此，用于播放Sequence是要调用setReceiver上的方法Sequencer'sTransmitter对象，实际上将其输出连接到将利用回放数据的设备。有关更多详细信息Transmitters和Receivers ，请参考传输和接收MIDI信息 。

编辑序列

许多应用程序允许通过从文件中加载序列来创建音序，还有很多应用程序还允许通过从实时MIDI输入（即录制）中捕获音序来创建音序。但是，某些程序将需要以编程方式或响应用户输入从头开始创建MIDI序列。功能齐全的定序器程序允许用户手动构建新序列，以及编辑现有序列。

这些数据编辑操作不是通过Java Sound API来实现的Sequencer方法，但通过数据对象本身的方法： Sequence ， Track和MidiEvent 。您可以使用以下任一方法创建空序列Sequence构造函数，然后通过调用以下命令向其添加轨道Sequence方法：

    Track createTrack()

如果您的程序允许用户编辑序列，则需要此Sequence删除曲目的方法：

    boolean deleteTrack(Track track)

序列包含曲目后，您可以通过调用以下内容的方法来修改曲目的内容Track类。的MidiEvents包含在Track存储为java.util.Vector在里面Track对象，以及Track提供了一组用于访问，添加和删除列表中事件的方法。方法add和remove相当不言自明，添加或删除指定的MidiEvent从一个Track 。一种get提供了方法，该方法将索引Track's事件列表并返回MidiEvent存储在那里。此外，还有size和tick方法，分别返回MidiEvents在曲目中以及曲目的持续时间，表示为Ticks 。

要在将新事件添加到曲目之前创建一个新事件，您当然可以使用MidiEvent构造函数。要指定或修改事件中嵌入的MIDI消息，您可以调用setMessage适当的方法MidiMessage子类（ ShortMessage ， SysexMessage ， 要么MetaMessage ）。要修改事件应发生的时间，请调用MidiEvent.setTick 。

这些低级方法结合在一起，为功能齐全的音序器程序所需的编辑功能提供了基础。
