查看策略文件效果

现在，您已将所需的策略条目添加到了examplepolicy策略文件，您应该能够在执行GetProps具有安全管理器的应用程序，如下图所示。

每当您运行applet或具有安全管理器的应用程序时，默认情况下加载和使用的策略文件就是“安全属性文件” （位于以下目录之一）中指定的文件：

  • Windows ：java.home\lib\security\java.security
  • UNIX ：java.home/lib/security/java.security

注意： java.home环境变量为JRE的安装目录命名。

策略文件位置被指定为名称形式为以下属性的值：

policy.url.n

那里的变量n表示一个数字。在采用以下格式的行中指定每个属性值：

policy.url.n=URL

其中URL是URL规范。例如，默认策略文件（有时分别称为系统策略文件和用户策略文件）在安全属性文件中定义为

policy.url.1=file:${java.home}/lib/security/java.policy
policy.url.2=file:${user.home}/.java.policy
注意：符号的使用${propName}安全属性文件中的“指定”是一种指定属性值的方法。从而${java.home}将在运行时由的实际值替换"java.home"属性，它指示JRE的安装目录，以及${user.home}将被替换为"user.home"属性，例如， C:\Windows 。

您可以通过两种方式获得examplepolicy除了安全属性文件中指定的策略文件之外，文件还被视为整体策略的一部分。您可以如方法1中所述在传递给运行时系统的属性中指定其他策略文件，也可以如方法2中所述在安全属性文件中添加一行以指定其他策略文件。

方法1

您可以使用-Djava.security.policy解释程序命令行参数，用于指定除安全属性文件中指定的策略文件之外还应使用的策略文件。

确保您在包含以下内容的目录中GetProps.class和examplepolicy 。然后您可以运行GetProps申请并通过examplepolicy在一行上键入以下命令，将策略文件发送给解释器：

java -Djava.security.manager -Djava.security.policy=examplepolicy GetProps

注意：请记住-Djava.security.manager要使用安全管理器运行应用程序是必需的，如“ 如何限制应用程序”步骤中所示。

该程序报告的值"user.home"和"java.home"属性。

如果应用程序仍然报告错误，则策略文件中有问题。使用策略工具来检查您刚刚在“ 设置策略文件以授予必需的权限”步骤中创建的策略条目。

方法2

您可以在中指定许多URL policy.url.n安全属性文件中的所有属性，将加载所有指定的策略文件。所以让你的一种方式examplepolicy文件考虑的文件策略条目java解释器将在安全属性文件中添加指定该策略文件的条目。

重要提示：如果您正在运行自己的JDK副本，则可以轻松地编辑安全属性文件。如果您正在运行与其他用户共享的版本，则只有在您具有对该文件的写权限或要求系统管理员在适当的时候修改该文件的情况下，您才能修改该系统范围的安全属性文件。但是，您可能不适合为此教程测试修改系统范围的策略文件。我们建议您阅读以下内容以了解其完成方式，或者安装自己的专用JDK版本以用于本教程。

要修改安全属性文件，请在适合于编辑ASCII文本文件的编辑器中将其打开。然后在包含以下内容的行之后添加以下行policy.url.2 ：如果您使用的是Windows系统，请添加

policy.url.3=file:/C:/Test/examplepolicy

如果您使用的是UNIX系统，请添加

policy.url.3=file:${user.home}/test/examplepolicy

在UNIX系统上，您也可以显式指定主目录，如下所示：

policy.url.3=file:/home/jones/test/examplepolicy

运行应用程序

现在您应该能够成功运行以下内容。

java -Djava.security.manager GetProps

与方法1一样，如果仍然遇到安全异常，则策略文件中有问题。使用策略工具来检查您刚刚在“ 设置策略文件以授予必需的权限”步骤中创建的策略条目。然后修复所有错别字或其他错误。

重要提示：您不需要包括examplepolicy文件，除非您正在运行本教程课程。要排除此文件，请打开安全属性文件，然后删除刚添加的行。

在继续之前，您可能希望删除刚添加到安全属性文件中的行（或将其注释掉），因为您可能不希望examplepolicy不运行教程课程时包含的文件。
