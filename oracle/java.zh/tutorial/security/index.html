Trail：Java SE中的安全功能

在本教程中，您将学习内置的Java™安全功能如何保护您免受恶意程序的侵害。您将看到如何使用工具来控制对资源的访问，生成和检查数字签名以及创建和管理签名生成和检查所需的密钥。您还将看到如何将加密服务（例如数字签名生成和检查）合并到程序中。

Java开发工具包（JDK™）提供的安全功能面向各种读者：

  • 用户正在运行程序 ：

    内置的安全功能可保护您免受恶意程序（包括病毒）的侵害，维护文件和有关您的信息的隐私，并验证每个代码提供者的身份。您可以根据需要对应用程序和小程序进行安全控制。

  • 开发人员 ：

    您可以使用API方法将安全功能合并到程序中，包括加密服务和安全检查。API框架使您可以定义和集成自己的权限（控制对特定资源的访问），密码服务实现，安全管理器实现和策略实现。此外，还提供了用于管理您的公钥/私钥对和来自您信任的人的公钥证书的类。

  • 系统管理员，开发人员和用户 ：

    JDK工具管理您的密钥库（密钥和证书的数据库）；为JAR文件生成数字签名，并验证此类签名的真实性和签名内容的完整性；并创建和修改定义安装安全策略的策略文件。

注意：对于要创建小程序和Java Web Start应用程序的开发人员，请参阅Java小程序以获取安全性信息。

经验教训

创建策略文件显示了策略文件如何控制资源访问。有关策略配置文件的最新信息，请参阅“ 策略指南”页面。

控制应用程序快速浏览以上一课为基础，显示了如何除非安全策略文件中的权限明确允许，否则不允许在安全管理器下运行的应用程序进行资源访问（例如读取或写入文件）。

用于安全代码和文件交换的API和工具定义了数字签名，证书和密钥库，并讨论了为什么需要它们。它还回顾了适用于接下来的三课的信息，这些信息涉及使用工具或API生成签名，导出/导入证书等通常所需的步骤。

对代码进行签名和授予权限说明了所有与安全性相关的工具的使用。它显示了开发人员将要签名和分发代码以供其他人运行的步骤。该课程还显示了将运行代码的人员（或系统管理员）如何在策略文件中添加条目以授予代码权限以访问其所需的资源。

交换文件表明一个人使用工具来签署重要文件（例如合同），并导出与用于签署合同的私钥相对应的公钥的公钥证书。然后，该课程说明了接收合同，签名和公钥证书的另一个人如何导入证书并验证签名。

生成和验证签名将逐步引导您完成一个示例，该示例使用JDK Security API编写Java程序以生成密钥，使用私钥生成数据的数字签名，以及将公钥和签名导出到文件。然后，该示例显示编写第二个程序，该程序可以预期在其他人的计算机上运行，该程序可以导入公钥并验证签名的真实性。最后，该示例讨论了基本程序使用的方法的潜在弱点，并演示了提供和导入密钥（包括证书）的可能替代方法和方法。

实现自己的权限演示了如何编写定义其自己的特殊权限的类。

想要查询更多的信息
