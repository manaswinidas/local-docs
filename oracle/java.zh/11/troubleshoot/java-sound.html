15 Java声音

本章描述了Java声音技术可能引起的一些问题，并提出了原因和解决方法。

以下主题描述了解决Java声音问题的方案。

解决Java声音问题，例如系统声音配置，音频文件格式，音频格式以及溢出和欠载情况。

  • 系统声音配置：

    确保正确配置了音频系统（用于Windows的声卡驱动程序/ DirectSound，用于Linux的ALSA，用于Oracle Solaris的Audio Mixer）。此外，请确保已连接扬声器，并且已将声卡的音量和静音状态调整为适当的值。要测试您的声音配置，请运行任何本地声音应用程序并通过它播放一些声音。

    在Oracle Solaris和Linux操作系统上，您可能无法播放声音，因为应用程序（或声音守护程序，例如esd要么artsd ）专门打开音频设备，从而拒绝Java Sound对设备的访问。

  • 音频文件格式：

    Java Sound支持一组音频文件格式，例如AU，AIF和WAV。大多数文件格式只是容器，可以包含各种压缩音频格式的音频数据。Java Sound文件阅读器支持某些格式（未压缩的PCM，a-law，mu-law），但不支持ADPCM，MP3和其他格式。

    Java Sound还通过服务提供商接口（SPI）支持文件读取器和写入器的插件。您可以使用Sun，第三方或您自己的插件来读取各种音频文件。在任何情况下，您都必须处理插件的存在，例如，通过在应用程序中分发所需的插件，或者要求在客户端Java环境中安装插件。

  • 音频格式：

    Java Sound支持各种音频格式，但是它们的可用性取决于操作系统。要使用某种音频格式进行录制或播放，系统（声卡驱动程序）必须支持该格式。尽可能使用受支持的格式：PCM； 8或16位； 8000、11025、22050、44100 Hz。大多数声卡都支持这些格式。大多数声卡仅支持PCM格式，即使驱动程序支持mu-law，也需要对软件进行一些修改。如果您需要播放或记录mu-law数据，则首选方法是通过格式转换器将其转换为PCM格式。

    看到AudioSystem.getAudioInputStream有关格式转换的详细信息的文档。

  • 超限和欠限条件：

    记录的数据保存在DataLine缓冲。如果您很长时间没有从该行读取数据，则会发生溢出情况，并且较旧的数据将被新数据替换。这将在记录的音频数据中产生伪像。

    播放时也会发生类似情况。如果已播放了缓冲区中的所有数据，并且没有新数据写入该行，则将发生欠载情况，并且将播放静音，直到将音频数据的新部分写入该行为止。

    记录的首选方法是在单独的线程中读取数据，以防止其他任务的可能影响（例如，UI处理）。如果您使用SourceDataLine对于播放，则使用单独的线程将数据写入该行也是首选方法。如果您使用Clip玩，然后Clip实现本身会创建这种类型的线程。
