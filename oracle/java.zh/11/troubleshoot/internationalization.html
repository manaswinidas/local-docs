国际化和本地化疑难解答

解决国际化与本地化之间的差异。

在进行故障排除之前，请确保您了解国际化和本地化之间的区别：

  • 国际化是设计软件的过程，因此可以以经济高效的方式轻松地将其适应（本地化）各种语言和地区，而无需更改软件。此过程通常涉及隔离依赖于语言和文化的程序部分。例如，错误消息的文本与程序源代码保持分开，因为在本地化期间必须翻译错误消息。

  • 本地化是使程序适应特定语言环境的过程。语言环境是具有相同语言和习俗的地理或政治区域。本地化包括文本的翻译，例如用户界面标签，错误消息和联机帮助。它还包括特定于文化的数据项格式，例如货币值，时间，日期和数字。

Java SE平台中的用户界面库支持开发丰富的交互式应用程序。国际化方面包括文本输入，文本显示和用户界面布局。以下描述显示了国际化与AWT，Java 2D和Swing API提供的功能之间的关系：

  • 文本输入是通过在键盘上键入或通过前端软件（例如输入法，手写识别或语音输入）将新文本输入文档的过程。

  • 文本显示是一个多步骤过程，包括选择字体，将文本排列到段落和行中，为字符或字符序列选择字形以及渲染这些字形。一些书写系统需要双向文本布局或复杂的字符到字形映射。文本显示由Java 2D图形系统和Swing工具箱（用于轻量级用户界面组件）以及AWT（对等用户界面组件）处理。

  • 用户界面布局需要适应由本地化引起的文本扩展或收缩，并与用户书写系统的方向相匹配。
