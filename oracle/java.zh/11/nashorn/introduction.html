1 Nashorn引擎简介

本节提供有关Nashorn引擎以及如何将其用于在Java应用程序中或从命令行解释JavaScript代码的介绍性信息。

注意：

Nashorn引擎jjs JDK 11中不推荐使用jdk.scripting.nashorn和jdk.scripting.nashorn.shell模块，以准备在将来的版本中删除。

Nashorn引擎是ECMAScript版本5.1语言规范的实现 。它还实现了ECMAScript 6中引入的许多新功能，包括模板字符串。 let ， const ，并阻止作用域；迭代器和for..of循环; Map ， Set ， WeakMap和WeakSet数据类型；符号;以及二进制和八进制文字。

作为Nashorn项目的一部分，它完全使用Java语言开发。该代码基于Da Vinci Machine的新功能，该功能是Java Specification Request（JSR）292（在Java平台上支持动态类型语言）的参考实现。

Nashorn引擎包含在Java SE开发工具包（JDK）中。您可以使用Java脚本API从Java应用程序调用Nashorn来解释嵌入式脚本，也可以将脚本传递给jjs要么jrunscript工具。

注意：

Nashorn是JDK中包含的唯一JavaScript引擎。但是，您可以使用任何符合JSR 223的脚本引擎：Java平台的脚本或实现您自己的脚本引擎。请参阅《 Java平台标准版Java脚本程序员指南》中的“ 脚本语言和Java ” 。

从Java代码调用Nashorn

要在Java应用程序中调用Nashorn，请使用Java脚本API创建Nashorn引擎的实例。

要获取Nashorn引擎的实例：

  1. 导入javax.script包。

    Java脚本API由此程序包中的类和接口组成。

  2. 创建一个ScriptEngineManager宾语。

    的ScriptEngineManager类是Java脚本API的起点。一种ScriptEngineManager对象用于实例化ScriptEngine对象并维护它们共享的全局变量值。

  3. 得到ScriptEngine经理使用的对象getEngineByName()方法。

    这种方法需要一个String参数与脚本引擎的名称。要获取Nashorn引擎的实例，请传入"nashorn" 。或者，您可以使用以下任一方法： "Nashorn" ， "javascript" ， "JavaScript" ， "js" ， "JS" ， "ecmascript" ， "ECMAScript" 。

拥有Nashorn引擎实例后，可以使用它来评估语句和脚本文件，设置变量等。下面的例子， EvalScript.java ，提供了简单的Java应用程序代码来评估print("Hello, World!");使用Nashorn的声明。 

import javax.script.*;

public class EvalScript {
    public static void main(String[] args) throws Exception {
        // create a script engine manager
        ScriptEngineManager factory = new ScriptEngineManager();
        // create a Nashorn script engine
        ScriptEngine engine = factory.getEngineByName("nashorn");
        // evaluate JavaScript statement
        try {
            engine.eval("print('Hello, World!');");
        } catch (final ScriptException se) { se.printStackTrace(); }
    }
}

注意：

的eval()方法抛出一个ScriptException必须正确处理。

从命令行调用Nashorn

有两种命令行工具可用于调用Nashorn引擎：

  • jrunscript

    这是一个通用命令，可调用任何符合JSR 223的可用脚本引擎。默认情况下，没有任何选项， jrunscript调用Nashorn引擎，因为它是JDK中的默认脚本引擎。

  • jjs

    这是推荐的工具，专门为Nashorn创建。要使用Nashorn评估脚本文件，请将脚本文件的名称传递给jjs工具。要启动可解释使用标准输入传递的语句的交互式外壳，请启动jjs工具，无需指定任何脚本文件。

Nashorn解析器API

Nashorn解析器API使应用程序（尤其是IDE和服务器端框架）能够解析和分析ECMAScript代码。

使用Parser类中的方法从字符串，URL或文件中解析ECMAScript代码。这些方法返回CompilationUnitTree的实例，该实例将ECMAScript代码表示为抽象语法树。软件包jdk.nashorn.api.tree包含Nashorn解析器API。
