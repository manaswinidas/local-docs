1 Java虚拟机技术概述

本章描述了Java虚拟机（JVM）的实现以及Java HotSpot技术的主要功能：

  • 自适应编译器 ：标准解释器用于启动应用程序。当应用程序运行时，将对代码进行分析以检测性能瓶颈或热点 。Java HotSpot VM会编译代码的性能关键部分以提高性能，但不会编译很少使用的代码（大多数应用程序）。Java HotSpot VM使用自适应编译器来决定如何使用内联之类的技术来优化已编译的代码。
  • 快速的内存分配和垃圾收集 ：Java HotSpot技术为对象和快速，高效，最新的垃圾收集器提供了快速的内存分配。
  • 线程同步 ：Java HotSpot技术提供了一种线程处理功能，该功能旨在扩展用于大型共享内存多处理器服务器。

在Oracle Java Runtime Environment（JRE）8和更早版本中，对于通常用作客户端，服务器和嵌入式系统的配置，支持JVM的不同实现（客户端VM，服务器VM和最小VM）。由于现在大多数系统都可以利用服务器VM，因此Oracle Java Runtime Environment（JRE）9仅提供该VM实现。
