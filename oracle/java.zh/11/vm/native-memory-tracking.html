8本机内存跟踪

本章介绍了本机内存跟踪（NMT）功能。NMT是Java Hotspot VM功能，可跟踪HotSpot VM的内部内存使用情况。您可以使用来访问NMT数据jcmd效用。NMT不跟踪第三方本机代码和Oracle Java Development Kit（JDK）类库的内存分配。NMT不包括NMT MBean在HotSpot for Java Mission Control（JMC）中。

主要特征

当您将本机内存跟踪与jcmd ，您可以跟踪不同级别的Java虚拟机（JVM）或HotSpot VM内存使用情况。NMT仅跟踪JVM或HotSpot VM使用的内存，而不跟踪用户的本机内存。NMT没有提供有关类数据共享（CDS）存档所使用的内存的完整信息。

默认情况下，HotSpot VM的NMT已关闭。您可以使用JVM命令行选项打开NMT。见的Java在Java平台标准版工具参考有关高级运行选项的信息。

您可以使用jcmd效用。请参阅使用jcmd访问NMT数据 。您可以使用来停止NMT jcmd实用程序，但是您不能使用来启动或重新启动NMT jcmd实用。

NMT支持以下功能：

  • 生成摘要和详细报告。

  • 建立一个早期基准，以便以后进行比较。

  • 使用JVM命令行选项在JVM出口处请求内存使用情况报告。请参阅VM出口处的NMT 。

使用本机内存跟踪

您必须启用NMT，然后使用jcmd实用程序以访问NMT数据。

启用NMT

要启用NMT，请使用以下命令行选项：

-XX:NativeMemoryTracking=[off | summary | detail]

注意：

启用NMT会导致5％-10％的性能开销。

下表描述了NMT命令行用法选项：

表8-1 NMT使用选项

NMT选项 描述
off

NMT已开启off默认。

summary

仅收集子系统汇总的内存使用情况。
detail

收集各个呼叫站点的内存使用情况。

使用jcmd访问NMT数据

使用jcmd转储收集的数据，并可以选择将数据与最后一个基准进行比较。

jcmd VM.native_memory [summary | detail | baseline | summary.diff | detail.diff | shutdown] [scale= KB | MB | GB]

表8-2 jcmd NMT选项

jcmd NMT选项 描述
summary

打印摘要，按类别汇总。
detail
  • 打印内存使用情况，按类别汇总
  • 打印虚拟内存图
  • 打印内存使用情况，按呼叫站点汇总
baseline

创建一个新的内存使用情况快照以进行比较。
summary.diff

根据最后一个基准打印新的摘要报告。
detail.diff

根据最后一个基准打印新的详细报告。
shutdown

停止NMT。

在VM出口获取NMT数据

要获取有关VM退出时最后一次使用内存的数据，请在启用本机内存跟踪后使用以下VM诊断命令行选项。详细程度基于跟踪级别。

-XX:+UnlockDiagnosticVMOptions -XX:+PrintNMTStatistics

有关如何监视VM内部内存分配和诊断VM内存泄漏的信息，请参阅《 Java Platform，Standard Edition故障排除指南》中的“本机内存跟踪” 。
