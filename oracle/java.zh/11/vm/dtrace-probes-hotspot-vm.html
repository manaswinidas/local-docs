HotSpot VM中的9个 DTrace探针

本章介绍了Oracle HotSpot VM中的DTrace支持。使用hotspot和hotspot_jni提供程序，您可以访问探针，这些探针可用于监视与Java虚拟机（JVM）的内部状态和活动一起运行的Java应用程序。所有探针都是USDT探针，您可以使用JVM进程的process-id进行访问。

使用热点提供商

hotspot提供程序使您可以访问探针，这些探针可用于跟踪VM的寿命，线程启动和停止事件，垃圾收集器（GC）和内存池统计信息，方法编译以及监视活动。启动标志可以启用可用于监视正在运行的Java程序的其他探针，例如对象分配以及方法输入和返回探针。hotspot探针起源于VM库（libjvm.so），因此它们是从嵌入VM的程序中提供的。

提供程序中的许多探针都有用于提供有关VM状态的更多详细信息的参数。这些参数中有许多是不透明的ID，可用于将探针触发彼此链接。但是，还提供了字符串和其他数据。提供字符串值时，它们始终成对出现：指向未终止的修改后的UTF-8数据的指针（请参阅JVM Specification ），以及指示该数据范围的长度值。不能保证字符串数据以NUL字符结尾，因此必须使用以长度结尾的copyinstr()内部读取字符串数据。即使所有字符都没有超出ASCII范围也是如此。

监控探针

当Java应用程序运行时，线程会进入和退出监视器，在监视器上等待并执行通知。探针可用于所有等待和通知事件，以及竞争的监视器进入和退出事件。

当一个线程试图进入监视器而另一个线程在监视器中时，发生竞争的监视器条目。当一个线程离开监视器而其他线程正在等待进入监视器时，发生竞争监视器退出事件。竞争的监视器入口事件和竞争的监视器出口事件可能相对于遇到这些事件的线程彼此不匹配，尽管一个线程的竞争出口应与另一个线程上的竞争输入匹配（等待进入线程的线程）。监控）。

监视器事件提供线程ID，监视器ID和对象类的类型作为参数。线程ID和类类型可以映射回Java程序，而监视器ID可以提供探针触发之间的匹配信息。

虚拟机中存在这些探针会降低性能，并且仅在-XX:+ExtendedDTraceProbes标志在Java命令行上设置。该标志在运行时通过使用jinfo效用。

如果该标志为off，则可以从Dtrace获取的探针列表中包含监视探针，但是探针保持休眠状态并且不会启动。计划在将来的VM版本中删除此限制，并且将启用这些探针，而不会影响性能。

以下探针可用于监视事件：

探测 描述

当线程成功进入竞争监视器时启动的探针

monitor-contended-exit

当线程离开监视器并且其他线程正在等待进入时启动的探针

monitor-wait

通过使用以下命令在线程开始在监视器上等待时启动的探针Object.wait() 。额外的论据， args[4]是一个长值，指示正在使用的超时。

使用hotspot_jni提供程序

为了从本机代码调用Java代码，由于将VM嵌入应用程序中或在Java应用程序中执行本机代码，本机代码必须通过Java本机接口（JNI）进行调用。JNI提供了许多调用Java代码和检查VM状态的方法。每种方法的入口点和返回点都提供了DTrace探针。探针由hotspot_jni提供程序提供。探针的名称是JNI方法的名称，后跟-entry用于进入探针，以及-return用于返回探针。每个入口探针可用的参数是提供给函数的参数，但Invoke*方法，它忽略了传递给Java方法的参数。返回探测器将方法的返回值作为参数（如果可用）。

DTrace探针样本

  probe vm-init-begin();
  probe vm-init-end();
  probe vm-shutdown();
  probe class-loaded(
      char* class_name, uintptr_t class_name_len, uintptr_t class_loader_id, bool is_shared);
  probe class-unloaded(
      char* class_name, uintptr_t class_name_len, uintptr_t class_loader_id, bool is_shared);
  probe gc-begin(bool is_full);
  probe gc-end();
  probe mem-pool-gc-begin(
      char* mgr_name, uintptr_t mgr_name_len, char* pool_name, uintptr_t pool_name_len, 
      uintptr_t initial_size, uintptr_t used, uintptr_t committed, uintptr_t max_size);
  probe mem-pool-gc-end(
      char* mgr_name, uintptr_t mgr_name_len, char* pool_name, uintptr_t pool_name_len, 
      uintptr_t initial_size, uintptr_t used, uintptr_t committed, uintptr_t max_size);
  probe thread-start(
      char* thread_name, uintptr_t thread_name_length, 
      uintptr_t java_thread_id, uintptr_t native_thread_id, bool is_daemon);
  probe thread-stop(
      char* thread_name, uintptr_t thread_name_length,

      char* signature, uintptr_t signature_len);
