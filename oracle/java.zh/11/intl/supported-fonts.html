5种支持的字体

在Java平台的不同实现之间，受支持的字体集有所不同。有关使用的术语，请参见Font类说明。

  • 支持物理字体
  • 支持逻辑字体
  • Lucida字体

支持物理字体

JRE支持TrueType和PostScript Type 1字体。

物理字体需要安装在Java运行时环境已知的位置。JRE在两个位置查找： lib/fonts JRE本身中的目录，以及主机操作系统定义的常规字体位置。如果两个位置都存在相同名称的字体，则lib/fonts使用目录。

您可以通过以主机操作系统支持的方式安装使用支持的字体技术的物理字体来添加它们。建议在Solaris或Linux上添加每用户字体的位置是$HOME/.fonts平台的搜索目录libfontconfig ，然后由JRE使用。

请勿根据lib JRE中的目录，因为自JDK 9开始不支持此目录。

支持逻辑字体

逻辑字体名称以实现相关的方式映射到物理字体。通常，一个逻辑字体名称会映射到几种物理字体，以覆盖大范围的字符。JRE使用字体配置文件来定义映射。请参阅字体配置文件 。

Lucida字体

Oracle JRE包含“ Lucida”设计家族的几种物理字体。这些字体是物理字体，但是由于它们是JRE附带的，因此它们不依赖于主机操作系统。这些字体也被许可在Java平台的其他实现中使用。使用它们可以带来跨平台和多种语言实现的一致外观的好处。

有三种不同类型的族： "Lucida Sans" ， "Lucida Sans Typewriter"和"Lucida Bright" 。每个家庭都有简单，粗体，斜体和粗斜体样式。并非所有JRE实现中都存在所有这些。例如，用于Windows的JRE的默认下载包仅包含Lucida Sans纯字体（但请注意，应用程序开发人员可以包括所有Lucida字体以及他们随应用程序重新分发的JRE）。
