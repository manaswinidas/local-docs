1国际化增强

JDK的最新版本包括对国际化过程的增强，以支持更新的标准。

话题：

  • JDK 11中的国际化增强功能

  • JDK 10中的国际化增强功能

  • JDK 9中的国际化增强功能

JDK 11中的国际化增强功能

JDK 11的国际化增强功能包括：

  • Unicode 10.0.0

Unicode 10.0.0

已添加对Unicode 10.0.0的支持。Java平台标准版（Java SE）9和10支持Unicode 8.0。

Unicode 10.0标准包括自Unicode 8.0以来引入的16,018个字符和10个脚本，Java SE 11支持所有这些脚本。

JDK 10中的国际化增强功能

JDK 10的国际化增强功能包括：

  • 其他Unicode语言标签扩展

其他Unicode语言标签扩展

自Java SE 7开始， Locale类已支持IETF BCP（最佳实践）47种语言标签标准，其中包括Unicode扩展子标签。从Java SE 9开始，只有-ca （日历）和-nu支持（扩展名）。

Java SE 10在相关的JDK类中增加了对以下附加扩展的支持：

  • -cu （货币类型）

  • -fw （一周的第一天）

  • -rg （区域覆盖）

  • -tz （时区）

在JDK 10中，如果应用程序指定的语言环境为en-US-u-cu-EUR ，表示美国英语和欧元货币java.util。Currency.getInstance（locale）实例化欧元Currency 。如果语言环境是en-US-u-cu-JPY ，日元Currency被实例化。

JDK 9中的国际化增强功能

Oracle Java Development Kit 9的国际化增强功能包括：

  • Unicode 8.0

  • 默认情况下启用了CLDR语言环境数据

  • UTF-8属性文件

Unicode 8.0

已添加对Unicode 8.0的支持。Java平台标准版（Java SE）8支持Unicode 6.2。

Unicode的6.3 ， 7.0 ，以及8.0标准引入10555个字符，29个脚本，和42块，所有这些在Java SE 9支撑。

默认情况下启用了CLDR语言环境数据

首次在JDK 8中添加的Unicode通用语言环境数据存储库（CLDR）的基于XML的语言环境数据是JDK 9中的默认语言环境数据。在以前的版本中，默认值为JRE 。

语言环境数据有四个不同的来源，由以下关键字标识：

  • CLDR表示Unicode CLDR项目提供的语言环境数据。

  • HOST表示当前用户对基础操作系统设置的自定义。它仅适用于用户的默认语言环境，并且可自定义的设置可能会因操作系统而异。但是，主要支持日期，时间，数字和货币格式。

  • SPI表示由已安装的服务提供商接口（SPI）提供程序实现的对语言环境敏感的服务。

  • COMPAT （以前称为JRE ）表示与JDK 9之前的发行版兼容的语言环境数据。 JRE仍可以用作值，但是COMPAT是首选。

要选择语言环境数据源，请使用java.locale.providers系统属性，以首选顺序列出数据源。如果提供者无法提供请求的语言环境数据，则搜索将按顺序进行到下一个提供者。例如：

java.locale.providers=HOST,SPI,CLDR,COMPAT

如果未设置此属性，则默认行为等效于以下设置：

java.locale.providers=CLDR,COMPAT,SPI

要启用与JDK 8兼容的行为，请设置java.locale.providers系统属性值COMPAT至左侧CLDR 。

有关受支持的语言环境，请参见JDK 9和JRE 9支持的语言环境页面。参见java.util.spi。相关API的LocaleServiceProvider API规范 。

UTF-8属性文件

在Java SE 9中，属性文件以UTF-8编码加载。在以前的版本中，ISO-8859-1编码用于加载属性资源束。UTF-8是表示非拉丁字符的一种更为方便的方法。

大多数现有的属性文件都不会受到影响：UTF-8和ISO-8859-1的ASCII字符编码相同，并且人类可读的非ASCII ISO-8859-1编码无效的UTF-8。如果检测到无效的UTF-8字节序列，则Java运行时将自动重新读取ISO-8859-1中的文件。

如果有问题，请考虑以下选项：

  • 将属性文件转换为UTF-8编码。

  • 为属性文件的编码指定运行时系统属性，如以下示例所示：
    java.util.PropertyResourceBundle.encoding=ISO-8859-1

参见java.util。PropertyResourceBundle 。
