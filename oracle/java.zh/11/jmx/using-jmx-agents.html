4使用JMX代理

Java管理扩展（JMX）代理是在Java虚拟机（JVM）中运行的管理实体，并充当受管Bean（MBean）与管理应用程序之间的联络人。以下各节概述了JMX代理的各种组件：

  • MBean服务器
  • 代理服务
  • 协议适配器和连接器

MBean服务器

MBean服务器是JMX代理的核心组件。它是JMX代理中暴露给管理操作的对象的注册表。向MBean服务器注册的对象对于管理应用程序是可见的。MBean服务器仅公开MBean的管理接口，而不公开其直接对象引用。

您想要从代理JVM外部管理的任何资源都必须在服务器中注册为MBean。MBean服务器提供了用于访问同一JVM中的MBean的标准化接口，从而为本地对象提供了处理可管理资源的所有好处。可以通过以下方式实例化和注册MBean：

  • 另一个MBean
  • 代理本身
  • 远程管理应用程序

注册MBean时，必须为其分配一个唯一的对象名称。管理应用程序使用对象名称来标识要在其上执行管理操作的对象。MBean上可用的操作包括：

  • 发现MBean的管理界面
  • 读取和写入其属性值
  • 执行由MBean定义的操作
  • 获取MBean发出的通知
  • 使用对象名称或属性值查询MBean

代理服务

代理服务是可以在向MBean服务器注册的MBean上执行管理操作的对象。通过将管理智能纳入代理，JMX可帮助您构建更强大的管理解决方案。代理服务也可以是MBean，从而可以通过MBean服务器控制它们及其功能。JMX规范定义了以下代理服务：

  • 通过管理小程序（m-let）服务动态加载类，可以检索并实例化从网络动态下载的新类和本机库。
  • 监视MBean属性的数字或字符串值，并可以将几种类型的更改通知其他对象。
  • 计时器提供了一种调度机制，可以按预定的时间间隔发送通知。
  • 关系服务定义MBean之间的关联并维护关系的一致性。

协议适配器和连接器

协议适配器和连接器使代理可以从远程管理应用程序访问。它们提供了通过实例化并在MBean服务器上注册的MBean的特定协议的视图。它们使JVM外部的管理应用程序能够：

  • 获取或设置现有MBean的属性
  • 在现有MBean上执行操作
  • 实例化并注册新的MBean
  • 注册并接收MBean发出的通知

因此，为了使JMX代理易于管理，它必须至少包含一个协议适配器或连接器。Java SE平台包括标准的远程方法调用（RMI）连接器。代理可以包括一个或多个协议适配器和连接器，从而允许通过不同协议同时进行远程管理和监视。

协议适配器

协议适配器通过给定协议提供JMX代理的管理视图。它们使MBean和MBean服务器的操作适应给定协议中的表示形式，并可能适应不同的信息模型，例如SNMP。 Java SE平台不包含任何标准协议适配器。

连接到协议适配器的管理应用程序通常特定于给定协议。对于依赖特定管理协议的旧式管理解决方案，通常就是这种情况。他们不是通过MBean服务器的远程表示，而是通过映射到MBean服务器的操作访问JMX代理。

连接器

连接器用于将代理与启用了JMX技术的远程管理应用程序（即使用JMX规范的分布式服务开发的管理应用程序）连接。这种通信涉及代理中的连接器服务器和管理器中的连接器客户端。

这些组件通过特定协议透明地点对点传达管理操作。JMX远程API提供了到MBean服务器的远程接口，管理应用程序可以通过该接口执行操作。连接器特定于给定协议，但是管理应用程序可以使用任何连接器，因为它们具有相同的远程接口，因此可以无差异地使用它们。

有关标准JMX连接器的更多信息，请参见使用JMX连接器远程管理资源 。
