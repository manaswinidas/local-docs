5使用JMX连接器远程管理资源

Java管理扩展工具和代理规范定义了连接器的概念。连接器使基于Java技术的远程客户端可以访问Java Management Extensions（JMX）技术MBean服务器。连接器的客户端导出的接口基本上与MBean服务器相同。

连接器由连接器客户端和连接器服务器组成。连接器服务器连接到MBean服务器，并侦听来自客户端的连接请求。连接器客户端与连接器服务器建立连接。连接器客户端通常与连接器服务器位于不同的Java虚拟机（JVM）中，并且通常在不同的计算机上运行。

连接器的许多不同实现方式都是可能的。特别是，用于通过客户端和服务器之间的连接进行通信的协议有很多可能性。

连接器服务器通常具有一个地址，用于在连接器客户端和连接器服务器之间建立连接。或者，某些连接器可以提供连接桩以建立连接。建立连接的方式取决于您使用的发现和查找技术。请参阅发现和查找服务 。

本章在以下各节中概述了JMX远程API规范定义的不同类型的连接器及其使用的协议：

  • RMI连接器 ：符合JMX远程API标准的每个实现都必须支持标准的远程方法调用（RMI）协议。
  • 通用连接器 ：JMX远程API标准还定义了一个直接基于TCP套接字的可选协议，称为JMX消息协议（JMXMP）。该标准的实现可以忽略JMXMP连接器，但不能忽略RMI连接器。Java SE平台不包括可选的JMXMP连接器。
  • 用户定义的协议 ：连接器还可以实现JMX技术中未定义的协议。

RMI连接器

JMX远程API标准定义了基于RMI的标准协议。在JMX远程API的每个实现中都必须存在RMI连接器。

RMI连接器支持Java远程方法协议（JRMP）传输。

JRMP上的RMI连接器提供了一种简单的机制，用于保护和认证客户端与服务器之间的连接。该机制为使用RMI连接器的环境提供了基本的安全性。请注意，通用JMXMP连接器提供了更高级的安全性。

您可以通过使用RMI套接字工厂来提高通过JRMP的RMI连接器的安全性，以便客户端和服务器之间的连接使用安全套接字层（SSL）。

通用连接器

JMX远程API规范定义了一个可选的通用连接器，该连接器未包含在Java SE平台中。可以通过添加可插拔模块来定义以下内容来配置此连接器：

  • 传输协议，用于从客户端向服务器发送请求，以及从服务器向客户端发送响应和通知
  • 从客户端发送到服务器的对象的对象包装，其类装入器可以取决于目标MBean

JMX消息传递协议（JMXMP）连接器是通用连接器的配置，其中传输协议基于TCP，对象包装是本机Java序列化。安全性比RMI连接器更高。安全性基于Java安全套接字扩展（JSSE），Java身份验证和授权服务（JAAS）和简单身份验证和安全层（SASL）。

通用连接器及其JMXMP配置是可选的，这意味着它们并不总是包含在JMX Remote API的实现中。 Java SE平台不包括可选的通用连接器。

用户定义的协议

JMX远程API规范并未为每个协议定义连接器。您可以基于JMX Remote API标准中未定义的协议来实现连接器。例如，您可以基于使用HTTP / S的协议来实现连接器。 JMX规范描述了如何基于用户定义的协议实现连接器。
