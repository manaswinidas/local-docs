在64位Windows平台上安装JDK的系统要求

有关受支持的处理器和浏览器，请参见《 Oracle JDK认证的系统配置》 。

Windows的JDK安装说明符号

对于本文档中包含以下符号的任何文本，必须替换适当的更新版本号：

interim.update.patch

例如，如果要下载用于64位系统的JDK安装程序以更新11 Interim 0，Update 0和Patch 0，则文件名jdk-11.interim.update.patch_windows-x64_bin.exe变成jdk-11_windows-x64_bin.exe 。

Windows的JDK安装说明

您运行一个自我安装的可执行文件来解压缩并在Windows计算机上安装JDK。

通过执行以下主题中描述的操作，在Windows计算机上安装JDK：

  • 下载JDK安装程序

  • 运行JDK安装程序

  • 静默安装JDK

  • 设置PATH环境变量

下载JDK安装程序

访问Java SE下载页面，然后单击接受许可协议 。在下载菜单中，单击对应的下载链接.exe适用于您的Windows版本。

下载档案jdk-11.interim.update.patch_windows-x64_bin.exe 。

注意：

通过比较下载页面上的文件大小和本地驱动器，验证文件下载是否成功完成。或者，您可以确保下载的文件的校验和与Java SE Downloads页面上提供的校验和匹配。

运行JDK安装程序

您必须具有管理员权限才能在Microsoft Windows上安装JDK。
要运行JDK安装程序：
  1. 通过在下载位置中双击安装程序的图标或文件名来启动JDK 11安装程序。
  2. 请遵循安装向导提供的说明。
  3. 安装完成后，删除下载的文件以恢复磁盘空间。

静默安装JDK

您无需双击或打开JDK安装程序，而可以使用命令行参数执行静默的，非交互式的JDK安装。

下表列出了示例安装方案以及执行它们所需的命令。jdk表示下载的安装程序文件的基本名称，例如jdk-11_windows-x64_bin.exe 。

安装方案 命令
以静默方式安装JDK。 
jdk.exe /s
以静默方式安装开发工具和源代码。 
jdk.exe /s ADDLOCAL="ToolsFeature,SourceFeature"

设置PATH环境变量

设置PATH该变量对于JDK 11是永久的，因此在重新启动后将保持不变。

如果不设置PATH变量，那么您必须在每次运行可执行文件时指定其完整路径。例如：

C:\> "C:\Program Files\Java\jdk-11\bin\javac" MyClass.java
设置PATH永久变量，添加jdk-11\bin目录到PATH变量。通常，完整路径为：
C:\Program Files\Java\jdk-11\bin
设置PATH在Microsoft Windows上的变量：
  1. 选择控制面板 ，然后选择系统 。
  2. 单击高级 ，然后单击环境变量 。
  3. 添加位置bin JDK安装的文件夹到PATH 系统变量中的变量 。

    注意：

    的PATH环境变量是由分号（;）分隔的一系列目录，不区分大小写。Microsoft Windows在PATH目录，从左到右。

    你应该只有一个bin路径中一次JDK的目录。那些在第一个实例之后的将被忽略。

    如果不确定在哪里添加JDK路径，请附加它。

    设置路径后，新路径将在您打开的每个新命令窗口中生效。 PATH变量。

    以下是PATH变量：

    C:\WINDOWS\system32;C:\WINDOWS;"C:\Program Files\Java\jdk-11\bin"

开始使用JDK

使用Windows“ 开始”菜单中的Java开发工具包来访问与参考文档有关的信息。

在JDK安装过程中，Java菜单项被添加到Windows的“ 开始”菜单中，以方便地访问作为在线文档网页的参考文档。

在JDK安装和卸载过程中，将更新适当的开始菜单项，以便它们与系统上的最新JDK版本相关联

注意：

Windows 7和Windows 10具有“ 开始”菜单。但是，该菜单在Windows 8和Windows 8.1中不可用。Windows 8和Windows 8.1中的JDK和Java信息位于以下“开始”目录中： %ALLUSERSPROFILE%\Microsoft\Windows\Start Menu\Programs 。

在Windows上卸载JDK

要卸载JDK 11 ，请使用Microsoft Windows“ 控制面板”中的“ 添加/删除程序”实用程序。

JDK安装故障排除

以下各节提供了在安装JDK时解决问题的技巧。

减压期间系统错误

如果看到错误消息： system error during decompression ，则包含该磁盘的磁盘上可能没有足够的空间TEMP目录。

程序无法在DOS模式下运行

如果看到错误消息： This program cannot be run in DOS mode ，然后执行以下操作：

  1. 打开MS-DOS外壳程序或命令提示符窗口。

  2. 右键单击标题栏。

  3. 选择属性 。

  4. 选择程序选项卡。

  5. 单击高级 。

  6. 确保未选中“ 阻止基于MS-DOS的程序检测Windows”项 。

  7. 选择确定 。

  8. 再次选择确定 。

  9. 退出MS-DOS外壳程序。

  10. 重启你的电脑。

记事本中的源文件

在Microsoft Windows中，当您在Microsoft记事本中创建一个新文件然后首次保存时，记事本通常会添加.txt文件名的扩展名。因此，您命名的文件Test.java实际上保存为Test.java.txt 。请注意，您看不到.txt扩展名，除非您打开文件扩展名的查看功能（在Microsoft Windows资源管理器中，取消选中“ 文件 夹选项”下的“ 隐藏已知文件类型的文件扩展名” ）。为了防止.txt扩展名，将文件名括在引号中，例如"Test.java"在“ 另存为”对话框中输入信息时。

不属于系统代码页的字符

可以使用不属于系统语言环境代码页的字符来命名目录。如果这样的目录是安装路径的一部分，则会发生一般错误1722，并且安装未完成。错误1722是Windows安装程序的错误代码。表示安装过程失败。目前尚不清楚此错误的确切原因。

为避免此问题，请确保用户和系统区域设置相同，并且安装路径仅包含属于系统区域设置代码页的字符。可以在“ 区域选项”或“ 区域设置”控制面板中设置用户和系统区域设置 。

相关的错误号是4895647。
