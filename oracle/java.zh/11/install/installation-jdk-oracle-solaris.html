在Oracle Solaris上安装JDK的系统要求

Oracle Solaris SPARC（64位）11版或更高版本支持JDK 11 。

有关受支持的处理器和浏览器，请参见《 Oracle JDK认证的系统配置》 。

适用于Oracle Solaris的JDK安装说明符号

安装JDK的说明包含interim.update.patch表示更新版本号的符号。

有关包含符号的JDK安装说明interim.update.patch ，替换为适当的更新版本号。例如，如果要安装JDK 11 ，Interim 0，Update 0和Patch 0，则以下字符串表示捆绑包的名称：

JDK-11.interim.update.patch_solaris-sparcv9_bin.tar.gz

该字符串变为：

JDK-11_solaris-sparcv9_bin.tar.gz

从存档二进制文件在Oracle Solaris上安装JDK

您可以通过存档二进制文件在Oracle Solaris平台上安装JDK 11 （ .tar.gz ）和映像打包系统（IPS），后者是Solaris更新存储库。

以下列表提供有关下载捆绑软件的详细信息：

  • 下载文件：jdk-11.interim.update.patch_solaris-sparcv9_bin.tar.gz

  • 架构：64位SPARC

  • 谁可以安装：任何人

使用这些文件的安装说明由.tar.gz文件类型。此技术使您可以在任何位置为当前用户安装JDK的私有版本，而不会影响其他JDK的安装。但是，它涉及手动步骤以使某些功能正常工作。

注意：

有关建议的方法，请参见从IPS软件包在Oracle Solaris 11上安装JDK 。

从存档二进制文件在Oracle Solaris上安装JDK

您可以将JDK归档二进制文件安装在可以写入的任何位置。它不能替代Oracle Solaris OS提供的Java平台的系统版本。
从存档二进制文件安装JDK（ .tar.gz ）在Oracle Solaris上：
  1. 下载捆绑软件。

    下载文件之前，请确保您接受许可协议。任何人都可以在任何可写入的位置安装存档二进制文件。

    您可以同时解压缩并提取.tar.gz一步来归档文件（也称为tarball）。

  2. 将目录更改为要安装JDK的位置。
  3. 移动.tar.gz将二进制文件归档到当前目录。
  4. 解压缩tarball并将JDK安装在SPARC处理器上：
    $ gzip -dc jdk-11.interim.update.patch_solaris-sparcv9_bin.tar.gz | tar xf -

    JDK安装在名为jdk–11.interim.update.patch在当前目录中。例如，JDK 11 Interim 0，Update 0和Patch 0的目录名为：

    jdk-11

通过IPS软件包在Oracle Solaris 11上安装JDK

您可以使用IPS软件包在Oracle Solaris上安装JDK 11 。

使用以下命令在Oracle Solaris 11上安装JDK 11： jdk-11包：
  1. 确保jdk-11您可以从IPS发布者处获得该软件包。
    例如：

    $ pkg list -a jdk-11

    名称（发布者） 版 IFO
    developer/java/jdk-11 11-0.183.0.0.0.0.0 一世 - -

    • 名称：包装名称。如果安装（或可用）软件包的发布者不是发布者搜索顺序中的第一位，则发布者名称将在软件包名称后的括号中列出。

    • 版本：软件包的发行版本和分支版本。

    • IFO：

      显示软件包状态的一组标志：

      • i “ I”列中的表示已安装软件包。

      • f F列中的表示包装已冻结。

      • o在O列中的表示包装已过时。

      • r O列中的表示包已重命名。

    可从Oracle Solaris发行商处获得此软件包，网址为： pkg.oracle.com以及其他发布商来源。如果看到未找到此类软件包的消息，请使用pkg publisher命令以检查您的发布者来源，并与系统管理员或Oracle支持代表联系。

  2. 确保您具有安装IPS软件包的权限：

    • 使用profiles命令以列出分配给您的权限配置文件。如果您具有软件安装权限配置文件，则可以使用pfexec命令来安装和更新软件包。 

      $ pfexec pkg install jdk-11

      其他权限配置文件也提供安装特权，例如系统管理员权限配置文件。

    • 根据您站点的安全策略，您可能可以使用sudo命令和您的用户密码一起执行特权命令：

      $ sudo pkg install jdk-11

    • 使用Roles命令列出分配给您的角色。如果您具有root角色，则可以使用su使用root密码执行root角色的命令：

      $ pkg install jdk-11

      注意：

      通过SVR4软件包安装JDK 11 （通过pkgadd命令）。使用IPS软件包安装JDK 11 。

指定默认的Java平台

如果在同一映像上安装了多个Java版本，则可以使用以下命令指定默认或首选Java版本Mediator 。一种Mediator包含一组指向应用程序不同实现的链接，其中每个链接具有相同的介体名称和链接路径，但目标链接路径不同。

查看当前的默认版本

使用pkg mediator命令以显示图像中的所有介体和当前的默认Java版本。
$ /usr/bin/pkg mediator | grep java

设置默认Java版本

使用pkg set-mediator命令将Java 11设置为默认或首选版本。
$ usr/bin/pkg set-mediator -V 11 java
