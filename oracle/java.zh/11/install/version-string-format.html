2版本字符串格式

Java SE平台采用了基于时间的发布模型，而JDK每六个月发布一次。

从JDK 10和更高版本开始，版本字符串的格式为Java SE平台基于时间的发行模型。 $FEATURE.$INTERIM.$UPDATE.$PATCH 。

  • $FEATURE是每个功能发行版递增的版本号。该功能版本包含新功能以及对Java SE平台规范所指定的现有功能的更改。版本号每六个月递增一次。例如，2018年3月发行版的版本号是10,2018年9月发行版的版本号是11，依此类推。

  • $INTERIM是每个临时版本递增的版本号，其中包含错误修复和增强功能。临时版本不包含不兼容的更改，功能删除或对标准API的任何更改。临时版本的版本号始终为零（0），因为六个月的发布模型不包含临时版本。但是，此版本号保留给以后的临时版本（如果有）。

  • $UPDATE是针对更新版本增加的版本号，其中包括针对安全性问题，回归和新功能中的错误的修复程序。版本号在$FEATURE发布，此后每三个月发布一次。例如，十月更新版本的完整版本字符串为11 .0.1，一月更新版本的完整版本字符串为11 .0.2，依此类推。

  • $PATCH是为紧急补丁程序发行而解决的关键问题增加的版本号。

版本字符串没有尾随零个元素。例如，如果$FEATURE是11的值$INTERIM为0，则值$UPDATE是1，且值$PATCH为0，则完整版本字符串为11 .0.1。
