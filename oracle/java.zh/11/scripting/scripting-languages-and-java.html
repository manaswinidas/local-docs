1脚本语言和Java

本节描述脚本语言的特性以及Java程序员如何使用它们。

脚本语言是支持编写脚本功能的编程语言。与其他编程语言的源文件必须在运行之前先编译成字节码不同，脚本是由运行时环境（在这种情况下，由脚本引擎）直接评估的。

大多数脚本语言是动态键入的。这使您可以创建新变量而无需声明变量类型（解释器根据与该变量关联的对象的类型来分配类型），并且可以将同一变量重用于不同类型的对象（类型转换是自动执行的） 。

脚本语言通常具有简单的语法。它们允许在相对较少的步骤中执行复杂的任务。

尽管脚本语言通常在运行时进行解释，但是可以将它们编译为Java字节码，然后可以在Java虚拟机（JVM）上执行。对于某些问题，脚本语言可以更快，更容易使用，因此Java应用程序开发人员有时会选择它。但是，如果使用脚本语言编写Java应用程序，则将失去Java语言的优势（例如类型安全性和对类库的访问权限）。

Java规范请求（JSR）223：Java平台的脚本解决了Java和脚本语言集成的问题。它定义了一个标准框架和应用程序编程接口（API），以将脚本嵌入Java应用程序中并从脚本访问Java对象。

通过将脚本嵌入Java代码中，可以自定义和扩展Java应用程序。例如，您可以将配置参数，业务逻辑，数学表达式和其他外部部件编写为脚本。开发Java应用程序时，无需选择脚本语言。如果您使用Java脚本API（由JSR 223定义）编写应用程序，则用户可以使用任何与JSR 223兼容的语言编写脚本。请参阅Java脚本API 。

使用符合JSR 223的语言编写脚本时，您可以访问整个标准Java库。请参阅从脚本使用Java 。
