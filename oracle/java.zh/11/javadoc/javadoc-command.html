的javadoc命令行简介是javadoc [options] [packagenames] [sourcefiles] [@files] 。这些选项可以是Doclet选项，也可以是Standard Doclet选项。的javadoc命令也可以以编程方式运行。

Javadoc Doclets

您使用javadoc工具及其选项，可从Java源文件生成API文档的HTML页面。

Javadoc Doclet选项

的javadoc该命令具有doclet选项。标准Doclet提供了其他选项。

的javadoc该命令使用doclet确定其输出，并使用默认的Standard Doclet，除非使用-doclet选项。虽然选项名称不区分大小写，但它们的参数是区分大小写的。选项在Java平台标准版工具参考的javadoc一章中进行了描述。

处理源文件

的javadoc命令处理文件，在源文件的扩展名和其他文件端中所述源文件 。如果您运行javadoc通过输入单独的源文件名命令，然后您可以确切确定要处理的源文件。但是，这不是大多数开发人员想要的工作方式，因为传递包名称更简单。的javadoc命令可以通过三种方式运行，而无需显式指定源文件名。您可以传递软件包名称，使用-subpackages选项，或使用带有源文件名的通配符。在这些情况下， javadoc仅当文件满足以下所有要求时，该命令才会处理源文件：

  • 文件名前缀（带有。 java已删除）是有效的类名。

  • 分隔符转换为点后，相对于源树的根的路径名是有效的程序包名称。

  • package语句包含有效的软件包名称。

处理链接

在跑步过程中javadoc该命令将交叉引用链接添加到程序包，类和成员名称中，这些文件在该运行中被记录。链接出现在以下位置：

  • 声明（返回类型，参数类型和字段类型）

  • 另请参见@see标签

  • 从生成的内联文本{@link}标签

  • 从产生的例外名称@throws标签

  • 由指向接口成员的链接指定，并覆盖与类成员的链接

  • 摘要表列出了程序包，类和成员

  • 包和类继承树

  • 指标

处理细节

的javadoc该命令每次运行都会产生一个完整的文档。它不会执行增量构建来修改或直接合并早期运行的结果。然而javadoc命令可以链接到其他运行的结果。

的javadoc命令的实现需要并依赖Java编译器。的javadoc命令调用部分javac命令来编译声明并忽略成员实现。的javadoc该命令将构建包含类层次结构的丰富的内部表示形式，并使用关系生成HTML文档。的javadoc该命令还会从源代码中的文档注释中提取用户提供的文档。

的javadoc命令可以在没有方法主体的纯存根文件的源文件上运行。这意味着您可以编写文档注释并运行javadoc在API实施之前的设计早期命令。

依靠编译器可确保HTML输出与实际实现完全对应，后者可能依赖于隐式而非显式的源代码。例如， javadoc命令记录了已编译的类文件中存在但源代码中没有的默认构造函数。

在许多情况下， javadoc命令使您可以使用不完整或错误的代码生成源文件的文档。您可以在完成任何调试和故障排除之前生成文档。的javadoc该命令对文档注释进行原始检查。

当。。。的时候javadoc命令将为文档建立其内部结构，并加载所有引用的类。因此， javadoc命令必须能够找到所有引用的类，以及它们是引导类，扩展还是用户类。

Javadoc Doclets

您可以自定义广告内容和格式javadoc带有doclet的命令输出。的javadoc该命令具有一个默认的内置doclet，称为“标准Doclet”，它会生成HTML格式的API文档。您可以编写自己的doclet来生成HTML，XML，MIF，RTF或所需的任何输出格式。

如果未使用来指定自定义doclet -doclet选项， javadoc命令使用默认的标准Doclet。的javadoc该命令具有几个可用选项，而与使用哪个doclet无关。标准Doclet添加了一组补充的命令行选项。

你用-link您的代码所引用但未在当前文档中记录的类的选项javadoc命令运行。

要链接到有效页面，您必须知道这些HTML页面的位置，并使用extdocURL选项。这允许第三方文档链接到Java。省略-link您想要的选项javadoc该命令仅在当前运行中生成的文档中创建指向API的链接。没有-link选项， javadoc该命令不会创建指向外部参考文档的链接，因为它不知道该文档是否存在或存在于何处。的-link选项可以在生成的文档中的多个位置创建链接。请参阅Javadoc Doclets 。另一个用途是在软件包组之间进行交叉链接：执行javadoc对一组软件包执行命令，然后运行javadoc在另一组软件包上再次命令，在两组软件包之间创建双向链接。

包装清单

的-link选项需要一个名为package-list ，由javadoc命令，位于您使用-link选项。在JDK 8中， package-list文件是一个简单的文本文件，列出了在该位置记录的软件包名称。

什么时候javadoc没有运行-link选项，并且遇到一个名称，该名称属于外部引用的类，它将不带链接地显示该名称。但是，当-link使用选项， javadoc命令搜索package-list该包名称的指定extdocURL位置的文件。找到包名称时，会在名称前加上extdocURL 。

拼写检查标签名称

一些开发人员将自定义标签放在他们并不总是希望将其作为输出的源代码中。在这些情况下，列出源代码中的所有标签，启用要输出的标签和禁用不需要的标签非常重要。存在X禁用标签，而缺少标签则启用标签。这给了javadoc命令足够的信息来知道它遇到的标签是否未知，这很可能是印刷错误或拼写错误的结果。的javadoc在这种情况下，命令将显示警告。你可以加X到已经存在的展示位置值，这样，当您要启用代码时，只需删除X 。例如，如果@todo tag是要在输出中禁止显示的标签，则可以使用： -tag todo:Xcmf:"To Do:" 。如果您希望保持简单，请使用以下命令： -tag todo:X 。语法-tag todo:X即使在@todo标签由标签定义。

标签顺序

的顺序-tag和-taglet选项确定标签的生成顺序。您可以将自定义标签与标准标签混合使用，以散布它们。标准标签的标签选项仅是占位符，用于确定顺序。它们仅采用标准标签的名称。标准标记的子标题不能更改。例如，如果-tag选项丢失，则该位置-taglet选项确定其顺序。如果它们都存在，则以最后出现在命令行中的命令为准。发生这种情况是因为标签和小标签是按照它们在命令行上显示的顺序进行处理的。例如，如果-taglet和-tag选项的名称todo值，然后在命令行上最后出现的那个确定顺序。

生成的文件

您使用javadoc命令作为生成HTML格式文档的标准Doclet。

标准Doclet会生成基本内容，交叉引用和支持页面。每个HTML页面对应一个单独的文件。的javadoc命令生成两种类型的文件。第一种类型以类和接口命名。第二种类型包含连字符（例如package-summary.html ），以防止与第一种文件类型发生冲突。

基本内容页面

  • 一个类或界面页面（ classname.html ），用于记录的每个类或接口。

  • 一包页面（ package-summary.html ）记录的每个包装。的javadoc该命令包括文件中提供的任何HTML文本，其名称为package.html要么package-info.java在里面package源树的目录。

  • 一个概述页面（ overview-summary.html ）用于整个软件包。概述页面是生成的文档的首页。的javadoc该命令包括由以下命令指定的文件中提供的任何HTML文本： -overview选项。仅当您将两个或多个软件包名称传递给javadoc命令。请参阅HTML框架和Javadoc Doclet选项 。

交叉引用页面

  • 整个包集的一个类层次结构页面（ overview-tree.html ）。要查看层次结构页面，请在导航栏中单击概述 ，然后单击树 。

  • 每个包的一个类层次结构页面（ package-tree.html ）。要查看层次结构页面，请转到特定的程序包，类或接口页面，然后单击“ 树”以显示该程序包的层次结构。

  • 每个包装一个使用页面（ package-use.html ），以及每个类和界面的单独使用页面（ class-use/classname.html ）。“使用”页面描述了哪些包，类，方法，构造函数和字段使用指定类，接口或包的任何部分。例如，给定一个类或接口A ，其使用页面包含以下子类A ，声明为的字段A ，返回的方法A ，以及参数类型为的方法和构造函数A 。要查看“使用”页面，请转到包，类或界面，然后单击导航栏中的“ 使用”链接。

  • 已弃用的API页面（ deprecated-list.html ），其中列出了所有不推荐使用的API及其建议的替换。避免使用不推荐使用的API，因为它们可以在将来的实现中删除。

    常量字段值页面（ constant-values.html ）以获取静态字段的值。

  • 序列化的表单页面（ serialized-form.html ）提供有关可序列化和可外部化的类以及字段和方法描述的信息。此页面上的信息是重新实现者感兴趣的，而不是想要使用API的开发人员感兴趣的信息。要访问序列化表单页面，请转到任何序列化的类，然后在类注释的另请参阅部分中单击序列化的表单 。标准Doclet生成一个序列化的表单页面，该页面列出了实现的任何类（公共或非公共） Serializable与它readObject和writeObject方法，序列化的字段以及来自@serial ， @serialField和@serialData标签。上市Serializable可以通过标记它们（或它们的包）来排除类@serial exclude和package-private Serializable可以通过在类（或它们的包）上用@serial include 。您可以运行以下命令来为公共和私有类生成完整的序列化表格javadoc命令而不指定-private选项。请参阅Javadoc Doclet选项 。

  • 索引页（ index-*.html ），按字母顺序排列所有类，接口，构造函数，字段和方法的名称。索引页已针对Unicode进行了国际化，可以针对每个起始字符（例如，英语的A–Z）生成为单个文件或单独的文件。

支持页面

  • 帮助页面（ help-doc.html ），它描述了导航栏和前几页。使用-helpfile用您自己的自定义帮助文件覆盖默认帮助文件。

  • 一index.html用于创建要显示的HTML框架的文件。加载此文件以显示带有框架的首页。的index.html文件不包含文本内容。

  • 几个框架文件（ *-frame.html ），其中包含程序包，类和接口的列表。框架文件显示HTML框架。

  • 一种package-list所使用的文件-link和-linkoffline选项。软件包列表文件是无法通过链接访问的文本文件。

  • 样式表文件（ stylesheet.css ），该控件可控制生成的页面上有限数量的颜色，字体系列，字体大小，字体样式和位置信息。

  • 一种doc-files目录，其中包含要复制到目标目录的图像，示例，源代码或其他文件。这些文件不会由javadoc命令。除非源目录树中存在该目录，否则不会对其进行处理。

请参阅Javadoc Doclet选项 。

生成的API声明

的javadoc命令在该API项目的每个类，接口，字段，构造函数和方法描述的开头生成一个声明。例如，对于Boolean类是：

public final class Boolean
extends Object
implements Serializable

的声明Boolean.valueOf方法是：

public static Boolean valueOf(String s)

的javadoc命令可以包含修饰符public ， protected ， private ， abstract ， final ， static ， transient和volatile ， 但不是synchronized要么native 。的synchronized和native修饰符被视为实现细节，而不是API规范的一部分。

而不是依靠关键字synchronized ，API应该在注释的主要描述中记录其并发语义。例如，描述可能是：
A single enumeration cannot be used by multiple threads concurrently.
该文档不应描述如何实现这些语义。再举一个例子， Hashtable option应该是线程安全的，没有理由指定通过同步其所有导出方法来实现。最好保留内部同步的权限以获得更高的并发性。

简单的例子

以下是运行javadoc整个软件包或单个源文件上的命令。每个软件包名称都有一个对应的目录名称。

Oracle Solaris，Linux和macOS ：在以下示例中，源文件位于/home/src/java/awt/*.java 。目标目录是/home/html 。

Windows ：在以下示例中，源文件位于C:\home\src\java\awt\*java 。目标目录是C:\home\html 。

记录一个或多个软件包 ：要记录一个软件包，该软件包的源文件必须位于与该软件包同名的目录中。

Oracle Solaris，Linux和macOS ：

  • 包裹名称是否具有多个标识符（用点号分隔，例如java.awt.color ），则每个后续标识符都必须对应于更深的子目录（例如java/awt/color ）。

  • 您可以将单个程序包的源文件分成两个位于不同位置的目录树，只要-sourcepath选项指向他们两个。例如， src1/java/awt/color和src2/java/awt/color 。

Windows ：

  • 包裹名称是否具有多个标识符（用点号分隔，例如java.awt.color ），则每个后续标识符都必须对应于更深的子目录（例如java\awt\color ）。

  • 您可以将单个程序包的源文件分成两个位于不同位置的目录树，只要-sourcepath选项指向他们两个。例如， src1\java\awt\color和src2\java\awt\color 。

您可以运行javadoc通过更改目录（使用cd命令）或使用-sourcepath选项。以下示例说明了这两种选择：

示例1从一个或多个包进行递归运行

这个例子使用-sourcepath所以javadoc可以从任何目录运行该命令以进行递归。它遍历Java目录的子包，但不包括植于根目录下的包java.net和java.lang 。请注意，这不包括java.lang.ref ，是java.lang 。要还遍历其他软件包树，请将其名称附加到-subpackages争论，例如java:javax:org.xml.sax 。 

javadoc -d /home/html -sourcepath /home/src -subpackages java -exclude

  1. 转到标准软件包的父目录。

  2. 跑过javadoc命令，其中包含要记录的一个或多个软件包的名称：

示例4在多棵树的显式包上从任何目录运行

跑过javadoc命令并使用-sourcepath选项，并以冒号分隔的形式列出了每棵树的根的路径。提供您要记录的一个或多个软件包的名称。指定软件包的所有源文件都不需要位于单个根目录下，但是必须在源路径的某个位置找到它们。
