2个源文件

的javadoc工具会生成源自以下类型源文件的输出：类的Java语言源文件（ .java ），打包注释文件，概述注释文件和其他未处理的文件。

本主题描述了源文件，测试文件和模板文件，这些文件也可以位于源树中，但必须确保不要对其进行记录。

类源文件

每个类的源文件可以有自己的文档注释。

概述注释文件

您要记录的每个应用程序或软件包集都可以有自己的概述文档注释，该注释保存在自己的源文件中， javadoc然后，该工具合并到生成的概述页面中。您通常会在此注释中包括适用于整个应用程序或软件包集的任何文档。

您可以将文件命名为任意名称，例如overview.html放在任何地方典型位置在源代码树的顶部。

Oracle Solaris，Linux和macOS ：例如，如果java.math软件包包含在/home/user/src/java/math目录，那么您可以在中创建概述注释文件/home/user/src/overview.html 。

Windows ：例如，如果java.math软件包包含在C:\user\src\java\math目录，那么您可以在中创建概述注释文件C:\user\src\overview.html 。

如果您想运行同一组源文件，则可以有多个概述注释文件。 javadoc在不同的包装上多次使用该工具。例如，您可以运行javadoc工具一次-private内部文档的选项，对于公共文档则没有该选项。在这种情况下，您可以在每个概述注释文件的第一句话中将文档描述为公共文档或内部文档。

概述注释文件的内容是用HTML编写的一大文档注释。使第一句话成为有关应用程序或软件包集的摘要。请勿在标题之间插入标题或其他任何文字标签和第一句话。除内联标签外的所有标签，例如{@link}标记，必须出现在主要说明之后。如果您添加@see标记，则它必须具有完全限定的名称。

当您运行javadoc工具，使用-overview选项。然后，该文件的处理与软件包注释文件的处理类似。的javadoc该工具执行以下操作：

  • 在之间复制所有内容和标签进行处理。

  • 处理存在的概述标签。

  • 将已处理的文本插入到生成的概述页面的底部。

  • 将概述注释的第一句复制到概述摘要页面的顶部。

未处理的文件

您的源文件可以包括您想要的任何文件。 javadoc工具复制到目标目录。这些文件通常包括图形文件，示例Java源代码和类文件，以及带有大量内容的独立HTML文件，这些内容会使典型的Java源文件的文档注释不堪重负。

要包括未处理的文件，请将它们放在一个目录中doc-files 。的doc-files directory可以是任何包含源文件的软件包目录的子目录。你可以有一个doc-files每个包的子目录。

Oracle Solaris，Linux和macOS ：例如，如果要在java.awt.Button类文档，然后将图像文件放在/home/user/src/java/awt/doc-files/目录。请勿放置doc-files目录位于/home/user/src/java/doc-files ，因为java不是包装。它不包含任何源文件。

Windows ：例如，如果您要在按钮中包含按钮的图像java.awt.Button类文档，然后将图像文件放在\src\java\awt\doc-files目录。请勿放置doc-files目录位于\src\java\doc-files ，因为java不是包。它不包含任何源文件。

所有未处理文件的链接都必须包含在代码中，因为javadoc工具不会查看文件。的javadoc该工具会将目录及其所有内容复制到目标位置。以下示例显示了链接中的Button.java文档注释可能如下所示：

/**
 * <p> This button looks like this:</p> 
 * <img src="doc-files/Button.gif"/>
 */
测试和模板文件

您可以将测试文件和模板文件存储在与源文件所在目录相同的目录中或位于其子目录中的源树中。为防止测试文件和模板文件被处理，请运行javadoc工具并显式传递单个源文件名。

测试文件是有效的，可编译的源文件。模板文件不是有效的兼容源文件，但它们通常具有.java后缀。

  • 测试文件 ：如果您希望测试文件属于未命名的软件包或属于源文件所在的软件包以外的其他软件包，则将测试文件放在源文件下面的子目录中，并为该目录指定无效的名称。如果您将测试文件与源文件放在同一目录中，然后调用javadoc命令行参数指示其程序包名称的工具，然后测试文件会导致警告或错误。如果文件位于名称无效的子目录中，那么将跳过测试文件目录，并且不会发出错误或警告。例如，要为以下文件中的源文件添加测试文件com.package1 ，将它们放在无效软件包名称的子目录中。以下目录名称无效，因为它包含连字符：

    • Oracle Solaris，Linux和macOS ：com/package1/test-files/

    • Windows ：com\package1\test-files\

    如果您的测试文件包含文档注释，则可以单独运行javadoc通过使用通配符传递测试源文件名来生成测试文件文档的工具，例如com/package1/test-files/*.java 。

  • 模板文件 ：如果您希望模板文件位于源目录中，但在执行javadoc工具，然后给它一个无效的文件名，例如Buffer-Template.java以防止其被处理。的javadoc当工具删除了带有名称的源文件时， .java后缀，即有效的类名。
