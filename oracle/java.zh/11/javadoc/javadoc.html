1个 Javadoc

的javadoc工具是一个程序，可以将源文件的集合读取为内部形式。

结构为： (source-files)->[javadoc-tool:doclet]->(generated files) 。

Javadoc doclet就像一个可插拔的后端，它会出于某种目标而分析这种内部形式。根据doclet，它可以生成HTML，MIF或XML文件。

生成的文件的内容特定于doclet。标准doclet生成HTML文档，但是例如，其他doclet可以生成拼写错误的单词或语法错误的报告。

如果您指定标准doclet以外的其他doclet，则本指南中的描述可能不适用于该doclet或生成的文件（如果有）的操作。

要使用javadoc工具，您必须：

  • 编写包含文档注释的源代码。文档注释以HTML格式设置，并散布在javadoc工具。

  • 跑过javadoc工具。您需要指定一个doclet来分析文档注释和任何其他特殊标记。但是，如果您未指定任何doclet，则默认情况下会包括标准Doclet 。您指定了许多命令行选项，其中一些直接针对javadoc工具本身，其中一些特定于所选doclet。命令行帮助显示并区分适用于当前所选doclet的工具选项。使用标准doclet时，标准doclet生成的输出包含一系列HTML页面。如果指定其他doclet，则该doclet的操作以及生成的文件（如果有）可能与本指南中描述的标准doclet相同或不相同。

Javadoc功能

Javadoc功能包括：Javadoc搜索，对生成HTML5输出的支持，对模块系统中文档注释的支持以及简化的Doclet API。

搜索

的javadoc工具运行可能会生成输出的doclet。标准doclet生成的输出可让您搜索生成的文档。搜索框可用于生成的API，并提供以下内容：

  • 您可以搜索API中定义的元素和其他关键短语

  • 结果，包括与输入字符完全匹配的结果，然后是包含字符串中任意位置的输入字符的结果。多个结果显示为搜索框下方的简单滚动列表。为了方便分类和适当地选择用户，将结果分类如下：

    • 模组

    • 配套

    • 种类

    • 会员

    • 搜索标签

    如果搜索词或短语是使用继承的，则会显示具有不同程序元素名称的多个结果。 @inheritDoc标签。

  • 基于用户选择的页面重定向。

您可以搜索以下内容：

  • 声明的模块，包，类型和成员的名称 ：因为方法可以重载，所以方法参数类型的简单名称也会被索引并可以搜索。方法参数名称无法建立索引。

  • 使用新的内联标签索引的搜索字词或词组， @index ：其他内联标签不能嵌套在其中@index 。您只能搜索标有的词组或搜索词@index在声明的javadoc评论。例如，特定领域的术语ulps在整个java.lang.Math类，但不会出现在任何类或方法声明名称中。为了帮助Math API的用户，API设计者可以标记各种出现的ulps在班级javadoc注释或方法级别javadoc评论。标记使用{@index ulps} 。术语ulps由索引javadoc工具。

模块系统

的javadoc该工具支持模块声明中的文档注释。一些Javadoc命令行选项使您可以指定要记录的模块集，并为要记录的所有模块生成新的摘要页面。它具有新的命令行选项来配置要记录的模块集，并为要记录的所有模块生成新的摘要页面。请参阅《 Java平台，标准版工具参考》中的javadoc章节。

HTML 5支持

您可以生成HTML5输出。要获得完全兼容的HTML5输出，请确保文档注释中提供的任何HTML内容均与HTML5兼容。

简化的Doclet API

Doclet API使用功能强大的API，可以更好地表示所有语言功能。请参阅Javadoc标准Doclet 。
