卡塔布

您使用ktab用于管理存储在本地密钥表中的主体名称和服务密钥的工具。

概要

ktab commands options
commands options

列出密钥表名称和条目，向密钥表添加新的密钥项，删除现有的密钥项并显示说明。请参阅命令和选项 。

描述

的ktab使用户能够管理存储在本地密钥表中的主体名称和服务密钥。密钥表中列出的主体和密钥对使主机上运行的服务可以向密钥分发中心（KDC）进行身份验证。

在配置服务器以使用Kerberos之前，必须在运行服务器的主机上设置密钥表。请注意，使用ktab工具不会影响Kerberos数据库。

密钥表是主机自己的密钥表的副本，类似于用户的密码。需要向密钥分发中心（KDC）进行身份验证的应用程序服务器必须具有包含其自己的主体和密钥的密钥表。如果您更改了密钥表中的密钥，则还必须对Kerberos数据库进行相应的更改。的ktab该工具使您可以列出，添加，更新或删除密钥表中的主体名称和密钥对。这些操作均不影响Kerberos数据库。

安全警报

不要在命令行上指定密码。这样做可能会带来安全风险。例如，攻击者可能在运行UNIX时发现您的密码ps命令。

就像保护用户密码很重要一样，保护主机的密钥表也同样重要。您应该始终将keytab文件存储在本地磁盘上，并使它们只能由root用户读取。绝对不要通过网络发送密钥表文件。

命令和选项

-l [-e] [-t]

列出密钥表名称和条目。什么时候-e指定后，将显示每个条目的加密类型。什么时候-t指定后，将显示每个条目的时间戳。

-a principal_name [password] [-n kvno] [-append]

使用给定的主体名称将新的密钥条目添加到给定主体名称的密钥表中password 。如果一个kvno指定后，新密钥的密钥版本号等于该值，否则，将自动递增密钥版本号。如果-append如果指定，则将新密钥附加到密钥表，否则，将删除相同主体的旧密钥。

无需对Kerberos数据库进行任何更改。不要在命令行或脚本中指定密码。 如果未指定密码，此工具将提示您输入密码。

-d principal_name [—f] [-e etype] [kvno | all| old]

从密钥表中删除指定主体的密钥条目。无需对Kerberos数据库进行任何更改。

  • 如果kvno指定后，该工具将删除其密钥版本号与kvno匹配的密钥。如果all指定后，删除所有键。

  • 如果old指定后，该工具会删除除键值最高的所有键之外的所有键kvno 。默认动作是all 。

  • 如果etype如果指定，则该工具仅删除此加密类型的密钥。 etype应指定为数值etype在RFC 3961第8节中定义。除非出现提示，否则显示确认删除的提示-f已指定。

什么时候etype如果提供，则仅删除与此加密类型匹配的条目。否则，将删除所有条目。

-help

显示说明。

常用选项

此选项可与-l ， -a要么-d命令。

-k keytab name

指定密钥表名称和路径FILE:字首。

例子

列出默认密钥表中的所有条目
ktab -l
向密钥表添加新的主体（请注意，系统将提示您输入密码）
ktab -a duke@example.com
从密钥表中删除主体
ktab -d duke@example.com
