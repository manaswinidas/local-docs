统计

您使用jstat命令以监视JVM统计信息。此命令是实验性的，不受支持。

概要

jstat generalOptions
jstat -outputOptions [ -t] [-hlines] vmid [interval [count] ]
generalOptions

单个常规命令行选项。请参阅常规选项 。

outputOptions

期权报告的-options选项。一个或多个输出选项，其中包含一个statOption ，再加上任何-t ， -h和-J选项。请参见jstat命令的输出选项 。

-t

将时间戳列显示为输出的第一列。时间戳是自目标JVM启动时间以来的时间。

-h n

每显示一个列标题n样本（输出行），其中n是一个正整数。默认值为0 ，显示第一行数据的列标题。

vmid

虚拟机标识符，它是指示目标JVM的字符串。请参阅虚拟机标识符 。

interval

采样间隔，单位为秒（s）或毫秒（ms）。默认单位是毫秒。这必须是一个正整数。指定时， jstat命令在每个时间间隔产生其输出。

count

要显示的样本数。默认值为无穷大，这会导致jstat命令显示统计信息，直到目标JVM终止或jstat命令终止。该值必须是一个正整数。

描述

的jstat该命令显示已检测的Java HotSpot VM的性能统计信息。目标JVM由其虚拟机标识符标识，或者vmid选项。

的jstat命令支持两种类型的选项，常规选项和输出选项。一般选项会导致jstat命令以显示简单的用法和版本信息。输出选项确定统计输出的内容和格式。

所有选项及其功能都可能在将来的版本中更改或删除。

常规选项

如果指定常规选项之一，则不能指定任何其他选项或参数。

-help

显示帮助信息。

-options

显示静态选项列表。请参见jstat命令的输出选项 。

jstat命令的输出选项

如果不指定常规选项，则可以指定输出选项。输出选项确定内容的内容和格式jstat命令的输出，并且由单个statOption ，以及其他任何输出选项（ -h ， -t和-J ）。的statOption必须先来。

输出格式为表格，列之间用空格隔开。带标题的标题行描述了这些列。使用-h选项设置显示标题的频率。列标题名称在不同选项之间是一致的。通常，如果两个选项提供的列名称相同，则两列的数据源是相同的。

使用-t选项以显示一个时间戳列，将时间戳记为输出的第一列。“时间戳记”列包含自目标JVM启动以来经过的时间（以秒为单位）。时间戳的分辨率取决于各种因素，并且由于在重负载系统上的线程调度延迟而会发生变化。

使用interval和count参数来分别确定频率和次数jstat命令显示其输出。

注意：

不要编写脚本来解析jstat命令的输出，因为格式可能在将来的版本中更改。如果您编写用于解析jstat命令输出，然后期望针对此工具的将来版本对其进行修改。

-statOption

确定统计信息jstat命令显示。以下列出了可用的选项。使用-options常规选项以显示特定平台安装的选项列表。请参阅统计信息选项和输出 。

class ：显示有关类加载器行为的统计信息。

compiler ：显示有关Java HotSpot VM即时编译器行为的统计信息。

gc ：显示有关垃圾收集堆行为的统计信息。

gccapacity ：显示有关世代容量及其相应空间的统计信息。

gccause ：显示有关垃圾收集统计信息的摘要（与-gcutil ），以及最近（当前）垃圾收集事件的原因。

gcnew ：显示有关新一代行为的统计信息。

gcnewcapacity ：显示有关新一代大小及其相应空间的统计信息。

gcold ：显示有关旧时代行为的统计信息和元空间统计信息。

gcoldcapacity ：显示有关上一代的大小的统计信息。

gcmetacapacity ：显示有关元空间大小的统计信息。

-JjavaOption

通行证javaOption Java应用程序启动器。例如， -J-Xms48m将启动内存设置为48 MB。有关选项的完整列表，请参见java 。

虚拟机标识符

的语法vmid字符串对应于URI的语法：

[protocol:][//]lvmid[@hostname[:port]/servername]

的语法vmid字符串对应于URI的语法。的vmid字符串可以从代表本地JVM的简单整数到指定通信协议，端口号和其他特定于实现的值的更复杂的构造。

protocol

通信协议。如果protocol省略value且未指定主机名，则默认协议是特定于平台的优化本地协议。如果protocol省略value并指定主机名，则默认协议为rmi 。

lvmid

目标JVM的本地虚拟机标识符。的lvmid是特定于平台的值，用于唯一标识系统上的JVM。的lvmid是虚拟机标识符的唯一必需组件。的lvmid通常是但不一定是目标JVM进程的操作系统进程标识符。您可以使用jps命令确定lvmid前提是JVM进程未在单独的docker实例中运行。您还可以确定lvmid在Oracle Solaris，Linux和macOS平台上使用ps命令，并在Windows上使用Windows Task Manager。

注意：

当附加到在单独的docker进程中运行的Java进程时，JDK 10添加了对使用Attach API的支持。然而jps该命令将不会列出在单独的docker实例中运行的JVM进程。如果您尝试将Linux主机与Docker容器中的虚拟机连接，则必须使用以下工具ps查找JVM的PID。
hostname

指示目标主机的主机名或IP地址。如果hostname如果省略value，则目标主机为本地主机。

port

与远程服务器通信的默认端口。如果hostname值被忽略或protocol值指定一个优化的本地协议，然后port值将被忽略。否则，治疗port参数是特定于实现的。对于默认rmi协议，端口值表示端口号rmiregistry在远程主机上。如果port值被省略，并且protocol值指示rmi ，则使用默认的rmiregistry端口（1099）。

servername

的治疗servername参数取决于实现。对于优化的本地协议，将忽略此字段。为了rmi协议，它表示远程主机上RMI远程对象的名称。
