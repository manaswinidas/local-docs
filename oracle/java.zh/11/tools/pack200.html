包装200

您使用pack200命令，使用Java gzip压缩程序将Java存档（JAR）文件转换为压缩的pack200文件。

概要

注意：

pack200和unpack200已被弃用，并可能在将来的JDK版本中删除。

pack200 [-opt... | --option=value] x.pack[.gz] JAR-file
-opt... | --option=value

选项可以是任何顺序。命令行或属性文件中的最后一个选项将取代所有先前指定的选项。请参阅pack200命令的选项 。

x.pack[.gz]

输出文件的名称。

file.jar

输入文件的名称。

描述

的pack200 command是将JAR文件转换为压缩文件的Java应用程序pack200 Java gzip压缩程序压缩文件。此命令将JAR文件打包到压缩文件中pack200用于Web部署的文件。的pack200文件是高度压缩的文件，可以直接部署以节省带宽并减少下载时间。

以下示例显示了典型用法，其中myarchive.pack.gz产生与默认pack200命令设置：

pack200 myarchive.pack.gz myarchive.jar

注意：

此命令不应与以下命令混淆pack 。的pack和pack200命令是单独的产品。如果有差异，则JDK随附的Java SE API规范是取代权限。

退出状态

返回以下退出值：0表示成功完成，而数字则表示发生错误时大于0。

pack200命令的选项

的pack200命令有几个选项可以微调和设置压缩引擎。以下示例显示了典型用法，其中myarchive.pack.gz产生与默认pack200命令设置：

pack200 myarchive.pack.gz myarchive.jar
-r要么--repack

通过打包和解压缩JAR文件来生成JAR文件。生成的文件可用作输入jarsigner工具。以下示例打包和解压缩myarchive.jar文件：

pack200 --repack myarchive-packer.jar myarchive.jar
pack200 --repack myarchive.jar
-g要么--no-gzip

产生一个pack200文件。使用此选项时，必须使用合适的压缩器，并且目标系统必须使用相应的解压缩器。 

pack200 --no-gzip myarchive.pack myarchive.jar
--gzip

（默认）使用以下命令对包输出进行后压缩gzip 。

-G要么--strip-debug

从输出中剥离调试属性。这些包括SourceFile ， LineNumberTable ， LocalVariableTable和LocalVariableTypeTable 。删除这些属性会减少下载和安装的大小，还会降低调试器的实用性。

--keep-file-order

保留输入文件中文件的顺序。这是默认行为。

-O要么--no-keep-file-order

重新排序并传输所有元素。加壳程序还可以删除JAR目录名以减小下载大小。但是，某些JAR文件优化（例如索引编制）可能无法正常工作。

-SN要么--segment-limit=N

该值是估计的目标大小N每个归档段的字节数（以字节为单位）。如果单个输入文件要求超过N个字节，然后提供其自己的存档段。在特殊情况下，值为-1生成包含所有输入文件的单个大段，而值为0则为每个类生成一个段。较大的存档段可减少碎片并提高压缩率，但是处理它们需要更多的内存。

通过计算段中要传输的每个输入文件的大小及其名称和其他传输属性的大小，可以估算每个段的大小。

默认是-1 ，这意味着打包程序将创建一个单段输出文件。在生成非常大的输出文件的情况下，强烈建议用户使用分段或将输入文件分解为较小的JAR文件。

没有此限制的10 MB JAR压缩包通常会缩小约10％，但打包程序可能需要更大的Java堆（约为段限制的10倍）。

-Evalue要么--effort=value

如果该值设置为一个十进制数字，则打包程序将使用指定的工作量来压缩存档。级别1可能会产生更大的尺寸和更快的压缩速度，而级别9会花费更长的时间，但可以产生更好的压缩效果。特殊值0指示pack200命令直接在没有压缩的情况下直接复制原始JAR文件。JSR 200标准要求任何拆包程序都将这种特殊情况理解为整个存档的传递。

默认值为5，以花费适度的时间来产生合理的压缩。

-Hvalue要么--deflate-hint=value

覆盖默认值，该默认值保留输入信息，但可能导致传输的档案更大。可能的值为： true ， false ， 要么keep 。

如果value是true或为假，则packer200命令在输出归档文件中相应地设置放气提示，并且不传输归档元素的各个放气提示。

的keep值保留在输入JAR中观察到的放气提示。这是默认值。

-mvalue要么--modification-time=value

可能的值为latest和keep 。

如果值为latest ，然后打包程序将尝试确定原始归档文件中所有可用条目中的最新修改时间，或该段中所有可用条目的最新修改时间。该单个值作为段的一部分发送，并应用于每个段中的所有条目。这样可以稍微减少归档文件的传输大小，但要以将所有已安装文件设置为单个日期为代价。

如果值为keep ，则保留在输入JAR中观察到的修改时间。这是默认值。

-P file要么--pass-file=file

指示文件应不压缩地按字节传递。通过重复该选项，可以指定多个文件。没有路径名转换，除了将系统文件分隔符替换为JAR文件分隔符正斜杠（/）外。结果文件名必须与在JAR文件中出现的字符串完全匹配。如果file是目录名，则传递该目录下的所有文件。

-U action要么--unknown-attribute=action

覆盖默认行为，这意味着包含未知属性的类文件将使用指定的属性传递action 。动作的可能值为error ， strip ， 要么pass 。

如果值为error ，然后整个pack200命令操作失败并给出适当的解释。

如果值为strip ，然后删除该属性。删除必需的Java虚拟机（JVM）属性可能会导致类加载器失败。

如果值为pass ，则整个类就像资源一样被传输。

-C attribute-name=layout要么--class-attribute=attribute-name=action

（用户定义的属性），请参阅说明—Dattribute-name=layout 。

-Fattribute-name=layout要么--field-attribute=attribute-name=action

（用户定义的属性），请参阅说明—Dattribute-name=layout

-Mattribute-name=layout 要么--method-attribute=attribute-name=action

（用户定义的属性），请参阅说明—Dattribute-name=layout

—Dattribute-name=layout要么--code-attribute=attribute-name=action

（用户定义的属性）可以为类实体指定属性布局，例如class-attribute ， field-attribute ， method-attribute和code-attribute 。的attribute-name是为其定义布局或操作的属性的名称。的可能值action是some-layout-string ， error ， strip ， pass 。

some-layout-string ：布局语言是在JSR 200规范中定义的，例如： --class-attribute=SourceFile=RUH 。

如果值为error ，然后pack200操作失败，并给出解释。

如果值为strip ，然后从输出中删除该属性。删除JVM必需的属性可能会导致类加载器失败。例如， --class-attribute=CompilationID=pass使包含此属性的类文件直接传递，而打包程序无需进一步操作。

如果值为pass ，则整个类就像资源一样被传输。

-fpack.properties要么--config-file=pack.properties

表示可以在命令行上指定一个配置文件，其中包含用于初始化打包程序的Java属性。

pack200 -f pack.properties myarchive.pack.gz myarchive.jar
more pack.properties
# Generic properties for the packer.
modification.time=latest
deflate.hint=false
keep.file.order=false
# This option will cause the files bearing new attributes to
# be reported as an error rather than passed uncompressed.
unknown.attribute=error
# Change the segment limit to be unlimited.
segment.limit=-1
-v要么--verbose

输出最少的消息。对该选项的多种说明将创建更多详细消息。

-q要么--quiet

指定无消息的安静操作。

-lfilename要么--log-file=filename

指定用于输出消息的日志文件。

-? ， -h ， 要么--help

打印有关此命令的帮助信息。

-V要么--version

打印有关此命令的版本信息。

-Joption

将指定的选项传递给Java虚拟机。例如， -J-Xms48m将启动内存设置为48 MB。
