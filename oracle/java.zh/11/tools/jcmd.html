jcmd

您使用jcmd实用程序将诊断命令请求发送到正在运行的Java虚拟机（JVM）。

概要

jcmd [pid | main-class] command...|PerfCounter.print| 
-f filename
jcmd -l
jcmd -h

注意：

Java Flight Recorder（JFR）与jcmd该实用程序是一种商业产品，必须在使用前启用。JVM运行后， jcmd命令VM.unlock_commercial_features用于解锁商业功能和能够使用所描述的JFR命令的命令- jcmd 。

pid

使用时， jcmd实用程序将诊断命令请求发送到Java进程的进程ID。

main-class

使用时， jcmd实用程序使用指定的主类名称将诊断命令请求发送到所有Java进程。

command

的command必须是有效的jcmd所选JVM的命令。可用命令列表jcmd通过运行help命令（ jcmd pid help ）哪里pid是正在运行的Java进程的进程ID。如果pid是0 ，命令将发送到所有Java进程。main class参数将用于部分或完全匹配用于启动Java的类。如果未提供任何选项，它将列出正在运行的Java进程标识符，以及用于启动进程的主类和命令行参数（与使用-l ）。

Perfcounter.print

打印指定的Java进程公开的性能计数器。

—f filename

从指定的文件读取并执行命令， filename 。

-l

显示不在单独的docker进程中运行的Java虚拟机进程标识符的列表，以及用于启动该进程的主类和命令行参数。如果JVM在docker进程中，则必须使用以下工具ps查找PID。

注意：

使用jcmd没有参数与使用相同jcmd -l 。

-h

显示jcmd实用程序的命令行帮助。

描述

的jcmd实用程序用于将诊断命令请求发送到JVM。它必须在运行JVM的同一台计算机上使用，并且必须具有用于启动JVM的相同有效用户和组标识符。每个诊断命令都有其自己的参数集。要显示诊断命令的描述，语法和可用参数列表，请使用命令名称作为参数。例如

jcmd pid help command

如果参数包含空格，则必须用单引号或双引号引起来（ '要么" ）。此外，您必须在单引号或双引号之间加上反斜杠（ \ ），以防止操作系统外壳处理引号。或者，可以用单引号然后是双引号（或用双引号然后是单引号）将这些参数引起来。

如果您指定进程标识符（ pid ）或主要班级（ main-class ）作为第一个参数，然后是jcmd实用程序将诊断命令请求发送到具有指定标识符的Java进程，或者发送给具有指定主类名称的所有Java进程。您还可以通过以下方式将诊断命令请求发送到所有可用的Java进程： 0作为进程标识符。
