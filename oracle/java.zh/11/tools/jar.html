罐

您可以使用jar命令来为类和资源创建档案，并从档案中操纵或还原单个类或资源。

概要

 jar [OPTION...] [ [--release VERSION] [-C dir] files] ...

描述

的jar command是基于ZIP和ZLIB压缩格式的通用归档和压缩工具。最初， jar该命令旨在打包Java applet（自JDK 11开始不支持）或应用程序；但是，从JDK 9开始，用户可以使用jar命令来创建模块化JAR。对于运输和部署，通常将模块打包为模块化JAR更为方便。

的语法jar该命令类似于tar命令。它具有几种主要操作模式，由强制操作参数之一定义。其他参数是修改操作行为的选项，或者是执行操作所必需的。

将应用程序的模块或组件（文件，图像和声音）组合到单个存档中时，可以由Java代理（例如浏览器）在单个HTTP事务中下载它们，而无需为每个组件都建立新的连接片。这大大缩短了下载时间。的jar该命令还压缩文件，从而进一步缩短了下载时间。的jar命令还允许对文件中的各个条目进行签名，以便可以验证其来源。无论是否压缩，JAR文件都可以用作类路径条目。

当您包含模块描述符时，归档文件将变为模块化JAR， module-info.class ，位于给定目录的根目录或JAR归档文件的根目录中。仅在创建和更新模式下有效的操作修饰符中描述的以下操作仅在创建或更新模块化JAR或更新现有的非模块化JAR时有效：

  • --module-version

  • --hash-modules

  • --module-path

注意：

长选项的所有必需或可选参数对于任何对应的短选项也是必需或可选的。

主要操作模式

使用时jar命令，必须指定要执行的操作。您可以为jar通过包括本节中描述的适当操作参数来命令。您可以将运算参数与其他一个字母的选项混合使用。通常，操作参数是在命令行上指定的第一个参数。

—c要么--create

创建档案。

—i=FILE要么--generate-index=FILE

为指定的JAR文件生成索引信息。

—t要么--list

列出存档的目录。

—u要么--update

更新现有的JAR文件。

—x要么--extract

从存档中提取命名的（或所有）文件。

-d要么--describe-module

打印模块描述符或自动模块名称。

在任何模式下均有效的操作修饰符

您可以使用以下选项来自定义自定义工具中包含的任何操作模式的操作jar命令。

-C DIR

更改指定的目录并包括files在命令行末尾指定。 

 jar [OPTION...] [ [--release VERSION] [-C dir] files]
-f=FILE要么--file=FILE

指定存档文件名。

--release VERSION

创建一个多发行版JAR文件。将选项后指定的所有文件放入名为JAR文件的版本目录中META-INF/versions/VERSION/ ，其中VERSION必须为必须为9或更大的正整数。

在运行时，如果JAR中存在一个以上版本的类，则JDK将使用找到的第一个版本，最初在目录树中搜索VERSION数字与JDK的主版本号匹配。然后，它将依次查找目录中较低的目录VERSION数字，最后看一下JAR的根。

-v要么--verbose

将详细输出发送或打印到标准输出。

操作修饰符仅在创建和更新模式下有效

您可以使用以下选项来自定义创建和更新主要操作模式的操作：

—e=CLASSNAME要么--main-class=CLASSNAME

指定捆绑到模块化或可执行模块化JAR文件中的独立应用程序的应用程序入口点。

-m=FILE要么--manifest=FILE

包括来自给定清单文件的清单信息。

-M要么--no-manifest

不为条目创建清单文件。

--module-version=VERSION

在创建或更新模块化JAR文件或更新非模块化JAR文件时，指定模块版本。

--hash-modules=PATTERN

计算并记录与给定模式匹配的模块的哈希，这些哈希直接或间接取决于要创建的模块化JAR文件或要更新的非模块化JAR文件。

-p path要么--module-path path

指定用于生成哈希的模块相关性的位置。

@file

读jar选项和文本文件中的文件名。

选项文件是一个文本文件，其中包含您通常会在命令提示符下输入的选项和值。选项可以显示在一行或几行上。您不能为路径名指定环境变量。您可以通过在行的开头加上井号（＃）来注释掉行。

操作修饰符仅在创建，更新和生成索引模式下有效

您可以使用以下选项来自定义创建操作（ —c要么--create ） 更新 （ —u要么--update ）和generate-index（ —i要么--generate-index=FILE ）主要操作模式：

-0要么--no-compress

无需使用ZIP压缩即可存储。

其他选择

以下选项可被jar命令，不与操作模式一起使用：

-h要么--help[:compat]

显示命令行帮助jar命令或兼容性帮助。

--help-extra

显示有关其他选项的帮助。

--version

打印程序版本。

jar命令语法的示例

创建档案， classes.jar ，其中包含两个类文件， Foo.class和Bar.class 。 
 jar --create --file classes.jar Foo.class Bar.class
创建档案， classes.jar ，通过使用现有清单， mymanifest ，其中包含目录中的所有文件foo/ 。 
 jar --create --file classes.jar --manifest mymanifest -C foo/
创建一个模块化JAR存档， foo.jar ，模块描述符位于其中classes/module-info.class 。 
 jar --create --file foo.jar --main-class com.foo.Main --module-version 1.0 -C foo/classes resources
更新现有的非模块化JAR， foo.jar到模块化JAR文件。 
 jar --update --file foo.jar --main-class com.foo.Main --module-version 1.0 -C foo/module-info.class
创建版本或多版本的JAR， foo.jar ，将文件放在classes JAR根目录中的目录，以及classes—10目录中META-INF/versions/10 JAR的目录。

在此示例中， classes/com/foo目录包含两个类， com.foo.Hello （入口点类）和com.foo.NameProvider ，都是针对JDK 8编译的。的classes-10/com/foo目录包含其他版本的com.foo.NameProvider类，该类包含针对JDK 10的特定代码并针对JDK 10进行了编译。

通过此设置，创建一个多发行版JAR文件foo.jar通过从包含目录的目录中运行以下命令classes和classes-10 。 

 jar --create --file foo.jar --main-class com.foo.Hello -C classes . --release 10 -C classes-10 .

JAR文件foo.jar现在包含：

% jar -tf foo.jar

META-INF/ 
META-INF/MANIFEST.MF 
com/ 
com/foo/ 
com/foo/Hello.class 
com/foo/NameProvider.class 
META-INF/versions/10/com/ 
META-INF/versions/10/com/foo/ 
META-INF/versions/10/com/foo/NameProvider.class

以及其他信息，文件META-INF/MANIFEST.MF ，将包含以下几行内容，以表明这是入口为的多版本JAR文件com.foo.Hello 。 

...
Main-Class: com.foo.Hello
Multi-Release: true

假设com.foo.Hello类在上调用方法com.foo.NameProvider类，使用JDK 10运行程序将确保com.foo.NameProvider班级是其中之一META-INF/versions/10/com/foo/ 。使用JDK 8运行程序将确保com.foo.NameProvider class是JAR的根源之一com/foo 。

创建档案， my.jar ，通过从文件中读取选项和类文件列表classes.list 。

注意：

缩短或简化jar命令，您可以在单独的文本文件中指定参数，然后将其传递给jar使用at符号（ @ ）作为前缀。 

 jar --create --file my.jar @classes.list
