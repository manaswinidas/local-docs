jdeps

您使用jdeps命令以启动Java类依赖关系分析器。

概要

jdeps [options] path ...
options

命令行选项。有关可以使用的选项的详细说明，请参见

  • 可能的选择

  • 模块依赖性分析选项

  • 筛选依赖项的选项

  • 筛选要分析的类别的选项

path

的路径名.class文件，目录或JAR文件进行分析。

描述

的jdeps命令显示Java类文件的包级或类级依赖关系。输入类可以是指向.class文件，目录，JAR文件，也可以是分析所有类文件的标准类名。这些选项确定输出。默认情况下， jdeps命令将相关性写入系统输出。该命令可以使用DOT语言生成依赖关系（请参见-dotoutput选项）。

可能的选择

-dotoutput dir要么--dot-output dir

指定DOT文件输出的目标目录。如果指定了此选项，则jdeps命令生成一个.dot每个分析的档案的文件名为archive-file-name.dot列出了依赖项，还有一个名为summary.dot列出了存档文件之间的依赖关系。

-s要么-summary

仅打印依赖项摘要。

-v要么-verbose

打印所有类级别的依赖项。这相当于

 -verbose:class -filter:none
-verbose:package

打印软件包级别的依赖关系，默认情况下，排除同一软件包内的依赖关系。

-verbose:class

打印类级别的依赖关系，默认情况下，不包含同一归档文件中的依赖关系。

-apionly要么--api-only

将分析限制为API，例如，对签名的依赖public和protected公共类的成员，包括字段类型，方法参数类型，返回类型和检查的异常类型。

-jdkinternals要么--jdk-internals

在JDK内部API中查找类级别的依赖关系。默认情况下，此选项分析在--classpath选项和输入文件，除非您指定了-include选项。您不能将此选项与-p ， -e和-s选项。

警告 ：无法访问JDK内部API。

-cp path ， -classpath path ， 要么--class-path path

指定在何处查找类文件。

--module-path module-path

指定模块路径。

--upgrade-module-path module-path

指定升级模块路径。

--system java-home

指定备用系统模块路径。

--add-modules module-name [, module-name...]

将模块添加到根集中以进行分析。

--multi-release version

指定处理多发行版JAR文件时的版本version应该是大于等于9的整数或基数。

—q要么-quite

不显示缺少来自的依赖–generate-module-info输出。

-version要么--version

打印版本信息。

模块依赖性分析选项

–m module-name要么--module module-name

指定要分析的根模块。

--generate-module-info dir

产生module-info.java在指定目录下。指定的JAR文件将被分析。此选项不能与--dot-output要么--class-path选项。使用--generate-open-module开放模块的选项。

--generate-open-module dir

产生module-info.java将指定目录下的指定JAR文件作为打开模块。此选项不能与--dot-output要么--class-path选项。

--check module-name [, module-name...]

分析指定模块的依赖性。它打印模块描述符，分析后生成的模块相关性以及转换减少后的图形。它还标识任何未使用的合格出口。

--list-deps

列出模块依赖性以及JDK内部API的软件包名称（如果已引用）。

--list—reduced-deps

和...一样--list-deps没有列出模块图中的隐含读取边。如果模块M1读取M2，并且M2需要在M3上传递，则隐含M1读数M3，并且未在图中显示。

--print-module-deps

和...一样--list-reduced-deps打印以逗号分隔的模块依赖关系列表。输出可以由jlink --add-modules创建包含这些模块及其传递依赖关系的自定义图像。

筛选依赖项的选项

-p pkg name ， -package pkg name ， 要么--package pkg name

查找与指定程序包名称匹配的依赖项。您可以为不同的软件包多次指定此选项。的-p和-e选项是互斥的。

-e regex ， -regex regex ， 要么--regex regex

查找与指定模式匹配的依赖项。的-p和-e选项是互斥的。

--require module-name

查找与给定模块名称匹配的依赖项（可以多次给出）。的--package ， --regex和--require选项是互斥的。

-f regex要么-filterregex

过滤与给定模式匹配的依赖项。如果多次，将选择最后一个。

-filter:package

过滤同一包中的依赖关系。这是默认值。

-filter:archive

过滤同一存档中的依赖关系。

-filter:module

过滤同一模块内的依赖关系。

-filter:none

没有-filter:package和-filter:archive过滤。通过指定的过滤-filter该选项仍然适用。

筛选要分析的类别的选项

-include regex

将分析限制为类匹配模式。此选项过滤要分析的类的列表。可以与-p和-e ，将模式应用于依赖项。

-P要么-profile

显示包含软件包的配置文件。

-R要么-recursive

递归遍历所有运行时依赖项。的-R选项暗示-filter:none 。如果-p ， -e ， 要么-f指定了选项，仅分析匹配的依赖性。

-I要么-inverse

分析其他给定选项的依赖性，然后找到直接和间接依赖于匹配节点的所有工件。这等效于编译时视图分析和打印依赖项摘要的逆过程。此选项必须与--require ， --package ， 要么--regex选项。

--compile-time

分析可传递依赖项的编译时视图，例如可移植依赖项的编译时视图-R选项。分析其他指定选项的依赖性。如果从目录，JAR文件或模块中找到依赖项，则将分析包含归档文件的所有类。
