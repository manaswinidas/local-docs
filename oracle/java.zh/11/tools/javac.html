javac

您可以使用javac工具及其选项，以读取Java类和接口定义并将其编译为字节码和类文件。

概要

javac [ options ] [ sourcefiles ]
options

命令行选项。请参见javac选项概述 。

sourcefiles

一个或多个要编译的源文件（例如MyClass.java ）或处理注释（例如MyPackage.MyClass ）。

描述

的javac命令读取用Java编程语言编写的类和接口定义，并将其编译为字节码类文件。的javac该命令还可以处理Java源文件和类中的注释。

一个新的启动器环境变量， JDK_JAVAC_OPTIONS是在JDK 9中引入的，它的内容位于命令行之前， javac 。请参见使用JDK_JAVAC_OPTIONS环境变量 。

有两种方法可以将源代码文件名传递给javac 。

  • 对于少量源文件，您可以在命令行上列出文件名。

  • 对于大量的源文件，您可以使用@filename在选项javac命令行以包含列出源文件名的文件。有关选项的说明，请参见“ 标准选项” ；有关以下内容的说明，请参见javac命令行参数文件 ： javac参数文件。

源代码文件名必须具有.java后缀，类文件名必须具有.class后缀，并且源文件和类文件都必须具有标识该类的根名称。例如，一个名为MyClass将被写入名为MyClass.java并编译成一个字节码类文件，称为MyClass.class 。

内部类定义会生成其他类文件。这些类文件的名称结合了内部和外部类的名称，例如MyClass$MyInnerClass.class 。

您应将源文件放在反映其包树的目录树中。例如：

  • Oracle Solaris，Linux和macOS ：如果所有源文件都在/workspace ，然后放入com.mysoft.mypack.MyClass在/workspace/com/mysoft/mypack/MyClass.java 。

  • Windows：如果所有源文件都在\workspace ，然后放入com.mysoft.mypack.MyClass在\workspace\com\mysoft\mypack\MyClass.java 。

默认情况下，编译器将每个类文件与其源文件放在同一目录中。您可以使用-d 标准选项中所述的选项 。

程序界面

的javac该命令支持新的Java Compiler API，该API由的类和接口定义javax.tools包。

要编译一组源文件，编译器可能需要隐式加载其他源文件。请参阅搜索类型 。此类文件当前未进行注释处理。默认情况下，当进行批注处理并且编译任何隐式加载的源文件时，编译器会发出警告。的-implicit选项提供了一种抑制警告的方法。

使用JDK_JAVAC_OPTIONS环境变量

的内容JDK_JAVAC_OPTIONS环境变量，由空格（）或空格字符（ \n ， \t ， \r ， 要么\f ）放在传递给的命令行参数之前javac作为参数列表。

环境变量的编码要求与javac系统上的命令行。 JDK_JAVAC_OPTIONS处理环境变量内容的方式与在命令行中指定的方式相同。

单引号（ ' ）或双引号（ " ）可用于包含空格字符的参数。只需删除一对引号，即可保留开引号和第一个匹配的闭引号之间的所有内容。如果找不到匹配的报价，启动器将中止并显示错误消息。 @files支持，因为它们在命令行中指定。但是，就像@files ，不支持使用通配符。

引用包含空格的参数的示例：

export JDK_JAVAC_OPTIONS='@"C:\white spaces\argfile"'
export JDK_JAVAC_OPTIONS='"@C:\white spaces\argfile"'
export JDK_JAVAC_OPTIONS='@C:\"white spaces"\argfile'

  • 如果--class-path ， -classpath ， 要么-cp未指定，则用户类路径为当前目录。

  • 如果-processorpath如果未指定option，则还将在类路径中搜索注释处理器。

-d directory

设置类文件的目标目录。如果类是包的一部分，则javac将类文件放在反映包名称的子目录中，并根据需要创建目录。例如：

  • Oracle Solaris，Linux和macOS ：如果指定-d/home/myclasses这个班叫做com.mypackage.MyClass ，则类文件为/home/myclasses/com/mypackage/MyClass.class 。

如果-d未指定选项，则javac将每个类文件与生成它的源文件放在同一目录中。

注意：

由指定的目录-d选项不会自动添加到您的用户类路径中。

-deprecation

显示不赞成使用的成员或类的每次使用或覆盖的说明。没有-deprecation选项， javac显示使用或覆盖不赞成使用的成员或类的源文件的摘要。的-deprecation选项是的简写-Xlint:deprecation 。

-endorseddirs directories

警告丢失serialVersionUID可序列化类的定义。例如：

如果可序列化的类未明确声明一个名为serialVersionUID ，则序列化运行时环境将计算默认值serialVersionUID该值基于类的各个方面，如Java对象序列化规范中所述。但是，强烈建议所有可序列化的类显式声明serialVersionUID值，因为默认的计算过程serialVersionUID值对类详细信息高度敏感，类详细信息可能会根据编译器的实现而有所不同。结果，这可能会导致意外InvalidClassExceptions在反序列化期间。保证一致serialVersionUID如果要在不同的Java编译器实现中实现值，则可序列化的类必须声明一个显式的serialVersionUID值。

unchecked

批注处理

的javac该命令直接为注释处理提供支持，取代了对单独注释处理命令的需要， apt 。

注释处理器的API在javax.annotation.processing和javax.lang.model包和子包。

批注处理的工作方式

除非使用-proc:none选项，编译器将搜索可用的所有注释处理器。可以使用以下命令指定搜索路径-processorpath选项。如果未指定路径，则使用用户类路径。通过名为的服务提供商配置文件来定位处理器META-INF/services/javax.annotation.processing 。搜索路径上的处理器。此类文件应包含要使用的任何注释处理器的名称，每行列出一个。另外，也可以使用-processor选项。

在命令行上扫描源文件和类以确定存在哪些批注之后，编译器将查询处理器以确定它们处理的批注。找到匹配项后，将调用处理器。处理器可以声明其处理的注释，在这种情况下，无需进一步尝试为这些注释找到任何处理器。声明所有注释后，编译器不会搜索其他处理器。

如果有任何处理器生成新的源文件，则将进行另一轮注释处理：扫描所有新生成的源文件，并像以前一样处理注释。在前几轮中调用的任何处理器也会在所有后一轮中调用。这将继续直到没有新的源文件生成。

在没有生成新源文件的回合之后，最后一次调用注释处理器，以使它们有机会完成所有剩余的工作。最后，除非-proc:only使用option时，编译器将编译原始文件和所有生成的源文件。

搜索类型

要编译源文件，编译器通常需要有关类型的信息，但是类型定义不在命令行上指定的源文件中。

编译器需要在源文件中使用，扩展或实现的每个类或接口的类型信息。这包括在源文件中未明确提及的类和接口，但它们通过继承提供信息。

例如，当您创建的子类时java.awt.Window ，您还使用了的祖先类Window ： java.awt.Container ， java.awt.Component和java.lang.Object 。

当编译器需要类型信息时，它将搜索定义该类型的源文件或类文件。编译器首先在引导类和扩展类中搜索类文件，然后在用户类路径（默认情况下为当前目录）中搜索。用户类别路径是通过设置CLASSPATH环境变量或通过使用-classpath选项。

如果您设定-sourcepath选项，然后编译器会在指示的路径中搜索源文件。否则，编译器将在用户类路径中搜索类文件和源文件。

您可以使用以下命令指定其他引导程序或扩展程序类-bootclasspath和-extdirs选项。请参阅javac的交叉编译选项 。

成功的类型搜索可能会产生类文件，源文件或两者。如果两者都找到，则可以使用-Xprefer选项指示编译器使用哪个。如果newer指定后，编译器将使用两个文件中的较新版本。如果source指定后，编译器将使用源文件。默认是newer 。

如果类型搜索本身或由于设置的结果而找到了所需类型的源文件。 -Xprefer选项，然后编译器读取源文件以获取所需的信息。默认情况下，编译器还会编译源文件。您可以使用-implicit选项来指定行为。如果none指定，则不会为源文件生成任何类文件。如果class指定后，将为源文件生成类文件。

直到注释处理完成后，编译器才可能发现需要一些类型信息。当在源文件中找到类型信息并且没有-implicit如果指定了option，则编译器会警告该文件正在编译中，而不会进行注释处理。要禁用警告，请在命令行上指定文件（以便对其进行批注处理）或使用-implicit选项，指定是否应为此类源文件生成类文件。
