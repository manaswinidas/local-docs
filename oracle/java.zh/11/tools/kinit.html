金尼特

您使用kinit工具及其用于获取和缓存Kerberos票证授予票证的选项。

该工具的功能类似于kinit在其他Kerberos实现中通常可以找到的工具，例如SEAM和MIT Reference实现。用户必须先在密钥分发中心（KDC）中注册为主体，然后才能运行kinit 。

概要

初始票务请求：

 kinit [-A] [-f] [-p] [-c cache_name] [-l lifetime] [-r renewable_time] [[-k [-t keytab_file_name]] [principal] [password]

续订票证：

 kinit -R [-c cachename] [principal]

描述

默认情况下，在Windows上，一个名为USER_HOME\krb5cc_USER_NAME生成。

标识符USER_HOME从java.lang.System属性user.home 。 USER_NAME从java.lang.System属性user.name 。如果USER_HOME为null，高速缓存文件存储在程序正在运行的当前目录中。 USER_NAME是操作系统的登录用户名。该用户名可以与用户的主体名不同。例如，在Windows上，缓存文件可能是C:\Windows\Users\duke\krb5cc_duke ，其中duke是个USER_NAME和C:\Windows\Users\duke是个USER_HOME 。

默认情况下，密钥表名称是从Kerberos配置文件中检索的。如果未在Kerberos配置文件中指定密钥表名称，则kinit工具将假定该名称为USER_HOME\krb5.keytab

如果您未使用password命令行上的选项kinit工具会提示您输入密码。

注意：

的password此选项仅用于测试目的。不要在脚本中指定密码，也不要在命令行中提供密码。这样做会破坏您的密码。

指令

您可以指定以下命令之一。在命令之后，为其指定选项。

-A

不包含地址。

-f

发行可转让票证。

-p

发行可代理票。

-c cache_name

缓存名称（例如， FILE:D:\temp\mykrb5cc ）。

-l lifetime

设置票证的生命周期。

-r renewable_time

设置票证可以更新的总寿命。

-R

续订票证。

-k

使用密钥表

-t keytab_filename

密钥标签名称（例如， D:\winnt\profiles\duke\krb5.keytab ）。

principal

主体名称（例如， duke@example.com ）。

password

的principal的Kerberos密码。不要在命令行或脚本中指定它。

-help

显示说明。

例子

向默认客户端从当前客户端主机请求对身份验证有效的凭据，以将凭据缓存存储在默认位置（ C:\Windows\Users\duke\krb5cc_duke ）：

kinit duke@example.com

请求其他主体的代理凭据，并将这些凭据存储在指定的文件缓存中：

kinit -p -c FILE:C:\Windows\Users\duke\credentials\krb5cc_cafebeef
    cafebeef@example.com

请求不同主体的可代理凭证和可转发凭证，并将这些凭证存储在指定的文件缓存中：

kinit -f -p -c FILE:C:\Windows\Users\duke\credentials\krb5cc_cafebeef
    cafebeef@example.com

显示帮助菜单kinit工具：

kinit -help
