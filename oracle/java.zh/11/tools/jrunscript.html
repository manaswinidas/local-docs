简写

您使用jrunscript命令来运行支持交互式和批处理模式的命令行脚本外壳。

概要

注意：

此工具是实验性的 ，不受支持。 

jrunscript [options] [arguments]
options

这代表jrunscript可以使用的命令行选项。请参阅jrunscript命令的选项 。

arguments

使用参数时，请紧随其后的是选项或命令名称。参见参数 。

描述

的jrunscript command是独立于语言的命令行脚本外壳。的jrunscript该命令同时支持交互（read-eval-print）模式和批处理（ -f选项）脚本执行模式。默认情况下，JavaScript是使用的语言，但是-l选项可用于指定其他语言。通过使用Java编写语言交流的脚本， jrunscript命令支持探索性编程风格。

如果使用JavaScript，则在评估用户定义的脚本之前， jrunscript命令初始化某些内置函数和对象，这些函数和对象在API规范中记录jrunscript JavaScript内置函数。

jrunscript命令的选项

-cp path要么-classpath path

指示脚本需要访问任何类文件的位置。

-Dname=value

设置Java系统属性。

-Jflag

通行证flag直接到Java虚拟机jrunscript命令正在运行。

-l language

使用指定的脚本语言。默认情况下，使用JavaScript。要使用其他脚本语言，必须使用以下命令指定相应脚本引擎的JAR文件： -cp要么-classpath选项。

-e script

评估指定的脚本。此选项可用于运行在命令行上完全指定的单行脚本。

-encoding encoding

指定用于读取脚本文件的字符编码。

-f script-file

评估指定的脚本文件（批处理模式）。

-f -

进入交互模式以从标准输入中读取和评估脚本。

-help要么-?

显示帮助消息并退出。

-q

列出所有可用的脚本引擎并退出。

争论

是否存在参数，如果不存在-e要么-f使用option时，第一个参数是脚本文件，其余参数（如果有）作为脚本参数传递。如果参数和-e或者-f选项被使用，然后所有参数作为脚本参数传递。如果参数-e和-f缺少，则使用交互模式。

执行内联脚本的示例

jrunscript -e "print('hello world')"
jrunscript -e "cat('http://www.example.com')"

使用指定语言和评估脚本文件的示例

jrunscript -l js -f test.js

互动模式的例子

jrunscript
js> print('Hello World\n');
Hello World
js> 34 + 55
89.0
js> t = new java.lang.Thread(function() { print('Hello World\n'); })
Thread[Thread-0,5,main]
js> t.start()
js> Hello World
 
js>

使用脚本参数运行脚本文件

在此示例中， test.js文件是脚本文件。的arg1 ， arg2和arg3参数传递到脚本。脚本可以使用arguments数组访问这些参数。 

jrunscript test.js arg1 arg2 arg3
