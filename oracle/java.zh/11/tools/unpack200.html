开箱200

您使用unpack200命令将打包文件转换为JAR文件以进行Web部署。

概要

注意：

pack200和unpack200已被弃用，并可能在将来的JDK版本中删除。

unpack200 [ options  ] input-file JAR-file
options

命令行选项。请参阅unpack200命令的选项 。

input-file

输入文件的名称，可以是pack200 gzip文件或pack200文件。输入也可以是由pack200努力地0 ，在这种情况下，输入文件的内容将通过以下命令复制到输出JAR文件： pack200标记。

JAR-file

输出JAR文件的名称。

描述

的unpack200该命令是本机实现，可以转换由pack200放入用于Web部署的JAR文件中。以下是典型用法的示例。在以下示例中， myarchive.jar文件产生于myarchive.pack.gz使用默认unpack200命令设置。 

unpack200 myarchive.pack.gz myarchive.jar

unpack200命令的选项

-Hvalue要么--deflate-hint=value

将通缩设置为true ， false ， 要么keep在JAR文件中的所有条目上。默认模式是keep 。如果值为true要么false ，然后--deflate=hint选项将覆盖默认行为，并在输出JAR文件中的所有条目上设置放气模式。

-r要么--remove-pack-file

除去输入包文件。

-v要么--verbose

显示最少的消息。此选项的多个规范显示更多详细消息。

-q要么--quiet

指定无消息的安静操作。

-l filename或--log-file=filename

指定用于记录输出消息的日志文件。

-?要么-h要么--help

打印有关的帮助信息unpack200命令。

-V要么--version

打印有关的版本信息unpack200命令。

-Joption

将选项传递给Java虚拟机，其中option是Java应用程序启动器参考页上描述的选项之一。例如， -J-Xms48m将启动内存设置为48 MB。

笔记

此命令不应与unpack命令。它们是截然不同的产品。

如有差异，JDK随附的Java SE API规范是取代权限。
