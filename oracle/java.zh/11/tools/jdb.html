数据库

您使用jdb命令及其用于查找和修复Java平台程序中的错误的选项。

概要

jdb [options] [classname] [arguments]
options

这代表jdb命令行选项。请参见jdb命令的选项 。

classname

这代表要调试的主类的名称。

arguments

这代表传递给main()类的方法。

描述

Java调试器（JDB）是Java类的简单命令行调试器。的jdb命令及其选项将调用JDB。的jdb该命令演示了Java平台调试器体系结构，并提供了对本地或远程JVM的检查和调试。

启动JDB会话

有很多方法可以启动JDB会话。最常用的方法是让JDB使用要调试的应用程序的主类启动新的JVM。为此，请替换jdb的命令java命令行中的命令。例如，如果您的应用程序的主类是MyClass ，然后使用以下命令在JDB下对其进行调试：

jdb MyClass

以这种方式启动时， jdb该命令使用指定的参数调用第二个JVM，加载指定的类，并在执行该类的第一条指令之前停止JVM。

另一种使用方式jdb命令是通过将其附加到已经在运行的JVM上。用于启动JVM的语法jdb运行JVM时，将附加以下命令。这将加载进程内调试库并指定要建立的连接类型。 

java -agentlib:jdwp=transport=dt_socket,server=y,suspend=n MyClass

然后，您可以附加jdb使用以下命令将命令传递给JVM：

jdb -attach 8000

8000是正在运行的JVM的地址。

的MyClass参数未在中指定jdb命令行在这种情况下，因为jdb命令正在连接到现有JVM，而不是启动新JVM。

有许多其他方法可以将调试器连接到JVM，并且所有这些方法均受支持。 jdb命令。Java平台调试器体系结构中有关于这些连接选项的其他文档。

断点

可以在JDB中的行号或方法的第一条指令中设置断点，例如：

  • 命令stop at MyClass:22在包含以下内容的源文件的第22行的第一条指令处设置断点MyClass 。

  • 命令stop in java.lang.String.length在方法开始时设置一个断点java.lang.String.length 。

  • 命令stop in MyClass.用途标识用于的静态初始化代码MyClass 。

重载方法时，还必须指定其参数类型，以便可以为断点选择适当的方法。例如， MyClass.myMethod(int,java.lang.String)要么MyClass.myMethod() 。

的clear该命令使用以下语法删除断点： clear MyClass:45 。使用clear要么stop不带参数的命令显示当前设置的所有断点的列表。的cont命令继续执行。

踏步

的step命令将执行前进到下一行，无论是在当前堆栈帧中还是在被调用的方法中。的next命令将执行推进到当前堆栈帧的下一行。

例外情况

当发生异常时，没有catch如果在抛出线程的调用堆栈中的任何位置执行该语句，则JVM通常会打印异常跟踪并退出。但是，当在JDB下运行时，控制权将在有问题的时候返回JDB。然后，您可以使用jdb命令以诊断异常原因。

使用catch使已调试的应用程序在其他引发的异常处停止的命令，例如： catch java.io.FileNotFoundException要么catchmypackage.BigTroubleException 。作为指定类或子类的实例的任何异常都将在引发异常时停止应用程序。

的ignore命令否定了较早版本的效果catch命令。的ignore该命令不会导致调试的JVM忽略特定的异常，而只会忽略调试器。

jdb命令的选项

当您使用jdb命令而不是java命令行上的命令jdb该命令接受许多与java命令..

以下选项被接受jdb命令：

-help

显示帮助信息。

-sourcepath dir1:dir2: . . .

使用指定的路径在指定的路径中搜索源文件。如果未指定此选项，则使用点（。）的默认路径。

-attach address

使用默认的连接机制将调试器附加到正在运行的JVM。

-listen address

等待正在运行的JVM使用标准连接器连接到指定地址。

-listenany

等待运行的JVM使用标准连接器连接到任何可用地址。

-launch

启动后立即启动调试的应用程序jdb命令。的-launch选项消除了对run命令。调试的应用程序将启动，然后在加载初始应用程序类之前停止。此时，您可以设置任何必要的断点并使用cont命令以继续执行。

-listconnectors

列出此JVM中可用的连接器。

-connect connector-name:name1=value1....

使用命名的连接器和列出的参数值连接到目标JVM。

-dbgtrace [flags]

打印信息以调试jdb命令。

-tclient

在Java HotSpot VM客户端中运行应用程序。

-tserver

在Java HotSpot VM服务器中运行该应用程序。

-Joption

通行证option到JVM，其中option是Java应用程序启动器的参考页上描述的选项之一。例如， -J-Xms48m将启动内存设置为48 MB。请参阅Java选项概述 。

以下选项转发到调试进程：

-v -verbose[:class|gc|jni]

打开详细模式。

-Dname=value

设置系统属性。

-classpath dir

列出用冒号分隔的目录，在其中查找类。

-X option

非标准目标JVM选项。

支持其他选项，以提供用于将调试器连接到要调试的JVM的备用机制。
