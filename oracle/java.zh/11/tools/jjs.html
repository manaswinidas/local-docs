js

您使用jjs命令行工具来调用Nashorn引擎。

概要

注意：

的jjs JDK 11中不推荐使用该工具和Nashorn引擎，以准备在将来的版本中删除。 
jjs [options] script-files [-- arguments]
options

这代表了一个或多个选项jjs命令，以空格分隔。请参阅jjs命令的选项 。

script-files

这表示您要使用Nashorn引擎解释的一个或多个脚本文件，以空格分隔。如果未指定文件，则启动交互式外壳程序。

arguments

双连字符标记（ -- ）作为参数传递给脚本或交互式外壳。这些值可以通过使用arguments属性。

描述

的jjs命令行工具用于调用Nashorn引擎。您可以使用它来解释一个或几个脚本文件，或运行交互式外壳程序。

jjs命令的选项

的选项jjs命令控制Nashorn引擎解释脚本的条件。

-Dname=value

通过为属性名称分配值来设置要传递给脚本的系统属性。以下示例显示了如何以交互方式调用Nashorn引擎并进行分配myValue到名为myKey ：

>> jjs -DmyKey=myValue
jjs> java.lang.System.getProperty("myKey")
myValue
jjs>

可以重复此选项以设置多个属性。

--add-modules modules

指定root用户Java模块。

-cp path要么-classpath path

指定支持类文件的路径。要设置多个路径，可以重复该选项，也可以用以下字符分隔每个路径：

  • Oracle Solaris，Linux和macOS ：冒号（:)

  • Windows：分号（;）

-doe=[true|false]要么-dump-on-error=[true|false]

发生错误时提供完整的堆栈跟踪。默认情况下，仅打印简短的错误消息。默认参数是false 。

-fv=[true|false]要么-fullversion=[true|false]

打印完整的Nashorn版本字符串。默认参数是false 。

-fx=[true|false]

将脚本作为JavaFX应用程序启动。默认参数是false 。

注意：

您必须显式添加JavaFX模块以将脚本作为JavaFX应用程序启动。以下示例指定了JavaFX模块的位置，并将它们与--module-path和--add-modules选项：

jjs -fx --module-path /SOMEDIR/javafx-sdk-11/lib --add-modules javafx.controls HelloWorld.js

以下示例使用jlink命令以创建包含JavaFX模块的自定义运行时映像。然后，该示例将脚本作为JavaFX应用程序启动，而未在脚本中指定JavaFX模块。 jjs命令：

jlink --module-path /SOMEDIR/javafx-jmods-11 --add-modules jdk.scripting.nashorn,jdk.scripting.nashorn.shell,javafx.controls --output /SOMEDIR/myjdk

/SOMEDIR/myjdk/bin/jjs -fx HelloWorld.js

如果您未明确指定JavaFX模块，则jjs命令显示一条消息并退出：

jjs -fx HelloWorld.js

JavaFX is not available.

-h要么-help

打印选项列表及其说明。

--language=[es5|es6]

指定ECMAScript语言版本。默认版本为ES5。

--module-path path

指定在何处查找用户Java模块。

-ot=[true|false]要么-optimistic-types=[true|false]

通过取消优化重新编译来启用或禁用乐观类型假设。这使编译器尝试对在编译时无法证明其类型的任何程序符号进行尽可能狭窄而原始的键入。如果运行时由于符号类型太窄而遇到错误，则将生成更宽的方法，直到达到稳定状态为止。尽管这会产生尽可能最佳的Java字节码，但错误的类型猜测将导致更长的预热时间。当前默认情况下启用乐观类型，但可以禁用它以提高启动性能。默认参数是true 。

-scripting=[true|false]

启用外壳脚本功能。默认参数是true 。

-strict=[true|false]

启用严格模式，以加强对标准（ECMAScript Edition 5.1）的遵守，从而更易于检测常见的编码错误。默认参数是false 。

-t=zone要么-timezone=zone

设置脚本执行的指定时区。它会覆盖操作系统中设置的时区，并由Date宾语。默认值zone是America/Los_Angeles 。

-v=[true|false]要么-version=[true|false]

打印Nashorn版本字符串。默认参数是false 。

使用Nashorn运行脚本的示例

jjs script.js

在交互模式下运行Nashorn的示例

>> jjs
jjs> println("Hello, World!")
Hello, World!
jjs> quit()
>>

将参数传递给Nashorn的示例

>> jjs -- a b c
jjs> arguments.join(", ")
a, b, c
jjs>
