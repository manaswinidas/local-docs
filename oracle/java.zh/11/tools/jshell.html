壳

您使用jshell该工具可在读评估打印循环（REPL）中交互式评估Java编程语言的声明，语句和表达式。

概要

jshell [options] [load-files]
options

命令行选项，以空格分隔。请参阅jshell选项 。

load-files

启动该工具时要运行的一个或多个脚本。脚本可以包含任何有效的代码段或JShell命令。

该脚本可以是本地文件，也可以是以下预定义脚本之一：

DEFAULT

加载默认条目，这些条目通常用作导入。

JAVASE

导入所有Java SE软件包。

PRINTING

定义print ， println和printf如jshell工具中使用的方法。

对于多个脚本，请使用空格分隔名称。脚本按照在命令行中输入的顺序运行。命令行脚本在启动脚本之后运行。要在JShell启动后运行脚本，请使用/open命令。

要接受来自标准输入的输入并禁止交互式I / O，请为load-files输入连字符（-）。使用此选项可以使用jshell管道链中的工具。

描述

JShell提供了一种交互式评估Java编程语言的声明，语句和表达式的方法，从而使学习该语言，探索不熟悉的代码和API以及对复杂代码进行原型开发变得更加容易。接受Java语句，变量定义，方法定义，类定义，导入语句和表达式。输入的代码位称为代码段。

输入摘要后，便会对其进行评估并提供反馈。根据输入的代码段和选择的反馈方式，反馈的内容从结果和对动作的说明到什么都没有。不管反馈模式如何，都会描述错误。从详细模式开始，以在学习该工具时获得最多的反馈。

命令行选项可用于在启动JShell时配置初始环境。在JShell中，命令可用于根据需要修改环境。

可以从文件加载现有的代码片段以初始化JShell会话，或在会话内的任何时间。可以在会话中修改代码段，以尝试不同的变体并进行更正。要保留摘要供以后使用，请将其保存到文件中。

jshell的选项

--add-modules module[,module...]

除了初始模块外，还指定要解析的根模块。

-Cflag

提供一个标志以传递给编译器。要传递多个标志，请为每个需要的标志或标志参数提供此选项的实例。

--class-path path

指定搜索以查找类文件的目录和档案。此选项会覆盖CLASSPATH环境变量。如果未设置环境变量且未使用此选项，则搜索当前目录。对于Oracle Solaris，Linux和macOS，请使用冒号（:)分隔路径中的项目。对于Windows，请使用分号（;）分隔各项。

--feedback mode

设置响应于输入内容而提供的初始反馈级别。可以使用以下命令在会话中覆盖初始级别/set feedback mode命令。默认是normal 。

以下值适用于mode ：

verbose

提供条目的详细反馈。有关执行的操作的其他信息将在操作结果之后显示。下一个提示与反馈之间用空白行分隔。

normal

提供平均数量的反馈。下一个提示与反馈之间用空白行分隔。

concise

提供最少的反馈。下一个提示将紧随代码片段或反馈之后。

silent

不提供反馈。下一个提示紧跟在代码片段之后。

custom

根据如何定义模式提供自定义反馈。使用以下命令在JShell中创建自定义反馈模式： /set mode命令。

--help要么-h要么-?

打印标准选项的摘要并退出该工具。

--help-extra要么-X

打印非标准选项的摘要并退出该工具。非标准选件如有更改，恕不另行通知。

-Jflag

提供一个标志以传递给运行时系统。要传递多个标志，请为每个需要的标志或标志参数提供此选项的实例。

--module-path modulepath
指定在哪里找到应用程序模块。对于Oracle Solaris，Linux和macOS，请使用冒号（:)分隔路径中的项目。对于Windows，请使用分号（;）分隔各项。
--no-startup

阻止启动JShell启动时启动脚本。使用此选项可以在启动JShell时仅运行在命令行上输入的脚本，或者，如果没有输入任何脚本，则在没有任何预加载信息的情况下启动JShell。如果--startup使用选项。

-q

将反馈模式设置为concise ，与输入相同--feedback concise 。

-Rflag

提供一个标志以传递到远程运行时系统。要传递多个标志，请为每个要传递的标志或标志参数提供此选项的实例。

-s

将反馈模式设置为silent ，与输入相同--feedback silent 。

--show-version

打印版本信息并进入工具。

--startup file

覆盖此会话的默认启动脚本。该脚本可以包含任何有效的代码段或命令。

该脚本可以是本地文件，也可以是以下预定义脚本之一：

DEFAULT

加载默认条目，这些条目通常用作导入。

JAVASE

导入所有Java SE软件包。

PRINTING

定义print ， println和printf如jshell工具中使用的方法。

对于多个脚本，请为每个脚本提供此选项的单独实例。首次启动JShell以及使用以下命令重新启动会话时，将运行启动脚本： /reset ， /reload ， 要么/env命令。启动脚本按在命令行中输入的顺序运行。

如果--no-startup使用选项。

-v

将反馈模式设置为verbose ，与输入相同--feedback verbose 。

--version

打印版本信息并退出工具。

jshell命令

内jshell工具，命令用于修改环境和管理代码段。

/drop {name|id|startID-endID}[ {name|id|startID-endID}...]

删除按名称，ID或ID范围标识的摘要，使其变为非活动状态。对于一系列ID，请提供以连字符分隔的开始ID和结束ID。要提供列表，请用空格分隔列表中的项目。 使用/list命令以查看代码段的ID。

/edit [option]

打开一个编辑器。如果未输入任何选项，则编辑器将打开并显示活动代码段。

以下选项有效：

{name|id|startID-endID}[ {name|id|startID-endID}...]

用名称，ID或ID范围标识的代码段打开编辑器。对于一系列ID，请提供以连字符分隔的开始ID和结束ID。要提供列表，请用空格分隔列表中的项目。 使用/list命令以查看代码段的ID。

-all

使用所有代码片段打开编辑器，包括启动代码片段和失败，被覆盖或删除的代码片段。

-start

用启动JShell启动时评估的启动摘要打开编辑器。

要退出编辑模式，请关闭编辑器窗口，或者在出现以下情况时响应提供的提示： -wait设置编辑器时使用了该选项。

使用/set editor命令以指定要使用的编辑器。如果未设置编辑器，那么将依次检查以下环境变量： JSHELLEDITOR ， VISUAL和EDITOR 。如果在JShell中未设置任何编辑器，并且未设置任何编辑器环境变量，则使用简单的默认编辑器。

/env [options]

显示环境设置，或更新环境设置并重新启动会话。如果未输入任何选项，则显示当前环境设置。如果输入了一个或多个选项，则会话将重新启动，如下所示：

  • 使用提供的选项更新环境设置。

  • 重置执行状态。

  • 运行启动脚本。

  • 以输入的顺序静默重放历史记录。历史记录包括所有有效的摘要或/drop在输入的命令jshell提示符，在命令行中输入的脚本，或在命令行中输入的脚本/open命令。

在命令行中输入的环境设置或先前提供的环境设置/reset ， /env ， 要么/reload除非保留命令option输入将覆盖设置。

以下选项有效：

--add-modules module[,module...]

除了初始模块外，还指定要解析的根模块。

--add-exports source-module/package=target-module[,target-module]*

增加了出口package从source-module至target-module 。

--class-path path

指定搜索以查找类文件的目录和档案。此选项会覆盖CLASSPATH环境变量。如果未设置环境变量且未使用此选项，则搜索当前目录。对于Oracle Solaris，Linux和macOS，请使用冒号（:)分隔路径中的项目。对于Windows，请使用分号（;）分隔各项。

--module-path modulepath
指定在哪里找到应用程序模块。对于Oracle Solaris，Linux和macOS，请使用冒号（:)分隔路径中的项目。对于Windows，请使用分号（;）分隔各项。
/exit [integer-expression-snippet]

退出工具。如果未输入任何代码段，则退出状态为零。如果输入了一个代码段，并且该代码段的结果是整数，则该结果将用作退出状态。如果发生错误，或者代码段的结果不是整数，则会显示错误，并且工具将保持活动状态。

/history

显示在此会话中输入的内容。

/help [command|subject]

显示有关命令和主题的信息。如果未输入任何选项，则将显示所有命令的信息摘要和可用主题列表。如果提供了有效命令，那么将显示该命令的扩展信息。如果输入了有效的主题，则会显示有关该主题的信息。

以下值subject有效：

描述可用于配置环境的选项。

提供对该工具的介绍。

描述用于完成命令和摘要的击键。请参阅输入快捷方式 。

/imports

显示当前的活动导入，包括来自启动脚本的内容以及启动JShell时在命令行上输入的脚本的内容。

/list [option]

/methods [option]

{name|id|startID-endID}[ {name|id|startID-endID}...]

显示通过名称，ID或ID范围标识的方法的信息。对于一系列ID，请提供以连字符分隔的开始ID和结束ID。要提供列表，请用空格分隔列表中的项目。 使用/list命令以查看代码段的ID。

-all

显示所有方法的信息，包括启动JShell时添加的方法，失败，被覆盖或删除的方法。

以下选项有效：

-default

将启动设置设置为默认设置。

-none

指定不使用启动设置。

使用-retain选项以跨会话保存开始设置。

truncation mode length selector

设置显示值的最大长度。如果未输入任何模式，则将显示所有反馈模式的当前截断值。如果仅输入一种模式，则显示该模式的当前截断值。

要定义截断值，需要以下参数：

mode

指定应用截断值的反馈模式。仅使用/set mode命令可以修改。

length

指定无符号整数，用作指定选择器的最大长度。

selector

指定在其中应用截断值的上下文。在线帮助中描述了选择器，可以使用/help /set truncation命令。

/types [option]

显示输入的类，接口和枚举。如果未输入任何选项，那么将显示所有当前的活动类，接口和枚举。

以下选项有效：

{name|id|startID-endID}[ {name|id|startID-endID}...]

显示有关通过名称，ID或ID范围标识的类，接口和枚举的信息。对于一系列ID，请提供以连字符分隔的开始ID和结束ID。要提供列表，请用空格分隔列表中的项目。 使用/list命令以查看代码段的ID。

-all

显示所有类，接口和枚举的信息，包括启动JShell时添加的信息，失败，被覆盖或删除的类，接口和枚举。

-start

显示有关启动类，接口和JShell启动时添加的枚举的信息。

/vars [option]

显示输入的变量的名称，类型和值。如果未输入任何选项，则显示所有当前活动变量。

以下选项有效：

{name|id|startID-endID}[ {name|id|startID-endID}...]

显示有关由名称，ID或ID范围标识的变量的信息。对于一系列ID，请提供以连字符分隔的开始ID和结束ID。要提供列表，请用空格分隔列表中的项目。 使用/list命令以查看代码段的ID。

-all

显示所有变量的信息，包括启动JShell时添加的变量，失败，被覆盖或删除的变量。

-start

显示有关启动JShell时添加的启动变量的信息。

/?

与相同/help命令。

/!

重新运行最后一个代码段。

/{id|startID-endID}[ {id|startID-endID|name}...]

重新运行由ID，ID范围或名称标识的代码段。对于一系列ID，请提供以连字符分隔的开始ID和结束ID。要提供列表，请用空格分隔列表中的项目。 列表中的第一项必须是ID或ID范围。使用/list命令以查看代码段的ID。

/-n

重新运行第n个前一片段。例如，如果输入了15个代码段，则/-4运行第11个代码段。命令不包括在计数中。

输入快捷方式

下表描述了可用于在JShell中输入命令和代码段的快捷方式。

捷径 用法

制表符补全

输入代码段，命令，子命令，命令参数或命令选项时，请使用Tab键自动完成该项。如果根据输入的内容无法确定该项目，则提供可能的选项。

输入方法调用时，请在方法调用的左括号后使用Tab键查看方法的参数。如果该方法具有多个签名，则将显示所有签名。第二次按Tab键将显示方法说明和第一个签名的参数。继续按Tab键以获取所有其他签名的描述。

Shift + V

输入完整的表达式后，使用此键序列将表达式转换为由表达式的类型确定的类型的变量声明。

Shift + M

输入完整的表达式或语句后，使用此键序列将表达式或语句转换为方法声明。如果输入了表达式，则返回类型取决于表达式的类型。

Shift + I

输入无法解析的标识符时，请使用此键序列显示可能的导入，这些导入可基于指定的类路径的内容来解析标识符。

命令缩写

如果缩写唯一地标识命令，则接受该缩写。例如， /l被认为是/list命令。然而， /s不是有效的缩写，因为无法确定/set要么/save命令的意思。使用/se为了/set命令或/sa为了/save命令。

子命令，命令参数和命令选项也接受缩写。例如使用/m -a显示所有方法。

历史导航

跨会话维护输入内容的历史记录。使用向上和向下箭头在当前和过去的会话中滚动浏览命令和片段。使用Ctrl键和向上和向下箭头可以跳过多行代码段的第一行以外的所有内容。

历史搜索

使用Ctrl + R组合键在历史记录中搜索输入的字符串。提示更改为显示字符串和匹配项。Ctrl + R通过较早的条目从历史记录中的当前位置向后搜索。Ctrl + S会从历史记录中的当前位置向前搜索，但以后会输入。

代码段是Java语句，变量定义，方法定义，类定义，导入语句和表达式。如果缺少分号，则会将它们自动添加到完整代码段的末尾。

以下示例显示了两个变量和一个已定义的方法，以及正在运行的方法。请注意，由于未提供任何变量，因此会自动创建临时变量来保存结果。

jshell> int a=4
a ==> 4

b ==> 8

   ...> return i1 * i1;
   ...> }

创建自定义反馈模式的示例

反馈模式确定所显示的提示，输入作为摘要提供的反馈消息以及显示值的最大长度。提供了预定义的反馈模式。还提供了用于创建自定义反馈模式的命令。

使用/set mode命令以创建新的反馈模式。在下面的示例中，新模式mymode ，它基于预定义的反馈模式， normal ，并显示验证命令反馈：

jshell> /set mode mymode normal -command
|  Created new feedback mode: mymode

因为新模式基于normal模式下，提示是相同的。下面的示例演示如何查看使用了哪些提示，然后将提示更改为自定义字符串。第一个字符串代表标准的JShell提示符。第二个字符串表示多行代码段中其他行的提示。 

jshell> /set prompt mymode
|  /set prompt mymode "\njshell> " "   ...> "

jshell> /set prompt mymode "\nprompt$ " "   continue$ "

显示值的最大长度由截断设置控制。不同类型的值可以具有不同的长度。下面的示例将整体截断值设置为72，对于可变值表达式将截断值设置为500：

jshell> /set truncation mymode 72

jshell> /set truncation mymode 500 varvalue

输入摘录后显示的反馈信息受格式设置控制，并且取决于输入的摘录类型和对该摘录采取的操作。在预定义模式下normal ，字符串created创建方法时显示。以下示例显示了如何将字符串更改为defined ：

jshell> /set format mymode action "defined" added-primary

使用/set feedback命令以开始使用刚创建的反馈模式。以下示例显示了正在使用的自定义模式：

jshell> /set feedback mymode
|  Feedback mode: mymode

prompt$ int square (int num1){
   continue$ return num1*num1;
   continue$ }
|  defined method square(int)

prompt$
