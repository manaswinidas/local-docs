资讯

您使用jinfo命令以生成指定Java进程的Java配置信息。此命令是实验性的，不受支持。

概要

jinfo [option] pid
option

这代表jinfo命令行选项。请参阅jinfo命令的选项 。

pid

要为其打印配置信息的进程ID。该进程必须是Java进程。要获取计算机上运行的Java进程的列表，请使用ps命令，或者，如果JVM进程未在单独的docker实例中运行，则使用jps命令。

注意：

当附加到在单独的docker进程中运行的Java进程时，JDK 10添加了对使用Attach API的支持。然而jps该命令将不会列出在单独的docker实例中运行的JVM进程。如果您尝试将Linux主机与Docker容器中的虚拟机连接，则必须使用以下工具ps查找JVM的PID。

描述

的jinfo命令显示指定Java进程的Java配置信息。配置信息包括Java系统属性和JVM命令行标志。如果指定的进程在64位JVM上运行，则可能需要指定-J-d64选项，例如：

 jinfo -J-d64 -sysprops pid

jinfo命令的选项

注意：

如果不使用以下任何选项，则将同时打印命令行标志和系统属性名称/值对。

-flag name

打印指定命令行标志的名称和值。

-flag [+|-]name

启用或禁用指定的布尔命令行标志。

-flag name=value

将指定的命令行标志设置为指定的值。

-flags

打印传递给JVM的命令行标志。

-sysprops

将Java系统属性打印为名称/值对。

-h要么-help

打印帮助信息。
