捷联

您可以使用jlink工具，用于将一组模块及其相关性组装和优化为自定义运行时映像。

概要

jlink [options] --module-path modulepath --add-modules module [,module...]
options

命令行选项用空格分隔。请参阅jlink选项 。

modulepath

的路径jlink工具发现可观察的模块。这些模块可以是模块化JAR文件，JMOD文件或分解模块。

module

要添加到运行时映像的模块的名称。的jlink工具会添加这些模块及其传递依赖项。

描述

的jlink工具将一组模块及其传递依赖关系链接在一起，以创建自定义运行时映像。

注意：

开发人员负责更新其自定义运行时映像。

jlink选项

--add-modules mod [,mod...]

添加命名模块， mod设置为默认的根模块集。根模块的默认集合为空。

--bind-services

链接服务提供者模块及其依赖性。

-c ={0|1|2}要么--compress={0|1|2}

启用资源压缩：

  • 0 ：无压缩
  • 1 ：常量字符串共享
  • 2 ： 压缩
--disable-plugin pluginname

禁用指定的插件。有关受支持插件的列表，请参见jlink插件。

--endian {little|big}

指定生成的图像的字节顺序。默认值为系统体系结构的格式。

-h要么--help

打印帮助消息。

--ignore-signing-information

当在运行时映像中链接签名的模块化JAR时，可以抑制致命错误。签名的模块化JAR的与签名相关的文件不会复制到运行时映像。

--launcher command=module要么--launcher command=module/main

指定模块的启动器命令名称或模块和主类的命令名称（模块和主类名称之间用斜杠（ / ））。

--limit-modules mod [,mod...]

将可观察模块的范围限制为已命名模块的传递闭包中的那些模块， mod ，再加上主要模块（如果有），再加上--add-modules选项。

--list-plugins

列出可用的插件，您可以通过命令行选项访问这些插件；请参阅jlink插件 。

-p要么--module-path modulepath

指定模块路径。

如果未指定此选项，则默认模块路径为$JAVA_HOME/jmods 。该目录包含java.base模块以及其他标准和JDK模块。如果指定了此选项，但是无法从中解析java.base模块，则jlink命令追加$JAVA_HOME/jmods到模块路径。

--no-header-files

排除头文件。

--no-man-pages

不包括手册页。

--output path

指定生成的运行时映像的位置。

--save-opts filename

节省jlink指定文件中的选项。

--suggest-providers [name, ...]

建议从模块路径实现给定服务类型的提供程序。

--version

打印版本信息。

@filename

从指定文件中读取选项。

选项文件是一个文本文件，其中包含您通常会在命令提示符下输入的选项和值。选项可以显示在一行或几行上。您不能为路径名指定环境变量。您可以通过在井号前面加一个注释来注释掉行（ # ）到该行的开头。

以下是一个选项文件的示例jlink命令：

#Wed Dec 07 00:40:19 EST 2016
--module-path mlib
--add-modules com.greetings
--output greetingsapp

jlink插件

注意：

不支持本节中未列出的插件，它们可能会发生变化。

对于需要pattern-list ，值是逗号分隔的元素列表，每个元素使用以下一种形式：

  • glob-pattern
  • glob:glob-pattern
  • regex:regex-pattern
  • @filename
    • filename是包含要使用的模式的文件名，每行一个模式。

有关所有可用插件的完整列表，请运行以下命令jlink --list-plugins 。

表2-4 jlink插件列表

插件名称 选项 描述

compress

--compress={0|1|2}[:filter=pattern-list]

压缩输出图像中的所有资源。

  • 0级：无压缩
  • 级别1：常量字符串共享
  • 2级：ZIP

可选的pattern-list可以指定过滤器以列出要包含的文件的模式。

include-locales

--include-locales=langtag[,langtag]*

包括以下区域的列表langtag是BCP 47语言标签。此选项支持RFC 4647中定义的语言环境匹配。使用此选项时，请确保添加模块jdk.localedata 。

例：

--add-modules jdk.localedata --include-locales=en,ja,*-IN

order-resources

--order-resources=pattern-list

按优先顺序对指定路径进行排序。如果@filename被指定，那么每一行pattern-list必须与要订购的路径完全匹配。

例：

--order-resources=**/module-info.class,@classlist,/java.base/java/lang/**

strip-debug

--strip-debug

从输出映像中剥离调试信息。

jlink示例

以下命令在目录中创建运行时映像greetingsapp 。此命令链接模块com.greetings ，其模块定义包含在目录中mlib 。 

jlink --module-path mlib --add-modules com.greetings --output greetingsapp

以下命令列出了运行时映像中的模块greetingsapp ：

greetingsapp/bin/java --list-modules
com.greetings
java.base@11
java.logging@11
org.astro@1.0

以下命令在目录中创建运行时映像compressedrt去除了调试符号，使用压缩来减少空间，并包括法语语言环境信息：

jlink --add-modules jdk.localedata --strip-debug --compress=2 --include-locales=fr --output compressedrt

下面的示例比较运行时映像的大小compressedrt与fr_rt ，它没有去除调试符号并且不使用压缩：

jlink --add-modules jdk.localedata --include-locales=fr --output fr_rt

du -sh ./compressedrt ./fr_rt
23M     ./compressedrt
36M     ./fr_rt

以下示例列出了实现java.security的提供程序。提供者 ：

以下示例创建一个名为的自定义运行时映像mybuild其中仅包含java.naming和jdk.crypto.cryptoki及其依赖项，但不包括其他提供程序。请注意，这些依赖项必须存在于模块路径中：

jlink --add-modules java.naming,jdk.crypto.cryptoki --output mybuild

以下命令类似于创建名为greetingsapp ，除了它将通过服务绑定链接从根模块解析的模块；请参见Configuration.resolveAndBind方法。 

jlink --module-path mlib --add-modules com.greetings --output greetingsapp --bind-services
