映射

您使用jmap命令以显示指定进程的详细信息。此命令是实验性的，不受支持。

概要

jmap [options] pid
options

这代表jmap命令行选项。请参阅jmap命令的选项 。

pid

进程ID，由其指定的信息options将被打印。该进程必须是Java进程。要获取计算机上运行的Java进程的列表，请使用ps命令，或者，如果JVM进程未在单独的docker实例中运行，则使用jps命令。

注意：

当附加到在单独的docker进程中运行的Java进程时，JDK 10添加了对使用Attach API的支持。然而jps该命令将不会列出在单独的docker实例中运行的JVM进程。如果您尝试将Linux主机与Docker容器中的虚拟机连接，则必须使用以下工具ps查找JVM的PID。

jmap命令的选项

-clstats pid

连接到正在运行的进程并打印Java堆的类加载器统计信息。

-finalizerinfo pid

连接到正在运行的进程，并在等待完成的对象上打印信息。

-histo[:live] pid

连接到正在运行的进程，并打印Java对象堆的直方图。如果live如果指定了suboption，则它仅计算活动对象。

-dump:dump options pid

连接到正在运行的进程并转储Java堆。的dump options包括：

  • live —指定时，仅转储活动对象；如果未指定，则转储堆中的所有对象。

    format=b —转储Java堆。在hprof二进制格式

  • file=filename —将堆转储到filename

例：jmap -dump:live,format=b,file=heap.bin pid
