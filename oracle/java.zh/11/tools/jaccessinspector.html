jaccessinspector

您使用jaccessinspector Java Accessibility Utilities API的可访问性评估工具，用于检查有关Java虚拟机中对象的可访问信息。

描述

的jaccessinspector该工具使您可以选择不同的方法来检查对象可访问性信息：

  • 当发生事件时，例如焦点改变，鼠标移动，属性改变，菜单选择以及弹出菜单的显示

  • 当鼠标悬停在对象上方时按F1键，或者鼠标悬停在窗口上时按F2键

选择对象进行检查后， jaccessinspector工具显示对该对象调用Java Accessibility API方法的结果。

运行jaccessinspector工具

要使用jaccessinspector工具，启动jaccessinspector启动Java应用程序后的工具。推出jaccessinspector ，运行以下命令：

%JAVA_HOME%\bin\jaccessinspector.exe

现在，您打开了两个窗口：Java应用程序窗口和jaccessinspector窗口。的jaccessinspector窗口包含五个菜单：

  • 文件菜单

  • UpdateSettings菜单

  • JavaEvents菜单

  • 辅助功能事件菜单

  • 选项菜单

通过UpdateSettings ， JavaEvents和AccessibilityEvents菜单中的项目，您可以通过多种方式查询Java应用程序。

UpdateSettings菜单

本节介绍UpdateSettings菜单项。

从鼠标更新

确定鼠标的x和y坐标（假设jaccessinspector工具窗口位于最顶部），则当鼠标停止移动后，再向Java应用程序查询鼠标下方的可访问对象，并将输出转储到jaccessinspector窗口。

使用F2更新（鼠标HWND）

确定鼠标的x和y坐标（假设jaccessinspector工具窗口位于最顶部），然后在Java应用程序中查询鼠标下方HWND的可访问对象，并将输出转储到jaccessinspector窗口。

使用F1更新（鼠标点）

确定鼠标的x和y坐标（假设jaccessinspector工具窗口位于最顶部），然后在Java应用程序中查询光标下方的可访问对象，然后将输出转储到jaccessinspector窗口。

JavaEvents菜单

本节描述JavaEvents菜单项。

跟踪鼠标事件

向Java应用程序注册所有Java Mouse Entered事件，并在收到事件后查询由光标输入的对象，并将输出转储到jaccessinspector窗口。

注意：

如果鼠标快速移动，则更新显示的信息可能会有些延迟。
跟踪焦点事件

向Java应用程序注册所有Java Focus Gained事件，并在接收到事件后，查询接收到焦点的对象，并将输出转储到jaccessinspector窗口。

跟踪插入事件

向Java应用程序注册所有Java Caret Update事件，并在接收到事件后，查询在其中更新了插入符号的对象，并将输出转储到jaccessinspector窗口。

注意：

因为根据定义，包含插入符号的对象几乎是带格式文本对象的对象，所以这似乎不像其他事件跟踪选项那样敏感。在实际使用中，在“插入符号更新”情况下（例如，只需在插入符号位置获取新的字母，单词，句子），拨打较少的辅助功能呼叫，这将明显更快。
选择“轨道”菜单|取消选择取消的活动

向Java应用程序注册所有Menu事件，并在接收到事件后，查询插入符号已更新的对象，并将输出转储到jaccessinspector窗口。

可见的轨道弹出窗口|隐形取消的活动

向Java应用程序注册所有Popup Menu事件，并在接收到事件后，查询插入符号已更新的对象，并将输出转储到jaccessinspector窗口。

跟踪关闭事件

在Java应用程序终止时，向Java应用程序注册以接收属性更改事件。

辅助功能事件菜单

本节介绍AccessibilityEvents菜单项。

注意：

AccessibilityEvents菜单中列出的项目对于测试应用程序尤其是辅助技术应用程序最为重要。
曲目名称属性事件

向Java应用程序注册所有Java属性已更改事件，特别是在Name属性已更改的可访问对象上，并且在接收到事件后，将输出以及有关已更改属性的信息转储到滚动窗口中。

轨道描述属性事件

向Java应用程序注册所有Java属性更改事件，特别是在Description属性已更改的可访问对象上，并在接收到事件后将输出转储到jaccessinspector窗口，以及有关已更改属性的信息。

跟踪状态属性事件

向Java应用程序注册所有Java属性更改事件，特别是在State属性已更改的可访问对象上，并在接收到事件后将输出转储到jaccessinspector窗口，以及有关已更改属性的信息。

跟踪值属性事件

向Java应用程序注册所有Java属性更改事件，特别是在Value属性已更改的可访问对象上，并且在接收到事件后，将输出以及有关更改的属性的信息转储到滚动窗口中。

曲目选择属性事件

向Java应用程序注册所有Java属性更改事件，特别是在其中Selection属性已更改的可访问对象上，并在接收到事件后将输出转储到jaccessinspector窗口，以及有关已更改属性的信息。

跟踪文本属性事件

向Java应用程序注册所有Java属性更改事件，特别是在Text属性已更改的可访问对象上，并在接收到一个事件后，将输出转储到jaccessinspector窗口，以及有关已更改属性的信息。

跟踪插入号属性事件

向Java应用程序注册所有Java属性更改事件，特别是在其中插入符号属性已更改的可访问对象上，并在接收到事件后将输出转储到jaccessinspector窗口，以及有关已更改属性的信息。

跟踪VisibleData属性事件

向Java应用程序注册所有Java属性更改事件，特别是在其中VisibleData属性已更改的可访问对象上，并在接收到事件后将输出转储到jaccessinspector窗口，以及有关已更改属性的信息。

跟踪子代财产事件

向Java应用程序注册所有Java属性更改事件，特别是针对其中Child属性已更改的可访问对象，并在接收到事件后将输出转储到jaccessinspector窗口，以及有关已更改属性的信息。

跟踪活动后代属性事件

向Java应用程序注册所有Java属性更改事件，特别是在Active Descendent属性已更改的可访问对象上，并在接收到事件后将输出转储到jaccessinspector窗口，以及有关已更改属性的信息。

跟踪表模型更改属性事件

向Java应用程序注册所有“属性更改”事件，特别是在“表模型更改”属性已更改的可访问对象上，并在接收到事件后将输出转储到jaccessinspector窗口，以及有关已更改属性的信息。
