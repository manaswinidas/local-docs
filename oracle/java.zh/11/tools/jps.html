jps

您使用jps命令以列出目标系统上已检测的JVM。 此命令是实验性的，不受支持。

概要

jps [ -q ] [ -mlvV ][hostid ]
jps [ -help ]
-q

禁止输出类名，JAR文件名以及传递给的参数main方法，仅生成本地JVM标识符的列表。

-mlvV

  • -m显示传递给main方法。输出可能是null用于嵌入式JVM。

  • -l显示应用程序的完整软件包名称main类或应用程序JAR文件的完整路径名。

  • -v显示传递给JVM的参数。

  • -V禁止输出类名，JAR文件名以及传递给的参数main方法，仅生成本地JVM标识符的列表。

hostid

应为其生成过程报告的主机的标识符。的hostid可以包括指示通信协议，端口号和其他实现特定数据的可选组件。请参阅主机标识符 。

-help

显示有关jps命令。

描述

的jps该命令列出了目标系统上已检测到的Java HotSpot VM。该命令仅限于报告有关其具有访问权限的JVM的信息。

注意：

当附加到在单独的docker进程中运行的Java进程时，JDK 10添加了对使用Attach API的支持。然而jps工具无法查看在单独的docker实例中运行的JVM进程。如果您尝试将Linux主机与Docker容器中的虚拟机连接，则必须使用以下工具ps查找JVM的PID，然后在接受PID的工具的命令行上指定PID。

如果jps运行命令时未指定hostid ，然后在本地主机上搜索已检测的JVM。如果以hostid ，然后使用指定的协议和端口在指示的主机上搜索JVM。一种jstatd假定进程正在目标主机上运行。

的jps命令报告本地JVM标识符，或者lvmid ，用于在目标系统上找到的每个已检测的JVM。的lvmid通常是（但不是必须）JVM进程的操作系统进程标识符。没有选择， jps命令列出每个Java应用程序的lvmid然后是应用程序的类名或jar文件名的缩写。类名或JAR文件名的缩写形式省略了类的包信息或JAR文件路径信息。

的jps该命令使用Java启动器来查找传递给main方法的类名和参数。如果目标JVM是使用自定义启动器启动的，则为类或JAR文件名，以及main方法不可用。在这种情况下， jps命令输出字符串Unknown用于类名或JAR文件名，以及用于main方法。

由JVM生成的JVM列表jps命令可以受到授予运行该命令的主体的权限的限制。该命令仅列出由特定于操作系统的访问控制机制确定的主体对其具有访问权限的JVM。

主机标识符

主机标识符，或hostid ，是指示目标系统的字符串。的语法hostid字符串对应于URI的语法：

[protocol:][[//]hostname][:port][/servername]
protocol

通信协议。如果protocol被省略，一个hostname未指定，则默认协议是特定于平台的优化本地协议。如果省略了协议并指定了主机名，则默认协议为rmi 。

hostname

指示目标主机的主机名或IP地址。如果您省略hostname参数，则目标主机为本地主机。

port

与远程服务器通信的默认端口。如果hostname参数被省略或protocol参数指定优化的本地协议，然后port参数将被忽略。否则，治疗port参数是特定于实现的。对于默认rmi协议port参数指示端口号rmiregistry在远程主机上。如果port参数被省略，并且protocol参数指示rmi ，然后是默认值rmiregistry港口 （ 1099 ） 用来。

servername

此参数的处理方式取决于实现方式。对于优化的本地协议，此字段将被忽略。为了rmi协议，此参数是一个字符串，代表远程主机上RMI远程对象的名称。参见jstatd命令-n选项。

jps命令的输出格式

输出jps命令具有以下模式：

lvmid [ [ classname | JARfilename | "Unknown"] [ arg* ] [ jvmarg* ] ]

所有输出令牌都由空格分隔。一个arg尝试将参数映射到其实际位置参数时，包含嵌入式空格的值引入了歧义。

注意：

建议您不要编写脚本来解析jps输出，因为格式可能在将来的版本中更改。如果您编写可解析的脚本jps输出，然后期望对其进行修改以用于此工具的将来版本。

例子

本节提供了jps命令。

列出本地主机上已检测的JVM：

jps
18027 Java2Demo.JAR
18032 jps
18005 jstat

以下示例列出了远程主机上的检测JVM。本示例假设jstat服务器及其内部RMI注册表或单独的外部rmiregistry进程正在默认端口（端口）上的远程主机上运行1099 ）。它还假定本地主机具有访问远程主机的适当权限。此示例包括-l选项输出类名或JAR文件名的长格式。 

jps -l remote.domain
3002 /opt/jdk1.7.0/demo/jfc/Java2D/Java2Demo.JAR
2857 sun.tools.jstatd.jstatd

以下示例列出了具有RMI注册表非默认端口的远程主机上的检测JVM。本示例假设jstatd服务器，内部RMI注册表绑定到端口2002 ，正在远程主机上运行。本示例还使用了-m包含传递给的参数的选项main列出的每个Java应用程序的方法。 

jps -m remote.domain:2002
3002 /opt/jdk1.7.0/demo/jfc/Java2D/Java2Demo.JAR
3102 sun.tools.jstatd.jstatd -p 2002
