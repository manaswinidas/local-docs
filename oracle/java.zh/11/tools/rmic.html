Rmic

您使用rmic编译器使用Java远程方法协议（JRMP）生成存根和骨架类文件。

概要

rmic [ options ] package-qualified-class-names
options

这代表命令行options为了rmic编译器。请参阅rmic编译器的选项 。

package-qualified-class-names

包含其软件包的类名称，例如， java.awt.Color 。

描述

注意：

不支持静态生成Java远程方法协议（JRMP）存根和框架。Oracle建议您改用动态生成的JRMP存根，这样就无需在基于JRMP的应用程序中使用此工具。

的rmic编译器使用JRMP生成存根和骨架类文件。

注意：

Rmic编译器已更新，可以删除-idl和-iiop选项，并且不再生成IDL或IIOP存根和联系类。

JRMP类文件是从作为远程对象实现类的已编译Java编程语言类生成的。远程实现类是实现接口的类java.rmi.Remote 。中的类名称rmic该命令必须适用于已成功编译的类javac命令，并且必须完全符合包装要求。例如，运行rmic关于类文件名的命令HelloImpl如这里所示创建HelloImpl_Stub.class文件在hello子目录（以类的包命名）：

rmic hello.HelloImpl

远程对象的框架是JRMP协议服务器端实体，该实体具有一种将调用调度到远程对象实现的方法。

存根是远程对象的客户端代理，负责将远程对象上的方法调用传递到实际的远程对象实现所驻留的服务器。因此，客户端对远程对象的引用实际上是对本地存根的引用。

默认情况下， rmic命令生成仅使用1.2 JRMP存根协议版本的存根类，就像-v1.2选项已指定。请参阅rmic编译器的选项 。

存根仅实现远程接口，而不实现远程对象也实现的本地接口。因为JRMP存根实现了与远程对象相同的一组远程接口，所以客户端可以使用Java编程语言内置的运算符进行转换和类型检查。

Rmic编译器的选项

-bootclasspath path

覆盖引导类文件的位置。

-classpath path

指定路径rmic命令用于查找类。此选项将覆盖默认值或CLASSPATH设置时的环境变量。目录由冒号或分号分隔，具体取决于您的操作系统。以下是通用格式path ：

  • Oracle Solaris，Linux和macOS ： .:your_path ， 例如：.:/usr/local/java/classes

  • 视窗： .;your_path ， 例如： .;/usr/local/java/classes

-d directory

指定生成的类层次结构的根目标目录。您可以使用此选项为存根，框架和领带文件指定目标目录。

  • Oracle Solaris，Linux和macOS ：例如，以下命令将存根和框架类派生自MyClass进入目录/java/classes/exampleclass ：

    rmic -d /java/classes exampleclass.MyClass

  • Windows：例如，以下命令放置从中派生的存根和骨架类MyClass进入目录C:\java\classes\exampleclass ：

    rmic -d C:\java\classes exampleclass.MyClass

如果-d未指定选项，则默认行为类似于-d已指定。目标类的包层次结构在当前目录中创建，并且stub / tie / skeleton文件位于其中。

-g

启用所有调试信息的生成，包括局部变量。默认情况下，仅生成行号信息。

-Jargument

与任何Java命令一起使用， -J选项通过argument紧随其后（ -J和参数）传递给Java解释器。

-keep要么-keepgenerated

保留生成的.java stub，skeleton和tie类的源文件，并将它们写入与.class文件。

-nowarn

关闭警告。当。。。的时候-nowarn使用options时，编译器不会输出警告。

-nowrite

不将已编译的类写入文件系统。

-vcompat （已弃用）

生成与1.1和1.2 JRMP存根协议版本兼容的存根和骨架类。在5.0之前的版本中，此选项是默认选项。生成的存根类在JDK 1.1虚拟机中加载时使用1.1存根协议版本，在加载到1.2（或更高版本）虚拟机中时使用1.2存根协议版本。生成的框架类同时支持1.1和1.2存根协议版本。生成的类相对较大，以支持两种操作模式。

注意：

此选项已被弃用。请参阅说明 。

-verbose

使编译器和链接器打印有关正在编译哪些类和正在加载哪些类文件的消息。

-v1.1 （已弃用）

仅为1.1 JRMP存根协议版本生成存根和骨架类。的-v1.1该选项仅用于生成与现有的，静态部署的存根类序列化兼容的存根类，该存根类由rmic JDK 1.1中的命令无法升级（并且不使用动态类加载）。

注意：

此选项已被弃用。请参阅说明 。

-v1.2 （已弃用）

（默认）仅针对1.2 JRMP存根协议版本生成存根类。没有生成骨架类，因为骨架类未与1.2存根协议版本一起使用。将生成的存根类加载到JDK 1.1虚拟机中后将无法使用。

注意：

此选项已被弃用。请参阅说明 。

环境变量

CLASSPATH

用于向系统提供用户定义的类的路径。

  • Oracle Solaris，Linux和macOS ：目录之间用冒号分隔，例如： .:/usr/local/java/classes 。

  • Windows：目录之间用冒号分隔，例如： .;C:\usr\local\java\classes 。
