7常见迁移问题和解决方案

本主题描述了从Oracle JRockit迁移到HotSpot时可能发生的一些常见问题及其解决方案。

下表列出了在迁移过程中可能会发生的一些常见问题及其解决方案：

表7-1迁移问题和解决方案

问题 Oracle JRockit选件 热点选项 笔记

迁移到JDK 7后性能下降。

该问题已通过使用解决-XX:ReservedCodeCacheSize=1g

-XX:+ReserveCodeMemory

默认值：

  • 使用时-XX:+UseLargePagesForCode ：64 MB

  • 使用时-XX:-UseLargePagesForCode ：1024 MB

-XX:ReservedCodeCacheSize

大多数平台上的默认值为48 MB。

使用HotSpot VM，可以发现在某些情况下， ReservedCodeCacheSize值，例如-XX:ReservedCodeCacheSize=1g ，大大提高了性能。

切换到HotSpot后观察到的锁定/解锁事件增加。

禁用UseBiasedLocking帮助改善了整体表现。

-XX:-UseLazyUnlocking （禁用）

-XX:-UseBiasedLocking （禁用）

的UseBiasedLocking选项提高了无竞争同步的性能。默认情况下启用此选项。

但是，如果应用程序具有较高的竞争性同步，则禁用UseBiasedLocking有利于性能。

如果在迁移到HotSpot后由于锁定或同步而遇到性能问题，则禁用此选项可能会带来一些性能提升。
