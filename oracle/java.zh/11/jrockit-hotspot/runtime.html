本主题描述了控制HotSpot的运行时行为的重要选项。

运行时选项

下表列出了Oracle JRockit和HotSpot中运行时子系统的重要等效选项：

表3-1运行时选项

甲骨文JRockit 热点 笔记

-XX:+UseLazyUnlocking

-XX:+UseBiasedLocking

UseBiasedLocking提高了无竞争同步的性能。默认情况下启用此选项。但是，如果应用程序具有较高的竞争同步性，请禁用UseBiasedLocking增强性能的选项。

-XlargePages

-XX:+UseLargePages

在HotSpot中，默认情况下在Solaris平台上启用此选项。在Linux平台上，此选项从版本7u60开始被禁用。

使用-XX:+UseLargePages以便在默认情况下处于禁用状态的平台上启用大页面。然而， -XX:+UseLargePages不允许在MetaSpace中使用大页面。要启用此选项，请添加-XX:+UseLargePagesInMetaspace 。

-XX:MaxLargePageSize

-XX:LargePageSizeInBytes=size

设置用于Java堆的大页面的最大大小（以字节为单位）。默认情况下，大小设置为0，这意味着JVM自动为大页面选择大小。

-XXcompressedRefs

-XX:+UseCompressedOops

用于Compressed Oops是64位HotSpot进程的默认值-Xmx未指定，且的值-Xmx小于32 GB。
