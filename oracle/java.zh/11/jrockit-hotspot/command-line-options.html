将Oracle JRockit映射到HotSpot命令行选项

某些Oracle JRockit命令行选项类似于HotSpot选项。

本节提供了Oracle JRockit选项到HotSpot选项的一对一映射，或者使您参考了本文档的其他部分。有某些Oracle JRockit选项没有对应的HotSpot JVM选项。另外，某些映射的HotSpot选项与Oracle JRockit选项并不完全等效，并且可能在HotSpot上提供略有不同的行为。

迁移时，不建议仅将与Oracle JRockit一起使用的每个选项转换为类似的HotSpot选项。特别是对于与性能相关的选项，最佳实践是从仅指定Java堆大小和垃圾收集器（例如CMS或G1）开始。如有必要，必须根据使用HotSpot完成的新基准测试和性能分析完成HotSpot的任何其他调整。不建议假定针对Oracle JRockit配置做出的大多数JVM级调整决定（如果有的话）也将适用于HotSpot配置。

-XX:+AggressiveHeap启用Java堆优化。根据计算机（RAM和CPU）的配置，这会将各种参数设置为最适合长时间运行且需要大量内存分配的作业的参数。默认情况下，该选项处于禁用状态，并且未优化堆。

-XX:+AggressiveOpts启用其他与堆无关的优化。
