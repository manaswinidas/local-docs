热点记录选项

这些是HotSpot中可用的一些常见日志记录选项，可用于为HotSpot JVM中的特定子系统启用诊断输出。

表5-2日志记录选项

热点记录选项 笔记
-Xlog

为所有JVM组件启用通用日志记录系统。

-Xloggc:
设置必须将详细的GC事件信息重定向到该文件以进行记录的文件。写入此文件的信息类似于-verbose:gc以及从每个记录的事件之前的第一个GC事件开始经过的时间。的-Xloggc选项覆盖-verbose:gc ，如果两者都具有相同的java命令。

注意：

JDK 9中不推荐使用此选项。

-XX:LogFile=

设置写入日志数据的路径和文件名。

-XX:+LogCompilation

启用将编译活动记录到名为的文件中hotspot.log在当前工作目录中。您可以使用以下命令指定其他日志文件路径和名称-XX:LogFile选项。的-XX:+LogCompilation选项必须与-XX:UnlockDiagnosticVMOptions用于解锁诊断JVM选项的选项。

-XX:+PrintCommandLineFlags

启用打印出现在命令行上的选定JVM标志。

-XX:+PrintGC
启用在每个GC上打印消息的功能。

注意：

JDK 9中不推荐使用此选项。

-XX:+PrintGCDetails
启用在每个GC上打印详细消息的功能。

注意：

JDK 9中不推荐使用此选项。

-XX:+PrintNMTStatistics

启用本机内存跟踪时，启用在JVM出口处打印收集的本机内存跟踪数据。

-XX:+PrintNMTStatistics

启用本机内存跟踪时，启用在JVM出口处打印收集的本机内存跟踪数据。

-XX:+PrintAssembly

启用通过使用外部JIT编译Java字节码产生的汇编代码的打印功能disassembler.so图书馆。此选项使您可以查看生成的代码，从而帮助您诊断性能问题。此选项必须与-XX:UnlockDiagnosticVMOptions用于解锁诊断JVM选项的选项。

-XX:+PrintCompilation

每次编译方法时，通过将消息打印到控制台来启用JVM的详细诊断输出。

-XX:+PrintInlining

启用内联决策的打印。此选项使您可以查看要内联的方法。

-XX:+PrintClassHistogram

在Control + C事件（SIGTERM）之后启用类实例直方图的打印。默认情况下，此选项是禁用的。

-XX:+PrintConcurrentLocks

允许打印java.util.concurrent在Control + C事件（SIGTERM）之后锁定。默认情况下，此选项是禁用的。

表5-3 GC日志记录选项

GC记录选项 笔记
-Xlog:gc

启用在每个GC上打印消息的功能。的gc是记录所有GC相关信息的主要标记。的gc标签与其他标签结合使用以记录特定信息。以下行中列出了一些标签。

-Xlog:gc+region=trace

启用有关已分配区域以及G1收集器回收的区域的信息的打印。

-Xlog:gc+ergo*=trace

允许打印有关自适应发电规模的信息。

-Xlog:safepoint

启用从上次暂停（例如，GC暂停）开始经过的时间的打印。

-Xlog:gc+task=trace

为每个单独的GC工作线程任务启用时间戳打印。

-Xlog:gc::uptime,tid

使用装饰器可以在每个GC上打印时间戳uptime和tid 。

-Xlog:gc+stringdedup

打印详细的重复数据删除统计信息。

-Xlog:gc+age=trace

启用打印保有权年龄信息。

-Xlog:gc*

启用在每个GC上打印详细消息的功能。

-Xlog:gc:file=

记录带有gc标记为指定的文件名。例如选项-Xlog:gc:file=gc.txt将消息记录到gc.txt文件。

请参阅《 Java平台，标准版工具参考》指南中的使用JVM统一日志记录框架启用日志记录 。
