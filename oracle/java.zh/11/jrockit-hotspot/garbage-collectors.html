2个垃圾收集器

本主题描述了Oracle JRockit和HotSpot中可用的垃圾收集调整选项，并比较了它们的功能和性能。

调优垃圾回收

下表列出了Oracle JRockit和HotSpot中可用的重要垃圾收集（GC）调整选项：

表2-1垃圾收集器

Oracle JRockit垃圾收集器 热点垃圾收集器 笔记

使用以下任何选项设置吞吐量收集器：

  • -Xgc:throughput

  • -Xgc:genpar

  • -Xgc:singlepar

  • -Xgc:parallel

吞吐量收集器：

—XX:+UseParallelGC -对年轻人和老年人使用平行压实收集器

-XX:+UseParallelGC -XX:-UseParallelOldGC -对年轻一代使用并行压缩收集器，对于老一代使用串行标记扫描

 可以使用以下命令控制并行GC线程的数量

-XX:ParallelGCThreads=n 。

使用以下任一选项设置低延迟收集器：

-Xgc:pausetime

-Xgc:gencon

-Xgc:singlecon

 -XX:+UseG1GC

要么

-XX:+UseConcurrentMarkSweepGC

注意：

CMS收集器在JDK 9中已弃用。

HotSpot提供了两个主要同时存在的收集器之间的选择：

  • 垃圾优先（G1）垃圾收集器是服务器型收集器，用于具有大内存的多处理器计算机。它在实现高吞吐量的同时极有可能满足垃圾收集暂停时间目标
  • 并发标记扫描（CMS）收集器适用于那些希望更短的垃圾收集暂停并能够与垃圾收集共享处理器资源的应用程序

垃圾优先垃圾收集器是默认的收集器。
要了解HotSpot随附的各种垃圾收集器，请参阅《 Java平台中的 垃圾收集调整简介 》，《标准版HotSpot虚拟机垃圾收集调整指南》 。
