1 Java SE监视和管理概述

本主题介绍为Java平台标准版（Java SE平台）提供监视和管理服务的功能和实用程序。

关键监控和管理功能

Java SE平台包含重要的监视和管理功能。这些功能分为四大类：

  • Java虚拟机检测

  • 监控和管理API

  • 监控和管理工具

  • Java管理扩展技术

Java虚拟机检测

Java虚拟机（Java VM）用于监视和管理，启用了内置（或ready-to-use ）管理功能，可以远程和本地访问这些功能。

请参阅使用JMX技术进行监视和管理 。

Java VM包括一个平台MBean服务器和平台MBean，供符合Java管理扩展（JMX）规范的管理应用程序使用。这些平台是监视和管理API的实现。 平台MXBean和平台MBean服务器主题中介绍了平台MXBean和MBean服务器 。

监控和管理API

Java SE包括以下用于监视和管理的API：

  • java.lang.management ：启用监视和管理Java虚拟机及其底层操作系统的功能。该API使应用程序能够进行自我监视，并使符合JMX的工具能够在本地和远程监视和管理虚拟机。通过此API，可以访问以下类型的信息：

    • 加载的类数和正在运行的线程

    • Java VM正常运行时间，系统属性和VM输入参数

    • 线程状态，线程竞争统计信息和活动线程的堆栈跟踪

    • 内存消耗

    • 垃圾收集统计

    • 低内存检测

    • 按需死锁检测

    • 操作系统信息

  • java.util.logging。LoggingMXBean ：使您能够检索和设置日志记录信息。

  • 附加 ：允许将管理代理动态加载到虚拟机上。

  • JConsole ：提供访问JConsole的编程接口，例如添加JConsole插件。JTop是JConsole插件的示例，可从以下位置获得： JDK_HOME/demo/management/JTop目录。

监控和管理工具

Java SE平台提供了一个称为JConsole的图形监视工具。JConsole实现了JMX API，并使您能够监视Java VM和任何检测到的应用程序的性能。它提供信息以帮助您优化性能。

JConsole中的一些增强功能如下：

  • JConsole插件支持，它允许您构建自己的插件以与JConsole一起运行。例如，您可以添加一个自定义选项卡来访问应用程序的MBean。

  • 动态附加功能允许您将JConsole连接到支持附加API的任何应用程序。

  • 增强的用户界面，使数据更易于访问。

  • 新的概述和VM摘要选项卡可更好地呈现有关Java VM的常规信息。

  • HotSpot Diagnostic MBean，它提供API以在运行时请求堆转储，还可以更改某些VM选项的设置。

  • 改进了MBean的表示方式，以使其更易于访问MBean的操作和属性。

在“ 使用JConsole”主题中详细介绍了JConsole 。

Java SE平台还提供了其他命令行工具。

Java管理扩展技术

Java SE平台版本11包含Java管理扩展（JMX）规范1.4版。JMX API允许您对应用程序进行检测以进行监视和管理。远程方法调用（RMI）连接器使该仪器可以远程访问，例如，使用JConsole。

请参阅《 Java平台标准版Java管理扩展指南》中的JMX技术文档 。

以下各节简要介绍了JMX API的主要组件。

什么是MBean？

JMX技术MBean是managed beans ，即表示要管理的资源的Java对象 。MBean的management interface包含以下内容：

  • 可以读取和写入的命名和类型化属性。

  • 可以调用的命名和类型操作。

  • MBean可以发出的类型化通知。

例如，表示应用程序配置的MBean可以具有表示不同配置参数的属性，例如CacheSize 。阅读CacheSize属性将返回缓存的当前大小。写作CacheSize更新缓存的大小，可能会更改正在运行的应用程序的行为。诸如save持久存储当前配置。MBean可以发送诸如ConfigurationChangedNotification当配置更改时。

MBean可以是标准的也可以是动态的。标准MBean是符合从JavaBeans组件模型派生的设计模式的Java对象。动态MBean在运行时定义其管理接口。另一种类型的MBean，称为MXBean，已添加到Java平台。

  • standard MBean通过其属性和操作公开要直接管理的资源。通过以下方式公开属性getter和setter方法。操作是管理人员可用的该类的其他方法。所有这些方法都是在MBean接口中静态定义的，并且对于JMX代理通过自省可见。此方法是使新资源易于管理的最直接方法。

  • dynamic MBean是在运行时定义其管理接口的MBean。例如，配置MBean通过解析XML文件来确定其公开的属性的名称和类型。

  • MXBean是一种MBean，它提供了一种简单的方法来编写仅引用预定义类型集的MBean。这样，您可以确保任何客户端都可以使用MBean。它包括远程客户端，而不要求客户端有权访问特定于模型的类，这些类代表您的MBean的类型。平台MBean都是MXBean。

MBean服务器

为了有用，必须在MBean服务器中注册MBean。MBean服务器是MBean的存储库。每个MBean在MBean服务器中都注册有唯一的名称。通常，对MBean的唯一访问是通过MBean服务器。换句话说，代码不会直接访问MBean，而是通过MBean服务器按名称访问MBean。

Java SE平台包括一个内置的平台MBean服务器。请参阅使用平台MBean服务器和平台MXBean 。

平台MBean服务器

平台MBean服务器可以由在同一Java VM中运行的不同托管组件共享。您可以使用以下方法访问平台MBean服务器ManagementFactory.getPlatformMBeanServer() 。对该方法的第一次调用将创建平台MBean服务器，并使用其唯一的对象名称注册平台MXBean。随后，此方法返回最初创建的平台MBeanServer实例。

动态创建和销毁的MXBean（例如，内存池和管理器）将自动在平台MBean服务器中注册和注销。如果系统属性javax.management.builder.initial设置后，将通过指定的方式创建平台MBean服务器MBeanServerBuilder参数。

您可以使用平台MBean服务器注册除平台MXBean之外的其他MBean。这使所有MBean都可以通过同一MBean服务器发布，并使网络发布和发现更加容易。
