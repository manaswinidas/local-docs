样本信任库

该捆绑软件包括与示例代码一起使用的示例密钥库文件。它们以“ JKS”密钥库格式存储，这是Oracle JDK实现使用的默认格式。（如果需要其他密钥库格式，则需要将JDK配置为识别新的默认格式。）

JSSE使用以下证书密钥库文件来验证客户端和服务器。

  • */testkeys

    这些文件被代码示例用作公用/专用密钥和证书材料的来源。在客户端程序目录中， testkeys文件包含Java吉祥物的证书条目Duke 。在服务器程序目录中（ ./sockets/server和rmi ），该文件包含服务器的证书条目localhost 。

    该示例代码期望testkeys文件位于当前工作目录中。

    注意：

    这些是非常简单的证书，不适用于生产环境，但是它们足以在此处运行示例。

    这些密钥库的密码为：passphrase

  • samplecacerts

    该信任库文件与库存JDK非常相似cacerts文件，其中包含来自多个供应商的信任证书。它还包含来自上述Duke和localhost的受信任证书。

    此密钥库的密码与JDK相同cacert的初始密码：changeit

    请参阅提供商的文档，以了解如何配置受信任证书文件的位置。

    注意：

    JDK的用户可以使用以下方法之一指定信任库的位置：

    1. 系统属性：

      java -Djavax.net.ssl.trustStore=samplecacerts \
     -Djavax.net.ssl.trustStorePassword=changeit Application

    2. 将文件安装到：

      <java-home>/lib/security/jssecacerts

    3. 将文件安装到：

      <java-home>/lib/security/cacerts

    如果选择（2）或（3），请确保将此文件替换为生产文件cacerts部署前文件。

实用程序keytool可用于生成备用证书和密钥库文件。

注意：

确保您验证了cacerts文件。由于您信任CA中的cacerts文件作为实体以用于向其他实体签名和颁发证书，您必须管理cacerts仔细归档。的cacerts文件应仅包含您信任的实体和CA的证书。您有责任验证捆绑在其中的受信任的根CA证书。 cacerts归档并做出自己的信任决定。从以下位置删除不受信任的CA证书： cacerts文件，请使用keytool命令。你可以找到cacerts jre安装目录中的文件。如果您没有编辑此文件的权限，请与系统管理员联系。
