使用JAAS登录实用程序

前面的两个教程JAAS Authentication和JAAS Authorization展示了如何使用LoginContext和Subject类将程序编写为

  • 验证用户身份以验证其身份，并
  • 将表示用户的特定类的实例（ 主题实例）与访问控制上下文相关联，这样，如果当前策略向用户授予所需的权限，则将允许后续的安全敏感操作。

本教程描述了一个登录实用程序，该实用程序执行上述操作，然后以经过身份验证的用户身份执行任何指定的应用程序。

本教程演示了如何将Login实用程序与示例应用程序一起使用。下一个教程“ 使用JAAS登录实用程序和Java GSS-API进行安全消息交换” （使用Java GSS-API的客户端/服务器应用程序）也将使用“登录”实用程序。

在阅读前两个教程之前，没有必要先阅读有关JAAS身份验证和授权的两个教程。但是，您可能希望参考这些教程中的某些部分，以获得有关某些主题（例如JAAS Authorization）的更多详细信息。您还应该阅读附录B：JAAS登录配置文件，以获取有关什么是登录配置文件的信息，因为本教程和本系列所有其他教程都需要一个。

与本系列教程中的所有教程一样，用于支持身份验证的基础技术是Kerberos。请参见Kerberos要求 。

您需要了解的有关登录实用程序的信息

您不需要了解其中包含的代码Login.java ;您可以按原样使用它。但是，您需要了解有关其功能的一些事实，以便您的程序，策略文件和登录配置文件可以正常使用。以下是这些事实的摘要，随后是带有更多信息和示例的部分。

的Login该类执行以下操作：

  • 假定将它作为应用程序的顶级类名称作为参数传递，然后是应用程序可能需要的任何参数。
  • 假定顶级应用程序类的类名也用作要在登录配置文件中查找的条目的名称。
  • 指定TextCallbackHandler类（从com.sun.security.auth.callback包）作为与用户交流时要使用的类。此类可以提示用户输入用户名和密码。
  • 使用LoginContext验证用户。LoginContext调用适当的身份验证技术或LoginModule来执行身份验证。LoginModules根据需要使用CallbackHandler （在我们的示例中为TextCallbackHandler ）与用户进行通信。
  • 允许用户尝试三次成功登录。
  • 创建一个实例MyAction类（也在Login.java ），并向其传递应用程序参数（如果有）。
  • 调用Subject.doAsPrivileged ，将代表用户的主题传递给MyAction实例，以及一个空的AccessControlContext 。结果是，您的应用程序中的public static main方法被调用，并且您的应用程序代码被视为代表用户执行。

申请书和其他文件要求

要使用登录实用程序来验证用户身份并执行您的应用程序，您可能需要对登录配置文件和策略文件进行少量添加或修改，如下所述。

  • 申请条件
  • 登录配置文件要求
  • 策略文件要求

申请条件

为了利用Login实用程序，您的应用程序代码不需要任何特殊的东西。您所需要做的就是使应用程序的入口成为main照常编写类的方法。

调用Login的方式，以便它将对用户进行身份验证然后实例化MyAction调用您的应用程序如下：

java <options> Login <AppName> <app arguments>

哪里是应用程序的顶级类名称，并且是应用程序所需的任何参数。有关用于本教程的完整命令，请参见使用Login Utility运行示例程序 。

登录配置文件要求

每当使用LoginContext验证用户身份时，就需要一个登录配置文件来指定所需的登录模块。有关什么是登录配置文件及其包含的内容的更多信息，请参见JAAS身份验证教程中的“登录配置”部分。

使用登录实用程序时，登录配置文件条目的名称必须与顶级应用程序类名称完全相同。有关示例，请参见本教程中的“登录配置文件” 。

策略文件要求

每当使用安全管理器运行应用程序时，都需要一个策略来指示授予特定代码或特定用户（或多个用户）正在执行的特定代码的权限。指定策略的一种方法是grant策略文件中的语句。有关更多信息，请参见策略文件 。

如果使用Login实用程序来调用您的应用程序，则需要向其授予各种权限，如Login和MyAction类所需的权限中所述 。

示例应用程序

的Sample.java本教程使用的应用程序执行的操作与SampleAction.java该应用程序在上一期（ JAAS授权 ）教程中做了。它执行以下操作：

  • 读取并打印java.home系统属性，
  • 读取并打印user.home系统属性，以及
  • 确定是否将文件命名为foo.txt存在于当前目录中。

登录配置文件

的sample.conf本教程的登录配置文件包含一个条目，就像上一个（ JAAS Authorization ）教程的登录配置文件一样。输入内容是相同的，因为在两种情况下实现所需身份验证技术的类都是Krb5LoginModule 。 com.sun.security.auth.module包。

唯一的区别是用于输入的名称。在上一个教程中，我们使用名称“ JaasSample”，因为这是JaasAzn类用于查找条目的名称。当您在应用程序中使用Login实用程序时，它期望您的登录配置文件条目的名称与顶级应用程序类的名称相同。本教程的该应用程序类名为“样本”，因此它也必须是登录配置文件条目的名称。因此，登录配置文件如下所示：

Sample {
   com.sun.security.auth.module.Krb5LoginModule required;
};

“必需”表示必须成功使用Krb5LoginModule登录，才能使身份验证成功。仅当成功使用用户提供的名称和密码将用户登录到Kerberos KDC时， Krb5LoginModule才会成功。

请参阅Krb5LoginModule Javadoc API文档，以获取有关可以传递给Krb5LoginModule的所有可能选项的信息。

政策档案

Login ， MyAction和Sample类均执行一些对安全性敏感的操作，因此策略文件中需要相关权限， sample.policy ，以便执行操作。

Login和MyAction类所需的权限

对于本教程，您将创建一个Login.jar包含Login.class和MyAction.class文件。您需要授予Login.jar各种权限，特别是调用安全敏感方法所需的权限Login.jar类调用，以及应用程序所需的所有权限。否则，访问控制检查将失败。

我们建议的最简单的方法是授予Login.jar AllPermission 。对于本教程， Login.jar假定文件位于当前目录中，并且策略文件包括以下内容：

grant codebase "file:./Login.jar" {
   permission java.security.AllPermission;
};

样本所需的权限

（注意：本节实质上是上一期（ JAAS授权 ）教程中SampleAction所需权限的修改后的副本，因为Sample和SampleAction执行相同的操作，因此需要相同的权限。）

的Sample代码执行需要权限的三个操作。它

  • 读取java.home系统属性。
  • 读取user.home系统属性。
  • 检查以查看是否有一个名为foo.txt存在于当前目录中。

这些操作所需的权限如下：

permission java.util.PropertyPermission "java.home", "read";
permission java.util.PropertyPermission "user.home", "read";
permission java.io.FilePermission "foo.txt", "read";

我们需要向这些代码授予这些权限Sample.class ，我们将其放置在一个名为Sample.jar 。但是，我们的grant语句将不仅向code授予权限，还向执行code的特定经过身份验证的用户授予权限。这说明了如何在grant语句，用于将代码中对安全性敏感的操作的执行限制为特定用户，而不是向执行代码的所有用户授予权限。

因此，如JAAS授权中所述 ，我们的grant语句如下所示：

grant codebase "file:./Sample.jar",
    Principal javax.security.auth.kerberos.KerberosPrincipal 
        "your_user_name@your_realm"  {

   permission java.util.PropertyPermission "java.home", "read";
   permission java.util.PropertyPermission "user.home", "read";
   permission java.io.FilePermission "foo.txt", "read";
};

注意：

重要提示：您必须用Kerberos用户名（用“ @”和领域完整）替换为your_user_name@your_realm 。

例如，如果您的用户名是mjones而你的境界是KRBNT-OPERATIONS.ABC.COM ，您将使用mjones@KRBNT-OPERATIONS.ABC.COM 。

使用登录实用程序运行示例程序

执行Sample登录实用程序的应用程序，请执行以下操作：

  1. 将以下文件放入目录：
    • 的Login.java源文件。
    • 的Sample.java源文件。
    • 的sample.conf登录配置文件。
    • 的sample.policy策略文件。
  2. 更换your_user_name@your_realm在sample.policy与您的用户名和领域。
  3. 编译Login.java和Sample.java ：
    javac Login.java Sample.java

    注意Login.java包含两个类，因此进行编译Login.java创造Login.class和MyAction.class 。

  4. 创建一个名为的JAR文件Login.jar包含Login.class and MyAction.class ：
    jar -cvf Login.jar Login.class MyAction.class
  5. 创建一个名为的JAR文件Sample.jar包含Sample.class ：
    jar -cvf Sample.jar Sample.class
  6. 执行Login类，指定
    • 通过适当的-classpath子句，应该在Login.jar和Sample.jar JAR文件，
    • 通过-Djava.security.manager应该安装安全管理器，
    • 通过-Djava.security.krb5.realm=您的Kerberos领域是指定的领域。
    • 通过-Djava.security.krb5.kdc=您的Kerberos KDC是指定的那个。
    • 通过-Djava.security.policy=sample.policy要使用的策略文件是sample.policy和
    • 通过-Djava.security.auth.login.config=sample.conf要使用的登录配置文件是sample.conf 。

    您传递应用程序的名称（在这种情况下， Sample ）作为Login 。然后，您可以将应用程序所需的所有参数添加为参数，但是在我们的情况下Sample不需要任何。

    以下是可用于Windows，Solaris，Linux和macOS的完整命令。唯一的区别是，在Windows上，您使用分号来分隔类路径项，而在Solaris，Linux和macOS上，则使用冒号来实现此目的。 请务必更换与您的Kerberos领域，以及与您的Kerberos KDC。

    这是Windows的完整命令：

    java -classpath Login.jar;Sample.jar 
 -Djava.security.manager 
 -Djava.security.krb5.realm=<your_realm> 
 -Djava.security.krb5.kdc=<your_kdc> 
 -Djava.security.policy=sample.policy 
 -Djava.security.auth.login.config=sample.conf Login Sample

    这是针对Solaris，Linux和macOS的完整命令：

    java -classpath Login.jar:Sample.jar 
 -Djava.security.manager 
 -Djava.security.krb5.realm=<your_realm> 
 -Djava.security.krb5.kdc=<your_kdc> 
 -Djava.security.policy=sample.policy 
 -Djava.security.auth.login.config=sample.conf Login Sample

    在一行上键入完整的命令。为了清晰起见，此处使用多行。如果该命令对于您的系统而言太长，则可能需要将其放置在.bat文件（对于Windows）或.sh文件（对于Solaris，Linux和macOS）中，然后运行该文件以执行命令。

    系统将提示您输入Kerberos用户名和密码，并且在登录配置文件中指定的基础Kerberos登录模块将使您登录Kerberos。身份验证成功完成后， Sample代码将代表您（用户）执行。的sample.policy策略文件会授予您所需的权限，因此您将看到您的值的显示java.home和user.home系统属性以及关于您是否具有名为的文件的声明foo.txt在当前目录中。

    有关登录故障排除的建议，请参阅故障排除 。
