第一部分：使用Java身份验证和授权服务（JAAS）的安全身份验证

练习1：使用JAAS API

本练习的目的

本练习的目的是学习如何使用Java身份验证和授权（JAAS）API进行身份验证。

本练习的背景

JAAS为Java平台提供了一个标准的可插入身份验证框架（PAM）。应用程序使用JAAS API进行身份验证 -验证使用该应用程序的用户的身份并将其身份信息收集到称为主题的容器中的过程。然后，应用程序可以使用主题中的身份信息以及JAAS API来做出授权决策，以决定是否允许已认证的用户访问受保护的资源或执行受限制的操作。本练习演示了JAAS认证。它不显示JAAS授权。

  • JAAS教程

遵循的步骤

  • 阅读Jass.java示例代码。该代码执行以下任务：

    1. 定义一个回调处理程序或使用预定义的处理程序。
    2. 创建一个LoginContext ，其名称标识要使用的JAAS配置条目。
    3. 执行身份验证。
    4. 定义认证用户要执行的任务。
    5. 以经过身份验证的用户身份执行操作。
    6. 登出。

    Subject.doA将运行在中定义的代码MyAction作为经过身份验证的用户[第14-15行]。这有两个目的。首先，在MyAction需要身份信息进行服务验证的身份可以从主题中获取。本练习演示了这种用法。第二，如果MyAction如果访问任何受保护的资源/操作，则当前主题中的身份信息将用于做出相应的访问控制决策。本练习未涵盖第二个方面。

  • 确保%JAVA_HOME%/bin在PATH环境变量中。

  • 编译修改后的示例代码。完成一些设置后，您将在后续练习中运行此代码。到此结束。

摘要

本练习介绍了JAAS API的主要类： LoginContext和Subject 。您学习了如何使用LoginContext验证用户并在Subject中收集其身份信息。然后，您学习了如何使用主题以经过身份验证的用户身份执行操作。

练习2：配置JAAS进行Kerberos身份验证

本练习的目的

本练习的目的是学习如何配置JAAS应用程序以使用Kerberos进行身份验证。

本练习的Kerberos背景

Kerberos是RFC 4120中定义的用于受信任第三方身份验证的Internet标准协议。它在当今的大多数现代计算平台上都可用，包括Solaris，Windows和Linux。

Kerberos体系结构围绕着称为密钥分发中心或KDC的可信身份验证服务。 Kerberos环境中的用户和服务称为主体。每个主体与KDC共享一个秘密（例如密码）。主体通过向KDC证明它知道共享机密来对Kerberos进行身份验证。如果身份验证成功，则KDC会向主体颁发票证授予票证（TGT）。当委托人随后想要对网络上的服务（例如目录服务或文件服务）进行身份验证（从而充当服务的“客户端”）时，它将TGT交给KDC以获取服务票证与服务进行通信。服务凭单不仅指示客户端和服务主体的身份，还包含会话密钥，客户端和服务可使用该会话密钥来随后建立安全通信。为了对服务进行身份验证，客户端将服务票证发送到服务。当服务收到票证时，它将使用与KDC共享的机密对票证进行解码。

在此体系结构中，主体仅直接（一次）向KDC进行身份验证。它通过使用服务票证间接向所有其他服务进行身份验证。服务票证是KDC担保委托人身份的方式。主体通过仅执行一次显式身份验证来访问多个安全服务的能力称为单点登录。

本练习的JAAS背景

在JAAS中，对于客户端主体，“登录Kerberos”意味着获取TGT并将其放置在Subject中，以便可以将其用于客户端将访问的服务的身份验证。对于服务主体，“登录到Kerberos”表示获取服务解码进入的客户端身份验证请求所需的密钥。

遵循的步骤

  1. 检查jaas-krb5.conf配置文件。

    该文件包含两个条目，一个命名为client ，一个命名为server 。客户端条目指示LoginContext必须使用com.sun.security.auth.module。Krb5LoginModule 。服务器条目指示LoginContext必须使用相同的登录模块，并使用来自sample.keytab主体host / machineName的文件。

  2. 通过执行hostname命令确定计算机的主机名。

  3. 编辑此文件，然后更改服务器主体的条目以使用您的计算机名称。例如，如果您的计算机名称是j1hol-001，则配置文件中的这一行应如下所示：

    principal="host/j1hol-001"

  4. 通过键入以下命令来执行客户端身份验证：

    % java -Djava.security.auth.login.config=jaas-krb5.conf Jaas client

    系统将提示您输入密码。您应该看到以下输出。用安全的密码替换密码 。 

    Kerberos password for test: password
Authenticated principal: [test@J1LABS.EXAMPLE.COM]
Performing secure action...

  5. 通过键入以下命令来执行服务器身份验证：

    % java -Djava.security.auth.login.config=jaas-krb5.conf Jaas server

    您应该看到以下输出：

    Authenticated principal: [host/j1hol-001@J1LABS.EXAMPLE.COM] Performing secure action...
