第四部分：使用更强大的加密算法的安全通信

练习7：配置为在Kerberos环境中使用更强大的加密算法来保护通信

本练习的目的

本练习的目的是学习如何使用各种Kerberos加密算法来保护通信。Java GSS / Kerberos提供了广泛的加密算法，包括AES256，AES128、3DES，RC4-HMAC和DES。

注意：

默认情况下，禁用基于DES的加密类型。

以下是Java SE中Java GSS / Kerberos提供程序支持的所有加密类型的列表：

  • AES256-CTS

  • AES128-CTS

  • RC4-HMAC

  • DES3-CBC-SHA1

  • DES-CBC-MD5

  • DES-CBC-CRC

遵循的步骤

  1. 配置密钥分发中心（KDC）并更新Kerberos数据库。

    首先，您需要更新以使用支持必需的Kerberos加密类型的KDC，例如最新版本的Solaris或MIT发行版中的最新版本的Kerberos。如果在Windows平台上使用Active Directory，则最新版本还支持RC4-HMAC和AES加密类型。

    您需要更新Kerberos数据库以使用更强大的加密算法来生成新密钥。默认情况下，Solaris 10 KDC将为列出的所有上述加密类型生成密钥。现在，您可以创建一个密钥表，其中将包含上述所有加密类型的所有密钥。

    注意：

    如果要使用Windows 2000 KDC，则可以配置为使用Windows上可用的DES和RC4-HMAC加密类型。

  2. 编辑Kerberos配置文件（ krb5.conf ）。

    您将需要编辑Kerberos配置文件，以选择所需的加密类型。您需要在以下位置插入的必需参数libdefaults下面列出了Kerberos配置文件的“部分”。在本练习中，所有必需的条目已添加到该练习随附的示例Kerberos配置文件中，并且这些条目已被注释掉。要启用所需的加密类型，只需要取消注释必填项即可。

    • 要启用AES256-CTS加密类型，请添加以下内容：

      [libdefaults]
default_tkt_enctypes = aes256-cts
default_tgs_enctypes = aes256-cts
permitted_enctypes = aes256-cts

      注意：

      默认情况下，Solaris 10 11/06和更早版本不支持AES256。您将需要安装以下软件包：SUNWcry，SUNWcryr，SUNWcryptoint。

    • 要启用AES128-CTS加密类型，请添加以下内容：

      [libdefaults]
default_tkt_enctypes = aes128-cts
default_tgs_enctypes = aes128-cts
permitted_enctypes = aes128-cts

    • 要启用RC4-HMAC加密类型，请添加以下内容：

      [libdefaults]
default_tkt_enctypes = rc4-hmac
default_tgs_enctypes = rc4-hmac
permitted_enctypes = rc4-hmac

    • 要启用DES3-CBC-SHA1加密类型，请添加以下内容：

      [libdefaults]
default_tkt_enctypes = des3-cbc-sha1
default_tgs_enctypes = des3-cbc-sha1
permitted_enctypes = des3-cbc-sha1

    注意：

    • 如果未将上述参数添加到Kerberos配置文件中，则Solaris 10 11/06及更早版本将默认使用AES128编码类型。如果已安装可导出的加密软件包，则默认使用AES256编码类型。

    • 如下所述，从票证缓存中销毁票证缓存中的所有先前存在的Kerberos TGT：

      % kdestroy

  3. 启动一个新窗口，并使用更新后的服务器启动服务器krb5.conf如下：

    % java -Djava.security.auth.login.config=jaas-krb5.conf \
-Djava.security.krb5.conf=krb5.conf GSSServer

  4. 使用更新的运行客户端应用程序krb5.conf 。GSSClient类采用两个参数：服务名称和在其上运行服务的服务器的名称。例如，如果服务是在机器j1hol-001上运行的主机，请使用以下命令（在出现提示时提供安全密码）：

    % java -Djava.security.auth.login.config=jaas-krb5.conf \
-Djava.security.krb5.conf=krb5.conf \
GSSClient host j1hol-001

摘要

在本练习中，您学习了如何编写使用Java GSS API的客户端服务器应用程序，以使用更强大的Kerberos加密算法来安全地进行身份验证和通信。您可以启用Kerberos调试（ -Dsun.security.krb5.debug=true ），以获取有关所用Kerberos加密类型的信息。
