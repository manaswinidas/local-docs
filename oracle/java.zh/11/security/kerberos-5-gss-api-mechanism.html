Kerberos 5 GSS-API机制

本节描述并列出了有关Kerberos 5的Java通用安全服务（Java GSS）的安全功能。它还描述了Kerberos V5机制的对象标识符（OID），加密类型以及krb5.conf Java GSS支持的设置。

通用安全服务应用程序接口（GSS-API）机制由RFC 1964定义，并在Internet Standards流程下补充了RFC 4121 。

Kerberos V5机制的OID

根据RFC 1964第1节，针对Kerberos 5的Java通用安全服务（Java GSS）的OID定义为1.2.840.113554.1.2.2；另请参见Java安全标准算法名称中的 GSSAPI机制 。

Java GSS / Kerberos支持的加密类型

下表列出了Java GSS / Kerberos支持的加密类型的首选顺序。

表7-1 Java GSS / Kerberos支持的加密类型

名称 别名 输入号码
aes256-cts-hmac-sha1-96 aes256-sha1，aes256-cts 18岁
aes128-cts-hmac-sha1-96 aes128-sha1，aes128-cts 17
aes256-cts-hmac-sha384-192 aes256-sha2 20
aes128-cts-hmac-sha256-128 aes128-sha2 19
des3-cbc-sha1 des3-hmac-sha1 16
arcfour-hmac-md5 arcfour-hmac，rc4-hmac 23
des-cbc-crc 没有 1个
des-cbc-md5 没有 3

注意：

默认情况下启用AES-256加密类型。基于DES的加密类型，包括des-cbc-crc和dec-cbc-md5 ，默认情况下处于禁用状态。

用户可以出于以下目的限制加密的使用： krb5.conf ， 在里面[libdefaults]部分。

支持的krb5.conf设置

支持以下参数：

include FILENAME
includedir DIRNAME

[libdefaults]
default_realm
allow_weak_crypto
 
dns_lookup_kdc
dns_lookup_realm
dns_fallback
 
default_checksum
safe_checksum_type
ap_req_checksum_type
default_keytab_name
 
default_tkt_enctypes
permitted_enctypes
default_tgs_enctypes
 
no_addresses
noaddresses
 
renewable
proxiable
forwardable
 
kdc_default_options
clockskew
 
kdc_timeout
udp_preference_limit

max_retries
renew_lifetime
ticket_lifetime
 
[realms]
  REALM.NAME = {
      kdc
      kdc_timeout
      udp_preference_limit
      max_retries
  }
 
[capaths]
  A = {
    I = .
    B = I
  }
 
[domain_realm]
  domain=REALM

以下是默认值krb5.conf文件参数：

no_addresses = true
noaddresses = true
dns_lookup_kdc = true 
dns_lookup_realm = false
allow_weak_crypto = false
kdc_timeout = 30s
max_retries = 3
udp_preference_limit = 1465
clockskew = 300
renewable = false
proxiable = false
forwardable = false
