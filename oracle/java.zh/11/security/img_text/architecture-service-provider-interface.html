图例说明应用程序如何检索“ AES”密码实例/服务提供商接口体系结构的示例

该图由五个框组成：

  • 应用程序：此框包含以下伪代码：

    c:Cipher.getInstance("AES");

  • JCA / JCE：此框包含以下列表：

    • 签名
    • 信息摘要
    • 按键脚本生成器
    • 重点工厂
    • 算法参数
    • 密码
    • 关键协议
    • 密钥生成器
    • 秘密工厂
    • 苹果电脑

  • CSP，CSP2，CSP3：这些框代表加密服务提供商

  • 第五个方框代表CSP3。它包含以下标头和伪代码：

    • Provider.class

      "Cipher.AES" -> "com.foo.AESCipher"

    • com.foo。AESCipher.class

      package com.foo:
  class AESCipher extends CipherSPi {
    .
    .
    .
  }

箭头将框连接如下：

  • 从申请到JCA / JCE
  • 从JCA / JCE到CSP1，CSP2和CSP3（箭头分支）

虚线将CSP3连接到标题Provider.class和com.foo。AESCipher.class
