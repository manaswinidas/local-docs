JDK 1.1安全模型

该图说明了JDK 1.1安全模型，该模型引入了“签名小程序”的概念。它由两层组成。顶层是JVM。该层包含一个沙箱图标。底层包含一个图标，代表有价值的资源，例如文件。

箭头从本地代码图标指向JVM层。相信本地代码可以完全访问重要的系统资源（例如文件系统）。箭头从远程代码图标指向。该箭头指向两个箭头。一个箭头指向沙盒图标。另一个箭头指向JVM层。该分支被标记为“可信任” 。受信任的箭头表示正确数字签名的小程序，将其视为受信任的本地代码。指向沙盒图标的箭头表示不受信任的代码，例如未签名的applet。
