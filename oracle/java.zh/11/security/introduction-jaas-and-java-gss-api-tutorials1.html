JAAS和Java GSS-API教程简介

该页面链接到一系列教程，这些教程演示了JAAS（Java身份验证和授权服务）和Java GSS-API的使用的各个方面。

JAAS可以用于两个目的：

  • 用于authentication用户authentication ，以可靠，安全地确定当前正在执行Java代码的人员，以及
  • authorization用户以确保他们具有执行对安全敏感的操作所需的访问控制权（权限）。

Java GSS-API用于在通信应用程序之间securely exchanging messages 。Java GSS-API包含RFC 5653中定义的通用安全服务应用程序接口（GSS-API）的Java绑定。GSS-API为应用程序程序员提供了对各种基础安全机制（包括Kerberos）之上的安全服务的统一访问。

注意：JSSE是另一个可用于安全通信的API。有关两者之间的区别，请参阅何时使用Java GSS-API与JSSE 。

同时显示JAAS和Java GSS-API教程的原因是，通常在使用Java GSS-API进行安全通信之前执行JAAS身份验证。因此，JAAS和Java GSS-API是相关的，并且经常一起使用。但是，应用程序可以在不使用Java GSS-API的情况下使用JAAS，也可以在不使用JAAS的情况下使用Java GSS-API。此外，JAAS本身可以简单地用于身份验证或身份验证和授权。
