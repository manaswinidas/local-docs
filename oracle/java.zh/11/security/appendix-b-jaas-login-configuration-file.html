附录B：JAAS登录配置文件

JAAS身份验证以可插入方式执行，因此Java应用程序可以保持与基础身份验证技术无关。在运行时指定配置信息，例如所需的身份验证技术。配置信息的来源（例如，文件或数据库）取决于当前的javax.security.auth.login。配置实施。默认值Configuration实施， ConfigFile从登录配置文件中获取其配置信息。有关默认登录的详细信息Configuration JAAS随附的实现，请参见com.sun.security.auth.login.ConfigFile类。

登录配置文件的结构和内容

登录配置文件由一个或多个条目组成，每个条目指定应针对一种或多种特定应用程序使用哪种基础身份验证技术。每个条目的结构如下：

<name used by application to refer to this entry> { 
    <LoginModule> <flag> <LoginModule options>;
    <optional additional LoginModules, flags and options>;
};

因此，每个登录配置文件条目包含一个名称，后跟一个或多个特定于LoginModule条目，其中每个LoginModule特异性条目由分号和特异性的LoginModule条目的整组封端的被包围在括号。每个配置文件条目均以分号终止。

示例6-1 JAAS身份验证教程的登录配置文件

例如，用于“ JAAS认证教程”教程的登录配置文件仅包含一个条目，即

Sample {
   sample.module.SampleLoginModule required debug=true;
};

在这里，条目被命名为Sample这就是“ JAAS认证”教程应用程序（ SampleAcn.java ）用于引用该条目。该条目指定用于执行用户身份验证的LoginModule是SampleLoginModule在里面sample.module包和这个SampleLoginModule必须“成功”才能使认证成功。的SampleLoginModule仅当用户提供的名称和密码与预期的名称和密码相同时，该命令才会成功（ testUser和testPassword ， 分别）。

在登录配置文件中的条目名称是应用程序用来参考条目时，他们实例化一个LoginContext，如在名字JAAS认证教程 JAAS认证教程。该名称可以是应用程序开发人员希望使用的任何名称。这里，术语“应用程序”是指JAAS登录的任何代码。

指定的LoginModule （如下所述）用于控制身份验证过程。身份验证按照指定的确切顺序从列表中向下进行，如Configuration类中所述。

每个特定于LoginModule的条目的子部分如下：

  • LoginModule ：这指定实现所需身份验证技术的类。具体来说，该类必须是LoginModule类的子类，该类位于javax.security.auth.spi包。典型的LoginModule可能会提示您输入并验证用户名和密码，如SampleLoginModule （在里面sample.module包）用于这些教程。任何供应商都可以提供您可以使用的LoginModule实现。Oracle的JRE提供了一些实现。您可以在以下位置查看各种LoginModule的参考文档。 com.sun.security.auth包：

    • JndiLoginModule
    • KeyStoreLoginModule
    • Krb5LoginModule
    • NTLoginModule
    • Unix登录模块

  • flag ：标志值指示前面的LoginModule是否成功required ， requisite ， sufficient ， 要么optional 。如果只有一个特定于LoginModule的条目（如我们的教程中所述），则应“要求”该标志。这些选项在Configuration类中有更详细的描述。

  • LoginModule选项 ：如果指定的LoginModule实现具有可以设置的选项，则可以在此处指定任何所需的选项值。这是一个用空格分隔的值列表，这些值直接传递到基础LoginModule 。选项由LoginModule本身定义，并控制其中的行为。例如， LoginModule可以定义选项以支持调试/测试功能。

    在配置文件中指定选项的正确方法是使用名称/值对，例如debug=true ，其中的选项名称（在这种情况下， debug ）和值（在这种情况下， true ）应以等号分隔。

示例6-2登录配置文件演示必需，足够，必需和可选标志

以下是一个示例登录配置文件，该文件演示了required ， sufficient ， requisite和optional标志。见Configuration类，以获取有关这些标志的更多信息。 

Login1 {
       sample.SampleLoginModule required debug=true;
    };

    Login2 {
       sample.SampleLoginModule required;
       com.sun.security.auth.module.NTLoginModule sufficient;
       com.foo.SmartCard requisite debug=true;
       com.foo.Kerberos optional debug=true;
    };

应用程序Login1仅配置了一个LoginModule ， SampleLoginModule 。因此， 只有在以下情况下， Login1尝试对主题（用户或服务）进行身份验证的尝试才会成功。 SampleLoginModule成功。

在何处指定应使用哪个登录配置文件

可以使用以下两种方式之一指定要使用的配置文件：

  1. 在命令行上。

    您可以使用-Djava.security.auth.login.config解释程序命令行参数，用于指定应使用的登录配置文件。我们在所有教程中都使用这种方法。例如，我们运行SampleAcn JAAS Authentication Tutorial中的Java应用程序使用以下命令，该命令指定配置文件是sample_jaas.config当前目录中的文件：

    java -Djava.security.auth.login.config==sample_jaas.config sample.SampleAcn

    注意：

    如果您使用单个等号（ = ）与java.security.auth.login.config系统属性，然后由该系统属性和java.security文件被使用。

  2. 在Java安全属性文件中。

    指定登录配置文件位置的另一种方法是将其URL指示为a的值。 login.config.url.n安全属性文件中的属性。安全属性文件是java.security文件位于conf/security JDK的目录。

    这里， n表示从1开始的连续编号的整数。因此，如果需要，您可以通过指定一个文件的URL来指定多个登录配置文件。 login.config.url.1属性，第二个文件的URL login.config.url.2财产，等等。如果指定了多个登录配置文件（即n > 1），然后读取文件并将其串联为单个配置。

    这是一个示例，该示例将需要添加到安全属性文件中以指示sample_jaas.config本教程使用的登录配置文件。本示例假定文件位于C:\AcnTest Windows上的目录：

    login.config.url.1=file:C:/AcnTest/sample_jaas.config

    （请注意，无论用户运行哪种操作系统，URL都始终使用正斜杠。）
