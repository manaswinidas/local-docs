第五部分：使用SPNEGO Java GSS机制的安全身份验证

练习8：将Java通用安全服务（GSS）API与SPNEGO一起使用

Java GSS是一个可以支持多种安全机制的框架。需要一种在GSS-API下协商安全机制的方法。可通过SPNEGO获得。

SPNEGO在RFC 4178中由IETF标准化。 它是用于协商基础安全机制的伪安全机制。它为客户端和服务器提供了灵活地安全协商通用GSS安全机制的灵活性。

Microsoft大量使用SPNEGO。 SPNEGO可用于通过HTTP与Microsoft Server进行互操作，以通过协商协议支持基于HTTP的跨平台身份验证。

当前，当将Java GSS与Kerberos一起使用时，我们指定Kerberos OID如下：

Oid krb5Oid = new Oid("1.2.840.113554.1.2.2");

为了使用SPNEGO，您只需指定SPNEGO OID，如下所示：

Oid spnegoOid = new Oid("1.3.6.1.5.5.2");

然后，您可以在创建GSSCredential ， GSSContext等时使用SPNEGO OID。

本练习的目的

当前，Java GSS可用的唯一安全机制是Kerberos。本练习的目标是学习如何使用其他Java GSS机制（例如，简单和受保护的GSS-API协商机制（SPNEGO））来保护关联。

遵循的步骤

  1. 阅读GssSpNegoClient.java码。

  2. 编译示例代码：

    % javac GssSpNegoClient.java

  3. 阅读GssSpNegoServer.java码。

  4. 编译示例代码：

    % javac GssSpNegoServer.java

  5. 启动一个新窗口并启动服务器：

    % java -Djava.security.auth.login.config=jaas-krb5.conf GssSpNegoServer

  6. 运行客户端应用程序。 GssSpNegoClient接受两个参数：服务名称和运行该服务的服务器的名称。例如，如果服务是在机器j1hol-001上运行的主机，请使用以下命令（在出现提示时提供安全密码）：

    % java -Djava.security.auth.login.config=jaas-krb5.conf \
GssSpNegoClient host j1hol-001

摘要

在本练习中，您学习了如何编写将Java GSS API与SPNEGO一起使用的客户端服务器应用程序，以协商诸如Kerberos之类的基础安全机制，并使用Kerberos作为基础认证系统进行安全通信。

注意：

Microsoft已经实现了SPNEGO协议的某些变体，因此为了与Microsoft互操作，我们通过新的系统属性添加了单独的模式sun.security.spnego.msinterop 。默认情况下，此属性启用为true。要禁用它，您需要将该属性显式设置为false。要启用SPNEGO调试，可以设置系统属性sun.security.spnego.debug=true 。
