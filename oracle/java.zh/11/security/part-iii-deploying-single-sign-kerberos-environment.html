第三部分：在Kerberos环境中部署单点登录

练习6：部署单点登录

本练习的目的

本练习的目的是学习如何配置使用Kerberos进行身份验证的JAAS应用程序以实现单点登录。单一登录意味着用户只需要向系统或一组服务进行一次身份验证。初始身份验证之后，用户可以使用与初始身份验证相同的身份访问系统中的其他服务。

单一登录可用于描述不同类型的身份验证。有基于HTTP的网络单点登录协议。网络服务有基于Kerberos的单点登录。在此特定练习中，我们通过显示如何从底层本机操作系统中导入已经获取的Kerberos凭据，来说明如何在基于Kerberos的系统中实现单点登录。

本练习的背景和资源

请参阅在Java中使用Kerberos进行单点登录 。此外，请参阅练习2：为Kerberos身份验证配置JAAS和练习4：使用Java SASL API中提供的信息，以获取有关Kerberos和Java GSS的背景信息。

遵循的步骤

  1. 编辑jaas-krb5.conf配置文件。

    该文件包含两个条目：一个命名的客户端和一个命名的服务器 。添加行useTicketCache=true到客户端条目。

  2. 执行Kerberos登录到本机操作系统。要登录到Kerberos，请使用kinit命令，如下所示：

    % kinit test

    提供安全密码。

  3. 在练习1至5中运行客户端和服务器程序，您会注意到客户端应用程序不再要求您输入密码。
