5外部代码

外部类是通过类路径从JShell会话访问的。通过模块路径，其他模块设置和模块导出设置可以访问外部模块。

话题

  • 设置课程路径

  • 设置模块选项

设置课程路径

您可以使用可通过JShell会话中的类路径访问的外部代码。

如下例所示，在命令行上设置类路径：

 % jshell --class-path myOwnClassPath

将类路径指向具有您要访问的软件包的目录或JAR文件。该代码必须编译到类文件中。无法从JShell访问默认程序包中的代码（也称为未命名程序包）。设置类路径后，可以将这些包导入到会话中：

jshell> import my.cool.code.*

您也可以使用/env命令设置类路径，如以下示例所示：

jshell> /env --class-path myOwnClassPath
|  Setting new options and restoring state.

的/env命令将重置执行状态，并使用该命令输入的新类路径设置或其他环境设置重新加载当前的所有代码段。

设置模块选项

JShell支持模块。可以设置模块路径，指定要解析的其他模块，并提供模块导出。

可以在选项中提供模块选项/env命令或命令行，如以下示例所示：

 % jshell --module-path myOwnModulePath  --add-modules my.module

要查看当前环境设置，请使用/env没有选择。以下示例包括在设置类路径中设置的类路径信息 ：

 jshell> /env
 |     --add-modules my.module
 |     --module-path myOwnModulePath
 |     --class-path myOwnClassPath

有关选项的详细信息，请输入以下命令：

jshell> /help context
