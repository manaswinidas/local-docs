4编辑

JShell支持在jshell提示并在您选择的外部编辑器中进行编辑。

使用Shell编辑，您可以在输入代码段和命令时对其进行编辑，以及检索和更改先前输入的代码段和命令。外部编辑器提供了另一种编辑和创建代码段的方式，使用多行代码段时更容易。

话题

外壳编辑

通过在命令提示符下编辑输入，可以轻松地更正输入以及检索和修改先前输入的命令和代码片段。

JShell中的Shell编辑基于JLine2构建，其功能类似于BSD editline和GNU readline在Emacs模式下。请参阅JLine2用户信息和GNU Readline文档 。

输入线导航

支持Shell编辑，用于编辑当前行或通过JShell的先前会话访问历史记录。

为了导航输入行，在组合键中使用了Ctrl键和Meta键。如果您的键盘没有Meta键，则通常将Alt键映射为提供Meta键功能。

对于一行中的基本导航，请使用向右和向左箭头键，或者向后使用Ctrl + B，向前使用Ctrl + F。要在历史记录中的各行之间导航，请使用向上和向下箭头键。按下向上箭头一次，将当前行替换为先前的命令或代码段行。再按一次向上箭头，将转到上一行。历史记录包含命令和摘要行。如果代码段包含多行，则上下箭头会浏览代码段的每一行。

历史导航

片段和命令的历史记录在JShell会话之间进行维护。通过此历史记录，您可以访问在当前和先前会话中输入的项目。

要重新输入或编辑先前的输入，请使用向上，向下，向左和向右箭头浏览历史记录。输入的文本将插入到光标处。Delete键用于删除文本。按Enter键重新输入历史记录行（已修改或未修改）。

向上和向下箭头键一次在历史记录中前后移动一次，例如：

jshell> class C {
   ...>    int x;
   ...> }
|  created class                                                                                           
jshell> /list

   1 : class C 
         int x;
       }

jshell> <up arrow>

向上箭头键显示以下行：

jshell> /list

再按一次向上箭头将显示类定义的最后一行：

jshell> }

按向下箭头返回到/list命令。按Enter执行它：

jshell> /list

   1 : class C {
         int x;
       }

Ctrl +向上箭头按代码片段向上移动。对于单行代码段，Ctrl +向上箭头的行为与向上箭头相同。适用于多行代码段，例如class C ，Ctrl +向上箭头会跳过其他行，并转到代码段的顶部。

只需输入文本即可在当前光标位置添加文本。有关在行内移动光标的键，请参见输入行导航 。

搜索和更多

搜索历史记录是JShell的一项功能，它使查找所需的行变得更加容易，而无需一次浏览一行历史记录。

要开始搜索，请按Ctrl-R。在提示符下，输入搜索字符串。搜索从您最近的条目开始向后进行，并且包括以前的JShell会话。以下示例显示了按Ctrl-R后出现的提示：

jshell> <Ctrl+R>
((reverse-i-search)`':

根据“ 历史记录导航”中的示例，输入class更改显示以显示带有文字的最新行class ：

(reverse-i-search)`class': class C {

搜索是渐进式的，因此仅以第一个字符c检索该行。您可以通过反复按Ctrl + R继续在历史记录中进行更早的搜索。Ctrl + S将搜索向前移动到当前位置。

您可以通过输入以下内容来定义键盘宏Ctrl-x ( ，然后输入您的文字，最后输入Ctrl-x ) 。要使用您的宏，请输入Ctrl+x e 。

外部编辑器

在命令提示符下进行编辑的另一种方法是使用外部编辑器。该编辑器可用于编辑和创建代码片段，对于多行代码片段特别有用。您可以配置JShell以使用您选择的编辑器。

要在编辑器中一次编辑所有现有片段，请使用/edit没有选择。要在编辑器中编辑特定代码段，请使用/edit带有代码段名称或ID的命令。使用/list命令以获取代码段ID。以下示例打开一个编辑器来编辑名为的代码段volume ， 在前向引用中定义：

jshell> /edit volume

您也可以在编辑器中输入新的代码段。当您保存在编辑器中时，所有更改或新增的代码段都将输入到JShell会话中。来自片段的反馈显示在JShell窗口中，但是，没有显示JShell提示。在关闭编辑器之前，您无法在JShell窗口中输入命令或代码段。

如果您未指定编辑器，那么将依次检查以下环境变量： JSHELLEDITOR ， VISUAL和EDITOR 。如果未设置任何一个，则使用一个简单的默认编辑器。要设置JShell以打开您选择的编辑器，请使用/set editor命令。的论点/set editor command是启动要使用的外部编辑器所需的命令。以下示例集kwrite作为编辑器，并使用所有现有的片段打开编辑器：

jshell> /set editor kwrite
|  Editor set to: kwrite

jshell> /edit

定义x在外部编辑器窗口中并保存更改，将在JShell窗口中生成以下输出：

|  created variable x of type int with initial value 6

关闭外部编辑器将还原JShell提示符。
