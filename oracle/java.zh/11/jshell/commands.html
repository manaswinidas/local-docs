命令介绍

JShell命令控制环境并在会话中显示信息。

命令与代码段之间的区别在于前导斜杠（/）。有关当前变量，方法和类型的信息，请使用/vars ， /methods和/types命令。有关输入的摘要的列表，请使用/list命令。以下示例基于“ 尝试片段”中的示例显示了这些命令：

jshell> /vars
|    int x = 45
|    int $3 = 4
|    String $5 = "OceanOcean"

jshell> /methods
|    twice (String)String

jshell> /list

   1 : System.out.println("Hi");
   2 : int x = 45;
   3 : 2 + 2
   4 : String twice(String s) {
         return s + s;
       }
   5 : twice("Ocean")

注意，将显示变量的类型和值以及方法的类型签名。

JShell具有默认的启动脚本，该脚本在JShell启动之前会自动静默执行，因此您可以快速开始工作。除非您使用启动脚本请求，否则不会列出启动脚本中的条目。 /list -start要么/list -all命令：

jshell> /list -all

  s1 : import java.util.*;
  s2 : import java.io.*;
  s3 : import java.math.*;
  s4 : import java.net.*;
  s5 : import java.util.concurrent.*;
  s6 : import java.util.prefs.*;
  s7 : import java.util.regex.*;
   1 : System.out.println("Hi");
   2 : int x = 45;
   3 : 2 + 2
   4 : String twice(String s) {
         return s + s;
       }
   5 : twice("Ocean")

默认的启动脚本包含几个常见的导入。您可以使用/set start命令。有关此命令的信息，请输入/help /set start 。的/save -start该命令将当前启动脚本保存为您自己的启动脚本的起点。

其他重要命令包括/exit离开JShell， /save保存您的摘录，以及/open从文件中输入摘要。

输入/help有关JShell命令的列表。

命令的制表符完成

与代码段完成类似，当您输入命令和命令选项时，请使用Tab键自动完成命令或选项。如果无法根据输入的内容确定完成，则提供可能的选择。

以下示例显示了在命令的前斜杠（/）之后按Tab时的反馈：

jshell> /<Tab>
/!          /?          /drop       /edit       /env        /exit       /help
/history    /imports    /list       /methods    /open       /reload     /reset      
/save       /set        /types      /vars       

<press tab again to see synopsis>

jshell> /

独特的完成是就地完成的。例如，输入/l然后按Tab键，该行将替换为/list ：

jshell> /l<Tab>

jshell> /list

选项卡补全也适用于命令选项。以下示例说明了如何使用Tab键来显示/list命令：

jshell> /list -<Tab>
-all       -history   -start     

<press tab again to see synopsis>

jshell> /list -

请注意有关再次按下Tab键以显示命令摘要的消息，这是该命令的简短说明。再次按Tab键以显示帮助文档。以下示例显示了第二次和第三次按下Tab的结果：

jshell> /list -<Tab>
list the source you have typed

<press tab again to see full documentation>

jshell> /list -<Tab>
Show the source of snippets, prefaced with the snippet id.

/list
    List the currently active snippets of code that you typed or read with /open

/list -start
    List the automatically evaluated start-up snippets

/list -all
    List all snippets including failed, overwritten, dropped, and start-up

/list <name>
    List snippets with the specified name (preference for active snippets)

/list <id>
    List the snippet with the specified snippet id

jshell> /list -

唯一参数的完成就位。例如，输入/list -a ， -all选项会自动显示：

jshell> /list -a<Tab>

jshell> /list -all

片段名称也可以使用Tab来完成。例如，如果您定义了volume JShell会话中较早的方法名称，然后在开始输入方法名称后按Tab键将显示完整的方法名称：

jshell> /ed v<Tab>

jshell> /ed volume

在命令的文件参数位置中使用Tab键可显示可用文件：

jshell> /open <Tab>
myfile1      myfile2    definitions.jsh

<press tab again to see synopsis>

jshell> /open

唯一文件名的填写已完成：

jshell> /open d<Tab> 

jshell> /open definitions.jsh

命令缩写

减少使用缩写的键入量。命令/set子命令，命令参数和命令选项都可以缩写，只要缩写是唯一的即可。

唯一以开头的命令/l是/list ，唯一/list开头的选项-a是-all 。因此，您可以使用以下缩写输入/list -all命令：

jshell> /l -a

同样，唯一以开头的命令/se是/set ， 唯一的/set以开头的子命令fe是feedback ，以及唯一以开头的反馈模式v是verbose ，假设没有以开头的自定义反馈模式v存在。因此，您可以使用以下缩写将反馈模式设置为详细：

jshell> /se fe v

注意/s还不够缩写，因为/save和/set两者都以相同的字母开头。如有疑问，可以使用制表符补全来查看选项。
